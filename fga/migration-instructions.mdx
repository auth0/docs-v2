# Notice of Upcoming Changes to the Fine-Grained Authorization Token Issuer 
<Info>
**Note**

As of April 30th, 2025, the FGA API token issuer URL has been changed to [auth.fga.dev](https://auth.fga.dev). If you have any questions, please reach out to [FGA support](https://docs.fga.dev/support).
</Info>
At Auth0/Okta, we are continuously improving the reliability, security and performance of the Fine-Grained Authorization (FGA) service to ensure the best experience for our customers. As part of these enhancements, we are migrating to a new token issuer, which will further strengthen the security and resilience of your applications connected to the FGA store.

## What does this mean for you?
To continue using your FGA store without any interruptions, there are a few key updates that need to be made to your environment. Specifically, you will need to update the API Token Issuer (env: `FGA_API_TOKEN_ISSUER`) value in your configuration from the current `fga.us.auth0.com` to the new `auth.fga.dev`.

After **April 30, 2025**, the current token issuer (fga.us.auth0.com) will no longer be supported. To avoid any disruption to your services, we strongly recommend updating your settings before this deadline.

## Steps to Update Your Environment
If you have set up your SDK client as described in our [How To Setup The SDK Client](https://docs.fga.dev/integration/setup-sdk-client) docs, you must change the `FGA_API_TOKEN_ISSUER` environment variable to `auth.fga.dev`:

   ![Image showing the model explorer](/img/migration-instruction.png)

If you are not using an environment variable [as described in our docs](https://docs.fga.dev/integration/setup-sdk-client), please see below for the changes you need to make for each SDK.

<Tabs>
<Tab title="Node.js">

Update the value of `apiTokenIssuer` to reflect the new token issuer value when the FGA client is created:

```javascript
const openFga = new OpenFgaClient({
    // ..
    credentials: {
        method: CredentialsMethod.ClientCredentials,
        config: {
          // ..
          apiTokenIssuer: "auth.fga.dev"
        },
    }
});
```

</Tab>
<Tab title="Go">
Update the value of `ClientCredentialsApiTokenIssuer` to reflect the new token issuer value when the FGA client is created:
```go
fgaClient, err := NewSdkClient(&ClientConfiguration{
        // ..
        Credentials: &credentials.Credentials{
            Method: credentials.CredentialsMethodClientCredentials,
            Config: &credentials.Config{
                // ..
                ClientCredentialsApiTokenIssuer: "auth.fga.dev",
            },
        },
    })
```

</Tab>
<Tab title=".NET">

    Update the value of `ApiTokenIssuer` to reflect the new token issuer value when the FGA client is created:

```csharp
var configuration = new ClientConfiguration() {
    // ..
    Credentials = new Credentials() {
        Method = CredentialsMethod.ClientCredentials,
        Config = new CredentialsConfig() {
            // ..
            ApiTokenIssuer = "auth.fga.dev"
        }
    }
};
```

</Tab>
<Tab title="Python">
    Update the value of `api_issuer` to reflect the new token issuer value when the FGA client is created:

```python
credentials = Credentials(
    method='client_credentials',
    configuration=CredentialConfiguration(
        api_issuer= 'auth.fga.dev'
        # ..
    )
)
```

</Tab>
<Tab title="Java">
Update the value provided to `.apiTokenIssuer` to reflect the new token issuer when the FGA client is created:

```java
var config = new ClientConfiguration()
  // ..
  .credentials(new Credentials(
      new ClientCredentials()
        // ..
        .apiTokenIssuer("auth.fga.dev")
));
```

</Tab>
</Tabs>
