# Managing User Access

In this guide you will learn how to grant a [user](/intro/fga-concepts#what-is-a-user) access to a particular [object](/intro/fga-concepts#what-is-an-object).

<Card title="When to use" appearance="filled">

Granting access with a _[relationship tuple](/intro/fga-concepts#what-is-a-relationship-tuple)_ is a core part of Auth0 FGA. Without any relationship tuples, any _[check](/intro/fga-concepts#what-is-a-check-request)_ will fail. You should use:

- _authorization model_ to represent what **relation**s are possible between the users and objects in your system
- _relationship tuples_ to represent the facts about the relationships between users and objects in your system.

</Card>

## Before you start

In order to understand this guide correctly you must be familiar with some FGA concepts and know how to develop the things that we will list below.

### Direct access

You need to know how to create an authorization model and create a relationship tuple to grant a user access to an object. [Learn more →](../modeling/direct-access.mdx)

### Auth0 FGA concepts

- A [Type](/intro/fga-concepts#what-is-a-type): a class of objects that have similar characteristics
- A [User](/intro/fga-concepts#what-is-a-user): an entity in the system that can be related to an object
- A [Relation](/intro/fga-concepts#what-is-a-relation): is a string defined in the type definition of an authorization model that defines the possibility of a relationship between an object of the same type as the type definition and a user in the system
- An [Object](/intro/fga-concepts#what-is-an-object): represents an entity in the system. Users' relationships to it can be define through relationship tuples and the authorization model
- A [Relationship Tuple](/intro/fga-concepts#what-is-a-relationship-tuple): a grouping consisting of a user, a relation and an object stored in Auth0 FGA

</details>
