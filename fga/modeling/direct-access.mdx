# Modeling Basics

This article describes how to grant a [user](/intro/fga-concepts#what-is-a-user) access to an [object](/intro/fga-concepts#what-is-an-object) in Auth0 FGA.

<Card title="When to use" appearance="filled">

Granting access with [relationship tuples](/intro/fga-concepts#what-is-a-relationship-tuple) is a core part of Auth0 FGA. Without relationship tuples, any [checks](/intro/fga-concepts#what-is-a-check-request)_ will fail. You should use:

- **authorization model** to represent what relations are possible between the users and objects in the system
- **relationship tuples** to represent the facts about the relationships between users and objects in your system.

</Card>

## Before you start

Familiarize yourself with FGA concepts to understand how to develop a relationship tuple and authorization model.

In addition, you will need to know the following:

### Auth0 FGA Concepts

- A [Type](/intro/fga-concepts#what-is-a-type): a class of objects that have similar characteristics
- A [User](/intro/fga-concepts#what-is-a-user): an entity in the system that can be related to an object
- A [Relation](/intro/fga-concepts#what-is-a-relation): a string defined in the type definition of an authorization model that defines the possibility of a relationship between an object of the same type as the type definition and a user in the system
- An [Object](/intro/fga-concepts#what-is-an-object): represents an entity in the system. Users' relationships to it can be define through relationship tuples and the authorization model
- A [Relationship Tuple](/intro/fga-concepts#what-is-a-relationship-tuple): a grouping consisting of a user, a relation and an object stored in Auth0 FGA

</details>

## Step By Step

For an application to understand that **user x** has access to **document y**, it must provide Auth0 Fine Grained Authorization that information with [relationship tuples](/intro/fga-concepts#what-is-a-relationship-tuple).
Each relationship tuple has three basic parameters: a **[user](/intro/fga-concepts#what-is-a-user)**, a **[relation](/intro/fga-concepts#what-is-a-relation)** and an **[object](/intro/fga-concepts#what-is-an-object)**.

### 02. Check That The Relationship Exists

Once you add that relationship tuple to Auth0 FGA, you can [check](/intro/fga-concepts#what-is-a-check-request) if the relationship is valid by asking if bob is an editor of document:meeting_notes.doc:

Checking whether `bob` is an `viewer` of `document:meeting_notes.doc` returns **false** because that relationship tuple does not exist in Auth0 FGA yet.

<Warning>
When creating relationship tuples for Auth0 Fine Grained Authorization, use unique ids for each object and user within your application domain. We're using first names and simple ids to as an easy-to-follow example.
</Warning>
