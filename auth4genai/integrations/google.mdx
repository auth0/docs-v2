The Google Integration allows your GenAI application to authenticate users with their Google accounts and access Google services on their behalf through Token Vault.

## Overview

The Google Integration is one of the most popular social integrations, providing access to a comprehensive suite of Google services. This connection enables users to log in using their personal Google accounts and grants your AI agents secure access to their Google data.

## Configuration

To configure the Google Integration for your GenAI application:

<Steps>
  <Step title="Create a Google Cloud Project">
    1. Go to the [Google Cloud Console](https://console.cloud.google.com/)
    2. Create a new project or select an existing one
    3. Enable the APIs you want to use (Gmail, Calendar, Drive, etc.)
  </Step>
  <Step title="Configure OAuth 2.0">
    1. Go to **APIs & Services** > **Credentials**
    2. Click **Create Credentials** > **OAuth 2.0 Client IDs**
    3. Configure the authorized redirect URIs
    4. Note your **Client ID** and **Client Secret**
  </Step>
  <Step title="Set up Auth0 Integration">
    1. In Auth0 Dashboard, go to **Authentication** > **Social**
    2. Select **Google** and configure with your credentials
    3. Set the required scopes for your application
    4. Enable **Token Vault** in the Advanced settings
  </Step>
  <Step title="Enable for your Application">
    Ensure the Google Integration is enabled for your Auth0 application under **Applications** > **[Your App]** > **Connections**.
  </Step>
</Steps>

## Required Scopes

Configure these scopes based on the Google services your AI agents need to access:

| Service | Scope | Description |
|---------|-------|-------------|
| Profile | `openid profile email` | Basic user profile information |
| Gmail | `https://www.googleapis.com/auth/gmail.modify` | Read, send, and manage emails |
| Calendar | `https://www.googleapis.com/auth/calendar` | Manage calendar events |
| Drive | `https://www.googleapis.com/auth/drive` | Access and manage Drive files |
| Contacts | `https://www.googleapis.com/auth/contacts` | Read and manage contacts |
| Docs | `https://www.googleapis.com/auth/documents` | Create and edit Google Docs |
| Sheets | `https://www.googleapis.com/auth/spreadsheets` | Read and write spreadsheet data |

<Note>
Use the most restrictive scopes possible for your use case. For example, use `gmail.readonly` if you only need to read emails.
</Note>

## Example Use Cases

### Email Management
Your AI agent can help users manage their inbox by reading emails, drafting responses, and sending messages.

```javascript
// Example: Check for unread emails
const response = await fetch('/api/gmail/messages', {
  headers: {
    'Authorization': `Bearer ${userToken}`
  }
});
```

### Calendar Scheduling
AI agents can check availability and schedule meetings automatically.

```javascript
// Example: Find available time slots
const availability = await fetch('/api/calendar/freebusy', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${userToken}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    timeMin: '2024-01-15T00:00:00Z',
    timeMax: '2024-01-15T23:59:59Z'
  })
});
```

### Document Collaboration
Access and modify Google Docs and Sheets for collaborative AI workflows.

## Best Practices

<Warning>
Always request the minimum scopes necessary for your application's functionality.
</Warning>

1. **Scope Management**: Only request scopes that your application actively uses
2. **Error Handling**: Implement proper error handling for API rate limits and authentication failures
3. **User Consent**: Clearly explain to users what data your AI agent will access
4. **Token Refresh**: Token Vault automatically handles token refresh, but monitor for any authentication issues
