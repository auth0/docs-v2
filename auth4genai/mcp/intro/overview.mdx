The [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) is an open standard for connecting AI applications to external data sources, tools, and workflows through a consistent interface. It lets agents use your SaaS apps and internal APIs, run organization-specific workflows, and reuse integrations instead of building one-offs.

**Auth for MCP** implements the OAuth 2.1 and OpenID Connect authorization requirements of the MCP specification. It handles client authentication, discovery, registration, and token management so your MCP servers can validate every incoming request and enforce granular access controls.

### Secure MCP Client Access

MCP servers require valid access tokens from all clients. Auth0 issues these tokens after validating the user's identity through social login, enterprise IdP, or custom credentials. The token includes the user's identity and permissions, which your MCP server can check on every request.

For enterprise deployments, Auth0 connects to your organization's identity provider (Okta, Entra ID, Ping, Google Workspace) via Single Sign-On. This means your employees authenticate with their corporate credentials, and every MCP interaction is auditable under your organization's access policies.

### Client Discovery and Registration

MCP clients discover your authorization server and register themselves automatically using standard OAuth 2.1 endpoints. Auth0 provides the required discovery document and handles Dynamic Client Registration (DCR), so clients can register without manual dashboard configuration.

### Call your APIs on user's behalf

Your MCP server often needs to call your own internal APIs on behalf of the connected user. Auth0's token exchange feature lets your server exchange the client's token for a new token scoped to your internal API. This maintains the user's identity through the entire flow while keeping your API tokens short-lived and scoped appropriately.

### Call third-party APIs on user's behalf

Your MCP server may need to connect to third-party services like Google, Microsoft, Jira, or Notion on behalf of your users. Auth0's Token Vault stores and manages third-party API tokens securely, handling rotation and revocation automatically. This eliminates the need to manage credentials directly in your application code.
