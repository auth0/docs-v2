Vous pouvez surveiller Auth0 en tant qu'application Web standard à l'aide de System Center Operations Manager (SCOM) ou de tout outil prenant en charge les transactions synthétiques. Nous vous recommandons d'ajouter une transaction de connexion synthétique qui inclut les extensions sur lesquelles vos applications s'appuient (telles que les règles qui exécutent un code personnalisé pour l'intégration avec les autres services de votre société).

## Configurer SCOM

1. Ajoutez une nouvelle instance SCOM à l’aide de l’ **Assistant d’ajout de surveillance** :

<table class="table"><thead>
<tr>
<th>Champ</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>Nom</td>
<td>Nom de description pour l’instance SCOM.</td>
</tr>
<tr>
<td>Description</td>
<td>Description de ce que surveille cette instance SCOM.</td>
</tr>
<tr>
<td>Sélectionner le pack de gestion de destination</td>
<td>Pack de gestion par défaut</td>
</tr>
</tbody>
</table>

   Une fois terminé, cliquez sur **Suivant** pour continuer.
2. Cliquez sur **Ajouter** pour saisir les URL que doit surveiller SCOM. Une fois terminé, cliquez sur **Suivant** pour continuer.
3. Cliquez sur **Ajouter** pour définir un emplacement à partir duquel effectuer la surveillance.
4. Dans la boîte de dialogue, cherchez **Emplacement interne - Agent**. Sélectionnez l’adresse appropriée, puis cliquez sur **Ajouter**. Cliquez ensuite sur **OK** une fois l’emplacement sélectionné. Une fois terminé, cliquez sur **Suivant** pour continuer.
5. Définissez la fréquence à laquelle SCOM doit collecter les données de chaque point de terminaison :

   <table class="table"><thead>
<tr>
<th>Données</th>
<th>Fréquence</th>
</tr>
</thead>
<tbody>
<tr>
<td>Fréquence de test</td>
<td>60 secondes</td>
</tr>
<tr>
<td>Intervalle de collecte de données de performances</td>
<td>60 secondes</td>
</tr>
<tr>
<td>Délai d’expiration test</td>
<td>30 secondes</td>
</tr>
<tr>
<td>Code d’état HTTP</td>
<td>Supérieur ou égal à 400</td>
</tr>
</tbody>
</table>
6. Une fois terminé, cliquez sur **Suivant** pour continuer.

## Exécuter les tests SCOM

1. Cliquez sur **Exécuter un test** pour tester chaque point de terminaison et vous assurer que les paramètres de connexion sont corrects.
2. Une fois que vous avez configuré votre instance SCOM, vous pouvez voir son activité dans l’onglet **Suivi**.

## Examiner le résultat des tests

Cliquez sur **Statut des applications Web** pour afficher les informations récupérées par SCOM.
