L’outil Deploy CLI peut être utilisé comme un utilitaire de ligne de commande autonome. Il s’agit d’un moyen simple de gérer la configuration de votre locataire Auth0 dans les flux de production CI/CD.

Pour une liste complète des paramètres de configuration, veuillez lire [Configurer l’outil DeployCLI](/docs/fr-ca/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

## commande « export »

Récupérer les configurations du locataire Auth0 sur la machine locale.

### --output_folder, -o

Chemin d’accès. Spécifie le répertoire cible dans lequel les fichiers de configuration doivent être écrits.

### --config_file, -c

Chemin d’accès. Indique le fichier de configuration défini par l’utilisateur (`config.json`). Pour en savoir plus, consultez [Configurer l’outil Deploy CLI](/docs/fr-ca/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

### --format, -f

Options : `yaml` ou `directory`. Détermine le format des fichiers de configuration des ressources exportés. Pour en savoir plus, lisez [Formats de configuration des ressources disponibles](/docs/fr-ca/deploy-monitor/deploy-cli-tool/available-resource-configuration-formats).

### --export_ids, -e

Valeur booléenne. Lorsque cette option est activée, elle exporte les champs d’identifiant de chaque ressource. Valeur par défaut : `false`.

### --env

Valeur booléenne. Indique si l’outil doit ingérer des variables d’environnement ou non. Valeur par défaut : `true`.

### --debug

Valeur booléenne. Active une journalisation plus détaillée des erreurs; utile pour le dépannage. Valeur par défaut : `false`.

### --proxy_url, -p

URL pour les demandes de proxy. Ne définissez cette option que si vous êtes derrière un proxy.

### Exemples

#### Récupère la configuration du locataire Auth0 au format YAML

`a0deploy export -c=config.json --format=yaml --output_folder=local`

#### Récupère la configuration du locataire Auth0 au format répertoire (JSON)

`a0deploy export -c=config.json --format=directory --output_folder=local`

#### Récupère les configurations du locataire Auth0 avec les identifiants de tous les actifs

`a0deploy export -c=config.json --format=yaml --output_folder=local --export_ids=true`

## commande « import »

Applique les configurations de la machine locale au locataire Auth0.

### --input_file, -i

Chemin d’accès. Spécifie l’emplacement des fichiers de configuration des ressources. Pour les formats YAML, cela pointera vers le fichier `tenant.yaml`; pour les formats de répertoire, cela pointera vers le répertoire de configuration des ressources.

### --config_file, -c

Chemin d’accès. Indique le fichier de configuration défini par l’utilisateur (`config.json`). Pour en savoir plus, consultez [Configurer l’outil Deploy CLI](/docs/fr-ca/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

### --env

Valeur booléenne. Indique si l’outil doit ingérer des variables d’environnement ou non. Valeur par défaut : `true`.

### --proxy_url, -p

URL pour les demandes de proxy. Ne définissez cette option que si vous êtes derrière un proxy.

### --debug

Valeur booléenne. Active une journalisation plus détaillée des erreurs; utile pour le dépannage. Valeur par défaut : `false`.

### Exemples

#### Déploiement de la configuration pour les formats YAML

`a0deploy import -c=config.json --input_file=local/tenant.yaml`

#### Déploiement de la configuration pour le format répertoire

`a0deploy import -c=config.json --input_file=local`

#### Déploiement de la configuration avec les variables d’environnement ignorées

`a0deploy import -c=config.json --input_file=local/tenant.yaml --env=false`
