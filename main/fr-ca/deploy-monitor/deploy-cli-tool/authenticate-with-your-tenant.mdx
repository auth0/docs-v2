Il existe trois méthodes d’authentification de l’outil Auth0 Deploy CLI avec votre locataire.

## Identifiants clients

L’authentification par [Client credentials (Identifiants clients)](/docs/fr-ca/get-started/authentication-and-authorization-flow/client-credentials-flow) exige que vous fournissiez directement à l’application une paire d’ID client et de secret client. Cette option est simple et permet la configuration la plus rapide de l’outil.

Pour configurer l’authentification des identifiants clients, définissez les champs suivants dans votre [configuration file (fichier de configuration)](/docs/fr-ca/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli) :

<table class="table"><thead>
<tr>
<th><strong>Champ</strong></th>
<th><strong>Valeur</strong></th>
</tr>
</thead>
<tbody>
<tr>
<td>`AUTH0_CLIENT_ID`</td>
<td>ID Client de votre application Auth0.</td>
</tr>
<tr>
<td>`AUTH0_CLIENT_SECRET`</td>
<td>Secret client de votre application Auth0.</td>
</tr>
</tbody>
</table>

## Clé privée JWT

La [Clé privée JWT](/docs/fr-ca/get-started/applications/configure-private-key-jwt) exige la configuration d’une clé privée pour l’application et d’une clé publique pour le locataire distant. Cette option est idéale si vous ne souhaitez pas stocker les identifiants à distance sur Auth0.

Pour configurer l’authentification par clé privée <Tooltip href="/docs/fr-ca/glossary?term=json-web-token" tip="Jeton Web JSON (JWT)
Format standard de jeton d’ID (et souvent de jeton d’accès) utilisé pour représenter en toute sécurité des demandes entre deux parties." cta="Voir le glossaire">JWT</Tooltip>, définissez les champs suivants dans votre [fichier de configuration](/docs/fr-ca/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli) :

<table class="table"><thead>
<tr>
<th><strong>Champ</strong></th>
<th><strong>Valeur</strong></th>
</tr>
</thead>
<tbody>
<tr>
<td>`AUTH0_CLIENT_SIGNING_KEY_PATH`</td>
<td>Chemin relatif au certificat de clé de connexion JWT.</td>
</tr>
<tr>
<td>`AUTH0_CLIENT_SIGNING_ALGORITHM`</td>
<td>Facultatif. Algorithme de signature JWT utilisé pour le certificat.</td>
</tr>
</tbody>
</table>

## Jeton d’accès

L’authentification par [Access token (Jeton d’accès)](/docs/fr-ca/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) exige que vous fournissiez directement à l’application un jeton d’accès de <Tooltip href="/docs/fr-ca/glossary?term=management-api" tip="Management API
Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire">Management API</Tooltip> d’Auth0. Cette option fait peser une plus grande responsabilité sur les développeurs, mais elle permet de mettre en place des flux de travail flexibles et spécifiques lorsque cela est nécessaire.

Pour configurer l’authentification par jeton d’accès, transmettez un jeton d’accès Management API Auth0 via la `variable d’environnement`[AUTH0_ACCESS_TOKEN](/docs/fr-ca/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).
