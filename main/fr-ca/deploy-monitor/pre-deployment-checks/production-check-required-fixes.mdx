Les contrôles suivants permettent de vérifier si vous avez rempli toutes les conditions requises pour un déploiement en production réussi.

Pour plus d’informations sur les résultats des contrôles, voir [Exécuter les contrôles de production](/docs/fr-ca/deploy-monitor/pre-deployment-checks/how-to-run-production-checks).

<table class="table"><thead>
<tr>
<th>Vérification</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>Toutes les actions utilisent une [version recommandée de NodeJS]https://auth0.com/docs/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-actions-nodejs-16-to-nodejs-18)</td>
<td>Définir toutes les <a href="https://auth0.com/docs/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-actions-nodejs-16-to-nodejs-18">Actions</a> sur une version recommandée de NodeJS pour profiter des correctifs de sécurité et de nouvelles fonctionnalités.</td>
</tr>
<tr>
<td>L’option <a href="/docs/fr-ca/migrations/guides/calling-api-with-idtokens">Autorisation des jetons d’ID pour l’authentification de Management API v2</a> est désactivée</td>
<td>Les capacités d’utilisation des jetons d’ID pour autoriser certains des points de terminaison des utilisateurs et des identifiants de périphériques du management API sont obsolètes. Après avoir terminé la migration vers les jetons d’accès, assurez-vous que l’option <a href="https://manage.auth0.com/#/tenant/advanced">Allow ID Tokens for Management API v2 Authentication (Autorisation des jetons d’ID pour l’authentification du management API v2)</a> est désactivée. Si vous ne voyez pas ce paramètre, c’est que votre locataire a été créé après la suppression de cette fonctionnalité, qui est donc déjà désactivée par défaut.</td>
</tr>
<tr>
<td>allowLegacyRoGrantTypes</td>
<td>L’ancien <a href="https://auth0.com/docs/troubleshoot/product-lifecycle/past-migrations/migration-oauthro-oauthtoken">point de terminaison oauth/ro</a> dans les paramètres de migration du locataire est obsolète. Après avoir mis à jour vos applications pour utiliser des points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.</td>
</tr>
<tr>
<td>allowLegacyRoGrantTypes</td>
<td>L’ancien <a href="https://auth0.com/docs/api/authentication#social-with-provider-s-access-token">point de terminaison oauth/access_token</a> dans les paramètres de migration du locataire est obsolète. Après avoir mis à jour vos applications pour utiliser les points de terminaison pris en charge, rassurez-vous de désactiver ce point de terminaison.</td>
</tr>
<tr>
<td>allowLegacyTokenInfoEndpoint</td>
<td>L’ancien <a href="https://auth0.com/docs/troubleshoot/product-lifecycle/past-migrations#tokeninfo-endpoint-validation">point de terminaison tokeninfo</a> dans les paramètres de migration du locataires sont obsolètes. Après avoir mis à jour vos applications pour utiliser des points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.</td>
</tr>
<tr>
<td>allowLegacyRoGrantTypes</td>
<td>L’ancien <a href="https://auth0.com/docs/authenticate/login/oidc-conformant-authentication/oidc-adoption-delegation">point de terminaison delegation</a> dans les paramètres de migration du locataire est obsolète. Après avoir mis à jour vos applications pour utiliser des points de terminaison pris en charge, assurez-vous de désactiver ce point de terminaison.</td>
</tr>
<tr>
<td>Les <a href="/docs/fr-ca/protocols/oauth2/redirect-users">URL de rappel autorisées</a> ne doivent pas inclure Localhost</td>
<td>Valide que les <a href="https://manage.auth0.com/#/applications">URL de rappel autorisées d’application)</a> ne pointent pas vers`localhost`, `127.0.0.1`, etc.</td>
</tr>
<tr>
<td><a href="/docs/fr-ca/dashboard/reference/settings-application">Allowed Web Origins (Origines Web autorisées)</a> ne doit pas inclure`localhost`</td>
<td>Valide que les <a href="https://manage.auth0.com/#/applications">URL d’origine Web autorisées</a> ne pointent pas vers `localhost`.</td>
</tr>
<tr>
<td><a href="/docs/fr-ca/email/providers">Fournisseur d’email</a> doit être configuré</td>
<td>Vérifie que le <a href="https://manage.auth0.com/#/emails/provider">fournisseur d’email personnalisé</a> a été configuré.</td>
</tr>
<tr>
<td>Les crochets sont obsolètes et doivent être <a href="https://auth0.com/docs/customize/actions/migrate/migrate-from-hooks-to-actions">migrés vers les Actions(</a></td>
<td>Les crochets sont obsolètes. Nous vous recommandons vivement de commencer à utiliser des <a href="https://auth0.com/docs/customize/actions/migrate/migrate-from-hooks-to-actions">Actions</a> et d’initier le processus de migration. Les crochets existants resteront actifs jusqu’au 18 novembre 2024.</td>
</tr>
<tr>
<td><a href="/docs/fr-ca/mfa/guides/configure-sms">Authentification multifacteur (MFA)</a> doit être configuré (Dépendance : AMF configurée)</td>
<td>Si vous utilisez l’AMF avec le <a href="https://manage.auth0.com/#/mfa">facteur Message téléphonique</a>,configurez-la avec un fournisseur de livraison Twilio ou personnalisé pour éviter les limites anti-attaques du fournisseur de livraison par défaut.</td>
</tr>
<tr>
<td>Le <a href="/docs/fr-ca/guides/migration-legacy-flows#user-profiles">Profil utilisateur hérité</a> doit être désactivé</td>
<td>Les flux d’authentification hérités qui permettent aux jetons d’ID et au point de terminaison `/userinfo` d’inclure le profil utilisateur complet sont obsolètes. Après avoir terminé la migration vers les nouvelles API conformes à l’OIDC, assurez-vous que le commutateur du <a href="https://manage.auth0.com/#/account/advanced">Profil utilisateur hérité</a> est désactivé.</td>
</tr>
<tr>
<td>Les règles sont obsolètes et doivent être <a href="https://auth0.com/docs/customize/actions/migrate/migrate-from-rules-to-actions">migrées vers les Actions(</a></td>
<td>Les règles sont obsolètes. Nous vous recommandons vivement de commencer à utiliser des <a href="https://auth0.com/docs/customize/actions/migrate/migrate-from-rules-to-actions">Actions</a> et d’initier le processus de migration. Les Règles existantes resteront actives jusqu’au 18 novembre 2024.</td>
</tr>
<tr>
<td>Les <a href="/docs/fr-ca/connections/social/devkeys">Connexions sociales</a> ne doivent pas utiliser les Clés de développement Auth0</td>
<td>Vérifie que les <a href="https://manage.auth0.com/#/connections/social">Connexions sociales</a> n’utilisent pas les clés de développement Auth0 par défaut.</td>
</tr>
<tr>
<td>L’adresse courriel d’assistance doit être configuré</td>
<td>Vérifie que l’<a href="https://manage.auth0.com/#/account">adresse courriel d’assistance</a> est configurée dans les Paramètres du locataire.</td>
</tr>
<tr>
<td>L’URL d’assistance est configurée</td>
<td>Vérifie que l’<a href="https://manage.auth0.com/#/account">URL d’assistance</a> est configurée dans les Paramètres du locataire.</td>
</tr>
<tr>
<td>L’étiquette d’environnement du locataire doit être configurée</td>
<td>Vérifie que l’<a href="https://support.auth0.com/tenants/public">étiquette d’environnement du locataire</a> est définie de manière appropriée sur Production, Staging ou Development. Les modifications apportées à cette balise affectent la limite anti-attaques de votre locataire.</td>
</tr>
<tr>
<td>Le locataire est configuré pour utiliser une version par défaut recommandée de <a href="https://auth0.com/docs/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-actions-nodejs-16-to-nodejs-18">NodeJS</a></td>
<td>Mettez à jour votre locataire vers une <a href="https://auth0.com/docs/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-actions-nodejs-16-to-nodejs-18">version recommandée de NodeJS</a> pour tirer parti des correctifs de sécurité et des nouvelles fonctionnalités.</td>
</tr>
<tr>
<td>URI de connexion du locataire</td>
<td>Vérifie que <a href="https://manage.auth0.com/#/tenant/advanced">L’URI de connexion du locataire</a> est configurée dans les Paramètres du locataire.</td>
</tr>
<tr>
<td>Utiliser un Domaine personnalisé dans les modèles de courriel de marque</td>
<td>Assurez-vous que vos modèles de courriels utilisent un <a href="https://auth0.com/docs/customize/custom-domains/configure-features-to-use-custom-domains#use-custom-domains-in-emails">domaine personnalisé</a>. Nous vous recommandons vivement de personnaliser tous les courriels destinés aux utilisateurs pour utiliser votre domaine personnalisé et le branding de votre entreprise.</td>
</tr>
</tbody>
</table>
