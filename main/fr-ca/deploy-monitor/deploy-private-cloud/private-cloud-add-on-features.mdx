L’option de déploiement du Nuage privé comprend deux fonctionnalités supplémentaires : Basculement géographique et conformité PCI. Si vous avez des questions sur ces fonctionnalités, veuillez [contacter le service Ventes Auth0](https://auth0.com/get-started?place=documentation%20post&type=link&text=contact%20auth0%20sales).

## Basculement-géographique du Nuage privé

Le basculement géographique est une configuration de basculement actif-passif qui permet aux clients de spécifier une région de basculement secondaire à utiliser en cas d’une panne régionale majeure. En général, toute région du même fournisseur de nuage dont la latence est inférieure à 100 ms peut être sélectionnée comme région de basculement. Dans certains cas, cependant, les fournisseurs de nuage préfèrent des régions de basculement spécifiques. L’équipe d’intégration Auth0 travaillera avec vous pour sélectionner les meilleures régions pour votre cas d’utilisation.

De plus, une troisième région « tie break » est requise pour l’un de nos magasins de données. Cette troisième région ne reçoit jamais de trafic en direct mais est tenue d’élire un nouveau leader en cas de basculement. Auth0 peut également vous aider à sélectionner la meilleure option pour cette région.

Si une panne régionale est détectée, le basculement géographique se déclenche au nom du client par Auth0. L’ensemble du processus est automatisé une fois déclenché. Lorsque la confirmation que la région ne rencontre plus de problèmes est reçue, Auth0 basculera vers la région principale.

## Conformité PCI

La conformité PCI est disponible en tant que module complémentaire pour les clients qui en ont besoin.
