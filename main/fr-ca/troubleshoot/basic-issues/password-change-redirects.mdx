L’URI vers laquelle l’utilisateur est redirigé après une réinitialisation de mot de passe réussie dépend de toutes les conditions suivantes :

1. si votre locataire utilise la connexion Universelle ou Classique, et
2. si vous générez directement un ticket de changement de mot de passe via Management API, ou
3. si vous envoyez un courriel de changement de mot de passe depuis Authentication API, la page de connexion, ou une autre source.

Le diagramme de flux suivant est un guide pour définir l’URI de redirection de votre locataire en fonction de sa configuration.
