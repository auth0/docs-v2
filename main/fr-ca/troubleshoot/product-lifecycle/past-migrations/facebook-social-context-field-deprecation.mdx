À partir du **30 juillet 2020**, les connexions Facebook qui demandent le champ `context` ne fonctionneront pas. Par conséquent, Auth0 cessera de le demander pour toutes les connexions.

Le 30 avril, [Facebook a supprimé](https://developers.facebook.com/docs/graph-api/changelog/4-30-2019-endpoint-deprecations) l'utilisation du champ **Contexte social** pour les nouvelles applications. Auth0 a continué à demander ce champ par défaut pour les connexions Facebook créées avant le 30 avril 2019. Vous pouvez vous assurer que le champ n'est pas demandé avant le 30 juillet en décochant le champ **Contexte social** de la section de connexion Données utilisateur :

<Frame>![Facebook Context Field Deprecation](/images/fr-ca/cdy7uua7fh8z/4oi3foRuwYYOveDlq2vJM/a57cd954ceeaf6d7722796be7c743b76/facebook-context.png)</Frame>

Une fois que vous aurez décoché le champ **Contexte social**, les données du profil n’incluront pas le champ contextuel. Le champ contient le contenu suivant :

```text lines
"context": {
      "mutual_likes": {"data": [],"summary": {"total_count": 0}},
      "id": "dXNlcl9...UZD"
    }
```

## Actions

Si vous n'utilisez pas le champ `context` dans le profil Facebook renvoyé par Auth0 dans votre application, celle-ci continuera de fonctionner sans modification. Sinon, vous devrez modifier le code de votre application pour qu'il ne dépende pas de ce champ.

Pour vous assurer que votre application n'est pas affectée le 30 juillet, nous vous conseillons de décocher le champ **Contexte social** dans les propriétés de la connexion Facebook.
