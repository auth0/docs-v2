À compter du 21 juin 2023, Auth0 ne prendra plus en charge la connexion aux bases de données Oracle à partir des fonctionnalités d’extensibilité Node.js. Auth0 supportait auparavant la connexion aux bases de données Oracle via le module `oracledb` Node.js.

## Flux de travail affectés

Passez en revue vos fonctionnalités d’extensibilité pour le code qui contient `require(oracledb)` ou  `require('oracledb@4.2.0')`.

Consulter les flux ou les fonctionnalités suivants qui permettent le code d’extensibilité pour les références à [oracledb](https://www.npmjs.com/package/oracledb) :

* [Crochets](https://auth0.com/docs/customize/hooks)
* [Règles](https://auth0.com/docs/customize/rules)
* [Scripts de base de données personnalisés](https://auth0.com/docs/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Connexions sociales personnalisées](https://auth0.com/docs/authenticate/identity-providers/social-identity-providers/oauth2)

## Mises à jour recommandées

Il n’y a pas de remplacement officiel prévu pour cette fonction. Auth0 recommande de remplacer la logique `oracledb` par un service distinct au sein de votre propre infrastructure. Après avoir créé le service dans votre infrastructure, vous pouvez utiliser une fonction d’extensibilité pour effectuer une requête HTTP afin de déclencher la logique de base de données Oracle.

Pour en savoir plus sur la façon d’implémenter le module `oracledb` dans votre propre infrastructure, consultez le [Guide de démarrage rapide OracleDB](https://node-oracledb.readthedocs.io/en/latest/user_guide/installation.html#quickstart).

Par exemple, dans un script de connexion à une base de données personnalisée utilisant `oracledb`, supprimez la fonction qui appelle `oracledb` avec un client HTTP générique pour capturer la réponse.

Pour plus d’informations sur la création d’Actions ou la migration vers des Actions, [consultez la documentation Auth0](https://auth0.com/docs/customize/actions).
