Pour intégrer Auth0 à une application machine-machine (M-M), vous devez d’abord enregistrer votre application auprès d’Auth0 à l’aide du <Tooltip href="/docs/fr-ca/glossary?term=auth0-dashboard" tip="Auth0 Dashboard
Principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire">Auth0 Dashboard</Tooltip> (Tableau de bord Auth0).

1. Naviguez vers [Dashboard (Tableau de bord) > Applications > Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application (Créer une application**).
2. Saisissez un nom descriptif pour votre application, sélectionnez **Machine to Machine Applications (Application de communication machine à machine)** et cliquez sur **Create (Créer)**.
3. Sélectionnez Management API pour autoriser la communication entre machines, car c’est ainsi que l’OAG interagit avec Auth0.

4. Cliquez sur **Authorize (Autoriser)**.

5. Chaque application de communication entre machines qui accède à une API doit se voir accorder un ensemble de permissions (ou portées) par l’utilisateur authentifié. Sélectionnez les **Permissions (Autorisations)** que vous voulez accorder à l’aide du jeton d’accès de votre application. OAG impose les autorisations suivantes pour le **client** et les **utilisateurs :**

* `read:client, write:client, update:client, delete:client`
* `read:users`

6. Cliquez sur **Authorize (Autoriser)**.

7. Sélectionnez l’onglet **Credentials (Identifiants)**  de votre application.

8. Cliquez sur **<Tooltip href="/docs/fr-ca/glossary?term=client-id" tip="ID client
Identifiant attribué par Auth0 à votre ressource enregistrée." cta="Voir le glossaire">Client ID</Tooltip> (ID client)** et copiez-le dans une application d’édition de texte.

9. Cliquez sur **<Tooltip href="/docs/fr-ca/glossary?term=client-secret" tip="Secret client
Secret utilisé par un client (application) pour s’authentifier auprès du serveur d’autorisation; il ne doit être connu que du client et du serveur d’autorisation et doit être suffisamment aléatoire pour ne pas être deviné." cta="Voir le glossaire">Client Secret</Tooltip> (Secret client)** et copiez-le dans une application d’édition de texte.

10. Poursuivez avec la procédure [Utiliser Customer Identity Cloud comme IdP pour un Access Gateway](https://help.okta.com/okta_help.htm?type=oag&id=ext_oag_config_idp_okta).
