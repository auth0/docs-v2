Pour intégrer Auth0 à une application de communication entre machines, vous devez d’abord enregistrer votre application auprès d’Auth0 à l’aide d’Auth0 Dashboard. Ces applications peuvent inclure des applications non interactives, comme des outils de ligne de commande, des démons, des appareils IoT ou des services s’exécutant sur l’ordinateur principal.

Les applications M-M sont liées à une API et à ses autorisations ou[permissions](/docs/fr-ca/get-started/apis/scopes/api-scopes), qui sont sélectionnées à partir de valeurs prédéfinies. Assurez-vous d’avoir déjà [enregistré l’API associée](/docs/fr-ca/get-started/auth0-overview/set-up-apis) auprès d’Auth0 et défini les permissions pour l’API avant de commencer cette inscription.

Si vous voulez autoriser votre application à n’accéder qu’à l’Auth0 Management API, vous n’avez rien à faire. Cette API est pré-renseignée.

Pour intégrer Auth0 à une application de communication entre machines, vous devez d’abord enregistrer votre application auprès d’Auth0 à l’aide d’Auth0 Dashboard.

1. Naviguez vers [Dashboard (Tableau de bord) > Applications > Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application (Créer une application**).
2. Saisissez un nom descriptif pour votre application, sélectionnez **Machine to Machine Applications (Application de communication machine à machine)** et cliquez sur **Create (Créer)**.
3. Sélectionnez l’API que vous souhaitez pouvoir appeler à partir de votre application.
4. Chaque application M-M qui accède à une API doit se voir accorder un ensemble de permissions (ou autorisations), accordées par l’utilisateur authentifié. Pour savoir comment ajouter des autorisations, consultez [Ajouter des autorisations d’API](/docs/fr-ca/get-started/apis/add-api-permissions). Sélectionnez les **Permissions (Autorisations)** que vous souhaitez inclure dans le jeton d’accès de votre application, puis cliquez sur **Authorize (Autoriser)**.

   <Warning>

   Utilisez les permissions `update:client_grants` et `create:client_grants` uniquement avec des applications à privilèges élevés, car elles permettent au client de s’accorder des autorisations supplémentaires.

   </Warning>

## Prochaines étapes

Une fois que vous avez enregistré et configuré votre application, voici quelques étapes courantes à suivre :

* Configurez une connexion et activez-la pour votre application.
* Modifiez le code de votre application pour utiliser votre application enregistrée avec Auth0. Voir nos [Guides de deémarrage rapide Auth0](/docs/fr-ca/quickstarts), où vous trouverez des instructions détaillées et des échantillons pour une variété de technologies. Vous apprendrez également comment implémenter la connexion et la déconnexion, gérer vos sessions utilisateur, récupérer et afficher les informations de profil utilisateur, et plus encore.
* Utilisez les [API Auth0](/docs/fr-ca/api).

  + L'[Authentication API](/docs/fr-ca/api/authentication) gère toutes les fonctions principales liées à l’identité (par exemple, la connexion, la déconnexion et l’obtention du profil utilisateur). La plupart des utilisateurs utilisent cette API à travers nos guides de démarrage rapide, la [bibliothèque Auth0.js](/docs/fr-ca/libraries/auth0js) ou le [widget Lock](/docs/fr-ca/libraries/lock). Cependant, si vous construisez entièrement votre interface utilisateur d’authentification manuellement, vous devrez interagir directement avec cette API.
  + Le [Management API](/docs/fr-ca/api/management/v2) vous permet d’automatiser diverses tâches qui sont également accessibles via le tableau de bord dans Auth0 (par exemple : création d’utilisateurs, configuration des types d’autorisation de l’application).
