Auth0 fournit des connexions d’entreprise pour authentifier les utilisateurs auprès de fournisseurs d’identité fédérés (<Tooltip href="/docs/fr-ca/glossary?term=idp" tip="Fournisseur d’identité (IdP)
Service de stockage et de gestion des identités numériques." cta="Voir le glossaire">IdP</Tooltip>) externes, tels que Azure AD, Google Workspace, PingFederate, etc.

<Card title="La disponibilité varie selon le plan Auth0">

La disponibilité de cette fonctionnalité dépend de votre plan Auth0 ou de votre accord personnalisé. Pour en savoir plus, lisez [Page de tarification d’Auth0](https://auth0.com/pricing).

</Card>

## Créer une connexion d’entreprise dans Auth0

Auth0 prend en charge de nombreux fournisseurs d’identité. Pour en savoir plus, consultez la page [Fournisseurs d’identité de l’entreprise](/docs/fr-ca/authenticate/identity-providers/enterprise-identity-providers).

#### Protocole OpenID Connect (OIDC)

Les connexions d’entreprise utilisant <Tooltip href="/docs/fr-ca/glossary?term=openid" tip="OpenID
Norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans avoir à collecter leurs informations de connexion." cta="Voir le glossaire">OpenID</Tooltip> Connect ou Okta Workforce en tant que fournisseur d’identité peuvent prendre en charge les éléments suivants :

* Proof Key for Code Exchange (PKCE)

  + Pour plus d’informations sur PKCE, consultez [Protéger les applications avec la PKCE](https://www.oauth.com/oauth2-servers/pkce/).
* Correspondance des demandes d’attributs et mappage des jetons d’accès
* Intégration de UserInfo

Vous pouvez actuellement implémenter ces fonctionnalités pour les connexions [Connexion OpenID](/docs/fr-ca/authenticate/identity-providers/enterprise-identity-providers/oidc) ou [Okta Workforce](/docs/fr-ca/authenticate/identity-providers/enterprise-identity-providers/okta). Pour de plus amples renseignements, consultez [Configurer la PKCE et le mappage des demandes pour les connexions OIDC](/docs/fr-ca/authenticate/identity-providers/enterprise-identity-providers/configure-pkce-claim-mapping-for-oidc).

## Afficher les connexions d’entreprise

<Tabs><Tab title="Tableau de bord">

Accédez à [**Auth0 Dashboard > Authentication > Enterprise (Tableau de bord > Authentification &gt > Entreprise)**](https://manage.auth0.com/#/connections/enterprise) pour afficher tous les types de connexion d’entreprise disponibles. Sélectionnez un type de connexion (par exemple, SAML) pour déterminer s’il existe des connexions configurées de ce type.

Vous pouvez aussi sélectionner une connexion configurée et vérifier dans l’onglet **Applications** si elle est activée pour quelque application que ce soit.

</Tab><Tab title="Management API">

Appeler l’Auth0 Management API [Get all Connections endpoint (Obtenir tous les points de terminaison de connexion)](https://auth0.com/docs/api/management/v2#!/Connections/get_connections) pour obtenir des informations sur vos connexions. Inclure le paramètre `strategy` pour filtrer par type de connexion.

Les valeurs pour le paramètre `strategy` pour le type de connexion d’entreprise sont les suivantes :

* `ad` (Active Directory/LDAP)
* `adfs` (ADFS)
* `auth0-adldap`
* `google-apps` (Google Workspace)
* `ip`
* `office365`
* `oidc` (OpenID Connect)
* `pingfederate` (PingFederate)
* `samlp` (SAML)
* `sharepoint`
* `waad` (Microsoft Azure AD)

</Tab></Tabs>

## Qu’est-ce qu’une connexion d’entreprise active?

Une connexion d’entreprise est considérée active si, au cours du dernier mois, elle :

* a été activée pour accéder à une application
* est associée à des activités (par exemple, une connexion, une inscription ou l’échange de jetons).

Si une connexion d’entreprise n’a jamais été activée pour aucune application, ou si elle a été activée mais n’a fait l’objet d’aucune activité par les utilisateurs pendant le mois en cours, elle n’est pas considérée comme active.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

La connexion Okta Workforce Enterprise n’est pas prise en compte dans la limite des connexions actives de votre plan Enterprise.

</Callout>
