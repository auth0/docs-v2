Si vous avez besoin d’intégrer l’interface utilisateur de connexion dans votre application, vous pouvez le faire en utilisant [notre API intégrée sans mot de passe](/docs/fr-ca/authenticate/passwordless/implement-login/embedded-login/relevant-api-endpoints) ou nos trousses SDK.

Selon le type d’application que vous voulez créer, vous devrez l’implémenter de différentes manières :

* Pour les applications à page unique (SPA) (par exemple, Angular ou React), consultez [Authentification intégrée sans mot de passe pour les applications à page unique](/docs/fr-ca/authenticate/passwordless/implement-login/embedded-login/spa).
* Pour les applications natives (par exemple, iOS, Android, ou les applications de bureau), consultez [Connexion intégrée sans mot de passe pour les applications natives](/docs/fr-ca/authenticate/passwordless/implement-login/embedded-login/native).
* Pour les applications Web courantes (par exemple, NodeJS, Java, Rails ou .NET), consultez [Connexion intégrée sans mot de passe pour les applications Web courantes](/docs/fr-ca/authenticate/passwordless/implement-login/embedded-login/webapps).

Cependant, les connexions intégrées dans les applications web avec Auth0 utilisent l’authentification Cross-origin, ce que nous ne recommandons pas. Pour en savoir plus, consultez [Authentification Cross-origin](/docs/fr-ca/authenticate/login/cross-origin-authentication).
