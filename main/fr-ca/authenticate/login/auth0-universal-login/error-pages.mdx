Si votre URL de rappel est valide, lorsqu’une erreur d’autorisation se produit, le serveur d’autorisations renvoie les paramètres d’erreur et d’état appropriés à votre URL de rappel. Si l’URL de rappel n’est pas valide, votre application affichera la page d’erreur générique Auth0 par défaut. Vous pouvez également personnaliser les pages d’erreur. Pour en savoir plus, consultez les [Pages d’erreur personnalisées](/docs/fr-ca/customize/login-pages/custom-error-pages).

Pour connaître le nom par défaut de la page d’erreur générique, voir [Utiliser le contrôle de version pour gérer vos pages de connexion universelles](/docs/fr-ca/customize/login-pages/classic-login/version-control).

## Affichage

La page d’erreur générique affiche les informations suivantes si vous avez configuré les champs associés dans vos [Tenant Settings (Paramètres du locataire)](https://manage.auth0.com/#/tenant/).

<table class="table"><thead>
<tr>
<th>Champ</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>Nom convivial</td>
<td>Nom de votre société</td>
</tr>
<tr>
<td>Logo URL</td>
<td>URL du logo de votre société</td>
</tr>
<tr>
<td>Courriel d’assistance</td>
<td>Adresse courriel de l’équipe d’assistance de votre société</td>
</tr>
<tr>
<td>Assistance URL</td>
<td>URL de la page d’assistance de votre société</td>
</tr>
</tbody>
</table>

## Paramètres

Le serveur d’autorisations renvoie les paramètres ajoutés à l’URL sous la forme d’une chaîne de requête.

<table class="table"><thead>
<tr>
<th><strong>Paramètre</strong></th>
<th><strong>Description</strong></th>
</tr>
</thead>
<tbody>
<tr>
<td>`client_id`</td>
<td>Identifiant de l’application Auth0.</td>
</tr>
<tr>
<td>`connection`</td>
<td>Connexion utilisée au moment de l’erreur.</td>
</tr>
<tr>
<td>`lang`</td>
<td>Langue utilisée au moment de l’erreur.</td>
</tr>
<tr>
<td>`error`</td>
<td>Code d’erreur de l’erreur.</td>
</tr>
<tr>
<td>`error_description`</td>
<td>Description de l’erreur.</td>
</tr>
<tr>
<td>`tracking`</td>
<td>Identifiant utilisé par Auth0 pour trouver des erreurs dans les journaux internes.</td>
</tr>
</tbody>
</table>

Les paramètres présentés varient en fonction du type d’erreur et sont spécifiques à la demande. Par exemple, si la demande ne contient pas `client_id` et qu’elle aboutit à une erreur, la page d’erreur ne renverra pas d’ID client.
