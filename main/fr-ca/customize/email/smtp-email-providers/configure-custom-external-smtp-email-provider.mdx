<Card title="Avant de commencer">

* Le fournisseur doit prendre en charge l'[authentification](https://en.wikipedia.org/wiki/SMTP_Authentication) LOGIN.
* Le fournisseur doit prendre en charge [TLS 1.2](https://en.wikipedia.org/wiki/STARTTLS) ou une version supérieure.
* Le fournisseur doit disposer d’un certificat signé par une autorité de certification (CA) publique.
* Configurez votre fournisseur pour qu’il [autorise les connexions entrantes depuis Auth0](https://auth0.com/docs/secure/security-guidance/data-security/allowlist). Pour voir la liste des adresses IP requises, les clients Public Cloud (Nuage public) peuvent naviguer vers [Auth0 Dashboard > Marque > Fournisseur de messagerie électronique](https://manage.auth0.com/#/templates/provider). Les clients du nuage privé peuvent communiquer avec l’[assistance Auth0](https://support.auth0.com) pour demander des adresses IP propres à leur instance.

</Card>

Vous pouvez configurer votre propre fournisseur de messagerie SMTP pour gérer, surveiller et dépanner vos communications par courriel, ainsi que personnaliser les modèles de courriels.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

Lors de la configuration d’un fournisseur SMTP personnalisé, Auth0 prend en charge Basic Auth (authentification de base). Bien que depuis le 1er octobre 2022 Microsoft ne prenne en charge que Modern Auth, vous pouvez toujours utiliser Basic Auth pour l’authentification SMTP dans votre configuration Auth0. Si vous êtes intéressé par la prise en charge par Auth0 de Modern Auth pour Office 365, veuillez [envoyer une demande de fonctionnalité](https://auth0.com/feedback).

</Callout>

## Configurer un serveur SMTP personnalisé

1. Accédez à [Auth0 Dashboard > Image de marque > Fournisseur de messagerie électronique](https://manage.auth0.com/#/templates/provider).
2. Activez la bascule **Utiliser mon propre fournisseur de messagerie**.
3. Sélectionnez **SMTP**.
4. Saisissez une adresse courriel **From (De)**, puis saisissez **Hôte**, **Port**, **Nom d’utilisateur**, et **Mot de passe** pour votre serveur SMTP. Ports fréquemment utilisés : 25 et 587. Évitez d’utiliser le port 25 si possible, car de nombreux fournisseurs ont des limitations sur ce port.
5. Cliquez sur **Enregistrer**.

## Tester la livraison des courriels

Vous pouvez maintenant envoyer un courriel de test en utilisant le bouton **Envoyer un courriel de test**. Si vous avez tout configuré correctement, vous recevrez un courriel de confirmation. Si vous ne recevez pas de courriel après quelques minutes, veuillez vérifier vos [Journaux Auth0](https://manage.auth0.com/#/logs) pour toute erreur.

SMTP permet de configurer des services de test qui vous aident à vérifier le bon fonctionnement de votre configuration sans envoyer de courriels indésirables à vos utilisateurs. Pour en savoir plus, veuillez consulter [Configurer des serveurs SMTP de test](/docs/fr-ca/email/testing).
