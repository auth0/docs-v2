L’objet `event` du déclencheur d’actions `custom-email-provider` fournit des informations contextuelles sur les données du courriel.

<table class="table">
<thead>
<tr>
<th>Propriété</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>
<p>`event.client`</p>
</td>
<td>
<p>
          Informations sur le client avec lequel la transaction de connexion
          a été initiée.
        </p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`client_id` <em>Chaîne</em>.
            <span>L’ID client de l’application à laquelle l’utilisateur se connecte.</span>
</li>
<li>
`metadata` <em>Dictionnaire</em>.
            <span>Un objet qui contient les autres propriétés d’une application.</span>
</li>
<li>
`name` <em>Chaîne</em>.
            <span>Le nom de l’application (comme défini dans le Dashboard).</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.connection`</p>
<p><em>(Facultatif)</em></p>
</td>
<td>
<p>
          Détails sur la connexion utilisée pour authentifier l’utilisateur.
        </p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`id` <em>Chaîne facultative</em>.
            <span>L’identifiant unique de la connexion.</span>
</li>
<li>
`metadata` <em>Dictionnaire facultatif</em>.
            <span>Métadonnées associées à la connexion.</span>
</li>
<li>
`name` <em>Chaîne facultative</em>.
            <span>Le nom de la connexion utilisée pour authentifier l’utilisateur (par ex. :
              `twitter` ou `some-g-suite-domain`).</span>
</li>
<li>
`strategy` <em>Chaîne facultative</em>.
            <span>Le type de connexion. Pour les connexions sociales,
              `event.connection.strategy === event.connection.name`.
              Pour les connexions d’entreprise, la stratégie est la suivante :
              `waad` (Windows Entra ID), `ad` (Active
              Directory/LDAP), `auth0` (connexions aux bases de données),
              etc.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.notification`</p>
</td>
<td>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`html` <em>Chaîne</em>.
            <span>Modèle HTML rendu.</span>
</li>
<li>
`from` <em>Chaîne</em>.
            <span>Adresse courriel de l’expéditeur pour les courriels.</span>
</li>
<li>
`subject` <em>Chaîne</em>.
        <span>Objet à joindre au courriel.</span>
</li>
<li>
`locale` <em>Chaîne facultative</em>.
            <span>La langue dans laquelle nous avons rendu le message, par exemple
              `en_US`, telle que définie dans la spécification BCP-47.</span>
</li>
<li>
`message_type` <em>Chaîne</em>.
            <p>
              Le type de message envoyé, par exemple
              `verify_email` ou `welcome_email`.
            </p>
<p>Les valeurs possibles sont les suivantes :</p>
<ul>
<li>`verify_email`</li>
<li>`verify_email_by_code`</li>
<li>`reset_email`</li>
<li>`reset_email_by_code`</li>
<li>`welcome_email`</li>
<li>`verification_code`</li>
<li>`mfa_oob_code`</li>
<li>`enrollment_email`</li>
<li>`blocked_account`</li>
<li>`stolen_credentials`</li>
<li>`try_provider_configuration_email`</li>
<li>`organization_invitation`</li>
</ul>
</li>
<li>
`text` <em>Chaîne</em>.
            <span>Modèle de texte rendu.</span>
</li>
<li>
`to` <em>Chaîne</em>.
            <span>Adresse courriel du destinataire.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.organization`</p>
<p><em>(Facultatif)</em></p>
</td>
<td>
<p>
          Détails sur l’Organization associée à la transaction
          en cours.
        </p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`display_name` <em>Chaîne</em>.
            <span>Le nom convivial de l’Organization.</span>
</li>
<li>
`id` <em>Chaîne</em>.
            <span>L‘identifiant de l’Organization.</span>
</li>
<li>
`metadata` <em>Dictionnaire</em>.
            <span>Métadonnées associées à l’Organization.</span>
</li>
<li>
`name` <em>Chaîne</em>.
            <span>Le nom de l’Organization.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.request`</p>
<p><em>(Facultatif)</em></p>
</td>
<td>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`geoip` <em>Objet facultatif</em>.
            <p>Comprend les propriétés suivantes :</p>
<ul>
<li>`countryCode` <em>Chaîne facultative</em>.</li>
<li>`countryCode3` <em>Chaîne facultative</em>.</li>
<li>`cityName` <em>Chaîne facultative</em>.</li>
<li>`latitude` <em>Chaîne facultative</em>.</li>
<li>`longitude` <em>Chaîne facultative</em>.</li>
<li>`timeZone` <em>Chaîne facultative</em>.</li>
<li>`subdivisionCode` <em>Chaîne facultative</em>.</li>
<li>`subdivisionName` <em>Chaîne facultative</em>.</li>
<li>`continentCode` <em>Chaîne facultative</em>.</li>
</ul>
</li>
<li>
`ip` <em>Chaîne facultative</em>.
            <span>L’adresse IP initiale de la requête.</span>
</li>
<li>
`query` <em>Dictionnaire facultatif</em>.
            <span>Les paramètres de la chaîne de requête envoyés à la demande d’autorisation. Pris en charge uniquement pour le flux de réinitialisation des mots de passe.</span>
</li>
<li>
`user_agent` <em>Chaîne facultative</em>.
            <span>La valeur de l’en-tête `User-Agent` reçu lorsque de la transaction
              a été initiée.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.tenant`</p>
</td>
<td>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`friendly_name` <em>Chaîne facultative</em>.
            <span>Le nom convivial du locataire, généralement une version
              plus lisible de l’ID.</span>
</li>
<li>
`home_url` <em>Chaîne facultative</em>.
            <span>L’URL d’accueil du locataire, si elle est définie et telle qu’elle figure dans ses
               paramètres.</span>
</li>
<li>
`id` <em>Chaîne</em>.
            <span>Le nom du locataire.</span>
</li>
<li>
`logo_url` <em>Chaîne facultative</em>.
            <span>L’URL du logo du locataire, si elle est définie et telle qu’elle figure dans ses
              paramètres.</span>
</li>
<li>
`support_email` <em>Chaîne facultative</em>.
            <span>Le courriel du service de soutien du locataire, s’il est défini et tel qu’il figure
              dans ses paramètres.</span>
</li>
<li>
`support_url` <em>Chaîne facultative</em>.
            <span>L’URL du service de soutien du locataire, si elle est définie et tel qu’elle figure
              dans ses paramètres.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.user`</p>
</td>
<td>
<p>
          Objet décrivant l’utilisateur au nom duquel la transaction en cours
          a été initiée.
        </p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`app_metadata` <em>Dictionnaire</em>.
            <span>Champs personnalisés qui stockent des informations sur un utilisateur et qui influencent
              l’accès de l’utilisateur, comme le plan de soutien, les rôles de sécurité ou les groupes
              de contrôle d’accès.</span>
</li>
<li>
`email` <em>Chaîne facultative</em>.
            <span>(unique) Adresse de courriel de l’utilisateur.</span>
</li>
<li>
`email_verified` <em>Booléen</em>.
            <span>Indique si l’utilisateur a vérifié son adresse
              de courriel.</span>
</li>
<li>
`family_name` <em>Chaîne facultative</em>.
            <span>Nom de famille de l’utilisateur.</span>
</li>
<li>
`given_name` <em>Chaîne facultative</em>.
            <span>Prénom de l’utilisateur.</span>
</li>
<li>
`name` <em>Chaîne facultative</em>.
            <span>Nom complet de l’utilisateur.</span>
</li>
<li>
`nickname` <em>Chaîne facultative</em>.
            <span>Pseudonyme de l’utilisateur.</span>
</li>
<li>
`picture` <em>Chaîne facultative</em>.
            <span>URL pointant vers la
              <a href="https://auth0.com/docs/users/change-user-picture">photo de profil de l’utilisateur</a>.</span>
</li>
<li>
`user_id` <em>Chaîne</em>.
            <span>(unique) Identifiant unique de l’utilisateur.</span>
</li>
<li>
`user_metadata` <em>Dictionnaire</em>.
            <span>Champs personnalisés qui stockent des informations sur un utilisateur qui n’ont pas d’incidence
              sur les données auxquelles il peut ou ne peut pas accéder, comme l’adresse professionnelle, l’adresse
              personnelle ou les préférences de l’utilisateur.</span>
</li>
<li>
`username` <em>Chaîne facultative</em>.
            <span>(unique) Nom d’utilisateur</span>
</li>
</ul>
</td>
</tr>
</tbody>
</table>
