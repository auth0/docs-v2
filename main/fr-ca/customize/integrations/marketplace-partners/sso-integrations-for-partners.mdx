Les intégrations authentification unique (<Tooltip href="/docs/fr-ca/glossary?term=single-sign-on" tip="Authentification unique (SSO)
Service qui, après qu’un utilisateur se soit connecté à une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire">SSO</Tooltip>) utilisent une identité Auth0 pour se connecter, via <Tooltip href="/docs/fr-ca/glossary?term=security-assertion-markup-language" tip="Security Assertion Markup Language (SAML)
Protocole normalisé permettant à deux parties d’échanger des informations d’authentification sans mot de passe." cta="Voir le glossaire">SAML</Tooltip>, à un service externe. Cela permet aux clients ayant des applications internes d’utiliser SSO pour accéder à des fournisseurs tiers en utilisant l’identité Auth0.

Les [intégrations SSO disponibles sur Marketplace](https://marketplace.auth0.com/features/sso-integrations) définissent à la fois les valeurs de configuration SAML standard nécessaires pour connecter Auth0 aux fournisseurs externes et les champs nécessaires pour les valeurs supplémentaires spécifiques au client. Lors de ces intégrations, Auth0 agit en tant que fournisseur d’identité dans un scénario initié par le fournisseur de services. Pour savoir comment configurer Auth0 pour ce type d’intégration, voir [Configurer Auth0 en tant que fournisseur d’identité SAML](/docs/fr-ca/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider).

La partie fournie par nos partenaires est la configuration SAML par défaut qui permet à cette transaction de fonctionner. Il s’agit de valeurs ou de champs demandés par le client pour une partie ou la totalité des attributs énumérés dans les [Personnaliser les assertions SAML : attributs des assertions SAML](/docs/fr-ca/authenticate/protocols/saml/saml-configuration/customize-saml-assertions).

Pour créer et tester une intégration, [créez un locataire](https://auth0.com/signup) dans Auth0 et suivez la documentation ci-dessus pour créer une intégration SSO fonctionnelle avec votre service.

De plus, vous pouvez utiliser notre [intégration SSO](https://github.com/auth0-marketplace/TEMPLATE-sso) pour vous aider à concevoir votre intégration.

## Soumettre votre intégration SSO

Une fois que l’intégration fonctionne, Auth0 travaillera avec vous pour publier l’intégration dans [Auth0 Marketplace (Place de marché Auth0)](http://marketplace.auth0.com/). Lorsque vous soumettez votre intégration, vous devez fournir les informations suivantes :

1. Le contenu commercial et marketing pour votre inscription dans la Place de marché Auth0. Cela inclut la description de votre entreprise et votre logo, ainsi qu’une explication de ce qu’effectue l’intégration.
2. Le code et la configuration que vous avez utilisés pour l’intégration. Les partenaires soumettent généralement le code en utilisant GitHub.

**Nous aurons besoin de la configuration et de la documentation suivantes :**

* Le JSON utilisé pour configurer le module complémentaire SAML de l’application.
* La liste des champs qui doivent être montrés au client chargé de configurer l’intégration (comme **URL de rappel**, **<Tooltip href="/docs/fr-ca/glossary?term=audience" tip="Audience
Identifiant unique de 'audience d’un jeton émis. Appelé aud dans un jeton, sa valeur contient l’identifiant soit d’une application (Client ID) pour un jeton d’ID, soit d’une API (API Identifier) pour un jeton d’accès." cta="Voir le glossaire">Audience</Tooltip>**, et **Certificat de signature**).
* Un guide d’installation rédigé en Markdown.
