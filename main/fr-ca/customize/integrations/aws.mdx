Il existe de nombreuses façons d’intégrer Auth0 aux produits et services Amazon, en fonction de vos besoins spécifiques.

## Intégrations de produits

Vous pouvez lire plus en détail comment utiliser les produits ci-dessous en plus de vos services Auth0 et AWS :

* [CloudFront](/docs/fr-ca/customize/custom-domains/self-managed-certificates/configure-aws-cloudfront-for-use-as-reverse-proxy) : à utiliser comme proxy inverse avec votre domaine personnalisé.
* [Simple Email Service (SES)](/docs/fr-ca/customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider) : gérer les communications par courriel entre vos utilisateurs.
* [EventBridge](https://marketplace.auth0.com/integrations/amazon-log-streaming) : flux de journaux vers EventBridge.
* [Cognito](/docs/fr-ca/customize/integrations/aws/amazon-cognito) : à utiliser comme système dorsal pour votre application.
* AWS S3 et DynamoDB :

  + Auth0 fournit des points de terminaison de passerelle VPC pour accéder à AWS S3 et DynamoDB, qui peuvent être nécessaires pour les [Actions](/docs/fr-ca/customize/actions). Cela nécessite des politiques IAM basées sur `aws:SourceVpc`, comme décrit dans [Points de terminaison de passerelle pour Amazon S3](https://docs.aws.amazon.com/vpc/latest/privatelink/vpc-endpoints-s3.html) et [Points de terminaison de passerelle pour Amazon DynamoDB](https://docs.aws.amazon.com/vpc/latest/privatelink/vpc-endpoints-ddb.html). Pour plus de détails sur les identifiants VPC, veuillez contacter le [service d’assistance d’Auth0](/docs/fr-ca/www.support.auth0.com).

## Intégration de la gestion des identités et des accès (IAM)

Nous proposons plusieurs articles expliquant comment configurer AWS pour qu’il fonctionne avec Auth0 :

* [Points de terminaison Sécuriser la passerelle API AWS à l’aide d’autorisateurs personnalisés](/docs/fr-ca/customize/integrations/aws/aws-api-gateway-custom-authorizers) : acceptez les jetons d’accès émis par Auth0.
* [Utiliser les balises de session AWS avec les API et les ressources AWS](/docs/fr-ca/customize/integrations/aws/session-tags) : ajoutez des balises aux ressources et attribuez aux utilisateurs des paires clé/valeur pour le RBAC.
* [Configurer AWS pour l’authentification unique](/docs/fr-ca/customize/integrations/aws/configure-amazon-web-services-for-sso) : autorisez vos utilisateurs à se connecter à AWS en utilisant n’importe quel fournisseur d’identité pris en charge.
* [Ajout d’une connexion Amazon à votre application](/docs/fr-ca/connections/social/amazon) : permettez aux utilisateurs de se connecter à votre application en utilisant leur profil Amazon.
