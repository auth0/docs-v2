L’implémentation propre à votre connexion et votre plan Auth0 ou accord personnalisé que vous utilisez déterminent si cette fonctionnalité est disponible. Pour en savoir plus, lisez [Tarification](https://auth0.com/pricing).

Pour configurer un produit Akamai en tant que proxy inverse, suivez les étapes suivantes :

1. Vérifiez la propriété de votre domaine personnalisé. Pour en savoir plus, consultez [Vérifier la propriété](https://auth0.com/docs/customize/custom-domains/auth0-managed-certificates#verify-ownership).
2. Allez dans [Dashboard > Paramètres > Domaines personnalisés](https://manage.auth0.com/#/tenant/custom_domains) pour rassembler les paramètres de distribution, d’en-tête personnalisé d’origine, de nom d’hôte d’origine et de comportement de cache par défaut. Ensuite, utilisez ces informations pour mettre à jour les paramètres de votre domaine personnalisé Akamai afin de pointer vers le domaine du serveur proxy inverse. Pour en savoir plus, consultez [Configurer le proxy inverse](https://auth0.com/docs/customize/custom-domains/self-managed-certificates#configure-reverse-proxy).
3. Mettez à jour l’en-tête d’IP client du domaine personnalisé à l’aide de Management API avec le point de terminaison API PATCH `/api/v2/custom-domains/{id}` comme suit :

   ```json lines
   { 
   "custom_client_ip_header": "true-client-ip" 
   }
   ```

   

   

   Pour en savoir plus, lisez [Mettre à jour la configuration du domaine personnalisé](https://auth0.com/docs/api/management/v2#!/Custom_Domains/patch_custom_domains_by_id).

Pour des détails sur la façon de configurer Akamai en tant que proxy inverse en utilisant les informations ci-dessus, reportez-vous à la documentation d’Akamai ou à l’assistance client.
