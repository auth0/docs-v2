<Card title="La disponibilité varie selon le plan Auth0">

L’implémentation propre à votre connexion et votre plan Auth0 ou accord personnalisé que vous utilisez déterminent si cette fonctionnalité est disponible. Pour en savoir plus, lisez [Tarification](https://auth0.com/pricing).

</Card>

Vous pouvez configurer AWS CloudFront pour l'utiliser comme proxy inverse avec des noms de domaine personnalisés pour votre locataire Auth0.

1. Connectez-vous à AWS et accédez à [CloudFront](https://console.aws.amazon.com/cloudfront).
2. Cliquez sur **Créer une distribution**.
3. Vous pouvez choisir la méthode de distribution pour votre contenu. Cliquez sur **Commencer** dans la section **Web**.
4. Configure your distribution settings. Here are the values you’ll need to change.

   <table class="table"><thead>
   <tr>
   <th>Paramètre</th>
   <th>Valeur</th>
   </tr>
   </thead>
   <tbody>
   <tr>
   <td>Nom du domaine d’origine</td>
   <td>Définissez ceci sur la valeur du <strong>Nom du domaine d’origine</strong> obtenue à partir du Auth0 Dashboard lors du processus de configuration des domaines personnalisés</td>
   </tr>
   <tr>
   <td>ID d’origine</td>
   <td>Une description de l’origine. Cette valeur vous permet de distinguer plusieurs origines dans la même distribution et doit donc être unique.</td>
   </tr>
   <tr>
   <td>Politique de protocole d’origine</td>
   <td>Définir sur `HTTPS Only`</td>
   </tr>
   <tr>
   <td>Noms de domaine alternatifs (CNAMEs)</td>
   <td>Définir sur le nom de domaine personnalisé (celui que vous avez configuré dans Auth0 Dashboard)</td>
   </tr>
   <tr>
   <td>Certificat SSL</td>
   <td>Définir sur le certificat SSL pour votre domaine personnalisé stocké dans AWS Certificate Manager (ACM) dans la région USA Est (Virginie du Nord) ou dans IAM.</td>
   </tr>
   </tbody>
   </table>
5. Provide information on the **Origin Custom Headers** (the **Header Name** and **Value** fields appear only after you’ve provided an **Origin Domain Name**)

   <table class="table"><thead>
   <tr>
   <th>Paramètre</th>
   <th>Valeur</th>
   </tr>
   </thead>
   <tbody>
   <tr>
   <td>Nom de l’en-tête</td>
   <td>Défini sur `cname-api-key`</td>
   </tr>
   <tr>
   <td>Valeur</td>
   <td>Définie sur la valeur de clé API CNAME que vous avez reçue immédiatement suite à la vérification de propriété de votre nom de domaine avec Auth0</td>
   </tr>
   </tbody>
   </table>
6. Configure the **Default Cache Behavior Settings**. Here are the values you’ll need to change

   <table class="table"><thead>
   <tr>
   <th>Paramètre</th>
   <th>Valeur</th>
   </tr>
   </thead>
   <tbody>
   <tr>
   <td>Politique de protocole du visualiseur</td>
   <td>Sélectionnez <strong>Rediriger HTTP vers HTTPS</strong></td>
   </tr>
   <tr>
   <td>Méthodes HTTP autorisées</td>
   <td>Sélectionnez <strong>GET, HEAD, OPTIONS, PUT, POST, PATCH, DELETE</strong></td>
   </tr>
   <tr>
   <td>Cache et paramètres de requête d’origine</td>
   <td>Sélectionnez <strong>Paramètres de cache hérités</strong></td>
   </tr>
   <tr>
   <td>Cache basé sur les en-têtes de requête sélectionnés</td>
   <td>Sélectionnez <strong>Inclure les en-têtes suivants</strong></td>
   </tr>
   <tr>
   <td>Ajoutez des en-têtes</td>
   <td>Entrez `User-Agent` et cliquez sur <strong>Ajout personnalisé &gt;&gt;</strong> pour ajouter l’en-tête personnalisée. Faites de même pour les en-têtes `Authorization`, `Origin`, `Referer`, `Accept-Language`, et `Accept`.</td>
   </tr>
   <tr>
   <td>Transférer les cookies</td>
   <td>Sélectionner <strong>Tous</strong></td>
   </tr>
   <tr>
   <td>Chaînes de requête</td>
   <td>Sélectionner <strong>Tous</strong></td>
   </tr>
   </tbody>
   </table>
7. Scroll to the bottom of the page and click **Create Distribution**.
   You’ll see your newly-created distribution in your CloudFront Distributions list. Note that the Status will reflect `In progress` until the distribution is Deployed.
8. Ajoutez un nouvel enregistrement CNAME à votre DNS pour votre domaine personnalisé pointant vers le nom de domaine CloudFront de votre distribution. Vous pouvez le trouver en cliquant sur votre ID de distribution, sous l'onglet Général, Nom de domaine (par exemple, `e2zwy42nt1feu7.cloudfront.net`).
