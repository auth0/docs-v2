Les déclencheurs sont des points dans le processus d’Auth0 où des actions peuvent être ajoutées pour répondre aux demandes de personnalisation. Plusieurs actions, exécutées les unes après les autres, peuvent être associées à un déclencheur. Les déclencheurs peuvent être synchrones (bloquant) ou asynchrones (non bloquant).
Chaque déclencheur dispose d’objets et d’API spécifiques. Ils peuvent varier selon votre mode de connexion (par ex. sans mot de passe).

Pour explorer les objets de référence et les plans pour des Actions spécifiques pour un déclencheur, sélectionnez le déclencheur ci-dessous, en notant la définition Auth0 d’une [connexion sans mot de passe](/docs/fr-ca/authenticate/passwordless).

#### Inscription et connexion

Lorsqu’un utilisateur est ajouté à une base de données ou une connexion sans mot de passe (Inscription), ou dans le cadre d’un processus d’authentification (Connexion).

<table class="table"><thead>
<tr>
<th>Déclencheur</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>`pre-user-registration`</td>
<td>Se déclenche avant que l’utilisateur soit créé.</td>
</tr>
<tr>
<td>`post-user-registration`</td>
<td>Se déclenche de manière asynchrone après la création d’un utilisateur.</td>
</tr>
<tr>
<td>`post-login`</td>
<td>Se déclenche après l’authentification de l’utilisateur, mais avant l’émission d’un jeton.</td>
</tr>
</tbody>
</table>
