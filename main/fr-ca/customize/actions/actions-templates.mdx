La galerie de modèles d’Actions offre une grande variété de modèles de départ pour créer une Action. Ces modèles fournissent des exemples de code couvrant divers déclencheurs et cas d’utilisation, tels que l’authentification multifacteur (<Tooltip href="/docs/fr-ca/glossary?term=multifactor-authentication" tip="Authentification multifacteur (MFA)
Processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, tel qu’un code par SMS." cta="Voir le glossaire">MFA</Tooltip>), le contrôle d’accès et l’enrichissement de profil.

Les modèles fournissent un code de départ que vous pouvez modifier et utiliser comme bloc de construction dans le cadre d’un déclencheur. Une fois que vous avez sélectionné un modèle pour votre cas d’utilisation spécifique, vous pouvez personnaliser le code pour répondre aux besoins individuels de votre application.

Pour un guide étape par étape sur l’utilisation des modèles d’Actions, consultez [Programmer votre première Action](/docs/fr-ca/customize/actions/write-your-first-action).

## Meilleures pratiques

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

Les Modèles d’action sont conçus exclusivement dans des buts éducatifs.

Ces modèles sont constitués de parties de code incomplètes, et ils ne devraient en aucune circonstance être utilisés en environnement de production. Les développeurs sont responsables du développement sécurisé et de compléter le Modèle d’action avant de déployer un environnement de production.

</Callout>

* Assurez-vous que les modèles sont adaptés à votre environnement spécifique. Le code étant incomplet, vous devez personnaliser et compléter les exemples avant de les importer dans un environnement de production.
* Ne déployez pas de modèle sans l’avoir modifié et complété au préalable.
* Les modèles d’Actions contiennent souvent des balises dans les commentaires, qui ne doivent pas être modifiées ou supprimées.
* Utilisez toujours la dernière version d’un package `npm`.
* Les modèles d’Actions ciblent par défaut les dernières versions de Node.js. Après avoir créé un modèle, vous pouvez le modifier pour prendre en charge d’autres versions de Node.js, si nécessaire.

Veuillez également consulter toutes les [Directives de codage des actions](/docs/fr-ca/customize/actions/action-coding-guidelines), car elles s’appliquent également ici.
