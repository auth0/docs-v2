Pour profiter des fonctionnalités fournies par Auth0 Actions, vous pouvez devoir migrer votre code actuel depuis des Règles, des crochets ou des versions antérieures d’Actions. Cependant, vous devez être conscient des limitations actuelles lors de la migration entre les offres d’extensibilité héritées et les Actions.

Pour plus d’informations, consulter [Migrer vers les Actions](/docs/fr-ca/customize/actions/migrate).

## Différences spécifiques aux règles

* La réutilisation des fonctions entre les Actions n’est pas prise en charge actuellement, car chaque exécution d’Action est autonome. Vous devez définir explicitement les fonctions requises pour chaque Action.
* L’objet [`accessToken`](/docs/fr-ca/libraries/lock/lock-api-reference#getuserinfo-) est actuellement limité à 100 permissions.
* L’objet événement dans les Actions ne fournit pas actuellement d’alternative pour les [attributs de contexte](https://auth0.com/docs/customize/rules/context-object)suivants à l’intérieur d’une règle :

  + `context.connectionOptions`
* Les actions ne prennent en charge que les propriétés suivantes dans [`event.user.identities`](/docs/fr-ca/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-event-object) :

  + `connection`
  + `provider`
  + `user_id`
  + `profileData`
  + `isSocial`

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

Si vous utilisez des propriétés supplémentaires dans l’objet `event.user.identities` dans les Règles, elles ne sont pas prises en charge dans une Action. Cela inclut tout jeton provenant de l’IdP en amont pour les utilisateurs fédérés.

</Callout>

* Les demandes standard d’un `idToken` ou d’un `accessToken` ne peuvent pas être supprimées à l’aide d’une Action.
* Les Actions ne peuvent renvoyer que des codes d’erreur `access denied`et ne permettent pas de renvoyer un code d’erreur `unauthorized`.
* Les Actions permettent uniquement de modifier l’utilisateur principal entre les comptes liés.

## Différences spécifiques aux crochets

* Vous ne pouvez pas modifier les permissions dans un flux des identifiants client de communication entre machines comme il était possible de le faire avec un crochet.
