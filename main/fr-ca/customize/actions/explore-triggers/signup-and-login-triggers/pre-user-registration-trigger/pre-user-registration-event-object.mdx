L’objet `event` pour le déclencheur d’actions de pré-enregistrement de l’utilisateur fournit des informations contextuelles sur la demande d’enregistrement d’un nouvel utilisateur.

<table class="table">
<thead>
<tr>
<th>Propriété</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>
<p>`event.client`</p>
<p><em>(Facultatif)</em></p>
</td>
<td>
<p>
          Informations sur le client avec lequel la translation a
          été initiée.
        </p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`client_id` <em>Chaîne</em>.
            <span>L’ID client de l’application à laquelle l’utilisateur se connecte.</span>
</li>
<li>
`metadata` <em>Dictionnaire</em>.
            <span>Un objet qui contient les autres propriétés d’une application.</span>
</li>
<li>
`name` <em>Chaîne</em>.
            <span>Le nom de l’application (comme défini dans le Dashboard).</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.connection`</p>
</td>
<td>
<p>Détails à propos de la connexion utilisée pour enregistrer l’utilisateur.</p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`id` <em>Chaîne</em>.
            <span>L’identifiant unique de la connexion.</span>
</li>
<li>
`metadata` <em>Dictionnaire facultatif</em>.
            <span>Métadonnées associées à la connexion.</span>
</li>
<li>
`name` <em>Chaîne</em>.
            <span>Le nom de la connexion utilisée pour authentifier l’utilisateur (par ex. :
              `twitter` ou `some-g-suite-domain`).</span>
</li>
<li>
`strategy` <em>Chaîne</em>.
            <span>Le type de connexion. Pour les connexions sociales,
              `event.connection.strategy === event.connection.name`.
              Pour les connexions d’entreprise, la stratégie est la suivante :
              `waad` (Windows Entra ID), `ad` (Active
              Directory/LDAP), `auth0` (connexions de base de données),
              etc.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.request`</p>
</td>
<td>
<p>Détails sur la demande à l’origine de la transaction.</p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`body` <em>Dictionnaire</em>.
            <span>Le corps de la requête POST. Ces données ne seront disponibles
              que pendant les flux d’échange de jetons d’actualisation et de flux d’échange d’identifiants clients,
              ainsi que lors de l’action PreUserRegistration.</span>
</li>
<li>
`geoip` <em>Objet</em>.
            <p>Comprend les propriétés suivantes :</p>
<ul>
<li>`cityName` <em>Chaîne facultative</em>.</li>
<li>`continentCode` <em>Chaîne facultative</em>.</li>
<li>`countryCode` <em>Chaîne facultative</em>.</li>
<li>`countryCode3` <em>Chaîne facultative</em>.</li>
<li>`countryName` <em>Chaîne facultative</em>.</li>
<li>`latitude` <em>Numéro facultatif</em>.</li>
<li>`longitude` <em>Numéro facultatif</em>.</li>
<li>`subdivisionCode` <em>Chaîne facultative</em>.</li>
<li>`subdivisionName` <em>Chaîne facultative</em>.</li>
<li>`timeZone` <em>Chaîne facultative</em>.</li>
</ul>
</li>
<li>
`hostname` <em>Chaîne facultative</em>.
            <span>Le nom d’hôte utilisé pour le flux
              d’authentification.</span>
</li>
<li>
`ip` <em>Chaîne</em>.
            <span>L’adresse IP initiale de la demande.</span>
</li>
<li>
`language` <em>Chaîne facultative</em>.
            <span>La langue demandée par le navigateur.</span>
</li>
<li>
`method` <em>Chaîne</em>.
            <span>La méthode HTTP utilisée pour la demande.</span>
</li>
<li>
`user_agent` <em>Chaîne facultative</em>.
            <span>La valeur de l’en-tête `User-Agent` reçu lorsque de la transaction
              a été initiée.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.tenant`</p>
</td>
<td>
<p>Détails sur le locataire associé à la transaction en cours.</p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`id` <em>Chaîne</em>.
            <span>Nom du locataire.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.transaction`</p>
<p><em>(Facultatif)</em></p>
</td>
<td>
<p>Détails à propos de la transaction en cours.</p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`acr_values` <em>Tableau de chaînes</em>.
            <span>Toutes les acr_values fournies dans la demande
              d’authentification initiale.</span>
</li>
<li>
`locale` <em>Chaîne</em>.
            <span>La langue locale à utiliser pour cette transaction, déterminée en
              comparant les langues demandées par le navigateur aux paramètres
              linguistiques du locataire.</span>
</li>
<li>
`login_hint` <em>Chaîne facultative</em>.
            <span>Indication au serveur d’autorisation de l’identifiant de connexion que
              l’utilisateur final pourrait utiliser pour se connecter (si nécessaire).</span>
</li>
<li>
`prompt` <em>Tableau de chaînes facultatif</em>.
            <span>Liste d’instructions indiquant si l’utilisateur peut être invité à s’authentifier
              à nouveau et à donner son consentement.</span>
</li>
<li>
`protocol` <em>Chaîne facultative</em>.
            <p>Les valeurs possibles sont les suivantes :</p>
<ul>
<li>
`oidc-basic-profile`
<span>Connexion Web la plus fréquemment utilisée.</span>
</li>
<li>
`oidc-implicit-profile`
<span>Utilisée sur les appareils mobiles et les applications à page unique.</span>
</li>
<li>
`samlp` <span>Protocole SAML utilisé pour les applications SaaS.</span>
</li>
<li>
`wsfed`
<span>Protocole WS-Federation utilisé sur les produits Microsoft comme
                  Office365.</span>
</li>
<li>
`wstrust-usernamemixed`
<span>Identifiants utilisateur/mot de passe WS-trust utilisés sur CRM et Office365.</span>
</li>
<li>
`oauth2-webauthn`
<span>Inscription par le biais de l’API Clés d’identification natives.</span>
</li>
<li>
`oauth2-device-code`
<span>Transaction utilisant le flux d’autorisation de l’appareil.</span>
</li>
<li>
`oauth2-resource-owner`
<span>Identifiants utilisateur/mot de passe généralement utilisés pour les connexions
                  aux bases de données.</span>
</li>
<li>
`oauth2-resource-owner-jwt-bearer`

</li>
<li>
`oauth2-password`
<span>Connexion utilisant l’échange de mots de passe.</span>
</li>
<li>
`oauth2-access-token`
<span>Actualisation d’un jeton à l’aide de l’échange de jetons d’actualisation.</span>
</li>
<li>
`oauth2-refresh-token`
<span>Actualisation d’un jeton à l’aide de l’échange de jetons d’actualisation.</span>
</li>
<li>`oauth2-token-exchange`</li>
<li>
`oidc-hybrid-profile`
<span>Permet à votre application d’avoir un accès immédiat à un jeton d’ID
                  tout en assurant une récupération sûre et sécurisée des
                  jetons d’accès et d’actualisation.</span>
</li>
</ul>
</li>
<li>
`redirect_uri` <em>Chaîne facultative</em>.
            <span>L’URL vers laquelle Auth0 redirigera le navigateur une fois
              la transaction terminée.</span>
</li>
<li>
`requested_scopes` <em>Tableau de chaînes</em>.
            <span>Les permissions demandées (le cas échéant) lors du lancement de ce
              flux d’authentification.</span>
</li>
<li>
`response_mode` <em>Chaîne facultative</em>.
            <p>
              Informe le serveur d’autorisation du mécanisme à utiliser pour
              renvoyer les paramètres à partir du point de terminaison d’autorisation.
            </p>
<p>Les valeurs possibles sont les suivantes :</p>
<ul>
<li>`query`</li>
<li>`fragment`</li>
<li>`form_post`</li>
<li>`web_message`</li>
</ul>
</li>
<li>
`response_type` <em>Tableau de chaînes facultatif</em>.
            <p>Les valeurs possibles sont les suivantes :</p>
<ul>
<li>`code`</li>
<li>`token`</li>
<li>`id_token`</li>
</ul>
</li>
<li>
`state` <em>Chaîne facultative</em>.
            <span>Une chaîne alphanumérique arbitraire et opaque que votre application ajoute à la
              requête initiale incluse par Auth0 lors de la redirection vers votre
              application.</span>
</li>
<li>
`ui_locales` <em>Tableau de chaînes</em>.
            <span>Les ui_locales fournies dans la demande d’authentification
              initiale.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.user`</p>
</td>
<td>
<p>Objet décrivant l’utilisateur qui tente de s’enregistrer.</p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`app_metadata` <em>Dictionnaire facultatif</em>.
            <span>Champs personnalisés qui stockent des informations sur un utilisateur qui
              influencent l’accès de l’utilisateur, comme le plan de soutien, les rôles de sécurité ou les groupes
              de contrôle d’accès.</span>
</li>
<li>
`email` <em>Chaîne facultative</em>.
            <span>(unique) Adresse de courriel de l’utilisateur.</span>
</li>
<li>
`family_name` <em>Chaîne facultative</em>.
            <span>Nom de famille de l’utilisateur.</span>
</li>
<li>
`given_name` <em>Chaîne facultative</em>.
            <span>Prénom de l’utilisateur.</span>
</li>
<li>
`name` <em>Chaîne facultative</em>.
            <span>Nom complet de l’utilisateur.</span>
</li>
<li>
`nickname` <em>Chaîne facultative</em>.
            <span>Pseudonyme de l’utilisateur.</span>
</li>
<li>
`phone_number` <em>Chaîne facultative</em>.
            <span>Numéro de téléphone de l’utilisateur.</span>
</li>
<li>
`picture` <em>Chaîne facultative</em>.
            <span>URL pointant vers la
              <a href="https://auth0.com/docs/users/change-user-picture">photo de profil de l’utilisateur</a>.</span>
</li>
<li>
`user_metadata` <em>Dictionnaire facultatif</em>.
            <span>Champs personnalisés qui stockent des informations sur un utilisateur qui n’ont pas d’incidence
              sur les données auxquelles il peut ou ne peut pas accéder, comme l’adresse professionnelle,
              l’adresse personnelle ou les préférences de l’utilisateur.</span>
</li>
<li>
`username` <em>Chaîne facultative</em>.
            <span>(unique) Nom d’utilisateur</span>
</li>
</ul>
</td>
</tr>
