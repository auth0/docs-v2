L’objet `event` pour le déclencheur d’actions post-change-password fournit des informations contextuelles sur l’utilisateur nouvellement créé.

<table class="table">
<thead>
<tr>
<th>Propriété</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>
<p>`event.connection`</p>
</td>
<td>
<p>
          Information sur la connexion utilisée pour la transaction
          en cours.
        </p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`id` <em>Chaîne</em>.
            <span>L’identifiant unique de la connexion.</span>
</li>
<li>
`metadata` <em>Dictionnaire optionnel</em>.
            <span>Métadonnées associées à la connexion.</span>
</li>
<li>
`name` <em>Chaîne</em>.
            <span>Le nom de la connexion utilisée pour authentifier l’utilisateur (tel que :
              `twitter` ou `some-g-suite-domain`).</span>
</li>
<li>
`strategy` <em>Chaîne</em>.
            <span>Le type de connexion. Pour les connexions sociales,
              `event.connection.strategy === event.connection.name`.
              Pour les connexions d’entreprise, la stratégie est la suivante :
              `waad` (Windows Entra ID), `ad` (Active
              Directory/LDAP), `auth0` (connexions aux bases de données),
              etc.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.request`</p>
</td>
<td>
<p>Détails sur la requête qui a initié la transaction.</p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`geoip` <em>Objet</em>.<span>Contient des informations géographiques sur la requête.</span>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>`cityName` <em>Chaîne facultative</em>.</li>
<li>`continentCode` <em>Chaîne facultative</em>.</li>
<li>`countryCode` <em>Chaîne facultative</em>.</li>
<li>`countryCode3` <em>Chaîne facultative</em>.</li>
<li>`countryName` <em>Chaîne facultative</em>.</li>
<li>`latitude` <em>Numéro facultatif</em>.</li>
<li>`longitude` <em>Numéro facultatif</em>.</li>
<li>`subdivisionCode` <em>Chaîne facultative</em>.</li>
<li>`subdivisionName` <em>Chaîne facultative</em>.</li>
<li>`timeZone` <em>Chaîne facultative</em>.</li>
</ul>
</li>
<li>
`hostname` <em>Chaîne facultative</em>.
            <span>Le nom d’hôte utilisé pour le flux
              d’authentification.</span>
</li>
<li>
`ip` <em>Chaîne</em>.
            <span>L’adresse IP d’origine de la requête.</span>
</li>
<li>
`language` <em>Chaîne facultative</em>.
            <span>La langue demandée par le navigateur.</span>
</li>
<li>
`method` <em>Chaîne</em>.
            <span>La méthode HTTP utilisée pour la requête</span>
</li>
<li>
`user_agent` <em>Chaîne facultative</em>.
            <span>La valeur de l’en-tête `User-Agent` reçu lors de l’initiation
              de la transaction.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.tenant`</p>
</td>
<td>
<p>Détails sur le locataire associé à la transaction en cours.</p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`id` <em>Chaîne</em>.
            <span>Le nom du locataire.</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.user`</p>
</td>
<td>
<p>
          Objet décrivant l’utilisateur au nom duquel la transaction en cours
          a été initiée.
        </p>
<p>Comprend les propriétés suivantes :</p>
<ul>
<li>
`email` <em>Chaîne facultative</em>.
            <span>(unique) Adresse courriel de l’utilisateur.</span>
</li>
<li>
`email_verified` <em>Booléen facultatif</em>.
            <span>Indique si l’utilisateur a vérifié son adresse
              courriel.</span>
</li>
<li>
`last_password_reset` <em>Chaîne facultative</em>.
            <span>Horodatage indiquant la dernière fois que le mot de passe de l’utilisateur a été
              réinitialisé/modifié. Lors de la création de l’utilisateur, ce champ n’existe pas. Cette
              propriété n’est disponible que pour les connexions aux bases de données.</span>
</li>
<li>
`phone_number` <em>Chaîne facultative</em>.
            <span>(unique) Numéro de téléphone de l’utilisateur.</span>
</li>
<li>
`phone_verified` <em>Booléen facultatif</em>.
            <span>Indique si l’utilisateur a vérifié son numéro de téléphone.</span>
</li>
<li>
`user_id` <em>Chaîne facultative</em>.
            <span>(unique) Identifiant unique de l’utilisateur.</span>
</li>
<li>
`username` <em>Chaîne facultative</em>.
            <span>(unique) Nom d’utilisateur de l’utilisateur.</span>
</li>
</ul>
</td>
</tr>
</tbody>
</table>
