## Prérequis

Avant de personnaliser vos modèles de téléphone, configurez votre fournisseur de téléphonie à l’aide d’un service tiers comme Twilio. Pour en savoir plus, veuillez consulter [Configurer les fournisseurs de téléphonie](/docs/fr-ca/customize/phone-messages/configure-phone-messaging-providers).

## Débuter

Accédez à [Auth0 Dashboard](https://manage.auth0.com/#/phone). Sous **Branding (Image de marque) > Phone Templates (Modèles de téléphone)**, Auth0 propose les modèles de téléphone personnalisables suivants :

* **Blocked Account (Compte bloqué)** : alerte le client que son compte a été bloqué à la suite d’une activité suspecte.
* **Change Password (Modifier le mot de passe)** : envoie au client son code de changement de mot de passe.
* **One Time Password - Enrollment (Mot de passe à usage unique – Inscription)** : envoie au client le code de vérification pour l’inscription au mot de passe à usage unique (OTP).
* **One Time Password - Verify (Mot de passe à usage unique – Vérifier** : envoie au client le code de vérification pour la vérification du mot de passe à usage unique (OTP).

Une fois que vous avez sélectionné un modèle de téléphone, activez-le en basculant son **Status (État)**. Vous pouvez désormais personnaliser le numéro de téléphone **From (De)** et le **Text Message Template (Modèle de message SMS)**.

## Numéro de téléphone « De »

La valeur par défaut du numéro de téléphone **From (De)** est celle que vous avez définie lors de la configuration de votre fournisseur de téléphonie. Pour en savoir plus, veuillez consulter [Configurer les fournisseurs de téléphonie](/docs/fr-ca/customize/phone-messages/configure-phone-messaging-providers).

Pour définir un numéro de téléphone personnalisé **From (De)** pour un modèle de téléphone, saisissez le numéro de téléphone à partir duquel les clients recevront des messages.

**Remarque :** Si vous avez configuré Twilio comme fournisseur de téléphonie et sélectionné [Messaging Services (Services de messagerie)](https://www.twilio.com/docs/messaging/services) comme source SMS, vous ne pourrez pas définir un paramètre de numéro de téléphone personnalisé **From (De)**. Twilio sélectionne à la place l’un des numéros de téléphone de l’ensemble des expéditeurs associés à votre service de messagerie, en fonction de votre configuration Twilio. Pour en savoir plus, veuillez consulter la documentation de Twilio portant sur les [Messaging Services (Services de messagerie)](https://www.twilio.com/docs/messaging/services).

## Modèle de message SMS

Dans le corps du **Text Message Template (Modèle de message SMS)**, Auth0 vous permet de générer dynamiquement du contenu à l’aide du langage de création de modèles Liquid. À l’aide de la syntaxe Liquid, vous pouvez utiliser un certain nombre de variables contextuelles qu’Auth0 remplace par les valeurs appropriées lors de la génération des messages téléphoniques. Pour en savoir plus sur Liquid, veuillez consulter [Liquid pour les designers](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) sur GitHub.

Auth0 met à disposition le même ensemble de [variables contextuelles](/docs/fr-ca/customize/email/email-templates#common-variables), accessibles via la syntaxe Liquid, aussi bien pour les modèles de messages téléphoniques que pour ceux de courriels. Les règles de formatage pour personnaliser les modèles de messages téléphoniques avec la syntaxe Liquid sont identiques à celles utilisées pour personnaliser les modèles de courriels. Pour savoir comment utiliser la syntaxe Liquid pour personnaliser un modèle de téléphone, veuillez consulter [Utiliser la syntaxe Liquid dans les modèles de courriels](/docs/fr-ca/customize/email/email-templates/use-liquid-syntax-in-email-templates).

## Modèle de téléphone de test

Une fois que vous avez terminé de personnaliser votre modèle de téléphone, cliquez sur le bouton **Try (Essayer)**. Auth0 envoie le message téléphonique à une application par défaut, dont le nom correspond à celui de votre locataire. Pour tester les modèles de différentes applications, créez un exemple d’utilisateur et passez en revue les flux de messages téléphoniques.
