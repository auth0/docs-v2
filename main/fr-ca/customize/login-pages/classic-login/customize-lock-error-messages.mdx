Vous pouvez personnaliser les messages d’erreur qui seront affichés dans certaines situations en fournissant une option `languageDictionary`. Une liste complète des champs `languageDictionary` disponibles qui peuvent être personnalisés se trouve dans le repertoire GitHub [Dictionnaire anglais pour Lock](https://github.com/auth0/lock/blob/master/src/i18n/en.js). Voici un exemple de messages d’erreur personnalisés :



```javascript lines
// Examples of customized error messages in the languageDictionary option
var options = {
  languageDictionary: {
    error: {
      login: {
        "lock.invalid_email_password": "Custom message about invalid credentials",
        "lock.network": "Custom message indicating a network error and suggesting the user check connection",
        "lock.unauthorized": "Custom message about a failure of permissions",
        "too_many_attempts": "Custom message indicating the user has failed to login too many times."
      },
      signUp: {
        "invalid_password": "Custom message indicating a password was invalid",
        "user_exists": "Custom message indicating that a user already exists"
      }
    }
  }
};

// Initiating our Auth0Lock
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);
```

## Erreurs personnalisées dans les Règles

Si vous renvoyez des codes d’erreur personnalisés à partir d’une [règle](/docs/fr-ca/customize/rules) ou d’un script de base de données personnalisé, vous pouvez gérer les erreurs personnalisées :

* Dans l’URL de redirection de votre application, en lisant les paramètres de chaîne de requête `error` et `error_mesage`.
* En redirigeant l’utilisateur vers vos pages hébergées avec un message d’erreur personnalisé et en affichant le message avec un [message flash](/docs/fr-ca/libraries/lock/lock-api-reference).
