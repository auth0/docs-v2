Voici le scénario : Après l’authentification de l’utilisateur, il est nécessaire de stocker ses informations d’identification et de les utiliser tant qu’elles restent valides. Lorsque ces informations expirent, il est recommandé de les renouveler à l’aide du `refreshToken`, afin d’éviter de rediriger l’utilisateur de nouveau, vers la page de connexion. Au lieu de procéder automatiquement, vous demanderez à l’utilisateur de valider son identité via son empreinte digitale ou la reconnaissance faciale.

Pour simplifier la gestion des identifiants utilisateurs et effectuer l’authentification biométrique, vous pourrez utiliser l’utilitaire [Gestionnaire des informations d’identification](https://github.com/auth0/Auth0.swift/blob/master/Auth0/CredentialsManager.swift) intégré dans [Auth0.swift](https://github.com/auth0/Auth0.swift/).

## Démarrage

Tout d’abord, importez le module `Auth0` :

`import Auth0`

### Gestionnaire des identifiants

Avant de récupérer vos identifiants, vous pouvez activer l’authentification biométrique (Face ID ou Touch ID) disponible sur votre appareil iOS.

Commencez par configurer le Gestionnaire des informations d’identification. Activez ensuite la biométrie. Vous pouvez également transmettre un titre à afficher dans l’invite.

```swift lines
let credentialsManager = CredentialsManager(authentication: Auth0.authentication())
credentialsManager.enableBiometrics(withTitle: "Touch ID / Face ID Login")
```

Nous vous recommandons fortement d’ajouter le paramètre [NSFaceIDUsageDescription](https://developer.apple.com/library/archive/documentation/General/Reference/InfoPlistKeyReference/Articles/CocoaKeys.html#//apple_ref/doc/uid/TP40009251-SW75) à votre projet `Info.plist` pour afficher une raison pour l’utilisation de Face ID. Dans certains cas, si vous ne fournissez pas de chaîne de description, une tentative d’authentification via Face ID peut échouer.

```swift lines
...
<key>NSFaceIDUsageDescription</key>
<string>Reason why we use Face ID here</string>
...
```

### Connexion

Affichez la page de connexion universelle. Une fois l’authentification réussie, transmettez les informations d’identification au Gestionnaire des informations d’identification.

```swift lines
Auth0
    .webAuth()
    .scope("openid profile offline_access")
    .audience("https://{yourDomain}/userinfo")
    .start {
        switch $0 {
        case .failure(let error):
            // Handle error
        case .success(let credentials):
            // Store credentials securely with the Credentials Manager
            credentialsManager.store(credentials: credentials)
        }
}
```

### Renouveler les informations d’identification de l’utilisateur

Lorsque vous devez renouveler les informations d’identification de l’utilisateur, vous pouvez appeler la méthode `credentials` du Gestionnaire d’informations d’identification.

```swift lines
credentialsManager.credentials { error, credentials in
    guard error == nil, let credentials = credentials else {
        // Handle error
        // Fallback to login screen
    }
    // Continue routing the user as authentication was successful
}
```

Il n’est pas nécessaire de stocker manuellement les nouvelles informations d’identification, car cette opération est gérée par le Gestionnaire d’informations d’identification lors du renouvellement.
