Chaque propriété de la clé est définie par la spécification JWK [RFC 7517 Section 4](https://tools.ietf.org/html/rfc7517#section-4) ou, pour des propriétés propres aux algorithmes, dans [RFC 7518](https://tools.ietf.org/html/rfc7518).

<table class="table"><thead>
<tr>
<th>Nom de la propriété</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>`alg`</td>
<td>L’algorithme cryptographique spécifique utilisé avec la clé. Il s’agit d’un paramètre facultatif. Par défaut, Auth0 inclut l’algorithme de signature défini au niveau du locataire dans l’ensemble de clés Web JSON (JWKS), qui est ensuite publié. Pour permettre l’utilisation de clés avec plusieurs algorithmes plutôt qu’avec un seul algorithme, c’est-à-dire RS256, désactivez <strong>Include Signing Algorithms in JSON Web Key Set</strong> sous Paramètres avancés du locataire dans le tableau de bord. Cela supprime le paramètre `alg` et oblige les consommateurs du JWKS à interpréter les algorithmes de signature selon les besoins.</td>
</tr>
<tr>
<td>`kty`</td>
<td>La famille d’algorithmes cryptographiques utilisés avec la clé.</td>
</tr>
<tr>
<td>`use`</td>
<td>Comment la clé était censée être utilisée; `sig` représente la signature.</td>
</tr>
<tr>
<td>`x5c`</td>
<td>La chaîne de certificat x.509. La première entrée du tableau est le certificat à utiliser pour la vérification du jeton; les autres certificats peuvent être utilisés pour vérifier ce premier certificat.</td>
</tr>
<tr>
<td>`n`</td>
<td>Les modules pour la <a href="https://tools.ietf.org/html/rfc7518#page-30">Clé publique RSA</a>.</td>
</tr>
<tr>
<td>`e`</td>
<td>L’exposant pour le <a href="https://tools.ietf.org/html/rfc7518#page-30">Clé publique RSA</a>.</td>
</tr>
<tr>
<td>`kid`</td>
<td>L’identifiant unique de la clé.</td>
</tr>
<tr>
<td>`x5t`</td>
<td>L’empreinte numérique du certificat x.509 (empreinte numérique SHA-1).</td>
</tr>
</tbody>
</table>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

Auth0 ne prend en charge que HMAC et RSA; il ne prend pas actuellement en charge Elliptic Curve (chiffrement par courbe elliptique ECDSA).

</Callout>

Vous trouverez un exemple qui utilise l’ensemble de clés Web JSON pour vérifier une signature de jeton Web JSON dans l’article [Utiliser RS256 et l’ensemble de clés Web JSON](https://auth0.com/blog/navigating-rs256-and-jwks/) (à l’aide de Node.js). Vous pouvez aussi consulter nos [guides de démarrage rapide des API/Systèmes dorsaux](https://auth0.com/docs/quickstart/backend).
