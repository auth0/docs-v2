Vous pouvez activer l’authentification multifacteur (<Tooltip href="/docs/fr-ca/glossary?term=multifactor-authentication" tip="Authentification multifacteur (MFA)
Processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, tel qu’un code par SMS." cta="Voir le glossaire">MFA</Tooltip>) dans la section [Tableau de bord > Sécurité> Authentification multifacteur (MFA)](https://manage.auth0.com/#/security/mfa). Vous pouvez personnaliser les pages MFA en utilisant les options d’image de marque de base d’Auth0 Dashboard. Vous pouvez personnaliser davantage la page MFA en y ajoutant votre propre code HTML. La connexion classique utilise le gadget logiciel MFA d’Auth0 avec les limites suivantes lorsque vous utilisez votre propre code HTML :

* La MFA par courriel n’est pas prise en charge.
* Les utilisateurs inscrits à plus d’un facteur ne peuvent pas choisir celui qu’ils souhaitent utiliser. Le gadget logiciel MFA leur demandera de se connecter avec le facteur le plus sûr.
* Les fonctionnalités d’internationalisation de la connexion universelle ne sont pas utilisées. Pour en savoir plus, consultez [Internationalisation de la connexion universelle](/docs/fr-ca/customize/internationalization-and-localization/universal-login-internationalization).

## Personnaliser le code HTML de la page MFA

Pour personnaliser la page MFA, rendez-vous à l’onglet [Dashboard > Image de marque > Connexion universelle > Authentification multifacteur (MFA)](https://manage.auth0.com/#/mfa_page) et activez l’option **Personnaliser la page MFA**. Vous pouvez ensuite utiliser l’éditeur de texte pour modifier votre code HTML, donner du style à votre page à l’aide des feuilles de style CSS et modifier le code JavaScript employé pour récupérer les variables personnalisées. N’oubliez pas de cliquer sur **Enregistrer les modifications** une fois que vous avez effectué vos modifications.

Les pages MFA fonctionnent sans personnalisation et Auth0 mettra à jour les scripts inclus si nécessaire. Cependant, une fois que vous avez activé la personnalisation, vous êtes responsable de la mise à jour et de la maintenance du script (notamment de la modification des numéros de version, comme celui du gadget logiciel MFA). Auth0 ne peut plus le mettre à jour automatiquement.

Pour en savoir plus sur la façon de remplacer le texte dans de nombreux domaines du processus MFA pour la connexion classique, consultez [Dictionnaire des langues des thèmes MFA](/docs/fr-ca/secure/multi-factor-authentication/customize-mfa/mfa-theme-language-dictionary). Pour en savoir plus sur les options des thèmes du gadget logiciel MFA, consultez [Options des thèmes du gadget logiciel MFA](/docs/fr-ca/secure/multi-factor-authentication/customize-mfa/mfa-widget-theme-options).

## Réinitialiser la page

Deux options s’offrent à vous si vous souhaitez revenir à un modèle antérieur :

* Revenir au dernier modèle enregistré en cliquant sur **Reset to Last (Revenir au dernier)**.
* Revenir au modèle par défaut fourni par Auth0 en cliquant sur **Reset to Default (Revenir au modèle par défaut)**.

## Rendu des invitations à l’inscription

Il existe deux scénarios possibles pour rendre (afficher) la page. Le `ticket` de propriété sera affiché si un utilisateur a été dirigé vers cette page expressément pour s’inscrire (par exemple, à partir d’un courriel contenant un lien d’inscription). Sinon, la propriété `requestToken` sera affichée.

## HTML et syntaxe Liquid

La page hébergée utilise la syntaxe Liquid pour la création de modèles. Pour en savoir plus, consultez [Liquid pour les concepteurs](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) sur GitHub. Les paramètres suivants sont proposés pour faciliter le rendu de votre page :

* `userData.email`
* `userData.friendlyUserId`
* `userData.tenant`
* `userData.tenantFriendlyName`
* `iconUrl`

La plupart des paramètres employés dans le gadget logiciel MFA doivent être transmis à Guardian comme indiqué dans le modèle pars défaut fourni dans la zone de personnalisation. Pour une personnalisation plus poussée, vous pouvez utiliser [auth0-guardian.js](https://github.com/auth0/auth0-guardian.js/tree/master/example).

Pour en savoir plus, consultez [Personnaliser les pages utilisateur MFA](/docs/fr-ca/secure/multi-factor-authentication/customize-mfa).
