Vous pouvez gérer les inscriptions des utilisateurs en créant des billets d’inscription à l’aide du point de terminaison [`/Guardian/post_ticket`](https://auth0.com/docs/api/management/v2#!/Guardian/post_ticket) de l’API <Tooltip href="/docs/fr-ca/glossary?term=multifactor-authentication" tip="Authentification multifacteur (MFA)
Processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, tel qu’un code par SMS." cta="Voir le glossaire">MFA</Tooltip>. Cette API renverra un billet d’inscription contenant un `ticket_id` et une `ticket_url`, qui peuvent être utilisés pour inscrire un utilisateur. La `ticket_url` ne peut être utilisée qu’une seule fois. Les inscriptions suivantes doivent être effectuées par l’intermédiaire de l’API MFA.

Si vous utilisez la connexion universelle classique et que vous avez besoin de personnaliser l’apparence de la page lorsque l’utilisateur navigue vers la `ticket_url`,vous pouvez éditer la page MFA. Allez dans l’onglet [Dashboard > Marque > Connexion universelle> Authentification multifacteur (MFA)](https://manage.auth0.com/#/mfa_page), et modifiez la variable `ticket` Voici un exemple :

```html lines
{% if ticket %}
<h4 class="message">Welcome, {{ userData.email }}, enroll your device below</h4>
{% else %}
<h4 class="message">Welcome back, {{ userData.email }}, authenticate below</h4>
{% endif %}
```
