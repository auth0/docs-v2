Pour paginer les résultats de la recherche de l’utilisateur, utilisez les paramètres `page`, `per_page` et `include_totals` dans votre demande.

<table class="table"><thead>
<tr>
<th>Paramètre</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>`page`</td>
<td>Le numéro de page, basé sur zéro. Si ce paramètre n’est pas défini, nous renvoyons un maximum de 50 enregistrements, quel que soit le nombre d’enregistrements existants.</td>
</tr>
<tr>
<td>`per_page`</td>
<td>Le nombre d’utilisateurs par page.</td>
</tr>
<tr>
<td>`include_totals`</td>
<td>Défini à `true` pour inclure un résumé de la requête dans le résultat.</td>
</tr>
</tbody>
</table>

## Exemple de réponse

Un exemple de corps de réponse pour les valeurs définies dans l’exemple de demande ci-dessus est le suivant :

```json lines
{
    "start": 20,
    "limit": 10,
    "length": 10,
    "users": [
        {
            ...
        }
    ],
    "total": 79
}
```

## Limites

Auth0 limite le nombre total d’utilisateurs que vous pouvez récupérer à 1 000 (p. ex., 100 utilisateurs par page pour 10 pages). Si le paramètre `page` n’est pas défini, nous renvoyons un maximum de 50 enregistrements, quel que soit le nombre d’enregistrements existants.
