Les fonctionnalités Authorization Core d’Auth0 permettent un contrôle d'accès basé sur les rôles (RBAC) de vos API.

Pour que le RBAC fonctionne correctement, vous devez l’activer pour votre API à l’aide du Tableau de bord  ou de <Tooltip href="/docs/fr-ca/glossary?term=management-api" tip="Management API
Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire">Management API</Tooltip>. Pour en savoir plus, lisez [Activer le contrôle d'accès basé sur les rôles (RBAC) pour les API](/docs/fr-ca/get-started/apis/enable-role-based-access-control-for-apis).

La fonctionnalité Authorization Core est différente de Authorization Extension (Extension d’autorisation). Pour une comparaison entre les deux produits, lisez [Authorization Core par rapport à Authorization Extension](/docs/fr-ca/manage-users/access-control/authorization-core-vs-authorization-extension).

Pour utiliser la fonctionnalité de base le plus efficacement possible, vous devez procéder comme suit :

1. [Enregistrer l’API auprès d’Auth0](/docs/fr-ca/get-started/auth0-overview/set-up-apis)
2. [Définir les autorisations pour l’API](/docs/fr-ca/get-started/apis/add-api-permissions)
3. [Créer des rôles](/docs/fr-ca/manage-users/access-control/configure-core-rbac/roles/create-roles)
4. [Attribuer des rôles aux utilisateurs](/docs/fr-ca/manage-users/access-control/configure-core-rbac/rbac-users/assign-roles-to-users)
5. [Attribuer des autorisations aux utilisateurs](/docs/fr-ca/manage-users/access-control/configure-core-rbac/rbac-users/assign-permissions-to-users), le cas échéant.
