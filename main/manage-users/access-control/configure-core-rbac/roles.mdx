<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

The [Authorization Core](/manage-users/access-control/rbac) feature set and [Authorization Extension](/customize/extensions/authorization-extension) are completely separate features. To manage groups, roles, or permissions, you need to use the feature they were originally created in.

</Callout>

For role-based access control (RBAC) to work properly, you must enable it for your API using either the Dashboard or the <Tooltip tip="Management API: A product to allow customers to perform administrative tasks." cta="View Glossary" href="/glossary?term=Management+API">Management API</Tooltip>. The Authorization Core functionality is different from the Authorization Extension. For a comparison, read [Authorization Core vs. Authorization Extension](/manage-users/access-control/authorization-core-vs-authorization-extension).

We provide various functions to help you manage your roles, which you can access through either the <Tooltip tip="Auth0 Dashboard: Auth0's main product to configure your services." cta="View Glossary" href="/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> or the Auth0 Management API:

* [Create roles](/manage-users/access-control/configure-core-rbac/roles/create-roles)
* [Edit role definitions](/manage-users/access-control/configure-core-rbac/roles/edit-role-definitions)
* [Add permissions to roles](/manage-users/access-control/configure-core-rbac/roles/add-permissions-to-roles)
* [Remove permissions from roles](/manage-users/access-control/configure-core-rbac/roles/remove-permissions-from-roles)
* [View role permissions](/manage-users/access-control/configure-core-rbac/roles/view-role-permissions)
* [View role users](/manage-users/access-control/configure-core-rbac/roles/view-users-assigned-to-roles)
* [Delete roles](/manage-users/access-control/configure-core-rbac/roles/delete-roles)
