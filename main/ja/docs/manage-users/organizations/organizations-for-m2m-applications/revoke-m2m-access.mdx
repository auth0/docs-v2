title: M2M アクセスを取り消す
---

M2M アクセスを取り消す、またはアプリケーションの組織へのアクセスを削除するには、そのクライアントグラントと組織の関連付けを削除する必要があります。この関連付けを削除すると、アプリケーションは、そのグラントで定義された API の新しいトークンをリクエストする際に、その組織を使用できなくなります。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  M2M アクセスを取り消しても、いずれかの組織へのアクセスがすでに付与されているアプリケーションには影響しません。詳細は、[組織の動作を定義する](/ja/docs/manage-users/organizations/organizations-for-m2m-applications/configure-your-application-for-m2m-access#define-organization-behavior)を参照してください。
</Callout>

<Tabs>
  <Tab title="Auth0 Dashboard">
    [Auth0 Dashboard](https://manage.auth0.com/) でアプリケーションのクライアントグラントと組織の関連付けを削除するには、次の手順を実行します。

    1. **Organizations** に移動し、関連付けを削除する組織を選択します。
    2. **Machine-to-Machine Access** タブを選択します。
    3. アプリケーションを選択します。
    4. 組織との関連付けを解除する API の横にあるチェックボックスをオフにします。
    5. **Save** をクリックします。

    <Frame>![](/docs/images/cdy7uua7fh8z/5MFv5NAoBDswloVweCVU1D/9057b94d5c5fd35e25afc7170ac645f3/Acme_Bot_-_Travel_Api_-_English.png)</Frame>
  </Tab>

  <Tab title="Management API">
    アプリケーションのクライアントグラントと組織の関連付けは、[Disassociate client grants from organizations](https://auth0.com/docs/api/management/v2/organizations/delete-client-grants-by-grant-id) エンドポイントを使用して削除できます。

    ```bash lines
    curl -X DELETE --location "https://{yourDomain}/api/v2/organizations/{ORG_id}/client-grants/{GRANT_ID}" \
      --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' ;
    ```
  </Tab>
</Tabs>
