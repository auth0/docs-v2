<Card title="利用可否は Auth0 プランによって異なります">
  この機能を利用できるかどうかは、ご利用のログイン実装と、Auth0 プランまたはカスタム契約の両方によって決まります。詳しくは、[Pricing](https://auth0.com/pricing)を参照してください。
</Card>

[組織](/ja/docs/manage-users/organizations/organizations-overview)にメンバーを追加したいものの、ユーザーがまだデータストアに存在しない場合 (またはデータストアには存在していても、まだ組織のメンバーではない場合) は、そのユーザーを組織に招待できます。この場合、ユーザーには、アカウントを作成するかログインして組織に参加するためのリンクを含むメールが送信されます。必要に応じて事前定義されたロールを付与することもでき、さらにアプリケーションにリダイレクトされるため、すぐに利用を開始できます。

<Warning>
  組織メンバーのロールは、Auth0 の[ロールベースアクセス制御](/ja/docs/manage-users/access-control/rbac)でユーザーに割り当てるロールとは別のものです。組織への招待を承諾したユーザーに事前定義されたロールを適用した場合、そのロールは、ユーザーがその組織経由でログインしているときにのみ有効です。Dashboard では、[組織メンバーのロールを確認](/ja/docs/manage-users/organizations/configure-organizations/retrieve-member-roles)できます。
</Warning>

招待フローを設定するには、次の手順を実行します。

1. [メンバーシップ招待フローをアプリケーションに統合する](#integrate-the-membership-invitation-flow-into-your-application)
2. [ユーザー招待フローをカスタマイズする](#customize-the-user-invitation-flow)
3. [メンバーシップ招待を送信する](#send-membership-invitations)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **制限事項**

  * Auth0 で送信できるユーザー招待は、メール経由のみです。別の方法で招待を送信したい場合は、招待 URL のみを生成することもできます。
  * 招待されたユーザーは、招待の送信先メールアドレスでログインするか、アカウントを作成する必要があります。
</Callout>

<div id="integrate-the-membership-invitation-flow-into-your-application">
  ## メンバーシップ招待フローをアプリケーションに統合する
</div>

ユーザー招待を機能させるには、招待受諾フローの一部としてアプリケーションにルートを設定し、テナントまたはアプリケーションの[関連するデフォルトのログインルートを設定](/ja/docs/authenticate/login/auth0-universal-login/configure-default-login-routes)する必要があります。設定した URI へのリンクは、ユーザーに送信される招待メールに含まれます。

<div id="configure-the-uri">
  ### URI を設定する
</div>

この URI をすべてのアプリケーションで共通して使用する場合は、[Auth0 Dashboard &gt; Settings &gt; Advanced](https://manage.auth0.com/#/tenant/advanced) で **Tenant Login URL** を設定できます。そうでない場合は、[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) から開ける各アプリケーションの設定で **Application Login URI** を使用して、アプリケーションごとに URI を設定できます。

<div id="specify-route-behavior">
  ### ルートの動作を指定する
</div>

アプリケーション内のルートでは、クエリ文字列内の `invitation` パラメーターと `organization` パラメーターを受け取り、その後、両方のパラメーターを指定して Auth0 Authentication API の [認可](https://auth0.com/docs/api/authentication#authorize-application) エンドポイントを呼び出す必要があります。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  アプリケーションで組織名をサブドメインまたはパス変数として使用するマルチテナントシナリオ (たとえば `acme.myapp.com`) をサポートするため、ユーザーに送信される招待リンクには `organization_name` パラメーターも含まれます。このパラメーターを Authentication API の [認可](https://auth0.com/docs/api/authentication#authorize-application) エンドポイントに送信する必要はありません。
</Callout>

たとえば、**Application Login URI** が `https://myapp.com/login` に設定された組織対応アプリケーションがある場合、エンドユーザーに送信される招待メール内のリンクは次のようになります: `https://myapp.com/login?invitation={invitation_ticket_id}&organization={organization_id}&organization_name={organization_name}`。

<div id="customize-the-user-invitation-flow">
  ## ユーザー招待フローをカスタマイズする
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ユーザー招待フローをカスタマイズするには、カスタムドメインを使用する必要があります。無料プランでは 1 つのカスタムドメインを無料で利用できます。追加のカスタムドメインは、いずれの有料プランでも利用できます。
</Callout>

ユーザー招待時のプロンプトと、ユーザーが受け取るメールをカスタマイズするには、<Tooltip tip="Universal Login: アプリケーションは、ユーザーの本人確認を行うために、Auth0 の認可サーバーでホストされる Universal Login にリダイレクトされます。" cta="用語集を見る" href="/ja/docs/glossary?term=Universal+Login">Universal Login</Tooltip> のプロンプトとメールテンプレートを変更します。詳細については、次のページを参照してください。

* [Universal Login ページテンプレート](/ja/docs/customize/login-pages/universal-login/customize-templates)
* [Universal Login の国際化](/ja/docs/customize/internationalization-and-localization/universal-login-internationalization)
* [メールテンプレートをカスタマイズする](/ja/docs/customize/email/email-templates)

<div id="localization">
  ### ローカライズ
</div>

招待されたユーザーのロケールは事前にはわからないため、<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を使用して、招待エンドポイントにオプションの `user_metadata` パラメーターを送信できます。ユーザー招待フローをローカライズする場合は、**User Invitation** メールテンプレートからこのパラメーターにアクセスし、[他の多言語メールテンプレートと同じロジックを使用できます](/ja/docs/customize/email/email-templates)。

招待されたユーザーに適切な言語で招待を表示するには、アプリケーションから `ui_locales` パラメーターを送信するか、招待されたユーザーのブラウザーから適切な `Accept-Language` ヘッダーが送信される必要があります。詳しくは、[メールテンプレートのカスタマイズ: 多言語メールテンプレート](/ja/docs/customize/email/email-templates)を参照してください。

<div id="send-membership-invitations">
  ## メンバーシップ招待を送信する
</div>

ユーザーへの[組織メンバーシップ招待の送信](/ja/docs/manage-users/organizations/configure-organizations/send-membership-invitations)には、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または Management API を使用できます。独自のメールサービスを使用して招待を送信する場合は、招待 URL のみを生成することもできます。それ以外の場合、ユーザーにはリンクを含むメールが送信され、そのリンクを使用して設定済みの接続経由で組織にログインできます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ユーザーが招待を承諾すると、そのメールアドレスは確認済みとしてマークされます。

  ユーザーがフェデレーションログイン経由で招待を承諾する場合、IDプロバイダーは招待の送信先メールアドレスと一致するメールアドレスを返す必要があります。以後のログインでは、メールアドレス確認が IDプロバイダー から返された値で上書きされることはありません。

  ユーザーが有効期限までに招待を承諾しなかった場合、その招待は期限切れとしてマークされ、最終的に[Auth0 Dashboard](/ja/docs/get-started/auth0-overview/dashboard)から削除されます。
</Callout>
