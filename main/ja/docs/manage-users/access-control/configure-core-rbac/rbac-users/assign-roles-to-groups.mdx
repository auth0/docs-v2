Auth0 では、Auth0 Dashboard または Management API を使用して、<Tooltip tip="System for Cross-domain Identity Management (SCIM): システム間でユーザーのアイデンティティとグループをプロビジョニングおよび管理するための標準化されたプロトコルです。" cta="用語集を表示" href="/ja/docs/glossary?term=SCIM">SCIM</Tooltip> でプロビジョニングされたエンタープライズグループに[ロール](/ja/docs/manage-users/access-control/rbac)を割り当てたり削除したりできます。グループにロールを割り当てると、すべてのメンバーがログイン時にそのロールを継承します。グループのロールは、次の 2 つのスコープで管理できます。

* **組織グループ** — 特定の[組織](/ja/docs/manage-users/organizations)内のエンタープライズグループ。ロールの継承を有効にするには、エンタープライズ接続で[自動メンバーシップ](/ja/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership)を有効にしておく必要があります。
* **エンタープライズグループ** — Auth0 テナントに属するエンタープライズグループ。

<div id="assign-roles">
  ## ロールの割り当て
</div>

<div id="organization-groups">
  ### 組織グループ
</div>

組織内で、SCIM によりプロビジョニングされた エンタープライズグループ にロールを割り当てます。エンタープライズグループ は、SCIM を介してエンタープライズ IDプロバイダー (Okta や Microsoft Entra ID など) から同期されます。組織のメンバーがログインすると、自分が属している エンタープライズグループ に割り当てられたロールを継承します。

グループのロール継承を有効にするには、組織のエンタープライズ接続で [自動メンバーシップが有効](/ja/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership) を有効にしておく必要があります。エンタープライズグループ がどのようにプロビジョニングされるかについて詳しくは、[インバウンド SCIM を設定する](/ja/docs/authenticate/protocols/scim/configure-inbound-scim#use-groups-with-organizational-roles) を参照してください。

<Tabs>
  <Tab title="Dashboard">
    1. [Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) に移動し、組織を選択します。
    2. **Groups** タブを選択します。
    3. ロールを割り当てるグループを選択します。
    4. **Assign Roles** を選択します。
    5. 割り当てるロールを検索して選択し、**Assign** を選択します。

    割り当てられたロールは、グループの詳細に表示されます。
  </Tab>

  <Tab title="Management API">
    [Assign Organization Group Roles](/ja/docs/api/management/v2) エンドポイントに `POST` リクエストを送信します。必要な情報は次のとおりです。

    | パラメーター                       | 説明                                                                                                                                   |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
    | `org_id`                     | 組織の ID。                                                                                                                              |
    | `group_id`                   | エンタープライズグループ の ID。                                                                                                               |
    | `role_id`                    | 割り当てる組織ロールの ID。複数のロールを一度に割り当てる場合は、配列内で繰り返し指定します。                                                                                     |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | `create:organization_group_roles` スコープを持つ [Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens)。 |

    ```bash wrap lines
    curl --request POST \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/organizations/{ORG_ID}/groups/{GROUP_ID}/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    リクエストが成功すると、`204 No Content` レスポンスが返されます。
  </Tab>
</Tabs>

<div id="enterprise-groups">
  ### エンタープライズグループ
</div>

SCIM でプロビジョニングされたエンタープライズグループにロールを割り当てます。グループにロールを割り当てると、そのグループのすべてのメンバーはログイン時に割り当てられたロールを継承します。

エンタープライズグループがエンタープライズ IDプロバイダーからどのようにプロビジョニングされるかについては、[インバウンド SCIM を構成する](/ja/docs/authenticate/protocols/scim/configure-inbound-scim)を参照してください。

<Tabs>
  <Tab title="Dashboard">
    1. [Dashboard &gt; User Management &gt; エンタープライズグループ](https://manage.auth0.com/#/groups) に移動します。
    2. ロールを割り当てるグループを選択します。
    3. **ロール** タブを選択し、**Assign Roles** を選択します。
    4. 割り当てるロールを検索して選択し、**Assign** を選択します。

    割り当てられたロールは、グループの **ロール** タブに表示されます。
  </Tab>

  <Tab title="Management API">
    [Assign Roles to a Group](/ja/docs/api/management/v2) エンドポイントに `POST` リクエストを送信します。必要な情報は次のとおりです。

    | Parameter                    | Description                                                                                                        |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------ |
    | `group_id`                   | エンタープライズグループの ID。                                                                                                 |
    | `role_id`                    | 割り当てるロールの ID。一度に複数のロールを割り当てるには、配列内で繰り返し指定します。                                                                      |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | `update:groups` スコープを持つ [Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens)。 |

    ```bash wrap lines
    curl --request POST \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/groups/{group_id}/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    リクエストが成功すると、`204 No Content` レスポンスが返されます。
  </Tab>
</Tabs>

<div id="remove-roles">
  ## ロールを削除
</div>

<div id="organization-groups">
  ### 組織グループ
</div>

組織内の、SCIM でプロビジョニングされたエンタープライズグループからロールを削除します。ロールを削除すると、そのグループに属する組織メンバーは、ログイン時にそのロールを継承しなくなります。

グループのロール継承を機能させるには、組織のエンタープライズ接続で [Auto-Membership を有効にする](/ja/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership) 必要があります。

<Tabs>
  <Tab title="Dashboard">
    1. [Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) に移動し、組織を選択します。
    2. **Groups** タブを選択します。
    3. 更新するグループを選択します。
    4. 削除するロールを見つけ、その横にあるごみ箱アイコンを選択します。
    5. 削除を確認します。

    グループの詳細から、そのロールが表示されなくなります。
  </Tab>

  <Tab title="Management API">
    [Delete Organization Group Roles](/ja/docs/api/management/v2) エンドポイントに `DELETE` リクエストを送信します。必要な情報は次のとおりです。

    | Parameter                    | Description                                                                                                                          |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
    | `org_id`                     | 組織の ID。                                                                                                                              |
    | `group_id`                   | エンタープライズグループの ID。                                                                                                                    |
    | `role_id`                    | 削除する組織ロールの ID。一度に複数のロールを削除するには、配列内で繰り返して指定します。                                                                                       |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | `delete:organization_group_roles` スコープを持つ [Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens)。 |

    ```bash wrap lines
    curl --request DELETE \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/organizations/ORG_ID/groups/GROUP_ID/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    リクエストが成功すると、`204 No Content` レスポンスが返されます。
  </Tab>
</Tabs>

<div id="enterprise-groups">
  ### エンタープライズグループ
</div>

SCIM でプロビジョニングされたエンタープライズグループからロールを削除します。ロールを削除すると、以後グループメンバーはログイン時にそのロールを継承しなくなります。

<Tabs>
  <Tab title="Dashboard">
    1. [Dashboard &gt; User Management &gt; Enterprise Groups](https://manage.auth0.com/#/groups) に移動します。
    2. 更新するグループを選択します。
    3. **ロール** タブを選択します。
    4. 削除するロールを見つけて、その横にあるゴミ箱アイコンを選択します。
    5. 削除を確認します。

    削除したロールは、グループの **ロール** タブに表示されなくなります。
  </Tab>

  <Tab title="Management API">
    [Remove ロール from a Group](/ja/docs/api/management/v2) エンドポイントに `DELETE` リクエストを送信します。必要な情報は次のとおりです。

    | パラメーター                       | 説明                                                                                                                 |
    | ---------------------------- | ------------------------------------------------------------------------------------------------------------------ |
    | `group_id`                   | エンタープライズグループの ID。                                                                                                  |
    | `role_id`                    | 削除するロールの ID。複数のロールを一度に削除するには、配列内に複数指定します。                                                                          |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | `update:groups` スコープを持つ [Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens)。 |

    ```bash wrap lines
    curl --request DELETE \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/groups/GROUP_ID/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    リクエストが成功すると、`204 No Content` レスポンスが返されます。
  </Tab>
</Tabs>
