title: プログレッシブプロファイリングの仕組みを理解する
---

プログレッシブプロファイリングとは、ユーザーがウェブサイトやアプリケーションを利用する過程で、追加情報を段階的に収集していく手法です。ユーザーの行動に応じて必要な場合にのみ、より詳細なユーザー情報を収集できます。

たとえば、サインアップ時にユーザーの名前、メールアドレス、パスワードを収集し、次回ログイン時に会社名や役職を尋ねることができます。[Auth0 Forms](/ja/docs/customize/forms/configure-progressive-profile-form) を使用して、ユーザー情報を収集するフローを設計することもできます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ユーザーに追加情報を求める際は、すでに取得している可能性のある情報を改めて尋ねないようにしてください。たとえば、ユーザーが OpenID Connect プロトコルを使用するソーシャル接続 (Google や Facebook など) を通じてサインアップした場合、IDプロバイダー (IdP) から想定以上に多くのユーザー情報が返されることがあります。詳しくは、[OpenID Connect Scopes](/ja/docs/get-started/apis/scopes/openid-connect-scopes#standard-claims) を参照してください。
</Callout>

プログレッシブプロファイリングを実装すると、次のようにユーザー体験が向上します。

* 登録フォームを短くできる
* 同じ質問を繰り返さずに済む
* より関連性の高い情報を収集できる
* コンバージョン率を高められる

<div id="how-it-works">
  ## 仕組み
</div>

ユーザーが Auth0 経由で認証されるたびに、Auth0 はそのユーザープロフィールを更新します。プロフィールの更新に使用されるデータは、さまざまなソースから取得できます。

* <Tooltip tip="IDプロバイダー（IdP）: デジタル ID を保存・管理するサービス。" cta="用語集を見る" href="/ja/docs/glossary?term=IdP">IdP</Tooltip> から提供される属性。
* [Auth0 Actions](/ja/docs/customize/actions) で動的に作成される属性。
* [FullContact](https://www.fullcontact.com/) や [Clearbit](https://clearbit.com/) などの API を呼び出して取得される属性。
* アプリケーションで管理される属性。

<div id="use-action-triggers">
  ## Action トリガーを使用する
</div>

`post-login` Action トリガーを使用すると、ログインフロー中にユーザーの `user_metadata` フィールドと `app_metadata` フィールドを変更できます。これは、アプリケーション固有のデータをユーザーのプロフィールに保存したり、特定の操作がユーザーに対して実行されたかどうかを記録したり、負荷の高い処理の結果をユーザーのプロフィールにキャッシュして今後のログインで再利用できるようにしたりする場合に役立ちます。詳しくは、[Manage User Metadata with the post-login Action Trigger](/ja/docs/manage-users/user-accounts/metadata/manage-user-metadata) を参照してください。

<div id="use-the-management-api">
  ## Management API を使用する
</div>

Auth0 の <Tooltip tip="Management API: お客様が管理タスクを実行するための製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> の [ユーザーを更新する](https://auth0.com/docs/api/management/v2/#!/Users/patch_users_by_id) エンドポイントを使用して、認証済みユーザーのプロフィールデータ (`app_metadata` フィールドや `user_metadata` フィールドを含む) を更新できます。詳しくは、[Management API を使用してメタデータを管理する](/ja/docs/manage-users/user-accounts/metadata/manage-metadata-api) を参照してください。

<div id="use-the-lock-library">
  ## Lock ライブラリを使用する
</div>

`additionalSignUpFields` オプションを使用すると、ユーザーのサインアップフォームにカスタムフィールドを追加できます。ユーザーがカスタムフィールドにデータを入力すると、Auth0 はその値をユーザープロファイルの `user_metadata` フィールドに保存します。詳細については、[Lock Configuration Options](/ja/docs/libraries/lock/lock-configuration#additionalsignupfields-array-) を参照してください。
