---
title: テナント管理者がユーザーをブロックおよびブロック解除する方法
sidebarTitle: ユーザーのブロックとブロック解除
description: テナント管理者によるブロックを追加または削除して、ユーザーのアプリケーションへのログイン可否を制御します。
---

テナント管理者は、ユーザーがアプリケーションにアクセスできないようにブロックできます (ブロックを解除すると、再びアクセスできるようになります) 。

ブロックされたユーザーがアプリケーションにログインしようとすると、URL にエラーメッセージ &quot;user is blocked&quot; が含まれた状態でアプリケーションからリダイレクトされます。

<div id="add-tenant-administrator-blocks">
  ## テナント管理者によるブロックを追加する
</div>

Auth0 Dashboard を使用してテナント管理者によるブロックを追加するには、次の手順に従います。

1. [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) に移動します。
2. ブロックするユーザーの名前を選択します。
3. **Details** タブで、**Block user** の横にある **Block** を選択します。

このオプションは、ユーザー名の右側にある **...** メニューからも利用できます。

Management API を使用してテナント管理者によるブロックを追加するには、[Update a User endpoint](/ja/docs/api/management/v2/users/patch-users-by-id) (`PATCH /users/{id}`) を使用して、ユーザーの [`blocked` 属性](/ja/docs/api/management/v2/users/patch-users-by-id#body-blocked) を `true` に設定します。

<div id="remove-tenant-administrator-blocks">
  ## テナント管理者によるブロックの解除
</div>

テナント管理者がユーザーをブロックした場合、そのブロックが自動的に解除されることはありません。アプリケーションへのアクセスを復元するには、テナント管理者がユーザーのブロックを解除する必要があります。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  [ブルートフォース保護](/ja/docs/secure/attack-protection/brute-force-protection)によるブロックは、テナント管理者によるブロックとは別に処理されます。詳しくは、[ブルートフォース保護によるブロックの解除](/ja/docs/secure/attack-protection/brute-force-protection#removing-brute-force-protection-blocks)を参照してください。
</Callout>

Auth0 Dashboard を使用してテナント管理者によるブロックを解除するには、次の手順を実行します。

1. [Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) に移動します。
2. テナント管理者によるブロックが設定されているユーザーの名前を選択します。
3. **Details** タブで、**Block user** の横にある **Unblock** を選択します。

このオプションは、ユーザー名の右側にある **...** メニューからも利用できます。

Management API を使用してテナント管理者によるブロックを解除するには、[Update a User endpoint](/ja/docs/api/management/v2/users/patch-users-by-id) (`PATCH /users/{id}`) を使用して、ユーザーの [`blocked` 属性](/ja/docs/api/management/v2/users/patch-users-by-id#body-blocked) を `false` に設定します。

Management API を使用してユーザーのパスワードをリセットした場合も、テナント管理者によるブロックは解除されます。
