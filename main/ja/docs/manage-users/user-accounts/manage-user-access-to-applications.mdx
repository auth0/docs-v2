title: アプリケーションへのユーザーアクセスを管理する
---

1 つの Auth0 テナントに関連付けられているすべてのユーザーは、そのテナント内のアプリケーション間で共有されます (そのため、各アプリケーションにアクセスできます) 。

* ユーザーを分離してアクセスを制限するには、特定のユーザーグループにアクセスさせたいアプリケーションを含む[追加のテナントを作成する](/ja/docs/get-started/auth0-overview/create-tenants/create-multiple-tenants)ことをお勧めします。

* ユーザーの特定のアプリケーションへのアクセスを制限するには、[Rules](/ja/docs/customize/rules)を使用できます。

  * Rule 内では、ユーザーがどのアプリケーションを使ってログインしているかを確認するために、`context.clientName` と `context.clientID` 変数を使用できます。
  * 例については、[この Rule](https://github.com/auth0/rules/blob/aeaf93bc058408e260192d0941a688963449d6be/src/rules/simple-user-whitelist-for-app.js)を参照してください。

* 新しい接続を設定し、特定のアプリケーションにのみアクセスを許可することで、ユーザーが利用できるアプリケーションを制限できます。

  * [Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/connections/social) に移動し、接続をクリックします。
  * **Applications** タブで、アプリケーションを有効または無効にします。

* アプリケーションへのユーザーアクセスを無効にするには、Dashboard で[ユーザーをブロックまたはブロック解除](/ja/docs/manage-users/user-accounts/block-and-unblock-users)できます。
