このドキュメントでは、API から想定外のレスポンスが返された場合に、設定をトラブルシューティングする方法を説明します。アカウント用に設定されたサンプルを使ってこのクイックスタートを進められるよう、ログインしておくことをお勧めします。

JWT 検証が正しく設定されていれば、リクエスト時に API から適切なレスポンスを取得できます。一方、API から 401 (Unauthorized) レスポンスが返される場合は、JWT ミドルウェアの設定が渡された JWT と一致していないことが原因です。

このドキュメントは、JWT ミドルウェアの設定をトラブルシューティングする際に役立ちます。

<div id="how-does-a-token-get-validated">
  ## トークンはどのように検証されますか？
</div>

JWT を検証する際には、考慮すべき点がいくつかあります。

1. **トークンの形式は正しいですか？** つまり、このトークンは JSON Web Token (JWT) の構造に準拠していますか？ JWT の構造の詳細については、[JWT の構造に関するこちらのセクション](/ja/docs/secure/tokens/json-web-tokens#what-is-the-json-web-token-structure-) を参照してください。
2. **トークンは改ざんされていませんか？** JWT の最後の部分は署名です。この署名は、トークンが実際に送信者によって署名されており、途中で一切変更されていないことを検証するために使用されます。
3. **トークンは有効期間内に受信されましたか？** JWT は指定された期間内でのみ有効です (`exp` クレームで表されます) 。
4. **トークンは想定された Authority から送信されていますか？** これには 2 つの要素があります。

   * **署名の検証**: JWT が、発行元の Authority から発行されたキーを使用して正しく署名されていることを確認できますか？
   * **Issuer 値**: Issuer は `iss` クレームで定義されます。このクレームは、アプリケーションが想定している値と一致していますか？
5. **トークンは現在のアプリケーション向けですか？** つまり、JWT の `aud` クレームは、アプリケーションが想定している値と一致していますか？

<div id="inspecting-a-token">
  ## トークンの確認
</div>

JWT を手早く確認するには、[JWT.io](https://jwt.io/) の Web サイトを使用する方法があります。便利なデバッガーが用意されており、JWT の形式が正しいかどうかをすばやく確認できるほか、各クレームの値も確認できます。

<Frame>![JWT.io で JWT をデバッグする](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/aspnet-core-webapi/jwt-io-debugger-rs256.png)</Frame>

上のスクリーンショットでは、トークンが **RS256** アルゴリズムで署名されていることがわかります。トークンの **Issuer** は **[https://jerrie.auth0.com/](https://jerrie.auth0.com/)**、**Audience** は **[https://rs256.test.api](https://rs256.test.api)** です。

つまり、JWT 検証構成の登録で使用するこれらの値は、Issuer の末尾のスラッシュも含めて、**完全に** 一致している必要があります。次のように指定します。

```
audience = "https://rs256.test.api",
domain = "https://jerrie.auth0.com/"
};
```

HS256 で署名されたトークンの場合、デバッガービューの表示は少し異なります。

<Frame>![JWT.io で JWT をデバッグしている様子](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/aspnet-core-webapi/jwt-io-debugger-hs256.png)</Frame>

上のスクリーンショットから、このトークンが **HS256** アルゴリズムで署名されていることがわかります。トークンの **Issuer** は **[https://jerrie.auth0.com/](https://jerrie.auth0.com/)**、**Audience** は **[https://hs256.test.api](https://hs256.test.api)** です。
