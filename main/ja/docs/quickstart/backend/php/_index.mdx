    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 は、PHP アプリに Auth0 の認証と認可を実装する際の作業を簡略化する [PHP
      SDK](https://github.com/auth0/auth0-PHP) (Auth0-PHP) を提供しています。

      Auth0 PHP SDK では、ネットワークリクエストを処理する互換性のある HTTP ライブラリとして、[PSR-17](https://www.php-fig.org/psr/psr-17/) と [PSR-18](https://www.php-fig.org/psr/psr-18/) をインストールしておく必要があります。利用可能なライブラリがない
      場合は、ターミナルで次のコマンドを実行して、信頼できるライブラリをインストールできます。

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      次に、ターミナルで次のコマンドを実行して、Auth0 PHP SDK をインストールします。

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Auth0 SDK を設定する

      SDK が正しく機能するようにするには、初期化時に Auth0 SDK で次のプロパティを設定する必要があります。

      * `strategy`: この設定は、アプリケーションのユースケースに応じた SDK の動作を決定するためのものです。この
        場合は、次の定数に設定します。

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
      * domain: Auth0 テナントのドメインです。通常は、Auth0 Dashboard の Application&#39;s
        Settings にある Domain フィールドで確認できます。カスタムドメインを使用している場合は、代わりに
        そのカスタムドメインの値を設定します。
      * clientId: このクイックスタートの前の手順で設定した Auth0 アプリケーションの ID です。これは、
        Auth0 Dashboard の Application&#39;s Settings にある クライアントID フィールドで確認できます。
      * clientSecret: このクイックスタートの前の手順で作成した Auth0 アプリケーションのシークレットです。クライアントシークレットは、
        Auth0 Dashboard の Application&#39;s Settings にある クライアントシークレット フィールドで確認できます。
      * audience: 上で登録した Auth0 API の識別子です。これは配列として指定する必要があります。

      次に、ベアラートークンを取得して処理できるように、アプリケーションを拡張します。ベアラートークンは、クライアントが
      ユーザーに代わって API に送信するリクエストに含まれるアクセストークンです。アクセストークンは、アプリケーション内のルートへの
      アクセスを許可または拒否します。これをエンドポイント認可と呼びます。

      リクエストからアクセストークンを取得する最も簡単な方法は、PHP SDK の `getBearerToken()`
      メソッドを使用することです。このメソッドは、GET パラメーター、POST ボディ、リクエストヘッダー、その他のソースからトークンを取得します。この
      場合、PHP SDK は、GET リクエストの `token` パラメーター、または HTTP
      `Authorization` ヘッダーで渡されたトークンを処理します。

      Auth0 アプリケーションと Auth0 PHP SDK の設定が完了し、アプリケーションがリクエストからベアラートークンを取得できるようになったら、次はプロジェクトのエンドポイント認可を設定します。上で実装した
