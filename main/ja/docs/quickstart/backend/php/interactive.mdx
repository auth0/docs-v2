    Auth0を使用すると、ほぼあらゆる種類のアプリケーションにトークンベースのエンドポイント認可を素早く追加できます。このガイドでは、Auth0 PHP SDKを使ってAuth0を統合し、トークンベースの認可を追加して、アプリケーションのルートを保護する方法を説明します。

    このクイックスタートを使用するには、以下が必要です。

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Auth0 のサービスを利用するには、Auth0 Dashboard にアプリケーションを登録する必要があります。Auth0 アプリケーションでは、
      プロジェクトで認証をどのように機能させるかを設定します。

      ### アプリケーションを設定する

      インタラクティブなセレクターを使用して、新しい Auth0 アプリケーションを作成するか、統合するプロジェクトに対応する既存のアプリケーションを選択します。Auth0 の各アプリケーションには、英数字の一意のクライアントIDが割り当てられ、
      アプリケーションのコードは SDK を介して Auth0 API を呼び出す際にこの値を使用します。

      このクイックスタートで行った設定は、[Dashboard](https://manage.auth0.com/#/) のアプリケーションに自動的に反映されます。ここでは、
      今後アプリケーションを管理できます。

      構成全体を確認したい場合は、代わりにサンプルアプリケーションを参照できます。

      ### API を設定する

      同様に、[Dashboard](https://manage.auth0.com/#/) から、統合するプロジェクトに対応する新しい Auth0 API を作成するか、
      既存の API を使用する必要があります。API には一意の識別子を選択し、控えておいてください。以下でアプリケーションを設定するには、
      その識別子が必要です。
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 では、PHP アプリに Auth0 の認証と認可を実装する作業を簡素化するための [PHP
      SDK](https://github.com/auth0/auth0-PHP) (Auth0-PHP) を提供しています。

      Auth0 PHP SDK では、ネットワークリクエストを処理する互換性のある HTTP ライブラリとして、[PSR-17](https://www.php-fig.org/psr/psr-17/) と [PSR-18](https://www.php-fig.org/psr/psr-18/) がインストールされている必要があります。利用可能なライブラリが
      ない場合は、ターミナルで次のコマンドを実行して、信頼性の高いライブラリをインストールできます。

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      次に、ターミナルで次のコマンドを実行し、Auth0 PHP SDK をインストールします。

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Auth0 SDK を設定する

      SDK を正しく機能させるには、初期化時に Auth0 SDK で次のプロパティを設定する必要があります。

      * `strategy`: アプリのユースケースに応じて SDK の動作を制御するための設定です。この
        場合は、これを次の定数に設定します。

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
      * domain: Auth0 テナントのドメインです。通常、Auth0 Dashboard の Application&#39;s
        Settings にある Domain フィールドで確認できます。カスタムドメインを使用している場合は、代わりに
        そのカスタムドメインの値を設定します。
      * clientId: このクイックスタートの前の手順で設定した Auth0 Application の ID です。これは
        Auth0 Dashboard の Application&#39;s Settings にある Client ID フィールドで確認できます。
      * clientSecret: このクイックスタートの前の手順で作成した Auth0 Application のシークレットです。Client Secret は
        Auth0 Dashboard の Application&#39;s Settings にある Client Secret フィールドで確認できます。
      * audience: 上で登録した Auth0 API の識別子です。これは配列として指定する必要があります。

      <Note>
        ##### チェックポイント

        Auth0 SDK は正しく設定されました。アプリケーションを実行して、次のことを確認してください。

        * SDK が正しく初期化されること。
        * アプリケーションで Auth0 関連のエラーが発生しないこと。
      </Note>
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      次に、ベアラートークンを取得して処理できるように、アプリケーションを拡張します。ベアラートークンは、ユーザーに代わってクライアントが
      API に送信するリクエストに含まれるアクセストークンです。アクセストークンは、アプリケーション内のルートへのアクセスを許可または拒否するために使用されます。
      これはエンドポイント認可と呼ばれます。

      リクエストからアクセストークンを取得する最も簡単な方法は、PHP SDK の `getBearerToken()`
      メソッドを使用することです。このメソッドは、GET パラメーター、POST ボディ、リクエストヘッダー、その他のソースからトークンを取得します。この
      場合、PHP SDK は GET リクエストの `token` パラメーター、または HTTP
      `Authorization` ヘッダーから渡されたトークンを処理します。
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      ここで、受信したリクエストをアプリケーションに振り分けるためのルーティングライブラリをインストールします。これは必須の手順ではありませんが、
      このクイックスタートでは、アプリケーション構成をシンプルにするのに役立ちます。

      ```bash lines
      composer require steampixel/simple-php-router
      ```

      アプリケーション内に `router.php` という新しいファイルを作成し、ルートを定義します。右側のインタラクティブパネルの **router.php** タブにある
      コードをコピーしてください。
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Auth0 アプリケーションと Auth0 PHP SDK を設定し、アプリケーションがリクエストからベアラートークンを取得できるようになったら、次のステップはプロジェクトのエンドポイント認可を設定することです。上で実装した
      `getBearerToken()` メソッドは、リクエストに付与されたアクセス権の詳細を含む `Token` クラスを返します。

      `getBearerToken()` メソッドは受信リクエストを自動的に検証して正当性を確認するため、アプリケーションはこのメソッドの戻り値を評価することでアクセストークンの詳細を判断します。戻り値が
      null の場合は、有効なトークンが提供されていません。それ以外の場合は、戻り値の内容を調べることでリクエストの詳細を確認できます。

      右側のインタラクティブ パネルでは、戻り値が null かどうかを確認する処理によって、`/api/private` ルートへのアクセスを制限する例を確認できます。
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      場合によっては、アクセストークンで要求されたスコープに基づいて、特定のルートへのアクセスを
      制御したいことがあります。右側のインタラクティブパネルに示されているように、
      `getBearerToken()` メソッドのレスポンスに含まれる &#39;scope&#39; プロパティの内容を確認し、アクセストークンに付与されたスコープを確認します。
    </Section>
