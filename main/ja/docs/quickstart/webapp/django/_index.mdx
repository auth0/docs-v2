    Auth0を使用すると、アプリケーションに認証を追加し、ユーザープロファイル情報にアクセスできるようになります。このガイドでは、[Authlib](https://authlib.org/) SDKを使用して、Auth0をPython [Django](https://www.djangoproject.com/)アプリケーションに統合する方法を説明します。

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Auth0 のサービスを利用するには、まず Auth0 Dashboard でアプリケーションを設定する必要があります。Auth0 アプリケーションでは、開発中のプロジェクトで認証をどのように動作させるかを設定します。

      ### アプリケーションを設定する

      インタラクティブなセレクターを使用して、新しい Auth0 アプリケーションを作成するか、統合するプロジェクトに対応する既存のアプリケーションを選択します。Auth0 の各アプリケーションには、英数字の一意なクライアントID が割り当てられます。アプリケーションのコードは、SDK を通じて Auth0 API を呼び出す際にこのクライアントID を使用します。

      このクイックスタートで行った設定は、[Dashboard](https://manage.auth0.com/dashboard/us/auth0-dsepaid/) 内のアプリケーションに自動的に反映されます。以後、アプリケーションはそこで管理できます。

      設定済みの完全な構成を確認したい場合は、代わりにサンプルアプリケーションを参照できます。

      ### コールバックURLを設定する

      コールバックURL は、認証後に Auth0 がユーザーをリダイレクトする先として、アプリケーション内で指定する URL です。これが設定されていない場合、ユーザーはログイン後にアプリケーションへ戻れません。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを
        `http://localhost:3000/callback` に設定してください。
      </Info>

      ### ログアウトURLを設定する

      ログアウトURL は、ログアウト後に Auth0 がユーザーをリダイレクトする先として、アプリケーション内で指定する URL です。これが設定されていない場合、ユーザーはアプリケーションからログアウトできず、エラーが表示されます。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      この統合では、Authlib など、いくつかのライブラリ依存関係を追加します。プロジェクトディレクトリに
      `requirements.txt` ファイルを作成し、次の内容を記載してください。

      ```text lines
      authlib ~= 1.0
      django ~= 4.0
      python-dotenv ~= 0.19
      requests ~= 2.27
      ```

      以下の依存関係を利用できるようにするには、シェルで次のコマンドを実行します。

      ```text lines
      pip install -r requirements.txt
      ```

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7" isSingleColumn>
      この例では、アプリケーションに 4 つのルート (login、callback、logout、index) を追加します。

      * `login` - アプリの訪問者が `/login` ルートにアクセスすると、Auth0 に移動して
        認証フローを開始します。
      * `callback` - ユーザーが Auth0 でのログインを完了すると、アプリケーションの
        `/callback` ルートに戻ります。このルートはユーザーのセッションを保存するため、再度アクセスしたときに
        もう一度ログインする必要がありません。
      * `logout` - `/logout` ルートは、ユーザーをアプリケーションからサインアウトさせます。このルートは
        アプリ内のユーザーセッションを消去し、セッションが保持されたままにならないように
        Auth0 のログアウトエンドポイントにリダイレクトします。その後、アプリケーションはユーザーをホームルートにリダイレクトします。
      * `index` - ホームルートでは、認証済みユーザーの詳細を表示するか、訪問者がサインインできるようにします。
