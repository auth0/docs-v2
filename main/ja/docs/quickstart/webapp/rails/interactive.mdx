    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Auth0 のサービスを利用するには、Auth0 Dashboard でアプリケーションを設定しておく必要があります。Auth0 アプリケーションでは、
      開発中のプロジェクトで認証をどのように機能させるかを設定します。

      ### アプリケーションを設定する

      インタラクティブなセレクターを使用して、新しい Auth0 アプリケーションを作成するか、
      連携するプロジェクトに対応する既存のアプリケーションを選択します。Auth0 のすべてのアプリケーションには、
      英数字から成る一意のクライアントID が割り当てられており、アプリケーションコードはそれを使用して SDK 経由で Auth0 API を呼び出します。

      このクイックスタートで行った設定は、[Dashboard](https://manage.auth0.com/#/) 内のアプリケーションに自動的に反映されます。ここでは、
      今後アプリケーションを管理できます。

      設定済みの完全な構成を確認したい場合は、代わりにサンプルアプリケーションを参照できます。

      ### コールバック URL を設定する

      コールバック URL とは、認証後に Auth0 がユーザーをリダイレクトする先として
      アプリケーション内で指定する URL です。設定しない場合、ユーザーはログイン後にアプリケーションへ戻れません。

      <Info>
        サンプルプロジェクトを使って進めている場合は、これを
        `http://localhost:3000/auth/auth0/callback` に設定してください。
      </Info>

      ### ログアウト URL を設定する

      ログアウト URL とは、ログアウト後に Auth0 がユーザーをリダイレクトする先として
      アプリケーション内で指定する URL です。設定しない場合、ユーザーはアプリケーションからログアウトできず、エラーが発生します。

      <Info>
        サンプルプロジェクトを使って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>

      ### 許可する Web オリジンを設定する

      許可する Web オリジンとは、認証フローへのアクセスを許可する URL です。ここには
      プロジェクトの URL を含める必要があります。これが適切に設定されていない場合、プロジェクトで認証トークンを
      サイレントに更新できないため、ユーザーは次回アプリケーションにアクセスしたときやページを更新したときに
      ログアウトされます。

      <Info>
        サンプルプロジェクトを使って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      認証フローの処理には、カスタムの[OmniAuthストラテジー](https://github.com/intridea/omniauth#omniauth-standardized-multi-provider-authentication)である`omniauth-auth0`を使用します。

      次の依存関係を`Gemfile`に追加します。

      ```gem lines
      gem 'omniauth-auth0', '~> 3.0'
      gem 'omniauth-rails_csrf_protection', '~> 1.0' # 偽造された認証リクエストを防ぐ
      ```

      gemを追加したら、`bundle install` を実行してインストールします。
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Auth0 Dashboard のアプリケーション **Settings** にある Auth0 のドメイン、クライアントID、クライアント
      シークレットの値を指定するため、設定ファイル `./config/auth0.yml` を作成します。
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      次の初期化ファイル `./config/initializers/auth0.rb` を作成し、前の手順で作成した設定ファイルを使用して **OmniAuth** ミドルウェアを[構成](https://github.com/auth0/omniauth-auth0/blob/master/EXAMPLES.md#send-additional-authentication-parameters)します。

      `callback_path` が、Auth0 アプリケーションの &quot;Allowed Callback URLs&quot; 設定で指定した値と一致していることを確認してください。
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      認証コールバック、`logout` アクション、および
      ログアウト URL を構築するメソッドを処理する Auth0 コントローラーを作成します。

      次のコマンドを実行します。
      `rails generate controller auth0 callback failure logout --skip-assets --skip-helper --skip-routes --skip-template-engine`.

      callback メソッド内で、
      `request.env['omniauth.auth']` として返されるユーザー情報のハッシュを現在のセッションに格納します。

      ログアウトを設定するには、`logout` アクション内で `reset_session`
      メソッドを呼び出し、セッションに保存されているすべてのオブジェクトをクリアします。その後、Auth0 のログアウトエンドポイントにリダイレクトします。`reset_session` の詳細については、
      [Ruby on Rails ActionController documentation](http://api.rubyonrails.org/classes/ActionController/Base.html#M000668) を参照してください。
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      これらのルートを `./config/routes.rb` ファイルに追加します。

      Rails が各種 Auth0 コールバック URL を前の手順で作成した Auth0 コントローラーにルーティングできるように、ルートを設定しておく必要があります。

      <Info>
        ##### チェックポイント

        アプリケーションを実行し、意図したとおりに引き続き動作すること、および Auth0 関連のエラーが発生していないことを確認します。
      </Info>
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      ユーザーは `/auth/auth0` エンドポイントにアクセスすると、アプリケーションにログインできます。

      <Warning>
        [認証リクエストの偽造を防ぐ](https://github.com/cookpad/omniauth-rails_csrf_protection)には、`link_to` または
        `button_to` ヘルパーメソッドで `:post` メソッドを使用してください。
      </Warning>

      ```ruby lines
      <!-- アプリケーションの任意の場所にログインボタンを配置する -->
      <%= button_to 'Login', '/auth/auth0', method: :post %>
      ```

      <Info>
        ##### チェックポイント

        選択するとユーザーが `/auth/auth0` エンドポイントにリダイレクトされるボタンをアプリケーションに追加します。ログインのために Auth0 にリダイレクトされ、認証が正常に完了するとアプリに戻ることを確認してください。
      </Info>
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Rails アプリケーションにログインできるようになったので、次は
      ログアウトする方法が必要です。ユーザーを `auth/logout` アクションにリダイレクトしてログアウトさせます。このアクションは
      Auth0 のログアウトエンドポイントにリダイレクトします。

      <Info>
        前の手順の後でこれをテストするには、セッションをクリアしてから、ユーザー
        を Auth0 のログアウトエンドポイントにリダイレクトする必要がある場合があります。
      </Info>

      ```ruby lines
      <!-- アプリケーションの任意の場所にログアウトボタンを配置してください -->
      <%= button_to 'Logout', 'auth/logout', method: :get %>
      ```

      <Info>
        ##### チェックポイント

        選択するとユーザーが `/auth/logout` エンドポイントにリダイレクトされるボタンをアプリケーションに追加します。
        Auth0 にリダイレクトされた後、すぐにアプリケーションに戻り、ログイン状態が解除されていることを確認します。
      </Info>
    </Section>

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9">
      ユーザーのプロフィールを表示するには、アプリケーションで保護されたルートを用意する必要があります。複数の
      コントローラー間で共有できるルートへのアクセスを制御するには、[Concern](https://guides.rubyonrails.org/getting_started.html#using-concerns) を使用できます。ユーザーが未認証の場合、Concern は自動的に Auth0 にリダイレクトするようにする必要があります。それ以外の
      場合は、Concern は現在のユーザープロフィールを返すようにする必要があります。

      Concern を作成したら、ログイン済みユーザーが必要なすべてのコントローラーにそれをインクルードします。すると、次の例のように、セッション `session[:userinfo]` から
      ユーザー情報にアクセスできます。

      ```ruby lines
      class DashboardController < ApplicationController
      include Secured
      def show
      @user = session[:userinfo]

      end
      end
      ```

      セッションからユーザーが読み込まれたら、そのユーザー情報を使ってフロントエンドに情報を表示します。

      ```html lines
      <div>
      <p>正規化されたユーザープロファイル:<%= JSON.pretty_generate(@user[:info])%></p>
      <p>完全なユーザープロファイル:<%= JSON.pretty_generate(@user[:extra][:raw_info])%></p>
      </div>
      ```

      <Info>
        ##### チェックポイント

        アプリに `Secured` concern を追加し、認証済みのユーザーのみがアクセスできる
        コントローラーにこれを含めます。認証済みのユーザーがその
        コントローラー内のアクションにアクセスできること、および未認証のユーザーが認証のために Auth0 にリダイレクトされることを確認します。
      </Info>
    </Section>
