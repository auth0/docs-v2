    Auth0を使用すると、ほぼあらゆる種類のアプリケーションに素早く認証を追加できます。このガイドでは、Auth0 PHP SDKを使用してPHPアプリケーションにAuth0を統合し、認証を追加して、ユーザープロファイル情報を表示する方法を説明します。

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Auth0 のサービスを利用するには、Auth0 Dashboard でアプリケーションを設定しておく必要があります。Auth0 アプリケーションでは、
      開発中のプロジェクトで認証をどのように機能させるかを設定します。

      ### アプリケーションを設定する

      インタラクティブなセレクターを使って、新しい Auth0 アプリケーションを作成するか、
      連携するプロジェクトに対応する既存のアプリケーションを選択します。Auth0 のすべてのアプリケーションには、
      英数字から成る一意のクライアントIDが割り当てられます。アプリケーションコードは、このクライアントIDを使って SDK 経由で Auth0 API を呼び出します。

      このクイックスタートで行った設定は、[Dashboard](https://manage.auth0.com/#/) のアプリケーションに自動的に反映されます。今後はここで、
      アプリケーションを管理できます。

      設定一式をまとめて確認したい場合は、代わりにサンプルアプリケーションを参照することもできます。

      ### コールバックURLを設定する

      コールバックURLとは、認証後に Auth0 がユーザーをリダイレクトする先として
      アプリケーション内で指定する URL です。これを設定していない場合、ユーザーはログイン後にアプリケーションへ戻れません。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、
        これを `http://localhost:3000/callback` に設定してください。
      </Info>

      ### ログアウトURLを設定する

      ログアウトURLとは、ログアウト後に Auth0 がユーザーをリダイレクトする先として
      アプリケーション内で指定する URL です。これを設定していない場合、ユーザーはアプリケーションからログアウトできず、エラーが発生します。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>

      ### 許可する Web オリジンを設定する

      許可する Web オリジンとは、認証フローへのアクセスを許可する URL です。ここには必ず
      プロジェクトの URL を含める必要があります。これが正しく設定されていない場合、プロジェクトは認証トークンをサイレントに更新できなくなるため、
      ユーザーは次回アプリケーションにアクセスしたとき、またはページを再読み込みしたときにログアウトされます。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 は、PHP アプリに Auth0 の認証と認可を実装する作業を簡素化するための [PHP
      SDK](https://github.com/auth0/auth0-PHP) (Auth0-PHP) を提供しています。

      Auth0 PHP SDK では、ネットワークリクエストを処理するために、[PSR-17](https://www.php-fig.org/psr/psr-17/) および [PSR-18](https://www.php-fig.org/psr/psr-18/) に対応した HTTP ライブラリをインストールしておく必要があります。利用可能なライブラリが
      ない場合は、ターミナルで次のコマンドを実行して、信頼性の高いライブラリをインストールできます。

      ```text lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      次のコマンドをターミナルで実行して、Auth0 PHP SDK をインストールします。

      ```text lines
      composer require auth0/auth0-php
      ```

      ### Auth0 SDK を設定する

      アプリケーションに `index.php` という新しいファイルを作成し、右側のインタラクティブ
      パネルの **index.php** タブにあるコードをコピーします。

      SDK が正しく動作するように、初期化時に Auth0 SDK で次のプロパティを設定する必要があります。

      * `domain`: Auth0 テナントのドメインです。通常、この値は Auth0 Dashboard の
        Application の Settings にある Domain フィールドで確認できます。[カスタムドメイン](https://auth0.com/docs/custom-domains) を使用している場合は、代わりに
        カスタムドメインの値を設定してください。
      * `clientId`: このクイックスタートの前の手順で設定した Auth0 Application の ID です。これは
        Auth0 Dashboard の Application の Settings にある Client ID フィールドで確認できます。
      * `clientSecret`: このクイックスタートの前の手順で設定した Auth0 Application のシークレットです。これは
        Auth0 Dashboard の Application の Settings にある Client Secret フィールドで確認できます。
      * `redirectUri`: ユーザーの認証後に、Auth0 から
        リダイレクトするアプリケーション内の URL です。これは、このクイックスタートの前の手順で設定したコールバック URL に対応します。この値は
        Auth0 Dashboard の Application の Settings にある Callback URLs フィールドでも確認できます。コードに入力する値が前の手順で設定したものと一致していることを
        必ず確認してください。一致していない場合、ユーザーにエラーが表示されます。
      * `cookieSecret`: セッション cookie の暗号化に使用する長いシークレット値です。適切な
        文字列は、ターミナルで `openssl rand -hex 32` を実行して生成できます。

      <Info>
        ##### チェックポイント

        これで Auth0 SDK は正しく設定されているはずです。アプリケーションを実行して、次の点を確認してください。

        * SDK が正しく初期化されること。
        * アプリケーションで Auth0 に関連するエラーが発生しないこと。
      </Info>

      ユーザーがログインやログアウトできるようになったら、次は認証済みユーザーに関連付けられた[プロフィール情報](https://auth0.com/docs/users/concepts/overview-user-profile)
      を取得できるようにしたいはずです。たとえば、プロジェクト内にログイン中のユーザーの名前や
      プロフィール画像を表示したい場合があります。

      Auth0 PHP SDK では、`getCredentials()` メソッドを通じてユーザー情報を取得できます。使用方法の例は、
      インタラクティブパネルで `profile.php` のコードを確認してください。

      このメソッドにはユーザーの識別情報に関する機密情報が含まれているため、利用できるかどうかは
      ユーザーの認証状態によって決まります。レンダリングエラーを防ぐため、アプリケーションで結果を利用する前に、
      `getCredentials()` メソッドが `object` または `null` を返すかどうかを必ず確認し、
      Auth0 がそのユーザーを認証済みかどうかを判断してください。

      <Info>
        ##### チェックポイント

        次を確認してください。

        * ログイン後に、`nickname` またはその他のユーザープロパティを正しく表示
          できること。
      </Info>
