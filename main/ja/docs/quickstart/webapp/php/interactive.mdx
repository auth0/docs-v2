  <Content>
    Auth0を使用すると、ほぼあらゆる種類のアプリケーションに素早く認証を追加できます。このガイドでは、Auth0 PHP SDKを使用してPHPアプリケーションにAuth0を統合し、認証を追加して、ユーザープロファイル情報を表示する方法を説明します。

    このクイックスタートを使用するには、以下が必要です：

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Auth0 のサービスを利用するには、Auth0 Dashboard でアプリケーションを設定しておく必要があります。Auth0 アプリケーションでは、
      開発中のプロジェクトで認証をどのように機能させるかを設定します。

      ### アプリケーションを設定する

      インタラクティブ セレクターを使用して、新しい Auth0 アプリケーションを作成するか、
      統合するプロジェクトに対応する既存のアプリケーションを選択します。Auth0 のすべてのアプリケーションには、一意の英数字のクライアントIDが割り当てられます。
      アプリケーション コードは、SDK を通じて Auth0 API を呼び出す際にこのクライアントIDを使用します。

      このクイックスタートで行った設定は、[Dashboard](https://manage.auth0.com/#/) 内のアプリケーションに自動的に反映されます。今後はここで
      アプリケーションを管理できます。

      構成済みの完全な設定例を確認したい場合は、代わりにサンプルアプリケーションを参照できます。

      ### コールバック URL を設定する

      コールバック URL とは、認証後に Auth0 がユーザーをリダイレクトする
      アプリケーション内の URL です。これを設定しないと、ユーザーはログイン後にアプリケーションへ戻れません。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを
        `http://localhost:3000/callback` に設定してください。
      </Info>

      ### ログアウト URL を設定する

      ログアウト URL とは、ログアウト後に Auth0 がユーザーをリダイレクトする
      アプリケーション内の URL です。これを設定しないと、ユーザーはアプリケーションからログアウトできず、エラーが発生します。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>

      ### 許可された Web オリジンを設定する

      許可された Web オリジンは、認証フローへのアクセスを許可する
      URL です。ここにはプロジェクトの URL を含める必要があります。これが適切に設定されていないと、プロジェクトは認証トークンをサイレントに更新できなくなるため、
      ユーザーは次回アプリケーションにアクセスしたとき、またはページを更新したときにログアウトされます。

      <Info>
        サンプルプロジェクトに沿って進めている場合は、これを `http://localhost:3000` に設定してください。
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 は、PHP アプリに Auth0 の認証と認可を実装するプロセスを簡素化するための [PHP
      SDK](https://github.com/auth0/auth0-PHP) (Auth0-PHP) を提供しています。

      Auth0 PHP SDK では、ネットワーク リクエストを処理するために、[PSR-17](https://www.php-fig.org/psr/psr-17/) および [PSR-18](https://www.php-fig.org/psr/psr-18/) に準拠した HTTP ライブラリをインストールしておく必要があります。利用可能なライブラリが
      ない場合は、ターミナルで次のコマンドを実行して、信頼性の高いライブラリをインストールできます。

      ```text lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      次に、ターミナルで次のコマンドを実行して、Auth0 PHP SDK をインストールします。

      ```text lines
      composer require auth0/auth0-php
      ```

      ### Auth0 SDK を設定する

      アプリケーションに `index.php` という新しいファイルを作成し、右側のインタラクティブ
      パネルの **index.php** タブにあるコードをコピーします。

      SDK を正しく動作させるには、初期化時に Auth0 SDK で次のプロパティを設定する必要があります。

      * `domain`: Auth0 テナントのドメインです。通常、この値は Auth0 Dashboard の
        Application の Settings にある Domain フィールドで確認できます。[カスタムドメイン](https://auth0.com/docs/custom-domains) を使用している場合は、代わりに
        カスタムドメインの値を設定してください。
      * `clientId`: このクイックスタートで先ほど設定した Auth0 Application の ID です。この値は
        Auth0 Dashboard の Application の Settings にある Client ID フィールドで確認できます。
      * `clientSecret`: このクイックスタートで先ほど設定した Auth0 Application のシークレットです。この値は
        Auth0 Dashboard の Application の Settings にある Client Secret フィールドで確認できます。
      * `redirectUri`: ユーザーの認証後に Auth0 が
        リダイレクトするアプリケーション内の URL です。これは、このクイックスタートで先ほど設定したコールバック URL に対応します。この値は
        Auth0 Dashboard の Application の Settings にある Callback URLs フィールドでも確認できます。コードに入力する値が、
        先ほど設定した値と一致していることを確認してください。一致していない場合、ユーザーにエラーが表示されます。
      * `cookieSecret`: セッションクッキーの暗号化に使用する長いシークレット値です。適切な
        文字列は、ターミナルで `openssl rand -hex 32` を実行して生成できます。

      <Info>
        ##### チェックポイント

        これで Auth0 SDK は正しく設定されているはずです。アプリケーションを実行して、次の点を確認してください。

        * SDK が正しく初期化されること。
        * アプリケーションで Auth0 に関連するエラーが発生していないこと。
      </Info>
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      次に、受信リクエストをアプリケーションにルーティングするためのライブラリをインストールします。これは必須の手順ではありませんが、
      このクイックスタートでは、アプリケーションの構成を簡潔にするために導入します。

      ```text lines
      composer require steampixel/simple-php-router
      ```

      ルートを定義するため、アプリケーションに `router.php` という新しいファイルを作成し、右側のインタラクティブ パネルからコードをコピーします。
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Auth0 アプリケーションと Auth0 PHP SDK の設定が完了したら、次はプロジェクトでログインを設定します。これを行うには、SDK の `login()` メソッドを使用して、ユーザーを Auth0 Universal Login ページにリダイレクトするログインボタンを作成します。ユーザーが正常に認証されると、このクイックスタートで先ほど設定したコールバック URL にリダイレクトされます。

      アプリケーションに `login.php` という新しいファイルを作成してログイン処理を実装し、右側のインタラクティブパネルにあるコードをコピーしてください。このコードには、ログインに必要なロジックが含まれています。

      <Info>
        ##### チェックポイント

        これで、username とパスワードを使用してログインまたはサインアップできるようになります。

        ログインリンクをクリックして、次の点を確認してください。

        * アプリケーションが Auth0 Universal Login ページにリダイレクトされること。
        * ログインまたはサインアップできること。
        * SDK の設定に使用した `redirectUri` の値を使って、Auth0 からアプリケーションにリダイレクトされること。
      </Info>
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      プロジェクトにログインするユーザーには、ログアウトする手段も必要です。ここでは、SDK の
      `logout()` メソッドを使用してログアウトボタンを実装します。ユーザーがログアウトすると、[Auth0 logout](https://auth0.com/docs/api/authentication?http#logout) エンドポイントにリダイレクトされ、
      その後、このクイックスタートの前の手順で設定したログアウト URL にすぐにリダイレクトされます。

      この処理を行うため、アプリケーションに `logout.php` という新しいファイルを作成し、ログアウトに必要なロジックを含むインタラクティブパネルの
      コードをコピーしてください。次に、ログアウトボタンを追加するように
      `index.php` ファイルを更新してください。

      <Info>
        ##### チェックポイント

        アプリケーションを実行してログアウトボタンをクリックし、次のことを確認してください。

        * アプリケーションが、Application Settings の Allowed Logout URLs の 1 つとして指定したアドレスにリダイレクトされること。
        * アプリケーションでログインしていない状態になっていること。
      </Info>
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      ユーザーがログインおよびログアウトできるようになったら、次は認証済みユーザーに関連付けられた[プロフィール情報](https://auth0.com/docs/users/concepts/overview-user-profile)を
      取得したくなるはずです。たとえば、プロジェクト内にログイン中のユーザーの名前や
      プロフィール画像を表示したい場合があります。

      Auth0 PHP SDK では、`getCredentials()` メソッドを通じてユーザー情報を利用できます。使用方法の例については、
      インタラクティブパネルで `profile.php` のコードを確認してください。

      このメソッドにはユーザーのIDに関する機微な情報が含まれるため、利用できるかどうかは
      ユーザーの認証状態によって決まります。レンダリングエラーを防ぐには、
      アプリケーションで結果を利用する前に、`getCredentials()` メソッドが `object` または `null` を返すかどうかを必ず確認し、
      Auth0 によってユーザーが認証されているかどうかを判断してください。

      <Info>
        ##### チェックポイント

        次のことを確認してください。

        * ログイン後に、`nickname` またはその他のユーザープロパティを正しく表示できること。
      </Info>
    </Section>
