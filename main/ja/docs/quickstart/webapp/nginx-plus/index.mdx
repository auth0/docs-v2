このチュートリアルでは、`nginx-openid-connect` モジュールを使用して NGINX サーバーに認証と認可を追加する方法を説明します。アカウント向けに設定された例を使ってこのクイックスタートを進められるよう、事前にログインしておくことをお勧めします。

<Note>
  ### システム要件

  このチュートリアルとシードプロジェクトは、以下の環境でテストされています。

  * NGINX Plus R24
</Note>

**Auth0 および OpenID Connect と連携するように [**NGINX Plus**](https://www.nginx.com/products/nginx/) を使用してアプリケーションを構成するには、以下の手順に従ってください。**

<div id="install-and-enable-nginx-plus-module-njs-module">
  ## nginx-plus-module-njs モジュールのインストールと有効化
</div>

まず、NGINX Plus 用の nginx-plus-module-njs モジュールをインストールする必要があります。ホスト OS にパッケージをインストールするには、[動的モジュールのインストールガイド](https://docs.nginx.com/nginx/admin-guide/dynamic-modules/nginscript)に従ってください。
`yum` パッケージマネージャーを使用する Linux ディストリビューションでは、次のようにインストールします。

```bash lines
sudo yum install nginx-plus-module-njs jq
```

インストール後、NGINX で有効にするには、`/etc/nginx/nginx.conf` ファイルの先頭付近に次の行を追加する必要があります。

```bash lines
load_module modules/ngx_http_js_module.so;
```

<div id="checkout-nginx-openid-connect-template-repository">
  ## nginx-openid-connect テンプレートリポジトリをチェックアウトする
</div>

[`nginx-openid-connect` GitHub リポジトリ](https://github.com/nginxinc/nginx-openid-connect)をクローンします。このリポジトリにはテンプレート設定が含まれています。

```bash lines
git clone https://github.com/nginxinc/nginx-openid-connect
```

<div id="configure-with-your-auth0-application-information">
  ## Auth0 アプリケーション情報を使用して設定する
</div>

Auth0 アプリケーションのテンプレート設定を反映するには、nginx-openid-connect フォルダー内で `configure.sh` スクリプトを実行します。

export const codeExample1 = `./configure.sh --auth_jwt_key request \
  --client_id {yourClientId} \
  --pkce_enable \
  https://{yourDomain}/.well-known/openid-configuration`;

<div id="set-accept-encoding-type-for-token-and-jwks-endpoints">
  ## トークンエンドポイントと JWKS エンドポイントの Accept-Encoding を設定する
</div>

`openid_connect.server_conf` に `Accept-Encoding` ヘッダーを追加します

```conf lines
# openid_connect.server_conf
location = /_jwks_uri {
    internal;
    ...
    proxy_set_header    Content-Length "";           
    proxy_set_header    Accept-Encoding "gzip";          # これは必須です
    ...
}

location = /_token {
    internal;
    ...
    proxy_set_header    Content-Type "application/x-www-form-urlencoded";
    proxy_set_header    Accept-Encoding "gzip";          # これは必須です
    ...
}
```

<div id="copy-openid-connect-config-files-to-nginx-server">
  ## OpenID Connect 設定ファイルを NGINX サーバーにコピーする
</div>

4 つのファイルを NGINX サーバーマシンの設定フォルダーにコピーする必要があります

```sh lines
sudo cp openid_connect.js \ 
   frontend.conf \
   openid_connect_configuration.conf \
   openid_connect.server_conf /etc/nginx/conf.d
```

<div id="configuring-auth0-settings">
  ## Auth0 設定の構成
</div>

アプリケーション設定で、`https://server-fqdn/_codexch` を新しい Allowed Callback URLs として追加します。

次に、Auth0 のアプリケーションで &quot;Token Endpoint Authentication Method&quot; を &quot;None&quot; に変更します。これは PKCE 認可コードフローに必要です。

<div id="passing-headers-to-upstream-application">
  ## アップストリームアプリケーションにヘッダーを渡す
</div>

`/etc/nginx/conf.d/frontend.conf` を編集し、`id_token` から追加のヘッダーをアップストリームターゲットに渡すよう設定します。

```conf lines
# frontend.conf
# auth_jwt_claim_set $claim_name https://namespace/key;

server {
    include conf.d/openid_connect.server_conf; # 認可コードフローとリライングパーティの処理
    error_log /var/log/nginx/error.log debug;  # 必要に応じてログレベルを下げてください

    listen 8010; # 本番環境ではSSL/TLSを使用してください
    
    location / {
        # このサイトはOpenID Connectで保護されています
        auth_jwt "" token=$session_jwt;
        error_page 401 = @do_oidc_flow;

        #auth_jwt_key_file $oidc_jwt_keyfile; # ファイル名を使用する場合は有効にしてください
        auth_jwt_key_request /_jwks_uri; # URLを使用する場合は有効にしてください

        # 認証に成功したユーザーはバックエンドにプロキシされ、
        # 'sub' クレームがHTTPヘッダーとして渡されます
        proxy_set_header username $jwt_claim_sub;
        proxy_set_header x-email $jwt_claim_email;
        #proxy_set_header x-custom $claim_name;             # 名前空間付きクレーム

        proxy_pass http://my_backend; # バックエンドのサイト/アプリ

        access_log /var/log/nginx/access.log main_jwt;
    }
}
```
