Auth0 では、[ボット検知](/ja/docs/secure/attack-protection/bot-detection) 用に複数のサードパーティ製 キャプチャプロバイダーとの統合をサポートしています。

<div id="google-recaptcha-enterprise-on-google-cloud-platform-gcp">
  ## Google Cloud Platform (GCP) での Google reCAPTCHA Enterprise
</div>

Google reCAPTCHA Enterprise は、月間評価件数が 100 万件を超えるトラフィック向けの <Tooltip tip="ボット検知: Auth0 がログインプロセス中に CAPTCHA を有効にすることで、不審なボットトラフィックをブロックする攻撃対策の一種です。" cta="用語集を見る" href="/ja/docs/glossary?term=Bot+Detection">ボット検知</Tooltip> サービスとして使用できます。

GCP で reCAPTCHA Enterprise を設定する方法については、[Google ドキュメントの Set up reCAPTCHA Enterprise ...](https://cloud.google.com/recaptcha-enterprise/docs/authenticate-apis) を参照してください。

<div id="locate-your-gcp-project-id">
  ### GCP のプロジェクト ID を確認する
</div>

1. [Google Cloud Dashboard](https://console.cloud.google.com/) にアクセスします。
2. ドロップダウン メニューでプロジェクトを選択し、対応する **Project ID** を確認します。
3. 後で必要になるため、**Project ID** を控えておいてください。

<div id="enable-recaptcha-enterprise">
  ### reCAPTCHA Enterprise を有効にする
</div>

1. [Google Cloud Dashboard](https://console.cloud.google.com/) で、検索バーに reCAPTCHA Enterprise API と入力し、Marketplace の検索結果から選択します。
2. API を有効にするには、**Enable** を選択します。

<div id="create-a-site-key">
  ### サイトキーを作成する
</div>

1. [Google Cloud Dashboard](https://console.cloud.google.com/) の検索バーに reCAPTCHA Enterprise と入力し、Marketplace の検索結果から選択します。
2. **Create Key** を選択します。
3. **Domain List** で、Auth0 テナントのドメインを追加します。
4. **Disable domain verification** のトグルを有効にします。
5. **Use checkbox challenge** のトグルを有効にして、Google の「私はロボットではありません」確認用チェックボックスを有効にします。
6. **Challenge Security** で、チャレンジの難易度を選択します。
7. **Create Key** を選択します。
8. 後で必要になるため、**Site Key** を控えておきます。

<div id="create-an-api-key">
  ### API キーを作成する
</div>

1. [Google Cloud Dashboard](https://console.cloud.google.com/) で、検索バーに「Credentials」と入力し、検索結果から選択します。
2. **Create Credentials &gt; API Key** を選択します。後で必要になるため、**API Key** は控えておいてください。
3. ポップアップで **Edit Key** を選択します。
4. **Application Restrictions** で **IP addresses (web servers, cron jobs, etc.)** を選択し、[許可リストに追加する Auth0 の IP アドレス](/ja/docs/secure/security-guidance/data-security/allowlist) を入力します。
5. **API Restrictions** で **Restrict Key** を選択し、ドロップダウンリストから **reCAPTCHA Enterprise API** を選択して、OK をクリックします。
6. **Save** を選択します。

<div id="hcaptcha">
  ## hCaptcha
</div>

hCaptcha ウィジェットは、ボットやスパム、その他の自動化された不正利用からアプリケーションを保護できます。

hCaptcha の詳細や設定情報の取得方法については、[hCaptcha Docs の「reCAPTCHA から hCaptcha への切り替え」](https://docs.hcaptcha.com/switch)を参照してください。

<div id="friendly-captcha">
  ## Friendly Captcha
</div>

Friendly Captcha は、Web サイト上のスパムを防止するためのシステムです。ユーザー体験への影響を最小限に抑えながら、Web アプリに Friendly Captcha ウィジェットを追加してスパム対策を行えます。

Friendly Captcha の詳細および設定情報の取得方法については、[Friendly Captcha Docs の Installation](https://docs.friendlycaptcha.com/#/installation?id=installation) を参照してください。

<div id="arkose">
  ## Arkose
</div>

Arkose Labs は、人間によるものを含む不正アクセスやボット攻撃を軽減するためのボット検知ソリューションを提供しています。キャプチャプロバイダーとして Arkose をシームレスに統合する方法について詳しくは、[Arkose Bot Manager](https://www.arkoselabs.com/arkose-bot-manager/) を参照してください。

<Warning>
  Arkose をキャプチャプロバイダーとして使用するには、Auth0.js v9.23 以降にアップグレードする必要があります。
</Warning>

<div id="configure-callback-action-with-auth0js">
  ### Auth0.js でコールバック処理を設定する
</div>

Arkose をキャプチャプロバイダーとして使用する場合、Auth0.js WebAuth クライアントの `renderCaptcha` メソッドにコールバック関数を渡せます。コールバックの 2 番目のパラメーターには、`triggerCaptcha` 関数を含むペイロードオブジェクトが渡されます。`triggerCaptcha` 関数を使用すると、アプリケーションでキャプチャのプロンプトをどのように処理し、レンダリングするかを設定できます。

`renderCaptcha` メソッドの設定方法の詳細については、Auth0.js ドキュメントの [renderCaptcha](https://auth0.github.io/auth0.js/WebAuth.html#renderCaptcha) を参照してください。
