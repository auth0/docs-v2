---

Tenant ACL ルールは、次の要素で構成されます。

* **Signal**: シグナルは、IP アドレス、位置情報、ユーザーエージェントなど、受信リクエストに含まれる識別情報です。

* **Condition**: 条件は、演算子 (`match` など) と値のセット (IP アドレスのリストなど) の組み合わせです。

* **Action**: アクションは、条件が満たされた場合にルールが実行する動作を指し、許可、ブロック、リダイレクトなどがあります。

* **Scope**: スコープは、そのルールが適用されるエンドポイントの範囲を示します。これには、Authentication API、Management API、またはテナント全体が含まれます。

* **Priority**: 優先度は、そのルールが他のルールに対してどの順序で実行されるかを定義します。

これらの Tenant ACL ルールのプロパティと使用方法の詳細については、[Management API Endpoint Parameter Reference](./reference)を参照してください。

<Note>
  テナント ACL では、一貫した[ルール評価ロジック](./rule-evaluation)を使用して、特定のリクエストにどのアクションを適用するかを判定します。

  ユースケースに応じたきめ細かなアクセス制御ポリシーを作成するため、優先度は慎重に割り当ててください。
</Note>

<div id="create-and-delete-rules">
  ## ルールの作成と削除
</div>

次の Management API エンドポイントを使用すると、Tenant ACL ルールを作成および削除できます。指定されたスコープを持つ [Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) が必要です。

| Action  | エンドポイント                                                                        | 必要なスコープ               |
| ------ | ------------------------------------------------------------------------------ | --------------------- |
| ルールを作成 | [アクセス制御リストを作成](/ja/docs/api/management/v2/network-acls/post-network-acls)         | `create:network_acls` |
| ルールを削除 | [アクセス制御リストを削除](/ja/docs/api/management/v2/network-acls/delete-network-acls-by-id) | `delete:network_acls` |

<div id="modify-rules">
  ## ルールを変更
</div>

次の Management API エンドポイントを使用して、Tenant ACL ルールを変更できます。指定されたスコープを持つ [Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) が必要です。

| Action      | エンドポイント                                                                           | 必要なスコープ               |
| ------- | --------------------------------------------------------------------------------- | --------------------- |
| ルールを更新  | [アクセス制御リストを部分的に更新](/ja/docs/api/management/v2/network-acls/patch-network-acls-by-id) | `update:network_acls` |
| ルールを上書き | [アクセス制御リストを更新](/ja/docs/api/management/v2/network-acls/put-network-acls-by-id)       | `update:network_acls` |

<div id="enable-or-disable-rules">
  ### ルールを有効または無効にする
</div>

<Tabs>
  <Tab title="Management API">
    Management API の [Update access control list](https://auth0.com/docs/api/management/v2/network-acls/put-network-acls-by-id) エンドポイントを使用すると、ルールを有効または無効にできます。

    `rule` オブジェクトの `active` プロパティを、`true` または `false` に設定します。
  </Tab>

  <Tab title="Dashboard">
    Dashboard でルールを有効または無効にするには、[Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control) に移動します。

    既存のルール一覧では、各ルールの横にあるボタンを切り替えることで、特定のルールを有効または無効にできます。

    ![Auth0 Dashboard で Tenant ACL ルールを有効または無効にする](/docs/images/cdy7uua7fh8z/29b92e102142d3cb20e69a40e292f6b7/Screenshot_2025-09-03_at_12.34.05.png)

    **Status** 列には、ルールのステータス (**Active** または **Disabled**) が表示されます。
  </Tab>
</Tabs>

<div id="view-rules">
  ## ルールを表示
</div>

<Tabs>
  <Tab title="Management API">
    以下の Management API エンドポイントを使用して、既存のTenant ACL ルールを表示できます。指定されたスコープを持つ [Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) が必要です。

    | Action         | エンドポイント                                                                                 | 必要なスコープ             |
    | ---------- | --------------------------------------------------------------------------------------- | ------------------- |
    | ルールを表示     | [テナントの特定のアクセス制御リストエントリを取得](/ja/docs/api/management/v2/network-acls/get-network-acls-by-id) | `read:network_acls` |
    | すべてのルールを表示 | [テナントのすべてのアクセス制御リストエントリを取得](/ja/docs/api/management/v2/network-acls/get-network-acls)      | `read:network_acls` |
  </Tab>

  <Tab title="Dashboard">
    既存のルールの一覧を表示するには、[Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control) に移動します。

    | 列名    | 説明                                                                          | 関連する API レスポンスフィールド |
    | ----- | --------------------------------------------------------------------------- | ------------------- |
    | 優先度  | ルールが評価される順序を示す序数                                                            | `priority`          |
    | 名前    | ルールの目的または機能の説明                                                              | `description`       |
    | Action | ルールが実行する[アクション](/ja/docs/secure/tenant-access-control-list/reference#actions)。 | `action`            |
    | ステータス | ルールが有効か無効かを示す値。                                                             | `active`            |
    | 最終更新  | ルールが最後に更新された日時。                                                             | `updated_at`        |

    ![Auth0 Dashboard で Tenant ACL ルールの一覧を表示](/docs/images/cdy7uua7fh8z/4093f9a39aba48bdf8d579c134dc4da5/Screenshot_2025-09-03_at_12.33.47.png)
  </Tab>
</Tabs>
