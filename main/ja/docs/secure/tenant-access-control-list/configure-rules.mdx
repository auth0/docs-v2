  テナント ACL ルールを設定するには、次のスコープを持つ [Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) が必要です。

Auth0 の<Tooltip tip="Management API: お客様が管理タスクを実行するための製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip>を使用すると、テナント ACL ルールを設定できます。

<div id="available-actions">
  ## 利用可能な操作
</div>

Management API を使用すると、テナント ACL ルールの表示、作成、更新、削除を実行できます。

| 操作         | エンドポイント                                                                                 | 必要なスコープ               |
| ---------- | --------------------------------------------------------------------------------------- | --------------------- |
| ルールを表示     | [テナントの特定のアクセス制御リストエントリを取得](/ja/docs/api/management/v2/network-acls/get-network-acls-by-id) | `read:network_acls`   |
| すべてのルールを表示 | [テナントのすべてのアクセス制御リストエントリを取得](/ja/docs/api/management/v2/network-acls/get-network-acls)      | `read:network_acls`   |
| ルールを作成     | [アクセス制御リストを作成](/ja/docs/api/management/v2/network-acls/post-network-acls)                  | `create:network_acls` |
| ルールを更新     | [アクセス制御リストを部分的に更新](/ja/docs/api/management/v2/network-acls/patch-network-acls-by-id)       | `update:network_acls` |
| ルールを上書き    | [アクセス制御リストを更新](/ja/docs/api/management/v2/network-acls/put-network-acls-by-id)             | `update:network_acls` |
| ルールを削除     | [アクセス制御リストを削除](/ja/docs/api/management/v2/network-acls/delete-network-acls-by-id)          | `delete:network_acls` |

<div id="parameters">
  ## パラメーター
</div>

テナント ACL のパラメーターと使用方法の詳細については、[Reference](./reference)を参照してください。

<ParamField path="description" type="string" required>
  ルールの目的または機能を説明します。

  例: `Only allow requests originating from the United States`
</ParamField>

<ParamField path="active" type="boolean" required>
  ルールを有効または無効にします。

  例: `true`
</ParamField>

<ParamField path="priority" type="number" required>
  ルールが評価される順序を決定する数値です。

  例: `1`
</ParamField>

<ParamField path="rule" type="object" required>
  ルールの条件とアクションが含まれます。

  <Expandable>
    <ParamField path="action" type="object" required>
      ルールが実行するアクションが含まれます。

      例: `{ allow: true }`
    </ParamField>

    <ParamField path="match" type="object">
      受信リクエストが満たす必要がある条件を定義します。

      例: `{ geo_country_codes: ["US"] }`
    </ParamField>

    <ParamField path="not_match" type="object">
      受信リクエストが満たしてはならない条件を定義します。

      例: `{ geo_country_codes: ["CN"] }`
    </ParamField>

    <ParamField path="scope" type="string" required>
      ルールが適用されるサービスまたはコンテキストです。

      例: `authentication`
    </ParamField>
  </Expandable>
</ParamField>

<div id="toggle-monitoring-mode-for-a-rule">
  ## Rule の監視モードを切り替える
</div>

`log` プロパティを `rule.action` オブジェクトに追加し、その値を `true` に設定します。
