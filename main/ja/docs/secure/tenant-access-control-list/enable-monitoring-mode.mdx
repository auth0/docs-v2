---
title: テナントアクセス制御リストの Rule のモニタリングモードを有効または無効にする
sidebarTitle: モニタリングモードを有効化
description: モニタリングモードを有効にすると、テナント ACL の現在の動作を変更せずに、Rule が受信トラフィックにどのような影響を与えるかを監査できます。
---

Rule がモニタリングモードの場合、テナント ACL はその [Rule を評価](./rule-evaluation) し、[テナントログイベント](./tenant-log-event-reference) を生成しますが、Rule のアクションは実行せず、後続の Rule やリストの評価も終了しません。

モニタリングモードを有効にするには、Rule の action プロパティ `log` を `true` に設定します。

<Tabs>
  <Tab title="Management API">
    次のボディを指定して、Management API の[アクセス制御リストエンドポイントの部分更新](/ja/docs/api/management/v2/network-acls/patch-network-acls-by-id)を呼び出します。

    ```json Management API request body highlight={4}
    {
      "rule": {
        "action": {
          "log": true
        },
        "scope": "authentication"
      }
    }
    ```
  </Tab>

  <Tab title="Go SDK">
    ```go highlight={20}
    package main

    import (
    	"context"
    	"log"

    	"github.com/auth0/go-auth0"
    	"github.com/auth0/go-auth0/management"
    )

    func main() {
    	mgmt, err := management.New("{yourDomain}", management.WithClientCredentials("{yourClientId}", "{yourClientSecret}"))
    	if err != nil {
    		log.Fatal(err)
    	}

    	networkACL := &management.NetworkACL{
    		Rule: &management.NetworkACLRule{
    			Action: &management.NetworkACLRuleAction{
    				Log: auth0.Bool(true),
    			},
    			Scope: auth0.String("authentication"),
    		},
    	}

    	err = mgmt.NetworkACL.Patch(context.Background(), "{yourTenantAclRuleId}", networkACL)
    	if err != nil {
    		log.Fatal(err)
    	}
    	log.Println("Network ACL has been updated to enable monitoring mode")
    ```
  </Tab>

  <Tab title="Node SDK">
    ```javascript highlight={4}
    const updateNetworkAclPayload: Management.UpdateNetworkAclRequestContent = {
      rule: {
        action: {
          log: true,
        },
        scope: "authentication"
      }
    };

    const updateNetworkAcl = await client.networkAcls.update("{yourTenantAclRuleId}", updateNetworkAclPayload);
    ```
  </Tab>

  <Tab title="Terraform">
    リソースブロックの [`rule.action`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/network_acl#nested-schema-for-ruleaction) で、`log` を `true` に設定します。

    ```terraform highlight={7}
    resource "auth0_network_acl" "example_acl" {
        description = "Example ACL"
        active = true
        priority = 1
        rule {
            action {
                block = true
                log = true
            }
            scope = "authentication"
            match {
              hostnames = ["example.com"]
            }
        }
    }
    ```
  </Tab>

  <Tab title="Deploy CLI">
    ```yaml highlight={8}
    exampleACLs:
      - description: Example ACL
        active: true
        priority: 1
        rule:
          action:
            block: true
            log: true
          match:
            hostnames:
              - "example.com"
          scope: authentication
    ```
  </Tab>

  <Tab title="Auth0 CLI">
    ```bash
    auth0 network-acl update {yourTenantAclRuleId} --action log
    ```
  </Tab>
</Tabs>

同様に、`log` プロパティを `false` に設定すると、モニタリングモードを無効にできます。
