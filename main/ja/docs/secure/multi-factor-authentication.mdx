<Card title="概要">
  多要素認証 (MFA) は、アプリケーションとユーザーのアイデンティティを保護するための追加のセキュリティ対策です。MFA では、ログイン時にセキュリティ層が 1 つ追加され、ユーザーはデジタルアイデンティティを証明するために複数の認証情報を提示する必要があります。認証要素には、次のようなものがあります。

  * ユーザー自身に関するもの - 生体認証など
  * ユーザーが知っているもの - パスワードなど
  * ユーザーが所有しているもの - デバイスなど

  この Q&amp;A を読んで、Auth0 テナントで MFA を使用することが適切かどうかを確認してください。
</Card>

<div id="what-is-multi-factor-authentication">
  ## 多要素認証とは何ですか？
</div>

<Tooltip tip="多要素認証（MFA）: ユーザー名とパスワードに加えて、SMS で送信されるコードなどの認証要素も使用するユーザー認証プロセス。" cta="用語集を表示" href="/ja/docs/glossary?term=Multi-factor+authentication">多要素認証</Tooltip> (MFA) は、複数の認証手段による確認を必要とする本人確認の方法です。ユーザー名とパスワードを入手された場合でも、<Tooltip tip="悪意のある攻撃者: 害を及ぼす意図を持って、ビジネスや環境に脅威をもたらす存在（個人またはグループ）。" cta="用語集を表示" href="/ja/docs/glossary?term=bad+actors">悪意のある攻撃者</Tooltip>によるアカウントへのアクセスを防ぎます。

<div id="why-use-multi-factor-authentication">
  ## 多要素認証を使用する理由
</div>

MFA は、多くの種類のサイバー攻撃の成功率を低下させます。第三者がユーザー名やパスワードを盗んだり、プログラムによってユーザーアカウントに攻撃を仕掛けたりすることは珍しくありません。指紋やワンタイムパスワードなどの MFA の追加の認証要素があれば、こうした不正行為を防ぐことができます。

<div id="how-does-multi-factor-authentication-work">
  ## 多要素認証はどのように機能しますか？
</div>

MFA では、追加の確認情報 (認証要素) による検証が求められます。ユーザーは、ユーザー名とパスワードだけではログインできません。顔認証やテキストメッセージによる通知など、本人確認のための追加の証明を提示する必要があります。

<div id="mfa-factors">
  ## MFA factors
</div>

<Warning>
  MFA ファクターの利用可否はプランによって異なり、一部は Professional プランおよび Enterprise プランでのみ利用できます。詳しくは、[Auth0 Pricing](https://auth0.com/pricing) を参照してください。
</Warning>

Auth0 では、次のようなさまざまな MFA ファクターをサポートしています。

* プッシュ通知
* SMS 通知
* 音声通知
* ワンタイムパスワード
* Security Keys を使用する WebAuthn
* Device Biometrics を使用する WebAuthn
* メール通知
* Cisco Duo セキュリティ
* リカバリーコード

詳しくは、[多要素認証の認証要素](/ja/docs/secure/multi-factor-authentication/multi-factor-authentication-factors) を参照してください。

<div id="enable-mfa">
  ## MFA を有効にする
</div>

MFA を有効にする方法については、[多要素認証を有効にする](/ja/docs/secure/multi-factor-authentication/enable-mfa)を参照してください。

<div id="customize-mfa">
  ## MFA をカスタマイズする
</div>

[Auth0 Actions](/ja/docs/customize/actions) を使用して、MFA フローをカスタマイズすることもできます。特定の状況でのみ MFA を必須にしたり、特定の認証要素の使用を必須にしたりできます。
