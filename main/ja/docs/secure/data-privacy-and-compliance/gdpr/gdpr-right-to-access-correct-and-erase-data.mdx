GDPR の第15条、第16条、第17条、および第19条に基づき、ユーザーは、処理中の自身の個人データの写しを取得し、そのデータが不正確な場合は訂正を求め、個人データの削除を要求する権利を有します。

Auth0 では、Dashboard を使用して手動で、または <Tooltip tip="Management API: お客様が管理タスクを実行するための製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を使用してプログラムから、ユーザー情報へのアクセス、編集、削除を行えます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  これらのドキュメントの内容は、法的助言を目的としたものではなく、法的支援に代わるものでもありません。GDPR を理解し、準拠する最終的な責任はお客様にありますが、Auth0 は可能な限り GDPR 要件への対応を支援します。
</Callout>

<div id="manual-process">
  ## 手動で行う場合
</div>

[Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users) では、ユーザー情報の表示、編集、削除を行えます。ユーザーを選択すると、そのユーザーの情報を表示できます。変更できる情報は次のとおりです。

| 項目              | 編集方法                                                                                                               |
| --------------- | ------------------------------------------------------------------------------------------------------------------ |
| **メールアドレス**     | **Edit** をクリックします。新しいメールアドレスを設定します。                                                                                |
| **メールアドレス確認済み** | **Εmail** フィールドの **Edit** をクリックします。**Set email as verified** リンクをクリックします。                                          |
| **メタデータ**       | `app_metadata` と `user_metadata` の両方のオブジェクトをこの画面から編集できます。**Metadata** セクションで JSON を編集し、変更を保存します。                   |
| **ブロック済み**      | 直接編集することはできません。この画面の右上にある **Actions &gt; Block User** をクリックします。ブロックを解除するには、**Actions &gt; Unblock User** をクリックします。 |
| **メールアドレス**     | 直接編集することはできません。この画面の右上にある **Actions &gt; Change Email** をクリックします。                                                  |
| **パスワード**       | 直接編集することはできません。この画面の右上にある **Actions &gt; Change Password** をクリックします。                                               |

ユーザーを削除するには、**Actions &gt; Delete User** をクリックします。

<div id="programmatic-process">
  ## プログラムによる処理
</div>

API を使用して、ユーザー情報の取得、更新、削除を行えます。要件に合ったエンドポイントを選択してください。

* ID を検索条件としてユーザーを取得する
* メールアドレスを検索条件としてユーザーを取得する
* 長時間実行されるジョブを使用して、すべてのユーザーをファイルにエクスポートする
* ユーザーを更新する。なお、すべてのフィールドを編集できるわけではありません (次のセクションを参照) 。また、次の点に注意してください。

  * 新しいオブジェクトのプロパティは、古いプロパティを置き換えます。`user_metadata` フィールドと `app_metadata` フィールドはこのルールの例外です。これらのプロパティは置き換えではなくマージされますが、マージされるのは第 1 階層のみです。
  * `email_verified`、`phone_verified`、`username`、または `password` を更新する場合は、`connection` パラメータを設定する必要があります。
  * `email` または `phone_number` を更新する場合は、`connection` パラメータと `client_id` パラメータを設定する必要があります。
* ID に基づいてユーザーを削除する

API エンドポイントを呼び出すには、有効な <Tooltip tip="アクセストークン: 不透明な文字列または JWT の形式をとる認可資格情報で、API へのアクセスに使用されます。" cta="用語集を表示" href="/ja/docs/glossary?term=access+token">アクセストークン</Tooltip> が必要です。このトークンには、エンドポイントごとに必要な権限が含まれている必要があります。Management API エクスプローラーでは、各エンドポイントに、そのエンドポイントへのアクセスにアクセストークンに含まれている必要があるスコープを示す section があります。アクセスするエンドポイントを特定し、有効なアクセストークンを取得したら、リクエストを送信できます。

<div id="editable-data">
  ## 編集可能なデータ
</div>

次のユーザー情報は、API を使用して更新できます。

* blocked
* email&#95;verified
* email
* verify&#95;email
* password
* phone&#95;number
* phone&#95;verified
* verify&#95;password
* user&#95;metadata
* app&#95;metadata
* username

次のユーザー情報は**編集できません**。

* given&#95;name
* family&#95;name
* name
* nickname
* picture

<div id="searchable-fields">
  ## 検索可能なフィールド
</div>

以下を使用してユーザーを検索できます。

* 正規化されたすべてのユーザープロファイル フィールド
* `user_metadata` オブジェクト配下のプロファイル情報:

  * name
  * nickname
  * given&#95;name
  * family&#95;name

Auth0 に接続されていないその他のデータベースについては、顧客の情報が確実に削除または更新されるようにする責任はお客様にあります。
