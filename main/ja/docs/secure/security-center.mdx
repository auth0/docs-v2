Security Center は、潜在的な攻撃傾向を把握し、リアルタイムで迅速に対応できるオブザーバビリティツールを提供します。Security Center では、Customer Identity and Access Management (CIAM) の異常検知メトリクスをリアルタイムで監視できるほか、同じ場所から攻撃緩和機能を設定することもできます。

<div id="real-time-monitoring">
  ## リアルタイムモニタリング
</div>

Security Center では、テナントのセキュリティ状況をリアルタイムで把握でき、Auth0 Management Dashboard 内から対処を実行できます。Security Center では、次のことができます。

* 総トラフィック量と脅威の総数を監視する
* 脅威行動の傾向を把握する
* 脅威行動の傾向に関連するアプリケーションを特定する
* ログインとサインアップのトラフィックを追跡する
* <Tooltip tip="Attack Protection: Auth0 が提供する、ブルートフォース保護、不審な IP のスロットリング、漏えいしたパスワードの検出、ボット検出、アダプティブ多要素認証など、攻撃を検出して緩和するための機能。" cta="用語集を見る" href="/ja/docs/glossary?term=Attack+Protection">Attack Protection</Tooltip> と <Tooltip tip="Attack Protection: Auth0 が提供する、ブルートフォース保護、不審な IP のスロットリング、漏えいしたパスワードの検出、ボット検出、アダプティブ多要素認証など、攻撃を検出して緩和するための機能。" cta="用語集を見る" href="/ja/docs/glossary?term=MFA">MFA</Tooltip> 機能によって検出された脅威を監視する

<div id="filtering-and-aggregation">
  ### フィルタリングと集計
</div>

Security Center では、利用可能なデータを必要に応じて絞り込めます。

データは、次の項目でフィルタリングできます。

* 期間 (最大で過去 14 日間)
* アプリケーション
* 接続

選択した期間に応じて、データは 1 分単位、1 時間単位、または 1 日単位で自動的に集計されます。

<div id="threat-behavior-trends">
  ### 脅威行動の傾向
</div>

Security Center では、次の脅威タイプについて脅威行動の傾向を確認できます。

* **Credential stuffing**: ユーザーアカウントの侵害を目的として認証情報を送信しようとする、マシンによる試行が関与していると見られる行動パターン。
* **Signup attack**: 新しいユーザーアカウントの作成を目的とする、マシンによる試行が関与していると見られる行動パターン。
* **MFA bypass**: ユーザーの多要素認証 (MFA) 保護の回避を目的とする、マシンによる試行が関与していると見られる行動パターン。

ビューでは、脅威タイプ別にデータを切り分けて、脅威行動の傾向に関連するアプリケーションを特定できます。

<div id="authentication-events">
  ### 認証イベント
</div>

Security Center では、ログイン試行やサインアップ試行などの認証イベントを確認できます。

<div id="login-attempts">
  #### ログイン試行
</div>

ログイン試行には、過去7日間のログイン トランザクションのうち、成功したものと失敗したものの両方が含まれます。

<div id="signup-attempts">
  #### サインアップ試行
</div>

サインアップ試行には、過去7日間における成功したサインアップ トランザクションと失敗したサインアップ トランザクションの両方が含まれます。

<div id="attack-protection-and-mfa-monitoring">
  ### Attack Protection と MFA のモニタリング
</div>

Security Center では、Attack Protection と MFA の機能によって特定された現在の攻撃傾向を把握でき、これらの機能を有効にして設定することで対策を講じることができます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Attack Protection 機能の設定時に対策を選択したかどうかにかかわらず、既知の攻撃タイプや関連するログイベントの兆候となることが多いパターンを特定します。
</Callout>

<Frame>![スクリーンショットには、Security Center の Threat Protection タブに複数の折れ線グラフが表示されています。](/docs/images/cdy7uua7fh8z/Threat_Monitoring/Threat_Monitoring.png)</Frame>

<div id="bot-detection">
  #### ボット検知
</div>

<Tooltip tip="Bot Detection: Auth0 がログイン処理中に CAPTCHA を有効にして、ボットと疑われるトラフィックをブロックする攻撃対策の一種です。" cta="用語集を見る" href="/ja/docs/glossary?term=Bot+detection">ボット検知</Tooltip>は、リクエストがボットによるものの可能性が高い場合にそれを検出し、スクリプト化された攻撃を軽減します。ボット検知では、過去 7 日間に検出されたボットの数も確認できます。

この機能の詳細については、[Bot Detection](/ja/docs/secure/attack-protection/bot-detection)を参照してください。

<div id="suspicious-ip-throttling">
  #### 不審な IP スロットリング
</div>

<Tooltip tip="Suspicious IP Throttling: 単一の IP アドレスから多数のアカウントを標的とした不審なログインからテナントを保護する、攻撃防御機能の一種です。" cta="用語集を見る" href="/ja/docs/glossary?term=Suspicious+IP+throttling">不審な IP スロットリング</Tooltip> は、短時間に過剰な回数のログインまたはサインアップを試行する IP アドレスからのトラフィックをブロックします。不審な IP スロットリングでは、過去 7 日間にブロックされた不審な IP の数も確認できます。

この機能の詳細については、[Suspicious IP Throttling](/ja/docs/secure/attack-protection/suspicious-ip-throttling) を参照してください。

<div id="brute-force-protection">
  #### ブルートフォース対策
</div>

<Tooltip tip="ブルートフォース対策: 単一のIPアドレスから単一のユーザーアカウントを標的にしたブルートフォース攻撃から保護する攻撃対策の一種です。" cta="用語集を表示" href="/ja/docs/glossary?term=Brute-force+protection">ブルートフォース対策</Tooltip>は、単一のIPアドレスによる単一のユーザーアカウントへの攻撃を防ぎます。ブルートフォース対策には、過去7日間にブロックされたブルートフォース攻撃の試行回数が含まれます。

この機能の詳細については、[Brute-Force Protection](/ja/docs/secure/attack-protection/brute-force-protection)を参照してください。

<div id="breached-password-detection">
  #### 漏えいパスワードの検知
</div>

<Tooltip tip="漏えいパスワードの検知: Auth0 が、サードパーティの Web サイトまたはアプリで発生したデータ漏えいにより漏えいした username とパスワードの組み合わせをユーザーが使用した場合に通知する、攻撃対策機能の一種です。" cta="用語集を見る" href="/ja/docs/glossary?term=Breached+password+detection">漏えいパスワードの検知</Tooltip> は、盗まれた認証情報を使って <Tooltip tip="漏えいパスワードの検知: Auth0 が、サードパーティの Web サイトまたはアプリで発生したデータ漏えいにより漏えいした username とパスワードの組み合わせをユーザーが使用した場合に通知する、攻撃対策機能の一種です。" cta="用語集を見る" href="/ja/docs/glossary?term=bad+actors">悪意のある攻撃者</Tooltip> がサインアップまたはログインするのを防ぎ、アプリケーションを保護します。漏えいパスワードの検知には、過去 7 日間のログインフローおよびサインアップフローで検出された漏えい認証情報の件数が含まれます。

この機能の詳細については、[Breached Password Detection](/ja/docs/secure/attack-protection/breached-password-detection) を参照してください。

<div id="multi-factor-authentication">
  #### 多要素認証
</div>

多要素認証 (MFA) では、複数の認証要素を要求してユーザーを検証します。MFA には、過去 7 日間に検出された MFA チャレンジの数と、成功または失敗した MFA チャレンジの数が含まれます。

この機能の詳細については、[Multi-Factor Authentication](/ja/docs/secure/multi-factor-authentication) を参照してください。
