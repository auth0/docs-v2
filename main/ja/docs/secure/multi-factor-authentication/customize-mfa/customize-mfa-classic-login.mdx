title: Classic Login の MFA をカスタマイズする
---

[Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa) セクションで、<Tooltip tip="多要素認証（MFA）: SMS で送信される code など、username と password に加えて別の要素を使用するユーザー認証プロセスです。" cta="用語集を表示" href="/ja/docs/glossary?term=multi-factor+authentication">多要素認証</Tooltip>を有効にできます。<Tooltip tip="多要素認証（MFA）: SMS で送信される code など、username と password に加えて別の要素を使用するユーザー認証プロセスです。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> の基本的なブランディングオプションを使用して、MFA ページをカスタマイズできます。さらにカスタマイズする場合は、MFA ページに独自の HTML を指定できます。独自の HTML を使用する場合、Classic Login は Auth0 MFA Widget を使用しますが、次の制限があります。

* メールアドレスを使用した MFA には対応していません。
* ユーザーが複数の要素を登録している場合、どの要素を使用するかは選択できません。MFA ウィジェットは、最も安全性の高い要素でログインするよう求めます。
* <Tooltip tip="Universal Login: アプリケーションは、ユーザーの本人確認のために、Auth0 の認可サーバーでホストされる Universal Login にリダイレクトされます。" cta="用語集を表示" href="/ja/docs/glossary?term=Universal+Login">Universal Login</Tooltip> の国際化機能は使用されません。詳しくは、[Universal Login の国際化](/ja/docs/customize/internationalization-and-localization/universal-login-internationalization) を参照してください。

<div id="customize-html-for-the-mfa-page">
  ## MFA ページの HTML をカスタマイズする
</div>

MFA ページをカスタマイズするには、[Dashboard &gt; ブランディング &gt; Universal Login &gt; Multi-factor Authentication](https://manage.auth0.com/#/mfa_page) タブに移動し、**Customize MFA Page** トグルを有効にします。その後、テキストエディターを使用して HTML を変更したり、CSS でページのスタイルを設定したり、カスタム変数の取得に使用する JavaScript を変更したりできます。変更が完了したら、必ず **Save Changes** をクリックしてください。

MFA ページはカスタマイズしなくても動作し、Auth0 は必要に応じて組み込まれているスクリプトを更新します。ただし、カスタマイズを有効にすると、Auth0 は自動更新できなくなるため、スクリプト (MFA Widget のバージョン番号の変更を含む) の更新と保守はお客様の責任となります。

Classic Login の MFA プロセスのさまざまな箇所のテキストを上書きする方法について詳しくは、[MFA Theme Language Dictionary](/ja/docs/secure/multi-factor-authentication/customize-mfa/mfa-theme-language-dictionary) を参照してください。MFA Widget のテーマオプションについて詳しくは、[MFA Widget Theme Options](/ja/docs/secure/multi-factor-authentication/customize-mfa/mfa-widget-theme-options) を参照してください。

<div id="reset-page">
  ## ページをリセット
</div>

以前のデザインに戻すには、次の 2 つの方法があります。

* **Reset to Last** をクリックして、最後に保存したテンプレートに戻します。
* **Reset to Default** をクリックして、Auth0 が提供するデフォルトのテンプレートに戻します。

<div id="render-enrollment-invitations">
  ## 登録招待を表示する
</div>

このページが表示されるケースは、2 つあります。ユーザーが登録のためにこのページへ直接誘導された場合 (たとえば、登録リンクが記載されたメールアドレスから) には、プロパティ `ticket` を利用できます。それ以外の場合は、プロパティ `requestToken` を利用できます。

<div id="html-and-liquid-syntax">
  ## HTML と Liquid の構文
</div>

ホストページでは、テンプレートに Liquid 構文を使用します。詳細については、GitHub の [Liquid for Designers](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) を参照してください。ページのレンダリングに役立つ次のパラメーターを利用できます。

* `userData.email`
* `userData.friendlyUserId`
* `userData.tenant`
* `userData.tenantFriendlyName`
* `iconUrl`

MFA Widget で使用するパラメーターの大半は、カスタマイズ領域で提供されるデフォルトテンプレートに示されているように、Guardian に渡す必要があります。より高度なカスタマイズが必要な場合は、[auth0-guardian.js](https://github.com/auth0/auth0-guardian.js/tree/master/example) を使用できます。

詳細については、[Customize MFA User Pages](/ja/docs/secure/multi-factor-authentication/customize-mfa) を参照してください。
