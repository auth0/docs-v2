Security Center は、既知の攻撃タイプの兆候であることが多いパターンを特定するために、テナントログイベントを使用します。テナントログイベントのパターンは、クレデンシャルスタッフィング の脅威、サインアップ攻撃の脅威、および <Tooltip tip="多要素認証（MFA）: SMS 経由の code など、username とパスワードに加えて認証要素を使用するユーザー認証プロセス。" cta="用語集を表示" href="/ja/docs/glossary?term=MFA">MFA</Tooltip> 回避の脅威というカテゴリに分類されます。

<Warning>
  イベントタイプ code の分類は変更される場合があります。現在のログイベント code の定義に依存するソリューションは実装しないでください。
</Warning>

<div id="credential-stuffing">
  ## クレデンシャルスタッフィング
</div>

次のイベントコードを使用して、1 時間以内のクレデンシャルスタッフィングの脅威を特定します。

| イベントコード     | イベント                                             |
| ----------- | ------------------------------------------------ |
| `f`         | ユーザーのログイン失敗                                      |
| `fu`        | 無効なユーザー名によるログイン失敗                                |
| `fp`        | 無効なパスワードによるログイン失敗                                |
| `pwd_leak`  | 漏えいしたパスワードを使用したログイン試行                            |
| `limit_wc`  | 1 つのアカウントに対するログイン失敗が 10 回を超えたため、IP をブロック         |
| `limit_sul` | 同一 IP アドレスから 1 分間に 20 回を超えるログインがあったため、ユーザーをブロック  |
| `limit_mu`  | ログイン失敗が 100 回を超えたか、サインアップ試行が 50 回を超えたため、IP をブロック |

<div id="signup-attack">
  ## サインアップ攻撃
</div>

次のイベントコードを使用して、1時間以内に発生したサインアップ攻撃の脅威を特定します。

| イベントコード | イベント     |
| ------- | -------- |
| `fs`    | サインアップ失敗 |

<div id="mfa-bypass">
  ## MFA バイパス
</div>

次のイベントコードを使用して、1 時間以内に MFA バイパスの脅威を特定します。

| イベントコード                         | イベント               |
| ------------------------------- | ------------------ |
| `gd_send_email`                 | メール送信              |
| `gd_send_pn`                    | プッシュ通知送信           |
| `gd_send_sms`                   | SMS 送信             |
| `gd_send_voice`                 | 音声通話の発信            |
| `gd_auth_failed`                | OTP 認証の失敗          |
| `gd_auth_rejected`              | OTP 認証の拒否          |
| `gd_otp_rate_limit_exceed`      | OTP 認証の失敗回数が上限を超過  |
| `gd_recovery_failed`            | リカバリーの失敗           |
| `gd_recovery_rate_limit_exceed` | リカバリーの失敗回数が上限を超過   |
| `gd_webauthn_challenge_failed`. | WebAuthn ブラウザー エラー |
