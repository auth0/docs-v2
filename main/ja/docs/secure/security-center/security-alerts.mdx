title: セキュリティ監視アラートの設定
---

**セキュリティ監視アラート**では、ユーザーが Security Center でセキュリティメトリクスのしきい値を設定できます。各しきい値に対して、アラート、警告、回復の数値を指定できるほか、脅威メトリクスが設定したしきい値を超えた際に通知するユーザー向けアラート通知を設定できます。

| オプション | 説明                                                      |
| ----- | ------------------------------------------------------- |
| アラート  | 評価対象のメトリクスが指定した値を超えたときに、アラート通知を生成する必須の数値です。             |
| 警告    | 指定した場合、評価対象のメトリクスがその値を超えたときに、警告通知を生成する任意の数値です。          |
| 回復    | 指定した場合、評価対象のメトリクスがしきい値超過ではない状態に戻ったときに、回復通知を生成する任意の数値です。 |

<div id="configure-and-update-alert-thresholds">
  #### アラートしきい値の設定と更新
</div>

しきい値は、対象メトリックの加重移動平均に基づいて計算され、<Tooltip tip="Auth0 のサービスを設定するための主要な製品である Auth0 Dashboard。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> でカスタマイズできます。定義した各しきい値は脅威モニターのメトリックチャートで確認でき、1 時間単位で集計されます。回復しきい値が設定されていない場合、回復値のデフォルトは、設定した警告またはアラートのしきい値をわずかに下回る値になります。

<Warning>
  しきい値は、フィルターが適用されていない元のメトリックに対して適用されます。フィルターを適用すると、元のしきい値とフィルター適用後のトレンドラインが表示されます。
</Warning>

<Frame>![Security Center のしきい値機能の 2 枚目のスクリーンショット](/docs/images/cdy7uua7fh8z/1Kd9d33WUCdlVa1SwEFkKQ/496722ce86bc00cc7f58d1e04e1fd006/MFA_success_rate_threshold_editor_-_english.png)</Frame>

1. [**Security &gt; Security Center &gt; Threat Monitoring**](https://manage.auth0.com/#/security/center/bot-detection) に移動し、メトリックチャートを選択します。
2. 右上にある **View Details** アイコンを選択します。
3. 詳細チャートビューの下に表示される Thresholds パネルに移動し、**Create** を選択します。
4. しきい値に名前を付けて、次の設定を構成します。

   * しきい値が警告をトリガーする条件
   * しきい値がアラートをトリガーする条件
   * しきい値が回復する条件
5. 通知先が設定されている場合は、次の操作を実行できます。

   * 通知先を選択して、メトリックのアラート、警告、回復の通知を受信する
   * **+** を選択して新しい通知先を作成する
   * **Mute Notifications** ドロップダウンで、すべてのしきい値通知先への通知を一時的または無期限にミュートする
6. **Save** を選択します。

しきい値は、展開ビュー画面で更新または削除することもできます。同じチャート上の別のしきい値には、右上の Threshold ラベルのキャレットからアクセスできます。

<div id="manage-notification-destinations">
  #### 通知先を管理する
</div>

通知先は、アラート、警告、回復の通知を配信するエンドポイントです。各テナントで設定できる送信先エンドポイントは 2 つまでです。通知メッセージをカスタマイズするには、サードパーティ製の webhook エディターを使用することを推奨します。

1. Dashboard の Thresholds 設定パネルから、または [**Security &gt; Security Center &gt; Manage Destinations**](https://manage.auth0.com/#/security/center/manage-destinations) に移動して、Dashboard の Manage Destinations ページを開きます
2. **New Destination** ボタンを選択し、次の情報を入力します。

   1. 名前
   2. 送信先 URL
   3. 認可トークン
3. **Save** を選択します。送信先エンドポイントを削除するには、**More Actions &gt; Delete** に移動します。

<div id="notification-payload-data">
  #### 通知ペイロードデータ
</div>

通知ペイロードに含まれるルートレベルのフィールドは次のとおりです。

* `id`: Security Center アラートの ID
* `time`: しきい値の超過が発生した UTC の日時
* `source`: Auth0 Security Center Alert History ダッシュボードの URL
* `data`: アラート通知に関連する情報を含むデータオブジェクト

以下は、通知ペイロードに含まれる `data` オブジェクトです。

* `id`: Security Center アラートの ID
* `tenant`: Security Center 通知の送信元であるテナント
* `evaluated_metric`: しきい値が適用される脅威メトリクス
* `state`: 通知の状態 (ALERT/WARN/RECOVERED)
* `metric_value`: 直前 60 分間における評価対象メトリクスの値
* `alert_threshold`: 脅威メトリクスに設定されたアラートしきい値の値
* `warn_threshold`: 脅威メトリクスに設定された警告しきい値の値
* `recovery_threshold`: 脅威メトリクスに設定された回復しきい値の値
* `triggered_at`: しきい値の超過が発生した UTC の日時

<div id="view-alert-history">
  #### アラート履歴を表示
</div>

発生したアラート、警告、回復の通知は、[**Security &gt; Security Center &gt; Alert History**](https://manage.auth0.com/#/security/center/alert-history) で確認できます。すべての通知は、設定された通知先にも送信されます。
