description: セキュリティ自動化を保護する 優先ログ Streams について詳しく説明します。
title: Prioritized Log Streams
---

**優先ログ Streams** は、セキュリティイベントを優先的に扱う Dashboard の機能です。攻撃発生時や、その他の要因でユーザーアクティビティが増加した場合でも、優先ログ Streams と事前定義されたイベントは引き続き有効なため、後続のセキュリティ自動化やプロセスに影響しません。

サポート対象のサービスを使用して新しい優先ログストリームを作成することも、すでに利用しているサービスにストリーミングするためのカスタム Webhook を設定することもできます。さらに、すべてのログをイベント分析サービスにエクスポートできます。優先ログ Streams は、料金プランに記載されているログストリーム数の上限にはカウントされません。

<div id="configure-a-prioritized-log-stream">
  #### 優先ログストリームを設定する
</div>

1. [ログストリームページ](/ja/docs/customize/log-streams) の手順に従って、新しいログストリームを設定します。
2. 新しいストリームの設定中に、Auth0 Dashboard の **Settings** タブに移動し、以下で説明する定義済みのセキュリティイベントを有効にするため、**優先ログ** チェックボックスを選択します。続いて **Save** を選択します。
3. **Stream** ページに移動し、**Prioritized** ラベルを開きます。

<div id="prioritized-logs-event-types">
  #### 優先ログイベントタイプ
</div>

次のイベントコードは、Auth0 によってセキュリティ関連イベントとして分類されています。

| イベントコード           | イベント                       |
| ----------------- | -------------------------- |
| `limit_mu`        | ブロックされた IP アドレス            |
| `limit_sul`       | ブロックされたアカウント (IP スロットリング)  |
| `limit_wc`        | ブロックされたアカウント (ブルートフォース保護)  |
| `pwd_leak`        | ログイン時の漏えいパスワード             |
| `ublkdu`          | ユーザーのログインブロック解除            |
| `signup_pwd_leak` | サインアップ時の漏えいパスワード           |
| `reset_pwd_leak`  | リセット時の漏えいパスワード             |
| `gd_send_sms`     | MFA SMS 送信                 |
