title: IDトークンを検証する
---

ユーザープロフィールの属性を含む[IDトークン](/ja/docs/secure/tokens/id-tokens)は、アプリで利用され、通常はユーザーインターフェースの表示に使用されます。Auth0 は、すべての<Tooltip tip="IDトークン: リソースへのアクセスではなく、クライアント自身を対象とした認証情報です。" cta="用語集を見る" href="/ja/docs/glossary?term=ID+tokens">IDトークン</Tooltip>を[JSON Web Token (JWT) ](/ja/docs/secure/tokens/json-web-tokens)形式で発行します。

これらのチェックのいずれかに失敗した場合、そのトークンは無効と見なされるため、リクエストは拒否する必要があります。

1. [JWT を検証します](/ja/docs/secure/tokens/json-web-tokens/validate-json-web-tokens)。
2. 追加の標準クレームを確認します。標準の JWT 検証を実行していれば、[JWT のペイロード](/ja/docs/secure/tokens/json-web-tokens/json-web-token-structure)はすでにデコード済みで、標準クレームも確認しています。IDトークンで追加で検証する必要があるクレームは次のとおりです。

   * **トークンのオーディエンス** (`aud`, string): トークンのオーディエンス値は、[アプリケーションの設定](https://manage.auth0.com/#/applications)の **Client ID** フィールドで定義されているアプリケーションのクライアントIDと一致している必要があります。
   * **Nonce** (`nonce`, string): リプレイ攻撃の防止に役立つため、トークンリクエストでは nonce を渡すことを推奨します (Implicit Flow では必須) 。トークン内の nonce 値は、リクエストで送信した元の nonce と完全に一致している必要があります。詳しくは、[リプレイ攻撃を軽減する](/ja/docs/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post/mitigate-replay-attacks-when-using-the-implicit-flow)を参照してください。
