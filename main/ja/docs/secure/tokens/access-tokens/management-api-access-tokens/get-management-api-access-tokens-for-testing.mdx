title: テスト用のManagement APIアクセストークンを取得する
---

<Warning>
  このアクセストークンの取得方法は、**テスト目的にのみ**使用してください。手動で長期間有効なトークンを取得してアプリケーションで使用すると、トークンが持つセキュリティ上の利点が失われます。
</Warning>

<Tooltip tip="Management API: お客様が管理タスクを実行するための製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> のテストトークンを最初に取得するのは、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> での設定を完了したときです。新しいテナントを作成しない限り、これを再度行う必要はありません。既存のトークンを再利用するのではなく、Management API へのアクセス認可専用のテストトークンを作成することをお勧めします。

<div id="prerequisites">
  ## 事前準備
</div>

* [テスト用の Machine-to-Machine アプリケーションを作成する](/ja/docs/get-started/apis/create-m2m-app-test)

<div id="get-access-tokens-manually">
  ## アクセストークンを手動で取得する
</div>

1. Auth0 Dashboard で **Applications &gt; APIs &gt;** [Auth0 Management API](https://manage.auth0.com/#/apis/management/explorer) に移動します。**API Explorer** タブを選択し、**Token** セクションで自動生成されたトークンを確認します。
2. トークンの右側にある **Copy** アイコンを選択します。これで、このトークンを使用して [Management API](https://auth0.com/docs/api/management/v2) に対する認可された呼び出しを行えるようになります。

   <Frame>![APIs - Management API アクセストークン - テスト - トークンをコピー](/docs/images/cdy7uua7fh8z/3Y4I9YJR77j73FCYDiegnC/44f32f76d27357bc39aad9ee7c4d5561/management-api-access-token.png)</Frame>
3. 必要に応じて有効期限を設定します。デフォルトでは、トークンの有効期限は 86400 秒 (24 時間) です。この期間を過ぎると、トークンは期限切れになり使用できなくなります。有効期限を変更するには、**Token Expiration (Seconds)** を更新し、**Update &amp; Regenerate Token** をクリックします。

<Warning>
  これらのトークンは**失効できない**ため、有効期限を長く設定することは推奨されません。代わりに、有効期限は短く設定し、必要な都度新しいトークンを発行することを推奨します。[jwt.io](https://jwt.io/) を使用してトークンをデコードすると、有効期限を確認できます。
</Warning>

<div id="use-access-tokens-for-testing">
  ## テストにアクセストークンを使用する
</div>

テスト用に、作成したばかりの <Tooltip tip="アクセストークン: 不透明な文字列または JWT の形式をとる認可資格情報で、API へのアクセスに使用されます。" cta="用語集を見る" href="/ja/docs/glossary?term=Access+Token">アクセストークン</Tooltip> を使用して、[Management API Explorer](https://auth0.com/docs/api/management/v2) からエンドポイントを手動で呼び出せます。

1. [Management API Explorer](https://auth0.com/docs/api/management/v2#!) にアクセスし、**Set API Token** ボタンを選択します。
2. **API Token** フィールドに入力します。すると、アプリケーションに付与されたスコープなどの追加情報がポップアップに表示されます。
3. **Set Token** を選択してトークンを保存し、ポップアップを閉じます。**Set API Token** ボタンの表示が **Authorized** に変わったことを確認します。
4. テストを開始するには、呼び出すエンドポイントに移動し、右側のフィールドに必要なパラメーターまたは任意のパラメーターを入力します。次に、**Test Endpoint** を選択します。
