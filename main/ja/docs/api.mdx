<div class="api_card_wrapper">
  <Card title="Authentication API" href="/ja/docs/api/authentication" arrow />
</div>

<Columns cols={2}>
  <Column>
    Authentication API は、Auth0 およびサポートされている ID プロトコル (OpenID Connect、OAuth、SAML など) の認証・ID 機能を提供します。

    通常、この API は [Auth0.js](/ja/docs/libraries/auth0js) などの Auth0 SDK や、[Lock](/ja/docs/libraries/lock) のようなライブラリを通じて利用します。ただし、認証 UI を手動で構築する場合は、Authentication API を直接呼び出す必要があります。
  </Column>

  <Column>
    たとえば、次のようなタスクがあります。

    * 認証時に [tokens](/ja/docs/secure/tokens) を取得する
    * [Access Token](/ja/docs/secure/tokens/access-tokens) を使用してユーザープロファイルを取得する
    * [Refresh Tokens](/ja/docs/secure/tokens/refresh-tokens) を新しい Access Token に交換する
    * [multi-factor authentication (MFA)](/ja/docs/secure/multi-factor-authentication) のチャレンジを要求する
  </Column>
</Columns>

<div class="api_card_wrapper">
  <Card title="Management API" href="/ja/docs/api/management/v2" arrow />
</div>

<Columns cols={2}>
  <Column>
    Management API を使用すると、Auth0 アカウントをプログラムから管理できるため、環境設定を自動化できます。Auth0 Management Dashboard で実行できるタスクの多くは、この API を使ってプログラムから実行することもできます。
  </Column>

  <Column>
    たとえば、次のようなタスクがあります。

    * アプリケーションと API を Auth0 に登録する
    * ユーザーが認証に使用できる [connections](/ja/docs/connections) を設定する
    * [ユーザーを管理する](/ja/docs/manage-users)
    * [ユーザーアカウントをリンクする](/ja/docs/manage-users/user-accounts/user-account-linking/link-user-accounts)
  </Column>
</Columns>

<div class="api_card_wrapper">
  <Card title="My Account API" href="/ja/docs/api/myaccount" arrow />
</div>

<Columns cols={2}>
  <Column>
    Auth0 My Account API は、ユーザーが自分のアカウント情報を管理するための専用エンドポイント群を提供します。これらの API を使用して、アプリケーションにセルフサービス機能を構築したり、ユーザーアカウントに詳細情報を段階的に追加したりできます。
  </Column>

  <Column>
    たとえば、次のようなタスクがあります。

    * エンドユーザーが自分の認証要素を管理できるようにする
    * 認証方法の詳細を更新する
    * 利用可能な認証方法を確認する
  </Column>
</Columns>

<div class="api_card_wrapper">
  <Card title="My Organization API" href="/ja/docs/api/myorganization" arrow />
</div>

<Columns cols={2}>
  <Column>
    Auth0 My Organization API は、Organization スコープの安全なインターフェースを提供し、ビジネス顧客が Auth0 テナント内で自社の Organizations を管理できるようにします。この API は、埋め込み型の委任管理や API ファーストの統合を支える技術的基盤として機能します。詳細については、[My Organzation API and Embeddable UI Components](/ja/docs/manage-users/my-organization-api) を参照してください。
  </Column>

  <Column>
    たとえば、次のようなタスクがあります。

    * Auth0 Organization の詳細 (名前、ブランディング、表示名)
    * Organization 固有の設定、所有権、関係
    * Identity Providers (IdPs) と SCIM プロビジョニング
    * ドメインと Home Realm Discovery (HRD) の設定
  </Column>
</Columns>

<div id="pagination">
  ## ページネーション
</div>

Public Cloud テナントでは、Management API エンドポイントが 1 回のレスポンスで返す結果は最大 50 件です。1 回のレスポンスで返せる件数を超える結果がある場合は、より多くの結果を取得するために、リクエストに `page` と `per_page` パラメーターを含めてください。

アプリケーションの更新方法については、[Migrate to Management API v2 Endpoint Paginated Queries](/ja/docs/troubleshoot/product-lifecycle/past-migrations/migrate-to-paginated-queries) を参照してください。
