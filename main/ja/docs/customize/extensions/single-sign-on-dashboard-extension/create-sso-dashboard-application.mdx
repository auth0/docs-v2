複数のエンタープライズアプリケーションに対するユーザーの SSO ログインを管理するには、<Tooltip tip="シングルサインオン（SSO）: ユーザーが 1 つのアプリケーションにログインすると、他のアプリケーションにも自動的にログインできるようにするサービス。" cta="用語集を表示" href="/ja/docs/glossary?term=Single+Sign-On">シングルサインオン</Tooltip> (SSO) Dashboard Extension を使用します。詳細については、[GitHub のこの拡張機能を参照してください](https://github.com/auth0-extensions/auth0-sso-dashboard-extension)。

<div id="create-auth0-application">
  ## Auth0アプリケーションを作成する
</div>

[SSO Dashboard 拡張機能を追加する](/ja/docs/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension)前に、Auth0でアプリケーションを作成する必要があります。

1. [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) に移動し、**+Create Application** を選択します。
2. アプリケーションにわかりやすい名前 (例: SSO Dashboard) を入力し、**Application Type** で **Single-Page Web Application** を選択して、**Create** を選択します。
3. **Settings** タブで、テナントのリージョンに応じて **Allowed Callback URLs** を設定します。

   1. **Admins** の場合: `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/admins/login`
   2. **Users** の場合: `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/login`

      | Region | Region Domain  |
      | ------ | -------------- |
      | US-1   | us.webtask.io  |
      | US-3   | us.webtask.run |
      | US-4   | us.webtask.run |
      | US-5   | us.webtask.run |
      | EU     | eu.webtask.io  |
      | EU-2   | eu.webtask.run |
      | AU     | au.webtask.io  |
      | JP-1   | jp.webtask.run |
      | UK-1   | uk.webtask.run |
      | CA-1   | ca.webtask.run |
4. **Client ID** の値をコピーします。
5. ページの下部に移動し、**Advanced Settings** を選択します。
6. **OAuth** タブを選択し、**Client ID** の値を **Allowed APPs / APIs** フィールドに貼り付けます。
7. **JsonWebToken Signature Algorithm** を **RS256** に設定します。
8. **Save Changes** を選択します。

デフォルトでは、ユーザーが SSO Dashboard にログインできるように、すべての接続タイプが有効になっています。変更する場合は、このアプリケーションの **Connections** タブに移動します。
