<Card title="開始する前に">
  1. Auth0 で [SSO Dashboard アプリケーションを作成](/ja/docs/customize/extensions/single-sign-on-dashboard-extension/create-sso-dashboard-application)します。
  2. SSO Dashboard アプリケーションの **クライアントID** の値をコピーします。
</Card>

<Tooltip tip="Single Sign-On（SSO）: ユーザーが 1 つのアプリケーションにログインすると、他のアプリケーションにも自動的にログインできるようにするサービスです。" cta="用語集を見る" href="/ja/docs/glossary?term=Single+Sign-On">シングルサインオン</Tooltip> (SSO) Dashboard Extension を使用すると、複数のエンタープライズアプリケーションでユーザーの SSO ログインを管理できます。詳細については、[GitHub のこの拡張機能を参照してください](https://github.com/auth0-extensions/auth0-sso-dashboard-extension)。

<div id="install-extension">
  ## 拡張機能をインストールする
</div>

<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> から SSO Dashboard 拡張機能をインストールできます。

1. [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) に移動し、提供されている拡張機能の一覧から **SSO Dashboard** を選択します。**Install Extension** ウィンドウが開きます。

   <Frame>![Dashboard - Extensions - SSO Dashboard - 拡張機能をインストール](/docs/images/cdy7uua7fh8z/75heEyuJAN6FGSszXx06Hm/1cf10550c770beb56731285709442afb/SSO_Extension_-_EN.png)</Frame>
2. 次の設定変数を指定します。

   | 変数                    | 必須 | 説明                                                     |
   | --------------------- | -- | ------------------------------------------------------ |
   | `EXTENSION_CLIENT_ID` | 必須 | 拡張機能用に作成した Auth0 アプリケーションの **クライアントID**。               |
   | `TITLE`               | 任意 | SSO Dashboard ページの上部に表示されるカスタムタイトル。                    |
   | `CUSTOM_CSS`          | 任意 | SSO Dashboard ページの見た目をカスタマイズするための CSS ファイルへのパス。        |
   | `FAVICON_PATH`        | 任意 | カスタム favicon へのパス。                                     |
   | `AUTH0_CUSTOM_DOMAIN` | 任意 | Auth0 で設定した[カスタムドメイン](/ja/docs/customize/custom-domains)。 |

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     `AUTH0_CUSTOM_DOMAIN` 変数を設定しても拡張機能の URL には影響せず、拡張機能で使用される認可エンドポイントの URL だけが変更されます。カスタムドメインを使用すると、拡張機能にログインするユーザーは、デフォルトの `https://{yourTenant}.{yourRegion}.auth0.com/login` ではなく `https://{yourAuth0CustomDomain}/login` にリダイレクトされます。
   </Callout>
3. **Install** を選択します。

[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) に移動すると、`auth0-sso-dashboard` という名前のアプリケーションが自動的に作成されていることがわかります。このアプリケーションは、拡張機能が <Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> へのアクセスを認可するために使用されるため、変更しないでください。

<div id="use-extension">
  ## 拡張機能を使用する
</div>

1. [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) に移動し、**Installed Extensions** ビューを選択します。
2. **SSO Dashboard** を選択します。インストール済みの拡張機能を初めて選択すると、必要な権限を付与するよう求められます。同意すると、カスタム **SSO Dashboard** に移動し、ページ上部に指定した **タイトル** が表示されます。カスタム CSS ファイルを指定している場合は、そのスタイルも適用されます。
3. SSO Dashboard にログインするには、次の URL を使用します。

   1. **管理者** の場合:
      `https://{yourTenant}.YOUR_REGION_KEY.webtask.io/auth0-sso-dashboard/admins/login`
   2. **ユーザー** の場合:
      `https://{yourTenant}.YOUR_REGION_KEY.webtask.io/auth0-sso-dashboard/login`

| リージョン | リージョンドメイン      |
| ----- | -------------- |
| US-1  | us.webtask.io  |
| US-3  | us.webtask.run |
| US-4  | us.webtask.run |
| US-5  | us.webtask.run |
| EU    | eu.webtask.io  |
| EU-2  | eu.webtask.run |
| AU    | au.webtask.io  |
| JP-1  | jp.webtask.run |
| UK-1  | uk.webtask.run |
| CA-1  | ca.webtask.run |
