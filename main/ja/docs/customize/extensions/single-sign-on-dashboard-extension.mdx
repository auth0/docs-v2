<Tooltip tip="シングルサインオン（SSO）: ユーザーが1つのアプリケーションにログインすると、他のアプリケーションにも自動的にログインできるようにするサービスです。" cta="用語集を見る" href="/ja/docs/glossary?term=Single+Sign-on">シングルサインオン</Tooltip> (SSO) Dashboard は、多くの組織が抱える共通の課題を解決するために設計された Web アプリケーションです。規模を問わず、組織では会計、人事、開発、サポートなど、さまざまな業務機能に対応するために多種多様なアプリケーションを運用しています。すべてのアプリケーションのユーザー名、パスワード、ログイン URL を覚えておくのは煩雑です。この拡張機能を使用すると、複数のエンタープライズアプリケーションでユーザーの SSO ログインを有効にし、認証エクスペリエンスを簡素化できます。また、ユーザーが SSO ログインで認証できるすべてのクラウドサービスの一覧を作成できます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ユーザーは、適切な権限 (グループ/ロール) を持たない限り、ダッシュボード経由かどうかにかかわらず、サービスプロバイダーにアクセスできないようにする必要があります。理想的には、アクセス権が付与されていないサービスプロバイダーは、ダッシュボード上に表示されないようにする必要があります。
</Callout>

SSO Dashboard は、次の 2 種類のユーザーをサポートしています。

* **ユーザー**: ダッシュボードにログインして、SSO でサインインするアプリケーションを選択できます。
* **管理者**: ユーザーに表示するアプリケーションを設定できます。

この拡張機能を設定するには、次の手順を実行します。

1. [SSO Dashboard アプリケーションを作成する](/ja/docs/customize/extensions/single-sign-on-dashboard-extension/create-sso-dashboard-application)
2. [SSO Dashboard 拡張機能をインストールする](/ja/docs/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension)
3. [SSO Dashboard にアプリケーションを追加する](/ja/docs/customize/extensions/single-sign-on-dashboard-extension/add-applications-to-the-sso-dashboard)
4. [SSO Dashboard 上のアプリケーションを更新する](/ja/docs/customize/extensions/single-sign-on-dashboard-extension/update-applications-on-the-sso-dashboard)

また、[GitHub でこの SSO Dashboard Extension を確認する](https://github.com/auth0-extensions/auth0-sso-dashboard-extension)こともできます。
