Authentication API Debugger 拡張機能を使用すると、Auth0 Authentication API の各種エンドポイントをテストできます。

<div id="install-the-extension">
  ## 拡張機能をインストールする
</div>

この拡張機能をインストールするには、次の手順に従います。

1. [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) に移動し、**Auth0 Authentication API Debugger** を選択します。**拡張機能のインストール**ウィンドウが開きます。
2. **インストール** を選択します。

<div id="authorize-the-extension">
  ## 拡張機能を認可する
</div>

拡張機能は、<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> とユーザーに代わって通信し、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> で設定したアプリケーションの詳細を取得するため、そのアクセスを認可する必要があります。

拡張機能を認可するには:

1. このガイドの手順どおりに進めている場合は、すでに [Auth0 Dashboard &gt; Extensions](https://manage.auth0.com/#/extensions) の [Installed Extensions](https://manage.auth0.com/#/extensions) ビューが表示されているはずです。拡張機能を起動するには、**Auth0 Authentication API Debugger** を選択します。アカウントへのアクセスを求める同意ダイアログが表示されます。
2. 拡張機能がアカウントにアクセスすることに同意します。同意すると、拡張機能が起動します。

<div id="configure-the-extension">
  ## 拡張機能を設定する
</div>

拡張機能をインストールしたら、基本的な設定を行います。拡張機能を設定するには、次の手順に従います。

1. このガイドに従っている場合は、すでに拡張機能を起動しているはずです。**Configuration** ビューを選択します。

   <Frame>![Dashboard - Extensions - Authentication API Debugger - Configuration](/docs/images/cdy7uua7fh8z/5riI24ArqUAKRXhZfNWIQ5/ab7d6825288752132d14e54db1e563d0/configuration.png)</Frame>
2. 次の設定パラメーターを指定し、**Save** を選択します。

* **Domain**: テナントのドメインです。このフィールドは読み取り専用で、参考情報として表示されます。
* **Application**: 認証フローを開始する対象の Application です。アプリケーション は [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) で管理できます。
* **Callback URL**: この拡張機能のコールバック URL です。この URL は、選択した Application の設定にある **Allowed Callback URLs** にも追加する必要があります。
* **State**: 認証フローとともに送信できるオプションの state 情報です。
* **Connection**: ログインに使用する接続名です。このパラメーターを使用すると、メインのログイン画面をスキップして、該当する <Tooltip tip="IDプロバイダー（IdP）: デジタルアイデンティティを保存および管理するサービス。" cta="用語集を表示" href="/ja/docs/glossary?term=Identity+Provider">IDプロバイダー</Tooltip> のログイン画面に直接移動できます。

<div id="execute-the-flows">
  ## フローを実行する
</div>

拡張機能を設定したら、**OAuth2/OIDC** ビューを選択して、任意のフローを実行できます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  フローによっては、基本設定に加えて追加の設定が必要になる場合があります。必要な設定は実行するフローによって異なるため、各フローで使用できるパラメーターの詳細については、[Auth0 Authentication API ドキュメント](/ja/docs/api)を参照してください。
</Callout>

<Frame>![Dashboard - Extensions - Authentication API Debugger - OAuth OIDC 設定](/docs/images/cdy7uua7fh8z/72iYPxZU9OiHkKkHtgPTjI/fc166f3ae74e595c8d916276e78edb99/oauth-oidc-settings.png)</Frame>

特定のフローを実行すると、Auth0 から返された情報を表示する画面が表示されます。表示される情報は、実行したフローによって異なります。通常の **OAuth2/OIDC Login** フローを実行した場合の結果例を次に示します。

<Frame>![Dashboard - Extensions - Authentication API Debugger - フロー実行結果](/docs/images/cdy7uua7fh8z/13TRS8SQUd4OsfLYRWMoe6/3478778905ad253160022db1f1f6f31a/flow-executed.png)</Frame>

別のフローを実行するには、**Login** ビューを選択して **Configuration** 画面に戻ります。
