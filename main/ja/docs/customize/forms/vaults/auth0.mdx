<div id="get-access-to-auth0-management-api">
  ## Auth0 Management API へのアクセスを取得する
</div>

Auth0 Management API へのアクセスが許可された [Machine-to-Machine (M2M) Application](/ja/docs/get-started/auth0-overview/create-applications/machine-to-machine-apps) を作成します。

<Warning>
  M2M Application が Management API にアクセスできるように許可し、最小権限の原則に従って適切なスコープを選択することを忘れないでください。

  * ユーザーの取得: `read:users`
  * ユーザーの作成: `create:users`
  * ユーザーの更新: `update:users`
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0テナントで設定済みのプロバイダーを通じて通知を送信する場合、Auth0 Management API の認可は不要です。
</Callout>

<div id="configure-your-vault-connection">
  ## Vault Connection を設定する
</div>

Auth0 Flow Actions 用の Auth0 Vault Connection を設定するには、事前に作成した M2M Application に基づいて、必要なパラメーターを設定します。

<Frame>![Dashboard > Actions > Forms > Vault > Auth0](/docs/images/customize/forms/vaults/auth0-vault-connection.png)</Frame>

<div id="input-settings">
  #### 入力設定
</div>

| Parameter           | Description                                                  |
| ------------------- | ------------------------------------------------------------ |
| Tenant Domain       | M2M Application のテナントドメイン。                                       |
| Client ID           | Auth0 Management API へのアクセスを許可された M2M Application のクライアント ID。    |
| Client Secret       | Auth0 Management API へのアクセスを許可された M2M Application のクライアントシークレット。 |
| Audience (Optional) | Auth0 Management API の Audience。                             |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  デフォルトでは、Audience は Tenant Domain をもとに設定されます。
</Callout>
