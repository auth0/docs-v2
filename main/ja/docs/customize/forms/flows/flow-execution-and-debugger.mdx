title: 実行とデバッガー
---

フローが実行されるたびに、Auth0 はその実行をデバッグするために使用できるエントリを保存します。これは、フローの実行エラーが発生した理由を把握するのに役立ちます。たとえば、プロパティの不足、タイプミス、誤った API キーなどが原因で、フローが失敗することがあります。

<div id="review-flow-executions">
  ## フロー実行の確認
</div>

フローの実行エントリを確認するには、次の手順に従います。

1. **Auth0 Dashboard &gt; Actions &gt; Forms** に移動し、**Flows** を選択します。
2. 一覧からフローを選択します。
3. **Executions** を選択します。

<Frame>![Dashboard > Actions > Forms > Flows > Execution](/docs/images/cdy7uua7fh8z/448jfjHtgCxfIeDOakuHyd/fba64deda0cf920f399de2190e94bdc9/Flows-execution.png)</Frame>

各実行エントリには 4 つのプロパティがあります。

* **Execution date**: フローが実行された日付です。
* **Execution ID**: フロー実行の一意の ID です。
* **Status**: フロー実行の現在の段階です。実行ステータスには次の 4 種類があります。

  * **Running**: 実行は現在進行中です。
  * **Failed**: フロー内のActionに必要な設定が不足しているなど、内部エラーにより実行が失敗しました。
  * **Completed**: 内部エラーなしで実行が完了しました。
* **View Details**: 実行のデバッグウィンドウを開きます。

<Frame>![Dashboard > Actions >  Forms > Flows > Executions](/docs/images/cdy7uua7fh8z/1VqWEaVZg5lrXatXe9dbFB/dbcf7edca5aabdfc544f04cb18be4149/Flow-execution-details.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Completed と Failed は最終ステータスです。実行がこれらのいずれかのステータスに達すると、フローの処理は完了しています。
</Callout>

<div id="debug-flow-executions">
  ## フロー実行をデバッグする
</div>

フローのデバッグやトラブルシューティングを行うには、実行の **View Details** にアクセスして、フローの入力、出力、エラーデータを確認できます。

<Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/5aeniTfHXplDyZfwCKBcgK/a2098a476edde76da10a527b0a606e95/Flow-error.png)</Frame>

フローの実行を中断するエラーをデバッグするには、次の手順に従います。

1. **Auth0 Dashboard &gt; Actions &gt; Forms** に移動します。
2. ステップノードに関連付けられているフローを見つけます。

   <Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/4LjULYyo0nhtNowm7J5okz/27ea3f8386bb5015d447c48facf6b90f/flow-error-linked.png)</Frame>
3. **Edit Flow** を選択します。
4. **Executions** を選択します。
5. 失敗した実行の **View Details** を選択します。

以下の例では、`Update user` の **Input** ビューに、`user_id` プロパティの値として null が表示されています。

<Frame>![Dashboard > Actions > Forms > Flow > Execution](/docs/images/cdy7uua7fh8z/6QNYKDqG9AIOEzQkP6Fnnr/db885e146179a0490eddd5ad6cb518dc/Flow-execution-failed.png)</Frame>

**Error** ビューでは、必須の `user_id` が不足しているために Action のスキーマ検証が失敗したことが示されています。フローは内部エラーを返し、その結果、フローの実行が中断されます。

<Frame>![Dashboard > Actions > Forms > Flow > Execution > Error](/docs/images/cdy7uua7fh8z/1kaKihtJtv8VvNAyTkW4cC/6b34bcecc8642d7d97aaf2c8dec51ebc/flow-execution-error-view.png)</Frame>

**Update user** Action を確認すると、`User ID` パラメーターに構文エラーがあり、正しい構文は `{{context.user.user_id}}` です。構文を修正したら、フローをテストして確認します。変数の構文の詳細については、[Variables](/ja/docs/customize/forms/variables) を参照してください。

<Frame>![Dashboard > Actions > Forms > Flows](/docs/images/cdy7uua7fh8z/4YLRVNbzK5LjJEQjFXIJXM/5003833784ce5b0d9f63730ed46ff047/Action-typo-error.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  フィールドが transient として設定されている場合や、その値が [Helper function](/ja/docs/customize/forms/variables#helper-functions) でマスクされている場合、その値はデバッグ時には利用できません。

  <Frame>![Dashboard > Actions > Forms > Flows > Execution](/docs/images/cdy7uua7fh8z/4MVtjnVD50cG63ocN5a458/c0337a94d1f4c21f28d06e28ec59121e/masked-information.png)</Frame>
</Callout>
