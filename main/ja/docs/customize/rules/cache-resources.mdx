Rules では、`global` オブジェクトを使用して高コストなリソースを保存できます。`global` オブジェクトは個々の実行をまたいで保持されるため、Rule の実行ごとに毎回作成し直す代わりに、そこに保存したものを再利用できます。ただし、Rules の実行環境はいつでも再初期化される可能性があるため、コードでは **必ず** `global` に想定どおりの内容が含まれていることを確認する必要があります。

次の例では、`global` オブジェクトを使って MongoDB 接続を保持する方法を示します。

```javascript lines
//dbオブジェクトが存在する場合は使用する。
if (global.db){
  return query(global.db, callback);
}

//存在しない場合はdbを取得する（この場合はmongodb）
mongo('mongodb://user:pass@mymongoserver.com/my-db',  function (db){
  global.db = db;
  return query(db, callback);
});

//実際の処理を実行する
function query(db, cb){
  // dbを使って処理を行う
  ...
  return cb(null, user, context);
}
```

MongoDB の使用方法について詳しくは、[MongoDB Manual](https://docs.mongodb.com/manual/) を参照してください。
