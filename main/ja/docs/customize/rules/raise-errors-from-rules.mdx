Auth0 のログインページでは、Rules からのエラーメッセージを表示できません。Rules で発生させたカスタムメッセージをユーザーに表示したい場合は、アプリケーション側でそのメッセージを表示する必要があります。

Rules によって発生したエラーは、プロトコルの標準レスポンスモードでコールバック URL に送られます。たとえば、アプリケーションが OIDC/OAuth2 を使用して認証をリクエストすると、`error` と `error_description` の結果値を含む標準的な OAuth2 エラーを受け取ります。

コールバック URL が [https://travel0.com/callback](https://travel0.com/callback) で、標準のクエリレスポンスモードを使用しているとします。この場合、次のレスポンスを受け取ります。

```http wrap lines
https://travel0.com/callback?error=access_denied&error_description=[The message you wrote in the rule]
```

ユーザーにメッセージを表示するのは、アプリの役割です。たとえば、次のようになります。

<div id="retry-login-after-a-rule-error">
  ## Rules エラーの後にログインを再試行する
</div>

次に考えられる手順は、新たに認可をリクエストすることです。ただし、ほとんどの場合、Auth0 はすでにユーザーを識別しているため、ログイン情報の入力を求めません。代わりに、Rules がただちに実行され、再び認可が拒否されます。その結果、無限ループに陥ることがあります。

ユーザーには、新しい認証情報を入力する機会が必要です。アプリでログイン ページを再表示するには、2 つの方法があります。

* ログイン ページを強制的に表示するよう、リクエストに指定を追加します。

  * OIDC/OAuth2 リクエストで `prompt=login` パラメーターを使用します。
  * SAML リクエストで `ForceAuthn="true"` 属性を使用します。

—or—

* 新しいリクエストを送信する前に、ユーザーを Auth0 からログアウトさせます。
