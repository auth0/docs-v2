メールプロバイダー、またはご利用中のメールサーバーの SMTP サーバー詳細を使用して、テナントを設定できます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  お使いのメールプロバイダーを Auth0 が直接サポートしている場合は、より安全な認証プロトコルに対応しているため、SMTP サーバー詳細ではなくその統合を使用することをお勧めします。
</Callout>

<div id="smtp-server-requirements">
  ## SMTP サーバーの要件
</div>

SMTP サーバーは、次の要件を満たす必要があります。

* LOGIN プロトコルを使用する [SMTP AUTH](https://en.wikipedia.org/wiki/SMTP_Authentication) をサポートしていること。
* [TLS 1.2](https://en.wikipedia.org/wiki/STARTTLS) 以降をサポートしていること。
* 公開証明機関 (CA) によって署名された証明書を使用していること。
* [Auth0 の IP アドレス](/ja/docs/secure/security-guidance/data-security/allowlist) からの受信接続を許可していること。

<div id="configure-tenants-with-an-smtp-server">
  ## SMTP サーバーを使用してテナントを設定する
</div>

Auth0 Dashboard または Management API を使用して、テナントに SMTP サーバーを設定できます。

<div id="using-the-auth0-dashboard">
  ### Auth0 Dashboard を使用する
</div>

Auth0 Dashboard を使用してテナントに SMTP サーバーを設定するには、次の手順に従います。

1. [Auth0 Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email-provider) に移動します。
2. **Use my own email provider** トグルを有効にします。
3. **SMTP** を選択します。
4. **From** のメールアドレスを入力し、次に SMTP サーバーの **Host**、**Port**、**Username**、**Password** を入力します。

   利用可能であれば、現在の既定の SMTP ポートであり、TLS をサポートしているポート 587 の使用を推奨します。プロバイダーによっては、古いポート 465 の使用が必要になる場合があります。元の SMTP ポートであるポート 25 は、多くのプロバイダーや ISP でブロックされているため、使用は推奨されません。
5. **Save** をクリックします。

その後、**Send Test Email** ボタンをクリックしてテストメールを送信できます。数分待っても確認メールが届かない場合は、[Auth0 ログ](https://manage.auth0.com/#/logs) で失敗が発生していないか確認してください。

<div id="using-the-management-api">
  ### Management API の使用
</div>

Management API を使用して SMTP サーバーでテナントを設定するには、`/api/v2/emails/provider` エンドポイントを使用します。

* 初めてメールプロバイダーを設定するには、[`POST` リクエストを `/api/v2/emails/provider` に送信](/ja/docs/api/management/v2/emails/post-provider)します。

* 既存のメールプロバイダーを更新するには、[`PATCH` リクエストを `/api/v2/emails/provider` に送信](/ja/docs/api/management/v2/emails/patch-provider)します。host、port、または username を変更する場合は、[password を指定する必要があります](/ja/docs/troubleshoot/product-lifecycle/deprecations-and-migrations#allow-omitting-password-on-smtp-email-provider-host-related-changes)。

<div id="additional-considerations">
  ## 追加の考慮事項
</div>

* LOGIN 以外の認証プロトコルへの対応が必要な場合は、[SMTP サーバーに接続する Action を作成](/ja/docs/customize/email/configure-a-custom-email-provider)できます。

* Microsoft Exchange Online では、2026 年 4 月に [SMTP AUTH の基本認証が廃止される](https://techcommunity.microsoft.com/blog/exchange/exchange-online-to-retire-basic-auth-for-client-submission-smtp-auth/4114750)予定です。これは、`smtp.office365.com` および `smtp-legacy.office365.com` エンドポイントに影響します。

  [Microsoft Office 365 Exchange Online](/ja/docs/customize/email/smtp-email-providers/configure-365-exchange-as-smtp-email-provider) および [Azure Communication Services Email](/ja/docs/customize/email/smtp-email-providers/configure-azure-comm-service-as-smtp-email-provider) との統合を使用することをお勧めします。
