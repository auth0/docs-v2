<div id="prerequisites">
  ## 前提条件
</div>

* [Resend](https://resend.com) のアカウントが必要です。

* メールを送信するには、事前に Resend で少なくとも 1 つのドメインを検証しておく必要があります。詳細については、Resend の[ドメイン検証に関するドキュメント](https://resend.com/docs/dashboard/domains/introduction)を参照してください。

* 特定の [Auth0 IP アドレス](/ja/docs/secure/security-guidance/data-security/allowlist) からの受信接続を許可するように、プロバイダーを設定する必要があります。

<div id="configure-resend">
  ## Resend を設定
</div>

1. [Resend](https://resend.com) アカウントにログインします。
2. **API Keys** に移動し、**Create API Key** をクリックします。
3. キーの名前を入力し、適切な権限レベルとドメインを選択して、**Add** をクリックします。

<Frame>
  <img src="/docs/images/customize/emails/providers/resend-create-api-key.png" />
</Frame>

4. API キーの値をコピーします。この値は再度表示できません。
5. Auth0 の [Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email_provider) に移動します。
6. **Use my own email provider** トグルをオンにします。
7. **Resend** ロゴを選択します。
8. **From** メールアドレスを入力し、コピーした Resend の **API Key** を貼り付けて、**Save** をクリックします。

**From** メールアドレスには、Resend で検証済みのドメインを使用する必要があります。[メールによるパスワードレス認証](/ja/docs/authenticate/passwordless#email) を使用している場合は、<Tooltip tip="パスワードレス: 第 1 要素としてパスワードに依存しない認証方式です。" cta="用語集を表示" href="/ja/docs/glossary?term=passwordless">パスワードレス</Tooltip> 接続の **From** メールアドレスの値が、Resend で使用する **From** メールアドレスの値と一致している必要があります。

<div id="send-a-test-email">
  ## テストメールを送信する
</div>

**Send Test Email** ボタンをクリックして、テストメールを送信します。設定が正しければ、確認メールが届きます。数分経ってもメールが届かない場合は、エラーコードを [Auth0ログ](https://manage.auth0.com/#/logs) で確認してください。

設定が完了すると、Resend の [Emails](https://resend.com/emails) ページに、ユーザーに送信されたすべてのメールの配信状況に関するインサイトが表示されます。
