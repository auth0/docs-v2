<Card title="利用可否は Auth0 プランによって異なります">
  この機能を利用できるかどうかは、ログイン実装の構成と、Auth0 プランまたはカスタム契約の内容によって異なります。詳しくは、[Pricing](https://auth0.com/pricing) を参照してください。
</Card>

Auth0 テナントの <Tooltip tip="Custom Domain: 特別な名前、または独自の名前を持つサードパーティドメイン。" cta="用語集を見る" href="/ja/docs/glossary?term=custom+domain">カスタムドメイン</Tooltip> 名で使用するリバースプロキシとして、AWS CloudFront を設定できます。

1. AWS にログインし、[CloudFront](https://console.aws.amazon.com/cloudfront) に移動します。
2. **Create Distribution** をクリックします。
3. コンテンツの配信方法を選択できます。**Web** セクションの **Get Started** をクリックします。
4. Distribution の設定を行います。変更が必要な値は次のとおりです。

   | Setting                         | Value                                                                                                          |
   | ------------------------------- | -------------------------------------------------------------------------------------------------------------- |
   | Origin Domain Name              | Custom Domains の設定時に Auth0 Dashboard で取得した **Origin Domain Name** の値を設定します                                     |
   | Origin ID                       | Origin の説明を入力します。この値は同じ Distribution 内の複数の Origin を区別するために使用されるため、一意である必要があります。                                |
   | Origin Protocol Policy          | `HTTPS Only` に設定します                                                                                            |
   | Alternate Domain Names (CNAMEs) | カスタムドメイン名 (Auth0 Dashboard で設定したものと同じもの) を設定します                                                                |
   | SSL Certificate                 | US East (N. Virginia) リージョンまたは IAM の AWS Certificate Manager (ACM) に保存されている、カスタムドメイン用の SSL Certificate を設定します。 |
5. **Origin Custom Headers** の情報を入力します (**Header Name** フィールドと **Value** フィールドは、**Origin Domain Name** を入力した後にのみ表示されます)

   | Setting     | Value                                                |
   | ----------- | ---------------------------------------------------- |
   | Header Name | `cname-api-key` に設定します                               |
   | Value       | Auth0 でドメイン名の所有権を確認した直後に付与された CNAME API Key の値を設定します |
6. **Default Cache Behavior Settings** を設定します。変更が必要な値は次のとおりです

   | Setting                                 | Value                                                                                                                                            |
   | --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
   | Viewer Protocol Policy                  | **Redirect HTTP to HTTPS** を選択します                                                                                                                |
   | Allowed HTTP Methods                    | **GET, HEAD, OPTIONS, PUT, POST, PATCH, DELETE** を選択します                                                                                          |
   | Cache and origin request settings       | **Legacy cache settings** を選択します                                                                                                                 |
   | Cache Based on Selected Request Headers | **Include the following headers** を選択します                                                                                                         |
   | Add Headers                             | `User-Agent` を入力し、**Add Custom &gt;&gt;** をクリックしてカスタムヘッダーを追加します。`Authorization`、`Origin`、`Referer`、`Accept-Language`、`Accept` ヘッダーについても同様に追加します。 |
   | Forward Cookies                         | **All** を選択します                                                                                                                                   |
   | Query Strings                           | **All** を選択します                                                                                                                                   |
7. ページの一番下までスクロールし、**Create Distribution** をクリックします。
   新しく作成した Distribution が CloudFront Distributions の一覧に表示されます。Distribution が Deployed になるまでは、Status に `In progress` と表示されることに注意してください。
8. カスタムドメイン用の DNS に新しい CNAME レコードを追加し、Distribution の CloudFront Domain Name を指すようにします。これは Distribution ID をクリックし、General タブの Domain Name (たとえば `e2zwy42nt1feu7.cloudfront.net`) で確認できます。ドメインに Route 53 が設定されている場合は、Alias レコードを使用することもできます。
