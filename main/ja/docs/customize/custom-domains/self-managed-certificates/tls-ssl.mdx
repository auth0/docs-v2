<Card title="始める前に">
  Auth0 のネットワークエッジでは、すべてのリクエストで Server Name Indication (SNI) が設定されている必要があります。ほとんどのクライアントでは SNI は既定で設定されますが、Web クライアントで設定されない場合は、SNI を手動で設定する方法について Web クライアントのドキュメントを参照してください。
</Card>

Auth0 のネットワークエッジでは、安全性が確認された SSL/TLS バージョンと暗号スイートの組み合わせのみを許可しています。自己管理証明書を使用するリバースプロキシを介して Auth0 サービスに接続する場合は、サポートされている TLS バージョンと暗号スイートを**必ず**使用する必要があります。TLS ハンドシェイク時に、サーバーとクライアント間の通信で使用する TLS バージョンと暗号スイートが決定されます。サポート対象のバージョンを使用していない場合は、接続に失敗する可能性があります。

<div id="supported-versions">
  ## サポート対象バージョン
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  カスタムドメインで自己管理証明書を使用している場合は、以下の TLS バージョンおよび暗号スイートのいずれかに対応している必要があります。セキュリティ上の理由により、プロトコルまたは暗号スイートのサポートが予告なく終了される場合があります。
</Callout>

Auth0 では、サポートされている暗号スイートで TLS バージョン 1.2 または 1.3 を使用する必要があります。

<div id="tls-13-supported-ciphers">
  #### TLS 1.3 でサポートされる暗号スイート
</div>

TLS&#95;AES&#95;128&#95;GCM&#95;SHA256
TLS&#95;AES&#95;256&#95;GCM&#95;SHA384
TLS&#95;CHACHA20&#95;POLY1305&#95;SHA256

</div>

<div id="tls-parameters">
  #### TLS パラメーター
</div>

詳細については、暗号スイートを含む登録済みパラメーターの一覧が掲載されている Internet Assigned Numbers Authority (IANA) の [Transport Layer Security (TLS) Parameters](https://www.iana.org/assignments/tls-parameters) を参照してください。
