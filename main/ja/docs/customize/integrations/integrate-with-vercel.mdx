<div id="overview">
  ## 概要
</div>

[Auth0 統合](https://vercel.com/integrations/auth0) を使用して、新規または既存の Auth0 インスタンスを Vercel プロジェクトに接続すると、アプリのセキュリティ、スケーラビリティ、カスタマイズ性を強化できます。この統合により、簡素化された顧客アイデンティティの仕組みをアプリケーションに直接実装できます。

この新しい統合を使用すると、Vercel プロジェクトのアイデンティティフローを構築してカスタマイズできます。アプリをすばやく安全に稼働させるために、この統合では次のことが可能です。

* 数回クリックするだけで、Auth0 テナントと統合された本番環境を Vercel に設定できます。
* Vercel プロジェクトに接続するアプリケーションを Auth0 テナント内に自動的に作成します。
* Auth0 の認証情報を Vercel に事前設定するため、このプロセスで必要な手動手順の一部を省略できます。

<div id="prerequisites">
  ### 前提条件
</div>

Auth0 を Vercel プロジェクトに統合するには、事前にいくつか準備が必要です。

* 統合対象のプロジェクトが含まれる Vercel アカウント。まだ Vercel アカウントをお持ちでない場合は、[Create your Vercel account](https://vercel.com/signup) にアクセスして作成してください。
* 最新の [Auth0 Next.js SDK](/ja/docs/quickstart/webapp/nextjs#install-the-auth0-next-js-sdk) をベースにした Next.js アプリケーション。

<div id="add-a-vercel-integration">
  ## Vercel 統合を追加する
</div>

既存の Vercel プロジェクトに Auth0 統合を追加するには、次の手順を実行します。

1. [Auth0 integration page](https://vercel.com/integrations/auth0) に移動します。
2. インストール ウィンドウが表示されたら、Auth0 統合をインストールする Vercel のチームとプロジェクトを選択します。

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     現在、統合を提供しているのは一度に 1 つのプロジェクトのみです。**All projects** を選択すると、プロジェクトの選択画面はワークフローの後半で表示されます。
   </Callout>
3. **Install** を選択して、Auth0 統合ポータルでのインストールに進みます。
4. 正しい Auth0 アカウントでログインしていること、および正しいテナントの Dashboard に対して統合しようとしていることを確認します。

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Auth0 にログインするか、正しい Auth0 アカウントでログインしていて、正しいテナントの Dashboard に対して統合しようとしていることを確認してください。

     * テナントが選択されていない場合は、テナントの作成画面にリダイレクトされます。
     * 新しいテナントを作成する場合は、標準のオンボーディング フローに進みます。
     * Auth0 を初めて使用する場合は、ここで Auth0 アカウントを作成します。
   </Callout>
5. **Accept** を選択します。
6. 次に、Auth0 と統合する Vercel プロジェクトを選択し、**Continue** を選択します。
7. アプリケーションの選択内容を確認し、**Continue** を選択します。
8. これで、Auth0 統合が Vercel プロジェクトに正常に追加されました。インストール済みの統合を表示するには、プロジェクト概要ページの **Integrations** タブに移動します。Auth0 がプロジェクトのインストール済み統合として表示されます。

<div id="manage-a-vercel-integration">
  ## Vercel 統合を管理する
</div>

Auth0 統合の Vercel プロジェクトへの設定が完了したら、Vercel のアプリケーション Dashboard からその統合を管理できます。

<div id="manage-from-vercel">
  ### Vercel から管理する
</div>

Vercel のダッシュボードから Auth0 アプリケーションを管理するには、プロジェクトの **Integration** タブで Auth0 統合を選択します。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  環境変数を設定できるのは、本番ドメインのみです。
</Callout>

<div id="manage-access">
  #### アクセスを管理する
</div>

Vercel アプリケーション内の特定のプロジェクトに対する統合のアクセスを管理するには、**Manage Access** を選択します。これにより、Auth0 統合に対してプロジェクトレベルのアクセス制限を適用できます。

<div id="remove-integration">
  #### 統合を削除
</div>

Auth0 の統合をアンインストールするには、**統合を削除**を選択します。すると、アプリケーションとそれに関連するすべての設定が <Tooltip tip="Auth0 Dashboard: サービスを構成するための Auth0 の主要製品。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip> から削除されます。この操作は元に戻せないため、削除した場合はアプリケーションを再インストールする必要があります。

<div id="manage-from-the-auth0-dashboard">
  ### Auth0 Dashboard で管理する
</div>

Vercel プロジェクトに Auth0 統合を追加すると、[Auth0 Dashboard](https://manage.auth0.com/) に新しいアプリケーションが作成され、CIAM ワークフローを管理およびカスタマイズできるようになります。

1. [Auth0 Dashboard](https://manage.auth0.com/) に移動します。
2. **Applications &gt; Applications &gt; [Your Vercel application]** を選択します。
3. **Settings &gt; Application URIs** で、Vercel アプリケーションの URI を確認します。

   <Frame>![](/docs/images/cdy7uua7fh8z/62UwXZPwqrjYly0rkXawYb/31a270f2e8b1413bac2fd879edd1dd6e/border_Screenshot_2024-05-30_at_10.54.28_AM.png)</Frame>
4. これで、Auth0 Dashboard 内の他のアプリケーションと同様に、このアプリケーションをカスタマイズおよび変更できます。
