---

Office365 は非推奨になりました。Office365 接続を Windows Azure AD 接続へ移行する必要があります。

Auth0 では早い段階から、Office365 でユーザーを認証できるようサポートしてきました。Office365 は以前からバックエンドで Windows Azure AD を使用していましたが、Windows Azure AD で「アプリケーション」を作成するための適切な UI がありませんでした。そのため、Seller Dashboard で作成する必要がありました。今後は Microsoft が Windows Azure AD の利用を推奨しており、Office365 アカウントとアプリケーションに関連付けられたディレクトリを簡単に作成できるようになりました。

<div id="how-to-migrate-to-azure">
  ## Azure への移行方法
</div>

1. Windows Azure AD のサブスクリプション (無料) を作成します
2. ディレクトリ (Office365 アカウントに関連付けられるもの) とアプリケーションを作成します。詳しくは、[アプリを Microsoft Azure Active Directory に接続する](/ja/docs/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2)を参照してください。

アプリケーションで `user_id` を使用している場合は、`office365|....some-guid....` から `waad|...email....` に変更されることに注意してください。
