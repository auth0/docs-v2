要件に応じて、Auth0 を Amazon の製品やサービスと統合する方法は数多くあります。

<div id="product-integrations">
  ## 製品統合
</div>

以下の製品を、Auth0 および AWS のサービスとあわせて利用する方法について詳しく確認できます。

* [CloudFront](/ja/docs/customize/custom-domains/self-managed-certificates/configure-aws-cloudfront-for-use-as-reverse-proxy): <Tooltip tip="カスタムドメイン: 固有名またはバニティ名を持つサードパーティのドメイン。" cta="用語集を表示" href="/ja/docs/glossary?term=custom+domain">カスタムドメイン</Tooltip>でリバースプロキシとして使用します。
* [Simple Email Service (SES)](/ja/docs/customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider): ユーザーとのメール通信を管理します。
* [EventBridge](https://marketplace.auth0.com/integrations/amazon-log-streaming): ログを EventBridge にストリーミングします。
* [Cognito](/ja/docs/customize/integrations/aws/amazon-cognito): アプリケーションのバックエンドとして使用します。
* AWS S3 と DynamoDB:

  * Auth0 では、AWS S3 と DynamoDB にアクセスするための VPC ゲートウェイエンドポイントを提供しています。これは [Actions](/ja/docs/customize/actions) で必要になる場合があります。これには、[Gateway endpoints for Amazon S3](https://docs.aws.amazon.com/vpc/latest/privatelink/vpc-endpoints-s3.html) および [Gateway endpoints for Amazon DynamoDB](https://docs.aws.amazon.com/vpc/latest/privatelink/vpc-endpoints-ddb.html) で説明されている `aws:SourceVpc` ベースの IAM ポリシーが必要です。VPC 識別子の詳細については、[Auth0 support](https://www.support.auth0.com) までお問い合わせください。

<div id="identity-and-access-management-iam-integrations">
  ## アイデンティティおよびアクセス管理 (IAM) の統合
</div>

Auth0 と連携するように AWS を設定するためのハウツー記事がいくつかあります。

* [カスタムオーソライザーを使用して AWS API Gateway エンドポイントを保護する](/ja/docs/customize/integrations/aws/aws-api-gateway-custom-authorizers): Auth0 が発行した <Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可資格情報。" cta="用語集を表示" href="/ja/docs/glossary?term=access+tokens">アクセストークン</Tooltip> を受け付けます。
* [AWS API およびリソースで AWS セッションタグを使用する](/ja/docs/customize/integrations/aws/session-tags): リソースにタグを付け、RBAC 用のキー/値ペアをユーザーに割り当てます。
* [シングルサインオン用に AWS を設定する](/ja/docs/customize/integrations/aws/configure-amazon-web-services-for-sso): サポートされている任意の <Tooltip tip="IDプロバイダー（IdP）: デジタルアイデンティティを保存および管理するサービス。" cta="用語集を表示" href="/ja/docs/glossary?term=identity+provider">IDプロバイダー</Tooltip> を使用して、ユーザーが AWS にログインできるようにします。
* [アプリに Amazon ログインを追加する](/ja/docs/customize/integrations/aws/configure-amazon-web-services-for-sso): ユーザーが Amazon プロフィールを使用してアプリにログインできるようにします。
