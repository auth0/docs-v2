description: Auth0 のビジネスパートナーが、サードパーティ製品を Auth0 のワークフローに組み込むゼロコードソリューションを作成するための手順
title: パートナー向け Actions Integrations
---

このようなパートナーは、Actions Integrations を使用して、自社製品を Auth0 のワークフローに組み込むゼロコードソリューションを構築します。Actions Integrations を使用すると、Auth0 の顧客はコードをまったく書かずに、本人確認や同意管理などのソリューションを実装できます。パートナーが Actions Integration を作成し、顧客はそれをワークフローにドラッグ＆ドロップします。

Actions Integrations は、Auth0 プラットフォーム内の指定したポイントで実行される自己完結型の関数です。JavaScript で記述されるクローズドソースのため、顧客はそのコードを変更できません。

<div id="create-and-publish-an-actions-integration">
  ## Actions Integration を作成して公開する
</div>

Actions Integration の作成では、計画、構築、テストを行います。[Auth0 Marketplace](https://marketplace.auth0.com/) で顧客が利用できるようにする前に、Auth0 による Actions Integration の審査が行われます。Auth0 Marketplace の詳細については、[Marketplace Partners](/ja/docs/customize/integrations/marketplace-partners) を参照してください。

<div id="step-1-determine-your-use-case">
  ### ステップ 1. ユースケースを明確にする
</div>

Actions Integration で対応するユースケースを明確にします。顧客のどのような課題を解決するのか。どのようにその課題を解決するのか。詳しくは、[Defining an Integration Use Case](/ja/docs/customize/integrations/marketplace-partners/defining-an-integration-use-case)を参照してください。

<div id="step-2-choose-a-flow">
  ### ステップ 2. フローを選択する
</div>

Action Integration は、1 つの [Action](/ja/docs/customize/actions) で構成されます。Action で実行できることは、Auth0 ランタイム環境内のどこで実行されるか、つまり Action の [フロー](/ja/docs/customize/actions/explore-triggers) によって決まります。利用可能な Action フローは次のとおりです。

* **Login flow** は、ユーザーが正常にサインアップまたはログインした後に同期的に実行されます。ログインイベントには、インタラクティブなフロー (username/password やフェデレーテッドログインなど) と、非インタラクティブなフロー (セッションチェックや <Tooltip tip="Refresh Token: ユーザーに再ログインを求めることなく、新しいアクセストークンを取得するために使用されるトークン。" cta="用語集を表示" href="/ja/docs/glossary?term=refresh+token">リフレッシュトークン</Tooltip> の交換など) が含まれます。このフローの詳細については、[Login Flow](/ja/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger) を参照してください。
* **Machine to Machine flow** は、アプリケーションがクライアントクレデンシャルグラントを使用して <Tooltip tip="Access Token: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可クレデンシャル。" cta="用語集を表示" href="/ja/docs/glossary?term=access+token">アクセストークン</Tooltip> を要求するときに同期的に実行されます。このフローの詳細については、[Machine to Machine Flow](/ja/docs/customize/actions/explore-triggers/machine-to-machine-trigger) を参照してください。
* **Pre User Registration flow** は、Auth0 がユーザーをデータベースまたは <Tooltip tip="Passwordless: 第 1 要素としてパスワードに依存しない認証方式。" cta="用語集を表示" href="/ja/docs/glossary?term=passwordless">パスワードレス</Tooltip> 接続に追加する前に同期的に実行されます。このフローは、エンタープライズ接続やソーシャル接続などのフェデレーテッドログインでは実行されません。このフローの詳細については、[Pre User Registration Flow](/ja/docs/customize/actions/explore-triggers/signup-and-login-triggers/pre-user-registration-trigger) を参照してください。
* **Post User Registration flow** は、Auth0 がユーザーをデータベースまたはパスワードレス接続に追加した後に非同期で実行されます。このフローは、エンタープライズ接続やソーシャル接続などのフェデレーテッドログインでは実行されません。このフローの詳細については、[Post User Registration Flow](/ja/docs/customize/actions/explore-triggers/signup-and-login-triggers/post-user-registration-trigger) を参照してください。
* **Post Change Password flow** は、データベース接続のユーザーのパスワードが変更された後に非同期で実行されます。このフローは、他の種類の接続でのパスワード変更では実行されません。このフローの詳細については、[Post Change Password Flow](/ja/docs/customize/actions/explore-triggers/password-reset-triggers/post-change-password-trigger) を参照してください。
* **Send Phone Message** **flow** は、カスタム <Tooltip tip="Multi-factor authentication (MFA): SMS 経由の code など、username と password に加えて別の要素を使用するユーザー認証プロセス。" cta="用語集を表示" href="/ja/docs/glossary?term=MFA">MFA</Tooltip> プロバイダーの一部として、音声通話または SMS メッセージを送信します。カスタム MFA プロバイダーの詳細については、[Customize Multi-factor Authentication SMS and Voice Messages](/ja/docs/customize/customize-sms-or-voice-messages) を参照してください。このフローの詳細については、[Send Phone Message Flow](/ja/docs/customize/actions/explore-triggers/mfa-notifications-trigger) を参照してください。

Action フローの詳細については、[Explore Flows and Triggers](/ja/docs/customize/actions/explore-triggers) を参照してください。ログイン中にユーザーをエンドポイントにリダイレクトする必要がある場合は、[Redirect Actions for Partners](/ja/docs/customize/integrations/marketplace-partners/redirect-actions-for-partners) を参照してください。

<div id="step-3-build-an-actions-integration">
  ## ステップ 3. Actions Integration を構築する
</div>

Action Integration のユースケースに適したフローを決めたら、テスト用テナントで Action を作成します。手順については、[初めての Action を作成する](/ja/docs/customize/actions/write-your-first-action)を参照してください。この Action は、Action Integration コードのベースになります。

また、統合の構築には、以下の Action Integration テンプレートを利用できます。

* [Post Login Action Integration Template](https://github.com/auth0-marketplace/TEMPLATE-action-post-login)
* [Machine-to-Machine Action Integration Template](https://github.com/auth0-marketplace/TEMPLATE-action-client-credentials)
* [Pre User Registration Action Integration Template](https://github.com/auth0-marketplace/TEMPLATE-action-pre-user-registration)
* [Post User Registration Action Integration Template](https://github.com/auth0-marketplace/TEMPLATE-action-post-user-registration)
* [Post Change Password Action Integration Template](https://github.com/auth0-marketplace/TEMPLATE-action-post-password-change)
* [Send Phone Message Action Integration Template](https://github.com/auth0-marketplace/TEMPLATE-action-phone-message)

Actions Integration を安全かつ安定して動作させる方法については、[Actions Coding Guidelines](/ja/docs/customize/actions/action-coding-guidelines)を参照してください。あわせて、Actions Integration 固有の次のガイドラインにも従ってください。

* テスト用テナントで Action を作成する際は、顧客が指定する必要のあるすべての値 (識別子、テナント名、しきい値など) に secrets を使用します。Actions Integration を送信すると、Auth0 はどのフィールドを保存時に暗号化する必要があるか (secrets のままにするか) 、またどのフィールドを編集可能なテキストフィールドにできるかを確認します。
* セットアップやトラブルシューティングに役立つログ出力には、デバッグ/テストモードの設定フラグを使用します。Actions Integration では、すべての実行でログを出力することはまれです。
* レビューしやすいように、secret 名は `ALL_CAPS_SNAKE_CASE` で記述します。
* Actions Integration はクローズドソースです。顧客はコードを閲覧することも変更することもできません。Action Integration の制御に必要な顧客固有の入力は、secrets または設定から取得する必要があります。

<div id="step-4-test-your-actions-integrations">
  ### ステップ 4. Actions Integration をテストする
</div>

Auth0 のレビュアーはお客様のサービスにアクセスできないため、レビューに提出する前に Action をテストする必要があります。正常に実行されるケースとエッジケースの両方を手動でテストしてください。次の項目を確認します。

* Action は何の通知もなく失敗しますか、それとも以降の実行をブロックしますか？
* HTTP リクエストが失敗した場合、Action はどのように動作しますか？
* ユーザーに表示されるエラーメッセージはありますか？ある場合、どのような内容ですか？
* どのような情報がログに記録されますか？
* Action は回避されたり、なりすまされたりする可能性がありますか？

提出プロセスでは、Auth0 は成功パスに対する単体テストのカバレッジを求めます。Auth0 では、これをできるだけ簡単に行えるよう、必要なツールと例をすべて提供しています。

<div id="step-5-submit-your-actions-integration">
  ### ステップ 5. Actions Integration を提出する
</div>

Actions Integration の作成と十分なテストが完了したら、Auth0 のレビューに提出します。[Auth0 Marketplace のサービスリクエスト](https://autheco.atlassian.net/servicedesk/customer/portal/1/group/1/create/43)を送信してください。初回レビューを通過すると、次にセキュリティ準拠の審査が行われます。

<div id="step-6-publish-your-actions-integration">
  ### ステップ 6. Actions Integration を公開する
</div>

Auth0 による Actions Integration の審査が完了したら、審査結果と公開予定時期をお知らせします。
