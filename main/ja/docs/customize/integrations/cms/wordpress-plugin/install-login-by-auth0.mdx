<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  プラグインをインストールまたはカスタマイズするには、セルフホストの WordPress.org サイトを使用する必要があります。WordPress.com サイトではプラグインをインストールできません。違いの詳細については、[wordpress.com の WordPress.com と WordPress.org](https://wordpress.com/support/com-vs-org/)を参照してください。
</Callout>

このプラグインは、wp-admin の **Plugins** 画面から WordPress サイトに追加できます。

1. 管理者として既存の WordPress サイトにログインします。
2. 左側の管理メニューで **Plugins &gt; Add New** に移動します。
3. 「Login by Auth0」を検索します。
4. Login by Auth0 プラグインの **Install Now** をクリックし、続けて **Activate** をクリックします。

この手順の詳細については、[wordpress.com の WordPress.com と WordPress.org](https://wordpress.com/support/com-vs-org/)を参照してください。

プラグインを有効化すると、すぐにセットアップウィザードの開始画面にリダイレクトされます。

まだ Auth0 アカウントをお持ちでない場合は、先に進む前に[**無料でサインアップしてアカウントを作成してください**](https://auth0.com/signup)。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  設定ページの Basic タブに **ドメイン**、**クライアントID**、**クライアントシークレット** を追加するまで、Auth0 はログインフォームを置き換えません。
</Callout>

<div id="multisite-setup">
  ## マルチサイトのセットアップ
</div>

Login by Auth0 プラグインは、WordPress のマルチサイトネットワークに対応しています。プラグインはネットワーク全体で有効化して、設定完了後に各サイトを自動的に保護することも、ネットワーク内の一部のサイトでのみ有効化することもできます。

Auth0 でサイトネットワークを設定する方法はいくつかあります。

1. **すべてのサイトで 1 つのアプリケーションと 1 つのデータベース接続を共有できます**

   1. メインサイトでセットアップウィザードの手順を最後まで実行します。
   2. Auth0 設定ページの Basic タブで、メインサイトの **ドメイン**、**クライアントID**、**クライアントシークレット** を使って、他のすべてのサイトを手動で設定します。
   3. アプリケーションの **Allowed Callback URLs**、**Allowed Web Origins**、**Allowed Logout URLs** を更新し、各サイトを追加します (ネットワークでサブドメインを使用している場合はワイルドカードを使用できます) 。
2. **各サイトがそれぞれ独自のアプリケーションを持ち、データベース接続を共有できます**

   1. メインサイトでセットアップウィザードの手順を最後まで実行します。
   2. 次に、[各サイト用のアプリケーションを手動で作成し](/ja/docs/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0)、それぞれを先に作成したデータベース接続に追加します。
   3. 各サイトの Auth0 設定ページの Basic タブに、**ドメイン**、**クライアントID**、**クライアントシークレット** の値を追加します。
3. **各サイトがそれぞれ独自のアプリケーションと独自のデータベース接続を持つこともできます。** この場合は、各サイトでセットアップウィザードの手順を最後まで実行します。

上記の各オプションには、それぞれトレードオフがあります。オプション 1 は、Auth0 で管理するエンティティの数を最も少なくできますが、ネットワークに何百ものサイトがあり、サブドメインを使用していない場合は、コールバック URL の数の制限に達する可能性があります。オプション 2 では多数の異なるアプリケーションを管理する必要がありますが、各サイトのアプリケーションを個別に設定できます。

この設定プロセスについて不明点がある場合は、[Community に投稿し、&quot;wordpress&quot; タグを付けてください](https://community.auth0.com/tags/wordpress)。

<div id="setup-wizard">
  ## セットアップウィザード
</div>

セットアップウィザードは、WordPress サイトで Auth0 を使用するために必要なコンポーネントをすべて作成しようとします。使用する既存のアプリケーションまたはデータベース接続がある場合は、以下の手動セットアップ手順を参照してください。

<div id="option-1-standard-setup">
  ### オプション 1: 標準セットアップ
</div>

これにより、このサイト用のアプリケーションとデータベース接続が作成され、設定されます。

まず、[Management API トークンを生成](https://auth0.com/docs/api/management/v2/get-access-tokens-for-test#get-access-tokens-manually)します。トークンを生成したら、**Settings** タブの **Identifier** フィールドで使用されているドメイン名を控えておいてください。たとえば、Identifier が `https://tenant-name.auth0.com/api/v2/` の場合、テナントのドメインは `tenant-name.auth0.com` です。テナントのドメインの詳細については、[Create Tenants](/ja/docs/get-started/auth0-overview/create-tenants) を参照してください。

WordPress 管理画面のセットアップウィザードに戻り、**Standard** をクリックします。表示されたモーダルで **Start Standard Setup** をクリックします。

先ほど確認したテナントのドメインと API トークンを入力します。このトークンはセットアップ処理でのみ使用され、データベースには保存されません。

セットアップの最初の部分が正常に完了すると、&quot;Configure your social connections&quot; 画面が表示されます。**Next** をクリックして、管理者アカウントを移行し、セットアップを続行します。

この手順では、ログインを承認する Auth0 ユーザーに WordPress ユーザーを関連付けます。管理者アカウントと同じパスワードを使用することも、別のパスワードを設定することもできますが、使用するデータベース接続のパスワード強度ポリシーに準拠している必要があります。詳しくは、[Password Strength in Auth0 Database Connections: Password Policies](/ja/docs/authenticate/database-connections/password-strength#password-policieslicies) を参照してください。

サイトを正しく設定するには、セットアップウィザードを最後まで完了する必要があります。&quot;setup successful&quot; 画面が表示される前に何らかの理由でウィザードが失敗した場合は、**wp-admin &gt; Auth0 &gt; Error Log** にあるプラグインのエラーログと以下の手順を確認して、問題を特定してください。

最初から完全にやり直すには、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> で作成されたアプリケーションまたはデータベース接続を削除します。WordPress で **Auth0 &gt; Settings &gt; Basic** に移動し、ドメイン、<Tooltip tip="Client Secret: クライアント（アプリケーション）が認可サーバーに対して認証を行うために使用するシークレットです。これはクライアントと認可サーバーのみが把握している必要があり、推測されないよう十分にランダムでなければなりません。" cta="用語集を見る" href="/ja/docs/glossary?term=Client+ID">クライアントID</Tooltip>、および <Tooltip tip="Client ID: 登録したリソースに Auth0 から付与される識別値です。" cta="用語集を見る" href="/ja/docs/glossary?term=Client+Secret">クライアントシークレット</Tooltip> の各フィールドを削除して、**Save** をクリックします。次に、管理メニューの **Setup Wizard** をクリックして、再度最初からセットアップを開始します。

それでもインストールできない場合は、Error Log に表示されているエラーメッセージを添えて、[Community にスレッドを投稿](https://community.auth0.com/tags/wordpress)してください。サポートいたします。

<div id="option-2-user-migration-setup">
  ### オプション 2: ユーザー移行のセットアップ
</div>

これにより、アプリケーション とデータベース接続が作成・設定され、WordPress データベースからのデータ移行も実行されます。これには Auth0 サーバーからのインバウンド接続が必要であり、後から変更するとデータが失われるため、変更できません。詳細については、[Login by Auth0 WordPress Plugin におけるユーザー移行](/ja/docs/customize/integrations/cms/wordpress-plugin/user-migration-in-login-by-auth0)を参照してください。

::: warning
Auth0 にカスタムデータベース接続が複数ある場合は、競合を避けるためにユーザーIDに名前空間が付与されていることを確認する必要があります。これは、バージョン 3.11.0 以降をインストールするサイトでは自動的に行われます。以前のバージョンで接続が作成されている、または作成されていた場合は、[Login by Auth0 WordPress Plugin におけるユーザー移行](/ja/docs/customize/integrations/cms/wordpress-plugin/user-migration-in-login-by-auth0)のトラブルシューティング手順を参照してください。
:::

設定プロセスが完了したら、WordPress サイトからログアウトし、Auth0 のログインフォームで既存の WordPress 認証情報を使って再度ログインしてみてください。これにより、WordPress アカウントに紐付けられた Auth0 ユーザーが作成されます。

<div id="option-3-manual-setup">
  ### オプション 3: 手動セットアップ
</div>

このオプションでは自動セットアップをスキップし、独自のアプリケーションとデータベース接続を作成して設定できます (以下を参照) 。このサイトで既存のアプリケーションまたはデータベース接続を使用する場合は、これを選択してください。

<div id="option-4-import-setup">
  ### オプション 4: 設定のインポート
</div>

別のサイトから設定をインポートして、このサイトをセットアップすることもできます。これは、環境間で移行する場合や、既に設定済みの類似した WordPress サイトがある場合に便利です。

<div id="setup-complete">
  ## セットアップ完了
</div>

「Done」画面が表示されたら、Auth0 は有効になっており、ログインと、設定されていればサインアップを受け付ける準備ができています。

デフォルト設定を変更する前に、サイトで基本的な動作に問題がないことを確認するのに適したタイミングです。

1. WordPress からログアウトし、`/wp-login.php` に Auth0 のフォームが表示されることを確認します。
2. 上で作成した Auth0 ユーザーでログインします。
3. ログアウトし、別のメールアドレスでアカウントを作成してみます (WordPress の一般設定で「Anyone can register」を有効にしている場合) 。
4. ソーシャル接続を使ってログインしてみます (有効にしている場合) 。

これで、[アプリケーションを設定](/ja/docs/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0)する準備は完了です。

<div id="manual-setup">
  ## 手動設定
</div>

このプラグインは、組み込みのセットアップウィザード (以下で説明) を使用して設定することも、アプリケーションを作成して接続を割り当てることで手動で設定することもできます。完全な手動設定は、ウィザードで問題が発生している場合、以前に設定を行ったことがある場合、またはアプリケーション間でデータベース接続を共有したい場合に利用できます。

1. [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#) に移動し、**Create** を選択します。
2. アプリケーションにわかりやすい名前を付け、**Regular Web Applications** を選択してから、**Create** を選択します。
3. WordPress のインストールに合わせてアプリケーションを設定します。設定方法については、[Configure Login by Auth0](/ja/docs/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0) を参照し、「Update Auth0 settings in WordPress」までのすべてのセクションに従ってください。

上記の手順が完了すると、サイトは設定され、使用できる状態になります。
