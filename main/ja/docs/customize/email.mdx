---
title: メールのカスタマイズ
description: アカウント確認、ウェルカムメッセージ、漏えいしたパスワードの検知など、Auth0 から送信されるメールを管理およびカスタマイズします。
---

Auth0 では、[ユーザーアカウントの確認](/ja/docs/manage-users/user-accounts/verify-emails) や [漏えいしたパスワードの検知](/ja/docs/secure/attack-protection/breached-password-detection) など、メールベースのワークフローがいくつか用意されています。

これらのワークフローを利用するには、まず、サポートされているプロバイダーの認証情報 (Mandrill API キーや Amazon SES のアクセスキーなど) または SMTP サーバーの詳細情報を使って、[外部 SMTP メールプロバイダー](/ja/docs/customize/email/smtp-email-providers) を設定します。

メールをさらに細かく管理およびカスタマイズするには、次の方法があります。

* [メールテンプレート](/ja/docs/customize/email/email-templates) と、オープンソースのテンプレート言語である Liquid を使用して、メールの**見た目と内容を変更**します。

* メールプロバイダーの API にリクエストを送信し、ユースケースに応じたカスタムロジックを実装する <Tooltip tip="Action: Auth0 ランタイムの定義された時点で実行される Node.js 関数です。" cta="用語集を見る" href="/ja/docs/glossary?term=Actions">Action</Tooltip> を記述して、**メール配信プロセスを完全に制御**します。

* Management API を使用して、**デフォルトのワークフロー以外でメールを送信したり、チケットを作成したり**します。

開発中は、[組み込みのメールプロバイダー](/ja/docs/customize/email/smtp-email-providers#use-the-built-in-smtp-email-provider-for-development) を使用して、メール配信を確認したり、テスト中のメール表示をプレビューしたりできます。
