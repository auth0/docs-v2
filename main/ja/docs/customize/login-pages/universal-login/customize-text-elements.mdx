<Tooltip tip="Universal Login: アプリケーションはユーザーの本人確認を行うために、Auth0 の認可サーバーでホストされている Universal Login にリダイレクトされます。" cta="用語集を表示" href="/ja/docs/glossary?term=Universal+Login">Universal Login</Tooltip> は、ログイン、<Tooltip tip="Universal Login: アプリケーションはユーザーの本人確認を行うために、Auth0 の認可サーバーでホストされている Universal Login にリダイレクトされます。" cta="用語集を表示" href="/ja/docs/glossary?term=multi-factor+authentication">多要素認証</Tooltip>要素の登録、パスワードのリセットなど、アカウント関連のさまざまな操作を行うための一連のページで構成されています。Auth0 は、これらのページのテキストを複数の言語で提供しています。

場合によっては、アプリケーションのトーンや特定のニーズにより適したものにするために、これらのページのテキストを変更したいことがあります。

[Universal Login](/ja/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience) ページに表示されるテキストは、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要プロダクトです。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要プロダクトです。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を通じて、サポート対象のすべての言語でカスタマイズできます。次の定義、値、および変数は、どちらの方法にも共通して適用されます。

<div id="configure-prompts-and-screens">
  ## プロンプトと画面を設定する
</div>

利用可能な[プロンプトと画面](#prompt-values)は、Auth0 Dashboard または [Management API](https://auth0.com/docs/api/management/v2) からカスタマイズできます。

<Tabs>
  <Tab title="ダッシュボードを使用する">
    ダッシュボードを使用して、New Universal Login のテキストプロンプトをカスタマイズできます。

    1. ダッシュボードで [**ブランディング &gt; Universal Login &gt; 詳細オプション**](https://manage.auth0.com/#/custom_text) に移動し、**カスタムテキスト** を選択します。
    2. 編集する **プロンプト** と **画面** を選択し、必要な変更を行います。右側でライブプレビューを確認できます。
    3. 変更を本番環境に反映する準備ができたら、**変更を保存** を選択します。
  </Tab>

  <Tab title="Management API を使用する">
    Management API を使用して、新しい Universal Login のテキストプロンプトをカスタマイズできます。

    1. `read:prompts` と `update:prompts` のスコープを含む [Management API アクセストークン](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) を取得します。
    2. 指定したプロンプトと言語の既存の設定データを取得するには、Management API の [Get custom text for a prompt](https://auth0.com/docs/api/management/v2#!/Prompts/get_custom_text_by_language) エンドポイントを呼び出します。
    3. 必要に応じて、[サポートされているパラメーター](#supported-parameters) と [プロンプト値](#prompt-values) を参照しながら、設定データを変更してください。
    4. 更新した設定データを使用して、Management API の [特定のプロンプトのカスタムテキストを設定する](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) エンドポイントを呼び出します。

       <Warning>
         Management API の [特定のプロンプトのカスタムテキストを設定する](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) エンドポイントでは、指定したプロンプトの既存の設定データが、`body` パラメーターで指定した設定データにすべて置き換えられます。

         指定したプロンプトに対して複数の `screen` 値を設定する場合は、それらをすべて 1 回の Management API 呼び出しに含める必要があります。
       </Warning>

    #### 例

    Management API の [Set custom text for a specific prompt](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) エンドポイントを呼び出すことで、`login` プロンプトの `login` 画面の `description` テキストコンポーネントを `Log in to ACME's website` に設定できます。次に例を示します。

      ```bash cURL
      curl --request PUT \
        --url 'https://{yourDomain}/api/v2/prompts/login/custom-text/en' \
        --header 'authorization: Bearer {yourMgmtApiAccessToken}' \
        --header 'content-type: application/json' \
        --data '{ "login": { "description": "Login to ACME'''s Website" } }'
      ```

      ```csharp C#
      var client = new RestClient("https://{yourDomain}/api/v2/prompts/login/custom-text/en");

      request.AddHeader("content-type", "application/json");
      request.AddHeader("authorization", "Bearer {yourMgmtApiAccessToken}");
      request.AddParameter("application/json", "{ "login": { "description": "Login to ACME's Website" } }", ParameterType.RequestBody);
      IRestResponse response = client.Execute(request);
      ```

      ```go Go
      package main

      import (
      	"fmt"
      	"strings"

      	req.Header.Add("content-type", "application/json")
      	req.Header.Add("authorization", "Bearer {yourMgmtApiAccessToken}")

      	res, _ := http.DefaultClient.Do(req)

      	defer res.Body.Close()
      	body, _ := ioutil.ReadAll(res.Body)

      	fmt.Println(res)
      	fmt.Println(string(body))

      }
      ```

    #### プロンプトのカスタムテキストを削除する

<div id="supported-parameters">
  ### サポートされているパラメーター
</div>

| パラメーター     | 定義                                                                                                                                                                             |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `prompt`   | ログインフロー内のステップです。使用可能な値の一覧については、[Prompt values](#prompt-values) を参照してください。                                                                                                      |
| `language` | カスタマイズが適用される言語です。使用可能な言語の一覧については、[Universal Login Internationalization](/ja/docs/customize/internationalization-and-localization/universal-login-internationalization) を参照してください。 |
| `screen`   | プロンプト内の特定の画面です。                                                                                                                                                                |
| `key`      | テキストコンポーネントの識別子です。                                                                                                                                                             |
| `text`     | ユーザーに表示されるメッセージです。                                                                                                                                                             |

<div id="prompt-values">
  ### プロンプトの値
</div>

`prompt` は、ログインフロー内の特定のステップを指します。各 `prompt` には少なくとも 1 つの `screen` があり、各 `screen` にはカスタマイズ可能な一連のテキストコンポーネントがあります。

使用できる `prompt` の値は次のとおりです。

<div id="variables">
  ## 変数
</div>

一部の画面では、コンテキスト情報に基づいて実行時に置き換えられる変数をテキスト内で使用できます。画面ごとに使用できる変数が異なるため、特定の変数が元の画面以外で動作するとは限りません。

| 変数                            | 説明                                         |
| ----------------------------- | ------------------------------------------ |
| $&#123;clientName&#125;       | Auth0 アプリケーション名                            |
| $&#123;connectionName&#125;   | 接続名 (例: Google)                            |
| $&#123;tenantName&#125;       | Auth0 テナント名                                |
| $&#123;organizationName&#125; | ユーザーが組織のコンテキストで認証している場合の組織名                |
| $&#123;companyName&#125;      | Auth0 テナント名、またはユーザーが組織のコンテキストで認証している場合は組織名 |
| $&#123;userName&#125;         | ログイン中のユーザー名                                |
| $&#123;email&#125;            | ログイン中のユーザーのメールアドレス                         |
| $&#123;appName&#125;          | カスタム Guardian Push アプリケーション名               |
