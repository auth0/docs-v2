<Warning>
  現時点では、Auth0 の開発は Universal Login に注力しており、Classic Login は現在アップデートされていません。ログインページをカスタマイズする場合は、Auth0 は [Universal Login の高度なカスタマイズ](/ja/docs/customize/login-pages/advanced-customizations) を推奨しています。
</Warning>

`languageDictionary` オプションを指定すると、特定の状況で表示されるエラーメッセージをカスタマイズできます。カスタマイズ可能な `languageDictionary` フィールドの一覧は、GitHub リポジトリの [Lock の英語辞書ファイル](https://github.com/auth0/lock/blob/master/src/i18n/en.js) にあります。以下に、カスタマイズしたエラーメッセージの例を示します。

{/* codeblockOld.header.login.logInButton codeblockOld.header.login.configureSnippet */}

export const codeExample = `// languageDictionary オプションでカスタマイズしたエラーメッセージの例
var options = {
  languageDictionary: {
    error: {
      login: {
        "lock.invalid_email_password": "無効な認証情報に関するカスタムメッセージ",
        "lock.network": "ネットワークエラーを示し、ユーザーに接続を確認するよう促すカスタムメッセージ",
        "lock.unauthorized": "権限不足に関するカスタムメッセージ",
        "too_many_attempts": "ユーザーのログイン失敗回数が多すぎることを示すカスタムメッセージ"
      },
      signUp: {
        "invalid_password": "無効なパスワードに関するカスタムメッセージ",
        "user_exists": "ユーザーがすでに存在することを示すカスタムメッセージ"
      }
    }
  }
};

// Auth0Lock を初期化
var lock = new Auth0Lock(
  '{yourClientId}',
  '{yourDomain}',
  options
);`;

<div id="custom-errors-in-rules">
  ## Rules のカスタムエラー
</div>

[rule](/ja/docs/customize/rules) またはカスタムデータベーススクリプトからカスタムエラーコードを返す場合は、カスタムエラーを次の方法で処理できます。

* `error` および `error_mesage` クエリ文字列パラメーターを読み取ることで、アプリケーションのリダイレクト URL で処理できます。
* カスタムエラーメッセージを付けてユーザーをホストされたページにリダイレクトし、そのメッセージを [フラッシュメッセージ](/ja/docs/libraries/lock/lock-api-reference) として表示することでも処理できます。
