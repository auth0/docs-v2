Core configurationページでは、Universal Login と Classic Login の以下の設定を管理できます。

* **ログインエンジン**: Universal Login と Classic Login を切り替えます。
* **Iframe 埋め込み**: 開発環境でログインフローを iframe 内に読み込めるようにします。

<div id="settings">
  ## 設定
</div>

<div id="universal-or-classic-login">
  ### Universal Login または Classic Login
</div>

テナントで使用するログインエクスペリエンスを選択します。この設定を変更すると、Auth0 のログインページにリダイレクトするすべてのアプリケーションに影響します。詳しくは、[Universal Login と Classic Login の比較](/ja/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login)を参照してください。

<Steps>
  <Step title="Core configuration に移動">
    **Branding &gt; Universal Login &gt; Pro-code customizations** に移動し、**Core configuration** を選択します。
  </Step>

  <Step title="ログインタイプを選択">
    **Settings** で **Universal Login** または **Classic Login** を選択します。
  </Step>
</Steps>

<div id="iframe-embedding">
  ### Iframe 埋め込み
</div>

Vercel のような生成 UI プラットフォームで、アプリケーションとログインフローが iframe に埋め込まれている場合に、Universal Login または Classic Login を読み込めるようにします。有効にすると、iframe 埋め込みは開発環境でのみ適用されます。

<Warning>
  iframe 埋め込みを有効にすると、クリックジャッキングを防ぐセキュリティヘッダーは無効になります。
</Warning>

<Steps>
  <Step title="Core configuration に移動">
    **Branding &gt; Universal Login &gt; Pro-code customizations** に移動し、**Core configuration** を選択します。
  </Step>

  <Step title="iframe サポートを有効にする">
    **Iframe settings** で、**Enable support for generative UI platforms** をオンにします。
  </Step>

  <Step title="許可するオリジンを設定する">
    **Allowed iframe URLs for Login** で、許可するオリジンを入力します。

    v0 の場合は、次のオリジンを入力します。

    * `https://v0.app`
    * `https://*.v0.app`
    * `https://*.vusercontent.net`

    利用可能な Auth0 統合を使用して Vercel のような生成 UI プラットフォームからアプリケーションを作成すると、Auth0 はそのプラットフォームから許可済みオリジンを事前設定します。漏れがないことを確認するため、事前設定されたオリジンは必ず確認してください。詳しくは、[Vercel と統合する](/ja/docs/customize/integrations/integrate-with-vercel) を参照してください。
  </Step>
</Steps>

<div id="login">
  ## Login
</div>

Classic Login のログインページをカスタマイズします。詳細は、[ログインページをカスタマイズする](/ja/docs/customize/login-pages/classic-login/customization-classic)を参照してください。

<div id="password-reset">
  ## パスワードリセット
</div>

Classic Login のパスワードリセットページをカスタマイズできます。詳細については、[パスワードリセットページのカスタマイズ](/ja/docs/customize/login-pages/classic-login/customize-password-reset-page)を参照してください。

<div id="multi-factor-authentication">
  ## 多要素認証
</div>

Classic Login の多要素認証ページをカスタマイズできます。詳しくは、[Classic Login の MFA をカスタマイズする](/ja/docs/secure/multi-factor-authentication/customize-mfa/customize-mfa-classic-login) を参照してください。
