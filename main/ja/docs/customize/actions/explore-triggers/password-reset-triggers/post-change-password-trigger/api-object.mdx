post-change-password Actions トリガーの API オブジェクトには、次の項目が含まれます。

<div id="apicache">
  ## `api.cache`
</div>

実行をまたいで保持されるデータを保存および取得します。

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

指定したキーに対応するキャッシュ値のレコードが存在する場合は、それを削除します。

キャッシュから値が削除された場合は、`type: "success"` を持つ `CacheWriteResult` オブジェクトを返します。操作に失敗した場合は `type: "error"` を返します。エラー時には、返されるオブジェクトに、失敗の内容を示す
`code` プロパティが含まれます。

| Parameter | Description                    |
| --------- | ------------------------------ |
| `key`     | *String*。キャッシュに保存されているレコードのキー。 |

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

指定した
`key` に対応するキャッシュ値のレコードが存在する場合は、それを取得します。レコードが見つかった場合、キャッシュされた値は返されるオブジェクトの `value` プロパティに格納されています。

指定した
`key` に対応する項目がキャッシュ内に見つかった場合は、キャッシュレコードを返します。キャッシュレコードはオブジェクトで、キャッシュされた値を保持する `value` プロパティと、Unix エポックからの経過ミリ秒でレコードの最大有効期限を示す `expires_at` プロパティを持ちます。

**重要:**
このキャッシュは、短期間だけ保持される一時的なデータ向けに設計されています。項目は、指定された有効期間内であっても、後続のトランザクションでは利用できない場合があります。

| パラメーター | 説明                          |
| ------ | --------------------------- |
| `key`  | *文字列*。キャッシュに格納されているレコードのキー。 |

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, [options])`
</div>

指定したキーに対応する文字列値をキャッシュに保存または更新します。

このキャッシュに保存された値は、設定された Trigger のスコープ内で有効です。
これらには
[Actions Cache Limits](/ja/docs/customize/actions/limitations)
が適用されます。

この方法で保存された値の保持期間は、指定した
`ttl` または `expires_at` の値までです。有効期間が
指定されていない場合は、デフォルトで 15 分間保持されます。保持期間は、
[Actions Cache Limits](/ja/docs/customize/actions/limitations)
に記載されている最大期間を超えることはできません。

| Parameter            | Description                                                                                                                                                                                               |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `key`                | *String*。キャッシュに保存するレコードのキー。                                                                                                                                                                               |
| `value`              | *String*。保存するレコードの値。                                                                                                                                                                                      |
| `options`            | *Optional object*。キャッシュの動作を調整するためのオプション。                                                                                                                                                                  |
| `options.expires_at` | *Optional number*。Unix エポックからの経過時間をミリ秒で表した絶対有効期限。キャッシュされたレコードはこれより前に削除される場合がありますが、指定した `expires_at` を過ぎて保持されることはありません。<br />*注:* `ttl` の値も指定する場合は、この値を指定しないでください。両方のオプションが指定された場合は、2 つのうち早い方の有効期限が使用されます。 |
| `options.ttl`        | *Optional number*。このキャッシュエントリの有効期間 (ミリ秒単位) 。キャッシュされた値はこれより前に削除される場合がありますが、指定した `ttl` を過ぎて保持されることはありません。<br />*注:* `expires_at` の値も指定する場合は、この値を指定しないでください。両方のオプションが指定された場合は、2 つのうち早い方の有効期限が使用されます。         |
