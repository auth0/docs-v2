<Warning>
  Actions Modules は現在 EA で提供されています。この機能を使用すると、Okta の [Master Subscription Agreement](https://www.okta.com/legal/) に記載されている該当する Free Trial 条項に同意したものとみなされます。Auth0 のリリースステージの詳細については、[Product Release Stages](/ja/docs/troubleshoot/product-lifecycle/product-release-stages) を参照してください。
</Warning>

Actions Modules を使用すると、Auth0 テナント内の複数の Actions 間で再利用可能なコードを作成、管理、共有できます。

従来は各 Action が独立して動作していたため、NPM Modules などの外部依存関係を使わずに共通の関数を共有するのは困難でした。Actions Modules を使用すると、次のことが可能になります。

* Auth0 でホストされる関数を含むカスタムモジュールを作成する。
* 各カスタムモジュールで定義した関数を、1 つ以上の Actions で使用する。
* 各カスタムモジュールでは、個別のシークレットを定義し、NPM モジュールへの依存関係を追加できる。

Actions Modules を使用した Actions の作成方法の詳細については、[Write Your First Action Module](/ja/docs/customize/actions/modules/write-your-first-action-module) を参照してください。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Actions Modules は特定の Trigger や Action タイプに紐付かないため、そこで定義した関数は、異なる Triggers に紐付いた Actions 間で再利用できます。
</Callout>

Actions Modules の制限事項の詳細については、[Actions Limitations](/ja/docs/customize/actions/limitations) を参照してください。

<div id="performance">
  ## パフォーマンス
</div>

[コーディングガイドライン](/ja/docs/customize/actions/action-coding-guidelines) に従って Actions Modules を採用すると、生成コストの高いオブジェクトを共有・再利用できるため、実行パフォーマンスを改善できます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ただし、改善が見込めるかどうかは、具体的なユースケース、実装、実行時の負荷や実行頻度によって異なるため、必ずしも保証されるものではありません。
</Callout>
