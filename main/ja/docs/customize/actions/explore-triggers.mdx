sidebarTitle: 概要
title: トリガーを確認する
---

トリガーは、必要なカスタマイズのために Auth0 の処理フロー内で Actions を追加できるポイントです。1 つのトリガーには複数の Actions を配置でき、順番に実行されます。カスタムトークン交換を除き、トリガーは同期型 (ブロッキング) または非同期型 (ノンブロッキング) のいずれかです。各トリガーには、それぞれ固有のオブジェクトと API があります。これらは、使用している接続 (<Tooltip tip="パスワードレス: 第 1 認証要素としてパスワードに依存しない認証の形式。" cta="用語集を表示" href="/ja/docs/glossary?term=passwordless">パスワードレス</Tooltip> 接続など) によって異なる場合があります。

特定のトリガーで使用できる Actions のリファレンスオブジェクトと API を確認するには、Auth0 における [パスワードレス接続](/ja/docs/authenticate/passwordless) の定義を踏まえたうえで、以下のトリガーを選択してください。

<div id="signup-and-login">
  #### サインアップとログイン
</div>

ユーザーがデータベース接続またはパスワードレス接続に追加されるとき (Signup) 、またはユーザーの認証トランザクションの一部であるとき (Login) 。

| トリガー                     | 説明                             |
| ------------------------ | ------------------------------ |
| `pre-user-registration`  | ユーザーが作成される前にトリガーされます。          |
| `post-user-registration` | ユーザーの作成後に非同期でトリガーされます。         |
| `post-login`             | ユーザーの認証後、トークンが発行される前にトリガーされます。 |

<div id="mfa-notifications">
  #### MFA 通知
</div>

SMS を <Tooltip tip="多要素認証（MFA）: SMS で送信されるコードなど、ユーザー名とパスワードに加えて別の認証要素を使用するユーザー認証プロセス。" cta="用語集を表示" href="/ja/docs/glossary?term=Multi-Factor+Authentication">多要素認証</Tooltip> (MFA) の認証要素として使用する場合、またはカスタムプロバイダーを設定する場合に使用されます。

| トリガー                 | 説明                                                    |
| -------------------- | ----------------------------------------------------- |
| `send-phone-message` | 登録およびチャレンジのプロセスで送信するメッセージにカスタムプロバイダーを使用する場合にトリガーされます。 |

<div id="password-reset">
  #### パスワードリセット
</div>

パスワードリセットのリクエストが開始されたときに使用されます。

| トリガー                            | 説明                                    |
| ------------------------------- | ------------------------------------- |
| `post-change-password`          | パスワードの変更後にトリガーされます。                   |
| `password-reset-post-challenge` | 最初のチャレンジの完了後、パスワードがリセットされる前にトリガーされます。 |

<div id="machine-to-machine">
  #### マシン間通信
</div>

ユーザーではなく、アプリケーションが認証・認可されて API を使用する場合。

| トリガー                   | 説明                     |
| ---------------------- | ---------------------- |
| `credentials-exchange` | アクセストークンが返される前にトリガーします。 |

<div id="custom-token-exchange">
  ### カスタムトークン交換
</div>

ユーザーのログインを行うためのカスタムトークン交換リクエストを受信したとき。

| トリガー                    | 説明                                                 |
| ----------------------- | -------------------------------------------------- |
| `custom-token-exchange` | トランザクションの最初のステップでトリガーされ、post-login トリガーより前に実行されます。 |
