Actions の使用には、次の制限事項があります。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  エンティティの制限の詳細については、[Actions Entity Limits](/ja/docs/troubleshoot/customer-support/operational-policies/entity-limit-policy#actions)を参照してください。
</Callout>

<div id="actions">
  ## Actions
</div>

* 各 Action のサイズは 100 kB を超えないようにしてください。サイズが大きいほど遅延が増加し、システムのパフォーマンスに影響する可能性があります。このサイズ制限には、`require` ステートメントで参照される `npm` モジュールは含まれません。

<div id="actions-modules">
  ## Actions モジュール
</div>

* 各 Action モジュールは NPM モジュールに依存できますが、他の Actions モジュールには依存できません。
* **Actions モジュールには独自のランタイムがありません。** これらは、それを含む Action のランタイムで実行されます。

<Warning>
  **Action モジュールをビルドすると、Auth0 は builder ランタイムでビルドします。ただし、モジュールの実行時には、それをインポートした Action のランタイムが使用されます。**

  *例:*

  *Action モジュールで Node.js 22 では非推奨の Node.js API 呼び出しを使用している場合、Node.js 18 ランタイムを使用する Actions では問題なく動作しますが、Node.js 22 ランタイムを使用する Actions では失敗します。*
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  サポート予定のすべてのランタイム バージョンと互換性のある Node.js API を使用して Actions モジュールを作成してください。
</Callout>

<div id="account-linking-setprimaryuser">
  ## アカウントリンク (setPrimaryUser)
</div>

* `primary_user_id` は 128 文字までに制限されています
* `setPrimaryUser` は 1 つのトランザクションにつき 1 回だけ呼び出せます
* `setPrimaryUser` と同じ Action で設定した userMetadata は破棄され、失われます。同じトランザクション内の後続の Actions では、新しいプライマリユーザーの userMetadata は保持されます。
* `setPrimaryUser` は、Rule が `context.primaryUser.` を設定するトランザクションでは使用できません

<div id="cached-data">
  ## キャッシュされたデータ
</div>

* キャッシュされた項目は、最大 24 時間保持されます。
* Trigger ごとにキャッシュできるエントリは最大 20 件です。
* キャッシュキーの最大サイズは 64 バイト、値の最大サイズは 4kB です。
* キャッシュされたキーとその値の合計サイズは、8kB を超えてはなりません。
* キャッシュは、単一の実行中であれば同じ Trigger 内のすべての Actions で確実に利用できますが、その後の実行 (別のフロー、別のユーザーのログイン、またはユーザーがリダイレクト Action から戻る場合など) では保証されません。
* 制御が呼び出し元に戻る Execution (リダイレクトなど) を実行する Actions では、その後のアクションが異なるキャッシュ状態を持つ別のインスタンスでスケジュールされることがあります。同じ実行であっても、Action ごとにキャッシュデータの整合性が取れない可能性があります。

<div id="executions">
  ## 実行
</div>

* トリガーの実行は、それぞれ20秒以内に完了する必要があります。これを超えると、処理はエラーで終了します。この制限時間内に収めるには、HTTP リクエストを制限するのが最も効果的です。
* トリガーの実行は、それぞれ20秒以内に完了する必要があります。これを超えると、処理はエラーで終了します。この制限時間内に収めるには、タイムアウトのない送信 HTTP リクエストなど、長時間実行されるプロセスを制限する必要があります。[ユーザーを外部ページにリダイレクトする](/ja/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/redirect-with-actions) Action には、リダイレクト前とリダイレクト後にそれぞれ別個のタイムアウトが適用されます。
* Action トリガーが中断され、その後再開されるたびに、新しい `event.request` オブジェクトが発行されます (たとえば、リダイレクトや <Tooltip tip="Management API: お客様が管理タスクを実行するための製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=MFA">MFA</Tooltip> チャレンジによる場合) 。

<div id="logs">
  ## ログ
</div>

* 各Actionで `console.log()` の出力として永続的に保存されるのは、最大256文字です。
* 実行ログは10日間保持されます。

<div id="programming-languages">
  ## プログラミング言語
</div>

* Actions では TypeScript はサポートされていません。ソースファイルは、デプロイ前に JavaScript で記述する必要があります。

<div id="secrets">
  ## シークレット
</div>

* 各シークレットキーの長さは最大128文字です。
* 各シークレット値の長さは最大4096文字です。

<div id="saml-attributes">
  ## SAML 属性
</div>

* Actions では、最大 100 個の <Tooltip tip="多要素認証（MFA）: SMS で送信されるコードなど、ユーザー名とパスワードに加えて別の要素を使用するユーザー認証プロセス。" cta="用語集を見る" href="/ja/docs/glossary?term=SAML">SAML</Tooltip> 属性を変更または追加できます。
* SAML 属性名の最大サイズは 1kB です。
* SAML 属性値の最大サイズは 2kB です。
* SAML アサーション全体の最大サイズは 10kB です。

<div id="saml-configuration">
  ## SAML 設定
</div>

* `audience` の最大サイズは 2kB です
* `recipient` の最大サイズは 2kB です
* `destination` の最大サイズは 2kB です
* `nameIdentifierFormat` の最大サイズは 0.5kB です
* `nameIdentifierProbes` は最大 10 個まで指定でき、各プローブの最大サイズは 0.5kB です
* `authnContextClassRef` の最大サイズは 0.5kB です
* `signingCert` の最大サイズは 4kB です
* `encryptionCert` の最大サイズは 4kB です
* `encryptionPublicKey` の最大サイズは 4kB です
* `cert` の最大サイズは 4kB です
* `key` の最大サイズは 4kB です

<div id="service-requests">
  ## サービスへのリクエスト
</div>

* Auth0 <Tooltip tip="Security Assertion Markup Language (SAML): パスワードなしで 2 者間が認証情報をやり取りできるようにする標準化されたプロトコル。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> の呼び出しとユーザーメタデータの更新には、[レート制限](/ja/docs/troubleshoot/customer-support/operational-policies/rate-limit-policy/management-api-endpoint-rate-limits)が適用されます。
* 外部サービスへの HTTP 呼び出しは、[これらの IP アドレス](/ja/docs/secure/security-guidance/data-security/allowlist) から送信されます。

<div id="transaction-metadata">
  ## トランザクションメタデータ
</div>

* [post-login Actions](/ja/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger)でのみ利用できます。
* 認証トリガーの完了後は保持されません。
* 同じトランザクション内のActions以外からはアクセスできません。
* キーは64文字までに制限されます。
* 値は8 KBまでに制限されます。
* 値としてサポートされる型は、`string`、`number`、`boolean`のみです。
* 同じトランザクション内でのメタデータの合計サイズの上限は16 KBです。
* 安全性チェックに合格しない数値は、有効な値として受け付けられません。安全でない整数値は、開発者が安全にシリアライズする必要があります。詳細については、[safe integers](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Number/isSafeInteger)を参照してください。

<div id="user-metadata-and-app-metadata">
  ## ユーザーメタデータとアプリメタデータ
</div>

* 各セッションで永続化できるユーザーメタデータとアプリメタデータは、それぞれ最大 32kB です。
