---
description: Rules と Hooks から Actions に移行する際の、移行固有の作業に関する制限事項
title: Actions への移行における制限事項
---

Auth0 Actions が提供する機能を活用するには、現在のコードを Rules、Hooks、または以前のバージョンの Actions から移行する必要があります。ただし、従来の拡張機能から Actions へ移行する際には、現時点での制限事項を把握しておく必要があります。

<div id="general-differences">
  ## 一般的な違い
</div>

* Actions で返せるエラー code は `access denied` のみで、`unauthorized` エラー code を返すことはできません。
* `accessToken` の標準クレームは、Action では削除できません。

<div id="rule-specific-differences">
  ## Rules 固有の違い
</div>

* Post Login Actions の event オブジェクトでは、現時点で、Rules で利用できる `context.connectionOptions` に相当するものは提供されていません。
* Post Login Actions の event オブジェクトでは、現時点で、Rules で利用できる `user.identities` オブジェクト経由での、フェデレーションユーザーに対する上流 IdP のトークンへのアクセスに相当する手段は提供されていません。
* `idToken` の標準クレームは、Action を使用して削除することはできません。
* Actions でサポートされているのは、リンク済みアカウント間でのプライマリユーザーの変更のみです。
