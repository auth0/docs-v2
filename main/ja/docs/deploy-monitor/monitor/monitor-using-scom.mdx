System Center Operations Manager (SCOM) または合成トランザクションをサポートする任意のツールを使用して、標準的な Web アプリケーションとして Auth0 を監視できます。アプリケーションが依存している拡張機能 (会社の他のサービスとの統合のためにカスタムコードを実行する Rules など) を含む、合成ログイン トランザクションを追加することをお勧めします。

<div id="set-up-scom">
  ## SCOM を設定する
</div>

1. **Add Monitoring Wizard** を使用して、新しい SCOM インスタンスを追加します。

| Field                              | Description                 |
| ---------------------------------- | --------------------------- |
| Name                               | SCOM インスタンスの説明用の名前です。       |
| Description                        | この SCOM インスタンスで監視する内容の説明です。 |
| Select destination management pack | Default Management Pack     |

完了したら、**Next** をクリックして続行します。

2. **Add** をクリックし、SCOM で監視する URL を入力します。完了したら、**Next** をクリックして続行します。

3. **Add** をクリックして、監視元のロケーションを設定します。

4. ポップアップ ダイアログで、**Internal location - Agent** を検索します。適切なアドレスを選択し、**Add** をクリックします。次に、**Ok** をクリックしてロケーションの選択を完了します。完了したら、**Next** をクリックして続行します。

5. SCOM が各エンドポイントからデータを収集する頻度を設定します。

| Data                                 | Frequency |
| ------------------------------------ | --------- |
| Test frequency                       | 60 秒      |
| Performance data collection interval | 60 秒      |
| Test time-out                        | 30 秒      |
| HTTP status code                     | 400 以上    |

6. 完了したら、**Next** をクリックして続行します。

<div id="run-scom-tests">
  ## SCOM テストを実行する
</div>

1. **Run Test** をクリックして各エンドポイントをテストし、指定した接続設定が正しいことを確認します。
2. SCOM インスタンスの設定が完了したら、**Monitoring** タブでアクティビティを確認できます。

<div id="review-test-results">
  ## テスト結果を確認する
</div>

SCOM が収集した情報を表示するには、**Web Application Status** をクリックします。
