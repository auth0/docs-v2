title: ログを表示
---

<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> を使用すると、ユーザー認証や、アプリケーション、接続、Rule の追加・更新などの管理アクションを含め、テナント内で発生するすべてのログを表示できます。管理アクションは、ログでは **API Operation** ログとして表示されます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  デフォルトでは、ログは実際の発生順どおりに表示されない場合があります。テナントログをストリーミングしていて時系列順での並べ替えが必要な場合は、ストリーム処理側でログの並べ替えを実装することを推奨します。
</Callout>

1. [Dashboard &gt; Monitoring &gt; Logs](https://manage.auth0.com/#/logs) に移動します。

   <Frame>![Dashboard Monitoring Logs](/docs/images/cdy7uua7fh8z/2l9LfKEjrprVmHfWt80VX0/4e48bc6ad3a8b591014e23fdb7353405/2025-01-28_15-07-43.png)</Frame>
2. 一覧には最新のログが表示されます。検索を実行するには、検索フィールドにカーソルを置き、クエリを入力して **Enter** キーを押します。たとえば、&quot;Pass&quot; を含む接続のログをすべて検索するには、`connection:*pass*` と入力します。検索クエリの構文と例の詳細については、[ログ検索クエリ構文](/ja/docs/deploy-monitor/logs/log-search-query-syntax) を参照してください。
3. ログタイプで検索をフィルタリングすることもできます。ドロップダウンリストからフィルターを選択し、**Apply** をクリックします。たとえば、ユーザーのログイン失敗ログで絞り込むには、**Failed Login** を選択します。ログイベントフィルターと例については、[ログイベントフィルター](/ja/docs/deploy-monitor/logs/log-event-filters) を参照してください。

   <Frame>![Dashboard Monitoring Logs Filters](/docs/images/cdy7uua7fh8z/3bfjw3WURyQRiNhj6f8VlJ/5c71550ff0a1d51f81b0c65a4d2cc22a/dashboard-monitoring-logs-filters.png)</Frame>
4. 検索の日付範囲を選択できます。日付フィールドの横にあるドロップダウンをクリックし、特定の日付または期間を選択して、**Apply** をクリックします。
