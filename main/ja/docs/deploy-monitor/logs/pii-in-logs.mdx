description: Auth0 がテナントログに書き込む可能性のある個人を特定できる情報（PII）について説明します。
title: Auth0ログにおける個人を特定できる情報
---

Auth0 のテナントログ内の多くのログには、個人を特定できる情報 (PII) が含まれる可能性があります。認証イベントによってトリガーされるログエントリには、ユーザーの PII が含まれることがあります。また、Auth0 の <Tooltip tip="Management API: 顧客が管理タスクを実行できるようにする製品。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> または Dashboard を使用してユーザーを追加すると、Auth0 はユーザーアカウントの詳細をログに記録します。

ログエントリ内の次のフィールドには、PII が含まれる可能性があります。

* 名前
* 電話番号 (<Tooltip tip="多要素認証 (MFA): SMS による code など、username と password に加えて認証要素を使用するユーザー認証プロセス。" cta="用語集を表示" href="/ja/docs/glossary?term=multi-factor+authentication">多要素認証</Tooltip> に使用)
* メールアドレス
* IP アドレス (位置情報が明らかになる可能性があります)
* ユーザーに対して定義したカスタム情報

Auth0 は、Auth0 またはいかなる <Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可認証情報。" cta="用語集を表示" href="/ja/docs/glossary?term=access+tokens">アクセストークン</Tooltip> もログに記録しません。<Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可認証情報。" cta="用語集を表示" href="/ja/docs/glossary?term=identity+provider">IDプロバイダー</Tooltip> が発行したものも同様です。認可コードの交換が発生した場合、ログには code の一部のみが表示されます (例: `code: 31XXXXX`) 。
