---

Auth0のテナントログには、ユーザーの認証アクションに関するトランザクションの詳細が記録されます。テナントログには、サインイン、ログイン、検証の各プロセスに関する情報が記録され、確認できます。ユーザーがどのように認証されたか、どの認証情報を提供したか、トランザクションの所要時間、接続のステータスを確認できます。

以下は、テナントログに記録されるデフォルト値です。

| 属性            | 説明                                                |
| ------------- | ------------------------------------------------- |
| `completedAt` | ユーザーの操作時間を含むステージの完了時刻                             |
| `elapsedTime` | `completedAt` と `initiatedAt` の差                  |
| `flow`        | ログインエクスペリエンス (例: Universal または Classic) によって異なります |
| `initiatedAt` | ステージの開始時刻                                         |
| `name`        | ステージ名                                             |

<div id="login">
  ## ログイン
</div>

ログインステージでは、次の値が提供されます。

| 属性          | 説明                                                |
| ----------- | ------------------------------------------------- |
| `flow`      | Universal Login または Classic Login に応じて、2 つの値を取ります |
| `user_id`   | このステージで認証されたユーザーの Auth0 ID                        |
| `user_name` | このステージで認証されたユーザーの名前                               |

以下の情報を含む追加フィールド:

| 属性              | 説明                     |
| --------------- | ---------------------- |
| `connection`    | ユーザーの認証に使用された接続        |
| `connection_id` | ユーザーの認証に使用された接続の ID    |
| `strategy`      | ユーザーの認証に使用された接続のストラテジー |
| `identity`      | ユーザーの認証済みアイデンティティ      |

ログインフローの追加イベントの値:

| 名前                             | Universal Login Experience | 説明                                                    |
| ------------------------------ | -------------------------- | ----------------------------------------------------- |
| `federated-authenticate`       | Universal                  | 認証に使用されたフェデレーション接続 (ソーシャル、リーガル、またはエンタープライズ)           |
| `lock-password-authenticate`   | Classic                    | username と password でユーザーが認証された                       |
| `oauth1-authenticate`          | Classic                    | oauth1 フェデレーションでユーザーが認証された                            |
| `oauth2-authenticate`          | Classic                    | oauth2 フェデレーションでユーザーが認証された                            |
| `oidc-authenticate`            | Classic                    | oidc フェデレーションでユーザーが認証された                              |
| `prompt-authenticate`          | Universal                  | username と password でユーザーが認証された                       |
| `prompt-authenticate-password` | Universal                  | `idfirst` フローで username と password によりユーザーが認証された      |
| `prompt-signup`                | Universal                  | username と password を使用してユーザーがサインアップした                |
| `prompt-signup-password`       | Universal                  | `idfirst` フローで username と password を使用してユーザーがサインアップした |
| `saml-authenticate`            | Classic                    | SAML フェデレーションでユーザーが認証された                              |
| `wsfed-authenticate`           | Classic                    | WS-FED フェデレーションでユーザーが認証された                            |

<div id="consent">
  ## consent
</div>

consent ステージでは、次の値を利用できます。

| 属性          | 説明                                             |
| ----------- | ---------------------------------------------- |
| `grantInfo` | ユーザーが許可した API の `{"{myApiIdentifier}"}` または識別子 |
| `audience`  | トークンのオーディエンス                                   |
| `id`        | データベースに永続化されたユーザーのグラント                         |
| `scope`     | ユーザーが承認したスコープ                                  |

<div id="login-email-verification">
  ## login-email-verification
</div>

メールアドレス確認プロンプトでは、以下の値を利用できます。

| 属性                          | 説明                           |
| --------------------------- | ---------------------------- |
| ` login-email-verification` | メールアドレス確認プロンプトが表示された場合に含まれます |

<div id="redirect">
  ## redirect
</div>

redirect ステージでは、次の値を使用できます。

| 属性    | 説明                 |
| ----- | ------------------ |
| `URL` | ユーザーがリダイレクトされた URL |

<div id="mfa">
  ## mfa
</div>

<Tooltip tip="多要素認証 (MFA): ユーザー名とパスワードに加えて、SMS 経由の code などの認証要素を使用するユーザー認証プロセス。" cta="用語集を表示" href="/ja/docs/glossary?term=Multi-factor+authentication">多要素認証</Tooltip> では、次の値を使用できます。

| 属性              | 説明                                                                                         |
| --------------- | ------------------------------------------------------------------------------------------ |
| `flow`          | Classic エクスペリエンス (`mfa`) と Universal エクスペリエンス (`universal-mfa`) のどちらで使用するかに応じて、2 つの値を取ります。 |
| `performed_acr` | `id_token` に含まれる `acr`。                                                                    |
| `performed_amr` | `id_token` に含まれる `amr`。                                                                    |
| `provider`      | mfa の実行に使用されるプロバイダー。                                                                       |
