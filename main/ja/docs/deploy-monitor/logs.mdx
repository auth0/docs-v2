---

Auth0 では、ビジネスニーズに応じて分析できるログを提供しています。次のことができます。

* テナント管理者が実行したアクションを表示する。
* <Tooltip tip="Management API: 顧客が管理タスクを実行できるようにする製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を介して実行された操作を表示する。
* ユーザーによる認証を表示する。
* 品質保証テストでは見つけにくい断続的なエラーを表示する。
* セキュリティ調査や異常検知トリガーの特定に役立つフォレンジックデータを取得する。
* 利用傾向のパターンを特定する。
* 認証エラーや Management API 呼び出しエラーを確認する。
* 自動ログスキャンを設定してレート制限エラーを確認し、レート制限に達するアクティビティがユーザーに影響する前に、事前に対処できるようにする。
* `X-Correlation-ID` ヘッダー値を照合することで、Management API イベントをその発生元のリクエストに関連付ける。

<div id="log-streaming-and-exporting">
  ## ログのストリーミングとエクスポート
</div>

ログストリーミングを使用すると、ログを任意の処理ツールや分析ツールにエクスポートできます。複数の分析サービスとの統合に加えて、Webhook を使用してカスタムログストリームを作成する方法も提供しています。詳しくは、[Log Streaming](/ja/docs/customize/log-streams)を参照してください。
