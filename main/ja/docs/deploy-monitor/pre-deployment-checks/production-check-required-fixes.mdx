description: 本番環境へのデプロイ前に Auth0アプリケーションで必ず対応する必要がある修正
title: '本番環境 Readiness Checks: 必須の修正'
---

以下のチェックでは、本番環境へのデプロイを正常に行うために必要な要件をすべて満たしているかどうかを確認します。

チェック出力の詳細については、[Production Checks の実行](/ja/docs/deploy-monitor/pre-deployment-checks/how-to-run-production-checks)を参照してください。

| チェック項目                                                                                                                                                            | 説明                                                                                                                                                                                                                                                                                                |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| すべての Actions が[推奨バージョンの NodeJS](/ja/docs/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)で実行されている                                      | セキュリティパッチや新機能を活用できるよう、すべての[Actions](/ja/docs/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)を推奨されるバージョンのNodeJSに設定します。                                                                                                                                                 |
| [Management API v2 Authentication で IDトークンを許可](/ja/docs/troubleshoot/product-lifecycle/past-migrations/migrate-to-calling-api-with-access-tokens) が無効                | Management API の Users および Device Credentials エンドポイントの一部で、認可に IDトークン を使用する機能は非推奨となっています。アクセストークン への移行が完了したら、[Allow ID Tokens for Management API v2 Authentication](https://manage.auth0.com/#/tenant/advanced) トグルがオフになっていることを確認してください。この設定が表示されない場合、そのテナントはこの機能が非推奨になった後に作成されているため、既定で無効になっています。 |
| allowLegacyRoGrantTypes                                                                                                                                           | テナント移行設定のレガシーな [oauth/ro endpoint](/ja/docs/troubleshoot/product-lifecycle/past-migrations/migration-oauthro-oauthtoken) は非推奨となります。アプリケーションをサポート対象のエンドポイントを使用するように更新したら、このエンドポイントをオフにしてください。                                                                                                         |
| allowOtherLegacyGrantTypes                                                                                                                                        | テナント移行設定のレガシー [oauth/access&#95;token endpoint](https://auth0.com/docs/api/authentication/login-legacy/social-with-providers-access-token) は非推奨化されています。サポートされているエンドポイントを使用するようにアプリケーションを更新したら、このエンドポイントを必ず無効にしてください。                                                                              |
| allowLegacyTokenInfoEndpoint                                                                                                                                      | テナント移行設定のレガシー [tokeninfo エンドポイント](/ja/docs/troubleshoot/product-lifecycle/past-migrations#tokeninfo-endpoint-validation) は非推奨となっています。アプリケーションをサポート対象のエンドポイントを使用するように更新したら、このエンドポイントを必ず無効にしてください。                                                                                                     |
| allowLegacyDelegationGrantTypes                                                                                                                                   | テナント移行設定にあるレガシーな[デリゲーションエンドポイント](/ja/docs/authenticate/login/oidc-conformant-authentication/oidc-adoption-delegation)は非推奨になっています。アプリケーションをサポート対象のエンドポイントを使用するように更新したら、このエンドポイントは必ず無効にしてください。                                                                                                         |
| [Allowed Callback URLs](/ja/docs/authenticate/login/redirect-users-after-login) に Localhost を含めない                                                                    | [アプリケーションの Allowed Callback URLs](https://manage.auth0.com/#/applications) に `localhost`、`127.0.0.1` などが含まれていないことを確認します。                                                                                                                                                                          |
| [許可されたオリジン (CORS) ](/ja/docs/authenticate/login/cross-origin-authentication) を Localhost に設定しないこと                                                                    | ページの [Location URL](https://manage.auth0.com/#/applications) が `localhost` を指していないことを確認します。                                                                                                                                                                                                       |
| [Allowed Web Origins](/ja/docs/get-started/applications/application-settings) に `localhost` を含めない                                                                    | [Allowed Web Origins URLs](https://manage.auth0.com/#/applications) が `localhost` を参照していないことを確認します。                                                                                                                                                                                               |
| [メールプロバイダー](/ja/docs/customize/email/smtp-email-providers) が設定されている必要があります                                                                                           | [カスタムメールプロバイダー](https://manage.auth0.com/#/emails/provider) が設定済みであることを確認します。                                                                                                                                                                                                                     |
| Hooks は非推奨のため、[Actions へ移行](/ja/docs/customize/actions/migrate/migrate-from-hooks-to-actions)する必要があります                                                               | Hooks は非推奨となります。[Actions](/ja/docs/customize/actions/migrate/migrate-from-hooks-to-actions) の利用を開始し、移行プロセスを開始することを強くお勧めします。既存の Hooks は 2024 年 11 月 18 日まで引き続き有効です。                                                                                                                                   |
| [多要素認証](/ja/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) を設定しておく必要があります (依存関係: MFA が設定済みであること) | MFA で [電話メッセージ認証要素](https://manage.auth0.com/#/mfa) を使用している場合は、デフォルトの配信プロバイダーのレート制限を回避するため、Twilio またはカスタム配信プロバイダーを使用するよう設定してください。                                                                                                                                                                 |
| [レガシー ユーザープロフィール](/ja/docs/troubleshoot/product-lifecycle/past-migrations/migrate-from-legacy-auth-flows#user-profiles)は無効にする必要があります                                 | IDトークンおよび`/userinfo`エンドポイントに完全なユーザープロフィールを含められるレガシー認証フローは、非推奨となります。新しいOIDC準拠のAPIへの移行が完了したら、[Legacy User Profile](https://manage.auth0.com/#/account/advanced)トグルがオフになっていることを確認してください。                                                                                                             |
| Rules は非推奨となるため、[Actions への移行](/ja/docs/customize/actions/migrate/migrate-from-rules-to-actions)が必要です                                                                | Rules は非推奨となる予定です。[Actions](/ja/docs/customize/actions/migrate/migrate-from-rules-to-actions) の利用を開始し、移行プロセスに着手することを強く推奨します。既存の Rules は 2024 年 11 月 18 日まで引き続き有効です。                                                                                                                                  |
| [ソーシャル接続](/ja/docs/authenticate/identity-providers/social-identity-providers/devkeys) で Auth0 の開発者キーを使用しないこと                                                         | [Social Connections](https://manage.auth0.com/#/connections/social) でデフォルトの Auth0 開発者キーが使用されていないことを確認します。                                                                                                                                                                                         |
| サポート用メールアドレスが設定されていること                                                                                                                                            | Tenant Settings で [Support Email](https://manage.auth0.com/#/account) が設定されていることを確認します。                                                                                                                                                                                                           |
| サポート URL が設定されていること                                                                                                                                               | [Support URL](https://manage.auth0.com/#/account) が Tenant Settings で設定されていることを確認します。                                                                                                                                                                                                             |
| テナント環境タグが設定されていること                                                                                                                                                | [テナント環境タグ](https://support.auth0.com/tenants/public)が、Production、Staging、または Development に適切に設定されていることを確認します。このタグの変更は、テナントのレート制限に影響します。                                                                                                                                                           |
| テナントが推奨されるデフォルトの [NodeJS バージョン](/ja/docs/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-nodejs-22) を使用するよう設定されている                                | セキュリティパッチと新機能を利用するために、テナントを[推奨される NodeJS バージョン](/ja/docs/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)に更新してください。                                                                                                                                                    |
| テナントログイン URI                                                                                                                                                      | Tenant Settings で [Tenant Login URI](https://manage.auth0.com/#/tenant/advanced) が設定済みであることを確認します。                                                                                                                                                                                                |
| ブランド化されたメールテンプレートでカスタムドメインを使用                                                                                                                                     | メールテンプレートで[カスタムドメイン](/ja/docs/customize/custom-domains/configure-features-to-use-custom-domains#use-custom-domains-in-emails)を使用するようにしてください。ユーザー向けのすべてのメールは、カスタムドメインと会社のブランディングを使用するようカスタマイズすることを強くお勧めします。                                                                                           |
