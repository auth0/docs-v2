---
description: 本番環境へのデプロイ前に Auth0 アプリケーションを改善するための推奨事項
title: 'Production Readiness Checks: 重要ではない修正'
---

以下のチェックでは、本番環境への正常なデプロイに向けた推奨事項 (任意) をすべて完了しているかどうかを確認します。

チェック結果の詳細については、[本番環境チェックを実行する](/ja/docs/deploy-monitor/pre-deployment-checks/how-to-run-production-checks)を参照してください。

| チェック                                                                                                              | 説明                                                                                                                                                                                                                                    |
| ----------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Log Streaming を設定                                                                                                 | テナントのログエントリを外部ストレージまたはプロバイダーにストリーミングするようにしてください。ログを 30 日を超えて保持し、より適切な分析とアラートを可能にするため、[ログストリーミングサービスを設定する](/ja/docs/customize/log-streams)ことを推奨します。                                                                                       |
| [カスタムドメイン](/ja/docs/customize/custom-domains)を設定する必要があります                                                            | エンドユーザーに最もシームレスで安全な体験を提供するため、[Universal Login でカスタムドメインを使用](https://manage.auth0.com/#/tenant/custom_domains)してください。                                                                                                                  |
| [カスタムエラーページ](/ja/docs/customize/login-pages/custom-error-pages)を設定する必要があります                                          | アプリケーション固有の詳細情報と企業のブランディングを含めて、[カスタムエラーページを設定](https://manage.auth0.com/#/account)してください。                                                                                                                                             |
| Universal Login を有効にする                                                                                            | 最新バージョンの [Universal Login](/ja/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login) を使用していることを確認してください。アクセシビリティとパフォーマンスの向上、コード不要のカスタマイズ、新機能を利用するため、ログインページを Classic エクスペリエンスから Universal Login に更新することを推奨します。 |
| [メールテンプレート](/ja/docs/customize/email/manage-email-flow)を設定する必要があります                                                  | アプリケーション固有の詳細情報と企業のブランディングを含めて、[カスタムメールテンプレートを設定](https://manage.auth0.com/#/emails)してください。                                                                                                                                           |
| [多要素認証](/ja/docs/secure/multi-factor-authentication)を設定する必要があります                                                     | ユーザーの認証をより安全に行うため、多要素認証の導入を検討することを推奨します。                                                                                                                                                                                              |
| [テナント管理者向け MFA](/ja/docs/get-started/manage-dashboard-access) が有効になっています                                             | テナント管理者に対して [多要素認証を有効にする](https://manage.auth0.com/#/account/admins) ことを推奨します。                                                                                                                                                        |
| [リダイレクト ログアウト URL](/ja/docs/authenticate/login/logout#set-the-allowed-logout-urls-at-the-account-level) を定義する必要があります | アプリケーションの [Allowed Redirect Logout URLs](https://manage.auth0.com/#/account/advanced) を確認してください。                                                                                                                                      |
| アプリケーションでは JSON Web Token (JWT) の署名アルゴリズムに HS256 ではなく RS256 を使用する必要があります                                          | 各アプリケーションで、JSON Web Token (JWT) の[署名アルゴリズム](/ja/docs/get-started/apis#signing-algorithms)を HS256 ではなく RS256 に設定してください。                                                                                                                   |
| [Application Login URI](/ja/docs/get-started/applications/application-settings#application-uris) を設定する               | すべてのインタラクティブなアプリケーションに対して、[Application Login URI](/ja/docs/get-started/applications/application-settings#application-uris) を指定してください。                                                                                                    |
| [Tenant Allowed Logout URL](/ja/docs/get-started/applications/application-settings#application-uris) を設定する           | 少なくとも 1 つの公開アクセス可能なログアウトルートを作成し、そのルートを指すように[テナント設定を更新](/ja/docs/get-started/applications/application-settings#application-uris)してください。                                                                                                   |
| [Tenant Login URI](/ja/docs/get-started/tenant-settings#login-and-logout) を設定する                                      | デフォルトのログインルートを公開し、そのルートを指すように[テナント設定を更新](/ja/docs/get-started/tenant-settings#login-and-logout)してください。                                                                                                                                   |
