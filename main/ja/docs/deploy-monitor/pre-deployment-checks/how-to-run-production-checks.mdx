title: Production Readiness Checks を実行する
---

本番環境をデプロイする前に、テナントの本番チェック結果を確認します。

1. Auth0 Management Dashboard に移動し、アカウントにログインします。
2. ドロップダウンから、Production Checks を実行する対象のテナントを選択します
3. 同じドロップダウンで **Run Readiness Check** をクリックしてインターフェイスを開きます

<Frame>![](/docs/images/cdy7uua7fh8z/nHS8TrcMAsgUUWOv5gcNO/66d641136222fe6634b1d53f0a455ffd/Screenshot_2023-06-02_at_10.53.40_AM.jpeg)</Frame><Frame>![デプロイ前の Auth0 サポート本番チェック](/docs/images/cdy7uua7fh8z/4EjetzpVocGQKy6HYVTHMw/a4a4bfb85e8db8aaf393a7e97aeb8bb4/2025-01-28_15-33-08_copy.png)</Frame>

任意:

1. **Filters** をクリックし、このテナントに関連付けられている [Applications](/ja/docs/get-started/applications) のうち、チェックを実行するものを 1 つ以上選択します。
2. **Apply** をクリックします。

チェックが完了すると、画面が自動的に更新され、結果が表示されます。

<div id="production-check-results">
  ## 本番環境チェック結果
</div>

チェック結果の修正は、次の 3 つのカテゴリに分かれます。

* [必須の修正事項](/ja/docs/deploy-monitor/pre-deployment-checks/production-check-required-fixes)
* [推奨される修正事項](/ja/docs/deploy-monitor/pre-deployment-checks/production-check-recommended-fixes)
* [ベストプラクティス](/ja/docs/deploy-monitor/pre-deployment-checks/production-checks-best-practices)

チェックで表示される結果には、次のものがあります。

* 合格
* 不合格
* ベストプラクティス

各結果セットの下には、アプリケーションが合格したチェック数と、確認が必要なチェック数が Auth0 によって表示されます。

アプリケーションが 1 つ以上のチェックで**不合格**になった場合は、次の内容が表示されます。

* チェックの名前
* チェックで確認している内容に関する情報
* アプリケーションがチェックに合格できるよう、必要な修正を行う該当箇所に移動するための **Configure** リンク
* 対応するドキュメントへのリンク

アプリケーションが**合格**したすべてのチェックは、結果セットの passed オブジェクトにまとめて表示されます。チェックの名前と内容を確認できるほか、関連ドキュメントを参照したり、必要に応じて対応する設定箇所へのリンクから変更を行ったりできます。
