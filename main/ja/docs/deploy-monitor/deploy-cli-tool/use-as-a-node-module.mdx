---
description: Auth0 Deploy CLI を Node モジュールとして使用する方法を説明します。
title: Node モジュールとして使用する
---

Deploy CLI は、スタンドアロン CLI として使用できるだけでなく、Node モジュールとしても使用できます。これにより、柔軟な Node スクリプト内で Auth0 リソースを管理できます。

設定項目の一覧については、[Deploy CLI の設定](/ja/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli) を参照してください。

<div id="dump-function">
  ## dump 関数
</div>

Auth0テナントの設定をローカルマシンに取得します。

<div id="argument-parameters">
  ### 引数
</div>

<div id="format">
  #### format
</div>

オプション: `yaml` または `directory`。エクスポートされるリソース設定ファイルの形式を指定します。詳細については、[利用可能なリソース設定形式](/ja/docs/deploy-monitor/deploy-cli-tool/available-resource-configuration-formats)を参照してください。

<div id="output_folder">
  #### output_folder
</div>

パス。設定ファイルの書き込み先ディレクトリを指定します。

<div id="config">
  #### config
</div>

オブジェクト。ユーティリティの動作を設定するために使用します。詳細については、[Deploy CLI の設定](/ja/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli)を参照してください。

<div id="config_file">
  #### config_file
</div>

パスです。ユーザー定義の設定ファイル (`config.json`) を指定します。詳しくは、[Deploy CLI の設定](/ja/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli)を参照してください。

<div id="export_ids">
  #### export_ids
</div>

ブール値。有効にすると、各リソースの 識別子フィールドをエクスポートします。デフォルトは `false` です。

<div id="env">
  #### env
</div>

ブール値。ツールが環境変数を読み込むかどうかを示します。既定値: `false`。

<div id="proxy_url">
  #### proxy_url
</div>

リクエストをプロキシ経由で送信するための URL。プロキシ環境下にある場合にのみ設定してください。

<div id="example">
  ### 例
</div>

```jsx lines
import { dump } from 'auth0-deploy-cli';

dump({
  output_folder: './local',
  format: 'yaml',
  config: {
    AUTH0_DOMAIN: '<YOUR_AUTH0_TENANT_DOMAIN>',
    AUTH0_CLIENT_ID: '<YOUR_AUTH0_CLIENT_ID>',
    AUTH0_CLIENT_SECRET: '<YOUR_AUTH0_CLIENT_SECRET>',
  },
})
  .then(() => {
    console.log('Auth0設定のエクスポートが成功しました');
  })
  .catch((err) => {
    console.log('Auth0設定のエクスポート中にエラーが発生しました:', err);
  });
```

<div id="deploy-function">
  ## deploy 関数
</div>

ローカルマシンから Auth0 テナントへ設定を適用します。

<div id="argument-parameters">
  ### 引数
</div>

<div id="input_file">
  #### input_file
</div>

パス。リソース設定ファイルの場所を指定します。YAML 形式の場合は `tenant.yaml` ファイルを、ディレクトリー形式の場合はリソース設定ディレクトリーを指定します。

<div id="config">
  #### config
</div>

オブジェクト。ユーティリティの動作を構成します。詳しくは、[Deploy CLI の設定](/ja/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli)を参照してください。

<div id="config_file">
  #### config_file
</div>

パス。ユーザー定義の設定ファイル (`config.json`) のパスを指定します。詳細については、[Deploy CLI の設定](/ja/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli)を参照してください。

<div id="export_ids">
  #### export_ids
</div>

ブール値。有効にすると、各リソースの識別子フィールドがエクスポートされます。デフォルト: `false`。

<div id="env">
  #### env
</div>

ブール値。ツールが環境変数を読み込むかどうかを示します。デフォルト: `false`。

<div id="proxy_url">
  #### proxy_url
</div>

リクエストをプロキシ経由で送信するための URL。プロキシ環境下にある場合にのみ設定してください。

<div id="example">
  ### 例
</div>

```jsx lines
import { deploy } from 'auth0-deploy-cli';

deploy({
  input_file: './local/tenant.yaml',
  config: {
    AUTH0_DOMAIN: '<YOUR_AUTH0_TENANT_DOMAIN>',
    AUTH0_CLIENT_ID: '<YOUR_AUTH0_CLIENT_ID>',
    AUTH0_CLIENT_SECRET: '<YOUR_AUTH0_CLIENT_SECRET>',
  },
})
  .then(() => {
    console.log('Auth0の設定がテナントに正常に適用されました');
  })
  .catch((err) => {
    console.log('Auth0テナントへの設定適用中にエラーが発生しました:', err);
  });
```
