<div id="use-automated-deployment-for-actions">
  ## Actions には自動デプロイを使用する
</div>

<Tooltip tip="Auth0 Dashboard: サービスを構成するための Auth0 の主要プロダクトです。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> エディター内で Action を作成することは、開発段階で実装やテストを行ううえで有効な方法です。ただし、自動テスト環境や本番環境にデプロイする段階では、より自動化された仕組みが必要になります。Auth0 テナント間でコードをコピー＆ペーストする方法は、適切とは言えません。

<div id="set-up-version-control-for-actions">
  ### Actions のバージョン管理を設定する
</div>

Auth0 Dashboard で Action に加えた変更は、デプロイされるまで下書きとして保存されます。デプロイするとバージョンが作成されますが、有効化するには Action をトリガーに関連付ける必要があります。変更管理やコードレビューを行うために、バージョン管理の利用を推奨します。[Actions は Dashboard でバージョン管理をネイティブにサポートしています](docs/customize/actions/manage-versions)。

Auth0 では、Auth0 のテナント環境間で Actions を自動デプロイする方法を提供しています。次の Auth0 統合を使用すると、バージョン管理システムから Actions を手動または自動で更新できます (たとえば、バージョン管理システム内の変更が検出された場合など) 。[GitHub Actions](https://marketplace.auth0.com/integrations/github-actions)、[GitLab Pipelines](https://marketplace.auth0.com/integrations/gitlab-pipeline)、[Bitbucket Pipelines](https://marketplace.auth0.com/integrations/bitbucket-pipeline)、[Microsoft Azure Pipelines](https://marketplace.auth0.com/integrations/azure-pipeline)。

<div id="use-deploy-cli-tool-for-action-import-and-export">
  ### Action のインポートとエクスポートに Deploy CLI ツールを使用する
</div>

Auth0 Deploy CLI ツールは、Auth0 テナント間での Actions のデプロイを自動化します。このツールは、ローカルの YAML ファイルまたはディレクトリベースのファイルを Auth0 の <Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> と組み合わせて使用し、Auth0 テナントから Actions をエクスポートしたり、Auth0 テナントにインポートしたりできます。さらに、このツールでは、デプロイ自動化の一環として、Action の順序や環境設定をプログラムで制御できます。詳細については、[Deploy CLI Tool](/ja/docs/deploy-monitor/deploy-cli-tool) を参照してください。

ベストプラクティスとして、テスト環境または本番環境へのデプロイが関係するほぼすべてのケースで Auth0 Deploy CLI を使用してください。Marketplace 統合では変更を自動検出できますが、Deploy CLI を使用すると、何を、いつ、どのようにデプロイするかを正確に制御できます。

<div id="set-up-test-tenants">
  ## テスト用テナントを設定する
</div>

本番環境にデプロイする前に Actions を安全にテストできるよう、Auth0 では別のテスト用テナントを設定することを推奨しています。詳細については、[複数の環境を設定する](/ja/docs/get-started/auth0-overview/create-tenants/set-up-multiple-environments)を参照してください。
