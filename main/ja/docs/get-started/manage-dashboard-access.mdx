Auth0 のテナント管理者は、テナントメンバーの管理を含め、Auth0 アカウントおよびテナント配下で行われるすべてのアクティビティに責任を負います。テナントメンバー (Dashboard ユーザー) は、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> または [Auth0 Teams Dashboard](/ja/docs/get-started/auth0-teams/tenant-member-management) から追加、変更、削除できます。Auth0 では、Auth0 テナントにアクセスできる Auth0 Dashboard のテナントメンバー一覧を定期的に見直し、次の点を確認することを推奨しています。

* 各メンバーにテナントメンバーとしてのアクセス権が必要な正当な理由があること。
* メンバーが会社のアカウントで登録されていること。
* 退職者がアクセスできないこと。
* Dashboard 管理者が複数いること。

以下は、特別なアクセス要件が必要になる可能性があるユーザーの例です。

* アプリのエンドユーザーのログイン問題をトラブルシューティングする必要があるサポート担当者。
* エンドユーザーにロールや権限を割り当てる必要があるサポート担当者／IT 担当者。
* 本番環境でアプリケーションをトラブルシューティングする必要がある開発者。
* アプリケーションの設定や利用状況を分析する必要があるプロダクトマネージャー。
* 自身のアプリケーションの設定を行う必要がある開発者。
* B2B のユースケースで顧客向けの接続を作成する必要があるサポート担当者／IT 担当者。
