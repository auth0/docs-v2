Auth0 Model Context Protocol (MCP) Server は、AI エージェントを Auth0 テナントに接続し、アプリケーションの作成、ユーザー管理、Actions のデプロイなど、Auth0 での複雑な複数ステップの操作を実行できるようにします。これは、AI エージェントが外部ツールや API と通信し、ユーザーに代わってタスクを実行できるようにする標準化されたプロトコルである [Model Context Protocol (MCP)](https://modelcontextprotocol.io/introduction) を活用しています。

Cursor、Windsurf、Claude Desktop などの AI 搭載ツールを使う開発者向けに設計された Auth0 MCP Server は、ID インフラストラクチャの管理方法を一変させ、Auth0 サービスをよりすばやく簡単に利用できるようにします。

<div id="authentication-and-credential-security">
  ## 認証と認証情報のセキュリティ
</div>

Auth0 MCP Server は、認証の管理、リクエストの検証、Auth0 の<Tooltip tip="Management API: お客様が管理タスクを実行するための製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip>との安全な通信の確保を通じて、堅牢なセキュリティを最優先しています。これには、次が含まれます。

* [OAuth 2.0 デバイス認可フロー](/ja/docs/get-started/authentication-and-authorization-flow/device-authorization-flow): Auth0 へのリクエストを安全に認証します。
* [認証情報の安全な保管](/ja/docs/get-started/applications/rotate-credentials): 認証情報はシステムのキーチェーンに安全に保存され、平文で公開されることはありません。
* [最小限の API Permissions](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens): 最小権限の原則に厳密に従い、必要最小限の Permissions のみを要求します。

<div id="supported-tools-and-integrations">
  ## 対応ツールと統合
</div>

Auth0 MCP Server は現在、次の [MCP クライアント](https://modelcontextprotocol.io/clients) と統合されています。

* [Claude Desktop](https://claude.ai/download)
* [Cursor](https://www.cursor.com/downloads)
* [Windsurf](https://www.windsurf.com/editor)

今後、主要な新しいクライアントの登場に合わせて、追加の統合が順次追加される予定です。サポートされているツールの一覧と各機能の詳細については、[Auth0 Model Context Protocol (MCP) Server Tools Reference](/ja/docs/get-started/auth0-mcp-server/auth0-mcp-tools-reference) を参照してください。
