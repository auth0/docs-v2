Auth0 MCP Server と任意の [MCP Client](https://modelcontextprotocol.io/clients) を使用して Auth0 テナントを操作するための自然言語の例については、このリファレンスガイドを参照してください。

<div id="application-management">
  ## アプリケーション管理
</div>

| ツール                         | 説明                                        | 自然言語の例                                                                                                                                                                                                                |
| --------------------------- | ----------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `auth0_list_applications`   | Auth0テナント内のすべてのアプリケーションを一覧表示するか、名前で検索します。 | 「Auth0のアプリケーションをすべて表示してください。」<br /><br />「名前に &#39;api&#39; を含むアプリケーションを検索してください。」<br /><br />「Auth0テナントにはどのアプリケーションがありますか？」                                                                                          |
| `auth0_get_application `    | 特定のAuth0アプリケーションの詳細を取得します。                | 「&#39;Customer Portal&#39; という名前のアプリケーションの詳細を表示してください。」<br /><br />「クライアントID {clientID} のアプリケーション情報を取得してください。」<br /><br />「&#39;Mobile App&#39; のcallback URLは何ですか？」                                                  |
| `auth0_create_application ` | 新しいAuth0アプリケーションを作成します。                   | 「&#39;Analytics Dashboard&#39; という名前の新しいシングルページアプリケーションを作成してください。」<br /><br />「&#39;iOS Client&#39; という名前の新しいネイティブモバイルアプリを作成してください。」<br /><br />「バックグラウンドサービス用のマシンツーマシンアプリケーションを作成してください。」                            |
| `auth0_update_application`  | 既存のAuth0アプリケーションを更新します。                   | 「&#39;Web App&#39; のcallback URLに `https://staging.example.com/callback` を追加してください。」<br /><br />「&#39;Customer Portal&#39; のログアウトURLを変更してください。」<br /><br />「&#39;Admin Dashboard&#39; アプリケーションに開発環境用のメタデータを追加してください。」 |

<div id="resource-server-api-management">
  ## リソースサーバー (API) の管理
</div>

| ツール                            | 説明                                       | 自然言語の例                                                                                                                                                        |
| ------------------------------ | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `auth0_list_resource_servers`  | Auth0 テナント内のすべてのリソースサーバー (API) を一覧表示します。 | 「Auth0 テナント内の API をすべて表示して。」<br /><br />「リソースサーバーを一覧表示して。」<br /><br />「Auth0 で設定している API には何がある？」                                                             |
| `auth0_get_resource_server`    | 特定の Auth0 リソースサーバーの詳細を取得します。             | 「『User API』の詳細を表示して。」<br /><br />「『Payment API』に定義されているスコープは？」<br /><br />「識別子 `https://api.example.com` のリソースサーバー情報を取得して。」                                   |
| `auth0_create_resource_server` | 新しい Auth0 リソースサーバー (API) を作成します。         | 「read スコープと write スコープを持つ『Inventory API』という新しい API を作成して。」<br /><br />「顧客データ API 用のリソースサーバーを設定して。」<br /><br />「識別子 `https://orders.example.com` の API を作成して。」 |
| `auth0_update_resource_server` | 既存の Auth0 リソースサーバーを更新します。                | 「『User API』に『admin』スコープを追加して。」<br /><br />「『Payment API』のトークン有効期間を 1 時間に更新して。」<br /><br />「API の署名アルゴリズムを RS256 に変更して。」                                       |

<div id="combining-operations">
  ## 操作を組み合わせる
</div>

[Claude](https://claude.ai/) に、一連の関連する操作を続けて実行するよう依頼することもできます。

<div id="set-up-a-new-api-with-client-access">
  #### クライアントアクセスを有効にした新しいAPIを設定する
</div>

```bash wrap lines
Create a new API called "Inventory API" with read and write scopes, then update my "Admin Dashboard" application to have access to it.
```

<div id="create-and-deploy-a-security-action">
  #### セキュリティ Action の作成とデプロイ
</div>

```bash wrap lines
社内ネットワーク外からログインするユーザーに MFA を要求する Action を作成し、Login フローにデプロイしてください。
```

<div id="analyze-security-events">
  #### セキュリティイベントの分析
</div>

```bash wrap lines
過去1週間のログイン失敗の試みをすべて表示し、パターンや不審なアクティビティを特定してください。
```

<div id="updating-branding-across-forms">
  #### フォーム全体でブランディングを更新する
</div>

```bash wrap lines
Update all our Auth0 forms to use our new logo at <https://example.com/new-logo.png> and change the primary color to #0066CC.
```

<div id="best-practices">
  ## ベストプラクティス
</div>

1. リクエストは具体的にする: 特定のリソースを指定する場合は、名前、ID、そのほかの識別情報を含めてください。
2. 自然言語を使う: ツール名やパラメーターを暗記する必要はありません。実現したいことをそのまま説明してください。
3. 確認前に見直す: Claude が変更を提案した場合は、確認する前に内容をよく見直してください。
4. まずは簡単なものから始める: Claude がリクエストをどのように解釈するかに慣れるため、まずは簡単な問い合わせから始めてください。

<div id="troubleshooting">
  ## トラブルシューティング
</div>

Claude が操作を実行できない場合は、次の点を確認してください。

1. 認証: Auth0 CLI (`auth0 login`) を使用してログインしていることを確認してください。
2. 権限: Auth0 で、アカウントに必要な権限が付与されていることを確認してください。
3. 具体性: リクエストの内容をより具体的にしてみてください。
