title: アプリケーションのシングルサインオン統合を有効にする
---

デフォルトでは、シームレスな<Tooltip tip="シングルサインオン（SSO）: ユーザーが1つのアプリケーションにログインすると、そのユーザーは自動的に他のアプリケーションにもログインされるサービスです。" cta="用語集を表示" href="/ja/docs/glossary?term=single+sign-on">シングルサインオン</Tooltip> (SSO) は、すべての新しいAuth0テナントで有効になっています。ただし、**レガシーテナント**では、[この機能をテナントレベルで有効にするかどうかを選択できます](/ja/docs/get-started/tenant-settings/enable-sso-for-legacy-tenants)。テナントレベルのSSOを有効にしていない場合は、アプリケーションごとに有効にできます。

アプリケーションのSSOは、<Tooltip tip="Auth0 Dashboard: サービスを設定するためのAuth0のメイン製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> を使用して有効にできます。

<div id="prerequisite">
  ## 前提条件
</div>

アプリケーションでSSOを有効にする前に、使用する各<Tooltip tip="IDプロバイダー（IdP）：デジタルアイデンティティを保存および管理するサービス。" cta="用語集を見る" href="/ja/docs/glossary?term=identity+provider">IDプロバイダー</Tooltip>用の接続を、事前に作成して設定する必要があります。ソーシャルIDプロバイダーの場合、[Universal Login experience](/ja/docs/authenticate/login/auth0-universal-login)を使用する場合は、その接続で開発者キーを使用していないことを確認してください。

<div id="enable-sso">
  ## SSO を有効にする
</div>

1. [Auth0 Dashboard &gt; Applications &gt; SSO Integrations](https://manage.auth0.com/#/externalapps) に移動し、表示する統合の名前を選択します。
2. **Settings** ページの最下部までスクロールし、**Use Auth0 instead of the IdP to do シングルサインオン** を見つけてトグルを有効にし、**Save Changes** を選択します。
