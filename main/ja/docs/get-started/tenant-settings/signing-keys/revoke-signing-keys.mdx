<div id="prerequisites">
  ## 前提条件
</div>

* 以前使用していた署名鍵を失効させるには、まずその鍵をローテーションしておく必要があります。詳細については、[署名鍵のローテーション](/ja/docs/get-started/tenant-settings/signing-keys/rotate-signing-keys)を参照するか、以下の**署名鍵のローテーションと失効**セクションを確認してください。
* 以前の鍵を失効させる前に、新しい鍵に合わせてアプリケーションまたは API を更新済みであることを確認してください。

<Warning>
  署名鍵は失効後に再利用できないため、失効させて問題ないことを必ず確認してください。
</Warning>

<div id="use-the-dashboard">
  ## Dashboard を使用する
</div>

<div id="revoke-previously-used-signing-key">
  ### 以前使用した署名鍵を失効する
</div>

1. [Dashboard &gt; Settings &gt; Signing Keys](https://manage.auth0.com/#/tenant/signing_keys) に移動します。
2. **List of Valid Keys** セクションで **Previously Used** キーを探し、その他のオプション (**...**) メニューから **Revoke Key** を選択します。
   **List of Valid Keys** セクションには、テナントで現在使用中の署名鍵と、署名鍵をローテーションした場合に次に割り当てられる署名鍵が表示されます。過去に署名鍵をローテーションしたことがある場合は、以前使用した鍵もこのセクションに表示されます。
   **List of Revoked Keys** セクションには、テナントで最後に失効した 3 つの鍵が表示されます。
3. 確認のため **Revoke** を選択します。

<div id="rotate-and-revoke-signing-key">
  ### 署名鍵のローテーションと失効
</div>

1. [Dashboard &gt; Settings &gt; Signing Keys](https://manage.auth0.com/#/tenant/signing_keys) に移動します。
2. **Rotation Settings** セクションで **Rotate &amp; Revoke Signing Key** を見つけ、**Rotate &amp; Revoke Key** を選択します。
3. 確認ダイアログで **Rotate &amp; Revoke** を選択します。

<div id="use-the-management-api">
  ## Management API を使用する
</div>

<Warning>
  失効できるのは、以前使用していた署名鍵のみです。
</Warning>

1. 署名鍵の一覧を取得するには、[Get all Application Signing Keys](https://auth0.com/docs/api/management/v2#!/Keys/get_signing_keys) エンドポイントに `GET` リクエストを送信します。
2. [Revoke an Application Signing Key by its Key ID](https://auth0.com/docs/api/management/v2#!/Keys/put_signing_keys) エンドポイントに `PUT` リクエストを送信します。プレースホルダー `{yourKeyId}` と `{yourMgmtApiAccessToken}` は、それぞれ署名鍵の ID と Management API アクセストークンに置き換えてください。
