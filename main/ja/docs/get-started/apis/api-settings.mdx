description: Auth0 Dashboard で利用できる API に関連する設定について説明します。
title: API 設定
---

認可済みアプリケーションから利用する登録済み API を設定するには、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> の [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) にある **Settings** タブを使用します。API の設定を構成するには、一覧内の API の横にある **...** をクリックして **Settings** を選択するか、API 名をクリックします。API を作成して登録する方法については、[API を登録する](/ja/docs/get-started/auth0-overview/set-up-apis) を参照してください。

<Frame>![Dashboard の Applications API 一覧](/docs/images/cdy7uua7fh8z/3rhmhghYZDSi6YWHRA5yMQ/c71340259481b0b6787d5f3887cfda0f/dashboard-apis-list.png)</Frame>

<div id="settings">
  ## 設定
</div>

このタブでは、トークンの有効期限、ロールベースアクセス制御 (RBAC) 、その他のアクセス設定を構成できます。変更を保存するには、タブの下部にある **Save** をクリックします。

<div id="general-settings">
  ### 一般設定
</div>

これらのフィールドは、Auth0 の <Tooltip tip="Management API: お客様が管理タスクを実行するための製品。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> の場合を除き、API を最初に登録した際に設定されるものです。変更できるのは **名前** のみです。

<Frame>![Dashboard アプリケーション API 設定タブ 一般設定](/docs/images/cdy7uua7fh8z/QCaXd3FSeP6SYqLfEJWBr/fbb1bb5b2e12fb099073c33fb39cf44a/dashboard-applications-apis-settings-general-settings.png)</Frame>

* **Id**: Auth0 によって生成される一意の英数字文字列です。この情報は読み取り専用で、[Auth0 の Management API のリソースサーバー endpoint](https://auth0.com/docs/api/management/v2#!/Resource_Servers/get_resource_servers_by_id) を直接操作する場合にのみ必要です。
* **名前**: API のわかりやすい名前です。機能には影響しません。次の文字は使用できません: `< >`。
* **識別子**: API の一意の識別子です。この値は API の作成時に設定され、その後は変更できません。URL を使用することを推奨しますが、公開されている URL である必要はありません。Auth0 が API を呼び出すことは一切ありません。

<div id="token-settings">
  ### トークン設定
</div>

<Frame>![Dashboard - API - Token Settings - Expiration](/docs/images/cdy7uua7fh8z/2FFfqqmntXL9CnGrSxea9M/810e5542553168ffcd6081a10bc2db2f/image2.png)</Frame>

* **最大<Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可資格情報。" cta="用語集を見る" href="/ja/docs/glossary?term=Access+Token">アクセストークン</Tooltip>有効期間 (秒)&#x20;**: アクセストークンの有効期限が切れるまでの時間 (秒数) です。デフォルト値は 86400 秒 (24 時間) です。設定可能な最大値は 2592000 秒 (30 日) です。
* **Implicit / Hybrid Flow アクセストークン有効期間 (秒)&#x20;**: Implicit Flow または Hybrid Flow で発行されたアクセストークンの有効期限が切れるまでの時間 (秒数) です。デフォルト値は 86400 秒 (24 時間) です。この値は、アクセストークンの最大有効期間を超えることはできません。
* **<Tooltip tip="JSON Web Token (JWT): 2 者間でクレームを安全に表現するために使用される、標準的な IDトークン 形式（多くの場合はアクセストークン形式でもあります）。" cta="用語集を見る" href="/ja/docs/glossary?term=JSON+Web+Token">JSON Web Token</Tooltip> (JWT) プロファイル**: このプロファイルは、API 用に発行されるアクセストークンの形式を決定します。使用できる値は `Auth0` と `RFC 9068` です。詳しくは、[アクセストークンプロファイル](/ja/docs/secure/tokens/access-tokens/access-token-profiles)をご覧ください。
* **JSON Web Token (JWT) <Tooltip tip="署名アルゴリズム: トークンが改ざんされていないことを保証するために、トークンにデジタル署名する際に使用されるアルゴリズム。" cta="用語集を見る" href="/ja/docs/glossary?term=Signing+Algorithm">署名アルゴリズム</Tooltip>**: トークンの署名に使用するアルゴリズムです。この署名は、JWT の送信者が名乗っている本人であることを検証し、メッセージが途中で変更されていないことを保証するために使用されます。使用できる値は [`HS256`](https://en.wikipedia.org/wiki/Symmetric-key_algorithm) と [`RS256`](https://en.wikipedia.org/wiki/Public-key_cryptography) です。`RS256` (推奨) を選択すると、トークンはテナントの秘密鍵で署名されます。この値は API の作成時に設定され、その後は変更できません。署名アルゴリズムと Auth0 での動作について詳しくは、[署名アルゴリズム](/ja/docs/get-started/applications/signing-algorithms)をご覧ください。

  <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
    署名は JWT の一部です。JWT の構造に不慣れな場合は、[JSON Web Token Structure](/ja/docs/secure/tokens/json-web-tokens/json-web-token-structure)をご覧ください。
  </Callout>
* **JSON Web Encryption (JWE):** 有効にすると、発行されるアクセストークンは JSON Web Encryption (JWE) 形式で暗号化されます。詳しくは、[JSON Web Encryption](/ja/docs/secure/tokens/access-tokens/json-web-encryption)をご覧ください。

<div id="rbac-settings">
  ### RBAC 設定
</div>

<Frame>![Auth0 Dashboard API 設定の RBAC トグル](/docs/images/cdy7uua7fh8z/65tKb6aj0ktc2qXLUVlV3e/641adef615d6af9e5a3b588ff397af87/dashboard-apis-edit_view-settings_rbac-settings.png)</Frame>

* **RBAC を有効にする**: API に対してロールベースアクセス制御のポリシーを適用するには、この設定を有効にします。詳細については、[ロールベースアクセス制御](/ja/docs/manage-users/access-control/rbac) および [API でロールベースアクセス制御を有効にする](/ja/docs/get-started/apis/enable-role-based-access-control-for-apis) を参照してください。トラブルシューティングについては、[ロールベースアクセス制御と認可のトラブルシューティング](/ja/docs/troubleshoot/authentication-issues/troubleshoot-rbac-authorization) を参照してください。
* **アクセストークンに Permissions を追加する**: アクセストークンに Permissions クレームを追加するには、この設定を有効にします。この設定は、RBAC を有効にした場合にのみ使用できます。権限は **Permissions** タブで設定できます。

<div id="access-settings">
  ### アクセス設定
</div>

<Frame>![Dashboard Applications APIs Settings Tab Access Settings](/docs/images/cdy7uua7fh8z/JNlWACqoyfmpE89JwWLKy/5a2b595f939167b0e6e1f72b485b08e7/2025-02-25_14-53-21.png)</Frame>

* **ユーザーの同意のスキップを許可**: この設定を有効にすると、「ファーストパーティ」として指定されたアプリケーションでは、API のユーザー同意をスキップできます。
* **オフラインアクセスを許可**: この設定を有効にすると、アプリケーションが API に対して <Tooltip tip="リフレッシュトークン: ユーザーに再度ログインさせることなく、新しいアクセストークンを取得するために使用されるトークンです。" cta="用語集を見る" href="/ja/docs/glossary?term=refresh+tokens">リフレッシュトークン</Tooltip> を要求できるようになります。

<div id="default-policy-setting">
  #### デフォルトポリシーの設定
</div>

<Warning>
  **My Account API** の認証保証は現在、単一オプションのポリシーで 早期アクセスとして提供されています。この機能を使用すると、Okta の [Master Subscription Agreement](https://www.okta.com/legal) に定められた該当する Free Trial 条項に同意したものとみなされます。Auth0 の製品リリースサイクルの詳細については、[Product Release Stages](/ja/docs/troubleshoot/product-lifecycle/product-release-stages)を参照してください。プログラムに参加するには、[Auth0 Support](https://support.auth0.com) までお問い合わせください。
</Warning>

<Frame>![デフォルトポリシー API 設定](/docs/images/api-settings-access-default-policy.png)</Frame>

* **2FA を必須にする**: 有効にすると、Auth0 は My Account API にデフォルトポリシーを適用し、[ステップアップ認証](/ja/docs/secure/multi-factor-authentication/step-up-authentication)を要求します。ユーザーは 15 分以内に第 2 の認証要素で認証する必要があります。動作の詳細と有効化の手順については、[My Account API: Default Policy](/ja/docs/manage-users/my-account-api#default-policy)を参照してください。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  デフォルトポリシーは Classic Login では利用できません。
</Callout>

<div id="permissions">
  ## Permissions
</div>

API で使用する権限 (スコープ) を定義するには、**Permissions** タブの設定を使用します。詳細については、[Add API Permissions](/ja/docs/get-started/apis/add-api-permissions) および [Delete API Permissions](/ja/docs/get-started/apis/delete-api-permissions) を参照してください。
