export const codeExample = `curl --request PATCH \\
  --url 'https://{yourDomain}/api/v2/resource-servers/API_ID' \\
  --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' \\
  --header 'content-type: application/json' \\
  --data '{"token_dialect": "TOKEN_DIALECT" }'
`;

API で使用する <Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可資格情報。" cta="用語集を表示" href="/ja/docs/glossary?term=access+token">アクセストークン</Tooltip> プロファイルとして、Auth0 トークンプロファイルまたは RFC 9068 トークンプロファイルを選択できます。設定したアクセストークンプロファイルによって、その API に対して発行されるアクセストークンの形式が決まります。

デフォルトでは、Auth0 は Auth0 トークンプロファイルを使用してアクセストークンを発行します。Auth0 トークンプロファイルでは、アクセストークンは [JSON Web Tokens (JWTs)](/ja/docs/secure/tokens/json-web-tokens) 形式で発行され、エンティティに関する情報がクレームとして含まれます。

Auth0 は RFC 9068 トークンプロファイルもサポートしています。RFC 9068 トークンプロファイルでは、アクセストークンが [IETF JWT Profile for OAuth 2.0 Access Tokens (RFC 9068)](https://datatracker.ietf.org/doc/html/rfc9068) に準拠した <Tooltip tip="JSON Web Token (JWT): 2 者間でクレームを安全に表現するために使用される標準の IDトークン形式（また、多くの場合はアクセストークン形式）。" cta="用語集を表示" href="/ja/docs/glossary?term=JWTs">JWT</Tooltip> 形式で発行されます。これらのトークンプロファイルの違いについて詳しくは、[Access Token Profiles](/ja/docs/secure/tokens/access-tokens/access-token-profiles) を参照してください。

[API を登録する](/ja/docs/get-started/auth0-overview/set-up-apis)ときに、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメインプロダクト。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> でアクセストークンプロファイルを選択できます。API の登録後も、[Management API](https://auth0.com/docs/api/management/v2) と [Auth0 Dashboard](https://manage.auth0.com/) を使用して、いつでもアクセストークンプロファイルを設定できます。

<div id="configure-access-token-profile-for-an-api">
  ## API のアクセストークンプロファイルを設定する
</div>

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. [Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis) に移動し、対象の API の名前をクリックします。

    2. **Access Token Settings** までスクロールし、**JSON Web Token (JWT) Profile** でアクセストークンプロファイルを選択します。選択したプロファイルによって、その API 用に発行されるアクセストークンの形式とクレームが決まります。サポートされている値は `Auth0` と `RFC 9068` です。2 つのプロファイルの違いについて詳しくは、[Access Token Profiles](/ja/docs/secure/tokens/access-tokens/access-token-profiles) を参照してください。

    3. **Save** をクリックします。
  </Tab>

  <Tab title="Management API">
    Management API では、アクセストークンプロファイルを token dialect と呼びます。アクセストークンプロファイルを設定するには、Management API を使用して API の `token_dialect` パラメーターを設定します。

    次のコードサンプルは、[Update a resource server endpoint](https://auth0.com/docs/api/management/v2/resource-servers/patch-resource-servers-by-id) に PATCH リクエストを送信します。

    <AuthCodeBlock children={codeExample} language="bash" />

    `API_ID`、`MGMT_API_ACCESS_TOKEN`、`TOKEN_DIALECT` を、次の表で説明するそれぞれの値に置き換えます。

    | パラメーター                  | 説明                                                                                                               |
    | ----------------------- | ---------------------------------------------------------------------------------------------------------------- |
    | `API_ID`                | トークンダイアレクトを更新する対象 API の ID。                                                                                      |
    | `MGMT_API_ACCESS_TOKEN` | スコープ `update:resource_servers` を持つ [Management API 用のアクセストークン](https://auth0.com/docs/api/management/v2/tokens)。 |
    | `TOKEN_DIALECT`         | 指定した API のアクセストークンダイアレクト。詳しくは、[Token dialect options](#token-dialect-options) を参照してください。                         |

    #### トークンダイアレクトのオプション

    Auth0 では、次のトークンダイアレクトをサポートしています。

    | 値                       | 説明                                                                                                                                                                                                                                                           |
    | ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
    | `access_token`          | デフォルトの Auth0 トークンプロファイルでは、JSON Web Token (JWT) 形式のアクセストークンが生成されます。詳しくは、[Access Token Profiles](/ja/docs/secure/tokens/access-tokens/access-token-profiles) を参照してください。                                                                                           |
    | `access_token_authz`    | `permissions` クレームを含む、デフォルトの Auth0 トークンプロファイル (`access_token`)。RBAC の権限について詳しくは、[Enable Role-Based Access Control for APIs](/ja/docs/get-started/apis/enable-role-based-access-control-for-apis) を参照してください。                                                     |
    | `rfc9068_profile`       | RFC 9068 トークンプロファイルでは、[IETF JWT Profile for OAuth 2.0 Access Tokens (RFC 9068)](https://datatracker.ietf.org/doc/html/rfc9068) に準拠した JWT 形式のアクセストークンが生成されます。詳しくは、[Access Token Profiles](/ja/docs/secure/tokens/access-tokens/access-token-profiles) を参照してください。 |
    | `rfc9068_profile_authz` | `permissions` クレームを含む RFC 9068 プロファイル (`rfc9068_profile`)。RBAC の権限について詳しくは、[Enable Role-Based Access Control for APIs](/ja/docs/get-started/apis/enable-role-based-access-control-for-apis) を参照してください。                                                          |
  </Tab>
</Tabs>
