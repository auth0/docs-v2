このチュートリアルでは、クラウドでホストされる通常の Web アプリが、既存の Active Directory ユーザーストアを利用しながら、認証に Auth0 を使用するというシンプルなシナリオを扱いました。<Tooltip tip="OpenID: アプリケーションがログイン情報を収集および保存することなく、ユーザーの本人確認を行えるようにする認証のためのオープン標準です。" cta="用語集を見る" href="/ja/docs/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) で何ができるのか、またこのユースケースでそれが適していた理由、認証フローがどのように機能するか、<Tooltip tip="OpenID: アプリケーションがログイン情報を収集および保存することなく、ユーザーの本人確認を行えるようにする認証のためのオープン標準です。" cta="用語集を見る" href="/ja/docs/glossary?term=ID+Token">IDトークン</Tooltip> とは何か、またそれをどのように検証し操作するか、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip> でアプリケーションと接続をどのように設定するか、Lock を使用してユーザーのログインとログアウトをどのように実装するか、そしてセッション管理とアクセス制御がどのように機能するかを学びました。

まず、ビジネスケースと要件を説明し、続いて各要件をどのように満たせるか、またそれぞれの選択の背景にある考え方を説明しました。

サンプルの Web アプリ実装には ASP.NET Core を使用しましたが、このドキュメントを読み終えるころには、お好みのフレームワークを使って同様の Web アプリを構築できるようになっているはずです。
