[Server + API Architecture Scenario](/ja/docs/get-started/architecture-scenarios/server-application-api) では、シンプルなマシン間シナリオを取り上げました。ExampleCo のシステムにタイムシートの記録をインポートするための API と、外部委託先がこの API を使ってタイムシートを送信するための cron ジョブです。

クライアントクレデンシャルグラントと<Tooltip tip="アクセストークン: API へのアクセスに使用される認可資格情報で、不透明な文字列または JWT の形式です。" cta="用語集を見る" href="/ja/docs/glossary?term=access+tokens">アクセストークン</Tooltip>、Auth0 で API を設定する方法、この API と安全に通信するためのマシン間アプリケーションの設定方法、API エンドポイントを定義して保護する方法、提供されているライブラリを使ってアクセストークンを検証する方法、そして Auth0 から新しいアクセストークンを取得する方法について学びました。

API の実装には Node.js を使用し、非対話型のサーバープロセスには Python を使用しました。この例が、お好みの技術で独自の実装を構築する助けになれば幸いです。
