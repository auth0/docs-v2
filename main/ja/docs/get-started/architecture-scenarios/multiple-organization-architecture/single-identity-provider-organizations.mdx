---

単一 <Tooltip tip="IDプロバイダー（IdP）: デジタル ID を保存および管理するサービス。" cta="用語集を見る" href="/ja/docs/glossary?term=Identity+Provider">IDプロバイダー</Tooltip> (IdP) 組織のシナリオでは、[Auth0 Organizations](/ja/docs/manage-users/organizations/organizations-overview) 機能を使用するすべての組織が、必ず 1 つの Auth0 接続を使用します。この機能により、各アプリケーションは Auth0 テナントで定義された適切な組織を選択でき、その組織がユーザーの資格情報を直接認証するか、適切な IdP に認証リクエストを転送して処理させます。

このシナリオでは、引き続き [Travel0 Corporate Booking](/ja/docs/get-started/architecture-scenarios/multiple-organization-architecture) の例を使用します。この例では、法人向け予約アプリケーションがあなたのアプリケーションに相当し、Travel0 があなたの会社に相当します。

<Frame>![Architecture Scenarios - Multitenancy - Diagram - Isolated users, Shared Apps, Orgs A\&B](/docs/images/cdy7uua7fh8z/1YBaMaMozamiTv8hXmNB2D/21e9270a59021a12209bdcb4345147cb/isolated-users_shared-apps_org-ab.png)</Frame>

<Warning>
  Auth0 Organizations 機能は現在、[SAML](/ja/docs/authenticate/protocols/saml/saml-configuration)、[Open ID Connect](/ja/docs/authenticate/protocols/openid-connect-protocol) プロトコル、および/または [OAuth 2](/ja/docs/authenticate/protocols/oauth) フレームワークを使用するアプリケーションをサポートしています。[WS-Fed](/ja/docs/authenticate/protocols/ws-fed-protocol) を必要とするアプリケーションはサポートされていません。
</Warning>

資格情報の処理方法が、データベース接続にユーザーの資格情報を保存する場合 (Hoekstra &amp; Associates のようなケース) でも、エンタープライズ接続 (MetaHexa Bank のようなケース) やソーシャル接続を介して検証する場合でも、アプリケーションの動作は同じです。[Organizations](/ja/docs/manage-users/organizations/organizations-overview) 機能を活用することで、エクスペリエンスを構成し、制御できます。
