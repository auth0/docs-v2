Auth0 では、製品やコミュニティを効果的に活用できるよう、さまざまなリソースを提供しています。この一覧では、利用可能なリソースへのリンクをカテゴリ別に紹介します。

<div id="get-started">
  ## はじめに
</div>

Auth0 の基本を学ぶために役立つリソースには、次のものがあります。

* [**Getting Started ドキュメント**](/ja/docs/get-started): <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> や、Auth0 サービスの各コンポーネントで使われる一般的な用語を確認できます。Auth0 の全体像を把握し、Auth0 の担当者とのやり取りやドキュメントを読む際に目にする用語を学べます。
* [**アーキテクチャシナリオ**](/ja/docs/get-started/architecture-scenarios): 一般的なアーキテクチャシナリオを確認し、Auth0 を使ってそれらを実装する方法を学べます。シナリオには、API を呼び出す Single-Page Application (SPA) など、一般的なアーキテクチャパターンに関するチュートリアルが含まれます。概要レベルの説明はアーキテクトに役立ち、チュートリアルは開発チームに役立ちます。
* [**実装ガイド**](/ja/docs/): ユーザー管理や <Tooltip tip="多要素認証（MFA）: SMS で送信される code など、username と password に加えて認証要素を使用するユーザー認証プロセス。" cta="用語集を表示" href="/ja/docs/glossary?term=multi-factor+authentication">多要素認証</Tooltip> (MFA) など、よく使われる機能の実装方法を学べます。この情報は、アーキテクトと開発者の両方に役立ちます

<div id="learn">
  ## 学ぶ
</div>

Auth0 では、学習にも、必要な情報をすばやく参照したい場合にも役立つ、数多くのチュートリアル、ガイド、ホワイトペーパー、ブログ記事を提供しています。

* [**Docs site**](/ja/docs/): ドキュメントを参照して幅広いトピックを確認したり、検索を使用してトピックや用語に関連するコンテンツをすばやく見つけたりできます。
* [**Feature descriptions and white papers**](https://auth0.com/learn/): 機能の簡単な説明、業界のケーススタディ、参考用のホワイトペーパーを確認できます。Auth0 Learn サイトでは、Auth0 の機能とそのビジネス価値の概要をすばやく把握できます。プロジェクトオーナーだけでなく、アーキテクトや開発者にも役立ちます。
* [**Blog posts**](https://auth0.com/blog/): さまざまなトピックについて専門家が執筆したブログ記事を読むことができます。長年通用してきた助言から、アイデンティティ分野の最新ニュースまで、幅広い内容を扱っています。多くのブログ記事は、アーキテクトや開発者向けです。

<div id="run-sample-code">
  ## サンプルコードを試す
</div>

開発チームで実装の準備が整ったら、Auth0 が提供するサンプルプログラム、SDK、ライブラリを活用して、プロジェクトを迅速に進められます。

* [**Quickstarts**](/ja/docs/quickstarts): 認証、セッション管理、ユーザープロファイルの更新、ログアウトなど、アプリケーションに組み込みたい主要な機能の実装方法を示す、豊富なサンプルプログラムを確認できます。Quickstarts を利用すれば、開発者はアプリケーションを Auth0 と統合する方法をすばやく把握できます。
* [Libraries and SDKs](/ja/docs/libraries): 豊富なライブラリや SDK を利用することで、ID プロトコルの詳細の多くを意識せずに済み、カスタムアプリケーションの開発を簡素化できます。Auth0 の SDK は多数の言語とフレームワークをサポートしており、Auth0 との統合を容易にします。また、iOS や Android など複数のプラットフォームで利用できる、ログインウィジェット Lock 用のライブラリも提供しています。
* [**Management API**](https://auth0.com/docs/api/management/v2): Auth0 内のオブジェクト、構成、設定を確認、操作できます。<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> Explorer を使用すると、個々のオブジェクトや設定をその場で素早く操作し、API 呼び出しをアプリケーションに実装する前にテストできます。
* [**Authentication API**](https://auth0.com/docs/api/authentication): OIDC、<Tooltip tip="OAuth 2.0: 認可プロトコルとワークフローを定義する認可フレームワーク。" cta="用語集を表示" href="/ja/docs/glossary?term=OAuth">OAuth</Tooltip>、および <Tooltip tip="OAuth 2.0: 認可プロトコルとワークフローを定義する認可フレームワーク。" cta="用語集を表示" href="/ja/docs/glossary?term=SAML">SAML</Tooltip> プロトコルを介して、ユーザーの認証と認可を行います。Authentication API Explorer を使用すると、認証や <Tooltip tip="Authorization Flow: OAuth 2.0 フレームワークで規定される認可グラント（またはワークフロー）。" cta="用語集を表示" href="/ja/docs/glossary?term=authorization+flows">認可フロー</Tooltip>を試し、API 呼び出しをアプリケーションに実装する前にテストできます。

<div id="try-out-features-and-api-calls">
  ## 機能と API 呼び出しを試す
</div>

開発中は、Auth0 のさまざまな機能を試してテストできます。

* **TRY ボタン/リンク**: Auth0 の機能をすばやく試せます。**TRY** ボタンは Auth0 全体に配置されており、接続、Rules、Hooks、メールテンプレートを試すことができます。
* [**Authentication API Explorer**](https://auth0.com/docs/api/authentication#introduction): 認証フローと認可フローを試し、アプリケーションに実装する前に API 呼び出しをテストできます。
* [**Management API Explorer**](https://auth0.com/docs/api/management/v2): 個々のオブジェクトや設定を必要に応じてすばやく操作し、アプリケーションに実装する前に API 呼び出しをテストできます。

<div id="get-help">
  ## サポートを利用する
</div>

実装のトラブルシューティングに役立つ Auth0 のリソースには、次のようなものがあります。

* [**Auth0 コミュニティフォーラム**](https://community.auth0.com/): Auth0 の投稿、FAQ、コミュニティ Q&amp;A を通じて、Auth0 ユーザーとつながることができます。アーキテクトや開発者にとって、学習や他のユーザーとの交流に加え、問題解決の支援を得るための貴重な情報源です。
* [**Support Center**](https://support.auth0.com/): サブスクリプションやテナントの表示と管理、サポートリクエストの送信と確認、テナントに対する自動本番チェックの実行、コンプライアンス情報の確認ができます。ドキュメントやフォーラムを検索しても問題や疑問を解決できない場合、有料プランの契約者にとって Support Center は有用なリソースです。

  * [**サポートケースを作成する**](/ja/docs/troubleshoot/customer-support/open-and-manage-support-tickets): サポートが必要な場合は、サポートケースを作成して送信します。
  * [**サポートプランとサービスレベル契約**](/ja/docs/troubleshoot/customer-support#defect-responses): 購入可能な複数のサポートレベルについて確認します。
  * [**トラブルシューティングのヒント**](/ja/docs/deploy-monitor/deploy-private-cloud/private-cloud-on-aws#what-to-check-before-logging-an-issue) および [**サポートケースに含める情報**](/ja/docs/deploy-monitor/deploy-private-cloud/private-cloud-on-aws#information-to-provide-when-logging-an-issue): 開発チームやサポートチームが問題を分析する際に役立つアドバイスを確認します。
* [**サポート対象バージョン**](/ja/docs/troubleshoot/customer-support/product-support-matrix): サポートされている SDK、ブラウザー、言語のバージョンを確認します。アーキテクトや開発者は、プロジェクトで使用する言語、ライブラリ、SDK によって実装が Auth0 で動作することを確認するために、これを確認する必要があります。
* [**Feedback Portal**](https://auth0.com/feedback): 製品に関する提案を送信します。 (過去に送信した内容をより見やすく確認したい場合は、Support Center から送信することもできます。) アーキテクトや開発者は、このサイトを使用して、今後の機能強化の検討材料となる Auth0 製品へのフィードバックを提供できます。
* [**Professional Services**](/ja/docs/get-started/professional-services): 世界各地のプロフェッショナルサービスチームを活用して、プロジェクトの成功を加速できます。プロジェクトオーナーにとっては、Auth0 のアイデンティティ専門家がどのようにプロジェクトの推進を支援し、一時的なスキル不足を補えるかを知るうえで役立ちます。

<div id="set-up-and-monitor-operations">
  ## 運用を整備して監視する
</div>

リリースを計画する段階になったら、Auth0 では次のリソースを利用できます。

* [**リリース前のアドバイス**](/ja/docs/deploy-monitor/pre-deployment-checks) と [**本番環境チェック**](/ja/docs/deploy-monitor/pre-deployment-checks/how-to-run-production-checks): リリース計画に役立つヒントやツールを確認できます。円滑なリリースに向けて、プロジェクトマネージャー、開発チーム、運用チームはこれらの情報を確認してください。
* [**運用ポリシー**](/ja/docs/troubleshoot/customer-support/operational-policies): Auth0 のポリシーを把握し、運用上の依頼に必要なリードタイムを確認してください。チーム全体に有用ですが、特にプロジェクトオーナーと運用チームは Auth0 の運用ポリシーを理解しておく必要があります。
* [**Status Dashboard**](https://status.auth0.com/): Auth0 サービスの可用性をすばやく確認し、ステータス更新を購読できます。サービス停止が発生することはまれですが、発生した場合は Auth0 が根本原因分析を実施し、その結果をこのサイトに公開します。運用チームとサポートチームは、Auth0 のステータス確認方法を把握しておく必要があります。
* [**エンドポイントを監視する**](/ja/docs/deploy-monitor): 監視用エンドポイントをお使いの監視基盤に統合する方法を確認できます。この情報は特に運用チームとプロジェクトオーナーに役立ちます。
* [**ログデータ**](/ja/docs/deploy-monitor/logs): Auth0 が提供するログの種類、ログデータの保持期間、さらに分析や長期保存のためにログデータを外部の分析ツールにエクスポートする際に使用できるツールについて確認できます。この情報は、開発者や運用チームに加え、データ保持に関心のあるコンプライアンスチームにも役立ちます。
* [**Dashboard の通知**](/ja/docs/get-started/architecture-scenarios/business-to-consumer/operations#notifications): Auth0 からの重要なお知らせを把握できます。Auth0 は随時、重要な情報を Auth0 Dashboard を通じて通知し、情報の重大度に応じて、登録済みの Auth0 Dashboard Admin にメールでも通知します。Dashboard に定期的にログインし、上部のベルアイコンで重要なお知らせを確認してください。

  ## プライバシー、セキュリティ、コンプライアンスの要件に対応する
