[Auth0 Dashboard](https://manage.auth0.com/#) では、Auth0 のサブスクリプションと設定のあらゆる側面を管理できます。

<Frame>![Auth0 Dashboard のアクティビティページ](/docs/images/cdy7uua7fh8z/6k0cRLNEPko6vY6Kw7nIQD/4590d1b9d3e79f8d9c9970d733b08d02/2025-01-28_14-34-41.png)</Frame>

この Dashboard は複数のセクションで構成されており、左側のサイドバーメニューから移動できます。

<div id="configure-implementation">
  ## 実装を設定する
</div>

次の表は、Dashboard の各セクションと、それぞれで実行できる内容の概要を示しています。

| Section             | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Applications**    | アプリケーション、API、シングルサインオン (SSO) 連携を管理します。<br />**Applications**: Auth0 でユーザーを認証する各アプリについて、アプリケーションを登録します。<br />**APIs**: Auth0 で保護する各 API について、API を登録します。新しい API を作成し、既存の API を管理します。<br />**SSO Integrations**: SSO 用の外部サービスを表示して有効にします。新しい SSO 連携を作成し、連携設定の構成、確認、管理を行います。                                                                                                                                                                                                                                                                                              |
| **Authentication**  | ユーザーがアプリに認証できるようにする IDプロバイダーを管理します。<br />**Database**: 識別子/パスワードの認証情報を、Auth0 のデータストアまたは独自のデータベースに安全に保存して管理します。認証のたびに Auth0 のサーバー上で実行されるテンプレートベースの JavaScript スクリプトを使用して、既存のデータベースに接続します。既存のレガシー認証情報データベースは、ユーザーの認証に合わせて Auth0 に段階的に移行できます (パスワードリセットは不要です) 。<br />**Social**: ユーザーがログインに使用できるソーシャル IDプロバイダー (Facebook、X、Github など) を設定します。<br />**Enterprise**: ユーザーが企業の認証情報を使用してログインできるようにするエンタープライズ IDプロバイダー (Active Directory、SAML、Office 365 など) を設定します。<br />**Passwordless**: パスワードの代わりに、ワンタイムパスコード (メールまたは SMS で配信) やワンクリックリンクを使用して、ユーザーがサインアップやログインを行えるようにします。 |
| **Organizations**   | 取引先の組織を管理し、それらのユーザーがアプリケーションにアクセスする際の体験をカスタマイズします。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **User Management** | ユーザーのアイデンティティと権限を管理します。<br />**Users**: ユーザープロフィールの表示と作成、パスワードリセットの実行、ユーザーのブロックや削除などを行います。<br />**Roles**: アプリのロールを作成して管理します。ロールには権限のセットが含まれ、ユーザーに割り当てることができます。                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Branding**        | **Universal Login**: ユーザーを認証のためにリダイレクトできるログインページを作成してカスタマイズします。<br />**Custom Domains**: ユーザーに一貫した体験を提供するためにカスタムドメインを作成します。<br />**Email Templates**: テンプレートを使用して、ウェルカムメール、パスワードリセット、アカウント確認のメールベースのワークフローを作成します。<br />**Email Provider**: カスタムメールプロバイダーの情報を指定して設定します。                                                                                                                                                                                                                                                                                                 |
| **Security**        | さまざまな種類の攻撃やユーザーアクセスの異常からユーザーを保護する防御機能を有効にして、追加のセキュリティレイヤーを構成します。<br />**Attack Protection**: ボット、IP スロットリング、ブルートフォース攻撃、漏えいパスワード攻撃に対する設定を管理します。<br />**Multi-factor Auth**: 不正アクセスを防ぐため、ログインプロセス中に追加の認証要素を要求します。<br />**Monitoring**: データ可視化やアラート用の連携機能を使用して、脅威インテリジェンスイベントを監視します。                                                                                                                                                                                                                                                                                      |
| **Actions**         | ログイン、マシン間、ユーザー登録、パスワードリセットなどのフローを設定します。フローで使用するカスタマイズされた Action を作成して管理します。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| **Auth Pipeline**   | **Rules**: 各ユーザー認証トランザクションの一部として Auth0 で実行されるカスタム JavaScript スニペットを設定します。外部 API の呼び出し、アプリケーションにログインできるユーザーの絞り込み、AllowList の使用、地理情報に基づくアクセスの設定などを行えます。<br />**Hooks**: Database Connections を使用する際の Auth0 の動作を、拡張ポイント (サーバーに備わる webhook に相当) で実行される Node.js コードを設定することでカスタマイズします。                                                                                                                                                                                                                                                                                       |
| **Monitoring**      | **Logs**: 	管理者が Dashboard で実行した操作やユーザーのログインに関するログデータを表示します。<br />**Streams**: 外部のデータ分析サービスへのログイベントストリーミングを作成して管理します。                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Marketplace**     | Auth0 をさらに活用するための統合を確認します。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **Extensions**      | 公式およびサードパーティの Add-on で Auth0 プラットフォームを拡張します。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Settings**        | テナントを設定し、Auth0 のサブスクリプションと支払いオプションを管理し、テナント管理者やその他のユーザーロールを制御します。カスタムドメイン、署名鍵、その他の高度な設定に関連するテナント設定も管理します。                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Get Support**     | [Support Center](https://support.auth0.com) にアクセスします。プランにサポートサービスへのアクセスが含まれていない場合は、[Auth0 Community](https://community.auth0.com/) を参照してください。                                                                                                                                                                                                                                                                                                                                                                                                                          |

<div id="manage-account-settings">
  ## アカウント設定を管理する
</div>

左上には、テナントの名前、アイコン、小さな矢印が表示されます。この矢印をクリックするとドロップダウンメニューが開き、アカウントのさまざまな項目を設定できます。

* [**設定**](/ja/docs/get-started/tenant-settings): テナントのさまざまな項目を設定します。
* [**メンバーを招待**](/ja/docs/get-started/manage-dashboard-access): 管理者やその他のロールとして、追加のユーザーをテナント設定に追加します。
* [**クォータ使用状況**](https://support.auth0.com/reports/quota): サブスクリプションとテナントのクォータ使用状況を確認できます。
* [**テナントを作成**](/ja/docs/get-started/auth0-overview/create-tenants): 新しいテナントを作成します。
* **テナントを切り替える**: [複数のテナント](/ja/docs/get-started/auth0-overview/create-tenants/create-multiple-tenants)がある場合は、このオプションで切り替えられます。あるテナント用に作成したアプリケーションは、別のテナントの一覧には表示されません。

右上には、クリックできるオプションがいくつか表示されます。

* **検索**: アプリ、Marketplace のエントリ、ユーザー、そのほかテナントに関連するコンテンツを検索します。
* **ニーズについて相談:** Auth0 のエキスパートに連絡して、Auth0 で目標を達成するための支援を受けられます。
* **ドキュメント:** 現在読んでいるドキュメントサイトへのリンクです。
* **通知ベル:** Auth0 からの新しい連絡や、テナント設定に関する通知を知らせます。
* **プロファイル:** ローカルテナントのプロファイルへのリンクに加え、[テーマを変更](/ja/docs/get-started/dashboard-profile/light-and-dark-themes)したりログアウトしたりするオプションもあります。
