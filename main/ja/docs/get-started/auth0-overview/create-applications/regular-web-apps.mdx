description: Auth Dashboard を使用して、通常の Web アプリケーションを登録および設定する方法について説明します。
title: 通常の Web アプリケーションを登録する
validatedOn: 2026-02-25
---

通常の Web アプリを Auth0 と統合するには、まず <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品です。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> を使用して、アプリを Auth0 に登録する必要があります。通常の Web アプリは、アプリケーション ロジックの大部分をサーバー側で実行する従来型の Web アプリケーションです (例: Express.js、ASP.NET) 。アプリケーションの種類の詳細については、[アプリケーションの種類](/ja/docs/get-started/applications) を参照してください。

1. [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) に移動し、**Create Application** をクリックします。
2. アプリケーションがわかる名前を入力し、**Regular Web Applications** を選択して、**Create** をクリックします。

   新しいアプリケーションが作成されると、次のタブを含む **アプリケーション設定** ページを表示できます。

   | 設定タブ                 | 説明                                                                                                                                                                                                                                                                                                                         |
   | -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**      | アプリケーション タイプで利用可能なすべてのドキュメントを表示します。                                                                                                                                                                                                                                                                                        |
   | **Settings**         | アプリケーションで使用できるすべての設定を表示します。デフォルトでは、ほとんどの設定は自動的に作成されます。詳細については、[アプリケーション設定](/ja/docs/get-started/applications/application-settings) を参照してください。                                                                                                                                                                       |
   | **Credentials**      | アプリケーションの認証方式と、設定済みの認証情報を表示します。詳細については、[Application Credentials](/ja/docs/secure/application-credentials) を参照してください。                                                                                                                                                                                                          |
   | **Add-ons**          | アプリケーションに関連付けられたプラグインを有効にできます。これらは、Auth0 がアクセストークンを生成する SAML または WS-Fed の Web アプリです。詳細については、[Enable SAML2 Web App Addon](/ja/docs/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) および [Configure WS-Fed Applications](/ja/docs/get-started/applications/update-application-connections) を参照してください。 |
   | **Connections**      | アプリケーションで接続を有効にできます。接続はユーザーのソースであり、複数のアプリケーション間で有効化して共有できます。詳細については、[Connections](/ja/docs/authenticate/identity-providers) を参照してください。                                                                                                                                                                                        |
   | **Organizations**    | ユーザーが所属する組織に対して認証できるようにします。詳細については、[Organizations](/ja/docs/manage-users/organizations) を参照してください。                                                                                                                                                                                                                            |
   | **Login Experience** | 組織内のユーザー向けログイン エクスペリエンスを設定できます。詳細については、[Understand how Auth0 Organizations Work](/ja/docs/manage-users/organizations/organizations-overview#login-experience) を参照してください。                                                                                                                                                      |

   詳細については、[アプリケーション設定](/ja/docs/get-started/applications/application-settings) を参照してください。
3. 下にスクロールして **Trust Token Endpoint IP Header** 設定を見つけます。アプリケーション アーキテクチャで対応している場合は、この設定を有効にして **Save Changes** をクリックします。この設定により、Auth0 がプロキシからの IP ヘッダーを信頼できるようになり、ブルートフォース攻撃からの保護に役立ちます。

<div id="next-steps">
  ## 次のステップ
</div>

アプリケーションの登録と設定が完了したら、次の手順に進んでください。

1. 接続を設定し、アプリケーションで有効化します。
2. Auth0 に登録したアプリケーションを使用して、アプリのコードに認証を実装します。[Auth0 クイックスタート](/ja/docs/quickstarts)には、さまざまな技術向けの詳細な手順とサンプルが用意されています。また、ログインとログアウトの実装、ユーザーセッションの処理、ユーザープロフィール情報の取得と表示などについても学べます。

* [Auth0 APIs](/ja/docs/api)を使用します。

  * [Authentication API](https://auth0.com/docs/api/authentication) は、主要な ID 関連機能 (たとえば、ログイン、ログアウト、ユーザープロフィールの取得) を処理します。ほとんどのユーザーは、この API をクイックスタート、[Auth0.js library](/ja/docs/libraries/auth0js)、または [Lock widget](/ja/docs/libraries/lock) を通じて利用します。ただし、認証 UI をすべて手動で構築する場合は、この API を直接利用することもできます。
  * [Management API](https://auth0.com/docs/api/management/v2) を使用すると、Auth0 Dashboard からも実行できるさまざまなタスク (たとえば、ユーザーの作成やアプリケーションの grant type の設定) を自動化できます。
