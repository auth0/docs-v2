Auth0 をマシン間 (M2M) アプリケーションと統合するには、まず <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要プロダクトです。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> を使用してアプリを Auth0 に登録する必要があります。これらのアプリには、コマンドラインツール、デーモン、IoT デバイス、バックエンドで実行されるサービスなどの非対話型アプリが含まれます。

M2M アプリは API とその権限、つまり事前定義された値から選択する[スコープ](/ja/docs/get-started/apis/scopes/api-scopes)に関連付けられます。この登録を始める前に、関連する API を Auth0 に[登録](/ja/docs/get-started/auth0-overview/set-up-apis)し、その API のスコープを定義しておいてください。

アプリケーションに Auth0 の <Tooltip tip="Management API: 顧客が管理タスクを実行できるようにするプロダクトです。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> のみにアクセスする認可を付与する場合は、別の API を登録する必要はありません。Management API はあらかじめ用意されています。

Auth0 をマシン間 (M2M) アプリケーションと統合するには、まず Auth0 Dashboard を使用してアプリを Auth0 に登録する必要があります。

1. [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) に移動し、**Create Application** をクリックします。
2. アプリケーションのわかりやすい名前を入力し、**Machine to Machine Applications** を選択して、**Create** をクリックします。
3. アプリケーションから呼び出せるようにする API を選択します。
4. API にアクセスする各 M2M アプリには、一連の権限 (スコープ) を付与する必要があります。権限を追加する方法については、[Add API Permissions](/ja/docs/get-started/apis/add-api-permissions) を参照してください。アプリケーションのアクセストークンの一部として発行する **Permissions** を選択し、**Authorize** をクリックします。

   <Warning>
     `update:client_grants` と `create:client_grants` のスコープは、高い権限を持つアプリケーションでのみ使用してください。これらを使用すると、クライアントが自分自身にさらに権限を付与できるためです。
   </Warning>

   新しいアプリケーションを作成したら、次のタブを含む **Application Settings** を設定できます。

   | 設定タブ            | 説明                                                                                                                                                                                                |
   | --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start** | ${application_type} アプリケーションで利用可能なすべてのサンプルを表示します。また、さまざまな技術を使用して API を呼び出す方法も確認できます。API でアクセストークンを受け入れて検証する方法については、[Backend/API クイックスタート](/ja/docs/quickstart/backend/nodejs/interactive) を参照してください。 |
   | **Settings**    | アプリケーションで利用可能なすべての設定を表示します。これらの設定の大半は、デフォルトで Auth0 によって作成されます。                                                                                                                                    |
   | **Credentials** | アプリケーションの認証方法と設定済みの認証情報を表示します。詳細については、[Application Credentials](/ja/docs/secure/application-credentials) を参照してください                                                                                   |
   | **APIs**        | アプリケーションで使用する追加の API を認可できます。                                                                                                                                                                     |

   詳細については、[Application Settings](/ja/docs/get-started/applications/application-settings) を参照してください。

<div id="next-steps">
  ## 次のステップ
</div>

アプリケーションの登録と設定が完了したら、一般的な次のステップは次のとおりです。

* IDプロバイダーの接続を設定し、アプリケーションで有効にします (ユースケースに応じて必要な場合) 。
* Auth0 に登録したアプリケーションを使用するように、アプリのコードを変更します。[Auth0 クイックスタート](/ja/docs/quickstarts) では、さまざまな技術向けの詳細な手順とサンプルを確認できます。また、ログインとログアウトの実装、ユーザーセッションの処理、ユーザープロフィール情報の取得と表示などについても学べます。
* [Auth0 APIs](/ja/docs/api) を使用します。

  * [Authentication API](https://auth0.com/docs/api/authentication) は、主要なID関連機能 (たとえば、ログイン、ログアウト、ユーザープロフィールの取得) を担います。ほとんどのユーザーは、この API をクイックスタート、[Auth0.js ライブラリ](/ja/docs/libraries/auth0js)、または [Lock ウィジェット](/ja/docs/libraries/lock) を通じて利用します。ただし、認証 UI を完全に手動で構築する場合は、この API を直接利用する必要があります。
  * [Management API](https://auth0.com/docs/api/management/v2) を使用すると、Auth0 の Dashboard からも実行できるさまざまなタスク (たとえば、ユーザーの作成やアプリケーションのグラントタイプの設定) を自動化できます。
