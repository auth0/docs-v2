title: シングルページ Web アプリケーションを登録する
validatedOn: 2026-02-25
---

Auth0 をシングルページ Web アプリに統合するには、まず Auth0 Dashboard を使用してアプリを Auth0 に登録する必要があります。

1. [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) に移動し、**Create Application** をクリックします。
2. アプリケーションのわかりやすい名前を入力し、**Single-Page Web Applications** を選択して、**Create** をクリックします。

   新しいアプリケーションが作成されると、**アプリケーション設定** ページを表示できます。このページには次のタブが含まれます。

   | Settings Tab         | Description                                                                                                                                                                                                                                                                                                              |
   | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Quick Start**      | アプリケーションの種類に応じて利用できるすべてのドキュメントを表示します。                                                                                                                                                                                                                                                                                    |
   | **Settings**         | アプリケーションで利用可能なすべての設定を表示します。デフォルトでは、ほとんどの設定は自動的に作成されます。詳しくは、[アプリケーション設定](/ja/docs/get-started/applications/application-settings) を参照してください。                                                                                                                                                                        |
   | **Credentials**      | アプリケーションの認証方法と設定済みの認証情報を表示します。詳しくは、[Application Credentials](/ja/docs/secure/application-credentials) を参照してください。                                                                                                                                                                                                            |
   | **Add-ons**          | アプリケーションに関連付けられたプラグインを有効にできます。これらは通常、SAML または WS-Fed の統合で使用されますが、SPA で必要になることはあまりありません。詳しくは、[Enable SAML2 Web App Addon](/ja/docs/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) と [Configure WS-Fed Applications](/ja/docs/get-started/applications/update-application-connections) を参照してください。 |
   | **Connections**      | アプリケーションの接続を有効にできます。接続はユーザーの認証元であり、複数のアプリケーションで有効化して共有できます。詳しくは、[Connections](/ja/docs/authenticate/identity-providers) を参照してください。                                                                                                                                                                                          |
   | **Organizations**    | ユーザーが所属する組織にログインできるようにします。詳しくは、[Organizations](/ja/docs/manage-users/organizations) を参照してください。                                                                                                                                                                                                                              |
   | **Login Experience** | 組織のユーザー向けのログイン体験を設定できます。詳しくは、[Understand how Auth0 Organizations Work](/ja/docs/manage-users/organizations/organizations-overview#login-experience) を参照してください。                                                                                                                                                              |

   詳しくは、[アプリケーション設定](/ja/docs/get-started/applications/application-settings) を参照してください。

<div id="next-steps">
  ## 次のステップ
</div>

アプリケーションを登録したら、次の設定手順を確認して完了してください。

* 少なくとも 1 つの接続 (データベース接続やソーシャルログインなど) を設定し、アプリケーションで有効化します。有効な接続がないと、ユーザーは認証できません。
* アプリのコードを変更して、Auth0 に登録したアプリケーションを使用するようにします。[Auth0 クイックスタート](/ja/docs/quickstarts)では、ログインとログアウトの実装方法、ユーザーのセッション管理、ユーザープロフィール情報の取得と表示などについて、詳しい手順を確認できます。
* [Auth0 API](/ja/docs/api) を使用します。

  * [Authentication API](https://auth0.com/docs/api/authentication) は、主要な ID 関連機能 (たとえば、ログイン、ログアウト、ユーザープロフィールの取得) を担います。ほとんどのユーザーは、この API をクイックスタート、[Auth0 SPA SDK](/ja/docs/libraries/auth0-spa-js)、または [Lock ウィジェット](/ja/docs/libraries/lock) を通じて利用します。レガシーアプリケーションについては、[Auth0.js ライブラリ](/ja/docs/libraries/auth0js) を参照してください。ただし、認証 UI をすべて手動で構築する場合は、この API を直接利用する必要があります。
  * [Management API](https://auth0.com/docs/api/management/v2) を使用すると、Auth0 の Dashboard からも実行できるさまざまなタスク (たとえば、ユーザーの作成やアプリケーションのグラントタイプの設定) を自動化できます。
