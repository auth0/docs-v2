Auth0 では、テナントユーザーが Auth0 Dashboard で検索可能なフィールドに対してフィルター式を作成でき、登録済みのアプリケーションや API をより直感的に検索できます。何百ものリソースが並ぶページ分割された一覧をスクロールする代わりに、テナントユーザーは次のようなフィールドで検索できます。

* アプリケーション名 (Travel0-native) 
* アプリケーションまたはユーザーメタデータ (permissions) 
* アプリケーションの種類 (SPA) 
* 名前による API
* ID による API

Auth0 Dashboard は、登録済みのすべてのリソースから一致する結果をリアルタイムで返します。

<div id="how-it-works">
  ## 仕組み
</div>

Auth0 Dashboard の [アプリケーション](https://manage.auth0.com/#/applications) と [APIs](https://manage.auth0.com/#/apis) のセクションにある検索バーは、ガイド付きフィルターインターフェイスとブール検索に対応しています。検索バーは、フィルターできるフィールドを一覧表示するドロップダウンメニューとして機能します。フィールドを選択して値を入力すると、その値が結果の上部に削除可能なタグとして適用されます。一度に組み合わせられるフィルターは最大 5 つです。

フィルターを追加するたびに、結果はさらに絞り込まれます。条件を細かく指定するには、異なるフィールド同士を `AND/OR` で組み合わせます。フィルターは URL にクエリーパラメーターとして反映されます。URL はいつでもコピーしてチームメイトと共有したり、ブックマークして後で同じ絞り込み表示に戻ったりできます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **ブール検索の例**

  **アプリケーションタイプ** を `Native` または `SPA`、**ファーストパーティクライアント** を `true` でフィルターすると、結果には Native または SPA に該当するファーストパーティのアプリケーションだけが表示されます。Regular Web Apps やサードパーティ アプリは除外されます。
</Callout>

<div id="search-behavior-by-field-type">
  ### フィールド型ごとの検索動作
</div>

Auth0 Dashboard では、フィールド型によって検索結果の一致方法が異なります。

* **String フィールド** (name, identifier): 部分一致がサポート対象です。`payments` と入力すると、「Payments-application」、「Corp Payments Service」などが見つかります。語順は問いません。特定の語句に一致する結果を除外するには、先頭に `NOT`、`-`、または `No` を付けます。たとえば、name フィールドに `NOT test` と入力すると、名前に「test」を含まないすべてのリソースが返されます。ワイルドカード検索には 3 文字以上が必要です。3 文字未満を入力すると、「Minimum 3 characters」というメッセージが表示され、クエリは送信されません。

* **完全一致フィールド**

完全な ID 値を入力する必要があります。ID の一部だけを入力しても結果は返されません。ログやサポートチケットに記載された特定のクライアントIDがある場合は、これらのフィールドを使用します。

* **メタデータ フィールド**

メタデータは、キーと値の組み合わせ、またはキーのみで検索できます。

* **キーと値**: キーと値を入力すると、そのキーがその値に完全一致するリソースを検索できます。たとえば、キー `environment` と値 `production` を指定すると、`app_metadata.environment` が `production` に等しいリソースだけが返されます。値では前方一致がサポート対象です。`prod` と入力すると、`production`、`prod-eu`、`prod-us` に一致します。
  * **キーのみ**: 値を入力せずにキーを入力し、**Enter** キーを押します。すると、値に関係なく、そのメタデータ キーが設定されているすべてのリソースが返されます。特定のキーが使われているリソースを探す場合に便利です。

ドット記法を使用すると、最大 2 階層までネストされたメタデータ キーも検索できます。たとえば、キー `tier.bronze` と値 `true` を指定すると、`app_metadata.tier.bronze` が `true` に等しいリソースに一致します。

特定のキーと値の組み合わせに一致する結果を除外するには、値の先頭に `NOT` を付けます。たとえば、キー `color` と値 `NOT orange` を指定すると、`color` が `orange` 以外に設定されているすべてのリソースが返されます。キーは完全一致である必要があります。メタデータでは、部分一致や後方一致はサポートされていません。値は 64 文字までインデックス化されるため、それを超える値は検索できません。

* **Enum フィールド**

事前定義された一覧から 1 つ以上のアプリケーションタイプを選択します。同じフィルター内で複数選択した場合は OR 条件として扱われます。たとえば、`Native` と `SPA` を選択すると、すべての Native アプリケーションと SPA アプリケーションが返されます。

* **Boolean フィールド**

`true` または `false` を選択して、ファーストパーティクライアントを検索します。これにより、ファーストパーティのアプリケーション (Consent 画面をスキップすることが許可されているもの) とサードパーティのアプリケーションを区別できます。

<div id="searchable-fields">
  ### 検索可能なフィールド
</div>

<div id="applications">
  #### アプリケーション
</div>

| **フィールド**                             | **説明**                                            | **一致タイプ**   |
| ------------------------------------- | ------------------------------------------------- | ----------- |
| **名前**                                | アプリケーションの名前                                       | 部分一致        |
| **クライアントID** (`client_id`)            | 一意のクライアントID                                       | 完全一致        |
| **外部クライアントID** (`external_client_id`) | アプリケーションの外部識別子                                    | 完全一致        |
| **メタデータ**                             | アプリケーションに保存されるカスタムのキーと値のメタデータ                     | 完全一致または前方一致 |
| **アプリケーションタイプ**                       | Native、SPA、Regular Web App、または Machine to Machine | 列挙値の選択      |
| **ファーストパーティクライアント**                   | アプリケーションがファーストパーティかサードパーティかどうか                    | 真偽値         |

<div id="apis">
  #### API
</div>

| **フィールド** | **説明**                                        | **一致タイプ** |
| --------- | --------------------------------------------- | --------- |
| **名前**    | API の名前                                       | 部分一致      |
| **識別子**   | 一意のオーディエンス識別子 (例: `https://api.example.com`)  | 部分一致      |
| **Id**    | システム内部の ID                                    | 完全一致      |

<div id="limitations">
  ### 制限事項
</div>

* 1 回の検索に適用できるフィルターは最大 5 つです。検索バーでは 6 つ目のフィルターを追加できません。
* インデックスは結果整合性で更新されるため、検索結果に直近数秒以内の変更が反映されない場合があります。
* 通常、結果は 2 秒以内に返されます。クエリがタイムアウトする場合は、フィルターの数を減らすか、より具体的な検索語句を使用してください。

<div id="example-queries">
  ## クエリ例
</div>

<div id="find-an-application-by-client-id">
  ### クライアントIDでアプリケーションを見つける
</div>

エラーログやサポートチケットで `client_id` を確認できており、該当するアプリケーションをすばやく特定したい場合に使用します。

1. **[Auth0 Dashboard &gt; アプリケーション &gt; アプリケーション](https://manage.auth0.com/#/applications)** に移動します。
2. 検索バーを選択し、**クライアントID** を選択します。
3. `client_id` の値全体を貼り付けます。一致するアプリケーションがすぐに表示されます。

<div id="find-applications-using-metadata">
  ### メタデータを使用してアプリケーションを検索する
</div>

アプリケーションにカスタムメタデータ (環境ラベル、プラン階層、外部 ID など) が保存されている場合は、それらの値で検索できます。

1. **[Auth0 Dashboard &gt; アプリケーション &gt; アプリケーション](https://manage.auth0.com/#/applications)** に移動します。
2. 検索バーを選択し、**メタデータ** を選択します。
3. メタデータキー (例: `environment`) を入力します。
4. 値 (例: `production`) を入力します。`metadata.environment` が `production` に一致するアプリケーションが結果に表示されます。前方一致で検索するには、`prod` のような部分的な値を入力します。

<div id="troubleshooting">
  ## トラブルシューティング
</div>

**結果が見つからない**

フィルター値のスペルが正しいことを確認してください。String フィールドの場合は、より短い語句や、より一般的な語句を試してください。メタデータ フィールドの場合は、キー名が正確であることを確認してください。キーでは大文字と小文字が区別されるため、完全に一致している必要があります。メタデータでは contains 形式の一致はサポートされていないため、代わりにプレフィックスを使用してください。Application ID などの完全一致フィールドでは、ID 値全体を貼り付けていることを確認してください。

**結果が古いように見える**

検索結果が反映されるまでに時間がかかることがあるため、ここ数秒以内に行った変更が反映されない場合があります。最近リソースを作成または更新した場合は、数秒待ってから再度検索してください。

**検索がタイムアウトした**

有効なフィルターの数を減らすか、より具体的な検索語を使用してください。問題が解決しない場合は、[Auth0 サポート](https://support.auth0.com) にお問い合わせください。
