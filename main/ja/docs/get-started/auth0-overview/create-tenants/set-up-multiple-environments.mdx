description: さまざまな環境を管理するために複数の Auth0 テナントを使用する方法を説明します。
title: 複数の環境を設定する
---

開発、ステージング、本番の各環境は、Auth0 で簡単に設定できます。各環境ごとに新しいテナントを作成するだけで、環境間の分離を確実にできます。Dashboard の左上メニューにあるテナント選択機能を使用すると、テナントを簡単に切り替えられます。また、各テナントに異なる管理者を設定することもできます。

本番用の[レート制限](/ja/docs/troubleshoot/customer-support/operational-policies/rate-limit-policy)は、`Production` としてタグ付けされたテナントにのみ適用されます。本番稼働の前に、テナントの環境タグが `Production` に設定されていることを確認し、テナント、環境、課金体系ごとのレート制限の違いに注意してください。

複数の環境には、任意の名前を付けることができます。本番環境では、[カスタムドメイン](/ja/docs/customize/custom-domains)の使用を強く推奨します。

Enterprise サブスクリプションプランをご利用の場合は、[複数のテナントを 1 つのサブスクリプションにリンクする](/ja/docs/get-started/auth0-overview/create-tenants/child-tenants)ことで子テナントを作成できます。各子テナントは、そのサブスクリプションに関連付けられた有料機能またはアップグレードされた機能を利用できます。開発、ステージング、テストの各環境をすべて同じ Enterprise サブスクリプションにリンクすることで、本番環境と同じ機能を利用できるようにできます。

<div id="tag-the-environment">
  ## 環境にタグを付ける
</div>

新しく作成する各テナントについて、その環境を指定する必要があります。テナントに Environment Tag を割り当てることで、開発、ステージング、本番の各環境を区別できます。

テナントが複数用途にまたがる場合は、より上位の環境を選択してください。たとえば、開発用と本番用の両方で使用するテナントは、**Production** に設定する必要があります。

1. テナントに Environment Tag を割り当てるには、[Dashboard &gt; Settings &gt; General](https://manage.auth0.com/#/tenant/general) に移動します。

   <Frame>![Dashboard の Tenant Settings の General Settings タブ](/docs/images/cdy7uua7fh8z/4okToiwlkNQBxwr8QGE3Rs/7e96db0efab4082ebb9f78cb43ee302c/Basic_Settings_-_EN.png)</Frame>
2. **Assign Environment Tag** で、テナントの環境として **Development**、**Staging**、または **Production** を選択します。

   <Frame>![undefined](/docs/images/cdy7uua7fh8z/27OH1jFXce97CSjk7TPeD7/deafc344f675fb11ff5d3d589ffadee9/2025-02-26_14-27-31.png)</Frame>
3. 環境を選択したら、**Save** をクリックします。

<div id="migration">
  ## 移行
</div>

[Management API v2](https://auth0.com/docs/api/management/v2) を使用すると、テナント間でアセット ([Rules](/ja/docs/customize/rules)、データベース[接続](/ja/docs/authenticate/identity-providers) など) の移行を自動化できます。

構成管理を簡単にするため、設定値は **rules** や **db connections** のスクリプトにハードコードするのではなく、[Dashboard](https://manage.auth0.com/#/rules) に保存してください。

たとえば、ログ用の URL を設定したいとします。その方法の 1 つは、Rule にハードコードすることです。

```javascript lines
function(user, context, callback){
      var log_url = 'https://someurl/log';
      ...
    }
```

ただし、このコードは移植性がありません。というのも、この URL は開発環境から本番環境へ移る際に変更される可能性が高いためです。

開発環境から本番環境へ移して使用する必要があるコードを扱うには、[Rules](https://manage.auth0.com/#/rules) セクションを使用することをお勧めします。まだ Rule を作成していない場合は、作成する必要があります。 (作成済みの場合は、手順 4 に進んでください。)

1. **Create Your First Rule** をクリックします。
2. **empty rule** テンプレートを選択します。
3. 新しい Rule の名前を入力し、**Save** をクリックします。
4. [Dashboard &gt; Rules](https://manage.auth0.com/#/rules) に移動し、ページの一番下までスクロールして設定値を指定します (キー名には `log_url`、値には `https://someurl/log` を使用します) 。その後、**Create** をクリックします。
5. これで、Rule を記述できます。作成した Rule を編集し、コード領域に次のコードを入力して、**Save** をクリックします。

   ```javascript lines
   function(user, context, callback){
         var log_url = configuration.log_url;
         ...
       }
   ```

このコードは移植可能であり、本番環境へ移行する際は、スクリプト内を検索する代わりにこの設定だけを変更すれば済みます。

<div id="adldap-connectors">
  ## AD/LDAP Connectors
</div>

AD/LDAP で複数の Auth0 テナントを使用する場合は、テナントごとに AD/LDAP 接続を作成し、AD/LDAP Connector を設定する必要があります。これは、各 AD/LDAP Connector が Auth0 テナント内の特定の接続に関連付けられているためです。

複数の AD/LDAP Connector で同じ AD または LDAP ディレクトリを参照できますが、各 AD/LDAP Connector を使用できるのは、1 つの Auth0 テナント内の 1 つの接続だけです。

ユーザーが認証に使用する AD/LDAP ディレクトリが複数ある場合 (たとえば、部門や顧客ごとにそれぞれ独自のディレクトリを使用する場合) は、各 Auth0 テナント内に複数の AD/LDAP Connector を設定できます。
