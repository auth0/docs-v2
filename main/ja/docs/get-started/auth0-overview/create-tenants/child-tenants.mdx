Auth0では、Enterpriseサブスクリプションをご利用のお客様向けに、単一のAuth0サブスクリプションに複数のテナントを紐付ける機能を提供しています (これらのリンクされたテナントは、子テナントと呼ばれることもあります) 。

この機能は、次のような場合に役立ちます。

* 開発用テナントと本番用テナントを分離しながら、本番用テナントで利用できる機能と同じ機能を開発用テナントでも利用できるようにする
* 単一のEnterpriseサブスクリプションで複数の本番用テナントを保有する

<div id="linking-tenants">
  ## テナントのリンク
</div>

Enterprise サブスクリプションでは、新しいテナントの作成時に **Create Under** ドロップダウンから **Custom Agreement** オプションを選択すると、既存の Auth0 サブスクリプション配下にテナントを自動的にリンクできます。

<Frame>![](/docs/images/cdy7uua7fh8z/7ggqdv5yw011z0TsdlaHOO/7707d826dd585e074bd0d8e25b30d9d6/publiccloudtenants.png)</Frame>

現在 Enterprise サブスクリプションに含まれていない、以前作成したテナントをリンクする必要がある場合は、担当のアカウントチームに連絡するか、[Auth0 Support](https://support.auth0.com) にサポートリクエストを送信してください。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  セルフサービスのサブスクリプション (Enterprise 以外) をご利用の場合でも、テナントをリンクできます。詳細については、[Tenant Management](/ja/docs/get-started/auth0-teams/tenant-management) を参照してください。
</Callout>

<div id="usage-consolidation">
  ## 使用状況の統合
</div>

リンクされたすべてのテナントの使用状況は、Enterprise サブスクリプションの上限に算入されます。使用状況レポートとクォータレポートでは、リンクされたすべてのテナントの集計使用状況を確認できます。
