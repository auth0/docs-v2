`POST /oauth/token`

このエンドポイントを使用して、認可時に取得したリフレッシュトークンを使い、アクセストークンを更新します。

<div id="learn-more">
  ## 詳しく見る
</div>

* [リフレッシュトークン](https://auth0.com/docs/secure/tokens/refresh-tokens)

<div id="parameters">
  ## パラメーター
</div>

<ParamField header="DPoP" type="string">
  リクエストの DPoP プルーフです。これは省略可能で、アプリケーションで Demonstrating Proof-of-Possession を使用している場合にのみ必要です。
</ParamField>

<div id="request-body">
  ## リクエストボディ
</div>

<ParamField body="grant_type" type="string" required>
  使用するフローを指定します。アクセストークンを更新する場合は、`refresh_token` に設定する必要があります。

  指定可能な値: `refresh_token`
</ParamField>

<ParamField body="client_id" type="string" required>
  リクエストを行うアプリケーションを識別する、アプリケーションのクライアントIDです。
</ParamField>

<ParamField body="client_secret" type="string">
  アプリケーションのクライアントシークレットです。Token Endpoint Authentication Method が `Post` または `Basic` に設定されている場合に必要です。
</ParamField>

<ParamField body="refresh_token" type="string" required>
  以前にクライアントに発行されたリフレッシュトークンです。このトークンは、新しいアクセストークンを取得するために使用されます。
</ParamField>

<ParamField body="scope" type="string">
  リクエストするスコープ権限をスペースで区切ったリストです。省略した場合は元のスコープが使用されます。指定した場合は、元のスコープより限定されたスコープをリクエストできます。なお、URL エンコードする必要があります。
</ParamField>

<div id="response">
  ## レスポンス
</div>

| ステータス | 説明                                            |
| ----- | --------------------------------------------- |
| 200   | アクセストークンが正常に更新されました。                          |
| 400   | 不正なリクエスト。必須パラメーターが不足しているか、リクエストの形式が正しくありません。  |
| 401   | 認証されていません。クライアントの認証情報が無効であるか、リフレッシュトークンが無効です。 |
| 403   | アクセスは禁止されています。リフレッシュトークンが無効であるか、有効期限が切れています。  |
