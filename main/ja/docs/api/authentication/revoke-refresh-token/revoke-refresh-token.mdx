---
title: "リフレッシュトークンを失効させる"
description: "リフレッシュトークンが侵害された場合は、このエンドポイントを使用して失効させることができます。"
---

`POST /oauth/revoke`

リフレッシュトークンが侵害された場合は、このエンドポイントを使用して失効させることができます。

このエンドポイントの挙動は、[Refresh Token Revocation Deletes Grant](https://auth0.com/docs/tokens/refresh-tokens/revoke-refresh-tokens#refresh-tokens-and-grants) トグルの状態によって異なります。
このトグルが有効な場合、各失効リクエストでは特定のトークンだけでなく、同じ認可グラントに基づく他のすべてのトークンも無効化されます。つまり、**同じユーザー、アプリケーション、オーディエンスに対して発行されたすべてのリフレッシュトークンが失効します**。
このトグルが無効な場合は、リフレッシュトークンのみが失効し、グラントはそのまま維持されます。

<div id="remarks">
  ## 備考
</div>

* クライアントシークレットを安全に保持できない非機密アプリケーション (たとえばネイティブアプリ) の場合、このエンドポイントではクライアントシークレットを渡さずに利用できますが、アプリケーション自体の `tokenEndpointAuthMethod` プロパティを `none` に設定しておく必要があります。これは、UI ([Dashboard &gt; Applications &gt; Application Settings](https://manage.auth0.com/dashboard)) から行うことも、[Management API](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id) を使用して行うこともできます。

<div id="parameters">
  ## パラメーター
</div>

<ParamField body="client_id" type="string" required>
  アプリケーションの `client_id`。
</ParamField>

<ParamField body="client_assertion" type="string">
  アプリケーションの認証情報で署名されたアサーションを含む JWT。アプリケーションの認証方法が Private Key JWT の場合に必要です。
</ParamField>

<ParamField body="client_assertion_type" type="string">
  値は `urn:ietf:params:oauth:client-assertion-type:jwt-bearer` です。アプリケーションの認証方法が Private Key JWT の場合に必要です。
</ParamField>

<ParamField body="client_secret" type="string">
  アプリケーションの `client_secret`。アプリケーションの認証方法が Client Secret Basic または Client Secret Post の場合に必要です。特に、Regular Web Application で**のみ**必要です。
</ParamField>

<ParamField body="token" type="string" required>
  失効するリフレッシュトークン。
</ParamField>

<div id="response">
  ## レスポンス
</div>

| ステータス   | 説明           |
| ------- | ------------ |
| 200     | (空のレスポンスボディ) |
| 400     | 不正なリクエスト     |
| 401     | 未認証          |
| default | 予期しないエラー     |
