<div id="endpoint">
  ## エンドポイント
</div>

`GET /authorize`

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  このエンドポイントは、アカウントリンクでは**非推奨**です。代わりに [POST /api/v2/users/{id}/identities](/ja/api/management/v2/users/post-identities) を使用してください。詳しくは、[Migration Notice](/ja/migrations/past-migrations#account-linking-removal) を参照してください。
</Callout>

ユーザーが 2 つ目の認証方法 (たとえば、Facebook と user/password データベース接続) をリンクしたい場合に、このエンドポイントを呼び出します。

このエンドポイントは、既存のアカウントに新しいアカウントをリンクするためのログインフローを開始します。これにより、現在のユーザーが追加しようとしている `connection` への 302 リダイレクトが返されます。ユーザーは、ログイン成功時に返されたアクセストークンによって識別されます。

<div id="remarks">
  ### 注意事項
</div>

* `redirect_uri` の値は、[アプリケーションの設定](https://manage.auth0.com/dashboard) で有効なコールバック URL として設定する必要があります。

<div id="query-parameters">
  ## クエリーパラメーター
</div>

<ParamField query="response_type" type="string" required>
  サーバーサイドフローでは `code`、クライアントサイドフローでは `token` を使用します。
</ParamField>

<ParamField query="client_id" type="string" required>
  アプリケーションの `client_id` です。
</ParamField>

<ParamField query="connection" type="string">
  アプリケーションに設定されている接続の名前です。null の場合は、[Auth0 Login Page](https://auth0.com/#/login_page) にリダイレクトされ、最初のデータベース接続を使用する Login Widget が表示されます。
</ParamField>

<ParamField query="redirect_uri" type="string" required>
  ユーザーが認可した後、Auth0 がブラウザーをリダイレクトする先の URL です。
</ParamField>

<ParamField query="access_token" type="string" required>
  ログイン中のユーザーのアクセストークンです。
</ParamField>

  ## レスポンスメッセージ
