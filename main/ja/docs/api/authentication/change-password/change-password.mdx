<div id="endpoint">
  ## エンドポイント
</div>

`POST /dbconnections/change_password`

ユーザーが指定したメールアドレスと`connection`を使用して、パスワード変更メールを送信します。

必要に応じて、[カスタマイズしたメールテンプレート](https://auth0.com/docs/customize/email/email-templates)で組織固有の変数をサポートし、**Redirect To** URL に `organization_id` と `organization_name` パラメーターを含めるために、組織 ID を指定できます。

**注:** このエンドポイントはデータベース接続でのみ機能します。

<div id="remarks">
  ### 注記
</div>

* ユーザーがパスワード変更リンクをクリックすると、新しいパスワードの入力を求めるページにリダイレクトされます。
* このエンドポイントは、レート制限に関する情報を提供する 3 つの HTTP レスポンスヘッダーを返します。
  * `X-RateLimit-Limit`: 1 分あたりに許可されるリクエスト数。
  * `X-RateLimit-Remaining`: 利用可能なリクエスト数。新しいリクエストがあるたびに、この数は 1 減少します。1 分経過するごとにリクエスト数が 1 ずつ戻るため、そのたびにこの数は 1 増加します。
  * `X-RateLimit-Reset`: レート制限 (`X-RateLimit-Limit`) がリセットされるまでの残り時間。値は [UTC エポック秒](https://en.wikipedia.org/wiki/Unix_time) です。

<div id="body-parameters">
  ## Body パラメーター
</div>

<ParamField body="client_id" type="string" required>
  クライアントの `client_id`。
</ParamField>

<ParamField body="email" type="string" required>
  ユーザーのメールアドレス。
</ParamField>

<ParamField body="connection" type="string" required>
  クライアントに設定されているデータベース接続の名前。
</ParamField>

<ParamField body="organization" type="string">
  ユーザーに関連付けられた組織の `organization_id`。
</ParamField>

<div id="response-schema">
  ## レスポンススキーマ
</div>

<ResponseSchema type="string">
  200 のレスポンス本文は、平文の文字列です。

  <br />

  例: `"パスワードをリセットするためのメールを送信しました。"`
</ResponseSchema>

<div id="response-messages">
  ## レスポンスメッセージ
</div>

| ステータス | 説明                             |
| ----- | ------------------------------ |
| 200   | パスワード変更メールが送信されました。            |
| 400   | 不正なリクエストです。パラメーターが不足しているか無効です。 |
| 404   | ユーザーが見つかりません。                  |
