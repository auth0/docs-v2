Actions リアルタイムログはサーバー送信イベント (SSE) を使用しているため、次のような要因に関連する接続の問題が発生する可能性があります。

* エンドポイント保護との競合
* ファイアウォール
* VPN ソフトウェア

<div id="diagnosis">
  ## 診断
</div>

これらの競合は、いくつかの形で現れることがあります。

* **接続のブロック:** 一部のソフトウェアでは、SSE の動作に必要な長時間維持される HTTP 接続が妨げられる場合があります。
* **データのバッファリング:** 一部のソフトウェアでは、受信データがバッファリングされ、HTTP リクエストが完全に完了した後でないとクライアントに渡されない場合があります。SSE のストリームは継続的であるため、この場合、ユーザーにはデータが表示されません。
* **Content-Type のブロック:** 一部のソフトウェアでは、SSE に不可欠な `content-type: text/event-stream` ヘッダーを明示的にブロックするよう設定されている場合があります。

<div id="potential-solutions">
  ## 考えられる解決策
</div>

* 別のデバイスまたは別のブラウザーで、テナントの Actions ログにアクセスします。
* 問題が解消するかどうかを確認するため、VPN またはエンドポイント保護ソフトウェアを一時的に無効にします。

<div id="contact-support">
  ## サポートへの問い合わせ
</div>

問題が解決しない場合は、[Auth0 Support](https://support.auth0.com) にお問い合わせください。機密情報を含まない `HAR` ファイルを含め、可能な限り多くの情報を提供してください。エラーやログがあると、Auth0 Support による問題のトラブルシューティングがより迅速になります。
