---

Auth0 のサブスクリプションプランによって、利用可能なユーザー数の上限と機能が決まります。

<div id="quota-utilization">
  ## クォータの使用状況
</div>

Auth0 Support Center の [Reports &gt; Quota Utilization](https://support.auth0.com/reports/quota) にアクセスすると、サブスクリプションの利用状況を確認できます。利用状況には、次の詳細が含まれます。

* ユーザー消費量

  * アクティブユーザー
* 機能消費量

  * エンタープライズ接続
  * マシン間認証
  * 多要素認証

<Frame>![サブスクリプション機能の使用状況を示すレポート](/docs/images/cdy7uua7fh8z/7g0PDAKl5D8YEf4vC5FjSI/188182aabbeb7e0af77b1bc9d292cfad/Screenshot_2023-02-07_at_5.35.40_PM.jpeg)</Frame>

各概要カードには、当月の利用状況が利用可能なサブスクリプション上限に対して表示されます。また、カードをクリックしてドリルダウンすると、Auth0 Private Instance (該当する場合) やテナントごとの使用量の内訳など、過去 12 か月分までの詳細を確認できます。

必要に応じて、このデータをエクスポートするための CSV をダウンロードすることもできます。

<Frame>![](/docs/images/cdy7uua7fh8z/7bBqDuycQWsnUdd9hlXN10/040a04d458bff06e8392c1432e3308ce/Screenshot_2023-02-07_at_6.07.01_PM.jpeg)</Frame>

<div id="active-users">
  ### アクティブユーザー
</div>

これを使用すると、特定の暦月にアクティブなイベント (ログインまたはサインアップ) があったユニークユーザー数を取得できます。さらに、これらのアクティブユーザーの内訳をテナントおよび Private Auth0 Instance (該当する場合) 別に、過去 12 か月分までドリルダウンして確認できます。

<div id="enterprise-connections">
  ### エンタープライズ接続
</div>

[エンタープライズ接続](/ja/docs/authenticate/enterprise-connections) は、 (当月中に) 次の両方を満たしている場合にアクティブと見なされます。

* アプリケーションに対して有効になっていること。
* ユーザーアクティビティ (たとえば、ログイン、サインアップ、トークン交換) があること。

エンタープライズ接続がどのアプリケーションでも一度も有効になっていない場合、または有効になっていても当月中にユーザーアクティビティがなかった場合は、アクティブとは見なされません。

このレポートを使用すると、特定の暦月におけるアクティブなエンタープライズ接続数を確認できます。また、Private Auth0 Instance ごとのアクティブな接続の内訳 (該当する場合) に加え、アクティブなエンタープライズ接続およびテナントごとのアクティブユーザー数も確認できます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Okta Workforce Enterprise connection は、アクティブなエンタープライズ接続数の上限にはカウントされません。
</Callout>

<div id="machine-to-machine-auth">
  ### マシン間認証
</div>

このレポートでは、[クライアントクレデンシャルフロー](/ja/docs/get-started/authentication-and-authorization-flow/client-credentials-flow) (API Authentication Calls とも呼ばれます) で発行された<Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可クレデンシャル。" cta="用語集を表示" href="/ja/docs/glossary?term=access+tokens">アクセストークン</Tooltip>の件数を、Auth0 <Tooltip tip="Management API: お客様が管理タスクを実行できるようにするプロダクト。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> 向けに発行されたトークンを除き、暦月ごとに確認できます。

Authentication Calls の件数は、Private Auth0 Instance (該当する場合) およびテナントごとに表示できます。

Auth0 Public Cloud デプロイメントオプションのテナントについては、このレポートでは過去 7 日間の Authentication Calls の内訳をテナントアプリケーションごとにも確認できます。

<div id="multi-factor-auth">
  ### 多要素認証
</div>

多要素認証 (<Tooltip tip="多要素認証（MFA）: SMS で送信される認証コードなど、ユーザー名とパスワードに加えて認証要素を使用するユーザー認証プロセス。" cta="用語集を見る" href="/ja/docs/glossary?term=MFA">MFA</Tooltip>) の使用状況に関するレポートは 2 種類あり、暦月ごとの MFA イベント (登録またはチャレンジ) 数が表示されます。

Enterprise MFA の件数には、プッシュ通知、WebAuthn、SMS、メールアドレス、または電話番号で送信されるワンタイムコードに関連する MFA イベントが含まれます。Pro MFA の件数には、Google Authentication や Cisco DUO などのワンタイムコードアプリに関連する MFA イベントが含まれます。

これら 2 種類のレポートでは、MFA 件数の内訳を Auth0 Private Instance* (該当する場合) およびテナント別に表示します。

*Auth0 Private Instance の MFA 使用状況は、2023 年 1 月 1 日以降のみ利用できます。

***

<div id="tenant-usage">
  ## テナントの使用状況
</div>

[Reports &gt; Usage](https://support.auth0.com/reports/usage) にアクセスし、Subscription (セルフサービス サブスクリプションの場合は テナント も) を選択すると、指定したテナントの合計ユーザー数とアクティブユーザー数を、Auth0 Support Center で暦月ごとに確認できます。表示される内訳は次のとおりです。

* アクティブユーザー: 月別およびユーザータイプ別 (Social、Password、<Tooltip tip="パスワードレス: 第1認証要素としてパスワードに依存しない認証方式。" cta="用語集を表示" href="/ja/docs/glossary?term=Passwordless">パスワードレス</Tooltip>、Enterprise) のアクティブユーザー数と Total Active の件数
* テナント別アクティブユーザー: 月別、テナント別、およびユーザータイプ別 (Social、Password、Passwordless、Enterprise) のアクティブユーザー数と Total Active の件数
* テナント別総ユーザー数

<Frame>![Support Center -> Reports -> Usage](/docs/images/cdy7uua7fh8z/2RmWp1ZIWwV2JCPJcftll5/acd9c3c71b8dbd9cca12a44588e015af/Screen_Shot_2022-10-27_at_1.21.35_PM.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  ユーザータイプを判別できないケースでは、Total Active または Total Users の数が、各ユーザータイプの合計を上回ることがあります。これは、アカウントリンクが有効になっている場合などに発生する可能性がありますが、これに限りません。
</Callout>
