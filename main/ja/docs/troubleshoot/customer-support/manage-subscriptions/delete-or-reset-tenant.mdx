<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> で、Auth0 テナントを削除したり、デフォルト設定にリセットしたりできます。Auth0 アカウントのデータは、Auth0 Support に依頼して削除できます。

<div id="delete-tenant">
  ## テナントを削除する
</div>

<Warning>
  テナントの削除は元に戻せない破壊的な操作であり、一度削除すると取り消せません。削除したテナントは復元できず、新しいテナントを作成する際にそのテナント名を再利用することもできません。また、そのテナントに関連するメトリクスも失われます。
</Warning>

テナントを削除する前に、次の選択肢を検討してください。

* [テナントの所有者を変更する](/ja/docs/get-started/manage-dashboard-access)。
* [テナントの設定をリセットする](#reset-tenant-configuration)。

<div id="remove-custom-domain">
  ### カスタムドメインを削除する
</div>

削除するテナントに <Tooltip tip="カスタムドメイン: 固有名、またはバニティ名を持つサードパーティドメイン。" cta="用語集を見る" href="/ja/docs/glossary?term=custom+domain">カスタムドメイン</Tooltip> を設定している場合は、先にカスタムドメインを削除する必要があります。そうしないと、そのカスタムドメインを別のテナントに設定できません。

<div id="determine-deletion-process">
  ### 削除手順を確認する
</div>

テナントの削除手順は、関連付けられている[サブスクリプションプラン](https://www.auth0.com/pricing)と、[テナントの種類](/ja/docs/get-started/auth0-overview/create-tenants/child-tenants) (メインまたは子) によって異なります。[Auth0 Dashboard](#delete-tenant-in-auth0-dashboard)でテナントを削除できる場合もあれば、[Auth0 Support または Community に問い合わせる](#contact-auth0-support-or-community)必要がある場合もあります。

|                     | Free plan               | セルフサービスプラン            | エンタープライズプラン           |
| ------------------- | ----------------------- | --------------------- | --------------------- |
| **メインテナント**         | Auth0 Dashboard         | Auth0 Dashboard       | Auth0 Support に問い合わせる |
| **子テナントを持つメインテナント** | Auth0 Community に問い合わせる | Auth0 Support に問い合わせる | Auth0 Support に問い合わせる |
| **子テナント**           | Auth0 Dashboard         | Auth0 Dashboard       | Auth0 Dashboard       |

<div id="downgrade-tenant">
  ### テナントのダウングレード
</div>

Dashboard でテナントを削除するには、事前に Free plan にダウングレードしておく必要があります。詳細については、[Auth0 サブスクリプションのダウングレードまたはキャンセル](/ja/docs/troubleshoot/customer-support/manage-subscriptions/downgrade-or-cancel-subscriptions)を参照してください。

<div id="delete-tenant-in-dashboard">
  ### Dashboard でテナントを削除する
</div>

[適切なアクセス権限](/ja/docs/get-started/manage-dashboard-access/feature-access-by-role)があれば、Dashboard でテナントを削除できます。

1. [Dashboard &gt; Settings](https://manage.auth0.com/#/tenant/advanced) に移動し、**Advanced** ビューに切り替えます。
2. ページ下部の **Danger Zone** を見つけ、**Delete** を選択します。
3. 確認プロンプトの内容を確認し、テナントを完全に削除することを確定します。
4. **Yes, Delete Tenant** を選択します。

<div id="contact-auth0-support-or-community">
  ### Auth0 Support または Auth0 Community に問い合わせる
</div>

テナントが Trial、Essential、Professional、または Enterprise プランで、Auth0 Dashboard からテナントを削除できない場合は、サポートを受けるために [Auth0 Support にサポートチケットを作成](/ja/docs/troubleshoot/customer-support/open-and-manage-support-tickets)してください。

テナントが Free plan で、Auth0 Dashboard から削除できない場合は、[Auth0 Community に投稿](https://community.auth0.com/)してサポートを依頼してください。

<div id="reset-tenant-configuration">
  ## テナント設定をリセットする
</div>

[Auth0 Deploy CLI](https://github.com/auth0/auth0-deploy-cli) を使用すると、テナントをデフォルトの設定にリセットできます。

Deploy CLI は、Auth0 テナント設定の管理に役立つツールです。スタンドアロンの CLI として、また Node.js モジュールとして開発ワークフローに組み込めます。Deploy CLI ツールと使い始める方法の詳細については、[Deploy CLI ツール](/ja/docs/deploy-monitor/deploy-cli-tool) を参照してください。

<div id="delete-auth0-account-data">
  ## Auth0 アカウントのデータを削除する
</div>

Auth0 アカウントのデータ (メールアドレス、請求情報など) を削除するには、テナントの削除手順に従った後、[Auth0 Community に投稿](https://community.auth0.com/)してサポートを依頼してください。

データ管理者としての Auth0 の責任について詳しくは、[Okta Privacy Policy](https://www.okta.com/privacy-policy/) を参照してください。
