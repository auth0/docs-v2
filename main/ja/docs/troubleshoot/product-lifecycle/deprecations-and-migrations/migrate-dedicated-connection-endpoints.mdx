title: 有効なクライアント管理を専用の接続エンドポイントに移行する
validatedOn: 2026-04-20
---

Auth0 は、Management API の [Get All Connections](/ja/docs/api/management/v2/connections/get-connections)、[Get Connections](/ja/docs/api/management/v2/connections/get-connections-by-id)、および [Update a Connection](/ja/docs/api/management/v2/connections/patch-connections-by-id) での `enabled_clients` フィールドを非推奨化します。Auth0 のお客様は、[Get enabled clients for a connection](https://auth0.com/docs/api/management/v2/connections/get-connection-clients) エンドポイントを使用して、接続で有効なクライアントを取得できます。接続にすでに設定されているクライアントを更新するには、[Update enabled clients for a connection](https://auth0.com/docs/api/management/v2/connections/patch-clients) を使用します。

この変更に備え、統合が引き続き円滑に機能するよう、以下の手順を確認して影響の有無を確認し、新しいエンドポイントに移行してください。

移行スケジュールの詳細については、[Deprecations and Migrations](/ja/docs/troubleshoot/product-lifecycle/deprecations-and-migrations) の項目を確認してください。

<div id="verify-affected-tenants">
  ## 影響を受けるテナントを確認する
</div>

<Warning>
  Private Cloud テナント管理者は、自身の [Private Cloud deployment](/ja/docs/deploy-monitor/deploy-private-cloud) がバージョン `v202608` 以降であれば、以下の手順に従うことができます。Private Cloud デプロイメントのバージョンの詳細については、[How to Check the Auth0 Private Cloud&#39;s Release Number and Deployment Date](https://support.auth0.com/center/s/article/How-to-check-the-Auth0-private-cloud-s-release-number-and-deployment-date) を参照してください。
</Warning>

Auth0 Dashboard を使用すると、テナントで非推奨の機能へのアクセスが許可されているかどうか、および移行が必要かどうかを確認できます。

1. [Auth0 Dashboard &gt; Tenant Settings &gt; Advanced](https://manage.auth0.com/dashboard/#/tenant/advanced) に移動します。
2. **Migrations** で、**Legacy Management of Connection&#39;s Enabled Clients toggle** を確認します。
   * トグルが **on**: 接続の取得および更新時に、テナントでは非推奨のフィールドへのアクセスが許可されています。期限までに移行する必要があります。
   * トグルが **off**: 追加の対応は必要ありません。

<div id="identify-affected-integrations">
  ## 影響を受ける統合を特定する
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Public Cloud テナントでは、2026 年 2 月前半からこれらの非推奨通知が送信されるようになりました。
  * Private Cloud テナントでは、Private Cloud のデプロイメントが `v202608` バージョンに達した時点から、これらの非推奨通知が送信されるようになりました。
</Callout>

どのアプリケーションと統合で非推奨のエンドポイントが使用されているかを確認するには、テナントの[ログ](/ja/docs/deploy-monitor/logs#logs)で非推奨通知を確認してください。

この非推奨に関する通知を検索するには、次のクエリを使用します。

```bash lines
type:depnote AND description:Legacy\ Management\ of\ Connection's\ Enabled\ Clients*
```

クエリ結果の `client_id` フィールドを確認し、影響を受ける可能性があるアプリケーションを特定します。

一部の Auth0 プラットフォームサービスおよび拡張機能により、テナントログに非推奨通知が記録されることがあります。

* `client_id` フィールドが空の場合、その通知は拡張機能または Auth0 Dashboard に起因します。以下の拡張機能の使用状況を確認したうえで、クライアントID がない通知は無視できます。
* 値がテナント内のどのクライアントアプリケーションとも一致しない場合、その通知はプラットフォームサービスに起因するため、無視できます。

さらに、特定のクライアントアプリケーションに対する非推奨通知が存在していても、そのアプリケーションが実際に影響を受けるとは限りません。たとえば、いずれかのアプリケーションが、非推奨の `enabled_clients` フィールドを含む `GET` 接続情報を取得していても、レスポンスで返されたそのフィールドを処理していない場合は、そのフィールドの廃止予定による影響はありません。

<div id="review-extensions-usage">
  ## 拡張機能の使用状況を確認する
</div>

次の拡張機能では、テナントログに非推奨に関する通知が表示されることがあります。

* [User Import/Export](/ja/docs/customize/extensions/user-import-export-extension#user-import-export-extension): この拡張機能は、非推奨の機能に依存していません。これらの通知は、拡張機能が接続情報を取得した際に発生する誤検知です。
* [Custom Social Connections](https://github.com/auth0/custom-social-connections): この拡張機能は、非推奨の機能に依存しているため、問題が発生します。
* [Delegated Administration Dashboard](/ja/docs/customize/extensions/delegated-administration-extension/install-delegated-admin-extension): この拡張機能の古いバージョンは、非推奨の機能に依存している可能性があり、問題が発生することがあります。Auth0 Dashboard からバージョン 4.8 以降に更新してください。

最初の 2 つの拡張機能は、それぞれのユースケースに対応する Auth0 Dashboard の機能に置き換えられており、現在は更新されていません。

* [一括ユーザーインポート / エクスポート](/ja/docs/manage-users/user-migration/bulk-user-import-export) の Dashboard 機能を使用して、User Import/Export 拡張機能を置き換えてください。
* [ソーシャル ID プロバイダー](/ja/docs/authenticate/identity-providers/social-identity-providers#auth0-dashboard) の Dashboard 機能を使用して、Custom Social Connections 拡張機能を置き換えてください。

<div id="update-your-integrations">
  ## 統合を更新する
</div>

影響を受ける接続エンドポイントを使用して `enabled_clients` フィールドを取得 (`GET`) または更新 (`PATCH`) しているアプリケーションや統合は、専用のクライアント管理エンドポイントを使用するように修正する必要があります。

* **読み取り:** [Get enabled clients for a connection](https://auth0.com/docs/api/management/v2/connections/get-connection-clients).
* **書き込み:** [Update enabled clients for a connection](https://auth0.com/docs/api/management/v2/connections/patch-clients).

<div id="update-sdks-and-tools">
  ## SDK とツールを更新する
</div>

次の SDK とツールのバージョンで、新しい endpoint がサポートされます。

* [node-auth0](https://github.com/auth0/node-auth0): `4.25.0+`
* [go-auth0](https://github.com/auth0/go-auth0/): `1.21.0+`
* [auth0-java](https://github.com/auth0/auth0-java): `2.21.0+`
* [auth0-python](https://github.com/auth0/auth0-python): `5.1.0+`
* [auth0.net](https://github.com/auth0/auth0.net): `7.38.0+`
* [Auth0 Deploy CLI](https://auth0.com/docs/deploy-monitor/deploy-cli-tool): `8.27.0+`
* [Auth0 Terraform Provider](https://auth0.com/docs/deploy-monitor/auth0-terraform-provider): `1.29.0+`

Auth0 Deploy CLI / Auth0 Terraform Provider では、データモデル内での接続およびそれぞれのクライアントの表現方法を変更する必要はありません。ツールで専用の endpoint を確実に使用するには、利用可能な最新バージョンを使用する必要があります。

Ruby および PHP の Management API SDK は現在更新中です。

<div id="opt-out-finalize-migration">
  ## オプトアウト (移行の完了)
</div>

統合が非推奨の動作に依存しなくなったことを確認したら、テナントで明示的にオプトアウトするよう設定してください。

* **なぜ早めにオプトアウトするのですか？** これにより、対象のテナントが新しい動作に切り替わるタイミングを正確に制御できます。想定外の事態が発生した場合でも、トグルをすぐに元に戻して、元の動作を復元できます。
* **オプトアウトする方法:** **[Auth0 Dashboard &gt; Tenant Settings &gt; Advanced](https://manage.auth0.com/dashboard/#/tenant/advanced)** に移動します。Migrations で、**Legacy Management of Connection&#39;s Enabled Clients** オプションをオフにします。
