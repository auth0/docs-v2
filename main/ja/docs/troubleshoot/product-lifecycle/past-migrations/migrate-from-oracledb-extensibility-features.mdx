2023 年 6 月 21 日以降、Auth0 では拡張機能で Node.js から Oracle Database に接続することはサポートされなくなります。Auth0 では以前、Node.js モジュール `oracledb` を介した Oracle Database への接続をサポートしていました。

<div id="affected-workflows">
  ## 影響を受けるワークフロー
</div>

`require(oracledb)` または `require('oracledb@4.2.0')` を含むコードがないか、拡張機能を確認してください。

拡張コードを使用できる次のフローまたは機能で、[oracledb](https://www.npmjs.com/package/oracledb) への参照がないか確認してください。

* [Hooks](/ja/docs/customize/hooks)
* [Rules](/ja/docs/customize/rules)
* [カスタムデータベーススクリプト](/ja/docs/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [カスタムソーシャル接続](/ja/docs/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## 推奨される更新
</div>

この機能について、公式に予定されている代替機能はありません。Auth0 では、`oracledb` のロジックをお客様自身のインフラストラクチャ内の別サービスに切り出すことを推奨しています。インフラストラクチャ内にそのサービスを作成したら、Extensibility 関数を使用して HTTP リクエストを送信し、Oracle Database のロジックをトリガーできます。

ご自身のインフラストラクチャで `oracledb` モジュールを実装する方法について詳しくは、[OracleDB Quickstart ガイド](https://node-oracledb.readthedocs.io/en/latest/user_guide/installation.html#quickstart)を参照してください。

たとえば、`oracledb` を使用する カスタムデータベース Login スクリプトでは、`oracledb` を呼び出す関数を削除し、代わりに汎用の HTTP クライアントを使用してレスポンスを取得します。

Actions の作成方法や Actions への移行について詳しくは、[Auth0 のドキュメントを参照してください](/ja/docs/customize/actions)。
