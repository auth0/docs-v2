2023 年 6 月 21 日以降、Auth0 は .NET および C# 向けの Node.js Extensibility 機能をサポートしなくなります。

<div id="affected-workflows">
  ## 影響を受けるワークフロー
</div>

`require(edge)` を含むコードがないか、Extensibility 機能を確認してください。

Extensibility コード内で `edge.js` が参照されていないか、次のフローまたは機能を確認してください。

* [Hooks](/ja/docs/customize/hooks)
* [Rules](/ja/docs/customize/rules)
* [Custom Database Scripts](/ja/docs/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Custom Social Connections](/ja/docs/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## 推奨される更新
</div>

この機能について、公式に予定されている代替機能はありません。

移行方法の 1 つとして、`edge` のロジックを自社インフラ内の別サービスに切り出すことが考えられます。インフラ内にそのサービスを作成したら、Extensibility 関数を使用して HTTP リクエストを送信し、必要な処理をトリガーできます。

たとえば、`edge` を使用する Custom Database Login スクリプトでは、`edge` を呼び出す関数を削除し、代わりに汎用の HTTP クライアントでレスポンスを取得します。

移行方法のもう 1 つは、Extensibility 機能を .NET/C# ではなく JavaScript で書き直すことです。

Actions の作成方法や Actions への移行について詳しくは、[Auth0 のドキュメントを参照してください](/ja/docs/customize/actions)。
