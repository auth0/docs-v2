description: 非推奨エラーのログを検索する方法を学びます
title: 非推奨エラー
---

Auth0 の機能が非推奨になると、アプリケーションでその機能が使用されていることを示すエラーや通知が、テナントのログに表示されることがあります。このガイドでは、非推奨に関連するメッセージをログから検索する方法に加え、特定の項目について考えられる原因と対処方法を説明します。

非推奨機能の使用を示す警告メッセージを検索する方法は 2 つあります。Dashboard を使用する方法と、<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を使用する方法です。どちらの場合も、[ログ保持期間](/ja/docs/deploy-monitor/logs) はアカウントのサブスクリプションレベルによって決まる点に注意してください。

詳しくは、[非推奨エラーのログを検索する](/ja/docs/troubleshoot/basic-issues/search-logs-for-deprecation-errors) を参照してください。

<div id="deprecation-log-messages">
  ## 非推奨に関するログメッセージ
</div>

<div id="legacy-lock-api">
  ### レガシー Lock API
</div>

**ログエントリ**: `up-idp-initiated`

**エラーメッセージ:** &quot;レガシー Lock API: この機能は非推奨です。アプリケーションの移行方法については、ドキュメントを参照してください。&quot;

**原因**: レガシー Lock API に関するテナントのログエントリには、リファラーや使用中の SDK に関する情報が含まれる場合があります。

**解決策**: この情報を使用して、いずれかのアプリケーションで古いライブラリが使われていないか確認してください。

<div id="ssodata-endpoint">
  ### SSOdata エンドポイント
</div>

**ログエントリ**: `ssodata`

**エラーメッセージ:** &quot;SSOdata endpoint: この機能は非推奨になります。アプリケーションの移行方法については、ドキュメントを参照してください。&quot;

**原因**: `/ssodata` エンドポイントを直接呼び出しているか、埋め込み版の古い Lock または Auth0.js SDK を使用して `/ssodata` エンドポイントを呼び出す関数を実行していることが原因です。

**解決策**: Lock SDK または Auth0.js SDK を最新バージョンに更新してください。
