---
description: 問題をトラブルシューティングするために、プラットフォームを確認する方法を説明します
title: プラットフォームを確認する
---

テクノロジースタックが現在のバージョンであることを確認してください。古いバージョンのプログラミング言語やライブラリを使用している場合は、現在のバージョンへの更新を検討してください。

* 使用している言語またはライブラリのバージョンがサポート対象かどうかを、[Auth0 サポートマトリクス](/ja/docs/troubleshoot/customer-support/product-support-matrix)で確認してください。
* ある環境 (開発環境など) では問題が発生する一方で、別の環境 (本番環境など) では発生しない場合は、2 つの環境のテクノロジースタックのバージョンを比較してください。
* 最新でない場合は、サポートリクエストを作成する前に、テクノロジースタックとブラウザーを最新バージョンにアップグレードして再度テストしてください。

<div id="check-the-scope-of-the-issue">
  ## 問題の範囲を確認する
</div>

異なるブラウザー、プラットフォーム、地域、ユーザーでテストすると、問題が発生したときに原因の切り分けに役立ちます。次のテストを実行し、詳細を確認するために、各テストの後で [Auth0 Dashboard のログ](https://manage.auth0.com/#/logs) を確認してください。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  問題のトラブルシューティングに役立つ各ログイベントの詳細については、[Monitoring ページ](/ja/docs/deploy-monitor/monitor)を参照してください。
</Callout>

<div id="test-with-different-browsers">
  ### 異なるブラウザーでテストする
</div>

* 問題は、すべてのブラウザーで発生しますか？ それとも一部のブラウザーだけですか？
* 新しいシークレットモードまたはプライベートブラウジングのセッション (以前の状態なし) でテストします。
* ブラウザーを最新バージョンに更新して、再度テストします。
* ブラウザーのプラグインを無効にして再度テストし、プラグインが問題の原因になっていないか確認します。

<div id="test-with-different-users">
  ### 別のユーザーでテストする
</div>

* 問題はすべてのユーザーで発生しますか？ それとも一部のユーザーだけですか？
* 一部のユーザーだけで発生する場合、何かパターンはありますか？ たとえば、場所、接続、ロールなどです。

<div id="test-with-different-connections-if-possible">
  ### 可能であれば、異なる接続でテストする
</div>

* 問題はすべての接続で発生しますか、それとも特定の 1 つだけで発生しますか？

<div id="test-with-different-client-applications-if-you-have-more-than-one">
  ### 別のクライアントアプリケーションでもテストする (複数ある場合)
</div>

* 問題はすべてのクライアントで発生しますか。それとも 1 つだけで発生しますか？

<div id="test-across-different-environments">
  ### 異なる環境でテストする
</div>

* アプリケーションの開発環境、ステージング環境、本番環境で試してください。
* 複数の Auth0 テナントで試し、違いがあるか確認してください。

<div id="check-if-the-issue-is-consistent-or-intermittent">
  ## 問題が常に発生するのか、断続的に発生するのかを確認する
</div>

* 問題が常に発生する場合は、確実に再現できる手順を記録します。
* 問題が断続的に発生する場合は、その原因となっていそうなパターンを記録します。
* これは新しい設定で発生している問題ですか？ それとも、以前は動作していて現在は動作しなくなっていますか？

  * 動作しなくなった直前に、どのような変更が加えられましたか？
  * その変更を元に戻すと、問題は解消しますか？

<div id="issues-that-affect-only-one-or-a-few-users">
  ## 1 人または少数のユーザーのみに影響する問題
</div>

* [ユーザーのプロファイル](/ja/docs/troubleshoot/authentication-issues/check-user-profiles)、ブラウザー、またはデバイスに問題がないか確認します。
* 影響を受けるユーザーで、すべてのブラウザーで発生するか (データの問題を示す) 、それとも特定の種類のブラウザーでのみ発生するか (ブラウザー固有の問題を示す) を確認します。
* ブラウザーで JavaScript と Cookie が有効になっていることを確認します。
* Caps Lock キーがオフになっていることを確認します。
* ユーザーがモバイルデバイスを使用している場合は、認証や認可に影響する可能性のあるソフトウェアがないか確認します (必要なソフトウェアの一部が実行されていない場合など) 。
* ユーザーが、<Tooltip tip="IDプロバイダー（IdP）: デジタルアイデンティティを保存および管理するサービス。" cta="用語集を表示" href="/ja/docs/glossary?term=identity+provider">IDプロバイダー</Tooltip> の <Tooltip tip="シングルサインオン（SSO）: ユーザーが 1 つのアプリケーションにログインすると、他のアプリケーションにも自動的にログインするサービス。" cta="用語集を表示" href="/ja/docs/glossary?term=Single+Sign-on">シングルサインオン</Tooltip> (SSO) URL など、アプリケーションの主要な URL の一部にアクセスできるか確認します (ネットワーク接続の問題を示す可能性があります) 。

<div id="issues-that-occur-after-go-live-worked-before-then-stopped-working">
  ## 本番稼働後に発生する問題 (以前は動作していたが、途中から動作しなくなった場合)
</div>

* アプリケーションや、呼び出している API に最近変更がなかったか確認します。
* 最近ネットワーク構成に変更がなかったか確認します (load balancer、firewall、プロキシ設定の変更など) 。
* 最近インフラ構成に変更がなかったか確認します (例: LDAP や DB サーバーの認証情報) 。
* アプリケーション、インフラ、または技術スタックにパッチ適用や更新がなかったか確認します。
* すべてのサーバーで NTP が稼働しており、時刻同期が正確であることを確認します
* いずれかの証明書の有効期限が切れていないか確認します。
* リモートの IDプロバイダー の管理者に、何か変更がなかったか確認します。

  * リモートの IDプロバイダー へのネットワーク接続を確認します。
* <Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 のメイン製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>/migrations の通知を確認します。見落としている変更はありませんか？
* Auth0 の変更ログを確認します。問題に関連する最近の設定変更はありませんか？
* 技術スタック内のいずれかのコンポーネントが最近更新またはパッチ適用されていないか確認します。
