title: 非推奨エラーの確認
---

Auth0 の機能が非推奨になると、アプリケーションでその機能が使われていることを示すエラーや通知がテナントログに記録される場合があります。このガイドでは、非推奨に関連するメッセージをログから検索する方法に加え、特定の項目について想定される原因と対処方法を説明します。

非推奨機能の使用を示す警告メッセージを検索する方法は 2 つあります。Dashboard を使用する方法と、<Tooltip tip="Management API: 顧客が管理タスクを実行するための製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> を使用する方法です。どちらの場合も、[ログの保持期間](/ja/docs/deploy-monitor/logs/log-data-retention) はアカウントのサブスクリプションレベルによって決まる点に注意してください。

<div id="use-the-dashboard">
  ## Dashboard を使用する
</div>

アプリケーションで非推奨機能を使用している場合、Dashboard の **Logs** セクションに **Deprecation Notice** メッセージが表示されます。非推奨通知は、非推奨機能を含む認証トランザクションごとではなく、1 時間に 1 回のみ (その 1 時間内に最初にエラーが発生した時点で) 表示されます。

1. [Dashboard &gt; Monitoring &gt; Logs](https://manage.auth0.com/#/logs) に移動し、クエリボックスに `type:depnote` と入力して、非推奨関連のメッセージを検索します。該当するものがある場合は、ログ内の非推奨関連の警告メッセージ一覧が表示されます。**Description** フィールドには、使用されている特定の非推奨機能に関する情報が表示されます。
2. **Type** 列の各項目のリンクをクリックすると、非推奨機能を使用しているクライアントアプリケーションを識別するクライアントIDなどの追加情報が表示されます。
3. ログエントリをクリックし、**Context Data** タブを選択すると、その項目の詳細が表示されます。

<div id="use-the-management-api">
  ## Management API を使用する
</div>

Management API を使用してログを検索し、`type:"depnote"` を探すことで、このようなメッセージを見つけることもできます。

1. [Management API](https://auth0.com/docs/api/management/v2) に移動します。まだ行っていない場合は、[API トークンを取得](/ja/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production)します。
2. 左側の [Logs &gt; ログイベントを検索](https://auth0.com/docs/api/management/v2/#!/Logs/get_logs) に移動し、**パラメーター** までスクロールします。
3. **q** フィールドに `type:"depnote"` と入力します。
4. **TRY** ボタンをクリックします。結果の **クライアントID** フィールドに、テナント内のどのアプリケーションが非推奨の機能を使用しているかが表示されます。
