title: 接続を確認する
---

認証トランザクションは、多くの場合複数の要素で構成されています。Auth0 には、問題の原因となっている可能性がある箇所を特定しやすくするために、トランザクションの各要素を個別に試せる方法が用意されています。

ほとんどの<Tooltip tip="IDプロバイダー（IdP）: デジタルアイデンティティを保存および管理するサービス。" cta="用語集を表示" href="/ja/docs/glossary?term=identity+provider">IDプロバイダー</Tooltip>接続には、接続が正常に動作しているかを確認するための **TRY** ボタンがあります。これが失敗した場合は、アプリケーションの他の部分を切り離して接続をデバッグできます。接続が機能している場合は、次にアプリケーションのデバッグを開始できます。

1. 接続の **TRY** ボタンを使用して、接続のみをテストします。
2. **TRY** ボタンでのテストが失敗した場合、問題はアプリケーションではなく、接続の設定または接続プロバイダーにあることがわかります。
3. **TRY** ボタンでのテストのレスポンスを確認し、有用なエラーメッセージやその他の情報が含まれていないか確認します。
4. 別の方法で同じサービスにログインしてみます。
5. 同じ問題が発生する場合は、アカウント側の問題であることがわかります。
6. カスタム DB 接続で **TRY** ボタンによるテストが失敗する場合、原因はカスタム DB スクリプトの問題であることがよくあります。`console.log` ステートメントを追加し、Console Log で出力を確認すると、デバッグに役立ちます。

<div id="verify-external-service-dependencies">
  ## 外部サービスの依存関係を確認する
</div>

1. 認証でソーシャルIDプロバイダーなどの外部サービスを使用していて、突然動作しなくなった場合は、[外部サービスのステータスを確認](/ja/docs/deploy-monitor/monitor/check-external-services-status)してください。
2. **TRY** ボタンを使用しても接続が機能しない場合は、その接続を確認してください。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  問題のトラブルシューティングに役立つ各ログイベントの詳細については、[Monitoring ページ](/ja/docs/deploy-monitor/monitor)を参照してください。
</Callout>
