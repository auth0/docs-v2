title: HAR ファイルの生成と分析
---

[HAR](https://en.wikipedia.org/wiki/.har) (HTTP Archive) ファイルは、Web ブラウザーと Web サーバーのやり取りを記録した JSON 形式のログです。

HAR ファイルには、ログイン トランザクション中に発生するリダイレクトの順序が記録されるため、認証の問題をデバッグする際に非常に役立ちます。[サポート リクエスト](https://support.auth0.com)に HAR ファイルを含めることで、トラブルシューティングを迅速に進めることができます。

<Warning>
  HAR ファイルを誰か (Auth0 を含む) と共有する前に、必ず次のような機密データをすべて削除するか、難読化してください。

  * 機密性の高いユーザー情報
  * 個人を特定できる情報 (PII)
  * 機密性の高いアプリケーション情報

  詳細については、[Auth0 Community](https://community.auth0.com) の以下の記事を参照してください。

  * [HTTP トレースのサニタイズ](https://community.auth0.com/t/sanitizing-http-traces/119488)
  * [HTTP トレース ファイルを自動的にサニタイズする方法](https://community.auth0.com/t/how-to-sanitize-a-http-trace-file-automatically/120583)
  * [機密情報を手動でマスキングする方法](https://community.auth0.com/t/how-to-manually-redact-sensitive-information/122554)
  * [HAR ファイルが大きすぎてサポートケースにアップロードできない](https://community.auth0.com/t/har-file-is-too-large-to-upload-to-the-support-case/122488)
</Warning>

<div id="generate-har-files">
  ## HAR ファイルを生成する
</div>

ほとんどのウェブブラウザーでは、ブラウザーに搭載されているデベロッパーツールやインスペクターツールを使用して HAR ファイルを生成できます。

<div id="google-chrome">
  ### Google Chrome
</div>

1. Google Chrome で開いているすべての**シークレット**ウィンドウを閉じます。
2. Google Chrome で**新しいシークレット**ウィンドウを開きます。
3. **View &gt; Developer &gt; 開発者ツール** を選択します。
4. 開発者ツール ペインで、**Network** タブを選択します。
5. すべての操作を記録するため、**Preserve Log** チェックボックスをオンにします。
6. ページにアクセスし、問題を再現する手順を実行します。
7. **Network** タブを選択します。
8. 下向き矢印をクリックして HAR ファイルをエクスポートします。
9. HAR ファイルを保存します。

<div id="safari">
  ### Safari
</div>

1. **Safari &gt; Preferences &gt; Advanced** で **Show Develop menu in menu bar** チェックボックスがオンになっていることを確認します。
2. **File &gt; Open New Private Window** を選択します。
3. **Developer** メニューで **Show Web Inspector** を選択し、**Network** タブを開きます。
4. Web Inspector の左上付近にある **All** の右側で **Filter button** をクリックし、**Preserve Log** を選択します。
5. 問題が発生する Web ページにアクセスします。
6. **Develop &gt; Show Web Inspector** を選択します。Web Inspector ウィンドウが表示されます。
7. 問題を再現するために、ページ上で必要な操作を行います。
8. **Network** タブを選択します。
9. ペイン右上の **Export** をクリックします。
10. HAR ファイルを保存します。

<div id="firefox">
  ### Firefox
</div>

1. Firefox ですべての**プライベート**ウィンドウを閉じます。
2. Firefox で**新しいプライベート**ウィンドウを開きます。
3. **Tools &gt; Developer &gt; Network** (**Ctrl + Shift + E**) を開きます。
4. **Network** タブをクリックし、**Persist Logs.** を選択します。
5. ページにアクセスし、問題を再現する手順を実行します。
6. **Network** タブを選択し、右クリックして **Save All As HAR** を選択します。
7. HAR ファイルを保存します。

<div id="microsoft-edge">
  ### Microsoft Edge
</div>

1. Microsoft Edge の **InPrivate** ウィンドウをすべて閉じます。
2. **新しい InPrivate** ウィンドウを開きます (**Ctrl + Shift + N**) 。
3. **設定など** (**...** アイコン) を開き、**その他のツール &gt; 開発者ツール** を選択します。
4. 開発者ツールの **Network** タブでプロファイリング セッションを開始します。
5. ページにアクセスし、問題を再現する手順を実行します。
6. **HAR としてエクスポート** (**Ctrl + S**) を選択し、HAR ファイルを保存します。

<div id="internet-explorer">
  ### Internet Explorer
</div>

1. Internet Explorer で開いている **InPrivate** ウィンドウをすべて閉じます。
2. Internet Explorer で **新しい InPrivate** ウィンドウを開きます (**Ctrl + Shift + P**) 。
3. **Tools &gt; F12 Developer Options &gt; Network** に移動します。
4. **Clear entries on navigate** がオフになっていることを確認します。
5. ページにアクセスし、問題を引き起こす手順を実行します。
6. **Network** タブを選択し、**Export as HAR** をクリックします (**Ctrl + S**) 。
7. HAR ファイルを保存します。

<div id="analyze-har-files">
  ## HAR ファイルを分析する
</div>

Auth0 は、HAR ファイルの分析にサードパーティアプリケーション、ツール、またはソフトウェアを使用しておらず、その使用も推奨していません。

HAR ファイルをサニタイズしたら、テキストエディターまたはブラウザーで開いて、記録されたイベントを確認できます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  問題のトラブルシューティングに役立つ各ログイベントの詳細については、[Monitoring ページ](/ja/docs/deploy-monitor/monitor)を参照してください。
</Callout>
