Authorization Core feature set を使用して [ロールベースアクセス制御 (RBAC) ](/ja/docs/manage-users/access-control/rbac) を実装する際によくある問題について、その解決策を紹介します。

<div id="role-based-access-control-is-enabled-for-my-api-but-the-scopes-claim-is-not-showing-what-you-say-it-should">
  ## API でロールベースアクセス制御を有効にしているのに、scopes クレームが説明どおりに表示されない
</div>

* Rule で `accessToken.scope` を設定していないことを確認してください。
* 設定されている[認可ルール](/ja/docs/manage-users/access-control/rules-for-authorization-policies)は、RBAC ベースの認可の判定が行われた後に実行されるため、デフォルトの動作を上書きする可能性があることに注意してください。
