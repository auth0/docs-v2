---
description: 問題のトラブルシューティングに向けてユーザープロフィールを確認する方法を説明します
title: ユーザープロフィールを確認する
---

1. ソース (認可サーバー) のユーザープロフィール情報は正しいですか？
2. [HAR file](/ja/docs/troubleshoot/troubleshooting-tools/generate-and-analyze-har-files) を生成して確認し、`id_token` を探します。

   <Warning>
     HAR ファイルを誰か (Auth0 を含む) と共有する前に、次のような機密データをすべて削除または難読化していることを確認してください。

     * 機密のユーザー情報
     * 個人を特定できる情報 (PII)
     * 機密のアプリケーション情報

     詳細については、[Auth0 Community](https://community.auth0.com) の以下の記事を参照してください。

     * [Sanitizing HTTP Traces](https://community.auth0.com/t/sanitizing-http-traces/119488)
     * [How to Sanitize an HTTP Trace File Automatically](https://community.auth0.com/t/how-to-sanitize-a-http-trace-file-automatically/120583)
     * [How to Manually Redact Sensitive Information](https://community.auth0.com/t/how-to-manually-redact-sensitive-information/122554)
     * [HAR File is Too Large to Upload to the Support Case](https://community.auth0.com/t/har-file-is-too-large-to-upload-to-the-support-case/122488)
   </Warning>
3. [JWT.io](https://jwt.io) で `id_token` をデコードし、正しい情報が含まれているか確認します。
4. カスタムデータベーススクリプトまたは Rule のロジックを確認します。
5. `/tokeninfo` エンドポイントを呼び出しており、Auth0 でカスタムドメインを設定しているか確認します。設定している場合は、代わりに `/userinfo` エンドポイントを使用する必要があります
6. `/userinfo` エンドポイントを正しく呼び出しているか確認します。アクセストークンを渡す必要があります。テナントでカスタムドメインが有効になっている場合でも、このエンドポイントはデフォルトの Auth0 ドメインで呼び出す必要があります。
7. アクセストークンを取得するために、正しい [スコープ](/ja/docs/get-started/apis/scopes) を指定しているか確認します。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  問題のトラブルシューティングに役立つ各ログイベントの詳細については、[Monitoring page](/ja/docs/deploy-monitor/monitor) を参照してください。
</Callout>
