想定するシナリオは次のとおりです。ユーザーの認証後、その認証情報を保存し、有効な間はそれを使い続けたいとします。有効期限が切れた場合は、ログインページを再表示しなくて済むように、`refreshToken` を使って認証情報を更新したいところです。これを自動的に行うのではなく、ユーザーに指紋または顔で本人確認してもらいます。

ユーザーの認証情報の管理を簡素化し、生体認証を行うために、[Auth0.swift](https://github.com/auth0/Auth0.swift/) の [Credentials Manager](https://github.com/auth0/Auth0.swift/blob/master/Auth0/CredentialsManager.swift) ユーティリティを使用します。

<div id="getting-started">
  ## はじめに
</div>

まず、`Auth0` モジュールをインポートします。

`import Auth0`

<div id="credentials-manager">
  ### Credentials Manager
</div>

認証情報を取得する前に、iOS デバイスでサポートされている生体認証 (Face ID または Touch ID) を利用することもできます。

まず、Credentials Manager を設定します。次に、生体認証を有効にします。必要に応じて、プロンプトに表示するタイトルを渡すこともできます。

```swift lines
let credentialsManager = CredentialsManager(authentication: Auth0.authentication())
credentialsManager.enableBiometrics(withTitle: "Touch ID / Face ID Login")
```

Face ID を使用する理由を表示するために、[NSFaceIDUsageDescription](https://developer.apple.com/library/archive/documentation/General/Reference/InfoPlistKeyReference/Articles/CocoaKeys.html#//apple_ref/doc/uid/TP40009251-SW75) 設定をプロジェクトの `Info.plist` に追加することを強く推奨します。場合によっては、説明文を指定していない状態でユーザーが Face ID 認証を試みると、認証に失敗する可能性があります。

```xml lines
...
<key>NSFaceIDUsageDescription</key>
<string>Reason why we use Face ID here</string>
...
```

<div id="login">
  ### ログイン
</div>

<Tooltip tip="Universal Login: ユーザーの本人確認を行うために、アプリケーションは Auth0 の認可サーバーでホストされている Universal Login にリダイレクトします。" cta="用語集を表示" href="/ja/docs/glossary?term=Universal+Login">Universal Login</Tooltip> ページを表示し、認証に成功したら認証情報を Credentials Manager に渡します。

export const codeExample = `Auth0
    .webAuth()
    .scope("openid profile offline_access")
    .audience("https://{yourDomain}/userinfo")
    .start {
        switch $0 {
        case .failure(let error):
            // エラーを処理する
        case .success(let credentials):
            // Credentials Manager を使用して認証情報を安全に保存する
            credentialsManager.store(credentials: credentials)
        }
}`;

<div id="renew-user-credentials">
  ### ユーザーの認証情報を更新する
</div>

ユーザーの認証情報を更新する必要がある場合は、Credentials Manager の `credentials` メソッドを呼び出せます。

```swift lines
credentialsManager.credentials { error, credentials in
    guard error == nil, let credentials = credentials else {
        // エラーを処理する
        // ログイン画面にフォールバックする
    }
    // 認証が成功したため、ユーザーのルーティングを続行する
}
```

更新時の処理は Credentials Manager が行うため、新しい認証情報を手動で保存する必要はありません。
