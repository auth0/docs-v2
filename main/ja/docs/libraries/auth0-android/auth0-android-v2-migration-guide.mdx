Auth0 Android SDK のバージョン 2 では、Auth0 を Android アプリケーションに統合する際の多くの改善が行われています。**このアップグレードには破壊的変更が含まれています。** v2 への移行時に必要な変更を把握するため、このガイドを確認してください。

<div id="requirements-changes">
  ## 要件の変更
</div>

v2 では、Android API バージョン 21 以降と Java 8 以降が必要です。

Android 用プラグインと Kotlin プラグインでそれぞれ Java 8 バイトコードをターゲットにするため、`build.gradle` には次の内容が必要です。

```kotlin lines
android {
  compileOptions {
      sourceCompatibility JavaVersion.VERSION_1_8
      targetCompatibility JavaVersion.VERSION_1_8
  }

  kotlinOptions {
      jvmTarget = '1.8'
  }
}
```

<div id="open-id-connect-only">
  ## OpenID Connect のみ
</div>

v2 では、OIDC 準拠のアプリケーションのみをサポートします。ユーザーの認証にこの SDK を使用する場合、現在の Auth0 認証パイプラインとエンドポイントのみが使用されます。`Auth0` クラスから `setOIDCConformant(boolean value)` メソッドは削除されました。

詳細は、[OpenID Connect プロトコル](/ja/docs/authenticate/protocols/openid-connect-protocol) を参照してください。

<div id="authorization-code-with-pkce">
  ## PKCE を使用した認可コード
</div>

v2 では、[Proof Key for Code Exchange (PKCE) を使用する認可コード](/ja/docs/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce) フローのみがサポートされています。`WebAuthProvider.Builder` の `withResponseType(@ResponseType int type)` メソッドは削除されました。

<div id="removal-of-webview-support">
  ## WebView サポートの削除
</div>

非推奨となっていた `WebView` コンポーネントを使用した認証機能は削除されました。セキュリティとユーザーの信頼性を向上させるため、代わりに常に外部ブラウザーアプリケーションが使用されます。

<div id="removal-of-legacy-authentication-api-support">
  ## レガシー Authentication API サポートの削除
</div>

レガシーに分類される [Authentication API](https://auth0.com/docs/api/authentication) を呼び出すためのメソッドとクラスを削除しました。完全な一覧については、以下の **削除されたクラスとインターフェース** セクションを参照してください。

<div id="request-interfaces-changes">
  ## Request インターフェースの変更
</div>

`com.auth0.android.request` パッケージでは、HTTP リクエストを構築して実行するための最上位インターフェースを定義しています。従来は、すべてのリクエスト型にカスタムヘッダーやリクエストパラメーターを追加できないことが、よくある問題の 1 つでした。この問題に対応するため、任意のリクエストに追加のヘッダーやリクエストパラメーターを設定できるよう、リクエストインターフェースをリファクタリングしました。なお、これらのパラメーターは現在 `String` 型である必要があります。

最上位の `Request` インターフェースでは、次のメソッドが定義されています。

* `public fun addParameters(parameters: Map<String, String>): Request<T, U>`
* `public fun addParameter(name: String, value: String): Request<T, U>`
* `public fun addHeader(name: String, value: String): Request<T, U>`

次のインターフェースは削除されました。

* `ParameterizableRequest`
* `AuthRequest`

これに伴い、`AuthenticationAPIClient` では多くのメソッドの戻り値の型が変更されています。以下はその例です。

* `ParameterizableRequest`、`TokenRequest`、または `DatabaseConnectionRequest` を返していたメソッドは、現在は `Request` を返します

* `AuthRequest` を返していたメソッドは、現在は `AuthenticationRequest` を返します

これらのメソッドの戻り値の型を直接使用している場合は、上記の説明どおりの型を想定するようにコードを変更する必要があります。リクエストを実行するために `start` または `execute` を続けて呼び出している場合は、変更不要です。

`AuthenticationRequest` インターフェースからは、`setAccessToken("{ACCESS-TOKEN}")` メソッドが削除されました。このメソッドは、2017 年 6 月に無効化された [Authentication API のレガシーエンドポイント `/oauth/access\_token`](https://auth0.com/docs/api/authentication#social-with-provider-s-access-token) へのリクエストに設定するトークン用のものでした。この名前のパラメーターを送信する必要がある場合は、`addParameter("access_token", "{ACCESS-TOKEN}")` を使用してください。

`ParameterizableRequest` または `AuthRequest` を実装していたクラスは、これらの変更に対応するよう更新されています。

完全な一覧については、以下の **削除されたクラスとインターフェース** セクションを参照してください。

<div id="networking-client-customization">
  ## ネットワーククライアントのカスタマイズ
</div>

v2 では、このライブラリの HTTP リクエスト処理のカスタマイズ性が向上しています。新しいインターフェース `NetworkingClient` は、HTTP リクエストを実行するための仕様を定義します。`Auth0` クラスには、`NetworkingClient` のインスタンスを設定できます。これにより、ネットワーククライアントを完全に制御するための独自実装を提供できます。さらに、このライブラリが提供するデフォルトのネットワーククライアントでは、一般的なカスタマイズ箇所を簡単に設定できるように、いくつかのカスタマイズポイントが用意されています。

<div id="timeout-configuration">
  ### タイムアウト設定
</div>

export const codeExample1 = `// 変更前
val account = Auth0("{yourClientId}", "{yourDomain}")

account.connectTimeoutInSeconds = 30
account.readTimeoutInSeconds = 30

val authAPI = AuthenticationAPIClient(account)

// 変更後
val netClient = DefaultClient(
    connectTimeout = 30,
    readTimeout = 30
)

val account = Auth0("{yourClientId}", "{yourDomain}")
account.networkingClient = netClient`;

<div id="detailed-change-listing">
  ## 変更の詳細一覧
</div>

<div id="files-that-changed-their-package">
  ### パッケージが変更されたファイル
</div>

* `com.auth0.android.authentication.request.ProfileRequest` クラスは `com.auth0.android.request.ProfileRequest` に移動されました。
* `com.auth0.android.authentication.request.SignUpRequest` クラスは `com.auth0.android.request.SignUpRequest` に移動されました。

<div id="internal-package">
  ### 内部パッケージ
</div>

このライブラリの以前のバージョンでは、`com.auth0.android.request.internal` パッケージ内のいくつかのクラスが公開されていました。これらはもともとライブラリの Public API の一部として意図されたものではなく、Java のドキュメントでも直接使用は推奨されていませんでした。コードベースが Kotlin に移行したため、現在は `internal` 修飾子を活用し、必要に応じて使用しています。これらのクラスは Java プロジェクトでは `public` クラスとして見える場合がありますが、使用しないでください。これらは現在もこのライブラリの Public API の一部ではありません。

これらのクラスに対するサポートは提供しません。必要に応じて予告なく変更される場合があります。

<div id="classes-and-interfaces-removed">
  ### 削除されたクラスとインターフェース
</div>

* `com.auth0.android.util.Base64` クラスは削除されました。代わりに `android.util.Base64` を使用してください。
* `com.auth0.android.request.ParameterizableRequest` インターフェースは削除されました。リクエストヘッダーとパラメーターを追加する機能は、`com.auth0.android.request.Request` インターフェースに移されました。
* `com.auth0.android.request.AuthRequest` インターフェースは削除されました。代わりに `com.auth0.android.request.AuthenticationRequest` インターフェースを使用してください。
* `com.auth0.android.provider.WebAuthActivity` クラスは削除されました。認証には常に外部ブラウザーアプリが使用されます。
* `com.auth0.android.result.Delegation` クラスは削除されました。これは、2017 年 6 月時点で無効化された Authentication API のレガシーエンドポイントへのリクエスト結果として使用されていました。
* `com.auth0.android.authentication.request.DelegationRequest` クラスは削除されました。これは、2017 年 6 月時点で無効化されたレガシー Authentication API エンドポイントへのリクエストを表すために使用されていました。
* `com.auth0.android.util.Telemetry` クラスは `com.auth0.android.util.Auth0UserAgent` に名前変更されました。
* `com.auth0.android.request.AuthorizableRequest` クラスは削除されました。同じことは `addHeader("Authorization", "Bearer {TOKEN_VALUE}")` を使用して実現できます。
* `com.auth0.android.authentication.request.TokenRequest` クラスは削除されました。Code Verifier と、任意のリクエストヘッダーおよびパラメーターを設定する機能は `com.auth0.android.request.Request` インターフェースに移されました。
* `com.auth0.android.authentication.request.DatabaseConnectionRequest` クラスは削除されました。任意のリクエストヘッダーおよびパラメーターを設定する機能は `com.auth0.android.request.Request` インターフェースに移されました。
* `com.auth0.android.provider.VoidCallback` クラスは削除されました。引数を取らないコールバックを使用する機能は、`Callback<Void, AuthenticationException>` に置き換えられました。
* `com.auth0.android.request.ErrorBuilder` インターフェースは削除されました。
* `com.auth0.android.RequestBodyBuildException` クラスは削除されました。
* `com.auth0.android.util.CheckHelper` クラスは削除されました。
* `com.auth0.android.util.JsonRequired` インターフェースは削除されました。
* `com.auth0.android.util.JsonRequiredTypeAdapterFactory` クラスは削除されました。

<div id="constants-removed">
  ### 削除された定数
</div>

* `ParameterBuilder.GRANT_TYPE_JWT` は削除されました。
* `ParameterBuilder.ID_TOKEN_KEY` は削除されました。
* `ParameterBuilder.DEVICE_KEY` は削除されました。
* `ParameterBuilder.ACCESS_TOKEN_KEY` は削除されました。
* `ResponseType.CODE`、`ResponseType.ID_TOKEN`、および `ResponseType.ACCESS_TOKEN` は削除されました。

<div id="classes-and-interfaces-changed">
  ### 変更されたクラスとインターフェース
</div>

* `SignupRequest` は、削除された `AuthRequest` ではなく、`AuthenticationRequest` を実装するようになりました
* `AuthorizableRequest` は、削除された `ParameterizableRequest` ではなく、`Request` を拡張するようになりました
* `BaseCallback` は非推奨になりました。代わりに `Callback` を使用してください。

<div id="constructors-changed">
  ### 変更されたコンストラクタ
</div>

* `Context` から `AuthenticationAPIClient` を生成することはできなくなりました。代わりに `AuthenticationAPIClient(auth0: Auth0)` を使用してください。`Context` を使用して `Auth0` のインスタンスを作成できます。
* `Context` から `UsersAPIClient` を生成することはできなくなりました。代わりに `UsersAPIClient(auth0: Auth0, token: String)` を使用してください。`Context` を使用して `Auth0` のインスタンスを作成できます。
* `SignupRequest` では、2 番目のパラメーターに `AuthenticationRequest` が必要になりました。
* `ProfileRequest` では、`AuthenticationRequest` と `Request<UserProfile, AuthenticationException>` が必要になりました。
* &quot;expires in&quot; の値を指定して `Credentials` を生成することはできなくなりました。代わりに、有効期限の日時、つまり &quot;expires at&quot; を指定してください。通常、`Credentials` を自分で生成することはありません。

<div id="methods-removed-or-changed">
  ### 削除または変更されたメソッド
</div>

<div id="auth0-methods-removed">
  #### 削除された Auth0 メソッド
</div>

* `setOIDCConformant(boolean enabled)` と `isOIDCConformant()` は削除されました。SDK は現在、OIDC 準拠アプリケーションのみをサポートしています。
* `doNotSendTelemetry()` は削除されました。代替手段はありません。
* `setWriteTimeoutInSeconds(seconds)` と `getWriteTimeoutInSeconds(seconds)` は削除されました。代替手段はありません。
* `setReadTimeoutInSeconds(seconds)` と `getReadTimeoutInSeconds(seconds)` は削除されました。読み取りタイムアウトは `DefaultClient` クラスで直接設定できます。`DefaultClient(readTimeout = 123)` を使用してください。
* `setConnectTimeoutInSeconds(seconds)` と `getConnectTimeoutInSeconds(seconds)` は削除されました。接続タイムアウトは `DefaultClient` クラスで直接設定できます。`DefaultClient(connectTimeout = 123)` を使用してください。
* `setLoggingEnabled(boolean enabled)` と `isLoggingEnabled()` は削除されました。ネットワークトラフィックロガーは `DefaultClient` クラスで直接有効にできます。`DefaultClient(enableLogging = true)` を使用してください。これはデバッグ目的でのみ使用し、本番環境では決して有効にしないでください。
* `setTLS12Enforced()` と `isTLS12Enforced()` は削除されました。SDK は現在、モダンな TLS をデフォルトでサポートしています。

<div id="authenticationapiclient-methods-removed-or-changed">
  #### AuthenticationAPIClient で削除または変更されたメソッド
</div>

* `setUserAgent(String)` は削除されました。送信するヘッダーは、各 `Request` インスタンスで `addHeader("{NAME}", "{VALUE}")` メソッドを使用して直接設定するか、`DefaultClient(defaultHeaders = mapOf())` コンストラクターパラメーターを使用してネットワーククライアントインスタンス全体に設定できます。

レガシーに分類される [Authentication APIs](https://auth0.com/docs/api/authentication) の呼び出し専用のメソッドとクラスは、v2 で削除されました。削除されたメソッドは次のとおりです。

* `delegation()`, `delegationWithIdToken("{ID-TOKEN}")`, `delegationWithRefreshToken("{REFRESH-TOKEN}")`, `delegationWithIdToken("{ID-TOKEN}", "{API-TYPE}}")`。レガシー Authentication API エンドポイントのサポートは削除されました。
* `loginWithOAuthAccessToken("{TOKEN}", {CONNECTION})`。一部のソーシャルプロバイダーでは、代わりに `loginWithNativeSocialToken("{TOKEN}", "{TOKEN-TYPE}")` を使用できます。
* `tokenInfo("{ID-TOKEN}")`。代わりに `userInfo("{ACCESS-TOKEN}")` を使用してください。

`ParameterizableRequest` を返していたメソッドは、現在は `Request` を返します。

* `userInfo("{ACCESS-TOKEN}")`
* `revokeToken("{REFRESH-TOKEN}")`
* `renewAuth("{REFRESH-TOKEN}")`
* `passwordlessWithEmail("{EMAIL}", PasswordlessType, "{CONNECTION}")`
* `passwordlessWithSMS("{PHONE-NUNBER}", PasswordlessType, "{CONNECTION}")`
* `fetchJsonWebKeys()`

`AuthRequest` を返していたメソッドは、現在は `AuthenticationRequest` を返します。

* `login("{USERNAME-OR-EMAIL}", "{PASSWORD}", "{REALM-OR-CONNECTION}")`
* `login("{USERNAME-OR-EMAIL}", "{PASSWORD}")`
* `loginWithOTP("{MFA-TOKEN}", "{OTP}")`
* `loginWithNativeSocialToken("{TOKEN}", "{TOKEN-TYPE}")`
* `loginWithPhoneNumber("{PHONE-NUMBER}", "{VERIFICATION-CODE}", "{REALM-OR-CONNECTION}")`
* `loginWithEmail("{EMAIL}", "{VERIFICATION-CODE}", "{REALM-OR-CONNECTION}")`

`DatabaseConnectionRequest` を返していたメソッドは、現在は `Request` を返します。

* `createUser("{EMAIL}", "{PASSWORD}", "{USERNAME}", "{CONNECTION}")`
* `resetPassword("{EMAIL}","{CONNECTION}")`

`TokenRequest` を返していたメソッドは、現在は `Request` を返します。

* `token("{AUTHORIZATION-CODE}", "{CODE-VERIFIER}", "{REDIRECT-URI}")`

<div id="authenticationrequest-methods-removed">
  #### AuthenticationRequest メソッドの削除
</div>

* `addAuthenticationParameters(parameters)` は削除されました。代わりに `addParameters(parameters)` を使用してください。
* `setDevice("{DEVICE}").` は削除されました。代わりに `addParameter("device", "{VALUE}")` を使用してください。

<div id="webauthproviderbuilder-methods-removed">
  #### WebAuthProvider.Builder メソッドの削除
</div>

* `useCodeGrant(boolean useCodeGrant)`。代替手段はありません。v2 では Code + PKCE フローのみをサポートしています。
* `useBrowser(boolean useBrowser)`。代替手段はありません。このライブラリでは WebView 認証をサポートしなくなりました。
* `useFullscreen(boolean useFullscreen)`。代替手段はありません。このライブラリでは WebView 認証をサポートしなくなりました。
* `withResponseType(@ResponseType int type)`。代替手段はありません。v2 では Code + PKCE フローのみをサポートしています。
* `start(activity: Activity, callback: AuthCallback, requestCode: Int)` は削除されました。代わりに `start(activity: Activity, callback: Callback<Credentials, AuthenticationException>)` を使用してください。

<div id="webauthproviderlogoutbuilder-methods-removed">
  #### WebAuthProvider.LogoutBuilder のメソッドを削除
</div>

* `start(context: Context, callback: VoidCallback)`。代わりに `start(context: Context, callback: Callback<Void, AuthenticationException>)` を使用してください。

<div id="webauthprovider-methods-removed">
  #### WebAuthProvider メソッドの廃止
</div>

* `init(account: Auth0)` は廃止されました。代わりに `login(account: Auth0)` を使用してください。
* `init(context: Context)` は廃止されました。代わりに `login(account: Auth0)` を使用してください。
* `resume(requestCode: Int, resultCode: Int, intent: Intent)` は廃止されました。代わりに `resume(intent: Intent)` を使用してください。
* `init(account: Auth0)` は廃止されました。代わりに `login(account: Auth0)` を使用してください。

<div id="parameterbuilder-methods-removed">
  #### ParameterBuilder のメソッドの削除
</div>

* `setDevice("{DEVICE}")` は削除されました。代わりに `set("device", "{VALUE}")` を使用します。

<div id="usersapiclient-methods-changed">
  #### UsersAPIClient のメソッドの変更
</div>

* `setUserAgent(String)` は削除されました。送信するヘッダーは、`addHeader("{NAME}", "{VALUE}")` メソッドを使用して各 `Request` インスタンスに直接設定するか、`DefaultClient(defaultHeaders = mapOf())` コンストラクターパラメーターでネットワーククライアントインスタンス全体に設定できます。

`ParameterizableRequest` を返していたメソッドは、`Request` を返すようになりました。

* `link("{PRIMARY-USER-ID}", "{SECONDARY-TOKEN}")`
* `unlink("{PRIMARY-USER-ID}", "{SECONDARY-USER-ID}", "{SECONDARY-PROVIDER}")`
* `updateMetadata("{USER-ID}", userMetadata)`
* `getProfile("{USER-ID}")`

<div id="requestfactory-methods-removed-or-changed">
  #### RequestFactory メソッドの削除または変更
</div>

`RequestFactory` クラスには、JSON リクエストボディとレスポンスのシリアライズおよびデシリアライズを含む、HTTP リクエストの実行を容易にするメソッドが用意されています。`com.auth0.android.request.internal` パッケージの一部であるため、公開利用は意図されていませんが、公開型であることから、変更の概要を以下に示します。

* すべてのリクエストメソッドは、特定の HTTP リクエストライブラリ (例: `OkHttp`) に依存しないようリファクタリングされました
* すべてのリクエストメソッドは `Request` を返すようになりました。
* すべてのリクエストメソッドが小文字になりました (例: `POST()` -&gt; `post()`) 。
* `authenticationPOST` メソッドは代替手段なしで削除されました。これは、すべての `Request` インスタンスに必要に応じて任意のヘッダーを設定できるためです。

<div id="profilerequest-methods-changed">
  #### ProfileRequest メソッドの変更
</div>

* `addParameters` メソッドでは、値として `String` から `Object` へのマップではなく、`String` から `String` へのマップが必要になりました (`addParameters(mapOf("key" to "val"))`)

<div id="signuprequest-methods-changed">
  #### SignUpRequest メソッドの変更
</div>

パラメータを設定するメソッドでは、値に `String to Object` ではなく `String to String` の Map を指定する必要があります。

* `addAuthenticationParameters(mapOf("key" to "val"))`
* `addSignupParameters(mapOf("key" to "val"))`
* `addParameters(mapOf("key" to "val"))`

さらに、`setDevice("device")` は削除されました。代わりに `addParameter("device", "{VALUE}")` を使用してください。
