ローカル開発環境は通常、デフォルトで http (例: `http://localhost`) で実行されます。このガイドでは、ローカルサーバーを https で実行すべきケースと、localhost で https を設定する方法について説明します。

<div id="when-to-use-a-local-https-server">
  ## ローカルの https サーバーを使用する場面
</div>

Web ブラウザーは http と https で挙動が異なるため、こうした違いを考慮すると、開発サーバーでも https 経由でページを配信する必要がある場合があります。たとえば、次のようなケースです。

* ページが https 経由で配信されている場合、http URL からリソースを読み込むリクエストはブロックされます。本番環境でのみ発生する不具合を避けるため、または機密情報を含む http リクエストを誤って送信しないようにするために、ローカルでもこの挙動を再現したい場合があります。
* Secure 属性を持つ Cookie や、[SameSite](https://auth0.com/blog/browser-behavior-changes-what-developers-need-to-know/) 属性が None に設定された Cookie は、安全でない通信経路では送信されないため、認証やその他の機能に支障が生じます。

<div id="how-to-set-up-a-secure-local-server">
  ## 安全なローカルサーバーを設定する方法
</div>

基本的な考え方は、ローカルの開発サーバーの前段に Web サーバーを立てて https リクエストを受け付け、それを http 経由でアプリケーションの Web サーバーに転送することで、アプリケーションへのリクエストをプロキシ経由で https で処理するというものです。これは https リバースプロキシと呼ばれ、[Caddy](https://caddyserver.com/v2) という既製の製品を使って構築できます。

手順は次のとおりです。

1. Caddy バイナリをダウンロードする
2. 通常どおり、アプリケーションの Web サーバーを http で実行する
3. Caddy バイナリをリバースプロキシとして実行し、トラフィックをアプリケーションの Web サーバーに転送するよう設定する
4. https 経由でリバースプロキシ Web サーバーにアクセスする

<div id="step-1-download-the-caddy-binary">
  ### ステップ 1. Caddy バイナリをダウンロードする
</div>

[https://caddyserver.com/download](https://caddyserver.com/download) にアクセスし、使用しているプラットフォームを選択して、ファイルをコンピューターにダウンロードします。

<div id="step-2-run-your-applications-web-server-over-http-as-normal">
  ### 手順 2. 通常どおり HTTP でアプリケーションの Web サーバーを実行します
</div>

たとえば $ PORT=3000 npm start のように、通常どおりアプリケーションのローカル開発サーバーを起動します

<div id="step-3-run-the-caddy-binary-as-a-reverse-proxy-telling-it-to-forward-traffic-to-your-applications-web-server">
  ### ステップ 3. Caddy バイナリをリバースプロキシとして実行し、トラフィックをアプリケーションの Web サーバーに転送するように設定します
</div>

ダウンロードした Caddy バイナリの場所を確認し、次のコマンドを実行してリバースプロキシを設定し、ポート 443 (https で使用されるデフォルトポート) へのリクエストをアプリケーションのポート (この場合は 3000) に転送します。

```bash wrap lines
/path/to/caddy reverse-proxy --from localhost:443 --to localhost:3000
```

<div id="step-4-now-visit-your-web-server-over-https">
  ### 手順 4. HTTPS 経由で Web サーバーにアクセスする
</div>

[https://localhost](https://localhost) にアクセスすると、アプリケーションへのリクエストは HTTPS 経由で処理されます

<div id="running-your-application-behind-a-proxy">
  ## プロキシの背後でアプリケーションを実行する
</div>

アプリケーションをプロキシの背後で実行する場合は、若干の調整が必要になることがあります。
