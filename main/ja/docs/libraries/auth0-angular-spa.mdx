title: シングルページアプリ向け Auth0 Angular SDK
---

Auth0 Angular SDK は、Auth0 を使用して Angular アプリに認証と認可を実装するための JavaScript ライブラリです。サービス、認証ガード、HTTP インターセプターを提供し、Angular アプリ内で一般的な認証タスクを実行できるようにします。

Auth0 Angular SDK は、グラントやプロトコルの詳細、トークンの有効期限と更新、トークンの保存とキャッシュを処理します。内部では、[Universal Login](/ja/docs/authenticate/login/auth0-universal-login) と [Authorization Code Grant Flow with PKCE](/ja/docs/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce) を実装しています。

このライブラリは [GitHub で公開](https://github.com/auth0/auth0-angular) されており、[API の詳細を確認](https://auth0.github.io/auth0-angular/) できます。

<div id="installation">
  ## インストール
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Angular SDK は、Angular SPA バージョン 14 以降に対応しています。
</Callout>

[Angular CLI](https://cli.angular.io/) を使用して、プロジェクトに SDK をインストールします。

`ng add @auth0/auth0-angular`

<div id="getting-started">
  ## はじめに
</div>

`AuthModule` をプロジェクトにインポートし、Auth0 のクライアントアプリ向けに設定して、SDK の統合を開始します。

```jsx lines
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { AppComponent } from './app.component';

// SDKからモジュールをインポートする
import { AuthModule } from '@auth0/auth0-angular';

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,

    // 設定とともにモジュールをアプリケーションにインポートする
    AuthModule.forRoot({
      domain: '{yourDomain}',
      clientId: 'YOUR_AUTH0_CLIENT_ID',
      authorizationParams: {
        redirect_uri: window.location.origin,
      },
    }),
  ],

  bootstrap: [AppComponent],
})
export class AppModule {}
```

<div id="login">
  ## ログイン
</div>

この SDK は、Angular の依存性注入システムを完全にサポートしています。まず、コンポーネントに `AuthService` 型をインポートし、コンストラクターに注入します。

次に、ユーザーのログインを開始するには、サービスの `loginWithRedirect` または `loginWithPopup` メソッドを呼び出します。

```jsx lines
import { Component } from '@angular/core';

// SDKからAuthServiceタイプをインポートする
import { AuthService } from '@auth0/auth0-angular';

@Component({
  selector: 'app-login',
  template: '<button (click)="login">Log in</button>',
  styles: [],
})
export class LoginComponent {
  // コンストラクター経由で認証サービスをコンポーネントにインジェクトする
  constructor(public auth: AuthService) {}

  login(): void {
    // これを呼び出してユーザーをログインページにリダイレクトする
    this.auth.loginWithRedirect();
  }
}
```

<div id="logout">
  ## ログアウト
</div>

ユーザーをログアウトするには、`logout` を使用します。`returnTo` が <Tooltip tip="サービスを設定するための Auth0 の主要な製品。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> の **Allowed Logout URLs** で指定されていることを確認してください。

```jsx lines
import { Component } from '@angular/core';

// SDKからAuthServiceタイプをインポートする
import { AuthService } from '@auth0/auth0-angular';

@Component({
  selector: 'app-logout',
  template: '<button (click)="logout">Log out</button>',
  styles: [],
})
export class LogoutComponent {
  // コンストラクター経由で認証サービスをコンポーネントにインジェクトする
  constructor(@Inject(DOCUMENT) private doc: Document, public auth: AuthService) {}

  login(): void {
    // これを呼び出してユーザーをログインページにリダイレクトする
    this.auth.logout({ 
      logoutParams: { 
        returnTo: this.doc.location.origin 
      }
    });
  }
}
```

<div id="authentication-state">
  ## 認証状態
</div>

認証状態は、`isAuthenticated$` オブザーバブルを介して公開されています。

```html lines
<div *ngIf="auth.isAuthenticated$ | async">
  <app-login></app-login>
</div>
```

<div id="user-profile">
  ## ユーザープロファイル
</div>

認証済みユーザーのユーザープロファイルは、`user$` オブザーバブルを通じて利用できます。

```html lines
<ul *ngIf="auth.user$ | async as user">
  <li>{{ user.name }}</li>
  <li>{{ user.email }}</li>
</ul>
```

内部的には、この オブザーバブル は `isAuthenticated$` オブザーバブル に連鎖しているため、ユーザーがログインしているかどうかを手動で確認する必要はありません。認証済みユーザーが利用可能になると、値の発行を開始します。

<div id="protect-a-route">
  ## ルートを保護する
</div>

組み込みの`AuthGuard`を使用して、ルートコンポーネントを保護します。未認証の状態でこのルートにアクセスすると、ユーザーはログインページにリダイレクトされ、ログイン後にこのページへ戻されます。

認証ガードは、`canActivate`フック (子ルートを使用している場合は`canActivateChild`) を使ってルート定義に追加する必要があります。

```jsx lines expandable
import { NgModule } from '@angular/core';
import { Routes, RouterModule } from '@angular/router';
import { HomeComponent } from './unprotected/unprotected.component';
import { ProtectedComponent } from './protected/protected.component';

// 認証ガードをインポートする
import { AuthGuard } from '@auth0/auth0-angular';

const routes: Routes = [
  {
    path: 'protected',
    component: ProtectedComponent,

    // `canActivate` フックに認証ガードを登録してルートを保護する
    canActivate: [AuthGuard],
  },
  {
    path: '',
    component: HomeComponent,
    pathMatch: 'full',
  },
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule],
})
export class AppRoutingModule {}
```

認証ガードは、ユーザーが認証されていない場合に遅延読み込みモジュールが読み込まれないようにする `canLoad` フックにも対応しています。

## API を呼び出す

SDK には組み込みの `HttpInterceptor` があり、Angular の `HttpClient` サービスを使用してリクエストを送信すると、`Authorization` ヘッダーを挿入して <Tooltip tip="アクセストークン: 不透明な文字列または JWT の形式を持つ認可クレデンシャルで、API へのアクセスに使用されます。" cta="用語集を表示" href="/ja/docs/glossary?term=access+tokens">アクセストークン</Tooltip> を自動的に付与します。

意図しない送信先へのアクセストークンの漏えいを防ぐため、どの API ルートにこのヘッダーを自動的に追加するかを指定するように SDK を設定する必要があります。

<div id="import-the-http-interceptor">
  ### HTTP インターセプターをインポートする
</div>

まず、Auth0 の HTTP インターセプターをアプリに統合するために必要な型をインポートします。

```jsx lines
// インターセプターモジュールと必要なAngularの型をインポートする
import { HttpClientModule, HTTP_INTERCEPTORS } from '@angular/common/http';
import { AuthHttpInterceptor } from '@auth0/auth0-angular';

// `providers` 配列にインターセプターをアプリモジュールへ登録する
@NgModule({
  declarations: [],
  imports: [
    BrowserModule,
    HttpClientModule,     // HttpClientでAPI呼び出しを行えるようにするために登録する
    AppRoutingModule,
    AuthModule.forRoot(...),
  ],
  providers: [
    { provide: HTTP_INTERCEPTORS, useClass: AuthHttpInterceptor, multi: true },
  ],
  bootstrap: [AppComponent],
})
```

**注:** このインターセプターは他のインターセプターと連鎖させて使用することを想定しているため、自動登録ではなく、アプリモジュールに手動で登録する必要があります。こちらで適切な位置に組み込まれることは保証できません。

<div id="configure-the-http-interceptor">
  ### HTTP インターセプターを設定する
</div>

これは、要件に応じてさまざまな方法で設定できます。ルートは文字列で指定できるほか、送信するトークンに影響するオプションを含むオブジェクトとして指定することもでき、使用する HTTP メソッドに応じてマッチさせることもできます。ルート URI には、ルートのグループを指定するワイルドカードを含めることもできます。

**注:** HTTP 呼び出しが行われても、設定で指定したルートに一致しない場合、インターセプターはバイパスされ、`Authorization` ヘッダーは含まれません。

この例では、どのルートに `Authorization` ヘッダーを含めるかを SDK に指定する一般的な方法を示します。

```jsx lines expandable
// httpInterceptor設定を含めるように既存のSDK設定を変更する
AuthModule.forRoot({
  domain: '{yourDomain}',
  clientId: 'YOUR_AUTH0_CLIENT_ID',
  authorizationParams: {
    redirect_uri: window.location.origin,
  },

  // AuthHttpInterceptorの設定
  httpInterceptor: {
    allowedList: [
      // '/api'へのすべての呼び出しにアクセストークンを付与する（完全一致）
      '/api',

      // '/api/'で始まるすべての呼び出しにアクセストークンを付与する
      '/api/*',

      // /api/accountsで始まるすべてのリクエストに一致させ、付与するアクセストークンに必要な
      // オーディエンスとスコープも指定する
      {
        uri: '/api/accounts/*',
        tokenOptions: {
          authorizationParams: {
            audience: 'http://my-api/',
            scope: 'read:accounts',
          }
        },
      },

      // HTTPメソッドによるマッチング
      {
        uri: '/api/orders',
        httpMethod: 'post',
        tokenOptions: {
          authorizationParams: {
            audience: 'http://my-api/',
            scope: 'write:orders',
          }
        },
      },

      // 絶対URIを使用する
      {
        uri: 'https://your-domain.auth0.com/api/v2/users',
        tokenOptions: {
          authorizationParams: {
            audience: 'https://your-domain.com/api/v2/',
            scope: 'read:users',
          }
        },
      },
    ],
  },
});
```

**注:** `tokenOptions` は、基盤となる SPA SDK の `getTokenSilently` メソッドに直接渡されます。指定可能なプロパティの詳細については、[ドキュメント](https://auth0.github.io/auth0-spa-js/interfaces/GetTokenSilentlyOptions.html)を参照してください。

<div id="call-an-api">
  ### API を呼び出す
</div>

Angular の `HttpClient` サービスを使用して API 呼び出しを行います。設定に一致するルートでは、アクセストークンが `Authorization` ヘッダーに自動的に含まれます。

```jsx lines
export class MyComponent {
  constructor(private http: HttpClient) {}

  callApi(): void {
    this.http.get('/api').subscribe(result => console.log(result));
  }
}
```

トークンやその他の認可情報を渡すために、呼び出し元で追加の対応を行う必要はないことに注意してください。これらはすべて、インターセプターとモジュール設定によって処理されます。
