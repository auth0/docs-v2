Auth0 ACUL JS SDK を使用すると、最小限の手間で Universal Login のカスタムエクスペリエンスを構築できます。ログイン、サインアップ、パスワードレス、またはパスキーの登録のどの場合でも、この SDK を使えば最新の認証フローを簡単に実装できます。

<div id="why-choose-the-auth0-acul-js-sdk">
  ### Auth0 ACUL JS SDK を選ぶ理由
</div>

* **迅速な統合:** `login-id` や `signup-password` などのあらかじめ用意された画面をそのまま導入できます。
* **モダンな機能:** パスワードレス、パスキー、ソーシャルログインに対応しています。
* **柔軟なモジュール:** 必要なものだけをインポートできます。
* **一元的な状態管理:** 単一のトランザクションオブジェクトで、あらゆる認証フローを処理できます。

<div id="architecture">
  ### アーキテクチャ
</div>

<div id="how-it-works">
  #### どのように機能しますか？
</div>

SDK は、モジュール性と拡張性を考慮して設計されています。構成は次のとおりです。

* **画面:** `login-id` や `signup-password` など、各認証フローの画面は独立したモジュールです。
* **送信ハンドラー:** フォームの送信と画面遷移を管理します。
* **動的フォーム:** 画面の設定に応じて、フィールドやバリデーションを調整します。
* **ヘルパープロパティ:** ブランディングやトランザクションなどの共通データに、統一された方法でアクセスします。
