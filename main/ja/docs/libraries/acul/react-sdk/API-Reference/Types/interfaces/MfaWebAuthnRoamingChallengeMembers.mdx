MfaWebAuthnRoamingChallengeMembers

````ts Example
export interface MfaWebAuthnRoamingChallengeMembers extends BaseMembers {
  /**
   * Screen-specific properties and data, including WebAuthn challenge options.
   * @type {ScreenMembersOnMfaWebAuthnRoamingChallenge}
   */
  screen: ScreenMembersOnMfaWebAuthnRoamingChallenge;

  /**
   * `screen.data.publicKeyChallengeOptions` に指定されたオプションを使用して `navigator.credentials.get()` を呼び出し、WebAuthnチャレンジを開始します。
   * 成功した場合、結果として得られたクレデンシャルを `action: "default"` とともにAuth0に送信します。
   * `navigator.credentials.get()` が既知のWebAuthn API エラー（`NotAllowedError` など）で失敗した場合、
   * このメソッドは内部的に `reportWebAuthnError` を呼び出してAuth0に通知します。
   *
   * @param {VerifySecurityKeyOptions} [options] - 検証プロセスのオプションパラメーター（`rememberDevice` やその他の追加のカスタムオプションなど）。
   * @returns {Promise<void>} 検証の試行が送信されたときに解決されるPromise。
   *                          操作が成功した場合は通常リダイレクトが発生するため、データを直接返しません。
   * @throws {Error} `screen.data.publicKeyChallengeOptions` が存在しない場合、
   *                 `navigator.credentials.get()` が予期しないエラーで失敗した場合、
   *                 またはAuth0へのフォーム送信が失敗した場合にエラーをスローします。
   *
   * @example
   * ```typescript
   * // Assuming 'sdk' is an instance of MfaWebAuthnRoamingChallenge
   * try {
   *   await sdk.verify({ rememberDevice: true });
   *   // On success, Auth0 handles redirection.
   * } catch (error) {
   *   console.error("Security key verification failed:", error);
   *   // Check sdk.transaction.errors for server-side validation messages if the page reloads.
   * }
   * ```
   */
  verify(options?: VerifySecurityKeyOptions): Promise<void>;

  /**
   * Reports a specific WebAuthn API error to Auth0.
   * This method should be used if `navigator.credentials.get()` fails in a way that
   * `verify()` doesn't automatically handle, or if the developer wants to explicitly
   * report an error before trying another action.
   * It submits the error details with `action: "showError::{errorDetails}"`.
   *
   * @param {ReportWebAuthnErrorOptions} options - Contains the `error` object with `name` and `message`
   * from the WebAuthn API, and any other custom options.
   * @returns {Promise<void>} A promise that resolves when the error report is submitted.
   * @throws {Error} Throws an error if the form submission fails.
   *
   * @example
   * ```typescript
   * // Assuming 'sdk' is an instance of MfaWebAuthnRoamingChallenge
   * // And webAuthnError is an error object from a failed navigator.credentials.get() call.
   * try {
   *   await sdk.reportWebAuthnError({
   *     error: { name: webAuthnError.name, message: webAuthnError.message }
   *   });
   * } catch (submitError) {
   *   console.error("Failed to report WebAuthn error:", submitError);
   * }
   * ```
   */
  reportWebAuthnError(options: ReportWebAuthnErrorOptions): Promise<void>;

  /**
   * Allows the user to opt-out of the WebAuthn challenge and select a different MFA方式.
   * This action submits `action: "pick-authenticator"` to Auth0, which should navigate
   * the user to an MFA factor selection 画面.
   *
   * @param {TryAnotherMethodOptions} [options] - Optional custom parameters for the request.
   * @returns {Promise<void>} A promise that resolves when the action is submitted.
   * @throws {Error} Throws an error if the form submission fails.
   *
   * @example
   * ```typescript
   * // Assuming 'sdk' is an instance of MfaWebAuthnRoamingChallenge
   * try {
   *   await sdk.tryAnotherMethod();
   *   // On success, Auth0 handles redirection to MFA selection.
   * } catch (error) {
   *   console.error("Failed to switch MFA方式:", error);
   * }
   * ```
   */
  tryAnotherMethod(options?: TryAnotherMethodOptions): Promise<void>;
}

<ParamField body="reportWebAuthnError" type="Promise<void>">
  特定の WebAuthn API エラーを Auth0 に報告します。
  このメソッドは、`navigator.credentials.get()` が
  `verify()` で自動処理されない形で失敗した場合、または開発者が別の操作を試す前に
  明示的にエラーを報告したい場合に使用します。
  エラーの詳細は `action: "showError::{errorDetails}"` とともに送信されます。

  エラー報告の送信が完了すると解決される Promise です。

  #### 送出

  フォームの送信に失敗した場合はエラーを送出します。

  ```typescript Example
  // 'sdk' は MfaWebAuthnRoamingChallenge のインスタンスであると仮定します
  // また、webAuthnError は navigator.credentials.get() の呼び出し失敗時に返されたエラーオブジェクトです。
  try {
    await sdk.reportWebAuthnError({
      error: { name: webAuthnError.name, message: webAuthnError.message }
    });
  } catch (submitError) {
    console.error("Failed to report WebAuthn error:", submitError);
  }
  ```

<ParamField body="tryAnotherMethod" type="Promise<void>">
  ユーザーが WebAuthn チャレンジを中止し、別の MFA方式 を選択できるようにします。
  この操作では `action: "pick-authenticator"` を Auth0 に送信し、
  ユーザーを MFA 認証要素の選択画面に移動させます。

  アクションの送信が完了すると解決される Promise です。

  #### 送出

  フォームの送信に失敗した場合はエラーを送出します。

  ```typescript Example
  // 'sdk' は MfaWebAuthnRoamingChallenge のインスタンスであると仮定します
  try {
    await sdk.tryAnotherMethod();
    // 成功すると、Auth0 が MFA 選択へのリダイレクトを処理します。
  } catch (error) {
    console.error("Failed to switch MFA method:", error);
  }
  ```

<ParamField body="verify" type="Promise<void>">
  `screen.data.publicKeyChallengeOptions` で指定された
  オプションを使って `navigator.credentials.get()` を呼び出し、WebAuthn チャレンジを開始します。
  成功した場合は、取得した認証情報を `action: "default"` とともに Auth0 に送信します。
  `navigator.credentials.get()` が既知の WebAuthn API エラー (`NotAllowedError` など) で失敗した場合、
  このメソッドは内部で `reportWebAuthnError` を呼び出して Auth0 に通知します。

  検証試行の送信が完了すると解決される Promise です。
  通常、成功時にはリダイレクトが発生するため、データを直接返すことはありません。

  #### 送出

  `screen.data.publicKeyChallengeOptions` が存在しない場合、
  `navigator.credentials.get()` が想定外のエラーで失敗した場合、
  または Auth0 へのフォーム送信に失敗した場合はエラーを送出します。

  ```typescript Example
  // 'sdk' は MfaWebAuthnRoamingChallenge のインスタンスであると仮定します
  try {
    await sdk.verify({ rememberDevice: true });
    // 成功すると、Auth0 がリダイレクトを処理します。
  } catch (error) {
    console.error("Security key verification failed:", error);
    // ページが再読み込みされた場合は、サーバー側の検証メッセージを sdk.transaction.errors で確認してください。
  }
  ```
