---
title: "型エイリアス: MfaPollingOptions"
---

```ts Example
type MfaPollingOptions = {
  intervalMs?: number;
  onCompleted?: () => void;
  onError?: (error: Error) => void;
}
```

<ParamField body="MfaPollingOptions" type="object">
  MFAプッシュ通知のポーリング処理を管理するための設定オプションです。

  ポーリングセッションの開始時にこれらのオプションを指定すると、チャレンジエンドポイントを問い合わせる頻度や、
  成功時または失敗時の処理方法を制御できます。

  <Expandable title="プロパティ">
    <ParamField body="intervalMs" type="number">
      連続するポーリングリクエストの間隔を **ミリ秒** 単位で指定する任意の値です。

      * 省略した場合は、SDK の内部デフォルト間隔が使用されます (通常は 5000 ms) 。
      * 応答性とレート制限への配慮のバランスが取れた値を選択してください。

      ```ts Default Value
      実装固有（一般的には 5000 ms）
      ```
    </ParamField>

    <ParamField body="onCompleted" type="() => void">
      MFAプッシュチャレンジが正常に
      承認され、ポーリングが完了した時点で実行される任意のコールバックです。

      * **1 回だけ** 呼び出され、その後ポーリングは自動的に停止します。
      * ログインフローを進める際に便利です (例: `continue()` の呼び出し) 。
    </ParamField>

    <ParamField body="onError" type="(error: Error) => void">
      ポーリング中にエラーが発生した場合に呼び出される任意のコールバックです。

      次を含む [ULError](/ja/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) オブジェクトを受け取ります。

      * `status` — 失敗したリクエストの HTTP ステータスコード

      * `responseText` — 利用可能な場合は、生のレスポンス本文

      * エラーイベントごとに 1 回呼び出されます。ポーリングはエラーの種類に応じて再試行されることがあります。

      * エラーメッセージを表示したり、カスタムの再試行 / 中断ロジックをトリガーしたりするために使用します。

      **プロパティ**

      <ParamField body="error" type={<a href="/ja/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error">Error</a>}>
        失敗したポーリングリクエストの HTTP ステータスコードと生のレスポンス本文を含むエラーオブジェクトです。
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>
