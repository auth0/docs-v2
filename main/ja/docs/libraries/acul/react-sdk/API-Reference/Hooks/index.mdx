<div id="methods">
  ## メソッド
</div>

* [useAuth0Themes](/ja/docs/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes): ブランディングコンテキストから取得したフラット化済み設定を含む、現在のテーマオプションを取得する React フック。
* [useChangeLanguage](/ja/docs/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage): 現在の ACUL 画面の表示言語を変更する関数を返す React フック。
* [useCurrentScreen](/ja/docs/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen): 現在の画面のコンテキストと状態を取得する React フック。
* [useErrors](/ja/docs/libraries/acul/react-sdk/API-Reference/Hooks/useErrors): ACUL (Universal Login の高度なカスタマイズ) でエラーを読み取り、管理するための React フック。検証エラーとサーバー側エラーを含む、すべてのエラーを扱います。エラーは次の 3 種類に分類されます: - `server` — Auth0 または独自のバックエンドから返されるエラー。 - `client` — クライアント側の検証 (例: 無効なフォーム入力) によるエラー。 - `developer` — 不正な統合や SDK の誤用によって発生するエラー。
* [useLoginIdentifiers](/ja/docs/libraries/acul/react-sdk/API-Reference/Hooks/useLoginIdentifiers): 認証フローまたはログインプロセスで現在使用されている、有効な識別子タイプ (`'email'`、`'phone'`、`'username'` など) の一覧を返します。
* [useMfaPolling](/ja/docs/libraries/acul/react-sdk/API-Reference/Hooks/useMfaPolling): Auth0 Advanced Customization of Universal Login (ACUL) 画面で、MFA プッシュのポーリング (例: プッシュ通知の承認待ち) を管理する React フック。
* [useResend](/ja/docs/libraries/acul/react-sdk/API-Reference/Hooks/useResend): ACUL 画面で「再送」アクション (例: 確認 code の再送) を管理する React フックです。
* [useSignupIdentifiers](/ja/docs/libraries/acul/react-sdk/API-Reference/Hooks/useSignupIdentifiers): 現在の画面のサインアップ識別子に基づいて、有効な識別子 (メールアドレス、電話番号、またはユーザー名) の一覧を、それぞれの `required` ステータスとともに返します。
* [usePasswordValidation](/ja/docs/libraries/acul/react-sdk/API-Reference/Hooks/usePasswordValidation): 現在の Auth0 パスワードポリシーに照らしてパスワードを検証し、各ルールを満たしているかどうかを示す構造化された結果を返す React フックです。
* [useUsernameValidation](/ja/docs/libraries/acul/react-sdk/API-Reference/Hooks/useUsernameValidation): 現在の Auth0 ユーザー名ポリシーに照らしてユーザー名を検証する React フック。
