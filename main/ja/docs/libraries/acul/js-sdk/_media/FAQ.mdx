<div id="where-can-i-see-errors-related-to-my-screen-and-how-to-handle-them">
  ## 自分の画面に関連するエラーはどこで確認でき、どのように処理できますか
</div>

SDK には、処理中に発生するエラーを検出して確認するための方法が複数用意されています。

1. **画面固有のエラー向けの `getErrors` メソッド**
   最上位の getter 関数である `getErrors` を使用すると、バックエンドの検証に失敗した場合に、詳細な認証エラーを取得できます。
   ```javascript
      import Login from "@auth0/auth0-acul-js/login";

      const loginIdManager = new Login();

      const errors = loginIdManager.getErrors();
      if (errors) {
      console.error("Login failed:", errors);
   }
   ```
2. **画面固有のエラー**
   各画面には `transaction.errors` プロパティがあり、その画面に固有のエラーの一覧を返します。これらのエラーはリクエストの最終検証後に生成されるため、対応が必要な問題を特定するのに役立ちます。

これらの方法を使用することで、SDK フロー全体を通じてエラーを容易に追跡し、解決できます。

<div id="can-i-use-this-sdk-from-a-cdn">
  ## このSDKはCDNから利用できますか？
</div>

現在、このSDKは npm レジストリ経由でインストールする場合のみ利用可能で、CDNから直接利用することはできません。

SDKをインストールするには、次のコマンドを実行します。

```sh
npm install @auth0/auth0-acul-js
```

インストール後は、JavaScript、TypeScript、または ReactJS のプロジェクトにインポートして使用できます。

<div id="can-i-build-a-reactjs-application-with-this-sdk">
  ## この SDK で ReactJS アプリケーションを構築できますか？
</div>

はい。この SDK は ReactJS アプリケーションに簡単に統合できます。

この SDK は Vanilla JavaScript で記述されていますが、React、Angular、Vue など、JavaScript ベースのあらゆるフレームワークと完全に互換性があります。React のコンポーネントやフック内でもシームレスに使用できます。
