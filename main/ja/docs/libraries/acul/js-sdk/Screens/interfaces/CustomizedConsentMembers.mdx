CustomizedConsentMembers

````ts Example
export interface CustomizedConsentMembers extends BaseMembers {
  /**
   * Customized Consent 画面の特定のプロパティとデータにアクセスします。
   * リクエストされている `scopes` と `authorizationDetails` のリストを含みます。
   * @type {ScreenMembersOnCustomizedConsent}
   */
  screen: ScreenMembersOnCustomizedConsent;

  /**
   * リクエストされた権限と認可の詳細を承諾（許可）するユーザーの決定を送信します。
   * このアクションは `action: "accept"` を指定して `/u/customized-consent` エンドポイントにPOSTします。
   * トランザクションの状態は自動的に含まれます。
   *
   * @param {CustomOptions} [payload] - 省略可能。リクエストと共に送信するカスタムキーと値のペアを含むオブジェクト。
   *                                    これらのパラメーターはサーバーに送信されるフォームデータに含まれます。
   * @returns {Promise<void>} 承諾アクションが正常に送信されると解決されるPromise。
   *                          成功した場合、Auth0は通常ユーザーをアプリケーションに戻すか、
   *                          認証フローの次のステップにリダイレクトします。
   * @throws {Error} フォームの送信に失敗した場合（例：ネットワークエラー、無効な状態）にエラーをスローします。
   *                 サーバー側のエラー（"invalid_request" など）は、JavaScriptエラーとしてスローされるのではなく、
   *                 操作後に `this.transaction.errors` に反映されます。
   *
   * @example
   * ```typescript
   * // 'customizedConsentManager' は CustomizedConsent SDK クラスのインスタンスであると仮定します
   * try {
   *   await customizedConsentManager.accept();
   *   // 成功した場合、ページは通常リダイレクトされます。
   * } catch (error) {
   *   // 送信中に発生した予期しないエラーを処理します。
   *   console.error("Failed to submit consent acceptance:", error);
   * }
   * // await の後、サーバー側のバリデーションエラーについては customizedConsentManager.transaction.errors を確認してください。
   * ```
   */
  accept(payload?: CustomOptions): Promise<void>;

  /**
   * リクエストされた権限と認可の詳細を拒否（却下）するユーザーの決定を送信します。
   * このアクションは `action: "deny"` を指定して `/u/customized-consent` エンドポイントにPOSTします。
   * トランザクションの状態は自動的に含まれます。
   *
   * @param {CustomOptions} [payload] - 省略可能。リクエストと共に送信するカスタムキーと値のペアを含むオブジェクト。
   *                                    これらのパラメーターはサーバーに送信されるフォームデータに含まれます。
   * @returns {Promise<void>} 拒否アクションが正常に送信されると解決されるPromise。
   *                          成功した場合、Auth0は通常ユーザーをリダイレクトし、アクセス拒否メッセージを表示するか、
   *                          アプリケーションにエラーを返す場合があります。
   * @throws {Error} フォームの送信に失敗した場合（例：ネットワークエラー、無効な状態）にエラーをスローします。
   *                 サーバー側のエラーは `this.transaction.errors` に反映されます。
   *
   * @example
   * ```typescript
   * // 'customizedConsentManager' は CustomizedConsent SDK クラスのインスタンスであると仮定します
   * try {
   *   await customizedConsentManager.deny({ reason_code: "user_rejected_details" }); // カスタムオプションの例
   *   // 成功した場合、ページは通常リダイレクトされます。
   * } catch (error) {
   *   console.error("Failed to submit consent denial:", error);
   * }
   * // await の後、サーバー側のバリデーションエラーについては customizedConsentManager.transaction.errors を確認してください。
   * ```
   */
  deny(payload?: CustomOptions): Promise<void>;
}
````

<ParamField body="accept" type="Promise<void>">
  要求された権限および認可の詳細を承諾 (付与) するというユーザーの意思を送信します。
  このアクションは、`action: "accept"` を指定して `/u/customized-consent` エンドポイントに送信されます。
  トランザクションの状態は自動的に含まれます。

  accept アクションが正常に送信されると解決される Promise です。
  成功すると、Auth0 は通常、ユーザーをアプリケーションへリダイレクトするか、
  認証フローの次のステップへ進めます。

  #### 例外

  フォーム送信に失敗した場合 (例: ネットワークの問題、無効な状態) にエラーをスローします。
  サーバー側のエラー (&quot;invalid&#95;request&quot; など) は、JavaScript エラーとしてスローされるのではなく、
  操作後に `this.transaction.errors` に反映されます。

  ```typescript Example
  // 'customizedConsentManager' が CustomizedConsent SDK クラスのインスタンスであると仮定します
  try {
    await customizedConsentManager.accept();
    // 成功した場合、通常はページがリダイレクトされます。
  } catch (error) {
    // 送信処理自体で発生した予期しないエラーを処理します。
    console.error("Failed to submit consent acceptance:", error);
  }
  // await の後で、サーバー側の検証に関する問題について customizedConsentManager.transaction.errors を確認します。
  ```

<ParamField body="deny" type="Promise<void>">
  要求された権限および認可の詳細を拒否 (却下) するというユーザーの意思を送信します。
  このアクションは、`action: "deny"` を指定して `/u/customized-consent` エンドポイントに送信されます。
  トランザクションの状態は自動的に含まれます。

  deny アクションが正常に送信されると解決される Promise です。
  成功すると、Auth0 は通常ユーザーをリダイレクトし、アクセス拒否メッセージを表示したり、
  アプリケーションにエラーを返したりすることがあります。

  #### 例外

  フォーム送信に失敗した場合 (例: ネットワークの問題、無効な状態) にエラーをスローします。
  サーバー側のエラーは `this.transaction.errors` に反映されます。

  ```typescript Example

    await customizedConsentManager.deny({ reason_code: "user_rejected_details" }); // カスタムオプションの例
    // 成功した場合、通常はページがリダイレクトされます。
  } catch (error) {
    console.error("Failed to submit consent denial:", error);
  }
  // await の後で、サーバー側の検証に関する問題について customizedConsentManager.transaction.errors を確認します。
  ```
