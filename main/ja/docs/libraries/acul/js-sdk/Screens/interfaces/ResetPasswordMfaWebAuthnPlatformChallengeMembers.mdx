<ParamField body="continueWithPasskey" type="Promise<void>">
  `screen.publicKey` で指定されたチャレンジオプションを使用して
  `navigator.credentials.get()` を呼び出し、WebAuthn プラットフォーム認証器のチャレンジを開始します。
  成功した場合は、生成された認証情報アサーションを `action: "default"` とともに Auth0 に送信します。

  検証の試行が送信されると解決される Promise です。
  通常、操作が成功するとリダイレクトされます。

  #### 例外

  `screen.publicKey` がない場合、
  `navigator.credentials.get()` が失敗した場合 (例: ユーザーによるキャンセル `NotAllowedError`) 、
  または Auth0 へのフォーム送信に失敗した場合にエラーを送出します。`navigator.credentials.get()` が
  `DOMException` で失敗した場合は、そのエラーを catch して `reportBrowserError` を呼び出すことを推奨します。

  ```typescript Example
  // 'sdk' が ResetPasswordMfaWebAuthnPlatformChallenge のインスタンスであると仮定します
  try {
    await sdk.continueWithPasskey({
      rememberDevice: true // ユーザーがチェックボックスをオンにしていて、sdk.screen.showRememberDevice が true の場合
    });
    // 成功すると、Auth0 がリダイレクトを処理します。
  } catch (error) {
    console.error("Platform authenticator verification failed:", error);
    // WebAuthn API エラー（DOMException）の場合は報告します
    if (error instanceof DOMException && error.name && error.message) {
      await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
    }
    // ページが再読み込みされた場合は、サーバー側の検証メッセージについて sdk.transaction.errors を確認します。
  }
  ```

<ParamField body="reportBrowserError" type="Promise<void>">
  `navigator.credentials.get()` API 呼び出し中に発生したブラウザー側のエラーを報告します。
  これは、ユーザーによるキャンセル、タイムアウト、サポートされていないハードウェアなどの問題を Auth0 に通知するために使用されます。
  エラーの詳細を `action: "showError::{errorDetailsJsonString}"` とともに送信します。

  エラーレポートが送信されると解決される Promise です。

  #### 例外

  フォーム送信に失敗した場合 (例: ネットワークエラー) 。

  ```typescript Example
  // 'sdk' が ResetPasswordMfaWebAuthnPlatformChallenge のインスタンスであり、
  // 'webAuthnError' が navigator.credentials.get() 呼び出し失敗時の DOMException であると仮定します。
  try {
    await sdk.reportBrowserError({
      error: { name: webAuthnError.name, message: webAuthnError.message }
    });
    // エラーの内容に応じて、Auth0 はページを再描画またはリダイレクトすることがあります。
  } catch (submitError) {
    console.error("Failed to report WebAuthn browser error:", submitError);
  }
  ```

<ParamField body="tryAnotherMethod" type="Promise<void>">
  ユーザーが WebAuthn のプラットフォームチャレンジを使用せず、別の MFA 方法を選択できるようにします。
  このアクションは `action: "pick-authenticator"` を Auth0 に送信し、
  パスワードリセットフローの MFA 要素選択画面へユーザーを遷移させます。

  `'pick-authenticator'` アクションの送信時に解決される Promise です。

  #### 例外

  フォームの送信に失敗した場合。

  ```typescript Example
  // 'sdk' は ResetPasswordMfaWebAuthnPlatformChallenge のインスタンスであるとします
  try {
    await sdk.tryAnotherMethod();
    // 成功すると、Auth0 が MFA 選択画面へのリダイレクトを処理します。
  } catch (error) {
    console.error("Failed to switch MFA method:", error);
  }
  ```
