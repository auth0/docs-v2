---
title: "registerPasskeyAutofill"
---

> **registerPasskeyAutofill**(`inputId`?: `string`): `Promise`&lt;`void`&gt;

`mediation: "conditional"` を指定した `navigator.credentials.get()` を使用して、パッシブな WebAuthn 認証情報リクエストを初期化します。サポートされている場合、ブラウザーは保存済みのパスキーを username フィールドのオートコンプリートのドロップダウンに直接表示できます。パスキーを選択すると、追加のユーザー操作なしで認証が完了します。

ログイン画面の初期化時 (たとえばページの読み込み時) に、これを 1 回呼び出します。ブラウザーが Conditional Mediation をサポートしていない場合、このメソッドは安全に何も行いません。

<div id="parameters">
  ## パラメーター
</div>

<ParamField body="inputId?" type="string">
  username 用 `<input>` 要素の ID (`#` なし) 。例: `"username"`。指定した場合、SDK はその要素が存在することを検証し、`<input>` 要素であることを確認したうえで、`autocomplete` 属性を `"username webauthn"` に上書きします。

  省略した場合は、`<input>` 要素を手動で設定する必要があります。

  ```html
  <input id="username" autocomplete="username webauthn" />
  ```
</ParamField>

<div id="returns">
  ## 戻り値
</div>

`Promise`&lt;`void`&gt;

Conditional UI の登録が完了すると、この Promise は resolve されます。

<div id="example">
  ## 例
</div>

```typescript
import LoginId from '@auth0/auth0-acul-js/login-id';

async function initializeLogin() {
  const loginId = new LoginId();
  // 関連するHTML inputが存在することを確認してください:
  // <input id="username" autocomplete="username webauthn" />
  await loginId.registerPasskeyAutofill('username');
}

initializeLogin().catch(console.error);
```

<div id="gotchas">
  ## 注意点
</div>

* `autocomplete` 属性には、**必ず完全に一致する形で** `"username webauthn"` を指定する必要があります。`"email"` や `"text"` などの無関係なトークンを含めると、ブラウザーでパスキーのドロップダウンが表示されなくなります。
* この属性を上書きするのは意図的なものであり、ブラウザー間で一貫した動作を確保するために必要です。既存の autocomplete 値をマージまたは拡張する前提にしないでください。
* ブラウザーが Conditional Mediation をサポートしていない場合、SDK は安全に何も行いません。

<div id="remarks">
  ## 備考
</div>

このメソッドは内部の `registerPasskeyAutofill()` ユーティリティに委譲され、リクエストの存続期間を管理するためのバックグラウンド `AbortController` を返します。ページのライフサイクルごとに、呼び出しは 1 回のみにしてください。
