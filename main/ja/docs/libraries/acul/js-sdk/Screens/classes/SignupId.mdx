SignupId クラスは、`signup-id` 画面の機能を実装します。この画面では、ユーザーの識別子を入力します。

  <img style={{maxHeight:"400px"}} src="/docs/images/cdy7uua7fh8z/5lzGYj0k4VcxvPCBYH37SO/ebd291cc0dbd673a70f93530f724bf65/Signup_ID.png" alt="Signup ID" />
</Frame>

<div id="constructors">
  ## コンストラクター
</div>

SignupId 画面マネージャーのインスタンスを作成します:

```typescript Example
import SignupId from '@auth0/auth0-acul-js/signup-id';
const signupIdManager = new SignupId();
signupIdManager.signup({
  email: 'test@example.com',
});
```

<div id="methods">
  ## メソッド
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  このメソッドは、Universal Login ページの表示言語を変更します。

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.changeLanguage({
    language: 'fr',
  });
  ```

  **メソッドパラメーター**

  <Expandable title="パラメーター">
    <ParamField body="options">
      [LanguageChangeOptions](/ja/docs/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions)。
    </ParamField>

    <ParamField body="language" type="string" required>
      変更先の言語のロケールコード (例: `'en'`、`'fr'`、`'es'`) 。
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      `'session'` に設定すると、選択した言語はセッション中保持されます。
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      `ulp-` で始まる追加のカスタムフィールド (例: `'ulp-custom-field'`) 。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="federatedSignup" type="Promise<void>">
  このメソッドは、Google や Facebook などの各種ソーシャル ID を使用したサインアップを処理します。

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.federatedSignup({
    connection: 'google-oauth2',
  });
  ```

  **メソッドパラメーター**

  <Expandable title="パラメーター">
    <ParamField body="options">
      [FederatedSignupOptions](/ja/docs/libraries/acul/js-sdk/Screens/interfaces/FederatedSignupOptions)。
    </ParamField>

    <ParamField body="connection" type="string" required>
      使用するソーシャル接続の名前。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/ja/docs/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  このメソッドは、コンテキストからトランザクションエラーの配列を取得します。エラーが存在しない場合は空の配列を返します。戻り値は、トランザクションコンテキスト内のエラーオブジェクトの配列です。
</ParamField>

<ParamField body="getSignupIdentifiers" type={<span><a href="/ja/docs/libraries/acul/js-sdk/Screens/interfaces/Identifier">Identifier</a>[] | null</span>}>
  このメソッドは、`signup-id` フォームで有効な識別子の一覧を返します。各識別子には、トランザクション設定に基づいて必須か任意かが示されます。戻り値は識別子オブジェクトの配列です (例: email、phone、username) 。

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  const identifiers = signupIdManager.getSignupIdentifiers();
  // [{ type: 'email', required: true }, { type: 'username', required: false }]
  ```
</ParamField>

<ParamField body="pickCountryCode" type="Promise<void>">
  このメソッドはユーザーを国番号の選択リストにリダイレクトし、そこで電話番号の国番号プレフィックスを更新できるようにします。

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.pickCountryCode();
  ```
</ParamField>

<ParamField body="signup" type="Promise<void>">
  このメソッドは `signup-id` に関連する設定に基づいて処理を行います。さまざまな識別子を使用して新しいユーザーをサインアップできます。

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  signupIdManager.signup({
    email: 'test@example.com',
  });
  ```

  **メソッドパラメーター**

  <Expandable title="パラメーター">
    <ParamField body="options">
      [SignupOptions](/ja/docs/libraries/acul/js-sdk/Screens/interfaces/SignupOptions)。
    </ParamField>

    <ParamField body="email?" type="string">
      ユーザーのメールアドレス。
    </ParamField>

    <ParamField body="username?" type="string">
      ユーザーの識別子。
    </ParamField>

    <ParamField body="phone?" type="string">
      ユーザーの電話番号。
    </ParamField>

    <ParamField body="captcha?" type="string">
      キャプチャの code、またはキャプチャプロバイダーからのレスポンスです。このプロパティは、Auth0 テナントで [Bot Detection](/ja/docs/secure/attack-protection/bot-detection) が有効になっている場合に必須です。
    </ParamField>

    <ParamField body="[`key`: `string`]" type="&#x22;string&#x22; | &#x22;number&#x22; | &#x22;boolean&#x22; | &#x22;undefined&#x22;">
      `ulp-` で始まる追加のカスタムフィールド (例: `'ulp-custom-field'`) 。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="validateUsername" type={<span><a href="/ja/docs/libraries/acul/js-sdk/Screens/interfaces/UsernameValidationResult">UsernameValidationResult</a></span>}>
  このメソッドは、指定された username を、トランザクションコンテキストで定義されている現在の username ポリシーに照らして検証します。username が有効かどうかと、その理由を示すオブジェクトを返します。

  ```typescript Example
  import SignupId from '@auth0/auth0-acul-js/signup-id';
  const signupIdManager = new SignupId();
  const result = signupIdManager.validateUsername('myusername');
  // result => { valid: true, errors: [] }
  ```

  **メソッドパラメーター**

  <Expandable title="パラメーター">
    <ParamField body="username" type="string" required>
      検証する username 文字列。
    </ParamField>
  </Expandable>
</ParamField>
