この記事では、シングルページアプリ (SPA) を [auth0.js](/ja/docs/libraries/auth0js) から [auth0-spa-js](/ja/docs/libraries/auth0-single-page-app-sdk) に移行する方法を説明します。以下に、auth0.js を使用する各シナリオと、それに対応する auth0-spa-js のコードを示します。

<div id="functionality-that-cannot-be-migrated">
  ## 移行できない機能
</div>

auth0.js のすべての機能を auth0-spa-js に直接移行できるわけではありません。直接移行できないシナリオには、次のものがあります。

* 埋め込みの[username/password ログイン](https://auth0.github.io/auth0.js/WebAuth.html#login)および埋め込みの[パスワードレスログイン](https://auth0.github.io/auth0.js/WebAuth.html#passwordlessLogin)
* ユーザーの[サインアップ](https://auth0.github.io/auth0.js/WebAuth.html#signup)
* [/userinfo エンドポイントからユーザープロファイルを取得する](https://auth0.github.io/auth0.js/Authentication.html#userInfo)
* [ユーザーのパスワード変更用メールをリクエストする](https://auth0.github.io/auth0.js/WebAuth.html#changePassword)
* [Management API を使用してユーザーをリンクする](https://auth0.github.io/auth0.js/Management.html#linkUser)
* [Management API を使用してユーザーを取得する](https://auth0.github.io/auth0.js/Management.html#getUser)
* [Management API を使用してユーザー属性を更新する](https://auth0.github.io/auth0.js/Management.html#patchUserAttributes)
* [Management API を使用してユーザーメタデータを更新する](https://auth0.github.io/auth0.js/Management.html#patchUserMetadata)
* また、Auth0.js で設定できても、auth0-spa-js には対応するものがないオプションもあります。その一例が `responseType` です。各オプションに 1 対 1 で直接対応していないのには理由があります。この場合、SDK は SPA でのみ使用するため、レスポンスタイプを変更する必要がなく、`responseType` は不要です。

<div id="authentication-parameters-are-not-modified-anymore">
  ## Authentication Parameters は変更されなくなりました
</div>

Auth0.js は、設定したカスタムパラメーターを内部で `camelCase` から `snake_case` に変換します。たとえば、`deviceType: 'offline'` を設定すると、auth0.js は実際にはサーバーに `device_type: 'offline'` を送信します。

auth0-spa-js を使用する場合、カスタムパラメーターは `camelCase` から `snake_case` に**変換されません**。送信したパラメーターが、そのまま <Tooltip tip="認可サーバー: ユーザーのアクセス境界の定義に関与する集中型サーバーです。たとえば、認可サーバーは、ユーザーが利用できるデータ、タスク、機能を制御できます。" cta="用語集を表示" href="/ja/docs/glossary?term=authorization+server">認可サーバー</Tooltip> に渡されます。

<div id="parse-the-hash-after-the-redirect">

window.addEventListener('load', () => {
  auth0.parseHash({ hash: window.location.hash }, function(err, authResult) {

<div id="get-the-user-information">
  ## ユーザー情報を取得する
</div>

<div id="auth0js">
  ### auth0.js
</div>

`userInfo()` 関数は、[`/userinfo` エンドポイント](https://auth0.com/docs/api/authentication#user-profile)を呼び出して、ユーザーのプロファイルを返します。

* [userInfo()](https://auth0.github.io/auth0.js/Authentication.html#userInfo)

```jsx lines
window.addEventListener('load', () => {
  auth0.client.userInfo(accessToken, function(err, user) {
    console.log(user)
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

auth0.js とは異なり、Auth0 SPA SDK はユーザープロファイルを取得するために [`/userinfo` エンドポイント](https://auth0.com/docs/api/authentication#user-profile) を呼び出しません。代わりに、`Auth0Client.getUser()` はデコード済みの `id_token` からユーザー情報を返します。

* [Auth0Client.getUser()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getUser)

```jsx lines
window.addEventListener('load', async () => {
  const user = await auth0.getUser();
  console.log(user);
});
```

<div id="refresh-tokens">
  ## リフレッシュトークン
</div>

<div id="auth0js">
  ### auth0.js
</div>

* [checkSession()](https://auth0.github.io/auth0.js/WebAuth.html#checkSession)

```jsx lines
document.getElementById('login').addEventListener('click', () => {
  auth0.checkSession({}, function(err, authResult) {
    // 認証トークンまたはエラー
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

Auth0 SPA SDK は、<Tooltip tip="アクセストークン: API へのアクセスに使用される、不透明な文字列または JWT 形式の認可資格情報。" cta="用語集を表示" href="/ja/docs/glossary?term=Access+Token">アクセストークン</Tooltip> の更新を自動的に処理します。`getTokenSilently` を呼び出すたびに、有効なアクセストークンが返されるか、Auth0 にセッションが存在しない場合はエラーが返されます。

* [Auth0Client.getTokenSilently()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenSilently)

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenSilently();
});
```

<div id="get-a-token-for-a-different-audience-or-with-more-scopes">
  ## 別のオーディエンス向けのトークン、または追加のスコープを持つトークンを取得する
</div>

<div id="auth0js">
  ### auth0.js
</div>

* [checkSession()](https://auth0.github.io/auth0.js/WebAuth.html#checkSession)

```jsx lines
document.getElementById('login').addEventListener('click', () => {
  auth0.checkSession({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  }, function(err, authResult) {
    // 認証トークンまたはエラー
  });
});
```

<div id="auth0-spa-js">
  ### auth0-spa-js
</div>

Auth0 SPA SDK では、アクセストークンの更新が自動的に処理されます。`getTokenSilently` を呼び出すたびに、有効なアクセストークンが返されるか、Auth0 にセッションが存在しない場合はエラーが返されます。

* [Auth0Client.getTokenSilently()](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenSilently)
* [GetTokenSilentlyOptions](https://auth0.github.io/auth0-spa-js/interfaces/GetTokenSilentlyOptions.html)

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenSilently({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  });
});
```

ポップアップを開き、ユーザーが新しい API への同意を行えるようにするには、[getTokenWithPopup](https://auth0.github.io/auth0-spa-js/classes/Auth0Client.html#getTokenWithPopup) を使用します。

```jsx lines
document.getElementById('login').addEventListener('click', async () => {
  await auth0.getTokenWithPopup({
    audience: 'https://mydomain/another-api/',
    scope: 'read:messages'
  });
});
```
