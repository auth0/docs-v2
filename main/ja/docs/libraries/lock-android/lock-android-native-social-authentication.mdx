<div id="native-provider-google">
  ## ネイティブプロバイダー - Google
</div>

**Lock** の有無にかかわらず、Google AuthProvider を使用してログインできます。[セットアップ](#setup) セクションの手順に必ず従ってください。

[Lock-Google.Android](https://github.com/okta-graveyard/Lock-Google.Android)
を使用するには、Android API 15 以降と Google Play Services 10.+ が必要です。

<div id="latest-version-of-lock-google">
  ### Lock-Google の最新バージョン
</div>

Lock-Google は [Maven Central](https://central.sonatype.com/) で提供されています。インストールするには、`build.gradle` に次の行を追加するだけです。

`compile 'com.auth0.android:lock-google:1.+'`

最新バージョンは、リポジトリの [README](https://github.com/okta-graveyard/Lock-Google.Android#install) または [Maven](https://central.sonatype.com/artifact/com.auth0.android/lock-google) で確認できます。

<div id="lock-google-setup">
  ### Lock-Google の設定
</div>

<div id="google-cloud-console">
  #### Google Cloud Console
</div>

1. [Google Cloud Console](https://console.cloud.google.com) に移動し、新しいプロジェクトを作成します。
2. [OAuth Consent Screen](https://console.cloud.google.com/apis/credentials/consent) を設定し、少なくとも有効なメールアドレスと名前を入力します。
3. 左側のナビゲーション ドロワーで、[Credentials](https://console.cloud.google.com/apis/credentials) をクリックします。
4. [Create Credentials](https://console.cloud.google.com/apis/credentials/oauthclient) ボタンをクリックし、**OAuth client ID** を選択して新しい認証情報を作成します。次に、**Web Application** を選択し、&quot;Auth0 Server Google-OAuth&quot; などの名前を付けます。**Authorized redirect URIs** には、`https://{yourDomain}/login/callback` のようなコールバック URL を入力します。フィールドを離れる前に必ず ENTER キーを押してから、Create ボタンをクリックしてください。後で使用するため、`CLIENT ID` と `CLIENT SECRET` の値を控えておきます。
5. [Create Credentials](https://console.cloud.google.com/apis/credentials/oauthclient) ボタンをクリックし、**OAuth client ID** を選択して新しい認証情報を作成します。次に、**Android** を選択し、&quot;Auth0 Android Google-OAuth&quot; などの名前を付けます。アプリケーションの署名に使用している証明書の **SHA-1** を取得し、最初のフィールドに入力します。SHA-1 の取得方法が必要な場合は、[こちら](#certificate-fingerprints) のセクションを参照してください。最後に、Android アプリケーションの **Package Name** を最後のフィールドに入力し、Create ボタンをクリックします。後で使用するため、`CLIENT ID` の値を控えておきます。

<div id="auth0-dashboard">
  #### Auth0 Dashboard
</div>

1. Auth0 Dashboard に移動し、[Social Connections](https://manage.auth0.com/#/connections/social) をクリックします。
2. **Google** をクリックすると、ダイアログが表示されます。
3. 上記の **Google Cloud Console** セクションの手順 4 で取得した `CLIENT ID` の値を「クライアントID」フィールドに入力します。
4. 上記の **Google Cloud Console** セクションの手順 4 で取得した `CLIENT SECRET` の値を「クライアントシークレット」フィールドに入力します。
5. 上記の **Google Cloud Console** セクションの手順 5 で取得した `CLIENT ID` を「Allowed Mobile Client IDs」フィールドに入力します。
6. **Save** ボタンをクリックします。
7. Auth0 Dashboard に移動し、[Applications](https://manage.auth0.com/#/applications) をクリックします。まだ作成していない場合は、先に作成してアプリケーションの設定ページを開きます。
8. ページの下部にある「Show Advanced Settings」リンクをクリックし、「Device Settings」タブを開きます。
9. Android セクションで、**Package Name** にアプリケーションのパッケージ名を入力します。次に、**Key Hashes** フィールドに、アプリケーションの署名に使用している証明書の SHA-256 を入力します。SHA-256 の取得方法については、[こちら](#certificate-fingerprints)のセクションを参照してください。「Save Changes」ボタンをクリックします。

<div id="android-application">
  #### Android アプリケーション
</div>

1. Android アプリケーションの `res/strings.xml` ファイルに、新しい文字列リソースを作成します。名前は `google_server_client_id` とし、値には上記の **Google Cloud Console** セットアップセクションの手順 5 で取得した `CLIENT_ID` を設定します。
2. `AndroidManifest.xml` ファイルの `application` タグ内に、Google Play Services のバージョンのメタデータを追加します。

   ```xml lines
   <meta-data
       android:name="com.google.android.gms.version"
       android:value="@integer/google_play_services_version" />
   ```
3. `AndroidManifest.xml` ファイルに、Android のインターネット権限を追加します。

   ```xml lines
   <uses-permission android:name="android.permission.INTERNET" />
   ```
4. `GoogleAuthProvider` の新しいインスタンスを作成する際は、先ほど取得した `google_server_client_id` の値を最初のパラメーターとして渡します。

   ```kotlin lines
   public class MainActivity extends AppCompatActivity {
     private GoogleAuthProvider provider;
     // ...

     @Override
     protected void onCreate(Bundle savedInstanceState) {
       Auth0 auth0 = new Auth0(getString(R.string.com_auth0_client_id), getString(R.string.com_auth0_domain));
       auth0.setOIDCConformant(true);
       AuthenticationAPIClient client = new AuthenticationAPIClient(auth0);
       provider = new GoogleAuthProvider(getString(R.string.google_server_client_id), client);
     }

     // ...
   }
   ```

Google SDK のセットアップについてさらにサポートが必要な場合は、Google の [Android 向け Sign-In ガイド](https://developers.google.com/identity/android-credential-manager)を参照してください。

<div id="usage-without-lock">
  ### Lock を使用しない場合
</div>

このプロバイダーを **Lock** を使わずに使用する場合は、必ず `onActivityResult()` メソッドをオーバーライドし、その呼び出しをプロバイダーのインスタンスに渡してください。最後に、認証プロセスを開始するには `start` を呼び出します。

```kotlin lines
// 独自のリクエストコードを定義する
private static final int RC_PERMISSIONS = 101;
private static final int RC_AUTHENTICATION = 102;

@Override
protected void onActivityResult(int requestCode, int resultCode, Intent data) {
    if (provider.authorize(requestCode, resultCode, data)) {
        return;
    }
    super.onActivityResult(requestCode, resultCode, data);
}

private void beginAuthentication(){
  provider.start(this, callback, RC_PERMISSIONS, RC_AUTHENTICATION);
}
```

これで完了です。アプリケーションを実行し、Googleネイティブプロバイダーを使用してログインできます！！

<div id="additional-options">
  ### その他のオプション
</div>

<div id="using-a-custom-connection-name">
  #### カスタム接続名を使用する
</div>

Auth0 での認可にカスタムのソーシャル接続名を使用するには、2 番目のコンストラクターを使って GoogleAuthProvider インスタンスを作成します。

```kotlin wrap lines
GoogleAuthProvider provider = new GoogleAuthProvider("my-connection", "google-server-client-id", client);
```

<div id="send-additional-authentication-parameters">
  #### 追加の認証パラメーターを送信する
</div>

認証リクエストで追加のパラメーターを送信するには、`setParameters` を使用します。

```kotlin lines
Map<String, Object> parameters = new HashMap<>();
//エントリを追加
provider.setParameters(parameters);
```

<div id="requesting-a-custom-google-scope">
  #### カスタム Google スコープをリクエストする
</div>

デフォルトでは、スコープ `Scopes.PLUS_LOGIN` がリクエストされます。`setScopes` に Scopes のリストを渡して呼び出すことで、スコープをカスタマイズできます。Google API (Auth、Drive、Plus..) ごとに、独自の Google スコープのリストが定義されています。

```kotlin wrap lines
provider.setScopes(Arrays.asList(new Scope(Scopes.PLUS_ME), new Scope(Scopes.PLUS_LOGIN)));
```

#### カスタム Android ランタイム権限のリクエスト

このプロバイダーでユーザーを認証する際に、特別な Android Manifest 権限は必要ありません。ただし、ユースケース上それらが必要な場合は、AuthProvider に処理を任せることができます。`setRequiredPermissions` メソッドを使用してください。

```kotlin wrap lines
provider.setRequiredPermissions(new String[]{"android.permission.GET_ACCOUNTS"});
```

Lock を使用していない場合は、権限リクエストの結果を自分で処理する必要があります。その場合は、アクティビティに `ActivityCompat.OnRequestPermissionsResultCallback` インターフェースを実装してください。`onRequestPermissionsResult` メソッドが呼び出されたら、`provider.onRequestPermissionsResult` を呼び出して、その結果をプロバイダーに渡します。

<div id="log-out-clear-account">
  #### ログアウト / アカウント情報をクリア。
</div>

ユーザーをログアウトし、次回は資格情報の入力を求めるようにするには、`clearSession` を呼び出します。これを行うと、プロバイダーの state は無効になり、結果を `authorize` しようとする前に、再度 `start` を呼び出す必要があります。`stop` を呼び出しても同じ効果があります。

`provider.clearSession();`

<div id="remember-the-last-login">
  #### 最後に使用したログインを記憶する
</div>

デフォルトでは、このプロバイダーは最後にログインに使用したアカウントを記憶します。この動作を変更するには、次のメソッドを使用します。

`provider.rememberLastLogin(false);`

<div id="certificate-fingerprints">
  #### 証明書のフィンガープリント
</div>

Google Cloud Console で新しい <Tooltip tip="OAuth 2.0: 認可プロトコルとワークフローを定義する認可フレームワーク。" cta="用語集を表示" href="/ja/docs/glossary?term=OAuth">OAuth</Tooltip> Credential を作成する際は、アプリケーションの署名に使用する証明書の SHA-1 を指定する必要があります。<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要な製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> でアプリケーションの設定を完了する際には、SHA-256 の値も指定する必要があります。以下に、その値を取得するためのターミナルコマンドの例と、出力例を示します。

コマンド:

`keytool -exportcert -alias androiddebugkey -keystore <PATH_TO_YOUR_KEYSTORE> -storepass android | openssl sha1 -binary | openssl base64`

出力例:

`no71633JAC3qgzQYCbskprUr55k=`

サポートが必要な場合は、この [Keystores Guide](/ja/docs/libraries/auth0-android/android-development-keystores-hashes) を参照してこれらの値を取得できます。

**Lock** の有無にかかわらず、Facebook AuthProvider を使用してログインできます。[セットアップ](#setup) セクションの手順に必ず従ってください。

[Lock-Facebook](https://github.com/okta-graveyard/Lock-Facebook.Android) には、Android API 15 以降と Facebook Android SDK 4 以降が必要です。

Lock-Facebook は [Maven Central](https://central.sonatype.com/) で提供されています。インストールするには、`build.gradle` に次の行を追加します。

最新バージョンは、リポジトリの [Readme](https://github.com/okta-graveyard/Lock-Facebook.Android#install) または [Maven](https://central.sonatype.com/artifact/com.auth0.android/lock-facebook) で確認できます。

<div id="lock-facebook-setup">
  ### Lock-Facebook の設定
</div>

<div id="facebook-developers-console">
  #### Facebook Developers Console
</div>

1. [Facebook Developers Console](https://developers.facebook.com/) に移動して、新しいアプリを作成します。「Android」を選択し、有効な名前を付けて、[Create new Facebook App ID] をクリックします。
2. アプリケーションの **Package Name** と、プロバイダーを使用する **Activity class** の名前を入力し、[Next] ボタンをクリックします。
3. アプリケーションの署名に使用する証明書の **SHA-1** Base64 エンコード済み Key Hashes を追加し、[Next] ボタンをクリックします。SHA-1 の取得方法については、[こちら](#certificate-fingerprints) のセクションを参照してください。
4. 最後に、ページ上部までスクロールして、[Skip Quickstart] ボタンをクリックし、Facebook アプリのページに移動します。
5. ページ上部に `APP ID` と `APP SECRET` の値があります。後で必要になるので、保存しておいてください。
6. 左側のナビゲーションドロワーで [Settings] をクリックし、次に [Basic] をクリックします。**Single Sign-On** スイッチを ON にして、[Save] ボタンをクリックします。
7. [Settings] をクリックし、次に [Advanced] をクリックします。**Native or desktop app?** スイッチを ON にします。

<div id="auth0-dashboard">
  #### Auth0 Dashboard
</div>

1. Auth0 Dashboard に移動し、[Social Connections](https://manage.auth0.com/#/connections/social) をクリックします。
2. **Facebook** をクリックすると、ダイアログが表示されます。
3. 上記の **Facebook Developers Console** セクションの手順 5 で取得した `APP ID` の値を &quot;App ID&quot; フィールドに入力します。
4. 上記の **Facebook Developers Console** セクションの手順 5 で取得した `APP SECRET` の値を &quot;App Secret&quot; フィールドに入力します。
5. &quot;Save&quot; ボタンをクリックします。
6. Auth0 Dashboard に移動し、[Applications](https://manage.auth0.com/#/applications) をクリックします。まだ作成していない場合は、先に作成してからアプリケーションの設定ページを開きます。
7. ページ下部の &quot;Show Advanced Settings&quot; リンクをクリックし、&quot;Device Settings&quot; タブに移動します。
8. Android セクションで、**Package Name** にアプリケーションのパッケージ名を入力します。最後に、**Key Hashes** フィールドに、アプリケーションの署名に使用している証明書の SHA-256 を入力します。SHA-256 の取得方法については、[こちら](#certificate-fingerprints)のセクションを参照してください。&quot;Save Changes&quot; ボタンをクリックします。

<div id="android-application">
  #### Android アプリケーション
</div>

1. Android アプリケーションの `res/strings.xml` ファイルに、新しい文字列リソースを作成します。名前は `facebook_app_id` とし、値には上記の **Facebook Developers Console** 設定セクションの手順 5 で取得した `APP ID` を設定します。
2. `AndroidManifest.xml` ファイルの `application` タグ内に、`FacebookActivity` と `facebook_app_id` のメタデータを追加します。

   ```xml lines
   <activity
       android:name="com.facebook.FacebookActivity"
       android:configChanges="keyboard|keyboardHidden|screenLayout|screenSize|orientation"
       android:label="@string/app_name"/>
   <meta-data
       android:name="com.facebook.sdk.ApplicationId"
       android:value="@string/facebook_app_id" />
   ```
3. `AndroidManifest.xml` ファイルに Internet 権限を追加します。

   ```xml lines
   <uses-permission android:name="android.permission.INTERNET" />
   ```
4. `FacebookAuthProvider` の新しいインスタンスを作成します。

   ```kotlin lines
   public class MainActivity extends AppCompatActivity {
     private FacebookAuthProvider provider;
     // ...

     @Override
     protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       Auth0 auth0 = new Auth0(getString(R.string.com_auth0_client_id), getString(R.string.com_auth0_domain));
       auth0.setOIDCConformant(true);
       AuthenticationAPIClient client = new AuthenticationAPIClient(auth0);
       provider = new FacebookAuthProvider(client);
     }

     // ...
   }
   ```

Facebook SDK の設定についてさらに詳しくは、Facebook の[スタートガイド](https://developers.facebook.com/docs/android/getting-started)を参照してください。

### 証明書のフィンガープリント

Facebook Developers Console で新しい OAuth Credential を作成する際は、アプリケーションの署名に使用する証明書の SHA-1 を指定する必要があります。さらに、Auth0 Dashboard でアプリケーションの設定を完了する際には、SHA-256 の値も指定する必要があります。以下に、この値を取得するためのターミナルコマンドの例と、その出力例を示します。

コマンド:

`keytool -exportcert -alias androiddebugkey -keystore <PATH_TO_YOUR_KEYSTORE> -storepass android | openssl sha1 -binary | openssl base64`

出力例:

SHA1: BB:0D:AC:74:D3:21:E1:43:07:71:9B:62:90:AF:A1:66:6E:44:5D:75
SHA256: 15:B9:F9:33:9F:E4:E3:68:C2:10:49:17:5D:A8:77:12:7C:8E:57:E9:FF:B7:23:EA:CC:DD:56:08:06:C9:5E:33

サポートが必要な場合は、これらの値を取得する方法について [Keystores Guide](/ja/docs/libraries/auth0-android/android-development-keystores-hashes) を参照してください。
