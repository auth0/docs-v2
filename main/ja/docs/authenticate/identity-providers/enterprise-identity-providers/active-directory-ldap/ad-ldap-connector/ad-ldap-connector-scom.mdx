description: System Center Operations Manager (SCOM) を使用して AD/LDAP Connector を監視する方法を説明します。
title: System Center Operations Manager で AD/LDAP Connector を監視する
---

Auth0 AD/LDAP Connector は、Windows ベースのマシン上でサービスとして実行できます。

サービスの状態は、他のサービスと同様に System Center を使用して監視できます。

1. **Add Monitoring Wizard** を開き、**Monitoring Type: Windows Service** を選択します。

   <Frame>![AD/LDAP Connector 監視ウィザード画面 1](/docs/images/cdy7uua7fh8z/6234mP3Hx1g3Iwf7y9mKH9/7a2ac217f7d22070c20a769939461f75/ss-2014-12-11T22-48-51.png)</Frame>
2. 名前と説明を入力します。

   <Frame>![AD/LDAP Connector 監視ウィザード画面 2](/docs/images/cdy7uua7fh8z/2KZkQoBuubAgAEWBPnkJx0/abdcdf5e1e0fad20829b255401db523f/ss-2014-12-11T22-49-57.png)</Frame>
3. AD/LDAP Connector がインストールされているサーバーを選択し、**Auth0 ADLDAP** を選択します。

   <Frame>![AD/LDAP Connector 監視ウィザード画面 3](/docs/images/cdy7uua7fh8z/DOQHiJYYiJKZrWzHYUIiB/21eae1ad8435a0754c6e221d5e33bab5/ss-2014-12-11T22-50-37.png)</Frame>
4. **CPU** と **Memory limits** のしきい値を設定します。たとえば、CPU 使用率 10% と RAM 200MB は、アラートを発生させるしきい値として適切です。

また、エンドツーエンドの認証を監視するために、合成トランザクションを設定することも推奨します。

詳しくは、[SCOM を使用した Auth0 の監視](/ja/docs/deploy-monitor/monitor/monitor-using-scom) を参照してください。
