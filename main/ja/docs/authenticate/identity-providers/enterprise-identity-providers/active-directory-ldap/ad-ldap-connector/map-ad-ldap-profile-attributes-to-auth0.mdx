description: Connector Admin Console の Profile Mapper を使用して、AD/LDAP のプロフィール属性を Auth0 ユーザープロフィールの属性にマッピングする方法を説明します。
title: AD/LDAP プロフィール属性を Auth0 ユーザープロフィールにマッピングする
---

AD/LDAP の属性から Auth0 ユーザープロフィールの属性へのマッピングは変更できます。

1. ブラウザーで `http://localhost:8357` にアクセスし、AD/LDAP Connector サーバー上の **Connector Admin Console** を起動します。
2. **Profile Mapper** をクリックして、ソースディレクトリーサービス (`raw_data` 変数で表されます) の属性を、Auth0 ユーザープロフィールの生成に使用される戻り値の変数にマッピングするコードエディター画面を開きます。
   関数の最初の部分では、`profile` という変数を生成し、Auth0 ユーザープロフィールの基本部分のマッピングを定義します。追加の属性は、その下に次の形式の構文で設定できます。
   `profile['department'] = raw_data['companydept'];`
   この例では、`department` は Auth0 ユーザープロフィール内の属性の名前であり、`companydept` はソースディレクトリーサービス (AD など) 内の属性の名前です。
3. LDAP 設定の変更が完了したら、**Save** をクリックします。
4. Connector によるテストの実行後、すべてのテストが緑色になっていることを確認します。

| テスト    | 説明                                                                   | トラブルシューティング                                 |
| ------ | -------------------------------------------------------------------- | ------------------------------------------- |
| Test 1 | 指定された LDAP サーバーおよびポートへの TCP 接続の確立を試みます。                              | 接続を妨げる可能性のある基本的なネットワーク接続とファイアウォールの設定を確認します。 |
| Test 2 | 指定された LDAP サーバーおよびポートに対して、指定された username とパスワードを使用した LDAP バインドを試みます。 | LDAP 接続文字列、検索パス、username、パスワードを確認します。       |
| Test 3 | ディレクトリーに対して LDAP 検索を実行し、指定された username の権限を確認します。                    | 対象ディレクトリー内の username の権限を確認します。             |
| Test 4 | Auth0 サーバーへの接続の確立を試みます。                                              | 接続を妨げる可能性のあるネットワーク接続とファイアウォールの設定を確認します。     |
