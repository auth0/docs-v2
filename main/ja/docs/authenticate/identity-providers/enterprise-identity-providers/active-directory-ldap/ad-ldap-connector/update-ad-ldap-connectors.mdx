description: AD/LDAP Connector を更新する方法について説明します。
title: AD/LDAP Connector の更新
---

デプロイ環境内に AD/LDAP Connector のインスタンスが複数ある場合は、同時に停止するインスタンスが 1 つだけになるよう、以下の手順を各インスタンスに対して 1 台ずつ実行することを Auth0 は推奨しています。

<div id="admin-console-windows">
  ## Admin console (Windows)
</div>

1. `http://localhost:8357/` にアクセスして Admin Console を開きます。Admin Console からインターネットへのアウトバウンドアクセスが可能な場合は、新しいバージョンが利用可能かどうかを確認し、ページ上部に次のように表示します。

   <Frame>![Update AD/LDAP Connectors Admin Console Screen](/docs/images/cdy7uua7fh8z/6Yp2uRnEU3RMeCqgHa6Ztl/b9c04ab83b9fd725910291b01ac9d05d/connector-update-available.png)</Frame>
2. **Update** タブに移動し、このマシン上のインストールを最新バージョンに更新します。更新には約 2 分かかり、アップデーターのログが Admin Console に表示されます。

<div id="updater-script-windows">
  ## 更新スクリプト (Windows)
</div>

更新スクリプトは、次の手順を実行して、コマンドラインから AD/LDAP Connector を更新します。

1. 利用可能な更新があるかどうかを確認します。
2. 既存の構成、証明書、および profileMapper.js をバックアップします。
3. AD/LDAP Connector をアンインストールします。
4. 更新をダウンロードします。
5. AD/LDAP Connector をインストールします。
6. 既存の構成、証明書、および profileMapper.js を復元します。
7. Windows サービスを起動します。
8. 更新スクリプトを実行するには、コマンドラインで次のコマンドを実行します。
   `@powershell -NoProfile -ExecutionPolicy Bypass -Command "iex ((new-object net.webclient).DownloadString('https://cdn.auth0.com/connector/windows/update-connector.ps1'))"`

更新スクリプトでは、Windows PowerShell 3.0 以降でのみ使用できる特定の PowerShell コマンドを使用します。Windows 2008 または Windows 2008 R2 で実行している場合は、先に PowerShell のバージョンを更新する必要がある可能性があります。

詳細については、[Download Window Management Framework](https://www.microsoft.com/en-us/download/details.aspx?id=34595) を参照してください。

<div id="manual-update-windowslinux">
  ## 手動更新 (Windows/Linux)
</div>

1. インストールされているバージョンを確認します。コンソールの Connector のステータスインジケーターにカーソルを合わせます。ツールチップに現在のステータスとインストール済みのバージョンが表示されます。
2. 最新バージョンの[インストーラー](https://cdn.auth0.com/connector/windows/adldap-5.0.10.msi?_ga=2.58973869.1675886805.1589821069-1123834949.1586212750)をダウンロードします。
   他のプラットフォームについては、GitHub リポジトリの[リリース版](https://github.com/auth0/ad-ldap-connector/releases/tag/v5.0.10)を使用してください。
3. 現在の設定を、Admin Console または手動でバックアップします。

   1. Admin Console で **Download** ボタンをクリックし、`config.json` ファイル、`certs` フォルダー、および `lib\\profileMapper.js` ファイルを含む .zip ファイルを生成します。
   2. 手動で行う場合は、上記と同じファイルを `%Program Files(x86)%\Auth0\AD LDAP Connector\` フォルダーからバックアップします。このパスは Windows のものです。他のプラットフォームではファイルの場所が異なる場合がありますが、含まれるファイルは同じです。
4. インストーラーを実行し、画面の指示に従います。
5. 何も変更せずに Configuration ダイアログを閉じます。
6. Admin Console または手動でファイルを復元します。

   1. Admin Console の **Import / Export** タブで .zip ファイルをアップロードします。
   2. 手動で行う場合は、事前にバックアップしたファイルをコピーします。
7. サービスコンソールから **Auth0 AD LDAP** サービスを再起動します。
