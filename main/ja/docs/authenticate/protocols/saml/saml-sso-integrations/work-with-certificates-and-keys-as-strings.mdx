description: Rules や Management API のリクエストで、ファイルの文字列表現が必要な場合に、証明書や鍵を扱う方法について説明します。
title: 証明書と鍵を文字列として扱う
---

UNIX シェルでは、次のコマンドを使用して、PEM 形式の証明書を `\n` (エスケープした改行) で連結できます。

```bash wrap lines
awk 'NF {sub(/\r/, ""); printf "%s\\n",$0;}' certificate.pem | pbcopy
```

これにより、値がクリップボードにコピーされるため、必要な場所に貼り付けることができます。

<div id="more-information">
  ### 詳細情報
</div>

Rules または <Tooltip tip="Management API: お客様が管理タスクを実行するためのプロダクト。" cta="用語集を見る" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> v2 リクエストで証明書や鍵を扱う場合は、通常、そのファイルの文字列表現が必要になります。

テキストエディターで証明書ファイル (`cer`、`pem`) を開くと、次のような内容が表示されます。

```text lines
-----BEGIN CERTIFICATE-----
MIICzDCCAbQCCQDH8GvxPIeH+DANBgkqhkiG9w0BAQsFADAoMQswCQYDVQQGEwJh
cjEZMBcGA1UEAwwQaHR0cHM6Ly9uaWNvLmNvbTAeFw0xOTA0MDgxODA3NTVaFw0y
//
// base64エンコードされた情報がさらに続く
//
nSWyabd+LiBGtLTMB+ZLbOIi3EioWPGw/nHOI8jzPrqhiCLuZCSQmiqrLQYNsc1W
-----END CERTIFICATE-----
```

`-----BEGIN CERTIFICATE-----` ヘッダーと `-----END CERTIFICATE-----` フッターの間の行には、Base64 エンコードされたバイナリ情報が含まれています。公開鍵や秘密鍵 (`.key` ファイル) も、ヘッダー/フッターが異なるだけで、見た目はほぼ同じです。

証明書/キーファイルの文字列表現には、ファイル内の実際の改行を `\n` (エスケープされた改行) シーケンスに置き換え、全体を 1 行に連結する必要があります。したがって、上記のサンプルは次のような文字列になります。

```text lines
"-----BEGIN CERTIFICATE-----
\nMIICzDCCAbQCCQDH8GvxPIeH+DANBgkqhkiG9w0BAQsFADAoMQswCQYDVQQGEwJh\n
[..all the other lines..]
-----END CERTIFICATE-----\n"
```
