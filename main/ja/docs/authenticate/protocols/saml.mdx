<Tooltip tip="Security Assertion Markup Language（SAML）: パスワードを使わずに 2 者間で認証情報を交換できる標準化されたプロトコルです。" cta="用語集を表示" href="/ja/docs/glossary?term=Security+Assertion+Markup+Language">Security Assertion Markup Language</Tooltip> (SAML) プロトコルは、パスワードを使わずに 2 者間で認証と認可を行うための、オープンスタンダードの XML ベースのフレームワークです。

* **サービスプロバイダー** (SP) は、<Tooltip tip="Security Assertion Markup Language（SAML）: パスワードを使わずに 2 者間で認証情報を交換できる標準化されたプロトコルです。" cta="用語集を表示" href="/ja/docs/glossary?term=identity+provider">IDプロバイダー</Tooltip>によるユーザー認証を信頼します。
* **IDプロバイダー** (IdP) はユーザーを認証し、そのユーザーが認証済みであることを示す認証アサーションをサービスプロバイダーに提供します。

Auth0 は SAML プロトコルをサポートしており、IdP、SP、またはその両方として機能できます。具体的には次のとおりです。

* SAML2 Web アプリケーション
* SAML <Tooltip tip="Single Sign-On（SSO）: ユーザーが 1 つのアプリケーションにログインすると、他のアプリケーションにも自動的にログインできるようにするサービスです。" cta="用語集を表示" href="/ja/docs/glossary?term=SSO">SSO</Tooltip> 統合
* IdP 起点の SSO
* Microsoft Active Directory Federation Services (ADFS)
* SAML リクエストの署名と暗号化

<div id="supported-saml-bindings-and-options">
  ## サポートされるSAMLバインディングとオプション
</div>

Auth0 は、次の SAML バインディングをサポートしています。

* HTTP Redirect
* HTTP POST

Auth0 は、次の SAML オプションをサポートしています。

* Web Browser SSO Profile
* Single Logout Profile
* Name Identifier Management Profile
* Name Identifier Mapping Profile
