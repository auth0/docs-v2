title: Oracle Eloqua Marketing Cloud を SAML サービスプロバイダーとして構成する
---

Oracle Eloqua Marketing Cloud では、次の <Tooltip tip="Security Assertion Markup Language（SAML）: パスワードを使わずに 2 者間で認証情報を交換するための標準化されたプロトコルです。" cta="用語集を見る" href="/ja/docs/glossary?term=SAML">SAML</Tooltip> 設定を使用します。

<div id="configure-saml-sso-in-auth0">
  ## Auth0 で SAML SSO を設定する
</div>

1. [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) に移動し、新しいアプリケーションを作成するか、更新するアプリケーション名をクリックします。
2. **Addons** タブに移動し、**SAML2 Web App** トグルを有効にします。
3. **Settings** タブで、**Application Callback URL** を `https://login.eloqua.com/auth/saml2/acs` に設定します。

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>
4. 次のコードを **Settings** テキストボックスに貼り付け、**Debug** をクリックします。

   ```json lines
   {
       "audience": "<Service Provider Entity Url from within Eloqua>",
        "recipient": "https://login.eloqua.com/auth/saml2/acs",
          "mappings": {
       "user_id": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier",
       "email": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress",
       "name": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name",
          },
        "nameIdentifierFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress",
        "nameIdentifierProbes": [
       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress"
       ],
        "destination": "https://login.eloqua.com/auth/saml2/acs"
         }
   ```

   `audience` は `http://foo` に設定されている点に注意してください。これは、Eloqua 側で IDプロバイダーの設定が完了したら変更されます。
5. タブの一番下までスクロールして、**Enable** をクリックします。
6. **Usage** タブで **Identity Provider Metadata** を見つけ、**Download** をクリックしてメタデータファイルをダウンロードします。これは、Eloqua で Auth0 を IDプロバイダーとして設定する際に必要です。

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Usage Tab](/docs/images/cdy7uua7fh8z/3T90BDpyTXFUWDp1JkncBU/86bdba6bed8130b34b42b25c9bf2d232/dashboard-applications-applications-addons-saml2-web-app-usage.png)</Frame>

<div id="configure-saml-sso-in-oracle-eloqua">
  ## Oracle Eloqua で SAML SSO を設定する
</div>

1. Eloqua に管理者としてログインし、**Settings &gt; Users &gt; Single Sign-On &gt; IDプロバイダー Settings** をクリックします。
2. **Upload IDプロバイダー from Metadata** をクリックします。Auth0 Dashboard の **Usage** タブからダウンロードした **IDプロバイダーのメタデータ** をアップロードします。
3. ダウンロードしたメタデータファイルを参照して選択します。
4. 新しく追加した IDプロバイダーを編集し、次の手順を実行します。

   1. **User Identity Mapping** を `Assertion contains the Email Address from the User object` に設定し、**Save** をクリックします。
   2. **IDプロバイダー Details** から **Service Provider Entity URL** をコピーします。
   3. ELOQUA の IdP 設定で、これがデフォルトの IdP の場合は、この IdP をデフォルトとしてマークします。

<div id="update-audience-restriction-in-auth0">
  ## Auth0 のオーディエンス制限を更新
</div>

Eloqua の <Tooltip tip="IDプロバイダー（IdP）：デジタル ID を保存および管理するサービス。" cta="用語集を表示" href="/ja/docs/glossary?term=identity+provider">IDプロバイダー</Tooltip> の設定でコピーした **Service Provider Entity URL** を使用して、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> で `audience` の制限を設定します。

1. [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) に移動し、更新するアプリケーション名をクリックします。
2. **Addons** タブに移動し、**SAML2 Web App** トグルを有効にします。
3. **Settings** タブで、`audience` を Eloqua からコピーした値に更新します。

<div id="test-configuration">
  ## テスト構成
</div>

**Auth0 を使用した Eloqua へのログイン** が有効になっているはずです。これで、IdP 起点ログインと SP 起点ログインの両方を使用して Eloqua にサインインできます。

* SP 起点ログインの場合は、[https://login.eloqua.com/auth/saml2](https://login.eloqua.com/auth/saml2)2&gt; にアクセスし、会社名を入力して、Auth0 による SAML ログインプロセスを開始します。
* IdP 起点ログインの場合は、**Dashboard &gt; Applications &gt; Applications** の **Addons** タブにある **SAML 2 Web App** の **Usage** タブで定義されている **IDプロバイダー Login URL** を使用します。
