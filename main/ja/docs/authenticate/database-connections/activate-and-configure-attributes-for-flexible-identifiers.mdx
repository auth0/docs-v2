<Warning>
  テナントで Flexible Identifiers を有効にすると、本番環境に破壊的変更が生じる可能性があります。想定される問題や制限事項の詳細については、[Flexible Identifiers and Attributes](/ja/docs/authenticate/database-connections/flexible-identifiers-and-attributes) を参照してください。
</Warning>

データベース接続では、メールアドレス、電話番号、または username などの属性は Flexible Identifiers の一部です。エンドユーザーは、これらの属性を使用してサインアップ、認証、パスワードのリセットを行います。各属性には固有の設定があります。

<div id="activate-attributes">
  ## 属性を有効にする
</div>

<Tabs>
  <Tab title="新しい接続" icon="database">
    1. [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database) に移動します。
    2. **+Create DB Connection** を選択します。
    3. 接続名を入力し、属性を選択します。
    4. 接続の認証方法を選択します。
  </Tab>

  <Tab title="既存の接続" icon="database">
    1. [**Auth0 Dashboard &gt; Authentication &gt; Database**](https://manage.auth0.com/#/connections/database) に移動します。
    2. 一覧から更新するデータベース接続を選択します。
    3. **Attributes** タブを選択し、**Activate** をクリックします。
    4. Development 環境で設定をテスト済みであることの確認を求められたら、**Proceed** を選択します。
    5. Attributes 画面で、**Add Attribute** を選択するか、既存の属性の **Configure** を選択します。
  </Tab>
</Tabs>

新しい接続にはデフォルト値が設定されており、接続の作成後に編集できます。

| 設定                                | 値            |
| --------------------------------- | ------------ |
| `\{attribute}` を識別子として使用          | `True`       |
| `\{attribute}` でのサインアップを許可        | 必須           |
| ユーザープロファイルで `\{attribute}` を必須にする | `True`       |
| Username の長さ                      | 最小: 1 最大: 15 |
| メールアドレス形式の username を許可           | `False`      |
| 電話番号形式の username を許可              | `False`      |

<div id="configure-attributes">
  ## 属性を設定する
</div>

接続の **Attributes** タブを選択します。使用可能な属性から選んで **Configure** を選択します。属性設定を更新したら、**Save** を選択します。

<div id="configure-or-remove-existing-attributes-on-a-connection">
  ### 接続の既存の属性を設定または削除する
</div>

すべての属性は、<Tooltip tip="サービスを設定するための Auth0 の主要製品である Auth0 Dashboard。" cta="用語集を見る" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> で変更または削除できます。削除された属性は、ユーザーのログイン、サインアップ、パスワードのリセットには使用できません。また、各接続では少なくとも 1 つの属性を有効なままにしておく必要があります。

属性を変更または削除するには、**Attributes** タブに移動し、**Remove** を選択します。

<div id="revert-a-connection">
  ### 接続を元に戻す
</div>

接続を元の状態に戻すには、以下のように `attributes` パラメーターを含まない `options` オブジェクトを指定して、<Tooltip tip="Management API: お客様が管理タスクを実行できるようにする製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Management+API">Management API</Tooltip> の [接続を更新する](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) エンドポイントを使用します。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * `options` パラメーターを使用すると、`options` オブジェクト全体が上書きされます。データの欠落やその他の問題を避けるため、接続の `options` オブジェクトの内容をコピーして PATCH 呼び出しに含め、現在のすべてのパラメーターが含まれていることを確認してください。削除するパラメーターは `attributes` のみです。
  * テナントで属性として `username` が有効になっている場合、以下の操作を行うには `username` の設定を変更する必要があります。この方法を選択する場合は、`username` の設定を再構成する必要があります。
</Callout>

```json lines
{
"options": {
       }
}
```
