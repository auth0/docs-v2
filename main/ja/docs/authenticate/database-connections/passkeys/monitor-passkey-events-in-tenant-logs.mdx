description: Auth0 Dashboard または Management API でパスキー固有のイベントコードを使用してフィルタリングすることで、Auth0 テナントログ内のパスキーの登録イベントと認証イベントを追跡します。
title: テナントログでパスキーイベントを監視する
---

[テナントログ](/ja/docs/deploy-monitor/logs)を使用して、データベース接続に関連するパスキーフローを監視できます。

ログイベントは、<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要製品です。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ([Monitoring &gt; Logs](https://manage.auth0.com/#/logs)) で確認するか、[Management API](https://auth0.com/docs/api/management/v2/logs/get-logs)を使用して取得できます。

| イベントコード                         | シナリオ                                  | 注記                                                                                                                                                           |
| ------------------------------- | ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `ss`                            | サインアップ時に、ユーザーが主要な認証方法としてパスキーを登録しました。  | このログイベントには、値 `passkey` を含む `details.authentication_methods` 配列フィールドが含まれます。<br /><br />ユーザーがパスワードで登録した場合、この配列フィールドには値 `pwd` が含まれます。                           |
| `s`                             | ログイン時に、ユーザーがパスキーで認証しました。              | このログイベントには `details.prompts` 配列が含まれており、ログイン中にユーザーがたどったプロンプトの一覧が記録されます。<br /><br />ユーザーがパスキーを使用して認証した場合、この配列には次のフィールドを持つ要素が含まれます: `"performed_amr": ["phr"]`。 |
| `gd_enrollment_complete`        | プログレッシブ登録フローまたはローカル登録フローが正常に完了しました。   | ユーザーがパスキーを登録した場合、このログイベントには値 `passkey` を持つ `details.authenticator.type` フィールドが含まれます。                                                                         |
| `gd_webauthn_enrollment_failed` | プログレッシブ登録フローまたはローカル登録フロー中にエラーが発生しました。 | ユーザーがパスキーを登録しようとした場合、このログイベントには値 `passkey` を含む `details.authentication_methods array` フィールドが含まれます。                                                           |
