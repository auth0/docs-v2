title: OIDC での デリゲーション
---

<Warning>
  デフォルトでは、2017 年 6 月 8 日時点でアドオンを使用していないテナントでは デリゲーション は無効になっています。現在、デリゲーション を必要とするアドオンを使用している既存のテナントでは、この機能を引き続き利用できます。今後、デリゲーション 機能が変更されたりサービスから削除されたりする場合は、現在利用しているお客様に事前に通知し、移行のための十分な時間が提供されます。
</Warning>

従来、デリゲーション は次の目的で使用されてきました。

* あるアプリケーションに対して発行された <Tooltip tip="IDトークン: リソースへのアクセスではなく、クライアント自体のための資格情報です。" cta="用語集を表示" href="/ja/docs/glossary?term=ID+token">IDトークン</Tooltip> を、別のアプリケーション向けに発行された新しいトークンと交換する。
* [refresh token](/ja/docs/secure/tokens/refresh-tokens) を使用して新しい IDトークン を取得する。
* IDトークン をサードパーティ (Firebase、AWS など) の API トークンと交換する。

OIDC 準拠パイプラインでは、IDトークン を API の保護に使用できなくなり、<Tooltip tip="リフレッシュトークン: ユーザーに再度ログインさせることなく、新しいアクセストークンを取得するために使用するトークンです。" cta="用語集を表示" href="/ja/docs/glossary?term=refresh+tokens">refresh tokens</Tooltip> は [`/oauth/token`](https://auth0.com/docs/api/authentication#refresh-token) エンドポイントでのみ使用する必要があるため、[`/delegation`](https://auth0.com/docs/api/authentication#delegation) エンドポイントは非推奨です。

OIDC 準拠アプリケーションは、デリゲーション リクエストのソースにもターゲットにもできません。

<div id="third-party-apis">
  ## サードパーティ API
</div>

サードパーティ (例: Firebase、AWS) の API トークンを取得するための OIDC 準拠の仕組みは存在しないため、サードパーティ API トークンの取得には引き続きデリゲーションを使用できます。
