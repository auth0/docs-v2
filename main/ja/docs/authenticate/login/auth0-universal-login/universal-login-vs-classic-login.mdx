title: Universal Login と Classic Login の比較

---

[Auth0 Universal Login](/ja/docs/authenticate/login/auth0-universal-login) は、<Tooltip tip="認可サーバー: ユーザーのアクセス範囲の定義に関与する中央集約型サーバーです。たとえば、認可サーバーはユーザーが利用できるデータ、タスク、機能を制御できます。" cta="用語集を見る" href="/ja/docs/glossary?term=authorization+server">認可サーバー</Tooltip> の中核機能であるログインフローを提供します。ユーザーがアプリケーションにアクセスするために本人確認を行う必要がある場合は、そのユーザーを <Tooltip tip="認可サーバー: ユーザーのアクセス範囲の定義に関与する中央集約型サーバーです。たとえば、認可サーバーはユーザーが利用できるデータ、タスク、機能を制御できます。" cta="用語集を見る" href="/ja/docs/glossary?term=Universal+Login">Universal Login</Tooltip> にリダイレクトし、認証プロセスは Auth0 に任せることができます。

Universal Login は、Auth0 の主要なホスト型ログインソリューションです。[Universal Login](/ja/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience) では使いやすいカスタマイズツールを利用でき、エンドユーザーによりシンプルで迅速な体験を提供できます。一方、Auth0 は [Classic Login](/ja/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/classic-experience) もサポートしており、こちらは JavaScript コントロールを使ってカスタマイズするホスト型ログインエクスペリエンスです。

<Warning>
  現在、Auth0 の開発は Universal Login に注力されており、Classic Login は更新されなくなっています。特別なユースケースで Classic エクスペリエンスが必要な場合を除き、Universal Login を実装することを推奨します。
</Warning>

ニーズに最適なソリューションを選べるよう、以下の表で 2 つのログインエクスペリエンスの機能を比較しています。

| **ノーコードソリューション**                                     | [ノーコードエディター](/ja/docs/customize/login-pages/universal-login/customize-themes)を使用して可能                                                                            | いいえ                                                                                                                                                                              |
