title: Classic Login エクスペリエンス
---

Classic Login は、カスタマイズに JavaScript を使用する Auth0 ホスト型のログインエクスペリエンスです。Classic Login の実装は、認証プロセスをアプリケーションに直接組み込む場合に比べて複雑さが低く、クロスオリジン認証に伴うリスクの軽減にも役立ちます。

Classic Login は、Auth0 の JavaScript ライブラリ ([Lock.js](/ja/docs/libraries/lock)、[auth0.js](/ja/docs/libraries/auth0js)、<Tooltip tip="多要素認証（MFA）: SMS による code など、username と password に加えて別の要素を使用するユーザー認証プロセス。" cta="用語集を表示" href="/ja/docs/glossary?term=MFA">MFA</Tooltip> Widget、Password Reset) をベースに構築されています。<Tooltip tip="Auth0 Dashboard: サービスを設定するための Auth0 の主要プロダクト。" cta="用語集を表示" href="/ja/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> で Classic Login ページをカスタマイズする際、デフォルトテンプレートでは同じ JavaScript ライブラリが使用されるため、デフォルトのユーザーインターフェースからカスタム UI への移行をより自然に行えます。

Classic Login ページ用のデフォルトテンプレートを選択したら、ニーズに合わせて変更できます。Lock widget の動作や外観に関するさまざまな要素もカスタマイズ可能です。Auth0.js テンプレートでは、カスタムユーザーインターフェースを作成し、アプリケーションのスタイルに合わせて調整できるため、さらに柔軟に対応できます。

<Warning>
  現在、Auth0 の開発は Universal Login に注力しており、Classic Login はすでに更新されていません。特定のユースケースで Classic エクスペリエンスが必要な場合を除き、Universal Login を実装することを推奨します。

  各ログインエクスペリエンスの機能比較については、[Universal Login vs. Classic Login](/ja/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login) を参照してください。
</Warning>
