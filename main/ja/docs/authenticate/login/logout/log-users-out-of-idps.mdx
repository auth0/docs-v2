---
description: ユーザーをIDプロバイダーから強制的にログアウトさせる方法を説明します。
title: IDプロバイダーからユーザーをログアウトさせる
---

一部のプロバイダーでは、ユーザーをその<Tooltip tip="Identity Provider (IdP): デジタルIDを保存および管理するサービス。" cta="用語集を見る" href="/ja/docs/glossary?term=identity+provider">IDプロバイダー</Tooltip>から強制的にログアウトさせることができます。Auth0 では通常、`/oidc/logout` エンドポイントへのリダイレクトに `federated` クエリ文字列パラメーターを追加することで、これを実現します。

これを行うには、ログアウト URL に `federated` クエリ文字列パラメーターを追加します。

`https://{yourDomain}/oidc/logout?federated`

<div id="federated-logout-support">
  ## フェデレーテッドログアウトのサポート
</div>

次のIDプロバイダーでフェデレーテッドログアウトがサポートされています。

* Evernote
* Facebook
* Fitbit
* GitHub
* Google

  * Google Workspace
  * OAuth 2.0
* Microsoft

  * Active Directory Federation Services
  * Microsoft 365
  * Microsoft Entra ID
  * Windows Live
* Okta
* OpenID Connect (OIDC)
* Salesforce/Salesforce Sandbox
* X
* Yahoo
* Microsoft Viva Engage

<div id="clear-application-session">
  ## アプリケーションのセッションをクリアする
</div>

Auth0 の Logout エンドポイントは、Auth0 からログアウトし、必要に応じて IDプロバイダー からもログアウトします。アプリケーションからはログアウトされません。これはお客様側で実装する必要があります。セッションをクリアして、アプリケーションからユーザーをログアウトさせる必要があります。

<div id="alternative-logout">
  ## 代替ログアウト
</div>

[代替ログアウトエンドポイント](https://auth0.com/docs/api/authentication#logout)にフェデレーテッドログアウトを追加するには、ログアウト URL に `federated` クエリパラメーターを追加します。

`https://{yourDomain}/v2/logout?federated`

<div id="alternative-logout-limitations">
  ### 代替ログアウトの制限事項
</div>

* `returnTo` パラメーターの値として指定された URL や、その URL に含まれるクエリ文字列またはハッシュ情報は、一切検証されません。
* ソーシャルプロバイダーでのフェデレーテッドログアウトの動作には一貫性がありません。各プロバイダーは `returnTo` パラメーターをそれぞれ異なる方法で処理するため、プロバイダーによっては機能しない場合があります。どのように動作するかは、利用しているソーシャルプロバイダーの設定を確認してください。
* Google や Facebook などのソーシャルIDプロバイダーを使用している場合は、ログアウトを正しく機能させるために、Dashboard でそれらのプロバイダーの <Tooltip tip="クライアントID: Auth0 から登録済みリソースに付与される識別値。" cta="用語集を表示" href="/ja/docs/glossary?term=Client+ID">クライアントID</Tooltip> と Secret を設定する必要があります。
* Auth0 Enterprise ユーザーの場合、通常は SharePoint、複数の .NET アプリケーション、複数の Java アプリケーション、Zendesk など、複数のアプリケーションで <Tooltip tip="シングルサインオン（SSO）: ユーザーが1つのアプリケーションにログインすると、他のアプリケーションにも自動的にログインされるサービス。" cta="用語集を表示" href="/ja/docs/glossary?term=SSO">SSO</Tooltip> が有効になっています。この場合、ユーザーがサインアウトする際には、すべてのアプリケーションでもサインアウトする必要が生じることがよくあります。
