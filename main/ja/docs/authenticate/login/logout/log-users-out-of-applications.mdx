description: Auth0 のログアウトエンドポイントを使用して、アプリケーションからユーザーを強制的にログアウトさせる方法を説明します。
title: アプリケーションからユーザーをログアウトする
---

エンタープライズユーザーでは、通常、複数のアプリケーション (例: SharePoint、いくつかの .NET アプリケーション、いくつかの Java アプリケーション、Zendesk) で <Tooltip tip="シングルサインオン（SSO）: ユーザーが 1 つのアプリケーションにログインすると、他のアプリケーションにも自動的にログインできるようにするサービスです。" cta="用語集を表示" href="/ja/docs/glossary?term=Single+Sign-on">シングルサインオン</Tooltip> (SSO) が有効になっています。この場合、ユーザーがサインアウトすると、そのユーザーが利用しているすべてのアプリケーションからもサインアウトさせる必要があることがよくあります。

Auth0 の [RP-initiated logout endpoint](/ja/docs/authenticate/login/logout/log-users-out-of-auth0) は、次の 2 つのいずれかの方法で動作します。

* Auth0 のシングルサインオン (SSO) Cookie を無効にします。 (Cookie 自体はブラウザーに残ります。)
* ユーザーを <Tooltip tip="IDプロバイダー（IdP）: デジタル ID を保存および管理するサービスです。" cta="用語集を表示" href="/ja/docs/glossary?term=identity+provider">IDプロバイダー</Tooltip> (IdP)  (ADFS や Google など) からサインアウトさせます。

ユーザーをログアウトエンドポイントにリダイレクトするだけでは、ユーザーが利用したすべてのアプリケーションからサインアウトする必要があるケースには対応できません。この機能が必要な場合は、次の 2 つのいずれかの方法で対応する必要があります。

* ローカルセッションのタイムアウトを短く設定し、短い間隔で Auth0 にリダイレクトして再認証します。これは、クライアントから `checkSession` を呼び出すことで実現できます。この呼び出しでは、非表示の iFrame でリダイレクトが行われます。非表示の iFrame を使用する場合は、[レート制限](/ja/docs/troubleshoot/customer-support/operational-policies/rate-limit-policy) とサードパーティ [cookie](/ja/docs/manage-users/cookies) の問題に注意する必要があります。
* ログアウトが発生したときに他のすべてのアプリケーションへ通知する仕組みを用意し、これをアプリケーションレベルで完全に処理します。

代替のログアウトエンドポイントを使用している場合は、[API ドキュメント](https://auth0.com/docs/api/authentication#logout) を確認してください。
