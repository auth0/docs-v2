Auth0 では、識別子 (メールアドレス、username、または電話番号) とパスワード、または [パスキー](/ja/docs/authenticate/database-connections/passkeys/configure-passkey-policy) を使用してユーザーを認証するためのデータベース接続を提供しています。これらの認証情報は、Auth0 のユーザーストアまたは独自のデータベースに安全に保存されます。

新しいデータベース接続の作成や既存の接続の管理は、[Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database) で行えます。

<div id="database-options">
  ## データベースオプション
</div>

<div id="use-auth0-user-store">
  ### Auth0 のユーザーストアを使用する
</div>

Auth0 では、ユーザーを保存するためのデータベースインフラストラクチャがデフォルトで提供されます。この方法では、すべてのデータが Auth0 に保存されるため、認証プロセスで最適なパフォーマンスが得られます。

Auth0 がホストするデータベースは非常に安全です。パスワードが平文で保存されたりログに記録されたりすることはなく、**bcrypt** でハッシュ化されます。また、パスワードセキュリティ要件も複数のレベルで適用できます。詳しくは、[Auth0 データベース接続のパスワード強度](/ja/docs/authenticate/database-connections/password-strength) を参照してください。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  データベース接続では、Auth0 はユーザーごとおよび IP アドレスごとに、ログインの再試行回数を制限します。詳しくは、[データベースのログイン制限](/ja/docs/troubleshoot/customer-support/operational-policies/rate-limit-policy#database-login-limits) を参照してください。
</Callout>

Auth0 のデータベースインフラストラクチャをユーザーストアとして使用する場合、ユーザーを Auth0 に移行する方法はいくつかあります。[自動移行](/ja/docs/manage-users/user-migration/configure-automatic-migration-from-your-database) 機能では、各ユーザーがログインした時点で 1 人ずつ Auth0 データベースに追加されるため、すべてのユーザーに同時にパスワードのリセットを求めずに済みます。あるいは、Management API を使用して [ユーザーをインポートする](/ja/docs/manage-users/user-migration/bulk-user-imports) ジョブを作成することもできます。詳しくは、移行例について説明している [ユーザー移行シナリオ](/ja/docs/manage-users/user-migration/user-migration-scenarios) を参照してください。

<div id="use-your-own-database">
  ### 独自のデータベースを使用する
</div>

既存のユーザーストアがある場合や、ユーザーの認証情報を独自のサーバーに保存したい場合は、Auth0 で [カスタムデータベース](/ja/docs/authenticate/database-connections/custom-db) またはリポジトリに接続し、それを IDプロバイダーとして使用できます。

<Frame>![](/docs/images/cdy7uua7fh8z/3SNKwx0KzkUwbodqUQbtvW/df720f8871b56dd7ec399a24aa621c36/Authentication>Database>use-my-own-database.png)</Frame>

このシナリオでは、ユーザーがログインを試みるたびに実行される、ユーザー認証用のログインスクリプトを用意します。必要に応じて、サインアップ、メールアドレス確認、パスワードリセット、ユーザー削除機能用の [スクリプトを作成](/ja/docs/authenticate/database-connections/custom-db/templates) することもできます。

これらのスクリプトは Node.js のコードです。Auth0 は、ASP.NET Membership Provider、MongoDB、MySQL、PostgreSQL、SQL Server、Windows Azure SQL Database、Basic Auth でアクセスする Web サービスなど、一般的なデータベースの多くに対応する [テンプレート](/ja/docs/authenticate/database-connections/custom-db/templates) を提供しています。つまり、カスタムスクリプトを使えば、ほぼあらゆる種類のデータベースや Web サービスに接続できます。
