```ts Example
export interface UserMembers {
  id: string | null;
  email: string | null;
  username: string | null;
  phoneNumber: string | null;
  picture: string | null;
  enrolledFactors: Array<string> | null;
  enrolledEmails: Array<EnrolledEmail> | null;
  enrolledPhoneNumbers: Array<EnrolledPhoneNumber> | null;
  enrolledDevices: Array<EnrolledDevice> | null;
  organizations: Organizations[] | null;
  userMetadata: { [key: string]: string } | null;
  appMetadata: { [key: string]: string } | null;
}
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
when the Auth0 server is not aware of the current user, User properties are not available. For example:

| Screens                                 | Screens                        |   
|-----------------------------------------|--------------------------------|
| Login screens                           | MFA OTP screens                | 
| Signup screens                          | MFA Phone screens              |  
| Passkey screens                         | MFA Push screens               |  
| Challenge screens                       | MFA Recovery Code screens      |  
| Email Identifier screens                | MFA WebAuthn screens           | 
| Phone Identifier screens                | Accept Invitation screens      |  
| InterstitialCaptcha screens             | Device Code screens            |  
| Reset Password screens                  | Email Verification screens     |  
| MFA Country Codes screens               | Logout Aborted screens         |  
| MFA Detect Browser Capabilities screens | Logout Complete screens        |
| MFA Enroll screens                      | Brute Force Protection screens |  

</Callout>
