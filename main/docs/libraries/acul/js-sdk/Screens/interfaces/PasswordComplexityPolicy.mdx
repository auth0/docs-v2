---
title: "PasswordComplexityPolicy"
---

Defines the password complexity rules enforced during signup or password reset flows.

```ts Example
export interface PasswordComplexityPolicy {
  character_type_rule: 'all' | 'at_least';
  character_types: ('number' | 'uppercase' | 'lowercase' | 'special')[];
  identical_characters: 'block' | 'allow';
  max_length_exceeded: 'allow' | 'error';
  min_length: number;
  sequential_characters: 'block' | 'allow';
}
```

## Properties

<ParamField body='character_type_rule' type='"all" | "at_least"' required>
Determines whether all specified character types must be present (`'all'`) or only some of them (`'at_least'`).
</ParamField>

<ParamField body='character_types' type='("number" | "uppercase" | "lowercase" | "special")[]' required>
Specifies which character categories to enforce in the password.
</ParamField>

<ParamField body='identical_characters' type='"block" | "allow"' required>
Controls whether repeated identical characters are permitted (`'allow'`) or blocked (`'block'`).
</ParamField>

<ParamField body='max_length_exceeded' type='"allow" | "error"' required>
Defines behavior when the password exceeds the maximum length — silently allow it or return an error.
</ParamField>

<ParamField body='min_length' type='number' required>
The minimum number of characters required for the password.
</ParamField>

<ParamField body='sequential_characters' type='"block" | "allow"' required>
Determines whether consecutive character sequences (e.g., `'abc'`, `'123'`) are permitted (`'allow'`) or blocked (`'block'`).
</ParamField>
