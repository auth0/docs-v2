The `mfa-phone-challenge` screen prompts the user to verify their identity by entering a code sent to their phone via SMS or voice call.

## Import

Each screen has its own set of hooks and methods. The SDK supports **partial import** and **root import** for each screen.
* Using partial import allows you include only the code you need for your specific use case.
* Using root import allows you to load all screens from a single bundle useful when you want a unified build to handle all possible screens.

```jsx Import Example
// root import
import { useMfaPhoneChallenge } from '@auth0/auth0-acul-react';

// partial import
import {
  useMfaPhoneChallenge,
  // Context hooks
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Common hooks
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Utility hooks
  useChangeLanguage,
  // Methods
  continueMethod,
  pickPhone,
  tryAnotherMethod,
} from "@auth0/auth0-acul-react/mfa-phone-challenge";

function MfaPhoneChallengeScreen() {
  const { continueMethod } = useMfaPhoneChallenge();
  return (
    <button onClick={() => continueMethod({ type: 'sms' })}>
      Send Code via SMS
    </button>
  );
}
```

## Methods

<ParamField body='continueMethod' type={<span>void | Promise&lt;void&gt;</span>}>

This method initiates the phone challenge by sending a verification code to the user's phone via the specified delivery method.

```jsx Example
import { useMfaPhoneChallenge } from '@auth0/auth0-acul-react/mfa-phone-challenge';

function PhoneChallengeForm() {
  const { continueMethod } = useMfaPhoneChallenge();
  return (
    <button onClick={() => continueMethod({ type: 'sms' })}>
      Send Code via SMS
    </button>
  );
}
```

<ParamField body='type' type='"sms" | "voice"'>

The type of challenge to perform.
'sms': Send the code via text message.
'voice': Send the code via a voice call.
