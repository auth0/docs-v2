The `email-otp-challenge` screen is displayed when a user needs to verify their identity by entering a one-time passcode sent to their email address. It allows the user to submit the code or resend it.

## Import

Each screen has its own set of hooks and methods. The SDK supports **partial import** and **root import** for each screen.
* Using partial import allows you to include only the code you need for your specific use case.
* Using root import allows you to load all screens from a single bundle useful when you want a unified build to handle all possible screens.

```jsx Import Example
// root import
import { useEmailOTPChallenge } from '@auth0/auth0-acul-react';

// partial import
import {
  useEmailOTPChallenge,
  // Context hooks
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Common hooks
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  useResend,
  // Utility hooks
  useChangeLanguage,
  // Methods
  submitCode,
  resendCode,
} from '@auth0/auth0-acul-react/email-otp-challenge';

function EmailOtpChallengeScreen() {
  const { submitCode } = useEmailOTPChallenge();
  return (
    <button onClick={() => submitCode({ code: '123456' })}>Submit</button>
  );
}
```

## Methods

<ParamField body='resendCode' type={<span>void | Promise&lt;void&gt;</span>}>

This method resends the one-time passcode to the user's email address.

```jsx Example
import { useEmailOTPChallenge } from '@auth0/auth0-acul-react/email-otp-challenge';

function ResendButton() {
  const { resendCode } = useEmailOTPChallenge();
  return (
    <button onClick={() => resendCode()}>
      Resend Code
    </button>
  );
}
```
