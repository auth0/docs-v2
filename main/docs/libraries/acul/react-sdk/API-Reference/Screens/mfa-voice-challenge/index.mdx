The `mfa-voice-challenge` screen prompts the user to enter a one-time code delivered via voice call to complete MFA verification.

## Import

Each screen has its own set of hooks and methods. The SDK supports **partial import** and **root import** for each screen.
* Using partial import allows you include only the code you need for your specific use case.
* Using root import allows you to load all screens from a single bundle useful when you want a unified build to handle all possible screens.

```jsx Import Example
// root import
import { useMfaVoiceChallenge } from '@auth0/auth0-acul-react';

// partial import
import {
  useMfaVoiceChallenge,
  // Context hooks
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Common hooks
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Utility hooks
  useChangeLanguage,
  useResend,
  // Methods
  continueMethod,
  pickPhone,
  resendCode,
  switchToSms,
  tryAnotherMethod,
} from "@auth0/auth0-acul-react/mfa-voice-challenge";

function MfaVoiceChallengeScreen() {
  const { continueMethod } = useMfaVoiceChallenge();
  return (
    <button onClick={() => continueMethod({ code: '123456' })}>
      Verify Code
    </button>
  );
}
```

## Methods

<ParamField body='continueMethod' type={<span>void | Promise&lt;void&gt;</span>}>

This method submits the voice call one-time code to complete the MFA challenge.

```jsx Example
import { useMfaVoiceChallenge } from '@auth0/auth0-acul-react/mfa-voice-challenge';

function VerifyCodeForm() {
  const { continueMethod } = useMfaVoiceChallenge();
  return (
    <button onClick={() => continueMethod({ code: '123456' })}>
      Verify Code
    </button>
  );
}
```
