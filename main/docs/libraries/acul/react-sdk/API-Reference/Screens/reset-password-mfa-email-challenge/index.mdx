The `reset-password-mfa-email-challenge` screen is displayed during the password reset flow when multi-factor authentication is required via email. It prompts the user to enter a verification code sent to their email address to continue the reset process.

## Import

Each screen has its own set of hooks and methods. The SDK supports **partial import** and **root import** for each screen.
* Using partial import allows you to include only the code you need for your specific use case.
* Using root import allows you to load all screens from a single bundle, which is useful when you want a unified build to handle all possible screens.

```jsx Import Example
// root import
import { useResetPasswordMfaEmailChallenge } from '@auth0/auth0-acul-react';

// partial import
import {
  useResetPasswordMfaEmailChallenge,
  // Context hooks
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Common hooks
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  useResend,
  // Utility hooks
  useChangeLanguage,
  // Methods
  continueMethod,
  resendCode,
  tryAnotherMethod,
} from '@auth0/auth0-acul-react/reset-password-mfa-email-challenge';

function ResetPasswordMfaEmailChallengeScreen() {
  const { continueMethod } = useResetPasswordMfaEmailChallenge();
  return (
    <button onClick={() => continueMethod({ code: '123456' })}>
      Verify Code
    </button>
  );
}
```

## Context Hooks

Screen-scoped hooks that provide read-only access to Auth0 context data on the `reset-password-mfa-email-challenge` screen. Import them from `@auth0/auth0-acul-react/reset-password-mfa-email-challenge`.

<ParamField body='useOrganization' type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>

This hook provides information about the user's Organization if the password reset is Organization-scoped. Returns `null` when no Organization context is present.

```jsx Example

}
```

## Methods

<ParamField body='continueMethod' type={<span>void | Promise&lt;void&gt;</span>}>

This method submits the email verification code to continue the MFA step in the password reset flow.

```jsx Example
import { useResetPasswordMfaEmailChallenge } from '@auth0/auth0-acul-react/reset-password-mfa-email-challenge';

function VerifyCodeForm() {
  const { continueMethod } = useResetPasswordMfaEmailChallenge();
  return (
    <button onClick={() => continueMethod({ code: '123456' })}>
      Verify Code
    </button>
  );
}
```

<ParamField body='code' type='string' required>
The verification code entered by the user.
</ParamField>

<ParamField body='resendCode' type={<span>void | Promise&lt;void&gt;</span>}>

This method resends the email verification code to the user.

```jsx Example
import { useResetPasswordMfaEmailChallenge } from '@auth0/auth0-acul-react/reset-password-mfa-email-challenge';

function ResendButton() {
  const { resendCode } = useResetPasswordMfaEmailChallenge();
  return (
    <button onClick={() => resendCode()}>
      Resend Code
    </button>
  );
}
```
