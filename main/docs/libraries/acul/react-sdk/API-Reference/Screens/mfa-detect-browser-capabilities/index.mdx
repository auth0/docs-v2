The `mfa-detect-browser-capabilities` screen automatically detects the browser's supported MFA capabilities and routes the user to the appropriate factor enrollment or challenge flow.

## Import

Each screen has its own set of hooks and methods. The SDK supports **partial import** and **root import** for each screen.
* Using partial import allows you include only the code you need for your specific use case.
* Using root import allows you to load all screens from a single bundle useful when you want a unified build to handle all possible screens.

```jsx Import Example
// root import
import { useMfaDetectBrowserCapabilities } from '@auth0/auth0-acul-react';

// partial import
import {
  useMfaDetectBrowserCapabilities,
  // Context hooks
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Common hooks
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Utility hooks
  useChangeLanguage,
  // Methods
  detectCapabilities,
} from "@auth0/auth0-acul-react/mfa-detect-browser-capabilities";

function MfaDetectBrowserCapabilitiesScreen() {
  const { detectCapabilities } = useMfaDetectBrowserCapabilities();
  return (
    <button onClick={() => detectCapabilities()}>
      Detect Capabilities
    </button>
  );
}
```

## Methods

<ParamField body='detectCapabilities' type={<span>void | Promise&lt;void&gt;</span>}>

This method triggers browser capability detection to determine which MFA factors are supported and advances the flow accordingly.

```jsx Example
import { useMfaDetectBrowserCapabilities } from '@auth0/auth0-acul-react/mfa-detect-browser-capabilities';

function DetectCapabilities() {
  const { detectCapabilities } = useMfaDetectBrowserCapabilities();
  return (
    <button onClick={() => detectCapabilities()}>
      Continue
    </button>
  );
}
```
