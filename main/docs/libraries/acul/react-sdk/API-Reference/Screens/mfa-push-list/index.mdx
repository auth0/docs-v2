The `mfa-push-list` screen displays a list of enrolled push notification devices and allows the user to select one to use for authentication.

## Import

Each screen has its own set of hooks and methods. The SDK supports **partial import** and **root import** for each screen.
* Using partial import allows you include only the code you need for your specific use case.
* Using root import allows you to load all screens from a single bundle useful when you want a unified build to handle all possible screens.

```jsx Import Example
// root import
import { useMfaPushList } from '@auth0/auth0-acul-react';

// partial import
import {
  useMfaPushList,
  // Context hooks
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Common hooks
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Utility hooks
  useChangeLanguage,
  // Methods
  goBack,
  selectMfaPushDevice,
} from "@auth0/auth0-acul-react/mfa-push-list";

function MfaPushListScreen() {
  const { selectMfaPushDevice } = useMfaPushList();
  return (
    <button onClick={() => selectMfaPushDevice({ deviceIndex: 0 })}>
      Select Device
    </button>
  );
}
```

## Methods

<ParamField body='goBack' type={<span>void | Promise&lt;void&gt;</span>}>

This method navigates back to the previous screen in the MFA flow.

```jsx Example
import { useMfaPushList } from '@auth0/auth0-acul-react/mfa-push-list';

function GoBackButton() {
  const { goBack } = useMfaPushList();
  return (
    <button onClick={() => goBack()}>
      Go Back
    </button>
  );
}
```

<ParamField body='selectMfaPushDevice' type={<span>void | Promise&lt;void&gt;</span>}>

This method selects a push notification device from the enrolled device list to use for the MFA challenge.

```jsx Example

      Use This Device

  );
}
```
