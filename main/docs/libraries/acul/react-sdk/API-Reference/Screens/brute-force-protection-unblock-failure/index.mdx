The `brute-force-protection-unblock-failure` screen is displayed when an attempt to unblock a user's account fails, informing the user that the unblock request could not be completed.

<Frame><img style={{maxHeight:"400px"}} src="/docs/images/cdy7uua7fh8z/ACUL/brute-force-protection-unblock-failure.png" alt="ACUL Brute Force Protection Unblock Failure"/></Frame>

## Import

Each screen has its own set of hooks and methods. The SDK supports **partial import** and **root import** for each screen.
* Using partial import allows you to include only the code you need for your specific use case.
* Using root import allows you to load all screens from a single bundle, which is useful when you want a unified build to handle all possible screens.

```jsx Import Example
// root import
import { useBruteForceProtectionUnblockFailure } from '@auth0/auth0-acul-react';

// partial import
import {
  useBruteForceProtectionUnblockFailure,
  // Context hooks
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Common hooks
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Utility hooks
  useChangeLanguage,
} from '@auth0/auth0-acul-react/brute-force-protection-unblock-failure';

function UnblockFailureScreen() {
  const errors = useErrors();
  return <p>Account unblock failed. Please try again or contact support.</p>;
}
```

## Context Hooks

Screen-scoped hooks that provide read-only access to Auth0 context data on the `brute-force-protection-unblock-failure` screen. Import them from `@auth0/auth0-acul-react/brute-force-protection-unblock-failure`.

<ParamField body='useBranding' type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>

This hook provides branding configurations, such as logo, colors, and theme settings displayed on the `brute-force-protection-unblock-failure` screen.

```jsx Example
import { useBranding } from '@auth0/auth0-acul-react/brute-force-protection-unblock-failure';
function CustomTheme() {
  const branding = useBranding();
}
```
</ParamField>

<ParamField body='useClient' type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>

This hook provides client-related configurations, such as `id`, `name`, and `logoUrl`, for the `brute-force-protection-unblock-failure` screen.

```jsx Example
import { useClient } from '@auth0/auth0-acul-react/brute-force-protection-unblock-failure';
function AppInfo() {
  const client = useClient();
}
```
</ParamField>

<ParamField body='useOrganization' type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>

This hook provides information about the user's Organization if the unblock flow is Organization scoped. Returns `null` when no Organization context is present.

```jsx Example
import { useOrganization } from '@auth0/auth0-acul-react/brute-force-protection-unblock-failure';
function OrgSelector() {
  const organization = useOrganization();
  if (!organization) {
    return <p>No organization context</p>;
  }
}
```
</ParamField>

<ParamField body='usePrompt' type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>

This hook contains data about the current prompt in the authentication flow.

```jsx Example
import { usePrompt } from '@auth0/auth0-acul-react/brute-force-protection-unblock-failure';
function FlowInfo() {
  const prompt = usePrompt();
}
```
</ParamField>
