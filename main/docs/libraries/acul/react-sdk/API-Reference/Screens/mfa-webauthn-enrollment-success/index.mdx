The `mfa-webauthn-enrollment-success` screen confirms that the user has successfully enrolled a WebAuthn security key and allows them to continue the authentication flow.

## Import

Each screen has its own set of hooks and methods. The SDK supports **partial import** and **root import** for each screen.
* Using partial import allows you include only the code you need for your specific use case.
* Using root import allows you to load all screens from a single bundle useful when you want a unified build to handle all possible screens.

```jsx Import Example
// root import
import { useMfaWebAuthnEnrollmentSuccess } from '@auth0/auth0-acul-react';

// partial import
import {
  useMfaWebAuthnEnrollmentSuccess,
  // Context hooks
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Common hooks
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Utility hooks
  useChangeLanguage,
  // Methods
  continueMethod,
} from "@auth0/auth0-acul-react/mfa-webauthn-enrollment-success";

function MfaWebAuthnEnrollmentSuccessScreen() {
  const { continueMethod } = useMfaWebAuthnEnrollmentSuccess();
  return (
    <button onClick={() => continueMethod()}>
      Continue
    </button>
  );
}
```

## Methods

<ParamField body='continueMethod' type={<span>void | Promise&lt;void&gt;</span>}>

This method advances the flow after the user has successfully enrolled a WebAuthn security key.

```jsx Example

function ContinueButton() {
  const { continueMethod } = useMfaWebAuthnEnrollmentSuccess();
  return (
    <button onClick={() => continueMethod()}>
      Continue
    </button>
  );
}
```
