The `mfa-webauthn-roaming-challenge` screen prompts the user to complete an MFA challenge using a roaming security key, such as a YubiKey or other FIDO2 hardware authenticator.

## Import

Each screen has its own set of hooks and methods. The SDK supports **partial import** and **root import** for each screen.
* Using partial import allows you include only the code you need for your specific use case.
* Using root import allows you to load all screens from a single bundle useful when you want a unified build to handle all possible screens.

```jsx Import Example
// root import
import { useMfaWebAuthnRoamingChallenge } from '@auth0/auth0-acul-react';

// partial import
import {
  useMfaWebAuthnRoamingChallenge,
  // Context hooks
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Common hooks
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Utility hooks
  useChangeLanguage,
  // Methods
  reportWebAuthnError,
  tryAnotherMethod,
  verify,
} from "@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge";

function MfaWebAuthnRoamingChallengeScreen() {
  const { verify } = useMfaWebAuthnRoamingChallenge();
  return (
    <button onClick={() => verify({ rememberDevice: true })}>
      Verify with Security Key
    </button>
  );
}
```

## Methods

<ParamField body='reportWebAuthnError' type={<span>void | Promise&lt;void&gt;</span>}>

This method reports a WebAuthn error encountered during the roaming security key challenge.

```jsx Example
import { useMfaWebAuthnRoamingChallenge } from '@auth0/auth0-acul-react/mfa-webauthn-roaming-challenge';

function HandleWebAuthnError({ error }) {
  const { reportWebAuthnError } = useMfaWebAuthnRoamingChallenge();
  return (
    <button onClick={() => reportWebAuthnError({ error })}>
      Report Error
    </button>
  );
}
```
