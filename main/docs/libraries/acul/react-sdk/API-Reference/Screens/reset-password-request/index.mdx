The `reset-password-request` screen is displayed at the start of the password reset flow. It prompts the user to enter their username or email address to receive a password reset link.

## Import

Each screen has its own set of hooks and methods. The SDK supports **partial import** and **root import** for each screen.
* Using partial import allows you to include only the code you need for your specific use case.
* Using root import allows you to load all screens from a single bundle useful when you want a unified build to handle all possible screens.

```jsx Import Example
// root import
import { useResetPasswordRequest } from '@auth0/auth0-acul-react';

// partial import
import {
  useResetPasswordRequest,
  // Context hooks
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Common hooks
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  useLoginIdentifiers,
  // Utility hooks
  useChangeLanguage,
  // Methods
  backToLogin,
  resetPassword,
} from '@auth0/auth0-acul-react/reset-password-request';

function ResetPasswordRequestScreen() {
  const { resetPassword } = useResetPasswordRequest();
  return (
    <button onClick={() => resetPassword({ username: 'user@example.com' })}>
      Send Reset Link
    </button>
  );
}
```

## Context Hooks

Screen-scoped hooks that provide read-only access to Auth0 context data on the `reset-password-request` screen. Import them from `@auth0/auth0-acul-react/reset-password-request`.

<ParamField body='useBranding' type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>

This hook provides branding configurations, such as logo, colors, and theme settings displayed on the `reset-password-request` screen.

```jsx Example
import { useBranding } from '@auth0/auth0-acul-react/reset-password-request';
function CustomTheme() {
  const branding = useBranding();
}
```
</ParamField>

<ParamField body='useClient' type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>

This hook provides client-related configurations, such as `id`, `name`, and `logoUrl`, for the `reset-password-request` screen.

```jsx Example
import { useClient } from '@auth0/auth0-acul-react/reset-password-request';
function AppInfo() {
  const client = useClient();
}
```
</ParamField>

<ParamField body='useOrganization' type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>

This hook provides information about the user's Organization if the password reset is Organization-scoped. Returns `null` when no Organization context is present.

```jsx Example
import { useOrganization } from '@auth0/auth0-acul-react/reset-password-request';
function OrgSelector() {
  const organization = useOrganization();
  if (!organization) {
    return <p>No Organization context</p>;
  }
}
```
</ParamField>

## Methods

<ParamField body='backToLogin' type={<span>void | Promise&lt;void&gt;</span>}>

This method navigates the user back to the login screen, cancelling the password reset flow.

```jsx Example
import { useResetPasswordRequest } from '@auth0/auth0-acul-react/reset-password-request';

function BackToLoginButton() {
  const { backToLogin } = useResetPasswordRequest();
  return (
    <button onClick={() => backToLogin()}>
      Back to Login
    </button>
  );
}
```
