---
title: "ErrorItem"
---

Represents a single error entry in the ACUL error store. Returned by [`useErrors`](/docs/libraries/acul/react-sdk/API-Reference/Hooks/useErrors) as part of the [`UseErrorsResult`](/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorsResult) object. Extends the base `Error` interface with an `id`, `kind`, and optional `label`.

```typescript
interface ErrorItem {
  id: string;
  code: string;
  message: string;
  kind?: ErrorType;
  field?: string;
  label?: string;
  rules?: PasswordComplexityRule[];
}
```

## Properties

<ParamField body='id' type='string' required>
  A unique identifier for this error instance. Used to target a specific error with `dismiss(id)`.
</ParamField>

<ParamField body='code' type='string' required>
  A machine-readable error code identifying the type of error. Example: `"invalid_password"`, `"username_exists"`.
</ParamField>

<ParamField body='message' type='string' required>
  A human-readable description of the error, suitable for display in the UI.
</ParamField>

<ParamField body='kind' type={<a href="/docs/libraries/acul/react-sdk/API-Reference/Types/type-aliases/ErrorType">ErrorType</a>}>
  Classifies the error as `"auth0"`, `"validation"`, or `"configuration"`. Used for filtering with `errors.byType()`.
</ParamField>

<ParamField body='field' type='string'>
  The form field associated with the error, if applicable. Example: `"username"`, `"email"`. Used for filtering with `errors.byField()`.
</ParamField>

<ParamField body='label' type='string'>
  An optional display label for the error, used when a human-friendly field name is needed alongside the error message.
</ParamField>

<ParamField body='rules' type={<span><a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/PasswordComplexityRule">PasswordComplexityRule</a>[]</span>}>
  Password complexity rules associated with this error, when the error relates to password validation.
</ParamField>
