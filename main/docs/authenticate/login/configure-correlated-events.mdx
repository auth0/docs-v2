Auth0 provides the capability to trace authentication transactions with Correlated Events. This tracking capability can reduce support response time and provide you with multi-system tracking downstream of authentication events. 

The `correlation_id` parameter adds a unique identifier generated by your application to the authorization URL. When authentication events occur, track and filter events in [Auth0 Tenant Logs](/docs/deploy-monitor/logs).

Correlation ID persists through the following events:
*  Signup
*  Login
*  Multi-Factor Authentication enrollment
*  Password reset

Correlated Events is supported with Auth0 Universal Login, Universal Login Page Templates, Flows, and Auth0 Actions.

## Configure correlation ID

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
Values for `correlation_id` should be strings no longer than 64 characters and **do not** contain any Personally Identifiable Information (PII).
</Callout>

Pass the `correlation_id` parameter in the `authorizationParams` object to appends the unique ID to the authorization URL similar to the following example:

```JavaScript
const { loginWithRedirect } = useAuth0();
loginWithRedirect({
  authorizationParams: {
    // Appends "&correlation_id=..." to the URL for tracking
    correlation_id: "txn_12345_xyz" 
  }
});
```

## Universal Login and page template

All Universal Login flows support `correlation_id`. If you customize your login experience with [Universal Login Page Templates](/docs/customize/login-pages/universal-login/customize-templates), add the `correlation_id` to your template to track authentication events with custom login:

```html
<!DOCTYPE html>
<html>
  <head>
    {%- auth0:head -%}
    <style>
      /* Ensure the debug box stays on top of the Auth0 widget */
      #debug-payload-container {
        position: relative;
        z-index: 9999; 
        margin: 20px;
        padding: 15px;
        background-color: #ffffff !important; /* Force white background */
        color: #000000 !important;           /* Force black text */
        border: 2px solid #ff0000;           /* Red border to make it obvious */
        border-radius: 8px;
        font-family: monospace;
        white-space: pre-wrap;
        box-shadow: 0px 4px 10px rgba(0,0,0,0.2);
      }
      .debug-label {
        font-weight: bold;
        display: block;
        border-bottom: 1px solid #ccc;
        margin-bottom: 10px;
        padding-bottom: 5px;
      }
    </style>
  </head>

  <body class="_widget-auto-layout">

    {%- auth0:widget -%}

    <div id="debug-payload-container">
      <span class="debug-label">DEBUG: Liquid Payload Data</span>
      <pre id="payload-display">Waiting for Javascript to load...</pre>
    </div>

    <script>
      window.addEventListener("load", function () {
        try {
          // Build JSON payload
          const payload = {
            application: "{{application.name}}",
            promptName: "{{prompt.name}}",
            screenName: "{{prompt.screen.name}}",
            correlationId: "{{correlation_id}}" // 
          };

          // Update the display
          const displayElement = document.getElementById("payload-display");
          if (displayElement) {
            // If the payload is empty, show a specific message
            const jsonString = JSON.stringify(payload, null, 2);
            displayElement.textContent = jsonString;
          }
        } catch (e) {
          document.getElementById("payload-display").textContent = "Error parsing data: " + e.message;
        }
      });
    </script>
  </body>
</html>
```

## Auth0 Actions

Use the `correlation_id` parameter with Auth0 Actions Signup and Login Triggers. The `event.transaction` object for the following triggers:

* [Pre-user-registration](/docs/customize/actions/explore-triggers/signup-and-login-triggers/pre-user-registration-trigger/pre-user-registration-event-object)
* [Post-user-registration](/docs/customize/actions/explore-triggers/signup-and-login-triggers/post-user-registration-trigger/post-user-registration-event-object)
* [Post-login](/docs/customize/actions/explore-triggers/signup-and-login-triggers/login-trigger/post-login-event-object)
* [Post-challenge](/docs/customize/actions/explore-triggers/password-reset-triggers/post-challenge-trigger/post-challenge-event-object)

## Management API

To use correlation_id with Management API, set the `correlation_id`in the X-Correlation-ID header of your Management API calls:
```
curl -H "Authorization: Bearer eyJhb..." -H "x-correlation-id: client1_xyz" https://@@TENANT@@/api/v2/users
```

To learn more, read [Management API: Request Correlation](https://auth0.com/docs/api/management/v2/introduction#request-correlation).
