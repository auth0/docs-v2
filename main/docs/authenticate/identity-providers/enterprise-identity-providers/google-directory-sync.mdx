Enabling Directory Sync for your [Google Workspace connection](/docs/authenticate/identity-providers/enterprise-identity-providers/google-apps) lets you synchronize the user profiles, group structures, and group membership from Google Workspace to Auth0. You can synchronize automatically or manually:

* Automatic synchronization runs every 30 minutes after the last sync completes. 

* Manual synchronization runs when you trigger it. If it has been less than 30 minutes since the last completed sync, it returns a 400 error.

## Enable Directory Sync

<Steps>

<Step title="Enable the admin directory API for your connection">
To enable Directory Sync, you must have a [Google Workspace connection](/docs/authenticate/identity-providers/enterprise-identity-providers/google-apps) with the admin directory API enabled.

You can enable the admin directory API either while creating a new connection or on the **Settings** tab of an existing connection.

The enterprise connection creation page and the **Settings** tab of existing Google Workspace connections both have an **Identity Provider API** section.

In this section, check the **Use Admin Directory API** box, then choose:

* **Users scopes** to enable Directory Sync only for users.

* **Users and Groups scopes** to enable Directory Sync for both users and groups.

Save these settings. Then, in the **Setup** tab of the connection, complete administrator consent.
</Step>

<Step title="Enable Directory Sync">

<Tabs>
<Tab title="Auth0 Dashboard">
1. From [Auth0 Dashboard > Authentication > Enterprise](https://manage.auth0.com/#/connections/enterprise), open your Google Workspace connection.

2. On the **Provisioning** tab, toggle **Provision Users Using Directory Sync**.

3. Configure **Attribute Mapping** to control which Google attributes are written into the Auth0 user profile.

4. Optionally, configure **Enable Automated Synchronizations** to enable automatic sync. You can run an on-demand sync by selecting **Synchronize Now**.
</Tab>

<Tab title="Management API">
To call the Management API, you need:

* A [Management API access token](/docs/secure/tokens/access-tokens/management-api-access-tokens#management-api-access-tokens) with the following scopes:
    * `create:directory_provisionings`
    * `read:directory_provisionings`
    * `update:directory_provisionings`
    * `delete:directory_provisionings`

* Your Google Workspace connection ID (`CONNECTION_ID`).

To enable Directory Sync with automatic synchronization, use the [Create a directory provisioning configuration](https://auth0.com/docs/api/management/v2/connections/post-directory-provisioning) endpoint.

Use the request body to enable automatic sync:

```json
{
    "mapping": [
        {
            "idp": "id",
            "auth0": "id"
        },
        {
            "idp": "primaryEmail",
            "auth0": "email"
        },
        [...]
        {
            "idp": "id",
            "auth0": "app_metadata.externalId"
        },
    ],
    "synchronize_automatically": true
}
```

* You can read the configuration settings with the [Get a directory provisioning configuration endpoint](https://auth0.com/docs/api/management/v2/connections/get-directory-provisioning)
* You can update the configuration settings with the [Patch a directory provisioning configuration endpoint](https://auth0.com/docs/api/management/v2/connections/patch-directory-provisioning).
* To trigger a manual sync, use the [Create a directory provisioning configuration endpoint](https://auth0.com/docs/api/management/v2/connections/post-synchronizations).
* To disable Directory Sync for the connection, call the [Delete a directory provisioning configuration endpoint](https://auth0.com/docs/api/management/v2/connections/delete-directory-provisioning).
</Tab>
</Tabs>

</Step>

</Steps>

## Best Practices

* For new Google Workspace connections, we recommend using the Google user `id` instead of email for the Auth0 `user_id`.

* If you enable Directory Sync, consider disabling **Sync User Profile Attributes at Login** to avoid conflicting updates from multiple sync methods.
