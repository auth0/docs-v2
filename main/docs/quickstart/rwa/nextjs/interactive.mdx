## Advanced Usage

### Authentication Routes

The SDK automatically mounts these routes via middleware:

| Route                      | Purpose                   |
| -------------------------- | ------------------------- |
| `/auth/login`              | Initiate login            |
| `/auth/logout`             | Logout user               |
| `/auth/callback`           | Handle Auth0 callback     |
| `/auth/profile`            | Get user profile          |
| `/auth/access-token`       | Get access token          |
| `/auth/backchannel-logout` | Handle backchannel logout |

### Protecting a Server-Side Rendered (SSR) Page

The Auth0 Next.js SDK v4 supports both App Router and Pages Router patterns for server-side rendered pages.

<Tabs>
  <Tab title="App Router">
    ```typescript app/protected/page.tsx
    import { auth0 } from "@/lib/auth0";
    
    // Requests to /protected without a valid session cookie 
    // will be redirected to the login page.
    export default auth0.withPageAuthRequired(
      async function Profile() {
        const { user } = await auth0.getSession();
        return (
    		<div>
        	  <h1>Protected Content</h1>
    	      <p>Welcome, {user.name}!</p>
    	    </div>
    	);
      },
      { returnTo: "/protected" }
    );
    // You need to provide a `returnTo` since Server Components aren't aware of the page's URL
    ```
  </Tab>
  <Tab title="Pages Router ">
    ```typescript pages/protected.tsx
    import { auth0 } from "@/lib/auth0";
    import { GetServerSideProps } from "next";
    
    // Requests to /pages/protected without a valid session cookie 
    // will be redirected to the login page.
    export default function ProtectedPage({ user }: { user: any }) {
      return (
        <div>
          <h1>Protected Content</h1>
          <p>Welcome, {user.name}!</p>
        </div>
      );
    }
    
    export const getServerSideProps: GetServerSideProps = auth0.withPageAuthRequired();
    ```
  </Tab>
</Tabs>

### **Protecting a Client-Side Rendered (CSR) Page**

To protect a Client-Side Rendered (CSR) page, you can use the `withPageAuthRequired` higher-order function. Requests to `/protected` without a valid session cookie will be redirected to the login page.

```typescript app/protected/page.tsx
import { withPageAuthRequired } from "@auth0/nextjs-auth0";

  return <div>Hello, {user.name}!</div>;
});
```

</CodeGroup>

### Accessing the authenticated user on the server-side

<Tabs>
  <Tab title="App Router">
    On the server, the `getSession()` helper can be used in Server Components, Server Routes, and Server Actions to get the session of the currently authenticated user and to protect resources.

    <Note>
      The getSession() method returns a complete session object containing the user profile and all available tokens (access token, ID token, and refresh token when present). Use this method for applications that only need user identity information without calling external APIs, as it provides access to the user's profile data from the ID token without requiring additional API calls. This approach is suitable for session-only authentication patterns. For API access, use getAccessToken() to get an access token, this handles automatic token refresh.
    </Note>
    <CodeGroup>

    ```typescript app/protected/page.tsx
    import { auth0 } from "@/lib/auth0";
    
    export default async function Home() {
      const session = await auth0.getSession();
    
      if (!session) {
        return <div>Not authenticated</div>;
      }
    
      return (
        <main>
          <h1>Welcome, {session.user.name}!</h1>
        </main>
      );
    }
    
    ```

    </CodeGroup>
  </Tab>
  <Tab title="Pages Router">
    On the server, the `getSession(req)` helper can be used in `getServerSideProps` and API routes to get the session of the currently authenticated user and to protect resources, like so:

    <CodeGroup>

    ```typescript pages/protected.tsx
    import type { GetServerSideProps, InferGetServerSidePropsType } from "next";
    
    import { auth0 } from "@/lib/auth0";
    
    export const getServerSideProps = (async (ctx) => {
      const session = await auth0.getSession(ctx.req);
    
      if (!session) return { props: { user: null } };
    
      return { props: { user: session.user ?? null } };
    }) satisfies GetServerSideProps<{ user: any | null }>;
    
    export default function Page({
      user
    }: InferGetServerSidePropsType<typeof getServerSideProps>) {
      if (!user) {
        return (
          <main>
            <p>Not authenticated!</p>
          </main>
        );
      }
    
      return (
        <main>
          <p>Welcome, {user.name}!</p>
        </main>
      );
    }
    ```

    </CodeGroup>
  </Tab>
  <Tab title="Middleware">
    In middleware, the `getSession(req)` helper can be used to get the session of the currently authenticated user and to protect resources, like so:

    <CodeGroup>

    ```typescript middleware.ts
    import { NextRequest, NextResponse } from "next/server";
    
    import { auth0 } from "@/lib/auth0";
    
    export async function middleware(request: NextRequest) {
      const authRes = await auth0.middleware(request);
    
      if (request.nextUrl.pathname.startsWith("/auth")) {
        return authRes;
      }
    
      const session = await auth0.getSession(request);
    
      if (!session) {
        // user is not authenticated, redirect to login page
        return NextResponse.redirect(
          new URL("/auth/login", request.nextUrl.origin)
        );
      }
    
      // the headers from the auth middleware should always be returned
      return authRes;
    }
    ```

    </CodeGroup>
  </Tab>
</Tabs>
