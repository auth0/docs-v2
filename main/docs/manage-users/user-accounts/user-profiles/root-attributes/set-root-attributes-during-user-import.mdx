This guide will show you how to set root attributes for a user during import using Auth0's <Tooltip tip="Management API: A product to allow customers to perform administrative tasks." cta="View Glossary" href="/docs/glossary?term=Management+API">Management API</Tooltip>. This allows you to minimize the number of API calls required to set root attributes when importing users. To see which attributes you can import, visit [Normalized User Profile Structure](/docs/manage-users/user-accounts/user-profiles/user-profile-structure).

Make a `POST` call to the [Create Job to Import Users endpoint](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_imports). Be sure to replace `MGMT_API_ACCESS_TOKEN`, `CONNECTION_ID`, and `JSON_USER_FILE_PATH` placeholder values with your Management API <Tooltip tip="Access Token: Authorization credential, in the form of an opaque string or JWT, used to access an API." cta="View Glossary" href="/docs/glossary?term=Access+Token">Access Token</Tooltip>, connection ID, and users filename, respectively.

</tr>
<tr>
<td><code>CONNECTION_ID</code></td>
<td>ID of the connection to which the users will be inserted. You can retrieve this info using the <a href="https://auth0.com/docs/api/management/v2#!/Connections/get_connections">Get All Connections endpoint</a>.</td>
</tr>
<tr>
<td><code>JSON_USER_FILE_PATH</code></td>
<td>Filename of the file that contains the users to be imported. File should be in JSON format and include root attributes for users. For a list of available attributes, see <a href="/docs/manage-users/user-accounts/user-profiles/user-profile-structure#attributes">User Profile Attributes</a>. For an example of the file format, see <a href="/docs/manage-users/user-migration/bulk-user-import-database-schema-and-examples">Bulk User Import Database Schema and Examples</a>.</td>
