L’[Auh0 Dashboard](https://manage.auth0.com/#) est l’endroit où vous gérez tous les aspects de votre abonnement et de votre configuration Auth0.

<Frame>![Auth0 Dashboard Activity page](/docs/images/fr-ca/cdy7uua7fh8z/6k0cRLNEPko6vY6Kw7nIQD/de19631d9c51fa36d227b5a3520d1c1a/2025-02-27_16-08-53_copy.png)</Frame>

Il se compose de plusieurs sections dans lesquelles vous pouvez naviguer à l’aide du menu latéral sur votre gauche.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

Pour connaître les meilleures pratiques concernant l’utilisation du tableau de bord Teams, voir [Meilleures pratiques de l’utilisation générale et des opérations](/docs/fr-ca/troubleshoot/general-usage-and-operations-best-practices).

</Callout>

## Configurer l’implémentation

Le tableau suivant contient un bref aperçu des différentes sections du Tableau de bord et de ce que vous pouvez faire dans chacune d’entre elles.

| Section | Description |
| --- | --- |
| **Applications** | Gérez vos applications, vos API et vos intégrations d’authentification unique (SSO).<br/>**Applications**: Pour chacune de vos applications pour lesquelles vous souhaitez authentifier les utilisateurs avec Auth0, enregistrez une application.<br/>**API**: Pour chacune de vos API que vous souhaitez sécuriser avec Auth0, enregistrez une API. Créez de nouvelles API et gérez les API existantes.<br/>**Intégrations SSO**: Afficher et activer les services externes pour SSO. Créer de nouvelles intégrations SSO et configurer, examiner et gérer les paramètres d’intégration. |
| **Authentification** | Gérer les fournisseurs d’identité à travers lesquels vous permettez aux utilisateurs de s’authentifier auprès de vos applications.<br/>**Base de données**: Stocker et gérer en toute sécurité les identifiants/mots de passe dans un magasin de données Auth0 ou dans votre propre base de données. Connectez-vous aux bases de données existantes en utilisant des scripts JavaScript basés sur des modèles qui s’exécutent sur le serveur d’Auth0 lors de chaque authentification. Migrer progressivement une base de données existante d’identifiants hérités vers Auth0 au fur et à mesure que les utilisateurs s’authentifient (aucune réinitialisation du mot de passe requise).<br/>**Social**: Configurer les fournisseurs d’identité sociale (tels que Facebook, X et Github) par l’intermédiaire desquels vos utilisateurs peuvent se connecter.<br/>**Enterprise**: Configurez les fournisseurs d’identité de l’entreprise (tels qu’Active Directory, SAML et Office 365) à travers lesquels vos utilisateurs peuvent se connecter en utilisant leurs identifiants d’entreprise.<br/>**Sans mot de passe**: Permettre à vos utilisateurs de s’inscrire et de se connecter en utilisant des codes d’accès uniques (envoyés par courriel ou SMS) ou des liens en un clic, au lieu de mots de passe. |
| **Organizations** | Gérer les organizations avec lesquelles vous faites affaire et personnaliser l’expérience de leurs utilisateurs lorsqu’ils accèdent à vos applications. |
| **Gestion des utilisateurs** | Gérez les identités et autorisations de vos utilisateurs.<br/>**Utilisateurs**: Afficher et créer des profils utilisateurs, réinitialiser les mots de passe, bloquer et supprimer des utilisateurs, etc..<br/>**Rôles**: Créer et gérer des rôles pour vos applications. Les rôles contiennent des collections d’autorisations et peuvent être attribués aux utilisateurs. |
| **Image de marque** | **Connexion universelle**: Créer et personnaliser une page de connexion à laquelle vous pouvez diriger les utilisateurs pour s’authentifier.<br/>**Domaines personnalisés**: Créez un domaine personnalisé pour maintenir une expérience cohérente pour vos utilisateurs.<br/>**Modèles de courriel**: Utiliser des modèles pour créer des flux de travail basés sur le courriel d’accueil, la réinitialisation du mot de passe et la vérification du compte.<br/>**Fournisseur de messagerie électronique**: Désigner et configurer les informations de votre fournisseur de messagerie électronique personnalisé. |
| **Sécurité** | Configurez des couches de sécurité supplémentaires en activant des boucliers qui protègent vos utilisateurs contre différents types d’attaques et d’anomalies d’accès utilisateur.<br/>**Protection contre les attaques**: Gérer les paramètres pour les attaques par robots, de limitation d’IP, par force brute et par mots de passe violés.<br/>**Authentification multifacteur (MFA)**: Exiger des facteurs supplémentaires pendant le processus de connexion pour empêcher l’accès non autorisé.<br/>**Surveillance**: Surveillez les événements liés aux renseignements sur les menaces grâce à l’une de nos intégrations de visualisation et d’alerte des données. |
| **Actions** | Configurer les flux tels que la connexion, de machine à machine, d’enregistrement d’utilisateur et de réinitialisation des mots de passe. Créer et gérer des actions personnalisées utilisées dans les flux. |
| **Pipeline Auth** | **Règles**: Configurer des fragments JavaScript personnalisés qui sont exécutés dans Auth0 dans le cadre de chaque transaction d’authentification utilisateur. Vous pouvez appeler des API externes, filtrer les utilisateurs qui peuvent se connecter à votre application, utiliser une AllowList, configurer l’accès géolocalisé, etc.<br/>**Hooks**: Personnalisez le comportement d’Auth0 lorsque vous utilisez les connexions de base de données en configurant le code Node.js qui est exécuté contre des points d’extensibilité (qui sont comparables aux liens de rappel HTTP fournis avec un serveur). |
| **Surveillance** | **Journaux**: 	Affichez les données de journal des actions prises dans le tableau de bord par les administrateurs et les connexions utilisateur.<br/>**Flux**: Créez et gérez le flux d’événements de journal vers des services externes d’analyse de données. |
| **Marketplace** | Explorez les intégrations qui aident votre entreprise à faire plus avec Auth0. |
| **Extensions** | Étendez la plateforme Auth0 avec des modules complémentaires officiels et tiers. |
| **Paramètres** | Configurez vos locataires, gérez vos options d’abonnement et de paiement Auth0, contrôlez vos administrateurs locataires et autres rôles utilisateurs. Gérez les autres paramètres des locataires liés à vos domaines personnalisés, clés de connexion et autres paramètres avancés. |
| **Obtenir de l’assistance** | Allez à notre [Centre d’assistance](https://support.auth0.com). Si votre plan n’a pas accès aux services de soutien, consultez la [Communauté Auth0](https://community.auth0.com/). |

## Gérer les paramètres de compte

En haut à gauche, vous pouvez voir le nom et l’icône de votre locataire, ainsi qu’une petite flèche. Cette flèche affiche un menu déroulant que vous pouvez utiliser pour configurer différents aspects de votre compte :

* [**Settings (Paramètres)**](/docs/fr-ca/get-started/tenant-settings) : Configurez plusieurs aspects de votre locataire.
* [**Invite a member (Inviter un membre)**](/docs/fr-ca/get-started/manage-dashboard-access) : Ajoutez un utilisateur supplémentaire en tant qu’administrateur ou autre rôle à la configuration de votre locataire.
* [**Quota Utilization (Utilisation du quota)**](https://support.auth0.com/reports/quota) : Informations sur l’utilisation des quotas de votre abonnement et de vos locataires.
* [**Create tenant (Créer un locataire)**](/docs/fr-ca/get-started/auth0-overview/create-tenants) : Créez un nouveau locataire.
* **Switch tenant (Changer de locataire)** : Si vous avez [plusieurs locataires](/docs/fr-ca/get-started/auth0-overview/create-tenants/create-multiple-tenants), utilisez cette option pour passer de l’un à l’autre. Si vous créez une application pour un locataire, elle ne sera pas répertoriée pour un autre locataire.

En haut à droite, vous voyez plusieurs options cliquables :

* **Search (Recherche)** : Cherchez des applications, des entrées sur la place de marché, des utilisateurs et d’autres contenus liés à votre locataire.
* **Discuter de vos besoins** : Contactez un expert Auth0 qui vous aidera à atteindre vos objectifs avec Auth0.
* **Docs (Documentation) :** Un lien vers le site de documentation que vous êtes en train de lire.
* **Notifications bell (Cloche de notifications) :** Vous informe des nouvelles communications d’Auth0 ou vos paramètres du locataire.
* **Profile (Profil) :** Un lien vers votre profil de locataire local, ainsi que des options pour [change your theme (changer votre thème)](/docs/fr-ca/get-started/dashboard-profile/light-and-dark-themes) et vous déconnecter.
