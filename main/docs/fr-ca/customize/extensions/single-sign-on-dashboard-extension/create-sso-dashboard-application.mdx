title: "Créer une application pour le Dashboard à authentification unique (SSO)"
'description': "Cet article explique comment créer une application à utiliser avec l’extension Dashboard à authentification unique pour activer la connexion à authentification unique pour vos applications."
---

Utilisez l’extension Dashboard à authentification unique pour gérer la connexion à authentification unique de vos utilisateurs sur plusieurs applications d’entreprise. Pour en savoir plus, [examinez cette extension sur GitHub](https://github.com/auth0-extensions/auth0-sso-dashboard-extension).

## Créer une application Auth0

Avant [d’ajouter l’extension Dashboard à authentification unique](/docs/fr-ca/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension), vous devez créer une application dans Auth0 :

1. Allez dans [Auth0 Dashboard > Applications > Applications](https://manage.auth0.com/#/applications) et sélectionnez **+ Créer une application**.
2. Saisissez un nom descriptif de votre application (p. ex., Dashboard SSO), et sélectionnez **Application Web à page unique** sous **Type d’application**, puis cliquez sur **Créer**.
3. Dans l’onglet **Paramètres**, définissez les **URL de rappel autorisées** en fonction de la région de votre locataire :

   1. Pour les **administrateurs** : `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/admins/login`
   2. Pour les **utilisateurs** : `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/login`

      | Région | Domaine de la région |
| --- | --- |
| US-1 | us.webtask.io |
| US-3 | us.webtask.run |
| US-4 | us.webtask.run |
| EU | eu.webtask.io |
| EU-2 | eu.webtask.run |
| AU | au.webtask.io |
| JP-1 | jp.webtask.run |
| UK-1 | uk.webtask.run |
| CA-1 | ca.webtask.run |
4. Sélectionnez et copiez la valeur **ID client**.
5. Faites défiler jusqu’au bas de la page, puis sélectionnez **Paramètres avancés**.
6. Sélectionnez l’onglet **OAuth** , puis copiez la valeur **ID client** dans le champ **Applications/API autorisées**.
7. Définissez **Algorithme de signature JsonWebToken** sur **RS256**.
8. Sélectionnez **Enregistrer les modifications**.

Par défaut, tous les types de connexion sont activés pour que les utilisateurs puissent se connecter au Dashboard à authentification unique. Si vous souhaitez modifier ces paramètres, allez dans l’onglet **Connexions** de l’application.

## Installer l’extension Dashboard à authentification unique

Ensuite, vous devrez [installer l’extension Dashboard à authentification unique](/docs/fr-ca/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension) et [ajouter les applications](/docs/fr-ca/customize/extensions/single-sign-on-dashboard-extension/add-applications-to-the-sso-dashboard) sur le tableau de bord.
