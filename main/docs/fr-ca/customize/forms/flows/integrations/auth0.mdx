Cette liste d’Auth0 Actions vous permet d’obtenir, de créer et de mettre à jour des utilisateurs.

## Configurer la connexion à votre coffre-fort

Pour configurer une connexion au coffre-fort pour vos Auth0 Actions, vous devrez [créer une application de communication entre machines](/docs/fr-ca/get-started/auth0-overview/create-applications/machine-to-machine-apps)  et utiliser **le domaine du locataire, l’ID et le secret client** :

<Frame>![](/docs/images/fr-ca/cdy7uua7fh8z/1IPtH5hD6QTRXsySyo2Ovr/3562828624e2fd4f308b6eb3b5042f34/auth0-vault-connection.png)</Frame>

<Warning>

N’oubliez pas d’autoriser votre application de communication entre machines à accéder au Management API et de sélectionner les bonnes permissions (par exemple, `update:users_app_metadata`).

</Warning>

Créer un utilisateur.

<Frame>![](/docs/images/fr-ca/cdy7uua7fh8z/6MQIKf2IpTtI2g7d5Pwn7u/222ec746cf9c0c1003d18ba4c0aa451e/create_user.png)</Frame>

### Paramètres d’entrée

| Paramètre | Description |
| --- | --- |
| Corps | La charge utile des propriétés utilisateur Auth0 à ajouter. Pour en savoir plus, lisez la documentation API [Créer un utilisateur](https://auth0.com/docs/api/management/v2/users/post-users). |

### Paramètres de sortie

Vous trouverez les réponses attendues et les codes d’état dans la documentation de l’API [Créer un utilisateur](https://auth0.com/docs/api/management/v2/users/post-users#response-messages).

## Mettre à jour l’utilisateur

Mettre à jour un utilisateur.

<Frame>![](/docs/images/fr-ca/cdy7uua7fh8z/3tymSpaeod6EksWRJp1sge/1122b244d756d754fe41baac209a132b/Screenshot_2024-11-18_at_16.49.56.png)</Frame>

### Paramètres d’entrée

| Paramètre | Description |
| --- | --- |

### Paramètres de sortie

Vous trouverez la réponse attendue et les codes d’état dans la documentation de l’API [Mettre à jour un utilisateur](https://auth0.com/docs/api/management/v2/users/patch-users-by-id#response-messages).

## Demande d’API personnalisée

Envoyer une demande d’API personnalisée aux points de terminaison de l’API Auth0.

<Frame>![Dashboard > Customize > Flows > Integrations > Auth0](/docs/images/fr-ca/cdy7uua7fh8z/66uV9iuhwfnXUdE1quLOyx/2ae3ff26335acbcd172cbb40618a5a90/Screenshot_2025-01-24_at_18.50.26.png)</Frame>

### Paramètres d’entrée

| Parameter | Description |
| --- | --- |
| Path | The API endpoint absolute pathname.<br/>( Example: `/api/v2/organizations` )<br/>**Note:** Some endpoints, for example: [ Get a user ](https://auth0.com/docs/api/management/v2/users/get-users-by-id), require URI encoded parameters when not using alphanumeric characters, such as: `-`, `.`, `_`, or `~`, [ characters ](https://datatracker.ietf.org/doc/html/rfc3986#section-2.3)).<br/>To do this, you can use the encodeURIComponent function:<br/>`{{ functions.encodeURIComponent(context.user.user_id) }} ` |
| Method | Available HTTP methods include: <ul> <li>`GET`</li> <li>`POST`</li> <li>`PUT`</li> <li>`PATCH`</li> <li>`DELETE`</li> </ul> |
| Body | The body of the request. |
| Headers | The headers of the request. |
| URL Params | The URL parameters. These parameters are automatically URL-encoded. |

### Paramètres de sortie
