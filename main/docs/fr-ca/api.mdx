    <Column>
    La Authentication API expose les fonctionnalités d'identité pour Auth0 et les protocoles d'identité pris en charge (notamment OpenID Connect, OAuth, et SAML).

    Typiquement, vous devriez consommer cette API à travers l'une des trousses SDK Auth0, telles que [Auth0.js](/docs/libraries/auth0js), ou une librairie comme [Lock](/docs/libraries/lock). Toutefois, si vous créez votre interface graphique d'authentification manuellement, vous devrez appeler l'Authentication API directement.
    </Column>

    <Column>
    Quelques exemples de tâches comprennent:

    * Obtenir des [jetonss](/docs/secure/tokens) pendant l'authentification
    * Demander le profil d'un utilisateur en utilisant un [Jeton d'accès](/docs/secure/tokens/access-tokens)
    * Échanger des [jetons d'actualisation](/docs/secure/tokens/refresh-tokens) contre de nouveau jetons d'accès
    * Demander un défi d'authentification pour [l'authentification multifacteur (MFA)](/docs/secure/multi-factor-authentication)
    </Column>
</Columns>

## Pagination

Pour les locataires du nuage public, les points de terminaison Management API renvoient un maximum de 50 résultats par réponse. Si le nombre de résultats dépasse la capacité de la réponse, ajoutez les paramètres `page` et `per_page` à votre requête pour en obtenir davantage.

Consultez [Migrer vers les requêtes paginées des points de terminaison de la Management API v2](/docs/troubleshoot/product-lifecycle/past-migrations/migrate-to-paginated-queries) pour obtenir plus d'informations concernant la mise à jour de vos applciations.
