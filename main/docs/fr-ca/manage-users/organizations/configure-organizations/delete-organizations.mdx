Lors de la suppression d’une [organisation](/docs/fr-ca/manage-users/organizations/organizations-overview), le lien entre les utilisateurs et l’organisation concernée est supprimé, mais les utilisateurs eux-mêmes ne sont pas supprimés de votre locataire.

<Warning>

Une fois confirmée, cette opération ne peut être annulée.

</Warning>

Vous pouvez supprimer des organisations à l’aide d’<Tooltip href="/docs/docs/fr-ca/glossary?term=auth0-dashboard" tip="Auth0 Dashboard
Principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire">Auth0 Dashboard</Tooltip> ou de <Tooltip href="/docs/docs/fr-ca/glossary?term=management-api" tip="Management API
Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire">Management API</Tooltip>.

## Auth0 Dashboard

Pour supprimer une organisation à partir d’Auth0 Dashboard :

1. Naviguez vers [Auth0 Dashboard > Organizations](https://manage.auth0.com/#/organizations), et sélectionnez l’organisation que vous souhaitez supprimer.
2. Faites défiler pour localiser la section **Zone de danger**, sélectionnez **Supprimer** et confirmez.

## Management API

Effectuez un appel `GET` au point de terminaison`Delete Organization`. Veillez à remplacer les valeurs de remplacement `ORG_ID` et `MGMT_API_ACCESS_TOKEN` par l’identifiant de votre organisation et le jeton d’accès à Management API, respectivement.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

**Trouvez votre domaine Auth0**

Si votre domaine Auth0 est le nom de votre locataire, votre sous-domaine régional (sauf si votre locataire se trouve aux États-Unis et a été créé avant juin 2020), est `.auth0.com`. Par exemple, si votre nom de locataire est `travel0`, votre nom de domaine Auth0 sera `travel0.us.auth0.com`. (Si votre locataire est aux États-Unis et qu’il a été créé avant juin 2020, votre nom de domaine Auth0 sera `https://travel0.auth0.com`.)

Si vous utilisez des domaines personnalisés, votre domaine est le nom de votre domaine personnalisé.

#### Codes d’état des réponses

Les codes d’état de réponse possibles sont les suivants  :

| Code d’état | Code d’erreur | Message | Cause |
| - | - | - - | |
| `204` | | L’organisation a été supprimée. | |
| `400` | `invalid_uri` | URI de demande non valide. Le message variera en fonction de la cause. | Le chemin n’est pas valide. |
| `401` | | Jeton non valide. | |
| `401` | | Signature non valide reçue pour la validation du jeton Web JSON. | |
| `403` | `insufficient_scope` | Permission insuffisante; on s’attend à l’un des éléments suivants : `delete:organizations`. | Tentative de lire/écrire un champ qui n’est pas autorisé avec les permissions de jeton du porteur fourni. |
| `404` | | L’organisation n’existe pas. | |
| `429` | | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. | |
