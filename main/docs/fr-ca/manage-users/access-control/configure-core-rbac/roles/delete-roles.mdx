Vous pouvez supprimer un [rôle](/docs/fr-ca/manage-users/access-control/rbac) avec <Tooltip href="/docs/docs/fr-ca/glossary?term=auth0-dashboard" tip="Auth0 Dashboard
Principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire">Auth0 Dashboard</Tooltip> ou <Tooltip href="/docs/docs/fr-ca/glossary?term=management-api" tip="Management API
Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire">Management API</Tooltip>. Les rôles sont utilisés avec l’ensemble de fonctionnalités API Authorization Core.

Pour que le contrôle d'accès basé sur les rôles (RBAC) fonctionne correctement, vous devez l’activer pour votre API en utilisant soit le Dashboard, soit Management API. La fonctionnalité centrale de l’autorisation est différente de l’extension d’autorisation. Pour une comparaison, consultez [Authorization Core par rapport à Authorization Extension](/docs/fr-ca/manage-users/access-control/authorization-core-vs-authorization-extension).

## Dashboard

1. Allez à [Dashboard > Gestion des utilisateurs > Rôles](https://manage.auth0.com/#/roles) et cliquez sur le nom du rôle à afficher.
2. Cliquez sur **Supprimer ce rôle**, puis confirmez.

## Management API

Faites un appel `DELETE` au [point de terminaison Supprimer le rôle](https://auth0.com/docs/api/management/v2#!/Roles/delete_roles_by_id). Assurez-vous de remplacer les valeurs des paramètres fictifs `ROLE_ID` et `MGMT_API_ACCESS_TOKEN` avec votre identifiant de rôle et votre jeton d’accès de Management API, respectivement.

  --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'
