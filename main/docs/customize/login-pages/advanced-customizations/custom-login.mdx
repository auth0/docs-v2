ACUL supports various tech stacks to fully customize the login experience for your end-users and to fit numerous business cases with advanced customization. Use the following scenarios to customize your Universal Login [themes](/docs/customize/login-pages/universal-login/customize-themes), [templates](/docs/customize/login-pages/universal-login/customize-templates), and [text elements](/docs/customize/login-pages/universal-login/customize-text-elements). ## Use cases

### Create a theme with CSS variables
Universal Login provides [a number of prompts](https://auth0.com/docs/customize/login-pages/universal-login/customize-signup-and-login-prompts#terminology), or steps in the authentication process, along with at least one screen. Use ACUL to apply a theme to the screen. The most commonly used prompt, the Signup and Login prompts, are standard with end-user login.  For example:

<Frame>![ACUL Login](/docs/images/cdy7uua7fh8z/5be9XfnulShqeXa4MXfXky/f3a5dded3539d4d683a525dfb6ad8042/Screenshot_2025-01-28_at_10.49.07.png)</Frame>

In this example, we want to apply a theme to this Login screen. The following example uses Tailwind CSS v4.

All theme and branding customizations are located in: src/index.css.

	1. Use the Auth0 CLI tool to create the project and a scaffold `acul-screens` directory with the screens you want to apply a new theme.

```bash
auth0 acul init acul-screens  && cd acul-screens
```
<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
If you have not completed the ACUL Quickstart Guide, review Initialize ACUL with Auth0 Deploy CLI.
</Callout>

	2. Open the Universal Login Context Inspector to edit and view your screen updates.

```bash
npm run screen login-passwordless-email-code
```
	3. Edit the CSS file: src/index.css

```css
/* In src/index.css */
:root {
  /* Change these HSL values to match your brand's primary color */
  --brand-hue: 244;
  --brand-saturation: 100%;
  --brand-lightness: 65%;

  /* You can also override specific component colors directly */
  --card-background-color: white;
  --body-background-color: hsl(240 10% 95%);
  --button-primary-background-color: hsl(
    var(--brand-hue) var(--brand-saturation) var(--brand-lightness)
  );
  /* ... and many more */
}
``` 

### Integrate a component library

With ACUL, you can use your choice of component libraries to customize your Universal Login prompt screens. The following examples uses [Shadcn](https://ui.shadcn.com/docs), a library of reusable components and the Auth0 login-passwordless-email-code screen. In this example, you will replace the default screen with Shadcn’s [InputOTP](https://ui.shadcn.com/docs/components/input-otp) component.

	1. Use the Auth0 CLI tool to create the project and a scaffold `acul-screens` directory with the screens you want to apply a new theme:

```bash
auth0 acul scaffold login-passwordless-email-code && cd acul-screens
```
	2. Open the Universal Login Context Inspector to edit and view your screen updates.

```bash
npm run screen login
```

	3. Initialize Shadcn in the root of your project:

```bash
npx shadcn-ui@latest init
```

	4. Follow the CLI prompts to create the components.json file to hold the configuration for your project and a src/lib/utils.ts file.

	5. Add the component files to src/components/ui/input-otp.tsx:

```bash
npx shadcn-ui@latest add input-otp
```

	6. Integrate the component:
		a. Navigate to src/screens/login-passwordless-email-code/LoginPasswordlessEmailCode.tsx and open the file.
		b. Import the InputOTP components and replace the existing input field. You also need to manage the state of the OTP code and use the correct SDK hook.

```javascript
```bash
// In LoginPasswordlessEmailCode.tsx
import { useState } from 'react';
import { useEmailOtpChallenge } from '@auth0/auth0-acul-react'; // Correct hook for this screen
import {
  InputOTP,
  InputOTPGroup,
  InputOTPSlot,
} from '@/components/ui/input-otp'; // Import from ShadCN

// ... inside your component
const { submit } = useEmailOtpChallenge(); // The method is 'submit'
const [otp, setOtp] = useState('');

const handleSubmit = (e) => {
  e.preventDefault();
  submit({ code: otp }); // Call the submit method with the code
};

return (
  <form onSubmit={handleSubmit}>
    {/* ... other UI elements ... */}
    <InputOTP maxLength={6} value={otp} onChange={setOtp}>
      <InputOTPGroup>
        <InputOTPSlot index={0} />
        <InputOTPSlot index={1} />
        <InputOTPSlot index={2} />
        <InputOTPSlot index={3} />
        <InputOTPSlot index={4} />
        <InputOTPSlot index={5} />
      </InputOTPGroup>
    </InputOTP>
    <Button type="submit">Verify Code</Button>
  </form>
);
```

	7. Run the screen locally with the ACUL Context Inspector to see your new component:

```bash
npm run screen login-passwordless-email-code
```

To customize the Passwordless Login page fields and text, read [Customize Universal Login Page Template](https://auth0.com/docs/customize/login-pages/universal-login/customize-templates).

### Configure dynamic branding

When a user logs in to your application, you might want the authentication screens to change based on the end-user’s Auth0 Organization or application.

With ACUL, you can use two properties to drive a multi-brand experiences:

	* **client.name**: Use properties from `client.name` to determine branding based on which application the user is logging into. To learn more, read [Dynamic Client Registration](/docs/api/authentication/dynamic-application-client-registration/dynamic-application-registration).
	* **organization.name**: Use properties from `organization.name` to determine branding based on the customer or business unit the user belongs to. To learn more, read [Configure Organizations](/docs/manage-users/organizations/configure-organizations).

To accomplish dynamic branding:
	* Create asset paths to load different UI bundles based on the end-user's context
	* In-app logic

#### Create multiple asset paths
This example uses [Vite](https://vite.dev/guide/) with Typescript for a more complex configuration and separate build process. The authentication screen in use is the `login-passwordless-email-code` screen.

	1. Create your Vite project.

```bash
npm create vite@latest
```

	2. Use the Auth0 CLI tool to create a scaffold `acul-screens` directory with the screens you want to apply a new theme:

```bash
auth0 acul scaffold login-passwordless-email-code && acul-screens
```

	3. Open the Universal Login Context Inspector to edit and view your screen updates.

```bash
npm run screen login-passwordless-email-code
```

	4. To organize your source code by brand, create separate directories for each Auth0 Organization `organization.name` or application client.name in the /src folder.
	Example:


```bash
src/
|–brands/
|–– `organization_a`/
|–––– index.ts/
|–––– style.css/
|–– `organization_b`/
|–––– index.ts/
|–––– style.css/
|–– shared/
|––– utils.ts/
```

	5. In the index.ts files, configure each Organization:

```javascript
import './styles.css';
import { renderLoginForm } from '../../shared/components/LoginForm';

// Organization A specific logic and configuration
const brandConfig = {
    theme: 'dark',
    title: "Welcome to Organization A Portal",
};

document.addEventListener('DOMContentLoaded', () => {
    // Inject brand-specific styles and title
    document.title = brandConfig.title;

    // Use a shared component, passing in brand-specific data
    const appRoot = document.getElementById('app-root');
    if (appRoot) {
        renderLoginForm(appRoot, brandConfig);
    }
});
```	 

	6. In the style.css, configure the UI. You can use the example below:

```css
/* Unique Styles for Organization A */
:root {
    --primary-color: #007bff; /* Blue theme */
    --background-color: #f8f9fa;
}

body {
    font-family: 'Arial', sans-serif;
    background-color: var(--background-color);
}

.login-card {
    border: 1px solid var(--primary-color);
    box-shadow: 0 4px 8px rgba(0, 123, 255, 0.1);
}

.login-button {
    background-color: var(--primary-color);
}
```

	7. Navigate to the `vite.config.ts` file in the root folder.


	8. To deploy multiple bundles, add the organization.name directories and `index.ts` files to the `rollupOptions.input` in the configuration file:

```javascript
import { defineConfig } from 'vite';
import { resolve } from 'path';

export default defineConfig({
  build: {
    // Specify the base directory for assets relative to the output folder
    // This setting (or similar) ensures consistent naming of assets within the bundle output
    // which aligns with the dynamic loading approach.
    assetsDir: '', // An empty string can keep assets at the root of the brand folder

    rollupOptions: {
      input: {
        // Define one input for each brand, using the brand's slugified name as the key
        // This key will determine the output filename (e.g., client_a.js).
        organization_a_a: resolve(__dirname, 'src/brands/client_a/index.ts'),
        organization_b: resolve(__dirname, 'src/brands/client_b/index.ts'),
      },
      output: {
        // Use a function to ensure the main bundle for each brand is named 'bundle.js'//
        entryFileNames: (chunkInfo) => {
          // This configuration assumes the CI/CD will place the bundle in the
          return 'bundle.js';
        },
      },
    },
  },
});
```

	9. Run the screen locally with the ACUL Context Inspector to see your new component:

```bash
npm run screen login-passwordless-email-code
```

	10. (Optional) To load the correct bundle at runtime, call the [Management API](/docs/api/management/v2/prompts/put-partials). Universal Login supports [Liquid syntax](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) to load your custom configuration:



```html
{
  "head_tags": [{
    "tag": "script",
    "src": "https://cdn.myapp.com/screens/{{ organization.name | slugify }}/bundle.js"
  }]
}
```

To learn more, read [Customize Signup and Login Prompts](/docs/customize/login-pages/universal-login/customize-signup-and-login-prompts).

#### Integrate in-app logic with React hooks

The following use case supports in-app customization with minor variations between Organization A and Organization B. The ACUL React SDK supports root import and uses a single deployed asset bundle with logic to render a different UI based on the context. In this case, by Organization.

	1. Navigate to your React project.
	2. Install the ACUL React SDK.

```bash
npm install @auth0/auth0-acul-react
```

	3. Install for peer dependencies:

```bash
npm install react
```

	4. Add the ACUL React SDK `useOrganization` hook to your project:

```javascript
import { useOrganization } from '@auth0/auth0-acul-react';

// Inside your React component
const organization = useOrganization();

// Now you can safely access organization properties
const organizationName = organization?.name; // e.g., "Organization_a"
const logoUrl = organization?.branding?.logo_url;
```

	5. The `useOrganization` hook relies on the Organization ID to read your custom configuration.
