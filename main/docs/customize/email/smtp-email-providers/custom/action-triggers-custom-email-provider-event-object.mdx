The `event` object for the `custom-email-provider` Actions trigger provides contextual information about email data.

| Name | Description | Properties |
|------|-------------|------------|
| `event.client` | Information about the client with which this login transaction was initiated. | <ul><li>`client_id` *String*. The <Tooltip tip="Identification value given to your registered resource from Auth0." cta="View Glossary" href="/docs/glossary?term=client+ID">client ID</Tooltip> of the application the user is logging in to.</li> <li>`metadata` *Dictionary*. An object for holding other application properties.</li> <li>`name` *String*. The name of the application (as defined in the Dashboard).</li></ul> |
| `event.connection` *(Optional)* | Details about the connection used to authenticate the user. | <ul><li>`id` *Optional string*. The connection's unique identifier.</li> <li>`metadata` *Optional dictionary*. Metadata associated with the connection.</li> <li>`name` *Optional string*. The name of the connection used to authenticate the user (such as `twitter` or `some-g-suite-domain`).</li> <li>`strategy` *Optional string*. The type of connection. For social connections, `event.connection.strategy === event.connection.name`. For enterprise connections, the strategy is `waad` (Windows Azure AD), `ad` (Active Directory/LDAP), `auth0` (database connections), and so on.</li></ul> |
| `event.custom_domain` *(Optional)* | | <ul><li>`domain` *Optional string*. Available in MCD EA. The custom domain name.</li></ul> | 
| `event.notification` | | <ul><li>`html` *String*. Rendered HTML template.</li> <li>`from` *String*. Email address of the sender for the email.</li> <li>`subject` *String*.Subject to be attached to the email.</li> <li>`locale` *Optional string*. The locale we rendered the message in, for example `en_US`, as defined in the BCP-47 specification.</li> <li>`message_type` *String*. The type of message that is being send, like `verify_email` or `welcome_email`. Possible values include: <ul><li>`verify_email`</li> <li>`verify_email_by_code`</li> <li>`reset_email`</li> <li>`reset_email_by_code`</li> <li>`welcome_email`</li> <li>`verification_code`</li> <li>`mfa_oob_code`</li> <li>`enrollment_email`</li> <li>`blocked_account`</li> <li>`stolen_credentials`</li> <li>`try_provider_configuration_email`</li> <li>`organization_invitation`</li></ul></li> <li>`text` *String*. Rendered text template.</li> <li>`to` *String*. Email address of the recipient.</li></ul> | 
| `event.organization` *(Optional)* | Details about the Organization associated with the current transaction. | <ul><li>`display_name` *String*. The friendly name of the Organization.</li> <li>`id` *String*. The Organization identifier.</li> <li>`metadata` *Dictionary*. Metadata associated with the Organization.</li> <li>`name` *String*. The name of the Organization.</li></ul> | 
| `event.request` *(Optional)* | | <ul><li>`geoip` *Optional object*. Includes the following properties: <ul><li>`countryCode` *Optional string*.</li> <li>`countryCode3` *Optional string*.</li> <li>`cityName` *Optional string*.</li> <li>`latitude` *Optional string*.</li> <li>`longitude` *Optional string*.</li> <li>`timeZone` *Optional string*.</li> <li>`subdivisionCode` *Optional string*.</li> <li>`subdivisionName` *Optional string*.</li> <li>`continentCode` *Optional string*.</li></ul></li> <li>`hostname` * Optional string*. Available in MCD EA. The hostname that is being used for the authentication flow.</li> <li>`ip` *Optional string*. The originating IP address of the request.</li> <li>`query` *Optional dictionary*. The query string parameters sent to the authorization request. Only supported for the Password Reset Flow.</li> <li>`user_agent` *Optional string*. The value of the `User-Agent` header received when initiating the transaction.</li></ul> | 
| `event.tenant` | | <ul><li>`friendly_name` *Optional string*. The friendly name for the tenant, usually a more human-readable version of the ID.</li> <li>`home_url` *Optional string*. The home URL for the tenant, if defined and as found in its settings.</li> <li>`id` *String*. The name of the tenant.</li> <li>`logo_url` *Optional string*. The logo URL for the tenant, if defined and as found in its settings.</li> <li>`support_email` *Optional string*. The email to the tenant's support service, if defined and as found in its settings.</li> <li>`support_url` *Optional string*. The url to the tenant's support service, if defined and as found in its settings.</li></ul> | 
| `event.user` | An object describing the user on whose behalf the current transaction was initiated. | <ul><li>`app_metadata` *Dictionary*. Custom fields that store information about a user that influences the user's access, such as support plan, security roles, or access control groups.</li> <li>`email` *Optional string*. (unique) User's email address.</li> <li>`email_verified` *Boolean*. Indicates whether the user has verified their email address.</li> <li>`family_name` *Optional string*. User's family name.</li> <li>`given_name` *Optional string*. User's given name.</li> <li>`name` *Optional string*. User's full name.</li> <li>`nickname` *Optional string*. User's nickname.</li> <li>`picture` *Optional string*. URL pointing to the <a href="/docs/manage-users/user-accounts/change-user-picture">user's profile picture</a>.</li> <li>`user_id` *String*. (unique) User's unique identifier.</li> <li>`user_metadata` *Dictionary*. Custom fields that store information about a user that does not impact what they can or cannot access, such as work address, home address, or user preferences.</li> <li>`username` *Optional string*. (unique) User's username.</li></ul> |
