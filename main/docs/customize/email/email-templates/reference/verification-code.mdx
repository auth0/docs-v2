If you enable the **Verification Email (Code)** template, this email is sent in the following scenarios where the user needs to prove they have access to the email on file for an account:

* You have enabled the code-based email verification flow and a user signs up or logs into the account for the first time

* You have enabled the Adaptive MFA policy and there is a low-confidence transaction for which account ownership must be verified

The email includes a code that users need to enter during the authentication.

## Template-Specific Variables

In addition to the [common Liquid variables](/docs/customize/email/email-templates/supported-liquid-syntax) available to all email templates, this email template has the following variables:

* `code`, the verification code
