[Auth0 Actions](/docs/customize/actions) allow you to customize authentication flows with custom code. When using [multiple custom domains](/docs/customize/custom-domains/multiple-custom-domains), you can access the custom domain information in your Actions to implement domain-specific logic, route users to organizations, enforce access policies, and more.

## Accessing custom domain information

Actions provide access to custom domain information through the `event.request.hostname` property. This property contains the fully-qualified domain name (FQDN) used for the current authentication request.

### Available in these triggers

Custom domain information is available in the following Action triggers:

* Post-Login
* Pre-User Registration
* Post-User Registration
* Password Reset - Post-Change Password
* Machine to Machine - Credentials Exchange

<Note>
The `event.request.hostname` property is only available in triggers that involve an HTTP request from an end user or client. Triggers such as Send Phone Message or custom database scripts do not have access to this property.
</Note>

### Event object structure

Using `login.example.com` domain configured with domain metadata:

| Key        | Value                     |
|------------|---------------------------|
| allow_list | example1.com,example2.com |

```javascript
exports.onExecutePostLogin = async (event, api) => {
  console.log(event.custom_domain);
};
```
Would log at the console:
```bash
{
  "domain": "login.example.com",
  "domain_metadata": {
    "allow_list": "example1.com,example2.com"
  }
}
```

## Common use cases

### Conditional logic by domain

Apply different rules based on whether a custom domain or canonical domain is used:

```javascript
exports.onExecutePostLogin = async (event, api) => {
  const domain = event.custom_domain?.domain;

  if (
    domain === undefined ||
    domain.includes('.auth0.com') || // Public Cloud
    domain.includes('.auth0app.com') // Private Cloud
  ) {
    return api.access.deny('Please use the custom domain to log in.');
  }
};
```

### Email domain-based access control

Enforce email domain-specific access policies using custom domain metadata:

```javascript
exports.onExecutePostLogin = async (event, api) => {
  const domain = event.custom_domain?.domain;
  
  if (domain === undefined) {
    return api.access.deny(
      `Access denied - Users cannot access without custom domain.`
    );
  }

  const email = event.user.email;
  
  if (email === undefined) {
    return api.access.deny(
      `Access denied - Users cannot access without email.`
    );
  }

  const domainAllowList = event.custom_domain?.domain_metadata?.allow_list?.split(',') || [];
  const emailDomain = email.split('@')[1];

  if (domainAllowList.includes(emailDomain) === false) {
    return api.access.deny(
      `Access denied - Users from ${emailDomain} cannot access ${domain}.`
    );
  }
};
```

### Application metadata-based access control

Enforce application and domain groups access policies using application and domain metadata:

```javascript
exports.onExecutePostLogin = async (event, api) => {
  const domain = event.custom_domain?.domain;
  
  if (domain === undefined) {
    return api.access.deny(
      `Access denied - Users cannot access without custom domain.`
    );
  }

  const applicationGroup = event.client.metadata?.domain_group?.split(',') || [];
  const domainGroup = event.custom_domain?.domain_metadata?.domain_group?.split(',') || [];
  const intersection = domainGroup.filter(x => new Set(applicationGroup).has(x));

  if (intersection.length === 0) {
    return api.access.deny(
      `Access denied - Users cannot access application ${event.client.name} from ${domain}.`
    );
  }
};
```

### Connection metadata-based access control

Enforce connection and domain groups access policies using connection and domain metadata:

```javascript
exports.onExecutePostLogin = async (event, api) => {

  if (domain === undefined) {
    return api.access.deny(
      `Access denied - Users cannot access without custom domain.`

  if (intersection.length === 0) {
    return api.access.deny(
      `Access denied - Users cannot access connection ${connection.name} from ${domain}.`
    );
  }

```

### Organization metadata-based access control

Enforce organization and domain groups access policies using organization and domain metadata:

```javascript
exports.onExecutePostLogin = async (event, api) => {
  const organization = event.organization;

  if (!organization) {
    return; // Skip for non-organization authentication
  }

  const domain = event.custom_domain?.domain;

  if (domain === undefined) {
    return api.access.deny(
      `Access denied - Users cannot access without custom domain.`
    );
  }

  const organizationGroup = organization.metadata?.domain_group?.split(',') || [];
  const domainGroup = event.custom_domain?.domain_metadata?.domain_group?.split(',') || [];  
  const intersection = domainGroup.filter(x => new Set(organizationGroup).has(x));

  if (intersection.length === 0) {
    return api.access.deny(
      `Access denied - Users cannot access organization ${organization.name} from ${domain}.`
    );
  }
};
```

### Region-based requests

Make requests to region-specific external services based on custom domain metadata:

```javascript
exports.onExecutePostLogin = async (event, api) => {
  const regionDomain = event.custom_domain?.domain_metadata?.regionDomain;

  if (regionDomain === undefined) {
    return api.access.deny(
      `Access denied - Users cannot access without custom domain.`
    );
  }

  const messageBody = {
    message: 'Hello World!'
  };

  try {
    await fetch(`https://${regionDomain}/api/resource`, {
      method: 'POST',
      headers: {
        'X-API-Key': event.secrets.API_KEY,
      },
      body: JSON.stringify(messageBody),
    });
  } catch (err) {
    api.notification.drop('External service failure');
  }
};
```
