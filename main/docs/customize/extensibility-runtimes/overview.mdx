## Supported Runtimes

The following table lists all Node.js runtimes currently considered by Auth0:

| Runtime     | Status                                                                        |
|:------------|:------------------------------------------------------------------------------|
| Node 26     | <Badge color="orange">Upcoming</Badge>                                        |
| Node 22     | <Badge color="blue">Active</Badge> <Badge color="green">Recommended</Badge>   |
| Node 18     | <Badge color="blue">Active</Badge>                                            |
| Node 16     | <Badge color="red">EOL</Badge> <Badge color="purple">Extended</Badge>         |
| Node 12     | <Badge color="red">EOL</Badge> <Badge color="purple">Extended</Badge>         |

### Status Definitions

| Runtime                                  | Status                                                                                                                                                                                                                                                                                            |
|:-----------------------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <Badge color="blue">Active</Badge>       | Node runtimes that receive bug fixes, security updates, and are compatible with Node.js Active and Maintenance stages.                                                                                                                                                                            |
| <Badge color="green">Recommended</Badge> | Node runtime is a currently <Badge color="blue">Active</Badge> and recommended version for all extensiiblity points in every development stage. This version is the one that receives all the upgrades, updates, and fixes regarding Node, but also the ones tha support latest Auth0 features.   |
| <Badge color="yellow">Deprecated</Badge> | Node runtimes only critical security patches and high-priority bug fixes. No new features.                                                                                                                                                                                                        |
| <Badge color="red">EOL</Badge>           | *(End-Of-Life)* Node runtimes are no longer supported, no fixes applied.                                                                                                                                                                                                                          |
| <Badge color="purple">Extended</Badge>   | Node runtimes receive an extended period of support related to security patches.                                                                                                                                                                                                                  |
|                                          | **Requires subscription to Advanced Extensibility SKU**.                                                                                                                                                                                                                                          |
| <Badge color="orange">Upcoming</Badge>   | Node runtime is the next version to be available once Node.js Community makes it Active.                                                                                                                                                                                                          |

## Features Node Runtimes

There are few Auth0 Extensibility features that use and configure Node runtimes at different levels:

| Feature                                                  | Configuration                                         |
|:---------------------------------------------------------|:------------------------------------------------------|
| Actions                                                  | <Badge color="green">Entity Level</Badge>             |
| Custom Databases                                         | <Badge color="blue">Tenant Level</Badge>              |
| Custom Social Connections                                | <Badge color="blue">Tenant Level</Badge>              |
| Rules <Badge color="yellow">Deprecated</Badge>           | <Badge color="gray">Tenant Level (Legacy)</Badge>     |
| Hooks <Badge color="yellow">Deprecated</Badge>           | <Badge color="gray">Tenant Level (Legacy)</Badge>     |
| Extensions                                               | <Badge color="purple">Per Instance (Internal)</Badge> |
| Custom Extensions                                        | <Badge color="orange">Per Instance</Badge>            |
| Custom Webtasks <Badge color="yellow">Deprecated</Badge> | <Badge color="orange">Per Instance</Badge>            |

### Configuration Definitions

| Configuration                                            | Description                                                                                                                                                              |
|:---------------------------------------------------------|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <Badge color="green">Entity Level</Badge>                | Node runtime is configured at each feature entity, instead of a general configuration at Tenant Level. Each feature entity can have a different Node runtime configured. |
| <Badge color="blue">Tenant Level</Badge>                 | Node runtime is configured at the Tenant Settings for group of features sharing the same runtime setup.                                                                  |
| <Badge color="gray">Tenant Level (Legacy)</Badge>        | Similar to <Badge color="blue">Tenant Level</Badge>, but focused on a groupd of Legacy features that are only available behind a feature flag or exception.              |
|                                                          | *This configuration, when present, works as an overwrite of the <Badge color="blue">Tenant Level</Badge>, otherwise it uses <Badge color="blue">Tenant Level</Badge>.*   |
| <Badge color="purple">Per Instance (Internal)</Badge>    | . |
| <Badge color="orange">Per Instance</Badge>               | . |


When Auth0 infrastructure encounters memory pressure, Actions running on end-of-life (EOL) runtimes (such as Node.js 12 and Node.js 16) may be deprioritized in favor of Actions running on supported LTS runtimes (Node.js 18 and Node.js 22). To ensure the best performance and reliability, migrate your Actions to the latest supported LTS runtime.
