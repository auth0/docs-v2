## Understanding the Lifecycle

Long-term Support (LTS) is a release strategy that provides:

<Steps>
  <Step title="Community Version Active">
    Auth0 consideres certain Node versions once they become Active.

        **Duration:** 12 months after initial LTS release
    
    During Active LTS, the runtime receives:
    - New features backported from Current
    - Bug fixes
    - Security updates
    - Documentation updates
    
    This is the recommended phase for production applications.
  </Step>
  <Step title="Auth0 Version Support">
    Around a month after the picked community version becomes Active, Auth0 starts supporting it.

        **Duration:** 18 months after Active LTS ends
    
    During Maintenance LTS, the runtime receives:
    - Critical bug fixes only
    - Security patches
    - No new features
    
    Consider planning your migration during this phase.
  </Step>
  <Step title="Auth0 Version Deprecation">
    6 months before the Community Version EOL, Auth0 announces the version Deprecation.
  </Step>
  <Step title="Community Version EOL">
    TBD
    
  </Step>
  <Step title="Auth0 Extended Version Support">
    At the same time of the Community Version EOL, Auth0 starts the Extended Version Support, where Auth0 would keep introducing security updates for a year to give additional time to customers to upgrade.
  </Step>
  <Step title="Auth0 Version EOL">
    After Maintenance LTS ends, the runtime is no longer supported:
      - No updates of any kind
      - Security vulnerabilities will not be patched
      - Auth0 will block deployments using EOL runtimes
    
    **You must migrate before this phase.**
  </Step>
</Steps>

When Auth0 adds support for a new Node.js runtime:

1. We announce support within 2 weeks of the official Node.js release
2. The runtime becomes available for new deployments
3. We recommend testing your applications before migrating production workloads
4. We provide migration guides and compatibility notes

## Notifications

Auth0 sends deprecation notifications through multiple channels:

| Channel | Timing | Content |
|---------|--------|---------|
| Email | 6, 3, 1 months before EOL | Detailed migration steps |
| Dashboard Banner | Ongoing after deprecation | Warning with documentation link |
| API Response Headers | After deprecation | `X-Runtime-Deprecated: true` |
| Webhook Events | Configurable | Custom integration support |

## Release Schedule

Auth0 follows the official Node.js release schedule. New major versions are released every 6 months, with even-numbered versions entering Long-term Support (LTS).
