# Auth0 CLI Command Reference: Experiment Center

:::note
This reference is maintained by the Auth0 CLI team. Command syntax is subject to change during Beta.
:::

This page covers the `auth0 experiment-center` command group, which lets you manage Experiment Center resources from the command line.

:::note
**Flag for CDT review:** The CLI command namespace (`auth0 experiment-center` vs. `auth0 experimentation` vs. another name) and the exact command syntax are pending confirmation from the Auth0 CLI (CDT) team. The structure shown here is based on the Management API shape and Auth0 CLI's established patterns. CDT to confirm or correct before publishing. See DRAFT-NOTES.md item 21.
:::

---

## Authentication setup

The `experiment-center` commands use your configured Auth0 CLI profile. Authenticate if you have not already:

```bash
auth0 login
```

Your token needs the following scopes for EC operations:

| Operation | Required scope |
|---|---|
| List and get resources | `read:experimentation` |
| Create resources | `create:experimentation` |
| Update and transitions | `update:experimentation` |
| Delete (archive) resources | `delete:experimentation` |

If a CLI command returns a `403` error, check that your Management API application has the required scopes.

**Fallback:** If a command is not yet available in the CLI build for Beta, use the Management API directly. See the [API reference](/docs/experiment-center/api-reference) for the equivalent endpoint.

## Command structure

```
auth0 experiment-center [resource] [action] [flags]
```

Resources: `feature-flags`, `variations`, `segments`, `experiments`

---

## Feature flags

### List feature flags

```bash
auth0 experiment-center feature-flags list [flags]
```

**Flags:**

| Flag | Description |
|---|---|
| `--status <status>` | Filter by status: `draft`, `active`, `archived` |
| `--type <type>` | Filter by type: `auth0`, `self` |
| `--take <n>` | Page size (default 25, max 50) |
| `--from <token>` | Pagination cursor |

**Example:**

```bash
auth0 experiment-center feature-flags list --status active
```

---

### Get a feature flag

```bash
auth0 experiment-center feature-flags get <feature-flag-id> [flags]
```

**Example:**

```bash
auth0 experiment-center feature-flags get flg_72jbvv7LfRKYp59gtRLtkn
```

---

### Create a feature flag

```bash
auth0 experiment-center feature-flags create [flags]
```

**Flags:**

| Flag | Description |
|---|---|
| `--name <name>` | Flag name (required) |
| `--description <desc>` | Optional description |
| `--parameter <name:type:value>` | Add a parameter (repeatable) |

**Example:**

```bash
auth0 experiment-center feature-flags create \
  --name "passkey-prompt" \
  --description "Controls passkey enrollment prompt display" \
  --parameter "show_passkey_prompt:boolean:false" \
  --parameter "prompt_style:string:inline"
```

---

### Update a feature flag

```bash
auth0 experiment-center feature-flags update <feature-flag-id> [flags]
```

**Flags:**

| Flag | Description |
|---|---|
| `--name <name>` | New name |
| `--description <desc>` | New description |

---

### Activate a feature flag

Transitions a feature flag from `draft` to `active`. Requires at least two variations.

```bash
auth0 experiment-center feature-flags activate <feature-flag-id>
```

---

### Archive a feature flag

Transitions a feature flag to `archived` (terminal state).

```bash
auth0 experiment-center feature-flags archive <feature-flag-id>
```

---

## Variations

### List variations for a flag

```bash
auth0 experiment-center variations list <feature-flag-id>
```

---

### Create a variation

```bash
auth0 experiment-center variations create <feature-flag-id> [flags]
```

**Flags:**

| Flag | Description |
|---|---|
| `--name <name>` | Variation name (required) |
| `--description <desc>` | Optional description |
| `--override <name:value>` | Override a parameter value (repeatable) |

**Example — control variation (no overrides):**

```bash
auth0 experiment-center variations create flg_72jbvv7LfRKYp59gtRLtkn \
  --name "control" \
  --description "No passkey prompt (baseline)"
```

**Example — treatment variation:**

```bash
auth0 experiment-center variations create flg_72jbvv7LfRKYp59gtRLtkn \
  --name "treatment-modal" \
  --description "Shows passkey prompt as modal" \
  --override "show_passkey_prompt:true" \
  --override "prompt_style:modal"
```

---

### Delete a variation

```bash
auth0 experiment-center variations delete <feature-flag-id> <variation-id>
```

## Segments

### List segments

```bash
auth0 experiment-center segments list [flags]
```

**Flags:**

| Flag | Description |
|---|---|
| `--type <type>` | Filter by type: `auth0`, `self` |

---

### Create a segment

```bash
auth0 experiment-center segments create [flags]
```

**Flags:**

| Flag | Description |
|---|---|
| `--name <name>` | Segment name (required) |
| `--description <desc>` | Optional description |
| `--rules <json>` | Segment rules as JSON (required) |

**Example:**

```bash
auth0 experiment-center segments create \
  --name "mobile-us" \
  --description "Mobile users from the United States" \
  --rules '[{"match_type":"all","conditions":[{"attribute":"device_type","operator":"equals","value":"mobile"},{"attribute":"country","operator":"equals","value":"US"}]}]'
```

---

### Delete a segment

```bash
auth0 experiment-center segments delete <segment-id>
```

## Experiments

### List experiments

```bash
auth0 experiment-center experiments list [flags]
```

**Flags:**

| Flag | Description |
|---|---|
| `--status <status>` | Filter by status: `draft`, `active`, `paused`, `completed`, `archived` |
| `--feature-flag-id <id>` | Filter by feature flag |

---

### Get an experiment

```bash
auth0 experiment-center experiments get <experiment-id>
```

---

### Create an experiment

```bash
auth0 experiment-center experiments create [flags]
```

**Flags:**

| Flag | Description |
|---|---|
| `--name <name>` | Experiment name (required) |
| `--description <desc>` | Optional description |
| `--feature-flag-id <id>` | Feature flag to test (required) |
| `--allocation-strategy <strategy>` | `percentage` or `segment` |
| `--subject <subject>` | Assignment subject: `device`, `user`, `session` |
| `--allocation <variation-id:weight:is-control>` | Allocation entry (repeatable, percentage strategy) |

**Example — 10/90 percentage split:**

```bash
auth0 experiment-center experiments create \
  --name "passkey-prompt-q2" \
  --feature-flag-id flg_72jbvv7LfRKYp59gtRLtkn \
  --allocation-strategy percentage \
  --subject device \
  --allocation "var_4Nq8mR2xPkTv6wYjHs3fBn:90:true" \
  --allocation "var_8Xp2qK5nWmJy9vDsHr4gAt:10:false"
```

---

### Validate an experiment

Runs the readiness check and returns any blockers.

```bash
auth0 experiment-center experiments validate <experiment-id>
```

**Example output:**

```
✓ Experiment is valid and ready to activate.
```

Or when not valid:

```
✗ Experiment is not valid. Errors:
  - feature_flag_not_active: Feature flag flg_72jbvv7LfRKYp59gtRLtkn must be active before activating this experiment.
```

---

### Activate an experiment

Transitions the experiment to `active`. Runs full validation first.

```bash
auth0 experiment-center experiments activate <experiment-id>
```

---

```bash
auth0 experiment-center experiments pause <experiment-id>
```

---

### Complete an experiment

```bash
auth0 experiment-center experiments complete <experiment-id>
```

---

### Archive an experiment

```bash
auth0 experiment-center experiments archive <experiment-id>
```

## Global flags

These flags apply to all `experiment-center` commands:

| Flag | Description |
|---|---|
| `--tenant <domain>` | Target tenant domain (overrides current profile) |
| `--output <format>` | Output format: `json`, `table` (default `table`) |

## Fallback: use the Management API

If a CLI command is not yet available in the Beta build, use the Management API directly:

```bash
# Example: get an experiment via API
curl -X GET \
  "https://YOUR_TENANT.us.auth0.com/api/v2/experimentation/experiments/EXPERIMENT_ID" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

See the [API reference](/docs/experiment-center/api-reference) for the full endpoint list.
