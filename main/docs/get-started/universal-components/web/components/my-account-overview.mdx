My Account Components let you embed a self-service authentication-management interface directly into your web application. Your users can enroll and remove multi-factor authentication (MFA) factors and manage passkeys without leaving your application or accessing the Auth0-hosted UI. Supported factor types include authenticator application, SMS, email one-time password (OTP), push notifications, and recovery codes.

<Note>
The My Account API enforces low rate limits, especially on free-tier tenants. This may cause errors while using these components.
</Note>

## How it works

My Account Components use the [My Account API's authentication methods](/docs/manage-users/my-account-api#manage-authentication-methods) endpoints to render an authentication-management UI inside your application. When an authenticated user opens their account settings screen, the Auth0 SDK retrieves an [access token](/docs/secure/tokens/access-tokens) scoped to the My Account API audience (`https://YOUR_TENANT.auth0.com/me/`). The components use this token to call [`/me/v1/authentication-methods`](/docs/api/myaccount/authentication-methods/get-authentication-methods) as the logged-in user, so each user can only view and modify their own authentication methods.

<Note>
* These components create **end-user self-service** interfaces. End users can enroll, list, and remove every authentication method on their own account: email OTP, SMS OTP, time-based one-time password (TOTP) authenticator application, push via Auth0 Guardian, passkeys, and recovery codes.

* For **delegated admin** interfaces in which a user manages an Auth0 Organization, read [Build a Delegated Admin Interface](/docs/get-started/universal-components/web/components/build-delegated-admin).
</Note>

### Available components

| **Component** | **Description** | **API endpoint** |
| :--- | :--- | :--- |
| [`UserMFAMgmt`](/docs/get-started/universal-components/web/components/user-mfa-management) | Lets users enroll and delete MFA factors: authenticator application (TOTP), SMS, email OTP, push notification, WebAuthn platform/roaming authenticator, and recovery codes. | `/me/v1/authentication-methods` |
| [`UserPasskeyMgmt`](/docs/get-started/universal-components/web/components/user-passkey-management) | Lets users register and revoke WebAuthn passkeys. | `/me/v1/authentication-methods` (type: `passkey`) |

## Prerequisites

Before using any My Account component, complete the following steps in the Auth0 Dashboard.

### Enable Auth0 My Account API

1. Navigate to [**Dashboard > Applications > APIs**](https://manage.auth0.com/#/apis).
2. Select **Activate My Account API** to enable it for your tenant.

### Create the application and configure My Account API permissions

<Info>
The steps below use a Single Page Application. For server-rendered React frameworks such as Next.js, choose **Regular Web Application** in step 2 and use the Next.js callback URLs.
</Info>

1. Navigate to [**Dashboard > Applications > Applications**](https://manage.auth0.com/#/applications). Select **Create Application**.
2. Choose **Single Page Web Applications**.
3. Select the **Settings** tab and add `http://localhost:5173` to the following fields:
    - Allowed Callback URLs
    - Allowed Logout URLs
    - Allowed Web Origins
4. Select the **API Access** tab.
5. Select **Edit** for the **Auth0 My Account API** to add the **User-delegated Access** permissions:

    ```text
    create:me:authentication_methods
    read:me:authentication_methods
    update:me:authentication_methods
    delete:me:authentication_methods
    read:me:factors
    ```

6. Select **Save** to save the permissions.

<Tip>
Because the user's access token only includes permissions that were granted during login, request all five scopes to allow users to enroll, review, and remove authentication methods.
</Tip>

### Set up the database and test user

1. Navigate to [**Dashboard > Authentication > Database**](https://manage.auth0.com/#/connections/database) to create a database connection.
2. In the **Applications** tab of the connection, enable your new application.
3. Create a test user in this database for initial testing.

## Configure your application

Choose your framework to install the SDK, configure environment variables, and wire up the component providers.

<Tabs>
<Tab title="React (SPA)">

### Install the SDK

<CodeGroup>
```bash pnpm wrap lines
pnpm add @auth0/universal-components-react
```

```bash npm wrap lines
npm install @auth0/universal-components-react
```
</CodeGroup>

<Note>
Either command also installs the `@auth0/universal-components-core` dependency for shared utilities and Auth0 integration.
</Note>

### Configure environment variables

Create a `.env` file in your project root:

```bash wrap lines
VITE_AUTH0_DOMAIN=YOUR_TENANT.auth0.com
VITE_AUTH0_CLIENT_ID=YOUR_SPA_CLIENT_ID
```

### Wire up the providers

Wrap your application with `Auth0Provider` and `Auth0ComponentProvider`. Set `interactiveErrorHandler="popup"` so MFA enrollment and deletion step-up challenges are handled in a popup rather than redirecting away from the page.

```tsx App.tsx wrap lines
import { Auth0Provider } from "@auth0/auth0-react";
import { Auth0ComponentProvider } from "@auth0/universal-components-react/spa";

const domain = import.meta.env.VITE_AUTH0_DOMAIN;
const clientId = import.meta.env.VITE_AUTH0_CLIENT_ID;

export default function App() {
  return (
    <Auth0Provider
      domain={domain}
      clientId={clientId}
      authorizationParams={{ redirect_uri: window.location.origin }}
      interactiveErrorHandler="popup"
    >
      <Auth0ComponentProvider domain={domain}>
        {/* your application */}
      </Auth0ComponentProvider>
    </Auth0Provider>
  );
}
```

<Check>
Users must be authenticated before any My Account component is rendered. Components automatically load the logged-in user's authentication methods from the My Account API. To learn more, read [Auth0 Universal Components](/docs/get-started/universal-components/universal-components-overview) for installation and `Auth0ComponentProvider` configuration.
</Check>

<Note>
You are responsible for ensuring that your use of the My Account API and Universal Components complies with your security policies and applicable laws, including any permissions granted to your end users.
</Note>

<Tab title="Next.js (RWA)">

### Create a Regular Web Application

Follow the [Prerequisites](#prerequisites) steps above, but in step 2 choose **Regular Web Application** instead of Single Page Web Applications. In step 3, add `http://localhost:3000` to **Allowed Callback URLs** and **Allowed Logout URLs** (Allowed Web Origins is not required for RWA).

### Install the SDK

<CodeGroup>
```bash pnpm wrap lines
pnpm add @auth0/universal-components-react
```

```bash npm wrap lines
npm install @auth0/universal-components-react
```
</CodeGroup>

<Note>
Either command also installs the `@auth0/universal-components-core` dependency.
</Note>

### Configure environment variables

Create a `.env.local` file in your Next.js project root:

```bash wrap lines
NEXT_PUBLIC_AUTH0_DOMAIN=YOUR_TENANT.auth0.com
NEXT_PUBLIC_AUTH0_CLIENT_ID=YOUR_RWA_CLIENT_ID
AUTH0_SECRET=YOUR_AUTH0_SECRET
AUTH0_ISSUER_BASE_URL=https://YOUR_TENANT.auth0.com
```

For complete Next.js SDK setup, follow the [Auth0 Next.js SDK documentation](/docs/quickstart/webapp/nextjs).

### Wire up the providers

Add `Auth0ComponentProvider` from the `/rwa` subpath to your root layout. Set `interactiveErrorHandler="popup"` so step-up auth challenges are handled in a popup.

```tsx app/layout.tsx wrap lines
import { Auth0ComponentProvider } from "@auth0/universal-components-react/rwa";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <Auth0ComponentProvider
          domain={process.env.NEXT_PUBLIC_AUTH0_DOMAIN!}
          interactiveErrorHandler="popup"
        >
          {children}
        </Auth0ComponentProvider>
      </body>
    </html>
  );
}
```
