<Recipe>
  <Content>
    Auth0 te permite agregar autenticación a casi cualquier tipo de aplicación de forma rápida. Esta guía muestra cómo
    integrar Auth0, agregar autenticación y mostrar información del perfil del usuario en cualquier aplicación .NET para Android e iOS
    usando los SDK de Auth0 para [Android](https://www.nuget.org/packages/Auth0.OidcClient.AndroidX/) e [iOS](https://www.nuget.org/packages/Auth0.OidcClient.iOS).

    <Info>
      Esta guía de inicio rápido se centra en .NET Android y iOS, ya que son la nueva generación de
      `Xamarin.Android` y `Xamarin.iOS`. Si aún usa
      `Xamarin.Android` y `Xamarin.iOS`, puede seguir esta guía, ya que la integración es
      idéntica y los SDK son compatibles.
    </Info>

    Para usar este inicio rápido, necesitarás:

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Para usar los servicios de Auth0, necesitas tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación en tu proyecto.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva &quot;Native Application&quot; o selecciona una aplicación existente que
      represente el proyecto que quieres integrar. A cada aplicación en Auth0 se le asigna un
      ID de cliente alfanumérico único que el código de tu aplicación usará para llamar a las API de Auth0 mediante el SDK.

      Cualquier configuración que realices con este inicio rápido se actualizará automáticamente para tu aplicación en el [Dashboard](https://manage.auth0.com/#/), donde
      podrás administrar tus aplicaciones en el futuro.

      Si prefieres revisar una configuración completa, puedes ver una aplicación de ejemplo.

      ### Configurar URL de callback

      Una URL de callback es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, establece esto en una de las siguientes URL según
        tu plataforma:

        * **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`
        * **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`

        **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`

        **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`
      </Info>

      ### Configurar URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión desde tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, establece esto en una de las siguientes URL según
        tu plataforma:

        * **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`
        * **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`

        **Android**: `YOUR_PACKAGE_NAME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback`

        **iOS**: `YOUR_BUNDLE_ID://{yourDomain}/ios/YOUR_BUNDLE_ID/callback`
      </Info>

      Por último, asegúrate de que el **Application Type** de tu aplicación esté establecido en **Native** en [Application
      Settings](https://manage.auth0.com/#/applications/).
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Auth0 proporciona SDK para [Android](https://www.nuget.org/packages/Auth0.OidcClient.AndroidX/) e [iOS](https://www.nuget.org/packages/Auth0.OidcClient.iOS) para simplificar la implementación de la
      autenticación con Auth0 en aplicaciones .NET para Android e iOS.

      Use el Administrador de paquetes NuGet (Tools -&gt; Library Package Manager -&gt; Package Manager Console) para instalar el
      paquete `Auth0.OidcClient.AndroidX` o `Auth0.OidcClient.iOS`, según si está
      compilando una aplicación para Android o iOS.

      Como alternativa, puede usar la Consola del Administrador de paquetes NuGet (`Install-Package`) o la
      CLI de `dotnet` (`dotnet add`).

      ```
      Install-Package Auth0.OidcClient.AndroidX
      Install-Package Auth0.OidcClient.iOS
      ```

      ```
      dotnet add Auth0.OidcClient.AndroidX
      dotnet add Auth0.OidcClient.iOS
      ```
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Para integrar Auth0 en tu aplicación, crea una instancia de la clase `Auth0Client` y pásale
      una instancia de `Auth0ClientOptions` que contenga tu dominio de Auth0 y tu ID de cliente.

      ```csharp
      using Auth0.OidcClient;
      var client = new Auth0Client(new Auth0ClientOptions {
      Domain = "{yourDomain}",
      ClientId = "{yourClientId}"
      }, this);
      ```

      De forma predeterminada, el SDK utilizará Chrome Custom Tabs en Android y ASWebAuthenticationSession en iOS.

      <Note>
        ##### Punto de control

        Tu `Auth0Client` ahora debería estar correctamente instanciado. Ejecuta la aplicación para verificar lo siguiente:

        * `Auth0Client` se instancia correctamente en la `Activity` (Android) o
          en el `UIViewController` (iOS).
        * La aplicación no muestra ningún error relacionado con Auth0.
      </Note>
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Después de que un usuario se autentique correctamente, se le redirigirá a la URL de callback que configuró antes en esta
      guía de inicio rápido.

      Para gestionar el callback en dispositivos Android, debe registrar un intent que maneje esta URL de callback. Una forma sencilla
      de hacerlo es registrar el intent en la misma actividad desde la que llamó al método LoginAsync para
      iniciar el flujo de autenticación.

      Asegúrese de reemplazar `YOUR_ANDROID_PACKAGE_NAME` en el ejemplo de código por el nombre de paquete real de su
      aplicación, como `com.mycompany.myapplication`, y de que todo el texto de
      `DataScheme`, `DataHost` y `DataPathPrefix` esté en minúsculas. Además, configure
      `LaunchMode = LaunchMode.SingleTask` para la `Activity`; de lo contrario, el sistema creará una nueva instancia
      de la actividad cada vez que se invoque la URL de callback.

      Además, debe gestionar el intent en el evento `OnNewIntent` de su clase `Activity`.
      Debe notificar al cliente OIDC de Auth0 para que finalice el flujo de autenticación llamando al método `Send`
      del singleton `ActivityMediator`, y pasar la URL recibida.

      ```charp
      protected override void OnNewIntent(Intent intent)
      {

            base.OnNewIntent(intent);

            ActivityMediator.Instance.Send(intent.DataString);

      }
      ```
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Después de que el usuario se autentique correctamente, se le redirigirá a la URL de callback que configuró antes en esta
      guía de inicio rápido.

      Para gestionar el callback en dispositivos iOS:

      1. Abra el archivo `Info.plist` de su aplicación en Visual Studio y vaya a la pestaña **Advanced **.
      2. En **URL Types**, haga clic en el botón **Add URL Type **.
      3. Establezca **Identifier **como Auth0, **URL Schemes **igual que el
         `Bundle Identifier` de su aplicación y **Role **como `None`.

      Este es un ejemplo de la representación XML de su archivo `info.plist` después de agregar el
      tipo de URL:

      ```xml
      <key>CFBundleURLTypes</key>
      <array>
      <dict>
      <key>CFBundleTypeRole</key>
      <string>None</string>
      <key>CFBundleURLName</key>
      <string>Auth0</string>
      <key>CFBundleURLSchemes</key>
      <array>
      <string>YOUR_BUNDLE_IDENTIFIER</string>
      </array>
      </dict>
      </array>
      ```

      Además, debe gestionar la URL de retorno en el evento `OpenUrl` de su
      clase `AppDelegate`. Debe notificar al cliente OIDC de Auth0 para que finalice el flujo de autenticación
      llamando al método `Send` del singleton `ActivityMediator` y pasándole la URL
      recibida.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Ahora que ha configurado la aplicación de Auth0 y el SDK de Auth0, debe configurar el inicio de sesión en su proyecto.
      Para ello, utilizará el método `LoginAsync()` del SDK para crear un botón de inicio de sesión que redirija a los usuarios
      a la página de Universal Login de Auth0.

      ```csharp
      var loginResult = await client.LoginAsync();
      ```

      Si no se produce ningún error, puedes acceder a `User`, `IdentityToken`,
      `AccessToken` y `RefreshToken` en el `LoginResult` que devuelve
      `LoginAsync()`.

      <Note>
        ##### Punto de control

        Ahora deberías poder iniciar sesión o registrarte con un username y una contraseña.

        Haz clic en el botón de inicio de sesión y verifica que:

        * Tu aplicación de Android o iOS te redirija a la página de Universal Login de Auth0.
        * Puedes iniciar sesión o registrarte.
        * Auth0 te redirige a tu aplicación.
      </Note>
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      Los usuarios que inicien sesión en su proyecto también necesitarán una forma de cerrar sesión. Cree un botón para cerrar sesión con el método del SDK
      `LogoutAsync()`. Cuando los usuarios cierren sesión, se les redirigirá al endpoint de cierre de sesión de Auth0,
      que luego los redirigirá de inmediato de vuelta a la URL de cierre de sesión que configuró anteriormente en esta guía de inicio rápido.

      ```csharp
      await client.LogoutAsync();
      ```

      <Note>
        ##### Verificación

        Ejecute la aplicación y haga clic en el botón Logout. Verifique que:

        * Su aplicación para Android o iOS lo redirija a la dirección que especificó como una de las direcciones URL de Logout permitidas en la configuración de la aplicación.
        * Ya no haya iniciado sesión en la aplicación.
      </Note>
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Ahora que sus usuarios pueden iniciar y cerrar sesión, probablemente querrá recuperar la información de perfil asociada a los usuarios autenticados.
      Por ejemplo, puede que quiera mostrar en su proyecto el nombre o la foto de perfil de un usuario que ha iniciado sesión.

      El SDK de Auth0 para Android e iOS proporciona información del usuario mediante la propiedad `LoginResult.User`.

      ```csharp
      if loginResult.IsError == false {
            var user = loginResult.User
            var name = user.FindFirst(c => c.Type == "name")?.Value
            var email = user.FindFirst(c => c.Type == "email")?.Value
            var picture = user.FindFirst(c => c.Type == "picture")?.Value
      }
      ```
    </Section>
