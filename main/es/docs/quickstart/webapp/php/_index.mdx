      Para usar los servicios de Auth0, deberá tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es

      ### Configurar una aplicación

      Use el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente

      ### Configurar URL de callback

      Una URL de callback es una URL de su aplicación a la que desea que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a su aplicación después de iniciar sesión.

      <Info>
        Si está siguiendo nuestro proyecto de ejemplo, configúrela como
        `http://localhost:3000/callback`.
      </Info>

      ### Configurar URL de cierre de sesión

      Una URL de cierre de sesión es una URL de su aplicación a la que desea que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión en su aplicación y recibirán un error.

      <Info>
        Si está siguiendo nuestro proyecto de ejemplo, configúrela como `http://localhost:3000`.
      </Info>

      ### Configurar orígenes web permitidos

      Un origen web permitido es una URL desde la que desea permitir el acceso a su flujo de autenticación. Debe
      incluir la URL de su proyecto. Si no se configura correctamente, su proyecto no podrá renovar silenciosamente
      los tokens de autenticación, por lo que sus usuarios cerrarán sesión la próxima vez que visiten su aplicación o actualicen una
      página.

      <Info>
        Si está siguiendo nuestro proyecto de ejemplo, configúrelo como `http://localhost:3000`.
      </Info>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 proporciona un [SDK
      de PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) para simplificar la implementación de la autenticación y la autorización de Auth0 en aplicaciones PHP.

      El SDK de PHP de Auth0 requiere bibliotecas HTTP compatibles con [PSR-17](https://www.php-fig.org/psr/psr-17/) y [PSR-18](https://www.php-fig.org/psr/psr-18/) instaladas para gestionar las solicitudes de red. Si
      no dispones de estas bibliotecas, puedes instalar opciones confiables ejecutando los siguientes comandos en tu
      terminal:

      ```text lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Ahora, instala el SDK de PHP de Auth0 ejecutando el siguiente comando en la terminal:

      ```text lines
      composer require auth0/auth0-php
      ```

      ### Configure el SDK de Auth0

      Cree un archivo nuevo en su aplicación llamado `index.php` y copie en él el código del panel interactivo
      de la derecha, en la pestaña **index.php**.

      Para que el SDK funcione correctamente, debe establecer las siguientes propiedades en el SDK de Auth0 durante la inicialización:

      * `domain`: El dominio de su inquilino de Auth0. Por lo general, puede encontrarlo en el Auth0 Dashboard,
        en Settings de su aplicación, en el campo Domain. Si usa un [dominio personalizado](https://auth0.com/docs/custom-domains), debe establecer esta propiedad con el
        valor de ese dominio personalizado.
      * `clientId`: El ID de la aplicación de Auth0 que configuró antes en esta guía de inicio rápido. Puede encontrarlo
        en el Auth0 Dashboard, en Settings de su aplicación, en el campo Client ID.
      * `clientSecret`: El secreto de la aplicación de Auth0 que configuró antes en esta guía de inicio rápido. Puede
        encontrarlo en el Auth0 Dashboard, en Settings de su aplicación, en el campo Client Secret.
      * `redirectUri`: La URL de su aplicación a la que desea que Auth0 redirija a los usuarios después de
        autenticarse. Esto corresponde a la URL de callback que configuró antes en esta guía de inicio rápido. También puede
        encontrar este valor en el Auth0 Dashboard, en Settings de su aplicación, en el campo Callback URLs. Asegúrese de que
        lo que ingrese en el código coincida con lo que configuró antes; de lo contrario, sus usuarios verán un error.
      * `cookieSecret`: Un valor secreto largo que se usa para cifrar la cookie de sesión. Puede generar una
        cadena adecuada ejecutando `openssl rand -hex 32` en la terminal.

      Ahora que sus usuarios pueden iniciar sesión y cerrar sesión, probablemente querrá poder obtener la [información del perfil](https://auth0.com/docs/users/concepts/overview-user-profile)
      asociada a los usuarios autenticados. Por ejemplo, puede que quiera mostrar el nombre o la
      foto de perfil de un usuario que haya iniciado sesión en su proyecto.

      El SDK de PHP de Auth0 proporciona información del usuario mediante el método `getCredentials()`. Revise el
      código de `profile.php` en el panel interactivo para ver un ejemplo de cómo usarlo.

      Dado que el método contiene información confidencial relacionada con la identidad del
      usuario, su disponibilidad depende del estado de autenticación del
      usuario. Para evitar errores de renderizado, siempre debe comprobar si el
      método `getCredentials()` devuelve un `object` o `null` para determinar si
      Auth0 ha autenticado al usuario antes de que su aplicación use los resultados.

        ##### Punto de control
