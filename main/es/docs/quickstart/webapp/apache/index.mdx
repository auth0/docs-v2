<div id="get-started">
  ## Primeros pasos
</div>

Este tutorial muestra cómo configurar Apache para agregar autenticación y autorización a tu aplicación web. Te recomendamos que inicies sesión para seguir esta guía de inicio rápido con ejemplos configurados para tu cuenta.

<Steps>
  <Step title="Instalar y habilitar el módulo mod_auth_openidc" stepNumber={1}>
    Primero, instala el módulo `mod_auth_openidc` para Apache.

    Puedes obtener los binarios desde [GitHub](https://github.com/OpenIDC/mod_auth_openidc/releases) e instalarlos en tu sistema operativo. Si tu sistema operativo no es compatible con ninguno de los binarios, también puedes [compilarlo desde el código fuente](https://github.com/OpenIDC/mod_auth_openidc/blob/master/INSTALL).

    Una vez que hayas instalado el módulo, habilítalo en Apache con el comando `a2enmod`. Para obtener más información, consulta [a2enmod on Ubuntu Manpage](https://manpages.ubuntu.com/manpages/focal/man8/a2enmod.8.html):

    ```shellscript
    a2enmod auth_openidc
    ```

    <Info>
      En Windows, puedes usar [este script de PowerShell](https://github.com/enderandpeter/win-a2enmod#installation) para hacer que `a2enmod` funcione en tu sistema.
    </Info>
  </Step>

  <Step title="Configurar el módulo con la información de tu cuenta de Auth0" stepNumber={2}>
    Actualiza tu nuevo archivo de configuración (`auth_openidc.conf`), ubicado en la carpeta `/etc/apache2/mods-available`.

    <Info>
      En Windows, debes usar el archivo `/apache/conf/httpd.conf`.
    </Info>

    <AuthCodeBlock children={configSnippet} language="apache" filename="auth_openidc.conf" />
  </Step>

  <Step title="Configurar Auth0" stepNumber={3}>
    En el [Auth0 Dashboard](https://manage.auth0.com/):

    1. Ve a **Applications** &gt; **Applications** y, a continuación, selecciona tu aplicación de la lista.
    2. Cambia a la vista **Settings** y luego localiza la sección **Application URIs**.
    3. Agrega el valor de `OIDCRedirectURI` a **Allowed Callback URLs**.
    4. Busca **Advanced Settings** en la parte inferior de la página.
    5. Cambia a la vista **OAuth**.
    6. Establece **JSON Web Token (JWT) Signature Algorithm** en `RS256`.
  </Step>

  <Step title="Autorización" stepNumber={4}>
    Puedes configurar Apache para proteger una ubicación específica según el valor de un claim en el token de ID del usuario; para ello, agrega un bloque `Location` a tu archivo `auth_openidc.conf`.

    Por ejemplo, podrías crear una Action que lea los roles del usuario y luego agregue un claim que conceda acceso a una ubicación protegida:

    ```js lines
    exports.onExecutePostLogin = async (event, api) => {
      const roles = event.authorization.roles; // ['user', 'admin']
      if (roles.includes('admin')) {
        api.idToken.setCustomClaim('folder', 'admin');
      }
    };
    ```
  </Step>
</Steps>
