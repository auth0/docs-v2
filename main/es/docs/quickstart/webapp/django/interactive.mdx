  <Content>
    Auth0 le permite agregar autenticación y acceder a la información del perfil del usuario en su aplicación. En esta guía
    se muestra cómo integrar Auth0 con una aplicación Python [Django](https://www.djangoproject.com/) mediante el SDK [Authlib](https://authlib.org/).

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Para usar los servicios de Auth0, necesitas tener una aplicación configurada en el Auth0 Dashboard. En la aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación para el proyecto que estás desarrollando.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación en Auth0 se le asigna un ID de cliente alfanumérico único
      que el código de tu aplicación usará para llamar a las API de Auth0 mediante el SDK.

      Cualquier configuración que realices con esta guía de inicio rápido se actualizará automáticamente en tu aplicación en el [Dashboard](https://manage.auth0.com/dashboard/us/auth0-dsepaid/), donde podrás administrar tus aplicaciones en el futuro.

      Si prefieres explorar una configuración completa, puedes ver una aplicación de ejemplo.

      ### Configurar URL de callback

      Una URL de callback es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como
        `http://localhost:3000/callback`.
      </Info>

      ### Configurar URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de cerrar
      sesión. Si no se configura, los usuarios no podrán cerrar sesión desde tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, configúrala como `http://localhost:3000`.
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Para esta integración, agregarás varias dependencias de bibliotecas, como Authlib. Crea un
      archivo `requirements.txt` en el directorio de tu proyecto e incluye lo siguiente:

      ```text lines
      authlib ~= 1.0
      django ~= 4.0
      python-dotenv ~= 0.19
      requests ~= 2.27
      ```

      Ejecute el siguiente comando en su shell para que estas dependencias estén disponibles:

      ```text lines
      pip install -r requirements.txt
      ```
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      A continuación, cree un archivo `.env` en el directorio de su proyecto. Este archivo contendrá las claves del cliente y otros
      detalles de configuración.

      ```text lines
      AUTH0_CLIENT_ID={yourClientId}
      AUTH0_CLIENT_SECRET={yourClientSecret}
      AUTH0_DOMAIN={yourDomain}
      ```
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Si ya tiene una aplicación de Django configurada, vaya al siguiente paso. Para un proyecto de aplicación nuevo, ejecute el
      siguiente comando:

      ```text lines
      django-admin startproject webappexample
      ```

      Ve a la nueva carpeta del proyecto:

      ```text lines
      cd webappexample
      ```
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Abra el archivo `webappexample/settings.py` para revisar los valores de `.env`.

      En la parte superior del archivo, agregue las importaciones de `os` y `dotenv`.

      A continuación, debajo de la definición de `BASE_DIR`, agregue la variable `TEMPLATE_DIR`.

      Luego, busque la variable `TEMPLATES` y actualice el valor de `DIRS` para añadir la
      cadena `TEMPLATE_DIR`. Esto determina la ruta de los archivos de plantilla, que creará en un
      paso posterior. Mantenga sin cambios cualquier otro contenido de este array.

      Al final de este archivo, agregue el código para cargar la configuración de Auth0.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Para empezar a crear su aplicación, abra el archivo `webappexample/views.py` en su IDE.

      Importe todas las bibliotecas que necesite su aplicación.

      Ahora puede configurar Authlib para que gestione la autenticación de su aplicación con Auth0.

      Para obtener más información sobre las opciones de configuración disponibles para el método OAuth `register()` de Authlib, consulte [su documentación.](https://docs.authlib.org/en/latest/client/frameworks.html#using-oauth-2-0-to-log-in)
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      En este ejemplo, agregará cuatro rutas para su aplicación: las rutas de inicio de sesión, callback, cierre de sesión e index.

      * `login` - Cuando los visitantes de su aplicación accedan a la ruta `/login`, llegarán a Auth0 para
        iniciar el flujo de autenticación.
      * `callback` - Después de que sus usuarios terminen de iniciar sesión con Auth0, volverán a su aplicación
        en la ruta `/callback`. Esta ruta guarda la sesión del usuario y evita que tenga
        que volver a iniciar sesión cuando regrese.
      * `logout` - La ruta `/logout` cierra la sesión de los usuarios en su aplicación. Esta ruta borra
        la sesión del usuario en su aplicación y redirige al endpoint de cierre de sesión de Auth0 para garantizar que la sesión ya no
        se conserve. Luego, la aplicación redirige al usuario a la ruta de inicio.
      * `index` - La ruta de inicio mostrará los datos de un usuario autenticado o permitirá a los visitantes iniciar sesión.
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Reemplaza el contenido del archivo `webappexample/urls.py` por el código de la derecha para conectar
      estas nuevas rutas.

      Esto hará que las rutas `/login`, `/callback`, `/logout` y `/`
      apunten a los controladores correctos.
    </Section>

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9">
      A continuación, creará un archivo de plantilla que se usará en la ruta de la página de inicio.

      Cree un nuevo subdirectorio dentro de la carpeta `webappexample` llamado `templates` y cree
      un archivo `index.html`.

      El archivo `index.html` contendrá código de plantilla para mostrar la información del usuario si ha iniciado sesión o
      mostrarle un botón de inicio de sesión si ha cerrado sesión.
    </Section>

    <Section id={sections[9].id} title={sections[9].title} stepNumber="10">
      ¡Ya puedes ejecutar tu aplicación! Desde el directorio de tu proyecto, abre una terminal y usa:

      ```text lines
      python3 manage.py migrate
      python3 manage.py runserver 3000
      ```

      Tu aplicación ya debería estar lista para abrirse en tu navegador en [http://localhost:3000](http://localhost:3000).

      <Info>
        ##### Punto de control

        Visita [http://localhost:3000](http://localhost:3000/) para comprobarlo. Deberías ver un botón de inicio de sesión que te redirige a
        Auth0 para iniciar sesión y luego de vuelta a tu aplicación, donde podrás ver la información de tu perfil.
      </Info>
    </Section>
