<div id="configure-auth0">
  ## Configura Auth0
</div>

<div id="get-your-application-keys">
  ### Obtén las claves de tu aplicación
</div>

Cuando te registraste en Auth0, se creó una nueva aplicación para ti, o es posible que hayas creado una nueva. Necesitarás algunos datos de esa aplicación para comunicarte con Auth0. Puedes obtenerlos en la sección [configuración de la aplicación](https://manage.auth0.com/#/applications) del Auth0 Dashboard.

<Frame>![Panel de la aplicación](https://cdn2.auth0.com/docs/1.14550.0/media/articles/dashboard/client_settings.png)</Frame>

Necesitas la siguiente información:

* **Dominio**
* **ID de cliente**
* **Secreto del cliente**

<Info>
  Si descargas el ejemplo de la parte superior de esta página, estos datos ya vendrán completos.
</Info>

<div id="configure-callback-urls">
  ### Configurar las URL de callback
</div>

Una URL de callback es una URL de tu aplicación a la que Auth0 redirige al usuario después de autenticarse. La URL de callback de tu aplicación debe añadirse al campo **Allowed Callback URLs** en la [configuración de la aplicación](https://manage.auth0.com/#/applications). Si este campo no está configurado, los usuarios no podrán iniciar sesión en la aplicación y recibirán un error.

<Info>
  Si estás siguiendo el proyecto de ejemplo que descargaste desde la parte superior de esta página, la URL de callback que debes agregar al campo **Allowed Callback URLs** es `http://localhost:3000/callback`.
</Info>

<div id="configure-logout-urls">
  ### Configurar las URL de cierre de sesión
</div>

Una URL de cierre de sesión es una URL de su aplicación a la que Auth0 puede redirigir después de que el usuario haya cerrado sesión en el servidor de autorización. Esto se especifica en el parámetro de consulta `returnTo`. La URL de cierre de sesión de su aplicación debe añadirse al campo **Allowed Logout URLs** en la [Configuración de la aplicación](https://manage.auth0.com/#/applications). Si este campo no está configurado, los usuarios no podrán cerrar sesión en la aplicación y recibirán un error.

<Info>
  Si está siguiendo el proyecto de ejemplo que descargó desde la parte superior de esta página, la URL de cierre de sesión que debe añadir al campo **Allowed Logout URLs** es `http://localhost:3000`.
</Info>

<div id="create-application">
  ## Crear una aplicación
</div>

En esta guía, mostramos cómo integrar Auth0 con una aplicación de Python mediante el [framework Django](https://www.djangoproject.com/) y [Authlib](https://authlib.org/). Para empezar, asegúrese de que Django esté instalado en su sistema. Desde la shell, ejecute el siguiente comando:

```bash lines
pip install django~=4.0
```

Si ya tiene una aplicación de Python configurada, puede pasar al siguiente paso. De lo contrario, vamos a crear el nuevo proyecto de la aplicación. Desde la consola, ejecute el siguiente comando y cambie a la carpeta del nuevo proyecto:

```bash lines
django-admin startproject webappexample
cd webappexample
```

<div id="install-dependencies">
  ## Instala las dependencias
</div>

Para esta integración, necesitarás algunas dependencias de bibliotecas, como Authlib. Crea un archivo `requirements.txt` en el directorio de tu proyecto e incluye lo siguiente:

```txt lines
# 📁 requirements.txt -----

authlib ~= 1.0
django ~= 4.0
python-dotenv ~= 0.19
requests ~= 2.27
```

Ahora debe ejecutar `pip install -r requirements.txt` en su terminal para que estas dependencias estén disponibles en su proyecto.

<div id="configure-your-env-file">
  ## Configura tu archivo `.env`
</div>

A continuación, crea un archivo `.env` en el directorio de tu proyecto. Este archivo contendrá las claves del cliente y otros detalles de configuración.

export const codeExample = `# 📁 .env -----

<div id="update-settingspy">
  ## Actualiza `settings.py`
</div>

Deberás hacer algunos cambios menores en tu archivo `webappexample/settings.py` para leer esos valores de `.env`. En la parte superior del archivo, agrega estas importaciones:

```py lines
# 📁 webappexample/settings.py -----

import os
from dotenv import load_dotenv, find_dotenv
```

A continuación, debajo de la definición de &#39;BASE&#95;DIR&#39;, añada lo siguiente:

```py lines
# 📁 webappexample/settings.py -----

# Agrega la siguiente línea después de 'BASE_DIR = ...'
TEMPLATE_DIR = os.path.join(BASE_DIR, "webappexample", "templates")
```

A continuación, busca la variable `TEMPLATES` y actualiza el valor de `DIRS` para añadir la cadena `TEMPLATE_DIR`. Esto le indica a Django dónde buscar los archivos de plantilla cuando los creemos. Mantén sin cambios cualquier otro contenido de este arreglo.

```py lines
# 📁 webappexample/settings.py -----

TEMPLATES = [
    {
        # Deja las demás líneas como están; solo actualizamos `DIRS`.
        "DIRS": [TEMPLATE_DIR],
    },
]
```

Por último, al final de este archivo, agrega lo siguiente:

```py lines
# 📁 webappexample/settings.py -----

# Cargar el archivo de definición de entorno
ENV_FILE = find_dotenv()
if ENV_FILE:
    load_dotenv(ENV_FILE)


# Cargar la configuración de la aplicación Auth0 en memoria
AUTH0_DOMAIN = os.environ.get("AUTH0_DOMAIN")
AUTH0_CLIENT_ID = os.environ.get("AUTH0_CLIENT_ID")
AUTH0_CLIENT_SECRET = os.environ.get("AUTH0_CLIENT_SECRET")
```

<div id="setup-your-application">
  ## Configura tu aplicación
</div>

Ahora ya puedes empezar a escribir tu aplicación. Abre el archivo `webappexample/views.py` en el directorio de tu proyecto.

Empieza importando todas las bibliotecas que utilizará tu aplicación:

```py lines
# 📁 webappexample/views.py -----

import json
from authlib.integrations.django_client import OAuth
from django.conf import settings
from django.shortcuts import redirect, render
from django.urls import reverse
from urllib.parse import quote_plus, urlencode
```

Ahora puedes configurar Authlib para encargarse de la autenticación de tu aplicación con Auth0:

```py lines
# 👆 Continuamos desde los pasos anteriores. Añade esto a tu archivo webappexample/views.py.

oauth = OAuth()

oauth.register(
    "auth0",
    client_id=settings.AUTH0_CLIENT_ID,
    client_secret=settings.AUTH0_CLIENT_SECRET,
    client_kwargs={
        "scope": "openid profile email",
    },
    server_metadata_url=f"https://{settings.AUTH0_DOMAIN}/.well-known/openid-configuration",
)
```

Puedes obtener más información sobre las opciones de configuración disponibles para el método OAuth `register()` de Authlib en [la documentación de Authlib.](https://docs.authlib.org/en/latest/client/frameworks.html#using-oauth-2-0-to-log-in)

<div id="setup-your-route-handlers">
  ## Configura tus manejadores de ruta
</div>

Para esta demostración, agregaremos 4 rutas a tu aplicación: las rutas de inicio de sesión, callback, cierre de sesión e inicio.

<div id="triggering-authentication-with-login">
  ### Activar la autenticación con /login
</div>

Cuando los visitantes de tu aplicación accedan a la ruta `/login`, se les redirigirá a Auth0 para iniciar el flujo de autenticación.

```py lines
# 👆 Continuamos con los pasos anteriores. Agrega esto a tu archivo webappexample/views.py.

def login(request):
    return oauth.auth0.authorize_redirect(
        request, request.build_absolute_uri(reverse("callback"))
    )
```

<div id="finalizing-authentication-with-callback">
  ### Completar la autenticación con /callback
</div>

Después de que sus usuarios terminen de iniciar sesión con Auth0, volverán a su aplicación en la ruta `/callback`. Esta ruta es la encargada de guardar la sesión del usuario para que, cuando vuelvan a visitarla más adelante, no tengan que iniciar sesión otra vez.

```py lines
# 👆 Continuamos desde los pasos anteriores. Agrega esto a tu archivo webappexample/views.py.

def callback(request):
    token = oauth.auth0.authorize_access_token(request)
    request.session["user"] = token
    return redirect(request.build_absolute_uri(reverse("index")))
```

<div id="clearing-a-session-with-logout">
  ### Borrar una sesión con /logout
</div>

Como era de esperar, esta ruta se encarga de cerrar la sesión de un usuario en tu aplicación. Borra la sesión del usuario en tu aplicación y lo redirige brevemente al endpoint de cierre de sesión de Auth0 para garantizar que su sesión se borre por completo antes de devolverlo a tu ruta de inicio (que se explica a continuación).

```py lines
# 👆 Continuamos desde los pasos anteriores. Añade esto a tu archivo webappexample/views.py.

def logout(request):
    request.session.clear()

    return redirect(
        f"https://{settings.AUTH0_DOMAIN}/v2/logout?"
        + urlencode(
            {
                "returnTo": request.build_absolute_uri(reverse("index")),
                "client_id": settings.AUTH0_CLIENT_ID,
            },
            quote_via=quote_plus,
        ),
    )
```

<div id="theres-no-place-like-home">
  ### No hay lugar como /home
</div>

Por último, la ruta de inicio servirá para mostrar los detalles de un usuario autenticado o permitir que los visitantes inicien sesión.

```py lines
# 👆 Continuamos desde los pasos anteriores. Agrega esto a tu archivo webappexample/views.py.

def index(request):
    return render(
        request,
        "index.html",
        context={
            "session": request.session.get("user"),
            "pretty": json.dumps(request.session.get("user"), indent=4),
        },
    )
```

<div id="register-your-routes">
  ### Registra tus rutas
</div>

Por último, debes indicarle a Django cómo conectar estas nuevas rutas. Reemplaza el contenido del archivo `webappexample/urls.py` por lo siguiente:

```py lines
# 📁 webappexample/urls.py -----

from django.urls import path

from . import views

urlpatterns = [
    path("", views.index, name="index"),
    path("login", views.login, name="login"),
    path("logout", views.logout, name="logout"),
    path("callback", views.callback, name="callback"),
]
```

<div id="add-templates">
  ## Agregar plantillas
</div>

Ahora solo necesitamos crear los archivos de plantilla sencillos que se usan en la ruta `about` (durante las llamadas a `render()`).

Cree un nuevo subdirectorio dentro de la carpeta `webappexample` llamado `templates` y cree un archivo `index.html`:

```html lines
# 📁 webappexample/templates/index.html -----

<html>
  <head>
    <meta charset="utf-8" />
    <title>Auth0 Example</title>
  </head>
  <body>
    {% if session %}
    <h1>Welcome {{session.userinfo.name}}!</h1>
    <p><a href="{% url 'logout' %}">Logout</a></p>
    <div><pre>{{pretty}}</pre></div>
    {% else %}
    <h1>Welcome Guest</h1>
    <p><a href="{% url 'login' %}">Login</a></p>
    {% endif %}
  </body>
</html>
```

<div id="run-your-application">
  ## Ejecute su aplicación
</div>

Ya puede ejecutar su aplicación. Desde el directorio de su proyecto, abra una consola y use:

```bash lines
python3 manage.py migrate
python3 manage.py runserver 3000
```

Tu aplicación ya debería estar lista para abrirse en tu navegador en `http://localhost:3000`.
