    Este tutorial muestra cómo usar el módulo nginx-openid-connect para agregar autenticación y autorización a
    su servidor NGINX. Le recomendamos iniciar sesión para seguir este inicio rápido con ejemplos configurados para su cuenta.

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Este tutorial y el proyecto base se han probado con lo siguiente:

      * NGINX Plus R24
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Primero, debe instalar el módulo `nginx-plus-module-njs` para NGINX Plus.

      Siga la guía de instalación de módulos dinámicos para instalar los paquetes en el sistema operativo de su host.

      En las distribuciones de Linux que usan el gestor de paquetes `yum`, puede instalar el módulo de la siguiente manera:

      ```sh lines
      sudo yum install nginx-plus-module-njs jq
      ```

      Una vez instalado, debe habilitarlo en NGINX agregando la siguiente línea cerca del principio del archivo
      `/etc/nginx/nginx.conf`:

      ```sh lines
      load_module modules/ngx_http_js_module.so;
      ```
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Clone el repositorio `nginx-openid-connect`. Este repositorio incluye una plantilla de configuración.

      ```sh lines
      git clone https://github.com/nginxinc/nginx-openid-connect
      ```
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Ejecute el script configure.sh dentro de la carpeta nginx-openid-connect para completar la plantilla de configuración de
      su aplicación de Auth0:

      ```sh lines
      ./configure.sh --auth_jwt_key request \
      --client_id {yourClientId} \
      --pkce_enable \
      https://{yourDomain}/.well-known/openid-configuration
      ```

      A continuación, agregue la [URL de cierre de sesión de su inquilino](https://auth0.com/docs/api/authentication#auth0-logout) a su
      archivo `openid_connect_configuration.conf`.
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Agregue el encabezado `Accept-Encoding` y establezca su valor en `gzip` en su
      archivo `openid_connect.server_conf`.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Copie sus cuatro archivos de configuración a la carpeta `conf.d`.

      ```sh lines
      sudo cp openid_connect.js \ 
      frontend.conf \
      openid_connect_configuration.conf \
      openid_connect.server_conf /etc/nginx/conf.d
      ```
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      En el Auth0 Dashboard:

      1. Vaya a **Applications** &gt; **Applications** y seleccione su aplicación de la lista.
      2. Cambie a la vista **Settings**.
      3. En la sección **Application URIs**, agregue `https://{yourDomain}/_codexch`
         a **Allowed Callback URLs**.
      4. Cambie a la vista **Credentials**.

      En la sección **Application Authentication**, establezca **Authentication Method** en None.
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Agregue encabezados adicionales del token de ID (JWT) al destino upstream en su
      archivo `/etc/nginx/conf.d/frontend.conf`.
    </Section>
