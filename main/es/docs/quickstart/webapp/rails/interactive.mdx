    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Para usar los servicios de Auth0, necesitas tener una aplicación configurada en el Auth0 Dashboard. La aplicación de Auth0 es
      donde configurarás cómo quieres que funcione la autenticación para el proyecto que estás desarrollando.

      ### Configurar una aplicación

      Usa el selector interactivo para crear una nueva aplicación de Auth0 o seleccionar una aplicación existente que represente
      el proyecto que quieres integrar. A cada aplicación de Auth0 se le asigna un ID de cliente alfanumérico único
      que el código de tu aplicación usará para llamar a las API de Auth0 a través del SDK.

      Cualquier configuración que realices con esta guía de inicio rápido se actualizará automáticamente en tu aplicación en el [Dashboard](https://manage.auth0.com/#/), que es donde
      podrás administrar tus aplicaciones más adelante.

      Si prefieres ver una configuración completa, puedes consultar una aplicación de ejemplo.

      ### Configurar URL de callback

      Una URL de callback es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que se hayan
      autenticado. Si no se configura, los usuarios no volverán a tu aplicación después de iniciar sesión.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, establece esto en
        `http://localhost:3000/auth/auth0/callback`.
      </Info>

      ### Configurar URL de cierre de sesión

      Una URL de cierre de sesión es una URL de tu aplicación a la que quieres que Auth0 redirija a los usuarios después de que hayan cerrado
      sesión. Si no se configura, los usuarios no podrán cerrar sesión en tu aplicación y recibirán un error.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, establece esto en `http://localhost:3000`.
      </Info>

      ### Configurar orígenes web permitidos

      Un origen web permitido es una URL desde la que quieres permitir el acceso a tu flujo de autenticación. Debe
      contener la URL de tu proyecto. Si no se configura correctamente, tu proyecto no podrá renovar silenciosamente
      los tokens de autenticación, por lo que tus usuarios cerrarán sesión la próxima vez que visiten tu aplicación o actualicen la
      página.

      <Info>
        Si estás siguiendo nuestro proyecto de ejemplo, establece esto en `http://localhost:3000`.
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Usa `omniauth-auth0`, una [estrategia personalizada de OmniAuth](https://github.com/intridea/omniauth#omniauth-standardized-multi-provider-authentication), para gestionar el flujo de autenticación.

      Agrega las siguientes dependencias a tu `Gemfile`:

      ```gem lines
      gem 'omniauth-auth0', '~> 3.0'
      gem 'omniauth-rails_csrf_protection', '~> 1.0' # evita solicitudes de autenticación falsificadas
      ```

      Una vez que hayas agregado las gemas, instálalas con `bundle install`.
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Cree un archivo de configuración `./config/auth0.yml` para especificar los valores del dominio de Auth0, el ID de cliente y el secreto del cliente, que se encuentran en la **Configuración** de la aplicación en Auth0 Dashboard.
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Cree el siguiente archivo de inicialización `./config/initializers/auth0.rb` y [configure](https://github.com/auth0/omniauth-auth0/blob/master/EXAMPLES.md#send-additional-authentication-parameters) el middleware de **OmniAuth** con el archivo de configuración
      que creó en el paso anterior.

      Asegúrese de que `callback_path` coincida con el valor especificado en la configuración &quot;Allowed Callback URLs&quot; de su
      aplicación de Auth0.
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Cree un controlador de Auth0 para gestionar el callback de autenticación, la acción `logout` y los métodos para
      construir la URL de cierre de sesión.

      Ejecute el comando:
      `rails generate controller auth0 callback failure logout --skip-assets --skip-helper --skip-routes --skip-template-engine`.

      Dentro del método callback, asigne el hash con la información del usuario, que se devuelve como
      `request.env['omniauth.auth']`, a la sesión activa.

      Para configurar el cierre de sesión, elimine todos los objetos almacenados en la sesión llamando al método `reset_session`
      dentro de la acción `logout`. Luego, redirija al endpoint de cierre de sesión de Auth0. Para obtener más información sobre
      `reset_session`, consulte la [documentación de ActionController de Ruby on Rails](http://api.rubyonrails.org/classes/ActionController/Base.html#M000668).
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Añade estas rutas al archivo `./config/routes.rb`.

      Las rutas deben estar definidas para que Rails sepa cómo dirigir las distintas URL de devolución de llamada de Auth0 al controlador de Auth0 que
      creaste en el paso anterior.

      <Info>
        ##### Punto de control

        Ejecuta la aplicación para verificar que sigue funcionando según lo previsto y que no recibes ningún error
        relacionado con Auth0.
      </Info>
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      Ahora, un usuario puede iniciar sesión en su aplicación visitando el endpoint `/auth/auth0`.

      <Warning>
        Para [evitar solicitudes de autenticación fraudulentas](https://github.com/cookpad/omniauth-rails_csrf_protection), use los métodos auxiliares `link_to` o
        `button_to` con el método `:post`.
      </Warning>

      ```ruby lines
      <!-- Coloca un botón de inicio de sesión en cualquier parte de tu aplicación -->
      <%= button_to 'Login', '/auth/auth0', method: :post %>
      ```

      <Info>
        ##### Punto de control

        Agregue un botón a su aplicación que redirija al usuario al endpoint `/auth/auth0` cuando
        se seleccione. Observe que se le redirige a Auth0 para iniciar sesión y luego de vuelta a su aplicación
        después de autenticarse correctamente.
      </Info>
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Ahora que ya puede iniciar sesión en su aplicación Rails, necesita una
      forma de cerrar sesión. Para cerrar la sesión de un usuario, rediríjalo a la acción `auth/logout`, que lo redirige
      al endpoint de cierre de sesión de Auth0.

      <Info>
        Para probar esto después del paso anterior, es posible que deba borrar la sesión y luego redirigir al usuario
        al endpoint de cierre de sesión de Auth0.
      </Info>

      ```ruby lines
      <!-- Coloca un botón de cierre de sesión en cualquier lugar de tu aplicación -->
      <%= button_to 'Logout', 'auth/logout', method: :get %>
      ```

      <Info>
        ##### Punto de control

        Agrega un botón a tu aplicación que redirija al usuario al endpoint `/auth/logout` cuando lo
        seleccione. Verifica que se te redirige a Auth0 y luego rápidamente de vuelta a tu aplicación, y que ya no has
        iniciado sesión.
      </Info>
    </Section>

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9">
      Para mostrar el perfil del usuario, la aplicación debe proporcionar una ruta protegida. Puede usar un [Concern](https://guides.rubyonrails.org/getting_started.html#using-concerns) para controlar el acceso a rutas que pueden compartirse entre varios
      controladores. El concern debe redirigir automáticamente a Auth0 cuando el usuario no esté autenticado. De lo contrario, el
      concern debe devolver el perfil del usuario actual.

      Una vez que tenga un Concern, inclúyalo en cualquier controlador que requiera que el usuario haya iniciado sesión. Luego podrá acceder al
      usuario desde la sesión `session[:userinfo]`, como en el siguiente ejemplo:

      ```ruby lines
      class DashboardController < ApplicationController
      include Secured
      def show
      @user = session[:userinfo]

      end
      end
      ```

      Una vez que el usuario se haya cargado desde la sesión, úsalo para mostrar información en tu frontend:

      ```html lines
      <div>
      <p>Normalized User Profile:<%= JSON.pretty_generate(@user[:info])%></p>
      <p>Full User Profile:<%= JSON.pretty_generate(@user[:extra][:raw_info])%></p>
      </div>
      ```

      <Info>
        ##### Punto de control

        Agrega el concern `Secured` a tu aplicación y luego inclúyelo en el controlador que requiera un
        usuario autenticado para acceder. Verifica que un usuario autenticado tenga acceso a las acciones dentro de ese
        controlador y que los usuarios no autenticados sean redirigidos a Auth0 para autenticarse.
      </Info>
    </Section>
