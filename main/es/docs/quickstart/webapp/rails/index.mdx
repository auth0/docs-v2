<div id="configure-auth0">
  ## Configura Auth0
</div>

<div id="get-your-application-keys">
  ### Obtén las claves de tu aplicación
</div>

Cuando te registraste en Auth0, se creó una nueva aplicación para ti, o puede que hayas creado una nueva. Necesitarás algunos datos de esa aplicación para comunicarte con Auth0. Puedes obtenerlos en la sección [configuración de la aplicación](https://manage.auth0.com/#/applications) del Auth0 Dashboard.

<Frame>![Panel de la aplicación](https://cdn2.auth0.com/docs/1.14550.0/media/articles/dashboard/client_settings.png)</Frame>

Necesitas la siguiente información:

* **Dominio**
* **ID de cliente**
* **Secreto del cliente**

<Info>
  Si descargas el ejemplo desde la parte superior de esta página, estos datos ya estarán completados.
</Info>

<div id="configure-callback-urls">
  ### Configurar las URL de callback
</div>

Una URL de callback es una URL de su aplicación a la que Auth0 redirige al usuario después de autenticarse. La URL de callback de su aplicación debe añadirse al campo **Allowed Callback URLs** en la [Configuración de la aplicación](https://manage.auth0.com/#/applications). Si este campo no está configurado, los usuarios no podrán iniciar sesión en la aplicación y recibirán un error.

<Info>
  Si está siguiendo el proyecto de ejemplo que descargó desde la parte superior de esta página, la URL de callback que debe añadir al campo **Allowed Callback URLs** es `http://localhost:3000/auth/auth0/callback`.
</Info>

<div id="configure-logout-urls">
  ### Configurar las URL de cierre de sesión
</div>

Una URL de cierre de sesión es una URL de su aplicación a la que Auth0 puede redirigir al usuario después de que haya cerrado sesión en el servidor de autorización. Esto se especifica en el parámetro de consulta `returnTo`. La URL de cierre de sesión de su aplicación debe agregarse al campo **Allowed Logout URLs** en la [configuración de la aplicación](https://manage.auth0.com/#/applications). Si este campo no está configurado, los usuarios no podrán cerrar sesión en la aplicación y recibirán un error.

<Info>
  Si está siguiendo el proyecto de ejemplo que descargó de la parte superior de esta página, la URL de cierre de sesión que debe agregar al campo **Allowed Logout URLs** es `http://localhost:3000`.
</Info>

  ## Instala el SDK de Auth0
</div>

<div id="install-dependencies">
  ### Instalar dependencias
</div>

Este tutorial usa `omniauth-auth0`, una [estrategia personalizada de OmniAuth](https://github.com/intridea/omniauth#omniauth-standardized-multi-provider-authentication), para gestionar el flujo de autenticación. Agrega las siguientes dependencias a tu `Gemfile`:

```bash lines
gem 'omniauth-auth0', '~> 3.0'
gem 'omniauth-rails_csrf_protection', '~> 1.0' # evita solicitudes de autenticación falsificadas
```

Una vez que hayas agregado las gems, instálalas con `bundle install`:

<div id="initialize-auth0-configuration">
  ### Inicializa la configuración de Auth0
</div>

Crea un archivo de configuración de Auth0 `./config/auth0.yml`

export const codeExample = `# ./config/auth0.yml
development:
  auth0_domain: {yourDomain}
  auth0_client_id: {yourClientId}
  auth0_client_secret: <TU SECRETO DEL CLIENTE DE AUTH0>`;

<AuthCodeBlock children={codeExample} language="yml" />

Cree el siguiente archivo de inicialización `./config/initializers/auth0.rb` y [configure](https://github.com/auth0/omniauth-auth0/blob/master/EXAMPLES.md#send-additional-authentication-parameters) el middleware **OmniAuth**.

```rb lines
# ./config/initializers/auth0.rb
AUTH0_CONFIG = Rails.application.config_for(:auth0)

Rails.application.config.middleware.use OmniAuth::Builder do
  provider(
    :auth0,
    AUTH0_CONFIG['auth0_client_id'],
    AUTH0_CONFIG['auth0_client_secret'],
    AUTH0_CONFIG['auth0_domain'],
    callback_path: '/auth/auth0/callback',
    authorize_params: {
      scope: 'openid profile'
    }
  )
end
```

<div id="add-an-auth0-controller">
  ### Agrega un controlador de Auth0
</div>

Crea un controlador de Auth0 para manejar el callback de autenticación. Ejecuta el comando `rails generate controller auth0 callback failure logout --skip-assets --skip-helper --skip-routes --skip-template-engine`.

Dentro del método `callback`, asigna a la sesión activa el hash con la información del usuario, que se devuelve como `request.env['omniauth.auth']`.

```rb lines
# ./app/controllers/auth0_controller.rb
class Auth0Controller < ApplicationController
  def callback
    # OmniAuth almacena la información devuelta por Auth0 y el IdP en request.env['omniauth.auth'].
    # En este código, se extrae el raw_info proporcionado por el id_token y se asigna a la sesión.
    # Consulta https://github.com/auth0/omniauth-auth0/blob/master/EXAMPLES.md#example-of-the-resulting-authentication-hash para obtener información completa sobre el contenido de 'omniauth.auth'.
    auth_info = request.env['omniauth.auth']
    session[:userinfo] = auth_info['extra']['raw_info']

    # Redirige a la URL deseada tras una autenticación exitosa
    redirect_to '/dashboard'
  end

  def failure
    # Gestiona los errores de autenticación -- Muestra una página de error (también puedes manejarlo con una redirección)
    @error_msg = request.params['message']
  end

  def logout
    # completarás esto en un paso posterior
  end
end
```

Añade las siguientes rutas a tu archivo `./config/routes.rb`:

```rb lines
# ./config/routes.rb
Rails.application.routes.draw do
  # ..
  get '/auth/auth0/callback' => 'auth0#callback'
  get '/auth/failure' => 'auth0#failure'
  get '/auth/logout' => 'auth0#logout'
end
```

<div id="add-login-to-your-application">
  ## Agrega el inicio de sesión a tu aplicación
</div>

Ahora, un usuario puede iniciar sesión en tu aplicación accediendo al endpoint `/auth/auth0`.

<Warning>
  Para [evitar solicitudes de autenticación falsificadas](https://github.com/cookpad/omniauth-rails_csrf_protection), usa los métodos auxiliares `link_to` o `button_to` con el método `:post`
</Warning>

```erb lines
<!-- Coloca un botón de inicio de sesión en cualquier lugar de tu aplicación -->
  <%= button_to 'Login', '/auth/auth0', method: :post, data: { turbo: false } %>
```

<div id="add-logout-to-your-application">
  ## Agrega el cierre de sesión a tu aplicación
</div>

Ahora que puedes iniciar sesión en tu aplicación de Rails, necesitas [una forma de cerrar sesión](https://auth0.com/docs/logout/guides/logout-auth0). Vuelve al Auth0Controller que creaste antes y termina de completar el método de cierre de sesión que agregaste allí.

Tendrás que borrar la sesión y luego redirigir al usuario al endpoint de cierre de sesión de Auth0.

Para borrar todos los objetos almacenados en la sesión, llama al método `reset_session` dentro del método `auth0_controller/logout`. [Obtén más información sobre reset&#95;session aquí](http://api.rubyonrails.org/classes/ActionController/Base.html#M000668). Agrega el siguiente código a `./app/controllers/auth0_controller.rb`:

```rb lines
# ./app/controllers/auth0_controller.rb
class Auth0Controller < ApplicationController
  # ..
  # ..Inserta el código a continuación
  def logout
    reset_session
    redirect_to logout_url, allow_other_host: true
  end

  private

  def logout_url
    request_params = {
      returnTo: root_url,
      client_id: AUTH0_CONFIG['auth0_client_id']
    }

    URI::HTTPS.build(host: AUTH0_CONFIG['auth0_domain'], path: '/v2/logout', query: request_params.to_query).to_s
  end
end
```

<Info>
  La URL de destino final (el valor de `returnTo`) debe estar en la lista de `Allowed Logout URLs`. Consulte la [documentación sobre cierre de sesión](/es/docs/authenticate/login/logout/redirect-users-after-logout) para obtener más información.
</Info>

El usuario ahora podrá cerrar sesión en su aplicación accediendo al endpoint `/auth/logout`.

```erb lines
<!-- Coloca un botón de cierre de sesión en cualquier lugar de tu aplicación -->
  <%= button_to 'Logout', 'auth/logout', method: :get, data: { turbo: false } %>
```

<div id="show-user-profile-information">
  ## Mostrar la información del perfil del usuario
</div>

Para mostrar el perfil del usuario, la aplicación debe proporcionar una ruta protegida. Puede usar un `concern` del controlador para controlar el acceso a las rutas. Cree el siguiente archivo `./app/controllers/concerns/secured.rb`

```rb lines
# ./app/controllers/concerns/secured.rb
module Secured
  extend ActiveSupport::Concern

  included do
    before_action :logged_in_using_omniauth?
  end

  def logged_in_using_omniauth?
    redirect_to '/' unless session[:userinfo].present?
  end
end
```

Después de crear el concern del controlador seguro, inclúyelo en cualquier controlador que requiera que un usuario haya iniciado sesión. Luego puedes acceder al usuario desde la sesión `session[:userinfo]`.

```rb lines
# ./app/controllers/dashboard_controller.rb
class DashboardController < ApplicationController
  include Secured

  def show
    # session[:userinfo] se guardó anteriormente en Auth0Controller#callback
    @user = session[:userinfo]
  end
end
```

Ahora que has cargado el usuario desde la sesión, puedes usarlo para mostrar información en el frontend.

```erb lines
<!-- ./app/views/dashboard/show.html.erb -->
<div>
  <p>Normalized User Profile:<%= JSON.pretty_generate(@user[:info])%></p>
  <p>Full User Profile:<%= JSON.pretty_generate(@user[:extra][:raw_info])%></p>
</div>
```
