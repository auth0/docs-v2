<div id="by-evan-sims">
  ##### Por Evan Sims
</div>

El SDK de Laravel de Auth0 le permite añadir rápidamente autenticación, gestión de perfiles de usuario y control de acceso a rutas a su aplicación de Laravel. Esta guía muestra cómo integrar Auth0 con una aplicación de Laravel 11 o 12, nueva o existente. Le recomendamos iniciar sesión para seguir esta guía de inicio rápido con ejemplos configurados para su cuenta.

<div id="laravel-installation">
  ## Instalación de Laravel
</div>

**Si aún no tiene configurada una aplicación de Laravel**, abra una terminal en un directorio adecuado para un proyecto nuevo y ejecute el siguiente comando:

```bash lines
composer create-project laravel/laravel:^12 auth0-laravel-app
```

Todos los comandos de esta guía suponen que los ejecuta desde el directorio raíz de su proyecto de Laravel, por lo que debe usar `cd` para acceder al directorio del proyecto nuevo:

```bash lines
cd auth0-laravel-app
```

<div id="sdk-installation">
  ## Instalación del SDK
</div>

Ejecute el siguiente comando en el directorio de su proyecto para instalar el [SDK de Laravel de Auth0](https://github.com/auth0/laravel-auth0):

```bash lines
composer require auth0/login:^7 --update-with-all-dependencies
```

A continuación, genere un archivo de configuración del SDK para su aplicación:

```bash lines
php artisan vendor:publish --tag auth0
```

<div id="sdk-configuration">
  ## Configuración del SDK
</div>

Ejecute el siguiente comando desde el directorio del proyecto para descargar la [Auth0 CLI](https://github.com/auth0/auth0-cli):

```bash lines
curl -sSfL https://raw.githubusercontent.com/auth0/auth0-cli/main/install.sh | sh -s -- -b .
```

Luego, autentica la CLI con tu cuenta de Auth0 y elige &quot;como usuario&quot; cuando se te indique:

```bash lines
./auth0 login
```

A continuación, cree una aplicación nueva en Auth0:

```bash lines
./auth0 apps create \
  --name "My Laravel Application" \
  --type "regular" \
  --auth-method "post" \
  --callbacks "http://localhost:8000/callback" \
  --logout-urls "http://localhost:8000" \
  --reveal-secrets \
  --no-input \
  --json > .auth0.app.json
```

También debe crear una API nueva:

```bash lines
./auth0 apis create \
  --name "My Laravel Application's API" \
  --identifier "https://github.com/auth0/laravel-auth0" \
  --offline-access \
  --no-input \
  --json > .auth0.api.json
```

Esto genera dos archivos en el directorio del proyecto que configuran el SDK.

Como estos archivos contienen credenciales, es importante tratarlos como información confidencial. Asegúrate de no incluirlos en el control de versiones. Si usas Git, debes añadirlos a tu archivo `.gitignore`:

```bash lines
echo ".auth0.*.json" >> .gitignore
```

<div id="login-routes">
  ## Rutas de inicio de sesión
</div>

El SDK registra automáticamente todas las rutas necesarias para que los usuarios de su aplicación se autentiquen.

| Ruta        | Propósito                                  |
| ----------- | ------------------------------------------ |
| `/login`    | Inicia el flujo de autenticación.          |
| `/logout`   | Cierra la sesión del usuario.              |
| `/callback` | Procesa la devolución de llamada de Auth0. |

Si necesita un mayor control sobre estas rutas, o si entran en conflicto con rutas existentes en su aplicación, puede registrar manualmente los controladores del SDK. Consulte [el README del SDK](https://github.com/auth0/laravel-auth0) para integraciones avanzadas.

<div id="access-control">
  ## Control de acceso
</div>

Las funciones de autenticación de Laravel usan &quot;guards&quot; para definir cómo se autentica a los usuarios en cada solicitud. Puede usar el guard de autenticación del SDK de Auth0 para restringir el acceso a las rutas de su aplicación.

Para exigir que los usuarios se autentiquen antes de acceder a una ruta, puede usar el middleware `auth` de Laravel:

```php lines
Route::get('/private', function () {
  return response('Welcome! You are logged in.');
})->middleware('auth');
```

También puede exigir que los usuarios autenticados tengan [permisos](https://auth0.com/docs/manage-users/access-control/rbac) específicos al combinar esto con el middleware `can` de Laravel:

```php lines
Route::get('/scope', function () {
    return response('You have the `read:messages` permissions, and can therefore access this resource.');
})->middleware('auth')->can('read:messages');
```

<div id="user-information">
  ## Información del usuario
</div>

La información sobre el usuario autenticado está disponible a través de la fachada `Auth` de Laravel o de la función auxiliar `auth()`.

Por ejemplo, para recuperar el identificador y el correo electrónico del usuario:

```php lines
Route::get('/', function () {
  if (! auth()->check()) {
    return response('You are not logged in.');
  }

  $user = auth()->user();
  $name = $user->name ?? 'User';
  $email = $user->email ?? '';

  return response("Hello {$name}! Your email address is {$email}.");
});
```

<div id="user-management">
  ## Gestión de usuarios
</div>

Puede actualizar la información del usuario mediante la [Auth0 Management API](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md). Se puede acceder a todos los endpoints de Management a través del método `management()` del SDK.

**Antes de realizar llamadas a la Auth0 Management API, debe habilitar su aplicación para que se comunique con la Auth0 Management API.** Puede hacerlo desde la [página de API del Auth0 Dashboard](https://manage.auth0.com/#/apis/), donde debe elegir `Auth0 Management API` y seleccionar la pestaña &#39;Machine to Machine Applications&#39;. Autorice su aplicación de Laravel y, a continuación, haga clic en la flecha desplegable para elegir los alcances que desea otorgar.

Para el siguiente ejemplo, en el que actualizaremos los metadatos de un usuario y le asignaremos un color favorito aleatorio, debe otorgar los alcances `read:users` y `update:users`. Puede encontrar una lista de endpoints de API y los alcances requeridos en [la documentación de la Auth0 Management API](https://auth0.com/docs/api/management/v2).

```php lines
use Auth0\Laravel\Facade\Auth0;

Route::get('/colors', function () {
  $endpoint = Auth0::management()->users();

  $colors = ['red', 'blue', 'green', 'black', 'white', 'yellow', 'purple', 'orange', 'pink', 'brown'];

  $endpoint->update(
    id: auth()->id(),
    body: [
        'user_metadata' => [
            'color' => $colors[random_int(0, count($colors) - 1)]
        ]
    ]
  );

  $metadata = $endpoint->get(auth()->id());
  $metadata = Auth0::json($metadata);

  $color = $metadata['user_metadata']['color'] ?? 'unknown';
  $name = auth()->user()->name;

  return response("Hello {$name}! Your favorite color is {$color}.");
})->middleware('auth');
```

Hay disponible [aquí](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md) una guía de referencia rápida de todos los métodos de Auth0 Management API del SDK.

<div id="run-the-application">
  ## Ejecuta la aplicación
</div>

Ahora ya puedes iniciar tu aplicación de Laravel para que pueda aceptar solicitudes:

```bash lines
php artisan serve
```

<div id="checkpoint">
  ### Comprobación
</div>

Abre tu navegador web e intenta acceder a las siguientes rutas:

* `http://localhost:8000` para ver la ruta pública.
* `http://localhost:8000/private` para que se te pida autenticarte.
* `http://localhost:8000` para ver la ruta pública, ahora ya autenticado.
* `http://localhost:8000/scope` para comprobar si tienes el [permiso](https://auth0.com/docs/manage-users/access-control/rbac) `read:messages`.
* `http://localhost:8000/update` para actualizar el perfil del usuario.
* `http://localhost:8000/logout` para cerrar sesión.

Si tienes algún problema, aquí tienes un par de cosas que puedes probar:

* Prueba a ejecutar `php artisan optimize:clear` para borrar la caché de Laravel.
* Asegúrate de que tus archivos `.auth0.app.json` y `.auth0.api.json` estén en la raíz de tu proyecto.
* Asegúrate de haber habilitado tu aplicación de Laravel como una aplicación de máquina a máquina y de haberle concedido todos los alcances necesarios para la `Auth0 Management API` desde el [Auth0 Dashboard](https://manage.auth0.com/#/apis/).

¿Tienes problemas? Consulta la [documentación](https://github.com/auth0/laravel-auth0) del SDK o nuestro [centro de documentación](https://auth0.com/docs). También puedes visitar [la comunidad](https://community.auth0.com), donde nuestro equipo y otros miembros de la comunidad pueden ayudarte a responder tus preguntas.
