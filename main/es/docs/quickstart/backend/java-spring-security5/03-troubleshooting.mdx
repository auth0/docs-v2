<div id="by-jim-anderson">
  ##### Por Jim Anderson

Este documento le ayudará a solucionar problemas en su configuración si recibe respuestas inesperadas de su API. Le recomendamos iniciar sesión para seguir esta guía de inicio rápido con ejemplos configurados para su cuenta.

Si configuró correctamente la validación de JWT, podrá obtener respuestas correctas de su API al realizar solicitudes. Sin embargo, si recibe una respuesta 401 (No autorizado) de su API, se debe a que la configuración de su middleware de JWT no coincide con el JWT proporcionado.

Este documento le ayudará a solucionar problemas en la configuración de su middleware de JWT.

<div id="how-does-a-token-get-validated">
  ## ¿Cómo se valida un token?
</div>

Al validar un JWT, hay varios aspectos que debes tener en cuenta:

1. **¿El token tiene un formato válido?** En otras palabras, ¿este token se ajusta a la estructura de un JSON Web Token (JWT)? Para obtener más información sobre la estructura de un JWT, consulta [esta sección sobre la estructura de un JWT](/es/docs/secure/tokens/json-web-tokens#what-is-the-json-web-token-structure-)
2. **¿Se ha manipulado el token?** La última parte de un JWT es la firma. La firma se utiliza para verificar que el token fue firmado realmente por el emisor y que no ha sido alterado de ninguna manera.
3. **¿Se recibió el token dentro de su período de validez?** Los JWT solo son válidos durante un período de tiempo determinado (como se indica en el claim `exp`).
4. **¿El token proviene de la autoridad prevista?** Esto consta de 2 partes

   * **Verificación de la firma**: ¿Podemos confirmar que el JWT está correctamente firmado con la clave emitida por la autoridad emisora?
   * **Valor del emisor**: El emisor se define en el claim `iss`. Una vez más, ¿este claim coincide con lo que espera tu aplicación?
5. **¿El token está destinado a la aplicación actual?** Es decir, ¿el claim `aud` del JWT coincide con lo que espera tu aplicación?

<div id="inspecting-a-token">
  ## Inspeccionar un token
</div>

Una forma rápida de inspeccionar un JWT es usar el sitio web [JWT.io](https://jwt.io/). Tiene un práctico depurador que permite comprobar rápidamente si un JWT tiene el formato correcto y también inspeccionar los valores de las distintas claims.

<Frame>![Depuración de un JWT en JWT.io](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/aspnet-core-webapi/jwt-io-debugger-rs256.png)</Frame>

En la captura de pantalla anterior puede ver que el token se firmó con el algoritmo **RS256**. El **emisor** del token es **[https://jerrie.auth0.com/](https://jerrie.auth0.com/)** y la **Audiencia** es **[https://rs256.test.api](https://rs256.test.api)**.

En otras palabras, estos valores de la configuración de validación del JWT deben coincidir **exactamente**, incluida la barra diagonal final del emisor, por ejemplo

```
audience = "https://rs256.test.api",
domain = "https://jerrie.auth0.com/"
};
```

En el caso de un token firmado con HS256, la vista del depurador será un poco diferente:

<Frame>![Depuración de un JWT en JWT.io](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/aspnet-core-webapi/jwt-io-debugger-hs256.png)</Frame>

En la captura de pantalla anterior puede ver que el token se firmó con el algoritmo **HS256**. El **emisor** del token es **[https://jerrie.auth0.com/](https://jerrie.auth0.com/)** y la **Audiencia** es **[https://hs256.test.api](https://hs256.test.api)**.
