    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 proporciona un [SDK
      de PHP](https://github.com/auth0/auth0-PHP) (Auth0-PHP) para simplificar la implementación de la autenticación y la autorización de Auth0 en aplicaciones PHP.

      El SDK de PHP de Auth0 requiere que [PSR-17](https://www.php-fig.org/psr/psr-17/) y [PSR-18](https://www.php-fig.org/psr/psr-18/) estén instalados, junto con bibliotecas HTTP compatibles para gestionar las solicitudes de red. Si no
      dispone de bibliotecas, puede instalar opciones fiables ejecutando los siguientes comandos en su
      terminal:

      ```bash lines
      cd <your-project-directory>
      composer require symfony/http-client nyholm/psr7
      ```

      Ahora, instala el SDK de PHP de Auth0 ejecutando el siguiente comando en la terminal:

      ```bash lines
      composer require auth0/auth0-php
      ```

      ### Configura el SDK de Auth0

      Para que el SDK funcione correctamente, debes establecer las siguientes propiedades en el SDK de Auth0 durante la inicialización:

      * `strategy`: Esta estrategia ayuda a definir el comportamiento del SDK según el caso de uso de tu aplicación. En este
        caso, debes establecerla en la constante

        `Auth0\SDK\Configuration\SdkConfiguration::STRATEGY_API`
      * domain: El dominio de tu inquilino de Auth0. Por lo general, puedes encontrarlo en Auth0 Dashboard, en la
        configuración de la aplicación, en el campo Domain. Si usas un dominio personalizado, establece este valor
        con el de tu dominio personalizado.
      * clientId: El ID de la aplicación de Auth0 que configuraste anteriormente en esta guía de inicio rápido. Puedes encontrarlo en
        Auth0 Dashboard, en la configuración de tu aplicación, en el campo Client ID.
      * clientSecret: El secreto de la aplicación de Auth0 que creaste anteriormente en esta guía de inicio rápido. El secreto del cliente está en
        Auth0 Dashboard, en la configuración de tu aplicación, en el campo Client Secret.
      * audience: El identificador de la API de Auth0 que registraste anteriormente. Debe proporcionarse como un array.

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3" isSingleColumn>
      A continuación, amplíe su aplicación para recuperar y procesar tokens Bearer. Los tokens Bearer son tokens de acceso que se proporcionan a
      su API en solicitudes de clientes en nombre de los usuarios. Los tokens de acceso aprueban o deniegan el acceso a las rutas de su
      aplicación. Esto se conoce como autorización de endpoints.

      La forma más sencilla de recuperar tokens de acceso de una solicitud es usar el método `getBearerToken()`
      del SDK de PHP. Este método obtiene tokens de parámetros GET, cuerpos POST, encabezados de solicitud y otras fuentes. En este
      caso, el SDK de PHP procesa los tokens que se pasan en solicitudes GET mediante el parámetro `token` o mediante el encabezado HTTP
      `Authorization`.

      Ahora que ha configurado su aplicación de Auth0 y el SDK de PHP de Auth0, y que su aplicación recupera tokens bearer
      de las solicitudes, el siguiente paso es configurar la autorización de endpoints para su proyecto. El
      método `getBearerToken()` que implementó anteriormente devuelve una clase `Token` que incluye
      detalles sobre el acceso de la solicitud.

      Dado que el método `getBearerToken()` valida y verifica automáticamente la solicitud entrante, su
      aplicación determina los detalles del token de acceso evaluando la respuesta del método. Cuando la respuesta es
      null, no se ha proporcionado ningún token válido. De lo contrario, inspeccione el contenido de la respuesta para obtener más información sobre la
      solicitud.

      En el panel interactivo de la derecha, puede ver una comprobación de si la respuesta es null o no para controlar el acceso a
      su ruta `/api/private`.
