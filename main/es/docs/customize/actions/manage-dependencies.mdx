Actions te permite usar paquetes del [registro de `npm`](https://www.npmjs.com/). Puedes instalar y administrar dependencias para tus Actions con el editor de código de Actions en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o con la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de Auth0.

<div id="add-a-dependency-using-the-actions-code-editor">
  ## Agregar una dependencia con el Editor de código de Actions
</div>

Para agregar como dependencia un paquete del registro público de `npm`:

1. Vaya a [Auth0 Dashboard &gt; Actions &gt; Library](https://manage.auth0.com/#/actions/library) y seleccione su Action.
2. Busque el Editor de código de Actions y, en su barra lateral, seleccione Modules (icono de cubo).
3. Seleccione **Add Module** e introduzca el nombre del módulo. Para usar la versión más reciente disponible del paquete, puede dejar el campo de versión en blanco.
4. En su código de Actions, requiera el módulo. Por ejemplo, si agregó el paquete Axios, añadiría la siguiente línea al principio de su código en el Editor de código de Actions:

   ```javascript lines
   const axios = require('axios');
   ```

<div id="add-a-dependency-using-the-auth0-management-api">
  ## Agrega una dependencia con la Auth0 Management API
</div>

Como alternativa, puedes agregar una dependencia con el [endpoint Create Action](https://auth0.com/docs/api/management/v2#!/Actions/post_action) de la Auth0 Management API. Al llamar al endpoint, especifica las dependencias en la carga útil:

```bash lines
curl -H "Authorization: Bearer {managementApiToken}" \
     -X POST  -H "Content-Type: application/json" \
     -d '{ \
            "name":"my-action", \
            "supported_triggers":[{"id":"post-login","version":"v2"}], \
            "code":"module.exports = () => {}",\
            "dependencies": [{\
                "name":"lodash",\
                "version":"4.17.21"\
            }],\
            "runtime":"node14"\
        }' \
    https://{yourTenant}.com/api/v2/actions/actions
```

<div id="security-considerations">
  ## Consideraciones de seguridad
</div>

Los paquetes de terceros pueden contener vulnerabilidades y código malicioso, por lo que recomendamos auditar los paquetes antes de usarlos en tus Actions. También recomendamos supervisarlos de forma continua para detectar vulnerabilidades mediante herramientas de análisis de composición de software.

<div id="limitations-for-dependencies-in-actions">
  ## Limitaciones de las dependencias de Actions
</div>

<div id="no-native-module-support">
  ### Sin compatibilidad con módulos nativos
</div>

Los módulos nativos no se pueden usar en Actions. Al ejecutar una Action que depende de un módulo nativo, la Action mostrará un error con un mensaje que contiene `Compilation failed: Cannot find module`.

Para comprobar si un paquete de `npm` es nativo, ve al repositorio público de GitHub del paquete y revisa los lenguajes que usa el código en la barra lateral. Si entre los lenguajes figuran C/C++, es probable que el paquete sea nativo. Algunos ejemplos populares son `bcrypt` y `sqlite3`.

<div id="no-private-module-or-private-registry-support">
  ### No se admiten módulos privados ni registros privados
</div>

Los módulos privados de `npm` y los registros privados de `npm` no se pueden usar con Actions. Actions usa el [registro público de `npm`](https://registry.npmjs.org).
