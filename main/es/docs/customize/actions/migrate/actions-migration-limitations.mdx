---
description: Limitaciones de las actividades específicas de migración al migrar de Rules y Hooks a Actions
title: Limitaciones de la migración a Actions
---

Para aprovechar las funciones que ofrece Auth0 Actions, debe migrar su código actual desde Rules, Hooks o versiones anteriores de Actions. No obstante, debe tener en cuenta las limitaciones actuales al migrar entre las soluciones de extensibilidad heredadas y Actions.

<div id="general-differences">
  ## Diferencias generales
</div>

* Actions solo puede devolver códigos de error `access denied` y no admite devolver un código de error `unauthorized`.
* Los claims estándar de un `accessToken` no se pueden eliminar mediante una Action.

<div id="rule-specific-differences">
  ## Diferencias específicas de Rules
</div>

* Actualmente, el objeto event de las Actions de Post Login no ofrece una alternativa a `context.connectionOptions`, que sí está disponible en Rules.
* Actualmente, el objeto event de las Actions de Post Login no ofrece una alternativa para acceder, a través del objeto `user.identities`, a ningún token del IdP de origen para usuarios federados, como sí es posible en Rules.
* Los claims estándar de un `idToken` no se pueden eliminar mediante una Action.
* Actions solo permite modificar el usuario principal entre cuentas vinculadas.
