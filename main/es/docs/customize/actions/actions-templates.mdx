title: Plantillas de Actions
---

La galería de plantillas de Actions ofrece una variedad de plantillas de inicio para crear una Action. Estas plantillas proporcionan código de ejemplo para distintos triggers y casos de uso, como <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor adicional además de username y contraseña, como un code por SMS." cta="Ver glosario" href="/es/docs/glossary?term=Multi-factor+Authentication">Autenticación multifactor</Tooltip>, control de acceso y enriquecimiento de perfiles.

Las plantillas proporcionan código inicial que puede modificar y usar como bloque básico dentro de un trigger. Una vez que seleccione una plantilla para su caso de uso específico, podrá personalizar el código para adaptarlo a las necesidades concretas de su aplicación.

Para obtener una guía paso a paso sobre cómo usar las plantillas de Actions, consulte [Escriba su primera Action](/es/docs/customize/actions/write-your-first-action).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las plantillas de Actions también están disponibles en: [https://github.com/auth0/opensource-marketplace](https://github.com/auth0/opensource-marketplace).
</Callout>

<div id="best-practices">
  ## Prácticas recomendadas
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las plantillas de Action están diseñadas únicamente con fines educativos.

  Estas plantillas constan de fragmentos de código incompletos y no deben utilizarse en un entorno de producción bajo ninguna circunstancia. Los desarrolladores son responsables de desarrollar de forma segura y completar la plantilla de Action antes de implementarla en un entorno de producción.
</Callout>

* Asegúrese de que las plantillas se adapten a su entorno específico. Dado que el código está incompleto, debe personalizar y completar los ejemplos antes de importarlos en un entorno de producción.
* No implemente una plantilla sin editarla y completarla previamente.
* Las plantillas de Actions suelen contener etiquetas en los comentarios que no deben modificarse ni eliminarse.
* Utilice siempre la versión más reciente de cualquier paquete de `npm`.
* De forma predeterminada, las plantillas de Actions están orientadas a las versiones más recientes de Node.js. Después de crear una plantilla, puede modificarla para que admita versiones adicionales de Node.js según sea necesario.

Revise todas las [Directrices de codificación de Actions](/es/docs/customize/actions/action-coding-guidelines), ya que también se aplican aquí.
