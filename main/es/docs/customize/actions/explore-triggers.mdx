description: Aprende sobre los activadores, que son eventos que invocan automáticamente una Action cuando, en tiempo de ejecución, se produce una operación específica, como el inicio de sesión de un usuario.
sidebarTitle: Información general
title: Explora los activadores
---

Los activadores son puntos del flujo de Auth0 en los que se pueden agregar Actions para realizar la personalización necesaria. Se pueden colocar varias Actions en un activador, que se ejecutan en secuencia. Excepto Intercambio de tokens personalizado, los activadores pueden ser síncronos (bloqueantes) o asíncronos (no bloqueantes). Cada activador tiene objetos y API específicos. Estos pueden variar según tu conexión (como una conexión <Tooltip tip="Sin contraseña: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=passwordless">sin contraseña</Tooltip>).

Para explorar los objetos y las API de referencia de Actions específicas para un activador, selecciona el activador a continuación, teniendo en cuenta la definición de Auth0 de una [conexión sin contraseña](/es/docs/authenticate/passwordless).

<div id="signup-and-login">
  #### Registro e inicio de sesión
</div>

Cuando se agrega un usuario a una conexión de base de datos o sin contraseña (registro), o durante la transacción de autenticación de un usuario (inicio de sesión).

| Activador                | Descripción                                                                            |
| ------------------------ | -------------------------------------------------------------------------------------- |
| `pre-user-registration`  | Se activa antes de que se cree un usuario.                                             |
| `post-user-registration` | Se activa de forma asíncrona después de que se cree un usuario.                        |
| `post-login`             | Se activa después de que se autentica un usuario, pero antes de que se emita un token. |

<div id="mfa-notifications">
  #### Notificaciones de MFA
</div>

Al usar SMS como factor para <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor adicional al username y la contraseña, como un code enviado por SMS." cta="Ver glosario" href="/es/docs/glossary?term=Multi-Factor+Authentication">Autenticación multifactor</Tooltip> (MFA) o para configurar un proveedor personalizado.

| Activador            | Descripción                                                                                                            |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `send-phone-message` | Se activa cuando se usa un proveedor personalizado para enviar los mensajes del proceso de inscripción y verificación. |

<div id="password-reset">
  #### Restablecimiento de contraseña
</div>

Cuando se inicia una solicitud de restablecimiento de contraseña.

| Trigger                         | Descripción                                                                            |
| ------------------------------- | -------------------------------------------------------------------------------------- |
| `post-change-password`          | Se activa después de cambiar una contraseña.                                           |
| `password-reset-post-challenge` | Se activa después de completar el primer desafío y antes de restablecer la contraseña. |

<div id="machine-to-machine">
  #### Máquina a máquina
</div>

Cuando una aplicación, en lugar de un usuario, se autentica y se autoriza para usar sus API.

| Activador             | Descripción                                            |
| ---------------------- | ------------------------------------------------------ |
| `credentials-exchange` | Se activa antes de que se devuelva un token de acceso. |

<div id="custom-token-exchange">
  ### Intercambio de tokens personalizado
</div>

Cuando se recibe una solicitud de intercambio de tokens personalizado para iniciar la sesión de un usuario.

| Activador               | Descripción                                                                                       |
| ----------------------- | ------------------------------------------------------------------------------------------------- |
| `custom-token-exchange` | Se activa en el primer paso de la transacción, antes del activador posterior al inicio de sesión. |
