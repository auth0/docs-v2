La bóveda le permite almacenar de forma segura secretos o configuraciones comunes (por ejemplo: claves de API, <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+tokens">tokens de acceso</Tooltip>, <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=refresh+tokens">tokens de actualización</Tooltip>) para autenticar solicitudes con servicios externos o integraciones como conexiones de Vault.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Hay conexiones que requieren que el usuario proporcione credenciales (por ejemplo, en Auth0 con una aplicación de máquina a máquina (M2M), se genera un token de acceso con un tiempo de expiración y un token de actualización). La conexión almacena ambos tokens y genera un nuevo token de acceso antes de que expire.
  * Hay servicios externos que requieren autorizar alcances para generar credenciales de API. Si cambia los alcances, actualice la conexión de esos servicios para evitar problemas de conexión.
  * Cuando se agrega una conexión, está lista para usarse en sus flujos. Cualquier actualización de una conexión existente se propagará automáticamente.
</Callout>

<div id="create-and-modify-vault-connections">
  ## Crear y modificar conexiones de Vault
</div>

Cree y modifique conexiones de Vault desde la página de Vault o desde el editor de flujos cuando una acción requiera credenciales.

<div id="create-vault-connections-from-the-vault-page">
  ### Crear conexiones de Vault desde la página de Vault
</div>

Agregue una conexión de Vault desde la página de Vault siguiendo estos pasos:

1. Abra el editor de formularios seleccionando **Auth0 Dashboard &gt; Actions &gt; Forms.**
2. En la esquina superior izquierda, seleccione el menú del tenant y, luego, **Vault** para abrir la página de Vault.

   <Frame>![Dashboard > Forms > Vault](/docs/images/cdy7uua7fh8z/1sPNPKO6nuEDCuiYGRWSur/d85dc616170eaf2d7d5f321750863ccf/vault-connection.png)</Frame>
3. Seleccione **Add Vault Connection**.
4. Seleccione la acción deseada y **Continue**.
5. Agregue un **Connection name** de Vault y seleccione **Continue**.
6. Introduzca las credenciales.
   Si no tiene las credenciales específicas, puede seleccionar **Create an empty connection** y agregar las credenciales más tarde. Las conexiones vacías tienen un icono naranja que indica que faltan credenciales.
7. Seleccione **Add Vault Connection**.

<div id="create-vault-connections-from-the-flow-editor">
  ### Crear conexiones de Vault desde el editor de flujos
</div>

Agrega una conexión de Vault desde una acción de flujo, por ejemplo, de Twilio, siguiendo estos pasos:

1. Abre el editor de flujos seleccionando **Auth0 Dashboard &gt; Actions &gt; Forms &gt; Flows**
2. Selecciona un flujo y agrega una acción **Send SMS with Twilio**.
3. En la configuración **Vault connection** de la acción, selecciona el icono **+**.

   <Frame>![Dashboard > Forms > Vault ](/docs/images/cdy7uua7fh8z/1AsaznhRKxm5LZxPlcbfUw/10e5a8af700aab909f5b03fcd266da7e/twilio-vault-connection.png)</Frame>
4. Agrega un **Connection name** de Vault y selecciona **Continue**.
5. Ingresa las credenciales.
   Si no tienes las credenciales específicas, puedes seleccionar **Create an empty connection** y agregarlas más tarde. Las conexiones vacías tienen un icono naranja que indica que faltan credenciales.

   <Frame>![Dashboard > Forms > Vault > Configuración de la conexión de Twilio](/docs/images/cdy7uua7fh8z/yobXanutPRCO3FWek8cOk/bc29e39fb8593cff35e2738234bfe6fe/twilio-vault-settings.png)</Frame>
6. Selecciona **Add Vault Connection.**
