<div id="configure-your-http-vault-connection-options">
  ## Configura las opciones de tu conexión de HTTP Vault
</div>

La conexión de HTTP Vault se puede configurar con distintas opciones de autenticación y luego usar en tus Actions de flujo de solicitudes HTTP.

<Frame>![Dashboard > Actions > Formularios > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-options.png)</Frame>

<div id="configure-your-http-vault-connection-for-bearer-token">
  ## Configure su conexión de Vault HTTP para Bearer Token
</div>

Para configurar una conexión de Vault HTTP con autorización de **Bearer Token**, debe establecer los parámetros requeridos.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-bearer-token.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro    | Descripción                                                   |
| ------------ | ------------------------------------------------------------- |
| Bearer Token | Token estático que se usa para autorizar el servicio externo. |

<div id="configure-your-http-vault-connection-for-oauth-client-credentials">
  ## Configure su conexión de HTTP Vault para usar credenciales de cliente de OAuth
</div>

Para configurar una conexión de HTTP Vault mediante la autorización de **Credenciales de cliente**, debe establecer los parámetros obligatorios.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-oauth-client-credentials.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parameter            | Description                                                                                           |
| -------------------- | ----------------------------------------------------------------------------------------------------- |
| Endpoint del tenant  | Endpoint del tenant del Servidor de autorización que emite los tokens de acceso.                      |
| ID de cliente        | ID de cliente de tu aplicación M2M autorizada para la API personalizada.                              |
| Secreto del cliente  | Secreto del cliente de tu aplicación M2M autorizada para la API personalizada.                        |
| Audiencia (opcional) | Audiencia de la API personalizada. Normalmente se usa para las API personalizadas definidas en Auth0. |
| Recurso (opcional)   | Recurso de la API personalizada. Parámetro estándar con un significado similar al de la audiencia.    |
| Scope (opcional)     | Lista de scopes en formato de cadena, separada por espacios.                                          |

<div id="configure-your-http-vault-connection-for-api-key">
  ## Configura tu conexión de HTTP Vault para API Key
</div>

Para configurar una conexión de HTTP Vault con autorización de **API Key**, debes establecer los parámetros obligatorios.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-api-key.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro | Descripción                                                                                                              |
| --------- | ------------------------------------------------------------------------------------------------------------------------ |
| Nombre    | Nombre de la clave de API que se agregará como encabezado o parámetro de consulta.                                       |
| Valor     | Valor de la clave de API que se agregará como encabezado o parámetro de consulta.                                        |
| Agregar a | De forma predeterminada, es Header. Determina si la clave de API se agrega como encabezado o como parámetro de consulta. |

<div id="configure-your-http-vault-connection-for-basic-authentication">
  ## Configure su conexión de HTTP Vault para usar autenticación básica
</div>

Para configurar una conexión de HTTP Vault con **Basic Auth**, debe establecer los parámetros requeridos.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-basic-authentication.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro         | Descripción                                                                                                                           |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| Nombre de usuario | Nombre de usuario que se concatenará y codificará en base64, para luego añadirse como parámetro del encabezado de autorización Basic. |
| Contraseña        | Contraseña que se concatenará y codificará en base64, para luego añadirse como parámetro del encabezado de autorización Basic.        |

<Warning>
  La configuración de conexión de HTTP Vault con Basic Auth está pensada para sustituir la opción actualmente integrada en la Action HTTP Request de Flows.
</Warning>
