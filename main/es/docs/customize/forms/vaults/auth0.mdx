<div id="get-access-to-auth0-management-api">
  ## Obtén acceso a la Management API de Auth0
</div>

[Crea una aplicación de máquina a máquina (M2M)](/es/docs/get-started/auth0-overview/create-applications/machine-to-machine-apps) autorizada para la Management API de Auth0.

<Warning>
  Recuerda autorizar tu aplicación M2M para acceder a la Management API y selecciona los alcances correctos siguiendo el principio de privilegio mínimo:

  * Para obtener un usuario: `read:users`
  * Para crear un usuario: `create:users`
  * Para actualizar un usuario: `update:users`
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  No necesitas autorización de la Management API de Auth0 para enviar notificaciones a través de los proveedores configurados en tu inquilino de Auth0.
</Callout>

<div id="configure-your-vault-connection">
  ## Configurar la conexión de Vault
</div>

Para configurar una conexión de Vault de Auth0 para sus Auth0 Flow Actions, establezca los parámetros requeridos según la aplicación M2M creada anteriormente.

<Frame>![Dashboard > Actions > Forms > Vault > Auth0](/docs/images/customize/forms/vaults/auth0-vault-connection.png)</Frame>

<div id="input-settings">
  #### Configuración de entrada
</div>

| Parámetro             | Descripción                                                                    |
| --------------------- | ------------------------------------------------------------------------------ |
| Dominio del inquilino | Dominio del inquilino de su aplicación M2M.                                    |
| ID de cliente         | ID de cliente de su aplicación M2M autorizada para Auth0 Management API.       |
| Secreto del cliente   | Secreto del cliente de su aplicación M2M autorizada para Auth0 Management API. |
| Audiencia (opcional)  | Audiencia de la Management API de Auth0.                                       |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  De forma predeterminada, la audiencia se basará en el dominio del inquilino.
</Callout>
