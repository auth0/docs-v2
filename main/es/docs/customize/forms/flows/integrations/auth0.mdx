<div id="custom-api-request">
  ## Solicitud personalizada a la API
</div>

Envía una solicitud personalizada a los endpoints de la API de Auth0.

<Frame>![Dashboard > Customize > Flows > Integrations > Auth0](/docs/images/cdy7uua7fh8z/66uV9iuhwfnXUdE1quLOyx/2ae3ff26335acbcd172cbb40618a5a90/Screenshot_2025-01-24_at_18.50.26.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro         | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ruta              | La ruta de acceso absoluta del endpoint de la API.<br />( Ejemplo: `/api/v2/organizations` )<br />**Nota:** Algunos endpoints, por ejemplo: [ Obtener un usuario ](https://auth0.com/docs/api/management/v2/users/get-users-by-id), requieren parámetros codificados en URI cuando no se usan caracteres alfanuméricos, como: `-`, `.`, `_` o `~`, [ caracteres ](https://datatracker.ietf.org/doc/html/rfc3986#section-2.3)).<br />Para ello, puedes usar la función encodeURIComponent:<br />`{{ functions.encodeURIComponent(context.user.user_id) }}` |
| Método            | Los métodos HTTP disponibles son: <ul> <li>`GET`</li> <li>`POST`</li> <li>`PUT`</li> <li>`PATCH`</li> <li>`DELETE`</li> </ul>                                                                                                                                                                                                                                                                                                                                                                                                                             |
| Cuerpo            | El cuerpo de la solicitud.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| Encabezados       | Los encabezados de la solicitud.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| Parámetros de URL | Los parámetros de la URL. Estos parámetros se codifican automáticamente en la URL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |

<div id="output-object">
  ### Objeto de salida
</div>

Consulta la [Auth0 Management API](https://auth0.com/docs/api/management/v2/introduction) para ver las respuestas y los códigos de estado esperados.

<div id="send-email">
  ## Enviar correo electrónico
</div>

Envía un correo electrónico personalizado con tu [proveedor de correo electrónico configurado](/es/docs/customize/email).

<Warning>
  Debes habilitar [Email Provider](/es/docs/customize/email) antes de usar esta acción.
</Warning>

<Frame>![](/docs/images/forms/flows/integrations/auth0-notifications-send-email.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro               | Descripción                                                                                                                                                                                                                                                  |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Use custom sender email | De forma predeterminada, usaremos el correo electrónico del remitente que hayas configurado en los ajustes personalizados de tu proveedor de correo electrónico. Activa esta opción si quieres usar un correo electrónico o un nombre de remitente distinto. |
| Recipient email         | El correo electrónico del destinatario.                                                                                                                                                                                                                      |
| Custom variables        | Agrega variables personalizadas para insertarlas en tu plantilla de correo electrónico. Una vez configuradas, puedes hacer referencia a ellas mediante la sintaxis `{{custom_vars.your_variable_key}}`.                                                      |
| Subject                 | El asunto del correo electrónico. Admite variables personalizadas y sintaxis Liquid.                                                                                                                                                                         |
| Message                 | El mensaje del correo electrónico está en formato HTML, usa sintaxis Liquid y admite variables personalizadas.                                                                                                                                               |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Subject y Message pueden usar una combinación de sintaxis Liquid con [variables comunes](https://auth0.com/docs/customize/email/email-templates/supported-liquid-syntax) o variables personalizadas mediante la sintaxis `{{custom_vars.your_variable_key}}`.

  Usa variables personalizadas para pasar información entre el contexto de Forms / Flows y el contenido de Subject y Message.
</Callout>

<div id="output-object">
  ### Objeto de salida
</div>

| Property | Type    | Description                                                                                                                    |
| -------- | ------- | ------------------------------------------------------------------------------------------------------------------------------ |
| success  | Boolean | Indica si el correo electrónico se encoló correctamente para su envío, independientemente de que la entrega haya fallado o no. |

<div id="output-object-example">
  ### Ejemplo de objeto de salida
</div>

```json
{
  "success": true
}
```

<div id="send-sms">
  ## Enviar SMS
</div>

Envía un SMS personalizado con tu [proveedor telefónico configurado](https://auth0.com/docs/customize/phone-messages/configure-phone-messaging-providers).

<Warning>
  [Proveedor telefónico](https://auth0.com/docs/customize/phone-messages/configure-phone-messaging-providers) debe estar habilitado para el envío de mensajes de texto antes de usar esta acción.
</Warning>

<Frame>![](/docs/images/forms/flows/integrations/auth0-notifications-send-sms.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro        | Descripción                                                                                                                                                                                                                                        |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| From             | De forma predeterminada, usaremos el número de teléfono del remitente que hayas configurado en la configuración personalizada de tu proveedor telefónico. Introduce un valor personalizado si quieres usar otro número de teléfono como remitente. |
| To               | El número de teléfono del destinatario.                                                                                                                                                                                                            |
| Custom variables | Añade variables personalizadas para insertarlas en tu plantilla de teléfono. Una vez configuradas, puedes hacer referencia a ellas mediante la sintaxis `{{custom_vars.your_variable_key}}`.                                                       |
| Message          | El mensaje de teléfono usa la sintaxis Liquid y admite variables personalizadas.                                                                                                                                                                   |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Message puede usar una combinación de sintaxis Liquid con [variables comunes](https://auth0.com/docs/customize/email/email-templates/supported-liquid-syntax) o variables personalizadas mediante la sintaxis `{{custom_vars.your_variable_key}}`.

  Usa variables personalizadas para pasar información entre el contexto de Forms / Flows y el contenido de Message.
</Callout>

<div id="output-object">
  ### Objeto de salida
</div>

| Propiedad | Tipo    | Descripción                                                                                                                  |
| --------- | ------- | ---------------------------------------------------------------------------------------------------------------------------- |
| success   | Boolean | Indica si el mensaje telefónico se puso en cola correctamente para su envío, independientemente de si la entrega falló o no. |

<div id="output-object-example">
  ### Ejemplo de objeto de salida
</div>

```json
{
  "success": true
}
```
