  ## Configure la conexión de HTTP Vault

Aprenda a configurar una conexión de Vault para sus Actions del flujo de solicitudes HTTP en [conexión de HTTP Vault](/es/docs/customize/forms/vaults/http).

<div id="http-request">
  ## Solicitud HTTP
</div>

La acción de solicitud HTTP te permite llamar a una API o a un servicio externos y procesar la respuesta.

<Frame>![](/docs/images/cdy7uua7fh8z/6eIOv07XenyaazzKixigei/2e9efd632137829a8ddb03aacd3197df/http-request-action.png)</Frame>

<div id="input-settings">
  ### Configuración de entrada
</div>

| Parámetro            | Descripción                                                                                                                                                                     |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Método               | El método HTTP. Los valores disponibles incluyen: <ul> <li>`GET`</li> <li>`POST`</li> <li>`PUT`</li> <li>`PATCH`</li> <li>`DELETE`</li> </ul>                                   |
| Tipo de contenido    | El encabezado `Content-Type`. Los valores disponibles incluyen: <ul> <li>`application/json`</li> <li>`application/x-www-form-urlenconded`</li> <li>`application/xml`</li> </ul> |
| Cuerpo               | El cuerpo de la solicitud.                                                                                                                                                      |
| Encabezados          | Los encabezados de la solicitud.                                                                                                                                                |
| Parámetros de URL    | Los parámetros de la URL. Estos parámetros se codificarán automáticamente en la URL.                                                                                            |
| Habilitar Basic Auth | Habilita la autenticación con username y contraseña.                                                                                                                            |

<div id="output-object">
  ### Objeto de salida
</div>

| Propiedad | Tipo    | Descripción                                                        |
| --------- | ------- | ------------------------------------------------------------------ |
| `headers` | Object  | Los encabezados de la respuesta.                                   |
| `body`    | Object  | El cuerpo de la respuesta.                                         |
| `status`  | Number  | El código de estado de la respuesta.                               |
| `success` | Boolean | Devuelve `true` o `false` en función del `status` de la respuesta. |

<div id="output-object-example">
  ### Ejemplo del objeto de salida
</div>

```json lines
{
  "headers": {
    ...
  },
  "status": 200,
  "body": {
    ...
  },
  "success": true
}
```
