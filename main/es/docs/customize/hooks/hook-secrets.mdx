<Warning>
  La fecha de fin de vida (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para nuevos inquilinos creados a partir del **16 de octubre de 2023**. Los inquilinos existentes con Hooks activos conservarán el acceso al producto Hooks hasta su fin de vida.

  Te recomendamos encarecidamente que uses Actions para extender Auth0. Con Actions, tienes acceso a información de tipos más completa, documentación integrada y paquetes públicos de `npm`, y puedes conectar integraciones externas que mejoran tu experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulta [Comprende cómo funcionan las Actions de Auth0](/es/docs/customize/actions/actions-overview).

  Para ayudarte con la migración, ofrecemos guías que te ayudarán a [migrar de Rules a Actions](/es/docs/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/es/docs/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página dedicada, [Move to Actions](https://auth0.com/extensibility/movetoactions), que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarte en tu proceso de migración.

  Para obtener más información sobre la retirada de Rules y Hooks, lee nuestra entrada de blog: [Preparación para el fin de vida de Rules y Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Como tenemos previsto eliminar las funciones de Rules y Hooks en 2026, debes crear nuevas Rules o Hooks solo en tu entorno de desarrollo y únicamente para probar la migración a Actions.

  Para saber cómo migrar tus Rules a Actions, consulta [Migrar de Rules a Actions](/es/docs/customize/actions/migrate/migrate-from-rules-to-actions). Para saber cómo migrar tus Hooks a Actions, consulta [Migrar de Hooks a Actions](/es/docs/customize/actions/migrate/migrate-from-hooks-to-actions).
</Warning>

Hooks incorpora administración integrada de secretos para almacenarlos de forma segura y ponerlos cómodamente a disposición del código.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Para acceder a un secreto de Hook configurado desde un hook, usa `context.webtask.secrets.SECRET_NAME`.
</Callout>

<div id="manage-hook-secrets">
  ## Administrar secretos de Hook
</div>

Puede crear, actualizar, eliminar y ver secretos de Hook desde el Dashboard o mediante la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>. Para obtener más información, consulte:

* [Crear secretos de Hook](/es/docs/customize/hooks/hook-secrets/create-hook-secrets)
* [Actualizar secretos de Hook](/es/docs/customize/hooks/hook-secrets/update-hook-secrets)
* [Eliminar secretos de Hook](/es/docs/customize/hooks/hook-secrets/delete-hook-secrets)
* [Ver secretos de Hook](/es/docs/customize/hooks/hook-secrets/view-hook-secrets)

Los secretos de Hook también se pueden importar o exportar con la [herramienta Deploy Command-Line Interface (CLI).](/es/docs/deploy-monitor/deploy-cli-tool)
