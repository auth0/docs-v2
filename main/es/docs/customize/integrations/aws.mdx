Hay muchas formas de integrar Auth0 con productos y servicios de Amazon en función de sus necesidades específicas.

<div id="product-integrations">
  ## Integraciones de productos
</div>

A continuación, puede obtener más información sobre cómo usar los siguientes productos junto con sus servicios de Auth0 y AWS:

* [CloudFront](/es/docs/customize/custom-domains/self-managed-certificates/configure-aws-cloudfront-for-use-as-reverse-proxy): Úselo como proxy inverso con su <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre especializado o personalizado." cta="Ver glosario" href="/es/docs/glossary?term=custom+domain">dominio personalizado</Tooltip>.
* [Simple Email Service (SES)](/es/docs/customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider): Gestione las comunicaciones por correo electrónico con sus usuarios.
* [EventBridge](https://marketplace.auth0.com/integrations/amazon-log-streaming): Transmita logs a EventBridge.
* [Cognito](/es/docs/customize/integrations/aws/amazon-cognito): Úselo como backend para su aplicación.
* AWS S3 y DynamoDB:

  * Auth0 proporciona endpoints de gateway de VPC para acceder a AWS S3 y DynamoDB, que pueden ser necesarios para [Actions](/es/docs/customize/actions). Esto requiere políticas de IAM basadas en `aws:SourceVpc`, como se describe en [Gateway endpoints for Amazon S3](https://docs.aws.amazon.com/vpc/latest/privatelink/vpc-endpoints-s3.html) y [Gateway endpoints for Amazon DynamoDB](https://docs.aws.amazon.com/vpc/latest/privatelink/vpc-endpoints-ddb.html). Póngase en contacto con [Auth0 Support](https://www.support.auth0.com) para obtener información sobre los identificadores de VPC.

<div id="identity-and-access-management-iam-integrations">
  ## Integraciones de gestión de identidades y accesos (IAM)
</div>

Tenemos varios artículos instructivos para configurar AWS para que funcione con Auth0:

* [Proteja los endpoints de AWS API Gateway con autorizadores personalizados](/es/docs/customize/integrations/aws/aws-api-gateway-custom-authorizers): Acepte <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o un JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+tokens">tokens de acceso</Tooltip> emitidos por Auth0.
* [Use etiquetas de sesión de AWS con las API y los recursos de AWS](/es/docs/customize/integrations/aws/session-tags): Etiquete recursos y asigne a los usuarios pares clave/valor para RBAC.
* [Configure AWS para el inicio de sesión único](/es/docs/customize/integrations/aws/configure-amazon-web-services-for-sso): Permita que sus usuarios inicien sesión en AWS mediante cualquier <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=identity+provider">proveedor de identidad</Tooltip> compatible.
* [Agregue el inicio de sesión con Amazon a su aplicación](/es/docs/customize/integrations/aws/configure-amazon-web-services-for-sso): Permita que los usuarios inicien sesión en su aplicación con su perfil de Amazon.
