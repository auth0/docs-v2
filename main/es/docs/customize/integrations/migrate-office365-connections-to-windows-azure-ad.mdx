description: Detalles sobre la migración de conexiones de Office365 a Windows Azure AD.
title: Migrar conexiones de Office365 a Windows Azure AD
---

Office365 ha quedado obsoleto. Debe migrar sus conexiones de Office365 a conexiones de Windows Azure AD.

Desde el principio, hemos admitido la autenticación de usuarios con Office365. Office365 siempre ha utilizado Windows Azure AD internamente, pero no existía una buena interfaz de usuario para crear una &quot;aplicación&quot; en Windows Azure AD. Por eso tenía que crearla en el Seller Dashboard. En adelante, Microsoft quiere que use Windows Azure AD, y ahora puede crear fácilmente un directorio asociado a su cuenta de Office365 y a la aplicación.

<div id="how-to-migrate-to-azure">
  ## Cómo migrar a Azure
</div>

1. Cree una suscripción a Windows Azure AD (gratuita)
2. Cree un directorio (que se asociará a su cuenta de Office365) y una aplicación. Para obtener más información, consulte [Conecte su aplicación a Microsoft Azure Active Directory](/es/docs/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

Si usaba `user_id` en su aplicación, tenga en cuenta que cambiará de `office365|....some-guid....` a `waad|...email....`.
