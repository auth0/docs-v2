Las integraciones de <Tooltip tip="Single Sign-On (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/es/docs/glossary?term=SSO">SSO</Tooltip> (inicio de sesión único) usan una identidad de Auth0 para iniciar sesión, mediante <Tooltip tip="Single Sign-On (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/es/docs/glossary?term=SAML">SAML</Tooltip>, en un servicio externo. Esto permite a los clientes con aplicaciones internas usar SSO para acceder a proveedores externos con la identidad de Auth0.

Las [integraciones de SSO disponibles en Marketplace](https://marketplace.auth0.com/features/sso-integrations) definen tanto los valores estándar de configuración de SAML necesarios para conectar Auth0 con proveedores externos como los campos necesarios para valores adicionales específicos del cliente. En estas integraciones, Auth0 actúa como <Tooltip tip="Identity Provider (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=identity+provider">proveedor de identidad</Tooltip> en un escenario iniciado por el proveedor de servicios (SP). Para obtener información sobre cómo configurar Auth0 para estas integraciones, consulte [Configure Auth0 as a SAML Identity Provider](/es/docs/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider).

La parte que proporcionarán nuestros partners es la configuración predeterminada de SAML para que esta transacción funcione. Esta consistirá en valores o campos solicitados por el cliente para algunos o todos los atributos enumerados en [Customize SAML Assertions: SAML Assertion Attributes](/es/docs/authenticate/protocols/saml/saml-configuration/customize-saml-assertions).

Para crear y probar una integración, [cree un inquilino](https://auth0.com/signup) en Auth0 y siga la documentación anterior para crear una integración de SSO funcional con su servicio.

Además, para ayudarle a crear su integración, puede usar nuestra [integración de SSO](https://github.com/auth0-marketplace/TEMPLATE-sso).

<div id="submit-your-sso-integration">
  ## Envía tu integración de SSO
</div>

Si estás listo para convertirte en partner y enviar tu integración, crea una cuenta de partner en el [portal de partners](https://partners.marketplace.auth0.com/) y haz lo siguiente:

1. Completa el perfil de partner, que incluye una breve descripción de tu empresa, contenido legal y de marketing, el logotipo de la empresa, los términos de servicio, la política de privacidad, la información de contacto y el correo electrónico de soporte.
2. Envía tu integración, que incluye el código de integración, la guía de instalación y detalles como el nombre y la descripción. De forma opcional, puedes añadir 3 propuestas de valor a tu integración.

**Necesitaremos la siguiente configuración y documentación:**

* El JSON utilizado para configurar la aplicación del complemento SAML funcional
* Una lista de campos que deben mostrarse al cliente que configura la integración (como **Callback URL**, **<Tooltip tip="Audiencia: identificador único de la audiencia de un token emitido. Se denomina aud en un token; su valor contiene el ID de una aplicación (ID de cliente) para un ID Token o de una API (Identificador de API) para un Token de acceso." cta="Ver glosario" href="/es/docs/glossary?term=Audience">Audiencia</Tooltip>**, y **Signing Certificate**)
* Una guía de instalación escrita en Markdown.
