Estas son algunas preguntas frecuentes sobre solución de problemas. Si los elementos siguientes no resuelven su problema, considere las siguientes alternativas:

* Si está configurando el plugin por primera vez o tiene problemas para que los usuarios inicien sesión, lea [Configure Login by Auth0](/es/docs/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0).
* Si encontró un error en el código del plugin, [envíe un issue](https://github.com/auth0/wordpress/issues) o [cree un pull request](https://github.com/auth0/wordpress/pulls) en GitHub.
* Si tiene preguntas sobre cómo usar Auth0 o el plugin, [publique su consulta en nuestro sitio de la comunidad](https://community.auth0.com/) o [cree una solicitud en el foro de soporte de WordPress del plugin](https://wordpress.org/support/plugin/auth0/).
* Para ver documentación adicional y encontrar más respuestas, [visite nuestro sitio de soporte](https://support.auth0.com/center/s/). Los clientes con un plan de pago de Auth0 pueden enviar un ticket de soporte para recibir una respuesta rápida.

<div id="im-seeing-the-error-message-invalid-state-that-prevents-me-from-logging-in">
  ## Veo el mensaje de error &quot;Invalid state&quot; que me impide iniciar sesión
</div>

La validación de estado es una función de seguridad añadida en la [versión 3.6.0](https://github.com/auth0/wordpress/releases/tag/3.6.0). Este mensaje de error suele deberse a una URL de devolución de llamada almacenada en caché (consulta la configuración de la aplicación para ver qué URL de devolución de llamada no deben almacenarse en caché). Para obtener más información, lee [Solucionar problemas de errores &quot;Invalid state&quot; del plugin de WordPress](/es/docs/customize/integrations/cms/wordpress-plugin/troubleshoot-wordpress-plugin-invalid-state-errors).

<div id="im-having-an-issue-logging-in-or-changing-emailpassword-using-a-custom-database">
  ## Tengo problemas para iniciar sesión o cambiar el correo electrónico/la contraseña con una base de datos personalizada
</div>

Para conocer las posibles soluciones, lee [Migración de usuario en el plugin de WordPress Login by Auth0](/es/docs/customize/integrations/cms/wordpress-plugin/user-migration-in-login-by-auth0).

<div id="im-seeing-the-error-message-invalid-id-token-or-expired-id-token-that-prevents-me-from-logging-in">
  ## Veo el mensaje de error &quot;Invalid ID token&quot; o &quot;Expired ID token&quot;, que me impide iniciar sesión
</div>

Esto suele deberse a que el servidor tiene una hora incorrecta. Si el mensaje de error incluye &quot;used too early,&quot;, la hora de su servidor está configurada en el futuro. Si indica que el token ha expirado, la hora del servidor está configurada demasiado en el pasado. Es habitual que haya una diferencia horaria entre dos servidores. Ejecute `echo date(DateTime::ISO8601)` en PHP en su servidor y compare ese valor, incluidos los segundos, con la hora UTC actual. Si la hora de su servidor difiere de la hora UTC en más de 60 segundos (el margen predeterminado), deberá establecer un margen mayor para compensar el desfase del reloj de su servidor. Puede pegar el siguiente código en el archivo `functions.php` de su tema o en cualquier otro lugar donde se ejecute después de que se cargue el plugin y antes de que se ejecute el hook de Login:

```js wrap lines
add_filter( 'auth0_jwt_leeway', function( $default_leeway ) { return 120; } );
```

Esto proporcionaría un margen de 120 segundos. Puede que necesite ajustarlo en función del desfase horario del servidor.

<div id="i-see-the-error-message-this-account-does-not-have-an-email-associated-that-prevents-me-from-logging-in">
  ## Veo el mensaje de error &quot;Esta cuenta no tiene un correo electrónico asociado...&quot; que me impide iniciar sesión
</div>

Si recibe este error, asegúrese de solicitar un correo electrónico a cada proveedor en el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, en **Connections &gt; Social** (expanda cada proveedor). Tenga en cuenta que no todos los proveedores devuelven direcciones de correo electrónico de los usuarios (por ejemplo, X). Si esto ocurre, siempre puede añadir una dirección de correo electrónico a cualquier usuario que haya iniciado sesión a través del Auth0 Dashboard (o la API). Consulte **Users &gt; Edit**.

Para las Conexiones que no proporcionan un indicador `email_verified` (algunas conexiones empresariales no lo incluyen) o para omitir esta validación en conexiones sociales específicas, agregue la estrategia de esa Conexión en el campo &quot;Skip Strategies&quot;. Este campo se encuentra debajo del interruptor **Require Verified Email**, al que puede acceder desde **wp-admin** &gt; **Auth0** &gt; **Settings** &gt; **Advanced**.

**Este campo solo debe usarse si es necesario, ya que omite las medidas de seguridad recomendadas por Auth0.**

<div id="i-see-the-error-message-there-is-a-user-with-the-same-email-that-prevents-me-from-logging-in">
  ## Veo el mensaje de error &quot;Hay un usuario con el mismo correo electrónico&quot; que me impide iniciar sesión
</div>

Esto significa que hay un usuario en WordPress con el mismo correo electrónico que se está usando para iniciar sesión, pero asociado a un usuario de Auth0 diferente. Si estás probando el plugin o quieres asociar el usuario existente con el nuevo usuario de Auth0:

1. Inicia sesión como administrador
2. Ve a **wp-admin &gt; Users** y busca el correo electrónico que se está usando
3. Abre el perfil del usuario y desplázate hasta el final
4. Haz clic en **Delete Auth0 Data** y confirma

Si tienes 2 cuentas de usuario en Auth0 con la misma dirección de correo electrónico, este mensaje de error seguirá apareciendo. Te recomendamos que [vincules cuentas de usuario](/es/docs/manage-users/user-accounts/user-account-linking/link-user-accounts).

<div id="i-see-the-error-message-failed-cross-origin-authentication-or-no-verifier-returned-from-client-in-my-browsers-console-logs-when-trying-to-log-in">
  ## Veo el mensaje de error &quot;Failed cross origin authentication&quot; o &quot;No verifier returned from client&quot; en los registros de la consola de mi navegador al intentar iniciar sesión
</div>

Ve a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), selecciona tu aplicación de WordPress para ver su configuración y comprueba que **Allowed Callback URLs** y **Allowed Origins (CORS)** coincidan con tu sitio de WordPress.

Si usas un navegador basado en Chromium, asegúrate de no tener desactivadas las cookies de terceros. Para obtener más información, consulta [Autenticación entre orígenes](/es/docs/authenticate/login/cross-origin-authentication).

<div id="i-need-to-rerun-the-setup-wizard-but-i-dont-see-that-menu-option-anymore">
  ## Necesito volver a ejecutar el asistente de configuración, pero ya no veo esa opción en el menú.
</div>

Esto significa que el plugin ya está configurado con un dominio, <Tooltip tip="ID de cliente: valor de identificación proporcionado a su recurso registrado por Auth0." cta="Ver glosario" href="/es/docs/glossary?term=Client+ID">ID de cliente</Tooltip> y <Tooltip tip="ID de cliente: valor de identificación proporcionado a su recurso registrado por Auth0." cta="Ver glosario" href="/es/docs/glossary?term=Client+Secret">Secreto del cliente</Tooltip>. Ejecutar el asistente de configuración por segunda vez puede producir resultados impredecibles. Si está configurando WordPress por primera vez y quiere empezar de nuevo antes de que se produzca cualquier inicio de sesión:

1. Vaya a **wp-admin** &gt; **Auth0** &gt; **Settings** &gt; **Basic**.
2. Elimine el dominio y el ID de cliente. Desplácese hacia abajo y haga clic en **Save Changes**.
3. Vaya a [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
4. Busque la aplicación que creó WordPress (su nombre debería ser el nombre del sitio de WordPress).
5. Seleccione la aplicación para ver su configuración. Desplácese hacia abajo y seleccione **Delete Application**.
6. Vaya a [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database).
7. Busque la conexión que creó WordPress (su nombre debería ser el nombre del sitio de WordPress con el prefijo &quot;DB-&quot;).
8. Seleccione la conexión para ver su configuración. Desplácese hacia abajo y seleccione **I Want To Delete This Connection.** Tenga en cuenta que esto eliminará la conexión y todos los usuarios que se hayan creado en ella.
9. Vuelva a WordPress. Ahora verá la opción del asistente de configuración en Auth0 en el menú de administración.

<div id="how-do-i-setup-passwordless-login">
  ## ¿Cómo configuro el inicio de sesión sin contraseña?
</div>

<Tooltip tip="Sin contraseña: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Passwordless">Sin contraseña</Tooltip> es posible en cualquier sitio web habilitado con Auth0 mediante correo electrónico o SMS. Para que esto funcione en tu sitio de WordPress:

1. Activa **Passwordless Login** en la vista **Features** de la configuración del plugin y guarda los cambios.
2. Ve a [Auth0 Dashboard &gt; Authentication &gt; Passwordless](https://manage.auth0.com/#/connections/passwordless).

* Para usar correo electrónico, activa la conexión **Email** y modifica la configuración si lo deseas. Esto habilitará el inicio de sesión con código por correo electrónico (se envía a los usuarios un código por correo electrónico, que luego escriben en el formulario de inicio de sesión de tu sitio).
* Para usar un &quot;magic link&quot; (el enlace enviado por correo electrónico iniciará sesión automáticamente a los usuarios), agrega `{passwordlessMethod: 'code'}` al campo &quot;Extra Settings&quot; en la pestaña **Advanced** de la configuración del plugin.
* Para usar el inicio de sesión por SMS, activa la conexión **SMS** y sigue los pasos para configurar una cuenta de desarrollador de Twilio (según el uso, esto puede requerir una cuenta de pago de Twilio).

El formulario de inicio de sesión de Auth0 seleccionará un método sin contraseña según la conexión que esté activada arriba. Si tienes ambas conexiones activas, usará correo electrónico de forma predeterminada. En este caso, desactiva la conexión de correo electrónico para mostrar SMS o agrega `sms` al campo **Connections** en la pestaña **Advanced** de la configuración del plugin.

<div id="i-have-two-accounts-for-the-same-user-in-wordpress">
  ## Tengo dos cuentas para el mismo usuario en WordPress
</div>

En algunas situaciones, es posible que un usuario termine teniendo dos cuentas. WordPress te permite fusionar usuarios eliminando una de las cuentas y asignando el contenido de esa cuenta a otro usuario. Ve a wp-admin &gt; Users, selecciona la cuenta que quieres eliminar y, en el cuadro de diálogo de confirmación, selecciona otro usuario para transferirle el contenido.

<div id="my-configuration-is-wrong-and-i-cant-authenticate-using-auth0-is-there-another-way-to-access-the-plugin">
  ## Mi configuración es incorrecta y no puedo autenticarme con Auth0. ¿Hay alguna otra forma de acceder al plugin?
</div>

Se puede acceder al plugin con credenciales válidas de WordPress mediante el inicio de sesión habitual de WordPress, agregando `?wle` a la URL de inicio de sesión. Por ejemplo: `http://yourdomain.com/wp-login.php?wle`.

<div id="i-am-having-problems-when-a-user-logs-in-where-can-i-find-a-log-of-what-is-happening">
  ## Tengo problemas cuando un usuario inicia sesión. ¿Dónde puedo encontrar un registro de lo que está sucediendo?
</div>

El plugin proporciona un registro de errores donde puedes consultar qué ha ocurrido. Accede a él a través de la subopción **Error Log** del menú del plugin **Auth0**. Puedes encontrar información adicional en [Auth0 Dashboard &gt; Monitoring &gt; Logs](https://manage.auth0.com/#/logs).

<div id="how-can-i-show-the-widget-or-shortcode-in-signup-mode-by-default">
  ## ¿Cómo puedo mostrar el widget o shortcode en modo de registro de forma predeterminada?
</div>

Puede usar la opción `Extra configuration` del widget (o el atributo `extra_conf` en el shortcode) y añadir este JSON `{"mode":"signup" }` para forzar que el plugin se muestre en este modo.

<div id="when-using-a-plugin-to-force-the-login-the-user-is-not-logged-in">
  ## Al usar un plugin para forzar el Login, el usuario no inicia sesión
</div>

Esto suele deberse a una página almacenada en caché después del Login. Consulta con tu proveedor de hosting qué estrategias pueden ayudar a mitigar este problema o prueba a añadir un parámetro a la URL para invalidar la caché. Para obtener más información, [consulta este gist en GitHub](https://gist.github.com/joshcanhelp/e3eb693749f0fe66aad097c3bbb3b415).

<div id="the-user-is-not-logged-in-when-using-the-force-login-plugin">
  ### El usuario no ha iniciado sesión al usar el plugin &quot;Force Login&quot;
</div>

Esto se debe a que la URL de devolución de llamada no se ha añadido a la AllowList. Prueba a añadir este código al filtro `my_forcelogin_whitelist`:

```javascript lines
function wp_auth0_forcelogin_whitelist( $whitelist ) {
  if ( ! empty( $_GET['auth0'] ) ) {
    $whitelist[] = site_url($_SERVER['REQUEST_URI']);
  }
  return $whitelist;
}
add_filter('v_forcelogin_whitelist', 'wp_auth0_forcelogin_whitelist', 10, 1);
```

<div id="how-can-i-redirect-the-users-to-a-specific-url-after-login">
  ## ¿Cómo puedo redirigir a los usuarios a una URL específica después de iniciar sesión?
</div>

Todas las redirecciones se validan con `wp_safe_redirect()` antes de realizarse. Si intentas redirigir a un dominio que no es tu dominio principal, agrega ese dominio a la validación mediante el filtro `allowed_redirect_hosts` del core de WordPress. Para obtener más información, consulta [apply&#95;filters( &#39;allowed&#95;redirect&#95;hosts&#39;, string[] $hosts, string $host ) en wordpress.org](https://developer.wordpress.org/reference/hooks/allowed_redirect_hosts/).

<div id="on-the-login-page">
  ### En la página de Login
</div>

Este complemento aprovecha las funcionalidades de WordPress para funcionar sin problemas con la configuración predeterminada. Para añadir una redirección, puedes agregar el parámetro de consulta `redirect_to` a la URL al dirigir al usuario a la página de Login. El complemento redirigirá al usuario a esta URL después de que inicie sesión correctamente.

También puedes usar la opción **URL de redirección de Login** en la página de configuración del complemento de Auth0. Esta URL se usará para redirigir al usuario cuando no se proporcione el parámetro `redirect_to`.

<div id="using-the-widget">
  ### Uso del widget
</div>

El widget redirigirá automáticamente a la misma página en la que estaba el usuario antes de la autenticación. Puede cambiar este comportamiento con la opción `Redirect after login:`.

<div id="using-the-shortcode">
  ### Uso del shortcode
</div>

El shortcode redirigirá automáticamente a la misma página en la que estaba el usuario antes de autenticarse. Puede sobrescribir este comportamiento mediante el atributo `redirect_to`.

<div id="how-can-i-migrate-my-wordpress-users-to-auth0">
  ## ¿Cómo puedo migrar mis usuarios de WordPress a Auth0?
</div>

La versión actual del plugin no ofrece una forma de migrar usuarios a Auth0 automáticamente, pero tiene algunas opciones:

* El plugin expone dos endpoints para marcar su conexión de base de datos personalizada en el modo de **importación a Auth0**. Para obtener más información sobre los endpoints, lea [Configure Automatic Migration from Your Database](/es/docs/manage-users/user-migration/configure-automatic-migration-from-your-database).
* Exporte sus datos de usuario a un archivo JSON y cárguelo para importarlos por lotes a Auth0. Inicialmente, sus usuarios tendrán que restablecer sus contraseñas al iniciar sesión con Auth0, ya que no hay forma de que Auth0 descifre las contraseñas de WordPress durante la migración. Para saber cómo generar el archivo JSON, lea [Bulk User Imports](/es/docs/manage-users/user-migration/bulk-user-imports). Después, deberá cargar el archivo mediante el [endpoint Import Users](https://auth0.com/docs/api/management/v2/jobs/post-users-imports) de la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de Auth0.
* [Use este script](https://gist.github.com/glena/b31716e3c8fe48927be2) y el endpoint XML-RPC de WordPress para configurar el flujo de migración mediante una conexión de base de datos personalizada. Para obtener más información sobre cómo configurar el flujo de migración, lea [Configure Automatic Migration from Your Database](/es/docs/manage-users/user-migration/configure-automatic-migration-from-your-database). Para obtener más información sobre el endpoint XML-RPC de WordPress, lea [XML-RPC Support on wordpress.org](https://codex.wordpress.org/XML-RPC_Support).

<div id="the-form_title-setting-is-ignored-when-i-set-up-the-dict-setting">
  ## La configuración form_title se ignora cuando configuras la opción dict
</div>

Internamente, el complemento usa la configuración dict para cambiar el título del widget de Auth0. Cuando configuras el campo dict, este sobrescribe el valor de form&#95;title.

Para cambiar form&#95;title en este caso, debes añadir el siguiente atributo al JSON de dict:

```json lines
{
    "signin": {
        "title": "The desired form title"
    }
}
```

<div id="how-can-i-modify-the-embedded-auth0-login-form">
  ## ¿Cómo puedo modificar el formulario de Login de Auth0 integrado?
</div>

Hay muchas opciones en la pestaña **Appearance** de la página de configuración del plugin que pueden cambiar la apariencia y el funcionamiento del formulario de Login integrado en tu sitio (página `wp-login/php`, shortcodes o widgets). Para obtener más información sobre estas opciones, consulta [Configure Login by Auth0](/es/docs/customize/integrations/cms/wordpress-plugin/configure-login-by-auth0). Esto no afectará al formulario de Login de la página de <Tooltip tip="Universal Login: tu aplicación redirige a Universal Login, alojado en el Servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Universal Login</Tooltip> alojada por Auth0.

También hay un campo llamado &quot;Extra Settings&quot; en la pestaña **Advanced** que acepta un objeto JSON válido con toda la configuración que quieras aplicar. Esto anulará cualquier cambio realizado en la pestaña **Appearance**. Para ver todas las opciones de configuración posibles, consulta [Lock Configuration Options](/es/docs/libraries/lock/lock-configuration).

Las hojas de estilo externas y los archivos JS deben cargarse en tu tema mediante el hook `wp_enqueue_scripts` para shortcodes/widgets y el hook `login_enqueue_scripts` para `wp-login.php`. Para obtener más información, consulta [`do_action( 'wp_enqueue_scripts' )` en wordpress.org](https://developer.wordpress.org/reference/hooks/wp_enqueue_scripts/) y [`do_action( 'login_enqueue_scripts' )` en wordpress.org](https://developer.wordpress.org/reference/hooks/login_enqueue_scripts/).

<div id="the-session-expires-too-soon">
  ## La sesión expira demasiado pronto
</div>

El plugin de Auth0 no gestiona las sesiones; utiliza la configuración de WordPress. De forma predeterminada, las sesiones de los usuarios se mantienen activas durante dos días. Puede habilitar la opción `Remember users session` en la pestaña **Advanced** de la configuración del plugin para permitir que las sesiones sigan activas hasta 14 horas.

<div id="how-do-i-implement-a-refresh-token">
  ## ¿Cómo implemento un Token de actualización?
</div>

Incorporamos parámetros adicionales en los métodos de inicio de sesión que utiliza el plugin para admitir <Tooltip tip="Token de actualización: token que se usa para obtener un nuevo Token de acceso sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/es/docs/glossary?term=Refresh+Tokens">Tokens de actualización</Tooltip>. Usa el filtro `auth0_auth_scope` junto con la acción `auth0_user_login` para hacerlo.

<div id="profile-data-saved-in-wordpress-is-not-being-synced-to-the-auth0-user-account">
  ## Los datos de perfil guardados en WordPress no se sincronizan con la cuenta de usuario de Auth0.
</div>

Esta es una limitación actual del plugin, pero es algo que estamos evaluando para una versión futura. La única excepción es la contraseña del usuario. Si la contraseña se cambia en WordPress y cumple la política de seguridad establecida para la conexión de base de datos en uso, esa contraseña también se actualizará para el usuario de Auth0. En una versión futura, agregaremos un mensaje de error para detener el proceso si la contraseña no es lo suficientemente segura.

<div id="how-do-i-migrate-from-social-login-with-auth0-to-login-by-auth0">
  ## ¿Cómo migro de &quot;Social Login with Auth0&quot; a &quot;Login by Auth0&quot;?
</div>

Históricamente, Auth0 ha mantenido dos plugins de WordPress:

* [Login by Auth0](https://wordpress.org/plugins/auth0/)
* [Social Login with Auth0](https://wordpress.org/plugins/social-login-with-auth0/)

Estos dos plugins son, en la práctica, el mismo, pero **Social Login with Auth** no recibirá más actualizaciones después de la versión 3.7.0 (publicada el 13 de agosto de 2018). Migrar de **Social Login with Auth** a **Login by Auth0** es sencillo y no provocará ninguna pérdida de datos de Auth0 ni de WordPress.

Al pasar de **Social Login with Auth** a **Login by Auth0**, cambiará el número de versión que ves, así que asegúrate de probar este cambio primero en un servidor de pruebas o de desarrollo (igual que harías si estuvieras actualizando el plugin en wp-admin). Además, es posible que los inicios de sesión no funcionen durante el proceso de migración, así que asegúrate de usar un plugin de modo de mantenimiento o de completar la migración en horas de baja actividad.

La forma más sencilla de migrar es mediante (S)FTP:

1. Inicia sesión en el sitio de WordPress como administrador.
2. [Descarga Login by Auth0](https://downloads.wordpress.org/plugin/auth0.zip) y descomprímelo en tu equipo.
3. Desactiva el plugin **Social Login with Auth0** desde la pantalla WordPress admin &gt; Plugins.
4. Inicia sesión en el servidor al que quieres migrar y ve a `wp-content/plugins`.
5. Mueve la carpeta `social-login-with-auth0` fuera de la carpeta de plugins para hacer una copia de seguridad de su contenido.
6. Sube la nueva carpeta del plugin `auth0` al directorio de plugins.
7. Activa el nuevo plugin &quot;Login by Auth0&quot; desde la pantalla de WordPress **Admin** &gt; **Plugins**.

Si no puedes acceder al sitio mediante FTP, también puedes realizar el proceso directamente desde el administrador de WordPress:

1. Inicia sesión en el sitio de WordPress como administrador.
2. Ve a **Auth0** &gt; **Configuración de importación y exportación**.
3. Haz clic en **Exportar configuración** y luego en **Exportar**.
4. Desactiva el plugin **Social Login with Auth0** desde la pantalla de WordPress **Admin** &gt; **Plugins**.
5. Elimina el plugin **Social Login with Auth0** y confirma.
6. Ve a **Plugins** &gt; **Añadir nuevo** y busca &quot;Auth0&quot;.
7. En el plugin **Login by Auth0** (asegúrate de comprobar el nombre), haz clic en **Instalar ahora**.
8. Cuando termine, haz clic en **Activar**.
9. Comprueba en **Auth0** &gt; **Configuración** que se conserven los ajustes anteriores. Si no es así:

   1. Ve a **Auth0** &gt; **Configuración de importación y exportación**.
   2. Pega el JSON de configuración exportado anteriormente y haz clic en **Importar**.
10. Elimina por completo el archivo JSON exportado con la configuración (contiene información confidencial).

Ahora todo debería funcionar como se espera con el nuevo plugin y las actualizaciones se reanudarán con normalidad.
