<div id="language-selection">
  ### Selección de idioma
</div>

Auth0 selecciona el idioma en que se muestran las páginas en función de lo siguiente:

* Los idiomas compatibles con Auth0, que se enumeran arriba.
* La lista de idiomas configurados en la sección [**Tenant Settings**](https://manage.auth0.com/#/tenant) del Auth0 Dashboard, donde puede seleccionar los idiomas que admite su inquilino y elegir uno predeterminado. De forma predeterminada, en la lista solo está seleccionado el inglés, pero puede seleccionar los que necesite.
* El valor del parámetro `ui_locales` enviado al [endpoint de solicitud de autorización](https://openid.net/specs/openid-connect-core-1_0.html#AuthRequest), que puede usarse para restringir la lista de idiomas de una aplicación o sesión. Puede proporcionar una lista de configuraciones regionales separadas por espacios. La primera configuración regional de la lista debe coincidir con una configuración regional habilitada en su inquilino para que se refleje en la UI.
* El encabezado HTTP `Accept-Language` enviado por el navegador. Auth0 muestra las páginas en este idioma si está permitido por la configuración anterior. De lo contrario, las páginas se muestran en el idioma predeterminado.

Use la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de Auth0 para hacer una llamada `PATCH` al [endpoint Tenants](https://auth0.com/docs/api/management/v2/tenants/patch-settings) con el siguiente cuerpo, reemplazando `he` por el código de idioma que desea agregar:

```json lines
{
  "enabled_locales": [
    "en","he"
  ]
}
```

Después de añadirlo, especifique el idioma en el parámetro de consulta`ui_locales` de la solicitud de Login.

También puede especificar los idiomas habilitados para el inquilino a través de la Management API mediante el [endpoint Update Tenants](https://auth0.com/docs/api/management/v2#!/Tenants/patch_settings). El idioma predeterminado es el primero de la lista.

<div id="right-to-left-languages">
  ### Idiomas de derecha a izquierda
</div>

<Warning>
  La compatibilidad con idiomas de derecha a izquierda está disponible en Acceso anticipado. Para solicitar esta función, [contacte con el equipo de soporte de Auth0](https://support.auth0.com/).  Al usar esta función, acepta los términos aplicables de la prueba gratuita incluidos en el [Master Subscription Agreement](https://www.okta.com/legal/) de Okta. Para obtener más información sobre el ciclo de lanzamiento de productos de Auth0, consulte [Etapas de lanzamiento del producto](/es/docs/troubleshoot/product-lifecycle/product-release-stages).
</Warning>

La compatibilidad con idiomas de derecha a izquierda (RtL) permite que su inquilino muestre texto en árabe (estándar, egipcio y saudí), persa, hebreo y urdu. Los inquilinos que usan texto RtL deben cumplir con [WCAG 2.2 AA](/es/docs/authenticate/login/auth0-universal-login#accessibility) y, si su inquilino usa plantillas de página, debe actualizar la etiqueta HTML para incluir `dir`.

```liquid lines
{% assign resolved_dir = dir | default: "auto" %}
<html lang="{{locale}}" dir="{{resolved_dir}}">
```

<div id="limitations">
  ### Limitaciones
</div>

* El parámetro `ui_locales` solo se puede usar en flujos de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/es/docs/glossary?term=OAuth">OAuth</Tooltip>, ya que no está disponible con Lenguaje de marcado para aserciones de seguridad <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/es/docs/glossary?term=SAML">(SAML)</Tooltip> ni con WS-Federation.
* El parámetro `ui_locales` no se reenvía a los proveedores de identidad (IdP) externos <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=IdPs">IdPs</Tooltip>. Para obtener más información sobre cómo pasar parámetros a los IdP, consulta [Pasar parámetros a proveedores de identidad](/es/docs/authenticate/identity-providers/pass-parameters-to-idps).
* No es posible localizar los alcances en la página Consent.

<div id="known-issues">
  ### Problemas conocidos
</div>

* La página de Universal Login muestra el atributo HTML `lang` para el código de idioma `fr-FR` como `fr`.
* La página de Universal Login muestra el atributo HTML `lang` para el código de idioma `pt-PT` como `pt`.

<div id="classic-login-localization">
  ## Localización de Classic Login
</div>

En la experiencia de Classic Login, la localización usa los widgets de JavaScript de Auth0 para [Login](/es/docs/customize/internationalization-and-localization/lock-internationalization), la [página de restablecimiento de contraseña](/es/docs/customize/login-pages/classic-login/customize-password-reset-page) y las [políticas de contraseñas](/es/docs/customize/internationalization-and-localization/password-options-translation).

La [página de autenticación multifactor (MFA)](/es/docs/secure/multi-factor-authentication/customize-mfa/customize-mfa-classic-login) usa, de forma predeterminada, el widget de <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación que utiliza un factor adicional además de username y contraseña, como un code enviado por SMS." cta="Ver glosario" href="/es/docs/glossary?term=MFA">MFA</Tooltip> de Auth0, que no se puede localizar. Puede crear versiones localizadas con [guardian.js](https://github.com/auth0/auth0-guardian.js).

No se puede localizar la página de [Consent](/es/docs/customize/login-pages/customize-consent-prompts).
