---
description: Aprenda a usar Auth0 Actions para validar y almacenar datos del usuario final.
title: Usar Actions para validar y almacenar los datos del usuario final recopilados mediante personalizaciones de la pantalla de registro
---

Puede usar [Signup Prompt Customization](/es/docs/customize/login-pages/universal-login/customize-signup-and-login-prompts) junto con el trigger `pre-user-registration` para agregar a `user_metadata` datos proporcionados por el usuario final en la pantalla de registro (como el número de teléfono o la ubicación de un usuario). Opcionalmente, puede validar estos datos y mostrar un error de validación en la pantalla.

<div id="prerequisites">
  ### Requisitos previos
</div>

* El Tenant tiene un [Custom Domain](/es/docs/customize/custom-domains) verificado
* El Tenant tiene configurada una [Plantilla de página](/es/docs/customize/login-pages/universal-login/customize-templates)

<div id="add-a-field-to-the-signup-prompt">
  ### Agrega un campo a la pantalla de registro
</div>

Usa la [Management API](https://auth0.com/docs/api/management/v2/prompts/put-custom-text-by-language) para insertar un campo personalizado en una pantalla mediante uno de los [puntos de entrada](/es/docs/customize/login-pages/universal-login/customize-signup-and-login-prompts#-signup-prompt-entry-points). En este ejemplo, se añade el siguiente contenido al punto de inserción `ulp-container-form-content-start`:

```html lines
<div class="ulp-field">
  <label for="first-name">Nombre</label>
  <input type="text" name="ulp-first-name" id="first-name">
</div>
```

El resultado es una sección «Nombre» en la pantalla de registro:

<Frame>![](/docs/images/cdy7uua7fh8z/580LNGMxSkRlazJEdcVNpn/5fa98d4b414ac2133d1d6f4e327cea5a/Screenshot_2023-10-05_at_4.07.38_PM.png)</Frame>

<div id="create-an-action-in-the-pre-user-registration-trigger">
  ### Crear una Action en el trigger de prerregistro de usuario
</div>

Puede crear una Action personalizada de prerregistro de usuario desde [**Actions &gt; Library &gt; Build Custom**](https://manage.auth0.com/dashboard/us/dev-6endizjt/actions/library)**.**

<Frame>![](/docs/images/cdy7uua7fh8z/52VSznphe2TyoJQnAn4nQg/ed54ca48dea191916a3173b648463fd8/Create_Action_-_EN.png)</Frame>

En el editor de código, actualice el controlador `onExecutePreUserRegistration`:

```js lines
exports.onExecutePreUserRegistration = async (event, api) => {
  const firstName = event.request.body['ulp-first-name'];
  api.user.setUserMetadata("firstName", firstName);
};
```

De forma opcional, puede validar la entrada del usuario y enviar un error de validación llamando al método `api.validation.error`; luego, implemente la Action:

```js lines
exports.onExecutePreUserRegistration = async (event, api) => {
  const firstName = event.request.body['ulp-first-name'];
  if(!firstName) {
    api.validation.error("invalid_payload", "Missing first name");
    return;
  }

  api.user.setUserMetadata("firstName", firstName);
};
```

<div id="add-the-action-to-the-flow">
  ### Añada la Action al flujo
</div>

Vaya a [**Actions &gt; Flows &gt; Pre User Registration &gt; Add Action &gt; Custom**](https://manage.auth0.com/dashboard/us/dev-6endizjt/actions/flows/pre-user-signup/), luego arrastre y suelte la nueva Action en el flujo de registro y seleccione **Apply**.

<Frame>![](/docs/images/cdy7uua7fh8z/5JmjwcdZCX7pDukNa90cgD/dae853b10cf4c41d2d3543a8e167aae7/3333.png)</Frame>

<div id="test-the-action">
  ### Pruebe la Action
</div>

Registre una cuenta en el flujo de prueba y deje el campo **Nombre** en blanco. Verá un error al enviar el formulario:

<Frame>![](/docs/images/cdy7uua7fh8z/7qIWCsP8doFWPJGvMTNXJY/1225f9b3ff7ac01a7e9b59f03ca5e139/2024-01-31_16-10-10.png)</Frame>

Después de introducir un nombre en el campo **Nombre**, podrá enviar el formulario correctamente:

<Frame>![](/docs/images/cdy7uua7fh8z/6TaK1VQyKGELMGmhuRIbVA/b453fe2eb97c95d92e4fbd68ce3cbd6a/2024-01-31_16-12-01.png)</Frame>

<div id="verify-that-the-data-was-saved-on-user_metadata">
  ### Verifique que los datos se hayan guardado en user_metadata
</div>

Vaya a [**User Management &gt; Users**](https://manage.auth0.com/dashboard/us/dev-6endizjt/users) y confirme que los datos se hayan guardado en la pestaña Details:

<Frame>![](/docs/images/cdy7uua7fh8z/3x5lyrSJJgJQhbjsxoTmG6/033820476be7f67dc688da993fee7924/2024-01-31_16-13-35.png)</Frame>
