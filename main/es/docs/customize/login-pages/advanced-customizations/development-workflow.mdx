Hay dos modos principales para desarrollar sus pantallas de ACUL: desarrollo local y pruebas de integración en un inquilino en vivo. 

<div id="local-development">
  ## Desarrollo local
</div>

Personaliza localmente tus pantallas de ACUL con el comando `auth0 acul dev` de [Auth0 CLI](https://github.com/auth0/auth0-cli/). Haz iteraciones rápidas en tus pantallas de ACUL sin demoras ni dependencias del backend con [Universal Login Context Inspector](https://github.com/auth0/ul-context-inspector).

Universal Login Context Inspector proporciona datos de contexto simulados para probar tus pantallas de autenticación con configuraciones específicas del inquilino.

<Frame>
  <img src="/docs/images/cdy7uua7fh8z/ACUL/UniversalLoginContextInspector.png" alt="Universal Login Context Inspector" />
</Frame>

* Selecciona **Pantalla** para ver las pantallas de ACUL configuradas disponibles con datos de contexto simulados predeterminados.
  * Selecciona **Origen de datos** para elegir datos de contexto simulados servidos desde **Auth0 CDN** o datos de contexto simulados del directorio `public/screens/{prompt}/{screen}` de tu proyecto de **Desarrollo local**.
  * Selecciona el icono **Download JSON** para descargar los datos de contexto de la pantalla seleccionada. 

<div id="live-tenant-integration-testing">
  ## Pruebas de integración con un inquilino real
</div>

La integración con un inquilino real mediante el comando de Auth0 CLI `auth0 acul dev --connected` permite realizar pruebas integrales de las IU de sus pantallas personalizadas y verificar la integración con su inquilino de Auth0.
Use el modo `--connected` para las pruebas de integración cuando la IU esté prácticamente terminada, ya que cada recarga reinicia el flujo de autenticación.

Al usar `auth0 acul dev --connected`

* Seleccione el directorio de su proyecto de ACUL.
  * Seleccione `npm run build` para compilar el proyecto y configurar su inquilino de Auth0.
  * Seleccione el **puerto** desde el que se servirán sus recursos locales; de forma predeterminada, `55444`, e inicie el servidor local en segundo plano.
  * Opcionalmente, permita que el modo `--connected` supervise continuamente los cambios en el directorio `/dist/assets` del proyecto.

Abra una nueva terminal y ejecute `auth0 test login` para probar el flujo de inicio de sesión en su inquilino.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  En el modo `--connected`, el comando de Auth0 CLI supervisa los cambios en los archivos, recompila los recursos y actualiza automáticamente la configuración de su inquilino.
</Callout>

Para obtener más información, consulte [Auth0 CLI](https://auth0.github.io/auth0-cli/auth0_acul_dev.html) en GitHub.
