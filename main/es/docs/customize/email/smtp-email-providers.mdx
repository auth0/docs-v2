title: Descripción general del proveedor de correo electrónico SMTP
description: Configure un proveedor externo de correo electrónico SMTP para enviar correos electrónicos en producción y usar plantillas de correo electrónico para personalizar su apariencia y contenido.
---

Los flujos de trabajo de Auth0 basados en correo electrónico controlan cuándo se envían los correos electrónicos y qué contenido incluyen. Para enviar el correo saliente, necesita un servidor SMTP o un servicio de un proveedor externo de correo electrónico.

Puede configurar un único proveedor de correo electrónico para gestionar todos los correos electrónicos enviados desde su inquilino. Ofrecemos tres opciones de configuración: la integración con proveedores externos de correo electrónico compatibles, conexiones directas a servidores SMTP o una configuración personalizada con Actions.

<div id="supported-email-provider-integrations">
  ## Integraciones compatibles con proveedores de correo electrónico
</div>

Nuestras integraciones con proveedores de correo electrónico le permiten configurar su inquilino agregando credenciales de API de un proveedor externo compatible.

Admitimos los siguientes proveedores:

* [Amazon SES](/es/docs/customize/email/smtp-email-providers/amazon-ses)
* [Azure Communication Services](/es/docs/customize/email/smtp-email-providers/azure-communication-services)
* [Mandrill](/es/docs/customize/email/smtp-email-providers/mandrill)
* [Microsoft 365 Exchange Online](/es/docs/customize/email/smtp-email-providers/microsoft-365-exchange-online)
* [Resend](/es/docs/customize/email/smtp-email-providers/resend)
* [SendGrid](/es/docs/customize/email/smtp-email-providers/sendgrid)
* [SparkPost](/es/docs/customize/email/smtp-email-providers/sparkpost)
* [Mailgun](/es/docs/customize/email/smtp-email-providers/mailgun)

<div id="custom-configuration-with-actions">
  ## Configuración personalizada con Actions
</div>

Puede crear una <Tooltip tip="Action: Una función de Node.js que se ejecuta en puntos definidos del entorno de ejecución de Auth0." cta="Ver glosario" href="/es/docs/glossary?term=MFA">Action</Tooltip> para conectarse a cualquier proveedor externo de correo electrónico. Esto le permite implementar lógica personalizada para su caso de uso y tener un mayor control sobre el proceso de envío de correos electrónicos.

Para obtener más información, consulte [Configuración personalizada de proveedores de correo electrónico con Actions](/es/docs/customize/email/smtp-email-providers/custom).

<div id="smtp-server-configuration">
  ## Configuración del servidor SMTP
</div>

Puede configurar su inquilino agregando directamente la información del servidor SMTP. Esta es una buena opción si:

* No admitimos la integración con el proveedor de correo electrónico que eligió y no necesita ni quiere crear una Action para configurarlo.

* Quiere configurar un servidor SMTP de prueba (como [Debug Mail](https://debugmail.io/), [FakeSMTP](https://nilhcem.github.io/FakeSMTP/), o [smtp4dev](https://github.com/rnwood/smtp4dev)) para realizar pruebas.

No permitimos ciertos dominios ficticios que se usan habitualmente durante las pruebas. Use direcciones de correo electrónico reales para evitar interrupciones o errores.

Para obtener instrucciones, consulte [Configurar un proveedor de correo electrónico con información del servidor SMTP](/es/docs/customize/email/smtp-email-providers/smtp-server).

<div id="the-built-in-email-provider">
  ## El proveedor de correo electrónico integrado
</div>

El proveedor de correo electrónico integrado está disponible de forma predeterminada y no requiere ninguna configuración. Sin embargo, no admite el uso en producción y tiene los siguientes límites:

* No puede usar plantillas de correo electrónico personalizadas.

* No puede personalizar la **dirección del remitente**. Todos los correos electrónicos se envían desde `no-reply@auth0user.net`.

* Solo puede enviar 10 correos electrónicos por minuto, independientemente del tipo de correo electrónico.

* Si sus correos electrónicos tienen una tasa de rebote alta, podemos reducir o bloquear temporalmente la capacidad de enviar correo electrónico desde su inquilino.

Para eliminar estos límites y usar el correo electrónico en producción, configure un proveedor de correo electrónico SMTP externo.
