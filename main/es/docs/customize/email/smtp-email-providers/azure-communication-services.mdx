<div id="prerequisites">
  ## Requisitos previos
</div>

* Debe tener una cuenta de [Microsoft Azure](https://azure.microsoft.com/en-us).

* Debe configurar su proveedor para permitir conexiones entrantes desde direcciones IP específicas de [Auth0](/es/docs/secure/security-guidance/data-security/allowlist).

<div id="configure-azure-communication-services">
  ## Configurar Azure Communication Services
</div>

Primero, cree un recurso en Microsoft Azure:

1. Inicie sesión en su cuenta de [Microsoft Azure](https://azure.microsoft.com/en-us).
2. Escriba &quot;Communication Services&quot; en la barra de búsqueda.
3. Cree un recurso de [Communication Services](https://learn.microsoft.com/en-us/azure/communication-services/quickstarts/create-communication-resource) si todavía no tiene uno.
4. Seleccione **Create**.
5. En la configuración de **Tools**, seleccione **Keys**. Obtenga la [cadena de conexión de Communication Services](https://learn.microsoft.com/en-us/azure/communication-services/quickstarts/create-communication-resource).
6. En **Primary Key**, copie el valor de **Connection string**.
   Una cadena de conexión válida tiene el formato `endpoint=COMMUNICATION_SERVICES_URL;accesskey=ACCESS_KEY`.

A continuación, configure su proveedor de correo electrónico en Auth0:

1. Vaya a [Dashboard &gt; Branding &gt; Proveedor de correo electrónico](https://manage.auth0.com/#/branding/email_provider).

2. Seleccione **Use my own proveedor de correo electrónico**.

3. Seleccione **Azure Communication Services**.

4. Introduzca una dirección de correo electrónico en **From**.

   <Warning>
     Azure Communications Services solo admite una dirección de correo electrónico simple (`user@example.com`) en el campo **From**.
   </Warning>

5. Agregue la **Connection string** de Azure Communication Services.

6. Seleccione **Save**.

<div id="send-a-test-email">
  ## Envíe un correo electrónico de prueba
</div>

Envíe un correo electrónico de prueba con el botón **Send Test Email**. Si la configuración es correcta, recibirá un correo electrónico de confirmación. Si no recibe ningún correo electrónico después de unos minutos, consulte sus [registros de Auth0](https://manage.auth0.com/#/logs) para ver los códigos de error.

Una vez configurado, la página [Metrics Monitor](https://learn.microsoft.com/en-us/azure/communication-services/concepts/metrics#chat-api-request-metric-operations) de Communication Services muestra información de entrega de todos los correos electrónicos enviados a sus usuarios.
