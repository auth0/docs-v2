<div id="prerequisites">
  ## Requisitos previos
</div>

* Debe tener una cuenta de [Microsoft 365](https://www.microsoft.com/en-us/microsoft-365) con una suscripción activa.

* Su suscripción a Microsoft 365 debe incluir una licencia para enviar correos electrónicos mediante Exchange Online. Como mínimo, necesita una suscripción de pago al plan Microsoft Business Basic para usar la API de Microsoft Graph con Exchange Online.

* Debe configurar su proveedor para permitir conexiones entrantes desde [direcciones IP específicas de Auth0](/es/docs/secure/security-guidance/data-security/allowlist).

<div id="configure-microsoft-365-exchange-online">
  ## Configurar Microsoft 365 Exchange Online
</div>

1. Inicie sesión en su cuenta de [Microsoft Azure](https://azure.microsoft.com/) con su cuenta de Microsoft 365.
2. Registre un recurso de [aplicación de Microsoft Entra ID](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application) si aún no tiene uno.
3. Vaya al recurso de la aplicación de Active Directory. Seleccione **Services &gt; App registrations** y, a continuación, seleccione la aplicación que usará para enviar correos electrónicos.
4. Configure los permisos de aplicación necesarios; para ello, vaya a **Manage &gt; API permissions**.
5. Agregue el [permiso](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#api-permissions) `Mail.Send` seleccionando **Add a permission &gt; Microsoft Graph &gt; Application permissions**.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Antes de configurar **Application permissions**, necesitará consentimiento de administrador.
   </Callout>
6. Obtenga la configuración requerida del proveedor:

   1. Vaya a **Application Overview** para obtener el [ID de cliente y el ID del inquilino](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#application-id-client-id) de su aplicación.
   2. Vaya a **Manage &gt; Certificates &amp; secrets** y copie el **Value** para crear un [secreto](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#certificates--secrets) de aplicación.
   3. Obtenga el **User Email** de la organización que enviará los correos electrónicos a través de la aplicación desde **Search &gt; Users**. Seleccione **Services &gt; Users**, elija su usuario autorizado y, a continuación, copie el **User Email** de sus propiedades.
7. Vaya a Auth0 [Dashboard &gt; Branding &gt; Email Provider](https://manage.auth0.com/#/branding/email_provider).
8. Active el interruptor **Use my own email provider**.
9. Haga clic en el logotipo de **Microsoft 365**.
10. Proporcione la dirección de correo electrónico en **From**. Luego, introduzca el **ID del inquilino**, el **ID de cliente** y el **Client Secret** del paso 6 y haga clic en **Save**.

<div id="send-a-test-email">
  ## Enviar un correo electrónico de prueba
</div>

Envía un correo electrónico de prueba con el botón **Send Test Email**. Si la configuración es correcta, recibirás un correo electrónico de confirmación. Si no recibes ningún correo electrónico después de unos minutos, consulta tus [registros de Auth0](https://manage.auth0.com/#/logs) y revisa los códigos de error.

Una vez configurado, la página [Email Usage](https://admin.microsoft.com/#/reportsUsage/EmailActivity) de Microsoft Exchange muestra información de entrega de todos los correos electrónicos enviados a tus usuarios.
