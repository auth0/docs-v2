<div id="prerequisites">
  ## Requisitos previos
</div>

* Debe tener una cuenta de [Mailgun](https://mailgun.com).

* Debe configurar su proveedor para permitir conexiones entrantes desde [direcciones IP específicas de Auth0](/es/docs/secure/security-guidance/data-security/allowlist).

<div id="configure-mailgun">
  ## Configurar Mailgun
</div>

1. Inicie sesión en [Mailgun](https://mailgun.com).
2. Vaya a Mailgun **Sending &gt; Domains** y agregue su dominio personalizado. Mailgun solo permite enviar correos electrónicos desde dominios verificados.
3. Vaya a Mailgun **Settings &gt; API Keys**. Su clave de API se creó cuando registró su cuenta; cópiela desde **Private API Key**.
4. Vaya al [Dashboard &gt; Branding &gt; proveedor de correo electrónico](https://manage.auth0.com/#/branding/email_provider) de Auth0.
5. Active el interruptor **Use my own email provider**.
6. Haga clic en el logotipo de **Mailgun**.
7. Proporcione una dirección de correo electrónico en **From**, introduzca el **Domain** que agregó previamente a Mailgun, introduzca la **API Key** de Mailgun que copió previamente y seleccione su **Region**.
8. Haga clic en **Save**.

<div id="send-a-test-email">
  ## Enviar un correo electrónico de prueba
</div>

Envía un correo electrónico de prueba con el botón **Enviar correo electrónico de prueba**. Si la configuración es correcta, recibirás un correo electrónico de confirmación. Si no recibes ningún correo electrónico después de unos minutos, consulta tus [registros de Auth0](https://manage.auth0.com/#/logs) para ver los códigos de error.

Una vez configurado, el [panel de Mailgun](https://app.mailgun.com/app/dashboard) muestra información sobre la entrega de todos los correos electrónicos enviados a tus usuarios.
