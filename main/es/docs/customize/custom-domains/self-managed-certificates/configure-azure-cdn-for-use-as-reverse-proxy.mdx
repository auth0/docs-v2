description: Aprenda a configurar Azure CDN para usarlo como proxy inverso de dominio personalizado para Auth0.
title: Configurar Azure CDN como proxy inverso
---

<Card title="La disponibilidad varía según el plan de Auth0">
  Su plan de Auth0 o acuerdo personalizado determina si esta función está disponible. Para obtener más información, consulte [Precios](https://auth0.com/pricing).
</Card>

Para configurar Azure CDN como proxy inverso, se requiere un plan Azure CDN Premium.

1. [Configure dominios personalizados con certificados autoadministrados](/es/docs/customize/custom-domains/self-managed-certificates) si aún no lo ha hecho. Tome nota de los valores de **Origin Domain Name** y **cname-api-key**, ya que los necesitará más adelante.
2. Inicie sesión en el [Portal de Azure](https://portal.azure.com/).
3. [Cree un nuevo perfil y endpoint de Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-create-new-endpoint). En la configuración del endpoint de CDN, proporcione el **Origin Domain Name** desde Auth0 Dashboard.
4. [Agregue su dominio personalizado a su endpoint de Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-map-content-to-custom-domain?tabs=azure-dns%2Cazure-portal%2Cazure-portal-cleanup).
5. [Configure HTTPS para su dominio personalizado de Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-custom-ssl?tabs=option-1-default-enable-https-with-a-cdn-managed-certificate). Este proceso requiere que verifique la propiedad del dominio. Una vez hecho esto, la implementación del certificado en todas las ubicaciones POP de la CDN puede tardar hasta 6 horas.
6. Configure la comunicación del dominio personalizado con Auth0 mediante el motor de reglas de Azure CDN. (Para obtener más información, consulte [Invalidar el comportamiento HTTP mediante el motor de reglas de Verizon Premium para Azure CDN](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine) en la documentación de Microsoft). [Cree una nueva regla de Azure CDN](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine#tutorial) con la siguiente configuración:

   | Configuración       | Valor                                                                                                                |
   | ------------------- | -------------------------------------------------------------------------------------------------------------------- |
   | Nombre/Descripción  | Auth0 Custom Domain                                                                                                  |
   | Tipo de solicitudes | Seleccione la opción **Edge CName** y, a continuación, seleccione el nombre de su dominio personalizado de la lista. |
7. Agregue las siguientes **Features** a su regla de Azure CDN:
   Recomendamos crear otra regla de Azure CDN para denegar el uso del CNAME **azureedge.net**.

   | Configuración                                 | Valor                                                                                                                                       |
   | --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
   | Omitir caché                                  | **Habilitado**                                                                                                                              |
   | Modificar encabezado de solicitud del cliente | Seleccione **Override**, introduzca **cname-api-key** como nombre e introduzca como valor la clave de API de CNAME proporcionada por Auth0. |
8. Una vez aprobada la regla de Azure CDN, el estado cambiará de Pending XML a Active XML. En este punto, Azure CDN publicará las reglas y los certificados. Cuando Azure termine de procesar todos los cambios, podrá usar su dominio personalizado.
