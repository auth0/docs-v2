La funcionalidad de Auth0 Multiple <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre especializado o distintivo." cta="Ver glosario" href="/es/docs/glossary?term=Custom+Domains">dominios personalizados</Tooltip> (MCD) ofrece una gran flexibilidad, ya que permite a las organizaciones gestionar distintas marcas, requisitos regionales e integraciones con partners, todo desde un único inquilino de Auth0. Esto simplifica la gestión de identidades y, al mismo tiempo, ofrece una experiencia fluida y alineada con la marca para sus usuarios.

<div id="single-brand-multiple-top-level-domains-tlds">
  ### Una sola marca, varios dominios de nivel superior (TLD)
</div>

Algunas empresas, como las aerolíneas, requieren distintos nombres de dominio de nivel superior (por ejemplo: `.com`, `.co.uk`, `.com.au`) incluso con una sola base de usuarios. Esto suele deberse a regulaciones jurisdiccionales o a las preferencias de los clientes. Con MCD, puedes crear fácilmente esta configuración en Auth0, con un único inquilino y un almacén de usuarios unificado, mientras ofreces varios dominios de nivel superior. Por ejemplo, tu sitio web principal podría ser `exampleco.com`, pero también puedes operar `exampleco.co.uk` y `exampleco.com.au`.

<div id="multiple-brands-single-user-base">
  ### Varias marcas, una sola base de usuarios
</div>

Es habitual que las empresas tengan varias marcas, cada una con su propio dominio, y aun así se beneficien de un único almacén de usuarios y de otras capacidades unificadas dentro de un solo inquilino de Auth0. En el comercio minorista, por ejemplo, una empresa podría gestionar distintas marcas que ofrecen muebles para el hogar de alta gama (`brand1-home-furniture.com`) y muebles de oficina (`brand2-office-furniture.com`). MCD permite que estas marcas mantengan nombres de dominio independientes mientras comparten un almacén de usuarios centralizado y la configuración de Auth0.

<div id="white-labeled-b2b-saas">
  ### SaaS B2B de marca blanca
</div>

Muchos proveedores de SaaS ofrecen sus servicios con un modelo de marca blanca. Imagine job-saas.com, una plataforma para gestionar solicitudes de empleo. Las empresas cliente pueden ofrecer los servicios de `job-saas.com` bajo un subdominio propio y personalizar su aspecto y experiencia, como `jobs.client-a.com` y `jobs.client-b.com`. Con MCD, `job-saas.com` puede mantener toda su configuración de Auth0 en un único inquilino y, aun así, ofrecer diferentes organizaciones y dominios personalizados directamente a sus clientes.
