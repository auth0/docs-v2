El objeto de API del desencadenador de Actions custom-phone-provider incluye:

<div id="apicache">
  ## `api.cache`
</div>

Modifica la caché.

<div id="apicachedeletekey">
  ### `api.cache.delete(key)`
</div>

Elimina el registro que describe un valor almacenado en caché
para la clave proporcionada, si existe.

<ResponseField name="key" type="string">
  La clave del registro de caché que se va a eliminar.
</ResponseField>

<div id="apicachegetkey">
  ### `api.cache.get(key)`
</div>

Recupera un registro que describe un valor almacenado en caché en la clave indicada,
si existe. Si se encuentra un registro, el valor almacenado en caché se encuentra
en la propiedad `value` del objeto devuelto.

<ResponseField name="key" type="string">
  La clave del registro almacenado en la caché.
</ResponseField>

<div id="apicachesetkey-value-options">
  ### `api.cache.set(key, value, options)`
</div>

Almacena o actualiza un valor de cadena en la caché con la clave especificada.

Los valores almacenados en esta caché están limitados al Trigger en el que
se establecen. Están sujetos a los [Límites de caché de Actions](https://auth0.com/docs/customize/actions/limitations).

Los valores almacenados de esta manera tendrán una duración de *hasta* los valores
de `ttl` o `expires_at` especificados. Si no se especifica ninguna duración, se usará una
duración predeterminada de 15 minutos. Las duraciones no pueden superar la duración máxima
indicada en [Límites de caché de Actions](https://auth0.com/docs/customize/actions/limitations).

**Importante**: Esta caché está diseñada para datos efímeros y de corta duración. Es posible que los elementos no estén
disponibles en transacciones posteriores incluso si siguen dentro de la duración especificada.

<ResponseField name="key" type="string">
  La clave del registro que se almacenará.
</ResponseField>

<ResponseField name="value" type="string">
  El valor del registro que se almacenará.
</ResponseField>

<ResponseField name="options" type="cachesetoptions" post={["optional"]}>
  Opciones para ajustar el comportamiento de la caché.

  <Expandable title="propiedades de options" defaultOpen>
    <ResponseField name="expires_at" type="number" post={["optional"]}>
      La hora absoluta de expiración en milisegundos desde la época de Unix.
      Aunque los registros en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor de `expires_at` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se proporcionó un valor
      para `ttl`. Si se proporcionan ambas opciones, se usará la
      expiración más próxima de las dos.
    </ResponseField>

    <ResponseField name="ttl" type="number" post={["optional"]}>
      El valor de tiempo de vida de esta entrada de caché en milisegundos.
      Aunque los valores en caché pueden eliminarse antes, nunca
      permanecerán más allá del valor de `ttl` proporcionado.

      *Nota*: Este valor no debe proporcionarse si también se proporcionó un valor
      para `expires_at`. Si se proporcionan ambas opciones, se usará la
      expiración más próxima de las dos.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="apinotification">
  ## `api.notification`
</div>

Indica si debe considerarse que el evento de notificación debe reintentarse o descartarse.
Consulta cada uno de estos métodos para obtener más información sobre el comportamiento real.
Si se realizan varias llamadas, solo se tiene en cuenta la última.

<div id="apinotificationdropreason">
  ### `api.notification.drop(reason)`
</div>

Cuando se llama, el evento de notificación se considera fallido y no se volverá a intentar:
Registraremos un error para este evento, pero no volveremos a enviarlo a la acción.
Si necesitas que este evento de notificación se vuelva a intentar, considera llamar a retry en su lugar.

<ResponseField name="reason" type="string">
  este motivo formará parte de la entrada de registro, lo que te ayudará a analizar mejor el error. Ten en cuenta que este campo está limitado a 1024 caracteres y se truncará si es más largo.
</ResponseField>

<div id="apinotificationretryreason">
  ### `api.notification.retry(reason)`
</div>

Cuando se llama, el evento de notificación se considera fallido, pero recuperable:
Registraremos un error para este evento, pero volveremos a intentarlo hasta 5 veces en los próximos minutos.
Si considera que este evento de notificación no debe volver a intentarse, considere llamar a drop en su lugar.

<ResponseField name="reason" type="string">
  este motivo formará parte de la entrada de registro y le ayudará a analizar mejor el error. Tenga en cuenta que este campo está limitado a 1024 caracteres y se truncará si se supera ese límite.
</ResponseField>
