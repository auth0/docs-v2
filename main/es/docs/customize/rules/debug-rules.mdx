<Warning>
  La fecha de fin de vida (EOL) de Rules y Hooks será el **18 de noviembre de 2026**, y ya no están disponibles para nuevos inquilinos creados a partir del **16 de octubre de 2023**. Los inquilinos existentes con Hooks activos conservarán el acceso al producto Hooks hasta el fin de su vida útil.

  Le recomendamos encarecidamente que use Actions para ampliar Auth0. Con Actions, tiene acceso a abundante información de tipos, documentación integrada y paquetes `npm` públicos, y puede conectar integraciones externas que mejoran su experiencia general de extensibilidad. Para obtener más información sobre lo que ofrece Actions, consulte [Comprenda cómo funcionan las Auth0 Actions](/es/docs/customize/actions/actions-overview).

  Para ayudarle con la migración, ofrecemos guías que le ayudarán a [migrar de Rules a Actions](/es/docs/customize/actions/migrate/migrate-from-rules-to-actions) y [migrar de Hooks a Actions](/es/docs/customize/actions/migrate/migrate-from-hooks-to-actions). También contamos con una página dedicada de [Migrar a Actions](https://auth0.com/extensibility/movetoactions) que destaca comparaciones de funcionalidades, [una demostración de Actions](https://www.youtube.com/watch?v=UesFSY1klrI) y otros recursos para ayudarle en su proceso de migración.

  Para obtener más información sobre la descontinuación de Rules y Hooks, lea nuestra entrada del blog: [Preparación para el fin de vida de Rules y Hooks](https://auth0.com/blog/preparing-for-rules-and-hooks-end-of-life/).
</Warning>

<Warning>
  Exponemos direcciones IPv6 en nuestros endpoints públicos (por ejemplo, `travel0.us.auth0.com`). Si una solicitud proviene de una máquina compatible con IPv6, la propiedad `context.request.ip` contendrá una dirección IPv6. Si manipula manualmente direcciones IP, le sugerimos usar la [biblioteca ipaddr.js@1.9.0](https://www.npmjs.com/package/ipaddr.js/v/1.9.0).
</Warning>

Para depurar cualquier [Auth0 Rule](/es/docs/customize/rules) que haya creado, puede usar `console.log` dentro del código de su Rule. Puede ver la salida de `console.log` con **Save and Try**, consultando los registros disponibles en [Actions Real-time Logs](/es/docs/customize/actions/actions-real-time-logs) o, para clientes heredados, usando [Debug Rule CLI](#debug-rule-cli).

<div id="try-this-rule">
  ## Pruebe esta Rule
</div>

En el [Editor de Rules](https://manage.auth0.com/#/rules/create), el botón **Save and Try** le permite ejecutar una Rule de forma aislada con objetos **user** y **context** simulados. Al hacer clic en **TRY**, se ejecutará la Rule con esos dos objetos como entrada y se mostrará cualquier salida de `console.log`.

La función **Try this rule script** funciona fuera del contexto de un cliente específico. Al usar esta función, puede tener problemas si su Rule depende de datos que se proporcionarían al invocarla desde una aplicación real.

<div id="actions-real-time-logs">
  ## Actions Real-time Logs
</div>

Actions Real-time Logs muestra en tiempo real todos los registros de todo el código personalizado de su cuenta, incluida toda la salida de `console.log` y las excepciones. Para obtener más información, consulte [Actions Real-time Logs](/es/docs/customize/actions/actions-real-time-logs).
