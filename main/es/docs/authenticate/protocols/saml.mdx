El protocolo <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/es/docs/glossary?term=Security+Assertion+Markup+Language">Security Assertion Markup Language</Tooltip> (SAML) es un marco abierto basado en XML para la autenticación y autorización entre dos entidades sin necesidad de contraseña:

* El **proveedor de servicios** (SP) acepta confiar en el <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/es/docs/glossary?term=identity+provider">proveedor de identidad</Tooltip> para autenticar a los usuarios.
* El **proveedor de identidad** (IdP) autentica a los usuarios y proporciona a los proveedores de servicios una aserción de autenticación que indica que el usuario ha sido autenticado.

Auth0 admite el protocolo SAML y puede actuar como IdP, SP o ambos, lo que incluye:

* aplicaciones web SAML2
* integraciones de <Tooltip tip="Single Sign-On (SSO): Servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión en otras aplicaciones." cta="Ver glosario" href="/es/docs/glossary?term=SSO">SSO</Tooltip> con SAML
* SSO iniciado por el IdP
* Microsoft Active Directory Federation Services (ADFS)
* firma y cifrado de solicitudes SAML

<div id="supported-saml-bindings-and-options">
  ## Bindings y opciones de SAML compatibles
</div>

Auth0 admite los siguientes bindings de SAML:

* HTTP Redirect
* HTTP POST

Auth0 admite las siguientes opciones de SAML:

* Perfil de SSO en navegador web
* Perfil de cierre de sesión único
* Perfil de administración del identificador de nombre
* Perfil de asignación del identificador de nombre
