<Card title="Antes de comenzar">
  Asegúrate de que tu inquilino de Auth0 esté conectado a Azure AD mediante el [tipo de conexión Microsoft Azure AD](/es/docs/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

  Para esta integración, deberás registrar dos aplicaciones en Azure AD: la integración de OpenID Connect y la integración de SCIM. Para simplificar este proceso de configuración para tus clientes, considera [publicar tu aplicación en la galería de aplicaciones de Azure Active Directory](https://learn.microsoft.com/en-us/entra/identity/enterprise-apps/v2-howto-app-gallery-listing#submit-your-application).
</Card>

Auth0 puede integrarse con Microsoft Azure Active Directory (ahora conocido como Microsoft Entra ID) mediante el [tipo de conexión Microsoft Azure AD](/es/docs/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2), que usa el protocolo <Tooltip tip="OpenID: estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/es/docs/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) para la autenticación de usuarios. Estas instrucciones se aplican a las conexiones nuevas de Azure AD; para conexiones antiguas que usan identificadores de usuario por pares (`sub`), consulta [Conexiones antiguas](/es/docs/authenticate/protocols/scim/inbound-scim-for-older-azure-ad-connections).

<div id="configure-scim-settings-in-auth0">
  ## Configurar la configuración de SCIM en Auth0
</div>

1. Abra el [Auth0 Dashboard](https://manage.auth0.com/#/), vaya a **Authentication &gt; Enterprise &gt; Microsoft Azure AD &gt; [your-connection] &gt; Settings.**
2. Asegúrese de que **User ID Attribute Type** esté configurado como **User Object Identifier (oid)** y de que el interruptor **Use Common Endpoint** esté **Disabled**.
3. Seleccione la pestaña **Provisioning** y desactive **Sync user profile attributes at each login**, a menos que quiera [sincronizar atributos adicionales al iniciar sesión](/es/docs/authenticate/protocols/scim/configure-inbound-scim).
4. En la misma sección, active **Sync user profiles using SCIM**.
5. En la pestaña **Mapping**, asegúrese de que la opción **SCIM attribute containing the User ID** esté configurada como **externalId**.
6. Revise **Additional Mappings** para asegurarse de que los atributos extendidos de SCIM estén asignados a los atributos de Auth0 que prefiera. Consulte [mapeo de atributos](/es/docs/authenticate/protocols/scim/configure-inbound-scim#attribute-mapping) para obtener más información.

<div id="retrieve-scim-endpoint-url-and-token">
  ## Obtener la SCIM Endpoint URL y el token
</div>

En esta sección se usa el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, pero estos pasos también pueden realizarse con la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>. Consulte la sección [Pautas de implementación](/es/docs/authenticate/protocols/scim/configure-inbound-scim#deployment-guidelines) para conocer las prácticas recomendadas.

1. En el [Auth0 Dashboard](https://manage.auth0.com/#/), vaya a la pestaña **Setup**, copie la **SCIM Endpoint URL** y péguela en un lugar seguro.
2. Genere un token SCIM haciendo clic en **Generate New Token** y, si lo desea, establezca una fecha de vencimiento para el token.
3. Seleccione los alcances que desea permitir. Los alcances predeterminados que requiere Azure AD son `get:users`, `post:users`, `patch:users` y `delete:users`. Opcionalmente, puede seleccionar `get:groups`, `post:groups`, `patch:groups` y `delete:groups` para aprovisionar grupos.

<div id="configure-scim-in-azure-ad-for-oidc-apps">
  ## Configurar SCIM en Azure AD para aplicaciones OIDC
</div>

1. Confirme que ya se haya registrado una aplicación OpenID Connect para gestionar la autenticación de usuarios en la sección **Microsoft Entra ID &gt; App registrations** del [portal de Azure](https://portal.azure.com).
2. Confirme que su aplicación OpenID Connect tenga **Assignment Required** establecido en **Yes** en la sección **Microsoft Entra ID &gt; Enterprise applications &gt; [your-oidc-app] &gt; Manage &gt; Properties** y que tenga usuarios asignados en la pestaña **Users and Groups**.
3. A continuación, registre una nueva aplicación **Non-gallery** en el [portal de Azure](https://portal.azure.com). Para ello, vaya a **Microsoft Entra ID &gt; Enterprise applications &gt; New application &gt; Create your own application**, introduzca un nombre para la aplicación y seleccione **Create**.
4. Vaya a la pestaña **Users and Groups** y asigne los mismos usuarios y grupos de Azure AD que están asignados a la aplicación OpenID Connect registrada.
5. Seleccione la pestaña **Provisioning**, haga clic en **Get started** y elija **Automatic** como **Provisioning Mode.**
6. Seleccione **Admin Credentials** y, a continuación, introduzca el valor de **SCIM Endpoint URL** que guardó anteriormente como **Tenant URL**. Al final de la URL, agregue el parámetro de consulta `?aadOptscim062020` para corregir los [problemas conocidos de Azure AD que se describen aquí.](https://learn.microsoft.com/en-us/azure/active-directory/app-provisioning/application-provisioning-config-problem-scim-compatibility)
7. Pegue el valor del token en el campo **Secret Token** y seleccione **Save.**
8. Vaya a **Mappings**.
9. Para aprovisionar grupos de forma opcional, confirme que **Provision Entra ID Groups** esté establecido en **Enabled**. De lo contrario, establézcalo en **Disabled**.
10. Seleccione **Provision Microsoft Entra ID Users**; luego vaya a **Attribute Mappings** y edite los atributos de la línea que contiene `externalId` y `mailNickname`.
11. En la pantalla **Edit Attribute**, cambie **Source attribute** a **objectId** y luego seleccione **OK**.
12. Vuelva a **Attribute Mappings** y seleccione la línea que contiene `emails[type eq "work"].value and mail`.
13. En la pantalla **Edit Attribute**, cambie **Match objects using this attribute** a **Yes**, luego establezca **Matching precedence** en **2** y seleccione **OK.** La pantalla de asignación de atributos se verá así mientras sigue usando la sección **Attribute Mappings** para configurar atributos SCIM adicionales:

<Frame>![Asignación de atributos de SAML Azure](/docs/images/cdy7uua7fh8z/6JnGm4IeBglKiHNoVPX5vs/ddc17baeca48f02a6309b57ab87f845e/attributemap2.jpg)</Frame>

**Guarde** las asignaciones de atributos y, a continuación, seleccione **X** en la esquina superior derecha para volver a la pantalla **Provisioning**.

<div id="testing">
  ## Pruebas
</div>

1. En la pantalla de resumen de la aplicación empresarial, ve a **Manage &gt; Provisioning &gt; Provision on Demand** para probar la conexión SCIM.
2. Ve a **Select a user or group** e introduce el nombre de un usuario o grupo que hayas asignado a la aplicación; después, selecciona el elemento y elige **Provision**. Esto crea el usuario o grupo en el inquilino de Auth0.
3. Establece **Provisioning Status** en **On** para [aprovisionar todos los usuarios y grupos asignados](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/configure-automatic-user-provisioning-portal#provisioning-status).
