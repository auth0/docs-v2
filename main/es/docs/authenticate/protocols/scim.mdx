---
title: Sistema para la Gestión de Identidades entre Dominios (SCIM)
description: Describe el uso del esquema de System for Cross-domain Identity Management (SCIM) en la gestión de identidades
---

SCIM (System for Cross-domain Identity Management) es un conjunto de protocolos de nivel de aplicación para gestionar y comunicar de forma segura los datos de usuario entre múltiples dominios. Los clientes SCIM pueden integrarse para gestionar operaciones CRUD (crear, reemplazar, actualizar y eliminar), aplicar consultas y filtros, y crear grupos de usuarios dentro de su organización. SCIM permite automatizar el ciclo de vida de los usuarios y mantener cuentas de usuario en distintas plataformas.

Para leer la especificación de SCIM, consulte [System for Cross-domain Identity Management: Core Schema](https://datatracker.ietf.org/doc/rfc7643/).

<div id="scim-with-auth0">
  ## SCIM con Auth0
</div>

Auth0 ofrece un directorio extensible y flexible diseñado para admitir casos de uso de CIAM, y se centra en simplificar la gestión de identidades para aplicaciones orientadas al consumidor final y de software como servicio.

En un inquilino de Auth0, puede usar directorios aislados mediante conexiones empresariales, sociales o de base de datos. Para obtener más información sobre las conexiones disponibles, lea [Proveedores de identidad](/es/docs/authenticate/identity-providers) o [Conexiones de base de datos](/es/docs/authenticate/database-connections).

Auth0 admite SCIM entrante para determinados <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=Identity+Providers">Proveedores de identidad</Tooltip> empresariales. Para obtener más información, lea [Configurar SCIM entrante](/es/docs/authenticate/protocols/scim/configure-inbound-scim).
