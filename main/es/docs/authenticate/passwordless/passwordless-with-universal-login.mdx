La experiencia de <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Universal Login</Tooltip> admite [conexiones sin contraseña](/es/docs/authenticate/passwordless), que permiten a los usuarios introducir un número de teléfono o una dirección de correo electrónico y luego recibir un código de un solo uso (OTP) para completar la autenticación.

<div id="enable-passwordless-for-universal-login">
  ## Habilitar la autenticación sin contraseña para Universal Login
</div>

Para configurar Universal Login de modo que admita conexiones <Tooltip tip="Sin contraseña: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Passwordless">sin contraseña</Tooltip>, deberá establecer el perfil de autenticación para que use un flujo de inicio de sesión compatible y actualizar su aplicación para especificar la conexión durante la autenticación.

<div id="enable-identifier-first">
  ### Habilite Identifier First
</div>

En el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, vaya a [Authentication &gt; Authentication Profile.](https://manage.auth0.com/?/authentication-profiles)

Seleccione **Identifier First** o **Identifier First + Biometrics**.

<div id="update-your-application">
  ### Actualiza tu aplicación
</div>

Según el tipo de conexiones que hayas habilitado para tu aplicación, es posible que debas actualizarla para especificar una conexión sin contraseña durante el inicio de sesión:

| Conexión de base de datos | Conexión sin contraseña    | ¿Es necesario especificarla? | Experiencia del usuario                                                                                             |
| ------------------------- | -------------------------- | ---------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| Sí                        | SMS y/o correo electrónico | Sí                           | La conexión sin contraseña especificada se muestra durante el inicio de sesión.                                     |
| No                        | SMS y correo electrónico   | No                           | Se muestra la conexión sin contraseña que se creó primero.                                                          |
| No                        | SMS o correo electrónico   | No                           | La conexión sin contraseña se muestra (junto con cualquier conexión social habilitada) durante el inicio de sesión. |

Para especificar una conexión sin contraseña durante el inicio de sesión, debes pasar el parámetro `connection` y su valor (ya sea `sms` o `email`) al [endpoint de inicio de sesión de la API de autenticación de Auth0](https://auth0.com/docs/api/authentication#login):

`https://mytenant.us.auth0.com/authorize?client_id={id}&``connection={sms|email}``&scope=…&response_type=code&response_mode=query&state=…&redirect_uri=http%3A%2F%2Flocalhost%3A3000&code_challenge=…`

Si estás usando uno de nuestros SDKs, puedes especificar una conexión sin contraseña durante la inicialización. Por ejemplo:

```js lines
auth0 = await createAuth0Client({
     domain: config.domain,
     client_id: config.clientId,
     connection: "email"
  });

auth0 = await createAuth0Client({
     domain: config.domain,
     client_id: config.clientId,
     connection: "sms"
  });
```

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si usa sin contraseña con Universal Login y [inicio de sesión único](/es/docs/authenticate/single-sign-on), los parámetros de conexión `sms` y `email` no utilizan la sesión existente de Auth0, y se solicitará al usuario que inicie sesión.
</Callout>
