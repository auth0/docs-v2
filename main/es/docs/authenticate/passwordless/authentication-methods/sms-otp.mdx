Puede configurar la autenticación <Tooltip tip="Sin contraseña: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Passwordless">sin contraseña</Tooltip> para enviar un código de un solo uso (OTP) a un usuario mediante SMS. Para obtener más información, lea [Configurar correo electrónico o SMS para la autenticación sin contraseña](/es/docs/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

Auth0 admite el uso de [Twilio](https://www.twilio.com) y de un Phone Provider personalizado para enviar OTP. Si desea usar un Phone Provider personalizado con Auth0 Actions, lea [Configurar un Phone Provider personalizado](/es/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) o [Configurar un Phone Provider personalizado con Terraform](/es/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/configure-a-custom-phone-provider-with-terraform).

<div id="how-it-works">
  ## Cómo funciona
</div>

Cuando un nuevo usuario recibe un código y lo introduce por primera vez en su aplicación, su perfil de usuario se crea en la conexión `sms` antes de que Auth0 lo autentique.

Si el número de teléfono al que se envió el OTP coincide con el de un usuario existente, Auth0 autentica al usuario:

<div id="configure-the-phone-provider">
  ## Configurar el Phone Provider
</div>

1. En el Auth0 Dashboard, vaya a [Marca &gt; Phone Provider](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Active el interruptor **Phone Message Provider**.

   <Frame>![Auth0 Dashboard > Marca > Phone Provider > Activar Phone Message Provider](/docs/images/cdy7uua7fh8z/1k93jvdxDq2FZLiRdvr5aR/d917431cd0265943c5118174a8673118/2025-02-24_16-43-14.png)</Frame>
3. Seleccione su proveedor de mensajería telefónica.

   1. Para usar Twilio, siga [Configurar Twilio como proveedor de mensajería telefónica](/es/docs/customize/phone-messages/configure-phone-messaging-providers/configure-twilio-as-a-phone-messaging-provider).
   2. Para usar un proveedor personalizado, siga [Configurar un Phone Provider personalizado](/es/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider) para usar Auth0 Actions.
4. Seleccione el método de entrega.
5. **Guarde** los cambios.

Si el Phone Provider personalizado de Auth0 Action no se ajusta a su caso de uso, puede administrar la puerta de enlace SMS personalizada en las [conexiones SMS sin contraseña](/es/docs/authenticate/passwordless) con <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>. <div id="configure-passwordless-sms-settings">
  ### Configurar los ajustes de SMS sin contraseña
</div>

1. En **Message**, introduzca el texto del cuerpo del SMS.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     El marcador de posición `@@password@@` se reemplazará automáticamente por la contraseña de un solo uso que se envía al usuario.
   </Callout>
2. Ajuste la configuración de **OTP Expiry** y **OTP Length**.

   * Solo se aceptará la última contraseña de un solo uso (o enlace) emitida. Una vez emitida la más reciente, las demás quedarán invalidadas. Cuando se use, la más reciente también quedará invalidada.
   * Solo se permiten tres intentos fallidos para introducir la contraseña de un solo uso. Después de eso, deberá solicitarse un nuevo code.
   * La contraseña de un solo uso emitida será válida (de forma predeterminada) durante tres minutos antes de expirar.
   * Si decide ampliar el tiempo que tarda en expirar la contraseña de un solo uso, también debe aumentar la longitud del code de la contraseña de un solo uso. De lo contrario, un atacante tendrá una ventana de tiempo más amplia para intentar adivinar un code corto.
3. Decida si desea **Disable Sign Ups**. Si habilita esta configuración, puede permitir el acceso sin contraseña solo a los usuarios existentes, pero también podría exponer su aplicación a ataques de enumeración de usuarios. Para obtener más información, consulte [Prácticas recomendadas para las conexiones sin contraseña](/es/docs/authenticate/passwordless/best-practices).
4. Seleccione **Save**.

<div id="multi-language-support">
  #### Compatibilidad multilingüe
</div>

El área **Message** admite varios idiomas.

Para especificar un idioma, llama al [endpoint Get Code or Link de Auth0 Authentication API](https://auth0.com/docs/api/authentication#get-code-or-link) y establece el valor del encabezado `x-request-language`. Si no se establece este encabezado, el idioma se obtiene del encabezado `accept-language`, que el navegador configura automáticamente.

<div id="message-syntax">
  #### Sintaxis del mensaje
</div>

El área de **Message** admite la sintaxis Liquid. Puede usar esta sintaxis, combinada con valores de parámetros, para crear mediante programación elementos del mensaje.

Por ejemplo, puede hacer referencia al parámetro `request_language` para cambiar el idioma del mensaje:

```liquid lines
{% if request_language contains 'dutch' %}
   Hier is uw verificatie code: {{ password }}
{% endif %}
{% if request_language contains 'fr-FR' %}
   Ceci est votre code: {{ password }}
{% endif %}
```

Los siguientes parámetros están disponibles al definir la plantilla del mensaje:

| Parameter            | Description                                                   |
| -------------------- | ------------------------------------------------------------- |
| `password` or `code` | La contraseña que se debe usar.                               |
| `phone_number`       | El número de teléfono del usuario.                            |
| `application.name`   | El nombre de la aplicación con la que el usuario se registra. |
| `request_language`   | El idioma solicitado para el contenido del mensaje.           |

Para obtener más información sobre el uso de Liquid, consulta [Liquid for Designers on GitHub](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers).
