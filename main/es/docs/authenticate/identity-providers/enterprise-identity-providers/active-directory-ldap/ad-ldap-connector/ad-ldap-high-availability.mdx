title: Implementar conectores AD/LDAP en entornos de alta disponibilidad
---

Puede implementar varias instancias del Conector AD/LDAP para ofrecer alta disponibilidad a su conexión AD/LDAP.

<div id="overview">
  ## Descripción general
</div>

Para desplegar varias instancias del Conector AD/LDAP, deberá:

1. Instalar el Conector AD/LDAP en el servidor principal.
2. Copiar o exportar los archivos de configuración de la instalación inicial.
3. Instalar el Conector AD/LDAP en servidores adicionales.
4. Importar los archivos de configuración de la instalación inicial en los conectores adicionales.

<div id="configure-primary-server">
  ## Configurar el servidor principal
</div>

1. Instale y configure el Conector AD/LDAP en el primer servidor.
2. Abra la pantalla de solución de problemas (`http://localhost:8357/#troubleshoot`) y ejecute la prueba de diagnóstico. Asegúrese de que todas las pruebas se completen correctamente.

| Prueba   | Descripción                                                                                                                    | Solución de problemas                                                                                     |
| -------- | ------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------- |
| Prueba 1 | Intenta establecer una conexión TCP con el servidor LDAP y el puerto especificados.                                            | Verifique la conectividad básica de red y la configuración del firewall que podrían impedir esa conexión. |
| Prueba 2 | Intenta realizar un bind de LDAP en el servidor LDAP y el puerto especificados con el username y la contraseña proporcionados. | Verifique la cadena de conexión de LDAP, la ruta de búsqueda, el username y la contraseña.                |
| Prueba 3 | Intenta realizar una búsqueda de LDAP en el directorio para comprobar los privilegios del username especificado.               | Verifique los privilegios del username en el directorio de destino.                                       |
| Prueba 4 | Intenta establecer una conexión con el servidor de Auth0.                                                                      | Verifique la conectividad de red y la configuración del firewall que podrían impedir esa conexión.        |

3. Copie o exporte los archivos de configuración.

<div id="configure-additional-servers">
  ## Configurar servidores adicionales
</div>

1. Instale el Conector AD/LDAP en los servidores adicionales, pero no lo configure.
2. Importe los archivos de configuración del servidor principal.
3. Reinicie los servicios de Windows Auth0 AD/LDAP y Auth0 AD/LDAP Admin en los nuevos servidores.
4. Abra la pantalla de solución de problemas (`http://localhost:8357/#troubleshoot`) y ejecute la prueba de diagnóstico. Asegúrese de que todas las pruebas se superen.

Para obtener más información, consulte [Instalar y configurar el Conector AD/LDAP](/es/docs/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/install-configure-ad-ldap-connector) e [Importar y exportar configuraciones del Conector AD/LDAP](/es/docs/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/import-export-ad-ldap-connector-configs).

<div id="verify-connections">
  ## Verificar conexiones
</div>

En el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, vaya a [Authentication &gt; Enterprise &gt; Active Directory / LDAP](https://manage.auth0.com/#/connections/enterprise/ad) y confirme que la conexión esté activa.

<div id="using-kerberos-or-client-certificates">
  ## Uso de Kerberos o certificados de cliente
</div>

Si habilita Kerberos o certificados de cliente para la autenticación en su conexión AD/LDAP, los usuarios se comunican directamente con el Conector AD/LDAP, en lugar de pasar por el servidor de Auth0.

Si usa una configuración de alta disponibilidad con varios conectores, Auth0 recomienda ponerlos detrás de un equilibrador de carga de red:

1. Use el parámetro `SERVER_URL` para publicar la ubicación pública en la que el Conector AD/LDAP recibirá las solicitudes entrantes.
2. Haga que el `SERVER_URL` del equilibrador de carga de red apunte a todas las instancias internas de los Conectores AD/LDAP desplegados. No se requiere una política de distribución especial (por ejemplo, funciona un round-robin uniforme sin sesiones persistentes).

Para obtener más información, consulte [Configurar la autenticación del Conector AD/LDAP con Kerberos](/es/docs/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/configure-ad-ldap-connector-with-kerberos) o [Configurar la autenticación del Conector AD/LDAP con certificados de cliente](/es/docs/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap/ad-ldap-connector/configure-ad-ldap-connector-client-certificates).
