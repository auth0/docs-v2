Auth0 ofrece conexiones empresariales para autenticar usuarios con un <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=identity+provider">Proveedor de identidad</Tooltip> (IdP) externo y federado, como Azure AD, Google Workspace, PingFederate, entre otros.

<Card title="La disponibilidad varía según el plan de Auth0">
  Tu plan de Auth0 o tu acuerdo personalizado afectan la disponibilidad de esta función. Para obtener más información, consulta la [página de precios de Auth0](https://auth0.com/pricing).
</Card>

<div id="create-an-enterprise-connection">
  ## Crear una conexión empresarial
</div>

Auth0 admite numerosos proveedores de identidad integrados. Para obtener más información, consulte [Proveedores de identidad empresariales](/es/docs/authenticate/identity-providers/enterprise-identity-providers).

<div id="openid-connect-oidc-protocol">
  ### Protocolo OpenID Connect (OIDC)
</div>

Las conexiones empresariales que usan <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/es/docs/glossary?term=OpenID">OpenID</Tooltip> Connect u Okta Workforce como Proveedor de identidad pueden admitir lo siguiente:

* [Proof Key for Code Exchange (PKCE)](https://www.oauth.com/oauth2-servers/pkce/)

* Claims de atributos y asignación de <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip>

* Integración con UserInfo

Puede implementar estas funciones para las conexiones de [OpenID Connect](/es/docs/authenticate/identity-providers/enterprise-identity-providers/oidc) o [Okta Workforce](/es/docs/authenticate/identity-providers/enterprise-identity-providers/okta). Para obtener más información, consulte [Configurar PKCE y la asignación de claims para conexiones OIDC](/es/docs/authenticate/identity-providers/enterprise-identity-providers/configure-pkce-claim-mapping-for-oidc).

<div id="view-enterprise-connections">
  ## Ver conexiones empresariales
</div>

<Tabs>
  <Tab title="Dashboard">
    Vaya a [**Auth0 Dashboard &gt; Authentication &gt; Enterprise**](https://manage.auth0.com/#/connections/enterprise) para ver todos los tipos de conexiones empresariales disponibles. Seleccione un tipo de conexión (por ejemplo, SAML) para ver si hay conexiones configuradas de ese tipo.

    También puede seleccionar una conexión configurada y consultar la pestaña **Applications** para ver si está habilitada para alguna aplicación.
  </Tab>

  <Tab title="Management API">
    Llame al endpoint [Get all Connections](https://auth0.com/docs/api/management/v2#!/Connections/get_connections) de la Management API de Auth0 para obtener información sobre sus conexiones. Incluya el parámetro `strategy` para filtrar por tipo de conexión.

    Los valores de `strategy` para el tipo de conexión empresarial son:

    * `ad` (Active Directory/LDAP)
    * `adfs` (ADFS)
    * `auth0-adldap`
    * `google-apps` (Google Workspace)
    * `ip`
    * `office365`
    * `oidc` (OpenID Connect)
    * `pingfederate` (PingFederate)
    * `samlp` (SAML)
    * `sharepoint`
    * `waad` (Microsoft Azure AD)
  </Tab>
</Tabs>

<div id="what-is-an-active-enterprise-connection">
  ## ¿Qué es una conexión empresarial activa?
</div>

Una conexión empresarial se considera activa si, durante el mes en curso, cumple ambas condiciones:

* Ha estado habilitada para una aplicación.
* Ha tenido actividad de usuario (por ejemplo, inicio de sesión, registro o intercambio de tokens).

Si una conexión empresarial nunca se habilitó para ninguna aplicación, o se habilitó pero no tuvo actividad de usuario durante el mes en curso, no se considera activa.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La conexión empresarial de Okta Workforce no cuenta para el límite de conexiones empresariales activas.
</Callout>
