---
description: Aprenda cómo el flujo conforme a OIDC afecta al uso de la delegación.
title: Delegación con OIDC
---

<Warning>
  De forma predeterminada, la delegación está deshabilitada para los inquilinos que no tuvieran un complemento en uso a fecha del 8 de junio de 2017. Los inquilinos heredados que actualmente usan un complemento que requiere delegación pueden seguir usando esta función. Si la funcionalidad de delegación cambia o se elimina del servicio en algún momento, se notificará previamente a los clientes que la usan actualmente y se les dará tiempo suficiente para migrar.
</Warning>

Tradicionalmente, la delegación se usa para:

* Intercambiar un <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=ID+token">token de ID</Tooltip> emitido para una aplicación por otro nuevo emitido para una aplicación distinta.
* Obtener un token de ID nuevo mediante un [token de actualización](/es/docs/secure/tokens/refresh-tokens).
* Intercambiar un token de ID por un token de API de terceros (por ejemplo, Firebase, AWS).

Debido a que el flujo conforme a OIDC exige que los tokens de ID ya no se utilicen para proteger las API y que los <Tooltip tip="Token de actualización: Token usado para obtener un Token de acceso renovado sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/es/docs/glossary?term=refresh+tokens">tokens de actualización</Tooltip> se usen solo en el endpoint [`/oauth/token`](https://auth0.com/docs/api/authentication#refresh-token), el endpoint [`/delegation`](https://auth0.com/docs/api/authentication#delegation) ha quedado obsoleto.

Las aplicaciones conformes a OIDC no pueden ser ni el origen ni el destino de las solicitudes de delegación.

<div id="third-party-apis">
  ## API de terceros
</div>

Dado que no existe ningún mecanismo compatible con OIDC para obtener tokens de API de terceros (por ejemplo, Firebase o AWS), la delegación todavía puede usarse para obtenerlos.
