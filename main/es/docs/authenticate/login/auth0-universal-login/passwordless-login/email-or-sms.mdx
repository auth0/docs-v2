title: Configurar el correo electrónico o SMS para la autenticación sin contraseña
---

Auth0 te permite configurar <Tooltip tip="Universal Login: Tu aplicación redirige a Universal Login, alojado en el Servidor de autorización de Auth0, para verificar la identidad del usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Universal Login</Tooltip> para que los usuarios puedan autenticarse con un enlace mágico o una contraseña de un solo uso (OTP) por correo electrónico, o con un OTP por SMS.

Para habilitar la autenticación <Tooltip tip="sin contraseña: Forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Passwordless">sin contraseña</Tooltip> con SMS o correo electrónico, debes definir una [conexión sin contraseña](/es/docs/authenticate/passwordless) para tu aplicación.

Una vez que hayas creado la conexión, debes configurar Universal Login para que la use. Tienes tres opciones:

<div id="universal-login">
  ## Universal Login
</div>

Este método utiliza la experiencia de Universal Login, que admite de forma nativa las conexiones sin contraseña. Para obtener más información, consulte [sin contraseña con Universal Login](/es/docs/authenticate/passwordless/passwordless-with-universal-login).

<div id="universal-login-lock-passwordless">
  ## Universal Login + Lock (sin contraseña)
</div>

Este método usa una página de inicio de sesión personalizada con la plantilla **Lock (sin contraseña)** y autentica al usuario con el [SDK de Lock para Web](/es/docs/libraries/lock):

1. Ve a [Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/#/login_settings) y haz clic en la pestaña **Login**.
2. Activa el interruptor **Custom Login Page** y selecciona la plantilla **Lock (sin contraseña)**. La plantilla HTML se actualizará con código que usa el widget Lock con opciones de personalización para la autenticación sin contraseña.
3. Personaliza la plantilla y haz clic en **Save Changes**.

Puedes usar HTML y CSS para personalizar el formulario de inicio de sesión y obtener una vista previa de los cambios en el Dashboard. Para obtener más información sobre cómo personalizar la plantilla **Lock (sin contraseña)**, consulta la sección sobre autenticación sin contraseña en [SDK de Lock para Web](/es/docs/libraries/lock).

<div id="universal-login-custom-ui-auth0js">
  ## Universal Login + UI personalizada + Auth0.js
</div>

Este método utiliza una página de inicio de sesión personalizada con la plantilla **Custom Login Form** y autentica a los usuarios con el [SDK de Auth0.js](/es/docs/libraries/auth0js). Para obtener más información, consulta [Autenticación sin contraseña con Universal Login](/es/docs/authenticate/passwordless/implement-login/universal-login).
