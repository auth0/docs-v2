title: Inicio de sesión integrado
---

Embedded Login permite a sus usuarios iniciar sesión directamente en su aplicación y transmitir sus credenciales al servidor de Auth0 para su autenticación. No recomendamos usar Embedded Login. Para obtener más información, consulte [Universal Login centralizado vs. Embedded Login](/es/docs/authenticate/login/universal-vs-embedded-login).

Si decide usar Embedded Login, debe [configurar su aplicación para el intercambio de recursos entre orígenes](/es/docs/get-started/applications/set-up-cors) y debería [configurar un dominio personalizado](/es/docs/customize/custom-domains). Luego puede implementar el [SDK de Lock](/es/docs/libraries/lock) o el [SDK de Auth0.js](/es/docs/libraries/auth0js) en su aplicación, o llamar directamente a la [API de autenticación de Auth0](https://auth0.com/docs/api/authentication).

Auth0 admite Embedded Login con conexiones de <Tooltip tip="Sin contraseña: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Passwordless">sin contraseña</Tooltip> para varios tipos de aplicaciones:

* [Inicio de sesión integrado sin contraseña en aplicaciones nativas](/es/docs/authenticate/passwordless/implement-login/embedded-login/native)
* [Inicio de sesión integrado sin contraseña en aplicaciones web tradicionales](/es/docs/authenticate/passwordless/implement-login/embedded-login/webapps)
* [Autenticación integrada sin contraseña para aplicaciones de página única](/es/docs/authenticate/passwordless/implement-login/embedded-login/spa)
