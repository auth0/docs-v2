---
description: Aprenda a configurar Auth0 como proveedor de identidad mediante la SAML2 Web App para Google Workspace.
title: Configurar Google Workspace como proveedor de servicios SAML
---

Utilice la siguiente configuración de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin necesidad de una contraseña." cta="Ver glosario" href="/es/docs/glossary?term=SAML">SAML</Tooltip> para Google Workspace.

1. Vaya a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y cree una nueva aplicación o haga clic en el nombre de una aplicación para actualizarla.

2. Vaya a la pestaña **Addons** y habilite el interruptor **SAML2 Web App**.

3. En la pestaña **Settings**, establezca la **Application Callback URL** en: `https://www.google.com/a/{YOUR-GOOGLE-DOMAIN}/acs`.

   <Frame>![Pestaña Addons de Dashboard Applications Applications SAML2 Web App Pestaña Settings](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>

4. Pegue el siguiente código en el cuadro de texto **Settings** y haga clic en **Debug**.

   ```json lines
   {
         "audience": "https://www.google.com/a/{yourGoogleDomain}/acs",
         "mappings": {
       "nickname": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name",
         },
         "createUpnClaim": false,
         "passthroughClaimsWithNoMapping": false,
         "mapUnknownClaimsAsIs": false,
         "mapIdentities": false,
         "signatureAlgorithm": "rsa-sha256",
         "digestAlgorithm": "sha256",
         "nameIdentifierFormat": "urn:oasis:names:tc:SAML:2.0:nameid-format:email",
         "nameIdentifierProbes": [
       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress"
         ],
       }
   ```

5. Desplácese hasta la parte inferior de la página y haga clic en **Enable**.

6. En la pestaña **Usage**, ubique **Identity Provider Metadata** y haga clic en **Download** para descargar el archivo de metadatos. Lo necesitará cuando configure Auth0 como proveedor de identidad (IdP).
