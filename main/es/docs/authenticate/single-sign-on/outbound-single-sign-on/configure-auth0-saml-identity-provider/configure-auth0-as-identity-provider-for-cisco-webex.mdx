description: Aprenda a configurar Auth0 como proveedor de identidad mediante el complemento SAML2 Web App para Cisco WebEx.
title: Configurar Cisco WebEx como proveedor de servicios SAML
---

Use la siguiente configuración de <Tooltip tip="Security Assertion Markup Language (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin necesidad de una contraseña." cta="Ver glosario" href="/es/docs/glossary?term=SAML">SAML</Tooltip> para Cisco WebEx.

1. Vaya a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y cree una aplicación nueva o haga clic en el nombre de una aplicación para actualizarla.

2. Vaya a la pestaña **Addons** y active el interruptor **SAML2 Web App**.

3. En la pestaña **Settings**, configure la **Application Callback URL.** Tenga en cuenta que Webex ofrece una opción para aprovisionar automáticamente a los usuarios nuevos. Deberá enviar esa información como claims (`lastname`, entre otros). Si integra Auth0 con los servicios de Cisco Spark, consulte [Single Sign-On with Cisco Spark Services](https://collaborationhelp.cisco.com/article/en-us/lfu88u).

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/docs/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>

4. Pegue el siguiente código en el cuadro de texto **Settings** y haga clic en **Debug**.

   ```json lines
   {
          "mappings": {
        "email": ["email", "uid", "Name_ID"]
          },
         "createUpnClaim":       false,
         "passthroughClaimsWithNoMapping": false,
         "mapUnknownClaimsAsIs": false,
         "mapIdentities":        false,
         "nameIdentifierFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress",
         "nameIdentifierProbes": [
        "email",
          ]
       }
   ```

5. Desplácese hasta la parte inferior de la pestaña y haga clic en **Enable**.

6. En la pestaña **Usage**, ubique **Identity Provider Metadata** y haga clic en **Download** para descargar el archivo de metadatos. Lo necesitará cuando configure Auth0 como proveedor de identidad en Cisco WebEx.
