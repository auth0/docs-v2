Al implementar <Tooltip tip="Single Sign-On (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/es/docs/glossary?term=Single+Sign-on">inicio de sesión único</Tooltip> (SS0) y configurar sus opciones, los siguientes endpoints de la API te serán útiles.

<div id="authentication-api-endpoints">
  ## Endpoints de la Authentication API
</div>

<div id="post-logincallback">
  ### POST /login/callback
</div>

En los flujos de SSO iniciados por un <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> (IdP), el endpoint [POST /login/callback](https://auth0.com/docs/api/authentication#idp-initiated-single-sign-on-sso-flow) puede aceptar una solicitud de inicio de sesión <Tooltip tip="Lenguaje de Marcado para Confirmaciones de Seguridad (SAML): Protocolo estandarizado que permite a dos partes intercambiar información de autenticación sin contraseña." cta="Ver glosario" href="/es/docs/glossary?term=SAML">SAML</Tooltip> del proveedor de identidad.

<div id="management-api">
  ## Management API
</div>

La <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> incluye varios endpoints diseñados para ayudarle a gestionar el SSO.

Para llamar a cualquiera de los endpoints de la Management API, deberá [obtener un Token de acceso](https://auth0.com/docs/api/management/v2/tokens).

<div id="clients">
  ### Clientes
</div>

Todas sus aplicaciones cliente/integraciones de SSO contienen información relevante para su implementación de SSO. Puede recuperar o modificar esta información invocando uno de los endpoints de Clientes.

Los parámetros de interés para cada cliente incluyen:

* `sso`: Indicador aplicable solo a los clientes creados mediante integraciones de SSO. Si es `true`, Auth0 gestiona el SSO; si es `false`, el IdP gestiona el SSO.
* `sso_disabled`: Indicador utilizado para habilitar o deshabilitar el SSO. Si es `true`, el SSO está deshabilitado; si es `false`, el SSO está habilitado. Esta opción solo se puede establecer mediante la Management API.
* `app_type`: Tipo de aplicación. Si el cliente se creó mediante una de las integraciones de SSO predefinidas de Auth0, aparecerá el nombre de la integración de SSO (por ejemplo, `box` o `concur` en lugar de `native` o `spa`)

<div id="get-all-clients">
  #### Obtener todos los clientes
</div>

El endpoint [GET /api/v2/clients](https://auth0.com/docs/api/management/v2#!/Clients/get_clients) puede usarse para obtener información sobre las aplicaciones cliente que ha configurado para su inquilino.

<div id="create-a-client">
  #### Crear un cliente
</div>

El endpoint [POST /api/v2/clients](https://auth0.com/docs/api/management/v2#!/Clients/post_clients) puede utilizarse para crear una nueva aplicación cliente.

<div id="get-a-client">
  #### Obtener un cliente
</div>

El endpoint [GET /api/v2/clients/{id}](https://auth0.com/docs/api/management/v2#!/Clients/get_clients_by_id) puede usarse para obtener información sobre un cliente específico que hayas configurado para tu inquilino.

<div id="update-a-client">
  #### Actualizar un cliente
</div>

El endpoint [PATCH /api/v2/clients/{id}](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id) puede usarse para actualizar un cliente específico, incluidos los parámetros relacionados con SSO.

<div id="tenants">
  ### Inquilinos
</div>

Auth0 le permite controlar los siguientes parámetros a nivel de inquilino que pueden afectar a su implementación de SSO:

* `session_lifetime`: Período durante el cual la sesión de Auth0 del usuario seguirá siendo válida.
* `idle_session_lifetime`: Tiempo que puede transcurrir antes de que el usuario tenga que volver a iniciar sesión por inactividad.

<div id="get-tenant-settings">
  ### Obtener la configuración del inquilino
</div>

El endpoint [GET /api/v2/tenants/settings](https://auth0.com/docs/api/management/v2#!/Tenants/get_settings) obtiene la configuración de su inquilino.

<div id="update-tenant-settings">
  ### Actualizar la configuración del inquilino
</div>

El endpoint [POST /api/v2/connections](https://auth0.com/docs/api/management/v2#!/Tenants/patch_settings) permite actualizar la configuración de su inquilino.
