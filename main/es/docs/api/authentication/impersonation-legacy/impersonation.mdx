<div id="endpoint">
  ## Endpoint
</div>

`POST /users/{user_id}/impersonate`

Use este endpoint para obtener una URL de suplantación para iniciar sesión como otro usuario. Es útil para solucionar problemas.

<div id="remarks">
  ### Observaciones
</div>

* Este endpoint solo puede usarse con credenciales de **Global Client**.
* Para distinguir entre inicios de sesión reales e inicios de sesión por suplantación, el perfil del usuario suplantado contendrá propiedades adicionales `impersonated` e `impersonator`. Por ejemplo: `"impersonated": true, "impersonator": {"user_id": "auth0|...", "email": "admin@example.com"}`.
* En una aplicación web normal, debe configurar `additionalParameters`: establezca `response_type` en `code`, `callback_url` en la URL de devolución de llamada a la que Auth0 redirigirá con el código de autorización, y `scope` en los claim del JWT que quiera incluir en el JWT.
