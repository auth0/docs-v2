`GET /v2/logout`

Use este endpoint para cerrar la sesión de un usuario. Si quiere redirigir al usuario a una URL específica después de cerrar la sesión, establezca esa URL en el parámetro `returnTo`. La URL debe estar incluida en la lista correspondiente de `Allowed Logout URLs`:

* Si se incluye el parámetro `client_id`, la URL de `returnTo` debe figurar en la lista de `Allowed Logout URLs` configurada a nivel de la aplicación. Para obtener más información, consulte [Log Users Out of Applications](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-applications).
* Si NO se incluye el parámetro `client_id`, la URL de `returnTo` debe figurar en la lista de `Allowed Logout URLs` configurada a nivel del inquilino. Para obtener más información, consulte [Log Users Out of Auth0](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-auth0).
* Si se incluye el parámetro `client_id` y NO se establece la URL de `returnTo`, el servidor devuelve al usuario a la primera URL de `Allowed Logout URLs` configurada en el Dashboard. Para obtener más información, consulte [Log Users Out of Applications](https://auth0.com/docs/authenticate/login/logout/log-users-out-of-applications).

<div id="remarks">
  ### Observaciones
</div>

* Cerrar la sesión del usuario en su Proveedor de identidad no es una práctica habitual, así que tenga en cuenta la experiencia del usuario antes de usar el parámetro de cadena de consulta `federated`.
* El script de ejemplo de auth0.js usa la versión 8 de la biblioteca. Si usa la versión 7 de auth0.js, consulte esta [guía de referencia](https://auth0.com/docs/libraries/auth0js).

<div id="parameters">
  ## Parámetros
</div>

<ParamField query="client_id" type="string">
  El `client_id` de la aplicación.
</ParamField>

<ParamField query="returnTo" type="string" required>
  URL a la que se redirigirá al usuario después de cerrar sesión.
</ParamField>

<ParamField query="federated" type="string">
  Añada este parámetro de cadena de consulta a la URL de cierre de sesión para cerrar la sesión del usuario en su proveedor de identidad.
</ParamField>

<div id="response">
  ## Respuesta
</div>

| Estado | Descripción                                                             |
| ------ | ----------------------------------------------------------------------- |
| 200    | Cierre de sesión completado correctamente                               |
| 400    | Solicitud incorrecta, posiblemente por falta de parámetros obligatorios |
