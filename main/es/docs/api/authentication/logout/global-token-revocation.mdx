---
title: "Revocación global de tokens"
description: "Revoca las cookies de sesión y los tokens de actualización de un usuario en todas las aplicaciones mediante Universal Logout de Okta."
---

<div id="endpoint">
  ## Endpoint
</div>

`POST /oauth/global-token-revocation/connection/{connection_name}`

Use este endpoint junto con [Okta Workforce Identity Cloud Universal Logout](https://developer.okta.com/docs/guides/oin-universal-logout-overview/) para cerrar la sesión de los usuarios en sus aplicaciones. Una solicitud a este endpoint revoca las cookies de sesión y los tokens de actualización, pero no los tokens de acceso. Para obtener más información, consulte [Universal Logout](https://auth0.com/docs/authenticate/login/logout/universal-logout). La solicitud debe estar autenticada antes de revocar las sesiones de los usuarios. Consulte [Endpoint Authentication](https://developer.okta.com/docs/guides/oin-universal-logout-overview/#endpoint-authentication).

<div id="path-parameters">
  ## Parámetros de ruta
</div>

<ParamField path="connection_name" type="string" required>
  El nombre de la conexión cuya sesión de usuario desea revocar.
</ParamField>

<div id="body-parameters">
  ## Parámetros del cuerpo
</div>

<ParamField body="subject" type="object">
  <Expandable title="propiedades">
    <ParamField body="format" type="string" required />

    <ParamField body="iss" type="string" required>
      La URL del emisor (normalmente, la URL de tu Servidor de autorización).
    </ParamField>

    <ParamField body="sub" type="string" required>
      El identificador del sujeto (normalmente, el identificador del usuario).
    </ParamField>
  </Expandable>
</ParamField>

<div id="response-messages">
  ## Mensajes de respuesta
</div>

| Status | Description                                                                                                     |
| ------ | --------------------------------------------------------------------------------------------------------------- |
| 200    | Se revocaron correctamente las cookies de sesión y los tokens de actualización.                                 |
| 400    | Solicitud incorrecta, posiblemente debido a un formato de `subject` no válido o a la ausencia de autenticación. |
| 401    | No autorizado; la autenticación falló.                                                                          |
| 500    | Error interno del servidor.                                                                                     |
