`GET /authorize`

Después de llamar al endpoint `/oauth/par`, redirija al usuario final al endpoint `/authorize` mediante una solicitud `GET`.

<Note>
  El endpoint `/authorize` responderá en función de los parámetros enviados al endpoint `/oauth/par`. Si solicita un `response_type`, debería recibir un código de autorización para usar en el endpoint `/oauth/token`.
</Note>

<div id="parameters">
  ## Parámetros
</div>

<ParamField query="client_id" type="string" required>
  El `client_id` de la aplicación. Obligatorio.
</ParamField>

<ParamField query="request_uri" type="string" required>
  El valor de `request_uri` recibido del endpoint `/oauth/par`. Obligatorio.
</ParamField>

<div id="response">
  ## Respuesta
</div>

| Estado | Descripción                                    |
| ------ | ---------------------------------------------- |
| 302    | Redirige a la URI de redirección especificada. |
