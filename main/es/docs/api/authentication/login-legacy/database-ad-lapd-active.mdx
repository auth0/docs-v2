`POST /oauth/ro`

<Note>
  Este endpoint forma parte del flujo de autenticación legado y fue reemplazado por [Password Grant](#resource-owner-password). Para obtener más información sobre el flujo de autenticación más reciente, consulta [Introducción a la autenticación OIDC Conformant](/es/api-auth/intro).
</Note>

Usa este endpoint para la autenticación basada en API (activa). Dadas las credenciales del usuario y la `connection` especificada, realizará la autenticación con el proveedor y devolverá un JSON con el Token de acceso y el ID Token.

<div id="remarks">
  ### Observaciones
</div>

* Este endpoint solo funciona para conexiones de base de datos, conexiones sin contraseña, Active Directory/LDAP, Windows Azure AD y ADFS.

* La principal diferencia entre la autenticación pasiva y la activa es que la primera ocurre en el navegador a través de la [página de Login de Auth0](http://manage.auth0.com/login), mientras que la segunda puede invocarse desde cualquier lugar (un script, de servidor a servidor, etc.).

* El script de ejemplo de auth0.js usa la versión 8 de la biblioteca. Si usa la versión 7 de auth0.js, consulte esta [guía de referencia](https://auth0.com/docs/libraries/auth0js/v7).

<div id="parameters">
  ## Parámetros
</div>

<ParamField body="client_id" type="string" required>
  El `client_id` de tu aplicación.
</ParamField>

<ParamField body="username" type="string" required>
  El username o correo electrónico del usuario para iniciar sesión.
</ParamField>

<ParamField body="password" type="string" required>
  La contraseña del usuario para iniciar sesión.
</ParamField>

<ParamField body="connection" type="string" required>
  El nombre de la conexión que se usará para iniciar sesión.
</ParamField>

<ParamField body="scope" type="string">
  Establécelo en `openid` para obtener también un ID Token.
</ParamField>

<ParamField body="grant_type" type="string" required>
  Establécelo en `password` o `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>

<ParamField body="device" type="string">
  Obligatorio cuando `grant_type` es `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>

<ParamField body="id_token" type="string">
  Obligatorio cuando `grant_type` es `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>
