`POST /oauth/token`

Use este endpoint para renovar un Token de acceso con el Token de actualización que obtuvo durante la autorización.

<div id="learn-more">
  ## Más información
</div>

* [Tokens de actualización](https://auth0.com/docs/secure/tokens/refresh-tokens)

<div id="parameters">
  ## Parámetros
</div>

<ParamField header="DPoP" type="string">
  Una prueba de DPoP para la solicitud. Es opcional y solo se requiere si la aplicación usa Demonstrating Proof-of-Possession.
</ParamField>

<div id="request-body">
  ## Cuerpo de la solicitud
</div>

<ParamField body="grant_type" type="string" required>
  Especifica el flujo que se está utilizando. Para renovar un token de acceso, debe establecerse en `refresh_token`.

  Valores permitidos: `refresh_token`
</ParamField>

<ParamField body="client_id" type="string" required>
  El ID de cliente de su aplicación, que identifica la aplicación que realiza la solicitud.
</ParamField>

<ParamField body="client_secret" type="string">
  El Secreto del cliente de su aplicación. Es obligatorio cuando el método de autenticación del endpoint de token está configurado como `Post` o `Basic`.
</ParamField>

<ParamField body="refresh_token" type="string" required>
  El token de actualización emitido previamente al cliente. Este token se utiliza para obtener un nuevo token de acceso.
</ParamField>

<ParamField body="scope" type="string">
  Una lista de permisos de scope solicitados, separada por espacios. Si se omite, se usarán los alcances originales; de lo contrario, puede solicitar un conjunto reducido de alcances. Tenga en cuenta que debe codificarse para URL.
</ParamField>

<div id="response">
  ## Respuesta
</div>

| Estado | Descripción                                                                                          |
| ------ | ---------------------------------------------------------------------------------------------------- |
| 200    | El token de acceso se actualizó correctamente.                                                       |
| 400    | Solicitud incorrecta. Faltan parámetros obligatorios en la solicitud o su formato es incorrecto.     |
| 401    | No autorizado. Las credenciales del cliente no son válidas o el token de actualización no es válido. |
| 403    | Prohibido. El token de actualización no es válido o ha expirado.                                     |
