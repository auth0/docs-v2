`GET /wsfed/{clientId}`

Este endpoint acepta una solicitud de WS-Federation para iniciar sesión.

<div id="remarks">
  ### Observaciones
</div>

* El parámetro `wtrealm` debe tener uno de estos formatos:
  * `urn:clientID` (por ejemplo, urn:${account.clientId})
  * Si este parámetro no comienza con una urn, se usa el array `client.clientAliases` para la búsqueda. Esto solo se puede configurar con la Management API en [/api/v2/clients](https://auth0.com/docs/api/management/v2/clients/get-clients).
* El parámetro `whr` se asigna a la conexión de la siguiente manera: `urn:CONNECTION_NAME`. Por ejemplo, `urn:google-oauth2` indica el inicio de sesión con Google. Si no se incluye el parámetro `whr`, el usuario será redirigido a la [página de inicio de sesión de Auth0](https://auth0.com/docs/login_page).

<div id="parameters">
  ## Parámetros
</div>

<ParamField path="client-id" type="string" required>
  El `client-id` de su aplicación.
</ParamField>

<ParamField query="wtrealm" type="string">
  Puede usarse en lugar de `client-id`.
</ParamField>

<ParamField query="whr" type="string">
  El nombre de la conexión (se usa para saltar la página de inicio de sesión).
</ParamField>

<ParamField query="wctx" type="string">
  El estado de su aplicación.
</ParamField>

<ParamField query="wreply" type="string">
  La URL de devolución de llamada.
</ParamField>

  ## Respuesta
