`POST /oauth/ro`

Con las credenciales del usuario, este endpoint autenticará al usuario ante el proveedor y devolverá un objeto JSON con el Token de acceso y un ID Token.

<Note>
  Este endpoint forma parte del flujo de autenticación heredado y ha sido sustituido por la [Concesión de contraseña](#resource-owner-password). Para obtener más información sobre el flujo de autenticación más reciente, consulta [Introducción a la autenticación compatible con OIDC](/es/api-auth/intro).
</Note>

<div id="remarks">
  ### Observaciones
</div>

* Este endpoint solo funciona con conexiones de base de datos, conexiones sin contraseña, Active Directory/LDAP, Windows Azure AD y ADFS.
* El scope `profile` solicita acceso a las claims predeterminadas del perfil del usuario final, que incluyen `name`, `family_name`, `given_name` y otras.
* El scope `email` solicita acceso a las claims `email` y `email_verified`.
