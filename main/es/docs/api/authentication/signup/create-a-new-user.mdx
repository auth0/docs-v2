`POST /dbconnections/signup`

Dadas las credenciales de un usuario y una `connection`, este endpoint crea un nuevo usuario.

Este endpoint solo funciona con conexiones de base de datos.

<div id="remarks">
  ### Observaciones
</div>

* El script de ejemplo de auth0.js usa la versión 8 de la biblioteca. Si usa la versión 7 de auth0.js, consulte esta [guía de referencia](https://auth0.com/docs/libraries/auth0js/v7).

<div id="parameters">
  ## Parámetros
</div>

<ParamField body="client_id" type="string" required>
  El `client_id` de tu cliente.
</ParamField>

<ParamField body="email" type="string">
  La dirección de correo electrónico del usuario.
</ParamField>

<ParamField body="password" type="string" required>
  La contraseña deseada del usuario.
</ParamField>

<ParamField body="phone_number" type="string">
  El número de teléfono del usuario.
</ParamField>

<ParamField body="connection" type="string" required>
  El nombre de la base de datos configurada para tu cliente.
</ParamField>

<ParamField body="username" type="string">
  El username del usuario. Solo es válido si la conexión requiere un username.
</ParamField>

<ParamField body="given_name" type="string">
  El nombre o los nombres del usuario.
</ParamField>

<ParamField body="family_name" type="string">
  El apellido o los apellidos del usuario.
</ParamField>

<ParamField body="name" type="string">
  El nombre completo del usuario.
</ParamField>

<ParamField body="nickname" type="string">
  El apodo del usuario.
</ParamField>

<ParamField body="picture" type="string">
  Una URI que apunta a la imagen del usuario.
</ParamField>

<ParamField body="user_metadata" type="object">
  Los metadatos de usuario que se asociarán con el usuario. Si se establece, el campo debe ser un objeto que no contenga más de diez propiedades. Los nombres de las propiedades pueden tener un máximo de 100 caracteres y los valores de las propiedades deben ser cadenas de no más de 500 caracteres.
</ParamField>

<div id="response">
  ## Respuesta
</div>

| Estado | Descripción                                               |
| ------ | --------------------------------------------------------- |
| 200    | Usuario creado correctamente                              |
| 400    | Solicitud incorrecta; faltan parámetros o no son válidos. |
| 409    | Conflicto; el usuario ya existe.                          |
