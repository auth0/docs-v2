Se admiten tres métodos para autenticar el Auth0 Deploy CLI con su inquilino.

<div id="client-credentials">
  ## Credenciales de cliente
</div>

La autenticación mediante [credenciales de cliente](/es/docs/get-started/authentication-and-authorization-flow/client-credentials-flow) requiere que proporcione directamente a la aplicación un par de <Tooltip tip="ID de cliente: valor de identificación que Auth0 asigna a su recurso registrado." cta="Ver glosario" href="/es/docs/glossary?term=client+ID">ID de cliente</Tooltip> y <Tooltip tip="ID de cliente: valor de identificación que Auth0 asigna a su recurso registrado." cta="Ver glosario" href="/es/docs/glossary?term=client+secret">Secreto del cliente</Tooltip>. Esta opción es sencilla y ofrece la forma más rápida de configurar la herramienta.

Para configurar la autenticación mediante credenciales de cliente, establezca los siguientes campos en su [archivo de configuración](/es/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli):

| Campo                 | Valor                                          |
| --------------------- | ---------------------------------------------- |
| `AUTH0_CLIENT_ID`     | ID de cliente de su aplicación de Auth0.       |
| `AUTH0_CLIENT_SECRET` | Secreto del cliente de su aplicación de Auth0. |

<div id="private-key-jwt">
  ## JWT con clave privada
</div>

La autenticación [JWT con clave privada](/es/docs/get-started/applications/configure-private-key-jwt) requiere que configure una clave privada para la aplicación y una clave pública para el inquilino remoto. Esta opción es ideal si no desea almacenar credenciales en Auth0.

Para configurar la autenticación con clave privada <Tooltip tip="JSON Web Token (JWT): formato estándar de ID Token (y, a menudo, también de Token de acceso) utilizado para representar declaraciones de forma segura entre dos partes." cta="Ver glosario" href="/es/docs/glossary?term=JWT">JWT</Tooltip>, establezca los siguientes campos en su [archivo de configuración](/es/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli):

| Campo                            | Valor                                                               |
| -------------------------------- | ------------------------------------------------------------------- |
| `AUTH0_CLIENT_SIGNING_KEY_PATH`  | Ruta relativa al certificado de la clave de firma del JWT.          |
| `AUTH0_CLIENT_SIGNING_ALGORITHM` | Opcional. Algoritmo de firma del JWT utilizado para el certificado. |

<div id="access-token">
  ## Token de acceso
</div>

La autenticación con [token de acceso](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) requiere que proporciones directamente a la aplicación un <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip> de Auth0. Esta opción traslada más responsabilidad a los desarrolladores, pero puede permitir flujos de trabajo flexibles y específicos cuando sea necesario.

Para configurar la autenticación con token de acceso, pasa un token de acceso de Auth0 para la Management API mediante la variable de entorno [`AUTH0_ACCESS_TOKEN`](/es/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).
