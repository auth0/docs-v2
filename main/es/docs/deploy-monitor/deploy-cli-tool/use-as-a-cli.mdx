---
description: Aprenda a usar Auth0 Deploy CLI como una utilidad de línea de comandos independiente.
title: Usar como CLI
---

Deploy CLI se puede usar como una utilidad de línea de comandos independiente. Esto proporciona una forma sencilla de administrar la configuración de su inquilino de Auth0 en flujos de trabajo de CI/CD.

Para obtener una lista completa de opciones de configuración, lea [Configure the Deploy CLI](/es/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="export-command">
  ## comando export
</div>

Obtiene las configuraciones del inquilino de Auth0 en la máquina local.

<div id="output_folder-o">
  ### `--output_folder`, `-o`
</div>

Ruta. Especifica el directorio de destino donde se escribirán los archivos de configuración.

<div id="config_file-c">
  ### `--config_file`, `-c`
</div>

Ruta de acceso. Especifica el archivo de configuración definido por el usuario (`config.json`). Para obtener más información, consulta [Configure the Deploy CLI](/es/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="format-f">
  ### `--format`, `-f`
</div>

Opciones: `yaml` o `directory`. Determina el formato de los archivos de configuración de recursos exportados. Para obtener más información, consulte [Formatos de configuración de recursos disponibles](/es/docs/deploy-monitor/deploy-cli-tool/available-resource-configuration-formats).

<div id="export_ids-e">
  ### `--export_ids`, `-e`
</div>

Booleano. Cuando se habilita, exporta los campos de identificador de cada recurso. Valor predeterminado: `false`.

<div id="env">
  ### `--env`
</div>

Booleano. Indica si la herramienta debe leer las variables de entorno o no. Valor predeterminado: `true`.

<div id="debug">
  ### `--debug`
</div>

Booleano. Habilita un registro de errores más detallado; resulta útil para solucionar problemas. Valor predeterminado: `false`.

<div id="proxy_url-p">
  ### `--proxy_url`, `-p`
</div>

Una URL para canalizar las solicitudes a través de un proxy. Establécela solo si estás detrás de un proxy.

<div id="examples">
  ### Ejemplos
</div>

<div id="fetching-auth0-tenant-configuration-in-the-yaml-format">
  #### Obtención de la configuración del inquilino de Auth0 en formato YAML
</div>

`a0deploy export -c=config.json --format=yaml --output_folder=local`

<div id="fetching-auth0-tenant-configuration-in-directory-json-format">
  #### Obtener la configuración del inquilino de Auth0 en formato de directorio (JSON)
</div>

`a0deploy export -c=config.json --format=directory --output_folder=local`

<div id="fetching-auth0-tenant-configurations-with-ids-of-all-assets">
  #### Obtención de la configuración del inquilino de Auth0 con los ID de todos los recursos
</div>

`a0deploy export -c=config.json --format=yaml --output_folder=local --export_ids=true`

<div id="import-command">
  ## comando import
</div>

Aplica la configuración de la máquina local al inquilino de Auth0.

<div id="input_file-i">
  ### `--input_file`, `-i`
</div>

Ruta. Especifica la ubicación de los archivos de configuración de recursos. En los formatos YAML, apuntará al archivo `tenant.yaml`; en los formatos de directorio, al directorio de configuración de recursos.

<div id="config_file-c">
  ### `--config_file`, `-c`
</div>

Ruta. Especifica el archivo de configuración (`config.json`) definido por el usuario. Para obtener más información, consulta [Configurar el Deploy CLI](/es/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="env">
  ### `--env`
</div>

Booleano. Indica si la herramienta debe usar variables de entorno o no. Valor predeterminado: `true`.

<div id="proxy_url-p">
  ### `--proxy_url`, `-p`
</div>

Una URL para enrutar las solicitudes a través de un proxy. Configúrala solo si estás detrás de un proxy.

<div id="debug">
  ### `--debug`
</div>

Booleano. Habilita un registro de errores más detallado; útil para la resolución de problemas. Valor predeterminado: `false`.

<div id="examples">
  ### Ejemplos
</div>

<div id="deploying-configuration-for-yaml-formats">
  #### Despliegue de la configuración para formatos YAML
</div>

`a0deploy import -c=config.json --input_file=local/tenant.yaml`

<div id="deploying-configuration-for-directory-format">
  #### Despliegue de la configuración en formato de directorio
</div>

`a0deploy import -c=config.json --input_file=local`

<div id="deploying-configuration-with-environment-variables-ignored">
  #### Desplegar la configuración ignorando las variables de entorno
</div>

`a0deploy import -c=config.json --input_file=local/tenant.yaml --env=false`
