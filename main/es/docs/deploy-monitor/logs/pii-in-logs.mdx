description: Describe la posible información de identificación personal (PII) que Auth0 escribe en los registros del inquilino.
title: Información de identificación personal en los Logs de Auth0
---

Muchas entradas de los registros del inquilino de Auth0 pueden contener información de identificación personal (PII). Los eventos de autenticación desencadenan entradas de registro que pueden incluir la PII de los usuarios. Si usa la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de Auth0 o el Dashboard para agregar usuarios, Auth0 registra los detalles de la cuenta del usuario.

Estos campos de las entradas de registro pueden contener PII:

* Nombre
* Número de teléfono (utilizado para la <Tooltip tip="Autenticación multifactor (MFA): Proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip>)
* Dirección de correo electrónico
* Dirección IP (que puede revelar la ubicación)
* Cualquier información personalizada del usuario que defina

Auth0 no registra en ningún momento <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+tokens">tokens de acceso</Tooltip> de Auth0 ni de ningún <Tooltip tip="Token de acceso: Credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=identity+provider">proveedor de identidad</Tooltip>. Cuando se produce el intercambio de un código de autorización, los logs solo muestran una parte del código (por ejemplo: `code: 31XXXXX`).
