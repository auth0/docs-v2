description: Describe los atributos y valores de las pantallas del usuario en los registros del inquilino de Auth0.
title: Detalles de la pantalla en los registros del inquilino
---

Los registros del inquilino de Auth0 proporcionan detalles de la transacción para las acciones de autenticación del usuario. Estos registros capturan información de los procesos de inicio de sesión, login y verificación para su revisión. Puede revisar cómo se autenticó el usuario, qué credenciales proporcionó, la duración de la transacción y el estado de la conexión.

A continuación se muestran los valores predeterminados que se capturan en los registros del inquilino:

| Atributo      | Descripción                                                                     |
| ------------- | ------------------------------------------------------------------------------- |
| `completedAt` | Hora de finalización de la etapa, incluido el tiempo de interacción del usuario |
| `elapsedTime` | Diferencia entre `completedAt` y `initiatedAt`                                  |
| `flow`        | Varía según la experiencia de login (es decir, Universal o Classic)             |
| `initiatedAt` | Hora de inicio de la etapa                                                      |
| `name`        | Nombre de la etapa                                                              |

<div id="login">
  ## login
</div>

La etapa de login proporciona los siguientes valores:

| Atributo    | Descripción                                                           |
| ----------- | --------------------------------------------------------------------- |
| `flow`      | Puede tomar dos valores, según se use Universal Login o Classic Login |
| `user_id`   | ID de Auth0 del usuario autenticado en esta etapa                     |
| `user_name` | Nombre del usuario autenticado en esta etapa                          |

Campos adicionales con la siguiente información:

| Atributo        | Descripción                                                    |
| --------------- | -------------------------------------------------------------- |
| `connection`    | Conexión utilizada para autenticar al usuario                  |
| `connection_id` | ID de la conexión utilizada para autenticar al usuario         |
| `strategy`      | Estrategia de la conexión utilizada para autenticar al usuario |
| `identity`      | Identidad autenticada del usuario                              |

Valores para eventos adicionales en el flujo de login:

| Nombre                         | Experiencia de Universal Login | Descripción                                                                       |
| ------------------------------ | ------------------------------ | --------------------------------------------------------------------------------- |
| `federated-authenticate`       | Universal                      | Conexión federada utilizada para la autenticación: social, legal o empresarial    |
| `lock-password-authenticate`   | Classic                        | Usuario autenticado con nombre de usuario y contraseña                            |
| `oauth1-authenticate`          | Classic                        | Usuario autenticado con una federación OAuth 1.0                                  |
| `oauth2-authenticate`          | Classic                        | Usuario autenticado con una federación OAuth 2.0                                  |
| `oidc-authenticate`            | Classic                        | Usuario autenticado con una federación OIDC                                       |
| `prompt-authenticate`          | Universal                      | Usuario autenticado con nombre de usuario y contraseña                            |
| `prompt-authenticate-password` | Universal                      | Usuario autenticado con nombre de usuario y contraseña en el flujo `idfirst`      |
| `prompt-signup`                | Universal                      | Usuario registrado con nombre de usuario y contraseña                             |
| `prompt-signup-password`       | Universal                      | Usuario registrado con nombre de usuario y contraseña mediante el flujo `idfirst` |
| `saml-authenticate`            | Classic                        | Usuario autenticado con una federación SAML                                       |
| `wsfed-authenticate`           | Classic                        | Usuario autenticado con una federación WS-FED                                     |

<div id="consent">
  ## consent
</div>

La etapa de consent proporciona los siguientes valores:

| Atributo    | Descripción                                                                                |
| ----------- | ------------------------------------------------------------------------------------------ |
| `grantInfo` | `{"{myApiIdentifier}"}` o el identificador de la API para la que el usuario otorgó permiso |
| `audience`  | Audiencia del token                                                                        |
| `id`        | Autorización del usuario persistida en la base de datos                                    |
| `scope`     | Scope aprobado por el usuario                                                              |

<div id="login-email-verification">
  ## login-email-verification
</div>

La pantalla de verificación de correo electrónico proporciona los siguientes valores:

| Atributo                    | Descripción                                                                 |
| --------------------------- | --------------------------------------------------------------------------- |
| ` login-email-verification` | Se incluye si se presentó la pantalla de verificación de correo electrónico |

<div id="redirect">
  ## redirección
</div>

La etapa de redirección proporciona los siguientes valores:

| Atributo | Descripción                        |
| -------- | ---------------------------------- |
| `URL`    | URL a la que se redirió al usuario |

<div id="mfa">
  ## mfa
</div>

<Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación del usuario que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=Multi-factor+authentication">Autenticación multifactor</Tooltip> proporciona los siguientes valores:

| Atributo        | Descripción                                                                                                                            |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| `flow`          | Puede tomar dos valores, dependiendo de si se usa con la experiencia Classic (`mfa`) o con la experiencia Universal (`universal-mfa`). |
| `performed_acr` | `acr` que se incluirá en el `id_token`.                                                                                                |
| `performed_amr` | `amr` que se incluirá en el `id_token`.                                                                                                |
| `provider`      | Proveedor utilizado para realizar MFA.                                                                                                 |
