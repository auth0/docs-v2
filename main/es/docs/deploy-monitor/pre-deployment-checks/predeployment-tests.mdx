title: Ejecutar pruebas previas a la implementación
---

Debe ejecutar pruebas unitarias y de integración antes de implementar Auth0 en una aplicación o servicio en producción. Realizar pruebas en las API de Auth0 puede hacer que su cuenta quede sujeta a [límites de tasa](/es/docs/troubleshoot/customer-support/operational-policies/rate-limit-policy), por lo que recomendamos crear API de Auth0 simuladas durante las pruebas. Según su entorno de desarrollo, sus herramientas de prueba también pueden ofrecer funciones para simular API. También hay numerosas herramientas disponibles para simular API, como [MockServer](https://www.mock-server.com/) o [JSON Server](https://github.com/typicode/json-server), que puede usar para crear rápidamente API falsas para las pruebas.

Si tiene una suscripción Enterprise, puede solicitar [pruebas de carga](/es/docs/troubleshoot/customer-support/operational-policies/load-testing-policy) en Auth0.

<div id="performance-tests">
  ## Pruebas de rendimiento
</div>

Si encuentra problemas al realizar pruebas de rendimiento, utilice los siguientes métodos para solucionarlos.

<div id="auth0-dashboard">
  ### Auth0 Dashboard
</div>

La sección [Monitoring &gt; Logs](https://manage.auth0.com/#/logs) del Dashboard almacena datos sobre:

* Acciones realizadas por administradores en el Dashboard
* Autenticaciones realizadas por sus usuarios

También hay [extensiones](/es/docs/customize/extensions) que puede usar para fines de registro, incluida la exportación de registros a herramientas de terceros y la recopilación de información sobre el uso de código personalizado en su cuenta.

<div id="third-party-test-tools">
  ### Herramientas de prueba de terceros
</div>

Hay varias herramientas de prueba de terceros que puede usar para realizar pruebas de rendimiento en API RESTful. Estas son algunas opciones que podría considerar (tenga en cuenta que Auth0 no respalda ningún producto o herramienta en particular):

* [Apache JMeter](https://jmeter.apache.org/)
* [Artillery](https://www.artillery.io/)
* [Micro Focus LoadRunner](https://www.radview.com/)
* [Loader](https://loader.io/)
* [RadView Webload](https://www.radview.com/)
* [SmartBear LoadUI](https://smartbear.com/)
* [Vegeta](https://github.com/tsenart/vegeta)
* [Wrk](https://github.com/wg/wrk)

Estas herramientas proporcionan registros de actividad que le ayudan a identificar cualquier problema. Si necesita ayuda para interpretar su registro o identificar el posible problema, póngase en contacto con Soporte.

<div id="http-archive-files">
  ### Archivos HAR
</div>

Si detecta un problema que puede reproducir, le recomendamos que [cree un archivo de captura HTTP (HAR)](https://support.auth0.com/center/s/article/generate-and-analyze-har-files-to-troubleshoot-auth0-issues) y lo envíe a nuestro equipo de Soporte para obtener asistencia adicional.
