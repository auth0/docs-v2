---
description: Correcciones que debe realizar en su aplicación de Auth0 antes de implementar en producción
title: 'Comprobaciones de preparación para producción: correcciones críticas'
---

Las siguientes comprobaciones verifican si ha completado todos los requisitos para implementar correctamente en producción.

Para obtener más información sobre los resultados de las comprobaciones, consulte [Ejecutar comprobaciones de producción](/es/docs/deploy-monitor/pre-deployment-checks/how-to-run-production-checks).

| Verificación                                                                                                                                                                                          | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Todas las Actions se ejecutan en una [versión recomendada de NodeJS](/es/docs/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18)                                             | Configure todas las [Actions](/es/docs/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) para que usen una versión recomendada de NodeJS y así aprovechar los parches de seguridad y las nuevas funciones.                                                                                                                                                                                                                                                                                          |
| [Permitir ID Tokens para la autenticación de Management API v2](/es/docs/troubleshoot/product-lifecycle/past-migrations/migrate-to-calling-api-with-access-tokens) está deshabilitada                    | Se está dejando obsoleta la capacidad de usar ID Token para autorizar algunos de los endpoints de Users y Device Credentials de la Management API. Después de completar la migración a Access Tokens, asegúrese de que el selector [Permitir ID Token para la autenticación de Management API v2](https://manage.auth0.com/#/tenant/advanced) esté desactivado. Si no ve esta configuración, su inquilino se creó después de que esta funcionalidad quedara obsoleta, por lo que ya está desactivada de forma predeterminada. |
| allowLegacyRoGrantTypes                                                                                                                                                                               | El endpoint [oauth/ro legado](/es/docs/troubleshoot/product-lifecycle/past-migrations/migration-oauthro-oauthtoken) de la configuración de migración del inquilino está quedando obsoleto. Después de actualizar sus aplicaciones para usar endpoints compatibles, asegúrese de desactivar este endpoint.                                                                                                                                                                                                                        |
| allowOtherLegacyGrantTypes                                                                                                                                                                            | El [endpoint legado oauth/access&#95;token](https://auth0.com/docs/api/authentication/login-legacy/social-with-providers-access-token) de la configuración de migración del inquilino se está retirando. Después de actualizar sus aplicaciones para que usen endpoints compatibles, asegúrese de desactivar este endpoint.                                                                                                                                                                                                   |
| allowLegacyTokenInfoEndpoint                                                                                                                                                                          | El [endpoint tokeninfo legado](/es/docs/troubleshoot/product-lifecycle/past-migrations#tokeninfo-endpoint-validation) en la configuración de migración del inquilino se va a retirar. Después de actualizar sus aplicaciones para usar endpoints compatibles, asegúrese de desactivar este endpoint.                                                                                                                                                                                                                             |
| allowLegacyDelegationGrantTypes                                                                                                                                                                       | El [endpoint de delegación](/es/docs/authenticate/login/oidc-conformant-authentication/oidc-adoption-delegation) legado de la configuración de migración del inquilino se está retirando. Después de actualizar sus aplicaciones para usar endpoints compatibles, asegúrese de desactivar este endpoint.                                                                                                                                                                                                                         |
| [Allowed Callback URLs](/es/docs/authenticate/login/redirect-users-after-login) no debe incluir localhost                                                                                                | Valida que las [Allowed Callback URLs de la aplicación](https://manage.auth0.com/#/applications) no apunten a `localhost`, `127.0.0.1` ni similares.                                                                                                                                                                                                                                                                                                                                                                          |
| [Orígenes permitidos (CORS)](/es/docs/authenticate/login/cross-origin-authentication) no debe ser localhost                                                                                              | Valida que la [URL de ubicación](https://manage.auth0.com/#/applications) de la página no apunte a `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                               |
| [Allowed Web Origins](/es/docs/get-started/applications/application-settings) no debe incluir `localhost`                                                                                                | Valida que las [URL de Allowed Web Origins](https://manage.auth0.com/#/applications) no apunten a `localhost`.                                                                                                                                                                                                                                                                                                                                                                                                                |
| [Proveedor de correo electrónico](/es/docs/customize/email/smtp-email-providers) debe estar configurado                                                                                                  | Comprueba que se haya configurado el [proveedor de correo electrónico personalizado](https://manage.auth0.com/#/emails/provider).                                                                                                                                                                                                                                                                                                                                                                                             |
| Hooks están en desuso y deben [migrarse a Actions](/es/docs/customize/actions/migrate/migrate-from-hooks-to-actions)                                                                                     | Hooks quedarán obsoletos. Recomendamos encarecidamente que empiece a usar [Actions](/es/docs/customize/actions/migrate/migrate-from-hooks-to-actions) e inicie el proceso de migración. Los Hooks existentes seguirán activos hasta el 18 de noviembre de 2024.                                                                                                                                                                                                                                                                  |
| [Autenticación multifactor](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) debe estar configurada (Dependencia: MFA configurado) | Si utiliza MFA con el [factor Phone Message](https://manage.auth0.com/#/mfa), configúrelo con Twilio o con un proveedor de envío personalizado para evitar los límites de frecuencia del proveedor de envío predeterminado.                                                                                                                                                                                                                                                                                                   |
| [Perfil de usuario legado](/es/docs/troubleshoot/product-lifecycle/past-migrations/migrate-from-legacy-auth-flows#user-profiles) debe desactivarse                                                       | Los flujos de autenticación legados que permiten que los ID Token y el endpoint `/userinfo` incluyan el perfil de usuario completo están quedando obsoletos. Después de completar la migración a las nuevas API conformes a OIDC, asegúrese de que el interruptor [Perfil de usuario legado](https://manage.auth0.com/#/account/advanced) esté desactivado.                                                                                                                                                                   |
| Rules quedará obsoleto y debe [migrarse a Actions](/es/docs/customize/actions/migrate/migrate-from-rules-to-actions)                                                                                     | Rules quedarán obsoletas. Recomendamos encarecidamente que empiece a usar [Actions](/es/docs/customize/actions/migrate/migrate-from-rules-to-actions) e inicie el proceso de migración. Las Rules existentes seguirán activas hasta el 18 de noviembre de 2024.                                                                                                                                                                                                                                                                  |
| [Conexiones sociales](/es/docs/authenticate/identity-providers/social-identity-providers/devkeys) no deben usar las claves de desarrollador de Auth0                                                     | Comprueba que las [Conexiones sociales](https://manage.auth0.com/#/connections/social) no utilicen las claves de desarrollador predeterminadas de Auth0.                                                                                                                                                                                                                                                                                                                                                                      |
| El correo electrónico de soporte debe estar configurado                                                                                                                                               | Asegura que el [correo electrónico de soporte](https://manage.auth0.com/#/account) esté configurado en Tenant Settings.                                                                                                                                                                                                                                                                                                                                                                                                       |
| La URL de soporte está configurada                                                                                                                                                                    | Asegúrese de que la [Support URL](https://manage.auth0.com/#/account) esté configurada en Tenant Settings.                                                                                                                                                                                                                                                                                                                                                                                                                    |
| La etiqueta de entorno del Tenant debe estar configurada                                                                                                                                              | Garantiza que la [etiqueta de entorno del inquilino](https://support.auth0.com/tenants/public) esté configurada correctamente como Production, Staging o Development. Los cambios en esta etiqueta afectan al límite de tasa de su inquilino.                                                                                                                                                                                                                                                                                 |
| Tenant está configurado para usar la [versión predeterminada recomendada de NodeJS](/es/docs/troubleshoot/product-lifecycle/deprecations-and-migrations/migrate-nodejs-22)                               | Actualiza tu inquilino para usar una [versión recomendada de NodeJS](/es/docs/troubleshoot/product-lifecycle/past-migrations/migrate-nodejs-16-to-nodejs-18) y aprovechar los parches de seguridad y las nuevas funciones.                                                                                                                                                                                                                                                                                                       |
| URI de Login del Tenant                                                                                                                                                                               | Asegúrate de que [Tenant Login URI](https://manage.auth0.com/#/tenant/advanced) esté configurado en Tenant Settings.                                                                                                                                                                                                                                                                                                                                                                                                          |
| Use un dominio personalizado en las plantillas de correo electrónico con marca                                                                                                                        | Asegúrese de que sus plantillas de correo electrónico usen un [dominio personalizado](/es/docs/customize/custom-domains/configure-features-to-use-custom-domains#use-custom-domains-in-emails). Recomendamos encarecidamente personalizar todos los correos electrónicos dirigidos a los usuarios para que usen su dominio personalizado y la marca de su empresa.                                                                                                                                                               |
