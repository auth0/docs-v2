La opción de implementación de Private Cloud incluye dos funciones complementarias adicionales: geo-failover y cumplimiento de PCI. Si tiene alguna pregunta sobre estas funciones, [póngase en contacto con el equipo de ventas de Auth0](https://auth0.com/get-started?place=documentation%20post\&type=link\&text=contact%20auth0%20sales).

<div id="private-cloud-geo-failover">
  ## Geo-failover de Private Cloud
</div>

El geo-failover es una configuración de conmutación por error activo-pasivo que permite a los clientes especificar una región secundaria de failover para usar en caso de una interrupción regional importante. En general, se puede seleccionar como región de failover cualquier región del mismo proveedor de nube que tenga una latencia de hasta 100 ms. Sin embargo, en algunos casos, los proveedores de nube prefieren regiones de failover específicas. El equipo de onboarding de Auth0 trabajará con usted para seleccionar las regiones más adecuadas para su caso de uso.

Además, se requiere una tercera región de &quot;desempate&quot; para uno de nuestros almacenes de datos. Esta tercera región nunca recibe tráfico en producción, pero es necesaria para elegir un nuevo líder en caso de failover.

Si se detecta una falla regional, Auth0 activa el geo-failover en nombre del cliente. Una vez activado, todo el proceso es automatizado. Cuando se confirma que la región ya no presenta problemas, Auth0 realizará la conmutación de retorno a la región primaria.

  ## Cumplimiento de PCI
</div>

El cumplimiento de PCI está disponible como complemento para los clientes que lo necesiten.
