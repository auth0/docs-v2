title: Supervisar Auth0 con System Center Operations Manager
---

Puede supervisar Auth0 como una aplicación web estándar mediante System Center Operations Manager (SCOM) o cualquier herramienta compatible con transacciones sintéticas. Recomendamos agregar una transacción sintética de inicio de sesión que incluya las extensiones de las que dependen sus aplicaciones (como Rules que ejecutan código personalizado para integrarse con otros servicios de su empresa).

<div id="set-up-scom">
  ## Configurar SCOM
</div>

1. Agregue una nueva instancia de SCOM mediante el **Asistente para agregar Monitoring**:

| Campo                                              | Descripción                                             |
| -------------------------------------------------- | ------------------------------------------------------- |
| Nombre                                             | Nombre descriptivo de la instancia de SCOM.             |
| Descripción                                        | Descripción de lo que supervisa esta instancia de SCOM. |
| Seleccione el paquete de administración de destino | Paquete de administración predeterminado                |

Cuando termine, haga clic en **Next** para continuar.

2. Haga clic en **Add** para introducir las URL que desea que SCOM supervise. Cuando termine, haga clic en **Next** para continuar.

3. Haga clic en **Add** para configurar la ubicación desde la que desea supervisar.

4. En el cuadro de diálogo emergente, busque **Internal location - Agent**. Seleccione la dirección correspondiente y haga clic en **Add**. A continuación, haga clic en **Ok** para terminar de seleccionar la ubicación. Cuando termine, haga clic en **Next** para continuar.

5. Configure la frecuencia con la que SCOM recopila datos de cada endpoint:

| Datos                                             | Frecuencia            |
| ------------------------------------------------- | --------------------- |
| Frecuencia de prueba                              | 60 segundos           |
| Intervalo de recopilación de datos de rendimiento | 60 segundos           |
| Tiempo de espera de la prueba                     | 30 segundos           |
| Código de estado HTTP                             | Mayor o igual que 400 |

6. Cuando termine, haga clic en **Next** para continuar.

<div id="run-scom-tests">
  ## Ejecutar pruebas de SCOM
</div>

1. Haga clic en **Run Test** para probar cada endpoint y asegurarse de que la configuración de la conexión sea correcta.
2. Una vez que haya terminado de configurar su instancia de SCOM, puede ver la actividad en la pestaña **Monitoring**.

<div id="review-test-results">
  ## Revise los resultados de la prueba
</div>

Haga clic en **Web Application Status** para ver la información que SCOM ha recopilado.
