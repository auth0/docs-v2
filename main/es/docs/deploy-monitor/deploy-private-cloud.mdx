Los paquetes de Private Cloud son servicios administrados que puede usar si:

* No puede usar un servicio de Public Cloud multiinquilino en su organización
* Tiene requisitos de implementación geográfica para los que la opción de Public Cloud no resulta viable
* Requiere un nivel garantizado de solicitudes por segundo (RPS) o una capacidad superior a la disponible en Public Cloud

La nube privada de Auth0 se puede implementar de las siguientes maneras:

| Opción                                                                                         | Descripción                                                                                                                                                            |
| ---------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [**Private Cloud on AWS**](/es/docs/deploy-monitor/deploy-private-cloud/private-cloud-on-aws)     | Un servicio de nube administrado y dedicado que se ejecuta en AWS y proporciona aislamiento, mayor rendimiento, instancias de desarrollo, complementos de Geo-HA y más |
| [**Private Cloud on Azure**](/es/docs/deploy-monitor/deploy-private-cloud/private-cloud-on-azure) | Un servicio de nube dedicado que se ejecuta en Azure y proporciona aislamiento, mayor rendimiento, instancias de desarrollo, complementos y más                        |

También ofrecemos [funcionalidades complementarias](/es/docs/deploy-monitor/deploy-private-cloud/private-cloud-add-on-features) que pueden complementar su implementación de Auth0 Private Cloud.

Si tiene requisitos de soporte específicos o necesita más información sobre los Servicios Profesionales que ofrecemos, [póngase en contacto con el equipo de ventas de Auth0](https://auth0.com/contact-us).
