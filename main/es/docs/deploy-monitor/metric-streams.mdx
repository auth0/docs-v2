<div id="auth0api_requestcount">
  ### auth0.api_request.count
</div>

Cuenta las solicitudes a la API de tu inquilino de Auth0 con etiquetas detalladas de filtrado y agregaciĂłn. Se emite una nueva mĂŠtrica cada 10 segundos y ofrece una granularidad de un segundo.

<div id="available-tags">
  #### Etiquetas disponibles
</div>

| Nombre de etiqueta | Obligatorio | DescripciĂłn                                                                                                              |
| ------------------ | ----------- | ------------------------------------------------------------------------------------------------------------------------ |
| `api`              | SĂ­          | Tipo de API: `AUTHENTICATION` o `MANAGEMENT`                                                                             |
| `operation_name`   | SĂ­          | MĂŠtodo HTTP y ruta (p. ej., `POST /oauth/token`)                                                                         |
| `operation_result` | SĂ­          | Resultado: `SUCCESS` o `FAILURE`                                                                                         |
| `tenant`           | SĂ­          | Nombre de tu inquilino de Auth0                                                                                          |
| `failure_code`     | No          | Motivo del error: `RATE_LIMIT`, `ANOMALY_DETECTION` o `CLIENT_SERVER_ERROR`                                              |
| `country_code`     | No          | CĂłdigo ISO del paĂ­s a partir de la direcciĂłn IP de la solicitud                                                          |
| `client_id`        | No          | ID de cliente de la aplicaciĂłn (solo Authentication API)                                                                 |

<Warning>
  Las etiquetas `client_id` y `connection_id` pueden generar una cardinalidad alta en entornos con muchos clientes o conexiones distintos, lo que puede aumentar los costos de monitoreo. Usa estas etiquetas de forma selectiva.

<Note>
  Los Metric Streams no incluyen las llamadas a la API que se almacenan en cachĂŠ o que el Web Application Firewall (WAF) de Auth0 bloquea en el perĂ­metro para evitar la sobrecarga del sistema.

<div id="enum-values">
  #### Valores de enumeraciĂłn
</div>

**api**

* `AUTHENTICATION` - Solicitudes a la Authentication API
* `MANAGEMENT` - Solicitudes a la Management API

**operation&#95;result**

* `SUCCESS` - La solicitud se completĂł correctamente
* `FAILURE` - La solicitud fallĂł debido a un error del cliente o del servidor

**failure&#95;code** (solo estĂĄ presente cuando `operation_result` es `FAILURE`)

* `RATE_LIMIT` - La solicitud superĂł el lĂ­mite de tasa
* `ANOMALY_DETECTION` - Bloqueada por la protecciĂłn contra ataques de Auth0
* `CLIENT_SERVER_ERROR` - Error HTTP 4xx o 5xx

<div id="prerequisites">

* Una cuenta activa de [Datadog](https://www.datadoghq.com/)

### Crear flujo de mĂŠtricas

<Tabs>
  <Tab title="Dashboard">
    **Vaya a Metric Streams:**

    1. Vaya a [Auth0 Dashboard](https://manage.auth0.com) &gt; **Monitoring** &gt; **Metric Streams**.
    2. Seleccione **+ Create Metric Stream**.
    3. Seleccione **Datadog**.

    **Configure el flujo:**

    1. **Name**: Introduzca un nombre descriptivo (por ejemplo, `datadog-production`).
    2. **Datadog Site**: Seleccione su regiĂłn de Datadog en el menĂş desplegable:
       * `US1` para app.datadoghq.com
       * `US3` para us3.datadoghq.com
       * `US5` para us5.datadoghq.com
       * `EU` para app.datadoghq.eu
       * `AP1` para ap1.datadoghq.com
    3. **Datadog API Key**: Pegue la clave de API de Datadog.
    4. Seleccione **Save**.

    Esto crea y habilita automĂĄticamente su Metric Stream.
  </Tab>

  <Tab title="CLI">
    Use la CLI de Auth0 para crear un flujo de mĂŠtricas de Datadog:

    ```bash
    auth0 api post metric-streams \
      --data '{
        "name": "datadog-production",
        "subscriptions": [{ "metric_type": "auth0.api_request.count" }],
        "destination": {
          "type": "datadog",
          "configuration": {
            "api_key": "YOUR_DATADOG_API_KEY",
            "site": "app.datadoghq.com"
          }
        }
      }'
    ```

    **Respuesta:**

    ```json
    {
      "id": "mst_abc123xyz",
      "status": "enabled",
      "name": "datadog-production",
      "created_at": "2024-02-12T10:30:00Z",
      "updated_at": "2024-02-12T10:30:00Z",
      "destination": {
        "type": "datadog",
        "configuration": {
          "api_key": "***",
          "site": "app.datadoghq.com"
        }
      }
    }
    ```

    Guarde el valor de `id` para administrar este flujo mĂĄs adelante.
  </Tab>
</Tabs>

<div id="manage-streams">
  ### Administrar streams
</div>

<Tabs>
  <Tab title="Dashboard">

  <Tab title="CLI">

    **Volver a habilitar un stream:**

  </Tab>
</Tabs>

    4. Configura la expiraciĂłn del token.
    5. Selecciona **Create**.
    6. Copia y guarda de forma segura el valor del token. Lo necesitarĂĄs en los pasos posteriores.

<div id="understand-metric-cardinality">
  ## Comprender la cardinalidad de las mĂŠtricas
</div>

La cardinalidad se refiere a la cantidad de series temporales Ăşnicas que generan sus mĂŠtricas. Una cardinalidad mĂĄs alta incrementa los costos de monitoreo.

| Tag                | Cardinalidad | Impacto                                            |
| ------------------ | ------------ | -------------------------------------------------- |
| `api`              | Baja         | 2 valores Ăşnicos (Authentication, Management)      |
| `operation_name`   | Media        | VarĂ­a segĂşn las operaciones de API que se utilicen |
| `operation_result` | Baja         | 2 valores Ăşnicos (Success, Failure)                |
| `tenant`           | Media        | 1 por inquilino de Auth0                           |
| `failure_code`     | Baja         | 3 valores Ăşnicos (solo en caso de error)           |
| `country_code`     | Media-alta   | VarĂ­a segĂşn la distribuciĂłn geogrĂĄfica             |
| `client_id`        | **Alta**     | 1 por cliente de aplicaciĂłn                        |
| `connection_id`    | **Alta**     | 1 por conexiĂłn de Proveedor de identidad           |
| `grant_type`       | Baja         | Tipos de concesiĂłn de OAuth 2.0 limitados          |
| `batch_seq`        | Baja         | Tag temporal (se elimina despuĂŠs de la Beta)       |

<Warning>
  Use las tags `client_id` y `connection_id` con cuidado. En entornos con muchas aplicaciones o conexiones, estas tags pueden aumentar significativamente los costos. Considere filtrar por clientes especĂ­ficos o agrupar estas tags si no necesita ese nivel de granularidad.
</Warning>

<div id="high-monitoring-costs">
  ### Costos elevados de monitoreo
</div>

Si observas costos inesperadamente altos:

1. Revisa quĂŠ etiquetas estĂĄs usando en las consultas y los paneles.
2. Considera eliminar `client_id` y `connection_id` de las agregaciones.
3. Usa muestreo o filtrado en tu plataforma de monitoreo para reducir el volumen de mĂŠtricas ingeridas.
4. Agrupa por etiquetas de nivel superior, como `api` y `operation_result`, en lugar de usar etiquetas granulares.
