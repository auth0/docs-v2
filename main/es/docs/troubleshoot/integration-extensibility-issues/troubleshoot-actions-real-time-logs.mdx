Los Registros en tiempo real de Actions utilizan eventos enviados por el servidor (SSE) y podrían experimentar problemas de conectividad relacionados con:

* Conflictos con la protección de endpoint
* firewalls
* software de VPN

<div id="diagnosis">
  ## Diagnóstico
</div>

Estos conflictos pueden manifestarse de varias formas:

* **Conexiones bloqueadas:** Algunos componentes de software pueden impedir las conexiones HTTP de larga duración necesarias para que SSE funcione.
* **Búfer de datos:** Algunos componentes de software pueden almacenar en búfer los datos entrantes y solo entregarlos al cliente una vez que la solicitud HTTP se haya completado por completo. Como los flujos de SSE son continuos, esto hace que no se muestre ningún dato al usuario.
* **Bloqueo de Content-Type:** Algunos componentes de software pueden estar configurados para bloquear explícitamente el encabezado `content-type: text/event-stream`, que es esencial para SSE.

<div id="potential-solutions">
  ## Posibles soluciones
</div>

* Accede a los logs de Actions del inquilino desde otro dispositivo o desde otro navegador.
* Desactiva temporalmente cualquier VPN o software de protección de endpoint para comprobar si eso resuelve el problema.

<div id="contact-support">
  ## Contactar con soporte
</div>

Si sigues teniendo problemas, ponte en contacto con [Auth0 Support](https://support.auth0.com). Debes proporcionar toda la información posible, incluidos archivos `HAR` sin información confidencial. Los errores y los logs pueden ayudar a Auth0 Support a resolver tu problema con mayor rapidez.
