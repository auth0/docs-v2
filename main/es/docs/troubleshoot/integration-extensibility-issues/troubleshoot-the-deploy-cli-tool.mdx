<div id="warning-log-entries-after-a-google-apps-connection-is-recreated">
  ## Entradas de registro de advertencia después de recrear una conexión de Google Apps
</div>

**Síntomas**: verá advertencias en los logs con mensajes como `Unable to get extended attributes: unauthorized` o `Unable to get groups: unauthorized`.

Cuando crea una conexión de Google Apps por primera vez y habilita una de las casillas para obtener información ampliada sobre el usuario, deberá completar un flujo de consentimiento. Para ello, un administrador de Google Apps debe seguir el enlace que aparece debajo del botón &quot;Instrucciones de configuración&quot;, junto a la conexión. Después de completar este flujo, se almacena información del token en el objeto `options` de la conexión, que se utiliza para recuperar información ampliada cuando un usuario inicia sesión.

Si tiene una conexión de Google Apps, deberá asegurarse de que el flujo de consentimiento se complete antes de exportar la información de la conexión mediante la CLI. De este modo, los tokens necesarios se incluirán en el script exportado y el administrador no tendrá que completar el flujo de consentimiento cada vez que se vuelva a crear la conexión.

<Warning>
  Los tokens almacenados en el objeto `options` de la conexión son confidenciales y deben tratarse del mismo modo que cualquier otra información sensible.
</Warning>
