---
description: Describe la política de límites de entidades del inquilino de Auth0 para suscriptores.
title: Política de límites de entidades
---

Las entidades en Auth0 son elementos de configuración del inquilino, como Aplicaciones, Conexiones, Organizaciones, API <Tooltip tip="Resource Server: Servidor que aloja recursos protegidos. Los servidores de recursos aceptan y responden a solicitudes de recursos protegidos." cta="Ver glosario" href="/es/docs/glossary?term=resource+servers">servidores de recursos</Tooltip>, Permisos, Roles y Actions.

<Warning>
  Los límites de entidades son límites estrictos que se aplican por la salud de nuestra plataforma; no forman parte del precio, aunque pueden variar entre grupos de planes de precios. Los planes de precios individuales tienen limitaciones aparte, que se refieren a las prestaciones, como MAU y conexiones empresariales, que recibe al suscribirse a un plan determinado. (Para conocer las limitaciones de su plan de precios, consulte [Auth0 Pricing](https://auth0.com/pricing/).)
</Warning>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Cualquier uso del término &quot;unlimited&quot; significa que no hay límites estrictos, pero Okta establece límites del sistema para el umbral. Comuníquese con soporte si tiene alguna pregunta sobre estos límites.
</Callout>

<div id="global-subscription-plan-limits">
  ## Límites globales de los planes de suscripción
</div>

En esta sección se enumeran los límites que se aplican a todos los planes de suscripción.

<div id="organizations">
  ### Organizaciones
</div>

| Entidad                                              | Límite  |
| ---------------------------------------------------- | ------- |
| Organizaciones por Tenant                            | 100,000 |
| Miembros por organización                            | 100,000 |
| Conexiones por organización                          | 10      |
| Asignaciones de roles por miembro de la organización | 50      |
| M2M Client Grants por organización (nube pública)    | 100     |
| M2M Client Grants por organización (nube privada)    | 1000    |
| Perfiles personalizados de intercambio de tokens     | 100     |

Los clientes con planes Enterprise pueden solicitar aumentos de los límites de entidades para organizaciones por Tenant y miembros por organización [poniéndose en contacto con soporte](https://support.auth0.com/).

Estos límites pueden aumentarse hasta 2,000,000 de organizaciones por Tenant y 2,000,000 de miembros por organización en la nube pública. En las instancias de nube privada, estas entidades son ilimitadas.

<div id="authorization-core-role-based-access-control-rbac">
  ### Authorization Core Control de acceso basado en roles (RBAC)
</div>

| Entidad                                   | Límite |
| ----------------------------------------- | ------ |
| Roles por Tenant                          | 1000   |
| Alcances por API (servidores de recursos) | 1000   |
| Roles por usuario                         | 50     |
| Permisos por usuario                      | 1000   |
| Permisos por rol                          | 1000   |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Tenga en cuenta que los límites de permisos por usuario afectan directamente a los permisos asignados directamente.

  Técnicamente, un usuario podría tener más permisos de los indicados si asigna los permisos a distintos roles y luego asigna esos roles al usuario.
</Callout>

| Entidad                                         | Límite |
| ----------------------------------------------- | ------ |
| Token de actualización por usuario y aplicación | 200    |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si alcanza el límite de <Tooltip tip="Token de actualización: token utilizado para obtener un nuevo Token de acceso sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/es/docs/glossary?term=Refresh+tokens">tokens de actualización</Tooltip> y crea un nuevo token de actualización,
  el sistema revoca o elimina el token más antiguo de ese usuario en la
  aplicación.

  Los tokens revocados y los tokens vencidos no cuentan para el límite.
</Callout>

<div id="actions">
  ### Actions
</div>

| Entidad                                      | Límite |
| -------------------------------------------- | ------ |
| Actions por Tenant                           | 100    |
| Actions por disparador por Tenant            | 20     |
| Actions Modules por Tenant                   | 100    |
| Actions Modules por Action                   | 3      |
| Versiones por Action                         | 50     |
| Versiones por Action Module                  | 50     |
| Secrets por Action                           | 30     |
| Secrets por Action Module                    | 30     |
| Dependencias (módulos NPM) por Action        | 10     |
| Dependencias (módulos NPM) por Action Module | 10     |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Los límites de Actions y Actions Modules incluyen tanto las entidades desplegadas como las no desplegadas. Cuando se eliminan, dejan de contar para el límite correspondiente.

  Los límites de las versiones de Action y Action Module incluyen tanto borradores como versiones activas. Una vez que alcance el límite, cada versión adicional que agregue hará que Auth0 elimine la versión sin usar más antigua.
</Callout>
