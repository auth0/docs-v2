description: Describe cómo exportar datos de Auth0.
title: Exportar datos
---

Todos los datos de tu Tenant de Auth0 están siempre bajo tu control y puedes acceder a ellos en cualquier momento a través de la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>.

<div id="user-passwords">
  ## Contraseñas de usuario
</div>

La única información que no está disponible a través de la API (por motivos de seguridad) son los hashes de las contraseñas de los usuarios de base de datos alojados en Auth0 y las claves privadas. Aun así, puedes solicitar esta información abriendo un [ticket de soporte](https://support.auth0.com). Para obtener más información, consulta [Exportar hashes de contraseñas y secretos de MFA](/es/docs/manage-users/user-migration/export-password-hashes-and-mfa-secrets). Esta operación no está disponible en nuestro plan de suscripción Free, y no podemos aceptar ni garantizar solicitudes de exportación para una fecha y hora concretas.

Si almacenas las contraseñas de los usuarios en tu base de datos, configura una base de datos personalizada que Auth0 consultará cada vez que un usuario inicie sesión. En este caso, Auth0 nunca almacenará hashes de contraseñas, a menos que elijas migrar usuarios a Auth0 de forma gradual.

<div id="auth0-components">
  ## Componentes de Auth0
</div>

Auth0 utiliza principalmente <Tooltip tip="OpenID: estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/es/docs/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) como protocolo de autenticación, por lo que debería poder implementar una integración con su aplicación mediante bibliotecas estándar. Lo mismo ocurre al integrar Auth0 mediante [SAML](/es/docs/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider). Todos los SDK, las bibliotecas y los ejemplos de Auth0 [se publican en GitHub como software libre](https://github.com/auth0/).

<div id="social-identity-providers">
  ## Proveedores de identidad sociales
</div>

Si decide no usar Auth0 pero quiere seguir usando los mismos <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/es/docs/glossary?term=OAuth">OAuth</Tooltip> <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/es/docs/glossary?term=client+IDs">ID de cliente</Tooltip> y secretos para sus <Tooltip tip="Identity Provider (IdP): Servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=identity+providers">proveedores de identidad</Tooltip> sociales, mantendrá acceso a la información de los usuarios sin necesidad de mostrar nuevos cuadros de diálogo de consentimiento.

<div id="custom-code">
  ## Código personalizado
</div>

Todas las funcionalidades de código personalizado de Auth0 (Rules, scripts de base de datos personalizada, conexiones OAuth personalizadas, etc.) se ejecutan en un servicio de entorno aislado de Node.js. Todas las bibliotecas disponibles en este servicio también están disponibles en npm para usarlas con código estándar de Node.js.
