title: Comprobar mensajes de error
---

Compruebe si se muestran mensajes de error en cualquiera de las siguientes ubicaciones:

* Navegadores y respuestas de páginas HTML
* Pestañas Red y Consola de las herramientas para desarrolladores
* Respuestas del <Tooltip tip="Servidor de autorización: Servidor centralizado que contribuye a definir los límites del acceso de un usuario. Por ejemplo, su servidor de autorización puede controlar los datos, las tareas y las funciones disponibles para un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Authorization+Server">Servidor de autorización</Tooltip>
* Errores de obsolescencia

<div id="check-browser-errors">
  ## Revisa los errores del navegador
</div>

Un archivo HAR es un registro en formato JSON de las interacciones de un navegador web con un servidor web. Si la autenticación no funciona como debería, puedes [generar y analizar archivos HAR](/es/docs/troubleshoot/troubleshooting-tools/generate-and-analyze-har-files) para identificar problemas.

<Warning>
  Antes de compartir un archivo HAR con alguien (incluido Auth0), asegúrate de eliminar u ofuscar todos los datos confidenciales, como:

  * Información confidencial del usuario
  * Información de identificación personal (PII)
  * Información confidencial de la aplicación

  Para obtener más información, consulta los siguientes artículos en [Auth0 Community](https://community.auth0.com):

  * [Saneamiento de trazas HTTP](https://community.auth0.com/t/sanitizing-http-traces/119488)
  * [Cómo sanear automáticamente un archivo de traza HTTP](https://community.auth0.com/t/how-to-sanitize-a-http-trace-file-automatically/120583)
  * [Cómo ocultar manualmente información confidencial](https://community.auth0.com/t/how-to-manually-redact-sensitive-information/122554)
  * [El archivo HAR es demasiado grande para adjuntarlo al caso de soporte](https://community.auth0.com/t/har-file-is-too-large-to-upload-to-the-support-case/122488)
</Warning>

<div id="check-login-screen">
  ## Comprueba la pantalla de inicio de sesión
</div>

El widget de inicio de sesión Lock muestra mensajes de error para ciertos tipos de problemas, como un nombre de usuario o una contraseña incorrectos. Comprueba el enlace **Más información** si estás usando la página de error estándar de Auth0.

<div id="check-logs">
  ## Consultar registros
</div>

Auth0 almacena [datos de registro](/es/docs/deploy-monitor/logs), incluidas las acciones de los administradores del Dashboard, las autenticaciones de usuarios exitosas y fallidas, y las solicitudes de cambio de contraseña. Puede ver los registros en el [Dashboard](https://manage.auth0.com/#/logs).

Algunos tipos de errores no aparecen en los registros. Por ejemplo, si se produce un error en un <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> remoto, si la autenticación no se completa y el usuario nunca vuelve a Auth0, no habrá ninguna entrada en los registros.

Puede exportar los registros de Auth0 y almacenarlos usted mismo o enviarlos automáticamente a servicios externos de registros. Esta funcionalidad puede ayudarle a cumplir los requisitos de retención y análisis de registros. Puede instalar y configurar una extensión de Auth0 para exportar automáticamente los registros a otro proveedor, como Sumo Logic o Loggly. Para obtener más información, vaya a nuestras [soluciones de transmisión de registros](https://marketplace.auth0.com/features/log-streaming) en Auth0 Marketplace.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Consulte [la página de monitoreo](/es/docs/deploy-monitor/monitor) para obtener más detalles sobre cada uno de los eventos de registro que pueden ayudarle a solucionar problemas.
</Callout>

También puede usar la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> para exportar registros y almacenarlos. Hay dos endpoints disponibles; cada uno proporciona información ligeramente distinta.

<div id="search-all-logs-get_logs-endpoint">
  ### Buscar todos los registros en el endpoint /get_logs
</div>

El [endpoint de búsqueda de eventos de registro](https://auth0.com/docs/api/management/v2#!/Logs/get_logs) recupera las entradas de registro que coinciden con los criterios de búsqueda que proporcionaste. Si no proporcionas ningún criterio de búsqueda, obtendrás una lista de todas las entradas disponibles.

Puedes indicar criterios de búsqueda mediante el parámetro **q** y recuperar campos específicos mediante el parámetro **fields**.

Para acceder a la API, necesitas un [token de Management API v2](https://auth0.com/docs/api/management/v2/tokens).

Esta solicitud de ejemplo recupera todos los registros de inicios de sesión exitosos (el acrónimo del evento para un inicio de sesión exitoso es `s`). La lista de campos que recuperaremos por cada entrada de registro es: **date**, **description**, **client&#95;id** y **log&#95;id**.

Para obtener más información sobre los criterios de búsqueda que puedes usar y una lista de los acrónimos de eventos, consulta el [endpoint de búsqueda de eventos de registro](https://auth0.com/docs/api/management/v2#!/Logs/get_logs).

<div id="get-a-single-log-entry-get_logs_by_id-endpoint">
  ### Obtener una entrada de registro individual del endpoint /get_logs_by_id
</div>

El [endpoint para obtener un evento de registro por ID](https://auth0.com/docs/api/management/v2#!/Logs/get_logs_by_id) recupera la entrada de registro asociada al ID proporcionado.

Esta solicitud de ejemplo recupera una única entrada de registro con el ID `90020180129170850881585554625888895190928456277777449010`.

<div id="check-logs-for-deprecation-errors">
  ## Revise los registros para detectar errores de obsolescencia
</div>

Cuando las funciones de Auth0 quedan obsoletas, pueden aparecer errores o avisos en los registros del inquilino que indican que sus aplicaciones están usando funciones obsoletas. Puede [buscar en los registros entradas específicas de obsolescencia](/es/docs/troubleshoot/basic-issues/check-deprecation-errors) que puedan indicar que una función ha quedado obsoleta.

<div id="rate-limits-and-other-errors">
  ## Límites de tasa y otros errores
</div>

Auth0 proporciona un código de error único para los errores que se notifican cuando [se supera el límite de tasa](/es/docs/troubleshoot/customer-support/operational-policies/rate-limit-policy). Debe configurar un análisis automático de los registros para comprobar si hay errores de límite de tasa, de modo que pueda abordar de forma proactiva la actividad que alcanza esos límites antes de que cause demasiados problemas a sus usuarios. Auth0 también publica códigos de error para otros tipos de errores, y le resultará útil analizar los registros para detectar errores de autenticación, así como errores en las llamadas a Auth0 Management API (los códigos de error de Management API se muestran debajo de cada llamada en [Management API Explorer](https://auth0.com/docs/api/management/v2)).

Tenga en cuenta también que llamar a la Management API para recuperar información del perfil del usuario desde una Rule es una causa habitual de errores de límite de tasa, ya que esas llamadas a la API pueden ejecutarse en cada inicio de sesión, así como durante comprobaciones periódicas de la sesión.

<div id="check-actions-real-time-logs-error-console">
  ## Revisa la consola de errores de Actions Real-time Logs
</div>

Puedes incluir instrucciones `console.log()` en Rules, Hooks y scripts personalizados de base de datos. La salida de esas instrucciones puede verse en Actions Real-time Logs. Si usas Actions Real-time Logs, puedes abrir esta consola de registros en **Auth0 Dashboard &gt; Monitoring &gt; Actions Logs**.
