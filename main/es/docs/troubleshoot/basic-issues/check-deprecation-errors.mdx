description: Aprenda a buscar errores de obsolescencia en los registros
title: Errores de obsolescencia
---

Cuando las funciones de Auth0 quedan obsoletas, pueden aparecer errores o avisos en los registros del inquilino para indicar que sus aplicaciones están usando funciones obsoletas. Esta guía le ayudará a buscar en sus registros mensajes relacionados con la obsolescencia, así como a comprender las posibles causas y resoluciones de elementos concretos.

Hay dos formas distintas de buscar mensajes de advertencia que muestren el uso de funciones obsoletas: el Dashboard o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>. Tenga en cuenta que, en cualquiera de los dos casos, el [período de retención de logs](/es/docs/deploy-monitor/logs) se rige por el nivel de suscripción de su cuenta.

Consulte [Buscar registros de errores de obsolescencia](/es/docs/troubleshoot/basic-issues/search-logs-for-deprecation-errors) para obtener más información.

<div id="deprecation-log-messages">
  ## Mensajes de registro de obsolescencia
</div>

<div id="legacy-lock-api">
  ### API de Lock legado
</div>

**Entrada de registro**: `up-idp-initiated`

**Mensaje de error:** &quot;API de Lock legado: Esta funcionalidad quedará obsoleta. Consulte nuestra documentación para obtener información sobre cómo migrar su aplicación.&quot;

**Causa**: Las entradas de registro del inquilino relacionadas con la API de Lock legado pueden incluir el referrer y la información sobre el SDK utilizado.

**Resolución**: Use esta información para verificar si alguna de sus aplicaciones utiliza bibliotecas desactualizadas.

<div id="ssodata-endpoint">
  ### Endpoint de SSOdata
</div>

**Entrada de registro**: `ssodata`

**Mensaje de error:** &quot;Endpoint de SSOdata: Esta funcionalidad quedará obsoleta. Consulte nuestra documentación para saber cómo migrar su aplicación.&quot;

**Causa**: Llamar directamente al endpoint `/ssodata` o usar versiones antiguas de Lock integrado o del SDK de Auth0.js para llamar a una función que llamaba al endpoint `/ssodata`.

**Resolución**: Actualice el SDK de Lock o el SDK de Auth0.js a la versión más reciente.
