A continuación, encontrará los códigos de error y las posibles soluciones para varios errores que pueden producirse en la API de cambio de contraseña por parte del usuario.

<div id="error-format">
  ## Formato de los errores
</div>

Los mensajes de error se devuelven en el formato estándar:

<div id="error-codes">
  ## Códigos de error
</div>

<div id="invalid_request">
  ### invalid_request
</div>

Este error se produce cuando se proporcionan parámetros no válidos. Los mensajes de error describen el problema, como la falta de un parámetro obligatorio o un formato no válido.

<div id="invalid_user_password">
  ### invalid_user_password
</div>

Este error se produce cuando se envía una combinación incorrecta de `username`/`email` y `old_password`. Vuelva a intentar la solicitud con el nombre de usuario correcto y `old_password`.

<div id="change_password_error">
  ### change_password_error
</div>

Este error puede deberse a varias condiciones relacionadas con el <Tooltip tip="Proveedor de identidad (IdP): Servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=identity+provider">Proveedor de identidad (IdP)</Tooltip> subyacente. Por lo general, esto ocurre cuando usa una base de datos personalizada y no ha implementado el script de cambio de contraseña.
