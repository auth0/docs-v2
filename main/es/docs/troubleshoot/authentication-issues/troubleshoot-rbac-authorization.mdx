Aquí tienes algunas soluciones para problemas comunes al implementar el [control de acceso basado en roles (RBAC)](/es/docs/manage-users/access-control/rbac) con el conjunto de funciones de Authorization Core.

<div id="role-based-access-control-is-enabled-for-my-api-but-the-scopes-claim-is-not-showing-what-you-say-it-should">
  ## El control de acceso basado en roles está habilitado para mi API, pero el claim de alcances no muestra lo que debería
</div>

* Asegúrese de no establecer `accessToken.scope` en una Rule.
* Recuerde que cualquier [regla de autorización](/es/docs/manage-users/access-control/rules-for-authorization-policies) configurada se ejecuta después de que se tomen las decisiones de autorización basadas en RBAC, por lo que puede sobrescribir el comportamiento predeterminado.
