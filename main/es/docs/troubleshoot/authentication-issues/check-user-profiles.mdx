---
description: Aprende a comprobar perfiles de usuario para solucionar problemas
title: Comprobar perfiles de usuario
---

1. ¿La información del perfil de usuario es correcta en el origen (Servidor de autorización)?
2. Genera y revisa el [archivo HAR](/es/docs/troubleshoot/troubleshooting-tools/generate-and-analyze-har-files) y busca un `id_token`.

   <Warning>
     Antes de compartir un archivo HAR con cualquier persona (incluido Auth0), asegúrate de eliminar u ocultar todos los datos sensibles, como:

     * Información confidencial del usuario
     * Información de identificación personal (PII)
     * Información confidencial de la aplicación

     Para obtener más información, consulta los siguientes artículos en [Auth0 Community](https://community.auth0.com):

     * [Sanitizing HTTP Traces](https://community.auth0.com/t/sanitizing-http-traces/119488)
     * [How to Sanitize an HTTP Trace File Automatically](https://community.auth0.com/t/how-to-sanitize-a-http-trace-file-automatically/120583)
     * [How to Manually Redact Sensitive Information](https://community.auth0.com/t/how-to-manually-redact-sensitive-information/122554)
     * [HAR File is Too Large to Upload to the Support Case](https://community.auth0.com/t/har-file-is-too-large-to-upload-to-the-support-case/122488)
   </Warning>
3. Decodifica el `id_token` en [JWT.io](https://jwt.io) para comprobar si contiene la información correcta.
4. Revisa cualquier script de base de datos personalizada o lógica de Rule.
5. Comprueba si llamaste al endpoint `/tokeninfo` y si tienes un dominio personalizado configurado en Auth0. Si es así, debes usar el endpoint `/userinfo` en su lugar.
6. Comprueba si llamaste correctamente al endpoint `/userinfo`. Debes pasar un token de acceso. Debes llamar a este endpoint con el dominio predeterminado de Auth0, incluso si el inquilino tiene un dominio personalizado habilitado.
7. Comprueba si especificaste el [scope](/es/docs/get-started/apis/scopes) correcto para obtener un Token de acceso.
