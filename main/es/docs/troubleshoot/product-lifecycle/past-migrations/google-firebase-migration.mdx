title: Migrar de Google Cloud Messaging a Firebase Cloud Messaging
---

A partir del 11 de abril de 2019, [Google declaró obsoleto](https://firebase.googleblog.com/2018/04/time-to-upgrade-from-gcm-to-fcm.html) Google Cloud Messaging (GCM) y lo reemplazó por Firebase Cloud Messaging (FCM).

<div id="features-affected">
  ## Funciones afectadas
</div>

Los SDK de Guardian de Auth0 para iOS y Android le ayudan a crear aplicaciones móviles personalizadas con la funcionalidad de Guardian, lo que proporciona acceso seguro a la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA) con notificaciones push.

La biblioteca SDK de Guardian para Android de Auth0 se diseñó para enviar notificaciones push mediante GCM. Tenga en cuenta que las aplicaciones existentes deberían [seguir funcionando como hasta ahora](https://aws.amazon.com/blogs/messaging-and-targeting/the-end-of-google-cloud-messaging-and-what-it-means-for-your-apps/).

<div id="actions">
  ## Actions
</div>

Consulte la [documentación de Google](https://developers.google.com/cloud-messaging/android/android-migrate-fcm) para obtener información sobre cómo migrar de GCM a FCM.

La principal diferencia en la forma de enviar notificaciones a GCM y FCM está en la carga útil recibida en la notificación. Aunque los clientes existentes que usaban el SDK de Android podían adaptar la carga útil recibida antes de invocar el método del SDK, hemos actualizado la biblioteca para que acepte la nueva carga útil, lo que facilita la adopción de FCM. Más detalles [aquí](https://github.com/auth0/Guardian.Android/pull/84).

La versión 0.4.0 del SDK de Guardian para Android ya está disponible en Maven Central e incluye este cambio. La aplicación de ejemplo también se actualizó, por lo que puede probarse proporcionando el archivo `google-services.json` y un `guardian-url`.
