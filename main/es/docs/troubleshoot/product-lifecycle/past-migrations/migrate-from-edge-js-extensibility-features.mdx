---

A partir del 21 de junio de 2023, Auth0 dejará de admitir las funciones de extensibilidad de Node.js para .NET y C#.

<div id="affected-workflows">
  ## Flujos de trabajo afectados
</div>

Revise sus funcionalidades de extensibilidad para identificar código que contenga `require(edge)`.

Revise los siguientes flujos o características que admiten código de extensibilidad en busca de referencias a `edge.js`:

* [Hooks](/es/docs/customize/hooks)
* [Rules](/es/docs/customize/rules)
* [Scripts de base de datos personalizada](/es/docs/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Conexiones sociales personalizadas](/es/docs/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Actualizaciones recomendadas
</div>

No hay previsto un reemplazo oficial para esta funcionalidad.

Una vía de migración es sustituir la lógica de `edge` por un servicio independiente dentro de su propia infraestructura. Después de crear el servicio en su infraestructura, puede usar una función de extensibilidad para realizar una solicitud HTTP que desencadene la operación necesaria.

Por ejemplo, en un script de Login de una base de datos personalizada que use `edge`, elimine la función que llama a `edge` y use un cliente HTTP genérico para capturar la respuesta.

Otra vía de migración es reescribir la funcionalidad de extensibilidad en JavaScript en lugar de .NET/C#.

Para obtener más información sobre cómo crear Actions o migrar a Actions, [consulte la documentación de Auth0](/es/docs/customize/actions).
