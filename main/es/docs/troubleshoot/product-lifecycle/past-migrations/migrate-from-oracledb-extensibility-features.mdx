title: Migrar desde la funcionalidad de extensibilidad de oracledb
---

A partir del 21 de junio de 2023, Auth0 dejará de admitir conexiones a bases de datos Oracle desde Node.js en la funcionalidad de extensibilidad. Anteriormente, Auth0 admitía conexiones a bases de datos Oracle mediante el módulo `oracledb` de Node.js.

<div id="affected-workflows">
  ## Flujos de trabajo afectados
</div>

Revise sus funcionalidades de extensibilidad para identificar código que contenga `require(oracledb)` o `require('oracledb@4.2.0')`.

Revise los siguientes flujos o características que permiten código de extensibilidad para buscar referencias a [oracledb](https://www.npmjs.com/package/oracledb):

* [Hooks](/es/docs/customize/hooks)
* [Rules](/es/docs/customize/rules)
* [Scripts de base de datos personalizada](/es/docs/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Conexiones sociales personalizadas](/es/docs/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Actualizaciones recomendadas
</div>

No hay ningún reemplazo oficial previsto para esta funcionalidad. Auth0 recomienda trasladar la lógica de `oracledb` a un servicio independiente dentro de su propia infraestructura. Después de crear el servicio en su infraestructura, puede usar una función de extensibilidad para realizar una solicitud HTTP que active la lógica de Oracle Database.

Para obtener más información sobre cómo implementar el módulo `oracledb` en su propia infraestructura, consulte la [guía de Quickstart de OracleDB](https://node-oracledb.readthedocs.io/en/latest/user_guide/installation.html#quickstart).

Por ejemplo, en un script de Login de base de datos personalizada que usa `oracledb`, elimine la función que llama a `oracledb` y reemplácela por un cliente HTTP genérico para capturar la respuesta.

Para obtener más información sobre cómo crear Actions o migrar a Actions, [consulte la documentación de Auth0](/es/docs/customize/actions).
