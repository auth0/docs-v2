title: Cambios en la API Graph de Facebook
---

A partir del 1 de agosto de 2018, Facebook cambió los permisos y campos de la API Graph de Facebook que se pueden solicitar. Auth0 ha actualizado las Conexiones de Facebook para reflejar estos cambios y ha modificado la interfaz de la conexión para que sea más clara. Consulta [Facebook Login Changelog: Recent Changes to Facebook Login](https://developers.facebook.com/docs/facebook-login/changelog#2018-07-02) para obtener todos los detalles y las fechas clave.

<div id="features-affected">
  ## Características afectadas
</div>

Es posible que esta actualización no requiera cambios en tu código ni en tu configuración, pero tu aplicación podría recibir datos de perfil adicionales si los permisos existentes lo permiten. Ten en cuenta lo siguiente:

* Si tu conexión de Facebook está configurada para solicitar uno de los permisos eliminados, tu <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o un JWT, que se usa para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">Token de acceso</Tooltip> no los incluirá en el scope.
* Si tu aplicación de Facebook está marcada como &quot;development&quot;, es posible que sigas viendo un error temporalmente al probar la conexión.
* Si agregas nuevos permisos a la conexión, se pedirá el consentimiento de los usuarios finales la próxima vez que inicien sesión. Consulta la documentación de Facebook para saber cómo actuar con los usuarios que no tengan un permiso específico.

<div id="facebook-login-permissions">
  ### Permisos de Facebook Login
</div>

La aplicación solicita los [permisos de Facebook Login](https://developers.facebook.com/docs/facebook-login/permissions) cuando un usuario inicia sesión con Facebook. Si el usuario inicia sesión por primera vez o si los permisos han cambiado, Facebook le mostrará una ventana de consentimiento con los nuevos permisos solicitados. Una vez concedidos esos permisos, la aplicación podrá actuar en nombre de ese usuario con un token de acceso de Facebook.

La interfaz de la conexión de Facebook se ha actualizado para mostrar, en todos los permisos, tanto el nombre habitual como el nombre interno. Esto facilita encontrar los permisos que necesita y asociarlos con cualquier código que esté ejecutando con esos nombres de permisos.

</div>

Los siguientes permisos se movieron a la sección **Obsoletos** y no deben usarse con la versión más reciente de la Graph API:

* publish&#95;actions
* user&#95;managed&#95;groups

<div id="facebook-graph-api-fields">
  ### Campos de la API Graph de Facebook
</div>

La [API Graph de Facebook](https://developers.facebook.com/docs/graph-api/reference/v3.2/user) se utiliza después de que un usuario inicia sesión para recuperar los datos del perfil del usuario de Auth0. Los permisos de datos de usuario solicitados determinan qué información se recupera de la API Graph. Los campos que se devuelven dependen de los permisos solicitados y de que esos campos existan en el perfil de usuario de Facebook.

Este cambio actualizó la API Graph de la versión 2.8 a la 3.2 y solicitará los siguientes campos de datos de usuario al iniciar sesión:
