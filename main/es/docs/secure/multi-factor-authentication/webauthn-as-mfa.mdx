---
description: Breve descripción de cómo funciona WebAuthn como factor de MFA
title: WebAuthn como autenticación multifactor
---

Cuando los usuarios se autentican con WebAuthn, usan algo **que tienen** como factor de autenticación: una clave de seguridad o un dispositivo.

Tanto las Claves de seguridad como la Biometría del dispositivo admiten la **verificación de usuario**, que requiere que los usuarios aporten algo **que saben** (un PIN o un código de acceso) y algo **que son** (como rasgos biométricos).

Al usar la Biometría del dispositivo, la verificación de usuario siempre se realiza. Para realizar la verificación con Claves de seguridad, [debe configurar Auth0](/es/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa) para exigir un PIN. Luego, se pedirá a los usuarios que introduzcan un PIN, que solo se almacena en la clave de seguridad, para completar la autenticación. Cuando se realiza la verificación de usuario, los usuarios pueden iniciar sesión con WebAuthn como único método de autenticación y así lograr la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor adicional al nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip>.

Al usar WebAuthn para la autenticación junto con la verificación de usuario, no solo reemplaza la contraseña por algo mucho más fácil de usar, sino que también elimina la necesidad de contar con otro paso de autenticación cuando se requiere MFA.
