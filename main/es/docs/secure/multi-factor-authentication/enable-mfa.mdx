Puedes habilitar la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que usa un factor además de username y contraseña, como un code por SMS." cta="Ver glosario" href="/es/docs/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA) para tu inquilino en el <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que usa un factor además de username y contraseña, como un code por SMS." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

Cuando habilites MFA, selecciona los factores (como las notificaciones push o los mensajes por teléfono) que quieres poner a disposición de los usuarios y define una política para determinar cuándo exigirles que se autentiquen.

<div id="select-factors">
  ## Seleccione factores
</div>

Auth0 admite varios factores que puede habilitar para que sus usuarios completen la autenticación.

<div id="independent-factors">
  ### Factores independientes
</div>

Debe habilitar y configurar al menos un factor independiente para exigir MFA a los usuarios de su inquilino. Si lo desea, puede habilitar varios factores independientes al mismo tiempo.

Los factores independientes disponibles son:

* [WebAuthn con claves de seguridad FIDO](/es/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa)
* [Código de un solo uso (OTP)](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-otp-notifications-for-mfa)
* [Notificación push con Auth0 Guardian](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa)
* [Mensaje telefónico](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)
* [Cisco Duo Security](/es/docs/secure/multi-factor-authentication/configure-cisco-duo-for-mfa)

<div id="dependent-factors">
  ### Factores dependientes
</div>

Puede habilitar uno o varios factores dependientes una vez que haya habilitado y configurado al menos un factor independiente para los usuarios de su inquilino.

Los factores dependientes disponibles son:

* [WebAuthn con biometría FIDO](/es/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa)
* [Correo electrónico](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa)
* [Código de recuperación](/es/docs/secure/multi-factor-authentication/configure-recovery-codes-for-mfa)

<div id="enable-mfa-in-the-auth0-dashboard">
  ## Habilite MFA en el Auth0 Dashboard
</div>

1. Vaya a [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).
2. En la sección **Factors**, habilite y configure los factores que quiera poner a disposición de los usuarios.
3. En la sección **Define policies**, seleccione una política para determinar cuándo se solicitará MFA a los usuarios.

   * **Never**: No se requiere MFA para ningún inicio de sesión.
   * **Use MFA adaptativo**: Se requiere MFA en función de la evaluación de riesgo de Auth0. Para obtener más información, consulte [Enable MFA adaptativo](/es/docs/secure/multi-factor-authentication/adaptive-mfa).
   * **Always**: Se requiere MFA en todos los inicios de sesión.
4. Si elige la política **Never** o **Always**, aparecerá la sección **MFA Risk Assessors**. De forma predeterminada, la opción **Enable MFA adaptativo Risk Assessment** está deshabilitada. Active el interruptor para evaluar y registrar el riesgo de todas las transacciones de inicio de sesión en los registros del inquilino.
5. En la sección **Additional Settings**, configure las siguientes opciones según sea necesario:

   * **Show Multi-factor Authentication options:** Active este interruptor para permitir que los usuarios seleccionen factores de autenticación durante la inscripción. Cuando está deshabilitado, Auth0 presenta automáticamente los factores de autenticación más seguros a los usuarios durante la inscripción.

     <Frame>![Show Multi-factor Authentication options vista del usuario final](/docs/images/cdy7uua7fh8z/3Mo3EXUk3ZFK4hZKck0yOO/37268f33f70d1e09032f0db317ac11d3/MFA_Options_-_English.png)</Frame>
   * **Customize MFA Factors using Actions**: Active este interruptor para personalizar sus flujos de MFA mediante las `post-login` [Actions](/es/docs/customize/actions). Esto le permite solicitar a los usuarios un factor específico o una secuencia de factores, así como usar información contextual para crear experiencias más individualizadas. Para obtener más información, consulte [Customize MFA Selection for Universal Login](/es/docs/secure/multi-factor-authentication/customize-mfa/customize-mfa-selection-universal-login).
6. Haga clic en **Save**.

<div id="customize-mfa-flow">
  ## Personaliza el flujo de MFA
</div>

Puedes usar Auth0 Actions para personalizar el flujo de MFA para los usuarios de tu inquilino. Por ejemplo, puedes crear una Action para exigir MFA en circunstancias específicas o para forzar el uso de un factor determinado.

Para obtener más información, consulta [Personaliza el MFA adaptativo](/es/docs/secure/multi-factor-authentication/adaptive-mfa/customize-adaptive-mfa).
