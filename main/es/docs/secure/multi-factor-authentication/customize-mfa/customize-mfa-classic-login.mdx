title: Personalizar MFA para Classic Login
---

Puede habilitar la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además de username y contraseña, como un code por SMS." cta="Ver glosario" href="/es/docs/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> en la sección [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa). Puede personalizar las páginas de MFA mediante opciones básicas de marca en el <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que usa un factor además de username y contraseña, como un code por SMS." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Para una personalización más avanzada, puede proporcionar su propio HTML para la página de MFA. Al usar su propio HTML, Classic Login usa el widget de MFA de Auth0 con las siguientes limitaciones:

* No admite MFA con correo electrónico.
* Si los usuarios están inscritos en más de un factor, no pueden seleccionar cuál usar. El widget de MFA les pedirá que inicien sesión con el factor más seguro.
* No utiliza las funciones de internacionalización de <Tooltip tip="Universal Login: su aplicación redirige a Universal Login, alojado en el Servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Universal Login</Tooltip>. Para obtener más información, consulte [Internacionalización de Universal Login](/es/docs/customize/internationalization-and-localization/universal-login-internationalization).

<div id="customize-html-for-the-mfa-page">
  ## Personalizar el HTML de la página de MFA
</div>

Para personalizar la página de MFA, vaya a la pestaña [Dashboard &gt; Branding &gt; Universal Login &gt; Multi-factor Authentication](https://manage.auth0.com/#/mfa_page) y active el interruptor **Customize MFA Page**. Luego puede usar el editor de texto para modificar el HTML, aplicar estilos a la página con CSS y cambiar el JavaScript utilizado para recuperar variables personalizadas. Una vez realizados los cambios, asegúrese de hacer clic en **Save Changes**.

Las páginas de MFA funcionan sin necesidad de personalización y Auth0 actualizará los scripts incluidos según sea necesario. Sin embargo, una vez que active la personalización, usted será responsable de actualizar y mantener el script (incluido el cambio de los números de versión, como el del widget de MFA), ya que Auth0 ya no podrá actualizarlo automáticamente.

Para obtener más información sobre cómo sobrescribir el texto en muchas áreas del proceso de MFA para Classic Login, consulte [MFA Theme Language Dictionary](/es/docs/secure/multi-factor-authentication/customize-mfa/mfa-theme-language-dictionary). Para obtener más información sobre las opciones de tema del widget de MFA, consulte [MFA Widget Theme Options](/es/docs/secure/multi-factor-authentication/customize-mfa/mfa-widget-theme-options).

<div id="reset-page">
  ## Restablecer la página
</div>

Si quieres volver a un diseño anterior, tienes dos opciones:

* Volver a la última plantilla guardada haciendo clic en **Reset to Last**;
* Volver a la plantilla predeterminada proporcionada por Auth0 haciendo clic en **Reset to Default**.

<div id="render-enrollment-invitations">
  ## Mostrar invitaciones de inscripción
</div>

Hay dos escenarios posibles en los que se muestra la página. Si se ha dirigido a un usuario a esta página específicamente para la inscripción (por ejemplo, desde un correo electrónico con un enlace de inscripción), la propiedad `ticket` estará disponible. De lo contrario, la propiedad `requestToken` estará disponible.

<div id="html-and-liquid-syntax">
  ## Sintaxis de HTML y Liquid
</div>

La página alojada usa la sintaxis Liquid para las plantillas. Para obtener más información, consulta [Liquid for Designers](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) en GitHub. Los siguientes parámetros están disponibles para ayudarte a renderizar la página:

* `userData.email`
* `userData.friendlyUserId`
* `userData.tenant`
* `userData.tenantFriendlyName`
* `iconUrl`

La mayoría de los parámetros que se usan en el widget de MFA deben pasarse a Guardian, como se muestra en la plantilla predeterminada incluida en el área de personalización. Si necesitas un mayor nivel de personalización, puedes usar [auth0-guardian.js](https://github.com/auth0/auth0-guardian.js/tree/master/example).

Para obtener más información, consulta [Personalizar las páginas de usuario de MFA](/es/docs/secure/multi-factor-authentication/customize-mfa).
