<Card title="Información general">
  La autenticación multifactor (MFA) es otro método para proteger su aplicación y la identidad de sus usuarios. MFA añade una capa de seguridad durante el inicio de sesión, ya que exige que los usuarios proporcionen más de una credencial para demostrar su identidad digital. Los factores pueden ser:

  * Algo que es, como un dato biométrico
  * Algo que sabe, como una contraseña
  * Algo que posee, como un dispositivo

  Lea estas preguntas frecuentes para determinar si usar MFA con su instancia de Auth0 es la opción adecuada para usted.
</Card>

<div id="what-is-multi-factor-authentication">
  ## ¿Qué es la autenticación multifactor?
</div>

<Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuario que utiliza un factor adicional además del nombre de usuario y la contraseña, como un código enviado por SMS." cta="Ver glosario" href="/es/docs/glossary?term=Multi-factor+authentication">Autenticación multifactor</Tooltip> (MFA) es un método de verificación de usuario que requiere más de un tipo de validación. Evita que <Tooltip tip="Actores maliciosos: entidad (una persona o un grupo) que representa una amenaza para la empresa o el entorno con la intención de causar daño." cta="Ver glosario" href="/es/docs/glossary?term=bad+actors">actores maliciosos</Tooltip> accedan a una cuenta incluso si han obtenido el nombre de usuario y la contraseña.

<div id="why-use-multi-factor-authentication">
  ## ¿Por qué usar la autenticación multifactor?
</div>

La MFA reduce la probabilidad de muchos tipos de ciberataques. Es habitual que terceros roben nombres de usuario y contraseñas o ataquen de forma automatizada las cuentas de usuario. Un factor adicional de MFA, como una huella dactilar o una contraseña de un solo uso, dificulta este tipo de ataques.

<div id="how-does-multi-factor-authentication-work">
  ## ¿Cómo funciona la autenticación multifactor?
</div>

La MFA funciona al exigir información de verificación adicional (conocida como factores). Los usuarios no pueden iniciar sesión solo con nombres de usuario y contraseñas. Deben proporcionar una prueba adicional de identidad, como reconocimiento facial o mensajes de texto.

<div id="mfa-factors">
  ## Factores de MFA
</div>

<Warning>
  La disponibilidad de los factores de MFA depende del plan; algunos solo están disponibles en los planes Professional y Enterprise. Para obtener más información, consulte [Auth0 Pricing](https://auth0.com/pricing).
</Warning>

Auth0 admite varios factores de MFA, entre ellos:

* Notificaciones push
* Notificaciones por SMS
* Notificaciones de voz
* Contraseñas de un solo uso
* WebAuthn con llaves de seguridad
* WebAuthn con biometría del dispositivo
* Notificaciones por correo electrónico
* Seguridad de Cisco Duo
* Códigos de recuperación

Para obtener más información, consulte [Factores de la autenticación multifactor](/es/docs/secure/multi-factor-authentication/multi-factor-authentication-factors).
