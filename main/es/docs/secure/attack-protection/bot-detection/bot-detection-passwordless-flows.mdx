title: Añadir detección de bots a los flujos sin contraseña
---

Si utilizas flujos de inicio de sesión y registro <Tooltip tip="Sin contraseña: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=passwordless">sin contraseña</Tooltip>, puedes habilitar la <Tooltip tip="Sin contraseña: forma de autenticación que no depende de una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Bot+Detection">detección de bots</Tooltip> para mostrar un paso de captcha cuando Auth0 determine que una solicitud es de alto riesgo.

<Warning>
  Si utilizas el [SDK de Lock](/es/docs/libraries/lock) con la experiencia Classic Login, debes actualizar a la versión 11.35.0 o posterior para usar la detección de bots en los flujos sin contraseña.

  Si utilizas el [SDK de Auth0.js](/es/docs/libraries/auth0js), debes actualizar a la versión 9.20.0 o posterior para usar la detección de bots en los flujos sin contraseña.
</Warning>

<div id="configure-bot-detection-for-passwordless-flows">
  ## Configurar detección de bots para flujos sin contraseña
</div>

Puede usar el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> para configurar el comportamiento de la función captcha.

1. En [Auth0 Dashboard](https://manage.auth0.com/dashboard), vaya a [Security &gt; Attack Protection &gt; detección de bots](https://manage.auth0.com/dashboard/#/security/attack-protection/bot-detection).
2. Localice la sección **Response**.
3. En la opción **Enforce CAPTCHA for passwordless flows**, seleccione cuándo exigir CAPTCHA:

   * **Never**: Los usuarios no tienen que completar un CAPTCHA para iniciar sesión.
   * **When Risky**: Los usuarios deben completar un CAPTCHA si el inicio de sesión es de alto riesgo.
   * **Always**: Los usuarios siempre deben completar un CAPTCHA para iniciar sesión.
