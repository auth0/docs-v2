Auth0 admite varias integraciones con proveedores de captcha de terceros para la [Detección de bots](/es/docs/secure/attack-protection/bot-detection).

<div id="google-recaptcha-enterprise-on-google-cloud-platform-gcp">
  ## Google reCAPTCHA Enterprise en Google Cloud Platform (GCP)
</div>

Puede usar Google reCAPTCHA Enterprise como servicio de <Tooltip tip="Detección de bots: forma de protección contra ataques en la que Auth0 bloquea el tráfico sospechoso de bots al habilitar un captcha durante el proceso de inicio de sesión." cta="Ver glosario" href="/es/docs/glossary?term=Bot+Detection">Detección de bots</Tooltip> para tráfico que supere 1 millón de evaluaciones al mes.

Para saber cómo configurar reCAPTCHA Enterprise en GCP, consulte [Configura reCAPTCHA Enterprise ... en la documentación de Google](https://cloud.google.com/recaptcha-enterprise/docs/authenticate-apis).

<div id="locate-your-gcp-project-id">
  ### Encuentra el ID de tu proyecto de GCP
</div>

1. Ve al [panel de Google Cloud](https://console.cloud.google.com/).
2. Selecciona tu proyecto en el menú desplegable para ver el **ID del proyecto** correspondiente.
3. Anota el **ID del proyecto**, ya que lo necesitarás más adelante.

<div id="enable-recaptcha-enterprise">
  ### Habilitar reCAPTCHA Enterprise
</div>

1. En el [panel de Google Cloud](https://console.cloud.google.com/), escribe reCAPTCHA Enterprise API en la barra de búsqueda y, a continuación, selecciónala en los resultados de Marketplace.
2. Selecciona **Habilitar** para habilitar la API.

<div id="create-a-site-key">
  ### Crear una clave del sitio
</div>

1. En el [Google Cloud Dashboard](https://console.cloud.google.com/), escribe reCAPTCHA Enterprise en la barra de búsqueda y selecciónalo en los resultados de Marketplace.
2. Selecciona **Create Key**.
3. En **Domain List**, agrega el dominio de tu inquilino de Auth0.
4. Activa la opción **Disable domain verification**.
5. Activa la opción **Use checkbox challenge** para habilitar la casilla de verificación «No soy un robot» de Google.
6. En **Challenge Security**, selecciona un nivel de dificultad para el desafío.
7. Selecciona **Create Key**.
8. Anota la **Site Key**, ya que la necesitarás más adelante.

<div id="create-an-api-key">
  ### Crear una clave de API
</div>

1. En el [Google Cloud Dashboard](https://console.cloud.google.com/), escribe Credentials en la barra de búsqueda y selecciónalo en los resultados.
2. Selecciona **Create Credentials &gt; API Key**. Anota la **API Key**, ya que la necesitarás más adelante.
3. En la ventana emergente, selecciona **Edit Key**.
4. En **Application Restrictions**, selecciona **IP addresses (web servers, cron jobs, etc.)** y luego introduce las [Auth0 IP Addresses for Allow Lists](/es/docs/secure/security-guidance/data-security/allowlist).
5. En **API Restrictions**, selecciona **Restrict Key**, elige **reCAPTCHA Enterprise API** en la lista desplegable y haz clic en Aceptar.
6. Selecciona **Save**.

<div id="hcaptcha">
  ## hCaptcha
</div>

El widget de hCaptcha puede proteger sus aplicaciones contra bots, spam y otras formas de abuso automatizado.

Para obtener más información sobre hCaptcha y cómo conseguir los detalles de configuración, consulte [Switch from reCAPTCHA to hCaptcha on hCaptcha Docs](https://docs.hcaptcha.com/switch).

<div id="friendly-captcha">
  ## Friendly Captcha
</div>

Friendly Captcha es un sistema para prevenir el spam en su sitio web. Puede añadir el widget de Friendly Captcha a su aplicación web para combatir el spam, con un impacto mínimo en la experiencia del usuario.

Para obtener más información sobre Friendly Captcha y cómo obtener los detalles de configuración, consulte [Installation on Friendly Captcha Docs](https://docs.friendlycaptcha.com/#/installation?id=installation).

<div id="arkose">
  ## Arkose
</div>

Arkose Labs ofrece una solución de detección de bots para mitigar ataques originados por personas y por bots. Para obtener más información sobre cómo integrar Arkose sin problemas como proveedor de CAPTCHA, consulta [Arkose Bot Manager](https://www.arkoselabs.com/arkose-bot-manager/).

<Warning>
  Debes actualizar a Auth0.js v9.23+ para usar Arkose como proveedor de CAPTCHA.
</Warning>

<div id="configure-callback-action-with-auth0js">
  ### Configurar la acción de callback con Auth0.js
</div>

Al usar Arkose como proveedor de CAPTCHA, el método `renderCaptcha` del cliente WebAuth de Auth0.js le permite pasar una función de callback. El segundo parámetro del callback será un objeto de carga útil que contiene la función `triggerCaptcha`. La función `triggerCaptcha` puede usarse para configurar cómo su aplicación gestiona y muestra la pantalla de CAPTCHA.

Para obtener más información sobre cómo configurar el método `renderCaptcha`, consulte [renderCaptcha](https://auth0.github.io/auth0.js/WebAuth.html#renderCaptcha) en la documentación de Auth0.js.
