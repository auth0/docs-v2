  Supplemental Signals está disponible para clientes Enterprise, y debes solicitar el complemento Attack Protection Add-on. Ponte en contacto con [Auth0 Sales](https://auth0.com/get-started?place=header\&type=button\&text=talk%20to%20sales) para obtener más información.
</Warning>

<Card title="Antes de empezar">
  Para configurar Akamai para que envíe Supplemental Signals a tu tenant de Auth0, debes [configurar Akamai como proxy inverso](/es/docs/customize/custom-domains/self-managed-certificates).
</Card>

Si usas Akamai Bot Manager y/o Akamai Account Protector, puedes [configurar tu proxy inverso en Akamai](/es/docs/customize/custom-domains/self-managed-certificates/configure-akamai-for-use-as-reverse-proxy) para enviar los resultados a Auth0 y [usar esos resultados en Actions](/es/docs/secure/attack-protection/use-akamai-supplemental-signals-actions).

<div id="supported-login-flows">
  ## Flujos de inicio de sesión compatibles
</div>

Auth0 admite actualmente los siguientes flujos de inicio de sesión para Supplemental Signals:

* Universal Login: Identificador + contraseña
  * [Universal Login: Primero el identificador](/es/docs/authenticate/login/auth0-universal-login/identifier-first)
  * [Universal Login: Registro con conexiones de base de datos](/es/docs/authenticate/database-connections)
  * Universal Login: Restablecimiento de contraseña, incluidas las organizaciones de Auth0
  * [Resource Owner Password Grant (ROPG)](/es/docs/get-started/authentication-and-authorization-flow/resource-owner-password-flow)

<div id="step-1-register-an-api-in-akamai">
  # Paso 1: Registrar una API en Akamai
</div>

Para que Akamai sepa cuándo los intentos de inicio de sesión se completan correctamente o fallan, tendrás que registrar una API en Akamai y luego definir un endpoint para cada flujo de inicio de sesión de Auth0 que quieras admitir.

Para obtener más información, consulta [Register an API on Akamai TechDocs](https://techdocs.akamai.com/api-definitions/docs/register-api) y [Add API resources on Akamai TechDocs](https://techdocs.akamai.com/api-definitions/docs/add-api-resources).

<div id="step-2-forward-bot-results-from-akamai-to-auth0">
  # Paso 2: Reenviar los resultados de bots de Akamai a Auth0
</div>

Para configurar Akamai para reenviar los resultados de bots a Auth0, consulta [Forward Bot Results to Origin on Akamai TechDocs](https://techdocs.akamai.com/content-protector/docs/forward-bot-results-to-origin).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Es posible que el encabezado `akamai-user-risk` no esté presente en todas las solicitudes de inicio de sesión, ya que Akamai solo lo envía cuando genera una puntuación de riesgo del usuario.
</Callout>

<div id="step-3-enable-processing-of-akamai-headers-in-auth0">
  # Paso 3: Habilite el procesamiento de encabezados de Akamai en Auth0
</div>

Puede configurar Auth0 para aceptar las Supplemental Signals enviadas desde Akamai en el Auth0 Dashboard o mediante la Auth0 Management API.

<Tabs>
  <Tab title="Dashboard">
    Para habilitar Supplemental Signals en el Dashboard:

    1. Vaya a **[Auth0 Dashboard &gt; Security &gt; Attack Protection](https://manage.auth0.com/dashboard/#/security/attack-protection)**.
    2. Seleccione **Supplemental Signals**.
    3. Active el interruptor **Enable Akamai headers**.

    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      Si no ve la opción de Supplemental Signals en el Auth0 Dashboard, debe solicitar el complemento Attack Protection Add-on. Para obtener más información, póngase en contacto con [Auth0 Sales](https://auth0.com/get-started?place=header\&type=button\&text=talk%20to%20sales).
    </Callout>
  </Tab>

  <Tab title="Management API">
    Para habilitar Supplemental Signals con la Management API:

    1. [Obtenga un token de acceso para la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con el alcance `update:attack_protection`.
    2. Llame al endpoint de la Management API [Update the Supplemental Signals configuration for a tenant](/es/docs/api/management/v2/supplemental-signals/patch-supplemental-signals) con el siguiente cuerpo:

    ```json
    {
      "akamai_enabled": true
    }
    ```
  </Tab>
</Tabs>

<div id="step-4-test-and-verify-configuration">
  # Paso 4: Probar y verificar la configuración
</div>

Prueba tu configuración haciendo una llamada a cada endpoint para los flujos de inicio de sesión que hayas configurado.

Si has configurado todo correctamente:

* Los encabezados `akamai-user-risk` y `akamai-bot headers` estarán presentes en los registros de tu tenant para los eventos correspondientes.
* Puedes usar las siguientes propiedades en el objeto `event` de la Action posterior al inicio de sesión:
  * `authentication.riskAssessment.supplemental.akamai.akamaiBot`
  * `authentication.riskAssessment.supplemental.akamai.akamaiUserRisk`
