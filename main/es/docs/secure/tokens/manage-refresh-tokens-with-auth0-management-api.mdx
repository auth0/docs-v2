<div id="management-api-endpoints">
  ## Endpoints de la Management API
</div>

La <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de Auth0 proporciona endpoints para gestionar tokens de actualización de forma individual o en conjunto.

Estos endpoints complementan la [revocación alternativa de tokens de actualización](/es/docs/secure/tokens/refresh-tokens/revoke-refresh-tokens) y reemplazan los endpoints del recurso `device-credentials` al añadir propiedades ampliadas y operaciones de revocación masiva.

<div id="refresh-token-resource">
  ### Recurso de token de actualización
</div>

Puede ver o revocar un token de actualización específico mediante los siguientes endpoints:

| Nombre                                                 | URL                                       | Scopes requeridos       |
| ------------------------------------------------------ | ----------------------------------------- | ----------------------- |
| Ver la información de un token de actualización por ID | `GET /api/v2/refresh-tokens/{tokenId}`    | `read:refresh_tokens`   |
| Revocar un token de actualización por ID               | `DELETE /api/v2/refresh-tokens/{tokenId}` | `delete:refresh_tokens` |

<div id="user-resource">
  ### Recurso de usuario
</div>

Puede listar o revocar todos los tokens de actualización de un usuario determinado mediante los siguientes endpoints:

| Nombre                                                  | URL                                            | scopes requeridos       |
| ------------------------------------------------------- | ---------------------------------------------- | ----------------------- |
| Listar todos los tokens de actualización de un usuario  | `GET /api/v2/users/{userId}/refresh-tokens`    | `read:refresh_tokens`   |
| Revocar todos los tokens de actualización de un usuario | `DELETE /api/v2/users/{userId}/refresh-tokens` | `delete:refresh_tokens` |

<div id="refresh-token-properties">
  ## Propiedades del token de actualización
</div>

Los <Tooltip tip="Endpoint de token: endpoint del Servidor de autorización que se usa para solicitar tokens de forma programática." cta="Ver glosario" href="/es/docs/glossary?term=token+endpoints">endpoints de token</Tooltip> de actualización devuelven información relevante sobre el token y su historial.

| Campo                                             | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| ------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ID del Token de actualización                     | Un identificador único y específico del Tenant del token de actualización en la API. No es el token en sí, sino una credencial secreta que se intercambia en el endpoint de token de la Authentication API.<br /><br />Los tokens de actualización pueden seguir siendo válidos después de que una sesión termine. El token de actualización conserva esta información independientemente del ciclo de vida de la sesión.                                                                                                                                                                                                     |
| ID de sesión                                      | El ID de sesión es un identificador persistente de la sesión en el Tenant de Auth0. Ten en cuenta que el ID de sesión corresponde al claim `sid`, que ya está presente en los ID Token y Logout Token, y puede usarse para establecer referencias cruzadas entre estas entidades.<br /><br />El token de actualización contiene referencias a la sesión que se utilizó para crear el token. El ID de sesión puede usarse para recuperar información adicional con el endpoint de la Management API para [inspeccionar una sesión específica por ID](/es/docs/manage-users/sessions/manage-user-sessions-with-auth0-management-api). |
| Tiempo relevante                                  | Información sobre la creación y el vencimiento del token de actualización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Información del cliente y de los Resource Servers | Información resumida sobre las aplicaciones y las API asociadas a un token de actualización.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |

<div id="limitations">
  ## Limitaciones
</div>

* Las operaciones de revocación de tokens de actualización (`DELETE`) se ejecutan de forma asíncrona y tienen consistencia eventual.
* Los tokens de actualización emitidos a partir del 21-09-2023 (22-02-2024 para los Tenants de la región US-3) contienen la propiedad de ID de sesión (`session_id`) con el valor correspondiente. Los tokens de actualización emitidos antes de esa fecha contienen esta propiedad con un valor `null`.
