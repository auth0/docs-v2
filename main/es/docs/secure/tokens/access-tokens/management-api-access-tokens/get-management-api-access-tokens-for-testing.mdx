title: Obtener tokens de acceso de la Management API para pruebas
---

<Warning>
  Este método para obtener tokens de acceso es **solo para fines de prueba**. No obtenga manualmente tokens de larga duración ni los use en sus aplicaciones, ya que eso anula las ventajas de seguridad que ofrecen los tokens.
</Warning>

La primera vez que obtiene un token de prueba para la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> es al completar la configuración en el <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. No tendrá que volver a hacerlo, a menos que cree un inquilino nuevo. Le recomendamos que cree un token de prueba exclusivamente para autorizar el acceso a la Management API, en lugar de reutilizar otro que ya tenga.

<div id="get-access-tokens-manually">
  ## Obtener tokens de acceso manualmente
</div>

1. En el Auth0 Dashboard, ve a **Applications &gt; APIs &gt;** [Auth0 Management API](https://manage.auth0.com/#/apis/management/explorer). Selecciona la pestaña **API Explorer** y busca un token generado automáticamente en la sección **Token**.
2. Selecciona el icono **Copy** a la derecha del token. Ahora puedes hacer llamadas autorizadas a la [Management API](https://auth0.com/docs/api/management/v2) con este token.

   <Frame>![APIs - Management API access tokens - Testing - Copy token](/docs/images/cdy7uua7fh8z/3Y4I9YJR77j73FCYDiegnC/44f32f76d27357bc39aad9ee7c4d5561/management-api-access-token.png)</Frame>
3. Opcionalmente, configura el tiempo de expiración. De forma predeterminada, el token tiene un tiempo de expiración de 86400 segundos, es decir, 24 horas. Después de ese período, el token caduca y ya no puede usarse. Para cambiar el tiempo de expiración, actualiza **Token Expiration (Seconds)** y haz clic en **Update &amp; Regenerate Token**.

<Warning>
  Estos tokens **no se pueden revocar**, por lo que no se recomiendan tiempos de expiración largos. En su lugar, te recomendamos usar tiempos de expiración cortos y emitir uno nuevo cada vez que lo necesites. Puedes verificar la expiración decodificando el token con [jwt.io](https://jwt.io/).
</Warning>

<div id="use-access-tokens-for-testing">
  ## Usa tokens de acceso para hacer pruebas
</div>

Puedes usar el <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">Token de acceso</Tooltip> que acabas de crear para hacer pruebas y llamar manualmente a un endpoint a través del [Management API Explorer](https://auth0.com/docs/api/management/v2).

1. Ve al [Management API Explorer](https://auth0.com/docs/api/management/v2#!) y selecciona el botón **Set API Token**.
2. Completa el campo **API Token**. A continuación, se mostrará información adicional en la ventana emergente, como los alcances otorgados a la aplicación.
3. Selecciona **Set Token** para guardar tu token y cerrar la ventana emergente. Observa que el botón Set API Token ahora aparece como **Authorized**.
4. Para comenzar a hacer pruebas, ve al endpoint que quieras invocar e introduce los parámetros obligatorios u opcionales en los campos de la derecha. Luego, selecciona **Test Endpoint**.
