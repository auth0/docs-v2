title: Estructura del token de ID
---

<Tooltip tip="token de ID: Credencial destinada al propio cliente, no para acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=ID+tokens">token de ID</Tooltip> siguen el estándar [JSON Web Token (JWT)](/es/docs/secure/tokens/json-web-tokens), lo que significa que su estructura básica se ajusta a la [estructura típica de JWT](/es/docs/secure/tokens/json-web-tokens/json-web-token-structure) y que contienen [claims de JWT](/es/docs/secure/tokens/json-web-tokens/json-web-token-claims) estándar sobre el propio token.

Además de lo requerido para <Tooltip tip="token de ID: Credencial destinada al propio cliente, no para acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=JWT">JWT</Tooltip>, los tokens de ID también contienen claims sobre el usuario autenticado, predefinidos por el protocolo <Tooltip tip="OpenID: Estándar abierto de autenticación que permite a las aplicaciones verificar la identidad de los usuarios sin recopilar ni almacenar información de inicio de sesión." cta="Ver glosario" href="/es/docs/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC), y por ello se conocen como claims estándar de OIDC. Algunos claims estándar de OIDC incluyen:

* `name`
* `nickname`
* `picture`
* `email`
* `email_verified`

Para obtener una lista completa de los claims estándar de OIDC, consulte [la especificación de OIDC: Claims estándar](https://openid.net/specs/openid-connect-core-1_0.html#StandardClaims). También puede crear [claims personalizados](/es/docs/secure/tokens/json-web-tokens/json-web-token-claims), que son claims que usted define, controla y agrega a un token mediante Actions de Auth0.

Usted controla qué claims de OIDC se incluyen en el token de ID que consume su aplicación al incluir [Scopes de OpenID Connect](/es/docs/get-started/apis/scopes/openid-connect-scopes) específicos en un parámetro al solicitar tokens durante la autenticación de usuarios. Para saber cómo solicitar un token de ID, consulte [Obtener token de ID](/es/docs/secure/tokens/id-tokens/get-id-tokens).

<div id="sample-id-token">
  ## Ejemplo de ID Token
</div>

Este ejemplo muestra el contenido de un token de ID. Observe que el valor de <Tooltip tip="Audiencia: identificador único de la audiencia de un token emitido. Se denomina aud en un token y su valor contiene el ID de una aplicación (ID de cliente) para un ID Token o de una API (Identificador de API) para un Token de acceso." cta="Ver glosario" href="/es/docs/glossary?term=audience">audiencia</Tooltip> (ubicado en el claim `aud`) se establece en el identificador de la aplicación, lo que significa que solo esta aplicación específica debe usar el token. Para obtener más información sobre las claims de este ejemplo, consulte [Claims de JSON Web Token](/es/docs/secure/tokens/json-web-tokens/json-web-token-claims).

```json lines
{
  "iss": "http://my-domain.auth0.com",
  "sub": "auth0|123456",
  "aud": "my_client_id",
  "exp": 1311281970,
  "iat": 1311280970,
  "name": "Jane Doe",
  "given_name": "Jane",
  "family_name": "Doe",
  "gender": "female",
  "birthdate": "0000-10-31",
  "email": "janedoe@example.com",
  "picture": "http://example.com/janedoe/me.jpg"
}
```
