title: Validar tokens de ID
---

Un [token de ID](/es/docs/secure/tokens/id-tokens), que contiene atributos del perfil del usuario, lo consume una aplicación y, por lo general, se usa para mostrar la interfaz de usuario. Auth0 emite todos los <Tooltip tip="ID Token: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=ID+tokens">tokens de ID</Tooltip> en formato [JSON Web Token (JWT)](/es/docs/secure/tokens/json-web-tokens).

Si alguna de estas comprobaciones falla, el token se considera no válido y la solicitud debe rechazarse.

1. [Valide el JWT](/es/docs/secure/tokens/json-web-tokens/validate-json-web-tokens).
2. Verifique claims estándar adicionales. Si ya realizó la validación estándar del JWT, ya ha decodificado la [carga útil del JWT](/es/docs/secure/tokens/json-web-tokens/json-web-token-structure) y ha revisado sus claims estándar. Los claims adicionales que debe verificar en los tokens de ID incluyen:

   * **Audiencia del token** (`aud`, string): El valor de audiencia del token debe coincidir con el ID de cliente de la aplicación, tal como se define en la [configuración de la aplicación](https://manage.auth0.com/#/applications), en el campo **Client ID**.
   * **Nonce** (`nonce`, string): Se recomienda pasar un nonce en la solicitud del token (es obligatorio para el flujo implícito) para ayudar a prevenir ataques de repetición. El valor de nonce en el token debe coincidir exactamente con el nonce original enviado en la solicitud. Consulte [Mitigar ataques de repetición](/es/docs/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post/mitigate-replay-attacks-when-using-the-implicit-flow) para obtener más información.
