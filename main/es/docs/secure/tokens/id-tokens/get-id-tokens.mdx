title: Obtener token de ID
---

Para obtener un <Tooltip tip="token de ID: Credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=ID+token">token de ID</Tooltip>, debes solicitarlo al autenticar a los usuarios. Auth0 facilita la autenticación de usuarios en tu aplicación mediante:

* [Quickstarts](/es/docs/quickstarts): La forma más sencilla de implementar la autenticación, que puede mostrarte cómo usar [Universal Login](/es/docs/authenticate/login/auth0-universal-login), el widget Lock y los SDK de Auth0 específicos de cada lenguaje y framework. Nuestra [documentación de Lock](/es/docs/libraries/lock) y la [documentación de Auth0.js](/es/docs/libraries/auth0js) proporcionan detalles específicos sobre cómo obtener un token de ID después de la autenticación.
* [Authentication API](https://auth0.com/docs/api/authentication): Si prefieres crear tu propia implementación, puedes llamar directamente a nuestra API. Primero, debes saber [qué flujo usar](/es/docs/get-started/authentication-and-authorization-flow/which-oauth-2-0-flow-should-i-use) antes de seguir el tutorial del flujo correspondiente.

<div id="control-id-token-contents">
  ## Controle el contenido del token de ID
</div>

Puede controlar qué claims sobre el usuario autenticado se incluyen en el token de ID que consume su aplicación al incluir [alcances de OpenID Connect](/es/docs/get-started/apis/scopes/openid-connect-scopes) específicos en el parámetro `scope` cuando solicita tokens durante la autenticación de usuarios.

También puede crear [claims personalizados](/es/docs/secure/tokens/json-web-tokens/json-web-token-claims), es decir, claims que usted define, controla y agrega a un token mediante una Rule. Al igual que con otros <Tooltip tip="JSON Web Token (JWT): formato estándar de ID Token (y, con frecuencia, de Token de acceso) utilizado para representar claims de forma segura entre dos partes." cta="Ver glosario" href="/es/docs/glossary?term=JWTs">JWTs</Tooltip>, debe [validar un token de ID](/es/docs/secure/tokens/id-tokens/validate-id-tokens) antes de dar por sentado que se puede confiar en su contenido.

<div id="renew-id-tokens">
  ## Renovar token de ID
</div>

De forma predeterminada, un token de ID es válido durante 36000 segundos (10 horas). Si hay consideraciones de seguridad, puede [acortar el tiempo antes de que el token expire](/es/docs/secure/tokens/id-tokens/update-id-token-lifetime), pero recuerde que uno de los propósitos de este token es mejorar el rendimiento al almacenar en caché la información del usuario.

Una vez que un token de ID haya expirado, es posible que quiera renovarlo. Para renovar el token de ID, puede volver a autenticar al usuario con Auth0 o usar un [Token de actualización](/es/docs/secure/tokens/refresh-tokens).
