title: 'RGPD: derecho a acceder a los datos, rectificarlos y eliminarlos'

De acuerdo con los artículos 15, 16, 17 y 19 del RGPD, los usuarios tienen derecho a obtener una copia de los datos personales que procesas, a solicitar su rectificación si son inexactos y a pedirte que elimines sus datos personales.

Con Auth0, puedes acceder a la información del usuario, editarla y eliminarla manualmente mediante el Dashboard o de forma programática mediante la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>.

</Callout>

<div id="manual-process">
  ## Proceso manual
</div>

Puede ver, editar y eliminar la información de los usuarios en [Auth0 Dashboard &gt; User Management &gt; Users](https://manage.auth0.com/#/users). Seleccione un usuario para ver su información. La información que puede cambiar es la siguiente:

| Campo                             | Cómo editar                                                                                                                                                                                         |
| --------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Correo electrónico**            | Haga clic en **Edit**. Establezca el nuevo correo electrónico.                                                                                                                                      |
| **Correo electrónico verificado** | Haga clic en **Edit** en el campo **Εmail**. Haga clic en el enlace **Set email as verified**.                                                                                                      |
| **Metadatos**                     | Tanto los objetos `app_metadata` como `user_metadata` se pueden editar desde esta pantalla. Edite el JSON en la sección **Metadata** y guarde los cambios.                                          |
| **Bloqueado**                     | No se puede editar directamente. Haga clic en **Actions &gt; Block User** en la esquina superior derecha de esta pantalla. Para desbloquear al usuario, haga clic en **Actions &gt; Unblock User**. |
| **Correo electrónico**            | No se puede editar directamente. Haga clic en **Actions &gt; Change Email** en la esquina superior derecha de esta pantalla.                                                                        |
| **Contraseña**                    | No se puede editar directamente. Haga clic en **Actions &gt; Change Password** en la esquina superior derecha de esta pantalla.                                                                     |

Para eliminar un usuario, haga clic en **Actions &gt; Delete User**.

<div id="programmatic-process">
  ## Proceso programático
</div>

Puede recuperar, editar y eliminar información del usuario mediante nuestra API. Elija un endpoint que se ajuste a sus necesidades:

* Recuperar un usuario usando el ID como criterio de búsqueda
* Recuperar un usuario usando el correo electrónico como criterio de búsqueda
* Exportar todos los usuarios a un archivo mediante un trabajo de ejecución prolongada
* Actualizar un usuario. Tenga en cuenta que no todos los campos son editables (consulte la siguiente sección). Tenga presente que:

  * Las propiedades del objeto nuevo reemplazarán a las anteriores. Los campos `user_metadata` y `app_metadata` son una excepción a esta regla. Estas propiedades se combinan en lugar de reemplazarse, aunque la combinación solo ocurre en el primer nivel.
  * Si está actualizando `email_verified`, `phone_verified`, `username` o `password`, debe establecer el parámetro `connection`.
  * Si está actualizando `email` o `phone_number`, debe establecer los parámetros `connection` y `client_id`.
* Eliminar un usuario en función del ID

Para llamar a cualquiera de los endpoints de la API, necesitará un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip> válido. Este token debe tener los permisos requeridos para cada endpoint. Cada endpoint del explorador de Management API tiene una sección de scopes que enumera los scopes que debe contener el token de acceso para poder acceder a él. Una vez que sepa a qué endpoint quiere acceder y tenga un token de acceso válido, estará listo para enviar la solicitud.

<div id="editable-data">
  ## Datos editables
</div>

La siguiente información del usuario puede actualizarse mediante la API:

* blocked
* email&#95;verified
* email
* verify&#95;email
* password
* phone&#95;number
* phone&#95;verified
* verify&#95;password
* user&#95;metadata
* app&#95;metadata
* username

La siguiente información del usuario **no** se puede editar:

* given&#95;name
* family&#95;name
* name
* nickname
* picture

<div id="searchable-fields">
  ## Campos de búsqueda
</div>

Puede buscar usuarios por los siguientes campos:

* Todos los campos normalizados del perfil de usuario
* La información del perfil del objeto `user_metadata`:

  * name
  * nickname
  * given&#95;name
  * family&#95;name

Usted es responsable de asegurarse de que se elimine al cliente o de que se actualicen los datos en cualquier otra base de datos a la que Auth0 no esté conectado.
