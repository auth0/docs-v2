---
title: Evaluación de Rules para Tenant ACL
sidebarTitle: Evaluación de Rules
description: Conozca cómo el motor determinista de Tenant ACL evalúa las coincidencias y la ejecución de las Rules.
---

Tenant ACL usa un motor determinista para evaluar las solicitudes entrantes. Para determinar si se debe realizar una acción (permitir, bloquear o redirigir), el motor sigue estos pasos de evaluación:

1. Orden de evaluación (prioridad)

2. Coincidencia de condiciones (señal)

3. Finalización de coincidencias con excepción del modo de supervisión

<div id="evaluation-order">
  ## Orden de evaluación
</div>

Tenant ACL evalúa las reglas en orden ascendente según su número de prioridad. La evaluación comienza con el número más bajo, la prioridad 0, y continúa secuencialmente.

Si dos reglas pueden coincidir con una solicitud, la que tenga el número de prioridad más bajo se activará primero.

<div id="condition-matching">
  ## Coincidencia de condiciones
</div>

Para que una sola regla active su acción, debe cumplir la lógica de coincidencia. Esta lógica determina cómo el motor procesa varios valores y varios tipos de señales:

| Tipo de lógica          | Comportamiento | Descripción                                                                                                                                                                                                                                                   | Ejemplo                                                                           |
| ----------------------- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------- |
| Coincidencia de señales | OR             | Dentro de una señal específica, una solicitud debe coincidir con al menos un valor de la lista para cumplir ese requisito de señal.                                                                                                                           | Si `ipv4_cidrs` contiene varios CIDR de IP, al menos uno de ellos debe coincidir. |
| Varias señales          | AND            | Si una regla contiene varios tipos de señales diferentes, la solicitud debe coincidir con todos los tipos de señales proporcionados. Como excepción, si una regla contiene tanto `ipv4_cidrs` como `ipv6_cidrs`, solo es necesario que coincida uno de ellos. | Si una regla tiene tanto `ipv4_cidrs` como `asn`, ambos deben coincidir.          |

Los tipos de señales que no se proporcionen en una regla se ignoran y no afectan a la lógica de evaluación de la regla.

<div id="match-termination">
  ## Finalización de la coincidencia
</div>

Tenant ACL aplica solo la primera coincidencia. Una vez que se cumplen por completo las condiciones de una regla, el comportamiento depende de si la regla está en [modo de supervisión](./enable-monitoring-mode) o no.

| Comportamiento | Modo de supervisión deshabilitado                                                             | Modo de supervisión habilitado                                            |
| -------------- | --------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------- |
| **Registro**   | Se genera un evento de registro `acls_summary`.                                               | Se genera un evento de registro `acls_summary`.                           |
| **Ejecución**  | La acción definida de la regla se ejecuta de inmediato.                                       | La acción definida de la regla no se ejecuta.                             |
| **Evaluación** | La evaluación finaliza. No se evalúan reglas de Tenant ACL posteriores ni de menor prioridad. | La evaluación continúa con la siguiente regla de la lista de prioridades. |
