<Card path="Antes de comenzar">
  Para configurar una Rule de Tenant ACL, necesita un [token de acceso para Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con los siguientes alcances:

  * `create:network_acls`
  * `update:network_acls`
  * `read:network_acls`
  * `delete:network_acls`
</Card>

Puede configurar Rules de lista de control de acceso (ACL) del inquilino con la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> de Auth0.

<div id="available-actions">
  ## Acciones disponibles
</div>

Puede ver, crear, actualizar y eliminar Rules de Tenant ACL con la Management API.

| Acción               | Endpoint                                                                                                                                       | scope requerido       |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- | --------------------- |
| Ver una Rule        | [Obtener una entrada específica de la lista de control de acceso de un inquilino](/es/docs/api/management/v2/network-acls/get-network-acls-by-id) | `read:network_acls`   |
| Ver todas las Rules | [Obtener todas las entradas de la lista de control de acceso de un inquilino](/es/docs/api/management/v2/network-acls/get-network-acls)           | `read:network_acls`   |
| Crear una Rule      | [Crear una lista de control de acceso](/es/docs/api/management/v2/network-acls/post-network-acls)                                                 | `create:network_acls` |
| Actualizar una Rule | [Actualización parcial de una lista de control de acceso](/es/docs/api/management/v2/network-acls/patch-network-acls-by-id)                       | `update:network_acls` |
| Reemplazar una Rule | [Actualizar lista de control de acceso](/es/docs/api/management/v2/network-acls/put-network-acls-by-id)                                           | `update:network_acls` |
| Eliminar una Rule   | [Eliminar lista de control de acceso](/es/docs/api/management/v2/network-acls/delete-network-acls-by-id)                                          | `delete:network_acls` |

<div id="parameters">
  ## Parámetros
</div>

Para obtener información detallada sobre los parámetros de Tenant ACL y cómo usarlos, consulta [Referencia](./reference).

<ParamField path="description" type="string" required>
  Describe el propósito o la funcionalidad de la Rule.

  Ejemplo: `Permitir solo solicitudes originadas en Estados Unidos`
</ParamField>

<ParamField path="active" type="boolean" required>
  Habilita o deshabilita la Rule.

  Ejemplo: `true`
</ParamField>

<ParamField path="priority" type="number" required>
  Valor numérico que determina el orden en que se evalúa la Rule.

  Ejemplo: `1`
</ParamField>

<ParamField path="rule" type="object" required>
  Contiene las condiciones y acciones de la Rule.

  <Expandable>
    <ParamField path="action" type="object" required>
      Contiene la acción que ejecuta la Rule.

      Ejemplo: `{ allow: true }`
    </ParamField>

    <ParamField path="match" type="object">
      Define las condiciones que debe cumplir la solicitud entrante.

      Ejemplo: `{ geo_country_codes: ["US"] }`
    </ParamField>

    <ParamField path="not_match" type="object">
      Define las condiciones que no debe cumplir la solicitud entrante.

      Ejemplo: `{ geo_country_codes: ["CN"] }`
    </ParamField>

    <ParamField path="scope" type="string" required>
      Servicio o contexto en el que se aplica la Rule.

      Ejemplo: `authentication`
    </ParamField>
  </Expandable>
</ParamField>

  ## Activar o desactivar el modo de supervisión para una Rule

Agregue la propiedad `log` al objeto `rule.action` y establezca su valor en `true`.
