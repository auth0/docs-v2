title: Cómo configurar Rules de la lista de control de acceso del Tenant
sidebarTitle: Configurar Rules
description: Defina la señal, la condición, la acción, el scope y la prioridad de una Rule para establecer su comportamiento. Use la Management API o los SDK, las CLI o el proveedor de Terraform de Auth0 para configurar y administrar el Tenant ACL.
---

Una Rule de Tenant ACL se compone de los siguientes elementos:

* **Señal**: La señal es un dato identificador que proporciona la solicitud entrante, como la dirección IP, la geolocalización o el agente de usuario.

* **Condición**: La condición es la combinación de un operador (como `match`) y un conjunto de valores (como una lista de direcciones IP).

* **Acción**: La acción es la directiva que ejecuta su Rule si se cumplen los criterios, como permitir, bloquear o redirigir.

* **Scope**: El scope indica para qué conjuntos de endpoints se aplica la Rule, incluida la Authentication API, la Management API o todo su Tenant.

* **Prioridad**: La prioridad define el orden en que se ejecuta la Rule en relación con otras Rules.

Para obtener información detallada sobre estas propiedades de las Rules de Tenant ACL y cómo usarlas, consulte la [Referencia de parámetros del endpoint de la Management API](./reference).

<Note>
  Tenant ACL utiliza una [lógica uniforme de evaluación de Rules](./rule-evaluation) para determinar qué acción, si corresponde, aplicar a una solicitud determinada.

  Asigne cuidadosamente las prioridades para crear políticas de control de acceso granulares para su caso de uso.
</Note>

<div id="create-and-delete-rules">
  ## Crear y eliminar Rules
</div>

Puede crear y eliminar Rules de Tenant ACL con los siguientes endpoints de la Management API. Necesita un [token de acceso para la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con los alcances indicados.

| Acción            | Endpoint                                                                                              | Alcance requerido     |
| ----------------- | ----------------------------------------------------------------------------------------------------- | --------------------- |
| Crear una Rule    | [Crear lista de control de acceso](/es/docs/api/management/v2/network-acls/post-network-acls)            | `create:network_acls` |
| Eliminar una Rule | [Eliminar lista de control de acceso](/es/docs/api/management/v2/network-acls/delete-network-acls-by-id) | `delete:network_acls` |

<div id="modify-rules">
  ## Modificar Rules
</div>

Puede modificar Rules de Tenant ACL mediante los siguientes endpoints de la Management API. Necesita un [token de acceso para la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con los alcances indicados.

| Acción              | Endpoint                                                                                                                 | scope requerido       |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------ | --------------------- |
| Actualizar una Rule | [Actualización parcial de una lista de control de acceso](/es/docs/api/management/v2/network-acls/patch-network-acls-by-id) | `update:network_acls` |
| Reemplazar una Rule | [Actualizar lista de control de acceso](/es/docs/api/management/v2/network-acls/put-network-acls-by-id)                     | `update:network_acls` |

<div id="enable-or-disable-rules">
  ### Habilitar o deshabilitar Rules
</div>

<Tabs>
  <Tab title="Management API">
    Puede habilitar o deshabilitar una Rule con el endpoint [Update access control list](https://auth0.com/docs/api/management/v2/network-acls/put-network-acls-by-id) de Management API.

    Establezca la propiedad `active` del objeto `rule` en `true` o `false`, respectivamente.
  </Tab>

  <Tab title="Dashboard">
    Para habilitar o deshabilitar una Rule desde el Dashboard, vaya a [Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control).

    En la lista de Rules existentes, puede habilitar o deshabilitar Rules específicas con el interruptor situado junto a cada una.

    ![Habilitar o deshabilitar una Rule de Tenant ACL en el Auth0 Dashboard](/docs/images/cdy7uua7fh8z/29b92e102142d3cb20e69a40e292f6b7/Screenshot_2025-09-03_at_12.34.05.png)

    La columna **Status** muestra el estado de la Rule (**Active** o **Disabled**).
  </Tab>
</Tabs>

<div id="view-rules">
  ## Ver Rules
</div>

<Tabs>
  <Tab title="Management API">
    Puede ver las Rules de Tenant ACL existentes mediante los siguientes endpoints de la Management API. Necesita un [token de acceso para la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) con los alcances indicados.

    | Acción              | Endpoint                                                                                                                                       | scope requerido     |
    | ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- | ------------------- |
    | Ver una Rule        | [Obtener una entrada específica de la lista de control de acceso de un inquilino](/es/docs/api/management/v2/network-acls/get-network-acls-by-id) | `read:network_acls` |
    | Ver todas las Rules | [Obtener todas las entradas de la lista de control de acceso de un inquilino](/es/docs/api/management/v2/network-acls/get-network-acls)           | `read:network_acls` |
  </Tab>

  <Tab title="Dashboard">
    Vaya a [Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control) para ver una lista de sus Rules existentes.

    | Nombre de la columna | Descripción                                                                                 | Campo de respuesta de la API relacionado |
    | -------------------- | ------------------------------------------------------------------------------------------- | ---------------------------------------- |
    | Prioridad            | Número ordinal que indica cuándo se evalúa la Rule                                          | `priority`                               |
    | Nombre               | Descripción del propósito o la funcionalidad de la Rule                                     | `description`                            |
    | Acción               | La [acción](/es/docs/secure/tenant-access-control-list/reference#actions) que realiza la Rule. | `action`                                 |
    | Estado               | Valor que indica si la Rule está activa o deshabilitada.                                    | `active`                                 |
    | Última actualización | Marca de fecha y hora de la última actualización de la Rule.                                | `updated_at`                             |

    ![Ver la lista de Rules de Tenant ACL en el Auth0 Dashboard](/docs/images/cdy7uua7fh8z/4093f9a39aba48bdf8d579c134dc4da5/Screenshot_2025-09-03_at_12.33.47.png)
  </Tab>
</Tabs>
