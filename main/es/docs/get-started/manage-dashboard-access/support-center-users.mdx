title: Acceso al Dashboard por rol
---

Como administrador del inquilino, puedes asignar roles a tus colegas para otorgarles acceso limitado al <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Esto permite que los miembros del inquilino realicen su trabajo sin poner en riesgo las aplicaciones de producción, a la vez que se cumple el principio de privilegio mínimo.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Todos los roles tienen acceso a [Support Center](https://support.auth0.com/).
</Callout>

<div id="dashboard-roles">
  ## Roles del Dashboard
</div>

Puede asignar los siguientes roles para acceder a Auth0 Dashboard:

| Rol                             | Permisos                                                                                                                                                                                                  |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Admin**                       | Acceso de lectura y escritura a todos los recursos de Auth0 Dashboard, incluidas las extensiones.                                                                                                         |
| **Editor - Connections**        | Acceso de lectura, escritura y creación a todo tipo de conexiones.                                                                                                                                        |
| **Editor - Key Management**     | Crear y administrar claves criptográficas.                                                                                                                                                                |
| **Editor - Organizations**      | Acceso de lectura, escritura y creación a la configuración de Organización. Acceso de solo lectura a Usuarios, Roles y Conexiones.                                                                        |
| **Editor - Specific Apps**      | Acceso de lectura y escritura únicamente a aplicaciones específicas.                                                                                                                                      |
| **Editor - Users**              | Operaciones de administración de usuarios (crear, eliminar, bloquear, desbloquear, restablecer MFA, restablecer contraseñas, actualizar metadatos, asignar roles, etc.) y acceso a los logs.              |
| **Viewer - Users**              | Acceso de solo lectura a los usuarios y a los logs relacionados con ellos.                                                                                                                                |
| **Viewer - Config Settings**    | Acceso de solo lectura a todos los ajustes de configuración (aplicaciones, API, Rules, ajustes de seguridad, etc.), excepto a información confidencial, como secretos, facturación, usuarios y logs.      |
| **Support Access - Deprecated** | Acceso a tickets (enviar, ver y actualizar) y a métricas agregadas.                                                                                                                                       |
| **Elevated Support Access**     | Acceso de lectura, escritura y creación a **todos** los tickets de soporte creados por **todos los usuarios** de un inquilino (Subscripton Tickets en Auth0 Support Center). Acceso a métricas agregadas. |

<Card title="La disponibilidad varía según el plan de Auth0">
  Su plan de Auth0 o acuerdo personalizado afecta la disponibilidad de esta funcionalidad. Para obtener más información, consulte [Auth0&#39;s Pricing Page](https://auth0.com/pricing).
</Card>

Los roles de acceso limitado restringen la experiencia de un miembro del inquilino en el Dashboard solo a las secciones y acciones necesarias para su trabajo.

A continuación se describen los permisos de funcionalidad asociados a cada rol. Si no se indica un permiso específico para un rol, los miembros del inquilino a los que se les asigne ese rol no tendrán acceso a esa área de Auth0 Dashboard.

<div id="log-events-available-to-user-roles">
  ## Eventos de log disponibles para los roles de usuario
</div>

Los logs pueden contener datos confidenciales, como secretos, PII, etc. Es importante no divulgar datos confidenciales a usuarios cuyo rol no requiera esa información. Sin embargo, los roles **Editor - Users** y **Viewer - Users** necesitan cierto acceso a los logs para identificar problemas de los usuarios. Por ejemplo, para comprobar si el usuario se registró correctamente o si fue bloqueado.

Permitimos que **Editor - Users** y **Viewer - Users** accedan a un conjunto limitado de tipos de log relacionados con eventos de usuario. Los eventos de log de la lista proporcionan la información necesaria sobre las acciones de los usuarios, pero no revelan información confidencial sobre otras partes de la configuración del inquilino. Para obtener más información, consulte [Log Event Type Codes](/es/docs/deploy-monitor/logs/log-event-type-codes).

<div id="limitations">
  ## Limitaciones
</div>

* Los usuarios con el rol **Admin** pueden invitar a usuarios **Editor - Specific Apps** a una sola aplicación a la vez. Como alternativa, el usuario **Admin** puede [editar su rol](/es/docs/get-started/manage-dashboard-access/edit-dashboard-users) para asignarle varias aplicaciones después de que el usuario acepte la invitación.
* Los roles **Viewer - Users** y **Editor - Users** no tienen acceso a las secciones **Devices** y **Authorized Apps** de **Users**.
* La [New Activity Page](/es/docs/get-started/auth0-overview/dashboard/activity) solo es visible para los usuarios **Admins** y **Elevated Support Access**. **Editor - Users** y **Viewer - Users** pueden acceder a la actividad diaria (como inicios de sesión o registros) a través de la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Stats/get_daily).

<div id="private-cloud-requirements">
  ## Requisitos de Private Cloud
</div>

Los roles **Editor - Users** y **Viewer - Users** requieren que User Search v3 y Logs Search v3 estén habilitados en tu entorno. Si tus entornos no admiten estas versiones, esos dos roles no estarán disponibles.
