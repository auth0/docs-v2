Auth0 ofrece una amplia variedad de recursos para ayudarle a aprovechar eficazmente nuestro producto y nuestra comunidad. Esta lista incluye enlaces a los recursos disponibles, por categoría.

<div id="get-started">
  ## Primeros pasos
</div>

Los recursos diseñados para ayudarle a aprender los conceptos básicos de Auth0 incluyen:

* [**Documentación de primeros pasos**](/es/docs/get-started): Explore el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> y los términos habituales que se usan para los componentes del servicio de Auth0. Obtenga una visión general de Auth0 y familiarícese con la terminología que podría escuchar al trabajar con el equipo de Auth0 o al leer nuestra documentación.
* [**Escenarios de arquitectura**](/es/docs/get-started/architecture-scenarios): Revise escenarios de arquitectura comunes y aprenda a implementarlos con Auth0. Los escenarios incluyen tutoriales sobre patrones de arquitectura comunes, como una aplicación de una sola página (SPA) que consume una API. Las descripciones generales son útiles para arquitectos, mientras que los tutoriales ayudarán a los equipos de desarrollo.
* [**Guías de implementación**](/es/docs/): Aprenda a implementar funcionalidades habituales, como la gestión de usuarios y la <Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor además del nombre de usuario y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=multi-factor+authentication">autenticación multifactor</Tooltip> (MFA). Esta información es útil tanto para arquitectos como para desarrolladores

<div id="learn">
  ## Aprenda
</div>

Auth0 ofrece numerosos tutoriales, guías, documentos técnicos y publicaciones de blog pensados tanto para aprender como para consultar referencias rápidas.

* [**Sitio de documentación**](/es/docs/): Explore nuestra documentación para descubrir una amplia variedad de temas, o use la búsqueda para encontrar rápidamente contenido relacionado con un tema o término.
* [**Descripciones de funcionalidades y documentos técnicos**](https://auth0.com/learn/): Consulte descripciones breves de funcionalidades, casos prácticos del sector y documentos técnicos de referencia. El sitio Auth0 Learn ofrece una visión general rápida de las funcionalidades de Auth0 y su valor para el negocio; resulta útil tanto para responsables de proyectos como para arquitectos y desarrolladores.
* [**Publicaciones de blog**](https://auth0.com/blog/): Lea publicaciones de blog escritas por expertos sobre una gran variedad de temas, desde consejos de siempre hasta noticias de última hora en el ámbito de la identidad. Muchas de estas publicaciones están dirigidas a arquitectos y desarrolladores.

<div id="run-sample-code">
  ## Ejecuta código de ejemplo
</div>

Una vez que tu equipo de desarrollo esté listo para empezar a crear, Auth0 ofrece programas de ejemplo, SDK y bibliotecas para acelerar tu proyecto.

* [**Quickstarts**](/es/docs/quickstarts): Explora una amplia variedad de pequeños programas de ejemplo que muestran cómo implementar las funciones clave que querrás incluir en tu aplicación, como autenticación, gestión de sesiones, actualizaciones del perfil y cierre de sesión. Los Quickstarts permiten a los desarrolladores empezar con ventaja a entender cómo integrar aplicaciones con Auth0.
* [Bibliotecas y SDK](/es/docs/libraries): Simplifica el desarrollo de tu aplicación personalizada con nuestro amplio conjunto de bibliotecas y SDK, que abstraen muchos de los detalles de los protocolos de identidad. Los SDK de Auth0 son compatibles con numerosos lenguajes y frameworks para simplificar tu integración con Auth0. También ofrecemos una biblioteca para Lock, un widget de inicio de sesión, que puedes usar en varias plataformas, incluidas iOS y Android.
* [**Management API**](https://auth0.com/docs/api/management/v2): Explora y administra objetos, configuraciones y ajustes dentro de Auth0. El Management API Explorer de <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> te permite modificar rápidamente objetos y ajustes individuales según sea necesario y probar llamadas a la API antes de incorporarlas al código de tus aplicaciones.
* [**Authentication API**](https://auth0.com/docs/api/authentication): Autentica y autoriza usuarios mediante los protocolos OIDC, <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/es/docs/glossary?term=OAuth">OAuth</Tooltip> y <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/es/docs/glossary?term=SAML">SAML</Tooltip>. El Authentication API Explorer te permite experimentar con la autenticación y los <Tooltip tip="Flujo de autorización: Concesión de autorización (o flujo de trabajo) especificada en el marco de OAuth 2.0." cta="Ver glosario" href="/es/docs/glossary?term=authorization+flows">flujos de autorización</Tooltip> y probar llamadas a la API antes de incorporarlas al código de tus aplicaciones.

<div id="try-out-features-and-api-calls">
  ## Pruebe funciones y llamadas a la API
</div>

Durante el desarrollo, Auth0 le permite experimentar y probar varias funciones del producto.

* **Botones/enlaces TRY**: Pruebe rápidamente las funciones del producto de Auth0. Los botones **TRY** están disponibles en todo Auth0 y le permiten experimentar con conexiones, Rules, Hooks y plantillas de correo electrónico.
* [**Authentication API Explorer**](https://auth0.com/docs/api/authentication#introduction): Experimente con flujos de autenticación y autorización, y pruebe llamadas a la API antes de implementarlas en sus aplicaciones.
* [**Management API Explorer**](https://auth0.com/docs/api/management/v2): Modifique rápidamente objetos y configuraciones individuales de forma ad hoc, y pruebe llamadas a la API antes de implementarlas en sus aplicaciones.

<div id="get-help">
  ## Obtener ayuda
</div>

Los recursos de Auth0 que pueden ayudarle a solucionar problemas de su implementación incluyen:

* [**Foro de la comunidad de Auth0**](https://community.auth0.com/): Conéctese con el ecosistema de Auth0 mediante publicaciones de Auth0, preguntas frecuentes y preguntas y respuestas de la comunidad. Los arquitectos y desarrolladores encontrarán aquí una valiosa fuente de información para aprender, conectar con otras personas y obtener ayuda con problemas.
* [**Support Center**](https://support.auth0.com/): Vea y administre su suscripción y sus inquilinos, envíe y consulte solicitudes de soporte, ejecute comprobaciones automatizadas de producción en un inquilino y consulte información de cumplimiento. Los suscriptores de pago encontrarán en el Support Center un recurso valioso si un problema o una pregunta no pueden resolverse con la documentación o buscando en el foro.

  * [**Crear casos de soporte**](/es/docs/troubleshoot/customer-support/open-and-manage-support-tickets): Cree y envíe un caso de soporte si necesita ayuda.
  * [**Planes de soporte y acuerdos de nivel de servicio**](/es/docs/troubleshoot/customer-support#defect-responses): Obtenga información sobre los distintos niveles de soporte disponibles para su compra.
  * [**Consejos de solución de problemas**](/es/docs/deploy-monitor/deploy-private-cloud/private-cloud-on-aws#what-to-check-before-logging-an-issue) y [**Información para incluir en su caso de soporte**](/es/docs/deploy-monitor/deploy-private-cloud/private-cloud-on-aws#information-to-provide-when-logging-an-issue): Obtenga recomendaciones para ayudar a sus equipos de desarrollo y soporte a analizar los problemas.
* [**Versiones compatibles**](/es/docs/troubleshoot/customer-support/product-support-matrix): Comprenda qué versiones de SDK, navegadores y lenguajes son compatibles. Los arquitectos y desarrolladores deben revisar esto para asegurarse de que su proyecto usa lenguajes, bibliotecas y SDK que permitirán que su implementación funcione con Auth0.
* [**Portal de comentarios**](https://auth0.com/feedback): Envíe sugerencias sobre el producto. (También puede hacerlo a través del Support Center si quiere tener mejor visibilidad de lo que ha enviado a lo largo del tiempo). Los arquitectos y desarrolladores pueden usar este sitio para enviar comentarios sobre el producto Auth0 que puedan considerarse como mejoras en el futuro.
* [**Servicios profesionales**](/es/docs/get-started/professional-services): Recurra a nuestro equipo global de servicios profesionales para ayudar a acelerar el éxito de su proyecto. Los responsables de proyectos encontrarán esto útil para entender cómo los expertos en identidad de Auth0 pueden ayudar a acelerar su proyecto o cubrir carencias temporales de habilidades.

<div id="set-up-and-monitor-operations">
  ## Configure y supervise las operaciones
</div>

Cuando esté listo para planificar su lanzamiento, Auth0 le ofrece los siguientes recursos:

* [**Consejos previos al lanzamiento**](/es/docs/deploy-monitor/pre-deployment-checks) y [**Verificación de producción**](/es/docs/deploy-monitor/pre-deployment-checks/how-to-run-production-checks): Obtenga consejos y herramientas que le ayudarán a planificar su lanzamiento. Los gerentes de proyecto y los equipos de desarrollo y operaciones deberían consultar estas recomendaciones para facilitar un lanzamiento sin contratiempos.
* [**Políticas operativas**](/es/docs/troubleshoot/customer-support/operational-policies): Familiarícese con las políticas de Auth0 para conocer los plazos de las solicitudes operativas. Estas políticas son útiles para todo el equipo, pero los responsables del proyecto y, en particular, los equipos de operaciones deberían conocer las políticas operativas de Auth0.
* [**Status Dashboard**](https://status.auth0.com/): Determine rápidamente la disponibilidad de los servicios de Auth0 y suscríbase a las actualizaciones de estado. Aunque las interrupciones del servicio son poco frecuentes, cuando se produce una, Auth0 realiza un análisis de causa raíz y publica los resultados en este sitio. Los equipos de operaciones y soporte deberían saber cómo comprobar el estado de Auth0.
* [**Supervisar endpoints**](/es/docs/deploy-monitor): Aprenda a integrar nuestros endpoints de monitorización en su infraestructura de monitorización. Esta información es especialmente útil para los equipos de operaciones y los responsables del proyecto.
* [**Datos de registro**](/es/docs/deploy-monitor/logs): Conozca los tipos de registros que ofrece Auth0, la retención de datos de registro y las herramientas que puede usar para exportar datos de registro a herramientas analíticas externas para su análisis y almacenamiento a largo plazo. Esta información es útil para desarrolladores y equipos de operaciones, así como para equipos de cumplimiento interesados en la retención de datos.
* [**Avisos del Dashboard**](/es/docs/get-started/architecture-scenarios/business-to-consumer/operations#notifications): Manténgase informado sobre anuncios importantes de Auth0. De vez en cuando, Auth0 le notifica información importante a través de su Auth0 Dashboard y (según la gravedad de la información) por correo electrónico a los administradores registrados de su Auth0 Dashboard. Debería iniciar sesión regularmente en el Dashboard y comprobar el icono de campana en la parte superior para ver si hay avisos importantes.
