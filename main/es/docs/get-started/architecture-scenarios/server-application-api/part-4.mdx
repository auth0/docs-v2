En el [escenario de arquitectura de servidor + API](/es/docs/get-started/architecture-scenarios/server-application-api), abordamos un caso sencillo de comunicación de máquina a máquina: una API, utilizada para importar entradas de hojas de horas en los sistemas de ExampleCo, y una tarea cron, utilizada por contratistas externos para enviar sus hojas de horas mediante esta API.

Aprendimos sobre el flujo Client Credentials Grant y los <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+tokens">tokens de acceso</Tooltip>, cómo configurar una API en Auth0, cómo configurar una aplicación de máquina a máquina para comunicarse de forma segura con esta API, cómo definir y proteger los endpoints de nuestra API, cómo usar las bibliotecas proporcionadas para validar el token de acceso y cómo obtener uno nuevo de Auth0.

Usamos Node.js para implementar la API y Python para el proceso de servidor no interactivo. Esperamos que este ejemplo te ayude a crear tu propia implementación con las tecnologías que prefieras.
