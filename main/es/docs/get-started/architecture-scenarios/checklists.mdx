Haga clic en los enlaces a continuación para descargar una lista de verificación correspondiente a una fase del SDLC (ciclo de vida del desarrollo de software). Puede abrir la lista de verificación en cualquier aplicación de hojas de cálculo y personalizarla según sus necesidades.

<div id="analyze-checklist">
  ## Lista de verificación para el análisis
</div>

[Descargar la lista de verificación para el análisis](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Analyze-Checklist.xlsx)

En la fase de análisis, analice los requisitos de negocio de los usuarios finales y determine los objetivos del proyecto como parte del plan general. Convierta los requisitos y los objetivos en funciones del sistema que la organización pretende desarrollar. Las actividades incluyen:

* Recopilar requisitos de negocio
* Crear diagramas de procesos
* Realizar un análisis detallado
* Alinear con el plan del proyecto

<div id="design-checklist">
  ## Lista de verificación de diseño
</div>

[Descargar la lista de verificación de diseño](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Design-Checklist.xlsx)

En la fase de diseño, describa las funciones y operaciones previstas del sistema, incluidas las reglas de negocio, el pseudocódigo, los diseños de pantalla y cualquier otra documentación necesaria. Las actividades incluyen:

* Diseño de la infraestructura
* Diseño del modelo del sistema

<div id="build-checklist">
  ## Lista de verificación de desarrollo
</div>

[Descargar la lista de verificación de desarrollo](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Build-Checklist.xlsx)

En la fase de desarrollo, se desarrolla el sistema mediante la implementación de la infraestructura y el código. Las actividades incluyen:

* Implementación de la infraestructura
* Implementación del código

<div id="test-checklist">
  ## Lista de verificación de pruebas
</div>

[Descargar lista de verificación de pruebas](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Test-Checklist.xlsx)

En la fase de pruebas, integre e implemente todo el código desarrollado en la infraestructura del entorno de pruebas. A continuación, se siguen las actividades del ciclo de vida de las pruebas de software para comprobar si hay errores, fallos y defectos en el sistema, y verificar que las funcionalidades del sistema funcionen como se espera (o no). Las actividades incluyen:

* Escribir casos de prueba
* Ejecutar casos de prueba

<div id="deploy-checklist">
  ## Lista de verificación de Deploy
</div>

[Descargar la lista de verificación de Deploy](https://cdn2.auth0.com/docs/media/articles/architecture-scenarios/checklists/Deploy-Checklist.xlsx)

En la fase de Deploy, implemente el sistema en un entorno de pruebas o de producción, donde los usuarios reales comienzan a utilizarlo e interactuar con él.

En última instancia, implementará todos los componentes del sistema en el entorno de producción cuando haga una puesta en producción.

<div id="monitor-checklist">
  ## Lista de verificación de la fase Monitor
</div>

[Descargar la lista de verificación de la fase Monitor](https://assets.ctfassets.net/cdy7uua7fh8z/4FYZQXQHknEDkLflShHXBz/d8757810ace0fc9dbabad9b1715d0dbf/Monitor-Checklist.xlsx)

En la fase de Monitor, realice mejoras, correcciones y cambios para garantizar que el sistema siga funcionando y se mantenga actualizado para cumplir los objetivos del negocio y satisfacer las necesidades de los usuarios. Las actividades incluyen:

* Monitoreo
* Mantenimiento
* Cambios y ajustes
* Actualización y adaptación a necesidades futuras
