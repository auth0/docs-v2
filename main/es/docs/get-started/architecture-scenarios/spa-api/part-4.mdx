title: Conclusión (SPA + API)
---

En este documento cubrimos un escenario sencillo en el que una API es utilizada por una aplicación de una sola página (SPA) para permitir que los empleados completen sus registros de horas.

Aprendimos sobre el flujo implícito, qué es un <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">Token de acceso</Tooltip>, cómo configurar una API en Auth0, cómo configurar una aplicación SPA para comunicarse de forma segura con esta API, cómo definir y proteger los endpoints de nuestra API, cómo usar las bibliotecas proporcionadas para validar el Token de acceso y cómo recuperar uno nuevo de Auth0.

Comenzamos describiendo el caso de uso y los requisitos, y luego explicamos cómo puede cumplirse cada requisito y el razonamiento detrás de cada decisión que se tomó.

Usamos Node.js para la implementación de la API y Angular para la SPA. Aun así, esperamos que, después de leer este documento, puedas implementar esta solución con las tecnologías que prefieras.
