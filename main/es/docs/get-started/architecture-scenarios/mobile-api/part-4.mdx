En este documento cubrimos un escenario sencillo: una API que utiliza una aplicación móvil para que los empleados registren sus horas de trabajo.

Aprendimos sobre Authorization Code Grant con PKCE, qué es un <Tooltip tip="Token de acceso: credencial de autorización, en forma de cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">Token de acceso</Tooltip>, cómo configurar una API en Auth0, cómo configurar una aplicación móvil para que se comunique de forma segura con esta API, cómo definir y proteger los endpoints de nuestra API, cómo usar las bibliotecas proporcionadas para validar el Token de acceso y cómo obtener uno nuevo de Auth0.

Comenzamos describiendo el caso de uso y los requisitos, y luego explicamos cómo puede cumplirse cada requisito y el razonamiento detrás de cada decisión tomada.

Usamos Node.js para implementar la API y Android para la aplicación móvil. Aun así, esperamos que, después de revisar este documento, puedas crear esto con las tecnologías que prefieras.
