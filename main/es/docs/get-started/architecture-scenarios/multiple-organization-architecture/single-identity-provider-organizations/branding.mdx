Los elementos de marca asociados a una organización son extremadamente valiosos, ya que su uso brinda a los usuarios un entorno que conocen y en el que confían. El uso de elementos de marca reconocibles también aumenta la confianza de los usuarios en que la información que proporcionan (por ejemplo, las credenciales) se gestionará de forma segura. En nuestros escenarios de arquitectura, ofrecemos orientación general sobre varios temas de [Marca B2B](/es/docs/get-started/architecture-scenarios/business-to-business/branding), que recomendamos revisar junto con la orientación que se proporciona aquí.

<div id="universal-login">
  ## Universal Login
</div>

La personalización de la marca en un entorno con múltiples organizaciones puede ser más compleja que en un entorno estándar, en el que solo debe gestionar un único conjunto de elementos de marca. Sin embargo, [Organizaciones de Auth0](/es/docs/manage-users/organizations/organizations-overview) aprovecha la [experiencia de Universal Login](/es/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience), que ofrece varias [plantillas de página](/es/docs/customize/login-pages/universal-login/customize-templates) para simplificar la personalización de la marca. La configuración de la Organización de Auth0 también le permite [seleccionar una marca básica mediante un esquema de colores](/es/docs/manage-users/organizations/configure-organizations/create-organizations).

<div id="login-page">
  ### Página de Login
</div>

Al usar una [Conexión de base de datos o una Conexión de base de datos personalizada](/es/docs/authenticate/database-connections), la página de Login se utiliza para obtener las credenciales del primer factor de un usuario. Puedes configurar la página de Login mediante un conjunto de [variables de plantilla](/es/docs/manage-users/organizations/create-first-organization) disponibles para facilitar la personalización por organización.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Puedes configurar la página de Login para mostrar conexiones empresariales y sociales para la autenticación del primer factor, por lo que la marca de la página de Login también puede ser relevante en estos casos.
</Callout>

<div id="password-reset-page">
  ### Página de Restablecimiento de contraseña
</div>

Al igual que la [página de Login](#login-page), la página de Restablecimiento de contraseña puede configurarse en escenarios que usan una conexión de base de datos o una conexión de base de datos personalizada. En los escenarios que implican una conexión empresarial o una conexión social, los cambios de contraseña se gestionan fuera de Auth0 por el <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> (IdP) de origen, por lo que la página de Restablecimiento de contraseña de Auth0 nunca se muestra.

<div id="multi-factor-authentication-pages">
  ### Páginas de autenticación multifactor
</div>

<Tooltip tip="Autenticación multifactor (MFA): proceso de autenticación de usuarios que utiliza un factor adicional al username y la contraseña, como un código por SMS." cta="Ver glosario" href="/es/docs/glossary?term=Multi-Factor+Authentication">Autenticación multifactor</Tooltip> (MFA) en Auth0 es compatible con todos los métodos que se usan para obtener las credenciales del primer factor de un usuario (es decir, conexión de base de datos o conexión de base de datos personalizada, conexión empresarial o conexión social). Si usa MFA, probablemente querrá personalizar la marca de las páginas de la experiencia de <Tooltip tip="Universal Login: su aplicación redirige a Universal Login, alojado en el Servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Universal Login</Tooltip> asociadas con MFA, lo que puede hacerse de forma similar a la personalización de la [página de Login](#login-page).

<div id="email-templates">
  ## Plantillas de correo electrónico
</div>

Al igual que con las plantillas de página de Universal Login, hay un conjunto de [variables de plantilla de correo electrónico](/es/docs/customize/email/email-templates/supported-liquid-syntax#general-variables) disponibles para facilitar la personalización de los correos electrónicos por organización. Las plantillas de correo electrónico que deban personalizarse variarán según cómo se autentiquen los usuarios (es decir, mediante una conexión de base de datos o una conexión de base de datos personalizada, una conexión empresarial o una conexión social). Para obtener más información sobre la personalización de plantillas de correo electrónico al usar la funcionalidad de Organizations de Auth0, consulta [Create Your First Organization: Email Templates](/es/docs/manage-users/organizations/create-first-organization).

<div id="custom-domains">
  ## Dominios personalizados
</div>

La funcionalidad de Organizations de Auth0 solo admite un [dominio personalizado](/es/docs/customize/custom-domains), que se define a nivel del Tenant de Auth0. Por lo tanto, le recomendamos que use un nombre de <Tooltip tip="Dominio personalizado: dominio de terceros con un nombre específico o de marca." cta="Ver glosario" href="/es/docs/glossary?term=Custom+Domain">dominio personalizado</Tooltip> que sea independiente de la organización. Si necesita implementar nombres de dominio por organización, deberá desplegar varios tenants de Auth0 en producción. Dado que usar varios tenants tiene implicaciones que afectan al <Tooltip tip="Inicio de sesión único (SSO): servicio que, después de que un usuario inicia sesión en una aplicación, inicia automáticamente la sesión de ese usuario en otras aplicaciones." cta="Ver glosario" href="/es/docs/glossary?term=Single+Sign-On">inicio de sesión único</Tooltip> (SSO), a la gestión de perfiles de usuario y a otros aspectos, debe evaluar cuidadosamente esta opción antes de optar por este enfoque.
