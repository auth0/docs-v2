Auth0 ofrece a los usuarios del tenant una experiencia de búsqueda más intuitiva para las aplicaciones y API registradas, con la posibilidad de crear expresiones de filtro para campos de búsqueda en Auth0 Dashboard. En lugar de desplazarse por listas paginadas con cientos de recursos, los usuarios del tenant pueden hacer consultas por campos como:

* El nombre de su aplicación (Travel0-native)
* Los metadatos de la aplicación o del usuario (permissions)
* El tipo de aplicación (SPA)
* API por nombre
* API por ID

Auth0 Dashboard muestra resultados coincidentes en tiempo real de todos sus recursos registrados.

<div id="how-it-works">
  ## Cómo funciona
</div>

La barra de búsqueda de las secciones [Aplicaciones](https://manage.auth0.com/#/applications) y [APIs](https://manage.auth0.com/#/apis) del Auth0 Dashboard usa una interfaz de filtros guiados y búsqueda booleana. La barra de búsqueda funciona como un menú desplegable que muestra los campos por los que se puede filtrar. Cuando seleccionas un campo e introduces un valor, el filtro aplica ese valor como una etiqueta que se puede quitar encima de los resultados. Puedes combinar hasta cinco filtros a la vez.

Cada filtro que agregas reduce aún más los resultados. Para acotar los resultados, aplica la lógica `AND/OR` entre distintos campos. Los filtros se reflejan en la URL como parámetros de consulta. Copia la URL en cualquier momento para compartirla con un compañero de equipo o guárdala en los marcadores para volver más adelante a la misma vista filtrada.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Ejemplo booleano**

  Si filtras por **Tipo de aplicación** `Native` OR `SPA`, y **Cliente de primera parte** `true`, los resultados muestran solo aplicaciones de primera parte que son Native o SPA. Se excluyen las aplicaciones que son Regular Web Apps o de terceros.
</Callout>

<div id="search-behavior-by-field-type">
  ### Comportamiento de la búsqueda por tipo de campo
</div>

La búsqueda en Auth0 Dashboard encuentra coincidencias según el tipo de campo.

* **Campos de cadena** (nombre, identificador): Se admite la coincidencia parcial. Al introducir `payments`, se encuentran &quot;Payments-application&quot;, &quot;Corp Payments Service&quot; y resultados similares; el orden de las palabras no importa. Para excluir resultados que coincidan con un término, antepón `NOT`, `-` o `No`; por ejemplo, si introduces `NOT test` en el campo de nombre, se devuelven todos los recursos cuyo nombre no contiene &quot;test&quot;. Las búsquedas con comodines requieren al menos tres caracteres; si introduces menos, se muestra el mensaje &quot;Minimum 3 characters&quot; y no se envía la consulta.

* **Campos de coincidencia exacta**

Debes introducir el valor completo del ID. Los ID parciales no devuelven resultados. Usa estos campos cuando tengas un ID de cliente específico de un registro o de un ticket de soporte.

* **Campos de metadatos**

Puedes buscar en los metadatos por clave y valor, o solo por clave:

* **Clave y valor**: Introduce una clave y un valor para encontrar recursos en los que esa clave exacta coincida con ese valor. Por ejemplo, la clave `environment` y el valor `production` devuelven solo los recursos en los que `app_metadata.environment` es igual a `production`. Se admite la coincidencia por prefijo en los valores; al introducir `prod`, coincide con `production`, `prod-eu` y `prod-us`.
  * **Solo clave**: Introduce una clave y presiona **Enter** sin indicar un valor. Esto devuelve todos los recursos que tienen establecida esa clave de metadatos, independientemente de su valor. Es útil para encontrar recursos que usan una clave específica.

También puedes buscar claves de metadatos anidadas de hasta dos niveles de profundidad mediante notación de puntos. Por ejemplo, la clave `tier.bronze` y el valor `true` coinciden con recursos en los que `app_metadata.tier.bronze` es igual a `true`.

Para excluir resultados que coincidan con un par clave-valor específico, antepón `NOT` al valor. Por ejemplo, la clave `color` y el valor `NOT orange` devuelven todos los recursos en los que `color` está establecido con cualquier valor distinto de `orange`. Las claves deben coincidir exactamente. No se admiten coincidencias de tipo contiene ni termina en para metadatos. Los valores se indexan hasta 64 caracteres; los valores más largos no se pueden buscar.

* **Campos de enumeración**

Selecciona uno o varios tipos de aplicación de una lista predefinida. Varias selecciones dentro del mismo filtro usan lógica OR. Por ejemplo, seleccionar `Native` y `SPA` devuelve todas las aplicaciones Native y SPA.

* **Campos booleanos**

Busca clientes de primera parte seleccionando `true` o `false`. Usa esto para distinguir las aplicaciones de primera parte (las autorizadas para omitir la pantalla de consentimiento) de las de terceros.

<div id="searchable-fields">
  ### Campos de búsqueda
</div>

<div id="applications">
  #### Aplicaciones
</div>

| **Campo**                                        | **Descripción**                                                            | **Tipo de coincidencia**          |
| ------------------------------------------------ | -------------------------------------------------------------------------- | --------------------------------- |
| **Nombre**                                       | El nombre de la aplicación                                                 | Coincidencia parcial              |
| **ID de cliente** (`client_id`)                  | El ID de cliente único                                                     | Coincidencia exacta               |
| **ID de cliente externo** (`external_client_id`) | Un identificador externo de la aplicación                                  | Coincidencia exacta               |
| **Metadatos**                                    | Metadatos personalizados en pares clave-valor almacenados en la aplicación | Coincidencia exacta o por prefijo |
| **Tipo de aplicación**                           | Native, SPA, Regular Web App o Machine to Machine                          | Selección de enumeración          |
| **Cliente de primera parte**                     | Indica si la aplicación es propia o de terceros                            | Booleano                          |

<div id="apis">
  #### API
</div>

| **Campo**         | **Descripción**                                                                 | **Tipo de coincidencia** |
| ----------------- | ------------------------------------------------------------------------------- | ------------------------ |
| **Nombre**        | El nombre de la API                                                             | Coincidencia parcial     |
| **Identificador** | El identificador único de la audiencia (por ejemplo, `https://api.example.com`) | Coincidencia parcial     |
| **Id**            | El id interno del sistema                                                       | Coincidencia exacta      |

<div id="limitations">
  ### Limitaciones
</div>

* Puede aplicar un máximo de cinco filtros por búsqueda. La barra de búsqueda impide agregar un sexto filtro.
* Es posible que los resultados de búsqueda no reflejen los cambios realizados en los últimos segundos, ya que el índice tiene consistencia eventual.
* Los resultados se obtienen en un plazo de dos segundos en condiciones normales. Si una consulta supera el tiempo de espera, reduzca la cantidad de filtros o use un término de búsqueda más específico.

<div id="example-queries">
  ## Ejemplos de consultas
</div>

<div id="find-an-application-by-client-id">
  ### Buscar una aplicación por ID de cliente
</div>

Usa esta opción cuando tengas un `client_id` de un registro de errores o de un ticket de soporte y necesites localizar la aplicación rápidamente.

1. Ve a **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Selecciona la barra de búsqueda y elige **ID de cliente**.
3. Pega el valor completo de `client_id`. La aplicación correspondiente aparecerá de inmediato.

<div id="find-applications-using-metadata">
  ### Buscar aplicaciones por metadatos
</div>

Si sus aplicaciones almacenan metadatos personalizados, como etiquetas de entorno, niveles de plan o ID externos, puede buscar por esos valores.

1. Vaya a **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Seleccione la barra de búsqueda y elija **Metadato**.
3. Introduzca la clave de metadatos, por ejemplo, `environment`.
4. Introduzca el valor, por ejemplo, `production`. Las aplicaciones en las que `metadata.environment` es igual a `production` aparecen en los resultados. Para buscar por prefijo, introduzca un valor parcial, como `prod`.

<div id="troubleshooting">
  ## Solución de problemas
</div>

**No se encontraron resultados**

Compruebe que los valores del filtro estén escritos correctamente. Para los campos de cadena, pruebe con un término más corto o más general. Para los campos de metadatos, confirme el nombre exacto de la clave: las claves distinguen entre mayúsculas y minúsculas y deben coincidir exactamente. No se admite la coincidencia de tipo `contains` para los metadatos; use un prefijo en su lugar. Para los campos de coincidencia exacta, como Application ID, asegúrese de pegar el valor completo del ID.

**Los resultados parecen desactualizados**

Los resultados de búsqueda tienen consistencia eventual y es posible que no reflejen los cambios realizados en los últimos segundos. Si creó o actualizó recientemente un recurso, espere unos segundos y vuelva a intentar la búsqueda.

**La búsqueda superó el tiempo de espera**

Reduzca la cantidad de filtros activos o use un término de búsqueda más específico. Si el problema persiste, póngase en contacto con [Auth0 Support](https://support.auth0.com).
