Puede crear una aplicación manualmente o importarla desde una URL de [documento de metadatos de ID de cliente (CIMD)](/es/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd) en el Auth0 Dashboard.

<div id="create-manually">
  ## Crear manualmente
</div>

Cuando crea una aplicación manualmente, Auth0 le asigna un <Tooltip tip="ID de cliente: Cadena alfanumérica que sirve como identificador único de una aplicación." cta="Ver glosario" href="/es/docs/glossary?term=client+ID">ID de cliente</Tooltip>, una cadena alfanumérica que funciona como identificador único de la aplicación. Usará este ID en el código de su aplicación cuando llame a las API de Auth0. Este proceso de registro de aplicaciones se denomina registro manual de clientes.

Otro dato importante es el <Tooltip tip="Secreto del cliente: Secreto que usa un cliente (aplicación) para autenticarse con el Servidor de autorización; solo deben conocerlo el cliente y el Servidor de autorización, y debe ser lo bastante aleatorio como para que no pueda adivinarse." cta="Ver glosario" href="/es/docs/glossary?term=client+secret">Secreto del cliente</Tooltip>. Debe mantenerse confidencial en todo momento. Si alguien obtiene acceso al secreto del cliente de su aplicación, podrá hacerse pasar por su aplicación y acceder a recursos protegidos. Es importante que seleccione el tipo de aplicación correcto para ayudar a Auth0 a detectar determinados riesgos de seguridad.

<div id="import-from-url">
  ## Importar desde una URL
</div>

Puede registrar aplicaciones importando desde una URL un documento de metadatos de ID de cliente (CIMD) alojado externamente. CIMD es un archivo JSON que contiene metadatos del cliente y está alojado en un dominio HTTPS seguro controlado por la aplicación. Este proceso de registro de aplicaciones se denomina registro manual de CIMD. Para obtener más información, consulte [Registrar aplicaciones con CIMD](/es/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd).

<div id="manual-client-registration-vs-manual-cimd-registration">
  ## Registro manual de clientes vs. registro manual de CIMD
</div>

La siguiente tabla compara el registro manual de clientes con el registro manual de CIMD:

| Característica                | Registro manual de clientes              | Registro manual de CIMD                                  |
| ----------------------------- | ---------------------------------------- | -------------------------------------------------------- |
| **ID de cliente**             | Generado por Auth0                       | URL HTTPS (p. ej., `https://yourdomain.com/client.json`) |
| **Autenticación**             | Secreto del cliente o certificado        | Solo claves asimétricas (sin secretos)                   |
| **Verificación de identidad** | Ninguna                                  | Propiedad del dominio mediante alojamiento HTTPS         |
| **Gestión de metadatos**      | Se actualizan a través del Dashboard/API | CIMD es la fuente autorizada                             |
| **Rotación de claves**        | Manual, una sola clave                   | Se admiten varias claves en `jwks_uri`                   |
| **Tipo de aplicación**        | Todos los tipos                          | Solo aplicaciones de terceros                            |
| **Más adecuado para**         | Aplicaciones propias, servicios internos | Clientes MCP, integraciones de terceros                  |

<div id="create-application-in-auth0">
  ## Crear una aplicación en Auth0
</div>

Para crear una aplicación en Auth0:

1. Ve a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
2. Selecciona **Create Application**.
3. Selecciona **Create it manually** o **Import from URL**.
   * **Import from URL**: Sigue las instrucciones de [Register Applications with CIMD](/es/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd).
   * **Create it manually**: Asigna un nombre descriptivo a tu nueva aplicación. Luego, selecciona un tipo de aplicación:
     * **Native Applications**: Estas aplicaciones incluyen apps móviles, de escritorio o híbridas que se ejecutan de forma nativa en un dispositivo (por ejemplo, iOS o Android).
     * **Single-Page Web Applications**: Estas aplicaciones incluyen apps de JavaScript que realizan la mayor parte de la lógica de la interfaz de usuario en un navegador web y se comunican con un servidor web principalmente mediante APIs (por ejemplo, AngularJS + Node.js o React).
     * **Regular Web Applications**: Estas aplicaciones son aplicaciones web tradicionales que realizan la mayor parte de la lógica de la aplicación en el servidor (por ejemplo, Express.js o ASP.NET).
     * **Machine-to-Machine Applications**: Estas aplicaciones incluyen aplicaciones no interactivas, como herramientas de línea de comandos, demonios, dispositivos IoT o servicios que se ejecutan en tu backend.
4. Selecciona **Create**.
