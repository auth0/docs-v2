Para integrar Auth0 con una aplicación web de página única, primero debe registrar su aplicación en Auth0 mediante Auth0 dashboard.

1. Vaya a [dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y haga clic en **Create Application**.
2. Introduzca un nombre descriptivo para su aplicación, seleccione **Single-Page Web Applications** y haga clic en **Create**.

   Una vez creada la nueva aplicación, puede ver la página **configuración de la aplicación**, que incluye las siguientes pestañas:

   | Pestaña de configuración | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                   |
   | ------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**          | Muestra toda la documentación disponible para su tipo de aplicación.                                                                                                                                                                                                                                                                                                                                                          |
   | **Settings**             | Muestra toda la configuración disponible para su aplicación. De forma predeterminada, la mayoría de estos ajustes se crean automáticamente. Para obtener más información, lea [configuración de la aplicación](/es/docs/get-started/applications/application-settings).                                                                                                                                                          |
   | **Credentials**          | Muestra el método de autenticación de la aplicación y las credenciales configuradas. Para obtener más información, lea [Credenciales de la aplicación](/es/docs/secure/application-credentials)                                                                                                                                                                                                                                  |
   | **Add-ons**              | Le permite habilitar complementos asociados a una aplicación. Normalmente se usan para integraciones de SAML o WS-Fed, que no suelen ser necesarias para las SPA. Para obtener más información, lea [Habilitar el addon SAML2 Web App](/es/docs/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) y [Configurar aplicaciones WS-Fed](/es/docs/get-started/applications/update-application-connections). |
   | **Connections**          | Le permite habilitar conexiones para su aplicación. Las conexiones son fuentes de usuarios; pueden habilitarse y compartirse entre varias aplicaciones. Para obtener más información, lea [Conexiones](/es/docs/authenticate/identity-providers).                                                                                                                                                                                |
   | **Organizations**        | Le permite autenticar usuarios en las organizaciones a las que pertenecen. Para obtener más información, lea [Organizaciones](/es/docs/manage-users/organizations).                                                                                                                                                                                                                                                              |
   | **Login Experience**     | Le permite configurar la experiencia de inicio de sesión de los usuarios de su organización. Para obtener más información, lea [Comprenda cómo funcionan las organizaciones de Auth0](/es/docs/manage-users/organizations/organizations-overview#login-experience).                                                                                                                                                              |

   Para obtener más información, lea [configuración de la aplicación](/es/docs/get-started/applications/application-settings).

<div id="next-steps">
  ## Próximos pasos
</div>

Una vez que haya registrado su aplicación, revise y complete estos pasos de configuración:

* Configure al menos una conexión (como una conexión de base de datos o un inicio de sesión con redes sociales) y habilítela para su aplicación. Sin una conexión habilitada, los usuarios no pueden autenticarse.
* Modifique el código de su aplicación para usar la aplicación registrada en Auth0. Revise nuestros [Auth0 Quickstarts](/es/docs/quickstarts), donde encontrará instrucciones detalladas sobre cómo implementar el inicio de sesión y el cierre de sesión, gestionar las sesiones de usuario, recuperar y mostrar información del perfil de usuario y mucho más.
* Use [las API de Auth0](/es/docs/api).

  * La [Authentication API](https://auth0.com/docs/api/authentication) gestiona todas las funciones principales relacionadas con la identidad (por ejemplo, inicio de sesión, cierre de sesión y obtención del perfil de usuario). La mayoría de los usuarios usan esta API a través de nuestros Quickstarts, el [SDK de SPA de Auth0](/es/docs/libraries/auth0-spa-js) o el [widget Lock](/es/docs/libraries/lock). Para aplicaciones legadas, consulte la [biblioteca Auth0.js](/es/docs/libraries/auth0js). Sin embargo, si está creando manualmente toda la interfaz de usuario de autenticación, tendrá que interactuar directamente con esta API.
  * La [Management API](https://auth0.com/docs/api/management/v2) le permite automatizar varias tareas a las que también se puede acceder desde el dashboard de Auth0 (por ejemplo: crear usuarios y configurar los grant types de la aplicación).
