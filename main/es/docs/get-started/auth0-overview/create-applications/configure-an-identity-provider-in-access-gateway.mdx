---

Para integrar Auth0 con una aplicación de máquina a máquina (M2M), primero debe registrar su aplicación en Auth0 mediante el <Tooltip tip="Auth0 Dashboard: producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Vaya a [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) y haga clic en **Create Application**.

2. Introduzca un nombre descriptivo para su aplicación, seleccione **Machine to Machine Applications** y haga clic en **Create**.

3. Seleccione la Management API para autorizar la comunicación M2M, ya que así es como OAG interactúa con Auth0.

4. Haga clic en **Authorize**.

5. A cada aplicación M2M que accede a una API se le debe otorgar un conjunto de permisos (o scopes) que el usuario autenticado debe conceder. Seleccione los **Permissions** que desea que se emitan como parte del <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, utilizada para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip> de su aplicación. OAG requiere los siguientes permisos para el **cliente** y los **usuarios:**

* `read:client, write:client, update:client, delete:client`
* `read:users`

6. Haga clic en **Authorize**.

7. Seleccione la pestaña **Credentials** de su aplicación.

8. Haga clic en el **<Tooltip tip="ID de cliente: valor de identificación asignado a su recurso registrado por Auth0." cta="Ver glosario" href="/es/docs/glossary?term=Client+ID">Client ID</Tooltip>** y cópielo en un editor de texto.

9. Haga clic en el **<Tooltip tip="Secreto del cliente: secreto utilizado por un cliente (aplicación) para autenticarse con el Servidor de autorización; solo deben conocerlo el cliente y el Servidor de autorización, y debe ser lo suficientemente aleatorio como para que no se pueda adivinar." cta="Ver glosario" href="/es/docs/glossary?term=Client+Secret">Client Secret</Tooltip>** y cópielo en un editor de texto.

10. Continúe con el procedimiento [Configure an Identity Provider in Access Gateway](/docs/Configure an Identity Provider in Access Gateway).
