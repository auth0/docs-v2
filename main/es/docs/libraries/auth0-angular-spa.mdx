title: SDK de Auth0 para Angular para aplicaciones de página única
---

El SDK de Auth0 para Angular es una biblioteca de JavaScript para implementar autenticación y autorización en aplicaciones Angular con Auth0. Proporciona un servicio, un guard de autenticación y un interceptor HTTP para que puedas realizar tareas comunes de autenticación dentro de tus aplicaciones Angular.

El SDK de Auth0 para Angular gestiona los detalles del grant y del protocolo, la expiración y renovación de tokens, así como su almacenamiento y caché. Internamente, implementa [Universal Login](/es/docs/authenticate/login/auth0-universal-login) y el [flujo de código de autorización con PKCE](/es/docs/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce).

La biblioteca está [alojada en GitHub](https://github.com/auth0/auth0-angular), donde puedes [obtener más información sobre la API](https://auth0.github.io/auth0-angular/).

<div id="installation">
  ## Instalación
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  El SDK de Angular es compatible con las aplicaciones SPA de Angular versión 14 y posteriores.
</Callout>

Usa la [CLI de Angular](https://cli.angular.io/) para instalar el SDK en tu proyecto:

`ng add @auth0/auth0-angular`

<div id="getting-started">
  ## Primeros pasos
</div>

Comienza a integrar el SDK importando `AuthModule` en tu proyecto y configurándolo para tu aplicación cliente de Auth0:

```jsx lines
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { AppComponent } from './app.component';

// Importar el módulo desde el SDK
import { AuthModule } from '@auth0/auth0-angular';

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,

    // Importar el módulo en la aplicación, con configuración
    AuthModule.forRoot({
      domain: '{yourDomain}',
      clientId: 'YOUR_AUTH0_CLIENT_ID',
      authorizationParams: {
        redirect_uri: window.location.origin,
      },
    }),
  ],

  bootstrap: [AppComponent],
})
export class AppModule {}
```

<div id="login">
  ## Inicio de sesión
</div>

El SDK es totalmente compatible con el sistema de inyección de dependencias de Angular. Para empezar, importe el tipo `AuthService` en un componente e inyéctelo en el constructor.

Luego, para iniciar la sesión del usuario, llame a los métodos `loginWithRedirect` o `loginWithPopup` del servicio:

```jsx lines
import { Component } from '@angular/core';

// Importa el tipo AuthService desde el SDK
import { AuthService } from '@auth0/auth0-angular';

@Component({
  selector: 'app-login',
  template: '<button (click)="login">Log in</button>',
  styles: [],
})
export class LoginComponent {
  // Inyecta el servicio de autenticación en tu componente mediante el constructor
  constructor(public auth: AuthService) {}

  login(): void {
    // Llama a esto para redirigir al usuario a la página de inicio de sesión
    this.auth.loginWithRedirect();
  }
}
```

<div id="logout">
  ## Cerrar sesión
</div>

Usa `logout` para cerrar la sesión de los usuarios. Asegúrate de que `returnTo` esté especificado en **Allowed Logout URLs** en el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar tus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

```jsx lines
import { Component } from '@angular/core';

// Importa el tipo AuthService desde el SDK
import { AuthService } from '@auth0/auth0-angular';

@Component({
  selector: 'app-logout',
  template: '<button (click)="logout">Log out</button>',
  styles: [],
})
export class LogoutComponent {
  // Inyecta el servicio de autenticación en tu componente a través del constructor
  constructor(@Inject(DOCUMENT) private doc: Document, public auth: AuthService) {}

  login(): void {
    // Llama a esto para redirigir al usuario a la página de inicio de sesión
    this.auth.logout({ 
      logoutParams: { 
        returnTo: this.doc.location.origin 
      }
    });
  }
}
```

<div id="authentication-state">
  ## Estado de autenticación
</div>

El estado de autenticación se expone mediante el observable `isAuthenticated$`:

```html lines
<div *ngIf="auth.isAuthenticated$ | async">
  <app-login></app-login>
</div>
```

<div id="user-profile">
  ## Perfil del usuario
</div>

El perfil del usuario autenticado se expone a través del observable `user$`:

```html lines
<ul *ngIf="auth.user$ | async as user">
  <li>{{ user.name }}</li>
  <li>{{ user.email }}</li>
</ul>
```

Internamente, este observable está encadenado al observable `isAuthenticated$`, por lo que no es necesario comprobar manualmente si el usuario ha iniciado sesión. Empezará a emitir valores en cuanto haya un usuario autenticado disponible.

<div id="protect-a-route">
  ## Proteger una ruta
</div>

Protege un componente de ruta con el `AuthGuard` integrado. Si se accede a esta ruta sin autenticarse, se redirigirá al usuario a la página de inicio de sesión y, después de iniciar sesión, de vuelta a esta página.

La protección de autenticación debe añadirse a una definición de ruta mediante el hook `canActivate` (o `canActivateChild` si se usan rutas secundarias):

```jsx lines expandable
import { NgModule } from '@angular/core';
import { Routes, RouterModule } from '@angular/router';
import { HomeComponent } from './unprotected/unprotected.component';
import { ProtectedComponent } from './protected/protected.component';

// Importar el guard de autenticación
import { AuthGuard } from '@auth0/auth0-angular';

const routes: Routes = [
  {
    path: 'protected',
    component: ProtectedComponent,

    // Proteger una ruta registrando el guard de autenticación en el hook `canActivate`
    canActivate: [AuthGuard],
  },
  {
    path: '',
    component: HomeComponent,
    pathMatch: 'full',
  },
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule],
})
export class AppRoutingModule {}
```

El guard de autenticación también admite el hook `canLoad` para impedir la carga de módulos de carga diferida cuando el usuario no está autenticado.

<div id="call-an-api">
  ## Llamar a una API
</div>

El SDK incluye un `HttpInterceptor` integrado que agrega automáticamente <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se usa para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=access+tokens">tokens de acceso</Tooltip> al inyectar un encabezado `Authorization` cuando las solicitudes se realizan con el servicio `HttpClient` de Angular.

Debe configurar el SDK para especificar a qué rutas de la API se les debe agregar automáticamente este encabezado, a fin de evitar que el token de acceso se filtre a destinatarios no previstos.

<div id="import-the-http-interceptor">
  ### Importa el interceptor HTTP
</div>

Para comenzar, importa los tipos que necesitarás para integrar el interceptor HTTP de Auth0 en tu aplicación:

```jsx lines
// Importa el módulo interceptor y los tipos de Angular que necesitarás
import { HttpClientModule, HTTP_INTERCEPTORS } from '@angular/common/http';
import { AuthHttpInterceptor } from '@auth0/auth0-angular';

// Registra el interceptor en el módulo de tu aplicación en el array `providers`
@NgModule({
  declarations: [],
  imports: [
    BrowserModule,
    HttpClientModule,     // Regístralo para poder realizar llamadas a la API con HttpClient
    AppRoutingModule,
    AuthModule.forRoot(...),
  ],
  providers: [
    { provide: HTTP_INTERCEPTORS, useClass: AuthHttpInterceptor, multi: true },
  ],
  bootstrap: [AppComponent],
})
```

**Nota:** El interceptor debe registrarse manualmente en el módulo de su aplicación (en lugar de registrarse automáticamente), ya que está diseñado para encadenarse con otros interceptores y no podemos garantizar que se inserte en el lugar correcto.

<div id="configure-the-http-interceptor">
  ### Configurar el interceptor HTTP
</div>

Hay varias formas de configurar esto según tus necesidades. Puedes especificar una ruta mediante una cadena o un objeto con opciones que influyen en el token que se envía, y también hacer que coincida según el verbo HTTP que se utilice. Los URI de las rutas también pueden incluir un comodín para especificar un grupo de rutas.

**Nota:** Si se realiza una llamada HTTP pero no coincide con ninguna ruta especificada en la configuración, se omitirá el interceptor y no se incluirá el encabezado `Authorization`.

Este ejemplo muestra formas habituales de indicarle al SDK qué rutas deben incluir un encabezado `Authorization`:

```jsx lines expandable
// Modifica tu configuración existente del SDK para incluir la configuración de httpInterceptor
AuthModule.forRoot({
  domain: '{yourDomain}',
  clientId: 'YOUR_AUTH0_CLIENT_ID',
  authorizationParams: {
    redirect_uri: window.location.origin,
  },

  // La configuración de AuthHttpInterceptor
  httpInterceptor: {
    allowedList: [
      // Adjunta tokens de acceso a cualquier llamada a '/api' (coincidencia exacta)
      '/api',

      // Adjunta tokens de acceso a cualquier llamada que comience con '/api/'
      '/api/*',

      // Coincide con cualquier ruta que comience con /api/accounts, pero también especifica la audiencia y el scope que debe tener
      // el token de acceso adjunto
      {
        uri: '/api/accounts/*',
        tokenOptions: {
          authorizationParams: {
            audience: 'http://my-api/',
            scope: 'read:accounts',
          }
        },
      },

      // Coincidencia por método HTTP
      {
        uri: '/api/orders',
        httpMethod: 'post',
        tokenOptions: {
          authorizationParams: {
            audience: 'http://my-api/',
            scope: 'write:orders',
          }
        },
      },

      // Usando una URI absoluta
      {
        uri: 'https://your-domain.auth0.com/api/v2/users',
        tokenOptions: {
          authorizationParams: {
            audience: 'https://your-domain.com/api/v2/',
            scope: 'read:users',
          }
        },
      },
    ],
  },
});
```

**Nota:** `tokenOptions` se pasa directamente al método `getTokenSilently` del SDK de SPA subyacente. Consulta [la documentación](https://auth0.github.io/auth0-spa-js/interfaces/GetTokenSilentlyOptions.html) para obtener más información sobre las propiedades admitidas.

### Llama a una API

Haz la llamada a la API con el servicio `HttpClient` de Angular. Los tokens de acceso deberían incluirse automáticamente mediante el encabezado `Authorization` en las rutas que coincidan con la configuración.

```jsx lines
export class MyComponent {
  constructor(private http: HttpClient) {}

  callApi(): void {
    this.http.get('/api').subscribe(result => console.log(result));
  }
}
```

Ten en cuenta que no se requiere nada adicional en el lugar donde se hace la llamada para proporcionar tokens ni otros mecanismos de autorización; de todo eso se encargan el interceptor y la configuración del módulo.
