Auth0.Android puede configurarse con varias opciones, que se enumeran a continuación.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Las opciones que se indican a continuación han cambiado con respecto a la versión V1 anterior. Consulta la guía de migración para obtener más información.
</Callout>

<div id="withconnection">
  ## withConnection
</div>

La opción `withConnection` le permite especificar la conexión con la que desea autenticarse.

```kotlin lines
WebAuthProvider.login(account)
                .withConnection("twitter")
                .start(this, callback)
```

<div id="withscope">
  ## withScope
</div>

El uso de alcances le permite devolver claims específicas para campos concretos en su solicitud. Agregar parámetros a `withScope` le permite añadir más alcances. Consulte [Scopes](/es/docs/get-started/apis/scopes) para obtener más información.

```kotlin lines
WebAuthProvider.login(account)
                .withScope("openid profile email")
                .start(this, callback)
```

El scope predeterminado es `openid profile email`.

<div id="withconnectionscope">
  ## withConnectionScope
</div>

En algunos casos, puede que necesite autenticarse con scopes de conexión, o permisos, específicos del proveedor de autenticación correspondiente. Consulte [Agregar scopes para un IdP externo](/es/docs/authenticate/identity-providers/adding-scopes-for-an-external-idp). Sin embargo, si necesita acceso específico para una situación concreta en su aplicación, puede pasar parámetros a `withConnectionScope`. Puede encontrar una lista completa de los parámetros disponibles en la configuración de esa conexión en el [Dashboard](https://manage.auth0.com/#) o en la documentación del proveedor de autenticación. El scope solicitado aquí se agrega a los que ya están especificados en la configuración de la conexión en el Dashboard.

```kotlin lines
WebAuthProvider.login(account)
                .withConnectionScope("email", "profile", "calendar:read")
                .start(this, callback)
```

<div id="withparameters">
  ## withParameters
</div>

Para enviar parámetros adicionales durante la autenticación, use `withParameters.`

```kotlin lines
val parameters = mapOf("param1" to "value1")

WebAuthProvider.login(account)
                .withParameters(parameters)
                .start(this, callback)
```

<div id="withheaders">
  ## withHeaders
</div>

Para enviar encabezados personalizados al endpoint de autorización, use `withHeaders.`

```kotlin lines
val headers = mapOf("header1" to "value1")

WebAuthProvider.login(account)
                .withHeaders(headers)
                .start(this, callback)
```

<div id="withscheme">
  ## withScheme
</div>

Si no usa Android &quot;App Links&quot; o quiere usar un esquema diferente para el URI de redirección, use `withScheme`. Actualice el marcador de posición del manifiesto `auth0Scheme` en el archivo `app/build.gradle` y la lista de permitidos **Allowed Callback URLs** en el [Dashboard](https://manage.auth0.com/#), dentro de la configuración de la aplicación, para que coincidan con el esquema elegido.

```kotlin lines
WebAuthProvider.login(account)
                .withScheme("myapp")
                .start(this, callback)
```

<Warning>
  El esquema debe estar en minúsculas y no contener guiones bajos.
</Warning>

<div id="withaudience">
  ## withAudience
</div>

Para especificar una <Tooltip tip="Audiencia: identificador único de la audiencia de un token emitido. Se llama aud en un token, y su valor contiene el ID de una aplicación (ID de cliente) para un ID Token o de una API (identificador de API) para un Token de acceso." cta="Ver glosario" href="/es/docs/glossary?term=audience">audiencia</Tooltip>, usa `withAudience`.

```kotlin lines
WebAuthProvider.login(account)
                .withAudience("https://YOUR_DOMAIN/userinfo")
                .start(this, callback)
```
