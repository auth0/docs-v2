title: Keystores de desarrollo y hashes de clave en Android
---

Al crear una nueva credencial de <Tooltip tip="OAuth 2.0: Marco de autorización que define protocolos y flujos de trabajo de autorización." cta="Ver glosario" href="/es/docs/glossary?term=OAuth">OAuth</Tooltip> para muchas conexiones, deberá proporcionar el SHA-1 del certificado que utiliza para firmar su aplicación. Al completar la configuración de su aplicación en el [Auth0 Dashboard](https://manage.auth0.com/#), también deberá proporcionar el valor SHA-256.

Localice el certificado que utiliza para firmar su aplicación. Si no dispone de uno, puede generarlo. En el caso de las aplicaciones de producción, debería hacerlo.

Durante el desarrollo, puede firmar su aplicación con el certificado predeterminado `android.keystore` que se generó automáticamente al instalar el SDK. En este ejemplo, usaremos este keystore predeterminado. Para generar los hashes de clave con este keystore, siga los ejemplos a continuación.

<div id="generating-your-key-hashes">
  ## Generar los hashes de tu clave
</div>

**En Windows:**

`keytool -list -v -keystore "%USERPROFILE%\.android\debug.keystore" -alias androiddebugkey -storepass android -keypass android`

**En Linux / macOS:**

`keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android`

<div id="using-your-key-hashes">
  ## Uso de los hashes de clave
</div>

Una vez que tengas los hashes de clave, copia el valor SHA256 resultante y ve a la configuración de tu aplicación en el [Auth0 Dashboard](https://manage.auth0.com/#/applications). Haz clic en &quot;Show Advanced Settings&quot; y, en la pestaña &quot;Device Settings&quot;, en la sección &quot;Android&quot;, completa &quot;App Package Name&quot; con el nombre del paquete de tu aplicación y el campo &quot;Key Hashes&quot; con el valor SHA256 que copiaste. No olvides guardar los cambios.

Si no agregas la Callback URL a la configuración de la aplicación ni los Key Hashes a la configuración del dispositivo de la aplicación, el servidor de Auth0 no devolverá el resultado de la llamada a tu aplicación.
