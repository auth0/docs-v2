Cuando la autenticación se realiza con el scope `offline_access` incluido, se devuelve un <Tooltip tip="Token de actualización: token utilizado para obtener un nuevo Token de acceso sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/es/docs/glossary?term=Refresh+Token">Token de actualización</Tooltip> junto con las credenciales. Este valor puede usarse para solicitar un nuevo <Tooltip tip="Token de actualización: token utilizado para obtener un nuevo Token de acceso sin obligar a los usuarios a volver a iniciar sesión." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">Token de acceso</Tooltip> y evitar tener que volver a pedirle las credenciales al usuario.

Los tokens deben almacenarse en un almacenamiento seguro después de una autenticación exitosa. Tenga en cuenta que los Tokens de actualización **nunca expiran**. Para solicitar un nuevo token, use el `AuthenticationAPIClient` de A`uth0.Android`.

<div id="using-refresh-token">
  ## Uso del token de actualización
</div>

```kotlin lines
val refreshToken: String = // Recuperar el Token de actualización del almacenamiento seguro
val account = Auth0(this)

val client = AuthenticationAPIClient(account)
client.renewAuth(refreshToken)
  .start(object: Callback<Credentials, AuthenticationException> {
  override fun onFailure(exception: AuthenticationException) {
       // Error
   }

   override fun onSuccess(credentials: Credentials) {
       // Usar las credenciales
   }
})
```
