Este es el escenario: después de la autenticación del usuario, quieres almacenar sus credenciales y usarlas mientras sigan siendo válidas. Cuando caduquen, querrás renovarlas mediante `refreshToken` para evitar volver a mostrar la página de inicio de sesión. En lugar de hacerlo automáticamente, necesitas que el usuario se autentique con su huella digital o su rostro.

Usarás la utilidad [Credentials Manager](https://github.com/auth0/Auth0.swift/blob/master/Auth0/CredentialsManager.swift) de [Auth0.swift](https://github.com/auth0/Auth0.swift/) para simplificar la gestión de las credenciales del usuario y realizar la autenticación biométrica.

<div id="getting-started">
  ## Primeros pasos
</div>

Primero, importa el módulo `Auth0`:

`import Auth0`

<div id="credentials-manager">
  ### Credentials Manager
</div>

Antes de recuperar las credenciales, también puede usar la autenticación biométrica (Face ID o Touch ID) que admite su dispositivo iOS.

Comience por configurar el Credentials Manager. Luego, habilite la biometría. También puede indicar un título para mostrarlo en la pantalla.

```swift lines
let credentialsManager = CredentialsManager(authentication: Auth0.authentication())
credentialsManager.enableBiometrics(withTitle: "Touch ID / Face ID Login")
```

Te recomendamos encarecidamente que añadas la configuración [NSFaceIDUsageDescription](https://developer.apple.com/library/archive/documentation/General/Reference/InfoPlistKeyReference/Articles/CocoaKeys.html#//apple_ref/doc/uid/TP40009251-SW75) al archivo `Info.plist` de tu proyecto para mostrar el motivo por el que se usa Face ID. En algunos casos, si no proporcionas una cadena de descripción y el usuario intenta autenticarse con Face ID, es posible que el intento falle.

```xml lines
...
<key>NSFaceIDUsageDescription</key>
<string>Reason why we use Face ID here</string>
...
```

<div id="login">
  ### Inicio de sesión
</div>

Muestre la página de <Tooltip tip="Universal Login: Su aplicación redirige a Universal Login, alojado en el Servidor de autorización de Auth0, para verificar la identidad de un usuario." cta="Ver glosario" href="/es/docs/glossary?term=Universal+Login">Universal Login</Tooltip> y, tras una autenticación correcta, pase las credenciales al Credentials Manager.

export const codeExample = `Auth0
    .webAuth()
    .scope("openid profile offline_access")
    .audience("https://{yourDomain}/userinfo")
    .start {
        switch $0 {
        case .failure(let error):
            // Gestiona el error
        case .success(let credentials):
            // Almacena las credenciales de forma segura con Credentials Manager
            credentialsManager.store(credentials: credentials)
        }
}`;

<div id="renew-user-credentials">
  ### Renovar las credenciales del usuario
</div>

Cuando necesites renovar las credenciales del usuario, puedes llamar al método `credentials` del Credentials Manager.

```swift lines
credentialsManager.credentials { error, credentials in
    guard error == nil, let credentials = credentials else {
        // Manejar el error
        // Volver a la pantalla de inicio de sesión
    }
    // Continuar enrutando al usuario ya que la autenticación fue exitosa
}
```

No es necesario guardar manualmente las nuevas credenciales, ya que el Credentials Manager se encarga de ello durante la renovación.
