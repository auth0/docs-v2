Lock ofrece opciones para activar y desactivar fácilmente las funciones de registro, así como para ajustar otras opciones de configuración relacionadas con el registro.

<div id="loglevel">
  ## logLevel
</div>

De forma predeterminada, es `.off`; se admiten los niveles de registro de Syslog.

```swift lines
.withOptions {
  $0.logLevel = .all
}
```

<div id="loghttprequest">
  ## logHttpRequest
</div>

Esta opción le permite elegir si se registran o no las solicitudes a la API de Auth0.swift. De forma predeterminada, es `false`.

```swift lines
.withOptions {
  $0.logHttpRequest = true
}
```

<div id="loggeroutput">
  ## loggerOutput
</div>

Puede especificar un controlador de salida para el registrador; de forma predeterminada, se utiliza la instrucción `print`.

```swift lines
.withOptions {
  $0.loggerOutput = CleanroomLockLogger()
}
```

En el código anterior, `loggerOutput` se ha configurado para usar [CleanroomLogger](https://github.com/emaloney/CleanroomLogger). Por lo general, esto se consigue implementando el protocolo `loggerOutput`. Por supuesto, puede usar la biblioteca de registro que prefiera. A continuación, se muestra un ejemplo de uso para gestionar la salida del registrador con CleanroomLogger.

```swift lines
class CleanroomLockLogger: LoggerOutput {
  func message(_ message: String, level: LoggerLevel, filename: String, line: Int) {
    let channel: LogChannel?
    switch level {
    case .debug:
        channel = Log.debug
    case .error:
        channel = Log.error
    case .info:
        channel = Log.info
    case .verbose:
        channel = Log.verbose
    case .warn:
        channel = Log.warning
    default:
        channel = nil
    }
    channel?.message(message, filePath: filename, fileLine: line)
  }
}
```
