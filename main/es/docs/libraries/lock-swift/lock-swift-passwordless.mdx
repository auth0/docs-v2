Lock <Tooltip tip="Sin contraseña: forma de autenticación que no se basa en una contraseña como primer factor." cta="Ver glosario" href="/es/docs/glossary?term=Passwordless">sin contraseña</Tooltip> admite la autenticación sin contraseña mediante conexiones de correo electrónico y SMS. Para usar la autenticación sin contraseña, necesita Lock.Swift versión 2.14.0 o posterior.

Para mostrar Lock, agregue el siguiente fragmento a su `UIViewController`.

```swift lines
Lock
    .passwordless()
    .withOptions {
         $0.oidcConformant = true
    }
    // withConnections, withOptions, withStyle, etc.
    .onAuth { credentials in
      // Guardar el objeto Credentials
    }
    .present(from: self)
```

Sin contraseña solo puede usarse con una única conexión y dará prioridad a las conexiones de correo electrónico frente a las de SMS.

<div id="passwordless-method">
  ### Método sin contraseña
</div>

Al usar Lock sin contraseña, el valor predeterminado de `passwordlessMethod` es `.code`, que envía al usuario un código de un solo uso para iniciar sesión. Si desea usar [enlaces universales](/es/docs/get-started/applications/enable-universal-links-support-in-apple-xcode), puede agregar lo siguiente:

```swift lines
.withOptions {
    $0.passwordlessMethod = .magicLink
}
```

<div id="activity-callback">
  ### Callback de la actividad
</div>

Si usa Lock sin contraseña y ha especificado la opción `.magicLink` para enviar al usuario un enlace universal, deberá agregar lo siguiente a su `AppDelegate.swift`:

```swift lines
func application(_ application: UIApplication, continue userActivity: NSUserActivity, restorationHandler: @escaping ([Any]?) -> Void) -> Bool {
    return Lock.continueAuth(using: userActivity)
}
```
