<div id="methods">
  ## Métodos
</div>

* [useAuth0Themes](/es/docs/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes): Hook de React para obtener las opciones del tema actual con la configuración plana del contexto de marca.
* [useChangeLanguage](/es/docs/libraries/acul/react-sdk/API-Reference/Hooks/useChangeLanguage): Hook de React que devuelve una función para cambiar el idioma mostrado en la pantalla actual de ACUL.
* [useCurrentScreen](/es/docs/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen): Hook de React que obtiene el contexto y el estado de la pantalla actual.
* [useErrors](/es/docs/libraries/acul/react-sdk/API-Reference/Hooks/useErrors): Hook de React para leer y gestionar errores en ACUL (Advanced Customization of Universal Login). Incluye todos los errores de validación y del servidor. Agrupa los errores en tres tipos: - `server` — errores devueltos por Auth0 o por tu propio backend. - `client` — errores de validación del cliente (por ejemplo, entradas de formulario no válidas). - `developer` — errores causados por una integración incorrecta o por un uso indebido del SDK.
* [useLoginIdentifiers](/es/docs/libraries/acul/react-sdk/API-Reference/Hooks/useLoginIdentifiers): Devuelve una lista de tipos de identificador activos (como `'email'`, `'phone'` o `'username'`) que se usan actualmente en el flujo de autenticación o de inicio de sesión.
* [useMfaPolling](/es/docs/libraries/acul/react-sdk/API-Reference/Hooks/useMfaPolling): Hook de React para gestionar el sondeo de MFA por notificación push (por ejemplo, mientras se espera la aprobación de una notificación push) en una pantalla de Auth0 Advanced Customization of Universal Login (ACUL).
* [useResend](/es/docs/libraries/acul/react-sdk/API-Reference/Hooks/useResend): Este hook de React gestiona las acciones de reenvío (por ejemplo, reenviar un código de verificación) en las pantallas de ACUL.
* [useSignupIdentifiers](/es/docs/libraries/acul/react-sdk/API-Reference/Hooks/useSignupIdentifiers): Devuelve una lista de identificadores habilitados (correo electrónico, teléfono o username), cada uno con su estado `required`, en función de los identificadores de registro de la pantalla actual.
* [usePasswordValidation](/es/docs/libraries/acul/react-sdk/API-Reference/Hooks/usePasswordValidation): Este hook de React valida una contraseña según la política de contraseñas actual de Auth0 y devuelve un resultado estructurado que indica si la contraseña cumple cada regla.
* [useUsernameValidation](/es/docs/libraries/acul/react-sdk/API-Reference/Hooks/useUsernameValidation): Hook de React para validar un username según la política actual de Auth0 para username.
