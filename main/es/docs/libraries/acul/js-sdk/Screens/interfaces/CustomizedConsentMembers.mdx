CustomizedConsentMembers

````ts Example
export interface CustomizedConsentMembers extends BaseMembers {
  /**
   * Provides access to the specific properties and data of the Customized Consent screen,
   * including the list of `scopes` and `authorizationDetails` being requested.
   * @type {ScreenMembersOnCustomizedConsent}
   */
  screen: ScreenMembersOnCustomizedConsent;

  /**
   * Envía la decisión del usuario de aceptar (conceder) los permisos y detalles de autorización solicitados.
   * Esta acción realiza una solicitud POST al endpoint `/u/customized-consent` con `action: "accept"`.
   * El estado de la transacción se incluye automáticamente.
   *
   * @param {CustomOptions} [payload] - Opcional. Un objeto para pares clave-valor personalizados
   *                                    que se enviarán con la solicitud. Estos parámetros se
   *                                    incluirán en los datos del formulario enviados al servidor.
   * @returns {Promise<void>} Una promesa que se resuelve cuando la acción de aceptación se envía correctamente.
   *                          Si la operación es exitosa, Auth0 normalmente redirige al usuario de vuelta a la aplicación
   *                          o al siguiente paso en el flujo de autenticación.
   * @throws {Error} Lanza un error si el envío del formulario falla (p. ej., problema de red, estado no válido).
   *                 Los errores del lado del servidor (como "invalid_request") se reflejarán en `this.transaction.errors`
   *                 tras la operación, en lugar de lanzarse como errores de JavaScript.
   *
   * @example
   * ```typescript
   * // Assuming 'customizedConsentManager' is an instance of the CustomizedConsent SDK class
   * try {
   *   await customizedConsentManager.accept();
   *   // If successful, the page will typically redirect.
   * } catch (error) {
   *   // Handle unexpected errors during the submission itself.
   *   console.error("Failed to submit consent acceptance:", error);
   * }
   * // After the await, check customizedConsentManager.transaction.errors for server-side validation issues.
   * ```
   */
  accept(payload?: CustomOptions): Promise<void>;

  /**
   * Submits the user's decision to deny (reject) the requested permissions and authorization details.
   * This action posts to the `/u/customized-consent` endpoint with `action: "deny"`.
   * The transaction state is automatically included.
   *
   * @param {CustomOptions} [payload] - Optional. An object for any custom key-value pairs
   *                                    to be sent with the request. These parameters will be
   *                                    included in the form data submitted to the server.
   * @returns {Promise<void>} A promise that resolves when the deny action is successfully submitted.
   *                          On success, Auth0 typically redirects the user, potentially showing an
   *                          access denied message or returning an error to the application.
   * @throws {Error} Throws an error if the form submission fails (e.g., network issue, invalid state).
   *                 Server-side errors are reflected in `this.transaction.errors`.
   *
   * @example
   * ```typescript
   * // Assuming 'customizedConsentManager' is an instance of the CustomizedConsent SDK class
   * try {
   *   await customizedConsentManager.deny({ reason_code: "user_rejected_details" }); // Example custom option
   *   // If successful, the page will typically redirect.
   * } catch (error) {
   *   console.error("Failed to submit consent denial:", error);
   * }
   * // After the await, check customizedConsentManager.transaction.errors for server-side validation issues.
   * ```
   */
  deny(payload?: CustomOptions): Promise<void>;
}

<ParamField body="accept" type="Promise<void>">
  Envía la decisión del usuario de aceptar (conceder) los permisos solicitados y los detalles de autorización.
  Esta acción realiza una solicitud al endpoint `/u/customized-consent` con `action: "accept"`.
  El estado de la transacción se incluye automáticamente.

  Una promesa que se resuelve cuando la acción de aceptación se envía correctamente.
  Si la operación se completa correctamente, Auth0 normalmente redirige al usuario a la aplicación
  o al siguiente paso del flujo de autenticación.

  #### Lanza

  Lanza un error si falla el envío del formulario (por ejemplo, por un problema de red o un estado no válido).
  Los errores del lado del servidor (como &quot;invalid&#95;request&quot;) se reflejarán en `this.transaction.errors`
  después de la operación, en lugar de lanzarse como errores de JavaScript.

  ```typescript Example
  // Suponiendo que 'customizedConsentManager' es una instancia de la clase SDK CustomizedConsent
  try {
    await customizedConsentManager.accept();
    // Si se realiza correctamente, la página normalmente redirigirá.
  } catch (error) {
    // Gestiona errores inesperados durante el propio envío.
    console.error("Failed to submit consent acceptance:", error);
  }
  // Después del await, comprueba customizedConsentManager.transaction.errors para ver si hay problemas de validación del lado del servidor.
  ```

<ParamField body="deny" type="Promise<void>">
  Envía la decisión del usuario de denegar (rechazar) los permisos solicitados y los detalles de autorización.
  Esta acción realiza una solicitud al endpoint `/u/customized-consent` con `action: "deny"`.
  El estado de la transacción se incluye automáticamente.

  Una promesa que se resuelve cuando la acción de denegación se envía correctamente.
  Si la operación se completa correctamente, Auth0 normalmente redirige al usuario, posiblemente mostrando un
  mensaje de acceso denegado o devolviendo un error a la aplicación.

  #### Lanza

  Lanza un error si falla el envío del formulario (por ejemplo, por un problema de red o un estado no válido).
  Los errores del lado del servidor se reflejan en `this.transaction.errors`.

  ```typescript Example
  // Suponiendo que 'customizedConsentManager' es una instancia de la clase SDK CustomizedConsent
  try {
    await customizedConsentManager.deny({ reason_code: "user_rejected_details" }); // Ejemplo de opción personalizada
    // Si se realiza correctamente, la página normalmente redirigirá.
  } catch (error) {
    console.error("Failed to submit consent denial:", error);
  }
  // Después del await, comprueba customizedConsentManager.transaction.errors para ver si hay problemas de validación del lado del servidor.
  ```
