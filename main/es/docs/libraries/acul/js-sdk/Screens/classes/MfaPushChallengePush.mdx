La clase MfaPushChallengePush implementa la funcionalidad de la pantalla `mfa-push-challenge-push`. Esta pantalla se muestra cuando un usuario debe confirmar una notificación push durante MFA.

<ParamField body="pollingManager" type={<span><a href="/es/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Este método inicia y administra el sondeo de un desafío de MFA push.

  Crea una sesión de sondeo que consulta repetidamente el endpoint del desafío de MFA push en el intervalo especificado hasta que el desafío se apruebe o se produzca un error. Cuando se cumple la condición de aprobación, se invoca el callback [onCompleted](/es/docs/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions#oncompleted) proporcionado y el sondeo se detiene automáticamente.

  Utilice el valor devuelto [MfaPushPollingControl](/es/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) para iniciar, detener o comprobar el estado del proceso de sondeo en cualquier momento.

  Devuelve una instancia de [MfaPushPollingControl](/es/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) que expone:

  * `startPolling()` — Inicia o reanuda el sondeo.
  * `stopPolling()` — Cancela el sondeo de inmediato.
  * `isRunning()` — Indica si el sondeo está activo en ese momento.

  ```typescript Example
  import MfaPushChallengePush from '@auth0/auth0-acul-js/mfa-push-challenge-push';

  const mfaPushChallengePush = new MfaPushChallengePush();

  // Inicia el sondeo cada 5 segundos hasta que se apruebe el desafío push
  const control = mfaPushChallengePush.pollingManager({
    intervalMs: 5000,
    onCompleted: () => mfaPushChallengePush.continue(),
    onError: (error) => console.error('Error de sondeo:', error),
  });

  // Más adelante, cancela el sondeo si es necesario
  control.stopPolling();
  ```

  **Observaciones**

  * Las respuestas HTTP 429 (límite de tasa) se gestionan internamente: el sondeo espera automáticamente hasta que se restablece el límite de tasa antes de reintentarlo.
  * Es seguro llamar a `startPolling()` varias veces; no tiene efecto si ya está en ejecución.

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [MfaPollingOptions](/es/docs/libraries/acul/js-sdk/Screens/type-aliases/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      Intervalo opcional, en milisegundos, entre solicitudes de sondeo consecutivas. Si se omite, se usa el intervalo interno predeterminado del SDK (normalmente, 5000 ms).
    </ParamField>

    <ParamField body="onCompleted()?" type="() => void">
      Callback opcional que se ejecuta cuando el desafío de MFA push se aprueba correctamente y el sondeo finaliza. Se llama exactamente una vez, tras lo cual el sondeo se detiene automáticamente.
    </ParamField>

    <ParamField body="onError()?" type="(error) => void">
      Callback opcional que se invoca si se produce un error durante el sondeo. Recibe un objeto `ULError` que contiene `status` (código de estado HTTP) y `responseText` (cuerpo de la respuesta sin procesar).
    </ParamField>
  </Expandable>
</ParamField>
