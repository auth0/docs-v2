---
title: "EmailIdentifierChallenge"
description: "Describe todas las propiedades y los métodos disponibles para personalizar la pantalla `email-identifier-challenge` de Universal Login."
---

La clase EmailIdentifierChallenge implementa la funcionalidad de la pantalla `email-identifier-challenge`. Esta pantalla solicita al usuario que introduzca un código de verificación enviado a su correo electrónico para confirmar su identidad.

<div id="constructors">
  ## Constructores
</div>

Crea una instancia del administrador de la pantalla EmailIdentifierChallenge:

```typescript Example
import EmailIdentifierChallenge from '@auth0/auth0-acul-js/email-identifier-challenge';
const emailIdentifierChallengeManager = new EmailIdentifierChallenge();
await emailIdentifierChallengeManager.submitEmailChallenge({ code: 'your-code' });
```

<div id="methods">
  ## Métodos
</div>

<ParamField body="changeLanguage" type="Promise<void>">
  Este método cambia el idioma de visualización de la página de Universal Login.

  ```typescript Example
  import EmailIdentifierChallenge from '@auth0/auth0-acul-js/email-identifier-challenge';
  const emailIdentifierChallengeManager = new EmailIdentifierChallenge();
  emailIdentifierChallengeManager.changeLanguage({
    language: 'fr',
  });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [LanguageChangeOptions](/es/docs/libraries/acul/js-sdk/Screens/interfaces/LanguageChangeOptions).
    </ParamField>

    <ParamField body="language" type="string" required>
      El código de configuración regional del idioma deseado (por ejemplo, `'en'`, `'fr'`, `'es'`).
    </ParamField>

    <ParamField body="persist?" type="&#x22;session&#x22;">
      Cuando se establece en `'session'`, el idioma seleccionado se conserva durante toda la sesión.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Este método recupera del contexto el arreglo de errores de la transacción, o un arreglo vacío si no hay ninguno.
</ParamField>

<ParamField body="resendCode" type="Promise<void>">
  Este método reenvía el código de verificación a la dirección de correo electrónico del usuario.

  ```typescript Example
  import EmailIdentifierChallenge from '@auth0/auth0-acul-js/email-identifier-challenge';
  const emailIdentifierChallengeManager = new EmailIdentifierChallenge();
  await emailIdentifierChallengeManager.resendCode();
  ```
</ParamField>

<ParamField body="resendManager" type={<span><a href="/docs/libraries/acul/js-sdk/Screens/interfaces/ResendControl">ResendControl</a></span>}>
  Este método proporciona funcionalidad de reenvío con gestión del tiempo de espera. Devuelve un objeto [`ResendControl`](/es/docs/libraries/acul/js-sdk/Screens/interfaces/ResendControl) con un método `startResend()` que activa el reenvío y aplica un período de espera antes de que el usuario pueda volver a reenviar.

  ```typescript Example
  import EmailIdentifierChallenge from '@auth0/auth0-acul-js/email-identifier-challenge';
  const emailIdentifierChallengeManager = new EmailIdentifierChallenge();
  const { startResend } = emailIdentifierChallengeManager.resendManager({
    timeoutSeconds: 15,
  });
  await startResend();
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [StartResendOptions](/es/docs/libraries/acul/js-sdk/Screens/interfaces/StartResendOptions).
    </ParamField>

    <ParamField body="timeoutSeconds?" type="number">
      La duración del período de espera, en segundos, antes de que el usuario pueda reenviar el código de nuevo.
    </ParamField>

    <ParamField body="onStatusChange?" type={<span><a href="/docs/libraries/acul/js-sdk/Screens/type-aliases/OnStatusChangeCallback">OnStatusChangeCallback</a></span>}>
      Función de devolución de llamada que se invoca cada vez que cambia el estado del reenvío; devuelve los segundos restantes e indica si la funcionalidad de reenvío está deshabilitada en ese momento.
    </ParamField>

    <ParamField body="onTimeout?" type="() => void">
      Función de devolución de llamada que se invoca cuando el período de espera vence y el reenvío vuelve a estar disponible.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="returnToPrevious" type="Promise<void>">
  Este método lleva al usuario de vuelta a la pantalla anterior.

  ```typescript Example
  import EmailIdentifierChallenge from '@auth0/auth0-acul-js/email-identifier-challenge';
  const emailIdentifierChallengeManager = new EmailIdentifierChallenge();
  await emailIdentifierChallengeManager.returnToPrevious();
  ```
</ParamField>

<ParamField body="submitEmailChallenge" type="Promise<void>">
  Este método envía el código de verificación por correo electrónico para completar el desafío.

  ```typescript Example
  import EmailIdentifierChallenge from '@auth0/auth0-acul-js/email-identifier-challenge';
  const emailIdentifierChallengeManager = new EmailIdentifierChallenge();
  await emailIdentifierChallengeManager.submitEmailChallenge({ code: 'your-code' });
  ```

  **Parámetros del método**

  <Expandable title="Parámetros">
    <ParamField body="options">
      [EmailChallengeOptions](/es/docs/libraries/acul/js-sdk/Screens/interfaces/EmailChallengeOptions).
    </ParamField>

    <ParamField body="code" type="string" required>
      El código de verificación enviado a la dirección de correo electrónico del usuario.
    </ParamField>

    <ParamField body="captcha?" type="string">
      El token de respuesta CAPTCHA, si CAPTCHA está habilitado para este inquilino.
    </ParamField>
  </Expandable>
</ParamField>
