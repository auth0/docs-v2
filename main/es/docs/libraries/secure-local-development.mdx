Los entornos de desarrollo local suelen ejecutarse en http (es decir, `http://localhost`) de forma predeterminada. En esta guía se explica cuándo conviene ejecutar un servidor local en https y cómo configurar https en localhost.

<div id="when-to-use-a-local-https-server">
  ## Cuándo usar un servidor HTTPS local
</div>

Los navegadores web se comportan de forma diferente con HTTP y HTTPS; para tener en cuenta estas diferencias, puede que necesites que tu servidor de desarrollo sirva páginas a través de HTTPS. Por ejemplo:

* Cuando tu página se sirve a través de HTTPS, se bloquearán las solicitudes para cargar recursos desde una URL HTTP. Puede que quieras este comportamiento en tu entorno local para evitar errores que solo se producirían en el entorno de producción o porque no quieres hacer por error solicitudes HTTP que contengan información confidencial.
* Las cookies que usan el atributo Secure o tienen el atributo [SameSite](https://auth0.com/blog/browser-behavior-changes-what-developers-need-to-know/) establecido en None no se enviarán a través de canales no seguros, lo que interrumpirá la autenticación y otras funciones.

<div id="how-to-set-up-a-secure-local-server">
  ## Cómo configurar un servidor local seguro
</div>

La idea es ejecutar un servidor web delante de tu servidor de desarrollo local que acepte solicitudes HTTPS y las reenvíe al servidor web de tu aplicación a través de HTTP, de modo que las solicitudes de tu aplicación se sirvan por HTTPS mediante un proxy. Esto se conoce como un proxy inverso HTTPS, y podemos usar un producto estándar llamado [Caddy](https://caddyserver.com/v2) para crear uno.

El proceso es el siguiente:

1. Descarga el binario de Caddy
2. Ejecuta el servidor web de tu aplicación sobre HTTP como de costumbre
3. Ejecuta el binario de Caddy como proxy inverso e indícale que reenvíe el tráfico al servidor web de tu aplicación
4. Accede al servidor web del proxy inverso a través de HTTPS

<div id="step-1-download-the-caddy-binary">
  ### Paso 1. Descarga el binario de Caddy
</div>

Visita [https://caddyserver.com/download](https://caddyserver.com/download), selecciona tu plataforma y descarga el archivo en tu computadora.

<div id="step-2-run-your-applications-web-server-over-http-as-normal">
  ### Paso 2. Ejecuta el servidor web de tu aplicación mediante HTTP como de costumbre
</div>

Ejecuta el servidor de desarrollo local de tu aplicación como de costumbre; por ejemplo, $ PORT=3000 npm start

<div id="step-3-run-the-caddy-binary-as-a-reverse-proxy-telling-it-to-forward-traffic-to-your-applications-web-server">
  ### Paso 3. Ejecuta el binario de Caddy como proxy inverso e indícale que reenvíe el tráfico al servidor web de tu aplicación
</div>

Busca la ubicación del binario de Caddy que descargaste y ejecuta el siguiente comando para crear un proxy inverso y reenviar las solicitudes desde el puerto 443 (el puerto predeterminado que se usa para HTTPS) al puerto de tu aplicación (en este caso, 3000):

```bash wrap lines
/path/to/caddy reverse-proxy --from localhost:443 --to localhost:3000
```

<div id="step-4-now-visit-your-web-server-over-https">
  ### Paso 4. Ahora accede a tu servidor web mediante https
</div>

Visita [https://localhost](https://localhost) y este gestionará las solicitudes de tu aplicación mediante https

<div id="running-your-application-behind-a-proxy">
  ## Ejecutar la aplicación detrás de un proxy
</div>

Es posible que tengas que hacer algunos ajustes menores en tu aplicación cuando se ejecuta detrás de un proxy.

Por ejemplo, en Express.js, aunque la aplicación no dejará de ejecutarse, registrará por error la dirección IP del proxy como la dirección IP del cliente, a menos que se configure `trust proxy`; consulta [https://expressjs.com/en/guide/behind-proxies.html](https://expressjs.com/en/guide/behind-proxies.html)l&gt;
