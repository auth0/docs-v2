A cada miembro de una [organización](/es/docs/manage-users/organizations/organizations-overview) se le pueden asignar uno o varios roles, que se aplican cuando los usuarios inician sesión a través de la organización. Para obtener más información sobre los roles y su comportamiento, consulte [Control de acceso basado en roles](/es/docs/manage-users/access-control/rbac).

Puede agregar roles a los miembros de las organizaciones mediante el <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>.

Para habilitar un rol para un miembro de una organización, primero debe haber [creado el rol](/es/docs/manage-users/access-control/configure-core-rbac/roles/create-roles) en su inquilino.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Para agregar roles a un miembro de una organización desde el Auth0 Dashboard:

1. Vaya a [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y seleccione la organización cuya membresía desea configurar.
2. Seleccione la vista **Members** y, luego, el nombre del miembro al que desea asignar un rol.
3. Seleccione **Assign** role.
4. Introduzca el nombre del rol o de los roles que desea asignar al miembro y seleccione **Add role(s) to organization**.

<div id="management-api">
  ## Management API
</div>

Para agregar roles a un miembro de una organización mediante la Management API:
Haz una solicitud `POST` al endpoint `Create Organization Member Roles`. Asegúrate de reemplazar los valores de marcador de posición `ORG_ID`, `MGMT_API_ACCESS_TOKEN`, `USER_ID` y `ROLE_ID` por el ID de tu organización, el <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o un JWT, que se usa para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">Token de acceso</Tooltip> de la Management API, el ID del usuario y el ID del rol, respectivamente.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Encuentra tu dominio de Auth0**

  Tu dominio de Auth0 está formado por el nombre de tu inquilino, tu subdominio regional (a menos que tu inquilino esté en la región de EE. UU. y se haya creado antes de junio de 2020) y `.auth0.com`. Por ejemplo, si el nombre de tu inquilino fuera `travel0`, el nombre de tu dominio de Auth0 sería `travel0.us.auth0.com`. (Si tu inquilino estuviera en EE. UU. y se hubiera creado antes de junio de 2020, entonces el nombre de tu dominio sería `https://travel0.auth0.com`.)

  Si usas dominios personalizados, este debe ser el nombre de tu dominio personalizado.
</Callout>

| Valor                   | Descripción                                                                                                                                               |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                | ID de la organización para la que quieres agregar roles a un miembro.                                                                                     |
| `MGMT_API_ACCESS_TOKEN` | [Token de acceso para la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens) con el scope `create:organization_member_roles`. |
| `USER_ID`               | ID del usuario al que quieres agregar los roles especificados.                                                                                            |
| `ROLE_ID`               | ID del rol que quieres agregar al usuario especificado para la organización especificada. Máximo de 100 roles por usuario.                                |

<div id="response-status-codes">
  ##### Códigos de estado de la respuesta
</div>

Los posibles códigos de estado de la respuesta son los siguientes:

| Código de estado | Código de error        | Mensaje                                                                                                        | Causa                                                                                                      |
| ---------------- | ---------------------- | -------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| `204`            |                        | Los roles se asociaron correctamente al usuario.                                                               |                                                                                                            |
| `400`            | `invalid_body`         | Cuerpo de la solicitud no válido. El mensaje variará según la causa.                                           | La carga útil de la solicitud no es válida.                                                                |
| `400`            | `invalid_query_string` | Cadena de consulta de la solicitud no válida. El mensaje variará según la causa.                               | La cadena de consulta no es válida.                                                                        |
| `401`            |                        | Token no válido.                                                                                               |                                                                                                            |
| `401`            |                        | Se recibió una firma no válida para la validación de JSON Web Token.                                           |                                                                                                            |
| `401`            |                        | El cliente no es global.                                                                                       |                                                                                                            |
| `403`            | `insufficient_scope`   | scope insuficiente; se esperaba alguno de: `create:organization_member_roles`.                                 | Se intentó leer o escribir un campo que no está permitido con los alcances del Bearer Token proporcionado. |
| `429`            |                        | Demasiadas solicitudes. Consulte los encabezados X-RateLimit-Limit, X-RateLimit-Remaining y X-RateLimit-Reset. |                                                                                                            |
