Al trabajar con [organizaciones](/es/docs/manage-users/organizations/organizations-overview) de forma programática, es posible que necesite obtener una lista de los roles asignados a un miembro de una organización.

Puede ver los roles asignados a un miembro de una organización mediante <Tooltip tip="Auth0 Dashboard: el principal producto de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o <Tooltip tip="Management API: un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

1. Vaya a [Auth0 Dashboard &gt; Organizaciones](https://manage.auth0.com/#/organizations) y seleccione la organización cuya membresía desea consultar.
2. Seleccione la vista **Members** y, a continuación, el usuario cuyos roles desea consultar.

<div id="management-api">
  ## Management API
</div>

Haz una solicitud `GET` al endpoint `Get Organization Member Roles`. Asegúrate de reemplazar los valores de marcador de posición `ORG_ID`, `USER_ID` y `MGMT_API_ACCESS_TOKEN` por el ID de tu organización, el ID de usuario del miembro y el <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">Token de acceso</Tooltip> de Management API, respectivamente.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Encuentra tu dominio de Auth0**

  Tu dominio de Auth0 se compone del nombre de tu inquilino, tu subdominio regional (a menos que tu inquilino esté en la región de EE. UU. y se haya creado antes de junio de 2020) y `.auth0.com`. Por ejemplo, si el nombre de tu inquilino fuera `travel0`, el nombre de tu dominio de Auth0 sería `travel0.us.auth0.com`. (Si tu inquilino estuviera en EE. UU. y se hubiera creado antes de junio de 2020, entonces tu nombre de dominio sería `https://travel0.auth0.com`).

  Si usas dominios personalizados, este debe ser el nombre de tu dominio personalizado.
</Callout>

| Valor                   | Descripción                                                                                                                                          |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                | ID de la organización de la que desea recuperar los roles de un miembro.                                                                             |
| `USER_ID`               | ID de usuario del miembro del que desea recuperar los roles.                                                                                         |
| `MGMT_API_ACCESS_TOKEN` | [Token de acceso para Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens) con el scope `read:organization_member_roles`. |

<div id="response-status-codes">
  #### Códigos de estado de respuesta
</div>

Los posibles códigos de estado de respuesta son los siguientes:

| Código de estado | Código de error        | Mensaje                                                                                                      | Causa                                                                                             |
| ---------------- | ---------------------- | ------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------- |
| `200`            |                        | Roles recuperados correctamente.                                                                             |                                                                                                   |
| `400`            | `invalid_query_string` | La cadena de consulta de la solicitud no es válida. El mensaje variará según la causa.                       | La cadena de consulta no es válida.                                                               |
| `401`            |                        | Token no válido.                                                                                             |                                                                                                   |
| `401`            |                        | Se recibió una firma no válida durante la validación del JSON Web Token.                                     |                                                                                                   |
| `401`            |                        | El cliente no es global.                                                                                     |                                                                                                   |
| `403`            | `insufficient_scope`   | scope insuficiente; se esperaba alguno de: `read:organization_member_roles`.                                 | Se intentó leer o escribir un campo no permitido con los alcances del Bearer Token proporcionado. |
| `429`            |                        | Demasiadas solicitudes. Revise los encabezados X-RateLimit-Limit, X-RateLimit-Remaining y X-RateLimit-Reset. |                                                                                                   |
