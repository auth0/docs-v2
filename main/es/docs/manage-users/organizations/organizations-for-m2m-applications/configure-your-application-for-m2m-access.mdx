Después de crear la aplicación, configúrala para el acceso de máquina a máquina siguiendo estos pasos:

1. [Define el comportamiento de la organización](#define-organization-behavior) para cada API a la que necesites acceder.
2. Cuando la aplicación solo deba acceder a organizaciones específicas, es decir, no a todas, [Autoriza el acceso M2M](/es/docs/manage-users/organizations/organizations-for-m2m-applications/authorize-m2m-access) para cada API a la que necesite acceder en esas organizaciones específicas.

<div id="define-organization-behavior">
  ## Definir el comportamiento de la organización
</div>

Puede configurar, para cada API, cómo usa su aplicación las Organizaciones durante el flujo de credenciales del cliente; por ejemplo, si la aplicación debe usar una organización, si puede acceder a cualquier organización o solo a aquellas asociadas explícitamente.

La siguiente tabla explica los campos para definir el comportamiento de la organización para el acceso M2M:

<div id="define-organization-behavior-table">
  ### Tabla para definir el comportamiento de la organización
</div>

| Campo                                                         | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         | Asignación de API                                                                                                                                                                                                               |
| ------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Soporte para organizaciones                                   | Determina cómo esta aplicación puede usar organizaciones al acceder mediante el flujo de credenciales del cliente.<br /><br />Las opciones incluyen: <ul> <li>**Ninguno:** La aplicación *no debe* usar organizaciones. Este es el valor predeterminado.</li> <li>**Opcional:** La aplicación *puede decidir* si usa una organización.</li> <li>**Obligatorio:** La aplicación *debe* usar una organización.</li> </ul>                                                                                                             | Soporte para organizaciones se asigna a `organization_usage`.<br /><br />Opciones: <ul> <li>**Ninguno** se asigna a `deny`</li> <li>**Opcional** se asigna a `allow`</li> <li>**Obligatorio** se asigna a `required`</li> </ul> |
| Permitir acceso de máquina a máquina a cualquier organización | Determina si esta aplicación puede acceder a cualquier organización o si está limitada a un conjunto de organizaciones asociadas explícitamente cuando usa el flujo de credenciales del cliente. De forma predeterminada, este comportamiento no está permitido.<br /><br />**Advertencia:** Habilitar el acceso de máquina a máquina a cualquier organización permite que la aplicación acceda a cualquier organización sin estar asociada explícitamente con ella. Solo debe habilitarse para aplicaciones internas de confianza. | Permitir acceso de máquina a máquina a cualquier organización se asigna a `allow_any_organization`.<br /><br />Opciones: <ul> <li>Marcado se asigna a `true`</li> <li>Desmarcado se asigna a `false`</li> </ul>                 |

<div id="define-organization-behavior-for-an-application">
  ### Definir el comportamiento de la organización para una aplicación
</div>

Para definir el comportamiento de la organización para una aplicación, usa el [Auth0 Dashboard](https://manage.auth0.com/) o la [Management API](https://auth0.com/docs/api/management/v2).

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para una aplicación máquina a máquina, define el comportamiento de la organización desde el Auth0 Dashboard:

    1. Ve a **Auth0 Dashboard &gt; Applications** y selecciona la aplicación para la que quieres configurar las Organizaciones.
    2. Ve a la pestaña **APIs** y expande los detalles de `client_grant` de la API a la que quieres configurar el acceso.
    3. Configura los ajustes correspondientes como se explica en la [tabla Definir el comportamiento de la organización](#define-organization-behavior-table).
    4. Selecciona **Save**.

    Como alternativa, si no se trata de una aplicación máquina a máquina:

    1. Ve a **Auth0 Dashboard &gt; APIs** y selecciona la API para la que quieres configurar el acceso de Organizaciones.
    2. Ve a la pestaña **Machine To Machine Applications**. Muestra todas las aplicaciones, no solo las aplicaciones máquina a máquina. Busca tu aplicación.
    3. Para autorizar el acceso, haz clic en el interruptor de tu aplicación. Luego, haz clic en la aplicación para expandir los detalles de configuración.
    4. Configura los ajustes correspondientes como se explica en la [tabla Definir el comportamiento de la organización](#define-organization-behavior-table).
    5. Selecciona **Save**.
  </Tab>

  <Tab title="Management API">
    También puedes definir el comportamiento de la organización para una aplicación mediante la Management API. El objeto `client_grant` controla el acceso de tu aplicación a una API, por lo que debes configurar el `client_grant` para cada API a la que tu aplicación necesite acceder. Para configurar el `client_grant` para acceso M2M, usa los endpoints [Create client grant](https://auth0.com/docs/api/management/v2/client-grants/post-client-grants) o [Update client grant](https://auth0.com/docs/api/management/v2/client-grants/patch-client-grants-by-id), junto con la [tabla Definir el comportamiento de la organización](#define-organization-behavior-table), para actualizar los ajustes correspondientes.

<div id="set-default-organization">
  ## Establecer una organización predeterminada
</div>

Algunos clientes no admiten campos no estándar en el [flujo de credenciales del cliente](/es/docs/get-started/authentication-and-authorization-flow/client-credentials-flow). Por lo tanto, no pueden enviar el parámetro `organization` obligatorio al endpoint `/oauth/token`. Para estos clientes, puede establecer una organización predeterminada que se aplique automáticamente a cualquier solicitud de credenciales de cliente de la aplicación cuando se especifica `none` y la compatibilidad con organizaciones de la API está configurada como `required`.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La organización predeterminada se aplica cuando `organization_usage=require` para la API (es decir, la audiencia de destino).
</Callout>

La siguiente tabla explica los campos para establecer una organización predeterminada para el acceso de máquina a máquina:

<div id="set-default-organization-table">
  ### Tabla para configurar la organización predeterminada
</div>

| Campo                                                           | Descripción                                                                                                     | Asignación en la API                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| --------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Organización predeterminada para el acceso de máquina a máquina | Define qué organización se aplica a las solicitudes de credenciales de cliente que no incluyen una organización | La organización predeterminada para el acceso de máquina a máquina se asigna al objeto `default_organization` con las siguientes propiedades: <ul> <li>`organization_id`: El id de la organización que se utilizará</li> <li> `flows`: El flujo de autenticación que se utilizará para la organización predeterminada.<br /><br />Esta configuración está oculta en la interfaz de usuario y su valor predeterminado es `["client_credentials"]` </li> </ul> |

<div id="set-default-organization-for-an-application">
  ### Establecer la organización predeterminada para una aplicación
</div>

Para establecer la organización predeterminada para una aplicación, utilice el [Auth0 Dashboard](https://manage.auth0.com/) o la [Management API](https://auth0.com/docs/api/management/v2).

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para habilitar una organización predeterminada para una aplicación mediante el Auth0 Dashboard:

    1. Vaya a **Auth0 Dashboard &gt; Applications** y seleccione la aplicación que desea configurar.
    2. Seleccione la pestaña **Organizations**.
    3. Configure los ajustes correspondientes, como se explica en la [tabla Establecer organización predeterminada](#set-default-organization-table).
    4. Haga clic en **Save.**
