description: Aprenda a revocar el acceso M2M de una aplicación.
title: Revocar el acceso M2M
---

Para revocar el acceso M2M, o quitar el acceso de su aplicación a una organización, debe eliminar la asociación entre sus client grant(s) y la organización. Una vez eliminada esta asociación, la aplicación ya no podrá usar la organización al solicitar nuevos tokens para la API definida en el grant.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Revocar el acceso M2M no afecta a las aplicaciones a las que se les haya otorgado acceso a alguna organización. Para obtener más información, consulte [Definir el comportamiento de la organización](/es/docs/manage-users/organizations/organizations-for-m2m-applications/configure-your-application-for-m2m-access#define-organization-behavior).
</Callout>

<Tabs>
  <Tab title="Auth0 Dashboard">
    Para quitar la asociación entre el client grant de una aplicación y una organización mediante el [Auth0 Dashboard](https://manage.auth0.com/):

    1. Vaya a **Organizations** y elija la organización de la que desea quitar la asociación.
    2. Seleccione la pestaña **Machine-to-Machine Access**.
    3. Elija una aplicación.
    4. Desmarque la casilla junto a las API que desea desvincular de la organización.
    5. Haga clic en **Save**.

    <Frame>![](/docs/images/cdy7uua7fh8z/5MFv5NAoBDswloVweCVU1D/9057b94d5c5fd35e25afc7170ac645f3/Acme_Bot_-_Travel_Api_-_English.png)</Frame>
  </Tab>

  <Tab title="Management API">
    Las asociaciones entre los client grants de las aplicaciones y las organizaciones se pueden eliminar mediante el endpoint [Disassociate client grants from organizations](https://auth0.com/docs/api/management/v2/organizations/delete-client-grants-by-grant-id):

    ```bash lines
    curl -X DELETE --location "https://{yourDomain}/api/v2/organizations/{ORG_id}/client-grants/{GRANT_ID}" \
      --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN' ;
    ```
  </Tab>
</Tabs>
