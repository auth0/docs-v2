La **Expiración absoluta** de una sesión se establece al crear la sesión. Puede ajustar la expiración absoluta configurando los ajustes de la sesión mediante el <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> o la <Tooltip tip="Auth0 Dashboard: el producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>.

<div id="dashboard">
  ## Dashboard
</div>

1. Ve a [Dashboard &gt; Tenant Settings](https://manage.auth0.com/#/tenant) y selecciona la vista **Advanced**.
2. Desplázate hasta la sección **Session Expiration**, localiza **Idle Session Lifetime** y **Maximum Session Lifetime**, introduce la configuración que desees y selecciona **Save**.

   <Frame>![Dashboard Tenant Settings Advanced Login Session Management](/docs/images/cdy7uua7fh8z/q2fGPzoUqCcXj7OxcHhjy/de15afcd26e8ea80937733034ffbd387/Screenshot_2024-11-06_at_11.02.16.png)</Frame>

| Configuración            | Descripción                                                                                                                                                                                                                                                                                            |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Idle Session Lifetime    | Período (en minutos) tras el cual la sesión de un usuario expirará si no ha interactuado con el Servidor de autorización. Se aplicarán los límites del sistema si supera los 4.320 minutos (3 días) en los planes Essential o Professional, o los 144.000 minutos (100 días) en los planes Enterprise. |
| Maximum Session Lifetime | Período (en minutos) tras el cual un usuario tendrá que volver a iniciar sesión, independientemente de su actividad. Se aplicarán los límites del sistema si supera los 43.200 minutos (30 días) en los planes Essential o Professional, o los 525.600 minutos (365 días) en los planes Enterprise.    |

<div id="management-api">
  ## Management API
</div>

Realiza una llamada `PATCH` al [endpoint de configuración del inquilino](https://auth0.com/docs/api/management/v2#!/tenants/patch_settings). Asegúrate de reemplazar los valores de marcador de posición `MGMT_API_ACCESS_TOKEN`, `SESSION_LIFETIME` y `IDLE_SESSION_LIFETIME` por tu <Tooltip tip="Token de acceso: credencial de autorización, en forma de una cadena opaca o JWT, que se utiliza para acceder a una API." cta="Ver glosario" href="/es/docs/glossary?term=Access+Token">Token de acceso</Tooltip> de Management API, el valor de duración de la sesión y el valor de tiempo de inactividad de la sesión, respectivamente.

| Valor                         | Descripción                                                                                                                                                                                                                                                                                     |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `MGMT_API_ACCESS_TOKEN`       | [Token de acceso para la Management API](https://auth0.com/docs/api/management/v2/tokens) con el scope `update:tenant_settings`.                                                                                                                                                                |
| `IDLE_SESSION_LIFETIME_VALUE` | Período (en horas) tras el cual la sesión de un usuario expirará si no ha interactuado con el Servidor de autorización. Se aplicarán los límites del sistema si supera las 72 horas (3 días) para los planes Essential o Professional, o las 2.400 horas (100 días) para los planes Enterprise. |
| `SESSION_LIFETIME_VALUE`      | Período (en horas) tras el cual un usuario deberá volver a iniciar sesión, independientemente de su actividad. Se aplicarán los límites del sistema si supera las 720 horas (30 días) para los planes Essential o Professional, o las 8.760 horas (365 días) para los planes Enterprise.        |
