title: Administrar usuarios con la Management API
---

Además de usar el Dashboard, puede obtener, crear, actualizar o eliminar usuarios mediante la [Management API](https://auth0.com/docs/api/management/v2#!/Users/get_users). Si desea llamar directamente a la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>, primero deberá generar el <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=access+token">token de acceso</Tooltip> adecuado. Para obtener más información, consulte [Tokens de acceso para la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens). Como alternativa, puede usar un SDK para implementar la funcionalidad que necesite para llamar a la Management API desde su aplicación. Para ver una lista de los SDK disponibles, consulte [la sección de SDK de nuestra matriz de soporte](/es/docs/troubleshoot/customer-support/product-support-matrix).

Para obtener información sobre cómo administrar el acceso al <Tooltip tip="Auth0 Dashboard: El producto principal de Auth0 para configurar sus servicios." cta="Ver glosario" href="/es/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> para los miembros de su equipo, consulte [Administrar el acceso al Dashboard](/es/docs/get-started/manage-dashboard-access).

<div id="permissions">
  ## Permisos
</div>

Use los siguientes permisos para obtener un token de acceso para aplicaciones de máquina a máquina. Para obtener más información, consulte [Registrar aplicaciones de máquina a máquina.](/es/docs/get-started/auth0-overview/create-applications/machine-to-machine-apps)

| Permisos                    | Descripciones                                                                                                                                                                                                                   |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `read:users`                | Las aplicaciones pueden buscar y recuperar cualquier registro de usuario almacenado en un inquilino de Auth0. Esto incluye leer mediante los endpoints `/api/v2/users`, `/api/v2/users-by-email` y `/api/v2/jobs/users-export`. |
| `create:users`              | Las aplicaciones pueden crear usuarios en cualquier conexión de base de datos o conexión sin contraseña en un inquilino de Auth0. Esto incluye crear mediante los endpoints `/api/v2/users` y `/api/v2/jobs/users-import`.      |
| `update:users`              | Las aplicaciones pueden actualizar cualquier usuario almacenado en un inquilino de Auth0. Esto incluye actualizar mediante el endpoint `/api/v2/users`.                                                                         |
| `update:users_app_metadata` | Las aplicaciones pueden actualizar solo el atributo `user.app_metadata` de cualquier usuario almacenado en un inquilino de Auth0. Esto incluye actualizar mediante el endpoint `/api/v2/users endpoint`.                        |
| `delete:users`              | Las aplicaciones pueden eliminar cualquier usuario almacenado en un inquilino de Auth0. Esto incluye eliminar mediante el endpoint `/api/v2/users`.                                                                             |

<div id="limitations">
  ## Limitaciones
</div>

De forma predeterminada, los atributos del perfil de usuario proporcionados por <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=identity+providers">proveedores de identidad</Tooltip> distintos de Auth0 (como Google, Facebook o X) no se pueden editar directamente, porque se actualizan desde el proveedor de identidad cada vez que el usuario inicia sesión. Para obtener más información sobre los atributos de los proveedores de identidad, consulte [Configurar la conexión del Proveedor de identidad para las actualizaciones del perfil de usuario](/es/docs/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0).

Para poder editar los atributos raíz `name`, `nickname`, `given_name`, `family_name` o `picture` del perfil de usuario normalizado, debe [configurar la sincronización de su conexión con Auth0](/es/docs/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) para que los atributos del usuario se actualicen desde el proveedor de identidad solo al crear el perfil de usuario. Estos atributos raíz estarán disponibles para su edición individual o mediante importaciones masivas.

Puede establecer contraseñas mediante los endpoints `create` o `update`, pero, por motivos de seguridad, las contraseñas no se pueden ver con los comandos `get` o `list user`. En el lado derecho del explorador de API se indican los atributos del perfil de usuario que se pueden ver o modificar para una llamada determinada.

<div id="endpoints">
  ## Endpoints
</div>

* Usa el endpoint [`/users`](https://auth0.com/docs/api/management/v2/users/get-users) para [obtener información sobre todos los usuarios](https://auth0.com/docs/api/management/v2/users/get-users). También puedes incluir criterios de búsqueda para encontrar usuarios específicos.
* Usa [`/user_id`](https://auth0.com/docs/api/management/v2/users/get-users-by-id) para [obtener información sobre un usuario](https://auth0.com/docs/api/management/v2/users/get-users-by-id) en función del `user_id`. El `user_id` es un identificador interno que consta de un nombre de conexión y un identificador único del usuario. El `user_id` es diferente del <Tooltip tip="ID Token: Credencial destinada al propio cliente, en lugar de usarse para acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=ID+token">token de ID</Tooltip>.
* El endpoint [`/userinfo`](https://auth0.com/docs/api/authentication/reference#get-user-info) toma como entrada el [token de acceso de Auth0 y devuelve información del perfil del usuario](https://auth0.com/docs/api/authentication/reference#get-user-info). Este endpoint incluirá los resultados de cualquier Rule que haya podido alterar el perfil del usuario durante la transacción de autenticación, pero el perfil de usuario resultante no se filtrará en función de ningún [alcance](/es/docs/get-started/apis/scopes/api-scopes).
* El endpoint [`/tokeninfo`](https://auth0.com/docs/api/authentication/reference#get-token-info) toma como entrada el [token de ID de Auth0 y devuelve información del perfil del usuario](https://auth0.com/docs/api/authentication/reference#get-token-info). Este endpoint devolverá un resultado que no incluye los resultados de ninguna Rule que altere el perfil del usuario.
