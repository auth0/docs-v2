title: Comprenda cómo funciona el perfilado progresivo
---

El perfilado progresivo es el proceso de recopilar de forma incremental información adicional sobre sus usuarios a medida que interactúan con su sitio web o aplicación. Puede recopilar información detallada del usuario solo cuando sea relevante para sus actividades.

Por ejemplo, podría recopilar el nombre, el correo electrónico y la contraseña del usuario durante el registro, y luego pedirle el nombre de su empresa y su puesto de trabajo la próxima vez que inicie sesión. También puede usar [Auth0 Forms](/es/docs/customize/forms/configure-progressive-profile-form) para diseñar un flujo que recopile información del usuario.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Cuando solicite información adicional a los usuarios, evite pedir información que quizá ya tenga. Por ejemplo, si un usuario se registra mediante una conexión social que usa el protocolo OpenID Connect (como Google o Facebook), el Proveedor de identidad (IdP) puede devolver más información del usuario de la que imagina. Para obtener más información, consulte [OpenID Connect Scopes](/es/docs/get-started/apis/scopes/openid-connect-scopes#standard-claims).
</Callout>

Implementar el perfilado progresivo mejora la experiencia de sus usuarios al:

* Acortar los formularios de registro.
* Evitar preguntas repetitivas.
* Recopilar información más relevante.
* Aumentar las tasas de conversión.

<div id="how-it-works">
  ## Cómo funciona
</div>

Cada vez que un usuario se autentica a través de Auth0, Auth0 actualiza su perfil de usuario. Los datos que se usan para actualizar su perfil pueden provenir de distintas fuentes:

* Atributos proporcionados por el <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y gestiona identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=IdP">IdP</Tooltip>.
* Atributos creados dinámicamente con [Actions de Auth0](/es/docs/customize/actions).
* Atributos devueltos al llamar a API como [FullContact](https://www.fullcontact.com/) y [Clearbit](https://clearbit.com/).
* Atributos gestionados por la aplicación.

<div id="use-action-triggers">
  ## Usa desencadenadores de Action
</div>

Con el desencadenador de Action `post-login`, puedes modificar los campos `user_metadata` y `app_metadata` del usuario durante el flujo de inicio de sesión. Esto es útil para tareas como almacenar datos específicos de la aplicación en el perfil del usuario, registrar si ya se realizaron determinadas operaciones para un usuario o almacenar en caché en el perfil del usuario los resultados de operaciones costosas para reutilizarlos en futuros inicios de sesión. Para obtener más información, consulta [Administrar los metadatos del usuario con el desencadenador de Action post-login](/es/docs/manage-users/user-accounts/metadata/manage-user-metadata).

<div id="use-the-management-api">
  ## Use la Management API
</div>

Puede usar el endpoint de Auth0 <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip> [Update a User](https://auth0.com/docs/api/management/v2/#!/Users/patch_users_by_id) para actualizar los datos de perfil de un usuario autenticado, incluidos los campos `app_metadata` y `user_metadata`. Para obtener más información, consulte [Administrar metadatos mediante la Management API](/es/docs/manage-users/user-accounts/metadata/manage-metadata-api).

<div id="use-the-lock-library">
  ## Use la biblioteca Lock
</div>

Puede usar la opción `additionalSignUpFields` para agregar campos personalizados a los formularios de registro de usuarios. Cuando un usuario introduce datos en un campo personalizado, Auth0 almacena el valor en el campo `user_metadata` de su perfil. Para obtener más información, consulte [Opciones de configuración de Lock](/es/docs/libraries/lock/lock-configuration#additionalsignupfields-array-).
