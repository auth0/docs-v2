description: Aprenda a actualizar los perfiles de usuario al usar su propia base de datos como proveedor de identidad.
title: Actualizar perfiles de usuario usando su base de datos
---

Actualice los perfiles de usuario al [usar su propia base de datos como proveedor de identidad](/es/docs/authenticate/database-connections/custom-db) realizando las siguientes tareas:

* Use la [Management API](https://auth0.com/docs/api/management/v2#!/Users/patch_users_by_id).
* Actualice el usuario en su base de datos.
* [Configure la migración de usuarios](/es/docs/manage-users/user-migration/configure-automatic-migration-from-your-database) desde su base de datos a Auth0.

<div id="update-users-with-the-management-api">
  ## Actualiza usuarios con la Management API
</div>

Si usas tu propia base de datos para la autenticación, puedes usar la [Management API](https://auth0.com/docs/api/management/v2) para actualizar los siguientes campos:

* `app_metadata`
* `user_metadata`
* `blocked`

Si necesitas actualizar otros campos del usuario, tendrás que hacerlo directamente en tu base de datos.

<div id="update-users-in-your-database">
  ## Actualiza usuarios en tu base de datos
</div>

Puedes actualizar los perfiles de usuario en tu base de datos como lo haces normalmente, y Auth0 actualizará su perfil de usuario en caché la próxima vez que ese usuario inicie sesión.

El perfil de usuario en la base de datos personalizada puede implementarse con cualquier estructura de perfil de usuario, pero debes asignarlo en la llamada a Login a los atributos del perfil de usuario normalizado de Auth0, como se muestra en la plantilla &quot;Login&quot; de la base de datos personalizada.

Puedes acceder a las plantillas de base de datos personalizada en
Conexiones -&gt; Base de datos -&gt; Base de datos personalizada. Asegúrate de activar el interruptor &quot;Use my own database&quot; para habilitar la edición de los scripts.

Consulta la sección [Caché del perfil de usuario](#user-profile-cache) a continuación para ver una breve descripción general de cómo Auth0 almacena en caché los perfiles de usuario.

<div id="update-users-through-migration">
  ## Actualizar usuarios mediante la migración
</div>

Si ha [habilitado la migración de usuarios](/es/docs/manage-users/user-migration/configure-automatic-migration-from-your-database) y un usuario ya se migró a la base de datos de Auth0, Auth0 no volverá a consultar su base de datos para obtener el perfil de ese usuario. Por lo tanto, los cambios realizados en la base de datos personalizada de ese usuario no se reflejarán en Auth0.

Una vez que un usuario se haya migrado, también podrá actualizar campos como `name`, `nickname`, `given_name`, `family_name`, `picture`, `email` y `email_verified` mediante la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>.

Sin embargo, las reglas para actualizar otros campos de usuario seguirán aplicándose como se describe en el [Perfil de usuario normalizado](/es/docs/manage-users/user-accounts/user-profiles/normalized-user-profiles).

<div id="user-profile-cache">
  ## Caché del perfil de usuario
</div>

Auth0 almacena en caché el perfil del usuario recibido de una [conexión de base de datos](/es/docs/authenticate/database-connections) antes de enviarlo a la aplicación cliente. Esta caché se guarda en la base de datos de Auth0 y se actualiza cada vez que el usuario se autentica.

Los valores en caché de los campos del [Perfil de usuario normalizado](/es/docs/manage-users/user-accounts/user-profiles/normalized-user-profile-schema) se basan en los valores devueltos por el Login Script de tu conexión de base de datos personalizada.

El perfil del usuario se almacena en caché por varias razones. En primer lugar, el almacenamiento en caché te permite implementar [inicio de sesión único (SSO)](/es/docs/authenticate/single-sign-on) en la capa de Auth0 para evitar tener que consultar la conexión en cada solicitud. Además, esto aporta resiliencia si una conexión no está disponible temporalmente.
