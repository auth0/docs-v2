description: Comprenda cómo Auth0 normaliza las propiedades comunes de los usuarios en el perfil de usuario.
title: Perfiles de usuario normalizados
---

El perfil de usuario normalizado es una forma específica de Auth0 de estandarizar y almacenar información relacionada con el usuario. Dado que cada <Tooltip tip="Proveedor de identidad (IdP): servicio que almacena y administra identidades digitales." cta="Ver glosario" href="/es/docs/glossary?term=Identity+Provider">Proveedor de identidad</Tooltip> (IdP) proporciona un conjunto distinto de información sobre un usuario, Auth0 normaliza las propiedades comunes del perfil en una representación del usuario independiente del protocolo al almacenar claims relacionados con el usuario. Por ejemplo, `family_name` en el perfil de usuario normalizado contiene información que un IdP puede devolver como `surname` o `last_name`.

Por ello, los claims de Auth0 incluidos en el perfil de usuario normalizado difieren del conjunto estándar de claims que pueden devolverse en los <Tooltip tip="ID Token: credencial destinada al propio cliente, en lugar de para acceder a un recurso." cta="Ver glosario" href="/es/docs/glossary?term=ID+Tokens">ID Tokens</Tooltip> del [endpoint `oauth/token` de la API de autenticación](https://auth0.com/docs/api/authentication#get-token) o en la respuesta del endpoint [`/userinfo`](https://auth0.com/docs/api/authentication#user-profile) (ambos siguen los requisitos detallados en la especificación de OIDC).

Para obtener más información sobre los atributos incluidos en el perfil de usuario normalizado y entender cómo trabajar con ellos, consulte [Estructura del perfil de usuario](/es/docs/manage-users/user-accounts/user-profiles/user-profile-structure).
