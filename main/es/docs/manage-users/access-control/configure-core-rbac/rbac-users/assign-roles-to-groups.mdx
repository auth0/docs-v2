Auth0 te permite asignar y quitar [roles](/es/docs/manage-users/access-control/rbac) a grupos empresariales aprovisionados mediante <Tooltip tip="System for Cross-domain Identity Management (SCIM): un protocolo estandarizado para aprovisionar y gestionar identidades de usuario y grupos entre sistemas." cta="Ver glosario" href="/es/docs/glossary?term=SCIM">SCIM</Tooltip> con el Auth0 Dashboard o la Management API. Cuando se asignan roles a un grupo, todos los miembros heredan esos roles al iniciar sesión. Puedes gestionar los roles de los grupos en dos ámbitos:

* **grupos de la Organización** — Grupos empresariales dentro de una [Organización](/es/docs/manage-users/organizations) específica. Requiere que [membresía automática](/es/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership) esté habilitado en la conexión empresarial para que se aplique la herencia de roles.
* **Grupos empresariales** — Grupos empresariales del Tenant de Auth0.

<div id="assign-roles">
  ## Asignar roles
</div>

<div id="organization-groups">
  ### Grupos de la Organización
</div>

Asigne roles a Grupos empresariales aprovisionados mediante SCIM dentro de una Organización. Los Grupos empresariales se sincronizan desde un proveedor de identidad empresarial (como Okta o Microsoft Entra ID) a través de SCIM. Cuando un miembro de la Organización inicia sesión, hereda los roles asignados a los Grupos empresariales a los que pertenece.

La conexión empresarial de la Organización debe tener habilitada la [Membresía automática](/es/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership) para que la herencia de roles por grupo surta efecto. Para obtener más información sobre cómo se aprovisionan los Grupos empresariales, consulte [Configurar SCIM entrante](/es/docs/authenticate/protocols/scim/configure-inbound-scim#use-groups-with-organizational-roles).

<Tabs>
  <Tab title="Dashboard">
    1. Vaya a [Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y seleccione la Organización.
    2. Seleccione la pestaña **Groups**.
    3. Seleccione el grupo al que desea asignar roles.
    4. Seleccione **Assign Roles**.
    5. Busque y seleccione los roles que desea asignar y, a continuación, seleccione **Assign**.

    Los roles asignados aparecen en los detalles del grupo.
  </Tab>

  <Tab title="Management API">
    Realice una llamada `POST` al endpoint [Assign Organization Group Roles](/es/docs/api/management/v2). Necesita la siguiente información:

    | Parámetro                    | Descripción                                                                                                                                              |
    | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `org_id`                     | ID de la Organización.                                                                                                                                   |
    | `group_id`                   | ID del Grupo empresarial.                                                                                                                                |
    | `role_id`                    | ID del rol de la Organización que se va a asignar. Repítalo en la matriz para asignar varios roles a la vez.                                             |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso para la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens) con el scope `create:organization_group_roles`. |

    ```bash wrap lines
    curl --request POST \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/organizations/{ORG_ID}/groups/{GROUP_ID}/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Una solicitud exitosa devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>

<div id="enterprise-groups">
  ### Grupos empresariales
</div>

Asigne roles a los grupos empresariales aprovisionados con SCIM. Cuando se asigna un rol a un grupo, todos los miembros de ese grupo heredan el rol asignado al iniciar sesión.

Para obtener información sobre cómo se aprovisionan los grupos empresariales desde un proveedor de identidad empresarial, consulte [configurar SCIM entrante](/es/docs/authenticate/protocols/scim/configure-inbound-scim).

<Tabs>
  <Tab title="Dashboard">
    1. Vaya a [Dashboard &gt; User Management &gt; Enterprise Groups](https://manage.auth0.com/#/groups).
    2. Seleccione el grupo al que desea asignar roles.
    3. Seleccione la pestaña **Roles** y, a continuación, **Assign Roles**.
    4. Busque y seleccione los roles que desea asignar y, a continuación, seleccione **Assign**.

    Los roles asignados aparecen en la pestaña **Roles** del grupo.
  </Tab>

  <Tab title="Management API">
    Realice una llamada `POST` al endpoint [Assign Roles to a Group](/es/docs/api/management/v2). Necesita la siguiente información:

    | Parámetro                    | Descripción                                                                                                                            |
    | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
    | `group_id`                   | ID del grupo empresarial.                                                                                                              |
    | `role_id`                    | ID del rol que se va a asignar. Repítalo en el array para asignar varios roles a la vez.                                               |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso para la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens) con el scope `update:groups`. |

    ```bash wrap lines
    curl --request POST \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/groups/{group_id}/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Si la solicitud se realiza correctamente, devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>

<div id="remove-roles">
  ## Eliminar roles
</div>

<div id="organization-groups">
  ### Grupos de la Organización
</div>

Quite roles de los grupos empresariales aprovisionados mediante SCIM dentro de una Organización. Una vez que se quite un rol, los miembros de la Organización de ese grupo dejarán de heredarlo al iniciar sesión.

La conexión empresarial de la Organización debe tener la [membresía automática habilitada](/es/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership) para que la herencia de roles del grupo surta efecto.

<Tabs>
  <Tab title="Dashboard">
    1. Vaya a [Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) y seleccione la Organización.
    2. Seleccione la pestaña **Groups**.
    3. Seleccione el grupo que desea actualizar.
    4. Busque el rol que desea quitar y, a continuación, seleccione el icono de papelera que aparece junto a él.
    5. Confirme la eliminación.

    El rol ya no aparecerá en los detalles del grupo.
  </Tab>

  <Tab title="Management API">
    Haga una llamada `DELETE` al endpoint [Delete Organization Group Roles](/es/docs/api/management/v2). Necesita la siguiente información:

    | Parámetro                    | Descripción                                                                                                                                              |
    | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `org_id`                     | ID de la Organización.                                                                                                                                   |
    | `group_id`                   | ID del grupo empresarial.                                                                                                                                |
    | `role_id`                    | ID del rol de la Organización que desea quitar. Repítalo en el arreglo para quitar varios roles a la vez.                                                |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso para la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens) con el scope `delete:organization_group_roles`. |

    ```bash wrap lines
    curl --request DELETE \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/organizations/ORG_ID/groups/GROUP_ID/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Si la solicitud se realiza correctamente, devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>

<div id="enterprise-groups">
  ### Grupos empresariales
</div>

Elimine roles de los grupos empresariales aprovisionados mediante SCIM. Una vez que se elimina un rol, los miembros del grupo dejarán de heredarlo cuando inicien sesión.

<Tabs>
  <Tab title="Dashboard">
    1. Vaya a [Dashboard &gt; User Management &gt; Enterprise Groups](https://manage.auth0.com/#/groups).
    2. Seleccione el grupo que desea actualizar.
    3. Seleccione la pestaña **Roles**.
    4. Busque el rol que desea eliminar y, a continuación, seleccione el icono de la papelera junto a él.
    5. Confirme la eliminación.

    El rol ya no aparece en la pestaña **Roles** del grupo.
  </Tab>

  <Tab title="Management API">
    Haga una llamada `DELETE` al endpoint [Remove Roles from a Group](/es/docs/api/management/v2). Necesita la siguiente información:

    | Parámetro                    | Descripción                                                                                                                            |
    | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
    | `group_id`                   | ID del grupo empresarial.                                                                                                              |
    | `role_id`                    | ID del rol que se va a eliminar. Repítalo en el array para eliminar varios roles a la vez.                                             |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [token de acceso para la Management API](/es/docs/secure/tokens/access-tokens/management-api-access-tokens) con el scope `update:groups`. |

    ```bash wrap lines
    curl --request DELETE \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/groups/GROUP_ID/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Una solicitud exitosa devuelve una respuesta `204 No Content`.
  </Tab>
</Tabs>
