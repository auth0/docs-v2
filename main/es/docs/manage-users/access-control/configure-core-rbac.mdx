title: Configurar las funciones principales de Authorization para el control de acceso basado en roles
---

Las funciones principales de Authorization de Auth0 permiten aplicar control de acceso basado en roles (RBAC) a sus APIs.

Para que RBAC funcione correctamente, debe habilitarlo en su API mediante el Dashboard o la <Tooltip tip="Management API: Un producto que permite a los clientes realizar tareas administrativas." cta="Ver glosario" href="/es/docs/glossary?term=Management+API">Management API</Tooltip>. Para obtener más información, consulte [Habilitar el control de acceso basado en roles para las APIs](/es/docs/get-started/apis/enable-role-based-access-control-for-apis).

La funcionalidad de Authorization Core es diferente de Authorization Extension. Para comparar ambos productos, consulte [Authorization Core frente a Authorization Extension](/es/docs/manage-users/access-control/authorization-core-vs-authorization-extension).

Para usar la funcionalidad principal de la manera más eficiente, debe hacer lo siguiente:

1. [Registrar una API con Auth0](/es/docs/get-started/auth0-overview/set-up-apis)
2. [Definir permisos para la API](/es/docs/get-started/apis/add-api-permissions)
3. [Crear roles](/es/docs/manage-users/access-control/configure-core-rbac/roles/create-roles)
4. [Asignar roles a los usuarios](/es/docs/manage-users/access-control/configure-core-rbac/rbac-users/assign-roles-to-users)
5. [Asignar permisos a los usuarios](/es/docs/manage-users/access-control/configure-core-rbac/rbac-users/assign-permissions-to-users), si es necesario.
