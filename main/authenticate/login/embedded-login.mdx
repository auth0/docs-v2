Embedded Login allows your users to log directly into your application and transmit their credentials to the Auth0 server for authentication. We do not recommend using Embedded Login. To learn more, read [Centralized Universal Login vs. Embedded Login](/authenticate/login/universal-vs-embedded-login).

If you decide to use Embedded Login, you must [configure your application for Cross-Origin Resource Sharing](/get-started/applications/set-up-cors) and should [configure a custom domain](/customize/custom-domains). You can then implement the [Lock SDK](/libraries/lock) or [Auth0.js SDK](/libraries/auth0js) within your application, or call the [Auth0 Authentication API](https://auth0.com/docs/api/authentication) directly.

Auth0 supports Embedded Login with <Tooltip tip="Passwordless: Form of authentication that does not rely on a password as the first factor." cta="View Glossary" href="/glossary?term=Passwordless">Passwordless</Tooltip> connections for multiple application types:

* [Embedded Passwordless Login in Native Applications](/authenticate/passwordless/implement-login/embedded-login/native)
* [Embedded Passwordless Login in Regular Web Applications](/authenticate/passwordless/implement-login/embedded-login/webapps)
* [Embedded Passwordless Authentication for Single-Page Apps](/authenticate/passwordless/implement-login/embedded-login/spa)
