title: Auth0 Terraform Provider
---

Deploy CLI n’est pas le seul outil offert pour gérer la configuration de votre locataire Auth0; il existe aussi un [Terraform Provider officiellement pris en charge](https://github.com/auth0/terraform-provider-auth0). [Terraform](https://terraform.io/) est un outil tiers qui permet de représenter sous forme de code la configuration de vos ressources infonuagiques. Il repose sur un cadre de plugiciels bien établi qui prend en charge un vaste éventail de fournisseurs infonuagiques, y compris Auth0.

Deploy CLI et Terraform Provider vous aident tous deux à gérer les configurations de votre locataire Auth0, mais chacun a ses avantages et ses inconvénients.

Vous pourriez envisager Auth0 Terraform Provider si :

* Vos flux de travail de développement utilisent déjà Terraform
* Vos besoins de gestion du locataire sont détaillés ou ne concernent que quelques ressources précises

Vous pourriez ne pas envisager Auth0 Terraform Provider si :

* Votre flux de travail de développement n’utilise pas Terraform, ce qui exige une configuration supplémentaire au départ
* Vos flux de travail de développement visent surtout la gestion en bloc de vos locataires
* Votre locataire comprend déjà beaucoup de ressources, ce qui peut demander des efforts importants pour les « importer »
