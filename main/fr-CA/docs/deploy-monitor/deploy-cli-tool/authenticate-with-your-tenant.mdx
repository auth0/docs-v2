Il existe trois méthodes prises en charge pour authentifier le Deploy CLI d’Auth0 auprès de votre locataire.

<div id="client-credentials">
  ## Identifiants de l’application
</div>

L’authentification par [identifiants de l’application](/fr-CA/docs/get-started/authentication-and-authorization-flow/client-credentials-flow) exige que vous fournissiez directement à l’application une paire composée d’un <Tooltip tip="ID client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=client+ID">ID client</Tooltip> et d’un <Tooltip tip="Secret client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=client+secret">secret client</Tooltip>. Cette option est simple et offre le moyen le plus rapide de configurer l’outil.

Pour configurer l’authentification par identifiants de l’application, définissez les champs suivants dans votre [fichier de configuration](/fr-CA/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli) :

| Champ                 | Valeur                                    |
| --------------------- | ----------------------------------------- |
| `AUTH0_CLIENT_ID`     | ID client de votre application Auth0.     |
| `AUTH0_CLIENT_SECRET` | Secret client de votre application Auth0. |

<div id="private-key-jwt">
  ## Private key JWT
</div>

L’authentification [Private key JWT](/fr-CA/docs/get-started/applications/configure-private-key-jwt) nécessite que vous configuriez une clé privée pour l’application et une clé publique pour le locataire distant. Cette option est idéale si vous ne voulez pas stocker d’identifiants dans Auth0.

Pour configurer l’authentification avec une clé privée <Tooltip tip="JSON Web Token (JWT) : format standard de l’ID Token (et souvent du Jeton d’accès) utilisé pour représenter de façon sécurisée des claims entre deux parties." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=JWT">JWT</Tooltip>, définissez les champs suivants dans votre [fichier de configuration](/fr-CA/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli) :

| Champ                            | Valeur                                                              |
| -------------------------------- | ------------------------------------------------------------------- |
| `AUTH0_CLIENT_SIGNING_KEY_PATH`  | Chemin relatif vers le certificat de clé de signature JWT.          |
| `AUTH0_CLIENT_SIGNING_ALGORITHM` | Facultatif. Algorithme de signature JWT utilisé pour le certificat. |

<div id="access-token">
  ## Jeton d’accès
</div>

L’authentification par [jeton d’accès](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) exige que vous fournissiez directement à l’application un jeton d’accès Auth0 pour la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+token">jeton d’accès</Tooltip>. Cette option confère davantage de responsabilités aux développeurs, mais elle peut permettre des flux de travail flexibles et précis lorsque nécessaire.

Pour configurer l’authentification par jeton d’accès, transmettez un jeton d’accès Auth0 pour la Management API au moyen de la variable d’environnement `AUTH0_ACCESS_TOKEN` [variable d’environnement](/fr-CA/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).
