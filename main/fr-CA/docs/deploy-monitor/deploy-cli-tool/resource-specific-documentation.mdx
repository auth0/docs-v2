En règle générale, les fichiers de configuration des ressources du Deploy CLI correspondent étroitement aux schémas de payload de l’[Auth0 Management API](https://auth0.com/docs/api/management/v2), mais il y a certaines nuances importantes à connaître.

<div id="client-grants">
  ## Autorisations d’application
</div>

L’autorisation d’application propre au Deploy CLI n’est volontairement ni exportée ni configurable à elle seule. Cela vise à éviter des changements incompatibles; autrement, l’outil pourrait révoquer l’accès ou tomber en panne en cours d’importation. Dans un contexte multilocataire et multienvironnement, on s’attend à ce que les nouveaux locataires disposent déjà d’une application désignée pour le Deploy CLI, comme indiqué dans [Getting Started](/fr-CA/docs/deploy-monitor/deploy-cli-tool).

<div id="prompts">
  ## Prompts
</div>

Les invites de texte personnalisé multilingues suivent une hiérarchie particulière. Sous la propriété de ressource `prompts` à la racine se trouve une propriété propriétaire `customText`, qui sert à regrouper les traductions de texte personnalisé avec d&#39;autres paramètres d&#39;invite. Sous `customText` se trouve le code de langue à deux caractères. Viennent ensuite l&#39;ID de l&#39;invite, puis l&#39;ID de l&#39;écran, puis l&#39;ID du texte.

<div id="hierarchy">
  ### Hiérarchie
</div>

```yaml lines
prompts:
  customText:
    <LANGUAGE>: # code de langue à deux caractères
      <PROMPT_ID>: # ID d'invite
        <SCREEN_ID>: # ID d'écran d'invite
          <TEXT_ID>: 'Some text'
```

<div id="example">
  ### Exemple
</div>

```yaml lines
prompts:
  identifier_first: true
  universal_login_experience: classic
  customText:
    en:
      login:
        login:
          description: Description de connexion en anglais
          buttonText: Texte du bouton
      mfa:
        mfa-detect-browser-capabilities:
          pickAuthenticatorText: 'Essayer une autre méthode'
          reloadButtonText: 'Recharger'
          noJSErrorTitle: 'JavaScript requis'
        mfa-login-options:
          pageTitle: 'Se connecter à ${clientName}'
          authenticatorNamesSMS: 'SMS'
```

<div id="databases">
  ## Bases de données
</div>

Lors de la gestion des connexions de base de données, les valeurs de `options.customScripts` renvoient à des fichiers JavaScript précis, relatifs au chemin du dossier de sortie. Sinon, le payload correspond presque exactement à celle de la [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Connections/post_connections).

### Exemple en YAML

Structure des dossiers en mode YAML :

```text lines
./databases/
    /Username-Password-Authentication
        /change_password.js   
        /create.js   
        /delete.js   
        /get_user.js   
        /login.js   
        /verify.js   
./tenant.yaml
```

 Contenu du fichier `tenant.yaml` :

```yaml lines
databases:
  - name: Username-Password-Authentication
    # ...
    options:
      # ...
      customScripts:
        change_password: ./databases/Username-Password-Authentication/change_password.js
        create: ./databases/Username-Password-Authentication/create.js
        delete: ./databases/Username-Password-Authentication/delete.js
        get_user: ./databases/Username-Password-Authentication/get_user.js
        login: ./databases/Username-Password-Authentication/login.js
        verify: ./databases/Username-Password-Authentication/verify.js
```

### Exemple de répertoire

Structure des dossiers en mode répertoire :

```text lines
./database-connections/
    ./Username-Password-Authentication/
        ./change_password.js
        ./create.js
        ./database.json
        ./delete.js
        ./get_user.js
        ./login.js
        ./verify.js
```

Contenu de `database.json` :

```json lines
{
  "options": {
    "customScripts": {
      "change_password": "./change_password.js",
      "create": "./create.js",
      "delete": "./delete.js",
      "get_user": "./get_user.js",
      "login": "./login.js",
      "verify": "./verify.js"
    }
  }
}
```

<div id="universal-login">
  ## Universal Login
</div>

<div id="pages">
  ### Pages
</div>

Lorsque vous remplacez le <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Universal+Login">Universal Login</Tooltip> par du HTML personnalisé, le contenu des pages d’erreur, de connexion, d’<Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip> et de réinitialisation du mot de passe est organisé dans des pages HTML spécifiques.

<div id="yaml-example">
  #### Exemple en YAML
</div>

Structure des dossiers en mode YAML :

```text lines
./pages/
    /error_page.html
    /guardian_multifactor.html
    /login.html
    /password_reset.html
./tenant.yaml
```

Contenu de `tenant.yaml` :

```yaml lines
pages:
  - name: error_page
    html: ./pages/error_page.html
    show_log_link: false
    url: https://mycompany.org/error
  - name: guardian_multifactor
    enabled: true
    html: ./pages/guardian_multifactor.html
  - name: login
    enabled: false
    html: ./pages/login.html
  - name: password_reset
    enabled: true
    html: ./pages/password_reset.html
```

<div id="directory-example">
  #### Exemple de répertoire
</div>

Arborescence des dossiers en mode répertoire :

```text lines
./pages/
    ./error_page.html
    ./error_page.json
    ./guardian_multifactor.html
    ./guardian_multifactor.json
    ./login.html
    ./login.json
    ./password_reset.html
    ./password_reset.json
```

Contenu de `login.json` :

```json lines
{
  "name": "login",
  "enabled": false,
  "html": "./login.html"
}
```

Contenu du fichier `error_page.json` :

```json lines
{
  "html": "./error_page.html",
  "show_log_link": false,
  "url": "https://mycompany.org/error",
  "name": "error_page"
}
```

Contenu du fichier `guardian_multifactor.json` :

```json lines
{
  "enabled": true,
  "html": "./guardian_multifactor.html",
  "name": "guardian_multifactor"
}
```

Contenu de `password_reset.json` :

```json lines
{
  "enabled": true,
  "html": "./password_reset.html",
  "name": "password_reset"
}
```
