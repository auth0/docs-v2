description: Découvrez comment utiliser Auth0 Deploy CLI comme utilitaire de ligne de commande autonome.
title: Utiliser comme CLI
---

Auth0 Deploy CLI peut être utilisé comme utilitaire de ligne de commande autonome. Cela vous permet de gérer simplement la configuration de votre locataire Auth0 dans des flux de travail CI/CD.

Pour obtenir une liste complète des paramètres de configuration, consultez [Configurer Auth0 Deploy CLI](/fr-CA/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="export-command">
  ## commande export
</div>

Récupère les configurations du locataire Auth0 sur la machine locale.

<div id="output_folder-o">
  ### `--output_folder`, `-o`
</div>

Chemin. Indique le répertoire cible dans lequel les fichiers de configuration seront enregistrés.

<div id="config_file-c">
  ### `--config_file`, `-c`
</div>

Chemin. Indique le fichier de configuration défini par l’utilisateur (`config.json`). Pour en savoir plus, consultez [Configurer le Deploy CLI](/fr-CA/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="format-f">
  ### `--format`, `-f`
</div>

Options : `yaml` ou `directory`. Détermine le format des fichiers de configuration des ressources exportées. Pour en savoir plus, consultez [Formats de configuration des ressources offerts](/fr-CA/docs/deploy-monitor/deploy-cli-tool/available-resource-configuration-formats).

<div id="export_ids-e">
  ### `--export_ids`, `-e`
</div>

Booléen. Lorsqu’elle est activée, cette option exporte les champs d’identifiant de chaque ressource. Valeur par défaut : `false`.

<div id="env">
  ### `--env`
</div>

Booléen. Indique si l’outil doit utiliser les variables d’environnement ou non. Valeur par défaut : `true`.

<div id="debug">
  ### `--debug`
</div>

Booléen. Active une journalisation des erreurs plus verbeuse; utile lors du dépannage. Valeur par défaut : `false`.

<div id="proxy_url-p">
  ### `--proxy_url`, `-p`
</div>

URL de proxy utilisée pour acheminer les requêtes. Ne la définissez que si vous êtes derrière un proxy.

<div id="examples">
  ### Exemples
</div>

<div id="fetching-auth0-tenant-configuration-in-the-yaml-format">
  #### Récupération de la configuration du locataire Auth0 au format YAML
</div>

`a0deploy export -c=config.json --format=yaml --output_folder=local`

<div id="fetching-auth0-tenant-configuration-in-directory-json-format">
  #### Récupération de la configuration du locataire Auth0 au format de répertoire (JSON)
</div>

`a0deploy export -c=config.json --format=directory --output_folder=local`

<div id="fetching-auth0-tenant-configurations-with-ids-of-all-assets">
  #### Récupération de la configuration du locataire Auth0 avec les ID de toutes les ressources
</div>

`a0deploy export -c=config.json --format=yaml --output_folder=local --export_ids=true`

<div id="import-command">
  ## commande import
</div>

Applique les configurations de la machine locale au locataire Auth0.

<div id="input_file-i">
  ### `--input_file`, `-i`
</div>

Chemin. Indique l’emplacement des fichiers de configuration des ressources. Pour les formats YAML, ce paramètre pointe vers le fichier `tenant.yaml`; pour les formats de répertoire, il pointe vers le répertoire de configuration des ressources.

<div id="config_file-c">
  ### `--config_file`, `-c`
</div>

Chemin d’accès. Spécifie le fichier de configuration défini par l’utilisateur (`config.json`). Pour en savoir plus, consultez [Configurer le Deploy CLI](/fr-CA/docs/deploy-monitor/deploy-cli-tool/configure-the-deploy-cli).

<div id="env">
  ### `--env`
</div>

Booléen. Indique si l’outil doit prendre en compte les variables d’environnement. Valeur par défaut : `true`.

<div id="proxy_url-p">
  ### `--proxy_url`, `-p`
</div>

Une URL pour faire transiter les requêtes par un proxy. Ne la définissez que si vous êtes derrière un proxy.

<div id="debug">
  ### `--debug`
</div>

Booléen. Active une journalisation des erreurs plus verbeuse; utile lors du dépannage. Valeur par défaut : `false`.

<div id="examples">
  ### Exemples
</div>

<div id="deploying-configuration-for-yaml-formats">
  #### Déploiement de la configuration pour les formats YAML
</div>

`a0deploy import -c=config.json --input_file=local/tenant.yaml`

<div id="deploying-configuration-for-directory-format">
  #### Déployer la configuration au format de répertoire
</div>

`a0deploy import -c=config.json --input_file=local`

<div id="deploying-configuration-with-environment-variables-ignored">
  #### Déployer la configuration sans tenir compte des variables d’environnement
</div>

`a0deploy import -c=config.json --input_file=local/tenant.yaml --env=false`
