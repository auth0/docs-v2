Accédez à la page [Auth0 Public Cloud Status](https://status.auth0.com) pour vérifier la disponibilité des services d’Auth0 dans le cloud public.

Sélectionnez une **région** dans le menu déroulant pour voir l’état des services principaux (comme l’authentification des utilisateurs et l’authentification machine à machine) et des services complémentaires (comme <Tooltip tip="MFA : Processus d’authentification des utilisateurs qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code reçu par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=MFA">MFA</Tooltip>, <Tooltip tip="Management API : Produit qui permet aux clients d’effectuer des tâches d’administration." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>, les journaux du locataire et le CDN).

Pour en savoir plus sur les régions disponibles, consultez le tableau [Région, localité et sous-localité](/fr-CA/docs/get-started/auth0-overview/create-tenants#region-locality-and-sub-locality) dans [Créer des locataires](/fr-CA/docs/get-started/auth0-overview/create-tenants).

<div id="subscribe-to-public-cloud-status-updates">
  ## S’abonner aux mises à jour du statut du cloud public
</div>

Sélectionnez une **région**, puis **S’abonner** pour recevoir des mises à jour. Vous pouvez choisir l’une des deux options suivantes pour consulter le statut :

1. Suivez [@auth0status](https://x.com/auth0status) sur X pour obtenir les plus récentes mises à jour du statut.
2. Abonnez-vous au flux Atom ou RSS d’Auth0 pour recevoir les mises à jour du statut qui concernent votre locataire. Remplacez `{YOUR_TENANT_DOMAIN}` par le **Domaine** de votre locataire.

Pour trouver le **Domaine** de votre locataire, accédez à [Auth0 Dashboard &gt; Applications](https://manage.auth0.com/dashboard/#/applications), sélectionnez une application, puis accédez à **Settings**. Le **Domaine** de votre locataire figure sous **Basic Information**.

<div id="check-auth0-status-for-private-cloud">
  ## Vérifier l’état d’Auth0 pour le cloud privé
</div>

Accédez à [Auth0 Public Cloud Status](https://status.auth0.com), puis sélectionnez **Private Cloud Login**.

Une fois authentifié, vous pouvez consulter l’état de tout environnement autre qu’un environnement de développement (par exemple, `travel0-prod-1`) s’il est disponible sur la plateforme convergée d’Auth0.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour accéder à l’état de votre environnement de cloud privé, vous devez être membre du locataire de soutien du cloud public associé. Pour en savoir plus, consultez [Open and Manage Support Tickets](/fr-CA/docs/troubleshoot/customer-support/open-and-manage-support-tickets) et [Manage Dashboard Access](/fr-CA/docs/get-started/manage-dashboard-access).
</Callout>

Pour revenir à la page Auth0 Public Cloud Status, sélectionnez **Auth0 Public Cloud Status** dans la navigation en haut à droite.

<div id="subscribe-to-private-cloud-status-updates">
  ## S’abonner aux mises à jour d’état du cloud privé
</div>

Sélectionnez un environnement, puis sélectionnez **Subscribe**.

Abonnez-vous au flux Atom ou RSS d’Auth0 en copiant l’URL générée dans la zone « Subscribe » pour recevoir les mises à jour d’état qui touchent votre environnement.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  L’URL du flux ATOM / RSS générée est unique à chaque environnement de cloud privé et contient des jetons dont vous pouvez effectuer la rotation en [contactant le soutien technique](https://support.auth0.com/).
</Callout>
