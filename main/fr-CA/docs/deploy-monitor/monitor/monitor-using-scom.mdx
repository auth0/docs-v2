title: Surveiller Auth0 à l’aide de System Center Operations Manager
---

Vous pouvez surveiller Auth0 en tant qu’application web standard à l’aide de System Center Operations Manager (SCOM) ou de tout outil prenant en charge les transactions synthétiques. Nous vous recommandons d’ajouter une transaction synthétique de connexion qui inclut les extensions sur lesquelles vos applications s’appuient (comme les Rules qui exécutent du code personnalisé pour l’intégration à d’autres services de votre entreprise).

<div id="set-up-scom">
  ## Configurer SCOM
</div>

1. Ajoutez une nouvelle instance SCOM à l’aide de l’**assistant Add Monitoring Wizard** :

| Champ                                                | Description                                          |
| ---------------------------------------------------- | ---------------------------------------------------- |
| Nom                                                  | Nom descriptif de l’instance SCOM.                   |
| Description                                          | Description de ce que cette instance SCOM surveille. |
| Sélectionner le pack d’administration de destination | Pack d’administration par défaut                     |

Lorsque vous avez terminé, cliquez sur **Suivant** pour continuer.

2. Cliquez sur **Ajouter** pour saisir les URL que vous souhaitez faire surveiller par SCOM. Lorsque vous avez terminé, cliquez sur **Suivant** pour continuer.

3. Cliquez sur **Ajouter** pour configurer un emplacement à partir duquel vous souhaitez effectuer la surveillance.

4. Dans la boîte de dialogue contextuelle, recherchez **Internal location - Agent**. Sélectionnez l’adresse appropriée, puis cliquez sur **Ajouter**. Cliquez ensuite sur **OK** pour terminer la sélection de l’emplacement. Lorsque vous avez terminé, cliquez sur **Suivant** pour continuer.

5. Définissez la fréquence à laquelle SCOM collecte les données pour chaque point de terminaison :

| Données                                           | Fréquence               |
| ------------------------------------------------- | ----------------------- |
| Fréquence de test                                 | 60 secondes             |
| Intervalle de collecte des données de performance | 60 secondes             |
| Délai d’expiration du test                        | 30 secondes             |
| Code d’état HTTP                                  | Supérieur ou égal à 400 |

6. Lorsque vous avez terminé, cliquez sur **Suivant** pour continuer.

<div id="run-scom-tests">
  ## Exécuter les tests SCOM
</div>

1. Cliquez sur **Run Test** pour tester chaque point de terminaison et vous assurer que les paramètres de connexion fournis sont corrects.
2. Une fois la configuration de votre instance SCOM terminée, vous pouvez consulter l’activité dans l’onglet **Monitoring**.

<div id="review-test-results">
  ## Consulter les résultats des tests
</div>

Cliquez sur **État de l’application Web** pour afficher les informations recueillies par SCOM.
