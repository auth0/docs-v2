L’option de déploiement Private Cloud comprend deux options supplémentaires : le basculement géographique et la conformité PCI. Si vous avez des questions sur ces fonctionnalités, veuillez [communiquer avec l’équipe des ventes d’Auth0](https://auth0.com/get-started?place=documentation%20post\&type=link\&text=contact%20auth0%20sales).

<div id="private-cloud-geo-failover">
  ## Basculement géographique de Private Cloud
</div>

Le basculement géographique est une configuration de basculement actif-passif qui permet aux clients de définir une région secondaire à utiliser en cas de panne régionale majeure. En général, toute région du même fournisseur infonuagique offrant une latence inférieure à 100 ms peut être choisie comme région de basculement. Dans certains cas, toutefois, les fournisseurs infonuagiques privilégient des régions de basculement précises. L’équipe d’intégration d’Auth0 travaillera avec vous pour sélectionner les régions les mieux adaptées à votre cas d’utilisation.

De plus, une troisième région de « départage » est requise pour l’un de nos stockages de données. Cette troisième région ne reçoit jamais de trafic en production, mais elle est nécessaire pour élire un nouveau leader en cas de basculement.

Si une panne régionale est détectée, Auth0 déclenche le basculement géographique au nom du client. L’ensemble du processus est automatisé une fois déclenché. Lorsqu’il est confirmé que la région ne rencontre plus de problèmes, Auth0 effectuera le retour vers la région principale.

  ## Conformité PCI
