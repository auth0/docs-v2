title: Détails des invites dans les journaux du locataire
---

Les journaux du locataire Auth0 fournissent des détails de transaction pour les actions d’authentification des utilisateurs. Les journaux du locataire consignent des informations provenant des processus d’identification, de connexion et de vérification à des fins d’examen. Vous pouvez vérifier comment l’utilisateur s’est authentifié, quels identifiants il a fournis, la durée de la transaction et l’état de la connexion.

Vous trouverez ci-dessous les valeurs par défaut consignées dans les journaux du locataire :

| Attribut      | Description                                                                |
| ------------- | -------------------------------------------------------------------------- |
| `completedAt` | Heure de fin de l’étape, y compris le temps d’interaction de l’utilisateur |
| `elapsedTime` | Différence entre `completedAt` et `initiatedAt`                            |
| `flow`        | Varie selon le flux de connexion (c.-à-d. Universal ou Classic)       |
| `initiatedAt` | Heure de début de l’étape                                                  |
| `name`        | Nom de l’étape                                                             |

<div id="login">
  ## login
</div>

L’étape de connexion fournit les valeurs suivantes :

| Attribut    | Description                                                                         |
| ----------- | ----------------------------------------------------------------------------------- |
| `flow`      | Peut prendre deux valeurs, selon qu’il s’agit d’Universal Login ou de Classic Login |
| `user_id`   | ID Auth0 de l’utilisateur authentifié à cette étape                                 |
| `user_name` | Nom de l’utilisateur authentifié à cette étape                                      |

Champs supplémentaires contenant les informations suivantes :

| Attribut        | Description                                                        |
| --------------- | ------------------------------------------------------------------ |
| `connection`    | Connexion utilisée pour authentifier l’utilisateur                 |
| `connection_id` | ID de la connexion utilisée pour authentifier l’utilisateur        |
| `strategy`      | Stratégie de la connexion utilisée pour authentifier l’utilisateur |
| `identity`      | Identité authentifiée de l’utilisateur                             |

Valeurs des événements supplémentaires dans le flux de connexion :

| Nom                            | Expérience Universal Login | Description                                                                                     |
| ------------------------------ | -------------------------- | ----------------------------------------------------------------------------------------------- |
| `federated-authenticate`       | Universal                  | Connexion fédérée utilisée pour l’authentification, sociale, juridique ou d’entreprise          |
| `lock-password-authenticate`   | Classic                    | Utilisateur authentifié avec un nom d’utilisateur et un mot de passe                            |
| `oauth1-authenticate`          | Classic                    | Utilisateur authentifié au moyen d’une fédération OAuth 1                                       |
| `oauth2-authenticate`          | Classic                    | Utilisateur authentifié au moyen d’une fédération OAuth 2                                       |
| `oidc-authenticate`            | Classic                    | Utilisateur authentifié au moyen d’une fédération OIDC                                          |
| `prompt-authenticate`          | Universal                  | Utilisateur authentifié avec un nom d’utilisateur et un mot de passe                            |
| `prompt-authenticate-password` | Universal                  | Utilisateur authentifié avec un nom d’utilisateur et un mot de passe dans le flux `idfirst`     |
| `prompt-signup`                | Universal                  | Utilisateur inscrit à l’aide d’un nom d’utilisateur et d’un mot de passe                        |
| `prompt-signup-password`       | Universal                  | Utilisateur inscrit à l’aide d’un nom d’utilisateur et d’un mot de passe dans le flux `idfirst` |
| `saml-authenticate`            | Classic                    | Utilisateur authentifié au moyen d’une fédération SAML                                          |
| `wsfed-authenticate`           | Classic                    | Utilisateur authentifié au moyen d’une fédération WS-FED                                        |

<div id="consent">
  ## consent
</div>

L’étape de consentement fournit les valeurs suivantes :

| Attribut    | Description                                                                                                    |
| ----------- | -------------------------------------------------------------------------------------------------------------- |
| `grantInfo` | Le `{"{myApiIdentifier}"}` ou l’identificateur de l’API pour laquelle l’utilisateur a accordé une autorisation |
| `audience`  | Audience du jeton                                                                                              |
| `id`        | Autorisation accordée par l’utilisateur et enregistrée dans la base de données                                 |
| `scope`     | Scope approuvé par l’utilisateur                                                                               |

<div id="login-email-verification">
  ## login-email-verification
</div>

L’invite de vérification du courriel comporte les valeurs suivantes :

| Attribut                    | Description                                                       |
| --------------------------- | ----------------------------------------------------------------- |
| ` login-email-verification` | Est inclus si l’invite de vérification du courriel a été affichée |

<div id="redirect">
  ## redirection
</div>

L’étape de redirection fournit les valeurs suivantes :

| Attribut | Description                                    |
| -------- | ---------------------------------------------- |
| `URL`    | URL vers laquelle l’utilisateur a été redirigé |

<div id="mfa">
  ## mfa
</div>

<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification d’un utilisateur qui utilise un facteur supplémentaire en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Multi-factor+authentication">Authentification multifacteur</Tooltip> fournit les valeurs suivantes :

| Attribut        | Description                                                                                                                      |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `flow`          | Peut prendre deux valeurs selon qu’il est utilisé avec l’expérience Classic (`mfa`) ou l’expérience Universal (`universal-mfa`). |
| `performed_acr` | `acr` qui sera inclus dans le `id_token`.                                                                                        |
| `performed_amr` | `amr` qui sera inclus dans le `id_token`.                                                                                        |
| `provider`      | Fournisseur utilisé pour l’authentification multifacteur.                                                                        |
