description: Décrit les renseignements personnels identifiables (PII) qu’Auth0 peut consigner dans les journaux du locataire.
title: Renseignements personnels identifiables dans les journaux d’Auth0
---

De nombreux journaux du locataire Auth0 peuvent contenir des renseignements personnels identifiables (PII). Les événements d’authentification déclenchent des entrées de journal qui peuvent inclure des renseignements personnels identifiables sur les utilisateurs. De plus, si vous utilisez la <Tooltip tip="Management API : produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> ou l’Auth0 Dashboard pour ajouter des utilisateurs, Auth0 consigne les détails du compte utilisateur.

Ces champs des entrées de journal peuvent contenir des renseignements personnels identifiables :

* Nom
* Numéro de téléphone (utilisé pour l’<Tooltip tip="authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code transmis par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip>)
* Adresse de courriel
* Adresse IP (qui peut révéler l’emplacement)
* Tout renseignement utilisateur personnalisé que vous définissez

Auth0 ne consigne jamais dans les journaux les <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+tokens">jetons d’accès</Tooltip> d’Auth0 ni ceux de quelque <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> que ce soit. Lorsqu’un échange de code d’autorisation a lieu, les journaux n’affichent qu’un code partiel (par exemple : `code: 31XXXXX`).
