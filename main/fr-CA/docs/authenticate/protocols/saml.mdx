Le protocole <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des informations d’authentification sans mot de passe." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Security+Assertion+Markup+Language">Security Assertion Markup Language</Tooltip> (SAML) est un standard ouvert fondé sur XML pour l’authentification et l’autorisation entre deux entités, sans mot de passe :

* **fournisseur de services** (SP) accepte de faire confiance au <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des informations d’authentification sans mot de passe." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> pour authentifier les utilisateurs.
* **fournisseur d’identité** (IdP) authentifie les utilisateurs et fournit aux fournisseurs de services une assertion d’authentification indiquant qu’un utilisateur a bien été authentifié.

Auth0 prend en charge le protocole SAML et peut faire office d’IdP, de SP ou des deux, notamment pour :

* les applications Web SAML2
* les intégrations SAML de <Tooltip tip="Single Sign-On (SSO) : service qui, après la connexion d’un utilisateur à une application, connecte automatiquement cet utilisateur à d’autres applications." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=SSO">SSO</Tooltip>
* le SSO initié par l’IdP
* Microsoft Active Directory Federation Services (ADFS)
* la signature et le chiffrement des requêtes SAML

<div id="supported-saml-bindings-and-options">
  ## Liaisons et options SAML prises en charge
</div>

Auth0 prend en charge les liaisons SAML suivantes :

* HTTP Redirect
* HTTP POST

Auth0 prend en charge les options SAML suivantes :

* Profil SSO pour navigateur Web
* Profil de déconnexion unique
* Profil de gestion de l’identifiant de nom
* Profil de correspondance de l’identifiant de nom
