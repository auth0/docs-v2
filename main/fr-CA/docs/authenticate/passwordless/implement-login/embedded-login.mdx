---
title: Authentification Passwordless avec connexion intégrée
description: Décrit comment implémenter l’authentification Passwordless avec Universal Login.
---

Pour intégrer l’interface de connexion à votre application, vous pouvez utiliser notre [API Passwordless intégrée](/fr-CA/docs/authenticate/passwordless/implement-login/embedded-login/relevant-api-endpoints) ou nos SDK.

Selon le type d’application que vous souhaitez créer, vous devrez l’implémenter différemment :

* Pour les applications monopage (SPA) (par exemple, Angular ou React), consultez [Authentification Passwordless intégrée pour les applications monopage (SPA)](/fr-CA/docs/authenticate/passwordless/implement-login/embedded-login/spa).
* Pour les applications natives (par exemple, iOS, Android ou les applications de bureau), consultez [Connexion Passwordless intégrée dans les applications natives](/fr-CA/docs/authenticate/passwordless/implement-login/embedded-login/native).
* Pour les applications Web traditionnelles (par exemple, NodeJS, Java, Rails ou .NET), consultez [Connexion Passwordless intégrée dans les applications Web traditionnelles](/fr-CA/docs/authenticate/passwordless/implement-login/embedded-login/webapps).

La connexion intégrée dans les applications Web traditionnelles et les SPA nécessite une configuration de l’authentification inter-origine. Pour en savoir plus, consultez [Authentification inter-origine](/fr-CA/docs/authenticate/login/cross-origin-authentication).
