---
description: Décrit certaines limites de l’utilisation des connexions Passwordless.
title: Limites des connexions Passwordless
---

Le type d’authentification que vous souhaitez utiliser dépend de la version de <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Universal+Login">Universal Login</Tooltip> que vous avez configurée pour votre application. Si vous utilisez l’expérience Universal Login, vous pouvez configurer [l’authentification Passwordless avec WebAuthn et la biométrie de l’appareil](/fr-CA/docs/authenticate/login/auth0-universal-login/passwordless-login/webauthn-device-biometrics) ou utiliser des connexions <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=passwordless">Passwordless</Tooltip> comme décrit ci-dessous.

Vous pouvez configurer l’authentification Passwordless par courriel ou par SMS à l’aide des méthodes suivantes :

1. Universal Login + paramètres de connexion Passwordless
2. Classic Login + Lock
3. Classic Login + interface utilisateur personnalisée + Auth0.js

Pour en savoir plus sur ces implémentations, consultez [Configurer le courriel ou le SMS pour l’authentification Passwordless](/fr-CA/docs/authenticate/login/auth0-universal-login/passwordless-login/email-or-sms).

<div id="errors">
  ### Erreurs
</div>

Si vous utilisez une authentification intégrée, nous vous recommandons d’utiliser des <Tooltip tip="Domaine personnalisé : domaine tiers avec un nom spécialisé ou personnalisé." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=custom+domains">domaines personnalisés</Tooltip>. Pour en savoir plus, consultez [Domaines personnalisés](/fr-CA/docs/customize/custom-domains).

<div id="considerations">
  ## Points à considérer
</div>

* Vous ne pouvez pas créer d’utilisateurs Passwordless depuis l’<Tooltip tip="Auth0 Dashboard : console principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Créez-les directement à l’aide de la [Management API](https://auth0.com/docs/api/management/v2#!/Users/post_users).
* Les <Tooltip tip="Refresh Token : jeton utilisé pour obtenir un nouveau Jeton d’accès sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Refresh+tokens">Jetons d’actualisation</Tooltip> ne peuvent pas être obtenus lorsque vous utilisez un lien magique Passwordless par courriel; seul un OTP peut l’être.
