title: Surveiller les événements liés aux clés d’accès dans les journaux du locataire
---

Vous pouvez surveiller les flux de clés d’accès associés à vos connexions de base de données dans les [journaux du locataire](/fr-CA/docs/deploy-monitor/logs).

Vous pouvez consulter les événements de journal dans le <Tooltip tip="Auth0 Dashboard : le produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ([Monitoring &gt; Logs](https://manage.auth0.com/#/logs)) ou récupérer les journaux à l’aide de la [Management API](https://auth0.com/docs/api/management/v2/logs/get-logs).

| Code d’événement                | Scénario                                                                                                        | Remarques                                                                                                                                                                                                                                                                                                                |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `ss`                            | Lors de l’inscription, un utilisateur a enregistré une clé d’accès comme méthode d’authentification principale. | Cet événement de journal contient le champ de tableau `details.authentication_methods` avec la valeur `passkey`.<br /><br />Si un utilisateur s’est inscrit avec un mot de passe, ce champ de tableau contient la valeur `pwd`.                                                                                          |
| `s`                             | Lors de la connexion, un utilisateur s’est authentifié avec une clé d’accès.                                    | Cet événement de journal contient le tableau `details.prompts`, qui répertorie l’ensemble des invites qu’un utilisateur a parcourues pendant la connexion.<br /><br />Si un utilisateur s’est authentifié à l’aide d’une clé d’accès, le tableau contient un élément avec le champ suivant : `"performed_amr": ["phr"]`. |
| `gd_enrollment_complete`        | Un flux d’inscription progressif ou local s’est terminé avec succès.                                            | Cet événement de journal contient le champ `details.authenticator.type` avec la valeur `passkey` si un utilisateur a inscrit une clé d’accès.                                                                                                                                                                            |
| `gd_webauthn_enrollment_failed` | Une erreur s’est produite pendant un flux d’inscription progressif ou local.                                    | Cet événement de journal contient le champ `details.authentication_methods array` avec la valeur `passkey` si un utilisateur a tenté d’inscrire une clé d’accès.                                                                                                                                                         |
