---

Auth0 prend en charge la connexion d’entreprise pour les applications Web et les applications natives. La connexion d’entreprise est une méthode d’authentification qui permet aux utilisateurs de se connecter à une application à l’aide des identifiants existants d’un fournisseur d’identité d’entreprise, comme Google Workspace ou Microsoft Azure Active Directory (Entra ID). Cela est distinct de la connexion et de l’autorisation d’applications auprès d’un fournisseur externe pour leur permettre d’accéder à des API externes au nom de l’utilisateur. Pour en savoir plus, consultez [Authentification des utilisateurs et comptes connectés](/fr-CA/docs/secure/tokens/token-vault/connected-accounts-for-token-vault#user-authentication-vs-connected-accounts).

Auth0 prend en charge les fournisseurs d’entreprise suivants prêts à l’emploi :

* [Active Directory/LDAP](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/active-directory-ldap)
* [ADFS](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/adfs)
* [Azure Active Directory Native](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory-native)
* [Google Workspace](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/google-apps)
* [OpenID Connect](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/oidc)
* [Okta](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/okta)
* [PingFederate](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/ping-federate)
* [SAML](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/saml)
* [Azure Active Directory](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2)

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les forfaits B2B et Enterprise d’Auth0 permettent un nombre illimité de [connexions d’entreprise Okta](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/okta). Pour en savoir plus, consultez [la page de tarification d’Auth0](https://auth0.com/pricing).
</Callout>

Vous pouvez également explorer les connexions d’entreprise prises en charge par des partenaires dans le [Marketplace d’Auth0](https://marketplace.auth0.com/features/enterprise-connections).

<div id="openid-connect-oidc-protocol">
  ## Protocole OpenID Connect (OIDC)
</div>

Les connexions d’entreprise utilisant <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker les informations de connexion." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=OpenID">OpenID</Tooltip> Connect ou Okta Workforce comme <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker les informations de connexion." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> peuvent prendre en charge les éléments suivants :

* Proof Key for Code Exchange (PKCE)

  * Pour en savoir plus sur PKCE, consultez [Protecting Apps with PKCE](https://www.oauth.com/oauth2-servers/pkce/).
* Mappage des revendications d’attributs et du <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+token">jeton d’accès</Tooltip>
* Intégration à UserInfo

Vous pouvez actuellement implémenter ces fonctionnalités pour les connexions [OpenID Connect](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/oidc) ou [Okta Workforce](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/okta). Pour en savoir plus, consultez [Configurer PKCE et le mappage des revendications pour les connexions OIDC](/fr-CA/docs/authenticate/identity-providers/enterprise-identity-providers/configure-pkce-claim-mapping-for-oidc).
