description: Mettez à niveau un connecteur AD/LDAP Auth0 installé vers la version la plus récente sous Windows ou Linux, tout en préservant la configuration existante et le lien avec le locataire.
title: Mettre à jour les connecteurs AD/LDAP
---

S’il y a plusieurs instances du connecteur AD/LDAP dans votre déploiement, Auth0 recommande d’effectuer les étapes ci-dessous pour chaque instance, une à la fois, afin qu’au plus une seule instance soit hors service à un moment donné.

<div id="admin-console-windows">
  ## Console d’administration (Windows)
</div>

1. Ouvrez la console d’administration en accédant à `http://localhost:8357/`. Si la console d’administration a un accès sortant à Internet, elle vérifiera si une nouvelle version est disponible et l’indiquera en haut de la page :

   <Frame>![Écran de mise à jour de la console d’administration des connecteurs AD/LDAP](/docs/images/cdy7uua7fh8z/6Yp2uRnEU3RMeCqgHa6Ztl/b9c04ab83b9fd725910291b01ac9d05d/connector-update-available.png)</Frame>
2. Accédez à l’onglet **Update** pour mettre à jour l’installation sur cette machine vers la version la plus récente. La mise à jour prendra environ 2 minutes, et les journaux de mise à jour s’afficheront dans la console d’administration.

<div id="updater-script-windows">
  ## Script de mise à jour (Windows)
</div>

Le script de mise à jour permet de mettre à jour le connecteur AD/LDAP à partir de la ligne de commande en effectuant les étapes suivantes :

1. Vérifiez si une mise à jour est disponible.
2. Sauvegardez la configuration existante, les certificats et profileMapper.js.
3. Désinstallez le connecteur AD/LDAP.
4. Téléchargez la mise à jour.
5. Installez le connecteur AD/LDAP.
6. Restaurez la configuration existante, les certificats et profileMapper.js.
7. Démarrez le service Windows.
8. Pour exécuter le script de mise à jour, utilisez l’instruction suivante dans la ligne de commande :
   `@powershell -NoProfile -ExecutionPolicy Bypass -Command "iex ((new-object net.webclient).DownloadString('https://cdn.auth0.com/connector/windows/update-connector.ps1'))"`

Le script de mise à jour utilise des commandes PowerShell spécifiques qui ne sont disponibles que dans Windows PowerShell 3.0 et les versions ultérieures. Si vous utilisez Windows 2008 ou Windows 2008 R2, vous devrez peut-être d’abord mettre à jour votre version de PowerShell.

Pour en savoir plus, consultez [Download Window Management Framework](https://www.microsoft.com/en-us/download/details.aspx?id=34595).

<div id="manual-update-windowslinux">
  ## Mise à jour manuelle (Windows/Linux)
</div>

1. Vérifiez la version installée. Survolez l’indicateur d’état du connecteur dans la console. L’infobulle indiquera l’état actuel et la version installée.
2. Téléchargez la plus récente version du [programme d’installation](https://cdn.auth0.com/connector/windows/adldap-5.0.10.msi?_ga=2.58973869.1675886805.1589821069-1123834949.1586212750).
   Utilisez la [version](https://github.com/auth0/ad-ldap-connector/releases/tag/v5.0.10) du dépôt GitHub pour les autres plateformes.
3. Sauvegardez votre configuration actuelle à partir de la console d’administration ou manuellement.

   1. Dans la console d’administration, cliquez sur le bouton **Download** pour générer un fichier .zip contenant le fichier `config.json`, le dossier `certs` et le fichier `lib\\profileMapper.js`.
   2. Pour le faire manuellement, sauvegardez les mêmes fichiers indiqués ci-dessus à partir du dossier `%Program Files(x86)%\Auth0\AD LDAP Connector\`. Ce chemin fonctionne sous Windows. Sur les autres plateformes, les fichiers peuvent se trouver à un autre emplacement, mais il s’agira des mêmes éléments.
4. Exécutez le programme d’installation et suivez les instructions.
5. Fermez la boîte de dialogue de configuration sans rien modifier.
6. Restaurez les fichiers à partir de la console d’administration ou manuellement.

   1. Dans la console d’administration, téléversez le fichier .zip dans l’onglet **Import / Export**.
   2. Pour le faire manuellement, copiez les fichiers que vous avez sauvegardés précédemment.
7. Redémarrez le service **Auth0 AD LDAP** à partir de la console des services.
