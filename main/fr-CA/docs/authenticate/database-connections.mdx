Auth0 fournit des connexions de base de données pour authentifier les utilisateurs à l’aide d’un identifiant (courriel, nom d’utilisateur ou numéro de téléphone) et d’un mot de passe ou de [clés d’accès](/fr-CA/docs/authenticate/database-connections/passkeys/configure-passkey-policy). Ces identifiants sont stockés de façon sécurisée dans le répertoire d’utilisateurs Auth0 ou dans votre propre base de données.

Vous pouvez créer une nouvelle connexion de base de données et gérer les connexions existantes dans [Auth0 Dashboard &gt; Authentication &gt; Database](https://manage.auth0.com/#/connections/database).

<div id="database-options">
  ## Options de la base de données
</div>

<div id="use-auth0-user-store">
  ### Utiliser le répertoire d’utilisateurs d’Auth0
</div>

Par défaut, Auth0 fournit l’infrastructure de base de données nécessaire pour stocker vos utilisateurs. Ce scénario offre la meilleure performance pour le processus d’authentification, puisque toutes les données sont stockées dans Auth0.

La base de données hébergée par Auth0 est hautement sécurisée. Les mots de passe ne sont jamais stockés ni consignés en texte brut; ils sont hachés avec **bcrypt**. Il est également possible d’appliquer différents niveaux d’exigences de sécurité pour les mots de passe. Pour en savoir plus, consultez [la force du mot de passe dans les connexions de base de données Auth0](/fr-CA/docs/authenticate/database-connections/password-strength).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour les connexions de base de données, Auth0 limite le nombre de tentatives de connexion répétées par utilisateur et par adresse IP. Pour en savoir plus, consultez [les limites de connexion à la base de données](/fr-CA/docs/troubleshoot/customer-support/operational-policies/rate-limit-policy#database-login-limits).
</Callout>

Pour utiliser l’infrastructure de base de données d’Auth0 comme répertoire d’utilisateurs, plusieurs options s’offrent à vous pour migrer vos utilisateurs vers Auth0. La fonctionnalité de [migration automatique](/fr-CA/docs/manage-users/user-migration/configure-automatic-migration-from-your-database) ajoute vos utilisateurs à la base de données Auth0 un à la fois, au fur et à mesure qu’ils se connectent, et évite d’avoir à leur demander de réinitialiser leur mot de passe tous en même temps. Vous pouvez aussi utiliser la Management API pour créer une tâche d’[importation d’utilisateurs](/fr-CA/docs/manage-users/user-migration/bulk-user-imports). Pour en savoir plus et voir des exemples de migration, consultez [les scénarios de migration des utilisateurs](/fr-CA/docs/manage-users/user-migration/user-migration-scenarios).

<div id="use-your-own-database">
  ### Utiliser votre propre base de données
</div>

Si vous disposez déjà d’un répertoire d’utilisateurs ou souhaitez stocker les identifiants des utilisateurs sur votre propre serveur, Auth0 vous permet de vous connecter à une [base de données personnalisée](/fr-CA/docs/authenticate/database-connections/custom-db) ou à un dépôt et de l’utiliser comme fournisseur d’identité.

<Frame>![](/docs/images/cdy7uua7fh8z/3SNKwx0KzkUwbodqUQbtvW/df720f8871b56dd7ec399a24aa621c36/Authentication>Database>use-my-own-database.png)</Frame>

Dans ce scénario, vous fournissez le script de connexion qui authentifie l’utilisateur et s’exécute chaque fois qu’un utilisateur tente de se connecter. Vous pouvez aussi [créer des scripts](/fr-CA/docs/authenticate/database-connections/custom-db/templates) pour l’inscription, la vérification du courriel, la réinitialisation du mot de passe et la suppression d’utilisateurs.

Les scripts sont écrits en Node.js. Auth0 fournit des [modèles](/fr-CA/docs/authenticate/database-connections/custom-db/templates) pour les bases de données les plus courantes, comme ASP.NET Membership Provider, MongoDB, MySQL, PostgreSQL, SQL Server et Windows Azure SQL Database, ainsi que pour un service Web accessible par Basic Auth. En pratique, vous pouvez vous connecter à presque n’importe quel type de base de données ou de service Web à l’aide d’un script personnalisé.
