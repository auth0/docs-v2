title: Okta Access Gateway
---

**Okta Access Gateway (OAG)** est une solution conçue pour apporter l’identité moderne aux applications sur site héritées afin de protéger votre cloud hybride. Vous pouvez maintenant configurer facilement Okta Access Gateway pour utiliser Auth0 comme <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> afin d’authentifier les utilisateurs et bénéficier de l’<Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Single+Sign-On">authentification unique</Tooltip> (SSO) lors de l’accès aux applications sur site. Pour en savoir plus sur la configuration, consultez le [site Web d’Okta Access Gateway](https://help.okta.com/oag/en-us/content/topics/access-gateway/ag-main.htm).

Une fois la configuration terminée correctement, vous pouvez consulter vos applications OAG comme des **intégrations SSO** dans l’<Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.
