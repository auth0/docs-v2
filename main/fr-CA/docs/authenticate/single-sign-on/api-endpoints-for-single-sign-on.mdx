---

Lors de la mise en œuvre de <Tooltip tip="authentification unique (SSO) : service qui, lorsqu’un utilisateur ouvre une session dans une application, l’ouvre automatiquement dans d’autres applications." cta="Consulter le glossaire" href="/fr-CA/docs/glossary?term=Single+Sign-on">l’authentification unique</Tooltip> (SS0) et de la configuration de ses paramètres, les points de terminaison d’API suivants vous seront utiles.

<div id="authentication-api-endpoints">
  ## Points de terminaison de l’Authentication API
</div>

<div id="post-logincallback">
  ### POST /login/callback
</div>

Pour les flux SSO initiés par un <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP), le point de terminaison [POST /login/callback](https://auth0.com/docs/api/authentication#idp-initiated-single-sign-on-sso-flow) peut recevoir une requête <Tooltip tip="Security Assertion Markup Language (SAML) : protocole normalisé permettant à deux parties d’échanger des renseignements d’authentification sans mot de passe." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=SAML">SAML</Tooltip> d’authentification provenant du fournisseur d’identité.

<div id="management-api">
  ## Management API
</div>

Le <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> offre plusieurs points de terminaison conçus pour vous aider à gérer le SSO.

Pour appeler n’importe quel point de terminaison de la Management API, vous devrez [obtenir un Jeton d’accès](https://auth0.com/docs/api/management/v2/tokens).

<div id="clients">
  ### Applications
</div>

Toutes vos applications et intégrations SSO contiennent des renseignements pertinents pour votre mise en œuvre du SSO. Vous pouvez récupérer ou modifier ces renseignements en appelant l’un des points de terminaison Applications.

Les paramètres suivants sont particulièrement pertinents pour chaque application :

* `sso` : indicateur applicable uniquement aux applications créées au moyen d’intégrations SSO. Si `true`, Auth0 gère le SSO; si `false`, l’IdP gère le SSO.
* `sso_disabled` : indicateur utilisé pour activer ou désactiver le SSO. Si `true`, le SSO est désactivé; si `false`, le SSO est activé. Cette option ne peut être définie qu’au moyen de la Management API.
* `app_type` : type d’application. Si l’application a été créée à l’aide de l’une des intégrations SSO intégrées d’Auth0, le nom de l’intégration SSO serait indiqué (p. ex., `box` ou `concur` au lieu de `native` ou `spa`)

<div id="get-all-clients">
  #### Récupérer toutes les applications
</div>

Le point de terminaison [GET /api/v2/clients](https://auth0.com/docs/api/management/v2#!/Clients/get_clients) permet d’obtenir des renseignements sur les applications que vous avez configurées pour votre locataire.

<div id="create-a-client">
  #### Créer une application
</div>

Le point de terminaison [POST /api/v2/clients](https://auth0.com/docs/api/management/v2#!/Clients/post_clients) sert à créer une nouvelle application.

<div id="get-a-client">
  #### Récupérer une application
</div>

Le [point de terminaison GET /api/v2/clients/{id}](https://auth0.com/docs/api/management/v2#!/Clients/get_clients_by_id) permet d’obtenir des renseignements sur une application précise que vous avez configurée pour votre locataire.

<div id="update-a-client">
  #### Mettre à jour une application
</div>

Le point de terminaison [PATCH /api/v2/clients/{id}](https://auth0.com/docs/api/management/v2#!/Clients/patch_clients_by_id) permet de mettre à jour une application précise, y compris ses paramètres liés au SSO.

<div id="tenants">
  ### Locataires
</div>

Auth0 vous permet de contrôler les paramètres suivants du locataire, qui peuvent avoir une incidence sur votre implémentation du SSO :

* `session_lifetime` : durée pendant laquelle la session Auth0 de l&#39;utilisateur reste valide.
* `idle_session_lifetime` : durée pouvant s&#39;écouler avant que l&#39;utilisateur doive se connecter de nouveau en raison de l&#39;inactivité.

<div id="get-tenant-settings">
  ### Obtenir les paramètres du locataire
</div>

Le point de terminaison [GET /api/v2/tenants/settings](https://auth0.com/docs/api/management/v2#!/Tenants/get_settings) permet de récupérer les paramètres de votre locataire.

<div id="update-tenant-settings">
  ### Mise à jour des paramètres du locataire
</div>

Le point de terminaison [POST /api/v2/connections](https://auth0.com/docs/api/management/v2#!/Tenants/patch_settings) vous permet de mettre à jour les paramètres de votre locataire.
