title: Connexion intégrée
---

Embedded Login permet à vos utilisateurs de se connecter directement à votre application et de transmettre leurs identifiants au serveur Auth0 pour l’authentification. Nous ne recommandons pas d’utiliser Embedded Login. Pour en savoir plus, consultez [Centralized Universal Login vs. Embedded Login](/fr-CA/docs/authenticate/login/universal-vs-embedded-login).

Si vous décidez d’utiliser Embedded Login, vous devez [configurer votre application pour le partage de ressources entre origines multiples (CORS)](/fr-CA/docs/get-started/applications/set-up-cors) et il est recommandé de [configurer un domaine personnalisé](/fr-CA/docs/customize/custom-domains). Vous pouvez ensuite implémenter le [SDK Lock](/fr-CA/docs/libraries/lock) ou le [SDK Auth0.js](/fr-CA/docs/libraries/auth0js) dans votre application, ou appeler directement l’[API d’authentification Auth0](https://auth0.com/docs/api/authentication).

Auth0 prend en charge Embedded Login avec des connexions <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Passwordless">Passwordless</Tooltip> pour plusieurs types d’applications :

* [Connexion Passwordless intégrée dans les applications natives](/fr-CA/docs/authenticate/passwordless/implement-login/embedded-login/native)
* [Connexion Passwordless intégrée dans les applications Web traditionnelles](/fr-CA/docs/authenticate/passwordless/implement-login/embedded-login/webapps)
* [Authentification Passwordless intégrée pour les applications monopage](/fr-CA/docs/authenticate/passwordless/implement-login/embedded-login/spa)
