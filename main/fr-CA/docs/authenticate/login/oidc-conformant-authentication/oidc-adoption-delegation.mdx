---
description: Découvrez comment le pipeline conforme à OIDC affecte votre utilisation de la délégation.
title: Délégation avec OIDC
---

<Warning>
  Par défaut, la délégation est désactivée pour les locataires qui n’utilisaient aucun module complémentaire au 8 juin 2017. Les anciens locataires qui utilisent actuellement un module complémentaire nécessitant la délégation peuvent continuer à utiliser cette fonctionnalité. Si la fonctionnalité de délégation est modifiée ou retirée du service, les clients qui l’utilisent actuellement en seront avisés à l’avance et disposeront de suffisamment de temps pour migrer.
</Warning>

Traditionnellement, la délégation est utilisée pour :

* Échanger un <Tooltip tip="ID Token : jeton destiné à l’application cliente elle-même, plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=ID+token">jeton d’identité</Tooltip> émis pour une application contre un nouveau jeton émis pour une autre application.
* Obtenir un nouveau jeton d’identité à l’aide d’un [jeton d’actualisation](/fr-CA/docs/secure/tokens/refresh-tokens).
* Échanger un jeton d’identité contre un jeton d’API tiers (par exemple, Firebase, AWS).

Comme le pipeline conforme à OIDC exige que les jetons d’identité ne soient plus utilisés pour sécuriser les API et que les <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un Jeton d’accès renouvelé sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=refresh+tokens">jetons d’actualisation</Tooltip> soient utilisés uniquement au point de terminaison [`/oauth/token`](https://auth0.com/docs/api/authentication#refresh-token), le point de terminaison [`/delegation`](https://auth0.com/docs/api/authentication#delegation) est obsolète.

Les applications conformes à OIDC ne peuvent pas être la source ni la cible des demandes de délégation.

<div id="third-party-apis">
  ## API tierces
</div>

Comme il n’existe pas de mécanisme conforme à OIDC pour obtenir des jetons d’API de tiers (par exemple Firebase ou AWS), il est toujours possible d’utiliser la délégation pour obtenir ces jetons.
