title: Configurer le courriel ou le SMS pour l’authentification Passwordless
---

Auth0 vous permet de configurer <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Consulter le glossaire" href="/fr-CA/docs/glossary?term=Universal+Login">Universal Login</Tooltip> afin que les utilisateurs puissent s’authentifier à l’aide d’un lien magique ou d’un mot de passe à usage unique (OTP) envoyé par courriel, ou d’un OTP envoyé par SMS.

Pour activer l’authentification <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Consulter le glossaire" href="/fr-CA/docs/glossary?term=Passwordless">Passwordless</Tooltip> par SMS ou par courriel, vous devez définir une [connexion Passwordless](/fr-CA/docs/authenticate/passwordless) pour votre application.

Une fois la connexion créée, vous devez configurer Universal Login pour l’utiliser. Trois options s’offrent à vous :

<div id="universal-login">
  ## Universal Login
</div>

Cette méthode utilise l’expérience Universal Login, qui prend nativement en charge les connexions Passwordless. Pour en savoir plus, consultez [Passwordless avec Universal Login](/fr-CA/docs/authenticate/passwordless/passwordless-with-universal-login).

<div id="universal-login-lock-passwordless">
  ## Universal Login + Lock (Passwordless)
</div>

Cette méthode utilise une page de connexion personnalisée avec le modèle **Lock (Passwordless)** et authentifie l’utilisateur à l’aide du [SDK Lock for Web](/fr-CA/docs/libraries/lock) :

1. Accédez à [Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/#/login_settings), puis cliquez sur l’onglet **Login**.
2. Activez le bouton bascule **Custom Login Page**, puis sélectionnez le modèle **Lock (Passwordless)**. Le modèle HTML sera mis à jour avec du code qui utilise le widget Lock et des options de personnalisation Passwordless.
3. Personnalisez le modèle, puis cliquez sur **Save Changes**.

Vous pouvez utiliser HTML et CSS pour personnaliser le formulaire de connexion et prévisualiser les modifications dans le Dashboard. Pour en savoir plus sur la personnalisation du modèle **Lock (Passwordless)**, consultez la section Passwordless du [SDK Lock for Web](/fr-CA/docs/libraries/lock).

<div id="universal-login-custom-ui-auth0js">
  ## Universal Login + interface utilisateur personnalisée + Auth0.js
</div>

Cette méthode utilise une page de connexion personnalisée avec le modèle **Custom Login Form** et authentifie les utilisateurs à l’aide du [SDK Auth0.js](/fr-CA/docs/libraries/auth0js). Pour en savoir plus, consultez [Authentification Passwordless avec Universal Login](/fr-CA/docs/authenticate/passwordless/implement-login/universal-login).
