title: Page d’erreur Auth0 par défaut
---

Si votre URL de rappel est valide, lorsqu’une erreur d’autorisation se produit, le <Tooltip tip="Serveur d’autorisation : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre serveur d’autorisation peut contrôler les données, les tâches et les fonctionnalités accessibles à un utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Authorization+Server">serveur d’autorisation</Tooltip> renvoie à votre URL de rappel les paramètres d’erreur et de state appropriés. Si votre URL de rappel n’est pas valide, votre application affiche la page d’erreur générique Auth0 par défaut. Vous pouvez aussi personnaliser les pages d’erreur. Pour en savoir plus, consultez [Pages d’erreur personnalisées](/fr-CA/docs/customize/login-pages/custom-error-pages).

Pour trouver le nom de page par défaut de la page d’erreur générique, consultez [Utiliser le contrôle de version pour gérer vos pages Universal Login](/fr-CA/docs/customize/login-pages/classic-login/version-control).

<div id="display">
  ## Affichage
</div>

La page d’erreur générique affiche les renseignements suivants si vous avez configuré les champs correspondants dans vos [paramètres du locataire](https://manage.auth0.com/#/tenant/).

| Champ                 | Description                                                   |
| --------------------- | ------------------------------------------------------------- |
| Nom affiché           | Nom de votre entreprise                                       |
| URL du logo           | URL du logo de votre entreprise                               |
| Courriel d’assistance | Adresse courriel de l’équipe d’assistance de votre entreprise |
| URL d’assistance      | URL de la page d’assistance de votre entreprise               |

<div id="parameters">
  ## Paramètres
</div>

Le serveur d’autorisation renvoie des paramètres ajoutés à l’URL sous forme de chaîne de requête.

| Paramètre           | Description                                                                          |
| ------------------- | ------------------------------------------------------------------------------------ |
| `client_id`         | Identifiant de l’application Auth0.                                                  |
| `connection`        | Connexion utilisée au moment de l’erreur.                                            |
| `lang`              | Langue définie au moment de l’erreur.                                                |
| `error`             | Code de l’erreur.                                                                    |
| `error_description` | Description de l’erreur.                                                             |
| `tracking`          | Identifiant utilisé par Auth0 pour retrouver les erreurs dans les journaux internes. |

Les paramètres présentés varient selon le type d’erreur et sont propres à la requête. Par exemple, si la requête ne contient pas de `client_id` et entraîne une erreur, aucun <Tooltip tip="ID client : valeur d’identification attribuée par Auth0 à votre ressource enregistrée." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=client+ID">ID client</Tooltip> ne sera renvoyé par la page d’erreur.
