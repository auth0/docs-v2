<div id="endpoint">
  ## Point de terminaison
</div>

`GET /authorize`

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Ce point de terminaison est **obsolète** pour la liaison de comptes. Utilisez plutôt [POST /api/v2/users/{id}/identities](/fr-CA/api/management/v2/users/post-identities). Pour en savoir plus, consultez l’[avis de migration](/fr-CA/migrations/past-migrations#account-linking-removal).
</Callout>

Appelez ce point de terminaison lorsqu’un utilisateur souhaite lier une deuxième méthode d’authentification (par exemple, une connexion de base de données avec nom d’utilisateur et mot de passe, à Facebook).

Ce point de terminaison déclenchera le flux de connexion pour lier un compte existant à un nouveau. Il renverra une redirection 302 vers la `connection` que l’utilisateur actuel souhaite ajouter. L’utilisateur est identifié par le Jeton d’accès renvoyé une fois la connexion réussie.

<div id="remarks">
  ### Remarques
</div>

* La valeur `redirect_uri` doit être indiquée comme une URL de rappel valide dans les [paramètres de votre application](https://manage.auth0.com/dashboard).

<div id="query-parameters">
  ## Paramètres de requête
</div>

<ParamField query="response_type" type="string" required>
  Utilisez `code` pour les flux côté serveur, `token` pour les flux côté client.
</ParamField>

<ParamField query="client_id" type="string" required>
  Le `client_id` de votre application.
</ParamField>

<ParamField query="connection" type="string">
  Le nom de la connexion associée à votre application. Si cette valeur est `null`, l’utilisateur sera redirigé vers la [page de connexion Auth0](https://auth0.com/#/login_page) et le widget de connexion s’affichera avec la première connexion de base de données.
</ParamField>

<ParamField query="redirect_uri" type="string" required>
  L’URL vers laquelle Auth0 redirigera le navigateur une fois l’autorisation accordée par l’utilisateur.
</ParamField>

<ParamField query="access_token" type="string" required>
  Le Jeton d’accès de l’utilisateur connecté.
</ParamField>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                         |
| ------ | --------------------------------------------------- |
| 302    | Redirection vers la connexion pour lier des comptes |
