`GET /samlp/{clientId}`

Utilisez ce point de terminaison pour accepter une requête SAML afin d’initier une connexion.

Vous pouvez également inclure un paramètre `connection` pour vous connecter avec un fournisseur précis. Si aucune connexion n’est spécifiée, la [page de connexion Auth0](https://auth0.com/docs/authenticate/login/auth0-universal-login) s’affichera.

Les requêtes de connexion initiées par le SP peuvent également inclure un paramètre `organization` pour authentifier les utilisateurs dans le contexte d’une organisation. Pour en savoir plus, consultez [Organisations](https://auth0.com/docs/manage-users/organizations).

<div id="parameters">
  ## Paramètres
</div>

<ParamField path="clientId" type="string" required>
  ID client de votre application.
</ParamField>

<div id="request-body">
  ## Corps de la requête
</div>

<ParamField body="connection" type="string">
  Connexion à utiliser lors de la connexion.
</ParamField>

<ParamField body="organization" type="string">
  ID de l’organisation, si l’authentification se fait dans le contexte d’une organisation.
</ParamField>

<div id="response">
  ## Réponse
</div>

| Statut | Description            |
| ------ | ---------------------- |
| 200    | Requête SAML acceptée. |
