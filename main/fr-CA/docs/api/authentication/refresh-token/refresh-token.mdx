<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oauth/token`

Utilisez ce point de terminaison pour actualiser un Jeton d’accès à l’aide du Jeton d’actualisation obtenu lors de l’autorisation.

<div id="learn-more">
  ## Pour en savoir plus
</div>

* [Jetons d’actualisation](https://auth0.com/docs/secure/tokens/refresh-tokens)

<div id="headers">
  ## En-têtes
</div>

<ParamField header="DPoP" type="string">
  Une preuve DPoP pour la requête. Ce paramètre est facultatif et n’est requis que si votre application utilise Demonstrating Proof-of-Possession.
</ParamField>

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="grant_type" type="string" required>
  Indique le type de flux utilisé. Pour actualiser un jeton d’accès, cette valeur doit être définie sur `refresh_token`.

  Valeurs autorisées : `refresh_token`
</ParamField>

<ParamField body="client_id" type="string" required>
  L’ID client de votre application, qui identifie l’application qui effectue la requête.
</ParamField>

<ParamField body="client_secret" type="string">
  Le Secret client de votre application. Obligatoire lorsque la méthode d’authentification du point de terminaison de jeton est définie sur `Post` ou `Basic`.
</ParamField>

<ParamField body="refresh_token" type="string" required>
  Le jeton d’actualisation précédemment émis à l’application. Ce jeton est utilisé pour obtenir un nouveau jeton d’accès.
</ParamField>

<ParamField body="scope" type="string">
  Une liste de scopes demandés, séparés par des espaces. S’il est omis, les scopes d’origine seront utilisés; sinon, vous pouvez demander un ensemble réduit de scopes. Notez que cette valeur doit être encodée dans l’URL.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  <ResponseField name="access_token" type="string">
    Le nouveau jeton d’accès.
  </ResponseField>

  <ResponseField name="refresh_token" type="string">
    Le jeton d’actualisation.
  </ResponseField>

  <ResponseField name="id_token" type="string">
    Le jeton d’identité, le cas échéant.
  </ResponseField>

  <ResponseField name="token_type" type="string">
    Le type de jeton. Généralement `Bearer`.
  </ResponseField>

  <ResponseField name="expires_in" type="integer">
    La durée de vie du jeton d’accès, en secondes.
  </ResponseField>

  <ResponseField name="scope" type="string">
    Les scopes du jeton d’accès.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                                                                                               |
| ------ | ------------------------------------------------------------------------------------------------------------------------- |
| 200    | Jeton d’accès renouvelé avec succès.                                                                                      |
| 400    | Requête incorrecte. La requête ne contient pas les paramètres requis ou est mal formée.                                   |
| 401    | Non autorisé. Les informations d’identification de l’application sont invalides ou le jeton d’actualisation est invalide. |
| 403    | Accès interdit. Le jeton d’actualisation est invalide ou a expiré.                                                        |
