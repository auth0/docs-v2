`POST /users/{user_id}/impersonate`

Utilisez ce point de terminaison pour obtenir une URL d’usurpation d’identité afin de vous connecter sous l’identité d’un autre utilisateur. Utile pour le dépannage.

<div id="remarks">
  ### Remarques
</div>

* Ce point de terminaison ne peut être utilisé qu’avec les identifiants **Global Client**.
* Pour distinguer les ouvertures de session réelles des ouvertures de session par usurpation d’identité, le profil de l’utilisateur usurpé contiendra les propriétés supplémentaires `impersonated` et `impersonator`. Par exemple : `"impersonated": true, "impersonator": {"user_id": "auth0|...", "email": "admin@example.com"}`.
* Pour une application Web classique, vous devez définir `additionalParameters` : attribuez à `response_type` la valeur `code`, à `callback_url` l’URL de rappel vers laquelle Auth0 redirigera avec le code d’autorisation, et à `scope` les revendications du JWT que vous souhaitez inclure dans le JWT.
