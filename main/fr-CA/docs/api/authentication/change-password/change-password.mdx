<div id="endpoint">
  ## Point de terminaison
</div>

`POST /dbconnections/change_password`

Envoyez un courriel pour changer le mot de passe à l’adresse de courriel fournie par l’utilisateur, pour la `connection` indiquée.

Vous pouvez également fournir un ID d’organisation pour prendre en charge les variables propres à l’organisation dans les [modèles de courriel personnalisés](https://auth0.com/docs/customize/email/email-templates) et inclure les paramètres `organization_id` et `organization_name` dans l’URL **Redirect To**.

**Remarque :** Ce point de terminaison fonctionne uniquement avec les connexions de base de données.

<div id="remarks">
  ### Remarques
</div>

* Lorsque l’utilisateur clique sur le lien pour changer son mot de passe, il est redirigé vers une page où on lui demande d’entrer un nouveau mot de passe.
* Ce point de terminaison renvoie trois en-têtes de réponse HTTP qui fournissent des renseignements pertinents sur ses limites de débit :
  * `X-RateLimit-Limit` : Nombre de requêtes autorisées par minute.
  * `X-RateLimit-Remaining` : Nombre de requêtes disponibles. Chaque nouvelle requête réduit ce nombre de 1. Pour chaque minute écoulée, une requête est rajoutée; ce nombre augmente donc de 1 chaque fois.
  * `X-RateLimit-Reset` : Temps restant avant la réinitialisation de la limite de débit (`X-RateLimit-Limit`). La valeur est exprimée en [secondes de l’époque Unix (UTC)](https://en.wikipedia.org/wiki/Unix_time).

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="client_id" type="string" required>
  Le `client_id` de votre application.
</ParamField>

<ParamField body="email" type="string" required>
  L’adresse de courriel de l’utilisateur.
</ParamField>

<ParamField body="connection" type="string" required>
  Le nom de la connexion de base de données configurée pour votre application.
</ParamField>

<ParamField body="organization" type="string">
  L’`organization_id` de l’organisation associée à l’utilisateur.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema type="string">
  Le corps de la réponse 200 est une chaîne de texte brut.

  <br />

  Exemple : `"We've just sent you an email to reset your password."`
</ResponseSchema>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                            |
| ------ | ------------------------------------------------------ |
| 200    | Un courriel pour changer le mot de passe a été envoyé. |
| 400    | Requête incorrecte, paramètres manquants ou invalides. |
| 404    | Utilisateur introuvable.                               |
