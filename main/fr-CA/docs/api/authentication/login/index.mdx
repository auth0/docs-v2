`GET /authorize`

<div id="social">
  ## Social
</div>

Vous pouvez connecter votre service Auth0 à un fournisseur d’identité social et permettre à vos utilisateurs de se connecter à votre application avec Facebook, Google, Apple ou d’autres fournisseurs pris en charge. Pour en savoir plus sur les fournisseurs pris en charge, consultez [Marketplace](https://marketplace.auth0.com/features/social-connections).

Pour authentifier des utilisateurs avec un fournisseur social, effectuez une requête `GET` au point de terminaison `/authorize`. Elle renverra une redirection `302` vers le fournisseur social indiqué dans le paramètre `connection`.

<Note>
  Les connexions sociales ne prennent en charge que l’authentification dans le navigateur (passive), car la plupart des fournisseurs sociaux n’autorisent pas la saisie d’un nom d’utilisateur et d’un mot de passe dans des applications qui ne leur appartiennent pas. L’utilisateur sera donc redirigé vers la page de connexion du fournisseur.
</Note>

<div id="remarks">
  ### Remarques
</div>

* Si `response_type=token`, une fois l’utilisateur authentifié auprès du fournisseur, il est redirigé vers l’`URL de rappel` de votre application, avec le Jeton d’accès et l’ID Token transmis dans le `location.hash` de l’URL. Ce comportement est utilisé par les applications monopages ainsi que par les SDK mobiles natifs.

* Le script auth0.js fourni en exemple utilise la version 8 de la bibliothèque. Si vous utilisez auth0.js version 7, veuillez consulter ce [guide de référence](https://auth0.com/docs/libraries/auth0js).

<div id="parameters">
  ## Paramètres
</div>

<ParamField query="response_type" type="string" required>
  Indique le type de jeton (`code` ou `token`)

  Valeurs autorisées : `code`, `token`
</ParamField>

<ParamField query="client_id" type="string" required>
  Le client&#95;id de votre application
</ParamField>

<ParamField query="connection" type="string">
  Le nom d’un fournisseur d’identité social configuré pour votre application
</ParamField>

<ParamField query="redirect_uri" type="string" required>
  L’URL vers laquelle Auth0 redirigera après l’autorisation
</ParamField>

<ParamField query="state" type="string">
  Valeur opaque utilisée pour prévenir les attaques CSRF
</ParamField>

<div id="response">
  ## Réponse
</div>

| Statut | Description                                       |
| ------ | ------------------------------------------------- |
| 302    | Redirection vers le fournisseur d’identité social |
