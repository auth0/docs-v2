---
title: "Révoquer un jeton d’actualisation"
description: "Utilisez ce point de terminaison pour invalider un jeton d’actualisation s’il a été compromis."
---

`POST /oauth/revoke`

Utilisez ce point de terminaison pour invalider un jeton d’actualisation s’il a été compromis.

Le comportement de ce point de terminaison dépend de l’état de l’option [Refresh Token Revocation Deletes Grant](https://auth0.com/docs/tokens/refresh-tokens/revoke-refresh-tokens#refresh-tokens-and-grants).
Si cette option est activée, chaque demande de révocation invalide non seulement le jeton en question, mais aussi tous les autres jetons associés au même octroi d’autorisation. Cela signifie que **tous les jetons d’actualisation émis pour le même utilisateur, la même application et la même audience seront révoqués**.
Si cette option est désactivée, seul le jeton d’actualisation est révoqué, tandis que l’octroi reste intact.

<div id="remarks">
  ## Remarques
</div>

* Pour les applications non confidentielles qui ne peuvent pas protéger adéquatement le Secret client (par exemple, les applications natives), le point de terminaison permet de ne pas transmettre de Secret client, mais l’application elle-même doit avoir la propriété `tokenEndpointAuthMethod` définie sur `none`. Vous pouvez le faire soit dans l’interface utilisateur ([Auth0 Dashboard &gt; Applications &gt; paramètres de l’application](https://manage.auth0.com/dashboard)), soit à l’aide de la [Management API](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id).

<div id="parameters">
  ## Paramètres
</div>

<ParamField body="client_id" type="string" required>
  Le `client_id` de votre application.
</ParamField>

<ParamField body="client_assertion" type="string">
  Un JWT contenant une assertion signée avec les identifiants de votre application. Obligatoire si l’application utilise Private Key JWT comme méthode d’authentification.
</ParamField>

<ParamField body="client_assertion_type" type="string">
  La valeur est `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`. Obligatoire si l’application utilise Private Key JWT comme méthode d’authentification.
</ParamField>

<ParamField body="client_secret" type="string">
  Le `client_secret` de votre application. Obligatoire si l’application utilise Client Secret Basic ou Client Secret Post comme méthode d’authentification. Requis pour les Applications Web régulières **uniquement**.
</ParamField>

<ParamField body="token" type="string" required>
  Le Jeton d’actualisation que vous voulez révoquer.
</ParamField>

<div id="response">
  ## Réponse
</div>

| Statut  | Description             |
| ------- | ----------------------- |
| 200     | (corps de réponse vide) |
| 400     | Requête incorrecte      |
| 401     | Non autorisé            |
| default | Erreur inattendue       |
