<div id="endpoint">
  ## Point de terminaison
</div>

`POST /oidc/register`

Avec un nom et l’URL de redirection requise, vous pouvez enregistrer dynamiquement une application auprès d’Auth0. Aucun jeton n’est requis pour cette requête.

<div id="body-parameters">
  ## Paramètres du corps
</div>

<ParamField body="client_name" type="string">
  Le nom de l’application dynamique à créer.
</ParamField>

<ParamField body="grant_types" type="array">
  Liste des types d’octroi que l’application peut utiliser. Si aucune valeur n’est fournie, le type d’octroi `authorization_code` sera appliqué par défaut aux <a href="https://auth0.com/docs/get-started/applications/third-party-applications">applications tierces</a> avec des contrôles de sécurité renforcés.
</ParamField>

<ParamField body="redirect_uris" type="array" required>
  Tableau d’URL qu’Auth0 considérera comme valides pour la redirection à la fin d’un flux d’authentification.
</ParamField>

<ParamField body="token_endpoint_auth_method" type="string">
  Méthode utilisée pour l’authentification au point de terminaison de jeton.
</ParamField>

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  <ResponseField name="client_name" type="string">
    Le nom de la nouvelle application enregistrée.
  </ResponseField>

  <ResponseField name="client_id" type="string">
    L’identifiant unique de l’application.
  </ResponseField>

  <ResponseField name="client_secret" type="string">
    Le Secret client.
  </ResponseField>

  <ResponseField name="client_secret_expires_at" type="integer">
    La date d’expiration du Secret client.
  </ResponseField>
</ResponseSchema>

<div id="response-messages">
  ## Messages de réponse
</div>

| Statut | Description                                             |
| ------ | ------------------------------------------------------- |
| 200    | Réponse réussie contenant les détails de l’application. |
