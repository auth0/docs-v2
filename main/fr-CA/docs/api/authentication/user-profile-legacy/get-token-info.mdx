<div id="endpoint">
  ## Point de terminaison
</div>

`POST /tokeninfo`

<Note>
  Ce point de terminaison fait partie de l’ancien pipeline d’authentification et sera désactivé pour les utilisateurs de notre pipeline le plus récent, conforme à OIDC. Nous vous encourageons plutôt à utiliser le [point de terminaison /userinfo](#get-user-info). Pour en savoir plus sur le pipeline d’authentification le plus récent, consultez [Introducing OIDC Conformant Authentication](/fr-CA/api-auth/intro).
</Note>

Ce point de terminaison valide un JSON Web Token (JWT) (signature et expiration) et renvoie les informations de l’utilisateur associées à la propriété `sub` (id utilisateur) du jeton.

<div id="remarks">
  ### Remarques
</div>

* Ce point de terminaison renvoie trois en-têtes de réponse HTTP qui fournissent des renseignements pertinents sur ses limites de débit :
  * `X-RateLimit-Limit` : Nombre de requêtes autorisées par minute.
  * `X-RateLimit-Remaining` : Nombre de requêtes encore disponibles. Chaque nouvelle requête réduit ce nombre de 1. À chaque minute écoulée, des requêtes sont rétablies, de sorte que ce nombre augmente de 1 chaque fois.
  * `X-RateLimit-Reset` : Temps restant avant la réinitialisation de la limite de débit (`X-RateLimit-Limit`). La valeur est exprimée en [secondes d’époque UTC](https://en.wikipedia.org/wiki/Unix_time).

<div id="response-schema">
  ## Schéma de réponse
</div>

<ResponseSchema>
  <ResponseField name="email_verified" type="boolean">
    Indique si l’adresse de courriel de l’utilisateur a été vérifiée.
  </ResponseField>

  <ResponseField name="email" type="string">
    L’adresse de courriel de l’utilisateur.
  </ResponseField>

  <ResponseField name="clientID" type="string">
    L’ID client associé au jeton.
  </ResponseField>

  <ResponseField name="updated_at" type="string">
    Date et heure de la dernière mise à jour des informations de l’utilisateur.
  </ResponseField>

  <ResponseField name="name" type="string">
    Le nom complet de l’utilisateur.
  </ResponseField>

  <ResponseField name="picture" type="string">
    URL de l’image de profil de l’utilisateur.
  </ResponseField>

  <ResponseField name="user_id" type="string">
    L’identifiant unique de l’utilisateur.
  </ResponseField>

  <ResponseField name="nickname" type="string">
    Le surnom de l’utilisateur.
  </ResponseField>

  <ResponseField name="provider" type="string">
    Le fournisseur d’identité de l’utilisateur.
  </ResponseField>

  <ResponseField name="connection" type="string">
    La connexion utilisée pour authentifier l’utilisateur.
  </ResponseField>

  <ResponseField name="isSocial" type="boolean">
    Indique si l’utilisateur s’est authentifié au moyen d’un fournisseur social.
  </ResponseField>

  <ResponseField name="created_at" type="string">
    Date et heure de création de l’utilisateur.
  </ResponseField>

  <ResponseField name="global_client_id" type="string">
    L’ID client global.
  </ResponseField>
</ResponseSchema>
