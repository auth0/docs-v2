`POST /oauth/ro`

<Note>
  Ce point de terminaison fait partie de l’ancien pipeline d’authentification et a été remplacé par le [Password Grant](#resource-owner-password). Pour en savoir plus sur le plus récent pipeline d’authentification, consultez [Présentation de l’authentification conforme à OIDC](/fr-CA/api-auth/intro).
</Note>

Utilisez ce point de terminaison pour l’authentification active basée sur l’API. À partir des identifiants de l’utilisateur et de la `connection` spécifiée, il effectuera l’authentification auprès du fournisseur et renverra une réponse JSON contenant le Jeton d’accès et l’ID Token.

<div id="remarks">
  ### Remarques
</div>

* Ce point de terminaison fonctionne uniquement pour les connexions de base de données, les connexions Passwordless, Active Directory/LDAP, Windows Azure AD et ADFS.

* La principale différence entre l’authentification passive et l’authentification active est que la première se déroule dans le navigateur, au moyen de la [page de connexion Auth0](http://manage.auth0.com/login), tandis que la seconde peut être invoquée de n’importe où (dans un script, de serveur à serveur, etc.).

* L’exemple de script auth0.js utilise la version 8 de la bibliothèque. Si vous utilisez la version 7 d’auth0.js, veuillez consulter ce [guide de référence](https://auth0.com/docs/libraries/auth0js/v7).

<div id="parameters">
  ## Paramètres
</div>

<ParamField body="client_id" type="string" required>
  Le `client_id` de votre application.
</ParamField>

<ParamField body="username" type="string" required>
  Le nom d’utilisateur ou le courriel de l’utilisateur pour se connecter.
</ParamField>

<ParamField body="password" type="string" required>
  Le mot de passe de l’utilisateur pour se connecter.
</ParamField>

<ParamField body="connection" type="string" required>
  Le nom de la connexion à utiliser pour l’authentification.
</ParamField>

<ParamField body="scope" type="string">
  Définissez cette valeur sur `openid` pour récupérer également un ID Token.
</ParamField>

<ParamField body="grant_type" type="string" required>
  Définissez cette valeur sur `password` ou `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>

<ParamField body="device" type="string">
  Obligatoire lorsque `grant_type` est `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>

<ParamField body="id_token" type="string">
  Obligatoire lorsque `grant_type` est `urn:ietf:params:oauth:grant-type:jwt-bearer`.
</ParamField>
