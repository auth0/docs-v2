<Card title="Avant de commencer">
  Pour migrer vers l’expérience téléphonique unifiée, vous devez :

  * Avoir activé et configuré dans votre locataire les notifications MFA par SMS, par appel vocal ou par SMS+voix, ou
  * Avoir activé et configuré Passwordless SMS dans votre locataire

  Si vous n’avez configuré ni MFA ni Passwordless, consultez [Configurer l’expérience téléphonique unifiée d’Auth0](/fr-CA/docs/customize/phone-messages/unified-phone/configure-unified-phone) pour commencer à partir de zéro.

  Si vous avez à la fois des intégrations MFA et Passwordless avec des fournisseurs différents, vous devez utiliser un [fournisseur téléphonique personnalisé](/fr-CA/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider).

  Si vous utilisez Phone as ID, vous utilisez automatiquement l’expérience téléphonique unifiée d’Auth0 et aucune autre action n’est nécessaire.
</Card>

L’expérience téléphonique unifiée centralise toute la configuration de vos fournisseurs téléphoniques en un seul endroit dans votre locataire. Vous n’avez pas besoin de configurer plusieurs fois un fournisseur téléphonique pour différents facteurs d’authentification par téléphone. Pour en savoir plus sur les facteurs téléphoniques pour la <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification qui utilise un facteur supplémentaire en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=MFA">MFA</Tooltip>, consultez [Notifications SMS et vocales pour la MFA](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa).

Si vous avez déjà configuré un fournisseur téléphonique pour les [notifications SMS et vocales pour l’authentification multifacteur (MFA)](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa), vous pouvez migrer votre configuration vers l’expérience téléphonique unifiée et l’appliquer au niveau du locataire.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 n’indique pas à l’utilisateur final si un message téléphonique a été envoyé avec succès avec le fournisseur configuré au niveau du locataire. Pour résoudre les problèmes, consultez les [journaux du locataire](/fr-CA/docs/deploy-monitor/logs).
</Callout>

<div id="update-your-phone-provider">
  ## Mettre à jour votre fournisseur téléphonique
</div>

Pour utiliser l’expérience téléphonique unifiée, vous devez :

* Vérifier les paramètres actuels
* Configurer votre fournisseur :

  * Configurer Twilio
  * Configurer un fournisseur téléphonique personnalisé
* Mettre à jour vos facteurs téléphoniques et vos modes d’envoi
* Mettre à jour les points de terminaison de <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>
* Mettre à jour les déclencheurs d’Actions Auth0

<div id="verify-current-settings">
  ### Vérifier les paramètres actuels
</div>

1. Pour vérifier vos paramètres MFA actuels, accédez à [**Auth0 Dashboard &gt; Security &gt; Multi-factor Auth**](https://manage.auth0.com/#/security/mfa).
2. Sélectionnez **Phone Message**.
3. Les nouveaux locataires Auth0 peuvent utiliser l’expérience expérience téléphonique unifiée, et une bannière vous invite à configurer un fournisseur téléphonique au niveau du locataire :

   <Frame>![](/docs/images/cdy7uua7fh8z/1mZryDWIpdkpGmhxn4Gr8K/4bdad7fbcd81dd0bf4855d94b7205e8b/mfa-phone-message-upe.png)</Frame>

* Les locataires existants d’avant mars 2025 peuvent configurer les notifications par SMS et par appel vocal pour l’authentification multifacteur (MFA), en plus de l’expérience expérience téléphonique unifiée.

  <Frame>![](/docs/images/cdy7uua7fh8z/1mZryDWIpdkpGmhxn4Gr8K/4bdad7fbcd81dd0bf4855d94b7205e8b/mfa-phone-message-no-upe.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 recommande d’utiliser l’expérience expérience téléphonique unifiée. Si vous devez utiliser l’expérience [SMS and Voice notifications for Multi-Factor Authentication (MFA)](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa) pour configurer votre fournisseur téléphonique et que l’option **Use  Tenant-Level Messaging Provider** est absente, communiquez avec le [soutien Auth0](/fr-CA/docs/troubleshoot/customer-support).
</Callout>

4. Pour vérifier les paramètres de votre locataire, accédez à [**Dashboard &gt; Tenant Settings**](https://manage.auth0.com/#/tenant/advanced).
5. Sélectionnez **Advanced**.
6. Repérez **expérience téléphonique unifiée**.
   <Frame>![](/docs/images/cdy7uua7fh8z/1mZryDWIpdkpGmhxn4Gr8K/4bdad7fbcd81dd0bf4855d94b7205e8b/tenant-settings-upe.png)</Frame>
7. Activez l’option, au besoin.

<div id="configure-twilio-as-a-tenant-level-provider">
  ### Configurer Twilio comme fournisseur pour le locataire
</div>

Si vous utilisez Twilio comme fournisseur de téléphone pour l’authentification multifacteur, vous devez ajouter le SID Twilio et l’AuthToken à l’expérience téléphonique unifiée. Vous pouvez copier le SID Twilio à partir de votre configuration existante, mais vous devrez obtenir la valeur de l’AuthToken auprès de Twilio.

1. Accédez à [**Image de marque &gt; Fournisseur téléphonique**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Sélectionnez et configurez votre fournisseur :

   <Frame>![Image de marque > Fournisseur téléphonique > Twilio](/docs/images/cdy7uua7fh8z/6TESTIdmhUL0LEitACadIK/fcdc34e18a6b66b47370f6d754c085b5/2025-02-21_12-49-53.png)</Frame>

   1. Choisissez Twilio comme fournisseur de téléphone.
   2. Choisissez vos modes d’envoi.
   3. Saisissez les paramètres Twilio.

      <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
        Les facteurs MFA souhaités doivent faire partie des modes d’envoi que vous configurez au niveau du locataire.
      </Callout>
3. **Enregistrez** vos modifications.
4. Sélectionnez **Envoyer un message de test** pour tester le fournisseur au niveau du locataire avant de l’utiliser en production.
5. Activez un fournisseur au niveau du locataire.

   1. Pour MFA, accédez à [**Sécurité &gt; Authentification multifacteur &gt; Message téléphonique**.](https://manage.auth0.com/#/security/mfa)
   2. Sélectionnez **Utiliser le fournisseur de messagerie au niveau du locataire**.

      <Warning>
        Si aucune option n’est affichée dans votre locataire, vous utilisez déjà l’expérience téléphonique unifiée.
      </Warning>
6. **Enregistrez** vos modifications.

<div id="configure-a-custom-provider-with-auth0-actions">
  ### Configurer un fournisseur personnalisé avec Auth0 Actions
</div>

L’expérience téléphonique unifiée utilise le déclencheur Actions `custom-phone-provider` pour les notifications téléphoniques plutôt que le déclencheur Actions `send-phone-message` pour les notifications téléphoniques personnalisées.

Vous ne pouvez pas migrer directement votre déclencheur Actions `send-phone-message` vers le déclencheur Actions `custom-phone-provider` pour les raisons suivantes :

* Il n’existe pas de correspondance un à un entre les types de messages et les types de notifications.
* La migration des secrets et des dépendances associés au déclencheur Actions `send-phone-message` pourrait entraîner des problèmes de sécurité ou perturber le fonctionnement du locataire.
* Les Actions peuvent prendre en charge plusieurs déclencheurs et être difficiles à migrer. Par exemple, les Actions peuvent prendre en charge à la fois les déclencheurs `send-phone-message` et `post-login`.
* Les déclencheurs Actions `send-phone-message` et `custom-phone-provider` prennent en charge différents types d’événements :

  * `send-phone-message` prend en charge les types de messages SMS et vocaux. Pour en savoir plus, consultez [Action triggers: send-phone-message Event Object](/fr-CA/docs/customize/actions/explore-triggers/mfa-notifications-trigger/send-phone-message-event-object).
  * `custom-phone-provider` prend en charge les types de notifications `otp_verify`, `otp_enroll`, `blocked_account`, `change_password` et `password_breach`. Pour en savoir plus, consultez [Actions triggers: custom-phone-provider Event Object](/fr-CA/docs/customize/phone-messages/configure-phone-messaging-providers/configure-a-custom-phone-provider/actions-triggers-custom-phone-provider-event-object).
* Vous ne pouvez définir qu’une seule Action par fournisseur téléphonique personnalisé. Si vous avez déjà défini une Action pour un fournisseur téléphonique personnalisé, vous **devez** la désactiver avant de l’associer à une nouvelle Action.

Par conséquent, vous devez convertir votre `send-phone-message` existant en un nouveau `custom-phone-provider` à l’aide des nouvelles variables contextuelles :

Passez en revue l’exemple de fournisseur téléphonique personnalisé :

```javascript lines expandable
const { fetch } = require('undici');
/**
* Gestionnaire à exécuter lors de l'envoi d'une notification téléphonique
* @param {Event} event - Détails sur l'utilisateur et le contexte dans lequel il se connecte.
* @param {CustomPhoneProviderAPI} api - Méthodes et utilitaires pour modifier le comportement de l'envoi d'une notification téléphonique.
*/
exports.onExecuteCustomPhoneProvider = async (event, api) => {

  if (event.notification.message_type.startsWith('otp')) {
    const body = {
      from: event.notification.from,
      action: event.notification.message_type === 'otp_verify' ? 'second-factor-authenticator' : 'enrollment',
      language: event.notification.locale,
      recipient: event.notification.recipient,
      message_type: event.notification.delivery_method,
      text: event.notification.as_text,
      code: event.notification.code,
    };

    const response = await fetch(event.secrets['MY_ENDPOINT'], {
      method: 'POST',
      headers: {
        authorization: `Bearer ${event.secrets['MY_SECRET']}`,
      },
      body: JSON.stringify(body),
    });

    console.log(response.status); // 200
    console.log(await response.text()); // corps de la réponse de la requête !
  }

  return;
};
```

<div id="configure-mfa-custom-phone-provider">
  #### Configurer `custom-phone-provider` pour MFA
</div>

Pour poursuivre et configurer votre déclencheur MFA `custom-phone-provider` :

1. Accédez à [**Branding &gt; Phone Provider**](https://manage.auth0.com/#/phone/templates/phone/provider).
2. Sélectionnez votre fournisseur :

   1. Choisissez **Custom**.
   2. Sous **Provider Configuration**, ajoutez le code traduit de votre Action `send-phone-message` dans l’éditeur de code intégré.
3. **Enregistrez** vos modifications.
4. Sélectionnez **Send Test Message** pour tester le fournisseur au niveau du locataire avant de l’utiliser en production.
5. Retournez à [**Security &gt; Multi-factor Auth &gt; Phone Message**](https://manage.auth0.com/#/security/mfa) pour activer l’option **Use Tenant-level Messaging Provider**.
6. **Enregistrez** vos modifications.
