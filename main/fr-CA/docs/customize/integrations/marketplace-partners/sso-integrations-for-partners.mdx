<Tooltip tip="Single Sign-On (SSO) : service qui, après qu’un utilisateur s’est connecté à une application, connecte automatiquement cet utilisateur à d’autres applications." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=SSO">SSO</Tooltip> (authentification unique) permet d’utiliser une identité Auth0 pour se connecter, au moyen de <Tooltip tip="Single Sign-On (SSO) : service qui, après qu’un utilisateur s’est connecté à une application, connecte automatiquement cet utilisateur à d’autres applications." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=SAML">SAML</Tooltip>, à un service externe. Cela permet aux clients qui ont des applications internes d’utiliser le SSO pour accéder à des fournisseurs tiers à l’aide de l’identité Auth0.

Les [intégrations SSO offertes dans Marketplace](https://marketplace.auth0.com/features/sso-integrations) définissent à la fois les valeurs de configuration SAML standard requises pour connecter Auth0 à des fournisseurs externes et les champs nécessaires pour des valeurs supplémentaires propres au client. Dans ces intégrations, Auth0 agit comme le <Tooltip tip="Identity Provider (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> dans un scénario à l’initiative du fournisseur de services (SP). Pour savoir comment configurer Auth0 pour ces intégrations, consultez [Configurer Auth0 comme fournisseur d’identité SAML](/fr-CA/docs/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider).

Ce que nos partenaires doivent fournir, c’est la configuration SAML par défaut nécessaire au fonctionnement de cette transaction. Elle se compose de valeurs ou de champs demandés par le client pour une partie ou la totalité des attributs répertoriés dans [Personnaliser les assertions SAML : attributs d’assertion SAML](/fr-CA/docs/authenticate/protocols/saml/saml-configuration/customize-saml-assertions).

Pour créer et tester une intégration, [créez un locataire](https://auth0.com/signup) dans Auth0 et suivez la documentation ci-dessus pour créer une intégration SSO fonctionnelle avec votre service.

De plus, pour vous aider à créer votre intégration, vous pouvez vous appuyer sur notre [modèle d’intégration SSO](https://github.com/auth0-marketplace/TEMPLATE-sso).

<div id="submit-your-sso-integration">
  ## Soumettez votre intégration SSO
</div>

Si vous êtes prêt à devenir partenaire et à soumettre votre intégration, veuillez créer un compte partenaire dans le [portail partenaire](https://partners.marketplace.auth0.com/) et suivre les étapes suivantes :

1. Remplissez le profil du partenaire, qui comprend une brève description de votre entreprise, le contenu juridique et marketing, le logo de l’entreprise, les conditions d’utilisation, la politique de confidentialité, les coordonnées et l’adresse de courriel du soutien.
2. Soumettez votre intégration, qui comprend le code d’intégration, le guide d’installation et des renseignements comme le nom et la description. Vous pouvez aussi ajouter 3 propositions de valeur à votre intégration.

**Nous aurons besoin de la configuration et de la documentation suivantes :**

* Le JSON utilisé pour configurer l’application avec le module complémentaire SAML fonctionnel
* Une liste des champs à afficher au client qui configure l’intégration (comme **URL de rappel**, **<Tooltip tip="Audience : identifiant unique de l’audience visée par un jeton émis. Nommée aud dans un jeton, sa valeur contient l’ID d’une application (ID client) pour un ID Token, ou d’une API (identifiant d’API) pour un jeton d’accès." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Audience">Audience</Tooltip>**, et **certificat de signature**)
* Un guide d’installation rédigé en Markdown.
