<div id="eventsecurity_context">
  ## `event.security_context`
</div>

<ResponseField name="event.security_context" type="object" post={["optional"]}>
  Objet contenant des signatures d’empreinte. Il est disponible uniquement si l’application utilise Cloudflare. L’empreinte JA3/JA4 peut être nulle ou vide dans certains cas. Le cas le plus courant concerne les requêtes HTTP, car JA3 et JA4 sont calculées au niveau de TLS. Elle peut aussi être vide si le Worker envoie des requêtes dans la même zone ou vers une zone qui n’est pas proxifiée (ou vers un tiers).

  <Expandable title="propriétés de security_context" defaultOpen>
    <ResponseField name="ja3" type="string" post={["optional"]}>
      Signature d’empreinte JA3. Elle est disponible uniquement si l’application utilise une connexion TLS.
    </ResponseField>

    <ResponseField name="ja4" type="string" post={["optional"]}>
      Signature d’empreinte JA4. Elle est disponible uniquement si l’application utilise une connexion TLS.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="eventtransaction">
  ## `event.transaction`
</div>

<ResponseField name="event.transaction" type="dictionary" post={["optional"]}>
  Détails sur la transaction en cours.

  <Expandable title="propriétés de la transaction" defaultOpen>
    <ResponseField name="acr_values" type="array of strings">
      Toutes les valeurs acr&#95;values fournies dans la demande d’authentification d’origine.
    </ResponseField>

    <ResponseField name="locale" type="string">
      Le paramètre régional à utiliser pour cette transaction, déterminé en comparant les langues demandées par le navigateur aux paramètres linguistiques du locataire.
    </ResponseField>

    <ResponseField name="login_hint" type="string" post={["optional"]}>
      Indication fournie au serveur d’autorisation au sujet de l’identifiant de connexion que l’utilisateur final pourrait utiliser pour se connecter (au besoin).
    </ResponseField>

    <ResponseField name="prompt" type="array of strings" post={["optional"]}>
      Liste d’instructions indiquant si l’utilisateur peut être invité à se réauthentifier et à donner son consentement.
    </ResponseField>

    <ResponseField name="protocol" type="string" post={["optional"]}>
      Valeurs possibles :

      * `oidc-basic-profile`
      * `oidc-ciba`
      * `oidc-ciba-web-link`
      * `oidc-implicit-profile`
      * `oauth2-device-code`
      * `oauth2-resource-owner`
      * `oauth2-resource-owner-jwt-bearer`
      * `oauth2-password`
      * `oauth2-webauthn`
      * `oauth2-access-token`
      * `oauth2-refresh-token`
      * `oauth2-token-exchange`
      * `oidc-hybrid-profile`
      * `samlp`
      * `wsfed`
      * `wstrust-usernamemixed`
    </ResponseField>

    <ResponseField name="redirect_uri" type="string" post={["optional"]}>
      URL vers laquelle Auth0 redirigera le navigateur une fois la transaction terminée.
    </ResponseField>

    <ResponseField name="requested_scopes" type="array of strings">
      Les scopes demandés, s’il y a lieu, au démarrage de ce flux d’authentification.
    </ResponseField>

    <ResponseField name="response_mode" type="string" post={["optional"]}>
      Indique au serveur d’autorisation le mécanisme à utiliser pour renvoyer les paramètres à partir du point de terminaison d’autorisation.

      Valeurs autorisées : `query`, `fragment`, `form_post`, `web_message`
    </ResponseField>

    <ResponseField name="response_type" type="array of strings" post={["optional"]}>
      Indique le type de justificatif qu’Auth0 retournera.
    </ResponseField>

    <ResponseField name="state" type="string" post={["optional"]}>
      Chaîne alphanumérique opaque et arbitraire que votre application ajoute à la demande initiale et qu’Auth0 inclut lors de la redirection vers votre application.
    </ResponseField>

    <ResponseField name="ui_locales" type="array of strings">
      Les ui&#95;locales fournis dans la demande d’authentification d’origine.
    </ResponseField>

    <ResponseField name="correlation_id" type="string" post={["optional"]}>
      Un ID de corrélation peut être fourni dans la demande d’authentification initiale lorsque l’application redirige vers Universal Login. Vous pouvez utiliser cette valeur pour corréler les journaux et les requêtes de votre code Action avec le parcours de l’utilisateur.
    </ResponseField>
  </Expandable>
</ResponseField>
