sidebarTitle: Aperçu
title: Découvrir les déclencheurs
---

Les déclencheurs sont des points du processus Auth0 où vous pouvez ajouter des Actions afin d’apporter les personnalisations requises. Plusieurs Actions peuvent être placées dans un déclencheur et s’exécutent en séquence. À l’exception de Custom Token Exchange, les déclencheurs peuvent être synchrones (bloquants) ou asynchrones (non bloquants). Chaque déclencheur comporte des objets et des API spécifiques. Ils peuvent varier selon votre connexion (par exemple, une connexion <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=passwordless">Passwordless</Tooltip>).

Pour consulter les objets de référence et les API associés aux Actions propres à un déclencheur donné, sélectionnez le déclencheur ci-dessous, en vous référant à la définition d’Auth0 d’une [connexion Passwordless](/fr-CA/docs/authenticate/passwordless).

<div id="signup-and-login">
  #### Inscription et connexion
</div>

Lorsqu’un utilisateur est ajouté à une connexion de base de données ou Passwordless (inscription), ou au cours de la transaction d’authentification d’un utilisateur (connexion).

| Déclencheur              | Description                                                                               |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| `pre-user-registration`  | Se déclenche avant qu’un utilisateur soit créé.                                           |
| `post-user-registration` | Se déclenche de manière asynchrone après qu’un utilisateur a été créé.                    |
| `post-login`             | Se déclenche après l’authentification d’un utilisateur, mais avant qu’un jeton soit émis. |

<div id="mfa-notifications">
  #### Notifications d’authentification multifacteur
</div>

Lors de l’utilisation des SMS comme facteur pour <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Multi-Factor+Authentication">l’authentification multifacteur</Tooltip> (MFA) ou pour configurer un fournisseur personnalisé.

| Déclencheur          | Description                                                                                                                                   |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `send-phone-message` | Se déclenche lorsqu’un fournisseur personnalisé est utilisé pour envoyer les messages pendant les processus d’inscription et de vérification. |

<div id="password-reset">
  #### Réinitialisation du mot de passe
</div>

Lorsqu’une demande de réinitialisation du mot de passe est amorcée.

| Déclencheur                     | Description                                                                                           |
| ------------------------------- | ----------------------------------------------------------------------------------------------------- |
| `post-change-password`          | Se déclenche après la modification d’un mot de passe.                                                 |
| `password-reset-post-challenge` | Se déclenche une fois la première vérification terminée et avant la réinitialisation du mot de passe. |

<div id="machine-to-machine">
  #### Machine à machine
</div>

Lorsqu’une application, plutôt qu’un utilisateur, est authentifiée et autorisée à utiliser vos API.

| Déclencheur            | Description                                      |
| ---------------------- | ------------------------------------------------ |
| `credentials-exchange` | Se déclenche avant le renvoi d’un jeton d’accès. |

<div id="custom-token-exchange">
  ### Custom Token Exchange
</div>

Lorsqu’une demande de Custom Token Exchange est reçue pour authentifier un utilisateur.

| Déclencheur             | Description                                                                          |
| ----------------------- | ------------------------------------------------------------------------------------ |
| `custom-token-exchange` | Se déclenche à la première étape de la transaction, avant le déclencheur post-login. |
