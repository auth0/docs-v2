---
description: Limites des activités propres à la migration lors du passage de Rules et Hooks à Actions
title: Limites de la migration vers Actions
---

Pour tirer parti des fonctionnalités offertes par Actions d’Auth0, vous devez migrer votre code actuel depuis Rules, Hooks ou des versions antérieures d’Actions. Cependant, vous devez tenir compte des limites actuelles lors de la migration depuis ces anciennes options d’extensibilité vers Actions.

<div id="general-differences">
  ## Différences générales
</div>

* Actions ne peut renvoyer que des codes d’erreur `access denied` et ne prend pas en charge le renvoi d’un code d’erreur `unauthorized`.
* Les revendications standard d’un `accessToken` ne peuvent pas être supprimées à l’aide d’une Action.

<div id="rule-specific-differences">
  ## Différences propres aux Rules
</div>

* L’objet event des Actions Post Login ne fournit actuellement pas d’équivalent à `context.connectionOptions`, disponible dans Rules.
* L’objet event des Actions Post Login ne fournit actuellement pas d’équivalent permettant d’accéder, pour les utilisateurs fédérés, aux jetons de l’IdP en amont via l’objet `user.identities`, comme dans Rules.
* Les revendications standard d’un `idToken` ne peuvent pas être supprimées à l’aide d’une Action.
* Les Actions prennent uniquement en charge la modification de l’utilisateur principal entre des comptes liés.
