---

Dans l’application fournie par l’extension Delegated Administration, deux vues sont offertes : **Users** et **Logs**. Dans la vue **User**s, vous pouvez afficher et modifier les utilisateurs associés à votre compte Auth0.

Par défaut, tous les utilisateurs sont affichés, mais vous pouvez filtrer la liste en configurant un hook de filtrage.

Pour en savoir plus sur le hook de filtrage, consultez [Delegated Administration : hook de filtrage](/fr-CA/docs/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-filter-hook).

<div id="user-actions-in-delegated-administration-dashboard">
  ## Actions des utilisateurs dans le tableau de bord Delegated Administration
</div>

Pour limiter le nombre d’options offertes à une personne ayant accès au tableau de bord fourni par l’extension Delegated Administration, configurez un Access Hook.

Pour en savoir plus sur les Access Hooks, consultez [Delegated Administration: Access Hook](/fr-CA/docs/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-access-hook).

L’option **Change Profile** est disponible seulement si vous avez configuré des champs personnalisés.

Notez la nouvelle option **Reset Password** offerte par l’extension. Cette option enverra un courriel à l’utilisateur, lui permettant de choisir un nouveau mot de passe. Pour ce faire, sélectionnez un utilisateur, puis sélectionnez **Actions** &gt; **Reset Password**.

Un courriel sera alors envoyé à l’utilisateur, contenant un lien pour modifier le mot de passe.

Si votre profil indique que vous avez le rôle `Delegated Admin - Administrator`, la vue **Logs** vous permet de consulter une liste des authentifications effectuées par vos utilisateurs (cette vue n’est visible que pour les utilisateurs ayant le rôle `Delegated Admin - Administrator`). Le contenu de cette vue correspond à un sous-ensemble des données affichées dans [Auth0 Dashboard &gt; Monitoring &gt; Logs](https://manage.auth0.com/#/logs). La vue Logs affiche également des données sur les actions administratives effectuées dans le <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

<div id="create-users">
  ## Créer des utilisateurs
</div>

Vous pouvez créer un nouvel utilisateur en cliquant sur le bouton **+ Create User** dans la vue **Users**. Vous devez préciser le courriel et le mot de passe. Selon votre rôle, il se peut que vous ne puissiez pas définir le **Departmen**t auquel le nouvel utilisateur appartient.

Par exemple, les utilisateurs ayant le rôle `Delegated Admin - Administrator` peuvent voir le champ **Department** et en sélectionner n’importe quelle valeur.

<Frame>![Extensions - Administration Dashboard - Create admin](/docs/images/cdy7uua7fh8z/4vrbU6nRuIkF30ZPM4yHnG/73fd2060fe9618c9f465025cb56deb02/create-user-admin.png)</Frame>

À l’inverse, Kelly, qui a le rôle `Delegated Admin - User` et appartient au service des finances, ne peut pas voir le champ **Department**. L’utilisateur qu’elle crée sera automatiquement affecté au service des finances.

<Frame>![Extensions - Delegated Administration Dashboard - Create user](/docs/images/cdy7uua7fh8z/2CSdiPbyiyPsURHH9X5VVZ/8416222627933bd35c044aa15c22dfae/create-user-kelly.png)</Frame>

<Card title="Expiration de la session Delegated Administration">
  Par défaut, la durée d’expiration du jeton est de 10 heures. Toutefois, pour des raisons de sécurité, lorsque vous utilisez Delegated Administration, Auth0 n’enregistre pas de jeton dans les cookies ni dans `sessionStorage`. Vous devez ouvrir une nouvelle session chaque fois que la page est rechargée.
</Card>
