Utilisez l’extension de tableau de bord <Tooltip tip="Authentification unique (SSO) : service qui, une fois qu’un utilisateur s’est connecté à une application, le connecte automatiquement à d’autres applications." cta="Consulter le glossaire" href="/fr-CA/docs/glossary?term=Single+Sign-On">authentification unique</Tooltip> (SSO) pour gérer la connexion SSO de vos utilisateurs sur plusieurs applications d’entreprise. Pour en savoir plus, [consultez cette extension sur GitHub](https://github.com/auth0-extensions/auth0-sso-dashboard-extension).

<div id="create-auth0-application">
  ## Créer une application Auth0
</div>

Avant d’[ajouter l’extension SSO Dashboard](/fr-CA/docs/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension), vous devez créer une application dans Auth0 :

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications), puis sélectionnez **+Create Application**.
2. Entrez un nom descriptif pour votre application (par exemple, SSO Dashboard), sélectionnez **Single-Page Web Application** comme **Application Type**, puis sélectionnez **Create**.
3. Dans l’onglet **Settings**, définissez **Allowed Callback URLs** en fonction de la région de votre locataire :

   1. Pour les **administrateurs** : `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/admins/login`
   2. Pour les **utilisateurs** : `https://{yourTenant}.YOUR_REGION_DOMAIN/auth0-sso-dashboard/login`

      | Région | Domaine régional |
      | ------ | ---------------- |
      | US-1   | us.webtask.io    |
      | US-3   | us.webtask.run   |
      | US-4   | us.webtask.run   |
      | US-5   | us.webtask.run   |
      | EU     | eu.webtask.io    |
      | EU-2   | eu.webtask.run   |
      | AU     | au.webtask.io    |
      | JP-1   | jp.webtask.run   |
      | UK-1   | uk.webtask.run   |
      | CA-1   | ca.webtask.run   |
4. Sélectionnez et copiez la valeur de l’**ID client**.
5. Allez au bas de la page, puis sélectionnez **Advanced Settings**.
6. Sélectionnez l’onglet **OAuth**, puis collez la valeur de l’**ID client** dans le champ **Allowed APPs / APIs**.
7. Définissez **JsonWebToken Signature Algorithm** à **RS256**.
8. Sélectionnez **Save Changes**.

Par défaut, tous les types de connexion sont activés afin que les utilisateurs puissent se connecter au SSO Dashboard. Si vous souhaitez modifier ce comportement, accédez à l’onglet **Connections** de l’application.

<div id="install-sso-dashboard-extension">
  ## Installer l’extension SSO Dashboard
</div>

Ensuite, vous devrez [installer l’extension SSO Dashboard](/fr-CA/docs/customize/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension) et [ajouter des applications](/fr-CA/docs/customize/extensions/single-sign-on-dashboard-extension/add-applications-to-the-sso-dashboard) au tableau de bord.
