Vous pouvez configurer ACUL à l’aide du Auth0 Dashboard, de l’Auth0 Management API et d’outils d’Infrastructure as Code (IaC). Chaque méthode modifie le même objet de configuration de rendu propre à chaque écran.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour configurer ACUL, assurez-vous que vos fichiers de configuration personnalisés et vos ressources sont hébergés sur un CDN public.
</Callout>

Pour comprendre l’objet de configuration de rendu et ses propriétés, consultez la [référence de la configuration de rendu ACUL](/fr-CA/docs/customize/login-pages/advanced-customizations/configure/reference).

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Vous pouvez utiliser l’Auth0 Dashboard pour personnaliser vos écrans avec ACUL :

A. Accédez à [Auth0 Dashboard &gt; Image de marque &gt; Universal Login](https://manage.auth0.com/dashboard/#/universal-login/customizations-new)

1. Sélectionnez **Personnaliser les écrans d’authentification**.
   2. Sélectionnez un <code>screen</code> à personnaliser.
   3. Dans **Paramètres &gt; Mode de rendu**, sélectionnez **Mode avancé**.
   4. (Facultatif) Sélectionnez **Utiliser un modèle de page personnalisé**.
   5. (Facultatif) Sélectionnez **Filtres du mode avancé**.

<Frame>
  <img src="/docs/images/cdy7uua7fh8z/ACUL/aculDashboard.png" alt="Page ACUL dans Auth0 Dashboard" />
</Frame>

6. Sélectionnez **Head Tags** pour ajouter vos propres balises HTML `Head Tags`.
   7. (Facultatif) Sélectionnez **Données supplémentaires** pour choisir quelles données supplémentaires envoyer au navigateur.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Données supplémentaires** vous permet d’ajouter des propriétés de données contextuelles facultatives. Par exemple : les données contextuelles facultatives de [`user`](/fr-CA/docs/libraries/acul/js-sdk/Screens/interfaces/UserMembers).
</Callout>

B. Sélectionnez **Enregistrer et publier**.

<div id="infrastructure-as-code-iac">
  ## Infrastructure as Code (IaC)
</div>

Configurez ACUL à l’aide d’outils d’IaC, notamment :

* la commande [`auth0 acul`](https://auth0.github.io/auth0-cli/auth0_acul.html) d’**Auth0 CLI**.
  * la ressource [`auth0_prompt_screen_renderer`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/prompt_screen_renderer) d’**Auth0 Terraform Provider**.
  * la ressource [`prompts`](https://github.com/auth0/auth0-deploy-cli/blob/master/docs/resource-specific-documentation.md#prompts) d’**Auth0 Deploy CLI**.

<div id="auth0-management-api">
  ## Auth0 Management API
</div>

Utilisez les points de terminaison de l’Auth0 Management API pour configurer individuellement les écrans Universal Login ou mettre à jour plusieurs écrans en un seul appel d’API. Vous devez inclure un [jeton d’accès de la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens) dans la requête.

**Configuration individuelle d’un écran**

* Le point de terminaison [`GET /v2/prompts/{prompt}/screen/{screen}/rendering`](/fr-CA/docs/api/management/v2/prompts/get-rendering) récupère la configuration d’un seul écran.

  * Le point de terminaison [`PATCH /v2/prompts/{prompt}/screen/{screen}/rendering`](/fr-CA/docs/api/management/v2/prompts/patch-rendering) configure ACUL pour un seul écran.

**Configuration groupée des écrans**

* Le point de terminaison [`GET /v2/prompts/rendering`](/fr-CA/docs/api/management/v2/prompts/get-all-rendering) récupère la configuration de tous les écrans.

  * Le point de terminaison [`PATCH /v2/prompts/rendering`](/fr-CA/docs/api/management/v2/prompts/patch-bulk-rendering) met à jour les écrans en lot.
