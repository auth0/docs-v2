Auth0 propose plusieurs SDK à utiliser avec ACUL.

<div id="acul-sdks">
  ## SDK ACUL
</div>

* [SDK React ACUL d’Auth0](/fr-CA/docs/libraries/acul/react-sdk)
  Fournit des hooks React pour simplifier l’intégration aux applications React.

  * [SDK JS ACUL d’Auth0](/fr-CA/docs/libraries/acul/js-sdk)
    La bibliothèque JavaScript principale (écrite en TypeScript), qui convient à n’importe quel framework front-end ou au JavaScript pur.

<div id="acul-sample-applications">
  ## Applications d’exemple ACUL
</div>

Les [applications d’exemple ACUL](https://github.com/auth0-samples/auth0-acul-samples) comprennent une chaîne de compilation complète, des workflows de déploiement et des implémentations pour des écrans Universal Login courants. Ces applications d’exemple vous aident à développer vos écrans. Les exemples sont offerts en React avec les SDK ACUL React et JS.

Vous pouvez utiliser la commande `auth0 acul init` pour initialiser un projet React de base avec le SDK ACUL React ou JS. Pour en savoir plus, consultez [Auth0 CLI](https://github.com/auth0/auth0-cli) sur GitHub.

<div id="acul-resources">
  ## Ressources ACUL
</div>

Auth0 propose plusieurs ressources pour personnaliser votre expérience Universal Login.

<div id="auth0-management-api">
  ### Auth0 Management API
</div>

Mettez à jour des écrans Universal Login individuellement ou configurez plusieurs écrans à la fois à l’aide de la Management API. Pour utiliser l’Auth0 Management API, vous devez disposer d’un [jeton d’accès](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens).

**Configuration d’un écran individuel**

* Le point de terminaison [`GET /v2/prompts/{prompt}/screen/{screen}/rendering`](/fr-CA/docs/api/management/v2/prompts/get-rendering) vous permet de récupérer la configuration d’un écran.

  * Le point de terminaison [`PATCH /v2/prompts/{prompt}/screen/{screen}/rendering`](/fr-CA/docs/api/management/v2/prompts/patch-rendering) vous permet de configurer ACUL pour un écran.

**Configuration groupée des écrans**

* Le point de terminaison [`GET /v2/prompts/rendering`](/fr-CA/docs/api/management/v2/prompts/get-all-rendering) vous permet de récupérer la configuration de tous les écrans.

  * Le point de terminaison [`PATCH /v2/prompts/rendering`](/fr-CA/docs/api/management/v2/prompts/patch-bulk-rendering) vous permet de mettre à jour plusieurs écrans en lot.

<div id="auth0-developer-tools">
  ### Outils de développement Auth0
</div>

Auth0 propose plusieurs outils pour simplifier la gestion des locataires, y compris la configuration d’ACUL.

* **Auth0 CLI**<p />
  Utilisez les commandes `acul` pour initialiser des projets, effectuer le développement en local et déployer vos configurations d’écran. Pour en savoir plus, consultez [Auth0 CLI](https://github.com/auth0/auth0-cli) sur GitHub.

  * **Auth0 Deploy CLI**<p />
    Gérez toute la configuration de votre locataire sous forme de code, y compris les paramètres de rendu ACUL définis dans des fichiers JSON. Pour en savoir plus, consultez [Auth0 Deploy CLI](https://github.com/auth0/auth0-deploy-cli) sur GitHub.

  * **Auth0 Terraform provider**<p />
    Définissez vos configurations d’écran ACUL. Pour en savoir plus, consultez la [`ressource promt_screen_renderer`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/prompt_screen_renderer) sur Terraform.
