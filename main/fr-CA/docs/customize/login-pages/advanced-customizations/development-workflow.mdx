Il existe deux principaux modes de développement pour vos écrans ACUL : le développement local et les tests d’intégration avec un locataire réel. 

<div id="local-development">
  ## Développement local
</div>

Personnalisez vos écrans ACUL localement avec la commande `auth0 acul dev` de l’[Auth0 CLI](https://github.com/auth0/auth0-cli/). Itérez rapidement sur vos écrans ACUL, sans délai ni dépendance au backend, grâce à l’[Universal Login Context Inspector](https://github.com/auth0/ul-context-inspector).

L’Universal Login Context Inspector fournit des données de contexte simulées pour tester vos écrans d’authentification selon des configurations propres au locataire.

<Frame>
  <img src="/docs/images/cdy7uua7fh8z/ACUL/UniversalLoginContextInspector.png" alt="Universal Login Context Inspector" />
</Frame>

* Sélectionnez **Écran** pour voir vos écrans ACUL configurés avec les données de contexte simulées par défaut.
  * Sélectionnez **Source de données** pour choisir des données de contexte simulées fournies par **Auth0 CDN** ou des données de contexte simulées provenant du répertoire `public/screens/{prompt}/{screen}` de votre projet **Développement local**.
  * Sélectionnez l’icône **Télécharger JSON** pour télécharger les données de contexte de l’écran sélectionné. 

<div id="live-tenant-integration-testing">
  ## Tests d’intégration en direct avec le locataire
</div>

L’intégration en direct avec le locataire au moyen de la commande Auth0 CLI `auth0 acul dev --connected` permet d’effectuer des tests de bout en bout de vos interfaces d’écran personnalisées et de vérifier l’intégration avec votre locataire Auth0.
Utilisez le mode `--connected` pour les tests d’intégration une fois l’interface presque terminée, car chaque rechargement redémarre le flux d’authentification.

Lors de l’utilisation de `auth0 acul dev --connected`

* Sélectionnez le répertoire de votre projet ACUL.
  * Sélectionnez `npm run build` pour générer le projet et configurer votre locataire Auth0.
  * Sélectionnez le **port** qui servira à héberger vos ressources locales, soit `55444` par défaut, puis démarrez le serveur local en arrière-plan.
  * Au besoin, activez dans le mode `--connected` la surveillance continue des changements dans le répertoire `/dist/assets` du projet.

Ouvrez un nouveau terminal et exécutez `auth0 test login` pour tester le flux de connexion sur votre locataire.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  En mode `--connected`, la commande Auth0 CLI surveille les modifications de fichiers, regénère les ressources et met automatiquement à jour la configuration de votre locataire.
</Callout>

Pour en savoir plus, consultez [Auth0 CLI](https://auth0.github.io/auth0-cli/auth0_acul_dev.html) sur GitHub.
