La page de configuration principale vous permet de gérer les paramètres suivants pour Universal Login et Classic Login :

* **Moteur de connexion** : Basculez entre Universal Login et Classic Login.
* **Intégration dans des iframes** : Autorisez le chargement des flux de connexion dans des iframes pour les environnements de développement.

<div id="settings">
  ## Paramètres
</div>

<div id="universal-or-classic-login">
  ### Universal Login ou Classic Login
</div>

Choisissez l’expérience de connexion utilisée par votre locataire. La modification de ce paramètre touche toutes les applications qui redirigent vers votre page de connexion Auth0. Pour en savoir plus, consultez [Universal Login vs. Classic Login](/fr-CA/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login).

<Steps>
  <Step title="Accéder à la configuration principale">
    Accédez à **image de marque &gt; Universal Login &gt; Pro-code customizations** et sélectionnez **configuration principale**.
  </Step>

  <Step title="Sélectionner le type de connexion">
    Sous **Settings**, sélectionnez **Universal Login** ou **Classic Login**.
  </Step>
</Steps>

<div id="iframe-embedding">
  ### Intégration dans un iframe
</div>

Activez Universal Login ou Classic Login pour qu’ils se chargent lorsque l’application et le flux de connexion sont intégrés dans un iframe, comme avec des plateformes d’UI générative comme Vercel. Lorsque cette option est activée, l’intégration dans un iframe s’applique uniquement aux environnements de développement.

<Warning>
  Lorsque l’intégration dans un iframe est activée, les en-têtes de sécurité qui protègent contre le clickjacking sont désactivés.
</Warning>

<Steps>
  <Step title="Accéder à la configuration principale">
    Accédez à **Image de marque &gt; Universal Login &gt; Personnalisations pro-code** et sélectionnez **Configuration principale**.
  </Step>

  <Step title="Activer la prise en charge des iframes">
    Sous **Paramètres d’iframe**, activez **Activer la prise en charge des plateformes d’UI générative**.
  </Step>

  <Step title="Configurer les origines autorisées">
    Sous **URL d’iframe autorisées pour Login**, entrez les origines autorisées.

    Pour v0, entrez les origines suivantes :

    * `https://v0.app`
    * `https://*.v0.app`
    * `https://*.vusercontent.net`

    Lorsqu’une application est créée à partir d’une plateforme d’UI générative comme Vercel au moyen d’une intégration Auth0 disponible, Auth0 préremplit les origines autorisées à partir de la plateforme. Vérifiez toujours les origines préremplies pour vous assurer qu’elles sont complètes. Pour en savoir plus, consultez [Intégrer avec Vercel](/fr-CA/docs/customize/integrations/integrate-with-vercel).
  </Step>
</Steps>
