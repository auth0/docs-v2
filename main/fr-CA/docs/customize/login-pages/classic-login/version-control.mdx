title: Contrôle de version des pages Classic Login
---

<Warning>
  À l’heure actuelle, les efforts de développement d’Auth0 sont axés sur Universal Login, et Classic Login n’est plus mis à jour. Pour personnaliser vos pages de connexion, Auth0 recommande [Personnalisations avancées pour Universal Login](/fr-CA/docs/customize/login-pages/advanced-customizations).
</Warning>

Vous pouvez utiliser un logiciel de contrôle de version pour gérer le code source de vos pages Classic Login. L’utilisation du contrôle de version pour votre contenu peut être particulièrement utile si vous utilisez un flux CI/CD pour la configuration de votre locataire et si vous avez personnalisé le HTML d’une ou de plusieurs pages Classic Login.

Pour mettre en place le contrôle de version, utilisez l’intégration Auth0 Marketplace correspondant à la méthode de votre choix :

* [GitLab Pipelines](https://marketplace.auth0.com/integrations/gitlab-pipeline)
* [GitHub Actions](https://marketplace.auth0.com/integrations/github-actions)
* [Bitbucket Pipelines](https://marketplace.auth0.com/integrations/bitbucket-pipeline)
* [Microsoft Azure Pipelines](https://marketplace.auth0.com/integrations/azure-pipeline)

Le processus général de déploiement comprend les étapes suivantes :

1. Créez un dossier dans votre dépôt de contrôle de version avec le nom approprié (`pages`).
2. Créez une page HTML (`login.html`, `password_reset.html`, `guardian_multifactor.html` ou `error_page.html`) dans ce dossier.
3. Créez un fichier JSON portant le même nom que votre page HTML pour chaque page hébergée que vous souhaitez gérer sous contrôle de version. Pour activer la page, le fichier JSON doit contenir ce qui suit :

`{
"enabled": true
}`

Par exemple :
`your-repo/pages/error_page.html
your-repo/pages/error_page.json`
