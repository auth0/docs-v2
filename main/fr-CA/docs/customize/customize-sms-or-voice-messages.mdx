---
title: Personnaliser les messages SMS et vocaux d’authentification multifacteur
description: Découvrez comment personnaliser les messages SMS et vocaux envoyés par Auth0 lors de l’inscription et de la vérification.
---

Pour personnaliser les messages SMS et vocaux d’authentification multifacteur envoyés par Auth0 :

1. Accédez à [Auth0 Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa), puis cliquez sur **Phone Message**.
2. Dans la section **Enrollment Template** ou **Verification Template**, personnalisez votre message.
3. Cliquez sur **Save**.

Le **Enrollment Template** définit le message envoyé lorsqu’un utilisateur inscrit un nouvel appareil pour la première fois à l’aide de la MFA. Le **Verification Template** définit le message envoyé lorsqu’un utilisateur se connecte après l’inscription.

<div id="syntax">
  ## Syntaxe
</div>

La syntaxe [Liquid](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) est le moteur de modèles pris en charge pour accéder aux attributs de l’utilisateur dans les modèles SMS. Les attributs suivants sont disponibles :

| Attribute                       | Description                                                                                                                                                                                                                                               |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `message_type`                  | Indique le type de message envoyé : `sms` ou `voice`.                                                                                                                                                                                                     |
| `code`                          | Code d’inscription ou de vérification. Lors de l’envoi de messages vocaux, les valeurs des variables sont séparées par des points entre les chiffres (`1.2.3.4.5`) afin d’en assurer la prononciation correcte par les fournisseurs de messagerie vocale. |
| `locale`                        | Pour Universal Login ou l’API MFA. Indique la langue.                                                                                                                                                                                                     |
| `requestInfo.lang`              | Pour la localisation de Classic Login. En-tête `accept-language` du navigateur. Par exemple : `es-AR,es;q=8.0`, `en-US,en`.                                                                                                                               |
| `tenant.friendlyName`           | Le **Friendly Name** défini dans **Auth0 Dashboard &gt; Locataire Settings**.                                                                                                                                                                             |
| `custom_domain.domain`          | Le nom de domaine du locataire.                                                                                                                                                                                                                           |
| `custom_domain.domain_metadata` | Les champs de métadonnées du domaine personnalisé (sous forme de paires clé-valeur).                                                                                                                                                                      |

Voici un exemple d’utilisation de la syntaxe Liquid pour envoyer différents messages vocaux et SMS selon la langue :

```liquid lines
{% if message_type == "voice" %}
    {% if locale contains "fr" %}
        Bonjour, vous avez demandé à recevoir un code de vérification pour vous enregister avec {{tenant.friendly_name}}. Votre code est: {{pause}} {{code}}. Je répète, votre code est: {{pause}}{{code}}.
    {% elsif locale contains "es" %}
        Usted ha requerido un código de verificación para inscribirse con {{tenant.friendly_name}}. Su código es: {{pause}}{{code}}. Repito, su código es: {{pause}}{{code}}.
    {% else %}
        Hello, you requested a verification code to enroll with {{tenant.friendly_name}}. Your code is: {{pause}}{{code}}. I repeat, your code is: {{pause}}{{code}}.
    {% endif %}
{% else %}
    {% if locale contains "fr" %}
        {{code}} est votre code de vérification pour vous enregistrer avec {{tenant.friendly_name}}.
    {% elsif locale contains "es" %}
        {{code}} es su código para inscribirse con {{tenant.friendly_name}}.
    {% else %}
        {{code}} is your verification code to enroll with {{tenant.friendly_name}}.
    {% endif %}
{% endif %}
```

<div id="localization">
  ## Localisation
</div>

Dans l’Authentication API d’Auth0, le [point de terminaison Get Code or Link](https://auth0.com/docs/api/authentication/passwordless/get-code-or-link), le point de terminaison de [demande de défi](https://auth0.com/docs/api/authentication/muti-factor-authentication/request-mfa-challenge) MFA et le point de terminaison MFA [Add an authenticator](https://auth0.com/docs/api/authentication/muti-factor-authentication/add-an-authenticator) prennent en charge l’en-tête `x-request-language`.

Pour l’utiliser, définissez les langues prises en charge dans **Locataire &gt; Settings** (sous **Supported Languages**). Lorsque l’en-tête `x-request-lanuage` est envoyé, cette langue est accessible dans la variable `locale` des modèles de messages MFA par téléphone. Consultez [la prise en charge multilingue de Passwordless](/fr-CA/docs/authenticate/passwordless/authentication-methods/sms-otp#multi-language-support) pour en savoir plus.
