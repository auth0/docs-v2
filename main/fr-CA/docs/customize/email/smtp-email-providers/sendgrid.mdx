<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [SendGrid](https://sendgrid.com). Si vous avez un abonnement Microsoft Azure, vous pouvez obtenir un compte gratuit dans la Place de marché Azure.

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes à partir de certaines [adresses IP d’Auth0](/fr-CA/docs/secure/security-guidance/data-security/allowlist).

<div id="configure-sendgrid">
  ## Configurer SendGrid
</div>

1. Connectez-vous à votre compte [SendGrid](https://sendgrid.com).
2. Accédez à **Paramètres &gt; Clés API** dans SendGrid, puis cliquez sur **Créer une clé API**.
3. Entrez un nom pour votre clé et sélectionnez **Accès restreint**.
4. Sous **Détails de l’accès**, développez **Envoi de courriel**.
5. Réglez l’autorisation **Envoi de courriel** sur **Accès complet**.
6. Cliquez sur **Enregistrer** et copiez la valeur de la clé.
7. Accédez à Auth0 [Dashboard &gt; Branding &gt; fournisseur de courriel](https://manage.auth0.com/#/branding/email_provider).
8. Activez le bouton bascule **Utiliser mon propre fournisseur de courriel**.
9. Cliquez sur le logo **SendGrid**.
10. Entrez une adresse de courriel dans le champ **De**, collez la **Clé API** SendGrid que vous avez copiée, puis cliquez sur **Enregistrer**.

Si vous utilisez l’[authentification Passwordless par courriel](/fr-CA/docs/authenticate/passwordless#email), la valeur de l’adresse de courriel **De** de votre connexion <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=passwordless">Passwordless</Tooltip> doit correspondre à celle que vous utilisez pour SendGrid.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer un courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez pas de courriel après quelques minutes, consultez vos [journaux Auth0](https://manage.auth0.com/#/logs) afin d’y repérer les codes d’erreur.

Une fois la configuration effectuée, la page [Email Activity](https://sendgrid.com/logs/index) de SendGrid affiche des informations sur la livraison de tous les courriels envoyés à vos utilisateurs.
