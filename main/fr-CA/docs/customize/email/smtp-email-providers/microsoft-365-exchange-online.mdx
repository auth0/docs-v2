<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Microsoft 365](https://www.microsoft.com/en-us/microsoft-365) avec un abonnement actif.

* Votre abonnement Microsoft 365 doit inclure une licence qui permet d’envoyer des courriels via Exchange Online. Vous devez disposer au minimum d’un abonnement payant au forfait Microsoft Business Basic pour utiliser l’API Microsoft Graph pour Exchange Online.

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes provenant des [adresses IP d’Auth0](/fr-CA/docs/secure/security-guidance/data-security/allowlist).

<div id="configure-microsoft-365-exchange-online">
  ## Configurer Microsoft 365 Exchange Online
</div>

1. Ouvrez une session sur [Microsoft Azure](https://azure.microsoft.com/) avec votre compte Microsoft 365.
2. Enregistrez une ressource [Application Microsoft Entra ID](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application) si vous n’en avez pas déjà une.
3. Accédez à la ressource de votre application Active Directory. Sélectionnez **Services &gt; Inscriptions d’applications**, puis sélectionnez l’application que vous utiliserez pour envoyer des courriels.
4. Configurez les autorisations d’application requises en accédant à **Gérer &gt; Autorisations d’API**.
5. Ajoutez l’[autorisation](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#api-permissions) `Mail.Send` en sélectionnant **Ajouter une autorisation &gt; Microsoft Graph &gt; Autorisations d’application**.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Avant de configurer les **Autorisations d’application**, vous devrez obtenir un consentement administratif.
   </Callout>

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
     Par défaut, `Mail.Send` accorde l’autorisation d’envoyer des messages au nom de n’importe quel utilisateur de votre locataire. Pour restreindre l’accès à une boîte aux lettres précise, utilisez [RBAC for Applications in Exchange Online](https://learn.microsoft.com/en-us/exchange/permissions-exo/application-rbac) pour limiter la portée de l’autorisation à un principal de service précis.
   </Callout>
6. Récupérez les paramètres du fournisseur requis :

   1. Accédez à la **Vue d’ensemble de l’application** pour récupérer l’[ID client et l’ID du locataire](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#application-id-client-id) de votre application.
   2. Accédez à **Gérer &gt; Certificats et secrets** et récupérez la **Valeur** pour créer un [secret](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application#certificates--secrets) d’application.
   3. Récupérez le **Courriel de l’utilisateur** de l’organisation qui envoie les courriels au moyen de l’application. Accédez à **Utilisateurs &gt; Utilisateurs actifs**, sélectionnez votre utilisateur autorisé, puis récupérez son **Courriel de l’utilisateur** dans ses propriétés.
7. Accédez à [Auth0 Dashboard &gt; Image de marque &gt; Fournisseur de courriel](https://manage.auth0.com/#/branding/email_provider).
8. Activez le bouton bascule **Utiliser mon propre fournisseur de courriel**.
9. Cliquez sur le logo **Microsoft 365**.
10. Indiquez l’adresse de courriel **De**. Ensuite, entrez l’**ID du locataire**, l’**ID client** et le **Secret client** de l’étape 6, puis cliquez sur **Enregistrer**.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Send Test Email**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez pas de courriel après quelques minutes, consultez vos [journaux Auth0](https://manage.auth0.com/#/logs) pour vérifier s’il y a des codes d’erreur.

Une fois la configuration terminée, la page [Email Usage](https://admin.microsoft.com/#/reportsUsage/EmailActivity) de Microsoft Exchange affiche des renseignements sur la livraison de tous les courriels envoyés à vos utilisateurs.
