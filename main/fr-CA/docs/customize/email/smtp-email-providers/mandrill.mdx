<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Mandrill](https://www.mandrill.com/signup/).

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes à partir d’[adresses IP Auth0 spécifiques](/fr-CA/docs/secure/security-guidance/data-security/allowlist).

<div id="configure-mandrill">
  ## Configurer Mandrill
</div>

1. Connectez-vous à [Mandrill](https://www.mandrill.com/signup/).
2. Accédez aux [Settings](https://mandrillapp.com/settings) de Mandrill, puis cliquez sur **Add API key**. Copiez la valeur de cette clé.
3. Accédez à [Dashboard &gt; Branding &gt; fournisseur de courriel](https://manage.auth0.com/#/branding/email_provider) dans Auth0.
4. Activez le bouton bascule **Use my own email provider**.
5. Cliquez sur le logo **Mandrill**.
6. Indiquez une adresse de courriel **From** et saisissez la **API Key** Mandrill que vous avez copiée précédemment.
7. Cliquez sur **Save**.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Envoyer un courriel de test**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez pas de courriel après quelques minutes, consultez vos [journaux d’Auth0](https://manage.auth0.com/#/logs) pour voir les codes d’erreur.

Une fois la configuration effectuée, la page [Outbound Activity](https://mandrillapp.com/activity) de Mandrill affiche des informations sur la livraison de tous les courriels envoyés à vos utilisateurs.
