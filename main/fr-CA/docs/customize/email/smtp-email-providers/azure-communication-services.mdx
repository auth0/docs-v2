<div id="prerequisites">
  ## Prérequis
</div>

* Vous devez avoir un compte [Microsoft Azure](https://azure.microsoft.com/en-us).

* Vous devez configurer votre fournisseur pour autoriser les connexions entrantes provenant d’[adresses IP Auth0](/fr-CA/docs/secure/security-guidance/data-security/allowlist) précises.

<div id="configure-azure-communication-services">
  ## Configurer Azure Communication Services
</div>

Commencez par créer une ressource dans Microsoft Azure :

1. Connectez-vous à votre compte [Microsoft Azure](https://azure.microsoft.com/en-us).
2. Entrez « Communication Services » dans la barre de recherche.
3. Créez une ressource [Communication Services](https://learn.microsoft.com/en-us/azure/communication-services/quickstarts/create-communication-resource) si vous n’en avez pas déjà une.
4. Sélectionnez **Create**.
5. Sous les paramètres **Tools**, sélectionnez **Keys**. Récupérez votre [chaîne de connexion Communication Services](https://learn.microsoft.com/en-us/azure/communication-services/quickstarts/create-communication-resource).
6. Sous **Primary Key**, copiez la valeur de **Connection string**.
   Une chaîne de connexion valide suit le format `endpoint=COMMUNICATION_SERVICES_URL;accesskey=ACCESS_KEY`.

Ensuite, configurez votre fournisseur de courriel dans Auth0 :

1. Accédez à [Dashboard &gt; Image de marque &gt; Fournisseur de courriel](https://manage.auth0.com/#/branding/email_provider).

2. Sélectionnez **Use my own email provider**.

3. Sélectionnez **Azure Communication Services**.

4. Entrez une adresse de courriel **From**.

   <Warning>
     Azure Communications Services prend uniquement en charge une adresse de courriel brute (`user@example.com`) dans le champ **From**.
   </Warning>

5. Ajoutez la **Connection string** d’Azure Communication Services.

6. Sélectionnez **Save**.

<div id="send-a-test-email">
  ## Envoyer un courriel de test
</div>

Envoyez un courriel de test à l’aide du bouton **Send Test Email**. Si la configuration est correcte, vous recevrez un courriel de confirmation. Si vous ne recevez pas de courriel après quelques minutes, consultez vos [journaux Auth0](https://manage.auth0.com/#/logs) pour obtenir les codes d’erreur.

Une fois la configuration effectuée, la page [Metrics Monitor](https://learn.microsoft.com/en-us/azure/communication-services/concepts/metrics#chat-api-request-metric-operations) de Communication Services affiche des données sur la livraison de tous les courriels envoyés à vos utilisateurs.
