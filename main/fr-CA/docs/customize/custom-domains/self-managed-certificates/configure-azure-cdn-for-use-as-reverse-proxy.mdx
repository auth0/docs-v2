title: Configurer Azure CDN comme proxy inverse
---

<Card title="La disponibilité varie selon le forfait Auth0">
  Votre forfait Auth0 ou votre entente personnalisée détermine si cette fonctionnalité est offerte. Pour en savoir plus, consultez [Tarification](https://auth0.com/pricing).
</Card>

Pour configurer Azure CDN comme proxy inverse, un forfait Azure CDN Premium est requis.

1. [Configurez les domaines personnalisés avec des certificats autogérés](/fr-CA/docs/customize/custom-domains/self-managed-certificates) si ce n’est pas déjà fait. Notez les valeurs **Origin Domain Name** et **cname-api-key**, car vous en aurez besoin plus tard.
2. Ouvrez une session dans le [portail Azure](https://portal.azure.com/).
3. [Créez un profil Azure CDN et un point de terminaison](https://learn.microsoft.com/en-us/azure/cdn/cdn-create-new-endpoint). Dans les paramètres du point de terminaison CDN, indiquez la valeur **Origin Domain Name** provenant d’Auth0 Dashboard.
4. [Ajoutez votre domaine personnalisé à votre point de terminaison Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-map-content-to-custom-domain?tabs=azure-dns%2Cazure-portal%2Cazure-portal-cleanup).
5. [Configurez HTTPS pour votre domaine personnalisé Azure CDN](https://learn.microsoft.com/en-us/azure/cdn/cdn-custom-ssl?tabs=option-1-default-enable-https-with-a-cdn-managed-certificate). Ce processus vous oblige à vérifier que vous êtes propriétaire du domaine. Une fois cette étape terminée, le déploiement du certificat sur tous les points de présence du CDN peut prendre jusqu’à 6 heures.
6. Configurez la communication du domaine personnalisé avec Auth0 à l’aide du moteur de règles Azure CDN. (Pour en savoir plus, consultez [Remplacer le comportement HTTP à l’aide du moteur de règles Azure CDN Verizon Premium](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine) dans la documentation Microsoft.) [Créez une nouvelle règle Azure CDN](https://learn.microsoft.com/en-us/previous-versions/azure/cdn/cdn-verizon-premium-rules-engine#tutorial) avec les paramètres suivants :

   | Paramètre        | Valeur                                                                                                      |
   | ---------------- | ----------------------------------------------------------------------------------------------------------- |
   | Nom/Description  | Auth0 Custom Domain                                                                                         |
   | Type de requêtes | Sélectionnez l’option **Edge CName**, puis sélectionnez le nom de votre domaine personnalisé dans la liste. |
7. Ajoutez les **fonctionnalités** suivantes à votre règle Azure CDN :
   Nous vous recommandons de créer une autre règle Azure CDN afin de bloquer l’utilisation du CNAME **azureedge.net**.

   | Paramètre                               | Valeur                                                                                                                         |
   | --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
   | Contourner le cache                     | **Activé**                                                                                                                     |
   | Modifier l’en-tête de la requête client | Sélectionnez **Override**, entrez **cname-api-key** comme nom, puis saisissez comme valeur la clé API CNAME fournie par Auth0. |
8. Une fois la règle Azure CDN approuvée, son état passera de Pending XML à Active XML. À ce moment, Azure CDN publiera les règles et les certificats. Quand Azure aura terminé le traitement de toutes les modifications, vous pourrez utiliser votre domaine personnalisé.
