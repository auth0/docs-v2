<Card title="La disponibilité varie selon le forfait Auth0">
  La disponibilité de cette fonctionnalité dépend de votre implémentation de connexion et de votre forfait Auth0 ou de votre entente personnalisée. Pour en savoir plus, consultez [Pricing](https://auth0.com/pricing).
</Card>

Pour configurer un produit Akamai comme proxy inverse, procédez comme suit :

1. Vérifiez que vous êtes propriétaire de votre domaine personnalisé. Pour en savoir plus, consultez [Verify Ownership](/fr-CA/docs/customize/custom-domains/auth0-managed-certificates#verify-ownership).
2. Accédez à [Dashboard &gt; Settings &gt; Custom Domains](https://manage.auth0.com/#/tenant/custom_domains) pour récupérer les paramètres de distribution, l’en-tête personnalisé d’origine, le nom d’hôte d’origine et le comportement de cache par défaut. Utilisez ensuite ces renseignements pour mettre à jour vos paramètres de domaine personnalisé Akamai afin qu’ils pointent vers le domaine du serveur proxy inverse. Pour en savoir plus, consultez [Configurer le proxy inverse](/fr-CA/docs/customize/custom-domains/self-managed-certificates#configure-reverse-proxy).
3. Mettez à jour l’en-tête IP client du domaine personnalisé à l’aide du point de terminaison PATCH de la Management API  `/api/v2/custom-domains/{id}` comme suit :

   ```json lines
   { 
   "custom_client_ip_header": "true-client-ip" 
   }
   ```

   Pour en savoir plus, consultez [Update custom domain configuration](https://auth0.com/docs/api/management/v2#!/Custom_Domains/patch_custom_domains_by_id).

Pour savoir comment configurer Akamai comme proxy inverse à l’aide des renseignements ci-dessus, consultez la documentation d’Akamai ou son service à la clientèle.
