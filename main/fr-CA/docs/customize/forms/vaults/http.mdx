<div id="configure-your-http-vault-connection-options">
  ## Configurez les options de votre HTTP Vault Connection
</div>

Vous pouvez configurer HTTP Vault Connection avec différentes options d’authentification, puis l’utiliser dans vos Actions de flux de requêtes HTTP.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-options.png)</Frame>

<div id="configure-your-http-vault-connection-for-bearer-token">
  ## Configurez votre HTTP Vault Connection pour Bearer Token
</div>

Pour configurer une HTTP Vault Connection avec l’autorisation **Bearer Token**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP](/docs/images/customize/forms/vaults/http-vault-connection-bearer-token.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre    | Description                                                             |
| ------------ | ----------------------------------------------------------------------- |
| Bearer Token | Jeton statique utilisé pour l’autorisation auprès d’un service externe. |

<div id="configure-your-http-vault-connection-for-oauth-client-credentials">
  ## Configurez votre HTTP Vault Connection pour l’autorisation OAuth Client Credentials
</div>

Pour configurer une HTTP Vault Connection à l’aide de l’autorisation **Client Credentials**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-oauth-client-credentials.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre                         | Description                                                                                             |
| --------------------------------- | ------------------------------------------------------------------------------------------------------- |
| Point de terminaison du locataire | Point de terminaison du locataire du serveur d’autorisation qui émet les jetons d’accès.                |
| ID client                         | ID client de votre application M2M autorisée pour l’API personnalisée.                                  |
| Secret client                     | Secret client de votre application M2M autorisée pour l’API personnalisée.                              |
| Audience (facultatif)             | Audience de l’API personnalisée. Généralement utilisée pour les API personnalisées définies dans Auth0. |
| Ressource (facultatif)            | Ressource de l’API personnalisée. Paramètre standard dont le sens est semblable à celui d’Audience.     |
| Scope (facultatif)                | Liste de scopes sous forme de chaîne, séparée par des espaces.                                          |

<div id="configure-your-http-vault-connection-for-api-key">
  ## Configurez votre HTTP Vault Connection avec une clé API
</div>

Pour configurer une HTTP Vault Connection à l’aide de l’autorisation **Clé API**, définissez les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-api-key.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre | Description                                                                                                                     |
| --------- | ------------------------------------------------------------------------------------------------------------------------------- |
| Nom       | Nom de la clé API à ajouter comme en-tête ou paramètre de requête.                                                              |
| Valeur    | Valeur de la clé API à ajouter comme en-tête ou paramètre de requête.                                                           |
| Ajouter à | Par défaut, l’option En-tête est sélectionnée. Détermine si la clé API est ajoutée comme en-tête ou comme paramètre de requête. |

<div id="configure-your-http-vault-connection-for-basic-authentication">
  ## Configurer votre HTTP Vault Connection pour l’authentification de base
</div>

Pour configurer une HTTP Vault Connection à l’aide du mode d’autorisation **Basic Auth**, vous devez définir les paramètres requis.

<Frame>![Dashboard > Actions > Forms > Vault > HTTP ](/docs/images/customize/forms/vaults/http-vault-connection-basic-authentication.png)</Frame>

<div id="input-settings">
  ### Paramètres d’entrée
</div>

| Paramètre         | Description                                                                                                              |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------ |
| Nom d’utilisateur | Nom d’utilisateur à concaténer et à encoder en base64, puis à ajouter comme paramètre de l’en-tête d’autorisation Basic. |
| Mot de passe      | Mot de passe à concaténer et à encoder en base64, puis à ajouter comme paramètre de l’en-tête d’autorisation Basic.      |

<Warning>
  La configuration HTTP Vault Connection avec Basic Auth est destinée à remplacer l’option actuellement intégrée à Flows HTTP Request Action.
</Warning>
