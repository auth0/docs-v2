---
description: Décrit comment consulter les exécutions de flux et déboguer les exécutions de Forms.
title: Exécution et débogueur
---

Chaque fois qu&#39;un flux s&#39;exécute, Auth0 enregistre une entrée que vous pouvez utiliser pour déboguer l&#39;exécution. Cela permet de comprendre pourquoi une erreur d&#39;exécution du flux s&#39;est produite. Par exemple : un flux peut échouer en raison de propriétés manquantes, de fautes de frappe, de clés API incorrectes, etc.

<div id="review-flow-executions">
  ## Consulter les exécutions de flux
</div>

Pour consulter une entrée d’exécution pour un flux, suivez les étapes ci-dessous :

1. Accédez à **Auth0 Dashboard &gt; Actions &gt; Forms**, puis sélectionnez **Flows**.
2. Sélectionnez un flux dans la liste.
3. Sélectionnez **Executions**.

<Frame>![Dashboard > Actions > Forms > Flows > Execution](/docs/images/cdy7uua7fh8z/448jfjHtgCxfIeDOakuHyd/fba64deda0cf920f399de2190e94bdc9/Flows-execution.png)</Frame>

Chaque entrée d’exécution comporte quatre propriétés :

* **Date d’exécution** : La date à laquelle le flux a été exécuté.
* **ID d’exécution** : L’ID unique de l’exécution du flux.
* **Statut** : L’état actuel de l’exécution du flux. Il existe quatre statuts d’exécution possibles :

  * **En cours** : L’exécution est en cours.
  * **Échec** : L’exécution a échoué en raison d’une erreur interne, par exemple si des paramètres requis sont manquants dans une action du flux.
  * **Terminée** : L’exécution s’est terminée sans erreur interne.
* **View Details** : Ouvre la fenêtre de débogage de l’exécution.

<Frame>![Dashboard > Actions >  Forms > Flows > Executions](/docs/images/cdy7uua7fh8z/1VqWEaVZg5lrXatXe9dbFB/dbcf7edca5aabdfc544f04cb18be4149/Flow-execution-details.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les statuts Terminée et Échec sont des statuts finaux. Lorsqu’une exécution atteint l’un ou l’autre de ces statuts, le flux est entièrement traité.
</Callout>

<div id="debug-flow-executions">
  ## Débogage des exécutions de flux
</div>

Pour faciliter le débogage ou le dépannage d’un flux, vous pouvez examiner ses données d’entrée, de sortie et d’erreur en accédant à **View Details** de l’exécution.

<Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/5aeniTfHXplDyZfwCKBcgK/a2098a476edde76da10a527b0a606e95/Flow-error.png)</Frame>

Pour déboguer une erreur qui interrompt l’exécution du flux, suivez ces étapes :

1. Accédez à **Auth0 Dashboard &gt; Actions &gt; Forms**
2. Repérez le flux associé au nœud d’étape.

   <Frame>![Dashboard > Actions > Forms > Form](/docs/images/cdy7uua7fh8z/4LjULYyo0nhtNowm7J5okz/27ea3f8386bb5015d447c48facf6b90f/flow-error-linked.png)</Frame>
3. Sélectionnez **Modifier le flux.**
4. Sélectionnez **Executions**.
5. Sélectionnez **View Details** de l’exécution échouée.

Dans l’exemple ci-dessous, la vue **Entrée** de `Update user` affiche une valeur nulle pour la propriété `user_id`.

<Frame>![Dashboard > Actions > Forms > Flow > Execution](/docs/images/cdy7uua7fh8z/6QNYKDqG9AIOEzQkP6Fnnr/db885e146179a0490eddd5ad6cb518dc/Flow-execution-failed.png)</Frame>

La vue **Erreur** indique que la validation du schéma de l’action a échoué parce que le champ obligatoire `user_id` est manquant. Le flux renvoie une erreur interne qui interrompt son exécution.

<Frame>![Dashboard > Actions > Forms > Flow > Execution > Error](/docs/images/cdy7uua7fh8z/1kaKihtJtv8VvNAyTkW4cC/6b34bcecc8642d7d97aaf2c8dec51ebc/flow-execution-error-view.png)</Frame>

L’examen de l’action **Update user** révèle une erreur de syntaxe dans le paramètre `User ID` ; la syntaxe correcte est `{{context.user.user_id}}`. Une fois la syntaxe corrigée, testez le flux pour confirmer le résultat. Pour en savoir plus sur la syntaxe des variables, consultez [Variables](/fr-CA/docs/customize/forms/variables).

<Frame>![Dashboard > Actions > Forms > Flows](/docs/images/cdy7uua7fh8z/4YLRVNbzK5LjJEQjFXIJXM/5003833784ce5b0d9f63730ed46ff047/Action-typo-error.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Si un champ est configuré comme transitoire ou si sa valeur est masquée à l’aide d’une [fonction utilitaire](/fr-CA/docs/customize/forms/variables#helper-functions), la valeur n’est pas disponible pour le débogage.

  <Frame>![Dashboard > Actions > Forms > Flows > Execution](/docs/images/cdy7uua7fh8z/4MVtjnVD50cG63ocN5a458/c0337a94d1f4c21f28d06e28ec59121e/masked-information.png)</Frame>
</Callout>
