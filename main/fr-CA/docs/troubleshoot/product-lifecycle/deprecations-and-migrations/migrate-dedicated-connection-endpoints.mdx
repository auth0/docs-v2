---
description: Découvrez comment mettre à jour la configuration de votre connexion Auth0 pour passer du champ `enabled_clients` à des points de terminaison de connexion dédiés.
title: Migrer la gestion des applications activées vers des points de terminaison de connexion dédiés
validatedOn: 2026-04-20
---

Auth0 retire progressivement le champ `enabled_clients` des opérations [Get All Connections](/fr-CA/docs/api/management/v2/connections/get-connections), [Get Connections](/fr-CA/docs/api/management/v2/connections/get-connections-by-id) et [Update a Connection](/fr-CA/docs/api/management/v2/connections/patch-connections-by-id) du Management API. Les clients d’Auth0 peuvent récupérer les applications activées pour une connexion à l’aide du point de terminaison [Obtenir les applications activées pour une connexion](https://auth0.com/docs/api/management/v2/connections/get-connection-clients). Pour mettre à jour les applications déjà associées à une connexion, utilisez le point de terminaison [Mettre à jour les applications activées pour une connexion](https://auth0.com/docs/api/management/v2/connections/patch-clients).

Pour vous préparer à ce changement et vous assurer que vos intégrations continuent de fonctionner sans problème, passez en revue les étapes ci-dessous pour confirmer si vous êtes concerné et migrer vers les nouveaux points de terminaison.

Pour en savoir plus sur le calendrier des migrations, consultez l’article [Deprecations and Migrations](/fr-CA/docs/troubleshoot/product-lifecycle/deprecations-and-migrations).

<div id="verify-affected-tenants">
  ## Vérifier les locataires touchés
</div>

<Warning>
  Les administrateurs de locataires Private Cloud peuvent suivre ces instructions si leur [déploiement Private Cloud](/fr-CA/docs/deploy-monitor/deploy-private-cloud) utilise la version `v202608` ou une version ultérieure. Pour en savoir plus sur les versions de déploiement de Private Cloud, consultez [Comment vérifier le numéro de version et la date de déploiement d’Auth0 Private Cloud](https://support.auth0.com/center/s/article/How-to-check-the-Auth0-private-cloud-s-release-number-and-deployment-date).
</Warning>

Vous pouvez utiliser Auth0 Dashboard pour vérifier si un locataire a accès à la fonctionnalité obsolète et doit être migré.

1. Accédez à [Auth0 Dashboard &gt; Paramètres du locataire &gt; Avancé](https://manage.auth0.com/dashboard/#/tenant/advanced).
2. Sous **Migrations**, vérifiez le **bouton bascule Legacy Management of Connection&#39;s Enabled Clients**.
   * Le bouton bascule est **activé** : votre locataire autorise l’accès au champ obsolète lors de la récupération et de la mise à jour des connexions, et vous devez effectuer la migration avant la date limite.
   * Le bouton bascule est **désactivé** : aucune autre action n’est requise.

<div id="identify-affected-integrations">
  ## Identifier les intégrations touchées
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  * Les locataires Public Cloud ont commencé à recevoir ces avis d’obsolescence au cours de la première moitié de février 2026.
  * Les locataires Private Cloud ont commencé à recevoir ces avis d’obsolescence lorsque leurs déploiements Private Cloud ont atteint la version `v202608`.
</Callout>

Pour déterminer quelles applications et intégrations utilisent les points de terminaison obsolètes, consultez les [journaux](/fr-CA/docs/deploy-monitor/logs#logs) du locataire pour repérer les avis d’obsolescence.

Utilisez la requête suivante pour rechercher les avis liés à cette obsolescence :

```bash lines
type:depnote AND description:Legacy\ Management\ of\ Connection's\ Enabled\ Clients*
```

Examinez le résultat de la requête pour le champ `client_id` afin d’identifier les applications potentiellement touchées.

Certains services et extensions de la plateforme Auth0 peuvent entraîner des avis d’obsolescence dans les journaux du locataire :

* Si le champ `client_id` est vide, l’avis provient d’une extension ou de l’Auth0 Dashboard. Après avoir vérifié l’utilisation des extensions ci-dessous, vous pouvez ignorer les avis sans ID client.
* Si la valeur ne correspond à aucune des applications du locataire, l’avis provient d’un service de la plateforme et peut être ignoré.

De plus, la présence d’un avis d’obsolescence pour une application précise n’indique pas nécessairement que celle-ci est touchée. Par exemple, si l’une de vos applications récupère avec `GET` des informations sur une connexion, y compris le champ obsolète `enabled_clients`, mais ne traite pas le champ renvoyé dans la réponse, l’obsolescence planifiée de ce champ n’a aucune incidence.

<div id="review-extensions-usage">
  ## Vérifier l’utilisation des extensions
</div>

Les extensions suivantes peuvent déclencher des avis d’obsolescence dans les journaux du locataire :

* [User Import/Export](/fr-CA/docs/customize/extensions/user-import-export-extension#user-import-export-extension) : Cette extension ne dépend pas de la fonctionnalité obsolète. Ces avis sont des faux positifs qui se déclenchent lorsque l’extension récupère des informations sur la connexion.
* [Custom Social Connections](https://github.com/auth0/custom-social-connections) : Cette extension dépend de la fonctionnalité obsolète et entraîne des problèmes.
* [Delegated Administration Dashboard](/fr-CA/docs/customize/extensions/delegated-administration-extension/install-delegated-admin-extension) : Les anciennes versions de cette extension peuvent dépendre de la fonctionnalité obsolète et entraîner des problèmes. Mettez-la à jour vers la version 4.8 ou ultérieure dans Auth0 Dashboard.

Les deux premières extensions sont remplacées par des fonctionnalités d’Auth0 Dashboard qui répondent à leurs cas d’utilisation et ne sont plus mises à jour.

* Utilisez la fonctionnalité [Bulk User Import / Export](/fr-CA/docs/manage-users/user-migration/bulk-user-import-export) d’Auth0 Dashboard pour remplacer l’extension User Import/Export.
* Utilisez la fonctionnalité [Social Identity Providers](/fr-CA/docs/authenticate/identity-providers/social-identity-providers#auth0-dashboard) d’Auth0 Dashboard pour remplacer l’extension Custom Social Connections.

<div id="update-your-integrations">
  ## Mettez à jour vos intégrations
</div>

Pour les applications et les intégrations qui utilisent les points de terminaison de connexion concernés pour récupérer (`GET`) ou mettre à jour (`PATCH`) le champ `enabled_clients`, vous devez les modifier afin qu’ils utilisent les points de terminaison dédiés à la gestion des applications :

* **Pour lire :** [Obtenir les applications activées pour une connexion](https://auth0.com/docs/api/management/v2/connections/get-connection-clients).
* **Pour écrire :** [Mettre à jour les applications activées pour une connexion](https://auth0.com/docs/api/management/v2/connections/patch-clients).

<div id="update-sdks-and-tools">
  ## Mettre à jour les SDK et les outils
</div>

Les versions suivantes des SDK et des outils prennent en charge les nouveaux points de terminaison :

* [node-auth0](https://github.com/auth0/node-auth0): `4.25.0+`
* [go-auth0](https://github.com/auth0/go-auth0/): `1.21.0+`
* [auth0-java](https://github.com/auth0/auth0-java): `2.21.0+`
* [auth0-python](https://github.com/auth0/auth0-python): `5.1.0+`
* [auth0.net](https://github.com/auth0/auth0.net): `7.38.0+`
* [Auth0 Deploy CLI](https://auth0.com/docs/deploy-monitor/deploy-cli-tool): `8.27.0+`
* [Auth0 Terraform Provider](https://auth0.com/docs/deploy-monitor/auth0-terraform-provider): `1.29.0+`

Auth0 Deploy CLI / Auth0 Terraform Provider ne nécessitent aucune modification à la façon dont les connexions et les applications associées sont représentées dans leur modèle de données. Vous devez utiliser la version la plus récente offerte pour garantir que ces outils utilisent les points de terminaison dédiés.

Les SDK Ruby et PHP de la Management API n’ont pas encore été mis à jour.

<div id="opt-out-finalize-migration">
  ## Désactivation (finaliser la migration)
</div>

Une fois que vous avez vérifié que les intégrations ne dépendent plus du comportement obsolète, vous devriez configurer explicitement le locataire pour ne plus l’utiliser.

* **Pourquoi désactiver tôt ?** Cela vous permet de contrôler précisément le moment où un locataire donné passe au nouveau comportement. En cas de situation inattendue, vous pouvez rapidement réactiver l’option pour rétablir le comportement d’origine.
* **Comment désactiver :** Accédez à **[Auth0 Dashboard &gt; Paramètres du locataire &gt; Avancé](https://manage.auth0.com/dashboard/#/tenant/advanced).**  Sous Migrations, désactivez l’option **Legacy Management of Connection&#39;s Enabled Clients**.
