---

À compter du 21 juin 2023, Auth0 ne prendra plus en charge les fonctionnalités d’extensibilité Node.js pour .NET et C#.

<div id="affected-workflows">
  ## Flux concernés
</div>

Passez en revue vos fonctionnalités d’extensibilité pour repérer le code qui contient `require(edge)`.

Passez en revue les flux et fonctionnalités suivants qui autorisent le code d’extensibilité afin d’y repérer des références à `edge.js` :

* [Hooks](/fr-CA/docs/customize/hooks)
* [Rules](/fr-CA/docs/customize/rules)
* [Scripts de base de données personnalisés](/fr-CA/docs/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Connexions sociales personnalisées](/fr-CA/docs/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Mises à jour recommandées
</div>

Aucun remplacement officiel n’est actuellement prévu pour cette fonctionnalité.

Une option de migration consiste à remplacer la logique `edge` par un service distinct dans votre propre infrastructure. Une fois ce service créé dans votre infrastructure, vous pouvez utiliser une fonction d’extensibilité pour effectuer une requête HTTP afin de déclencher l’opération requise.

Par exemple, dans un script Custom Database Login qui utilise `edge`, supprimez la fonction qui appelle `edge` et utilisez plutôt un client HTTP générique pour récupérer la réponse.

Une autre option de migration consiste à réécrire la fonctionnalité d’extensibilité en JavaScript plutôt qu’en .NET/C#.

Pour en savoir plus sur la création d’Actions ou la migration vers Actions, [consultez la documentation d’Auth0](/fr-CA/docs/customize/actions).
