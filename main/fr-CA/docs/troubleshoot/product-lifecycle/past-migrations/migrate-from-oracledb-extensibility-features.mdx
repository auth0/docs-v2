title: Migrer depuis les fonctionnalités d’extensibilité oracledb
---

À compter du 21 juin 2023, Auth0 ne prendra plus en charge les connexions aux bases de données Oracle depuis Node.js dans les fonctionnalités d’extensibilité. Auparavant, Auth0 prenait en charge les connexions aux bases de données Oracle au moyen du module Node.js `oracledb`.

<div id="affected-workflows">
  ## Flux de travail concernés
</div>

Passez en revue vos fonctionnalités d’extensibilité pour repérer le code contenant `require(oracledb)` ou  `require('oracledb@4.2.0')`.

Passez en revue les flux ou fonctionnalités suivants qui prennent en charge le code d’extensibilité afin d’y repérer des références à [oracledb](https://www.npmjs.com/package/oracledb) :

* [Hooks](/fr-CA/docs/customize/hooks)
* [Rules](/fr-CA/docs/customize/rules)
* [Scripts de base de données personnalisés](/fr-CA/docs/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Connexions sociales personnalisées](/fr-CA/docs/authenticate/identity-providers/social-identity-providers/oauth2)

<div id="recommended-updates">
  ## Mises à jour recommandées
</div>

Aucun remplacement officiel n’est prévu pour cette fonctionnalité. Auth0 recommande plutôt de déplacer la logique `oracledb` vers un service distinct dans votre propre infrastructure. Une fois ce service créé, vous pouvez utiliser une fonction d’extensibilité pour envoyer une requête HTTP qui déclenchera la logique Oracle Database.

Pour en savoir plus sur la façon d’implémenter le module `oracledb` dans votre propre infrastructure, consultez le [guide de démarrage rapide OracleDB](https://node-oracledb.readthedocs.io/en/latest/user_guide/installation.html#quickstart).

Par exemple, dans un script Login de base de données personnalisée utilisant `oracledb`, supprimez la fonction qui appelle `oracledb` et remplacez-la par un client HTTP générique pour récupérer la réponse.

Pour en savoir plus sur la création d’Actions ou la migration vers Actions, [consultez la documentation d’Auth0](/fr-CA/docs/customize/actions).
