Vous trouverez ci-dessous les codes d’erreur et les solutions possibles aux différentes erreurs pouvant survenir avec l’API de changement de mot de passe en libre-service.

<div id="error-format">
  ## Format des messages d’erreur
</div>

Les messages d’erreur sont renvoyés au format standard :

<div id="error-codes">
  ## Codes d’erreur
</div>

<div id="invalid_request">
  ### invalid_request
</div>

Cette erreur se produit lorsque vous fournissez des paramètres non valides. Les messages d’erreur décrivent le problème, par exemple un paramètre requis ou un format non valide.

<div id="invalid_user_password">
  ### invalid_user_password
</div>

Cette erreur se produit lorsqu’une combinaison invalide de `username`/`email` et de `old_password` est envoyée. Réessayez la requête avec le bon nom d’utilisateur et le `old_password` approprié.

<div id="change_password_error">
  ### change_password_error
</div>

Cette erreur peut être causée par diverses situations liées au <Tooltip tip="fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> sous-jacent. En général, elle se produit lorsque vous utilisez une base de données personnalisée et que vous n’avez pas implémenté le script de changement de mot de passe.
