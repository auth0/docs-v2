Voici quelques solutions à des problèmes courants rencontrés lors de la mise en œuvre du [Contrôle d’accès basé sur les rôles (RBAC)](/fr-CA/docs/manage-users/access-control/rbac) à l’aide de l’ensemble de fonctionnalités Authorization Core.

<div id="role-based-access-control-is-enabled-for-my-api-but-the-scopes-claim-is-not-showing-what-you-say-it-should">
  ## Le Contrôle d’accès basé sur les rôles est activé pour mon API, mais la revendication scopes n’affiche pas ce qu’elle devrait
</div>

* Assurez-vous de ne pas définir `accessToken.scope` dans une Rule.
* N’oubliez pas que toutes les [règles d’autorisation](/fr-CA/docs/manage-users/access-control/rules-for-authorization-policies) configurées s’exécutent une fois que les décisions d’autorisation basées sur le RBAC ont été prises et peuvent donc remplacer le comportement par défaut.
