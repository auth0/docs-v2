Vérifiez les messages d’erreur affichés dans l’un des emplacements suivants :

* Navigateurs et réponses de pages HTML
* Onglets Réseau et Console des outils de développement
* Réponses du <Tooltip tip="Serveur d’autorisation : serveur centralisé qui contribue à définir les limites de l’accès d’un utilisateur. Par exemple, votre serveur d’autorisation peut contrôler les données, les tâches et les fonctionnalités accessibles à un utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Authorization+Server">serveur d’autorisation</Tooltip>
* Erreurs de dépréciation

<div id="check-browser-errors">
  ## Vérifier les erreurs du navigateur
</div>

Un fichier HAR est un journal au format JSON des échanges entre un navigateur Web et un serveur Web. Si l’authentification ne fonctionne pas comme prévu, vous pouvez [générer et analyser des fichiers HAR](/fr-CA/docs/troubleshoot/troubleshooting-tools/generate-and-analyze-har-files) pour repérer les problèmes.

<Warning>
  Avant de partager un fichier HAR avec qui que ce soit (y compris Auth0), assurez-vous de supprimer ou de masquer toutes les données sensibles, telles que :

  * Renseignements confidentiels sur l’utilisateur
  * Renseignements personnels identifiables (PII)
  * Renseignements confidentiels sur l’application

  Pour en savoir plus, consultez les articles suivants sur [Auth0 Community](https://community.auth0.com) :

  * [Nettoyer les traces HTTP](https://community.auth0.com/t/sanitizing-http-traces/119488)
  * [Comment nettoyer automatiquement un fichier de trace HTTP](https://community.auth0.com/t/how-to-sanitize-a-http-trace-file-automatically/120583)
  * [Comment masquer manuellement les renseignements sensibles](https://community.auth0.com/t/how-to-manually-redact-sensitive-information/122554)
  * [Le fichier HAR est trop volumineux pour être téléversé dans le dossier d’assistance](https://community.auth0.com/t/har-file-is-too-large-to-upload-to-the-support-case/122488)
</Warning>

<div id="check-login-screen">
  ## Vérifier l’écran de connexion
</div>

Le widget Lock de connexion affiche des messages d’erreur pour certains types de problèmes, comme un nom d’utilisateur ou un mot de passe incorrect. Vérifiez le lien **Plus d’informations** si vous utilisez la page d’erreur standard d’Auth0.

<div id="check-logs">
  ## Consulter les journaux
</div>

Auth0 stocke des [données de journalisation](/fr-CA/docs/deploy-monitor/logs), notamment les actions des administrateurs dans l’Auth0 Dashboard, les authentifications d’utilisateurs réussies ou échouées, ainsi que les demandes de changement de mot de passe. Vous pouvez consulter les journaux dans l’[Auth0 Dashboard](https://manage.auth0.com/#/logs).

Certains types d’erreurs n’apparaissent pas dans les journaux. Par exemple, si une erreur se produit auprès d’un <Tooltip tip="fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> distant, où l’authentification n’aboutit pas et que l’utilisateur n’est jamais renvoyé vers Auth0, aucune entrée ne sera consignée dans les journaux.

Vous pouvez exporter les journaux Auth0 et soit les stocker vous-même, soit les envoyer automatiquement vers des services de journalisation externes. Cette fonctionnalité peut vous aider à répondre aux exigences de conservation des données ainsi qu’aux exigences d’analyse des journaux. Vous pouvez installer et configurer une extension Auth0 pour exporter automatiquement les journaux vers un autre fournisseur comme Sumo Logic ou Loggly. Pour en savoir plus, consultez nos [solutions de Log Streaming](https://marketplace.auth0.com/features/log-streaming) dans Auth0 Marketplace.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Consultez [la page Monitoring](/fr-CA/docs/deploy-monitor/monitor) pour obtenir plus de détails sur chacun des événements du journal qui peuvent vous aider à résoudre des problèmes.
</Callout>

Vous pouvez également utiliser la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> pour exporter et stocker les journaux. Deux points de terminaison sont disponibles, chacun fournissant des informations légèrement différentes.

<div id="search-all-logs-get_logs-endpoint">
  ### Rechercher tous les journaux : point de terminaison /get_logs
</div>

Le [point de terminaison de recherche des événements du journal](https://auth0.com/docs/api/management/v2#!/Logs/get_logs) récupère les entrées du journal qui correspondent aux critères de recherche que vous avez fournis. Si vous ne fournissez aucun critère de recherche, vous obtiendrez la liste de toutes les entrées disponibles.

Vous pouvez définir des critères de recherche à l’aide du paramètre **q** et récupérer des champs précis à l’aide du paramètre **fields**.

Pour accéder à l’API, vous avez besoin d’un [jeton de la Management API v2](https://auth0.com/docs/api/management/v2/tokens).

Cet exemple de requête récupère tous les journaux de connexions réussies (l’acronyme d’événement pour une connexion réussie est `s`). La liste des champs récupérés pour chaque entrée du journal est : **date**, **description**, **client&#95;id** et **log&#95;id**.

Pour en savoir plus sur les critères de recherche que vous pouvez utiliser et pour obtenir la liste des acronymes d’événement, consultez le [point de terminaison de recherche des événements du journal](https://auth0.com/docs/api/management/v2#!/Logs/get_logs).

<div id="get-a-single-log-entry-get_logs_by_id-endpoint">
  ### Obtenir une seule entrée du journal à partir du point de terminaison /get_logs_by_id
</div>

Le [point de terminaison Obtenir un événement de journal par ID](https://auth0.com/docs/api/management/v2#!/Logs/get_logs_by_id) récupère l’entrée du journal associée à l’ID fourni.

Cet exemple de requête récupère une seule entrée du journal ayant l’ID `90020180129170850881585554625888895190928456277777449010`.

<div id="check-logs-for-deprecation-errors">
  ## Vérifier les journaux pour repérer les erreurs de dépréciation
</div>

Lorsque des fonctionnalités d’Auth0 font l’objet d’une dépréciation, des erreurs ou des avis peuvent apparaître dans les journaux du locataire pour indiquer que vos applications utilisent des fonctionnalités dépréciées. Vous pouvez [rechercher dans les journaux des entrées de dépréciation précises](/fr-CA/docs/troubleshoot/basic-issues/check-deprecation-errors) pouvant indiquer qu’une fonctionnalité a été dépréciée.

<div id="rate-limits-and-other-errors">
  ## Limites de débit et autres erreurs
</div>

Auth0 fournit un code d’erreur unique pour les erreurs signalées lorsque la [limite de débit est dépassée](/fr-CA/docs/troubleshoot/customer-support/operational-policies/rate-limit-policy). Vous devriez configurer une analyse automatique des journaux afin de repérer les erreurs de limite de débit, pour pouvoir intervenir de manière proactive avant que l’activité qui atteint ces limites ne cause trop de problèmes à vos utilisateurs. Auth0 publie également des codes d’erreur pour d’autres types d’erreurs, et il est aussi utile d’analyser les journaux pour repérer les erreurs d’authentification ainsi que celles provenant des appels à l’Auth0 Management API (les codes d’erreur de la Management API sont affichés sous chaque appel dans le [Management API Explorer](https://auth0.com/docs/api/management/v2)).

Notez également qu’appeler la Management API depuis une Rule pour récupérer des informations du profil utilisateur est une cause fréquente d’erreurs de limite de débit, car ces appels d’API peuvent être exécutés à chaque connexion ainsi que lors des vérifications périodiques de session.

<div id="check-actions-real-time-logs-error-console">
  ## Vérifier la console d’erreurs des journaux en temps réel des Actions
</div>

Vous pouvez ajouter des instructions `console.log()` dans les Rules, les Hooks et les scripts de base de données personnalisés. Le résultat de ces instructions est visible dans les journaux en temps réel des Actions. Si vous utilisez les journaux en temps réel des Actions, vous pouvez ouvrir cette console de journaux en accédant à **Auth0 Dashboard &gt; Monitoring &gt; Actions Logs**.
