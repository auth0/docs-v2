description: Découvrez comment rechercher les erreurs de dépréciation dans les journaux
title: Erreurs de dépréciation
---

Lorsque des fonctionnalités d’Auth0 sont obsolètes, des erreurs ou des avis peuvent apparaître dans les journaux du locataire pour indiquer que vos applications utilisent ces fonctionnalités. Ce guide vous aidera à rechercher dans vos journaux les messages liés à la dépréciation, ainsi qu’à comprendre les causes possibles et les solutions pour certains cas.

Il existe deux façons de rechercher les messages d’avertissement indiquant l’utilisation de fonctionnalités obsolètes : l’Auth0 Dashboard ou la <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>. Notez que, dans les deux cas, la [période de conservation des journaux](/fr-CA/docs/deploy-monitor/logs) dépend du niveau d’abonnement de votre compte.

Consultez [Rechercher dans les journaux les erreurs de dépréciation](/fr-CA/docs/troubleshoot/basic-issues/search-logs-for-deprecation-errors) pour en savoir plus.

<div id="deprecation-log-messages">
  ## Messages de journalisation de dépréciation
</div>

<div id="legacy-lock-api">
  ### API Lock ancienne
</div>

**Entrée de journal** : `up-idp-initiated`

**Message d’erreur** : &quot;API Lock ancienne : cette fonctionnalité sera bientôt obsolète. Veuillez consulter notre documentation pour savoir comment migrer votre application.&quot;

**Cause** : Les entrées de journal du locataire concernant l’API Lock ancienne peuvent inclure le référent ainsi que des renseignements sur le SDK utilisé.

**Résolution** : Utilisez ces renseignements pour vérifier si l’une de vos applications utilise des bibliothèques obsolètes.

<div id="ssodata-endpoint">
  ### Point de terminaison SSOdata
</div>

**Entrée du journal**: `ssodata`

**Message d’erreur :** &quot;Point de terminaison SSOdata : Cette fonctionnalité est en voie de devenir obsolète. Veuillez consulter notre documentation pour savoir comment migrer votre application.&quot;

**Cause** : appel direct du point de terminaison `/ssodata` ou utilisation d’anciennes versions du SDK Lock intégré ou du SDK Auth0.js pour appeler une fonction qui appelait le point de terminaison `/ssodata`.

**Résolution** : mettez à jour le SDK Lock ou le SDK Auth0.js vers la version la plus récente.
