---
description: Découvrez comment vérifier votre plateforme afin de résoudre les problèmes
title: Vérifiez la plateforme
---

Assurez-vous d’utiliser des versions à jour de votre pile technologique. Si vous utilisez une ancienne version d’un langage de programmation ou d’une bibliothèque, envisagez de la mettre à jour vers la version actuelle.

* Consultez la [matrice de prise en charge d’Auth0](/fr-CA/docs/troubleshoot/customer-support/product-support-matrix) pour vérifier si les versions du langage ou de la bibliothèque que vous utilisez sont prises en charge.
* Si un problème survient dans un environnement (comme le développement), mais pas dans un autre (comme la production), comparez les versions de la pile technologique dans les deux environnements.
* Sinon, mettez à niveau votre pile technologique et votre navigateur vers les versions les plus récentes, puis testez de nouveau avant de soumettre une demande d’assistance.

<div id="check-the-scope-of-the-issue">
  ## Évaluez l’étendue du problème
</div>

Effectuer des tests dans différents navigateurs, sur différentes plateformes, à partir de différents lieux et avec différents utilisateurs peut aider à cerner la source d’un problème lorsqu’il survient. Effectuez les tests suivants et consultez les [journaux de votre Auth0 Dashboard](https://manage.auth0.com/#/logs) après chaque test pour obtenir plus d’information.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Consultez la [page Monitoring](/fr-CA/docs/deploy-monitor/monitor) pour en savoir plus sur chacun des événements du journal qui peuvent vous aider à résoudre les problèmes.
</Callout>

<div id="test-with-different-browsers">
  ### Tester avec différents navigateurs
</div>

* Le problème se produit-il dans tous les navigateurs ou seulement dans certains?
* Faites un test dans une nouvelle session de navigation privée ou incognito (sans données de session antérieures).
* Mettez le navigateur à jour vers la version la plus récente, puis testez de nouveau.
* Désactivez les extensions du navigateur et testez de nouveau pour voir si l’une d’elles contribue au problème.

<div id="test-with-different-users">
  ### Tester avec différents utilisateurs
</div>

* Le problème se produit-il pour tous les utilisateurs ou seulement pour certains?
* Si c’est le cas pour certains, y a-t-il un point commun? Par exemple, l’emplacement, la connexion ou le rôle?

<div id="test-with-different-connections-if-possible">
  ### Testez avec différentes connexions (si possible)
</div>

* Le problème se produit-il avec toutes les connexions ou uniquement avec l’une d’elles ?

<div id="test-with-different-client-applications-if-you-have-more-than-one">
  ### Testez avec différentes applications clientes (si vous en avez plus d’une)
</div>

* Le problème se produit-il avec toutes les applications ou seulement avec l’une d’elles?

<div id="test-across-different-environments">
  ### Testez dans différents environnements
</div>

* Testez votre ou vos applications dans les environnements de développement, de préproduction et de production.
* Testez différents locataires Auth0 pour voir s’il y a des différences.

<div id="check-if-the-issue-is-consistent-or-intermittent">
  ## Vérifiez si le problème est constant ou intermittent
</div>

* Si le problème est constant, notez les étapes pour le reproduire de manière fiable.
* Si le problème est intermittent, notez tout facteur déclencheur ou toute tendance pouvant l’expliquer.
* S’agit-il d’un problème lié à une nouvelle configuration, ou est-ce que cela fonctionnait auparavant et que c’est maintenant défaillant ?

  * Si le problème vient tout juste d’apparaître, quels changements ont été apportés ?
  * Le fait d’annuler ces changements corrige-t-il le problème ?

<div id="issues-that-affect-only-one-or-a-few-users">
  ## Problèmes qui touchent seulement un ou quelques utilisateurs
</div>

* [Vérifiez le profil de l&#39;utilisateur](/fr-CA/docs/troubleshoot/authentication-issues/check-user-profiles), le navigateur ou l&#39;appareil pour repérer tout problème.
* Vérifiez si le problème se produit dans tous les navigateurs pour les utilisateurs touchés (ce qui indique un problème de données) ou seulement dans certains types de navigateurs (ce qui indique un problème propre au navigateur).
* Vérifiez que JavaScript et les témoins sont activés dans le navigateur.
* Vérifiez que la touche Verr. Maj. est désactivée.
* Si l&#39;utilisateur utilise un appareil mobile, vérifiez s&#39;il y a un logiciel qui pourrait nuire à l&#39;authentification et/ou à l&#39;autorisation (par exemple, si un logiciel requis n&#39;est pas en cours d&#39;exécution).
* Vérifiez si l&#39;utilisateur peut accéder à certaines URL clés de l&#39;application, comme l&#39;URL d&#39;<Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> pour l&#39;<Tooltip tip="Authentification unique (SSO) : service qui, après qu’un utilisateur ouvre une session dans une application, l’ouvre automatiquement dans d’autres applications." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Single+Sign-on">authentification unique</Tooltip> (SSO) (ce qui indique un problème de connectivité réseau).

<div id="issues-that-occur-after-go-live-worked-before-then-stopped-working">
  ## Problèmes qui surviennent après la mise en production (fonctionnait auparavant, puis a cessé de fonctionner)
</div>

* Vérifiez s’il y a eu des modifications récentes à votre application ou aux API que vous appelez.
* Vérifiez s’il y a eu des changements réseau récents (répartiteur de charge, pare-feu, modifications de la configuration du proxy).
* Vérifiez s’il y a eu des changements récents de l’infrastructure (p. ex. les identifiants des serveurs LDAP ou DB?)
* Vérifiez s’il y a eu des correctifs ou des mises à jour des applications, de l’infrastructure ou de la pile technologique.
* Vérifiez que tous vos serveurs exécutent NTP et que l’heure est synchronisée correctement
* Vérifiez si l’un de vos certificats a expiré.
* Vérifiez auprès des responsables des fournisseurs d’identité distants concernés si quelque chose a changé.

  * Vérifiez la connectivité réseau avec les fournisseurs d’identité distants concernés.
* Vérifiez les notifications dans <Tooltip tip="Auth0 Dashboard : produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>/migrations — y a-t-il des changements qui vous auraient échappé?
* Vérifiez le journal des modifications d’Auth0 — y a-t-il des changements de configuration récents liés à votre problème?
* Vérifiez si un composant de la pile technologique a été mis à jour ou corrigé récemment.
