title: Vérifier les connexions
---

Une transaction d’authentification comporte souvent plusieurs éléments. Auth0 fournit des méthodes qui vous permettent de tester des parties précises de la transaction afin de vous aider à repérer la source possible du problème.

La plupart des connexions à un <Tooltip tip="fournisseur d’identité (IdP) : Service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> comportent un bouton **TRY** pour vérifier si la connexion fonctionne. Si ce test échoue, vous pouvez déboguer la connexion sans faire intervenir le reste de l’application. Si la connexion fonctionne, vous pouvez alors commencer à déboguer l’application.

1. Utilisez le bouton **TRY** sur une connexion pour tester uniquement la connexion.
2. Si le test avec le bouton **TRY** échoue, vous savez que le problème ne vient pas de votre application, mais plutôt de la configuration de la connexion ou de son fournisseur.
3. Vérifiez la réponse du test avec le bouton **TRY** pour voir si elle contient un message d’erreur utile ou d’autres renseignements.
4. Essayez de vous connecter au même service par une autre voie.
5. Si le même problème se produit, vous saurez qu’il s’agit d’un problème lié au compte.
6. Si un test avec le bouton **TRY** échoue pour une connexion de base de données personnalisée, cela est souvent causé par un problème dans les scripts de base de données personnalisés. L’ajout d’instructions console.log dans ces scripts et la consultation de la sortie dans le Console Log peuvent aider à les déboguer.

<div id="verify-external-service-dependencies">
  ## Vérifier les dépendances des services externes
</div>

1. Si votre authentification utilise des services externes, comme des fournisseurs d’identité sociaux, et qu’elle cesse soudainement de fonctionner, [vérifiez l’état des services externes](/fr-CA/docs/deploy-monitor/monitor/check-external-services-status).
2. Si une connexion ne fonctionne pas, même avec le bouton **TRY**, vérifiez sa configuration.

  Consultez [la page Monitoring](/fr-CA/docs/deploy-monitor/monitor) pour obtenir plus de détails sur chacun des événements du journal pouvant vous aider à résoudre les problèmes.
</Callout>
