title: Vérifier les erreurs de dépréciation
---

Lorsqu’une fonctionnalité Auth0 fait l’objet d’une dépréciation, des erreurs ou des avis peuvent apparaître dans les journaux du locataire pour indiquer que vos applications utilisent encore des fonctionnalités obsolètes. Ce guide vous aidera à rechercher dans vos journaux les messages liés à la dépréciation, ainsi qu’à comprendre les causes possibles et les solutions pour certains cas.

Il existe deux façons de rechercher les messages d’avertissement signalant l’utilisation de fonctionnalités obsolètes : l’Auth0 Dashboard ou la <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>. Notez que, dans les deux cas, la [période de conservation des journaux](/fr-CA/docs/deploy-monitor/logs/log-data-retention) est déterminée par le niveau d’abonnement de votre compte.

<div id="use-the-dashboard">
  ## Utiliser l’Auth0 Dashboard
</div>

Si votre application utilise une fonctionnalité obsolète, un message **Deprecation Notice** s’affichera dans la section **Logs** de l’Auth0 Dashboard. Les avis de dépréciation ne sont affichés qu’une fois par heure (la première fois que l’erreur se produit durant cette heure), plutôt que pour chaque transaction d’authentification impliquant la fonctionnalité obsolète.

1. Accédez à [Dashboard &gt; Monitoring &gt; Logs](https://manage.auth0.com/#/logs) et recherchez les messages liés à la dépréciation en saisissant `type:depnote` dans le champ de recherche. Une liste des messages d’avertissement liés à la dépréciation apparaîtra dans vos journaux, le cas échéant. Le champ **Description** fournit des renseignements sur la fonctionnalité obsolète utilisée.
2. Cliquez sur le lien dans la colonne **Type** de chaque élément pour afficher des renseignements supplémentaires, comme l’ID client qui identifie l’application cliente utilisant la fonctionnalité obsolète.
3. Cliquez sur les entrées de journal et sélectionnez l’onglet **Context Data** pour afficher les détails de l’élément.

<div id="use-the-management-api">
  ## Utiliser la Management API
</div>

Vous pouvez aussi utiliser la Management API pour rechercher ce type de messages dans les journaux en cherchant `type:"depnote"`.

1. Accédez à la [Management API](https://auth0.com/docs/api/management/v2). Si ce n’est pas déjà fait, [obtenez un jeton d’API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production).
2. Dans le panneau de gauche, accédez à [Logs &gt; Search log events](https://auth0.com/docs/api/management/v2/#!/Logs/get_logs), puis faites défiler jusqu’à **Parameters**.
3. Dans le champ **q**, entrez : `type:"depnote"`.
4. Cliquez sur le bouton **TRY**. Le champ **ID client** dans les résultats indiquera quelle application de votre locataire utilise la fonctionnalité obsolète.
