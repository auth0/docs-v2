title: Générer et analyser des fichiers HAR
---

Un fichier [HAR](https://en.wikipedia.org/wiki/.har) (HTTP Archive) est un journal au format JSON des interactions entre un navigateur web et un serveur web.

Un fichier HAR montre la séquence des redirections qui se produisent pendant une transaction de connexion, ce qui en fait une excellente ressource pour déboguer les problèmes d’authentification. Joindre un fichier HAR à vos [demandes d’assistance](https://support.auth0.com) peut aider à accélérer le dépannage.

<Warning>
  Avant de partager un fichier HAR avec qui que ce soit (y compris Auth0), assurez-vous de supprimer ou de masquer toutes les données sensibles, comme :

  * Renseignements confidentiels sur l’utilisateur
  * Renseignements permettant d’identifier une personne (PII)
  * Renseignements confidentiels sur l’application

  Pour en savoir plus, consultez les articles suivants sur [Auth0 Community](https://community.auth0.com) :

  * [Nettoyer les traces HTTP](https://community.auth0.com/t/sanitizing-http-traces/119488)
  * [Comment nettoyer automatiquement un fichier de trace HTTP](https://community.auth0.com/t/how-to-sanitize-a-http-trace-file-automatically/120583)
  * [Comment caviarder manuellement des renseignements sensibles](https://community.auth0.com/t/how-to-manually-redact-sensitive-information/122554)
  * [Le fichier HAR est trop volumineux pour être téléversé dans le dossier d’assistance](https://community.auth0.com/t/har-file-is-too-large-to-upload-to-the-support-case/122488)
</Warning>

<div id="generate-har-files">
  ## Générer des fichiers HAR
</div>

Vous pouvez générer un fichier HAR dans la plupart des navigateurs Web à l’aide des outils de développement ou d’inspection intégrés.

<div id="google-chrome">
  ### Google Chrome
</div>

1. Fermez toutes les fenêtres de **navigation privée** dans Google Chrome.
2. Ouvrez une **nouvelle fenêtre de navigation privée** dans Google Chrome.
3. Accédez à **Affichage &gt; Développeur &gt; Outils de développement**.
4. Dans le volet Outils de développement, sélectionnez l’onglet **Réseau**.
5. Cochez la case **Preserve Log** pour enregistrer toutes les interactions.
6. Accédez à la page et effectuez les étapes qui déclenchent le problème.
7. Sélectionnez l’onglet **Réseau**.
8. Cliquez sur la flèche vers le bas pour exporter le fichier HAR.
9. Enregistrez le fichier HAR.

<div id="safari">
  ### Safari
</div>

1. Assurez-vous que la case **Afficher le menu Développement dans la barre des menus** est cochée sous **Safari &gt; Préférences &gt; Avancées**.
2. Choisissez **Fichier &gt; Ouvrir une nouvelle fenêtre privée**.
3. Choisissez **Afficher l’inspecteur Web** dans le menu **Développement**, puis accédez à l’onglet **Réseau**.
4. À droite de **Tout**, près du coin supérieur gauche de l’inspecteur Web, cliquez sur le **bouton Filtre** et choisissez **Conserver le journal**.
5. Accédez à la page Web où le problème se produit.
6. Choisissez **Développement &gt; Afficher l’inspecteur Web**. La fenêtre de l’inspecteur Web s’affiche.
7. Effectuez sur la page les étapes qui provoquent le problème.
8. Sélectionnez l’onglet **Réseau**.
9. Cliquez sur **Exporter** dans le coin supérieur droit du volet.
10. Enregistrez le fichier HAR.

<div id="firefox">
  ### Firefox
</div>

1. Fermez toutes les fenêtres **de navigation privée** dans Firefox.
2. Ouvrez une **nouvelle fenêtre de navigation privée** dans Firefox.
3. Accédez à **Outils &gt; Développement web &gt; Réseau** (**Ctrl + Maj + E**).
4. Cliquez sur l’onglet **Réseau** et choisissez **Conserver les journaux**.
5. Rendez-vous sur la page et effectuez les étapes qui déclenchent le problème.
6. Dans l’onglet **Réseau**, faites un clic droit, puis sélectionnez **Enregistrer tout sous HAR**.
7. Enregistrez le fichier HAR.

<div id="microsoft-edge">
  ### Microsoft Edge
</div>

1. Fermez toutes les fenêtres **InPrivate** de Microsoft Edge.
2. Ouvrez une **nouvelle fenêtre InPrivate** (**Ctrl + Shift + N**).
3. Accédez à **Paramètres et plus** (l’icône **...**), puis choisissez **Plus d’outils &gt; Outils de développement**.
4. Lancez une session de profilage dans l’onglet **Réseau** de la barre d’outils des outils de développement.
5. Accédez à la page et suivez les étapes qui reproduisent le problème.
6. Sélectionnez Export as HAR (**Ctrl + S**) et enregistrez le fichier HAR.

<div id="internet-explorer">
  ### Internet Explorer
</div>

1. Fermez toutes les fenêtres **InPrivate** dans Internet Explorer.
2. Ouvrez une **nouvelle fenêtre InPrivate** dans Internet Explorer (**Ctrl + Shift + P**).
3. Accédez à **Tools &gt; F12 Developer Options &gt; Réseau**.
4. Assurez-vous que l’option **Clear entries on navigate** est désactivée.
5. Accédez à la page et suivez les étapes qui provoquent le problème.
6. Sélectionnez l’onglet **Réseau**, puis **Export as HAR** (**Ctrl + S**).
7. Enregistrez le fichier HAR.

<div id="analyze-har-files">
  ## Analyser les fichiers HAR
</div>

Auth0 n’utilise pas et ne recommande pas l’utilisation d’applications, d’outils ou de logiciels tiers pour analyser les fichiers HAR.

Une fois le fichier HAR assaini, vous pouvez l’ouvrir dans un éditeur de texte ou un navigateur pour consulter les événements enregistrés.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Consultez [la page Monitoring](/fr-CA/docs/deploy-monitor/monitor) pour plus de détails sur chacun des événements du journal qui peuvent vous aider à résoudre des problèmes.
</Callout>
