Auth0 est une plateforme d’identité qui permet de gérer l’accès à vos applications.

Si vous débutez en gestion des identités et des accès (IAM), découvrez les notions de base et planifiez la solution qui convient le mieux à votre technologie et à vos besoins. Si vous connaissez déjà l’IAM, vous pouvez passer directement à la création.

***

<div id="start-building">
  ## Commencez à développer
</div>

Pour démarrer rapidement, choisissez votre type d’application et suivez un guide de démarrage rapide étape par étape.

<Columns cols={2}>
  <Card title="Application native ou mobile" href="/fr-CA/docs/quickstart/native">
    Application mobile ou de bureau qui s’exécute nativement sur un appareil, comme iOS ou Android.
  </Card>

  <Card title="Application monopage" href="/fr-CA/docs/quickstart/spa">
    Application Web JavaScript qui s’exécute dans le navigateur, comme AngularJS + Node.js ou React.
  </Card>

  <Card title="Application Web classique" href="/fr-CA/docs/quickstart/webapp">
    Application Web traditionnelle qui s’exécute sur le serveur, comme Express.js ou ASP.NET.
  </Card>

  <Card title="Backend/API" href="/fr-CA/docs/quickstart/backend">
    API ou service protégé par Auth0, comme une API Express.js ou une API ASP.NET.
  </Card>

  <Card title="Agent d’IA" href="https://auth0.com/ai/docs">
    Agent d’IA permettant d’appeler des API externes au nom de l’utilisateur et d’utiliser des mécanismes de supervision humaine, comme LangChain ou LlamaIndex.
  </Card>
</Columns>
