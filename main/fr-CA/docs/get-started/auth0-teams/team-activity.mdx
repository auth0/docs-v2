title: Activité de l’équipe
---

Activité de l’équipe permet aux propriétaires d’équipe de consulter les journaux d’audit associés aux activités des membres effectuées sur leur compte Auth0.

Le rapport Activité de l’équipe se trouve dans la section **Reports** du Teams Dashboard.

Activité de l’équipe fournit un aperçu du membre qui a généré l’événement, du type d’événement et de la date à laquelle il a été généré.

Vous pouvez sélectionner **View Details** pour obtenir plus d’informations, comme la description de l’événement, son statut, l’adresse IP de l’utilisateur qui l’a déclenché, les détails de l’agent utilisateur, etc.

<div id="event-types">
  ## Types d’événements
</div>

Les types d’événements liés à l’activité de l’équipe sont décrits ci-dessous.

| Type d’événement      | Description                                                                                |
| --------------------- | ------------------------------------------------------------------------------------------ |
| Invitation d’équipe   | Invitation d’un membre de l’équipe créée<br />Invitation d’un membre de l’équipe supprimée |
| Membre de l’équipe    | Membre de l’équipe supprimé<br />Rôle du membre de l’équipe mis à jour                     |
| Stratégie de sécurité | Stratégie de sécurité mise à jour<br />Connexions sociales mises à jour                    |
| Paramètre d’équipe    | Paramètres de l’équipe mis à jour                                                          |

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La prise en charge de l’authentification des utilisateurs à Auth0 Dashboard et à Teams Dashboard, ainsi que d’autres événements, sera offerte ultérieurement.
</Callout>

<div id="event-schema">
  ## Schéma de l’événement
</div>

Le schéma de l’événement d’activité de l’équipe est décrit ci-dessous.

```json lines
{  
    "id": "string", 
    "team_id": "string",
    "user_id": "string",
    "type": "string",
    "description": "string",
    "status": "string",
    "details": "object",
    "ip": "string",
    "hostname": "string",
    "user_agent": "string",
    "version": "string",
    "created_at": "date",
    "team_slug": "string",
    "user": "object"
}
```

| Nom           | Type     | Description                                                                              |
| ------------- | -------- | ---------------------------------------------------------------------------------------- |
| `id`          | string   | Identifiant unique du journal des événements.                                            |
| `team_id`     | string   | Identifiant unique de l’équipe.                                                          |
| `user_id`     | string   | Identifiant unique de l’utilisateur qui a généré l’événement.                            |
| `type`        | string   | Type d’événement.                                                                        |
| `description` | string   | Résumé du type d’événement.                                                              |
| `status`      | string   | Statut de l’événement. Les valeurs possibles sont `Success` ou `Failure`.                |
| `details`     | object   | Détails de l’événement consigné.                                                         |
| `ip`          | string   | Adresse IP de l’utilisateur qui a généré le journal des événements.                      |
| `hostname`    | string   | Nom de domaine complet (FQDN) de l’application Auth0 sur laquelle l’événement a eu lieu. |
| `user_agent`  | string   | Navigateur Web ainsi que le type et la version du système d’exploitation de l’appareil.  |
| `version`     | string   | Version du schéma selon laquelle le journal d’activité a été généré.                     |
| `created_at`  | datetime | Date et heure auxquelles le journal des événements a été généré.                         |
| `team_slug`   | string   | Identifiant textuel unique de l’équipe.                                                  |
| `user`        | object   | Contient le nom, l’URL de l’avatar et le statut d’appartenance à l’équipe (Boolean).     |

<div id="audit-log-retention">
  ## Conservation des journaux d’audit
</div>

La durée de conservation des journaux d’audit de l’activité de l’équipe varie selon le forfait. Consultez notre page [Tarification](https://auth0.com/pricing) pour en savoir plus.
