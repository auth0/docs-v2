Auth0 offre aux utilisateurs du locataire une fonction de recherche plus intuitive pour les Applications et les API enregistrées, avec la possibilité de créer des expressions de filtre pour les champs interrogeables dans Auth0 Dashboard. Au lieu de parcourir des listes paginées contenant des centaines de ressources, les utilisateurs de votre locataire peuvent effectuer des recherches sur des champs tels que :

* Le nom de votre application (Travel0-native)
* Les métadonnées de l’application ou de l’utilisateur (permissions)
* Le type d’application (SPA)
* Une API par nom
* Une API par ID

Auth0 Dashboard affiche en temps réel les résultats correspondants parmi toutes vos ressources enregistrées.

<div id="how-it-works">
  ## Fonctionnement
</div>

Dans les sections [Applications](https://manage.auth0.com/#/applications) et [APIs](https://manage.auth0.com/#/apis) de l’Auth0 Dashboard, la barre de recherche utilise une interface de filtres guidés et la recherche booléenne. Elle fonctionne comme un menu déroulant qui répertorie les champs pouvant être filtrés. Lorsque vous sélectionnez un champ et saisissez une valeur, celle-ci s’applique sous forme d’étiquette amovible au-dessus des résultats. Vous pouvez combiner jusqu’à cinq filtres à la fois.

Chaque filtre ajouté affine davantage les résultats. Pour préciser les résultats, appliquez la logique `AND/OR` entre différents champs. Les filtres sont reflétés dans l’URL sous forme de paramètres de requête. Vous pouvez copier l’URL à tout moment pour la partager avec un collègue ou l’ajouter à vos favoris afin de revenir plus tard à la même vue filtrée.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Exemple booléen**

  Si vous filtrez par **Type d’application** `Native` OR `SPA`, et par **Application propriétaire** `true`, les résultats n’affichent que les applications de première partie de type Native ou SPA. Les applications Web régulières ou tierces sont exclues.
</Callout>

<div id="search-behavior-by-field-type">
  ### Comportement de la recherche selon le type de champ
</div>

Dans Auth0 Dashboard, les résultats de recherche dépendent du type de champ.

* **Champs String** (`name`, `identifier`) : la correspondance partielle est prise en charge. La saisie de `payments` permet de trouver « Payments-application », « Corp Payments Service » et des résultats semblables — l’ordre des mots n’a pas d’importance. Pour exclure les résultats correspondant à un terme, faites-le précéder de `NOT`, `-` ou `No` — par exemple, la saisie de `NOT test` dans le champ `name` renvoie toutes les ressources dont le nom ne contient pas « test ». Les recherches avec caractère générique exigent au moins trois caractères ; si vous en saisissez moins, le message « Minimum 3 characters » s’affiche et la requête n’est pas envoyée.

* **Champs de correspondance exacte**

Vous devez saisir la valeur complète de l’ID. Les ID partiels ne renvoient aucun résultat. Utilisez ces champs lorsque vous disposez d’un ID client précis provenant d’un log ou d’un ticket d’assistance.

* **Champs de métadonnées**

Vous pouvez rechercher dans les métadonnées par clé et valeur, ou par clé seulement :

* **Clé et valeur** : saisissez une clé et une valeur pour trouver les ressources où cette clé exacte correspond à cette valeur. Par exemple, la clé `environment` et la valeur `production` renvoient uniquement les ressources où `app_metadata.environment` est égal à `production`. La correspondance par préfixe est prise en charge pour les valeurs — la saisie de `prod` correspond à `production`, `prod-eu` et `prod-us`.
  * **Clé seulement** : saisissez une clé et appuyez sur **Enter** sans entrer de valeur. Cela renvoie toutes les ressources pour lesquelles cette clé de métadonnées est définie, quelle que soit sa valeur. C’est utile pour trouver les ressources qui utilisent une clé précise.

Vous pouvez aussi rechercher des clés de métadonnées imbriquées jusqu’à deux niveaux de profondeur à l’aide de la notation par point. Par exemple, la clé `tier.bronze` et la valeur `true` correspondent aux ressources où `app_metadata.tier.bronze` est égal à `true`.

Pour exclure les résultats correspondant à une paire clé-valeur précise, faites précéder la valeur de `NOT`. Par exemple, la clé `color` et la valeur `NOT orange` renvoient toutes les ressources où `color` est défini à n’importe quelle valeur autre que `orange`. Les clés doivent correspondre exactement. Les correspondances de type contient et se termine par ne sont pas prises en charge pour les métadonnées. Les valeurs sont indexées jusqu’à 64 caractères ; les valeurs plus longues ne sont pas recherchables.

* **Champs d’énumération**

Sélectionnez un ou plusieurs types d’application dans une liste prédéfinie. Plusieurs sélections dans le même filtre utilisent une logique OR. Par exemple, sélectionner `Native` et `SPA` renvoie toutes les applications Native et SPA.

* **Champs Boolean**

Recherchez les applications propriétaires en sélectionnant `true` ou `false`. Utilisez cette option pour distinguer les applications de première partie (celles autorisées à ignorer l’écran Consent) des applications tierces.

<div id="searchable-fields">
  ### Champs interrogeables
</div>

<div id="applications">
  #### Applications
</div>

| **Champ**                                    | **Description**                                                   | **Type de correspondance**           |
| -------------------------------------------- | ----------------------------------------------------------------- | ------------------------------------ |
| **Nom**                                      | Le nom de l’application                                           | Correspondance partielle             |
| **ID client** (`client_id`)                  | L’ID client unique                                                | Correspondance exacte                |
| **ID client externe** (`external_client_id`) | Un identifiant externe pour l’application                         | Correspondance exacte                |
| **Métadonnées**                              | Métadonnées clé-valeur personnalisées stockées dans l’application | Correspondance exacte ou par préfixe |
| **Type d’application**                       | Native, SPA, application Web standard ou machine à machine        | Sélection d’énumération              |
| **Application propriétaire**                 | Indique si l’application est propriétaire ou tierce               | Boolean                              |

<div id="apis">
  #### APIs
</div>

| **Champ**       | **Description**                                                          | **Type de correspondance** |
| --------------- | ------------------------------------------------------------------------ | -------------------------- |
| **Nom**         | Le nom de l’API                                                          | Correspondance partielle   |
| **Identifiant** | L’identifiant d’audience unique (par exemple, `https://api.example.com`) | Correspondance partielle   |
| **Id**          | L’identifiant interne du système                                         | Correspondance exacte      |

<div id="limitations">
  ### Limitations
</div>

* Vous pouvez appliquer un maximum de cinq filtres par recherche. La barre de recherche empêche l’ajout d’un sixième filtre.
* Les résultats de recherche peuvent ne pas refléter les changements apportés au cours des dernières secondes, car l’index n’est mis à jour qu’après un certain délai.
* En conditions normales, les résultats sont renvoyés en moins de deux secondes. Si une requête expire, réduisez le nombre de filtres ou utilisez un terme de recherche plus précis.

<div id="example-queries">
  ## Exemples de requêtes
</div>

<div id="find-an-application-by-client-id">
  ### Trouver une application par ID client
</div>

Utilisez cette option lorsque vous avez un `client_id` provenant d’un journal d’erreurs ou d’un ticket d’assistance et que vous devez retrouver rapidement l’application.

1. Accédez à **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Sélectionnez la barre de recherche et choisissez **ID client**.
3. Collez la valeur complète de `client_id`. L’application correspondante s’affiche immédiatement.

<div id="find-applications-using-metadata">
  ### Trouver des applications à l’aide des métadonnées
</div>

Si vos applications stockent des métadonnées personnalisées — comme des étiquettes d’environnement, des niveaux de forfait ou des ID externes — vous pouvez effectuer une recherche à partir de ces valeurs.

1. Accédez à **[Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications)**.
2. Sélectionnez la barre de recherche et choisissez **métadonnée**.
3. Saisissez la clé de métadonnée, par exemple `environment`.
4. Saisissez la valeur, par exemple `production`. Les applications dont `metadata.environment` correspond à `production` s’affichent dans les résultats. Pour faire une recherche par préfixe, saisissez une valeur partielle, comme `prod`.

<div id="troubleshooting">
  ## Dépannage
</div>

**Aucun résultat trouvé**

Vérifiez que les valeurs de filtre sont correctement orthographiées. Pour les champs String, essayez un terme plus court ou plus général. Pour les champs de métadonnées, confirmez le nom exact de la clé — les clés sont sensibles à la casse et doivent correspondre exactement. La correspondance de type « contains » n’est pas prise en charge pour les métadonnées; utilisez plutôt un préfixe. Pour les champs à correspondance exacte comme l’ID d’application, assurez-vous de coller la valeur complète de l’ID.

**Les résultats semblent obsolètes**

Les résultats de recherche ne deviennent cohérents qu’après un certain délai et peuvent ne pas refléter les changements apportés au cours des dernières secondes. Si vous avez récemment créé ou mis à jour une ressource, attendez quelques secondes, puis réessayez la recherche.

**La recherche a expiré**

Réduisez le nombre de filtres actifs ou utilisez un terme de recherche plus précis. Si le problème persiste, communiquez avec l’[assistance Auth0](https://support.auth0.com).
