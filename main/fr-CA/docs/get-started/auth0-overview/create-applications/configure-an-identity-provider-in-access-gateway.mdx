title: Configurer un fournisseur d’identité dans Access Gateway
---

Pour intégrer Auth0 à une application machine à machine (M2M), vous devez d’abord enregistrer votre application auprès d’Auth0 à l’aide du <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application**.

2. Entrez un nom descriptif pour votre application, sélectionnez **Machine to Machine Applications**, puis cliquez sur **Create**.

3. Sélectionnez la Management API pour autoriser la communication M2M, car c’est ainsi qu’OAG interagit avec Auth0.

4. Cliquez sur **Authorize**.

5. Chaque application M2M qui accède à une API doit se voir accorder un ensemble d’autorisations (ou scopes) par l’utilisateur authentifié. Sélectionnez les **Permissions** que vous souhaitez inclure dans le <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+token">jeton d’accès</Tooltip> de votre application. OAG requiert les autorisations suivantes pour l’**application** et les **utilisateurs :**

* `read:client, write:client, update:client, delete:client`
* `read:users`

6. Cliquez sur **Authorize**.

7. Sélectionnez l’onglet **Credentials** de votre application.

8. Cliquez sur **<Tooltip tip="ID client : valeur d’identification attribuée à votre ressource enregistrée par Auth0." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Client+ID">Client ID</Tooltip>** et copiez-le dans un éditeur de texte.

9. Cliquez sur **<Tooltip tip="Secret client : secret utilisé par une application pour s’authentifier auprès du serveur d’autorisation; il ne doit être connu que de l’application et du serveur d’autorisation et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Client+Secret">Client Secret</Tooltip>** et copiez-le dans un éditeur de texte.

10. Poursuivez avec la procédure [Configurer un fournisseur d’identité dans Access Gateway](/docs/Configure an Identity Provider in Access Gateway).
