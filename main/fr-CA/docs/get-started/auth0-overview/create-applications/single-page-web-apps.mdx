Pour intégrer Auth0 à une application web monopage, vous devez d’abord enregistrer votre application dans l’Auth0 Dashboard.

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et cliquez sur **Create Application**.
2. Entrez un nom descriptif pour votre application, sélectionnez **Single-Page Web Applications**, puis cliquez sur **Create**.

   Une fois la nouvelle application créée, vous pouvez consulter la page **paramètres de l’application**, qui comprend les onglets suivants :

   | Onglet des paramètres | Description                                                                                                                                                                                                                                                                                                                                                                                                                                       |
   | --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Quick Start**       | Affiche toute la documentation disponible pour votre type d’application.                                                                                                                                                                                                                                                                                                                                                                          |
   | **Settings**          | Affiche tous les paramètres disponibles pour votre application. Par défaut, la plupart des paramètres sont créés pour vous. Pour en savoir plus, consultez [paramètres de l’application](/fr-CA/docs/get-started/applications/application-settings).                                                                                                                                                                                                           |
   | **Credentials**       | Affiche la méthode d’authentification de l’application et les identifiants configurés. Pour en savoir plus, consultez [Application Credentials](/fr-CA/docs/secure/application-credentials)                                                                                                                                                                                                                                                             |
   | **Add-ons**           | Vous permet d’activer les modules complémentaires associés à une application. Ils sont généralement utilisés pour les intégrations SAML ou WS-Fed, qui sont rarement nécessaires pour les SPA. Pour en savoir plus, consultez [Enable SAML2 Web App Addon](/fr-CA/docs/authenticate/protocols/saml/saml-sso-integrations/enable-saml2-web-app-addon) et [Configure WS-Fed Applications](/fr-CA/docs/get-started/applications/update-application-connections). |
   | **Connections**       | Vous permet d’activer des connexions pour votre application. Les connexions sont des sources d’utilisateurs; elles peuvent être activées et partagées entre plusieurs applications. Pour en savoir plus, consultez [Connections](/fr-CA/docs/authenticate/identity-providers).                                                                                                                                                                          |
   | **Organizations**     | Vous permet d’authentifier des utilisateurs dans les organisations auxquelles ils appartiennent. Pour en savoir plus, consultez [Organizations](/fr-CA/docs/manage-users/organizations).                                                                                                                                                                                                                                                                |
   | **Login Experience**  | Vous permet de configurer l’expérience de connexion des utilisateurs au sein de votre organisation. Pour en savoir plus, consultez [Understand how Auth0 Organizations Work](/fr-CA/docs/manage-users/organizations/organizations-overview#login-experience).                                                                                                                                                                                           |

   Pour en savoir plus, consultez [paramètres de l’application](/fr-CA/docs/get-started/applications/application-settings).

<div id="next-steps">
  ## Étapes suivantes
</div>

Une fois votre application enregistrée, passez en revue et effectuez les étapes de configuration suivantes :

* Configurez au moins une connexion (comme une connexion de base de données ou une connexion sociale) et activez-la pour votre application. Sans connexion activée, les utilisateurs ne peuvent pas s’authentifier.
* Modifiez le code de votre application pour utiliser l’application que vous avez enregistrée dans Auth0. Consultez nos [Auth0 Quickstarts](/fr-CA/docs/quickstarts), où vous trouverez des instructions détaillées pour implémenter la connexion et la déconnexion, gérer vos sessions utilisateur, récupérer et afficher les informations du profil utilisateur, et plus encore.
* Utilisez les [API Auth0](/fr-CA/docs/api).

  * L’[Authentication API](https://auth0.com/docs/api/authentication) gère toutes les principales fonctions liées à l’identité (par exemple, la connexion, la déconnexion et l’obtention du profil utilisateur). La plupart des utilisateurs utilisent cette API au moyen de nos Quickstarts, du [Auth0 SPA SDK](/fr-CA/docs/libraries/auth0-spa-js) ou du [widget Lock](/fr-CA/docs/libraries/lock). Pour les applications plus anciennes, consultez la [bibliothèque Auth0.js](/fr-CA/docs/libraries/auth0js). Toutefois, si vous créez manuellement toute votre interface utilisateur d’authentification, vous devrez interagir directement avec cette API.
  * La [Management API](https://auth0.com/docs/api/management/v2) vous permet d’automatiser diverses tâches également accessibles dans Auth0 Dashboard (par exemple : créer des utilisateurs, définir les types d’octroi de l’application).
