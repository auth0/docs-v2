Vous pouvez créer une application manuellement ou l’importer depuis l’URL d’un [document de métadonnées d’ID client (CIMD)](/fr-CA/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd) dans Auth0 Dashboard.

<div id="create-manually">
  ## Créer manuellement
</div>

Lorsque vous créez manuellement une application, Auth0 lui attribue un <Tooltip tip="ID client : chaîne alphanumérique qui sert d’identifiant unique pour une application." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=client+ID">ID client</Tooltip>, une chaîne alphanumérique qui sert d’identifiant unique à l’application. Vous utiliserez cet ID dans le code de votre application lorsque vous appellerez les API Auth0. Ce processus d’enregistrement de l’application s’appelle l’enregistrement manuel du client.

Un autre élément d’information important est le <Tooltip tip="Secret client : secret utilisé par une application pour s’authentifier auprès du serveur d’autorisation; il ne doit être connu que de l’application et du serveur d’autorisation et doit être suffisamment aléatoire pour ne pas pouvoir être deviné." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=client+secret">Secret client</Tooltip>. Il doit demeurer confidentiel en tout temps. Si quelqu’un obtient l’accès au secret client de votre application, cette personne peut usurper l’identité de votre application et accéder à des ressources protégées. Il est important de sélectionner le bon type d’application pour aider Auth0 à détecter certains risques de sécurité.

<div id="import-from-url">
  ## Importer à partir d’une URL
</div>

Vous pouvez enregistrer des applications en important, à partir d’une URL, un document de métadonnées d’ID client (CIMD) hébergé à l’externe. Le CIMD est un fichier JSON contenant les métadonnées de l’application, hébergé sur un domaine HTTPS sécurisé contrôlé par l’application. Ce processus est appelé enregistrement manuel par CIMD. Pour en savoir plus, consultez [Enregistrer des applications avec CIMD](/fr-CA/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd).

<div id="manual-client-registration-vs-manual-cimd-registration">
  ## Enregistrement manuel de l’application vs. enregistrement manuel CIMD
</div>

Le tableau suivant compare l’enregistrement manuel de l’application et l’enregistrement manuel CIMD :

| Fonctionnalité              | Enregistrement manuel de l’application        | Enregistrement manuel CIMD                               |
| --------------------------- | --------------------------------------------- | -------------------------------------------------------- |
| **ID client**               | Généré par Auth0                              | URL HTTPS (p. ex., `https://yourdomain.com/client.json`) |
| **Authentification**        | Secret client ou certificat                   | Clés asymétriques uniquement (sans secret)               |
| **Vérification d’identité** | Aucune                                        | Propriété du domaine par hébergement HTTPS               |
| **Gestion des métadonnées** | Mise à jour via Auth0 Dashboard/API           | CIMD fait foi                                            |
| **Rotation des clés**       | Manuelle, une seule clé                       | Plusieurs clés prises en charge dans `jwks_uri`          |
| **Type d’application**      | Tous les types                                | Applications tierces uniquement                          |
| **Convient le mieux à**     | Applications propriétaires, services internes | Applications MCP, intégrations tierces                   |

<div id="create-application-in-auth0">
  ## Créer une application dans Auth0
</div>

Pour créer une application dans Auth0 :

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications).
2. Sélectionnez **Create Application**.
3. Sélectionnez **Create it manually** ou **Importer à partir d’une URL**.
   * **Importer à partir d’une URL** : Suivez les instructions dans [Enregistrer des applications avec CIMD](/fr-CA/docs/get-started/auth0-overview/create-applications/register-applications-with-cimd).
   * **Create it manually** : Donnez un nom explicite à votre nouvelle application. Ensuite, sélectionnez un type d’application :
     * **application native** : Ces applications incluent les applications mobiles, de bureau ou hybrides qui s’exécutent directement sur un appareil (p. ex., iOS, Android).
     * **application monopage** : Ces applications incluent les applications JavaScript qui exécutent la majeure partie de leur logique d’interface utilisateur dans un navigateur Web et communiquent principalement avec un serveur Web au moyen d’API (p. ex., AngularJS + Node.js ou React).
     * **application Web régulière** : Ces applications sont des applications Web traditionnelles qui exécutent la majeure partie de leur logique sur le serveur (p. ex., Express.js, ASP.NET).
     * **Machine to Machine Application** : Ces applications incluent des applications non interactives, comme des outils en ligne de commande, des démons, des appareils IoT ou des services exécutés sur votre back-end.
4. Sélectionnez **Create**.
