<div id="application-architecture-scenarios">
  ## Scénarios d’architecture d’application
</div>

Ces scénarios décrivent les différents types d’architectures que votre application peut utiliser et comment Auth0 peut vous aider. Leur objectif est de vous guider tout au long du processus de mise en œuvre, du début à la fin.

| Lire...                                                                                                                                | Pour apprendre à...                                                                                                                                                                     |
| -------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Organisations multiples](/fr-CA/docs/get-started/architecture-scenarios/multiple-organization-architecture)                                 | Configurer une architecture d’application qui utilise soit des organisations avec un seul fournisseur d’identité, soit des organisations avec plusieurs fournisseurs d’identité.        |
| [Authentification unique pour les applications Web traditionnelles](/fr-CA/docs/get-started/architecture-scenarios/sso-for-regular-web-apps) | Configurer et mettre en œuvre une application Web qui utilise le SSO pour les employés utilisant Active Directory et les contractuels utilisant un référentiel d’utilisateurs distinct. |
| [Application serveur + API](/fr-CA/docs/get-started/architecture-scenarios/server-application-api)                                           | Configurer une API de feuille de temps à l’aide d’une tâche cron pour enregistrer des entrées de feuille de temps.                                                                      |
| [Application monopage + API](/fr-CA/docs/get-started/architecture-scenarios/spa-api)                                                         | Configurer une API de feuille de temps à l’aide d’une application monopage pour enregistrer des entrées de feuille de temps.                                                            |
| [Application mobile + API](/fr-CA/docs/get-started/architecture-scenarios/mobile-api)                                                        | Configurer une API de feuille de temps à l’aide d’une application mobile pour enregistrer des entrées de feuille de temps dans une base de données centralisée.                         |
