Les éléments de l’image de marque associés à une organisation sont extrêmement précieux, car ils offrent aux utilisateurs un environnement qu’ils connaissent et auquel ils font confiance. L’utilisation d’éléments de marque reconnus renforce également la confiance des utilisateurs quant au fait que les renseignements qu’ils fournissent (par exemple, des identifiants) seront traités de façon sûre et sécuritaire. Dans nos scénarios d’architecture, nous proposons des conseils généraux sur plusieurs sujets liés à l’[image de marque B2B](/fr-CA/docs/get-started/architecture-scenarios/business-to-business/branding), que nous vous recommandons de consulter en complément des conseils fournis ici.

<div id="universal-login">
  ## Universal Login
</div>

L’image de marque d’un environnement à organisations multiples peut être plus complexe que dans un environnement standard, où vous n’avez à gérer qu’un seul ensemble d’éléments d’image de marque. Les [organisations Auth0](/fr-CA/docs/manage-users/organizations/organizations-overview) tirent toutefois parti de l’[expérience Universal Login](/fr-CA/docs/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience), qui offre plusieurs [modèles de page](/fr-CA/docs/customize/login-pages/universal-login/customize-templates) pour simplifier la personnalisation de l’image de marque. La configuration d’une organisation Auth0 vous permet également de [sélectionner un jeu de couleurs de base pour l’image de marque](/fr-CA/docs/manage-users/organizations/configure-organizations/create-organizations).

<div id="login-page">
  ### Page de connexion
</div>

Lorsqu’une [connexion de base de données ou connexion de base de données personnalisée](/fr-CA/docs/authenticate/database-connections) est utilisée, la page de connexion sert à recueillir les identifiants du premier facteur d’un utilisateur. Vous pouvez configurer la page de connexion à l’aide d’un ensemble de [variables de modèle](/fr-CA/docs/manage-users/organizations/create-first-organization) disponibles pour faciliter la personnalisation par organisation.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Vous pouvez configurer la page de connexion pour afficher les connexions d’entreprise et sociales pour l’authentification de premier facteur; l’image de marque de la page de connexion peut donc aussi être pertinente dans ces cas.
</Callout>

<div id="password-reset-page">
  ### Page de réinitialisation du mot de passe
</div>

Comme la [page de connexion](#login-page), la page de réinitialisation du mot de passe peut être configurée dans les scénarios qui utilisent une connexion de base de données ou une connexion de base de données personnalisée. Dans les scénarios faisant intervenir une connexion d’entreprise ou une connexion sociale, les changements de mot de passe sont gérés à l’extérieur d’Auth0 par le <Tooltip tip="Fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP) en amont; une page de réinitialisation du mot de passe Auth0 n’est donc jamais affichée.

<div id="multi-factor-authentication-pages">
  ### Pages d’authentification multifacteur
</div>

<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur supplémentaire en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Multi-Factor+Authentication">Authentification multifacteur</Tooltip> (MFA) est prise en charge dans Auth0 pour toutes les méthodes utilisées pour obtenir les identifiants du premier facteur d’un utilisateur (c.-à-d. une connexion de base de données ou une connexion de base de données personnalisée, une connexion d’entreprise ou une connexion sociale). Si vous utilisez l’authentification multifacteur, vous voudrez probablement appliquer votre image de marque aux pages de l’expérience <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, pour vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Universal+Login">Universal Login</Tooltip> associées à la MFA, comme vous le feriez pour l’image de marque de la [page de connexion](#login-page).

<div id="email-templates">
  ## Modèles de courriel
</div>

Comme pour les modèles de page Universal Login, un ensemble de [variables des modèles de courriel](/fr-CA/docs/customize/email/email-templates/supported-liquid-syntax#general-variables) est disponible pour faciliter la personnalisation des courriels pour chaque organisation. Les modèles de courriel à personnaliser varient selon la façon dont les utilisateurs s’authentifient (autrement dit, au moyen d’une connexion de base de données, d’une connexion de base de données personnalisée, d’une connexion d’entreprise ou d’une connexion sociale). Pour en savoir plus sur la personnalisation des modèles de courriel lorsque vous utilisez la fonctionnalité organisations Auth0 d’Auth0, consultez [Créez votre première organisation : Modèles de courriel](/fr-CA/docs/manage-users/organizations/create-first-organization).

<div id="custom-domains">
  ## Domaines personnalisés
</div>

La fonctionnalité Organisations d’Auth0 ne prend en charge qu’un seul [Domaine personnalisé](/fr-CA/docs/customize/custom-domains), défini au niveau de l’Auth0 Tenant. Par conséquent, nous vous recommandons d’utiliser un nom de <Tooltip tip="Domaine personnalisé : domaine tiers avec un nom spécialisé ou personnalisé." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Custom+Domain">Domaine personnalisé</Tooltip> qui ne soit pas propre à une organisation. Si vous devez absolument mettre en place une convention de nommage de domaine par organisation, plusieurs Auth0 Tenants devront alors être déployés en production. Comme l’utilisation de plusieurs locataires a des répercussions sur l’<Tooltip tip="Authentification unique (SSO) : service qui, après la connexion d’un utilisateur à une application, le connecte automatiquement à d’autres applications." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Single+Sign-On">authentification unique</Tooltip> (SSO), la gestion du profil utilisateur, etc., vous devriez bien réfléchir avant d’opter pour cette approche.
