Dans le scénario des organisations à <Tooltip tip="fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP) unique, chaque organisation qui utilise la fonctionnalité [Organisations Auth0](/fr-CA/docs/manage-users/organizations/organizations-overview) utilise exactement une connexion Auth0. Cette fonctionnalité permet à chaque application de sélectionner l’organisation appropriée définie dans le locataire Auth0, qui authentifiera directement les identifiants de l’utilisateur ou transmettra la demande d’authentification à l’IdP approprié pour qu’il la prenne en charge.

Dans ce scénario, nous continuerons d’utiliser l’exemple [Travel0 Corporate Booking](/fr-CA/docs/get-started/architecture-scenarios/multiple-organization-architecture). Dans cet exemple, l’application de réservation d’entreprise correspond à votre application et Travel0 à votre entreprise.

<Frame>![Scénarios d’architecture - Multilocataire - Diagramme - Utilisateurs isolés, applications partagées, organisations A et B](/docs/images/cdy7uua7fh8z/1YBaMaMozamiTv8hXmNB2D/21e9270a59021a12209bdcb4345147cb/isolated-users_shared-apps_org-ab.png)</Frame>

<Warning>
  La fonctionnalité Organisations Auth0 prend actuellement en charge les applications qui utilisent les protocoles [SAML](/fr-CA/docs/authenticate/protocols/saml/saml-configuration), [OpenID Connect](/fr-CA/docs/authenticate/protocols/openid-connect-protocol) et/ou le framework [OAuth 2](/fr-CA/docs/authenticate/protocols/oauth). Les applications qui nécessitent [WS-Fed](/fr-CA/docs/authenticate/protocols/ws-fed-protocol) ne sont pas prises en charge.
</Warning>

Peu importe la façon dont les identifiants sont traités--qu’ils soient stockés dans une connexion de base de données (comme avec Hoekstra &amp; Associates), validés au moyen d’une connexion d’entreprise (comme avec MetaHexa Bank) ou d’une connexion sociale--votre application se comporte de la même manière. L’utilisation de la fonctionnalité [Organisations](/fr-CA/docs/manage-users/organizations/organizations-overview) vous permet de configurer et de contrôler l’expérience.
