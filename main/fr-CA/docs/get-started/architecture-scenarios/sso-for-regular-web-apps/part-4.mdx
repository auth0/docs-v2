Dans ce tutoriel, nous avons présenté un scénario simple : une application Web classique, hébergée dans le cloud, qui utilise Auth0 pour l’authentification tout en s’appuyant sur l’annuaire Active Directory existant. Nous avons vu ce que <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker les renseignements de connexion." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) offre et pourquoi il convenait mieux à ce cas d’utilisation, comment fonctionne le flux d’authentification, ce qu’est un <Tooltip tip="OpenID : norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans recueillir ni stocker les renseignements de connexion." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=ID+Token">ID Token</Tooltip> ainsi que la façon de le valider et de le manipuler, comment configurer les applications et les connexions dans <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+dashboard">Auth0 Dashboard</Tooltip>, comment implémenter la connexion et la déconnexion des utilisateurs à l’aide de Lock, et comment fonctionnent la gestion des sessions et le contrôle d’accès.

Nous avons commencé par décrire le cas d’utilisation et les exigences, puis nous avons expliqué comment répondre à chaque exigence et le raisonnement derrière chacun des choix effectués.

Nous avons utilisé ASP .NET Core pour l’implémentation de l’exemple d’application Web, mais nous espérons qu’après avoir parcouru ce document, vous serez en mesure de créer une telle application Web à l’aide du framework de votre choix.
