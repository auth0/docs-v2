Accélérez votre processus de développement en connectant directement des agents d’IA et des LLM à la base de connaissances d’Auth0. Obtenez des réponses contextualisées, des exemples de code et des références d’API avec des liens vers la documentation officielle.

Pour aider vos outils d’IA à indexer l’information dont vous avez besoin et à y effectuer des recherches, Auth0 fournit :

* LLM context files (`llms.txt`) : des fichiers Markdown structurés pour l’ingestion en lot.
* Auth0 Docs MCP Server : un serveur MCP hébergé permettant aux agents d’IA d’interroger dynamiquement la base de connaissances.

<div id="llm-context-files">
  ## LLM context files
</div>

Utilisez les fichiers de contexte LLM pour aider les LLM à comprendre et à assimiler efficacement l’ensemble du site Auth0.

| Nom             | URL                                    | Description                                                                      |
| --------------- | -------------------------------------- | -------------------------------------------------------------------------------- |
| `llms.txt`      | `https://auth0.com/llms.txt`           | Résumés concis de tous les articles ; idéal pour obtenir rapidement du contexte. |
| `llms-full.txt` | `https://auth0.com/docs/llms-full.txt` | Contenu complet de tous les articles ; idéal pour une indexation approfondie.    |

<div id="auth0-docs-mcp-server">
  ## Auth0 Docs MCP Server
</div>

Connectez vos outils d’IA à Auth0 Docs MCP Server pour accéder en temps réel à la documentation Auth0 la plus récente. Auth0 Docs MCP Server expose l’outil `SearchAuth0Docs`, qui permet aux agents d’IA d’effectuer des recherches dans la base de connaissances d’Auth0 et de renvoyer du contenu contextuel avec des liens pertinents vers la documentation produit.

Auth0 Docs MCP Server expose les métadonnées d’endpoint suivantes :

```
Server Name: Auth0 Docs
Server URL: https://auth0.com/docs/mcp
Transport Type: HTTP
```

Découvrez comment connecter vos outils d’IA à Auth0 Docs MCP Server en suivant nos guides d’intégration.

<Tabs>
  <Tab title="ChatGPT">
    Pour connecter ChatGPT à Auth0 Docs MCP Server :

    1. Accédez à [Settings &gt; Connectors](https://chatgpt.com/#settings/Connectors).
    2. Sélectionnez **Create**, puis fournissez les métadonnées de votre serveur MCP :
       * Nom du connecteur : Auth0 Docs MCP Server
       * Description : Indexe et effectue des recherches dans la base de connaissances d’Auth0
       * URL du connecteur : `https://auth0.com/docs/mcp`
    3. Sélectionnez **Create**.

    Si tout se passe bien, Auth0 Docs MCP Server devrait apparaître comme serveur MCP installé. Pour en savoir plus, consultez la documentation [Connect from ChatGPT](https://developers.openai.com/apps-sdk/deploy/connect-chatgpt/) d’OpenAI.
  </Tab>

  <Tab title="Claude">
    Pour connecter Claude à Auth0 Docs MCP Server :

    1. Accédez à [Settings &gt; Connectors](https://claude.ai/settings/connectors).
    2. Sélectionnez **Add custom connector**.
    3. Saisissez les éléments suivants pour ajouter Auth0 Docs MCP Server :
       * Nom : Auth0 Docs MCP Server
       * URL : `https://auth0.com/docs/mcp`
    4. Sélectionnez **Add**.

    Si l’ajout réussit, Auth0 Docs MCP Server devrait apparaître comme connecteur personnalisé. Pour en savoir plus, consultez la documentation du Model Context Protocol sur [Connect to remote MCP Servers](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server).
  </Tab>

  <Tab title="Claude Code">
    Pour connecter Claude Code à Auth0 Docs MCP Server :

    Dans votre terminal, exécutez :

    ```bash lines
    claude mcp add --transport http auth0-docs-mcp-server https://auth0.com/docs/mcp
    ```

    Pour vérifier, exécutez :

    ```bash lines
    claude mcp list
    ```

    Si l’opération réussit, Auth0 Docs MCP Server devrait apparaître comme serveur MCP installé. Pour en savoir plus, consultez la documentation de Claude sur l’[installation des serveurs MCP](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="Codex">
    Pour connecter Codex à Auth0 Docs MCP Server :

    Dans votre terminal, exécutez :

    ```bash lines
    codex mcp add auth0-docs-mcp-server --url https://auth0.com/docs/mcp
    ```

    Pour vérifier, exécutez :

    ```bash lines
    codex mcp list
    ```

    Si l’opération réussit, Auth0 Docs MCP Server devrait apparaître comme un serveur MCP installé. Pour en savoir plus, consultez la documentation de Codex sur le [Model Context Protocol](https://developers.openai.com/codex/mcp).
  </Tab>

  <Tab title="Cursor">
    Pour connecter Cursor au serveur Auth0 Docs MCP Server :

    1. Utilisez Command + Shift + P (Ctrl + Shift + P sous Windows) pour ouvrir la palette de commandes, puis recherchez **Open MCP settings**.
    2. Sélectionnez **New MCP Server** pour modifier votre `mcp.json`.
    3. Ajoutez le serveur Auth0 Docs MCP Server :

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "url": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    Si l’installation réussit, le Auth0 Docs MCP Server devrait s’afficher comme serveur MCP installé. Pour en savoir plus, consultez la documentation de Cursor sur [Installing MCP servers](https://cursor.com/docs/context/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="VS Code">
    Pour connecter VS Code au serveur MCP Auth0 Docs :

    1. Utilisez Command + Shift + P (Ctrl + Shift + P sous Windows) pour ouvrir la palette de commandes et rechercher **MCP: Add Server**.
    2. Sélectionnez **HTTP** comme type de serveur MCP.
    3. Saisissez l’URL du serveur MCP (`https://auth0.com/docs/mcp`) et définissez l’ID du serveur sur `auth0-docs-mcp-server`.
    4. Sélectionnez **Global** ou **Workspace** pour rendre le serveur MCP accessible dans tous vos espaces de travail ou seulement dans l’espace de travail actuel.
    5. Appuyez sur **Entrée** pour terminer la configuration. Vérifiez que le serveur MCP Auth0 Docs apparaît sous **Installed MCP Servers** dans **Extensions**.

    Pour en savoir plus, consultez [Add and manage MCP servers in VS Code](https://code.visualstudio.com/docs/copilot/chat/mcp-servers).
  </Tab>

  <Tab title="Windsurf">
    Pour connecter Windsurf au Auth0 Docs MCP Server :

    1. Utilisez Command + Shift + P (Ctrl + Shift + P sous Windows) pour ouvrir les paramètres de Windsurf et rechercher **MCP**.
    2. Sélectionnez **View Raw Config** pour modifier votre fichier `mcp_config.json`.
    3. Ajoutez le Auth0 Docs MCP Server :

    ```json lines
    {
      "mcpServers": {
        "auth0-docs-mcp-server": {
          "serverUrl": "https://auth0.com/docs/mcp"
        }
      }
    }
    ```

    4. Enregistrez et sélectionnez **Actualiser** pour charger la nouvelle configuration du MCP.

    Pour vérifier, demandez `What MCP servers do you have installed?` dans le clavardage de Cascade. Cascade devrait indiquer que Auth0 Docs MCP Server est installé comme serveur MCP. Pour en savoir plus, consultez la documentation de Windsurf sur le [Model Context Protocol](https://docs.windsurf.com/plugins/cascade/mcp#model-context-protocol-mcp).
  </Tab>
</Tabs>

* Ajoutez à mon agent d’IA un appel d’outil pouvant accéder au Google Calendar de l’utilisateur et répondre à des questions sur son horaire et ses réunions. Mettez à jour la configuration actuelle de mon serveur pour exiger OAuth 2.1 pour toutes les connexions entrantes et vérifiez qu’elle utilise le profil de compatibilité du paramètre de ressource pour les jetons d’accès.

Si vous relevez des écarts pouvant entraîner une fuite du contexte utilisateur, suggérez un correctif et fournissez le code mis à jour.

<div id="troubleshooting">
  ### Dépannage
</div>

* Aide-moi à résoudre l’erreur `refresh token doesn’t exist` lors de l’utilisation de l’échange de jeton d’actualisation. Utilise les instructions de configuration de Token Vault comme contexte.
* Le bouton de déconnexion n’efface pas la session côté Auth0. Recherche dans la documentation la structure correcte de l’URL de Federated Logout et mets à jour mon gestionnaire de déconnexion pour l’utiliser.
* Mon flux de connexion échoue avec une `401 Unauthorized error`. Utilise Auth0 Docs MCP Server pour rechercher les causes courantes de cette erreur avec le SDK `auth0-node` et vérifie s’il y a des incohérences dans mon fichier `.env` et l’initialisation de mon application.
