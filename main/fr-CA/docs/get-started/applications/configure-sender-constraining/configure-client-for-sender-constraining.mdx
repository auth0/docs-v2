<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La contrainte d’expéditeur des jetons au moyen de Demonstrating Proof-of-Possession (DPoP) est actuellement en accès anticipé. Pour demander l’accès à cette fonctionnalité, communiquez avec votre représentant Auth0.
</Callout>

La [contrainte d’expéditeur](/fr-CA/docs/secure/sender-constraining) est un mécanisme de sécurité <Tooltip tip="" cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip> et <Tooltip tip="" cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=OpenID">OpenID</Tooltip> Connect (OIDC) qui lie de façon cryptographique les jetons d’accès et les <Tooltip tip="" cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=refresh+tokens">jetons d’actualisation</Tooltip> à l’application précise qui les a obtenus, ce qui empêche le vol et l’utilisation abusive de jetons.

Auth0 prend en charge la contrainte d’expéditeur mTLS et Demonstrating Proof-of-Possession (DPoP). Si vous activez la contrainte d’expéditeur pour une application, vous devez aussi l’imposer pour le <Tooltip tip="" cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=resource+server">serveur de ressources</Tooltip> auquel vous envoyez des appels d’API.

Pour configurer la contrainte d’expéditeur dans Auth0, vous devez :

* [Configurer la contrainte d’expéditeur pour une application](#configure-sender-constraining-for-a-client-application).
* [Configurer la contrainte d’expéditeur pour un serveur de ressources](#configure-sender-constraining-for-a-resource-server).

<div id="how-it-works">
  ## Fonctionnement
</div>

<Tooltip tip="" cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Access+tokens">Les jetons d’accès</Tooltip> sont assortis d’une contrainte d’expéditeur dans Auth0 selon la façon dont vous configurez cette contrainte pour votre application et votre serveur de ressources :

1. **Audience demandée :** Dans une demande de jeton, le fait que l’audience demandée soit uniquement `/userinfo`, qu’elle soit destinée uniquement au point de terminaison [`/userinfo`](https://auth0.com/docs/api/authentication/user-profile/get-user-info), ou qu’il s’agisse d’une API personnalisée pouvant inclure `/userinfo` lorsque vous demandez aussi le scope openid, a une incidence sur le fait que les jetons d’accès soient ou non assortis d’une contrainte d’expéditeur.
2. **Application :** Le fait de définir la contrainte d’expéditeur sur `required` pour l’application.
3. **Serveur de ressources :** Le fait que vous configuriez la contrainte d’expéditeur pour le serveur de ressources :

   1. `none` : Vous n’avez pas configuré de contrainte d’expéditeur pour le serveur de ressources.
   2. `allowed` : Vous avez configuré une contrainte d’expéditeur pour le serveur de ressources en définissant une méthode de contrainte d’expéditeur.
   3. `required` : Vous avez configuré la contrainte d’expéditeur comme obligatoire pour le serveur de ressources, ce qui signifie que les jetons d’accès doivent être assortis d’une contrainte d’expéditeur pour une application. Nécessite une méthode de contrainte d’expéditeur.

* **Preuve de possession :** Le fait que l’application ait envoyé une assertion de preuve de possession dans la demande de jeton :

  * contrainte d’expéditeur mTLS : La preuve de possession est démontrée par la présentation réussie, par l’application, d’une clé privée précise (associée à un certificat client) pendant l’établissement de la connexion TLS.
  * DPoP : La preuve de possession est obtenue lorsque l’application crée un JWT de preuve DPoP en le signant de façon cryptographique avec sa clé privée et inclut le JWT de preuve `DPoP` dans l’en-tête HTTP DPoP de chaque requête où le jeton d’accès associé est utilisé.

<div id="configure-sender-constraining-for-a-client-application">
  ## Configurer la contrainte d’expéditeur pour une application
</div>

Lorsque vous exigez une contrainte d’expéditeur pour une application, les jetons d’accès sont limités à cette application. Auth0 vérifie les requêtes pour s’assurer que seule l’application qui a demandé le jeton peut l’utiliser pour accéder à la ressource associée.

Une fois qu’une application est configurée pour exiger une contrainte d’expéditeur, vous pouvez définir la méthode de contrainte d’expéditeur, soit mTLS, soit DPoP, lorsque vous configurez votre serveur de ressources.

Vous pouvez configurer la contrainte d’expéditeur pour une application à l’aide de l’<Tooltip tip="" cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="" cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>.

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications). Sélectionnez l’application que vous souhaitez configurer.
    2. Sous **Settings**, faites défiler jusqu’à **Token Sender-Constraining**.
    3. Activez **Require Sender Constraining**. Désactivez cette option pour supprimer l’exigence de Sender Constraining pour l’application.

    <Frame>![](/docs/images/cdy7uua7fh8z/3BoGW9NEYKPBkfk2tPBL7l/0f884ac3f64f17b23fd6a0e9436bb472/Screenshot_2024-07-23_at_1.30.19_PM.png)</Frame>
  </Tab>

  <Tab title="Management API">
    Pour configurer la contrainte d’expéditeur pour une application, utilisez la [Management API](https://auth0.com/docs/api/management/v2).

    Pour exiger Sender Constraining pour une application, envoyez une requête PATCH pour [mettre à jour les paramètres de l’application](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id). Définissez le paramètre `require_proof_of_possession` à `true`.

    ```json lines
    "require_proof_of_possession": true
    ```

    Pour supprimer l’exigence de Sender Constraining, définissez le paramètre `require_proof_of_possession` à `false`.

    ```json lines
    "require_proof_of_possession": false
    ```
  </Tab>
</Tabs>

<div id="configure-sender-constraining-for-a-resource-server">
  ## Configurer la contrainte d’expéditeur pour un serveur de ressources
</div>

Les jetons d’accès émis par Auth0 peuvent être liés à l’expéditeur (c.-à-d. l’application cliente) qui doit accéder aux API auprès d’un serveur de ressources.

Vous pouvez configurer la contrainte d’expéditeur pour un serveur de ressources dans Auth0 Dashboard ou avec la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Pour activer Token Binding ou la contrainte d’expéditeur, configurez les **paramètres de l’API** de votre API.

    1. Accédez à [Auth0 Dashboard &gt; Applications &gt; APIs](https://manage.auth0.com/#/apis).
    2. Sélectionnez l’API que vous voulez configurer.
    3. Sous l’onglet **Settings**, repérez la section **Token Sender-Constraining**.
    4. Configurez les éléments suivants :

       1. Méthode de contrainte d’expéditeur :

          1. **Aucune :** N’activez aucune méthode de contrainte d’expéditeur pour votre serveur de ressources.
          2. **mTLS** : Activez mTLS comme méthode de contrainte d’expéditeur pour votre serveur de ressources.
          3. **DPoP:** Activez DPoP comme méthode de contrainte d’expéditeur pour votre serveur de ressources.

    B. Activez **Require Token Sender Constraining**. Tous les jetons d’accès émis à une application pour cette API seront liés à cette application.

    <Frame>![Auth0 Dashboard > APIs > Settings > Liaison de jeton](/docs/images/cdy7uua7fh8z/3Dv98iZosdpJMcZXfXyurn/823fa368ab874218fbb737aacac9b262/Screenshot_2025-07-28_at_3.59.01_PM.png)</Frame>
  </Tab>

  <Tab title="Management API">
    Pour activer la contrainte d’expéditeur avec la Management API, envoyez une requête PATCH pour [mettre à jour le serveur de ressources](https://auth0.com/docs/api/management/v2/resource-servers/patch-resource-servers-by-id). Définissez les paramètres de l’objet `proof_of_possession` comme suit :

    | Paramètre   | Description                                                                                                                                                                                                                  |
    | ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `mechanism` | Définit la méthode de contrainte d’expéditeur : `none`, `mtls` ou `dpop`.                                                                                                                                                      |
    | `required`  | Lorsqu’il est défini à `true`, tous les jetons d’accès émis à une application pour cette API seront liés à cette application. Lorsqu’il est défini à `false`, la contrainte d’expéditeur n’est pas requise pour l’application. |

    L’exemple de code suivant montre un corps de requête qui configure un serveur de ressources pour la contrainte d’expéditeur mTLS :

    ```json lines
    "proof_of_possession": {
        "mechanism": "mtls",
        "required": true
      }
    ```
  </Tab>
</Tabs>
