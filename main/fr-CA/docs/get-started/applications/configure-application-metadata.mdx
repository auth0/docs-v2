description: Découvrez comment configurer, mettre à jour et supprimer les métadonnées d’application (client_metadata et clientMetadata) dans Applications > Advanced Settings de l’Auth0 Dashboard.
title: Configurer Application Metadata
---

Les métadonnées d’application sont facultatives et se composent de clés et de valeurs personnalisables (255 caractères max. chacune) que vous pouvez définir pour chaque application. Les métadonnées sont exposées dans l’objet `Client` sous la forme `client_metadata`, et dans Rules sous la forme `context.clientMetadata`. Vous pouvez, par exemple, y stocker l’URL de la page d’accueil de l’application (un champ qu’Auth0 ne fournit pas par défaut dans les paramètres de l’application).

Les métadonnées du client sont stockées dans les propriétés de l’application (client). Pour en savoir plus sur les types de données, les noms de champ et les limites de stockage, consultez [Metadata Field Names and Data Types](/fr-CA/docs/manage-users/user-accounts/metadata/metadata-fields-data).

<Card title="Où stocker les secrets client">
  L’endroit où stocker le secret dépend de sa portée :

  * S’agit-il d’un seul secret par application ? Dans ce cas, `client_metadata` est un bon choix.
  * S’agit-il du même secret pour l’ensemble du système (c.-à-d. pour toutes les applications ou pour plusieurs d’entre elles) ? Dans ce cas, les valeurs de configuration de la Rule pourraient être un meilleur choix.
  * S’agit-il d’un secret différent pour chaque utilisateur ? Dans ce cas, il peut être préférable de le stocker dans `app_metadata`.
</Card>

<Warning>
  Les revendications dans le jeton d’identité ne sont pas chiffrées. Selon le flux que vous utilisez, l’utilisateur pourrait donc obtenir le jeton et en examiner le contenu. Auth0 **ne** recommande **pas** de stocker un secret de cette façon.
</Warning>

Les applications existantes n’auront aucune valeur pour cette propriété.

Vous pouvez accéder aux métadonnées de l’application dans [Actions](/fr-CA/docs/customize/actions) :

```js lines
exports.onExecutePostLogin = async (event, api) => {
  if (event.client.metadata.SKIP_VERIFICATION === "yes"){
    return;
  }
  // ... continuer cette Action
}
```

... ou dans [Rules](/fr-CA/docs/customize/rules) :

```javascript lines
function applicationMetadataExample (user, context, callback){
  context.clientMetadata = context.clientMetadata || {};
  if (context.clientMetadata.SKIP_VERIFICATION === "yes"){
    return callback();
  }
  // ... continuer cette Rule
}
```

Vous pouvez lire les métadonnées de l’application et y ajouter des paires clé-valeur en utilisant soit l’Auth0 Dashboard, soit la <Tooltip tip="Management API : produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>. Pour savoir comment gérer les métadonnées de l’application avec la Management API, consultez [Gérer les métadonnées à l’aide de la Management API](/fr-CA/docs/manage-users/user-accounts/metadata/manage-metadata-api).

<div id="add-application-metadata-keyvalue-pairs">
  ## Ajouter des paires clé-valeur de métadonnées d’application
</div>

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et sélectionnez l’application.
2. Faites défiler la page vers le bas, puis cliquez sur **Advanced Settings**.
3. Dans l’onglet **Application Metadata**, saisissez le nom et la valeur de la clé, puis cliquez sur **Add**.

   <Frame>![Auth0 Dashboard Applications Applications Settings Tab Advanced Settings Application Metadata Tab](/docs/images/cdy7uua7fh8z/7GWHyQloNihda3fuYiVEWQ/b3cced7c53690e6f3cbf7b2e23ff8646/App_Metadata_-_English.png)</Frame>
4. Cliquez sur **Save Changes**.

<div id="update-application-metadata-value">
  ## Mettre à jour la valeur des métadonnées d’application
</div>

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et sélectionnez l’application.
2. Faites défiler la page vers le bas, puis cliquez sur **Advanced Settings**.
3. Dans l’onglet **Application Metadata**, saisissez le nom de la clé que vous souhaitez modifier et une nouvelle valeur, puis cliquez sur **Add**.
4. Cliquez sur **Save Changes**

<div id="delete-application-metadata">
  ## Supprimer les métadonnées de l’application
</div>

1. Accédez à [Auth0 Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications) et sélectionnez l’application.
2. Faites défiler la page, puis cliquez sur **Advanced Settings**.
3. Dans l’onglet **Application Metadata**, repérez la paire clé-valeur que vous souhaitez supprimer, puis cliquez sur l’icône de corbeille.
4. Confirmez la suppression.
5. Cliquez sur **Save Changes**.
