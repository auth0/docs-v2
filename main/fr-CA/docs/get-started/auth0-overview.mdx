title: Présentation d’Auth0
---

Auth0 est une solution flexible, prête à l’emploi, qui permet d’ajouter des services d’authentification et d’autorisation à vos applications. Votre équipe et votre organisation peuvent éviter les coûts, les délais et les risques associés au développement de leur propre solution pour authentifier et autoriser les utilisateurs.

Voici quelques cas d’utilisation d’Auth0 :

* Vous avez créé une application formidable et vous voulez y ajouter l’authentification et l’autorisation des utilisateurs. Vos utilisateurs devraient pouvoir se connecter soit avec un identifiant (nom d’utilisateur, courriel ou numéro de téléphone) et un mot de passe, soit avec leurs comptes sociaux (comme Facebook ou X). Vous voulez récupérer le profil de l’utilisateur après la connexion afin de personnaliser l’interface utilisateur et d’appliquer vos stratégies d’autorisation.
* Vous avez créé une API et vous voulez la sécuriser avec [OAuth 2.0](/fr-CA/docs/authenticate/protocols/oauth).
* Vous avez plus d’une application et vous voulez implémenter l’[authentification unique (SSO)](/fr-CA/docs/authenticate/single-sign-on).
* Vous avez créé une application frontale en JavaScript et une application mobile, et vous voulez qu’elles accèdent toutes deux de façon sécurisée à votre API.
* Vous avez une application Web qui doit authentifier les utilisateurs à l’aide de [Security Assertion Markup Language (SAML)](/fr-CA/docs/authenticate/protocols/saml/saml-configuration).
* Vous estimez que les mots de passe ne suffisent plus et vous voulez que vos utilisateurs se connectent avec des codes à usage unique envoyés par courriel ou par SMS.
* Si l’une des adresses de courriel de vos utilisateurs est compromise dans une fuite de données publique sur un site, vous voulez en être avisé, puis avertir les utilisateurs et/ou les empêcher de se connecter à votre application jusqu’à ce qu’ils réinitialisent leur mot de passe.
* Vous voulez agir de manière proactive pour bloquer les adresses IP suspectes si elles effectuent plusieurs tentatives de connexion échouées consécutives, afin d’éviter les attaques DDoS.
* Vous faites partie d’une grande organisation qui souhaite fédérer son service d’annuaire d’entreprise existant pour permettre aux employés de se connecter aux différentes applications internes et tierces à l’aide de leurs identifiants d’entreprise existants.
* Vous ne voulez pas (ou vous ne savez pas comment) implémenter votre propre solution de gestion des utilisateurs. Réinitialisation des mots de passe, création, provisionnement, blocage et suppression d’utilisateurs, ainsi que l’interface pour gérer tout cela. Vous voulez simplement vous concentrer sur votre application.
* Vous voulez imposer l’[authentification multifacteur (MFA)](/fr-CA/docs/secure/multi-factor-authentication) lorsque vos utilisateurs souhaitent accéder à des données sensibles.
* Vous recherchez une solution d’identité qui vous aidera à respecter les exigences de conformité en constante évolution de SOC2, du RGPD, de PCI DSS, de HIPAA et d’autres normes.
* Vous voulez suivre l’activité des utilisateurs sur votre site ou dans votre application. Vous prévoyez utiliser ces données pour créer des entonnoirs, mesurer la fidélisation des utilisateurs et améliorer votre flux d’inscription.
* Vous voulez une stratégie d’autorisation robuste pour permettre à vos utilisateurs d’accéder aux ressources selon [leur relation](https://docs.fga.dev/authorization-concepts) avec la ressource ou [leur rôle](/fr-CA/docs/manage-users/access-control/rbac) dans votre organisation.
