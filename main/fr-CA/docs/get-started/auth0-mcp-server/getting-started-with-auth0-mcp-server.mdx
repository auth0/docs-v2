<Card title="Avant de commencer">
  * Installez [Node.js v18 ou une version ultérieure](https://nodejs.org/en/download) sur votre système.
  * Créez un [compte Auth0](https://auth0.com/) actif avec des droits d’administration.
  * Installez au moins un client MCP pris en charge ([Claude Desktop](https://claude.ai/download), [Cursor](https://www.cursor.com/downloads) ou [Windsurf](https://www.windsurf.com/editor), par exemple).
</Card>

Outillez votre équipe et vos utilisateurs en installant le [serveur Auth0 Model Context Protocol (MCP)](/fr-CA/docs/get-started/auth0-mcp-server) et en l’intégrant à plusieurs [clients MCP](https://modelcontextprotocol.io/clients) couramment utilisés.

<div id="installation-and-configuration">
  ## Installation et configuration
</div>

Installez Auth0 MCP Server en exécutant l’une de ces commandes dans votre terminal :

<div id="for-claude-desktop-default">
  ### Pour Claude Desktop (par défaut)
</div>

```bash lines
npx @auth0/auth0-mcp-server init
```

<div id="for-cursor">
  ### Pour Cursor
</div>

```bash lines
npx @auth0/auth0-mcp-server init --client cursor
```

<div id="for-windsurf">
  ### Pour Windsurf
</div>

```bash lines
npx @auth0/auth0-mcp-server init --client windsurf
```

<div id="other-mcp-clients">
  ### Autres clients MCP
</div>

Pour utiliser Auth0 MCP Server avec n’importe quel autre client MCP, vous pouvez ajouter manuellement cette configuration au client, puis le redémarrer pour que les modifications prennent effet :

```json lines
{
  "mcpServers": {
    "auth0": {
      "command": "npx",
      "args": ["-y", "@auth0/auth0-mcp-server", "run"],
      "capabilities": ["tools"],
      "env": {
        "DEBUG": "auth0-mcp"
      }
    }
  }
}
```

<div id="authenticate-with-auth0">
  ## S’authentifier avec Auth0
</div>

Votre navigateur s’ouvrira automatiquement pour lancer le <Tooltip tip="OAuth 2.0 : Cadre d’autorisation qui définit les protocoles et les flux d’autorisation." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=OAuth+2.0">OAuth 2.0</Tooltip> <Tooltip tip="OAuth 2.0 : Cadre d’autorisation qui définit les protocoles et les flux d’autorisation." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=authorization+flow">flux d’autorisation</Tooltip> d’appareil. Connectez-vous à votre compte Auth0 et accordez les autorisations demandées.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les identifiants sont stockés de façon sécurisée dans le trousseau de votre système. Vous pouvez aussi vérifier leur présence à l’aide de votre outil de gestion du trousseau.
</Callout>

<div id="verify-your-integration">
  ## Vérifiez votre intégration
</div>

<div id="claude-desktop">
  ### Claude Desktop
</div>

1. Redémarrez Claude Desktop pour vous assurer qu’il reconnaît le Auth0 MCP Server nouvellement installé.
2. Dans Claude Desktop, cliquez sur l’icône **Tools** et vérifiez que les outils Auth0 (par exemple, `auth0_create_action` ou `auth0_delete_application`) s’affichent dans la liste **Available MCP Tools**.
3. Lancez une conversation avec Claude Desktop et demandez de l’aide pour gérer votre locataire Auth0.

<div id="cursor">
  ### Cursor
</div>

1. Redémarrez Cursor pour vous assurer qu’il reconnaît le serveur Auth0 MCP Server nouvellement installé.
2. Dans Cursor, ouvrez **Settings &gt; MCP** et vérifiez que **Auth0** apparaît dans la liste **MCP Servers**.
3. Lancez une conversation avec Cursor et demandez-lui de vous aider à gérer votre locataire Auth0.

<div id="windsurf">
  ### Windsurf
</div>

1. Redémarrez Windsurf pour vous assurer qu’il reconnaît le Auth0 MCP Server nouvellement installé.
2. Dans Windsurf, ouvrez **Windsurf Settings &gt; Cascade** et vérifiez qu’**Auth0** figure dans la liste **Model Context Protocol (MCP) Servers**.
3. Lancez une conversation avec Windsurf et demandez-lui de vous aider à gérer votre locataire Auth0.

<div id="common-troubleshooting-steps">
  ## Étapes de dépannage courantes
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  La plupart des problèmes de connexion peuvent être résolus en configurant le serveur MCP d’Auth0 et en redémarrant le client MCP.
</Callout>

<div id="get-command-line-help">
  #### Afficher l’aide de la ligne de commande :
</div>

Consultez la liste des commandes prises en charge et des exemples d’utilisation avec :

```bash lines
npx @auth0/auth0-mcp-server help
```

<div id="authentication-issues">
  #### Problèmes d’authentification
</div>

Assurez-vous que les autorisations sont correctement configurées dans votre locataire Auth0, puis essayez de réinitialiser avec :

```bash lines
npx @auth0/auth0-mcp-server init
```

<div id="connection-failures">
  #### Échecs de connexion :
</div>

1. Assurez-vous que le serveur MCP est bien en cours d’exécution avec :

   ```ps lines
   ps aux | grep auth0-mcp
   ```

2. Redémarrez votre client IA après avoir vérifié que le serveur est en cours d’exécution.

<div id="claude-cant-connect-to-the-server-error">
  #### Erreur « Claude ne parvient pas à se connecter au serveur » :
</div>

1. Redémarrez Claude Desktop après l’installation.
2. Vérifiez que le serveur est en cours d’exécution à l’aide de la commande suivante :

   ```ps lines
   ps aux | grep auth0-mcp
   ```

<div id="api-errors">
  #### Erreurs d’API :
</div>

Vérifiez les autorisations et la date d’expiration de votre jeton Auth0. Activez le mode de débogage (voir ci-dessous) pour consulter les journaux détaillés.

<div id="debug-configuration-issues">
  ## Débogage des problèmes de configuration
</div>

<div id="enable-debug-mode">
  #### Activer le mode de débogage
</div>

Activez le mode de débogage pour afficher des logs détaillés :

```bash lines
export DEBUG=auth0-mcp
```

<div id="debug-client-logs">
  #### Déboguer les journaux du client
</div>

Obtenez les journaux détaillés du client MCP dans Claude Desktop :

```bash lines
# Suivre les journaux en temps réel
tail -n 20 -F ~/Library/Logs/Claude/mcp*.log
```

Pour obtenir des journaux détaillés du MCP Server, exécutez le MCP Server d’Auth0 en mode débogage :

```bash lines
DEBUG=auth0-mcp npx @auth0/auth0-mcp-server run
```

<div id="advanced-debugging-using-mcp-inspector">
  #### Débogage avancé avec MCP Inspector
</div>

Pour un dépannage avancé, utilisez MCP Inspector :

```bash wrap lines
npx @modelcontextprotocol/inspector -e DEBUG='auth0-mcp' @auth0/auth0-mcp-server run__text in bold__
```
