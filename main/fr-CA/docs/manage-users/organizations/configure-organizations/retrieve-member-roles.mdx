Lorsque vous travaillez avec des [organisations](/fr-CA/docs/manage-users/organizations/organizations-overview) de manière programmatique, vous devrez peut-être récupérer la liste des rôles attribués à un membre d’une organisation.

Vous pouvez consulter les rôles associés à un membre d’une organisation à l’aide du <Tooltip tip="Auth0 Dashboard : console principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Management API : produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>.

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; Organisations](https://manage.auth0.com/#/organizations), puis sélectionnez l’organisation dont vous voulez consulter les membres.
2. Sélectionnez la vue **Membres**, puis sélectionnez l’utilisateur dont vous voulez afficher les rôles.

<div id="management-api">
  ## Management API
</div>

Effectuez un appel `GET` au point de terminaison `Get Organization Member Roles`. Assurez-vous de remplacer les valeurs de remplacement `ORG_ID`, `USER_ID` et `MGMT_API_ACCESS_TOKEN` par l’ID de votre organisation, l’ID de l’utilisateur membre et le <Tooltip tip="Jeton d’accès : informations d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisées pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Access+Token">Jeton d’accès</Tooltip> de la Management API, respectivement.

  curl --request GET \

    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Trouvez votre domaine Auth0**

  Si votre domaine Auth0 est composé du nom de votre locataire, ajoutez votre sous-domaine régional (à moins que votre locataire se trouve dans la région des États-Unis et ait été créé avant juin 2020), puis `.auth0.com`. Par exemple, si le nom de votre locataire était `travel0`, le nom de votre domaine Auth0 serait `travel0.us.auth0.com`. (Si votre locataire se trouvait aux États-Unis et avait été créé avant juin 2020, le nom de votre domaine serait alors `https://travel0.auth0.com`.)

  Si vous utilisez des domaines personnalisés, il doit s’agir du nom de votre domaine personnalisé.
</Callout>

| Valeur                  | Description                                                                                                                                         |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ORG_ID`                | ID de l’organisation dont vous voulez récupérer les rôles d’un membre.                                                                              |
| `USER_ID`               | ID utilisateur du membre dont vous voulez récupérer les rôles.                                                                                      |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens) avec le scope `read:organization_member_roles`. |

<div id="response-status-codes">
  #### Codes d’état de la réponse
</div>

Les codes d’état de réponse possibles sont les suivants :

| Code d’état | Code d’erreur          | Message                                                                                                | Cause                                                                                           |
| ----------- | ---------------------- | ------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------- |
| `200`       |                        | Rôles récupérés avec succès.                                                                           |                                                                                                 |
| `400`       | `invalid_query_string` | La chaîne de requête est invalide. Le message varie selon la cause.                                    | La chaîne de requête n’est pas valide.                                                          |
| `401`       |                        | Jeton invalide.                                                                                        |                                                                                                 |
| `401`       |                        | Signature invalide reçue pour la validation du JSON Web Token.                                         |                                                                                                 |
| `401`       |                        | L’application n’est pas globale.                                                                       |                                                                                                 |
| `403`       | `insufficient_scope`   | Scopes insuffisants ; l’un des scopes suivants est requis : `read:organization_member_roles`.          | Tentative de lire ou de modifier un champ non autorisé avec les scopes fournis du Bearer Token. |
| `429`       |                        | Trop de requêtes. Vérifiez les en-têtes X-RateLimit-Limit, X-RateLimit-Remaining et X-RateLimit-Reset. |                                                                                                 |
