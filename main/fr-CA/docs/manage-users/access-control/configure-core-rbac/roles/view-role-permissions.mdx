Vous pouvez consulter les [autorisations](/fr-CA/docs/manage-users/access-control/rbac) ajoutées à un rôle à l’aide du <Tooltip tip="Tableau de bord Auth0 : produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">tableau de bord Auth0</Tooltip> ou de l’<Tooltip tip="Tableau de bord Auth0 : produit principal d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">API de gestion</Tooltip>. Les autorisations et les rôles attribués sont utilisés avec les fonctionnalités API Authorization Core.

<div id="prerequisites">
  ## Prérequis
</div>

Pour que le contrôle d’accès fondé sur les rôles (RBAC) fonctionne correctement, vous devez l’activer pour votre API à l’aide du tableau de bord ou de l’API de gestion. La fonctionnalité Authorization Core diffère de l’Authorization Extension. Pour comparer les deux, consultez [Authorization Core vs. Authorization Extension](/fr-CA/docs/manage-users/access-control/authorization-core-vs-authorization-extension).

<div id="dashboard">
  ## Tableau de bord
</div>

1. Accédez à [Tableau de bord &gt; Gestion des utilisateurs &gt; Rôles](https://manage.auth0.com/#/roles), puis cliquez sur le nom du rôle à afficher.
2. Cliquez sur l’onglet **Autorisations**.

Les renseignements suivants s’affichent pour chaque autorisation :

| Colonne     | Description                                                              |
| ----------- | ------------------------------------------------------------------------ |
| Nom         | Nom de l’autorisation tel qu’il apparaît dans sa définition.             |
| Description | Description de l’autorisation telle qu’elle apparaît dans sa définition. |
| API         | Nom de l’API à laquelle l’autorisation est associée.                     |

<div id="management-api">
  ## API de gestion
</div>

Effectuez un appel `GET` au [point de terminaison Get Role Permissions](https://auth0.com/docs/api/management/v2#!/Roles/get_role_permission). Assurez-vous de remplacer les valeurs de remplacement `ROLE_ID` et `MGMT_API_ACCESS_TOKEN` par l’ID de votre rôle et votre <Tooltip tip="Jeton d’accès : information d’identification d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisée pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+token">jeton d’accès</Tooltip> de l’API de gestion, respectivement.

<AuthCodeGroup>
  ```bash cURL
  curl --request GET \
    --url 'https://{yourDomain}/api/v2/roles/ROLE_ID/permissions' \
    --header 'authorization: Bearer MGMT_API_ACCESS_TOKEN'
  ```

  ```csharp C#
  var client = new RestClient("https://{yourDomain}/api/v2/roles/ROLE_ID/permissions");
  var request = new RestRequest(Method.GET);
  request.AddHeader("authorization", "Bearer MGMT_API_ACCESS_TOKEN");
  IRestResponse response = client.Execute(request);
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"net/http"
  	"io/ioutil"
  )

  func main() {

  	url := "https://{yourDomain}/api/v2/roles/ROLE_ID/permissions"

  	req, _ := http.NewRequest("GET", url, nil)

  	req.Header.Add("authorization", "Bearer MGMT_API_ACCESS_TOKEN")

  	res, _ := http.DefaultClient.Do(req)

  	defer res.Body.Close()
  	body, _ := ioutil.ReadAll(res.Body)

  	fmt.Println(res)
  	fmt.Println(string(body))

  }
  ```

  ```java Java
  HttpResponse<String> response = Unirest.get("https://{yourDomain}/api/v2/roles/ROLE_ID/permissions")
    .header("authorization", "Bearer MGMT_API_ACCESS_TOKEN")
    .asString();
  ```

  ```javascript Node.JS
  var axios = require("axios").default;

  var options = {
    method: 'GET',
    url: 'https://{yourDomain}/api/v2/roles/ROLE_ID/permissions',
    headers: {authorization: 'Bearer MGMT_API_ACCESS_TOKEN'}
  };

  axios.request(options).then(function (response) {
    console.log(response.data);
  }).catch(function (error) {
    console.error(error);
  });
  ```

  ```php PHP
  $curl = curl_init();

  curl_setopt_array($curl, [
    CURLOPT_URL => "https://{yourDomain}/api/v2/roles/ROLE_ID/permissions",
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_ENCODING => "",
    CURLOPT_MAXREDIRS => 10,
    CURLOPT_TIMEOUT => 30,
    CURLOPT_HTTP_VERSION => CURL_HTTP_VERSION_1_1,
    CURLOPT_CUSTOMREQUEST => "GET",

      "authorization: Bearer MGMT_API_ACCESS_TOKEN"

  $response = curl_exec($curl);
  $err = curl_error($curl);

  curl_close($curl);

  if ($err) {
    echo "cURL Error #:" . $err;
  } else {
    echo $response;
  }
  ```

  ```python Python
  import http.client

  conn = http.client.HTTPSConnection("")

  headers = { 'authorization': "Bearer MGMT_API_ACCESS_TOKEN" }

  conn.request("GET", "/{yourDomain}/api/v2/roles/ROLE_ID/permissions", headers=headers)

  res = conn.getresponse()
  data = res.read()

  print(data.decode("utf-8"))
  ```

  ```ruby Ruby
  require 'uri'
  require 'net/http'
  require 'openssl'

  url = URI("https://{yourDomain}/api/v2/roles/ROLE_ID/permissions")

  http = Net::HTTP.new(url.host, url.port)
  http.use_ssl = true
  http.verify_mode = OpenSSL::SSL::VERIFY_NONE

  request = Net::HTTP::Get.new(url)
  request["authorization"] = 'Bearer MGMT_API_ACCESS_TOKEN'

  response = http.request(request)
  puts response.read_body
  ```
</AuthCodeGroup>

| Valeur                  | Description                                                                                                         |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------- |
| `ROLE_ID`               | ID du rôle dont vous voulez obtenir les autorisations.                                                              |
| `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour l’API de gestion](https://auth0.com/docs/api/management/v2/tokens) avec la portée `read:roles`. |
