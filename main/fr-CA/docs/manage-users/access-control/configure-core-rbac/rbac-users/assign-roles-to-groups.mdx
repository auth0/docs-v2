Auth0 vous permet d’attribuer et de supprimer des [rôles](/fr-CA/docs/manage-users/access-control/rbac) pour les groupes d’entreprise provisionnés par <Tooltip tip="System for Cross-domain Identity Management (SCIM) : protocole normalisé pour provisionner et gérer les identités et les groupes d’utilisateurs entre différents systèmes." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=SCIM">SCIM</Tooltip> à l’aide d’Auth0 Dashboard ou de la Management API. Lorsqu’un rôle est attribué à un groupe, tous les membres héritent de ce rôle à la connexion. Vous pouvez gérer les rôles de groupe à deux niveaux :

* **Groupes d’organisation** — Groupes d’entreprise au sein d’une [Organisation](/fr-CA/docs/manage-users/organizations). Nécessite que l’[adhésion automatique](/fr-CA/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership) soit activée sur la connexion d’entreprise pour que l’héritage des rôles prenne effet.
* **Groupes d’entreprise** — Groupes d’entreprise définis au niveau de votre locataire Auth0.

<div id="assign-roles">
  ## Attribuer des rôles
</div>

<div id="organization-groups">
  ### Groupes d’organisation
</div>

Attribuez des rôles aux groupes d’entreprise provisionnés par SCIM au sein d’une organisation. Les groupes d’entreprise sont synchronisés à partir d’un fournisseur d’identité d’entreprise (comme Okta ou Microsoft Entra ID) au moyen de SCIM. Lorsqu’un membre d’une organisation ouvre une session, il hérite de tous les rôles attribués aux groupes d’entreprise auxquels il appartient.

Pour que l’héritage des rôles de groupe prenne effet, la connexion d’entreprise de l’organisation doit avoir l’option [adhésion automatique activée](/fr-CA/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership). Pour en savoir plus sur le provisionnement des groupes d’entreprise, consultez [Configurer Inbound SCIM](/fr-CA/docs/authenticate/protocols/scim/configure-inbound-scim#use-groups-with-organizational-roles).

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Accédez à [Auth0 Dashboard &gt; Organisations](https://manage.auth0.com/#/organizations) et sélectionnez l’organisation.
    2. Sélectionnez l’onglet **Groupes**.
    3. Sélectionnez le groupe auquel vous voulez attribuer des rôles.
    4. Sélectionnez **Attribuer des rôles**.
    5. Recherchez et sélectionnez les rôles à attribuer, puis sélectionnez **Attribuer**.

    Les rôles attribués apparaissent dans les détails du groupe.
  </Tab>

  <Tab title="Management API">
    Effectuez un appel `POST` au point de terminaison [Assign Organization Group Roles](/fr-CA/docs/api/management/v2). Vous avez besoin des renseignements suivants :

    | Paramètre                    | Description                                                                                                                                           |
    | ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `org_id`                     | ID de l’organisation.                                                                                                                                 |
    | `group_id`                   | ID du groupe d’entreprise.                                                                                                                            |
    | `role_id`                    | ID du rôle d’organisation à attribuer. Répétez-le dans le tableau pour attribuer plusieurs rôles à la fois.                                           |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [jeton d’accès de la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens) avec le scope `create:organization_group_roles`. |

    ```bash wrap lines
    curl --request POST \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/organizations/{ORG_ID}/groups/{GROUP_ID}/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Si la requête réussit, une réponse `204 No Content` est renvoyée.
  </Tab>
</Tabs>

<div id="enterprise-groups">
  ### Groupes d’entreprise
</div>

Attribuez des rôles aux groupes d’entreprise provisionnés par SCIM. Lorsqu’un rôle est attribué à un groupe, tous les membres de ce groupe héritent du rôle attribué lorsqu’ils se connectent.

Pour savoir comment les groupes d’entreprise sont provisionnés par un fournisseur d’identité d’entreprise, consultez [Configurer Inbound SCIM](/fr-CA/docs/authenticate/protocols/scim/configure-inbound-scim).

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Accédez à [Auth0 Dashboard &gt; Gestion des utilisateurs &gt; Groupes d’entreprise](https://manage.auth0.com/#/groups).
    2. Sélectionnez le groupe auquel vous souhaitez attribuer des rôles.
    3. Sélectionnez l’onglet **Rôles**, puis **Attribuer des rôles**.
    4. Recherchez et sélectionnez les rôles à attribuer, puis sélectionnez **Attribuer**.

    Les rôles attribués s’affichent dans l’onglet **Rôles** du groupe.
  </Tab>

  <Tab title="Management API">
    Effectuez un appel `POST` au point de terminaison [Assign Roles to a Group](/fr-CA/docs/api/management/v2). Vous avez besoin des renseignements suivants :

    | Paramètre                    | Description                                                                                                                         |
    | ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
    | `group_id`                   | ID du groupe d’entreprise.                                                                                                          |
    | `role_id`                    | ID du rôle à attribuer. Répétez-le dans le tableau pour attribuer plusieurs rôles à la fois.                                        |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [jeton d’accès de la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens) avec le scope `update:groups`. |

    ```bash wrap lines
    curl --request POST \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/groups/{group_id}/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Si la requête réussit, elle renvoie une réponse `204 No Content`.
  </Tab>
</Tabs>

<div id="remove-roles">
  ## Retirer des rôles
</div>

<div id="organization-groups">
  ### Groupes d’organisation
</div>

Supprimez des rôles des groupes d’entreprise provisionnés par SCIM dans une organisation. Une fois un rôle supprimé, les membres de l’organisation de ce groupe n’en hériteront plus à leur connexion.

La connexion d’entreprise de l’organisation doit avoir l’[adhésion automatique activée](/fr-CA/docs/manage-users/organizations/configure-organizations/grant-just-in-time-membership) pour que l’héritage des rôles de groupe prenne effet.

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Accédez à [Dashboard &gt; organisation](https://manage.auth0.com/#/organizations) et sélectionnez l’organisation.
    2. Sélectionnez l’onglet **Groupes**.
    3. Sélectionnez le groupe que vous souhaitez mettre à jour.
    4. Repérez le rôle que vous souhaitez supprimer, puis sélectionnez l’icône de corbeille à côté.
    5. Confirmez la suppression.

    Le rôle n’apparaît plus dans les détails du groupe.
  </Tab>

  <Tab title="Management API">
    Effectuez un appel `DELETE` au point de terminaison [Delete Organization Group Roles](/fr-CA/docs/api/management/v2). Vous aurez besoin des renseignements suivants :

    | Paramètre                    | Description                                                                                                                                           |
    | ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `org_id`                     | ID de l’organisation.                                                                                                                                 |
    | `group_id`                   | ID du groupe d’entreprise.                                                                                                                            |
    | `role_id`                    | ID du rôle d’organisation à supprimer. Répétez-le dans le tableau pour supprimer plusieurs rôles à la fois.                                           |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [jeton d’accès de la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens) avec le scope `delete:organization_group_roles`. |

    ```bash wrap lines
    curl --request DELETE \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/organizations/ORG_ID/groups/GROUP_ID/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Une requête réussie renvoie une réponse `204 No Content`.
  </Tab>
</Tabs>

<div id="enterprise-groups">
  ### Groupes d’entreprise
</div>

Supprimez des rôles des groupes d’entreprise provisionnés par SCIM. Une fois un rôle supprimé, les membres du groupe n’en hériteront plus lorsqu’ils se connecteront.

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Accédez à [Auth0 Dashboard &gt; Gestion des utilisateurs &gt; Groupes d’entreprise](https://manage.auth0.com/#/groups).
    2. Sélectionnez le groupe que vous souhaitez mettre à jour.
    3. Sélectionnez l’onglet **Rôles**.
    4. Repérez le rôle que vous souhaitez supprimer, puis sélectionnez l’icône de corbeille à côté.
    5. Confirmez la suppression.

    Le rôle n’apparaît plus dans l’onglet **Rôles** du groupe.
  </Tab>

  <Tab title="Management API">
    Envoyez une requête `DELETE` au point de terminaison [Remove Roles from a Group](/fr-CA/docs/api/management/v2). Vous avez besoin des renseignements suivants :

    | Paramètre                    | Description                                                                                                                         |
    | ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
    | `group_id`                   | ID du groupe d’entreprise.                                                                                                          |
    | `role_id`                    | ID du rôle à supprimer. Répétez cette valeur dans le tableau pour supprimer plusieurs rôles à la fois.                              |
    | `YOUR_MGMT_API_ACCESS_TOKEN` | [jeton d’accès de la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens) avec le scope `update:groups`. |

    ```bash wrap lines
    curl --request DELETE \
      --url 'https://{YOUR_AUTH0_DOMAIN}/api/v2/groups/GROUP_ID/roles' \
      --header 'authorization: Bearer YOUR_MGMT_API_ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{ "roles": [ "ROLE_ID" ] }'
    ```

    Une requête réussie renvoie une réponse `204 No Content`.
  </Tab>
</Tabs>
