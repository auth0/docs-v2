Vous pouvez mettre à jour les préférences de connexion d’un <Tooltip tip="fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+provider">fournisseur d’identité</Tooltip> en amont afin de contrôler à quel moment les mises à jour des attributs racine du profil utilisateur sont autorisées à l’aide de l’<Tooltip tip="fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> ou de la <Tooltip tip="Auth0 Dashboard : principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>.

Par défaut, les attributs du profil utilisateur fournis par des fournisseurs d’identité autres qu’Auth0 (comme Google, Facebook ou X) ne peuvent pas être modifiés directement, car ils sont mis à jour par le fournisseur d’identité chaque fois que l’utilisateur se connecte.

Pour pouvoir modifier les attributs racine name, nickname, given&#95;name, family&#95;name ou picture dans le profil utilisateur normalisé, vous devez configurer la synchronisation de votre connexion avec Auth0 afin que les attributs utilisateur soient mis à jour par le fournisseur d’identité uniquement lors de la création du profil utilisateur.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  **Sync Users at Login** ne s’applique pas aux connexions Passwordless et ne peut pas être activé dans l’interface utilisateur du locataire ni par la Management API.
</Callout>

Vous pouvez modifier les [attributs racine individuellement](/fr-CA/docs/manage-users/user-accounts/user-profiles/root-attributes/update-root-attributes-for-users) ou par [importation en bloc](/fr-CA/docs/manage-users/user-accounts/user-profiles/root-attributes/set-root-attributes-during-user-import) à l’aide de la Management API.

<div id="use-the-dashboard">
  ## Utiliser le Auth0 Dashboard
</div>

1. Accédez à [Auth0 Dashboard &gt; Authentification](https://manage.auth0.com/#/connections), puis sélectionnez le type de connexion : Base de données, Sociale, Entreprise ou Passwordless.
2. Sélectionnez le nom d’une connexion pour afficher ses paramètres.
3. Repérez la section **Avancé**, basculez l’interrupteur **Synchroniser les attributs du profil utilisateur à chaque connexion** au réglage souhaité, puis sélectionnez **Enregistrer**.

<div id="use-the-management-api">
  ## Utiliser Management API
</div>

Avant de passer à cette étape, vous devez d’abord [récupérer les valeurs existantes de l’objet `options` de la connexion](/fr-CA/docs/authenticate/identity-providers/retrieve-connection-options) pour éviter d’écraser les valeurs actuelles. Sinon, les paramètres absents de l’objet d’origine seront perdus après la mise à jour.

Effectuez un appel `PATCH` à l’[endpoint Update a Connection](https://auth0.com/docs/api/management/v2#!/Connections/patch_connections_by_id). Assurez-vous d’inclure les valeurs d’origine de `options` dans l’appel pour éviter d’écraser les valeurs actuelles. Assurez-vous également de remplacer les valeurs d’espace réservé `CONNECTION_ID`, `MGMT_API_ACCESS_TOKEN` et `ATTRIBUTE_UPDATE_VALUE` par l’ID de votre connexion, votre <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+token">jeton d’accès</Tooltip> Management API et la nouvelle valeur de l’attribut, respectivement.

| Valeur                   | Description                                                                                                                                                                                                                                     |
| ------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CONNECTION_ID`          | ID de la connexion pour laquelle vous souhaitez autoriser la mise à jour des attributs racine.                                                                                                                                                  |
| `MGMT_API_ACCESS_TOKEN`  | [Jeton d’accès du Management API](https://auth0.com/docs/api/management/v2/tokens) avec le scope `update:connections`.                                                                                                                          |
| `ATTRIBUTE_UPDATE_VALUE` | Indique à quel moment vous souhaitez autoriser la mise à jour des attributs racine du profil utilisateur. Les valeurs valides sont `on_first_login` et `on_each_login`. La valeur par défaut pour les nouvelles connexions est `on_each_login`. |
