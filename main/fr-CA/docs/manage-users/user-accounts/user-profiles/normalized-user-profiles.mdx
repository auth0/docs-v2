description: Découvrez comment Auth0 normalise les propriétés utilisateur courantes dans le profil utilisateur.
title: Profils utilisateur normalisés
---

Le profil utilisateur normalisé est un mécanisme propre à Auth0 pour standardiser et stocker les renseignements liés à l’utilisateur. Comme chaque <Tooltip tip="fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> (IdP) fournit un ensemble différent de renseignements sur un utilisateur, Auth0 normalise les propriétés de profil courantes pour créer une représentation de l’utilisateur indépendante du protocole lorsqu’il stocke les claims liés à l’utilisateur. Par exemple, `family_name` dans le profil utilisateur normalisé contient des informations qui peuvent être renvoyées par un IdP sous les noms `surname` ou `last_name`.

Par conséquent, les claims Auth0 inclus dans le profil utilisateur normalisé diffèrent de l’ensemble standard de claims qui peuvent être renvoyés dans les <Tooltip tip="ID Token : information d’identification destinée à l’application elle-même, plutôt qu’à l’accès à une ressource." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=ID+Tokens">ID Tokens</Tooltip> par le [point de terminaison `oauth/token` de l’API d’authentification](https://auth0.com/docs/api/authentication#get-token) ou dans la réponse du point de terminaison [`/userinfo`](https://auth0.com/docs/api/authentication#user-profile) (les deux respectent les exigences détaillées dans la spécification OIDC).

Pour en savoir plus sur les attributs inclus dans le profil utilisateur normalisé et voir comment les utiliser, consultez [Structure du profil utilisateur](/fr-CA/docs/manage-users/user-accounts/user-profiles/user-profile-structure).
