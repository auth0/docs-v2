title: Gérer l’accès des utilisateurs aux applications
---

Tous les utilisateurs associés à un même locataire Auth0 sont partagés entre les applications du locataire (et ont donc accès à ces applications).

* Pour séparer les utilisateurs et restreindre leur accès, nous vous recommandons de [créer un locataire supplémentaire](/fr-CA/docs/get-started/auth0-overview/create-tenants/create-multiple-tenants) contenant les applications auxquelles vous souhaitez donner accès à certains groupes d’utilisateurs.

* Pour restreindre l’accès des utilisateurs à certaines applications, vous pouvez utiliser des [Rules](/fr-CA/docs/customize/rules).

  * Dans une Rule, les variables `context.clientName` et `context.clientID` sont disponibles pour vérifier quelle application l’utilisateur utilise pour se connecter.
  * Consultez [cette Rule pour obtenir un exemple](https://github.com/auth0/rules/blob/aeaf93bc058408e260192d0941a688963449d6be/src/rules/simple-user-whitelist-for-app.js).

* Pour empêcher des utilisateurs d’accéder à certaines applications, configurez une nouvelle connexion et accordez l’accès uniquement à l’application voulue.

  * Accédez à [Dashboard &gt; Authentication &gt; Social](https://manage.auth0.com/#/connections/social), puis cliquez sur une connexion.
  * Dans l’onglet **Applications**, activez ou désactivez les applications.

* Pour désactiver l’accès des utilisateurs à vos applications, vous pouvez [bloquer et débloquer des utilisateurs](/fr-CA/docs/manage-users/user-accounts/block-and-unblock-users) dans le Dashboard.
