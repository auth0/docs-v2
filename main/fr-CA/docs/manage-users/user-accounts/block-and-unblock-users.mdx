---
title: Comment bloquer et débloquer des utilisateurs en tant qu’administrateur du locataire
sidebarTitle: Bloquer et débloquer des utilisateurs
description: Contrôlez la capacité d’un utilisateur à se connecter à votre application en ajoutant ou en supprimant des blocages d’administrateur du locataire.
---

Les administrateurs du locataire peuvent bloquer des utilisateurs pour les empêcher d’accéder aux applications (et les débloquer pour leur redonner l’accès).

Si un utilisateur bloqué essaie de se connecter à une application, il est redirigé vers l’URL de callback avec les paramètres d’erreur suivants :

```
error=unauthorized&error_description=user%20is%20blocked
```

Cet événement est également consigné dans vos [logs du locataire](/fr-CA/docs/deploy-monitor/logs/log-event-type-codes) sous la forme d’un événement **Failed Login** (`f`) avec la description `user is blocked`.

<div id="add-tenant-administrator-blocks">
  ## Ajouter des blocages d’administrateur du locataire
</div>

Pour ajouter un blocage d’administrateur du locataire à l’aide de l’Auth0 Dashboard :

1. Accédez à [Auth0 Dashboard &gt; Gestion des utilisateurs &gt; Utilisateurs](https://manage.auth0.com/#/users).
2. Sélectionnez le nom de l’utilisateur que vous souhaitez bloquer.
3. Dans l’onglet **Détails**, à côté de **Bloquer l’utilisateur**, sélectionnez **Bloquer**.

Cette option est également accessible dans le menu **...** à droite du nom de l’utilisateur.

Pour ajouter un blocage d’administrateur du locataire à l’aide de la Management API, utilisez le [point de terminaison Update a User](/fr-CA/docs/api/management/v2/users/patch-users-by-id) (`PATCH /users/{id}`) pour définir l’[attribut `blocked`](/fr-CA/docs/api/management/v2/users/patch-users-by-id#body-blocked) de l’utilisateur sur `true`.

<div id="remove-tenant-administrator-blocks">
  ## Supprimer les blocages d’administrateur du locataire
</div>

Lorsqu’un administrateur du locataire bloque un utilisateur, le blocage n’expire pas automatiquement. Un administrateur du locataire doit débloquer l’utilisateur pour rétablir son accès aux applications.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les blocages liés à la [protection contre les attaques par force brute](/fr-CA/docs/secure/attack-protection/brute-force-protection) sont gérés séparément des blocages d’administrateur du locataire. Pour en savoir plus, consultez [la procédure de suppression des blocages liés à la protection contre les attaques par force brute](/fr-CA/docs/secure/attack-protection/brute-force-protection#removing-brute-force-protection-blocks).
</Callout>

Pour supprimer un blocage d’administrateur du locataire à l’aide de l’Auth0 Dashboard :

1. Accédez à [Auth0 Dashboard &gt; Gestion des utilisateurs &gt; Utilisateurs](https://manage.auth0.com/#/users).
2. Sélectionnez le nom de l’utilisateur visé par ce blocage.
3. Dans l’onglet **détail**, à côté de **bloquer l’utilisateur**, sélectionnez **Unblock**.

Cette option est aussi accessible dans le menu **...** à droite du nom de l’utilisateur.

Pour supprimer un blocage d’administrateur du locataire à l’aide de la Management API, utilisez le point de terminaison [Update a User](/fr-CA/docs/api/management/v2/users/patch-users-by-id) (`PATCH /users/{id}`) pour définir l’[attribut `blocked`](/fr-CA/docs/api/management/v2/users/patch-users-by-id#body-blocked) de l’utilisateur à `false`.

Si vous utilisez la Management API pour réinitialiser le mot de passe d’un utilisateur, cela supprime aussi les blocages d’administrateur du locataire.
