---
description: Comment afficher les utilisateurs et les détails de leur profil.
title: Afficher les détails d’un utilisateur
---

La page [Utilisateurs](https://manage.auth0.com/#/users) répertorie les utilisateurs associés à vos applications. Pour ouvrir un utilisateur en particulier, cliquez sur sa photo de profil ou sur son nom dans la colonne « Nom ». La page Détails de l’utilisateur s’ouvre et affiche les informations relatives à cet utilisateur.

La page Détails de l’utilisateur contient des liens vers cinq onglets :

1. Détails
2. Appareils
3. Historique
4. Emplacements
5. JSON brut

<div id="user-details-details-tab">
  ## Détails de l’utilisateur : onglet Détails
</div>

L’onglet Détails contient trois sections qui offrent un aperçu général des informations du profil de l’utilisateur :

* **Identité de l’utilisateur** présente en un coup d’œil des renseignements sur l’utilisateur, notamment l’identifiant qu’il a utilisé, les Connexions associées et ses droits d’accès.
* **Métadonnées** affiche les informations `app_metadata` et `user_metadata`. Vous pouvez modifier ces valeurs.
* **<Tooltip tip="fournisseur d’identité (IdP) : Service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Identity+Provider">fournisseur d’identité</Tooltip> Attributs** affiche les informations récupérées auprès du fournisseur d’authentification. Notez que les attributs du fournisseur d’identité sont en lecture seule.
* **Clés d’accès** affiche les [clés d’accès](/fr-CA/docs/authenticate/database-connections/passkeys) créées par l’utilisateur.
* **<Tooltip tip="Authentification multifacteur (MFA) : Processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Multi-Factor+Authentication">Authentification multifacteur</Tooltip>** affiche les facteurs [MFA](/fr-CA/docs/secure/multi-factor-authentication) auxquels l’utilisateur est inscrit.
  Si vous activez [Multiple Custom Domains](/fr-CA/docs/customize/custom-domains/multiple-custom-domains), vous disposez de l’option **Domaine** pour sélectionner le domaine de notification de votre locataire.

<div id="user-details-devices-tab">
  ## Détails de l’utilisateur : onglet Appareils
</div>

L’onglet Appareils répertorie les appareils à partir desquels l’utilisateur a demandé une authentification. Lorsqu’une autorisation est demandée sur un appareil, celui-ci est lié au compte de l’utilisateur.

Les informations de connexion de l’utilisateur sont associées au <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un nouveau Jeton d’accès sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Refresh+Token">Jeton d’actualisation</Tooltip> attribué à cet appareil. Pour révoquer le Jeton d’actualisation, cliquez sur **Dissocier** à côté de l’appareil.

<div id="user-details-history-tab">
  ## Détails de l’utilisateur : onglet Historique
</div>

L’onglet Historique affiche un journal de l’activité du compte de l’utilisateur des 2 derniers jours.

Les journaux contiennent des renseignements sur :

* Les événements survenus
* Le moment où les événements sont survenus
* Les applications associées aux événements
* Le fournisseur d’identité utilisé pour l’authentification
* Les adresses IP d’origine des événements
* L’endroit d’où les événements proviennent

<div id="user-details-locations-tab">
  ## Détails de l’utilisateur : onglet Emplacements
</div>

L’onglet Emplacements affiche une carte avec des repères indiquant l’emplacement ou les emplacements de l’utilisateur au moment où il s’est connecté aux applications.

<div id="user-details-raw-json-tab">
  ## Détails de l’utilisateur : onglet JSON brut
</div>

L’onglet JSON brut affiche toutes les données du profil de l’utilisateur au format JSON, ce qui vous permet de consulter rapidement toutes les informations disponibles sur l’utilisateur.
