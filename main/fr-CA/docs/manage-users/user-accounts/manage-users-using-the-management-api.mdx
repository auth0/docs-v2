title: Gérer les utilisateurs à l’aide de la Management API
---

En plus d’utiliser l’Auth0 Dashboard, vous pouvez récupérer, créer, mettre à jour ou supprimer des utilisateurs à l’aide de la [Management API](https://auth0.com/docs/api/management/v2#!/Users/get_users). Si vous voulez appeler directement la <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip>, vous devrez d’abord générer le <Tooltip tip="Management API : un produit qui permet aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+token">jeton d’accès</Tooltip> approprié. Pour en savoir plus, consultez [Jetons d’accès pour la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens). Vous pouvez aussi utiliser un SDK pour implémenter la fonctionnalité nécessaire afin d’appeler la Management API depuis votre application. Pour obtenir la liste des SDK offerts, consultez [la section SDK de notre matrice de support](/fr-CA/docs/troubleshoot/customer-support/product-support-matrix).

Pour savoir comment gérer l’accès à l’<Tooltip tip="Auth0 Dashboard : le principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> pour les membres de votre équipe, consultez [Gérer l’accès à l’Auth0 Dashboard](/fr-CA/docs/get-started/manage-dashboard-access).

<div id="permissions">
  ## Autorisations
</div>

Utilisez les autorisations suivantes pour obtenir un jeton d’accès pour les applications machine-to-machine. Pour en savoir plus, consultez [Enregistrer des applications machine-to-machine.](/fr-CA/docs/get-started/auth0-overview/create-applications/machine-to-machine-apps)

| Autorisations               | Descriptions                                                                                                                                                                                                                                                    |
| --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `read:users`                | Les applications peuvent rechercher et récupérer tous les enregistrements d’utilisateurs stockés dans un locataire Auth0. Cela comprend la lecture au moyen des points de terminaison `/api/v2/users`, `/api/v2/users-by-email` et `/api/v2/jobs/users-export`. |
| `create:users`              | Les applications peuvent créer des utilisateurs dans n’importe quelle connexion de base de données ou Passwordless d’un locataire Auth0. Cela comprend la création au moyen des points de terminaison `/api/v2/users` et `/api/v2/jobs/users-import`.           |
| `update:users`              | Les applications peuvent mettre à jour n’importe quel utilisateur stocké dans un locataire Auth0. Cela comprend la mise à jour au moyen du point de terminaison `/api/v2/users`.                                                                                |
| `update:users_app_metadata` | Les applications peuvent mettre à jour uniquement l’attribut `user.app_metadata` pour n’importe quel utilisateur stocké dans un locataire Auth0. Cela comprend la mise à jour au moyen de `/api/v2/users endpoint`.                                             |
| `delete:users`              | Les applications peuvent supprimer n’importe quel utilisateur stocké dans un locataire Auth0. Cela comprend la suppression au moyen du point de terminaison `/api/v2/users`.                                                                                    |

<div id="limitations">
  ## Limites
</div>

Par défaut, les attributs du profil utilisateur fournis par des <Tooltip tip="fournisseur d’identité (IdP) : service qui stocke et gère les identités numériques." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=identity+providers">fournisseurs d’identité</Tooltip> autres qu’Auth0 (comme Google, Facebook ou X) ne peuvent pas être modifiés directement, car ils sont mis à jour par le fournisseur d’identité chaque fois que l’utilisateur se connecte. Pour en savoir plus sur les attributs provenant des fournisseurs d’identité, consultez [Configurer la connexion du fournisseur d’identité pour les mises à jour du profil utilisateur](/fr-CA/docs/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0).

Pour pouvoir modifier les attributs racine `name`, `nickname`, `given_name`, `family_name` ou `picture` du profil utilisateur normalisé, vous devez [configurer la synchronisation de votre connexion avec Auth0](/fr-CA/docs/manage-users/user-accounts/user-profiles/configure-connection-sync-with-auth0) afin que les attributs utilisateur soient mis à jour à partir du fournisseur d’identité uniquement lors de la création du profil utilisateur. Ces attributs racine pourront alors être modifiés individuellement ou par importation en lot

Vous pouvez définir des mots de passe à l’aide des points de terminaison `create` ou `update`, mais, pour des raisons de sécurité, les mots de passe ne peuvent pas être affichés avec les commandes `get` ou `list user`. La partie droite de l’API Explorer fournit des indications sur les attributs du profil utilisateur qui peuvent être affichés ou modifiés pour un appel donné.

<div id="endpoints">
  ## Points de terminaison
</div>

* Utilisez le point de terminaison [`/users`](https://auth0.com/docs/api/management/v2/users/get-users) pour [obtenir des renseignements sur tous les utilisateurs](https://auth0.com/docs/api/management/v2/users/get-users). Vous pouvez aussi inclure des critères de recherche pour trouver des utilisateurs précis.
* Utilisez [`/user_id`](https://auth0.com/docs/api/management/v2/users/get-users-by-id) pour [obtenir des renseignements sur un utilisateur](https://auth0.com/docs/api/management/v2/users/get-users-by-id) à partir du `user_id`. Le `user_id` est un identifiant interne composé d’un nom de connexion et d’un identifiant unique pour l’utilisateur. Le `user_id` est différent du <Tooltip tip="ID Token : jeton destiné au client lui-même, et non à l’accès à une ressource." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=ID+token">jeton d’identité</Tooltip>.
* Le point de terminaison [`/userinfo`](https://auth0.com/docs/api/authentication/reference#get-user-info) prend en entrée le [jeton d’accès Auth0 et renvoie des renseignements sur le profil utilisateur](https://auth0.com/docs/api/authentication/reference#get-user-info). Ce point de terminaison inclut les résultats de toutes les Rules qui ont pu modifier le profil utilisateur pendant la transaction d’authentification, mais le profil utilisateur renvoyé ne sera soumis à aucune [restriction par scopes](/fr-CA/docs/get-started/apis/scopes/api-scopes).
* Le point de terminaison [`/tokeninfo`](https://auth0.com/docs/api/authentication/reference#get-token-info) prend en entrée le [jeton d’identité Auth0 et renvoie des renseignements sur le profil utilisateur](https://auth0.com/docs/api/authentication/reference#get-token-info). Ce point de terminaison renvoie un résultat qui n’inclut pas les résultats des Rules qui modifient le profil utilisateur.
