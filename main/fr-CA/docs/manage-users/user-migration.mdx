<Warning>
  L’**extension d’importation/exportation des utilisateurs** deviendra obsolète en septembre 2025. Vous pouvez dès maintenant accéder aux mêmes fonctionnalités [directement dans l’Auth0 Dashboard](/fr-CA/docs/manage-users/user-migration/bulk-user-import-export).
</Warning>

Auth0 prend en charge l’importation d’utilisateurs depuis des applications externes à l’aide de connexions de base de données personnalisées, de la <Tooltip tip="Management API : produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> ou de l’extension d’importation/exportation des utilisateurs.

<div id="automatic-migrations">
  ## Migrations automatiques
</div>

Auth0 prend en charge la migration automatique des utilisateurs d’une [connexion de base de données personnalisée](/fr-CA/docs/authenticate/database-connections/custom-db) vers Auth0. Cette pratique est parfois appelée **migration progressive** ou **migration différée**. En activant cette fonctionnalité, vos utilisateurs :

* Sont migrés vers Auth0 la première fois qu’ils se connectent après la configuration de l’intégration.
* N’ont pas à réinitialiser leur mot de passe en raison de la migration.

<Card title="La disponibilité varie selon le forfait Auth0">
  La disponibilité de cette fonctionnalité dépend à la fois de votre implémentation de connexion et de votre forfait Auth0 ou de votre entente personnalisée. Pour en savoir plus, consultez [Tarification](https://auth0.com/pricing).
</Card>

Lorsqu’un utilisateur s’authentifie au moyen d’une connexion de base de données personnalisée marquée pour l’importation vers Auth0, le processus suivant a lieu :

* Auth0 authentifie les utilisateurs migrés à l’aide de la base de données Auth0.
* Si l’utilisateur n’a pas encore été migré, Auth0 exécute votre script de connexion personnalisé et, si la connexion réussit, ajoute l’utilisateur à la base de données Auth0.
* Lors des connexions suivantes, les identifiants de l’utilisateur sont récupérés depuis Auth0, **ET NON** depuis votre base de données personnalisée.
* Les nouveaux utilisateurs sont automatiquement ajoutés à la base de données Auth0.

<div id="bulk-user-imports-with-the-management-api">
  ## Importation groupée d’utilisateurs avec la Management API
</div>

Si vous avez déjà une base de données d’utilisateurs, vous pouvez utiliser notre point de terminaison [`/post_users_imports`](https://auth0.com/docs/api/management/v2#!/Jobs/post_users_imports) de la Management API pour importer ces données dans une connexion de base de données.

<div id="migrate-users-with-the-user-importexport-feature">
  ## Migrer des utilisateurs avec la fonctionnalité d’importation/exportation des utilisateurs
</div>

La fonctionnalité d’importation/exportation des utilisateurs vous permet de :

* Importer en bloc les utilisateurs de votre base de données existante dans Auth0.
* Rechercher et exporter certains de vos utilisateurs de base de données Auth0 (ou tous).

Pour utiliser l’importation en bloc, vous devez avoir le rôle Dashboard Editor - Users ou être administrateur. Pour utiliser l’exportation en bloc, vous devez avoir le rôle Dashboard Viewer - Users, Editor - Users, ou être administrateur.

Vous pouvez importer et exporter des données des utilisateurs à l’aide de la [fonctionnalité d’importation/exportation des utilisateurs](/fr-CA/docs/manage-users/user-migration/bulk-user-import-export) accessible dans l’Auth0 Dashboard.
