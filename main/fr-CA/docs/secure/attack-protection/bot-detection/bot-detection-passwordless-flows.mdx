title: Ajouter la Bot Detection aux flux Passwordless
---

Si vous utilisez des flux de connexion et d’inscription <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=passwordless">Passwordless</Tooltip>, vous pouvez activer la <Tooltip tip="Passwordless : forme d’authentification qui ne repose pas sur un mot de passe comme premier facteur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Bot+Detection">Bot Detection</Tooltip> pour afficher une étape de captcha lorsqu’Auth0 détermine qu’une requête présente un risque élevé.

<Warning>
  Si vous utilisez le [SDK Lock](/fr-CA/docs/libraries/lock) avec l’expérience Classic Login, vous devez le mettre à jour vers la version 11.35.0 ou une version ultérieure pour utiliser la Bot Detection avec les flux Passwordless.

  Si vous utilisez le [SDK Auth0.js](/fr-CA/docs/libraries/auth0js), vous devez le mettre à jour vers la version 9.20.0 ou une version ultérieure pour utiliser la Bot Detection avec les flux Passwordless.
</Warning>

<div id="configure-bot-detection-for-passwordless-flows">
  ## Configurer Bot Detection pour les flux Passwordless
</div>

Vous pouvez utiliser le <Tooltip tip="Auth0 Dashboard : le principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip> pour configurer le comportement de la fonctionnalité CAPTCHA.

1. Dans le [Auth0 Dashboard](https://manage.auth0.com/dashboard), accédez à [Security &gt; Attack Protection &gt; Bot Detection](https://manage.auth0.com/dashboard/#/security/attack-protection/bot-detection).
2. Repérez la section **Response**.
3. Pour le paramètre **Enforce CAPTCHA for passwordless flows**, sélectionnez à quel moment exiger un CAPTCHA :

   * **Never** : les utilisateurs n&#39;ont pas à remplir un CAPTCHA pour se connecter.
   * **When Risky** : les utilisateurs doivent remplir un CAPTCHA si la connexion présente un risque élevé.
   * **Always** : les utilisateurs doivent toujours remplir un CAPTCHA pour se connecter.
