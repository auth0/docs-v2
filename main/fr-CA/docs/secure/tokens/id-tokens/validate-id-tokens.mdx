description: Décrit comment valider un jeton d’identité.
title: Valider les ID tokens
---

Un [jeton d’identité](/fr-CA/docs/secure/tokens/id-tokens), qui contient des attributs du profil utilisateur, est utilisé par une application, généralement à des fins d’affichage dans l’interface utilisateur. Auth0 émet tous les <Tooltip tip="ID Token : jeton destiné à l’application cliente elle-même, et non à l’accès à une ressource." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=ID+tokens">ID tokens</Tooltip> au format [JSON web token (JWT)](/fr-CA/docs/secure/tokens/json-web-tokens).

Si l’une de ces vérifications échoue, le jeton est considéré comme invalide et la requête doit être rejetée.

1. [Validez le JWT](/fr-CA/docs/secure/tokens/json-web-tokens/validate-json-web-tokens).
2. Vérifiez d’autres claims standards. Si vous avez effectué la validation JWT standard, vous avez déjà décodé le [payload du JWT](/fr-CA/docs/secure/tokens/json-web-tokens/json-web-token-structure) et examiné ses claims standards. Les claims supplémentaires à vérifier pour les ID tokens comprennent notamment :

   * **Audience du jeton** (`aud`, string) : la valeur d’audience du jeton doit correspondre à l’ID client de l’application, tel qu’il est défini dans les [paramètres de votre application](https://manage.auth0.com/#/applications), dans le champ **Client ID**.
   * **Nonce** (`nonce`, string) : il est recommandé d’inclure un nonce dans la requête de jeton (obligatoire pour le flux implicite) pour aider à prévenir les attaques par rejeu. La valeur du nonce dans le jeton doit correspondre exactement au nonce d’origine envoyé dans la requête. Pour en savoir plus, consultez [Atténuer les attaques par rejeu](/fr-CA/docs/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post/mitigate-replay-attacks-when-using-the-implicit-flow).
