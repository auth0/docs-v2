---

Une Rule d’ACL du locataire est composée des éléments suivants :

* **Signal** : Le signal est une information d’identification fournie par la requête entrante, comme l’adresse IP, la géolocalisation ou l’agent utilisateur.

* **Condition** : La condition est la combinaison d’un opérateur (comme `match`) et d’un ensemble de valeurs (comme une liste d’adresses IP).

* **Action** : L’action est l’instruction que votre Rule exécute si les critères sont remplis, par exemple autoriser, bloquer ou rediriger.

* **Scope** : Le scope indique à quels ensembles de points de terminaison la Rule s’applique, notamment l’Authentication API, la Management API ou l’ensemble de votre locataire.

* **Priority** : La priorité définit l’ordre dans lequel la Rule s’exécute par rapport aux autres Rules.

Pour en savoir plus sur les propriétés de ces Rules d’ACL du locataire et sur leur utilisation, consultez la [référence des paramètres du point de terminaison de la Management API](./reference).

<Note>
  L’ACL du locataire utilise une [logique uniforme d’évaluation des règles](./rule-evaluation) pour déterminer, s’il y a lieu, quelle action appliquer à une requête donnée.

  Attribuez soigneusement les priorités afin de créer des politiques de contrôle d’accès granulaires adaptées à votre cas d’utilisation.
</Note>

<div id="create-and-delete-rules">
  ## Créer et supprimer des Rules
</div>

Vous pouvez créer et supprimer des Rules d’ACL du locataire à l’aide des points de terminaison de la Management API suivants. Vous avez besoin d’un [jeton d’accès de la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) avec les scopes indiqués.

| Action             | Point de terminaison                                                                                      | Scope requis          |
| ------------------ | --------------------------------------------------------------------------------------------------------- | --------------------- |
| Créer une Rule     | [Créer une liste de contrôle d’accès](/fr-CA/docs/api/management/v2/network-acls/post-network-acls)             | `create:network_acls` |
| Supprimer une Rule | [Supprimer une liste de contrôle d’accès](/fr-CA/docs/api/management/v2/network-acls/delete-network-acls-by-id) | `delete:network_acls` |

<div id="modify-rules">
  ## Modifier les Rules
</div>

Vous pouvez modifier des Rules d’ACL du locataire au moyen des points de terminaison suivants de la Management API. Vous avez besoin d’un [jeton d’accès de la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) avec les scopes indiqués.

| Action                 | Endpoint                                                                                                                   | Scope requis          |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------- | --------------------- |
| Mettre à jour une Rule | [Mettre à jour partiellement une liste de contrôle d’accès](/fr-CA/docs/api/management/v2/network-acls/patch-network-acls-by-id) | `update:network_acls` |
| Remplacer une Rule     | [Mettre à jour une liste de contrôle d’accès](/fr-CA/docs/api/management/v2/network-acls/put-network-acls-by-id)                 | `update:network_acls` |

<div id="enable-or-disable-rules">
  ### Activer ou désactiver des règles
</div>

<Tabs>
  <Tab title="Management API">
    Vous pouvez activer ou désactiver une Rule à l’aide du point de terminaison [Mettre à jour une liste de contrôle d’accès](https://auth0.com/docs/api/management/v2/network-acls/put-network-acls-by-id) de la Management API.

    Définissez la propriété `active` de l’objet `rule` sur `true` ou `false`, respectivement.
  </Tab>

  <Tab title="Auth0 Dashboard">
    Pour activer ou désactiver une Rule dans Auth0 Dashboard, accédez à [Auth0 Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control).

    Dans la liste des Rules existantes, vous pouvez activer ou désactiver des Rules précises à l’aide du bouton bascule situé à côté de chacune d’elles.

    ![Activer ou désactiver une Rule de l’ACL du locataire dans Auth0 Dashboard](/docs/images/cdy7uua7fh8z/29b92e102142d3cb20e69a40e292f6b7/Screenshot_2025-09-03_at_12.34.05.png)

    La colonne **Statut** indique le statut de la Rule (**Active** ou **Désactivée**).
  </Tab>
</Tabs>

<div id="view-rules">
  ## Afficher les Rules
</div>

<Tabs>
  <Tab title="Management API">
    Vous pouvez afficher les Rules d’ACL du locataire existantes au moyen des points de terminaison suivants de la Management API. Vous avez besoin d’un [jeton d’accès de la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) avec les scopes indiqués.

    | Action                    | Endpoint                                                                                                                                 | Scope requis        |
    | ------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | ------------------- |
    | Afficher une Rule         | [Obtenir une entrée précise de la liste de contrôle d’accès d’un locataire](/fr-CA/docs/api/management/v2/network-acls/get-network-acls-by-id) | `read:network_acls` |
    | Afficher toutes les Rules | [Obtenir toutes les entrées de la liste de contrôle d’accès d’un locataire](/fr-CA/docs/api/management/v2/network-acls/get-network-acls)       | `read:network_acls` |
  </Tab>

  <Tab title="Dashboard">
    Accédez à [Dashboard &gt; Security &gt; Access Control](https://manage.auth0.com/#/security/access-control) pour afficher la liste de vos Rules existantes.

    | Nom de la colonne    | Description                                                                                 | Champ de réponse de l’API connexe |
    | -------------------- | ------------------------------------------------------------------------------------------- | --------------------------------- |
    | Priorité             | Nombre ordinal indiquant à quel moment la Rule est évaluée                                  | `priority`                        |
    | Nom                  | Description de l’objectif ou de la fonctionnalité de la Rule                                | `description`                     |
    | Action               | L’[action](/fr-CA/docs/secure/tenant-access-control-list/reference#actions) exécutée par la Rule. | `action`                          |
    | Statut               | Valeur indiquant si la Rule est active ou désactivée.                                       | `active`                          |
    | Dernière mise à jour | Horodatage indiquant quand la Rule a été mise à jour pour la dernière fois.                 | `updated_at`                      |

    ![Afficher la liste des Rules d’ACL du locataire dans l’Auth0 Dashboard](/docs/images/cdy7uua7fh8z/4093f9a39aba48bdf8d579c134dc4da5/Screenshot_2025-09-03_at_12.33.47.png)
  </Tab>
</Tabs>
