<Card path="Avant de commencer">
  Pour configurer une Rule d’ACL du locataire, vous avez besoin d’un [jeton d’accès à la Management API](/fr-CA/docs/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) avec les scopes suivants :

  * `create:network_acls`
  * `update:network_acls`
  * `read:network_acls`
  * `delete:network_acls`
</Card>

Vous pouvez configurer des Rules de liste de contrôle d’accès (ACL) du locataire à l’aide de la <Tooltip tip="Management API : un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Management+API">Management API</Tooltip> d’Auth0.

<div id="available-actions">
  ## Actions disponibles
</div>

Vous pouvez afficher, créer, mettre à jour et supprimer des Rules d’ACL du locataire au moyen de la Management API.

| Action                     | Endpoint                                                                                                                                 | Scope requis          |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | --------------------- |
| Afficher une Rule         | [Obtenir une entrée précise de la liste de contrôle d’accès d’un locataire](/fr-CA/docs/api/management/v2/network-acls/get-network-acls-by-id) | `read:network_acls`   |
| Afficher toutes les Rules | [Obtenir toutes les entrées de la liste de contrôle d’accès d’un locataire](/fr-CA/docs/api/management/v2/network-acls/get-network-acls)       | `read:network_acls`   |
| Créer une Rule            | [Créer une liste de contrôle d’accès](/fr-CA/docs/api/management/v2/network-acls/post-network-acls)                                            | `create:network_acls` |
| Mettre à jour une Rule    | [Mettre à jour partiellement une liste de contrôle d’accès](/fr-CA/docs/api/management/v2/network-acls/patch-network-acls-by-id)               | `update:network_acls` |
| Remplacer une Rule        | [Mettre à jour une liste de contrôle d’accès](/fr-CA/docs/api/management/v2/network-acls/put-network-acls-by-id)                               | `update:network_acls` |
| Supprimer une Rule        | [Supprimer une liste de contrôle d’accès](/fr-CA/docs/api/management/v2/network-acls/delete-network-acls-by-id)                                | `delete:network_acls` |

<div id="parameters">
  ## Paramètres
</div>

Pour en savoir plus sur les paramètres de l’ACL du locataire et sur leur utilisation, consultez la section [Référence](./reference).

<ParamField path="description" type="string" required>
  Décrit l’objectif ou la fonction de la Rule.

  Exemple : `Only allow requests originating from the United States`
</ParamField>

<ParamField path="active" type="boolean" required>
  Active ou désactive la Rule.

  Exemple : `true`
</ParamField>

<ParamField path="priority" type="number" required>
  Valeur numérique qui détermine l’ordre d’évaluation de la Rule.

  Exemple : `1`
</ParamField>

<ParamField path="rule" type="object" required>
  Contient les conditions et les actions de la Rule.

  <Expandable>
    <ParamField path="action" type="object" required>
      Contient l’action effectuée par la Rule.

      Exemple : `{ allow: true }`
    </ParamField>

    <ParamField path="match" type="object">
      Définit la ou les conditions auxquelles la requête entrante doit satisfaire.

      Exemple : `{ geo_country_codes: ["US"] }`
    </ParamField>

    <ParamField path="not_match" type="object">
      Définit la ou les conditions auxquelles la requête entrante ne doit pas satisfaire.

      Exemple : `{ geo_country_codes: ["CN"] }`
    </ParamField>

    <ParamField path="scope" type="string" required>
      Service ou contexte dans lequel la Rule est appliquée.

      Exemple : `authentication`
    </ParamField>
  </Expandable>
</ParamField>

<div id="toggle-monitoring-mode-for-a-rule">
  ## Activer ou désactiver le mode de surveillance pour une Rule
</div>

Vous pouvez activer le [mode de surveillance](../tenant-access-control-list) pour une Rule de l’ACL du locataire à l’aide du point de terminaison [Mettre à jour une liste de contrôle d’accès](https://auth0.com/docs/api/management/v2/network-acls/put-network-acls-by-id) de la Management API.

Ajoutez la propriété `log` à l’objet `rule.action` et définissez sa valeur sur `true`.
