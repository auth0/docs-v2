**Publié** : 31 mars 2020

**Numéros CVE** : CVE-2020-5391, CVE-2020-5392, CVE-2020-6753, CVE-2020-7948, CVE-2020-7947

**Crédit** : Muhamad Visat

<div id="overview">
  ## Vue d’ensemble
</div>

Auth0 a publié une nouvelle version majeure du [WordPress Plugin for Auth0](https://github.com/auth0/wp-auth0) pour corriger plusieurs vulnérabilités.

Nous vous recommandons de consulter les avis de sécurité suivants et de passer à cette nouvelle version majeure :

* Absence de protections CSRF pour le champ Domaine dans le plugin Auth0 WP : [CVE-2020-5391](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-5391)
* Vulnérabilité XSS persistante dans le plugin Auth0 WP (page Paramètres) : [CVE-2020-5392](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-5392)
* Vulnérabilité XSS persistante dans le plugin Auth0 WP (plusieurs pages) : [CVE-2020-6753](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-6753)
* Vulnérabilités d’injection CSV dans le plugin Auth0 WP : [CVE-2020-7947](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-7947)
* Référence directe non sécurisée à un objet dans le plugin Auth0 WP : [CVE-2020-7948](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-7948)

<div id="am-i-affected">
  ## Suis-je touché ?
</div>

Les clients qui utilisent la version 3.11.3 ou toute version antérieure du WordPress Plugin for Auth0 peuvent être touchés.

<div id="how-to-fix-that">
  ## Comment corriger ce problème ?
</div>

Les clients qui utilisent [WordPress Plugin for Auth0](https://github.com/auth0/wp-auth0) doivent passer à la version 4.0.0 ou ultérieure.

<div id="will-this-update-impact-my-users">
  ## Cette mise à jour aura-t-elle un impact sur mes utilisateurs ?
</div>

Les [notes de version](https://github.com/auth0/wp-auth0/blob/master/CHANGELOG.md) fournissent des renseignements plus détaillés sur les modifications apportées, et les [instructions de migration](https://github.com/auth0/wp-auth0/blob/master/MIGRATE-v3-TO-v4.md) donnent plus de détails sur le processus de mise à niveau.
