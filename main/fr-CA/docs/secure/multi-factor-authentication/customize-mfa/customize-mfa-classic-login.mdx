---

Vous pouvez activer l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip> dans la section [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa). Vous pouvez personnaliser les pages MFA à l’aide des options de base d’image de marque dans l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>. Pour aller plus loin dans la personnalisation, vous pouvez fournir votre propre code HTML pour la page MFA. Lorsque vous utilisez votre propre code HTML, Classic Login utilise le widget MFA d’Auth0 avec les limitations suivantes :

* Il ne prend pas en charge la MFA par courriel.
* Si les utilisateurs sont inscrits à plus d’un facteur, ils ne peuvent pas choisir celui à utiliser. Le widget MFA leur demandera de se connecter avec le facteur le plus sûr.
* Il n’utilise pas les fonctionnalités d’internationalisation de <Tooltip tip="Universal Login : votre application redirige vers Universal Login, hébergé sur le serveur d’autorisation d’Auth0, afin de vérifier l’identité d’un utilisateur." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Universal+Login">Universal Login</Tooltip>. Pour en savoir plus, consultez [Internationalisation d’Universal Login](/fr-CA/docs/customize/internationalization-and-localization/universal-login-internationalization).

<div id="customize-html-for-the-mfa-page">
  ## Personnaliser le HTML de la page MFA
</div>

Pour personnaliser la page MFA, accédez à l’onglet [Dashboard &gt; Branding &gt; Universal Login &gt; Multi-factor Authentication](https://manage.auth0.com/#/mfa_page) et activez le bouton bascule **Customize MFA Page**. Vous pouvez ensuite utiliser l’éditeur de texte pour modifier votre HTML, mettre en forme votre page à l’aide de CSS et modifier le JavaScript utilisé pour récupérer des variables personnalisées. Une fois vos modifications effectuées, assurez-vous de cliquer sur **Save Changes**.

Les pages MFA fonctionnent sans personnalisation et Auth0 mettra à jour les scripts inclus au besoin. Toutefois, une fois la personnalisation activée, il vous incombe de mettre à jour et de maintenir le script (y compris les numéros de version, comme celui du widget MFA), puisque Auth0 ne peut plus le mettre à jour automatiquement.

Pour en savoir plus sur la façon de remplacer le texte dans de nombreuses parties du processus MFA pour Classic Login, consultez [Dictionnaire linguistique du thème MFA](/fr-CA/docs/secure/multi-factor-authentication/customize-mfa/mfa-theme-language-dictionary). Pour en savoir plus sur les options de thème du widget MFA, consultez [Options de thème du widget MFA](/fr-CA/docs/secure/multi-factor-authentication/customize-mfa/mfa-widget-theme-options).

<div id="reset-page">
  ## Page de réinitialisation
</div>

Si vous souhaitez revenir à une version antérieure de la conception, vous avez deux options :

* Revenir au dernier modèle enregistré en cliquant sur **Reset to Last** ;
* Revenir au modèle par défaut fourni par Auth0 en cliquant sur **Reset to Default**.

<div id="render-enrollment-invitations">
  ## Afficher les invitations à l’inscription
</div>

La page peut s’afficher dans deux scénarios différents. Si un utilisateur a été dirigé vers cette page expressément pour l’inscription (par exemple, à partir d’un courriel contenant un lien d’inscription), la propriété `ticket` sera disponible. Sinon, la propriété `requestToken` sera disponible.

<div id="html-and-liquid-syntax">
  ## Syntaxe HTML et Liquid
</div>

La page hébergée utilise la syntaxe Liquid pour le templating. Pour en savoir plus, consultez [Liquid for Designers](https://github.com/Shopify/liquid/wiki/Liquid-for-Designers) sur GitHub. Les paramètres suivants sont disponibles pour faciliter le rendu de votre page :

* `userData.email`
* `userData.friendlyUserId`
* `userData.tenant`
* `userData.tenantFriendlyName`
* `iconUrl`

La plupart des paramètres utilisés dans le widget MFA doivent être transmis à Guardian, comme indiqué dans le modèle par défaut fourni dans la section de personnalisation. Si vous avez besoin d’un niveau de personnalisation plus poussé, vous pouvez utiliser [auth0-guardian.js](https://github.com/auth0/auth0-guardian.js/tree/master/example).

Pour en savoir plus, consultez [Personnaliser les pages utilisateur de la MFA](/fr-CA/docs/secure/multi-factor-authentication/customize-mfa).
