Vous pouvez activer l’<Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=multi-factor+authentication">authentification multifacteur</Tooltip> (MFA) pour votre locataire dans <Tooltip tip="Authentification multifacteur (MFA) : processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, comme un code envoyé par SMS." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

Lorsque vous activez la MFA, sélectionnez les facteurs (comme les notifications poussées ou les messages téléphoniques) que vous souhaitez mettre à la disposition des utilisateurs, puis définissez une stratégie pour déterminer à quel moment vous devez exiger une authentification.

<div id="select-factors">
  ## Sélectionnez les facteurs
</div>

Auth0 prend en charge divers facteurs que vous pouvez activer pour permettre à vos utilisateurs de s’authentifier.

<div id="independent-factors">
  ### Facteurs indépendants
</div>

Vous devez activer et configurer au moins un facteur indépendant pour imposer la MFA aux utilisateurs de votre locataire. Vous pouvez activer simultanément plusieurs facteurs indépendants, selon vos besoins.

Les facteurs indépendants disponibles sont :

* [WebAuthn avec des clés de sécurité FIDO](/fr-CA/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-security-keys-for-mfa)
* [Mot de passe à usage unique (OTP)](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-otp-notifications-for-mfa)
* [Notification poussée avec Auth0 Guardian](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-push-notifications-for-mfa)
* [Message téléphonique](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-sms-voice-notifications-mfa)
* [Cisco Duo Security](/fr-CA/docs/secure/multi-factor-authentication/configure-cisco-duo-for-mfa)

<div id="dependent-factors">
  ### Facteurs dépendants
</div>

Vous pouvez activer un ou plusieurs facteurs dépendants après avoir activé et configuré au moins un facteur indépendant pour les utilisateurs de votre locataire.

Les facteurs dépendants disponibles sont :

* [WebAuthn avec biométrie FIDO](/fr-CA/docs/secure/multi-factor-authentication/fido-authentication-with-webauthn/configure-webauthn-device-biometrics-for-mfa)
* [Courriel](/fr-CA/docs/secure/multi-factor-authentication/multi-factor-authentication-factors/configure-email-notifications-for-mfa)
* [Code de récupération](/fr-CA/docs/secure/multi-factor-authentication/configure-recovery-codes-for-mfa)

<div id="enable-mfa-in-the-auth0-dashboard">
  ## Activer MFA dans l’Auth0 Dashboard
</div>

1. Accédez à [Dashboard &gt; Security &gt; Multi-factor Auth](https://manage.auth0.com/#/security/mfa).
2. Dans la section **Factors**, activez et configurez les facteurs que vous souhaitez rendre disponibles aux utilisateurs.
3. Dans la section **Define policies**, sélectionnez une stratégie pour déterminer quand les utilisateurs devront utiliser MFA.

   * **Never** : MFA n’est requise pour aucune ouverture de session.
   * **Use MFA adaptative** : MFA est requise selon l’évaluation du risque effectuée par Auth0. Pour en savoir plus, consultez [Enable MFA adaptative](/fr-CA/docs/secure/multi-factor-authentication/adaptive-mfa).
   * **Always** : MFA est requise pour toutes les ouvertures de session.
4. Si vous choisissez la stratégie **Never** ou **Always**, la section **MFA Risk Assessors** s’affiche. Par défaut, le paramètre **Enable MFA adaptative Risk Assessment** est désactivé. Activez le bouton bascule pour évaluer et consigner le risque pour toutes les transactions de connexion dans les journaux de votre locataire.
5. Dans la section **Additional Settings**, configurez les options suivantes au besoin :

   * **Show Multi-factor Authentication options:** Activez ce bouton bascule pour permettre aux utilisateurs de sélectionner des facteurs d’authentification au moment de l’inscription. Lorsqu’il est désactivé, Auth0 présente automatiquement aux utilisateurs les facteurs d’authentification les plus sûrs au moment de l’inscription.

     <Frame>![Vue de l’utilisateur final de Show Multi-factor Authentication options](/docs/images/cdy7uua7fh8z/3Mo3EXUk3ZFK4hZKck0yOO/37268f33f70d1e09032f0db317ac11d3/MFA_Options_-_English.png)</Frame>
   * **Customize MFA Factors using Actions** : Activez ce bouton bascule pour personnaliser vos flux MFA à l’aide des [Actions](/fr-CA/docs/customize/actions) `post-login`. Cela vous permet d’imposer aux utilisateurs un facteur précis ou une séquence de facteurs, ainsi que d’utiliser des informations contextuelles pour créer des expériences plus personnalisées. Pour en savoir plus, consultez [Customize MFA Selection for Universal Login](/fr-CA/docs/secure/multi-factor-authentication/customize-mfa/customize-mfa-selection-universal-login).
6. Cliquez sur **Save**.

<div id="customize-mfa-flow">
  ## Personnaliser le flux MFA
</div>

Vous pouvez utiliser Auth0 Actions pour personnaliser le flux MFA pour les utilisateurs de votre locataire. Par exemple, vous pouvez créer une Action pour exiger l’utilisation de la MFA dans certaines circonstances ou pour imposer un facteur précis.

Pour en savoir plus, consultez [Personnaliser la MFA adaptative](/fr-CA/docs/secure/multi-factor-authentication/adaptive-mfa/customize-adaptive-mfa).
