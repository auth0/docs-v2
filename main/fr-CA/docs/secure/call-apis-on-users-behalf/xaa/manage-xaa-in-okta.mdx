Une fois votre environnement de test de bout en bout configuré, vous pouvez gérer la manière dont les applications XAA autorisées peuvent se connecter entre elles dans la console d’administration Okta.

<Frame>![](/docs/images/xaa/manage_xaa_in_okta.png)</Frame>

Dans la console d’administration Okta :

1. Accédez à **Applications &gt; Applications** et sélectionnez votre application ressource (par ex. Todo0).
2. Sous l’onglet **Manage Connections**, ajoutez :
   * Applications demandeuses : applications qui peuvent se connecter à votre application SaaS
   * Applications ressources : applications auxquelles votre application SaaS peut se connecter

Pour l’environnement de test de bout en bout, ajoutez Agent0, ou l’application que vous souhaitez utiliser pour les tests, comme application demandeuse autorisée.
