L’écran `mfa-push-list` affiche la liste des appareils de notification push inscrits et permet à l’utilisateur d’en sélectionner un pour l’authentification.

<div id="import">
  ## Import
</div>

Chaque écran a son propre ensemble de hooks et de méthodes. Le SDK prend en charge **l’importation partielle** et **l’importation à la racine** pour chaque écran.

* L’utilisation de l’importation partielle vous permet d’inclure uniquement le code dont vous avez besoin pour votre cas d’utilisation.
* L’utilisation de l’importation à la racine vous permet de charger tous les écrans à partir d’un seul bundle, ce qui est utile si vous souhaitez une build unifiée capable de gérer tous les écrans possibles.

```jsx Import Example
// importation racine
import { useMfaPushList } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useMfaPushList,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks utilitaires
  useChangeLanguage,
  // Méthodes
  goBack,
  selectMfaPushDevice,
} from "@auth0/auth0-acul-react/mfa-push-list";

function MfaPushListScreen() {
  const { selectMfaPushDevice } = useMfaPushList();
  return (
    <button onClick={() => selectMfaPushDevice({ deviceIndex: 0 })}>
      Select Device
    </button>
  );
}
```

<div id="context-hooks">
  ## Hooks de contexte
</div>

Hooks associés à l’écran qui fournissent un accès en lecture seule aux données de contexte Auth0 sur l’écran `mfa-push-list`. Importez-les depuis `@auth0/auth0-acul-react/mfa-push-list`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit les paramètres d’image de marque, comme le logo, les couleurs et les réglages du thème affichés sur l’écran `mfa-push-list`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/mfa-push-list';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>
  Ce hook fournit les configurations liées à l’application, comme `id`, `name` et `logoUrl`, pour l’écran `mfa-push-list`.

  ```jsx Example
  import { useClient } from '@auth0/auth0-acul-react/mfa-push-list';
  function AppInfo() {
    const client = useClient();
  }
  ```
</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des renseignements sur l’Organisation de l’utilisateur si le flux MFA est associé à une Organisation. Il retourne `null` lorsqu’aucun contexte d’Organisation n’est présent.

  ```jsx Example
  import { useOrganization } from '@auth0/auth0-acul-react/mfa-push-list';
  function OrgSelector() {
    const organization = useOrganization();
    if (!organization) {
      return <p>No organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>
  Ce hook contient des données sur l’invite actuelle dans le flux d’authentification.

  ```jsx Example
  import { usePrompt } from '@auth0/auth0-acul-react/mfa-push-list';
  function FlowInfo() {
    const prompt = usePrompt();
  }
  ```
</ParamField>

<ParamField body="useScreen" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ScreenMembers">ScreenMembers</a></span>}>
  Ce hook contient des détails propres à l’écran `mfa-push-list`, notamment sa configuration et son contexte.

  ```jsx Example
  import { useScreen } from '@auth0/auth0-acul-react/mfa-push-list';
  function ScreenDebug() {
    const screen = useScreen();
  }
  ```
</ParamField>

<ParamField body="useTenant" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/TenantMembers">TenantMembers</a></span>}>
  Ce hook contient des données liées au locataire, comme `id` et les métadonnées associées.

  ```jsx Example
  import { useTenant } from '@auth0/auth0-acul-react/mfa-push-list';
  function TenantInfo() {
    const tenant = useTenant();
  }
  ```
</ParamField>

<ParamField body="useTransaction" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/TransactionMembers">TransactionMembers</a></span>}>
  Ce hook fournit des données propres à la transaction pour l’écran `mfa-push-list`, comme l’état actuel du flux MFA.

  ```jsx Example
  import { useTransaction } from '@auth0/auth0-acul-react/mfa-push-list';
  function TransactionInfo() {
    const transaction = useTransaction();
  }
  ```
</ParamField>

<ParamField body="useUntrustedData" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UntrustedDataMembers">UntrustedDataMembers</a></span>}>
  Ce hook fournit les données non fiables transmises à l’écran, comme les valeurs préremplies provenant de paramètres d’URL.

  ```jsx Example
  import { useUntrustedData } from '@auth0/auth0-acul-react/mfa-push-list';
  function PrefilledForm() {
    const untrustedData = useUntrustedData();
  }
  ```
</ParamField>

<ParamField body="useUser" type={<span>() =&gt; <a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UserMembers">UserMembers</a></span>}>
  Ce hook fournit des renseignements sur l’utilisateur actif, notamment `username`, `email` et les méthodes d’authentification disponibles.

  ```jsx Example
  import { useUser } from '@auth0/auth0-acul-react/mfa-push-list';
  function UserProfile() {
    const user = useUser();
  }
  ```
</ParamField>

<ParamField body="useMfaPushList" type={<a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/MfaPushListMembers">MfaPushListMembers</a>}>
  Ce hook renvoie toutes les méthodes et le contexte disponibles dans l’écran `mfa-push-list`.
</ParamField>

<div id="methods">
  ## Méthodes
</div>

<ParamField body="goBack" type="Promise<void>">
  Cette méthode permet de revenir à l’écran précédent dans le flux MFA.

  ```jsx Example
  import { useMfaPushList } from '@auth0/auth0-acul-react/mfa-push-list';

  function GoBackButton() {
    const { goBack } = useMfaPushList();
    return (
      <button onClick={() => goBack()}>
        Go Back
      </button>
    );
  }
  ```
</ParamField>

<ParamField body="selectMfaPushDevice" type="Promise<void>">
  Cette méthode sélectionne un appareil de notification push dans la liste des appareils inscrits à utiliser pour la demande MFA.

  ```jsx Example
  import { useMfaPushList } from '@auth0/auth0-acul-react/mfa-push-list';

  function DeviceSelector() {
    const { selectMfaPushDevice } = useMfaPushList();
    return (
      <button onClick={() => selectMfaPushDevice({ deviceIndex: 0 })}>
        Use This Device
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [SelectMfaPushDeviceOptions](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/SelectMfaPushDeviceOptions).
    </ParamField>

    <ParamField body="deviceIndex" type="number" required>
      L’index de l’appareil à sélectionner dans la liste des appareils inscrits. L’index est basé sur 0.
    </ParamField>
  </Expandable>
</ParamField>
