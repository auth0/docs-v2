L’écran `reset-password-mfa-webauthn-roaming-challenge` s’affiche pendant le flux de réinitialisation du mot de passe lorsqu’une authentification multifacteur est requise au moyen d’une clé de sécurité itinérante. Il invite l’utilisateur à s’authentifier à l’aide de sa clé de sécurité physique externe pour poursuivre le processus de réinitialisation.

<div id="import">
  ## Importation
</div>

Chaque écran possède son propre ensemble de hooks et de méthodes. Le SDK prend en charge les **imports partiels** et les **imports racine** pour chaque écran.

* Un import partiel vous permet d’inclure uniquement le code nécessaire à votre cas d’usage.
* Un import racine vous permet de charger tous les écrans à partir d’un seul bundle, ce qui est utile si vous souhaitez un build unifié capable de gérer tous les écrans possibles.

```jsx Import Example
// importation racine
import { useResetPasswordMfaWebAuthnRoamingChallenge } from '@auth0/auth0-acul-react';

// importation partielle
import {
  useResetPasswordMfaWebAuthnRoamingChallenge,
  // Hooks de contexte
  useUser,
  useTenant,
  useBranding,
  useClient,
  useOrganization,
  usePrompt,
  useScreen,
  useTransaction,
  useUntrustedData,
  // Hooks communs
  useCurrentScreen,
  useAuth0Themes,
  useErrors,
  // Hooks utilitaires
  useChangeLanguage,
  // Méthodes
  showError,
  tryAnotherMethod,
  useSecurityKey,
} from '@auth0/auth0-acul-react/reset-password-mfa-webauthn-roaming-challenge';

function ResetPasswordMfaWebAuthnRoamingChallengeScreen() {
  const { useSecurityKey } = useResetPasswordMfaWebAuthnRoamingChallenge();
  return (
    <button onClick={() => useSecurityKey()}>
      S’authentifier avec une clé de sécurité
    </button>
  );
}
```

<div id="context-hooks">
  ## Hooks de contexte
</div>

Hooks propres à l’écran qui fournissent un accès en lecture seule aux données de contexte Auth0 sur l’écran `reset-password-mfa-webauthn-roaming-challenge`. Importez-les depuis `@auth0/auth0-acul-react/reset-password-mfa-webauthn-roaming-challenge`.

<ParamField body="useBranding" type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/BrandingMembers">BrandingMembers</a></span>}>
  Ce hook fournit des configurations d’image de marque, comme le logo, les couleurs et les paramètres de thème affichés sur l’écran `reset-password-mfa-webauthn-roaming-challenge`.

  ```jsx Example
  import { useBranding } from '@auth0/auth0-acul-react/reset-password-mfa-webauthn-roaming-challenge';
  function CustomTheme() {
    const branding = useBranding();
  }
  ```
</ParamField>

<ParamField body="useClient" type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ClientMembers">ClientMembers</a></span>}>


</ParamField>

<ParamField body="useOrganization" type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/OrganizationMembers">OrganizationMembers</a></span>}>
  Ce hook fournit des renseignements sur l’Organisation de l’utilisateur si la réinitialisation du mot de passe s’applique à une Organisation. Renvoie `null` lorsqu’aucun contexte d’Organisation n’est présent.

  ```jsx Example

    const organization = useOrganization();

      return <p>No Organization context</p>;
    }
  }
  ```
</ParamField>

<ParamField body="usePrompt" type={<span>() =&gt; <a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/PromptMembers">PromptMembers</a></span>}>

<div id="methods">
  ## Méthodes
</div>

<ParamField body="showError" type="Promise<void>">
  Cette méthode signale à Auth0 une erreur du navigateur WebAuthn lorsque le défi de la clé de sécurité échoue sur l’appareil de l’utilisateur.

  ```jsx Example
  import { useResetPasswordMfaWebAuthnRoamingChallenge } from '@auth0/auth0-acul-react/reset-password-mfa-webauthn-roaming-challenge';

  function HandleSecurityKeyError() {
    const { showError } = useResetPasswordMfaWebAuthnRoamingChallenge();
    return (
      <button onClick={() => showError({ error: { name: 'NotAllowedError', message: 'User cancelled' } })}>
        Signaler l’erreur
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [ResetPasswordMfaWebAuthnRoamingChallengeShowErrorOptions](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ResetPasswordMfaWebAuthnRoamingChallengeShowErrorOptions).
    </ParamField>

    <ParamField body="rememberDevice?" type="boolean">
      Si `true` et que `screen.showRememberDevice` est également `true`, la MFA est ignorée lors des connexions suivantes à partir du navigateur actuel.
    </ParamField>

    <ParamField body="error" type={<span><a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/WebAuthnErrorDetails">WebAuthnErrorDetails</a></span>} required>
      L’objet d’erreur provenant de l’API WebAuthn (`navigator.credentials.get()`) à signaler.

      <ParamField body="name" type="string" required>
        Le nom de l’erreur (par exemple, `'NotAllowedError'`).
      </ParamField>

      <ParamField body="message" type="string" required>
        Le message d’erreur décrivant le problème.
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="tryAnotherMethod" type="Promise<void>">
  Cette méthode permet à l’utilisateur de sélectionner une autre méthode MFA.

  ```jsx Example
  import { useResetPasswordMfaWebAuthnRoamingChallenge } from '@auth0/auth0-acul-react/reset-password-mfa-webauthn-roaming-challenge';

  function TryAnotherMethodButton() {
    const { tryAnotherMethod } = useResetPasswordMfaWebAuthnRoamingChallenge();
    return (
      <button onClick={() => tryAnotherMethod()}>
        Essayer une autre méthode
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [ResetPasswordMfaWebAuthnRoamingChallengeTryAnotherMethodOptions](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ResetPasswordMfaWebAuthnRoamingChallengeTryAnotherMethodOptions).
    </ParamField>

    <ParamField body="rememberDevice?" type="boolean">
      Si `true` et que `screen.showRememberDevice` est également `true`, la MFA est ignorée lors des connexions suivantes à partir du navigateur actuel.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="useSecurityKey" type="Promise<void>">
  Cette méthode lance le défi de la clé de sécurité itinérante pour poursuivre l’étape MFA dans le flux de Réinitialisation du mot de passe.

  ```jsx Example
  import { useResetPasswordMfaWebAuthnRoamingChallenge } from '@auth0/auth0-acul-react/reset-password-mfa-webauthn-roaming-challenge';

  function UseSecurityKeyButton() {
    const { useSecurityKey } = useResetPasswordMfaWebAuthnRoamingChallenge();
    return (
      <button onClick={() => useSecurityKey()}>
        S’authentifier avec une clé de sécurité
      </button>
    );
  }
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [ResetPasswordMfaWebAuthnRoamingChallengeUseSecurityKeyOptions](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ResetPasswordMfaWebAuthnRoamingChallengeUseSecurityKeyOptions).
    </ParamField>

    <ParamField body="rememberDevice?" type="boolean">
      Si `true` et que `screen.showRememberDevice` est également `true`, la MFA est ignorée lors des connexions suivantes à partir du navigateur actuel.
    </ParamField>
  </Expandable>
</ParamField>
