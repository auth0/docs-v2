---
title: "Alias de type : MfaPollingOptions"
---

```ts Example
type MfaPollingOptions = {
  intervalMs?: number;
  onCompleted?: () => void;
  onError?: (error: Error) => void;
}
```

<ParamField body="MfaPollingOptions" type="object">
  Options de configuration pour gérer un processus d’interrogation d’une notification poussée MFA.

  Fournissez ces options au démarrage d’une session d’interrogation pour contrôler la fréquence
  à laquelle le point de terminaison du défi est interrogé et la façon dont les réussites ou les échecs sont gérés.

  <Expandable title="propriétés">
    <ParamField body="intervalMs" type="number">
      Intervalle facultatif, en **millisecondes**, entre des requêtes d’interrogation consécutives.

      * S’il est omis, l’intervalle interne par défaut du SDK est utilisé (généralement 5000 ms).
      * Choisissez une valeur qui offre un bon équilibre entre la réactivité et les limites de débit.

      ```ts Valeur par défaut
      Dépend de l’implémentation (généralement 5000 ms)
      ```
    </ParamField>

    <ParamField body="onCompleted" type="() => void">
      Fonction de rappel facultative exécutée une fois que le défi MFA push a été approuvé avec succès
      et que l’interrogation est terminée.

      * Appelée exactement **une seule fois**, après quoi l’interrogation s’arrête automatiquement.
      * Utile pour faire avancer le flux de connexion (par ex., en appelant `continue()`).
    </ParamField>

    <ParamField body="onError" type="(error: Error) => void">
      Fonction de rappel facultative invoquée si une erreur survient pendant l’interrogation.

      Reçoit un objet [ULError](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) contenant :

      * `status` — le code d’état HTTP de la requête échouée

      * `responseText` — le corps brut de la réponse, s’il est disponible

      * Appelée une fois par événement d’erreur; l’interrogation peut être retentée selon le type d’erreur.

      * À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.

      **Propriétés**

      <ParamField body="error" type={<a href="/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error">Error</a>}>
        Objet d’erreur contenant le code d’état HTTP et le corps brut de la réponse de la requête d’interrogation échouée.
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>
