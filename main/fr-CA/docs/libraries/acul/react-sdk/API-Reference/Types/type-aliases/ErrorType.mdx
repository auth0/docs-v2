---
title: "ErrorType"
---

Type d’union de chaînes de caractères qui classe les erreurs gérées par [`useErrors`](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Hooks/useErrors). Utilisé pour filtrer les erreurs avec `errors.byType()`, il figure aussi dans chaque [`ErrorItem`](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/ErrorItem) sous la propriété `kind`.

```typescript
type ErrorType = "auth0" | "validation" | "configuration";
```

<div id="values">
  ## Valeurs
</div>

<ParamField body="&#x22;auth0&#x22;" type="ErrorType">
  Erreurs renvoyées par Auth0 ou votre propre backend pendant l’authentification ou l’autorisation.
</ParamField>

<ParamField body="&#x22;validation&#x22;" type="ErrorType">
  Erreurs de validation du formulaire côté client, par exemple une entrée invalide avant l’envoi d’une requête.
</ParamField>

<ParamField body="&#x22;configuration&#x22;" type="ErrorType">
  Erreurs causées par une mauvaise intégration du SDK ou une configuration incorrecte, comme l’appel d’un hook hors du contexte requis. S’affichent uniquement lorsque `includeDevErrors: true` est défini dans [`UseErrorOptions`](/fr-CA/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/UseErrorOptions).
</ParamField>
