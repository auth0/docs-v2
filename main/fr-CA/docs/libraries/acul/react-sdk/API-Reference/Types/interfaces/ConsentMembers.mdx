ConsentMembers

````ts Example
export interface ConsentMembers extends BaseMembers {
  /**
   * Provides access to the specific properties and data of the Consent screen,
   * including the list of `scopes` being requested and the `hideScopes` flag.
   * @type {ScreenMembersOnConsent}
   */
  screen: ScreenMembersOnConsent;

  /**
   * Soumet la décision de l'utilisateur d'accepter (donner son octroi pour) les autorisations demandées.
   * Cette action envoie une requête POST au point de terminaison `/u/consent` avec `action: "accept"`.
   *
   * @param {CustomOptions} [payload] - Facultatif. Un objet pour toute paire clé-valeur personnalisée
   *                                    à envoyer avec la requête. Ces paramètres seront
   *                                    inclus dans les données du formulaire soumis au serveur.
   * @returns {Promise<void>} Une promesse qui se résout lorsque l'action d'acceptation est soumise avec succès.
   *                          En cas de succès, Auth0 redirige généralement l'utilisateur vers l'application
   *                          ou vers l'étape suivante du flux d'authentification.
   * @throws {Error} Génère une erreur si la soumission du formulaire échoue (p. ex., problème réseau, état invalide).
   *                 Les erreurs côté serveur (comme « invalid_request ») seront reflétées dans `this.transaction.errors`
   *                 après l'opération, plutôt que d'être générées comme des erreurs JavaScript.
   *
   * @example
   * ```typescript
   * // En supposant que 'consentManager' est une instance de la classe SDK de l'écran Consent
   * try {
   *   await consentManager.accept();
   *   // En cas de succès, la page sera généralement redirigée.
   * } catch (error) {
   *   // Gérer les erreurs inattendues lors de la soumission elle-même.
   *   console.error("Échec de la soumission de l'acceptation du consentement :", error);
   * }
   * // Après l'attente, vérifier consentManager.transaction.errors pour les problèmes de validation côté serveur.
   * ```
   */
  accept(payload?: CustomOptions): Promise<void>;

  /**
   * Soumet la décision de l'utilisateur de refuser (rejeter) les autorisations demandées.
   * Cette action envoie une requête POST au point de terminaison `/u/consent` avec `action: "deny"`.
   *
   * @param {CustomOptions} [payload] - Facultatif. Un objet pour toute paire clé-valeur personnalisée
   *                                    à envoyer avec la requête. Ces paramètres seront
   *                                    inclus dans les données du formulaire soumis au serveur.
   * @returns {Promise<void>} Une promesse qui se résout lorsque l'action de refus est soumise avec succès.
   *                          En cas de succès, Auth0 redirige généralement l'utilisateur, en affichant potentiellement un
   *                          message d'accès refusé ou en retournant une erreur à l'application.
   * @throws {Error} Génère une erreur si la soumission du formulaire échoue (p. ex., problème réseau, état invalide).
   *                 Les erreurs côté serveur sont reflétées dans `this.transaction.errors`.
   *
   * @example
   * ```typescript
   * // En supposant que 'consentManager' est une instance de la classe SDK de l'écran Consent
   * try {
   *   await consentManager.deny({ reason: "user_declined" }); // Exemple d'option personnalisée
   *   // En cas de succès, la page sera généralement redirigée.
   * } catch (error) {
   *   console.error("Échec de la soumission du refus de consentement :", error);
   * }
   * // Après l'attente, vérifier consentManager.transaction.errors pour les problèmes de validation côté serveur.
   * ```
   */
  deny(payload?: CustomOptions): Promise<void>;
}

<ParamField body="accept" type="Promise<void>">
  Soumet la décision de l’utilisateur d’accepter (de donner son octroi pour) les autorisations demandées.
  Cette action envoie une requête au point de terminaison `/u/consent` avec `action: "accept"`.

  Une promesse résolue lorsque l’action d’acceptation est soumise avec succès.
  En cas de succès, Auth0 redirige généralement l’utilisateur vers l’application
  ou vers l’étape suivante du flux d’authentification.

  #### Lance

  Lance une erreur si l’envoi du formulaire échoue (p. ex., problème réseau, `state` invalide).
  Les erreurs côté serveur (comme &quot;invalid&#95;request&quot;) seront indiquées dans `this.transaction.errors`
  après l’opération, plutôt que d’être lancées comme des erreurs JavaScript.

  ```typescript Example
  // En supposant que 'consentManager' est une instance de la classe SDK de l’écran Consent
  try {
    await consentManager.accept();
    // En cas de succès, il y aura généralement une redirection.
  } catch (error) {
    // Gérer les erreurs inattendues pendant l’envoi lui-même.
    console.error("Failed to submit consent acceptance:", error);
  }
  // Après le await, vérifiez consentManager.transaction.errors pour les problèmes de validation côté serveur.
  ```

<ParamField body="deny" type="Promise<void>">
  Soumet la décision de l’utilisateur de refuser (de rejeter) les autorisations demandées.
  Cette action envoie une requête au point de terminaison `/u/consent` avec `action: "deny"`.

  Une promesse résolue lorsque l’action de refus est soumise avec succès.
  En cas de succès, Auth0 redirige généralement l’utilisateur, avec
  possibilité d’afficher un message d’accès refusé ou de renvoyer une erreur à l’application.

  #### Lance

  Lance une erreur si l’envoi du formulaire échoue (p. ex., problème réseau, `state` invalide).
  Les erreurs côté serveur sont indiquées dans `this.transaction.errors`.

  ```typescript Example
  // En supposant que 'consentManager' est une instance de la classe SDK de l’écran Consent
  try {
    await consentManager.deny({ reason: "user_declined" }); // Exemple d’option personnalisée
    // En cas de succès, il y aura généralement une redirection.
  } catch (error) {
    console.error("Failed to submit consent denial:", error);
  }
  // Après le await, vérifiez consentManager.transaction.errors pour les problèmes de validation côté serveur.
  ```
