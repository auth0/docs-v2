La classe ResetPasswordMfaPushChallengePush implémente la fonctionnalité de l’écran `reset-password-mfa-push-challenge-push`. Cet écran s’affiche lorsqu’une notification poussée a été envoyée à l’appareil de l’utilisateur dans le cadre de l’étape d’authentification multifacteur pendant un processus de réinitialisation du mot de passe.

<ParamField body="pollingManager" type={<span><a href="/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl">MfaPushPollingControl</a></span>}>
  Cette méthode démarre et gère l’interrogation du défi d’AMF par notification poussée. Elle vérifie à intervalles réguliers si la notification poussée a été approuvée et appelle le rappel `onCompleted` lorsque l’approbation est détectée. Les réponses HTTP 429 de limitation du débit sont gérées en interne — l’interrogation attend que la limite soit réinitialisée avant de réessayer.

  Le [`MfaPushPollingControl`](/fr-CA/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPushPollingControl) renvoyé expose `startPolling()`, `stopPolling()` et `isRunning()` pour gérer le cycle de vie de l’interrogation.

  ```typescript Example
  import ResetPasswordMfaPushChallengePush from '@auth0/auth0-acul-js/reset-password-mfa-push-challenge-push';
  const resetPasswordMfaPushChallengePushManager = new ResetPasswordMfaPushChallengePush();
  const control = resetPasswordMfaPushChallengePushManager.pollingManager({
    intervalMs: 5000,
    onCompleted: () => resetPasswordMfaPushChallengePushManager.continue(),
    onError: (error) => console.error('Erreur d’interrogation :', error),
  });

  control.startPolling();

  // Annuler l’interrogation au besoin
  control.stopPolling();
  ```

  **Paramètres de la méthode**

  <Expandable title="Paramètres">
    <ParamField body="options">
      [MfaPollingOptions](/fr-CA/docs/libraries/acul/js-sdk/Screens/interfaces/MfaPollingOptions).
    </ParamField>

    <ParamField body="intervalMs?" type="number">
      L’intervalle, en millisecondes, entre les tentatives d’interrogation.
    </ParamField>

    <ParamField body="onCompleted?" type="() => void">
      Rappel appelé lorsque le défi par notification poussée est approuvé et que l’interrogation s’arrête automatiquement.
    </ParamField>

    <ParamField body="onError?" type="(error: Error) => void">
      Rappel appelé si une erreur survient pendant l’interrogation.
    </ParamField>
  </Expandable>
</ParamField>
