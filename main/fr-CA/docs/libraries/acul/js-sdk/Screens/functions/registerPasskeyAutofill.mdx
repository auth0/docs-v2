---
title: "registerPasskeyAutofill"
---

> **registerPasskeyAutofill**(`inputId`?: `string`): `Promise`&lt;`void`&gt;

Initialise une demande passive d’identifiant WebAuthn à l’aide de `navigator.credentials.get()` avec `mediation: "conditional"`. Lorsque cette fonctionnalité est prise en charge, le navigateur peut afficher les clés d’accès enregistrées directement dans le menu déroulant de saisie semi-automatique du champ de nom d’utilisateur. La sélection d’une clé d’accès complète l’authentification sans autre interaction de l’utilisateur.

Appelez cette méthode une seule fois lorsque l’écran de connexion est initialisé (par exemple, au chargement de la page). Si la médiation conditionnelle n’est pas prise en charge par le navigateur, la méthode n’a simplement aucun effet.

<div id="parameters">
  ## Paramètres
</div>

<ParamField body="inputId?" type="string">
  L’id de l’élément `<input>` pour le nom d’utilisateur (sans `#`). Exemple : `"username"`. S’il est fourni, le SDK valide que l’élément existe, confirme qu’il s’agit bien d’un `<input>` et remplace la valeur de son attribut `autocomplete` par `"username webauthn"`.

  S’il est omis, vous devez configurer l’élément d’entrée manuellement :

  ```html
  <input id="username" autocomplete="username webauthn" />
  ```
</ParamField>

<div id="returns">
  ## Retourne
</div>

`Promise`&lt;`void`&gt;

Est résolue une fois l’enregistrement de l’interface utilisateur conditionnelle terminé.

<div id="example">
  ## Exemple
</div>

```typescript
import LoginId from '@auth0/auth0-acul-js/login-id';

async function initializeLogin() {
  const loginId = new LoginId();
  // Assurez-vous que l'élément HTML input associé existe :
  // <input id="username" autocomplete="username webauthn" />
  await loginId.registerPasskeyAutofill('username');
}

initializeLogin().catch(console.error);
```

<div id="gotchas">
  ## Pièges à éviter
</div>

* L’attribut `autocomplete` **doit contenir exactement** `"username webauthn"`. L’ajout de jetons non pertinents comme `"email"` ou `"text"` empêchera les navigateurs d’afficher le menu déroulant des clés d’accès.
* Le fait d’écraser l’attribut est intentionnel et nécessaire pour garantir un comportement cohérent d’un navigateur à l’autre. Ne vous fiez pas à la fusion ou à l’ajout de valeurs `autocomplete` existantes.
* Si la médiation conditionnelle n’est pas prise en charge par le navigateur, le SDK n’effectuera simplement aucune opération.

<div id="remarks">
  ## Remarques
</div>

Cette méthode délègue à l’utilitaire interne `registerPasskeyAutofill()` et renvoie un `AbortController` exécuté en arrière-plan pour gérer le cycle de vie de la requête. Elle ne doit être appelée qu’une seule fois par cycle de vie de la page.
