   * Cette action correspond à l'utilisateur qui saisit un nouveau surnom et clique sur un bouton « Enregistrer » ou « Continuer ».
   * Le SDK enverra ce nouveau surnom via POST au point de terminaison Auth0 `/u/mfa-webauthn-change-key-nickname`.

   *                 ou si le `FormHandler` rencontre un problème irrécupérable lors de la soumission (p. ex., erreur réseau).
   *                 Les erreurs de validation provenant d'Auth0 (comme un surnom invalide) ne sont pas générées en tant qu'erreurs JavaScript,
   *                 mais sont reflétées dans `this.transaction.errors` après l'opération.
   *
   * @example

   * const newNickname = "My Favorite YubiKey"; // Valeur provenant de la saisie de l'utilisateur

   *   await sdk.continueWithNewNickname({ nickname: newNickname });
   *   // En cas de succès, la page est redirigée.
   * } catch (error) {
   *   // Ce bloc catch est destiné aux erreurs inattendues lors de l'appel SDK lui-même.
   *   console.error("Failed to submit the new nickname:", error);
   * }
   * // Après l'await, vérifiez toujours sdk.transaction.errors pour les problèmes de validation côté serveur.
   * if (sdk.transaction.errors && sdk.transaction.errors.length > 0) {

   *       // p. ex., « Le nom est trop court », « Le nom est requis »

   *   });

   */

<ParamField body="continueWithNewNickname" type="Promise<void>">
  Soumet le nouveau surnom de la clé de sécurité WebAuthn.
  Cette action correspond au moment où l’utilisateur saisit un nouveau surnom et clique sur un bouton « Enregistrer » ou « Continuer ».
  Le SDK enverra ce nouveau surnom au point de terminaison Auth0 `/u/mfa-webauthn-change-key-nickname` au moyen d’une requête POST.

  Si le nouveau surnom est valide et que la mise à jour réussit, Auth0 redirigera généralement l’utilisateur
  vers l’écran approprié suivant (p. ex., retour à la gestion des facteurs MFA ou confirmation de réussite).
  Si le surnom n’est pas valide (p. ex., vide, trop long ou trop court), Auth0 affichera de nouveau l’écran
  &#39;mfa-webauthn-change-key-nickname&#39;, et le tableau `transaction.errors` dans le contexte du SDK
  sera mis à jour avec des détails sur l’échec de validation.

  Une promesse qui se résout lorsque l’envoi du formulaire est lancé.
  Elle ne renvoie pas directement de données à la résolution, puisqu’une redirection ou un
  nouveau rendu de la page est généralement le résultat attendu.

  #### Lance

  Lance une erreur si `payload.nickname` n’est pas fourni ou n’est pas une chaîne de caractères,
  ou si le `FormHandler` rencontre un problème irrécupérable pendant l’envoi (p. ex., une erreur réseau).
  Les erreurs de validation provenant d’Auth0 (comme un surnom invalide) ne sont pas lancées comme erreurs JavaScript,
  mais se reflètent plutôt dans `this.transaction.errors` après l’opération.

  ```typescript Example
  // En supposant que 'sdk' est une instance de MfaWebAuthnChangeKeyNickname
  const newNickname = "My Favorite YubiKey"; // Valeur provenant de la saisie de l’utilisateur
  try {
    await sdk.continueWithNewNickname({ nickname: newNickname });
    // En cas de réussite, la page redirige.
  } catch (error) {
    // Ce catch sert aux erreurs inattendues pendant l’appel du SDK lui-même.
    console.error("Échec de l’envoi du nouveau surnom :", error);
  }
  // Après le await, vérifiez toujours sdk.transaction.errors pour les problèmes de validation côté serveur.
  if (sdk.transaction.errors && sdk.transaction.errors.length > 0) {
    sdk.transaction.errors.forEach(err => {
      if (err.field === 'nickname') {
        // Affichez err.message lié au champ de saisie du surnom.
        // p. ex., "Le nom est trop court", "Le nom est requis"
      }
    });
  }
  ```
