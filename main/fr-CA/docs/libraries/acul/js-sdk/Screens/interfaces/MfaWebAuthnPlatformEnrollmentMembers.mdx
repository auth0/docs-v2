<ParamField body="submitPasskeyCredential" type="Promise<void>">
  Lance le processus de création des informations d’identification de plateforme WebAuthn à l’aide des options de clé publique
  disponibles dans `this.screen.publicKey`, puis transmet les informations d’identification obtenues au serveur.
  Cette méthode appelle en interne `createPasskeyCredentials` (qui encapsule `navigator.credentials.create()`).

  Une promesse qui se résout lorsque les informations d’identification sont créées et transmises avec succès.

  #### Lève

  Lève une erreur si `this.screen.publicKey` n’est pas disponible, si `createPasskeyCredentials` échoue
  (par ex. annulation par l’utilisateur, problèmes matériels), ou si l’envoi au serveur échoue.

  ```typescript Example
  // En supposant que 'sdk' est une instance de MfaWebAuthnPlatformEnrollment
  try {
    if (!sdk.screen.publicKey) { // Vérifie si les options sont disponibles
      throw new Error("Public key creation options are not available on the screen context.");
    }
    await sdk.submitPasskeyCredential(); // Inutile de transmettre publicKey explicitement
    // En cas de réussite, Auth0 gère la redirection.
  } catch (error) {
    console.error('Passkey enrollment failed:', error);
    // Gère l'erreur, potentiellement en appelant reportBrowserError s'il s'agit d'une erreur de l'API WebAuthn
    if (error.name && error.message) { // Vérifie si cela ressemble à une erreur WebAuthn
      await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
    }
  }
  ```
