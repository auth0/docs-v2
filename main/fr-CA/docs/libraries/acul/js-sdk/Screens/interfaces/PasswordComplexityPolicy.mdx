---
title: "PasswordComplexityPolicy"
---

Définit les règles de complexité du mot de passe appliquées lors des flux d’inscription ou de réinitialisation de mot de passe.

```ts Example
export interface PasswordComplexityPolicy {
  character_type_rule: 'all' | 'at_least';
  character_types: ('number' | 'uppercase' | 'lowercase' | 'special')[];
  identical_characters: 'block' | 'allow';
  max_length_exceeded: 'allow' | 'error';
  min_length: number;
  sequential_characters: 'block' | 'allow';
}
```

<div id="properties">
  ## Propriétés
</div>

<ParamField body="character_type_rule" type="&#x22;all&#x22; | &#x22;at_least&#x22;" required>
  Détermine si tous les types de caractères indiqués doivent être présents (`'all'`) ou seulement certains d’entre eux (`'at_least'`).
</ParamField>

<ParamField body="character_types" type="(&#x22;number&#x22; | &#x22;uppercase&#x22; | &#x22;lowercase&#x22; | &#x22;special&#x22;)[]" required>
  Précise quelles catégories de caractères doivent être exigées dans le mot de passe.
</ParamField>

<ParamField body="identical_characters" type="&#x22;block&#x22; | &#x22;allow&#x22;" required>
  Contrôle si les caractères identiques répétés sont autorisés (`'allow'`) ou bloqués (`'block'`).
</ParamField>

<ParamField body="max_length_exceeded" type="&#x22;allow&#x22; | &#x22;error&#x22;" required>
  Définit le comportement lorsque le mot de passe dépasse la longueur maximale — l’autoriser sans erreur ou renvoyer une erreur.
</ParamField>

<ParamField body="min_length" type="number" required>
  Nombre minimal de caractères requis pour le mot de passe.
</ParamField>

<ParamField body="sequential_characters" type="&#x22;block&#x22; | &#x22;allow&#x22;" required>
  Détermine si les suites de caractères consécutifs (p. ex., `'abc'`, `'123'`) sont autorisées (`'allow'`) ou bloquées (`'block'`).
</ParamField>
