Le SDK JS ACUL d’Auth0 vous permet de créer des expériences Universal Login personnalisées avec un minimum d’efforts. Que vous mettiez en œuvre la connexion, l’inscription, Passwordless ou l’inscription de clés d’accès, ce SDK est conçu pour simplifier les flux d’authentification modernes.

<div id="why-choose-the-auth0-acul-js-sdk">
  ### Pourquoi choisir le SDK JS ACUL d’Auth0 ?
</div>

* **Intégration rapide :** Intégrez des écrans prédéfinis comme `login-id` ou `signup-password`.
* **Fonctionnalités modernes :** Prise en charge de Passwordless, des passkeys et des connexions sociales.
* **Modules flexibles :** Importez uniquement ce dont vous avez besoin.
* **Gestion unifiée de l’état :** Utilisez un seul objet de transaction pour gérer tous les flux d’authentification.

<div id="architecture">
  ### Architecture
</div>

<div id="how-it-works">
  #### Comment ça fonctionne?
</div>

Le SDK est conçu dans un souci de modularité et d’évolutivité. Voici comment ses composantes s’articulent :

* **Écrans :** Chaque flux d’authentification, comme `login-id` et `signup-password`, constitue un module autonome.
* **Gestionnaires de soumission :** Gèrent l’envoi des formulaires et les transitions.
* **Formulaires dynamiques :** Adaptent les champs et les validations selon la configuration de l’écran.
* **Propriétés utilitaires :** Permettent d’accéder de façon cohérente à des données communes, comme l’image de marque ou la transaction.
