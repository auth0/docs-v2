Lorsque l’authentification est effectuée avec le scope `offline_access`, un <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Refresh+Token">Jeton d’actualisation</Tooltip> est renvoyé avec les informations d’identification. Cette valeur peut servir à demander un nouveau <Tooltip tip="Jeton d’actualisation : jeton utilisé pour obtenir un nouveau jeton d’accès sans obliger les utilisateurs à se reconnecter." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Access+Token">Jeton d’accès</Tooltip> et à éviter de redemander ses informations d’identification à l’utilisateur.

Les jetons doivent être stockés dans un espace de stockage sécurisé après une authentification réussie. Gardez à l’esprit que les jetons d’actualisation **n’expirent jamais**. Pour demander un nouveau jeton, utilisez `AuthenticationAPIClient` d’A`uth0.Android`.

<div id="using-refresh-token">
  ## Utiliser le jeton d’actualisation
</div>

```kotlin lines
val refreshToken: String = // Récupérer le Jeton d'actualisation depuis un stockage sécurisé
val account = Auth0(this)

val client = AuthenticationAPIClient(account)
client.renewAuth(refreshToken)
  .start(object: Callback<Credentials, AuthenticationException> {
  override fun onFailure(exception: AuthenticationException) {
       // Erreur
   }

   override fun onSuccess(credentials: Credentials) {
       // Utiliser les informations d'identification
   }
})
```
