<div id="native-provider-google">
  ## Fournisseur natif - Google
</div>

Vous pouvez utiliser Google AuthProvider pour vous connecter avec ou sans **Lock**. Assurez-vous de suivre les instructions de la section [configuration](#setup).

[Lock-Google.Android](https://github.com/okta-graveyard/Lock-Google.Android)
nécessite l’API Android 15 ou une version ultérieure, ainsi que Google Play Services 10.+

<div id="latest-version-of-lock-google">
  ### Dernière version de Lock-Google
</div>

Lock-Google est disponible sur [Maven Central](https://central.sonatype.com/). Pour l’installer, ajoutez simplement la ligne suivante à votre `build.gradle` :

`compile 'com.auth0.android:lock-google:1.+'`

Vous pouvez consulter la version la plus récente dans le fichier [Readme](https://github.com/okta-graveyard/Lock-Google.Android#install) du dépôt ou dans [Maven](https://central.sonatype.com/artifact/com.auth0.android/lock-google).

<div id="lock-google-setup">
  ### Configuration de Lock-Google
</div>

<div id="google-cloud-console">
  #### Console Google Cloud
</div>

1. Accédez à la [console Google Cloud](https://console.cloud.google.com) et créez un nouveau projet.
2. Remplissez l’[écran de consentement OAuth](https://console.cloud.google.com/apis/credentials/consent) en fournissant au minimum une adresse de courriel valide et un nom.
3. Dans le menu de navigation de gauche, cliquez sur [Identifiants](https://console.cloud.google.com/apis/credentials).
4. Créez un nouvel identifiant en cliquant sur le bouton [Créer des identifiants](https://console.cloud.google.com/apis/credentials/oauthclient) et en choisissant **ID client OAuth**. Ensuite, choisissez **Application Web** et donnez-lui un nom comme « Auth0 Server Google-OAuth ». Remplissez le champ **URI de redirection autorisées** avec votre URL de rappel, qui devrait ressembler à `https://{yourDomain}/login/callback`. Assurez-vous d’appuyer sur ENTRÉE avant de quitter le champ, puis cliquez sur le bouton Créer. Prenez en note les valeurs `CLIENT ID` et `CLIENT SECRET`, car nous les utiliserons plus tard.
5. Créez un nouvel identifiant en cliquant sur le bouton [Créer des identifiants](https://console.cloud.google.com/apis/credentials/oauthclient) et en choisissant **ID client OAuth**. Ensuite, choisissez **Android** et donnez-lui un nom comme « Auth0 Android Google-OAuth ». Obtenez le **SHA-1** du certificat que vous utilisez pour signer votre application et remplissez le premier champ avec cette valeur. Si vous avez besoin d’aide pour obtenir le SHA-1, consultez [cette](#certificate-fingerprints) section. Enfin, remplissez le dernier champ avec le **Package Name** de votre application Android, puis cliquez sur le bouton Créer. Prenez en note la valeur `CLIENT ID`, car nous l’utiliserons plus tard.

<div id="auth0-dashboard">
  #### Auth0 Dashboard
</div>

1. Allez dans l’Auth0 Dashboard et cliquez sur [Connexions sociales](https://manage.auth0.com/#/connections/social).
2. Cliquez sur **Google**; une boîte de dialogue s’ouvrira.
3. Remplissez le champ « ID client » avec la valeur `CLIENT ID` obtenue à l’étape 4 de la section **Google Cloud Console** ci-dessus.
4. Remplissez le champ « Secret client » avec la valeur `CLIENT SECRET` obtenue à l’étape 4 de la section **Google Cloud Console** ci-dessus.
5. Remplissez le champ « ID clients mobiles autorisés » avec le `CLIENT ID` obtenu à l’étape 5 de la section **Google Cloud Console** ci-dessus.
6. Cliquez sur le bouton Enregistrer.
7. Allez dans l’Auth0 Dashboard et cliquez sur [Applications](https://manage.auth0.com/#/applications). Si vous n’avez pas encore créé d’application, faites-le d’abord, puis accédez à la page de configuration de votre application.
8. Au bas de la page, cliquez sur le lien « Afficher les paramètres avancés » et accédez à l’onglet « Paramètres de l’appareil ».
9. Dans la section Android, remplissez le champ **Package Name** avec le Package Name de votre application. Enfin, remplissez le champ **Key Hashes** avec le SHA-256 du certificat que vous utilisez pour signer votre application. Si vous avez besoin d’aide pour obtenir le SHA-256, consultez [cette section](#certificate-fingerprints). Cliquez sur le bouton « Enregistrer les modifications ».

<div id="android-application">
  #### Application Android
</div>

1. Dans votre application Android, créez une nouvelle ressource de chaîne dans le fichier `res/strings.xml`. Nommez-la `google_server_client_id` et définissez sa valeur sur le `CLIENT_ID` obtenu à l’étape 5 de la section de configuration de **Google Cloud Console** ci-dessus.
2. Ajoutez les métadonnées de version de Google Play Services au fichier `AndroidManifest.xml`, à l’intérieur de la balise `application`.

   ```xml lines
   <meta-data
       android:name="com.google.android.gms.version"
       android:value="@integer/google_play_services_version" />
   ```
3. Ajoutez la permission d’accès à Internet à votre fichier `AndroidManifest.xml`.

   ```xml lines
   <uses-permission android:name="android.permission.INTERNET" />
   ```
4. Lors de la création d’une nouvelle instance de `GoogleAuthProvider`, passez la valeur `google_server_client_id` obtenue précédemment comme premier paramètre :

   ```kotlin lines
   public class MainActivity extends AppCompatActivity {
     private GoogleAuthProvider provider;
     // ...

     @Override
     protected void onCreate(Bundle savedInstanceState) {
       Auth0 auth0 = new Auth0(getString(R.string.com_auth0_client_id), getString(R.string.com_auth0_domain));
       auth0.setOIDCConformant(true);
       AuthenticationAPIClient client = new AuthenticationAPIClient(auth0);
       provider = new GoogleAuthProvider(getString(R.string.google_server_client_id), client);
     }

     // ...
   }
   ```

Si vous avez besoin d’aide supplémentaire pour configurer le SDK Google, consultez le [guide Sign-In for Android de Google](https://developers.google.com/identity/android-credential-manager).

<div id="usage-without-lock">
  ### Utilisation sans Lock
</div>

Si vous comptez utiliser ce fournisseur sans **Lock**, assurez-vous de surcharger la méthode `onActivityResult()` et de rediriger l’appel vers l’instance du fournisseur. Enfin, appelez start pour démarrer le processus d’authentification.

```kotlin lines
// Définissez vos propres codes de requête
private static final int RC_PERMISSIONS = 101;
private static final int RC_AUTHENTICATION = 102;

@Override
protected void onActivityResult(int requestCode, int resultCode, Intent data) {
    if (provider.authorize(requestCode, resultCode, data)) {
        return;
    }
    super.onActivityResult(requestCode, resultCode, data);
}

private void beginAuthentication(){
  provider.start(this, callback, RC_PERMISSIONS, RC_AUTHENTICATION);
}
```

Ça y est, vous êtes prêt à lancer l’application et à vous connecter avec le fournisseur natif de Google !!

<div id="additional-options">
  ### Options supplémentaires
</div>

<div id="using-a-custom-connection-name">
  #### Utiliser un nom de connexion personnalisé
</div>

Pour utiliser un nom personnalisé pour la connexion sociale lors de l’autorisation auprès d’Auth0, créez l’instance `GoogleAuthProvider` à l’aide du second constructeur :

```kotlin wrap lines
GoogleAuthProvider provider = new GoogleAuthProvider("my-connection", "google-server-client-id", client);
```

<div id="send-additional-authentication-parameters">
  #### Envoyer des paramètres d’authentification supplémentaires
</div>

Pour envoyer des paramètres supplémentaires lors de l’appel d’authentification, utilisez `setParameters`.

```kotlin lines
Map<String, Object> parameters = new HashMap<>();
//Ajouter des entrées
provider.setParameters(parameters);
```

<div id="requesting-a-custom-google-scope">
  #### Demander un scope Google personnalisé
</div>

Par défaut, le scope `Scopes.PLUS_LOGIN` est demandé. Vous pouvez personnaliser les scopes en appelant `setScopes` avec la liste des scopes. Chaque API Google (Auth, Drive, Plus..) spécifie sa propre liste de scopes Google.

```kotlin wrap lines
provider.setScopes(Arrays.asList(new Scope(Scopes.PLUS_ME), new Scope(Scopes.PLUS_LOGIN)));
```

<div id="requesting-custom-android-runtime-permissions">
  #### Demander des permissions d’exécution Android personnalisées
</div>

Ce fournisseur ne nécessite aucune permission spéciale dans le manifeste Android pour authentifier l’utilisateur. Toutefois, si votre cas d’utilisation l’exige, vous pouvez laisser l’`AuthProvider` les gérer pour vous. Utilisez la méthode `setRequiredPermissions`.

```kotlin wrap lines
provider.setRequiredPermissions(new String[]{"android.permission.GET_ACCOUNTS"});
```

Si vous n&#39;utilisez pas Lock, vous devrez gérer vous-même le résultat de la demande de permission. Pour ce faire, faites en sorte que votre activité implémente l&#39;interface `ActivityCompat.OnRequestPermissionsResultCallback`. Lorsque la méthode `onRequestPermissionsResult` est appelée, transmettez le résultat au provider en appelant `provider.onRequestPermissionsResult`.

<div id="log-out-clear-account">
  #### Se déconnecter / effacer le compte.
</div>

Pour déconnecter l’utilisateur afin qu’on lui demande de nouveau ses identifiants la prochaine fois, appelez `clearSession`. Après cela, l’état du fournisseur ne sera plus valide et vous devrez appeler `start` de nouveau avant d’essayer d’obtenir un résultat avec `authorize`. L’appel à `stop` a le même effet.

`provider.clearSession();`

<div id="remember-the-last-login">
  #### Se souvenir de la dernière connexion
</div>

Par défaut, ce fournisseur se souvient du dernier compte utilisé pour se connecter. Si vous souhaitez modifier ce comportement, utilisez la méthode suivante.

`provider.rememberLastLogin(false);`

<div id="certificate-fingerprints">
  #### Empreintes du certificat
</div>

Lorsque vous créez de nouvelles informations d’identification <Tooltip tip="OAuth 2.0 : cadre d'autorisation qui définit les protocoles et les flux d’autorisation." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=OAuth">OAuth</Tooltip> dans Google Cloud Console, vous devez fournir l’empreinte SHA-1 du certificat que vous utilisez pour signer votre application. Lorsque vous terminez la configuration de votre application dans l’<Tooltip tip="Auth0 Dashboard : principal produit Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>, vous devez également fournir la valeur SHA-256. Voici un exemple de commande à exécuter dans le terminal pour obtenir cette valeur, ainsi qu’un exemple de résultat.

Commande :

`keytool -exportcert -alias androiddebugkey -keystore <PATH_TO_YOUR_KEYSTORE> -storepass android | openssl sha1 -binary | openssl base64`

Exemple de sortie :

`no71633JAC3qgzQYCbskprUr55k=`

Si vous avez besoin d’aide, vous pouvez suivre ce [guide des keystores](/fr-CA/docs/libraries/auth0-android/android-development-keystores-hashes) pour obtenir ces valeurs.

<div id="lock-facebook-setup">
  ### Configuration de Lock-Facebook
</div>

<div id="facebook-developers-console">
  #### Console des développeurs Facebook
</div>

1. Accédez à la [console des développeurs Facebook](https://developers.facebook.com/) et créez une nouvelle application : choisissez « Android » et donnez-lui un nom valide. Cliquez sur « Create new Facebook App ID ».
2. Ajoutez le **Package Name** de votre application ainsi que le nom de la **classe Activity** dans laquelle vous utilisez le fournisseur, puis cliquez sur le bouton Next.
3. Ajoutez les **Key Hashes** **SHA-1** encodés en Base64 des certificats que vous utilisez pour signer votre application, puis cliquez sur le bouton Next. Si vous avez besoin d’aide pour obtenir le SHA-1, consultez [cette section](#certificate-fingerprints).
4. Enfin, revenez en haut de la page et cliquez sur le bouton Skip Quickstart pour accéder à la page de votre application Facebook.
5. En haut de la page, vous trouverez les valeurs `APP ID` et `APP SECRET`. Enregistrez-les, car vous en aurez besoin plus tard.
6. À gauche, vous trouverez le menu de navigation. Cliquez sur Settings, puis sur Basic. Activez l’option **Single Sign-On** et cliquez sur le bouton Save.
7. Cliquez sur Settings, puis sur Advanced. Activez l’option **Native or desktop app?**.

<div id="auth0-dashboard">
  #### Auth0 Dashboard
</div>

1. Accédez à l’Auth0 Dashboard et cliquez sur [Connexions sociales](https://manage.auth0.com/#/connections/social).
2. Cliquez sur **Facebook**; une boîte de dialogue s’ouvrira.
3. Remplissez le champ « App ID » avec la valeur `APP ID` obtenue à l’étape 5 de la section **Facebook Developers Console** ci-dessus.
4. Remplissez le champ « App Secret » avec la valeur `APP SECRET` obtenue à l’étape 5 de la section **Facebook Developers Console** ci-dessus.
5. Cliquez sur le bouton **Save**.
6. Accédez à l’Auth0 Dashboard et cliquez sur [Applications](https://manage.auth0.com/#/applications). Si vous n’en avez pas encore créé, faites-le d’abord, puis accédez à la page de configuration de votre application.
7. Au bas de la page, cliquez sur le lien « Show Advanced Settings », puis accédez à l’onglet « Device Settings ».
8. Dans la section Android, indiquez le **Package Name** de votre application. Enfin, remplissez le champ **Key Hashes** avec le SHA-256 du certificat que vous utilisez pour signer votre application. Si vous avez besoin d’aide pour obtenir le SHA-256, consultez [cette](#certificate-fingerprints) section. Cliquez sur le bouton « Save Changes ».

<div id="android-application">
  #### Application Android
</div>

1. Dans votre application Android, créez une nouvelle ressource de chaîne dans le fichier `res/strings.xml`. Nommez-la `facebook_app_id` et définissez sa valeur sur `APP ID`, obtenu à l’étape 5 de la section de configuration **Facebook Developers Console** ci-dessus.
2. Ajoutez `FacebookActivity` et les métadonnées `facebook_app_id` au fichier `AndroidManifest.xml`, à l’intérieur de la balise `application`.

   ```xml lines
   <activity
       android:name="com.facebook.FacebookActivity"
       android:configChanges="keyboard|keyboardHidden|screenLayout|screenSize|orientation"
       android:label="@string/app_name"/>
   <meta-data
       android:name="com.facebook.sdk.ApplicationId"
       android:value="@string/facebook_app_id" />
   ```
3. Ajoutez la permission d’accès à Internet à votre fichier `AndroidManifest.xml`.

   ```xml lines
   <uses-permission android:name="android.permission.INTERNET" />
   ```
4. Créez une nouvelle instance de `FacebookAuthProvider`.

   ```kotlin lines
   public class MainActivity extends AppCompatActivity {
     private FacebookAuthProvider provider;
     // ...

     @Override
     protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       Auth0 auth0 = new Auth0(getString(R.string.com_auth0_client_id), getString(R.string.com_auth0_domain));
       auth0.setOIDCConformant(true);
       AuthenticationAPIClient client = new AuthenticationAPIClient(auth0);
       provider = new FacebookAuthProvider(client);
     }

     // ...
   }
   ```

Si vous avez besoin d’aide supplémentaire pour configurer le SDK Facebook, consultez le [guide de démarrage](https://developers.facebook.com/docs/android/getting-started) de Facebook.
