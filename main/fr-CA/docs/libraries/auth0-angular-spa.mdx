---

Le SDK Auth0 Angular est une bibliothèque JavaScript qui permet d’implémenter l’authentification et l’autorisation dans des applications Angular avec Auth0. Il fournit un service, une garde d’authentification et un intercepteur HTTP pour vous permettre d’exécuter les tâches d’authentification courantes dans vos applications Angular.

Le SDK Auth0 Angular gère les détails des octrois et des protocoles, l’expiration et le renouvellement des jetons, ainsi que le stockage et la mise en cache des jetons. En coulisse, il implémente [Universal Login](/fr-CA/docs/authenticate/login/auth0-universal-login) et le [flux d’autorisation par code avec PKCE](/fr-CA/docs/get-started/authentication-and-authorization-flow/authorization-code-flow-with-pkce).

La bibliothèque est [hébergée sur GitHub](https://github.com/auth0/auth0-angular), où vous pouvez [en savoir plus sur l’API](https://auth0.github.io/auth0-angular/).

<div id="installation">
  ## Installation
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Le SDK Angular est compatible avec les applications monopages Angular version 14 et ultérieures.
</Callout>

Utilisez l’[Angular CLI](https://cli.angular.io/) pour installer le SDK dans votre projet :

`ng add @auth0/auth0-angular`

<div id="getting-started">
  ## Pour commencer
</div>

Commencez l’intégration du SDK en important `AuthModule` dans votre projet, puis configurez-le pour votre application Auth0 :

```jsx lines
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { AppComponent } from './app.component';

// Importer le module depuis le SDK
import { AuthModule } from '@auth0/auth0-angular';

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,

    // Importer le module dans l'application, avec la configuration
    AuthModule.forRoot({
      domain: '{yourDomain}',
      clientId: 'YOUR_AUTH0_CLIENT_ID',
      authorizationParams: {
        redirect_uri: window.location.origin,
      },
    }),
  ],

  bootstrap: [AppComponent],
})
export class AppModule {}
```

<div id="login">
  ## Connexion
</div>

Le SDK prend entièrement en charge le système d’injection de dépendances d’Angular. Pour commencer, importez le type `AuthService` dans un composant et injectez-le dans le constructeur.

Ensuite, pour lancer la procédure de connexion de l’utilisateur, appelez les méthodes `loginWithRedirect` ou `loginWithPopup` sur le service :

```jsx lines
import { Component } from '@angular/core';

// Importer le type AuthService depuis le SDK
import { AuthService } from '@auth0/auth0-angular';

@Component({
  selector: 'app-login',
  template: '<button (click)="login">Log in</button>',
  styles: [],
})
export class LoginComponent {
  // Injecter le service d'authentification dans votre composant via le constructeur
  constructor(public auth: AuthService) {}

  login(): void {
    // Appeler cette méthode pour rediriger l'utilisateur vers la page de connexion
    this.auth.loginWithRedirect();
  }
}
```

<div id="logout">
  ## Déconnexion
</div>

Utilisez `logout` pour déconnecter vos utilisateurs. Assurez-vous que `returnTo` figure dans **Allowed Logout URLs** de votre <Tooltip tip="Auth0 Dashboard : console principale d’Auth0 pour configurer vos services." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

```jsx lines
import { Component } from '@angular/core';

// Importer le type AuthService depuis le SDK
import { AuthService } from '@auth0/auth0-angular';

@Component({
  selector: 'app-logout',
  template: '<button (click)="logout">Log out</button>',
  styles: [],
})
export class LogoutComponent {
  // Injecter le service d'authentification dans votre composant via le constructeur
  constructor(@Inject(DOCUMENT) private doc: Document, public auth: AuthService) {}

  login(): void {
    // Appeler cette méthode pour rediriger l'utilisateur vers la page de connexion
    this.auth.logout({ 
      logoutParams: { 
        returnTo: this.doc.location.origin 
      }
    });
  }
}
```

<div id="authentication-state">
  ## État d’authentification
</div>

L’état d’authentification est exposé via l’observable `isAuthenticated$` :

```html lines
<div *ngIf="auth.isAuthenticated$ | async">
  <app-login></app-login>
</div>
```

<div id="user-profile">
  ## Profil de l’utilisateur
</div>

Le profil de l’utilisateur authentifié est accessible via l’observable `user$` :

```html lines
<ul *ngIf="auth.user$ | async as user">
  <li>{{ user.name }}</li>
  <li>{{ user.email }}</li>
</ul>
```

En interne, cet observable est enchaîné à l’observable `isAuthenticated$`; il n’est donc pas nécessaire de vérifier manuellement si l’utilisateur est connecté. Il commencera à émettre des valeurs dès qu’un utilisateur authentifié sera disponible.

<div id="protect-a-route">
  ## Protéger une route
</div>

Protégez un composant de route à l’aide du `AuthGuard` intégré. Les accès non authentifiés à cette route redirigent l’utilisateur vers la page de connexion, puis le ramènent à cette page après la connexion.

La garde d’authentification doit être ajoutée à une définition de route à l’aide du hook `canActivate` (ou `canActivateChild` si vous utilisez des routes enfants) :

```jsx lines expandable
import { NgModule } from '@angular/core';
import { Routes, RouterModule } from '@angular/router';
import { HomeComponent } from './unprotected/unprotected.component';
import { ProtectedComponent } from './protected/protected.component';

// Importer le garde d'authentification
import { AuthGuard } from '@auth0/auth0-angular';

const routes: Routes = [
  {
    path: 'protected',
    component: ProtectedComponent,

    // Protéger une route en enregistrant le garde d'authentification dans le hook `canActivate`
    canActivate: [AuthGuard],
  },
  {
    path: '',
    component: HomeComponent,
    pathMatch: 'full',
  },
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule],
})
export class AppRoutingModule {}
```

La garde d’authentification prend également en charge le hook `canLoad` pour empêcher le chargement des modules chargés à la demande lorsque l’utilisateur n’est pas authentifié.

## Appeler une API

Le SDK comprend un `HttpInterceptor` intégré qui ajoute automatiquement des <Tooltip tip="Jeton d’accès : justificatif d’autorisation, sous la forme d’une chaîne opaque ou d’un JWT, utilisé pour accéder à une API." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=access+tokens">jetons d’accès</Tooltip> en injectant un en-tête `Authorization` lorsque des requêtes sont effectuées à l’aide du service `HttpClient` d’Angular.

Vous devez configurer le SDK pour préciser quelles routes de l’API doivent recevoir automatiquement cet en-tête, afin d’éviter toute fuite de jeton d’accès vers des destinataires non prévus.

<div id="import-the-http-interceptor">
  ### Importer l’intercepteur HTTP
</div>

Pour commencer, importez les types dont vous aurez besoin pour intégrer l’intercepteur HTTP d’Auth0 à votre application :

```jsx lines
// Importez le module d'intercepteur et les types Angular dont vous aurez besoin
import { HttpClientModule, HTTP_INTERCEPTORS } from '@angular/common/http';
import { AuthHttpInterceptor } from '@auth0/auth0-angular';

// Enregistrez l'intercepteur dans votre module d'application dans le tableau `providers`
@NgModule({
  declarations: [],
  imports: [
    BrowserModule,
    HttpClientModule,     // Enregistrez ce module pour pouvoir effectuer des appels d'API avec HttpClient
    AppRoutingModule,
    AuthModule.forRoot(...),
  ],
  providers: [
    { provide: HTTP_INTERCEPTORS, useClass: AuthHttpInterceptor, multi: true },
  ],
  bootstrap: [AppComponent],
})
```

**Remarque :** Vous devez enregistrer manuellement l’intercepteur dans le module de votre application (plutôt que de l’enregistrer automatiquement), puisqu’il est conçu pour être chaîné avec d’autres intercepteurs et que nous ne pouvons pas garantir qu’il sera inséré au bon endroit.

<div id="configure-the-http-interceptor">
  ### Configurer l’intercepteur HTTP
</div>

Il existe plusieurs façons de le configurer selon vos besoins. Une route peut être définie à l’aide d’une chaîne de caractères ou d’un objet contenant des options qui influencent le jeton envoyé, et elle peut aussi être mise en correspondance selon la méthode HTTP utilisée. Les URI de route peuvent également inclure un caractère générique pour désigner un groupe de routes.

**Remarque :** Si un appel HTTP est effectué sans correspondre à une route spécifiée dans la configuration, l’intercepteur sera ignoré et l’en-tête `Authorization` ne sera pas inclus.

Cet exemple montre les façons les plus courantes d’indiquer au SDK quelles routes doivent inclure un en-tête `Authorization` :

```jsx lines expandable
// Modifiez votre configuration SDK existante pour inclure la config httpInterceptor
AuthModule.forRoot({
  domain: '{yourDomain}',
  clientId: 'YOUR_AUTH0_CLIENT_ID',
  authorizationParams: {
    redirect_uri: window.location.origin,
  },

  // La configuration AuthHttpInterceptor
  httpInterceptor: {
    allowedList: [
      // Joindre des jetons d'accès à tous les appels vers '/api' (correspondance exacte)
      '/api',

      // Joindre des jetons d'accès à tous les appels commençant par '/api/'
      '/api/*',

      // Correspondre à tout ce qui commence par /api/accounts, mais aussi spécifier l'audience et le scope que doit
      // avoir le jeton d'accès joint
      {
        uri: '/api/accounts/*',
        tokenOptions: {
          authorizationParams: {
            audience: 'http://my-api/',
            scope: 'read:accounts',
          }
        },
      },

      // Correspondance sur la méthode HTTP
      {
        uri: '/api/orders',
        httpMethod: 'post',
        tokenOptions: {
          authorizationParams: {
            audience: 'http://my-api/',
            scope: 'write:orders',
          }
        },
      },

      // Utilisation d'un URI absolu
      {
        uri: 'https://your-domain.auth0.com/api/v2/users',
        tokenOptions: {
          authorizationParams: {
            audience: 'https://your-domain.com/api/v2/',
            scope: 'read:users',
          }
        },
      },
    ],
  },
});
```

**Remarque :** `tokenOptions` est transmis directement à la méthode `getTokenSilently` du SDK SPA sous-jacent. Consultez [la documentation](https://auth0.github.io/auth0-spa-js/interfaces/GetTokenSilentlyOptions.html) pour en savoir plus sur les propriétés prises en charge.

<div id="call-an-api">
  ### Appeler une API
</div>

Effectuez votre appel à l’API à l’aide du service `HttpClient` d’Angular. Les jetons d’accès sont automatiquement inclus dans l’en-tête `Authorization` pour les routes qui correspondent à la configuration.

```jsx lines
export class MyComponent {
  constructor(private http: HttpClient) {}

  callApi(): void {
    this.http.get('/api').subscribe(result => console.log(result));
  }
}
```

Notez qu’aucune étape supplémentaire n’est nécessaire dans le code appelant pour fournir des jetons ou d’autres informations d’autorisation; tout cela est pris en charge par l’intercepteur et la configuration du module.
