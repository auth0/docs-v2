Les environnements de développement locaux fonctionnent généralement en http (c.-à-d. `http://localhost`) par défaut. Ce guide explique à quel moment vous devriez exécuter un serveur local en https et comment configurer https sur localhost.

<div id="when-to-use-a-local-https-server">
  ## Quand utiliser un serveur HTTPS local
</div>

Les navigateurs Web se comportent différemment en HTTP et en HTTPS; pour tenir compte de ces différences, votre serveur de développement devra peut-être servir les pages en HTTPS. Par exemple :

* Lorsque votre page est servie en HTTPS, les requêtes visant à charger des ressources à partir d’une URL HTTP sont bloquées. Vous pourriez vouloir reproduire ce comportement en local pour éviter des bogues qui ne se manifesteraient qu’en production, ou pour éviter d’envoyer par erreur des requêtes HTTP contenant des renseignements sensibles.
* Les cookies utilisant l’attribut Secure ou l’attribut [SameSite](https://auth0.com/blog/browser-behavior-changes-what-developers-need-to-know/) défini à None ne seront pas envoyés par des canaux non sécurisés, ce qui perturbera l’authentification et d’autres fonctionnalités.

<div id="how-to-set-up-a-secure-local-server">
  ## Comment configurer un serveur local sécurisé
</div>

L’idée est d’exécuter un serveur Web devant votre serveur de développement local afin qu’il accepte les requêtes https et les transfère au serveur Web de votre application par http, de sorte que les requêtes de votre application soient servies en https par l’intermédiaire d’un proxy. C’est ce qu’on appelle un proxy inverse https, et vous pouvez utiliser un produit prêt à l’emploi appelé [Caddy](https://caddyserver.com/v2) pour créer un proxy inverse https.

Le processus est le suivant :

1. Téléchargez le binaire Caddy
2. Exécutez le serveur Web de votre application sur http, comme d’habitude
3. Exécutez le binaire Caddy comme proxy inverse en lui indiquant de transférer le trafic vers le serveur Web de votre application
4. Accédez au serveur Web du proxy inverse en https

<div id="step-1-download-the-caddy-binary">
  ### Étape 1. Téléchargez le binaire Caddy
</div>

Accédez à [https://caddyserver.com/download](https://caddyserver.com/download), sélectionnez votre plateforme et téléchargez le fichier sur votre ordinateur.

<div id="step-2-run-your-applications-web-server-over-http-as-normal">
  ### Étape 2. Exécutez le serveur Web de votre application en HTTP, comme d’habitude
</div>

Exécutez le serveur de développement local de votre application comme d’habitude, par exemple : $ PORT=3000 npm start

<div id="step-3-run-the-caddy-binary-as-a-reverse-proxy-telling-it-to-forward-traffic-to-your-applications-web-server">
  ### Étape 3. Exécutez le binaire Caddy en tant que proxy inverse et indiquez-lui d’acheminer le trafic vers le serveur web de votre application
</div>

Repérez l’emplacement du binaire Caddy téléchargé et exécutez la commande suivante pour créer un proxy inverse et acheminer les requêtes du port 443 (le port par défaut utilisé pour HTTPS) vers le port de votre application (dans ce cas-ci 3000) :

```bash wrap lines
/path/to/caddy reverse-proxy --from localhost:443 --to localhost:3000
```

<div id="step-4-now-visit-your-web-server-over-https">
  ### Étape 4. Accédez maintenant à votre serveur Web en HTTPS
</div>

Accédez à [https://localhost](https://localhost) et il répondra aux requêtes de votre application via HTTPS

  ## Exécuter votre application derrière un proxy
</div>
