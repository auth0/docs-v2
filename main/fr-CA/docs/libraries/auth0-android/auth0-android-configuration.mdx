Auth0.Android peut être configuré à l’aide de diverses options, présentées ci-dessous.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Les options ci-dessous diffèrent de celles de la version V1 précédente. Consultez le guide de migration pour en savoir plus.
</Callout>

<div id="withconnection">
  ## withConnection
</div>

L’option `withConnection` vous permet de spécifier la connexion avec laquelle vous souhaitez vous authentifier.

```kotlin lines
WebAuthProvider.login(account)
                .withConnection("twitter")
                .start(this, callback)
```

<div id="withscope">
  ## withScope
</div>

L’utilisation de scopes vous permet de renvoyer des claims précis pour des champs particuliers dans votre requête. L’ajout de paramètres à `withScope` vous permet d’ajouter d’autres scopes. Consultez [Scopes](/fr-CA/docs/get-started/apis/scopes) pour en savoir plus.

```kotlin lines
WebAuthProvider.login(account)
                .withScope("openid profile email")
                .start(this, callback)
```

Le scope par défaut est `openid profile email`.

<div id="withconnectionscope">
  ## withConnectionScope
</div>

Il peut arriver que vous deviez vous authentifier avec des scopes de connexion spécifiques, c’est-à-dire des autorisations propres au fournisseur d’authentification concerné. Consultez [Ajouter des scopes pour un IdP externe](/fr-CA/docs/authenticate/identity-providers/adding-scopes-for-an-external-idp). Toutefois, si vous avez besoin d’un accès précis dans un contexte particulier de votre application, vous pouvez transmettre des paramètres à `withConnectionScope`. Vous trouverez la liste complète des paramètres disponibles dans les paramètres de cette connexion dans le [Dashboard](https://manage.auth0.com/#) ou dans la documentation du fournisseur d’authentification. Le scope demandé ici s’ajoute à ceux déjà indiqués dans les paramètres de la connexion dans le Dashboard.

```kotlin lines
WebAuthProvider.login(account)
                .withConnectionScope("email", "profile", "calendar:read")
                .start(this, callback)
```

<div id="withparameters">
  ## withParameters
</div>

Pour envoyer des paramètres supplémentaires dans la requête d’authentification, utilisez `withParameters.`

```kotlin lines
val parameters = mapOf("param1" to "value1")

WebAuthProvider.login(account)
                .withParameters(parameters)
                .start(this, callback)
```

<div id="withheaders">
  ## withHeaders
</div>

Pour envoyer des en-têtes personnalisés au point de terminaison d’autorisation, utilisez `withHeaders.`

```kotlin lines
val headers = mapOf("header1" to "value1")

WebAuthProvider.login(account)
                .withHeaders(headers)
                .start(this, callback)
```

<div id="withscheme">
  ## withScheme
</div>

Si vous n’utilisez pas les &quot;App Links&quot; d’Android ou si vous souhaitez utiliser un schéma différent pour l’URI de redirection, utilisez `withScheme`. Mettez à jour le paramètre fictif du manifeste `auth0Scheme` dans le fichier `app/build.gradle` ainsi que la liste d’autorisations **Allowed Callback URLs** dans le [Dashboard](https://manage.auth0.com/#), sous les paramètres de l’application, pour qu’ils correspondent au schéma choisi.

```kotlin lines
WebAuthProvider.login(account)
                .withScheme("myapp")
                .start(this, callback)
```

<Warning>
  Le schéma doit être en minuscules et ne peut pas contenir de traits de soulignement.
</Warning>

<div id="withaudience">
  ## withAudience
</div>

Pour spécifier une <Tooltip tip="Audience : identifiant unique du destinataire d’un jeton émis. Nommée aud dans un jeton, sa valeur contient l’ID d’une application (ID client) pour un ID Token ou d’une API (identifiant d’API) pour un jeton d’accès." cta="Voir le glossaire" href="/fr-CA/docs/glossary?term=audience">audience</Tooltip>, utilisez `withAudience`.

```kotlin lines
WebAuthProvider.login(account)
                .withAudience("https://YOUR_DOMAIN/userinfo")
                .start(this, callback)
```
