---
title: "Agent Skills pour Auth0"
description: "Utilisez les Agent Skills d’Auth0 pour intégrer l’authentification à vos applications avec des assistants de codage IA comme Claude Code, Cursor et GitHub Copilot"
---

Les Agent Skills sont des instructions structurées qui aident les assistants de codage IA à implémenter correctement l’authentification avec Auth0. Installez-les une seule fois, puis demandez à votre assistant d’ajouter la connexion, de protéger des routes, de sécuriser des API, de configurer la MFA, de migrer depuis un autre fournisseur ou de personnaliser votre page de connexion — il s’occupe du reste.

Fonctionne avec Claude Code, Cursor, GitHub Copilot et [plus de 40 autres agents](https://agentskills.io/clients) compatibles avec le format [Agent Skills](https://agentskills.io).

<div id="prerequisites">
  ## Prérequis
</div>

* Un [compte Auth0](https://auth0.com/signup) (niveau gratuit disponible)
* Votre **domaine** Auth0 et votre **ID client** dans l’[Auth0 Dashboard](https://manage.auth0.com)
* Un assistant de codage IA (Claude Code, Cursor ou GitHub Copilot)

<div id="install">
  ## Installation
</div>

<Tabs>
  <Tab title="Claude Code">
    Auth0 est offert sur le marketplace officiel des plugins Claude Code (`claude-plugins-official`). Assurez-vous d’abord que le marketplace est à jour, puis installez-le :

    ```bash
    /plugin marketplace update claude-plugins-official
    /plugin install auth0@claude-plugins-official
    ```

    Vous pouvez aussi taper `/plugin` dans une session, aller à l’onglet **Discover** et rechercher « Auth0 ».

    Vous pouvez également l’installer à partir de votre terminal sans démarrer de session :

    ```bash
    claude plugin marketplace update claude-plugins-official
    claude plugin install auth0@claude-plugins-official
    ```

    <Note>
      Si vous obtenez une erreur « not found », ajoutez d’abord le marketplace, puis réessayez l’installation :

      ```bash
      /plugin marketplace add anthropics/claude-plugins-official
      /plugin install auth0@claude-plugins-official
      ```
    </Note>
  </Tab>

  <Tab title="Cursor">
    Auth0 est offert sur le [marketplace Cursor](https://cursor.com/marketplace/auth0). Ouvrez la fiche et cliquez sur **Add** pour l’installer.

    Vous pouvez aussi l’installer via **Cursor Settings &gt; Rules &gt; Add Rule &gt; Remote Rule (GitHub)** et entrer l’[URL du dépôt Auth0 Agent Skills](https://github.com/auth0/agent-skills).
  </Tab>

  <Tab title="GitHub Copilot">
    Installez-le à l’aide de [Skills CLI](https://github.com/vercel-labs/skills) en ciblant GitHub Copilot :

    ```bash
    npx skills add auth0/agent-skills --agent github-copilot
    ```
  </Tab>

  <Tab title="Autres agents">
    [Skills CLI](https://github.com/vercel-labs/skills) fonctionne avec Claude Code, Cursor, Copilot, Codex et [plus de 40 autres agents](https://agentskills.io/clients) :

    ```bash
    npx skills add auth0/agent-skills --all
    ```

    Pour cibler des agents précis, utilisez `--agent` :

    ```bash
    npx skills add auth0/agent-skills --agent claude-code cursor
    ```
  </Tab>
</Tabs>

<div id="try-it-now">
  ## Essayez-le dès maintenant
</div>

Après l’installation, collez l’un des éléments suivants dans votre assistant de codage IA :

```
Add Auth0 login to my app
```

```
Secure my Express API with Auth0 JWT validation
```

```
Migrate from Firebase Auth to Auth0
```

<Tip>
  Fournissez du contexte dans votre invite — mentionnez votre framework, le type de routeur et si vous avez besoin d’une connexion, d’une protection d’API ou des deux. Plus la demande est précise, meilleur sera le résultat.
</Tip>

<div id="how-it-works">
  ## Fonctionnement
</div>

<Steps>
  <Step title="Décrivez ce dont vous avez besoin">
    Indiquez à votre assistant ce que vous voulez faire — ajouter l’authentification à votre application React, sécuriser une API Spring Boot, migrer depuis Cognito, ajouter MFA ou personnaliser votre page de connexion. Vous n’avez pas besoin de savoir quelle compétence ou quel SDK utiliser.
  </Step>

  <Step title="La bonne compétence s’active">
    L’assistant lit les fichiers de votre projet (`package.json`, `requirements.txt`, `build.gradle`, etc.), identifie votre framework et active la compétence correspondante parmi les 25 disponibles.
  </Step>

  <Step title="Vous obtenez du code fonctionnel">
    La compétence guide votre assistant tout au long de l’implémentation — installation du bon SDK, création des routes d’authentification, configuration des variables d’environnement, ajout de la protection des routes et application des pratiques exemplaires d’Auth0.
  </Step>
</Steps>

<Info>
  Vous n’avez pas à choisir les compétences manuellement. La détection du framework gère automatiquement les compétences liées au SDK, et vous pouvez invoquer directement des compétences de fonctionnalité comme MFA ou la migration en décrivant ce dont vous avez besoin.
</Info>
