<div id="configure-auth0">
  ## Configurer Auth0
</div>

<div id="get-your-application-keys">
  ### Obtenez les clés de votre application
</div>

Lorsque vous vous êtes inscrit à Auth0, une nouvelle application a été créée pour vous, ou vous avez peut-être créé une nouvelle application. Vous aurez besoin de certaines informations sur cette application pour communiquer avec Auth0. Vous pouvez obtenir ces informations dans la section [paramètres de l’application](https://manage.auth0.com/#/applications) de l’Auth0 Dashboard.

<Frame>![Tableau de bord de l’application](https://cdn2.auth0.com/docs/1.14550.0/media/articles/dashboard/client_settings.png)</Frame>

Vous avez besoin des renseignements suivants :

* **Domaine**
* **ID client**
* **Secret client**

<Info>
  Si vous téléchargez l’exemple en haut de cette page, ces renseignements sont déjà remplis.
</Info>

<div id="configure-callback-urls">
  ### Configurer les URL de rappel
</div>

Une URL de rappel est une URL de votre application vers laquelle Auth0 redirige l’utilisateur après son authentification. L’URL de rappel de votre application doit être ajoutée au champ **Allowed Callback URLs** dans les [paramètres de l’application](https://manage.auth0.com/#/applications). Si ce champ n’est pas défini, les utilisateurs ne pourront pas ouvrir de session dans l’application et recevront une erreur.

<Info>
  Si vous suivez l’exemple de projet téléchargé en haut de cette page, l’URL de rappel à ajouter au champ **Allowed Callback URLs** est `http://localhost:3000/callback`.
</Info>

<div id="configure-logout-urls">
  ### Configurer les URL de déconnexion
</div>

Une URL de déconnexion est une URL de votre application vers laquelle Auth0 peut rediriger l’utilisateur une fois qu’il a été déconnecté du serveur d’autorisation. Elle est spécifiée dans le paramètre de requête `returnTo`. L’URL de déconnexion de votre application doit être ajoutée au champ **Allowed Logout URLs** dans les [paramètres de l’application](https://manage.auth0.com/#/applications). Si ce champ n’est pas défini, les utilisateurs ne pourront pas se déconnecter de l’application et obtiendront une erreur.

<Info>
  Si vous suivez l’exemple de projet que vous avez téléchargé au haut de cette page, l’URL de déconnexion que vous devez ajouter au champ **Allowed Logout URLs** est `http://localhost:3000`.
</Info>

<div id="create-application">
  ## Créer une application
</div>

Dans ce guide, nous vous montrons comment intégrer Auth0 à une application Python à l’aide du framework [Django](https://www.djangoproject.com/) et d’[Authlib](https://authlib.org/). Commençons par vérifier que Django est installé sur votre système. Dans votre terminal, exécutez la commande suivante :

```bash lines
pip install django~=4.0
```

Si vous avez déjà configuré une application Python, vous pouvez passer à l’étape suivante. Sinon, créons le projet de notre nouvelle application. Dans votre terminal, exécutez la commande suivante, puis accédez au nouveau dossier du projet :

```bash lines
django-admin startproject webappexample
cd webappexample
```

<div id="install-dependencies">
  ## Installer les dépendances
</div>

Pour cette intégration, vous aurez besoin de quelques dépendances de bibliothèque, comme Authlib. Créez un fichier `requirements.txt` dans le répertoire de votre projet, puis ajoutez-y ce qui suit :

```txt lines
# 📁 requirements.txt -----

authlib ~= 1.0
django ~= 4.0
python-dotenv ~= 0.19
requests ~= 2.27
```

Vous devriez maintenant exécuter `pip install -r requirements.txt` dans votre terminal pour que ces dépendances soient disponibles pour votre projet.

<div id="update-settingspy">
  ## Mettez à jour settings.py
</div>

Vous devrez apporter quelques modifications mineures à votre fichier `webappexample/settings.py` afin de lire les valeurs du fichier `.env`. Au début du fichier, ajoutez les importations suivantes :

```py lines
# 📁 webappexample/settings.py -----

import os
from dotenv import load_dotenv, find_dotenv
```

Ensuite, sous la définition de &#39;BASE&#95;DIR&#39;, ajoutez ce qui suit :

```py lines
# 📁 webappexample/settings.py -----

# Ajoutez la ligne suivante après 'BASE_DIR = ...'
TEMPLATE_DIR = os.path.join(BASE_DIR, "webappexample", "templates")
```

Ensuite, repérez la variable `TEMPLATES` et mettez à jour la valeur `DIRS` pour y ajouter notre chaîne `TEMPLATE_DIR`. Cela indique à Django où trouver nos fichiers de modèle une fois que nous les aurons créés. Laissez tel quel tout autre contenu de ce tableau.

```py lines
# 📁 webappexample/settings.py -----

TEMPLATES = [
    {
        # Laissez les autres lignes telles quelles ; on met simplement à jour `DIRS`.
        "DIRS": [TEMPLATE_DIR],
    },
]
```

Enfin, à la fin de ce fichier, ajoutez ce qui suit :

```py lines
# 📁 webappexample/settings.py -----

# Charger le fichier de définition d'environnement
ENV_FILE = find_dotenv()
if ENV_FILE:
    load_dotenv(ENV_FILE)


# Charger les paramètres de l'application Auth0 en mémoire
AUTH0_DOMAIN = os.environ.get("AUTH0_DOMAIN")
AUTH0_CLIENT_ID = os.environ.get("AUTH0_CLIENT_ID")
AUTH0_CLIENT_SECRET = os.environ.get("AUTH0_CLIENT_SECRET")
```

<div id="setup-your-application">
  ## Configurez votre application
</div>

Vous êtes maintenant prêt à commencer à coder votre application. Ouvrez le fichier `webappexample/views.py` dans le répertoire de votre projet.

Commencez par importer toutes les bibliothèques que votre application utilisera :

```py lines
# 📁 webappexample/views.py -----

import json
from authlib.integrations.django_client import OAuth
from django.conf import settings
from django.shortcuts import redirect, render
from django.urls import reverse
from urllib.parse import quote_plus, urlencode
```

Vous pouvez maintenant configurer Authlib pour prendre en charge l’authentification de votre application avec Auth0 :

```py lines
# 👆 Nous poursuivons à partir des étapes précédentes. Ajoutez ceci à votre fichier webappexample/views.py.

oauth = OAuth()

oauth.register(
    "auth0",
    client_id=settings.AUTH0_CLIENT_ID,
    client_secret=settings.AUTH0_CLIENT_SECRET,
    client_kwargs={
        "scope": "openid profile email",
    },
    server_metadata_url=f"https://{settings.AUTH0_DOMAIN}/.well-known/openid-configuration",
)
```

Vous trouverez plus d’information sur les options de configuration offertes pour la méthode OAuth `register()` d’Authlib dans [sa documentation.](https://docs.authlib.org/en/latest/client/frameworks.html#using-oauth-2-0-to-log-in)

<div id="setup-your-route-handlers">
  ## Configurez vos gestionnaires de routes
</div>

Pour cette démonstration, nous allons ajouter 4 routes à votre application : les routes login, callback, logout et index.

<div id="triggering-authentication-with-login">
  ### Déclenchement de l’authentification avec /login
</div>

Lorsque les utilisateurs de votre application accèdent à la route `/login`, ils sont redirigés vers Auth0 afin d’amorcer le flux d’authentification.

```py lines
# 👆 Nous poursuivons à partir des étapes précédentes. Ajoutez ceci à votre fichier webappexample/views.py.

def login(request):
    return oauth.auth0.authorize_redirect(
        request, request.build_absolute_uri(reverse("callback"))
    )
```

<div id="finalizing-authentication-with-callback">
  ### Finaliser l’authentification avec /callback
</div>

Une fois que vos utilisateurs ont fini de se connecter avec Auth0, ils sont redirigés vers votre application sur la route `/callback`. Cette route est chargée d’enregistrer la session de l’utilisateur afin que, lors d’une prochaine visite, il n’ait pas à se reconnecter de nouveau.

```py lines
# 👆 Nous poursuivons à partir des étapes ci-dessus. Ajoutez ceci à votre fichier webappexample/views.py.

def callback(request):
    token = oauth.auth0.authorize_access_token(request)
    request.session["user"] = token
    return redirect(request.build_absolute_uri(reverse("index")))
```

<div id="clearing-a-session-with-logout">
  ### Vider une session avec /logout
</div>

Comme vous pouvez vous en douter, cette route gère la déconnexion d’un utilisateur de votre application. Elle efface la session de l’utilisateur dans votre application et le redirige brièvement vers le point de terminaison de déconnexion d’Auth0 afin de s’assurer que sa session est entièrement supprimée, avant de le renvoyer vers votre route d’accueil (abordée à la section suivante.)

```py lines
# 👆 Nous poursuivons à partir des étapes précédentes. Ajoutez ceci à votre fichier webappexample/views.py.

def logout(request):
    request.session.clear()

    return redirect(
        f"https://{settings.AUTH0_DOMAIN}/v2/logout?"
        + urlencode(
            {
                "returnTo": request.build_absolute_uri(reverse("index")),
                "client_id": settings.AUTH0_CLIENT_ID,
            },
            quote_via=quote_plus,
        ),
    )
```

<div id="theres-no-place-like-home">
  ### Rien ne vaut /home
</div>

Enfin, votre route d’accueil servira soit à afficher les renseignements d’un utilisateur authentifié, soit à inviter les visiteurs à se connecter.

```py lines
# 👆 Nous poursuivons à partir des étapes précédentes. Ajoutez ceci à votre fichier webappexample/views.py.

def index(request):
    return render(
        request,
        "index.html",
        context={
            "session": request.session.get("user"),
            "pretty": json.dumps(request.session.get("user"), indent=4),
        },
    )
```

<div id="register-your-routes">
  ### Enregistrez vos routes
</div>

Enfin, vous devez indiquer à Django comment relier ces nouvelles routes. Remplacez le contenu de votre fichier `webappexample/urls.py` par ce qui suit :

```py lines
# 📁 webappexample/urls.py -----

from django.urls import path

from . import views

urlpatterns = [
    path("", views.index, name="index"),
    path("login", views.login, name="login"),
    path("logout", views.logout, name="logout"),
    path("callback", views.callback, name="callback"),
]
```

<div id="add-templates">
  ## Ajouter des modèles
</div>

Nous devons maintenant simplement créer les fichiers de modèle utilisés par les routes about (lors des appels à `render()`).

Créez un nouveau sous-répertoire nommé `templates` dans le dossier `webappexample`, puis créez un fichier `index.html` :

```html lines
# 📁 webappexample/templates/index.html -----

<html>
  <head>
    <meta charset="utf-8" />
    <title>Auth0 Example</title>
  </head>
  <body>
    {% if session %}
    <h1>Welcome {{session.userinfo.name}}!</h1>
    <p><a href="{% url 'logout' %}">Logout</a></p>
    <div><pre>{{pretty}}</pre></div>
    {% else %}
    <h1>Welcome Guest</h1>
    <p><a href="{% url 'login' %}">Login</a></p>
    {% endif %}
  </body>
</html>
```

<div id="run-your-application">
  ## Exécutez votre application
</div>

Vous êtes maintenant prêt à exécuter votre application! Dans le répertoire de votre projet, ouvrez un terminal et utilisez :

```bash lines
python3 manage.py migrate
python3 manage.py runserver 3000
```

Votre application devrait maintenant être accessible dans votre navigateur à l’adresse `http://localhost:3000`.
