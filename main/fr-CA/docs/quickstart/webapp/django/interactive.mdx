  <Content>
    Auth0 vous permet d&#39;ajouter l&#39;authentification et d&#39;accéder aux informations du profil utilisateur dans votre application. Ce guide
    montre comment intégrer Auth0 à une application Python [Django](https://www.djangoproject.com/) à l&#39;aide du SDK [Authlib](https://authlib.org/).

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1">
      Pour utiliser les services Auth0, vous devez avoir une application configurée dans le tableau de bord Auth0. L’application Auth0 est
      l’endroit où vous configurerez le fonctionnement de l’authentification pour le projet que vous développez.

      ### Configurer une application

      Utilisez le sélecteur interactif pour créer une nouvelle application Auth0 ou sélectionner une application existante correspondant
      au projet que vous souhaitez intégrer. Chaque application dans Auth0 reçoit un identifiant client unique alphanumérique
      que le code de votre application utilisera pour appeler les API Auth0 au moyen du SDK.

      Tous les paramètres que vous configurez à l’aide de ce guide de démarrage rapide seront automatiquement mis à jour pour votre application dans le [tableau de bord](https://manage.auth0.com/dashboard/us/auth0-dsepaid/), où vous pourrez gérer vos applications par la suite.

      Si vous préférez consulter une configuration complète, vous pouvez plutôt voir un exemple d’application.

      ### Configurer les URL de rappel

      Une URL de rappel est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      authentification. Si elle n’est pas définie, les utilisateurs ne seront pas redirigés vers votre application après leur connexion.

      <Info>
        Si vous utilisez notre projet d’exemple, définissez cette valeur sur
        `http://localhost:3000/callback`.
      </Info>

      ### Configurer les URL de déconnexion

      Une URL de déconnexion est une URL de votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      déconnexion. Si elle n’est pas définie, les utilisateurs ne pourront pas se déconnecter de votre application et recevront une erreur.

      <Info>
        Si vous utilisez notre projet d’exemple, définissez cette valeur sur `http://localhost:3000`.
      </Info>
    </Section>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2">
      Pour cette intégration, vous ajouterez plusieurs dépendances de bibliothèque, comme Authlib. Créez un
      fichier `requirements.txt` dans le répertoire de votre projet et incluez ce qui suit :

      ```text lines
      authlib ~= 1.0
      django ~= 4.0
      python-dotenv ~= 0.19
      requests ~= 2.27
      ```

      Exécutez la commande suivante dans votre terminal pour rendre ces dépendances disponibles :

      ```text lines
      pip install -r requirements.txt
      ```
    </Section>

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3">
      Ensuite, créez un fichier `.env` dans le répertoire de votre projet. Ce fichier contiendra vos clés client et d’autres
      paramètres de configuration.

      ```text lines
      AUTH0_CLIENT_ID={yourClientId}
      AUTH0_CLIENT_SECRET={yourClientSecret}
      AUTH0_DOMAIN={yourDomain}
      ```
    </Section>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4">
      Si vous avez déjà configuré une application Django, passez à l’étape suivante. Pour créer une nouvelle application, exécutez la
      commande suivante :

      ```text lines
      django-admin startproject webappexample
      ```

      Accédez au nouveau dossier du projet :

      ```text lines
      cd webappexample
      ```
    </Section>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5">
      Ouvrez le fichier `webappexample/settings.py` pour vérifier les valeurs du `.env`.

      Au début du fichier, ajoutez les importations `os` et `dotenv`.

      Ensuite, sous la définition de `BASE_DIR`, ajoutez la variable `TEMPLATE_DIR`.

      Ensuite, repérez la variable `TEMPLATES` et mettez à jour la valeur `DIRS` pour y ajouter la
      chaîne `TEMPLATE_DIR`. Cela détermine le chemin d’accès des fichiers de gabarit, que vous créerez à une
      étape ultérieure. Conservez inchangé tout autre contenu de ce tableau.

      À la fin de ce fichier, ajoutez le code pour charger la configuration d’Auth0.
    </Section>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6">
      Pour commencer à créer votre application, ouvrez le fichier `webappexample/views.py` dans votre IDE.

      Importez toutes les bibliothèques nécessaires à votre application.

      Vous pouvez maintenant configurer Authlib pour gérer l’authentification de votre application avec Auth0.

      Pour en savoir plus sur les options de configuration offertes par la méthode OAuth `register()` d’Authlib, consultez [sa documentation.](https://docs.authlib.org/en/latest/client/frameworks.html#using-oauth-2-0-to-log-in)
    </Section>

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7">
      Dans cet exemple, vous ajouterez quatre routes à votre application : les routes de connexion, de rappel, de déconnexion et d’index.

      * `login` - Lorsque les visiteurs de votre application accèdent à la route `/login`, ils sont redirigés vers Auth0 pour
        lancer le processus d’authentification.
      * `callback` - Une fois la connexion à Auth0 terminée, les utilisateurs reviennent à votre application
        par la route `/callback`. Cette route enregistre la session de l’utilisateur et lui évite
        d’avoir à se reconnecter lors d’une prochaine visite.
      * `logout` - La route `/logout` déconnecte les utilisateurs de votre application. Cette route supprime
        la session utilisateur dans votre application et redirige vers le point de terminaison de déconnexion d’Auth0 pour s’assurer que la session n’est plus
        conservée. L’application redirige ensuite l’utilisateur vers votre route d’accueil.
      * `index` - La route d’accueil affichera les renseignements d’un utilisateur authentifié ou permettra aux visiteurs de se connecter.
    </Section>

    <Section id={sections[7].id} title={sections[7].title} stepNumber="8">
      Remplacez le contenu de votre fichier `webappexample/urls.py` par le code à droite afin de relier
      ces nouvelles routes.

      Cela acheminera les routes `/login`, `/callback`, `/logout` et `/`
      vers les gestionnaires correspondants.
    </Section>

    <Section id={sections[8].id} title={sections[8].title} stepNumber="9">
      Ensuite, vous allez créer un fichier de modèle utilisé dans la route de la page d’accueil.

      Créez un nouveau sous-répertoire nommé `templates` dans le dossier `webappexample`, puis créez
      un fichier `index.html`.

      Le fichier `index.html` contiendra le code du modèle pour afficher les renseignements de l’utilisateur s’il est connecté, ou lui présenter
      un bouton de connexion s’il est déconnecté.
    </Section>

    <Section id={sections[9].id} title={sections[9].title} stepNumber="10">
      Vous êtes prêt à lancer votre application ! Dans le répertoire de votre projet, ouvrez un terminal et utilisez :

      ```text lines
      python3 manage.py migrate
      python3 manage.py runserver 3000
      ```

      Votre application devrait maintenant être accessible dans votre navigateur à l’adresse [http://localhost:3000](http://localhost:3000).

      <Info>
        ##### Point de contrôle

        Visitez [http://localhost:3000](http://localhost:3000/) pour vérifier. Vous devriez voir un bouton de connexion qui vous redirige vers
        Auth0 pour vous connecter, puis de retour vers votre application pour afficher les renseignements de votre profil.
      </Info>
    </Section>
