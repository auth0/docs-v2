Le SDK Laravel d’Auth0 vous permet d’ajouter rapidement l’authentification, la gestion des profils utilisateur et le contrôle d’accès aux routes à votre application Laravel. Ce guide montre comment intégrer Auth0 à une application Laravel 11 ou 12, nouvelle ou existante. Nous vous recommandons de vous connecter pour suivre ce guide de démarrage rapide avec des exemples configurés pour votre compte.

<div id="laravel-installation">
  ## Installation de Laravel
</div>

**Si vous n’avez pas déjà mis en place une application Laravel**, ouvrez un terminal dans le répertoire où vous souhaitez créer le projet et exécutez la commande suivante :

```bash lines
composer create-project laravel/laravel:^12 auth0-laravel-app
```

Toutes les commandes de ce guide supposent que vous les exécutez depuis le répertoire racine de votre projet Laravel; vous devez donc utiliser `cd` pour vous placer dans le répertoire du nouveau projet :

```bash lines
cd auth0-laravel-app
```

<div id="sdk-installation">
  ## Installation du SDK
</div>

Exécutez la commande suivante dans le répertoire de votre projet pour installer le [SDK Auth0 pour Laravel](https://github.com/auth0/laravel-auth0) :

```bash lines
composer require auth0/login:^7 --update-with-all-dependencies
```

Ensuite, générez un fichier de configuration SDK pour votre application :

```bash lines
php artisan vendor:publish --tag auth0
```

<div id="sdk-configuration">
  ## Configuration du SDK
</div>

Exécutez la commande suivante depuis le répertoire de votre projet pour télécharger l’[Auth0 CLI](https://github.com/auth0/auth0-cli) :

```bash lines
curl -sSfL https://raw.githubusercontent.com/auth0/auth0-cli/main/install.sh | sh -s -- -b .
```

Authentifiez-vous ensuite dans la CLI avec votre compte Auth0 en choisissant &quot;as a user&quot; lorsqu’on vous le demande :

```bash lines
./auth0 login
```

Ensuite, créez une nouvelle application dans Auth0 :

```bash lines
./auth0 apps create \
  --name "My Laravel Application" \
  --type "regular" \
  --auth-method "post" \
  --callbacks "http://localhost:8000/callback" \
  --logout-urls "http://localhost:8000" \
  --reveal-secrets \
  --no-input \
  --json > .auth0.app.json
```

Vous devez aussi créer une nouvelle API :

```bash lines
./auth0 apis create \
  --name "My Laravel Application's API" \
  --identifier "https://github.com/auth0/laravel-auth0" \
  --offline-access \
  --no-input \
  --json > .auth0.api.json
```

Cela génère deux fichiers dans le répertoire de votre projet pour configurer le SDK.

Comme ces fichiers contiennent des données d’identification, il est important de les traiter comme des renseignements sensibles. Assurez-vous de ne pas les inclure dans votre système de contrôle de version. Si vous utilisez Git, ajoutez-les à votre fichier `.gitignore` :

```bash lines
echo ".auth0.*.json" >> .gitignore
```

<div id="login-routes">
  ## Routes de connexion
</div>

Le SDK enregistre automatiquement toutes les routes nécessaires à l&#39;authentification des utilisateurs de votre application.

| Route       | Utilité                               |
| ----------- | ------------------------------------- |
| `/login`    | Lance le flux d&#39;authentification. |
| `/logout`   | Déconnecte l&#39;utilisateur.         |
| `/callback` | Gère le retour d&#39;Auth0.           |

Si vous avez besoin de mieux les contrôler, ou si elles entrent en conflit avec des routes existantes dans votre application, vous pouvez plutôt enregistrer manuellement les contrôleurs du SDK. Veuillez consulter [le README du SDK](https://github.com/auth0/laravel-auth0) pour les intégrations avancées.

<div id="access-control">
  ## Contrôle d’accès
</div>

Les fonctionnalités d’authentification de Laravel utilisent des « guards » pour définir comment les utilisateurs sont authentifiés à chaque requête. Vous pouvez utiliser le guard d’authentification du SDK Auth0 pour restreindre l’accès aux routes de votre application.

Pour obliger les utilisateurs à s’authentifier avant d’accéder à une route, vous pouvez utiliser le middleware `auth` de Laravel :

```php lines
Route::get('/private', function () {
  return response('Welcome! You are logged in.');
})->middleware('auth');
```

Vous pouvez aussi exiger que les utilisateurs authentifiés disposent de [permissions](https://auth0.com/docs/manage-users/access-control/rbac) précises en combinant cette option avec le middleware `can` de Laravel :

```php lines
Route::get('/scope', function () {
    return response('You have the `read:messages` permissions, and can therefore access this resource.');
})->middleware('auth')->can('read:messages');
```

<div id="user-information">
  ## Renseignements sur l’utilisateur
</div>

Les renseignements sur l’utilisateur authentifié sont accessibles au moyen de la façade `Auth` de Laravel ou de la fonction utilitaire `auth()`.

Par exemple, pour récupérer l’identifiant et l’adresse de courriel de l’utilisateur :

```php lines
Route::get('/', function () {
  if (! auth()->check()) {
    return response('You are not logged in.');
  }

  $user = auth()->user();
  $name = $user->name ?? 'User';
  $email = $user->email ?? '';

  return response("Hello {$name}! Your email address is {$email}.");
});
```

<div id="user-management">
  ## Gestion des utilisateurs
</div>

Vous pouvez mettre à jour les renseignements d’un utilisateur à l’aide de la [Auth0 Management API](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md). Tous les endpoints de Management sont accessibles au moyen de la méthode `management()` du SDK.

**Avant d’effectuer des appels à la Auth0 Management API, vous devez autoriser votre application à communiquer avec la Auth0 Management API.** Pour ce faire, accédez à la [page API d’Auth0 Dashboard](https://manage.auth0.com/#/apis/), choisissez `Auth0 Management API`, puis sélectionnez l’onglet &#39;Machine to Machine Applications&#39;. Autorisez votre application Laravel, puis cliquez sur la flèche vers le bas pour choisir les scopes que vous souhaitez accorder.

Dans l’exemple suivant, où nous mettrons à jour les métadonnées d’un utilisateur et lui attribuerons une couleur préférée aléatoire, vous devez accorder les scopes `read:users` et `update:users`. Vous trouverez la liste des endpoints d’API et des scopes requis dans [la documentation de la Auth0 Management API](https://auth0.com/docs/api/management/v2).

```php lines
use Auth0\Laravel\Facade\Auth0;

Route::get('/colors', function () {
  $endpoint = Auth0::management()->users();

  $colors = ['red', 'blue', 'green', 'black', 'white', 'yellow', 'purple', 'orange', 'pink', 'brown'];

  $endpoint->update(
    id: auth()->id(),
    body: [
        'user_metadata' => [
            'color' => $colors[random_int(0, count($colors) - 1)]
        ]
    ]
  );

  $metadata = $endpoint->get(auth()->id());
  $metadata = Auth0::json($metadata);

  $color = $metadata['user_metadata']['color'] ?? 'unknown';
  $name = auth()->user()->name;

  return response("Hello {$name}! Your favorite color is {$color}.");
})->middleware('auth');
```

Un guide de référence rapide de toutes les méthodes de la Auth0 Management API du SDK est [accessible ici](https://github.com/auth0/laravel-auth0/blob/main/docs/Management.md).

<div id="run-the-application">
  ## Exécutez l’application
</div>

Vous êtes maintenant prêt à lancer votre application Laravel afin qu’elle puisse recevoir des requêtes :

```bash lines
php artisan serve
```

<div id="checkpoint">
  ### Vérification
</div>

Ouvrez votre navigateur Web et essayez d’accéder aux routes suivantes :

* `http://localhost:8000` pour afficher la route publique.
* `http://localhost:8000/private` pour être invité à vous authentifier.
* `http://localhost:8000` pour afficher la route publique, cette fois en étant authentifié.
* `http://localhost:8000/scope` pour vérifier si vous disposez de la [permission](https://auth0.com/docs/manage-users/access-control/rbac) `read:messages`.
* `http://localhost:8000/update` pour mettre à jour le profil de l’utilisateur.
* `http://localhost:8000/logout` pour vous déconnecter.

Si vous rencontrez des problèmes, voici quelques pistes à essayer :

* Essayez d’exécuter `php artisan optimize:clear` pour vider le cache de Laravel.
* Assurez-vous que vos fichiers `.auth0.app.json` et `.auth0.api.json` se trouvent à la racine de votre projet.
* Assurez-vous d’avoir activé votre application Laravel en tant qu’application Machine-to-Machine et de lui avoir accordé tous les scopes nécessaires pour la `Auth0 Management API` dans l’[Auth0 Dashboard](https://manage.auth0.com/#/apis/).

Vous rencontrez toujours des problèmes ? Consultez la [documentation](https://github.com/auth0/laravel-auth0) du SDK ou notre [portail de documentation](https://auth0.com/docs). Vous pouvez aussi visiter [la communauté](https://community.auth0.com), où notre équipe et d’autres membres de la communauté peuvent répondre à vos questions.
