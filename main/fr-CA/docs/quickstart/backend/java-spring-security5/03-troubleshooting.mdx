Ce document vous aidera à résoudre les problèmes de configuration si votre API renvoie des réponses inattendues. Nous vous recommandons de vous connecter pour suivre ce guide de démarrage rapide avec des exemples configurés pour votre compte.

Si vous avez correctement configuré la validation du JWT, votre API devrait renvoyer les réponses attendues lorsque vous effectuez des requêtes. Toutefois, si votre API renvoie une réponse 401 (Non autorisé), c’est que la configuration de votre middleware JWT ne correspond pas au JWT transmis.

Ce document vous aidera à résoudre les problèmes liés à la configuration de votre middleware JWT.

<div id="how-does-a-token-get-validated">
  ## Comment un jeton est-il validé ?
</div>

Pour valider un JWT, plusieurs éléments doivent être pris en compte :

1. **Le jeton est-il bien formé ?** Autrement dit, respecte-t-il la structure d’un JSON Web Token (JWT) ? Pour en savoir plus sur la structure d’un JWT, consultez [cette section sur la structure d’un JWT](/fr-CA/docs/secure/tokens/json-web-tokens#what-is-the-json-web-token-structure-)
2. **Le jeton a-t-il été altéré ?** La dernière partie d’un JWT est la signature. Elle sert à vérifier que le jeton a bien été signé par l’expéditeur et qu’il n’a été modifié d’aucune façon.
3. **Le jeton a-t-il été reçu pendant sa période de validité ?** Les JWT ne sont valides que pendant une période donnée (comme indiqué dans la revendication `exp`).
4. **Le jeton provient-il de l’autorité attendue ?** Cela comporte 2 volets

   * **Vérification de la signature** : Peut-on confirmer que le JWT est correctement signé à l’aide de la clé émise par l’autorité de délivrance ?
   * **Valeur de l’émetteur** : L’émetteur est défini dans la revendication `iss`. Cette revendication correspond-elle bien à ce que votre application attend ?
5. **Le jeton est-il destiné à l’application en cours ?** Autrement dit, la revendication `aud` du JWT correspond-elle à ce que votre application attend ?

<div id="inspecting-a-token">
  ## Inspection d’un jeton
</div>

Une façon rapide d’inspecter un JWT consiste à utiliser le site Web [JWT.io](https://jwt.io/). Il comprend un débogueur pratique qui vous permet de vérifier rapidement qu’un JWT est bien formé et d’examiner les valeurs des différentes revendications.

<Frame>![Débogage d’un JWT sur JWT.io](https://cdn2.auth0.com/docs/1.14550.0/media/articles/server-apis/aspnet-core-webapi/jwt-io-debugger-rs256.png)</Frame>

Dans la capture d’écran ci-dessus, vous pouvez voir que le jeton a été signé à l’aide de l’algorithme **RS256**. L’**émetteur** du jeton est **[https://jerrie.auth0.com/](https://jerrie.auth0.com/)** et l’**Audience** est **[https://rs256.test.api](https://rs256.test.api)**.

Autrement dit, ces valeurs de votre configuration de validation JWT doivent correspondre **exactement**, y compris la barre oblique finale de l’émetteur, comme suit

```
audience = "https://rs256.test.api",
domain = "https://jerrie.auth0.com/"
};
```

Pour un jeton signé avec HS256, l’affichage du débogueur sera légèrement différent :

Dans la capture d’écran ci-dessus, vous pouvez voir que le jeton a été signé avec l’algorithme **HS256**. L’**émetteur** du jeton est **[https://jerrie.auth0.com/](https://jerrie.auth0.com/)** et l’**Audience** est **[https://hs256.test.api](https://hs256.test.api)**.
