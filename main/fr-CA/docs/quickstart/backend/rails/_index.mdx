    <Section id={sections[2].id} title={sections[2].title} stepNumber="3" isSingleColumn>
      Créez une classe appelée `Auth0Client`. Cette classe décode et vérifie le jeton d’accès reçu,
      extrait de l’en-tête `Authorization` de la requête.

      La classe `Auth0Client` récupère la clé publique de votre locataire Auth0, puis s’en sert pour vérifier la
      signature du jeton d’accès. La structure `Token` définit une méthode `validate_permissions` pour
      rechercher un `scope` particulier dans un jeton d’accès en fournissant un tableau des scopes requis et en vérifiant s’ils
      sont présents dans la charge utile du jeton.

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4" isSingleColumn>
      Créez un Concern nommé `Secured` qui recherche le jeton d’accès dans l’en-tête `Authorization`
      d’une requête entrante.

      Si le jeton est présent, `Auth0Client.validate_token` utilise la gem `jwt` pour vérifier
      la signature du jeton et valider les claims du jeton.

      En plus de vérifier que le jeton d’accès est valide, le Concern inclut aussi un mécanisme pour confirmer que le
      jeton possède le **scope** nécessaire pour accéder aux ressources demandées. Dans cet exemple, nous définissons une
      méthode `validate_permissions` qui reçoit un bloc et vérifie les permissions en appelant la
      méthode `Token.validate_permissions` de la classe `Auth0Client`.

      Pour la route `/private-scoped`, les scopes définis seront comparés aux scopes présents dans le
      payload afin de déterminer s’il contient un ou plusieurs éléments de l’autre tableau.

    <Section id={sections[6].id} title={sections[6].title} stepNumber="7" isSingleColumn>
      Créez un contrôleur pour gérer les points de terminaison privés : `/api/private` et
      `/api/private-scoped`.

      `/api/private` est disponible pour les requêtes authentifiées contenant un jeton d&#39;accès sans
      scopes additionnels.

      `/api/private-scoped` est disponible pour les requêtes authentifiées contenant un jeton d&#39;accès avec le scope
      `read:messages` accordé

      Les points de terminaison protégés doivent appeler la méthode `authorize` du concern `Secured`.
      Pour ce faire, utilisez `before_action :authorize`, ce qui garantit que la méthode `Secured.authorize` est
      appelée avant chaque action dans le `PrivateController`.
