委任管理拡張機能によって表示されるアプリケーションには、 **［Users（ユーザー）］** と **［Logs（ログ）］** の2つのビューがあります。 **［User（ユーザー）］** ビューでは、Auth0アカウントに関連付けられたユーザーを表示して編集することができます。

デフォルトではすべてのユーザーが表示されますが、フィルターフックを構成することで、表示されるリストをフィルタリングできます。

フィルターフックの詳細については、「[委任管理：フィルターフック](/ja-jp/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-filter-hook)」をお読みください。

## 委任管理ダッシュボードでのユーザーアクション

委任管理拡張機能によってDashboardへのアクセス権を持つユーザーに表示されるオプション数を制限するには、アクセスフックを構成します。

アクセスフックの詳細については、「[委任管理：アクセスフック](/ja-jp/customize/extensions/delegated-administration-extension/delegated-administration-hooks/delegated-administration-access-hook)」をお読みください。

**［Change Profile（プロファイルの変更）］** オプションは、カスタムフィールドを構成した場合にのみ使用できます。

拡張機能を通じて使用できる、新しい **［Reset Password（パスワードのリセット）］** オプションがあります。このオプションでは、ユーザーに新たなパスワードを設定するためのメールが送信されます。そのためには、ユーザーを選択してから、 **［Actions（アクション）］** > **［Reset Password（パスワードのリセット）］** を選びます。

これにより、パスワード変更用のリンクを含むメールがユーザーに送信されます。

ご自身のプロファイルに、`Delegated Admin - Administrator（委任管理 - 管理者）`ロールがあることが表示されている場合は、 **［Logs（ログ）］** ビューで、ユーザーによって実行された認証のリストを表示することができます（このビューは、`Delegated Admin - Administrator（委任管理 - 管理者）`ロールのユーザーしか閲覧できません）。このビューに表示される内容は、[［Auth0 Dashboard］>［Monitoring（モニタリング）］>［Logs（ログ）］](https://manage.auth0.com/#/logs)に移動したときに表示されるデータのサブセットです。ログダッシュボードには、<Tooltip data-tooltip-id="react-containers-DefinitionTooltip-0" href="/ja-jp/glossary?term=auth0-dashboard" tip="Auth0 Dashboard: サービスを構成するためのAuth0の主製品。" cta="用語集の表示">Auth0 Dashboard</Tooltip>で実行された管理アクションに関するデータも表示されます。

## ユーザーの作成

新しいユーザーは、 **［Users（ユーザー）］** ビューにある **［+ Create User（ユーザーの作成）］** ボタンを選択して作成できます。メールアドレスとパスワードを指定する必要があります。ロールによっては、新しいユーザーが所属する **［Department（部門）］** を設定できないことがあります。

たとえば、`Delegated Admin - Administrator（委任管理 - 管理者）`ロールがあるユーザーには **［Department（部門）］** フィールドが表示され、任意の値を選択することができます。

しかしKellyは、`Delegated Admin - User（委任管理 - ユーザー）`ロールのため、 **［Department（部門）］** フィールドは表示されません。そして、財務部門に所属する彼女が作成するユーザーはすべて、自動的に財務部門に割り当てられます。
