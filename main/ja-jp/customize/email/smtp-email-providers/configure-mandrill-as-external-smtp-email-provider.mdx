Auth0では独自のSMTPメールプロバイダーを構成して、メール通信をより完全に管理、監視、トラブルシューティングできるようにすることができます。Auth0に組み込みのメールインフラストラクチャは、メールのテストにのみ使用してください。

## 前提条件

* Auth0からSMTPにメールが確実に送信されるように、ポートを開いて、特定のIPアドレスからのインバウンド接続を許可します。IPのリストを更新するには、[［Dashboard］>［Branding（ブランディング）］](https://manage.auth0.com/#/branding/email_provider)[>](https://manage.auth0.com/#/branding/email_provider)[［Email Provider（メールプロバイダー）］](https://manage.auth0.com/#/branding/email_provider)に移動します。詳細については、「[AllowListにIPアドレスを追加する](/ja-jp/secure/security-guidance/data-security/allowlist)」を参照してください。
* [Mandrill](https://www.mandrill.com/signup/)アカウントが必要です。

## Mandrillを構成する

1. [Mandrill](https://www.mandrill.com/signup/)にログインします。
2. Mandrillの[［Settings（設定）］](https://mandrillapp.com/settings)に移動して、 **［Add API key（APIキーの追加）］** をクリックします。キーの値をコピーします。
3. [［Auth0 Dashboard］>［Branding（ブランディング）］>［Email Provider（メールプロバイダー）］](https://manage.auth0.com/#/branding/email_provider)に移動します。
4. **［Use my own email provider（独自のメールプロバイダーを使用する）］** を有効にします。
5. **Mandrill** ロゴをクリックします。
6. **［From（差出人）］** のメールアドレスを入力して、前の手順でコピーしたMandrillの **APIキー** を貼り付けます。
7. **［Save（保存）］** をクリックします。

## メールをテストする

テストメールを送信するには、 **［Send Test Email（テストメールの送信）］** ボタンを使用します。すべてが正しく構成されていると、確認メールを受け取ります。数分経ってもメールを受信しない場合は、[Auth0ログ](https://manage.auth0.com/#/logs)でエラーを確認してください。

Mandrillの[［Outbound Activity（アウトバウンドアクティビティ）］](https://mandrillapp.com/activity)ページに、ユーザーに送信したすべてのメールの配信インサイトが表示されるようになります。
