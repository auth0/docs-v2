`custom-email-provider`アクショントリガーの`event`オブジェクトは、メールデータに関するコンテキスト情報を提供します。

<table class="table">
<thead>
<tr>
<th>プロパティ</th>
<th>説明</th>
</tr>
</thead>
<tbody>
<tr>
<td>
<p>`event.client`</p>
</td>
<td>
<p>
          このログイントランザクションを開始した相手であるクライアントの情報です。
        </p>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`client_id`<em>：文字列</em>。<span>ユーザーがログインしようとしているアプリケーションのクライアントIDです。</span>
</li>
<li>
`metadata`<em>：辞書</em>。<span>アプリケーションの他のプロパティを含むオブジェクトです。</span>
</li>
<li>
`name`<em>：文字列</em>。<span>アプリケーションの名前（ダッシュボードで定義）です。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.connection`</p>
<p><em>（任意）</em></p>
</td>
<td>
<p>
          ユーザーの認証に使用された接続についての詳細です。
        </p>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`id`<em>：任意の文字列</em>。<span>接続の一意の識別子です。</span>
</li>
<li>
`metadata`<em>：任意の辞書</em>。<span>接続に付けられているメタデータです。</span>
</li>
<li>
`name`<em>：任意の文字列</em>。<span>ユーザーの認証に使用した接続の名前（`twitter`や`some-g-suite-domain`など）です。</span>
</li>
<li>
`strategy`<em>：任意の文字列</em>。<span>接続の種類です。ソーシャル接続の場合は`event.connection.strategy === event.connection.name`です。エンタープライズ接続の場合は`waad`（Windows Azure AD）、`ad`（Active
              Directory/LDAP）、`auth0`（データベース接続）などです。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.notification`</p>
</td>
<td>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`html`<em>：文字列</em>。<span>レンダリングされたHTMLテンプレートです。</span>
</li>
<li>
`from`<em>：文字列</em>。<span>差出人のメールアドレスです。</span>
</li>
<li>`subject`<em>：文字列</em>。<span>メールに付ける件名です。</span></li>
<li>
`locale`<em>：任意の文字列</em>。<span>`en_US`など、メッセージをレンダリングするロケールで、BCP-47の仕様に従います。</span>
</li>
<li>
`message_type`<em>：文字列</em>。<p>`verify_email`や`welcome_email`など、送信するメッセージの種類です。</p>
<p>可能な値には以下が含まれます。</p>
<ul>
<li>`verify_email`</li>
<li>`verify_email_by_code`</li>
<li>`reset_email`</li>
<li>`reset_email_by_code`</li>
<li>`welcome_email`</li>
<li>`verification_code`</li>
<li>`mfa_oob_code`</li>
<li>`enrollment_email`</li>
<li>`blocked_account`</li>
<li>`stolen_credentials`</li>
<li>`try_provider_configuration_email`</li>
<li>`organization_invitation`</li>
</ul>
</li>
<li>
`text`<em>：文字列</em>。<span>レンダリングされたテキストのテンプレートです。</span>
</li>
<li>
`to`<em>：文字列</em>。<span>宛先のメールアドレスです。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.organization`</p>
<p><em>（任意）</em></p>
</td>
<td>
<p>
          現在のトランザクションに関連付けられているOrganizationについての詳細です。
        </p>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`display_name`<em>：文字列</em>。<span>Organizationのフレンドリー名です。</span>
</li>
<li>
`id`<em>：文字列</em>。<span>Organizationの識別子です。</span>
</li>
<li>
`metadata`<em>：辞書</em>。<span>Organizationに関連付けられているメタデータです。</span>
</li>
<li>
`name`<em>：文字列</em>。<span>Organizationの名前です。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.request`</p>
<p><em>（任意）</em></p>
</td>
<td>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`geoip`<em>：任意のオブジェクト</em>
<p>以下のプロパティを含みます。</p>
<ul>
<li>`countryCode`<em>：任意の文字列</em></li>
<li>`countryCode3`<em>：任意の文字列</em></li>
<li>`cityName`<em>：任意の文字列</em></li>
<li>`latitude`<em>：任意の文字列</em></li>
<li>`longitude`<em>：任意の文字列</em></li>
<li>`timeZone`<em>：任意の文字列</em></li>
<li>`subdivisionCode`<em>：任意の文字列</em></li>
<li>`subdivisionName`<em>：任意の文字列</em></li>
<li>`continentCode`<em>：任意の文字列</em></li>
</ul>
</li>
<li>
`ip`<em>：任意の文字列</em>。<span>要求の送信元のIPアドレスです。</span>
</li>
<li>
`query`<em>：任意の辞書</em>。<span>認可要求に送信されたクエリ文字列パラメーターです。パスワードリセットフローにのみ対応しています。</span>
</li>
<li>
`user_agent`<em>：任意の文字列</em>。<span>トランザクションの開始時に受信した`User-Agent`ヘッダーの値です。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.tenant`</p>
</td>
<td>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`friendly_name`<em>：任意の文字列</em>。<span>テナントのフレンドリー名です。通常は人間に読める形式のIDです。</span>
</li>
<li>
`home_url`<em>：任意の文字列</em>。<span>定義されている場合、設定にあるホームURLです。</span>
</li>
<li>
`id`<em>：文字列</em>。<span>テナントの名前です。</span>
</li>
<li>
`logo_url`<em>：任意の文字列</em>。<span>定義されている場合、設定にあるテナントのロゴURLです。</span>
</li>
<li>
`support_email`<em>：任意の文字列</em>。<span>定義されている場合、設定にあるテナントのサポートサービスのメールアドレスです。</span>
</li>
<li>
`support_url`<em>：任意の文字列</em>。<span>定義されている場合、設定にあるテナントのサポートサービスのURLです。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.user`</p>
</td>
<td>
<p>
          代理で始めた現在のトランザクションの対象ユーザーを記述したオブジェクトです。
        </p>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`app_metadata`<em>：辞書</em>。<span>サポートプラン、セキュリティロール、アクセス制御グループなど、ユーザーのアクセスに影響を与えるユーザー情報を保管しているカスタムフィールドです。</span>
</li>
<li>
`email`<em>：任意の文字列</em>。<span>ユーザーの（一意の）メールアドレスです。</span>
</li>
<li>
`email_verified`<em>：ブール値</em>。<span>ユーザーに確認済みのメールアドレスがあるかを示します。</span>
</li>
<li>
`family_name`<em>：任意の文字列</em>。<span>ユーザーの姓です。</span>
</li>
<li>
`given_name`<em>：任意の文字列</em>。<span>ユーザーの名です。</span>
</li>
<li>
`name`<em>：任意の文字列</em>。<span>ユーザーのフルネームです。</span>
</li>
<li>
`nickname`<em>：任意の文字列</em>。<span>ユーザーのニックネームです。</span>
</li>
<li>
`picture`<em>：任意の文字列</em>。<span><a href="https://auth0.com/docs/users/change-user-picture">ユーザーのプロフィール画像</a>をポイントするURLです。</span>
</li>
<li>
`user_id`<em>：文字列</em>。<span>ユーザーの（一意の）識別子です。</span>
</li>
<li>`user_metadata`<em em="">。<span>勤務先の住所、自宅の住所、ユーザー設定など、アクセス権に影響を与えないユーザー情報を保管するカスタムフィールドです。</span></em></li>
<li>
`username`<em>：任意の文字列</em>。
            <span>ユーザーの（一意の）ユーザー名です。</span>
</li>
</ul>
</td>
</tr>
</tbody>
</table>
