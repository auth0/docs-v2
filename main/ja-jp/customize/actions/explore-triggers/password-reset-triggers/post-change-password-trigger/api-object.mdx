post-change-passwordアクショントリガーのAPIオブジェクトには以下が含まれます。

## `api.cache`

実行間で維持されるデータを保存・取得します。

### `api.cache.delete(key)`

存在する場合、提供されたキーにキャッシュされた値について説明したレコードを削除します。

キャッシュから値が削除された場合に`type: "success"`のある`CacheWriteResult`オブジェクトを返します。失敗した操作は、`type: "error"`を返します。エラーについて、返されたオブジェクトは、失敗の性質を示す
`code`プロパティを持ちます。

### `api.cache.get(key)`

存在する場合、提供された
`key`にキャッシュされた値を記述するレコードを取得します。レコードがある場合には、
返されたオブジェクトの`value`プロパティにキャッシュされた値があります。

提供された
`key`のキャッシュにアイテムがある場合には、キャッシュレコードを返します。キャッシュレコードは、キャッシュされた値のある
`value`プロパティと、レコードの最大有効期限をUNIXエポックからのミリ秒単位で示す
`expires_at`プロパティとを持つ
オブジェクトです。

**重要：**
このキャッシュは、短命で一時的なデータ向けに設計されています。アイテムは、
提供されたライフタイム内であったとしても、
後のトランザクションで利用できないかもしれません。

### `api.cache.set(key, value, [options])`

指定されたキーのキャッシュ内の文字列の値を保存またはアップデートします。

このキャッシュに保存された値は、設定されているトリガーにスコープされます。[アクションのキャッシュ制限](/ja-jp/customize/actions/limitations)
の対象です。

この方法で保存された値は、指定の
`ttl`または`expires_at`値までのライフタイムを持ちます。ライフタイムが指定されない場合、
デフォルトでライフタイムは15分になります。ライフタイムは、
[アクションのキャッシュ制限](/ja-jp/customize/actions/limitations)に
リストされた最長期間を超過できません。
