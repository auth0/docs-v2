`post-challenge`アクショントリガーの`event`オブジェクトは、Auth0を使用してログインする際にパスワードをリセットするユーザーのコンテキスト情報を提供します。

**注意：** `event.user`オブジェクトは、外部のIDプロバイダーにより追加された最上位の属性は含みません。

### event.authentication

パスワードリセットフロー中に取得した認証についての詳細。

次のプロパティを含みます：

* `methods`オブジェクトの配列。

  セッション中にユーザーが完了した認証方法を含みます。

  次のオブジェクトの1つを返します：

  + オブジェクト。次のプロパティを含みます：

    - `name`文字列。

      セッション中に完了した第一要素の名前。可能な値は次のとおりです。

      * `federated` ソーシャルまたはエンタープライズ接続が第一要素として、ユーザーの認証に使用されました。
      * `pwd` データベース接続が第一要素として、ユーザーの認証に使用されました。
      * `sms` パスワードレスのSMS接続が、第一要素としてユーザーの認証に使用されました。
      * `email` パスワードレスのメール接続がユーザーの認証の第一要素として使用されました。または、パスワードリセットのためのメール検証に使用されました。
      * `mock` 内部テストに使用されました。
      * `string` URLで指定されたカスタム認証メソッド（2番目以降の要素）。
    - `timestamp`文字列。
  + オブジェクト。

    次のプロパティを含みます：

    - `name`値`mfa` 。ユーザーが完了した多要素認証（2番目以降の要素）。
    - `timestamp`文字列。
    - `type`任意の文字列。

      多要素認証（MFA）に使用された2番目（またはそれ以降）の要素の種類（使用可能な場合）。指定できる値は次のとおりです。

      * `email` メール要素を使用したMFA。
      * `otp` OTP要素を使用したMFA。
      * `push-notification` プッシュ通知要素を使用したMFA。
      * `recovery-code` 復旧コード要素を使用したMFA。
      * `phone` SMSまたは音声要素を使用したMFA。
      * `webauthn-roaming` WebAuthn要素を使用したMFA。
      * `webauthn-platform` WebAuthn要素を使用したMFA。

### event.authorization

ログインしているユーザーに付与された認可の説明についての情報を含むオブジェクト。

次のプロパティを含みます：

* `roles`文字列の配列。ユーザーに割り当てられたロールの名前を含む配列。

### event.client

パスワードリセットトランザクションが開始されたクライアントに関する情報。

次のプロパティを含みます：

* `client_id`文字列。ユーザーがログインしようとしているアプリケーションのクライアントID。
* `metadata`辞書。別のアプリケーションプロパティを保持するためのオブジェクト。
* `name`文字列。アプリケーション名（Dashboardに定義されているとおり）。

### event.connection

ユーザーの認証に使用される接続に関する詳細。

次のプロパティを含みます：

* `id`文字列。接続の一意の識別子。
* `metadata`任意の辞書。接続に関連付けられているメタデータ。
* `name`文字列。ユーザーの認証に使用される接続名（`X`または`some-g-suite-domain`）。
* `strategy`文字列。接続のタイプ。

  + ソーシャル接続の場合、`event.connection.strategy === event.connection.name`
  + エンタープライズ接続の場合、`waad`（Microsoft Entra ID）、`ad`（Active Directory/LDAP）、`auth0`（データベース接続）など。

### event.organization

現在のトランザクションに関連したOrganizationについての詳細。

次のプロパティを含みます：

* `display_name`文字列。Organizationのフレンドリー名。
* `id`文字列。Organizationの識別子。
* `metadata`辞書。Organizationに関連付けられたメタデータ。
* `display_name`文字列。Organizationの名前。

### event.request

トランザクションを開始した要求についての詳細。

次のプロパティを含みます：

* `body`辞書。POST要求のボディ。このデータは、リフレッシュトークンとクライアント資格情報交換フローでのみ使用できます。
* `geoip`オブジェクト。次のプロパティが含まれます。

  + `cityName`任意の文字列
  + `continentCode`任意の文字列
  + `countryCode`任意の文字列
  + `countryCode3`任意の文字列
  + `countryName`任意の文字列
  + `latitude`任意の数字
  + `longitude`任意の数字
  + `subdivisionCode`任意の文字列
  + `subdivisionName`任意の文字列
  + `timeZone`任意の文字列
* `hostname`任意の文字列。認証フローで使用するホスト名。
* `ip`文字列。要求の発信元IPアドレス。
* `language`任意の文字列。ブラウザーから要求された言語。
* `method`文字列。要求に使用されるHTTPメソッド。
* `query`辞書。認可要求に送信されるクエリ文字列パラメーター。
* `user_agent`任意の文字列。トランザクションを開始するときに受信した`User-Agent`ヘッダーの値。

### event.stats

現在のユーザーのログイン統計情報。

次のプロパティを含みます：

* `logins_count`数値。ユーザーがログインした回数。

### event.tenant

現在のトランザクションに関連したテナントについての詳細。

次のプロパティを含みます：

* `id`文字列。テナントの名前。

### event.transaction

現在のトランザクションに関する詳細。

次のプロパティを含みます：

* `locale`文字列。このトランザクションに使うロケールで、ブラウザーの要求言語とテナントの言語設定を比較して決定される。
* `login_hint`任意の文字列。必要に応じて、エンドユーザーがログイン時に使用する可能性のあるログイン識別子に関する認可サーバーへのヒント。
* `state`任意の文字列。Auth0がリダイレクトしてアプリケーションに戻る際に含まれ、アプリが初期要求に追加する不透明な任意の英数字の文字列。
* `ui_locales`文字列の配列。元の認証要求で提供されるui_locales。

### event.user

現在のトランザクションに関連付けられているユーザーを記述するオブジェクト。

次のプロパティを含みます：

* `App_metadata`辞書。サポートプラン、セキュリティロール、アクセス制御グループなどのユーザーアクセスに影響するユーザーについての情報を保存するカスタムフィールドです。
* `created_at`文字列。ユーザープロファイルが最初に作成された日時を示すタイムスタンプ。
* `email`任意の文字列。（一意）ユーザーのメールアドレス。
* `email_verified`ブール値。ユーザーがメールアドレスを検証したかどうかを示します。
* `enrolledFactors`オブジェクトの任意の配列。

  ユーザーが登録した認証要素の配列。空の配列は、ユーザーが要素を登録していないことを示します。`enrolledFactors`が定義されていない場合、システムは情報を取得できず、ユーザーが要素を登録していない可能性があります。

  次のプロパティを含みます：

  + `options`任意のオブジェクト。登録された要素のこのインスタンスを説明する追加オプション。
  + `type`文字列。`push-notification`、`phone`、`email`、`otp`、`webauthn-roaming`および`webauthn-platform`などの認証要素のタイプ。
* `family_name`任意の文字列。ユーザーの姓。
* `family_name`任意の文字列。ユーザーの名。
* `identities`オブジェクトの配列。

  ユーザーの認証に最初に使用されたIDプロバイダーから取得された情報が含まれます。ユーザーが自分のプロファイルを複数のIDプロバイダーにリンクしている場合、この配列にはそれらのIDも含まれます。IDプロバイダーオブジェクトの内容はプロバイダーによって異なります。

  次のプロパティを含みます：

  + `connection`任意の文字列。ユーザーの認証に使用されるAuth0接続の名前。
  + `isSocial`任意のブール値。ソーシャル接続が使用されたかどうかを示します。
  + `profileData`任意の辞書。接続に関連付けられているユーザー情報。ユーザープロファイルがリンクされている場合、`profileData`にはセカンダリアカウントに関連付けられているユーザーデータが含まれます。
  + `provider`任意の文字列。Facebook、Google、SAML、または独自のプロバイダーといった、ユーザーを認証するエンティティの名前。
  + `user_id`任意の文字列。この接続/プロバイダーのユーザーの一意の識別子。
* `last_password_reset`任意の文字列。ユーザーのパスワードが最後に更新された時刻を示すタイムスタンプ。このフィールドはユーザー作成時には存在しません。このプロパティは、データベース接続でのみ使用できます。
* `family_name`任意の文字列。ユーザーのフルネーム。
* `nickname`任意の文字列。ユーザーのニックネーム。
* `phone_number`任意の文字列。ユーザーの電話番号。SMS接続を使用するユーザーに対してのみ有効です。
* `phone_verified`任意のブール値。ユーザーが自分の電話番号を確認したかどうかを示します。SMS接続を使用するユーザーに対してのみ有効です。
* `picture`任意の文字列。[ユーザーのプロファイル写真](https://auth0.com/docs/users/change-user-picture)をポイントするURL。
* `created_at`文字列。ユーザーのプロファイルが最後に更新された日時を示すタイムスタンプ。
* `user_id`文字列。（一意）ユーザーの一意の識別子。
* `user_metadata`辞書。勤務先住所、自宅住所、ユーザー設定など、アクセスレベルに影響を与えないユーザー情報を保存するカスタムフィールド。
* `Username`任意の文字列。（一意）ユーザーのユーザー名。
