pre-user-registration アクショントリガーの`event`オブジェクトは、新規ユーザーの登録要求に関するコンテキスト情報を提供します。

<table class="table">
  <thead>
    <tr>
      <th>プロパティ</th>
      <th>説明</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <p>`event.client`</p>
        <p>
          <em>（任意）</em>
        </p>
      </td>
      <td>
        <p>このトランザクションが開始されたクライアントについての 情報。</p>
        <p>次のプロパティを含みます：</p>
        <ul>
          <li>
            `client_id` <em>文字列</em>。
            <span>
              ユーザーがログインしているアプリケーションのクライアントID。
            </span>
          </li>
          <li>
            `metadata` <em>辞書</em>。
            <span>
              別のアプリケーションプロパティを保持するためのオブジェクト。
            </span>
          </li>
          <li>
            `name` <em>文字列</em>。
            <span>アプリケーションの名前（Dashboardで定義された通り）。</span>
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <p>`event.connection`</p>
      </td>
      <td>
        <p>ユーザー登録に使用された接続の詳細。</p>
        <p>次のプロパティを含みます：</p>
        <ul>
          <li>
            `id` <em>文字列</em>。<span>接続の一意の識別子。</span>
          </li>
          <li>
            `metadata` <em>任意の辞書</em>。
            <span>接続に関連したメタデータ。</span>
          </li>
          <li>
            `name` <em>文字列</em>。
            <span>
              ユーザーの認証に使用される接続名（
              `twitter`または`some-g-suite-domain`など）。
            </span>
          </li>
          <li>
            `strategy` <em>文字列</em>。
            <span>
              接続のタイプ。ソーシャル接続の場合、 `event.connection.strategy
              === event.connection.name`。エンタープライズ接続では、
              `waad`（Microsoft Entra ID）、`ad`（Active
              Directory/LDAP）、`auth0`（データベース接続）など です。
            </span>
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <p>`event.request`</p>
      </td>
      <td>
        <p>トランザクションを開始した要求の詳細。</p>
        <p>次のプロパティを含みます：</p>
        <ul>
          <li>
            `body` <em>辞書</em>。
            <span>
              POST要求のボディ。このデータは、
              リフレッシュトークン、クライアント資格情報交換フロー、
              PreUserRegistrationアクション中のみ利用できます。
            </span>
          </li>
          <li>
            `geoip` <em>オブジェクト</em>。<p>次のプロパティを含みます：</p>
            <ul>
              <li>
                `cityName` <em>任意の文字列</em>。
              </li>
              <li>
                `continentCode` <em>任意の文字列</em>。
              </li>
              <li>
                `countryCode` <em>任意の文字列</em>。
              </li>
              <li>
                `countryCode3` <em>任意の文字列</em>。
              </li>
              <li>
                `countryName` <em>任意の文字列</em>。
              </li>
              <li>
                `latitude` <em>任意の数字</em>。
              </li>
              <li>
                `longitude` <em>任意の数字</em>。
              </li>
              <li>
                `subdivisionCode` <em>任意の文字列</em>。
              </li>
              <li>
                `subdivisionName` <em>任意の文字列</em>。
              </li>
              <li>
                `timeZone` <em>任意の文字列</em>。
              </li>
            </ul>
          </li>
          <li>
            `hostname` <em>任意の文字列</em>。
            <span>認証フローに使用されている ホスト名。</span>
          </li>
          <li>
            `ip` <em>文字列</em>。<span>要求の発生元IPアドレス。</span>
          </li>
          <li>
            `language` <em>任意の文字列</em>。
            <span>ブラウザーから要求された言語。</span>
          </li>
          <li>
            `method` <em>文字列</em>。<span>要求に使用されたHTTPメソッド</span>
          </li>
          <li>
            `user_agent` <em>任意の文字列</em>。
            <span>
              トランザクションの開始時に受け取った`User-Agent`の ヘッダーの値。
            </span>
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <p>`event.tenant`</p>
      </td>
      <td>
        <p>現在のトランザクションに関連するテナントの詳細。</p>
        <p>次のプロパティを含みます：</p>
        <ul>
          <li>
            `id` <em>文字列</em>。<span>テナント名。</span>
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <p>`event.transaction`</p>
        <p>
          <em>（任意）</em>
        </p>
      </td>
      <td>
        <p>現在のトランザクションについての詳細。</p>
        <p>次のプロパティを含みます：</p>
        <ul>
          <li>
            `acr_values` <em>文字列の配列</em>。
            <span>元の認証要求に提供されたすべてのacr_values。</span>
          </li>
          <li>
            `locale` <em>文字列</em>。
            <span>
              このトランザクションに使われるロケールで、
              ブラウザーの要求言語とテナントの言語設定を 比較して決定される。
            </span>
          </li>
          <li>
            `login_hint` <em>任意の文字列</em>。
            <span>
              エンドユーザーが必要な場合にログインに使用する可能性のある、
              ログイン識別子に関する認可サーバーへのヒント。
            </span>
          </li>
          <li>
            `prompt` <em>任意の文字列の配列</em>。
            <span>
              ユーザーに再認証と同意を求めるかどうかを示す 指示のリスト。
            </span>
          </li>
          <li>
            `protocol` <em>任意の文字列</em>。
            <p>含まれる可能性のある値は以下の通りです：</p>
            <ul>
              <li>
                `oidc-basic-profile`
                <span>最も使用される、Webベースのログイン。</span>
              </li>
              <li>
                `oidc-implicit-profile`
                <span>
                  モバイルデバイスおよびシングルページアプリで使用されます。
                </span>
              </li>
              <li>
                `samlp` <span>SaaSアプリで使用されるSAMLプロトコール。</span>
              </li>
              <li>
                `wsfed`
                <span>
                  Office365といったMicrosoft製品で使用されるWS-Federation 。
                </span>
              </li>
              <li>
                `wstrust-usernamemixed`
                <span>
                  CRMおよびOffice365で使用されるWS-trustユーザー/パスワードログイン。
                </span>
              </li>
              <li>
                `oauth2-device-code`
                <span>デバイスの認可フローで使用されるトランザクション。</span>
              </li>
              <li>
                `oauth2-resource-owner`
                <span>
                  通常、データベース接続に使用されるユーザー/パスワード
                  ログイン。
                </span>
              </li>
              <li>
                `oauth2-resource-owner-jwt-bearer`
                <span>
                  ユーザーの秘密鍵で署名されたBearer{" "}
                  <Tooltip tip="二者間のクレームを安全に表現するために使用される標準IDトークン形式（および多くの場合、アクセストークン形式）。">
                    JWT
                  </Tooltip>
                  を使用したログイン。
                </span>
              </li>
              <li>
                `oauth2-password`
                <span>パスワード交換を使用したログイン。</span>
              </li>
              <li>
                `oauth2-access-token`
                <span>
                  リフレッシュトークン交換を使用したトークンのリフレッシュ。
                </span>
              </li>
              <li>
                `oauth2-refresh-token`
                <span>
                  リフレッシュトークン交換を使用したトークンのリフレッシュ。
                </span>
              </li>
              <li>`oauth2-token-exchange`</li>
              <li>
                `oidc-hybrid-profile`
                <span>
                  アクセストークンとリフレッシュトークンをアプリケーションが
                  安全に取得できるようにしながら、
                  IDトークンに即座にアクセスできるようにします。
                </span>
              </li>
            </ul>
          </li>
          <li>
            `redirect_uri` <em>任意の文字列</em>。
            <span>
              トランザクションが完了した後にAuth0がリダイレクトするブラウザーのURL。
            </span>
          </li>
          <li>
            `requested_scopes` <em>文字列の配列</em>。
            <span>
              この認証フローを開始するときに要求（必要ある場合）される
              スコープ。
            </span>
          </li>
          <li>
            `response_mode` <em>任意の文字列</em>。
            <p>
              認可エンドポイントから返されたパラメーターに使用される
              認可サーバーのメカニズムに知らせます。
            </p>
            <p>含まれる可能性のある値は以下の通りです：</p>
            <ul>
              <li>`query`</li>
              <li>`fragment`</li>
              <li>`form_post`</li>
              <li>`web_message`</li>
            </ul>
          </li>
          <li>
            `response_type` <em>任意の文字列の配列</em>。
            <p>含まれる可能性のある値は以下の通りです：</p>
            <ul>
              <li>`code`</li>
              <li>`token`</li>
              <li>`id_token`</li>
            </ul>
          </li>
          <li>
            `state` <em>任意の文字列</em>。
            <span>
              Auth0がリダイレクトしてアプリケーションに戻る際に含まれ、
              アプリが初期要求に追加する不透明な 任意の英数字の文字列。
            </span>
          </li>
          <li>
            `ui_locales` <em>文字列の配列</em>。
            <span>元の認証要求で提供されたui_locales。</span>
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <p>`event.user`</p>
      </td>
      <td>
        <p>登録を試みているユーザーを説明するオブジェクト。</p>
        <p>次のプロパティを含みます：</p>
        <ul>
          <li>
            `app_metadata` <em>任意の辞書</em>。
            <span>
              サポートプラン、セキュリティロール、アクセス制御グループなどの
              ユーザーアクセスに影響するユーザーについての情報を保存する
              カスタムフィールドです。
            </span>
          </li>
          <li>
            `email` <em>任意の文字列</em>。
            <span>（一意）ユーザーのメールアドレス。</span>
          </li>
          <li>
            `family_name` <em>任意の文字列</em>。<span>ユーザーの姓。</span>
          </li>
          <li>
            `given_name` <em>任意の文字列</em>。<span>ユーザーの名。</span>
          </li>
          <li>
            `name` <em>任意の文字列</em>。<span>ユーザーの氏名。</span>
          </li>
          <li>
            `nickname` <em>任意の文字列</em>。
            <span>ユーザーのニックネーム。</span>
          </li>
          <li>
            `phone_number` <em>任意の文字列</em>。
            <span>ユーザーの電話番号。</span>
          </li>
          <li>
            `picture` <em>任意の文字列</em>。
            <span>
              <a href="https://auth0.com/docs/ja-jp/users/change-user-picture">
                ユーザーのプロファイル写真
              </a>
              を指すURL。
            </span>
          </li>
          <li>
            `user_metadata` <em>任意の辞書</em>。
            <span>
              勤務先住所、自宅住所、ユーザーの設定など、
              ユーザーのアクセスに影響を与えないユーザーの情報を
              保存するカスタムフィールド。
            </span>
          </li>
          <li>
            `username` <em>任意の文字列</em>。
            <span>（一意）ユーザーのユーザー名。</span>
          </li>
        </ul>
      </td>
    </tr>
  </tbody>
</table>
