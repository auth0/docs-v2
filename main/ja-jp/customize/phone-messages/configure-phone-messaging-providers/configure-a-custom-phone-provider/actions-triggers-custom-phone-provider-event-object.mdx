<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

電話メッセージプロバイダーは、現在早期アクセス版として提供されています。Auth0のリリースについては、「[製品のリリース段階](https://auth0.com/docs/troubleshoot/product-lifecycle/product-release-stages)」を参照してください。

</Callout>

`custom-phone-provider`アクショントリガーの`event`オブジェクトは、カスタム電話メッセージの送信と受信の要求に関するコンテキスト情報を提供します。

<table class="table">
<thead>
<tr>
<th>プロパティ</th>
<th>説明</th>
</tr>
</thead>
<tbody>
<tr>
<td>
<p>`event.client`</p>
</td>
<td>
<p>
          このログイントランザクションが開始されたクライアントに関する情報です。
        </p>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`client_id` <em>：文字列</em>。<span>ユーザーがログインしているアプリケーションのクライアントIDです。</span>
</li>
<li>
`metadata` <em>：辞書</em>。<span>他のアプリケーションプロパティを含むオブジェクトです。</span>
</li>
<li>
`name` <em>：文字列</em>。<span>アプリケーションの名前（Dashboardで定義）です。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.connection`</p>
<p><em>（任意）</em></p>
</td>
<td>
<p>
          ユーザーの認証に使用された接続についての詳細です。
        </p>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`id` <em>：任意の文字列</em>。<span>接続の一意の識別子です。</span>
</li>
<li>
`metadata` <em>：任意の辞書</em>。<span>接続に関連付けられているメタデータです。</span>
</li>
<li>
`name` <em>：任意の文字列</em>。<span>ユーザーの認証に使用される接続名（`twitter`や`some-g-suite-domain`など）です。</span>
</li>
<li>
`strategy` <em>：任意の文字列</em>。<span>接続の種類です。ソーシャル接続の場合は`event.connection.strategy === event.connection.name`です。エンタープライズ接続の場合、ストラテジーは`waad`（Windows Azure AD）、`ad`（Active
              Directory/LDAP）、`auth0`（データベース接続）などになります。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.notification`</p>
</td>
<td>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`from` <em>：文字列</em>。<span>‌‌E.164に準拠した送信者の電話番号です。</span>
</li>
<li>
`locale` <em>：文字列</em>。<span>メッセージをレンダリングしたロケール（`en_US`など）で、BCP-47の仕様に従います。</span>
</li>
<li>
`message_type` <em>：文字列</em>です。<p>送信されるメッセージの種類（`otp_verify`や`blocked_account`など）です。</p>
<p>以下の値を使用できます。</p>
<ul>
<li>`otp_verify`</li>
<li>`otp_enroll`</li>
<li>`blocked_account`</li>
<li>`change_password`</li>
<li>`password_breach`</li>
</ul>
</li>
<li>
`recipient` <em>：文字列</em>。<span>E.164に準拠した受信者の電話番号です。</span>
</li>
<li>
`delivery_method` <em>：列挙型</em>。<span>メッセージの送信方法です。`text`または`voice`になります。</span>
</li>
<li>
`code` <em>：文字列</em>。<span>一部の `message_types`（`otp_verify`や`otp_enroll`など）で使用するワンタイムパスワードです。</span>
</li>
<li>
`as_text` <em>：文字列</em>。<span>テキストメッセージとして送信可能なレンダリング済みのテキストです。</span>
</li>
<li>
`as_voice` <em>：文字列</em>。<span>音声テキストメッセージとして送信可能なレンダリング済みのテキストです。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.organization`</p>
<p><em>（任意）</em></p>
</td>
<td>
<p>
                    現在のトランザクションに関連付けられている組織の詳細です。
        </p>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`display_name` <em>：文字列</em>。<span>組織のフレンドリー名です。</span>
</li>
<li>
`id` <em>：文字列</em>。<span>組織の識別子です。</span>
</li>
<li>
`metadata` <em>：辞書</em>。<span>組織に関連付けられているメタデータです。</span>
</li>
<li>
`name` <em>：文字列</em>。<span>組織の名前です。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.request`</p>
<p><em>（任意）</em></p>
</td>
<td>
<p>トランザクションを開始した要求についての詳細です。</p>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`geoip` <em>：任意のオブジェクト</em>です。
            <p>以下のプロパティを含みます。</p>
<ul>
<li>`countryCode` <em>：任意の文字列</em></li>
<li>`countryCode3` <em>：任意の文字列</em></li>
<li>`cityName` <em>：任意の文字列</em></li>
<li>`latitude` <em>：任意の文字列</em></li>
<li>`longitude` <em>：任意の文字列</em></li>
<li>`timeZone` <em>：任意の文字列</em></li>
<li>`subdivisionCode` <em>：任意の文字列</em></li>
<li>`subdivisionName` <em>：任意の文字列</em></li>
<li>`continentCode` <em>：任意の文字列</em></li>
<li>`continentCode` <em>：任意の文字列</em></li>
</ul>
</li>
<li>
`ip` <em>：任意の文字列</em>。<span>要求の送信元のIPアドレスです。</span>
</li>
<li>
`hostname` <em>：任意の文字列</em>。<span>認証フローに使用されているホスト名です。</span>
</li>
<li>
`method` <em>：任意の文字列</em>。<span>要求に使用されたHTTPメソッドです。</span>
</li>
<li>
`language` <em>：任意の文字列</em>。<span>ブラウザーから要求された言語です。</span>
</li>
<li>
`user_agent` <em>：任意の文字列</em>。<span>トランザクションの開始で受信した`User-Agent`ヘッダーの値です。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.tenant`</p>
</td>
<td>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`friendly_name` <em>：任意の文字列</em>。<span>テナントのフレンドリー名です。通常は人間に読めるバージョンのIDです。</span>
</li>
<li>
`home_url` <em>：任意の文字列</em>。<span>定義されている場合、設定にあるテナントのホームURLです</span>
</li>
<li>
`id` <em>：文字列</em>。<span>テナントの名前です。</span>
</li>
<li>
`logo_url` <em>：任意の文字列</em>。<span>定義されている場合、設定にあるテナントのロゴURLです。</span>
</li>
<li>
`support_email` <em>：任意の文字列</em>。<span>定義されている場合、設定にあるテナントのサポートサービスのメールアドレスです。</span>
</li>
<li>
`support_url` <em>：任意の文字列</em>。<span>定義されている場合、設定にあるテナントのサポートサービスのURLです。</span>
</li>
</ul>
</td>
</tr>
<tr>
<td>
<p>`event.user`</p>
</td>
<td>
<p>
          代理で現在のトランザクションが開始されたユーザーを記述するオブジェクトです。
        </p>
<p>以下のプロパティを含みます。</p>
<ul>
<li>
`app_metadata` <em>：辞書</em>。<span>サポートプラン、セキュリティロール、アクセス制御グループなどのユーザーアクセスに影響するユーザー情報を保管するカスタムフィールドです。</span>
</li>
<li>
`email` <em>：任意の文字列</em>。<span>（一意）ユーザーのメールアドレスです。</span>
</li>
<li>
`email_verified` <em>：ブール値</em>。<span>ユーザーのメールアドレスが検証済みかを示します。</span>
</li>
<li>
`family_name` <em>：任意の文字列</em>。<span>ユーザーの姓です。</span>
</li>
<li>
`given_name` <em>：任意の文字列</em>。<span>ユーザーの名です。</span>
</li>
<li>
`name` <em>：任意の文字列</em>。<span>ユーザーのフルネームです。</span>
</li>
<li>
`nickname` <em>：任意の文字列</em>。<span>ユーザーのニックネームです。</span>
</li>
<li>
`picture` <em>：任意の文字列</em>。<span>
<a href="https://auth0.com/docs/ja-jp/users/change-user-picture">ユーザーのプロフィール画像をポイントするURL</a>です。</span>
</li>
<li>
`user_id` <em>：文字列</em>。<span>（一意）ユーザーの一意の識別子です。</span>
</li>
<li>
`user_metadata` <em>：辞書</em>。<span>勤務先の住所、自宅の住所、ユーザーの設定など、ユーザーのアクセスに影響を与えないユーザー情報を保管するカスタムフィールドです。</span>
</li>
<li>
`username` <em>：任意の文字列</em>。<span>（一意）ユーザーのユーザー名です。</span>
</li>
</ul>
</td>
</tr>
