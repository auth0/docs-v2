**2020年7月30日**付けで、`context`フィールドを要求するFacebook接続が失敗するようになるため、Auth0はこの要求をすべての接続について停止しました。

2019年4月30日に、新規アプリケーションの**ソーシャルコンテキスト**フィールドの使用が[Facebookによって廃止](https://developers.facebook.com/docs/graph-api/changelog/4-30-2019-endpoint-deprecations)されました。2019年4月30日よりも前に作成されたFacebook接続について、Auth0は引き続きデフォルトでこのフィールドを要求します。7月30日以前にこのフィールドが要求されないようにするには、接続の［User Data（ユーザーデータ）］セクションで**［Social context（ソーシャルコンテキスト）］**を選択解除します。

<Frame>![Facebook Context Field Deprecation](/images/ja-jp/cdy7uua7fh8z/4oi3foRuwYYOveDlq2vJM/a57cd954ceeaf6d7722796be7c743b76/facebook-context.png)</Frame>

**［Social context（ソーシャルコンテキスト）］**を選択解除すると、プロファイルデータにコンテキストフィールドが含まれなくなります。このフィールドには以下の内容があります。

```javascript lines
"context": {
      "mutual_likes": {"data": [],"summary": {"total_count": 0}},
      "id": "dXNlcl9...UZD"
    }
```

## アクション

Auth0が返すFacebookプロファイルの`context`フィールドをアプリケーション内で使用していない場合、アプリケーションは変更を必要とすることなく動作します。それ以外の場合には、使用しないようにアプリケーションのコードを修正する必要があります。

7月30日にアプリケーションが影響を受けないように、Facebook接続プロパティで**［Social context（ソーシャルコンテキスト）］**フィールドを選択解除することをお勧めします。
